\chapter{Event Task Matrix (ETM)}
\label{mod:evntaskmatrix}

%%% Overview %%%

\section{Overview}
The Event Task Matrix (ETM) peripheral contains 50 configurable channels. Each channel can map an event of any specified peripheral to a task of any specified peripheral. In this way, peripherals can be triggered to execute specified tasks without CPU intervention.


%%%%%%%%%%%%%%%%%%%%
%%% Features %%%
%%%%%%%%%%%%%%%%%%%%

\section{Features}
\label{sec:evntaskmatrix-features}

The Event Task Matrix has the following features:
\begin{itemize}
    \item Receive various events from multiple peripherals
    \item Generate various tasks for multiple peripherals
    \item 50 independently configurable ETM channels
    \item An ETM channel can be set up to receive any event, and map it to any task
    \item Each ETM channel can be enabled or disabled independently. If disabled, the channel will not respond to the configured event and generate the task mapped to that event
    \item Support for checking event and task status
    \item Peripherals supporting ETM include GPIO, LED PWM, general-purpose timers, RTC Timer, system timer, temperature sensor, ADC, I2S, GDMA, and PMU
\end{itemize}

Note that the 50 ETM channels are identical regarding their features and operations. Thus, in the following sections, ETM channels are collectively referred to as channel\regindex{n} (where \regindex{n} ranges from 0 to 49).

\section{Functional Description}
\label{sec:evntaskmatrix-func-descr}

\subsection{Architecture}



The Event Task Matrix has 50 independent channels. Each channel can select any event as input, and map the event to any task as output (see Section \ref{sec:evntaskmatrix-events} and Section \ref{sec:evntaskmatrix-tasks} respectively). Each channel has an individual enable bit (see Section \ref{sec:evntaskmatrix-channel-enable}).

\begin{figure}[H]
    \centering
    \includegraphics[width=0.7\textwidth]{\modulefiles/figures/etm_channel}
    \caption{ETM Channel\regindex{n} Architecture}
    \label{fig:etm-channel}
\end{figure}

Figure \ref{fig:etm-channel} illustrates the structure of an ETM channel. The \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID} field configures the MUX (multiplexer) to select one of the events as the input of channel\regindex{n}. The \hyperref[fielddesc:SOCETMCH0TASKID]{SOC\_ETM\_CH\regindex{n}\_TASK\_ID} field configures the DEMUX (demultiplexer) to map the event selected by channel\regindex{n} to one of the tasks. \hyperref[fielddesc:SOCETMCHENABLE0]{SOC\_ETM\_CH\_ENABLE\regindex{n}} and \hyperref[fielddesc:SOCETMCHDISABLE0]{SOC\_ETM\_CH\_DISABLE\regindex{n}} are used to enable or disable channel\regindex{n}. \hyperref[fielddesc:SOCETMCHENABLED0]{SOC\_ETM\_CH\_ENABLED\regindex{n}} is used to indicate the status of the channel\regindex{n}.

When a channel receives a valid event pulse signal, it generates the mapped task pulse signal.

For more detailed descriptions of a task, please refer to the chapter for the peripheral receiving this task.

Events from different channels can be optionally mapped to the same task. For example, field \hyperref[fielddesc:SOCETMCH0TASKID]{SOC\_ETM\_CH\regindex{n}\_TASK\_ID} of multiple channels can be configured with the same value, and field \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID} can be configured with the same or different values. In this case, when the event received by any of the channels is valid, the task will be generated. If events received by multiple channels are valid at the same time, the task will be generated only once.

\subsection{Event and Task Status}

The ETM module supports checking the status of events and tasks by reading registers as follows:

\begin{itemize}
    \item Event status can be checked by reading the \hyperref[regdesc:SOCETMEVTST0REG]{SOC\_ETM\_EVT\_ST\regindex{n}\_REG} register. If the field corresponding to an event is 1, it indicates that the event has been received by ETM. Otherwise, it indicates that the event has not been received by ETM. The fields in the \hyperref[regdesc:SOCETMEVTST0REG]{SOC\_ETM\_EVT\_ST\regindex{n}\_REG} register can be cleared by writing 1 to the corresponding fields in the \hyperref[regdesc:SOCETMEVTST0CLRREG]{SOC\_ETM\_EVT\_ST\regindex{n}\_CLR\_REG} register.
    \item Task status can be checked by reading the \hyperref[regdesc:SOCETMTASKST0REG]{SOC\_ETM\_TASK\_ST\regindex{n}\_REG} register. If the field corresponding to a task is 1, it indicates that the task has been generated by ETM. Otherwise, it indicates that the task has not been generated by ETM. The fields in the \hyperref[regdesc:SOCETMTASKST0REG]{SOC\_ETM\_TASK\_ST\regindex{n}\_REG} register can be cleared by writing 1 to the corresponding fields in the \hyperref[regdesc:SOCETMTASKST0CLRREG]{SOC\_ETM\_TASK\_ST\regindex{n}\_CLR\_REG} register.
\end{itemize}

\subsection{Timing Considerations}

Figure \ref{fig:etm-clk-archi} shows the structure of clocks that drive received events, sent tasks, and ETM channels.

\begin{figure}[H]
    \centering
    \includegraphics[width=1\textwidth]{\modulefiles/figures/etm_clk_archi}
    \caption{Event Task Matrix Clock Architecture}
    \label{fig:etm-clk-archi}
\end{figure}

ETM is running at the AHB\_CLK domain (see Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}}). Each event corresponds to a pulse signal generated by the corresponding peripheral in its clock domain, while each task is mapped by the ETM to a pulse signal under its corresponding peripheral clock domain. The peripherals generating events, the Event Task Matrix, and the peripherals receiving tasks do not necessarily operate on the same clock and therefore require synchronization. Therefore, to avoid event loss, there should be a delay between two consecutive event pulses which is dependent on the clock of the peripheral generating the event, on the ETM's clock, and on the clock of the peripheral executing the task.

To make sure the Event Task Matrix receives every event successfully, for peripherals generating event pulses, the interval between two consecutive pulses must be greater than one ETM clock cycle, namely $ceil(\frac{peripheral\_clock\_frequency}{ETM\_clock\_frequency})$ in the unit of peripheral clock cycles.

For example, assuming that peripheral A is in the 80 MHz clock domain (PLL\_F80M\_CLK), and the ETM runs in the 40 MHz clock domain (AHB\_CLK). Peripheral A generates event 1, which is received by the ETM. To ensure successful reception of each event 1, the interval between two consecutive occurrences of event 1 must exceed two clock cycles of Peripheral A (i.e., one ETM clock cycle).

Likewise, to make sure the Event Task Matrix maps the received event (i.e., event synchronized to the ETM's clock domain) successfully to a task, the interval between two consecutive event pulses in the ETM clock domain must be greater than one peripheral clock cycle, namely $ceil(\frac{ETM\_clock\_frequency}{peripheral\_clock\_frequency})$ in the unit of ETM clock cycles.

For example, assuming that peripheral B is in the 20 MHz clock domain (RC\_FAST\_CLK), and the ETM runs in the 40 MHz clock domain (AHB\_CLK). ETM maps an event to task 1, which should be received by peripheral B. To map each received event successfully to task 1, the interval between two consecutive events must be greater than two ETM clock cycles (i.e., one peripheral B clock cycle).

As a result, to map two consecutive events generated by peripheral A to peripheral B, the interval between these two events must be $ceil(\frac{peripheral\_A\_clock\_frequency}{ETM\_clock\_frequency})*ceil(\frac{ETM\_clock\_frequency}{peripheral\_B\_clock\_frequency})$ in the unit of peripheral A clock cycles.

For example, assuming that peripheral A is in the 80 MHz clock domain (PLL\_F80M\_CLK), peripheral B is in the 20 MHz clock domain (RC\_FAST\_CLK), and the ETM runs in the 40 MHz clock domain (AHB\_CLK). ETM maps event 1 generated by peripheral A to task 1 received by peripheral B. To successfully map each event 1 to task 1, the interval between two consecutive event 1 must be greater than $2*2=4$ peripheral A clock cycles.


\subsection{Channel Control}
\label{sec:evntaskmatrix-channel-enable}
Each ETM channel can be independently configured to be enabled or disabled. When channel\regindex{n} is enabled and receives the event configured via \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID}, it maps the event to the task configured via \hyperref[fielddesc:SOCETMCH0TASKID]{SOC\_ETM\_CH\regindex{n}\_TASK\_ID}. When channel\regindex{n} is disabled, even if it receives the event configured via \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID}, no task will be generated.

To enable ETM channel\regindex{n}, write 1 to \hyperref[fielddesc:SOCETMCHENABLE0]{SOC\_ETM\_CH\_ENABLE\regindex{n}}. To disable ETM channel\regindex{n}, Write 1 to \hyperref[fielddesc:SOCETMCHDISABLE0]{SOC\_ETM\_CH\_DISABLE\regindex{n}}.

The status of ETM channel\regindex{n} can be obtained by reading \hyperref[fielddesc:SOCETMCHENABLED0]{SOC\_ETM\_CH\_ENABLED\regindex{n}}. 1 indicates that channel\regindex{n} has been enabled, and 0 indicates disabled.

If \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID} or  \hyperref[fielddesc:SOCETMCH0TASKID]{SOC\_ETM\_CH\regindex{n}\_TASK\_ID} is configured to 0, ETM channel\regindex{n} will also be disabled.

The complete procedure to configure ETM channel\regindex{n} is as follows:
\begin{enumerate}
    \item Enable the ETM's clock by writing 1 to \hyperref[fielddesc:PCRETMCLKEN]{PCR\_ETM\_CLK\_EN}
    \item Select the event to be received by channel\regindex{n} via \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID}
    \item Select the task mapped to the received event via  \hyperref[fielddesc:SOCETMCH0TASKID]{SOC\_ETM\_CH\regindex{n}\_TASK\_ID}
    \item Enable channel\regindex{n} by setting  \hyperref[fielddesc:SOCETMCHENABLE0]{SOC\_ETM\_CH\_ENABLE\regindex{n}}
    \item When channel\regindex{n} no longer needs to map the selected event to the selected task, disable channel\regindex{n} by setting \hyperref[fielddesc:SOCETMCHDISABLE0]{SOC\_ETM\_CH\_DISABLE\regindex{n}}. To configure a new event and task mapping, repeat Steps 2 to 4. If no further configurations, channel\regindex{n} will remain disabled
    \item The whole ETM module (i.e., all ETM channels) can be reset by writing 1 and then 0 to the \hyperref[fielddesc:PCRETMRSTEN]{PCR\_ETM\_RST\_EN} field
\end{enumerate}
