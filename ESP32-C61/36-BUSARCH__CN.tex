\hypertarget{busarch}{}
\chapter{总线架构}
\label{mod:busarch}


\chipname{} SoC 采用多层总线矩阵作为通信架构的核心，通过高效互连的结构连接多个主设备和从设备。该架构支持并发访问，有效缓解访问冲突，使高性能的 RISC-V 核心、DMA 控制器、调试接口及各类外设能够在竞争最小的情况下同时运行。通过优化数据流与系统性能，即使多个高速外设同时工作，总线矩阵也能够确保系统高效运转。
\chipname{} SoC 的主系统由以下部分组成：

\begin{itemize}
    \item 主设备 (master)：
        \begin{itemize}
            \item 高性能 (HP) 32 位 RISC-V 处理器的 I-bus、D-bus 和 AHB 总线
            \item TRACE
            \item TCM MEM MONITOR
            \item PSRAM MEM MONITOR
            \item GDMA
            \item SDIO SLAVE
        \end{itemize}
    \item 从设备 (slave)：
        \begin{itemize}
            \item ROM
            \item HP SRAM
            \item Cache
            \item HP \& LP 外设
            \item CPU 外设
        \end{itemize}
\end{itemize}

图 \ref{fig:bus-arch} \textit{\nameref{fig:bus-arch}} 中的总线说明如下：

\begin{itemize}
    \item \textbf{HP CPU I-bus 总线}：连接高性能 (HP) 32 位 RISC-V 处理器的指令总线和总线矩阵，用于处理器取指。其可访问对象为包含代码的存储器（如 ROM、HP SRAM 或通过 MSPI 访问的外部存储器）。
    \item \textbf{HP CPU D-bus 总线}：连接高性能 (HP) 32 位 RISC-V 处理器的数据总线和总线矩阵，用于字面量加载和数据读写。其可访问对象为存储代码或数据的存储器（如 ROM、HP SRAM 或通过 MSPI 访问的外部存储器）。
    \item \textbf{HP CPU AHB 总线}：连接高性能 (HP) 32 位 RISC-V 处理器的 AHB 总线和总线矩阵，用于访问外设中的数据。可访问对象包括 HP \& LP 外设和 CPU 外设。
    \item \textbf{TRACE 总线}：连接 TRACE 模块和总线矩阵，用于执行调试操作。可访问对象为 HP SRAM。
    \item \textbf{TCM MEM MONITOR 总线}：连接 TCM MEM MONITOR 和总线矩阵，用于将 TCM MEM MONITOR 监控数据写入内存。可访问对象为 HP SRAM。
    \item \textbf{PSRAM MEM MONITOR 总线}：连接 PSRAM MEM MONITOR 和总线矩阵，用于将 PSRAM MEM MONITOR 记录的监控信息写入内存。可访问对象为 HP SRAM。
    \item \textbf{GDMA 总线}：提供一条连接至总线矩阵的线路和一条连接外部存储器的线路。GDMA 通过该总线进行高速数据传输，包括外设与存储器之间以及存储器与存储器之间的传输。可访问对象包括 HP SRAM 和外部存储器。针对 GDMA 访问外部存储器的场景，为确保数据吞吐率，建议尽可能设置较长的 burst 长度。
    \item \textbf{SDIO SLAVE 总线}：连接 SDIO SLAVE 和总线矩阵，用于访问内存或外设中的数据。可访问对象包括 HP SRAM、HP \& LP 外设和 CPU 外设。
\end{itemize}
