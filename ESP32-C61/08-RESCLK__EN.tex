\chapter{Reset and Clock}
\label{mod:resclk}

\section{Reset}
\subsection{Overview}
\chipname{} provides four types of reset that occur at different levels, namely CPU Reset, Core Reset, System Reset, and Chip Reset. All reset types mentioned above (except Chip Reset) preserve the data stored in internal memory. Figure \ref{figure:Reset} shows the scopes of affected subsystems by each type of reset.

\subsection{Architectural Overview}
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{\modulefiles/figures/reset-overview.eps}
    \caption{Reset Types}
    \label{figure:Reset}
\end{figure}
\chipname{}'s Digital System can be divided into two parts: \hyperref[mod:riscvcpu]{High Performance System (HP system)} that includes Digital Core, Wireless Mac and Baseband, and HP SRAM, and \hyperref[mod:lowpowm]{Low Power System (LP system)} that only includes some low-power peripherals and LP SRAM. See Figure \ref{figure:Reset} for details (note that HP SRAM and LP SRAM would not be reset, so they are not shown in the figure).




\subsection{Features}
\label{sec:resclk-features}
\begin{itemize}
    \item Four reset types:
    \begin{itemize}
    \item CPU Reset: resets CPU core. Once such a reset is released, the instructions from the CPU reset vector (0x40000000)  will be executed.
    \item Core Reset: resets the whole digital system except LP system, including CPU, peripherals, digital GPIOs, Wireless MAC and Baseband.
    \item System Reset: resets the whole digital system, including LP system.
    \item Chip Reset: resets the whole chip, including the digital system and analog system.
    \end{itemize}
    \item Software reset and hardware reset:
    \begin{itemize}
        \item Software Reset: triggered via software by configuring the corresponding registers of CPU, see Chapter \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}.
        \item Hardware Reset: triggered directly by the hardware.
    \end{itemize}
\end{itemize}

\subsection{Functional Description}
CPU will be reset immediately when any type of reset above occurs. Users can retrieve reset source codes by reading \hyperref[fielddesc:LPCLKRSTRESETCAUSE]{LP\_CLKRST\_RESET\_CAUSE} after the reset is released. The current reset source recorded in the \hyperref[fielddesc:LPCLKRSTRESETCAUSE]{LP\_CLKRST\_RESET\_CAUSE} register can be cleared by configuring \hyperref[fielddesc:LPCLKRSTCORE0RESETCAUSECLR]{LP\_CLKRST\_CORE0\_RESET\_CAUSE\_CLR}.

Table \ref{tab:resclk-reset-src} lists possible reset sources and the types of reset they trigger. When multiple reset sources are active simultaneously, the reset source recorded in \hyperref[fielddesc:LPCLKRSTRESETCAUSE]{LP\_CLKRST\_RESET\_CAUSE} is the most top one among the active reset sources listed in Table \ref{tab:resclk-reset-src}. When multiple reset sources are active but not at the same time, \hyperref[fielddesc:LPCLKRSTRESETCAUSE]{LP\_CLKRST\_RESET\_CAUSE} records the most recent reset source.

\subsection{Peripheral Reset}

Peripherals can be reset individually by configuring corresponding registers, or globally by core reset, system reset, or chip reset. After releasing peripherals from reset, the reset release status registers can be read to determine the reset status. These registers turning to 1 indicates that peripherals have been released from reset and can operate properly.

The reset registers of \chipname{}'s HP system peripherals are controlled by the Power/Clock/Reset (PCR) module. Please see \ref{subsec:resclk-pcr-reg-summ} \textit{\nameref{subsec:resclk-pcr-reg-summ}} for the reset registers of HP system peripherals and \ref{subsec:resclk-lp-reg-summ} \textit{\nameref{subsec:resclk-lp-reg-summ}} for the clock registers of low-power system peripherals.

\section{Clock}

\subsection{Overview}

\chipname{} clocks are mainly sourced from oscillator (OSC), RC, and PLL circuits, and then processed by the dividers or selectors, which allows most functional modules to select their working clock according to their power consumption and performance requirements. Figure \ref{figure:Clock} shows the system clock structure.

\subsection{Architectural Overview}


\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/sys_clk_struct.eps}
    \caption{System Clock}
    \label{figure:Clock}
\end{figure}


\subsection{Features}

\chipname{} clocks can be classified into two types depending on their frequencies:
\begin{itemize}
    \item High-performance (HP) clocks for devices working at a higher frequency, such as CPU and digital peripherals
    \begin{itemize}
        \item PLL\_F480M\_CLK (480 MHz): internal PLL clock. Its reference clock is XTAL\_CLK
        \item XTAL\_CLK (40 MHz): external crystal clock
    \end{itemize}
    \item Low-power (LP) clocks for low-power system and some peripherals working in low-power mode
    \begin{itemize}
        \item XTAL32K\_CLK (32 kHz): external crystal clock
	    \item RC\_FAST\_CLK (20 MHz by default): internal fast RC oscillator with adjustable frequency
	    \item RC\_SLOW\_CLK (130 kHz by default): internal slow RC oscillator with adjustable frequency
	    \item OSC\_SLOW\_CLK (32 kHz by default): external slow clock input through \hyperref[tab:ANA-IO-MUX-EN]{XTAL\_32K\_P}. After configuring this GPIO, also configure the Hold function (see Chapter \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}} >  \ref{sec:pad-hold} \textit{\nameref{sec:pad-hold}})
    \end{itemize}
\end{itemize}


\subsection{Functional Description}

\subsubsection{HP System Clock}

As Figure \ref{figure:Clock} shows, CPU\_CLK is the master clock for CPU and its frequency can be as high as 160 MHz. Alternatively, CPU can run at lower frequencies, such as at 2 MHz, to achieve lower power consumption. CPU\_CLK shares the same clock sources with AHB\_CLK and APB\_CLK. Users can select from XTAL\_CLK, PLL\_160M\_CLK, or RC\_FAST\_CLK as the clock source of CPU\_CLK by configuring  \hyperref[fielddesc:PCRSOCCLKSEL]{PCR\_SOC\_CLK\_SEL}. For details, see Table  \ref{tab:resclk-cpu-clk-src} and Table \ref{tab:resclk-cpu-clk-freq}. By default, the CPU clock is sourced from XTAL\_CLK with a division factor of 1.

When selecting the clock source of HP\_ROOT\_CLK, or configuring the clock divisor for CPU\_CLK and AHB\_CLK, please also set \hyperref[fielddesc:PCRBUSCLOCKUPDATE]{PCR\_BUS\_CLOCK\_UPDATE} to apply the new configuration, and read \hyperref[fielddesc:PCRBUSCLOCKUPDATE]{PCR\_BUS\_CLOCK\_UPDATE} to see if new configuration takes effect.
\end{tiplisting}

As shown in \ref{figure:Clock}, to generate APB\_CLK, AHB\_CLK might be divided twice. The first division is compulsory. That is, AHB\_CLK is always divided by the divisor (\hyperref[fielddesc:PCRAPBDIVNUM]{PCR\_APB\_DIV\_NUM} + 1). The second division (also called automatic frequency reduction) is optional. When there is no request from the host in the chip to access peripheral registers, AHB\_CLK will be further divided by (\hyperref[fielddesc:PCRAPBDECREASEDIVNUM]{APB\_DECREASE\_DIV\_NUM} + 1) to lower power consumption. If the host initiates a request to access peripheral registers, APB\_CLK will be restored to the frequency after the first division.

Note that the function of automatic frequency reduction can be disabled (already disabled by default) by configuring \hyperref[fielddesc:PCRAPBDECREASEDIVNUM]{APB\_DECREASE\_DIV\_NUM} to 0.




\subsubsection{LP System Clock}

The LP system can operate when most other clocks are disabled. LP system clocks include LP\_SLOW\_CLK and LP\_FAST\_CLK.

The clock sources for LP\_SLOW\_CLK and LP\_FAST\_CLK are low-frequency clocks:
\begin{itemize}
    \item LP\_SLOW\_CLK can be derived from:
    \begin{itemize}
        \item RC\_SLOW\_CLK
        \item XTAL32K\_CLK
        \item OSC\_SLOW\_CLK
    \end{itemize}
    \item LP\_FAST\_CLK can be derived from:
    \begin{itemize}
        \item 20 MHz XTAL\_D2\_CLK, which is XTAL\_CLK divided by 2
        \item RC\_FAST\_CLK
        \item XTAL\_CLK
    \end{itemize}
\end{itemize}


The clock source of LP\_DYN\_SLOW\_CLK is LP\_SLOW\_CLK. There is no frequency change from the clock source.

The clock source of LP\_DYN\_FAST\_CLK depends on the chip's power mode (see Chapter \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}).

\begin{itemize}
    \item Select LP\_FAST\_CLK as its clock source in Active and Modem-sleep mode
    \item Select LP\_SLOW\_CLK as its clock source in Light-sleep and Deep-sleep mode
\end{itemize}

\textbf{PLL\_F480M\_CLK}

PLL\_F480M\_CLK is a 480 MHz clock. PLL\_F160M\_CLK, PLL\_F120M\_CLK, PLL\_F80M\_CLK, PLL\_F48M\_CLK, PLL\_F40M\_CLK, and CRYPTO\_CLK are divided from PLL\_F480M\_CLK.

\textbf{CRYPTO\_CLK}

As shown in Figure \ref{tab:resclk-periph-clks-root-hp}, CRYPTO\_CLK can be derived from XTAL\_CLK, PLL\_F480M\_CLK, or RC\_FAST\_CLK, and its frequency is up to 160 MHz.


To protect encryption and decryption peripherals from DPA (Differential Power Analysis) attacks, a random divider strategy is implemented for the functional clock of encryption and decryption peripherals. Four security levels are available, depending on the range of random divider. Users can select the security level by configuring  \hyperref[regdesc:HPSYSTEMSECDPACONFREG]{HP\_SYSTEM\_SEC\_DPA\_CONF\_REG}. If  \hyperref[fielddesc:HPSYSTEMSECDPACFGSEL]{HP\_SYSTEM\_SEC\_DPA\_CFG\_SEL} is set to 1, the security level is determined by the configuration of \hyperref[fielddesc:EFUSESECDPALEVEL]{EFUSE\_SEC\_DPA\_LEVEL}, otherwise, by the value of  \hyperref[fielddesc:HPSYSTEMSECDPALEVEL]{HP\_SYSTEM\_SEC\_DPA\_LEVEL}.

\textbf{LED\_PWM Clock}

LEDC module uses PLL\_F80M\_CLK, RC\_FAST\_CLK or XTAL\_CLK as its clock source. When the system is in low-power mode (APB\_CLK is disabled), most peripherals are halted, but LEDC can still work via RC\_FAST\_CLK.

\textbf{Slow Clock for Timer Group 0}
\label{sec:pcr_peri_timg}

Using XTAL\_CLK as a reference, TIMG0 can calculate the frequency of slow clock sources provided by \chipname{}. The slow clock source can be selected by configuring \hyperref[fielddesc:PCR32KSEL]{PCR\_32K\_SEL}:

0: Select RC\_FAST\_CLK clock\\
1: Select XTAL32K\_CLK clock\\
2: Select OSC\_SLOW\_CLK clock\\
3: Select RC\_SLOW\_CLK clock\\

\section{Programming Procedures}
\label{sec:resclk-prog-procedures}

\subsection{HP System Clock Configuration}

When configuring \hyperref[fielddesc:PCRSOCCLKSEL]{PCR\_SOC\_CLK\_SEL} to select the clock source of HP\_ROOT\_CLK, or configuring the clock divisor for CPU\_CLK via \hyperref[fielddesc:PCRCPUDIVNUM]{PCR\_CPU\_DIV\_NUM} and AHB\_CLK via \hyperref[fielddesc:PCRAHBDIVNUM]{PCR\_AHB\_DIV\_NUM}, please also set the enable register \hyperref[fielddesc:PCRBUSCLOCKUPDATE]{PCR\_BUS\_CLOCK\_UPDATE} to apply the new configuration. To check whether the new configuration takes effect, read \hyperref[fielddesc:PCRBUSCLOCKUPDATE]{PCR\_BUS\_CLOCK\_UPDATE} and see if it is 0.

\subsection{LP System Clock Configuration}

The clock source of LP\_SLOW\_CLK can be configured via  \hyperref[fielddesc:LPCLKRSTSLOWCLKSEL]{LP\_CLKRST\_SLOW\_CLK\_SEL}.

The clock source of LP\_FAST\_CLK can be configured via  \hyperref[fielddesc:LPCLKRSTFASTCLKSEL]{LP\_CLKRST\_FAST\_CLK\_SEL}.

\subsection{Peripheral Clock Reset and Configuration}\label{sec:resclk-periperal-clock-configuration}

\begin{importantlisting}
\vspace{-0.5em}

\chipname{} features low power consumption. This is why some peripheral clocks are gated (disabled) by default. Before using any of these peripherals, it is mandatory to enable the clock for the given peripheral by setting the corresponding CLK\_EN bit to 1, and release the peripheral from reset state to make it operational by setting the RST\_EN bit to 0.
\end{importantlisting}

The clocks of most peripherals can be classified into two types:
\begin{itemize}
    \item Bus clock: used to configure peripheral registers.
    \item Functional clock: such as UART's reference clock, used by peripherals to operate.
\end{itemize}
The functional clock of most peripherals can be selected from multiple clock sources. For clock gating registers, whether they are used to gate the bus clock (AHB\_CLK, APB\_CLK) or the functional clock will be stated in the corresponding register descriptions.

Bus clock switches, functional clock switches, and configuration registers for clock source selection and clock frequency division are grouped into the PCR module. For more information, see Section \ref{sec:resclk-reg-summ} \textit{\nameref{sec:resclk-reg-summ}}.

When a peripheral is not working, users can turn off its functional clock by configuring related PCR registers. Turning off the peripheral’s functional clock does not affect the rest of the system.

Take the I2C clock configuration as an example.


\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{\modulefiles/figures/sys_clk_struct_pcr.eps}
    \caption{I2C Clock Configuration Example}
    \label{figure:Clock Config}
\end{figure}


Figure \ref{figure:Clock Config} shows the clock structure of I2C. The clock structure of other peripherals is similar to this one. CLK\_SWITCH is used to select a clock output and CLK\_GATE to turn on/off the clock.

In scenarios that require low power consumption, when the peripheral is not in use, in addition to turning off the functional clock, the bus clock of the peripheral can also be turned off to further lower the power consumption.

Note that if you turn off the bus clock first, the functional clock may continue working. Therefore, when turning off clocks, it is recommended to turn off the functional clock first and then the bus clock; when turning on clocks, it is recommended to  turn on the bus clock first and then the functional clock.

\begin{tiplisting}
In this chapter, all divisor configuration registers are configured with the actual divisor minus 1.
\end{tiplisting}

The addresses of the third and fourth to last registers with the LP\_AON prefix in this section are relative to the Low-power Always-on Register (LP\_AON) base address.The other addresses in this section are relative to the Low-power Clock/Reset Register (LP\_CLKRST) base address. For base address, please refer to Table  \ref{tab:sysmem-base-address} in Chapter \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}.
