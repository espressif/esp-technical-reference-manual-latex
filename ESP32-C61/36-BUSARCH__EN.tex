\chapter{Bus Architecture}
\label{mod:busarch}

The \chipname{} SoC uses a multi-layer bus matrix as the core of its communication architecture, connecting multiple masters and slaves efficiently. This architecture supports concurrent access, effectively mitigating conflicts so that the high-performance RISC-V core, DMA controllers, debug interfaces, and various peripherals can operate simultaneously with minimal contention. By optimizing data flow and system performance, the bus matrix ensures efficient system operation even when multiple high-speed peripherals are running at the same time.

The main system of the \chipname{} SoC consists of:

\begin{itemize}
    \item Masters:
        \begin{itemize}
            \item High-performance (HP) 32-bit RISC-V processor I-bus, D-bus, and AHB bus
            \item TRACE
            \item TCM MEM MONITOR
            \item PSRAM MEM MONITOR
            \item GDMA
            \item SDIO SLAVE
        \end{itemize}
    \item Slaves:
        \begin{itemize}
            \item ROM
            \item HP SRAM
            \item Cache
            \item HP \& LP peripherals
            \item CPU peripherals
        \end{itemize}
\end{itemize}

The bus architecture of the SoC is shown below.

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/bus_matrix.png}
    \begin{minipage}{0.9\textwidth}
    \small
    Note: \\
    1. Circles at intersections indicate that the bus (column) can access the corresponding slave device (row).\\
    2. The GDMA has a data path that bypasses the bus matrix and directly accesses the external memory.\\
    \end{minipage}
    \caption{\chipname{} Bus Architecture}
    \label{fig:bus-arch}
\end{figure}

Below is the description of each bus shown in Figure \ref{fig:bus-arch} \textit{\nameref{fig:bus-arch}}:

\begin{itemize}
    \item \textbf{HP CPU I-bus}: Connects the instruction bus of the high-performance (HP) 32-bit RISC-V processor to the bus matrix for instruction fetch. Accessible targets include memory that stores code (such as ROM, HP SRAM, or external memory accessed via MSPI).
    \item \textbf{HP CPU D-bus}: Connects the data bus of the high-performance (HP) 32-bit RISC-V processor to the bus matrix for literal loads and data read/write. Accessible targets include memory that stores code or data (such as ROM, HP SRAM, or external memory accessed via MSPI).
    \item \textbf{HP CPU AHB bus}: Connects the AHB bus of the high-performance (HP) 32-bit RISC-V processor to the bus matrix for accessing peripherals’ data. Accessible targets include HP \& LP peripherals and CPU peripherals.
    \item \textbf{TRACE bus}: Connects the TRACE module to the bus matrix for performing debug operations. Accessible targets include HP SRAM.
    \item \textbf{TCM MEM MONITOR bus}: Connects the TCM MEM MONITOR to the bus matrix for writing monitored data into memory. Accessible targets include HP SRAM.
    \item \textbf{PSRAM MEM MONITOR bus}: Connects the PSRAM MEM MONITOR to the bus matrix for writing monitored data into memory. Accessible targets include HP SRAM.
    \item \textbf{GDMA bus}: Provides one route connected to the bus matrix and another route connected to external memory. GDMA uses this bus for high-speed data transfers, including transfers between peripherals and memory, as well as memory-to-memory transfers. Accessible targets include HP SRAM and external memory. For scenarios where GDMA accesses external memory, it is recommended to use a longer burst length whenever possible to ensure high data throughput.
    \item \textbf{SDIO SLAVE bus}: Connects the SDIO SLAVE to the bus matrix for accessing data in memory or peripherals. Accessible targets include HP SRAM, HP \& LP peripherals, and CPU peripherals.
\end{itemize}
