\chapter{SPI 控制器 (SPI)}
\label{mod:spi}

\section{概述}

串行外设接口 (SPI) 是一种同步串行接口，可用于与外围设备进行通信。
\chipname{} 芯片集成了三个 SPI 控制器：
\begin{itemize}
  \item SPI0
  \item SPI1
  \item 通用 SPI2，即 GP-SPI2
\end{itemize}
SPI0 和 SPI1 控制器 (MSPI) 主要供内部使用以访问外部 flash 及 PSRAM。因此，本章节将主要介绍 GP-SPI2 控制器。


\subfile{\modulefiles/33-SPI-terms__CN.tex}

\vspace{-2em}
\section{特性}



GP-SPI2 具有以下特性：
\begin{itemize}
    \item 用作主机或用作从机
    \item 支持半双工通信和全双工通信
    \item 支持 CPU 控制的传输类型以及 DMA 控制的传输类型
    \item 支持多种数据模式：
        \begin{itemize}
            \item 1-bit SPI 模式
            \item 2-bit Dual SPI 模式
            \item 4-bit Quad SPI 模式
            \item QPI 模式
        \end{itemize}
    \item 时钟频率可配置：
        \begin{itemize}
        \item 用作主机时：时钟频率可达 80 MHz
            \item 用作从机时：时钟频率可达 40 MHz
        \end{itemize}
    \item 数据长度可配置：
    \begin{itemize}
        \item 在主机和从机 CPU 控制的传输下：数据长度为 1$\sim$64 B
        \item 在主机 DMA 控制的单次传输下：数据长度为 1$\sim$32 KB
        \item 在主机 DMA 控制的分段配置传输下：数据长度字节数无限制
        \item 在从机 DMA 控制的单次或连续传输下：数据长度字节数无限制
    \end{itemize}
    \item 读写数据的比特位顺序可配置
    \item 为 CPU 控制的传输和 DMA 控制的传输分别提供独立中断
    \item 时钟极性和相位可配置
    \item 四种 SPI 时钟模式：模式 0 $\sim$模式 3
    \item 用作主机时，提供六条 CS 线：CS0$\sim$CS5
    \item 支持访问 SPI 接口的传感器、显示屏控制器、flash 或 RAM 芯片
\end{itemize}

\begin{itemize}
    \item 在 CPU 控制的传输类型下：CPU <-> GP-SPI2 <-> SPI 设备
    \item 在 DMA 控制的传输类型下：GDMA <-> GP-SPI2 <-> SPI 设备
\end{itemize}

GP-SPI2 输入输出信号的前缀为 ``FSPI"。FSPI 总线信号可通过 GPIO 交换矩阵或 IO MUX 与 GPIO 管脚相连。更多信息，见章节  \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}}。

\subsection{非字节对齐传输}
\label{sec:unaligned-byte-transfer}

\begin{itemize}
    \item 用作主机时，GP-SPI2 主机可以按比特收发数据，比特长度等于   \hyperref[fielddesc:SPIMSDATABITLEN]{SPI\_MS\_DATA\_BITLEN} + 1，并且比特长度在 1/2/4/8 线模式下分别是 1/2/4/8 的倍数。%
    \begin{itemize}
        \item 发送长度不是 8 比特整数倍的数据时，软件需要填充最后不满 8 比特的数据为完整的 1 字节数据。
        \item 接收长度不是 8 比特整数倍的数据时，不足 1 字节的部分依然按照 1 字节进行接收。
    \end{itemize}
    \item 用作从机时，GP-SPI2 从机可以按比特收发数据，并且比特长度在 1/2/4/8 线模式下分别是 1/2/4/8 的倍数。
    \begin{itemize}
        \item 发送长度不是 8 比特整数倍的数据时，软件需要填充最后不满 8 比特的数据为完整的 1 字节数据。
        \item 接收长度不是 8 比特整数倍的数据时，不足 1 字节的部分依然按照 1 字节进行接收，从机接收总长度可以通过  \hyperref[fielddesc:SPISLVDATABITLEN]{SPI\_SLV\_DATA\_BITLEN} 获取，并且最后 1 字节中的有效比特可以通过 \hyperref[fielddesc:SPISLVLASTBYTESTRB]{SPI\_SLV\_LAST\_BYTE\_STRB} 确定。
    \end{itemize}
\end{itemize}
\begin{tiplisting}
无论 GP-SPI2 用作主机还是从机，不足 1 字节部分的接收/发送比特顺序仍然按照配置进行。
\end{tiplisting}

\subsection{CPU 控制的数据传输}\label{sec:cpu-controlled-data-transfer}
GP-SPI2 提供了 16 个 32-bit 的数据 buffer，即 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}，见图  \ref{Figure:SPI_BUFFER}。
CPU 控制的传输表示在该次传输中发送的数据来自数据 buffer 或接收的数据存入数据 buffer。
在这种传输类型下，每次传输事务均需要先配置相关寄存器，然后由 CPU 来触发。因此，CPU 控制的传输只能是单次传输，即仅包含一次传输事务。CPU 控制的传输支持全双工通信和半双工通信。

    \centering

    \caption{CPU 控制的传输中使用的数据 Buffer}
    \label{Figure:SPI_BUFFER}
\end{figure}

\subsubsection{CPU 控制的主机传输} \label{GP-SPI2 Master CPU}

GP-SPI2 用作主机时，无论全双工通信还是半双工通信，CPU 控制的数据传输均通过 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 完成。此外，用户可选择“高位模式”仅使用其中部分寄存器，具体见下方列表描述。


\begin{itemize}
    \item \textbf{TX 数据：}
    \begin{itemize}
        \item 未使能“高位模式”（\hyperref[fielddesc:SPIUSRMOSIHIGHPART]{SPI\_USR\_MOSI\_HIGHPART} 置 0）：此时，TX 数据取自 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}，且每传输一个字节，取 TX 数据的地址即递增 1。\textbf{如果数据长度大于 64 字节，\hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} $\sim$ \hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 中的数据会被多次发送。}\\
        以 256 个字节为一个周期：
        \begin{itemize}
            \item 字节 0$\sim$字节 63 依次发送 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 的数据。
            \item 字节 64$\sim$字节 255 重复发送 \hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}[31:24] 中的数据。
            \item 字节 256$\sim$字节 319（另一组数据的前 64 个字节）重新依次发送 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 的数据。以此类推。
        \end{itemize}
        \textbf{例如：}发送 258 个字节（字节 0$\sim$字节 257），则：
        \begin{itemize}
            \item 字节 0$\sim$字节 63 依次发送 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 的数据。
            \item 字节 64$\sim$字节 255 重复发送 \hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}[31:24] 中的数据。
            \item 字节 256$\sim$字节 257 重新依次发送地址 0 (\hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}[7:0]) 和地址 1 (\hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}[15:8]) 的数据：
                \begin{itemize}
                    \item 字节 256 取数据的地址为 256 对 64 取模的结果（256 \% 64 = 0），即地址 0 (\hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}[7:0])。
                    \item 字节 257 取数据的地址为 257 对 64 取模的结果（257 \% 64 = 1），即地址 1 (\hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}[15:8])。
                \end{itemize}
        \end{itemize}
        \item 使能“高位模式”（\hyperref[fielddesc:SPIUSRMOSIHIGHPART]{SPI\_USR\_MOSI\_HIGHPART} 置 1）：此时，TX 数据取自 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}，且每传输一个字节，取 TX 数据的地址即递增 1。\textbf{如果数据长度大于 32 字节，则 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG} $\sim$ \hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 中的数据会被多次发送。}\\
        以 256 个字节为一个周期：
        \begin{itemize}
            \item 字节 0$\sim$字节 31 依次发送 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 中的数据。
            \item 字节 32$\sim$字节 255 重复发送 \hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}[31:24] 中的数据。
            \item 字节 256$\sim$字节 287（另一组数据的前 32 个字节）重新依次发送 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 中的数据。以此类推。
        \end{itemize}
        \textbf{例如：}发送 258 个字节（字节 0$\sim$字节 257），则：
        \begin{itemize}
            \item 字节 0$\sim$字节 31 依次发送 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 的数据。
            \item 字节 32$\sim$字节 255 重复发送 \hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}[31:24] 中的数据。
            \item 字节 256$\sim$字节 257 重新依次发送地址 0 (\hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}[7:0]) 和地址 1 (\hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}[15:8]) 的数据：
                \begin{itemize}
                    \item 字节 256 取数据的地址为 256 对 32 取模的结果（256 \% 32 = 0），即地址 0 (\hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}[7:0])。
                    \item 字节 257 取数据的地址为 257 对 32 取模的结果（257 \% 32 = 1），即地址 1 (\hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}[15:8])。
                \end{itemize}
        \end{itemize}


    \end{itemize}
    \item \textbf{RX 数据：}
    \begin{itemize}
        \item 未使能“高位模式”（\hyperref[fielddesc:SPIUSRMISOHIGHPART]{SPI\_USR\_MISO\_HIGHPART} 置 0）：此时，RX 数据存入 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}，且每传输一个字节，存 RX 数据的地址即递增 1。\textbf{如果数据长度大于 64 字节，\hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} $\sim$ \hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 中的数据会被覆盖。}\\
        以 256 个字节为一个周期：
        \begin{itemize}
            \item 字节 0$\sim$字节 63 依次存入 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}。
            \item 字节 64$\sim$字节 255 重复存入 \hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}[31:24]。
            \item 字节 256$\sim$字节 319（另一组数据的前 64 个字节）重新依次存入 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}。以此类推。
        \end{itemize}
        \textbf{例如：}接收 258 个字节（字节 0$\sim$字节 257），则：
        \begin{itemize}
            \item 字节 0$\sim$字节 63 依次存入 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}。
            \item 字节 64$\sim$字节 255 重复存入 \hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}[31:24]。
            \item 字节 256$\sim$字节 257 依次存入地址 0 (\hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}[7:0]) 和地址 1 (\hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}[15:8])：
            \begin{itemize}
                \item 字节 256 存数据的地址为 256 对 64 取模的结果 (256 \% 64 = 0)，即地址 0 (\hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}[7:0])。
                \item 字节 257 存数据的地址为 257 对 64 取模的结果 (257 \% 64 = 1)，即地址 1 (\hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}[15:8])。
            \end{itemize}
        \end{itemize}
        \item 使能“高位模式”（\hyperref[fielddesc:SPIUSRMISOHIGHPART]{SPI\_USR\_MISO\_HIGHPART} 置 1）：此时，RX 数据存入 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}，且每传输一个字节，存 RX 数据的地址即递增 1。\textbf{如果数据长度大于 32 字节，则 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG} $\sim$ \hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 中的数据会被覆盖。}\\
        以 256 个字节为一个周期：
        \begin{itemize}
            \item 字节 0$\sim$字节 31 依次存入 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}。
            \item 字节 32$\sim$字节 255 重复存入 \hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}[31:24]。
            \item 字节 256$\sim$字节 287（另一组数据的前 32 个字节）依次存入 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}。以此类推。
        \end{itemize}
        \textbf{例如：}接收 258 个字节（字节 0$\sim$字节 257），则：
        \begin{itemize}
            \item 字节 0$\sim$字节 31 依次存入 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}。
            \item 字节 32$\sim$字节 255 重复存入 \hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}[31:24]。
            \item 字节 256$\sim$字节 257 依次存入地址 0 (\hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}[7:0]) 和地址 1 (\hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}[15:8])：
            \begin{itemize}
                \item 字节 256 存数据的地址为 256 对 32 取模的结果 (256 \% 32 = 0)，即地址 0 (\hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}[7:0])。
                \item 字节 257 存数据的地址为 257 对 32 取模的结果 (257 \% 32 = 1)，即地址 1 (\hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}[15:8])。
            \end{itemize}
        \end{itemize}
    \end{itemize}
\end{itemize}
\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
    \item 上述的 TX/RX 数据均按字节寻址。
    \begin{itemize}
        \item 如果未使能``高位模式"，地址 0 表示 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}[7:0]，地址 1 表示 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}[15:8]，以此类推。
        \item 如果使能了``高位模式"，地址 0 表示 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}[7:0]，地址 1 表示 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}[15:8]，以此类推。
    \end{itemize}
    最大地址为 \hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}[31:24]。
    \item 为避免 TX/RX 数据传输错误，如 TX 数据重复发送或 RX 数据被覆盖等问题，请确保寄存器配置正确。
\end{itemize}
\end{tiplisting}

\subsubsection{CPU 控制的从机传输} \label{GP-SPI2 slave CPU}

GP-SPI2 用作从机时，无论全双工通信或半双工通信，CPU 控制的数据传输均通过 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 完成，均采用按字节寻址。

\begin{itemize}
    \item 全双工通信方式下：\hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 地址从 0 开始，且每传输一个字节，地址即递增 1。如果数据地址大于 63，则 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 中的数据会被覆盖。覆盖规律同主机模式下的非“高位模式”。
    \item 半双工通信方式下：\hyperref[slave communication format]{通信格式}中的 ADDR 值即为 RX 数据或 TX 数据的起始地址，对应 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}。每传输一个字节，则 RX 或 TX 地址即递增 1。如果地址大于 63（即大于最高地址：\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}[31:24]），\hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 中的数据会被覆盖。覆盖规律同主机模式下的“高位模式”。
\end{itemize}



用户可根据具体应用，将 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}：
\begin{itemize}
    \item 全部用作数据 buffer
    \item 部分用作数据 buffer，部分用作状态 buffer
    \item 全部用作状态 buffer
\end{itemize}



\subsection{DMA 控制的数据传输}\label{sec:usage-of-gp-spi2-dma-controlled-mode}
在 DMA 控制的传输中，GDMA RX 模块接收数据，GDMA TX 模块发送数据。GP-SPI2 用作主机和从机时均支持这种传输类型。

DMA 控制的传输可以是：

\begin{itemize}
    \item 一次单次传输，仅包含一次传输事务。GP-SPI2 用作主机和从机时均支持这种单次传输。
    \item 分段配置传输，包含多个传输事务（即多个分段）。仅有 GP-SPI2 用作主机时支持这种分段配置传输。更多信息，见章节  \ref{sec:gp-spi-dma-seg-conf-trans}。
    \item 从机连续传输，包含多次传输事务。仅在 GP-SPI2 用作从机时支持这种从机连续传输。更多信息，见章节  \ref{sec:gp-spi2-slave-seg-trans}。
\end{itemize}

DMA 控制的传输只需由 CPU 触发一次即可完成多次传输事务。此类传输一旦被触发，GDMA 引擎从 DMA 链接的内存中发送数据，或将收到的数据存入 DMA 链接的内存中，无需 CPU 的干预。

DMA 控制的传输支持全双工通信、半双工通信以及章节 \ref{sec:gp-spi2-master-mode} 和章节 \ref{sec:gp-spi2-slave-mode} 所描述的功能。同时，GDMA RX 模块与 GDMA TX 模块互不影响，即支持四种全双工通信：

\begin{itemize}
    \item 在 DMA 控制模式下接收数据，并在 DMA 控制模式下发送数据；

    \item 在 DMA 控制模式下接收数据，但在 CPU 控制模式下发送数据；

    \item 在 CPU 控制模式下接收数据，但在 DMA 控制模式下发送数据；

    \item 在 CPU 控制模式下接收数据，并在 CPU 控制模式下发送数据。

\end{itemize}


\subsubsection{GDMA 配置}\label{sec:DMA-conf}


\begin{itemize}
    \item 选择 GDMA 通道 \textcolor{red}{\it{n}}，并配置 GDMA TX/RX 描述符，见章节 \ref{mod:dma} \textit{\nameref{mod:dma}}；

    \item 置位 \hyperref[fielddesc:GDMAINLINKSTARTCHN]{GDMA\_INLINK\_START\_CH\textcolor{red}{\it{n}}}/\hyperref[fielddesc:GDMAOUTLINKSTARTCHN]{GDMA\_OUTLINK\_START\_CH\textcolor{red}{\it{n}}}  启动 GDMA RX/TX 引擎；

    \item 如果置位 \hyperref[fielddesc:GDMAOUTLINKRESTARTCHN]{GDMA\_OUTLINK\_RESTART\_CH\textcolor{red}{\it{n}}}，则在所有 GDMA TX buffer 用完之前，或在 GDMA TX 引擎重置之前，新的 TX buffer 将会被添加到最后使用中的 TX buffer 结尾；

\subsubsection{GDMA TX/RX Buffer 长度控制}\label{sec:DMA-buffer-length}

配置的 GDMA TX/RX buffer 长度最好应等于实际传输数据的长度。如果配置的 GDMA TX/RX buffer 长度不等于实际传输数据的长度：

\begin{itemize}
    \item 如果配置的 GDMA TX buffer 长度小于实际传输的数据长度，则多出来的数据将与最后传输的TX buffer 数据相同。同时触发 \hyperref[int:SPIOUTFIFOEMPTYERRINT]{SPI\_OUTFIFO\_EMPTY\_ERR\_INT} 和 \hyperref[int:GDMAOUTEOFINT]{GDMA\_OUT\_EOF\_CH\textcolor{red}{\it{n}}\_INT} 中断。

    \item 如果配置的 GDMA TX buffer 长度大于实际传输的数据长度，则 TX buffer 中的数据未被完全使用，即使稍后链接了新的 TX buffer，上个 TX buffer 中剩余的数据也将参与后续传输。请特别注意这一点，或保存未使用的数据并重置 GDMA。

    \item 如果配置的 GDMA RX buffer 长度小于实际传输的数据长度，则多出来的数据将会丢失。同时触发 \hyperref[int:SPIINFIFOFULLERRINT]{SPI\_INFIFO\_FULL\_ERR\_INT} 和 \hyperref[int:SPITRANSDONEINT]{SPI\_TRANS\_DONE\_INT} 中断。但不会触发 \hyperref[int:GDMAINSUCEOFINT]{GDMA\_IN\_SUC\_EOF\_CH\textcolor{red}{\it{n}}\_INT} 中断。

    \item 如果配置的 GDMA RX buffer 长度大于实际传输的数据长度，则 RX buffer 未被使用的部分被丢弃，下次传输直接使用后面链接的 RX buffer。

\end{itemize}

\subsection{ GP-SPI2 用作主机和从机时的数据流控制}\label{Subsection:Data Flow Control in GP-SPI2 Master and Slave Mode}

GP-SPI2 用作主机和从机时均支持 CPU 控制的数据传输和 DMA 控制的数据传输。CPU 控制的数据传输发生在 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 和外围 SPI 设备之间。DMA 控制的数据传输发生在配置好的 GDMA TX/RX buffer 和外围 SPI 设备之间。用户可在传输开始之前，配置 \hyperref[fielddesc:SPIDMARXENA]{SPI\_DMA\_RX\_ENA} 和 \hyperref[fielddesc:SPIDMATXENA]{SPI\_DMA\_TX\_ENA} 来选择需要的传输类型。

图 \ref{fig:gp-spi-block-diagram} 所示为 GP-SPI2 主要的功能模块，包括：

    \item SPI Buffer: \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}，见图  \ref{Figure:SPI_BUFFER}。CPU 控制的传输，数据在 SPI buffer 中准备。

    \item SPI\_CLK 生成器 (SPI\_CLK Generator)：对 clk\_spi\_mst 进行分频生成 SPI\_CLK 信号。分频系数由 \hyperref[fielddesc:SPICLKCNTN]{SPI\_CLKCNT\_N} 和 \hyperref[fielddesc:SPICLKDIVPRE]{SPI\_CLKDIV\_PRE} 共同决定，见章节 \ref{sec:gp-spi2-clk}。
    \item SPI\_CLK\_out Mode Control：发送数据传输以及从机所需的 SPI\_CLK 信号。
    \item SPI\_CLK\_in Mode Control：当 GP-SPI2 用作从机时，用于捕获 SPI 主机发出的 SPI\_CLK 信号。

\end{itemize}

\subsubsection{GP-SPI2 用作主机时的数据流控制}\label{Subsection:Data Flow Control in Master Mode}

\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/GPSPI_Mst_Data_Flow.eps}}
    \caption{GP-SPI2 用作主机时的数据流控制}
    \label{fig:spi-master-data-flow-control}
\end{figure}

图 \ref{fig:spi-master-data-flow-control} 所示为 GP-SPI2 用作主机时的数据流。其控制逻辑如下：
\begin{itemize}
    \item RX 数据：时序模块捕获 FSPI 总线上的数据，然后 \emph{spi\_mst\_din\_ctrl} 模块将比特数据转化为字节数据，暂存于 \emph{spi\_rx\_afifo} 中，此后根据控制方式转存至不同的接收位置：
	\begin{itemize}
	    \item CPU 控制：转存至 SPI\_W0\_REG$\sim$SPI\_W15\_REG
	    \item DMA 控制：转存至 GDMA RX buffer
	\end{itemize}
    \item TX 数据：\emph{buf\_tx\_afifo} 模块暂存待发送数据。根据控制方式不同，待发送数据来自不同的位置：
    \begin{itemize}
        \item CPU 控制：TX 数据来自 SPI\_W0\_REG$\sim$SPI\_W15\_REG
        \item DMA 控制：TX 数据来自 GDMA TX buffer
    \end{itemize}
\end{itemize}

\emph{buf\_tx\_afifo} 中的数据会由时序模块以 1/2/4-bit 的模式发送出去。具体数据模式由 GP-SPI2 状态机控制。时序模块可用于时序补偿。更多信息，见章节 \ref{sec:gp-spi-timing-compensation}。

    \label{fig:spi-slave-data-flow-control}
\end{figure}

图 \ref{fig:spi-slave-data-flow-control} 所示为 GP-SPI2 用作从机时的数据流控制。其控制逻辑如下：
\begin{itemize}
    \item 在 CPU/DMA 控制的全双工/半双工传输下，当外部 SPI 主机发起 SPI 传输后，FSPI 总线上的数据将被捕获，然后由 \emph{spi\_slv\_din\_ctrl} 模块转换为字节，暂存于 \emph{spi\_rx\_afifo} 中。
    \begin{itemize}
        \item 在 CPU 控制的全双工传输中，暂存于 \emph{spi\_rx\_afifo} 中的 RX 数据之后会被相继转存到 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}。

        \item 在半双工 \hyperref[cmd:wrbuf]{Wr\_BUF} 传输中，收到地址值 (SLV\_ADDR[7:0]) 后，\emph{spi\_rx\_afifo} 中暂存的 RX 数据将转存至寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 的相应地址中。

        \item 在 DMA 控制的全双工传输中，或在半双工 \hyperref[cmd:wrdma]{Wr\_DMA} 传输中，\emph{spi\_rx\_afifo} 中暂存的 RX 数据将转存至配置好的 GDMA RX buffer 中。

    \end{itemize}

    \item 在 CPU 控制的全双工/半双工传输中，待发送的数据暂存在 \emph{buf\_tx\_afifo} 中；而在 DMA 控制的全双工/半双工传输中，待发送的数据暂存在 \emph{dma\_tx\_afifo} 中。因此，在一次从机连续传输中，CPU 控制的 \hyperref[cmd:rdbuf]{Rd\_BUF} 传输事务和 DMA 控制的 \hyperref[cmd:rddma]{Rd\_DMA} 传输事务可同时发生。
    \begin{itemize}
        \item 在 CPU 控制的全双工传输中，如果置位了 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE} 和 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN}，同时清零了 SPI\_DMA\_TX\_ENA，则 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 中的数据将被转存至 \emph{buf\_tx\_afifo} 中。

        \item 在 CPU 控制的半双工传输中，如果置位了 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE}，清零了 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN}，且收到指令 \hyperref[cmd:rdbuf]{Rd\_BUF} 和地址 SLV\_ADDR[7:0]，则 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 相应地址中的数据将被转存至 \emph{buf\_tx\_afifo} 中。

        \item 在 DMA 控制的全双工传输中，如果置位了 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE}、\hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN} 和 SPI\_DMA\_TX\_ENA，则 GDMA TX buffer 中的数据将被转存至 \emph{dma\_tx\_afifo} 中。

        \item 在 DMA 控制的半双工传输中，如果置位了 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE}，清零了 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN}，且收到指令 \hyperref[cmd:rddma]{Rd\_DMA}，则 GDMA TX buffer 中的数据将被转存至 \emph{dma\_tx\_afifo} 中。

    \end{itemize}

\emph{buf\_tx\_afifo} 或 \emph{dma\_tx\_afifo} 的数据将由 \emph{spi\_slv\_dout\_ctrl} 模块以 1/2/4-bit 的模式发送出去。
\end{itemize}

\subsection{GP-SPI2 用作主机}\label{sec:gp-spi2-master-mode}
清零 \hyperref[regdesc:SPISLAVEREG]{SPI\_SLAVE\_REG} 中 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE} 位可将 GP-SPI2 配置成主机。此时，GP-SPI2 提供时钟信号（GP-SPI2 模块时钟的分频时钟）和六条 CS 线 (CS0$\sim$CS5)。



\subsubsection{主机状态机}
\label{sec:gp-spi2-state-machine}

GP-SPI2 用作主机时，状态机在数据传输中控制其各个阶段，包括配置阶段 (CONF)、准备阶段 (PREP)、命令阶段 (CMD)、地址阶段 (ADDR)、空闲阶段 (DUMMY)、发送数据阶段 (DOUT) 和接收数据阶段 (DIN)。GP-SPI2 主要用于访问 1/2/4-bit SPI 设备，如 flash、外部 RAM 等。因此，GP-SPI2 各个阶段的命名规则应与 flash 以及外部 RAM 的时序名称保持一致。每个阶段的描述如下，GP-SPI2 状态机的工作流程见图  \ref{fig:spi-master-mode-flow-chart}。

\begin{enumerate}
    \item 空闲阶段 (IDLE)：GP-SPI2 未处于工作状态或正在用作从机。
    \item 配置阶段 (CONF)：仅用于 DMA 控制的\hyperref[sec:gp-spi-dma-seg-conf-trans]{分段配置传输}。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和 \hyperref[fielddesc:SPIUSRCONF]{SPI\_USR\_CONF} 使能该阶段。如果未使能该阶段，则说明当前传输为单次传输。
    \item 准备阶段 (PREP)：准备 SPI 传输事务，控制 SPI CS 建立时间。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和 \hyperref[fielddesc:SPICSSETUP]{SPI\_CS\_SETUP} 使能该阶段。
    \item\label{phase:cmd} 命令阶段 (CMD)：发送命令序列。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和 \hyperref[fielddesc:SPIUSRCOMMAND]{SPI\_USR\_COMMAND} 使能该阶段。
    \item\label{phase:addr} 地址阶段 (ADDR)：发送地址序列。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和 \hyperref[fielddesc:SPIUSRADDR]{SPI\_USR\_ADDR} 使能该阶段。
    \item 等待阶段 (DUMMY)：发送 DUMMY 序列。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和 \hyperref[fielddesc:SPIUSRDUMMY]{SPI\_USR\_DUMMY} 使能该阶段。
    \item\label{phase:data} 传输数据阶段 (DATA)：传输数据。
    \begin{itemize}
    \item DOUT：发送数据。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和 \hyperref[fielddesc:SPIUSRMOSI]{SPI\_USR\_MOSI} 使能该阶段。
    \item DIN：接收数据。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和 \hyperref[fielddesc:SPIUSRMISO]{SPI\_USR\_MISO} 使能该阶段。
    \end{itemize}
    \item 结束阶段 (DONE)：控制 SPI CS 保持时间。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 使能该阶段。
\end{enumerate}

\textbf{注意：\\}

主机状态机启动必须配置 \hyperref[fielddesc:SPIUSR]{SPI\_USR}。\hyperref[fielddesc:SPIMSTFDWAITDMATXDATA]{SPI\_MST\_FD\_WAIT\_DMA\_TX\_DATA} 可以控制 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 生效的时刻：
\begin{itemize}
    \item 当 \hyperref[fielddesc:SPIMSTFDWAITDMATXDATA]{SPI\_MST\_FD\_WAIT\_DMA\_TX\_DATA} 为 0 时，状态机相关状态在配置 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和其他控制寄存器后立即生效；
    \item 当 \hyperref[fielddesc:SPIMSTFDWAITDMATXDATA]{SPI\_MST\_FD\_WAIT\_DMA\_TX\_DATA} 为 1 时，如果状态机也配置了 DOUT 状态，则会等待 \emph{buf\_tx\_fifo} 中至少有数据后，\hyperref[fielddesc:SPIUSR]{SPI\_USR} 和其他控制寄存器才会生效，状态机开始启动。
\end{itemize}

\textbf{配置}

例如，当 GP-SPI2 读取数据时，且希望实现：
\begin{itemize}
    \item CMD 为 1-bit 模式
    \item ADDR 为 2-bit 模式
    \item DUMMY 为 8 个时钟周期
    \item DIN 为 4-bit 模式
\end{itemize}
则具体的寄存器配置如下：

\begin{enumerate}
    \item 配置 CMD 状态相关寄存器。
        \begin{itemize}
            \item 配置 \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE} 为需要的命令值；
            \item 配置 \hyperref[fielddesc:SPIUSRCOMMANDBITLEN]{SPI\_USR\_COMMAND\_BITLEN}。 \hyperref[fielddesc:SPIUSRCOMMANDBITLEN]{SPI\_USR\_COMMAND\_BITLEN} 为所需要的命令位长 - 1；
            \item 置位 \hyperref[fielddesc:SPIUSRCOMMAND]{SPI\_USR\_COMMAND}；
            \item 清除 \hyperref[fielddesc:SPIFCMDDUAL]{SPI\_FCMD\_DUAL} 和 \hyperref[fielddesc:SPIFCMDQUAD]{SPI\_FCMD\_QUAD}。
        \end{itemize}

    \item 配置 ADDR 状态相关寄存器。
        \begin{itemize}
            \item 配置 \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE} 为需要的地址值；
            \item 配置 \hyperref[fielddesc:SPIUSRADDRBITLEN]{SPI\_USR\_ADDR\_BITLEN}。 \hyperref[fielddesc:SPIUSRADDRBITLEN]{SPI\_USR\_COMMAND\_BITLEN} 为所需要的地址位长 - 1；
            \item 置位 \hyperref[fielddesc:SPIUSRADDR]{SPI\_USR\_ADDR} 和 \hyperref[fielddesc:SPIFADDRDUAL]{SPI\_FADDR\_DUAL}；
            \item 清除 \hyperref[fielddesc:SPIFADDRQUAD]{SPI\_FADDR\_QUAD}。
        \end{itemize}

    \item 配置 DUMMY 状态相关寄存器。
        \begin{itemize}
            \item 在 \hyperref[fielddesc:SPIUSRDUMMYCYCLELEN]{SPI\_USR\_DUMMY\_CYCLELEN} 中配置 DUMMY 周期，其中 \hyperref[fielddesc:SPIUSRDUMMYCYCLELEN]{SPI\_USR\_DUMMY\_CYCLELEN} 的值等于 DUMMY 阶段所需要的时钟周期数 - 1；
            \item 置位 \hyperref[fielddesc:SPIUSRDUMMY]{SPI\_USR\_DUMMY}。
        \end{itemize}

    \item 配置 DIN 状态相关寄存器。
        \begin{itemize}
            \item 在 \hyperref[fielddesc:SPIMSDATABITLEN]{SPI\_MS\_DATA\_BITLEN} 中配置读数据的位长； \hyperref[fielddesc:SPIMSDATABITLEN]{SPI\_MS\_DATA\_BITLEN} 的值等于所需要的位长 - 1；
            \item 置位 \hyperref[fielddesc:SPIFREADQUAD]{SPI\_FREAD\_QUAD} 和 \hyperref[fielddesc:SPIUSRMISO]{SPI\_USR\_MISO}；
            \item 清除 \hyperref[fielddesc:SPIFREADDUAL]{SPI\_FREAD\_DUAL}；
            \item 如果选择了 DMA 控制的传输类型，则需要配置 GDMA。如果选择了 CPU 控制的传输类型，则无需任何操作；
        \end{itemize}

    \item 清除 \hyperref[fielddesc:SPIUSRMOSI]{SPI\_USR\_MOSI}；
    \item 置位 \hyperref[fielddesc:SPIDMAAFIFORST]{SPI\_DMA\_AFIFO\_RST}、\hyperref[fielddesc:SPIBUFAFIFORST]{SPI\_BUF\_AFIFO\_RST} 和 \hyperref[fielddesc:SPIRXAFIFORST]{SPI\_RX\_AFIFO\_RST} 复位 buffer。

    \item 置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 开始 GP-SPI2 传输。
\end{enumerate}

\subsubsection{主机全双工通信（仅支持 1-bit 模式）}\label{sec:gp-spi2-1-bit}

\textbf{概述}

GP-SPI2 支持 SPI 全双工通信。在这种通信方式下，SPI 主机提供 CLK 和 CS 信号，然后与从机使用 1-bit 模式同时交换数据：MOSI（FSPID，发送），MISO（FSPIQ，接收）。用户可通过置位寄存器 \hyperref[regdesc:SPIUSERREG]{SPI\_USER\_REG} 中 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN} 位使能全双工通信。GP-SPI2 与从机使用全双工通信时的连接方式见图  \ref{fig:gp-spi2-full-duplex}。

\begin{figure}[H]
    \centering
    \includegraphics[width=0.4\textwidth]{\modulefiles/figures/gp-spi2-fd}
    \caption{GP-SPI2 主机使用全双工模式与 SPI 从机通信框图}
    \label{fig:gp-spi2-full-duplex}
\end{figure}

在全双工通信中，CMD、ADDR、DUMMY、DOUT 和 DIN 各个状态的具体行为可配置。通常，全双工通信跳过 CMD、ADDR 和 DUMMY 状态。传输数据的位长可在 \hyperref[fielddesc:SPIMSDATABITLEN]{SPI\_MS\_DATA\_BITLEN} 中配置。通信中使用的实际位长等于 (\hyperref[fielddesc:SPIMSDATABITLEN]{SPI\_MS\_DATA\_BITLEN} + 1)。

\textbf{配置}

按照以下操作步骤，开始数据传输：
\begin{itemize}
    \item 经 IO MUX 或 GPIO 交换矩阵配置 GP-SPI2 与外部 SPI 设备之间的 IO 通道；
    \item 配置 AHB 时钟、APB 时钟（即 AHB\_CLK、APB\_CLK，见章节 \ref{mod:resclk} \textit{\nameref{mod:resclk}})，并为 GP-SPI2 配置模块时钟 (clk\_spi\_mst)；
    \item 置位 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN} 同时清除 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE}，使能主机全双工通信方式；

    \item 配置表 \ref{tab:spi-usr1} 中所列的 GP-SPI2 寄存器；

    \item 配置 SPI CS 建立时间和保持时间，见章节  \ref{sec:gp-spi2-cs-timing}；

    \item 设置 FSPICLK 的极性，见章节  \ref{sec:gp-spi2-clk}；

    \item 根据选定的传输类型准备数据：
    \begin{itemize}
        \item 如果选择的传输类型为 CPU 控制的 MOSI 传输，则需要在 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 中准备数据。

        \item 如果选择了 DMA 控制的传输类型，则需要：
        \begin{itemize}
            \item 配置 \hyperref[fielddesc:SPIDMARXENA]{SPI\_DMA\_RX\_ENA}/\hyperref[fielddesc:SPIDMATXENA]{SPI\_DMA\_TX\_ENA}；
            \item 配置 GDMA TX/RX 链表；
            \item 启动 GDMA TX/RX 引擎，更多描述见章节  \ref{sec:usage-of-gp-spi2-dma-controlled-mode} 和章节  \ref{Subsection:Data Flow Control in GP-SPI2 Master and Slave Mode}。
        \end{itemize}
    \end{itemize}
    \item 配置中断，然后等待 SPI 从机做好传输准备；

    \item 置位 \hyperref[fielddesc:SPIDMAAFIFORST]{SPI\_DMA\_AFIFO\_RST}、\hyperref[fielddesc:SPIBUFAFIFORST]{SPI\_BUF\_AFIFO\_RST} 和 \hyperref[fielddesc:SPIRXAFIFORST]{SPI\_RX\_AFIFO\_RST} 复位 buffer；

    \item 置位寄存器 \hyperref[regdesc:SPICMDREG]{SPI\_CMD\_REG} 中 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 位，开始数据传输，然后等待之前配置的中断。

\end{itemize}
\subsubsection{主机半双工通信（支持 1/2/4-bit 模式）}
\label{sec:gp-spi2-1/2/8-bits}

\textbf{概述}

在半双工通信下，GP-SPI2 发送 CLK 和 CS 信号。在同一时刻，SPI 主机或从机只能有一个可以发送数据，另一个接收数据。用户可通过清除寄存器 \hyperref[regdesc:SPIUSERREG]{SPI\_USER\_REG} 中 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN} 位使能半双工通信。SPI 半双工通信的通用格式为 CMD + [ADDR +] [DUMMY +] [DOUT 或 DIN]。其中，ADDR、DUMMY、DOUT 和 DIN 状态非必选，可单独禁用或启用。

如章节  \ref{sec:gp-spi2-state-and-register-configuration} 所述，CMD、ADDR、DUMMY、DOUT 和 DIN 各个状态的周期、具体值和并行总线位模式等可独立配置。更多寄存器配置信息，见表  \ref{tab:spi-usr1}。

半双工 GP-SPI2 的详细属性如下：

\begin{enumerate}
    \item CMD：0$\sim$16 位，主机发送，从机接收 (MOSI)。
    \item ADDR：0$\sim$32 位，主机发送，从机接收 (MOSI)。
    \item DUMMY：0$\sim$256 个 FSPICLK 周期，主机发送，从机接收。
    \item DOUT：在 CPU 控制的传输中，可传输 0$\sim$512 位（64 字节）数据；在 DMA 控制的单次传输中，可传输 0$\sim$256 Kbit (32 KB)；在 DMA 控制的分段配置传输中，可传输的数据长度无限制。主机发送，从机接收。

    \item DIN：在 CPU 控制的传输中，可传输 0$\sim$512 位（64 字节）数据；在 DMA 控制的单次传输中，可传输 0$\sim$256 Kbit (32 KB)；在 DMA 控制的分段配置传输中，可传输的数据长度无限制。主机接收，从机发送。
\end{enumerate}

\textbf{配置}

具体的寄存器配置如下：
\begin{enumerate}
    \item 经 IO MUX 或 GPIO 交换矩阵配置 GP-SPI2 与外部 SPI 设备之间的 IO 通道；
    \item 配置 AHB 时钟、APB 时钟（即 AHB\_CLK、APB\_CLK），并为 GP-SPI2 配置模块时钟 (clk\_spi\_mst)；

    \item 清除 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN} 和 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE} 位，使能主机半双工通信方式；

    \item 配置表 \ref{tab:spi-usr1} 中所列的 GP-SPI2 寄存器；

    \item 配置 SPI CS 建立时间和保持时间，见章节  \ref{sec:gp-spi2-cs-timing}；

    \item 设置 FSPICLK 的极性，见章节  \ref{sec:gp-spi2-clk}；

    \item 根据选定的传输类型准备数据：
    \begin{itemize}
        \item 如果选择的传输类型为 CPU 控制的 MOSI 传输，则需要在 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 中准备数据。

        \item 如果选择了 DMA 控制的传输类型，则需要：
        \begin{itemize}
            \item 配置 \hyperref[fielddesc:SPIDMARXENA]{SPI\_DMA\_RX\_ENA}/\hyperref[fielddesc:SPIDMATXENA]{SPI\_DMA\_TX\_ENA}；
            \item 配置 GDMA TX/RX 链表；
            \item 启动 GDMA TX/RX 引擎，更多描述见章节  \ref{sec:usage-of-gp-spi2-dma-controlled-mode} 和章节  \ref{Subsection:Data Flow Control in GP-SPI2 Master and Slave Mode}。
        \end{itemize}
    \end{itemize}

    \item 配置中断，然后等待 SPI 从机做好传输准备；

    \item 置位 \hyperref[fielddesc:SPIDMAAFIFORST]{SPI\_DMA\_AFIFO\_RST}、\hyperref[fielddesc:SPIBUFAFIFORST]{SPI\_BUF\_AFIFO\_RST} 和 \hyperref[fielddesc:SPIRXAFIFORST]{SPI\_RX\_AFIFO\_RST} 复位 buffer；

    \item 置位寄存器 \hyperref[regdesc:SPICMDREG]{SPI\_CMD\_REG} 中 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 位，开始数据传输，然后等待之前配置的中断。
\end{enumerate}

\subsubsection{DMA 控制的分段配置传输} \label{sec:gp-spi-dma-seg-conf-trans}

\begin{tiplisting}
注意，由于跳过 CONF 阶段即可实现单次传输，因此不再另起章节单独介绍如何在 GP-SPI2 用作主机时配置单次传输。
\end{tiplisting}

\textbf{概述}

GP-SPI2 用作主机时，可采用 DMA 控制的分段配置传输。

DMA 控制的主机传输可以是：
\begin{itemize}
    \item 一次单次传输，仅包含一次传输事务。
    \item 分段配置传输，包括多个传输事务（即多个分段）。
\end{itemize}

如果选择了分段配置传输，则在每个分段中，寄存器均可单独配置。在分段配置传输下，仅需 CPU 触发一次，即可完成多次传输事务。具体工作流程见图 \ref{fig:spi-master-dma-seg-trans}。



\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=1.0\textwidth]{\modulefiles/figures/GPSPI_Configurable_Segmented_Transfer.eps}}
    \caption{DMA 控制的分段配置传输}
    \label{fig:spi-master-dma-seg-trans}
\end{figure}


如图 \ref{fig:spi-master-dma-seg-trans} 所示，在分段配置传输中的某个单次传输事务 (segment \textcolor{red}{\it{n}}) 开始前，GP-SPI2 可在 CONF 阶段将寄存器重新按照 Conf\_buf\textcolor{red}{\it{n}}  定义的内容进行配置。

建议为每个传输事务的 CONF 阶段提供单独的 GDMA CONF 链表和 CONF buffer（即图 \ref{fig:spi-master-dma-seg-trans} 中的 Conf\_buf\regindex{i}）。GDMA TX 链表将所有的 CONF buffer 和 TX data buffer（即图 \ref{fig:spi-master-dma-seg-trans} 中的 Tx\_buf\regindex{i}）链接起来，因此可以独立控制每个传输事务中的 FSPI 总线行为。

例如，在一次完整的分段配置传输中，传输事务 \textcolor{red}{\it i}、传输事务 \textcolor{red}{\it j} 和传输事务 \textcolor{red}{\it k} 可分别配置为全双工、半双工 MISO 和半双工 MOSI 模式。 \textcolor{red}{\it i}、 \textcolor{red}{\it j} 和  \textcolor{red}{\it k} 均为整数变量，代表传输事务的编号。

同时，每个传输事务中，GP-SPI2 所使用到的各个阶段、各个阶段的相关值和 FSPI 总线周期长、以及 GDMA 行为等，均可独立配置。当整个 DMA 控制的分段配置传输（包括多个传输事务）完成后，即触发 GP-SPI2 中断 \hyperref[int:SPIDMASEGTRANSDONEINT]{SPI\_DMA\_SEG\_TRANS\_DONE\_INT}。

\textbf{配置}

 \begin{enumerate}
    \item 经 IO MUX 或 GPIO 交换矩阵配置 GP-SPI2 与外部 SPI 设备之间的 IO 通道；
    \item 配置 AHB 时钟、APB 时钟 （AHB\_CLK、APB\_CLK），并为 GP-SPI2 配置模块时钟 (clk\_spi\_mst)；

    \item 清除 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN} 和 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE} 位，使能主机半双工通信方式；

    \item 配置表 \ref{tab:spi-usr1} 中所列的 GP-SPI2 寄存器；

    \item 配置 SPI CS 建立时间和保持时间，见章节  \ref{sec:gp-spi2-cs-timing}；

    \item 设置 FSPICLK 的相位和极性，见章节  \ref{sec:gp-spi2-clk}；

   \item 为每个传输事务准备 GDMA CONF buffer 描述符和 TX data 描述符（可选）。把 CONF buffer 描述符和几次传输事务需要的 TX buffer 链接成一个链表；

   \item 同样，为每个传输事务准备 RX buffer 描述符，并链接成一个链表；
   \item 在该 DMA 控制的分段配置传输开始之前，为每个传输事务配置所需的 CONF buffer、TX buffer 和 RX buffer；

   \item 配置 \hyperref[fielddesc:GDMAOUTLINKADDRCHN]{GDMA\_OUTLINK\_ADDR\_CH\textcolor{red}{\it{n}}}  指向 CONF 和 TX buffer 描述符链表的首地址，之后置位 \hyperref[fielddesc:GDMAOUTLINKSTARTCHN]{GDMA\_OUTLINK\_START\_CH\textcolor{red}{\it{n}}}，启动 TX GDMA；

   \item 清除 \hyperref[regdesc:SPIDMACONFREG]{SPI\_DMA\_CONF\_REG} 中 \hyperref[fielddesc:SPIRXEOFEN]{SPI\_RX\_EOF\_EN} 位。配置 \hyperref[fielddesc:GDMAINLINKADDRCHN]{GDMA\_INLINK\_ADDR\_CH\textcolor{red}{\it{n}}}  指向 RX buffer 描述符链表的首地址，之后置位 \hyperref[fielddesc:GDMAINLINKSTARTCHN]{GDMA\_INLINK\_START\_CH\textcolor{red}{\it{n}}}  启动 RX GDMA；

   \item 置位 \hyperref[fielddesc:SPIUSRCONF]{SPI\_USR\_CONF} 使能 CONF 阶段；
   \item 置位 \hyperref[fielddesc:SPIDMASEGTRANSDONEINTENA]{SPI\_DMA\_SEG\_TRANS\_DONE\_INT\_ENA} 使能 \hyperref[int:SPIDMASEGTRANSDONEINT]{SPI\_DMA\_SEG\_TRANS\_DONE\_INT} 中断。如需配置其他中断，请参考章节  \ref{sec:spi-interrupt}；

   \item 等待所有从机做好传输准备；

    \item 置位 \hyperref[fielddesc:SPIDMAAFIFORST]{SPI\_DMA\_AFIFO\_RST}、\hyperref[fielddesc:SPIBUFAFIFORST]{SPI\_BUF\_AFIFO\_RST} 和 \hyperref[fielddesc:SPIRXAFIFORST]{SPI\_RX\_AFIFO\_RST} 复位 buffer；

   \item 置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 开始本次 DMA 控制的分段配置传输；

   \item 等待 \hyperref[int:SPIDMASEGTRANSDONEINT]{SPI\_DMA\_SEG\_TRANS\_DONE\_INT} 中断，即 DMA 分段配置传输结束，数据已存储至相应内存。
 \end{enumerate}

\begin{tiplisting}
\vspace{-2em}
在 PREP 阶段需要准备好本次分段传输发送的数据。应当保证：
\vspace{1em}
    \[
    (\hyperref[fielddesc:SPICSSETUPTIME]{SPI\_CS\_SETUP\_TIME} + 1) \times T_{SPI\_CLK} >= 4 \times (T_{AHB\_CLK} + Tclk\_spi\_mst)
    \]
\end{tiplisting}

\textbf{配置 CONF Buffer 和 Magic 值}\label{sec:SCT-CONF-Buffer}

在 GP-SPI2 分段配置传输中，仅有较上次传输事务有变动的寄存器会在 CONF 阶段被重新配置。为节省时间和芯片资源，其他寄存器配置则保持不变。

GDMA CONF buffer\textcolor{red}{\it i} 中第一个字，即 SPI\_BIT\_MAP\_WORD，记录传输事务 \textcolor{red}{\it i} 中，寄存器是否有改动。SPI\_BIT\_MAP\_WORD 和待更新的 GP-SPI2 寄存器的对应关系见表  \ref{fig:spi-master-BM-table}，即位图 (BM) 表。如果位图表中某一位为 1，则在此次传输事务中，该位对应寄存器的值将被更新。如果其他寄存器不需要修改，则位图表中相应位应置为 0。

        \item 经比较，如果 SPI\_BIT\_MAP\_WORD[31:28] == \hyperref[fielddesc:SPIDMASEGMAGICVALUE]{SPI\_DMA\_SEG\_MAGIC\_VALUE}，则分段配置传输继续正常进行，整个传输过程结束则触发 \hyperref[int:SPIDMASEGTRANSDONEINT]{SPI\_DMA\_SEG\_TRANS\_DONE\_INT} 中断。
        \item 如果 SPI\_BIT\_MAP\_WORD[31:28] != \hyperref[fielddesc:SPIDMASEGMAGICVALUE]{SPI\_DMA\_SEG\_MAGIC\_VALUE}，则 GP-SPI2 状态，即 spi\_st 将返回至 IDLE 状态，分段配置传输立即结束。同时触发 \hyperref[int:SPIDMASEGTRANSDONEINT]{SPI\_DMA\_SEG\_TRANS\_DONE\_INT} 中断， \hyperref[fielddesc:SPISEGMAGICERRINTRAW]{SPI\_SEG\_MAGIC\_ERR\_INT\_RAW} 位也将置 1。
    \end{itemize}

\begin{itemize}
    \item \hyperref[fielddesc:SPIUSRCONF]{SPI\_USR\_CONF}：在置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 之前，需先置位 \hyperref[fielddesc:SPIUSRCONF]{SPI\_USR\_CONF}，以使能本次传输。
    \item \hyperref[fielddesc:SPIUSRCONFNXT]{SPI\_USR\_CONF\_NXT}：如果传输事务 \regindex{i} 不是本次 DMA 控制的分段配置传输中的最后一次传输事务，则需要置位 \hyperref[fielddesc:SPIUSRCONFNXT]{SPI\_USR\_CONF\_NXT}。
    \item \hyperref[fielddesc:SPICONFBITLEN]{SPI\_CONF\_BITLEN}：此外，在每个单独的传输事务中，GP-SPI2 的 CS 建立时间和保持时间可独立编程，更多配置信息见章节  \ref{sec:gp-spi2-cs-timing}。在每次传输事务中，CS 保持高电平的时长约为：
    \[
    (\hyperref[fielddesc:SPICONFBITLEN]{SPI\_CONF\_BITLEN} + 5) \times T_{AHB\_CLK}
    \]

如果 \hyperref[fielddesc:SPICONFBITLEN]{SPI\_CONF\_BITLEN} 大于 0x3FFFA，(\hyperref[fielddesc:SPICONFBITLEN]{SPI\_CONF\_BITLEN} + 5) 将溢出 (0x40000 - \hyperref[fielddesc:SPICONFBITLEN]{SPI\_CONF\_BITLEN} - 5)。
\end{itemize}

\subsection{GP-SPI2 用作从机}\label{sec:gp-spi2-slave-mode}

GP-SPI2 可用作从机与另一 SPI 主机进行通信。用作从机时，GP-SPI2 支持特定格式的 1-bit SPI、2-bit Dual SPI、4-bit Quad SPI 和 QPI 模式。用户可置位寄存器 \hyperref[regdesc:SPISLAVEREG]{SPI\_SLAVE\_REG} 中 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE} 位使能 GP-SPI2 从机。

在传输过程中，CS 信号应保持低电平，CS 信号的下降沿和上升沿代表一次单次传输或从机连续传输的开始和结束。以 CS 低电平有效为例，GP-SPI2 用作从机时，每次 SPI 传输之间的 CS 高电平无效持续时间应当不小于 8
$T_{\textrm{AHB\_CLK}}$，保证每次传输都能正常结束。

\subsubsection{可配置的通信格式}\label{slave communication format}

GP-SPI2 用作从机时支持全双工通信和半双工通信。用户可配置寄存器 \hyperref[regdesc:SPIUSERREG]{SPI\_USER\_REG} 中 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN} 位选择需要的通信方式。

全双工通信下，传输一开始，则数据同时输入和输出。此时，所有数据位均被视为输入/输出数据，即不需要命令、地址或 DUMMY 阶段。传输结束即触发 \hyperref[int:SPITRANSDONEINT]{SPI\_TRANS\_DONE\_INT} 中断。

在半双工通信下，通信格式为 CMD+ADDR+DUMMY+DATA（DIN 或 DOUT）。
\begin{itemize}
    \item ``DIN" 表示 SPI 主机从 GP-SPI2 中读取数据；
    \item ``DOUT" 表示 SPI 主机向 GP-SPI2 中写入数据。
\end{itemize}

每个阶段的详细特性如下：

\begin{enumerate}
   \item CMD：
        \begin{itemize}
            \item 表明 SPI 从机用于何种功能；
            \item 一个字节，主机输出，从机输入；
            \item 仅支持表 \ref{tab:spi-slave-cmd-spi-new} 和表 \ref{tab:spi-slave-cmd-qpi} 所列的命令值；
            \item 以 1-bit SPI 模式或 4-bit QPI 模式发送。
        \end{itemize}

   \item ADDR：
        \begin{itemize}
            \item 在 CPU 控制的传输中，可以为  \hyperlink{cmd:wrbuf}{Wr\_BUF} 和  \hyperlink{cmd:rdbuf}{Rd\_BUF} 命令提供地址，或在其他命令中用作占位符，具体由应用定义；
            \item 一个字节，主机输出，从机输入；
            \item 可根据命令，以 1-bit、2-bit 或 4-bit 模式发送；
        \end{itemize}

   \item DUMMY：
        \begin{itemize}
            \item DUMMY 的值无实际意义；SPI 从机在这个阶段准备数据；
            \item FSPI 总线的位模式在这里也没有实际意义；
            \item 持续八个 SPI\_CLK 时钟周期。
        \end{itemize}

   \item DIN 或 DOUT：

        \begin{itemize}
            \item 在 CPU 控制的传输下，可传输 0$\sim$64 字节数据；在 DMA 控制的传输下，传输数据长度无限制。
            \item 可根据具体的 CMD 值，以 1-bit、2-bit 或 4-bit 模式发送。
        \end{itemize}

   \end{enumerate}

\begin{tiplisting}
半双工通信下，ADDR 和 DUMMY 阶段不可跳过。
\end{tiplisting}
\vspace{-1em}
半双工传输结束后，传输的 CMD 和 ADDR 的值分别锁存至 \hyperref[fielddesc:SPISLVLASTCOMMAND]{SPI\_SLV\_LAST\_COMMAND} 和 \hyperref[fielddesc:SPISLVLASTADDR]{SPI\_SLV\_LAST\_ADDR}。如果 GP-SPI2 不支持传输的 CMD 值，\hyperref[fielddesc:SPISLVCMDERRINTRAW]{SPI\_SLV\_CMD\_ERR\_INT\_RAW} 将被置位。 \hyperref[fielddesc:SPISLVCMDERRINTRAW]{SPI\_SLV\_CMD\_ERR\_INT\_RAW} 仅可由软件清零。



\subsubsection{半双工通信支持的 CMD 值} \label{GP-SPI2 HD salve format}

在半双工传输中，CMD 定义的值将决定传输类型。不支持的 CMD 值及其相关数据传输均被忽略，且 \hyperref[fielddesc:SPISLVCMDERRINTRAW]{SPI\_SLV\_CMD\_ERR\_INT\_RAW} 将被置 1。传输格式为：CMD（8 位）+ ADDR（8 位）+ DUMMY（8 个 SPI\_CLK 周期）+ DATA，其中，DATA 的单位为字节。CMD[3:0] 的详细说明如下：


\begin{itemize}
    \item \label{cmd:wrbuf} 0x1 (Wr\_BUF)：CPU 控制的写操作。主机发送数据，GP-SPI2 接收数据。数据将存储至相应地址的寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}。

    \item \label{cmd:rdbuf} 0x2 (Rd\_BUF)：CPU 控制的读操作。主机接收 GP-SPI2 发送的数据。数据来自相应地址的寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG}。

    \item \label{cmd:wrdma} 0x3 (Wr\_DMA)：DMA 控制的写操作。主机发送数据，GP-SPI2 接收数据。数据将存储至 GP-SPI2 的 GDMA RX buffer 中。

    \item \label{cmd:rddma} 0x4 (Rd\_DMA)：DMA 控制的读操作。主机接收 GP-SPI2 发送的数据。数据来自 GP-SPI2 的 GDMA TX buffer。

    \item \label{cmd:CMD7} 0x7 (CMD7)：用于生成 \hyperref[int:SPISLVCMD7INT]{SPI\_SLV\_CMD7\_INT} 中断。在从机连续传输下，使用 DMA RX 链表时，也可用于生成 GDMA\_IN\_SUC\_EOF\_CH\textcolor{red}{\it{n}}\_INT  中断。但不会结束 GP-SPI2 的从机连续传输。

    \item 0x8 (CMD8)：仅用于生成 \hyperref[int:SPISLVCMD8INT]{SPI\_SLV\_CMD8\_INT} 中断，但不会结束 GP-SPI2 的从机连续传输。

    \item 0x9 (CMD9)：仅用于生成 \hyperref[int:SPISLVCMD9INT]{SPI\_SLV\_CMD9\_INT} 中断，但不会结束 GP-SPI2 的从机连续传输。

    \item 0xA (CMDA)：仅用于生成 \hyperref[int:SPISLVCMDAINT]{SPI\_SLV\_CMDA\_INT} 中断，但不会结束 GP-SPI2 的从机连续传输。

\end{itemize}

CMD7、CMD8、CMD9 和 CMDA 的具体用途可由用户自定义。这些命令可用作握手信号、某些特定功能的密码、或某些用户自定义操作的触发信号等。

CMD、ADDR 和 DATA 阶段均支持 1/2/4-bit 模式，具体由 CMD[7:4] 决定。DUMMY 仅支持 1-bit 模式，且持续八个 SPI\_CLK 时钟周期。CMD[7:4] 的具体定义如下：

\begin{itemize}
    \item 0x0：CMD、ADDR 和 DATA 阶段均为 1-bit 模式。
    \item 0x1：CMD 和 ADDR 均为 1-bit 模式。DATA 为 2-bit 模式。
    \item 0x2：CMD 和 ADDR 均为 1-bit 模式。DATA 为 4-bit 模式。
    \item 0x5：CMD 为 1-bit 模式。ADDR 和 DATA 均为 2-bit 模式。
    \item 0xA：CMD 为 1-bit 模式。ADDR 和 DATA 均为 4-bit 模式，或 QPI 模式。
\end{itemize}

此外，CMD[7:0] 的值为 0x05、0xA5、0x06 和 0xDD 时，将跳过 DUMMY 和 DATA 阶段。CMD[7:0] 的具体定义如下：

\begin{itemize}
    \item \label{cmd:ENDSEGTRANS}0x05 (End\_SEG\_TRANS)：主机发送 0x05 命令，结束 SPI 模式下从机连续传输。
    \item 0xA5 (End\_SEG\_TRANS)：主机发送 0xA5 命令，结束 QPI 模式下从机连续传输。
    \item 0x06 (En\_QPI)：GP-SPI2 接收到 0x06 命令后，进入 QPI 模式。此时，寄存器 \hyperref[regdesc:SPIUSERREG]{SPI\_USER\_REG} 中 \hyperref[fielddesc:SPIQPIMODE]{SPI\_QPI\_MODE} 置位。
    \item 0xDD (Ex\_QPI)：GP-SPI2 接收到 0xDD 命令后，退出 QPI 模式。此时， \hyperref[fielddesc:SPIQPIMODE]{SPI\_QPI\_MODE} 位清零。
\end{itemize}

\subsubsection{从机单次传输和从机连读传输}\label{sec:gp-spi2-slave-seg-trans}

GP-SPI2 用作从机时，支持由 DMA 和 CPU 控制的全双工和半双工通信。DMA 控制的从机传输，可以是一次单次传输，也可以是从机连续传输（包含多次传输事务）。CPU 控制的传输只能是单次传输，因为每次传输均需由 CPU 触发。

一次从机连续传输包含多个传输事务，每个传输事务可以是表 \ref{tab:spi-slave-cmd-spi-new} 和表 \ref{tab:spi-slave-cmd-qpi} 列出的任一传输类型。即在一次完整的连续传输过程中，可以包含 CPU 控制的数据传输，也可以包含 DMA 控制的数据传输。

在一次完整的连续传输过程中，推荐操作如下：

\begin{itemize}
    \item CPU 控制的数据传输可用于握手通信以及少量数据传输；

    \item DMA 控制的数据传输可用于大量数据传输。
\end{itemize}


\subsubsection{配置从机单次传输}\label{GP-SPI2 slave single transfer}

用作从机时，GP-SPI2 支持 CPU 控制的和 DMA 控制的全/半双工单次传输。具体的寄存器配置如下：
\begin{enumerate}
    \item 经 IO MUX 或 GPIO 交换矩阵配置 GP-SPI2 与外部 SPI 设备之间的 IO 通道；

    \item 配置 AHB 时钟、APB 时钟（AHB\_CLK、APB\_CLK）；

    \item 置位 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE} 使能从机传输；

    \item 配置 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN}：
    \begin{itemize}
        \item 1：使能全双工通信；
        \item 0：使能半双工通信。
    \end{itemize}

    \item 准备数据：
    \begin{itemize}
        \item 如果选择的传输类型为 CPU 控制的传输，且 GP-SPI2 发送数据，则在寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 中准备数据。
        \item 如果选择的传输类型为 DMA 控制的传输，则需要：
        \begin{itemize}
            \item 配置 \hyperref[fielddesc:SPIDMARXENA]{SPI\_DMA\_RX\_ENA}/\hyperref[fielddesc:SPIDMATXENA]{SPI\_DMA\_TX\_ENA} 和 \hyperref[fielddesc:SPIRXEOFEN]{SPI\_RX\_EOF\_EN}；
            \item 配置 GDMA TX/RX 链表；
            \item 启动 GDMA TX/RX 引擎，更多描述见章节  \ref{sec:usage-of-gp-spi2-dma-controlled-mode} 和章节  \ref{Subsection:Data Flow Control in GP-SPI2 Master and Slave Mode}。
        \end{itemize}
    \end{itemize}

    \item 置位 \hyperref[fielddesc:SPIDMAAFIFORST]{SPI\_DMA\_AFIFO\_RST}、\hyperref[fielddesc:SPIBUFAFIFORST]{SPI\_BUF\_AFIFO\_RST} 和 \hyperref[fielddesc:SPIRXAFIFORST]{SPI\_RX\_AFIFO\_RST} 复位 buffer；

    \item 清零寄存器 \hyperref[regdesc:SPIDMACONFREG]{SPI\_DMA\_CONF\_REG} 中 \hyperref[fielddesc:SPIDMASLVSEGTRANSEN]{SPI\_DMA\_SLV\_SEG\_TRANS\_EN} 使能从机单次传输；

    \item 置位寄存器 \hyperref[regdesc:SPIDMAINTENAREG]{SPI\_DMA\_INT\_ENA\_REG} 中 \hyperref[fielddesc:SPITRANSDONEINTENA]{SPI\_TRANS\_DONE\_INT\_ENA}，使能中断，并等待 \hyperref[int:SPITRANSDONEINT]{SPI\_TRANS\_DONE\_INT} 中断。在 DMA 控制的传输下，使用 DMA RX buffer 时，推荐等待 GDMA\_IN\_SUC\_EOF\_CH\textcolor{red}{\it{n}}\_INT 中断，即数据已存储至相应内存。其他中断见章节 \ref{sec:spi-interrupt}。
\end{enumerate}


\subsubsection{配置半双工通信下从机连续传输}\label{sec:gp-spi2-slave-hd-seg-trans}

此时必须使用 GDMA。具体的寄存器配置如下：

\begin{enumerate}
   \item 经 IO MUX 或 GPIO 交换矩阵配置 GP-SPI2 与外部 SPI 设备之间的 IO 通道；

   \item 配置 AHB 时钟、APB 时钟（AHB\_CLK、APB\_CLK）；

    \item 置位 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE} 使能从机传输；

    \item 清除 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN} 使能半双工通信方式；

    \item 根据需求，确定是否需要在寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}$\sim$\hyperref[regdesc:SPIW15REG]{SPI\_W15\_REG} 中准备数据；

    \item 置位 \hyperref[fielddesc:SPIDMAAFIFORST]{SPI\_DMA\_AFIFO\_RST}、\hyperref[fielddesc:SPIBUFAFIFORST]{SPI\_BUF\_AFIFO\_RST} 和 \hyperref[fielddesc:SPIRXAFIFORST]{SPI\_RX\_AFIFO\_RST} 复位 buffer；

    \item 置位 \hyperref[fielddesc:SPIDMARXENA]{SPI\_DMA\_RX\_ENA} 和 \hyperref[fielddesc:SPIDMATXENA]{SPI\_DMA\_TX\_ENA}。清零 \hyperref[fielddesc:SPIRXEOFEN]{SPI\_RX\_EOF\_EN}。配置 GDMA TX/RX 链表，并启动 GDMA TX/RX 引擎，更多描述见章节 \ref{sec:usage-of-gp-spi2-dma-controlled-mode} 和章节 \ref{Subsection:Data Flow Control in GP-SPI2 Master and Slave Mode}；

    \item 置位寄存器 \hyperref[regdesc:SPIDMACONFREG]{SPI\_DMA\_CONF\_REG} 中 \hyperref[fielddesc:SPIDMASLVSEGTRANSEN]{SPI\_DMA\_SLV\_SEG\_TRANS\_EN}，使能从机连续传输；

    \item 置位寄存器 \hyperref[regdesc:SPIDMAINTENAREG]{SPI\_DMA\_INT\_ENA\_REG} 中 \hyperref[fielddesc:SPIDMASEGTRANSDONEINTENA]{SPI\_DMA\_SEG\_TRANS\_DONE\_INT\_ENA}，使能中断，并等待 \hyperref[int:SPIDMASEGTRANSDONEINT]{SPI\_DMA\_SEG\_TRANS\_DONE\_INT} 中断。中断发生，即表明从机连续传输已结束，且数据已放入相应的内存中。其他中断见章节 \ref{sec:spi-interrupt}。
\end{enumerate}

GP-SPI2 收到 End\_SEG\_TRANS 命令（SPI 模式下为 0x05，QPI 模式下为 0xA5），从机连续传输结束，并触发 \hyperref[int:SPIDMASEGTRANSDONEINT]{SPI\_DMA\_SEG\_TRANS\_DONE\_INT} 中断。

\subsubsection{配置全双工通信下从机连续传输}\label{sec:gp-spi2-slave-fd-seg-trans}
在这一传输中，必须使用 GDMA。数据从 GDMA buffer 中输入输出。传输结束，触发 GDMA\_IN\_SUC\_EOF\_CH\textcolor{red}{\it{n}}\_INT  中断。具体的配置程序如下：

\begin{enumerate}
   \item 经 IO MUX 或 GPIO 交换矩阵配置 GP-SPI2 与外部 SPI 设备之间的 IO 通道；
   \item 配置 AHB 时钟、APB 时钟（AHB\_CLK、APB\_CLK）；

    \item 置位 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE} 和 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN}，使能从机全双工通信；

    \item 置位 \hyperref[fielddesc:SPIDMAAFIFORST]{SPI\_DMA\_AFIFO\_RST}、\hyperref[fielddesc:SPIBUFAFIFORST]{SPI\_BUF\_AFIFO\_RST} 和 \hyperref[fielddesc:SPIRXAFIFORST]{SPI\_RX\_AFIFO\_RST} 复位 buffer；

    \item 置位 \hyperref[fielddesc:SPIDMARXENA]{SPI\_DMA\_RX\_ENA} 和 \hyperref[fielddesc:SPIDMATXENA]{SPI\_DMA\_TX\_ENA}。配置 GDMA TX/RX 链表，并启动 GDMA TX/RX 引擎，更多描述见章节 \ref{sec:usage-of-gp-spi2-dma-controlled-mode} 和章节 \ref{Subsection:Data Flow Control in GP-SPI2 Master and Slave Mode}；

    \item 置位寄存器 \hyperref[regdesc:SPIDMACONFREG]{SPI\_DMA\_CONF\_REG} 中 \hyperref[fielddesc:SPIRXEOFEN]{SPI\_RX\_EOF\_EN}。在寄存器 \hyperref[regdesc:SPIMSDLENREG]{SPI\_MS\_DLEN\_REG} 的 \hyperref[fielddesc:SPIMSDATABITLEN]{SPI\_MS\_DATA\_BITLEN}[17:0] 中配置 DMA 接收数据长度（单位：字节）；

    \item 置位寄存器 \hyperref[regdesc:SPIDMACONFREG]{SPI\_DMA\_CONF\_REG} 中 \hyperref[fielddesc:SPIDMASLVSEGTRANSEN]{SPI\_DMA\_SLV\_SEG\_TRANS\_EN}，使能从机连续传输；

    \item 置位 \hyperref[fielddesc:GDMAINSUCEOFCHNINTENA]{GDMA\_IN\_SUC\_EOF\_CH\textcolor{red}{\it{n}}\_INT\_ENA}  使能中断，然后等待 \hyperref[int:GDMAINSUCEOFINT]{GDMA\_IN\_SUC\_EOF\_CH\textcolor{red}{\it{n}}\_INT} 中断。
\end{enumerate}


\section{CS 建立时间和保持时间控制}\label{sec:gp-spi2-cs-timing}

SPI CS 建立时间和保持时间对于满足各种 SPI 设备（如 flash 或 PSRAM）的时序要求非常重要。

CS 建立时间为 CS 下降沿至 SPI\_CLK 第一个锁存边沿的时间。模式 0 和模式 3 的第一锁存边沿为上升沿，模式 2 和模式 4 的第一锁存边沿为下降沿。

CS 保持时间为 SPI\_CLK 最后一个锁存边沿到 CS 上升沿之间的时间。

用作从机时，CS 建立时间和保持时间应大于 0.5 x T\_SPI\_CLK，否则 SPI 传输可能出错。这里的 T\_SPI\_CLK 指 SPI\_CLK 时钟周期。

用作主机时，CS 建立时间由寄存器 \hyperref[regdesc:SPIUSERREG]{SPI\_USER\_REG} 中的 \hyperref[fielddesc:SPICSSETUP]{SPI\_CS\_SETUP} 位和寄存器 \hyperref[regdesc:SPIUSER1REG]{SPI\_USER1\_REG} 中的 \hyperref[fielddesc:SPICSSETUPTIME]{SPI\_CS\_SETUP\_TIME} 位控制：
    \begin{itemize}
        \item 清零 \hyperref[fielddesc:SPICSSETUP]{SPI\_CS\_SETUP}，则 SPI CS 建立时间为 0.5 x T\_SPI\_CLK；
        \item 置位 \hyperref[fielddesc:SPICSSETUP]{SPI\_CS\_SETUP}，则 SPI CS 建立时间为(\hyperref[fielddesc:SPICSSETUPTIME]{SPI\_CS\_SETUP\_TIME} + 1.5) x T\_SPI\_CLK。
    \end{itemize}

CS 保持时间由寄存器 \hyperref[regdesc:SPIUSERREG]{SPI\_USER\_REG} 中的 \hyperref[fielddesc:SPICSHOLD]{SPI\_CS\_HOLD} 位和寄存器 \hyperref[regdesc:SPIUSER1REG]{SPI\_USER1\_REG} 中的 \hyperref[fielddesc:SPICSHOLDTIME]{SPI\_CS\_HOLD\_TIME} 位控制：
    \begin{itemize}
        \item 清零 \hyperref[fielddesc:SPICSHOLD]{SPI\_CS\_HOLD}，则 SPI CS 保持时间为 0.5 x T\_SPI\_CLK；
        \item 置位 \hyperref[fielddesc:SPICSHOLD]{SPI\_CS\_HOLD}，则 SPI CS 保持时间为 (\hyperref[fielddesc:SPICSHOLDTIME]{SPI\_CS\_HOLD\_TIME} + 1.5) x T\_SPI\_CLK。
    \end{itemize}

\section{GP-SPI2 时钟控制}\label{sec:gp-spi2-clk}
GP-SPI2 中有以下四个时钟：
\begin{itemize}
    \item clk\_hclk：GP-SPI2 模块的时序补偿时钟，与 clk\_spi\_mst 同源，一般是 clk\_spi\_mst 的 2 倍频；
    \item clk\_spi\_mst：GP-SPI2 模块时钟，在 GP-SPI2 用作主机时用于生成数据传输以及从机所需的 SPI\_CLK 信号，时钟频率最高不超过 80MHz；
    \item SPI\_CLK：主机输出时钟；
    \item AHB\_CLK：用于寄存器配置的时钟。
\end{itemize}

其中，clk\_spi\_mst 与 clk\_hclk 时钟的开关寄存器为 PCR\_SPI2\_CLK\_EN，时钟源选择寄存器为 PCR\_SPI2\_CLKMST\_SEL，clk\_hclk 时钟频率与如下源时钟保持相同，clk\_spi\_mst 可通过 PCR\_SPI2\_CLKM\_DIV\_NUM 设置：
\begin{itemize}
    \item 0：时钟源为 XTAL\_CLK；
    \item 1：时钟源为 PLL\_F160M\_CLK；
    \item 2：时钟源为 FOSC\_CLK。
\end{itemize}


用作主机时，GP-SPI2 最高输出时钟频率为 $f_{\textrm{clk\_spi\_mst}}$。如果需要较低的时钟频率，可以采用如下分频方式：
    \[
    f_{\textrm{SPI\_CLK}}=\frac{f_{\textrm{clk\_spi\_mst}}} {\textrm{(\hyperref[fielddesc:SPICLKCNTN]{SPI\_CLKCNT\_N} + 1)(\hyperref[fielddesc:SPICLKDIVPRE]{SPI\_CLKDIV\_PRE} + 1)}}
    \] \label{division-formula}

用户可配置寄存器 \hyperref[regdesc:SPICLOCKREG]{SPI\_CLOCK\_REG} 中 \hyperref[fielddesc:SPICLKCNTN]{SPI\_CLKCNT\_N} 和 \hyperref[fielddesc:SPICLKDIVPRE]{SPI\_CLKDIV\_PRE} 设置分频系数。寄存器 \hyperref[regdesc:SPICLOCKREG]{SPI\_CLOCK\_REG} 中 \hyperref[fielddesc:SPICLKEQUSYSCLK]{SPI\_CLK\_EQU\_SYSCLK} 位置 1 时，GP-SPI 的输出时钟频率为 $f_{\textrm{clk\_spi\_mst}}$。如果采用其他整数分频，则 \hyperref[fielddesc:SPICLKEQUSYSCLK]{SPI\_CLK\_EQU\_SYSCLK} 应置 0。

用作从机时，GP-SPI2 支持的输入时钟频率为 $f_{\textrm{SPI\_CLK}}$ <= $f_{\textrm{AHB\_CLK}}$。

\section{GP-SPI2 时序补偿} \label{sec:gp-spi-timing-compensation}

\textbf{概述}
根据 GP-SPI2 在用户 SPI 传输系统中的角色（主机或从机），提供如下时序补偿方案：
\begin{itemize}
    \item 在主机模式下，默认采用延后半拍 SPI\_CLK 的时序补偿方案，如图 \ref{Figure:SPI_CLK0_2} 和图 \ref{Figure:SPI_CLK1_3} 所示；也可通过配置寄存器切换为标准采样模式。
    \item 若使用串联寄存器补偿方案，则在默认延后半拍的基础上，进一步增强高速传输下的时序补偿能力。
    \item 在从机模式下，默认按标准 SPI 协议发送数据，可以通过寄存器设置提前半拍发送数据。
\end{itemize}

GP-SPI2 用作主机时，默认延后半拍 SPI\_CLK 采样从机返回的数据，可以有效补偿 GP-SPI2 IO 时序，也可置位 \hyperref[fielddesc:SPICLKEDGESEL]{SPI\_CLK\_EDGE\_SEL} 切换为符合 SPI 标准协议的采样模式。请注意，\hyperref[fielddesc:SPICLKEQUSYSCLK]{SPI\_CLK\_EQU\_SYSCLK} 位置 1 时，固定延后半拍 SPI\_CLK 采样从机返回的数据，此时可采用其他方法进行 IO 时序补偿。

SPI 输入输出信号可通过 GPIO 矩阵或 IO MUX 映射到芯片管脚，但 IO MUX 不支持时序调整。输入输出数据在 GPIO 矩阵模块中，可在上升沿或下降沿延迟 1 或 2 个 IO MUX 运行时钟周期。更多寄存器配置信息，见章节 \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}}。

图 \ref{fig:spi-master-timing-diagram} 所示为 GP-SPI2 用作主机时的时序补偿控制，包括以下路径：
\begin{itemize}
    \item ``CLK"：GP-SPI2 总线时钟信号的输出路径。时钟由 SPI\_CLK 输出控制模块发送，经过 GPIO 矩阵或 IO MUX，然后到达外部 SPI 设备。
    \item ``IN"：GP-SPI2 的数据输入路径，即图 \ref{fig:spi-master-timing-diagram} 中路径 3（紫色）。来自外部 SPI 设备的输入数据通过 GPIO 矩阵或 IO MUX，由时序模块（见图 \ref{fig:gp-spi-block-diagram}）进行调整，最后存储到 \textit{spi\_rx\_afifo}。
    \item ``OUT"：GP-SPI2 的数据输出路径，即图 \ref{fig:spi-master-timing-diagram} 中路径 2（玫红色）。输出数据发送到时序模块，经过 GPIO 矩阵或 IO MUX，最后由外部 SPI 设备捕获。
\end{itemize}


\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/GPSPI_Timing_Block_20221011.eps}}
    \caption{GP-SPI2 主机时序补偿控制图}

时序调制模块同时适用于输入数据和输出数据，数据可在时钟上升沿或下降沿延迟整数个 clk\_spi\_mst 周期，即整数个 $T_{\textrm{clk\_spi\_mst}}$。

\textbf{关键寄存器}

\begin{itemize}
    \item \hyperref[regdesc:SPIDINMODEREG]{SPI\_DIN\_MODE\_REG}：用于选择输入数据的锁存沿；
    \item \hyperref[regdesc:SPIDINNUMREG]{SPI\_DIN\_NUM\_REG}：用于选择输入数据的延迟周期；
    \item \hyperref[regdesc:SPIDOUTMODEREG]{SPI\_DOUT\_MODE\_REG}：用于选择输出数据的锁存沿。
\end{itemize}

    \caption{GP-SPI2 主机时序补偿示例}
    \label{fig:spi-master-timing-seq}
\end{figure}

图  \ref{fig:spi-master-timing-seq} 中，``p1" 为时序模块输入数据的时间点；``p2" 为输出数据的时间点。由于 FSPIQ 输入数据并未与 FSPID 输入数据对齐，如果没有时间补偿的话，GP-SPI2 读取数据将会出错。

为了正确读取数据，需要进行如下配置。其中，假设 $f_{clk\_spi\_mst}$ 等于 $f_{SPI\_CLK}$：

\begin{itemize}
    \item 将 FSPID 在 clk\_spi\_mst 下降沿延迟两个时钟周期
    \item 将 FSPIQ 在 clk\_spi\_mst 下降沿延迟一个时钟周期
    \item 增加一个额外的 DUMMY 周期
\end{itemize}


GP-SPI2 用作从机时，如果寄存器 \hyperref[regdesc:SPISLAVEREG]{SPI\_SLAVE\_REG} 中 \hyperref[fielddesc:SPIRSCKDATAOUT]{SPI\_RSCK\_DATA\_OUT} 置 1，则在锁存沿发送输出数据，即提前半个 SPI 时钟周期。上述功能可用于从机时序补偿。
