\begin{register}{H}{mnxti}{0x345}\label{regdesc:MNXTI_REG}
\regfield{MNXTI}{32}{0}{{0x0}}%
\reglabel{Reset}\regnewline%
\begin{regdesc}\begin{reglist}
\label{fielddesc:MNXTI}\item [MNXTI]
表示下一个机器模式中断入口地址，并配置中断使能状态。\\
软件可以使用该 CSR 处理下一个特权模式相同的水平中断，但该中断级别必须高于保存在 \hyperref[fielddesc:MCAUSE_MPIL]{{mcause.MPIL}} 中的中断上下文，也须高于相应特权模式的中断阈值，如此便减少了刷新中断流水线和保存/恢复上下文的开销。\\
此 CSR 使用 CSRRSI/CSRRCI 指令访问，读取的值是陷阱处理程序表中表项的指针，写回的是中断使能状态。此外，写入 mnxti会产生副作用：更新中断上下文状态。\\
使用 CSRR 读取 mnxti CSR 时，返回零表示没有合适的中断可处理或系统不在 CLIC 模式；返回陷阱处理程序表中表项的非零地址，用于软件陷阱向量处理。\\
如果访问 mnxti 的 CSR 指令包含写操作，则用于读-改-写操作的 CSR是 \hyperref[regdesc:MSTATUS]{mstatus} CSR，而 \hyperref[regdesc:MCAUSE]{mcause} 寄存器的 exccode 域和 \hyperref[fielddesc:MINTSTATUS_MIL]{mintstatus.MIL} 域也可以更新为新的中断 ID 和级别。如果中断是边沿触发，则挂起位也会被清零。\\
当初始中断被接受 (taken) 后，\hyperref[regdesc:MCAUSE]{mcause} 和 \hyperref[regdesc:MEPC]{mepc} 寄存器会更新为中断上下文和中断 ID，此时，中断处理程序会使用 mnxti CSR。\\
如果挂起的中断是边沿触发的，当访问 mnxti 的 CSR 指令包含写操作时，硬件将自动清除相应的挂起位。然而，如果 CSR 指令不包含写操作，则不会对任何 CSR 进行状态更新，因此中断挂起位不会被清零。这种行为允许软件使用 mnxti 优化中断的选择和执行。\\
(R/W)
\end{reglist}\end{regdesc}
\end{register}

\begin{register}{H}{unxti}{0x045}\label{regdesc:UNXTI_REG}

\begin{regdesc}\begin{reglist}
\label{fielddesc:UNXTI}\item [UNXTI]
表示下一个用户模式中断入口地址，并配置中断使能状态。\\
软件可以使用该 CSR 处理下一个特权模式相同的水平中断，但该中断级别必须高于保存在 \hyperref[fielddesc:UCAUSE_UPIL]{{ucause.UPIL}} 中的中断上下文，也须高于相应特权模式的中断阈值，如此便减少了刷新中断流水线和保存/恢复上下文的开销。\\
此 CSR 使用 CSRRSI/CSRRCI 指令访问，读取的值是陷阱处理程序表中表项的指针，写回的是中断使能状态。此外，写入 unxti 会产生副作用：更新中断上下文状态。\\
使用 CSRR 读取 unxti CSR 时，返回零表示没有合适的中断可处理或系统不在 CLIC 模式；返回陷阱处理程序表中表项的非零地址，用于软件陷阱向量处理。\\

如果挂起的中断是边沿触发的，当访问 unxti 的 CSR 指令包含写操作时，硬件将自动清除相应的挂起位。然而，如果 CSR 指令不包含写操作，则不会对任何 CSR 进行状态更新，因此中断挂起位不会被清零。这种行为允许软件使用 unxti 优化中断的选择和执行。\\

\begin{register}{H}{mexstatus}{0x7E1}\label{regdesc:MEXSTATUS}%

\regfield{PBEXE}{1}{10}{{1}}%

\regfield{EXPT\_VLD}{1}{5}{{0}}%
\regfield{(reserved)}{5}{0}{{0x00}}%
\reglabel{Reset}\regnewline%
\begin{regdesc}\begin{reglist}
\label{fielddesc:MEXSTATUS_EXPT_VLD}\item [EXPT\_VLD] 进入陷阱处理程序时，自动设置此位。\\
此位仅可从调试模式写入。\\
(RO)
\label{fielddesc:MEXSTATUS_LOCKUP}\item [LOCKUP] 当两个异常发生时，自动设置此位。\\
一旦锁定，核心将暂停运行，并等待调试器处理锁定。\\
此位仅可从调试模式写入。\\
(RO)
\label{fielddesc:MEXSTATUS_BUS_ERR}\item [BUS\_ERR] 由于总线错误异常进入加载/存储错误陷阱时，自动设置此位。\\
一旦进入陷阱处理程序，必须读取此位，确定加载/存储错误是否由总线错误造成。\\
若 \hyperref[fielddesc:MEXSTATUS_NMFT]{mexstatus.NMFT} 已启用，执行 MRET 指令时，此位立即自动清除。\\
\hyperref[fielddesc:MEXSTATUS_NMFT]{mexstatus.NMFT} 禁用且 \hyperref[fielddesc:MEXSTATUS_PBEXE]{mexstatus.PBEXE} 启用时，必须在恢复执行前手动清除此位，否则可能会重新触发挂起的总线错误异常。\\
(R/W)
\label{fielddesc:MEXSTATUS_PBEXE}\item [PBEXE] 配置是否启用挂起的总线错误异常。\\
0：禁用\\
1：启用\\
当 \hyperref[fielddesc:MEXSTATUS_PBEXE]{mexstatus.PBEXE} 启用且 \hyperref[fielddesc:MEXSTATUS_NMFT]{mexstatus.NMFT} 禁用时，设置 \hyperref[fielddesc:MEXSTATUS_BUS_ERR]{mexstatus.BUS\_ERR} 会生成同步异常。\\
如果 \hyperref[fielddesc:MEXSTATUS_NMFT]{mexstatus.NMFT} 启用，则 \hyperref[fielddesc:MEXSTATUS_PBEXE]{mexstatus.PBEXE} 无效。\\
进入陷阱时，\hyperref[fielddesc:MEXSTATUS_PBEXE]{mexstatus.PBEXE} 的值被写入 \hyperref[fielddesc:MEXSTATUS_PPBEXE]{mexstatus.PPBEXE}，并且 \hyperref[fielddesc:MEXSTATUS_PBEXE]{mexstatus.PBEXE} 自动设置为 0。\\
通过 MRET 退出陷阱时，\hyperref[fielddesc:MEXSTATUS_PPBEXE]{mexstatus.PPBEXE} 的值被写回 \hyperref[fielddesc:MEXSTATUS_PBEXE]{mexstatus.PBEXE}。\\
(R/W)
\label{fielddesc:MEXSTATUS_PPBEXE}\item [PPBEXE] 配置是否启用之前挂起的总线错误异常。\\
0：禁用\\
1：启用\\
进入陷阱时，\hyperref[fielddesc:MEXSTATUS_PBEXE]{mexstatus.PBEXE} 的值写入 \hyperref[fielddesc:MEXSTATUS_PPBEXE]{mexstatus.PPBEXE}。\\
通过 MRET 退出陷阱时，\hyperref[fielddesc:MEXSTATUS_PPBEXE]{mexstatus.PPBEXE} 的值被写回 \hyperref[fielddesc:MEXSTATUS_PBEXE]{mexstatus.PBEXE}，并且 \hyperref[fielddesc:MEXSTATUS_PPBEXE]{mexstatus.PPBEXE} 自动设置为 1。\\
(R/W)
\label{fielddesc:MEXSTATUS_PMP_ERR}\item [PMP\_ERR] 由于 PMP/PMA 异常进入加载/存储错误陷阱时，自动设置此位。\\
一旦进入陷阱处理程序，必须读取此位，确定加载/存储错误是否由 PMP/PMA 错误造成。\\
MRET 时此位自动清除。\\
(R/W)
\item [见下页……]
\end{reglist}\end{regdesc}
\end{register}
