\chapter{Permission Control (PMS)}
\label{mod:permctrl}

\section{Overview}

The \chipname{} employs a two-fold permission control strategy: Physical Memory Protection (PMP) and Access Permission Management (APM).

The PMP is located inside the HP CPU and controls the HP CPU's access to all address spaces. APM is an access permission management module located at the bus port. APM, together with PMP, manages permissions for the HP CPU to access parts of the memory and the peripheral registers. Additionally, APM manages DMA masters such as GDMA and MEM\_MONITOR to access parts of internal and external memory.

The distribution of management areas by PMP and APM is shown in Table \ref{tab:pmp-apm}.

\begin{ThreePartTable}

\begin{TableNotes}

    \item[2] Peripheral registers in the high-performance system, address range: 0x6000\_0000 – 0x600A\_FFFF
    \item[3] Peripheral registers in the low-power system, address range: 0x600B\_0000 – 0x600B\_FFFF
    \item[4] External memory, e.g., flash, PSRAM.

As shown in Table \ref{tab:pmp-apm}, when the HP CPU accesses ROM and EXT MEM, the access paths are solely managed by PMP. On the other hand, when the HP CPU accesses HP SRAM, HP CPU\_PERI, HP\_PERI, and LP\_PERI, the access paths are controlled by both PMP and APM. If the PMP check fails, the APM permission control will not be triggered.

PMP-related registers are located inside the HP CPU and can be read or configured with special instructions. For how to configure PMP, please refer to Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}} > \textit{\nameref{sec:riscvcpu-pmp}}.

The following sections will provide a detailed introduction to the APM module, covering its features, functions, and configurations.

\section{Introduction to APM}

The APM module contains two parts: the TEE (Trusted Execution Environment) controller and the APM controller. Each module contains its own register: the TEE register and APM register.

\begin{itemize}
    \item The TEE controller is responsible for configuring the security mode of a particular master in \chipname{} (such as GDMA) to access memory or peripheral registers. There are four security modes: TEE, REE0 (Rich Execution Environment), REE1, and REE2.
    \item The APM controller manages a master's access permissions when accessing memory and peripheral registers. By comparing the pre-configured address ranges and corresponding access permissions with the information carried on the bus, such as ID number (please refer to Table \ref{tab:master_source}), security mode, access address, access permissions, etc, the APM controller determines whether the access is allowed.
\end{itemize}

TEE-related registers configure the security mode of each master. The APM-related registers specify the access permission and access address range of each master in the configured security mode. With the TEE controller and APM controller, \chipname{} can precisely control the access permission of all masters to the memory and peripheral registers.

\section{Features}\label{sec:permctrl-features}

\chipname{} has one TEE controller: HP\_TEE.

\begin{itemize}
    \item HP\_TEE has the following features:
        \begin{itemize}
        \item Four security modes for the masters
        \item Configurable security mode for 32 masters
        \end{itemize}

\end{itemize}

\chipname{} has three APM controllers: HP\_APM\_CTRL, LP\_APM\_CTRL, and CPU\_APM\_CTRL.

\begin{itemize}
    \item HP\_APM\_CTRL has the following features:
        \begin{itemize}
        \item Access permission configuration for up to 16 address ranges
        \item Access management to HP SRAM, HP CPU\_PERI, HP\_PERI, and EXT MEM
        \item Interrupt function on illegal access
        \item Exception information record
        \end{itemize}
    \item LP\_APM\_CTRL has the following features:
        \begin{itemize}
        \item Access permission configuration for up to four address ranges
        \item Access permission management to LP\_PERI
        \item Interrupt function on illegal access
        \item Exception information record
        \end{itemize}
    \item CPU\_APM\_CTRL has the following features:
        \begin{itemize}
        \item Access permission configuration for up to eight address ranges
        \item Access permission management for HP CPU to access HP SRAM
        \item Interrupt function on illegal access
        \item Exception information record
        \end{itemize}
\end{itemize}

\section{Functional Description}

\subsection{TEE Controller Functional Description}

\chipname{} supports four security modes: TEE, REE0, REE1, and REE2.

For the HP CPU to access memory or peripheral registers, first select the machine mode or user mode for it, then configure its security mode with HP\_TEE registers. For the configuration of machine mode and user mode, please refer to RISC\-/V Instruction Set Manual, Volume II: Privileged Architecture, Version 1.10.

\begin{itemize}
    \item When the HP CPU is in machine mode, its security mode is TEE mode.
    \item When the HP CPU is in user mode, its security mode is REE mode. The specific REE0, REE1 or REE2 mode depends on \hyperref[fielddesc:TEEMNMODE]{TEE\_M0\_MODE} as follows:
    \begin{itemize}
        \item If \hyperref[fielddesc:TEEMNMODE]{TEE\_M0\_MODE} is 0, which is TEE mode, the valid mode that actually takes effect in the HP CPU user mode is REE0.
        \item if \hyperref[fielddesc:TEEMNMODE]{TEE\_M0\_MODE} is set to 1, 2, or 3, the security mode is REE0, REE1, and REE2 respectively.
    \end{itemize}
\end{itemize}

For masters other than the HP CPU to access memory or peripheral registers, configure their security mode with \hyperref[fielddesc:TEEMNMODE]{TEE\_M\regindex{n}\_MODE}. \regindex{n} represents the ID number of master as listed in Table \ref{tab:master_source}.

\subsubsection{Address Ranges}

\begin{itemize}
    \item The \hyperref[sec:permctrl-reg-hp]{HP\_APM\_REG} register module configures up to 16 address ranges (i.e., region \regindex{n}, \regindex{n}=0-15) for HP\_APM\_CTRL. The start and end address of each region (address range) are configured by  \hyperref[regdesc:HPAPMREGIONNADDRSTARTREG]{HP\_APM\_REGION\regindex{n}\_ADDR\_START} and \hyperref[regdesc:HPAPMREGIONNADDRENDREG]{HP\_APM\_REGION\regindex{n}\_ADDR\_END}, respectively. Configure the bit \regindex{n} of \hyperref[fielddesc:HPAPMREGIONFILTEREN]{HP\_APM\_REGION\_FILTER\_EN\_REG} to enable region \regindex{n}. Region 0 (i.e., the first address range) is enabled by default.

    \item The \hyperref[sec:permctrl-reg-lp]{LP\_APM\_REG} register module configures up to four address ranges (i.e., region \regindex{n}, \regindex{n}=0-3) for LP\_APM\_CTRL. The start and end address of each region are configured by \hyperref[fielddesc:LPAPMREGIONNADDRSTART]{LP\_APM\_REGION\regindex{n}\_ADDR\_START} and \hyperref[fielddesc:LPAPMREGIONNADDREND]{LP\_APM\_REGION\regindex{n}\_ADDR\_END}. Configure the bit \regindex{n} of \hyperref[fielddesc:LPAPMREGIONFILTEREN]{LP\_APM\_REGION\_FILTER\_EN\_REG} to enable region \regindex{n}. Region 0 (i.e., the first address range) is enabled by default.

    \item The \hyperref[sec:permctrl-reg-cpu]{CPU\_APM\_REG} register module configures up to eight address ranges (i.e., region \regindex{n}, \regindex{n}=0-7) for CPU\_APM\_CTRL. The start and end address of each region are configured by \hyperref[fielddesc:CPUAPMREGIONNADDRSTART]{CPU\_APM\_REGION\regindex{n}\_ADDR\_START} and \hyperref[fielddesc:CPUAPMREGIONNADDREND]{CPU\_APM\_REGION\regindex{n}\_ADDR\_END}. Configure the bit \regindex{n} of \hyperref[fielddesc:CPUAPMREGIONFILTEREN]{CPU\_APM\_REGION\_FILTER\_EN\_REG} to enable region \regindex{n}. Region 0 (i.e., the first address range) is enabled by default.
\end{itemize}

When configuring the address ranges, the address requires a 4-byte alignment (the lower two bits of the address are 0). For example, the address range could be set to 0x4080000C \verb+~+ 0x40808774 or 0x600C0008 \verb+~+ 0X600CFF70.

\subsubsection{Access Permissions of Address Ranges}

For each address range, the access permissions (read/write/execute) are configurable in different security modes:

\begin{itemize}

    \item Masters in TEE mode always have read, write, and execute permissions in the address range.

    \item For masters in REE0, REE1 or REE2 mode, access permissions are configured with \hyperref[regdesc:HPAPMREGIONNATTRREG]{HP\_APM\_REGION\regindex{n}\_ATTR\_REG}, \hyperref[regdesc:LPAPMREGIONNATTRREG]{LP\_APM\_REGION\regindex{n}\_ATTR\_REG}, or \hyperref[regdesc:CPUAPMREGIONNATTRREG]{CPU\_APM\_REGION\regindex{n}\_ATTR\_REG} depending on the access path.

\end{itemize}

Different access paths managed by the same register module share the configuration of address ranges and access permissions. For example, the permission management of data paths HP\_APM\_CTRL M0-M3 follows the address ranges and access permissions of the 16 address ranges configured in the \hyperref[sec:permctrl-reg-hp]{HP\_APM\_REG} register module. Likewise, the permission management of data path LP\_APM\_CTRL M0 follows the address ranges and access permissions of the four address ranges configured in the \hyperref[sec:permctrl-reg-lp]{LP\_APM\_REG} register module. The permission management of data paths CPU\_APM\_CTRL M0-M1 follows the address ranges and access permissions of the eight address ranges configured in the \hyperref[sec:permctrl-reg-cpu]{CPU\_APM\_REG} register module.

For HP SRAM, all masters access it through HP\_APM\_CTRL M1, except that the HP CPU accesses it through PMP and CPU\_APM\_CTRL. Suppose that \hyperref[fielddesc:HPAPMM1FUNCEN]{HP\_APM\_M1\_FUNC\_EN} is enabled and a master in REE1 mode needs to access HP SRAM. The whole process is as follows:

\begin{enumerate}
    \item HP\_APM\_CTRL M1 first determines whether the address requested to access is within the 16 address ranges configured in the \hyperref[sec:permctrl-reg-hp]{HP\_APM\_REG} register module.

    \item Assuming that the address requested to access is within the second address range, then HP\_APM\_CTRL M1 determines whether the address range is enabled, that is, whether bit 1 of \hyperref[fielddesc:HPAPMREGIONFILTEREN]{HP\_APM\_REGION\_FILTER\_EN} is 1.

    \item If the address range is enabled, HP\_APM\_CTRL M1 checks whether the master has read permission for the second address range, that is, whether \hyperref[fielddesc:HPAPMREGIONNR1R]{HP\_APM\_REGION1\_R1\_R} in \hyperref[regdesc:HPAPMREGIONNATTRREG]{HP\_APM\_REGION1\_ATTR\_REG} is 1. If valid, the read request will be allowed, otherwise, 0 will be returned.

\end{enumerate}

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
    \item When the chip is powered up, only the HP CPU is in TEE mode by default, and other masters are in REE2 mode. By default, the APM controller blocks access requests from all masters in REE0, REE1, and REE2 modes.
    \item All registers listed in \ref{sec:permctrl-reg-summ} \textit{\nameref{sec:permctrl-reg-summ}} can only be configured by the masters that are in TEE security mode.
    \item The configured address ranges may overlap. For example, if one region is set to be unreadable and another region readable, then the overlapping area of the two regions is readable. The same rules apply to write and execute permissions.
    \item Configure \hyperref[fielddesc:HPAPMREGIONNLOCK]{HP\_APM\_REGION\regindex{n}\_LOCK} and \hyperref[fielddesc:LPAPMREGIONNLOCK]{LP\_APM\_REGION\regindex{n}\_LOCK} to lock the configuration of the start and end address of REGION\regindex{n} and its access permission. Only chip reset, system reset, and core reset can unlock the configurations and restore all configurations in APM registers to their default values.
\end{itemize}
\end{tiplisting}

\section{Programming Procedure}

For a master to access memory or peripheral registers, follow the programming procedures below:

\begin{enumerate}

    \item Set the HP CPU to the machine mode (i.e., TEE mode).
    \item Configure \hyperref[fielddesc:TEEMNMODE]{TEE\_M\regindex{n}\_MODE} to choose the security mode of the master. The master ID \regindex{n} is defined in Table \ref{tab:master_source}.

    \item Configure \hyperref[regdesc:HPAPMREGIONNADDRSTARTREG]{HP\_APM\_REGION\regindex{n}\_ADDR\_START} and \hyperref[regdesc:HPAPMREGIONNADDRENDREG]{HP\_APM\_REGION\regindex{n}\_ADDR\_END}, \hyperref[regdesc:LPAPMREGIONNADDRSTARTREG]{LP\_APM\_REGION\regindex{n}\_ADDR\_START} and \hyperref[regdesc:LPAPMREGIONNADDRENDREG]{LP\_APM\_REGION\regindex{n}\_ADDR\_END}, or \hyperref[regdesc:CPUAPMREGIONNADDRSTARTREG]{CPU\_APM\_REGION\regindex{n}\_ADDR\_START} and \hyperref[regdesc:CPUAPMREGIONNADDRENDREG]{CPU\_APM\_REGION\regindex{n}\_ADDR\_END} to define the start and end address of the access address ranges.

    \item Configure \hyperref[regdesc:HPAPMREGIONNATTRREG]{HP\_APM\_REGION\regindex{n}\_ATTR\_REG}, \hyperref[regdesc:LPAPMREGIONNATTRREG]{LP\_APM\_REGION\regindex{n}\_ATTR\_REG}, or \hyperref[regdesc:CPUAPMREGIONNATTRREG]{CPU\_APM\_REGION\regindex{n}\_ATTR\_REG} to set the access permissions for each region.


    \item Configure bit \regindex{n} of \hyperref[fielddesc:HPAPMREGIONFILTEREN]{HP\_APM\_REGION\_FILTER\_ EN\_REG}, \hyperref[fielddesc:LPAPMREGIONFILTEREN]{LP\_APM\_REGION\_FILTER\_ EN\_REG}, or \hyperref[fielddesc:CPUAPMREGIONFILTEREN]{CPU\_APM\_REGION\_FILTER\_ EN\_REG} to enable permission checks for the corresponding address range.

    \item Configure \hyperref[regdesc:HPAPMFUNCCTRLREG]{HP\_APM\_ FUNC\_CTRL\_REG}, \hyperref[regdesc:LPAPMFUNCCTRLREG]{LP\_APM \_FUNC\_CTRL\_REG}, or \hyperref[regdesc:CPUAPMFUNCCTRLREG]{CPU\_APM \_FUNC\_CTRL\_REG} to enable the permission management for different access paths (enabled by default).

\end{enumerate}

Take I2S accessing HP SRAM via GDMA as an example, assuming that it is only allowed to read and write in the fourth address range 0x40805000 \verb+~+ 0x4080FFFF:

\begin{enumerate}

    \item Configure the HP CPU to machine mode (i.e., TEE mode).

    \item According to the master ID number in Table \ref{tab:master_source}, set \hyperref[fielddesc:TEEMNMODE]{TEE\_M19\_MODE} to 1, so that the security mode for I2S access via GDMA is REE0.

    \item Configure \hyperref[regdesc:HPAPMREGIONNADDRSTARTREG]{HP\_APM\_REGION3\_ADDR\_START} to 0x40805000 and \hyperref[regdesc:HPAPMREGIONNADDRSTARTREG]{HP\_APM\_REGION3\_ADDR\_END} to 0x4080EFFC, respectively.

    \item Set \hyperref[fielddesc:HPAPMREGIONNR0W]{HP\_APM\_REGION3\_R0\_W} and \hyperref[fielddesc:HPAPMREGIONNR0R]{HP\_APM\_REGION3\_R0\_R} to 1 to enable the access permissions to the address range.

    \item Set bit 3 of \hyperref[fielddesc:HPAPMREGIONFILTEREN]{HP\_APM\_REGION\_FILTER\_EN} to 1 to enable region 3 (i.e., the fourth address range).

    \item Set \hyperref[fielddesc:HPAPMM1FUNCEN]{HP\_APM\_M1\_FUNC\_EN} to 1 to enable the permission management for HP\_APM\_CTRL M1.

\end{enumerate}

Through this configuration, I2S can read and write in the address range of 0x40805000 to 0x4080FFFF in HP SRAM using GDMA.

\section{Illegal Access and Interrupts}

If the information carried on the bus is inconsistent with the configuration, \chipname{} will regard it as an illegal access and proceed as follows:

\begin{itemize}
    \item Denies the access request and returns the default value:
    \begin{itemize}
        \item Returns 0 on instruction execution and read operations
        \item Invalidate write operations
    \end{itemize}
    \item Triggers interrupt
\end{itemize}

The APM controller will automatically record relevant information about the illegal access, including the master ID, security mode, access address, reasons for illegal access (address out of bounds or permission restrictions), and permission management result of each access path. All this information is obtained from relevant registers listed in Section \ref{sec:permctrl-reg-summ} \textit{\nameref{sec:permctrl-reg-summ}}.

Take the access path HP\_APM\_CTRL M0 as an example. When an illegal access occurs:

\begin{itemize}
    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONID]{HP\_APM\_M0\_EXCEPTION\_ID} records the master ID.
    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONMODE]{HP\_APM\_M0\_EXCEPTION\_MODE} records the security mode.
    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONADDR]{HP\_APM\_M0\_EXCEPTION\_ADDR} records the access address.
    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONSTATUS]{HP\_APM\_M0\_EXCEPTION\_STATUS} records the reason for illegal access.
    \begin{itemize}
        \item If the address requested to access is not within the enabled address ranges, bit 1 will be set to 1, indicating the address is out of bounds.
        \item If the address requested to access is within the enabled address ranges, but the master does not have the read/write/execution permission within this region, then bit 0 will be set to 1, indicating permission restrictions.
    \end{itemize}

    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONREGION]{HP\_APM\_M0\_EXCEPTION\_REGION} records the permission management result of each address range. This register has a total of 16 bits, corresponding to 16 address ranges, and bit 0 corresponds to the first address range. When the address to access is within a particular enabled address range, but the master does not have the corresponding read/write/execute permission within this address range, the corresponding bit of this register will be set to 1 .

\end{itemize}

\chipname{}'s APM controller can generate the following interrupt signals and send them to the \hyperref[mod:intmtrx]{Interrupt Matrix}:

\begin{itemize}
    \item HP\_APM\_M0\_INTR
    \item HP\_APM\_M1\_INTR
    \item HP\_APM\_M2\_INTR
    \item HP\_APM\_M3\_INTR
    \item LP\_APM\_M0\_INTR
    \item CPU\_APM\_M0\_INTR
    \item CPU\_APM\_M1\_INTR
\end{itemize}

The interrupt signals correspond to the controlled access paths shown in Figure \ref{fig:APM_Ctrl_Location}. If an illegal access occurs in a controlled access path, the corresponding interrupt will be generated.
