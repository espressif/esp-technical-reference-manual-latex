\chapter{GDMA Controller (GDMA)}
\label{mod:dma}


\section{Overview}


General Direct Memory Access (GDMA) is a feature that allows peripheral-to-memory, memory-to-peripheral, and memory-to-memory data transfer at high speed. The CPU is not involved in the GDMA transfer and therefore is more efficient with less workload.

GDMA has four independent channels: two transmit channels and two receive channels. The GDMA channels are shared and can be assigned to ADC, SHA, I2S or general-purpose SPI (GP-SPI) to access internal or external memory.

GDMA uses configurable priority and weight arbitration schemes to manage peripherals' needs for bandwidth.

\begin{figure}[H]
\fbox{\includegraphics[width=0.5\textwidth]{\modulefiles/figures/ahb_dma_c61_new.pdf}}
\centering
\caption{Modules that Share GDMA Channels}
\label{Figure:Module_with_DMA_AHB}
\end{figure}

\section{Features}

GDMA has the following features:

\begin{itemize}
    \item AHB bus architecture
    \item Programmable length of data to be transferred in bytes
    \item Access via any address and size
    \item Alignment:
        \begin{itemize}
            \item Descriptor address: 1-word aligned
            \item Data address and length: no requirements
        \end{itemize}
    \item Linked list of descriptors
    \item INCR burst transfer when accessing memory
    \item Two transmit channels and two receive channels
    \item Software-configurable selection of peripheral requesting its service
    \item Configurable channel priority and weight arbitration
    \item Support for memory transfer
\end{itemize}


\section{Architecture}

In \chipname{}, all modules that need high-speed data transfer support GDMA. The GDMA controller and CPU data bus have access to the same address space in memory. Figure \ref{fig:gdma-controller} shows the basic architecture of the GDMA controller.

\begin{figure}[H]
\fbox{\includegraphics[width=1\textwidth]{\modulefiles/figures/gdma_controller_c61.pdf}}
\centering
\caption{GDMA controller Architecture}
\label{fig:gdma-controller}
\end{figure}


GDMA has four independent channels: two transmit channels and two receive channels. Every channel can be connected to different peripherals. In other words, channels are general-purpose, and shared by peripherals.

GDMA reads data from or writes data to memory via AHB\_BUS. Before this, GDMA uses configurable arbitration schemes for channels requesting read or write access. For the available address range of internal and external memory, please see Chapter \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}.

Software can use the GDMA through linked lists, which are stored in the internal memory. These linked lists consist of outlink\textcolor{red}{\it{n}} and inlink\textcolor{red}{\it{n}}, where \textcolor{red}{\it{n}} indicates the channel number (ranging from 0 to 1).
The GDMA reads an outlink\textcolor{red}{\it{n}} (i.e., a linked list of transmit descriptors) from memory and transmits data in the corresponding memory according to the outlink\textcolor{red}{\it{n}}, or reads an inlink\textcolor{red}{\it{n}} (i.e., a linked list of receive descriptors) and stores received data into specific address space in the memory according to the inlink\textcolor{red}{\it{n}}.


\section{Functional Description}

\subsection{Linked List}

\begin{figure}[H]
\includegraphics[width=1\textwidth]{\modulefiles/figures/link_strut}
\centering
\caption{Structure of a Linked List}
\label{fig:gdma-link-descriptor}
\end{figure}

Figure \ref{fig:gdma-link-descriptor} shows the structure of a linked list. An outlink and an inlink have the same structure. A linked list is formed by one or more descriptors, and each descriptor consists of three words. Linked lists should be stored in the memory for the GDMA to be able to use them. The meanings of a descriptor's fields are as follows:

\begin{itemize}
    \item owner (DW0) [31]: Specifies who is allowed to access the buffer that this descriptor points to.\\
    0: CPU can access the buffer. \\
    1: The GDMA controller can access the buffer.\\
    When the GDMA controller stops using the buffer, this bit in a receive descriptor is automatically cleared by hardware, while this bit in a transmit descriptor can only be automatically cleared by hardware if \hyperref[fielddesc:AHBDMAOUTAUTOWRBACKCHN]{AHB\_DMA\_OUT\_AUTO\_WRBACK\_CH\regindex{n}} is set to 1. Software can disable automatic clearing by hardware by setting the \hyperref[fielddesc:AHBDMAOUTLOOPTESTCHN]{AHB\_DMA\_OUT\_LOOP\_TEST\_CH\regindex{n}} or \hyperref[fielddesc:AHBDMAINLOOPTESTCHN]{AHB\_DMA\_IN\_LOOP\_TEST\_CH\regindex{n}} bit. When software loads a linked list, this bit should be set to 1.
    \\{\bfseries Note:} AHB\_DMA\_OUT is the prefix of transmit channel registers, and AHB\_DMA\_IN is the prefix of receive channel registers.
    \item suc\_eof (DW0) [30]: Specifies whether the AHB\_DMA\_IN\_SUC\_EOF\_CH\regindex{n}\_INT or AHB\_DMA\_OUT\_EOF\_CH\regindex{n}\_INT interrupt will be triggered when the data corresponding to this descriptor has been received or transmitted.\\
    0: No interrupt will be triggered after the current descriptor's successful transfer;\\
    1: An interrupt will be triggered after the current descriptor's successful transfer.\\
    For receive descriptors, software needs to clear this bit to 0, and hardware will set it to 1 after receiving data containing the EOF flag.\\
    For transmit descriptors, software needs to set this bit to 1 as needed.\\
    If software configures this bit to 1 in a descriptor, the GDMA will include the EOF flag in the data sent to the corresponding peripheral, indicating to the peripheral that this data segment marks the end of one transfer phase.
    \item reserved (DW0) [29]: Reserved. The value of this bit does not matter.
    \item err\_eof (DW0) [28]: Specifies whether the received data has errors.\\
    0: The received data does not have errors.\\
    1: The received data has errors.\\
    GDMA configures this bit in a receive descriptor based on the err\_eof provided by the peripheral when writing back the descriptor (Currently, no peripheral on \chipname{} supports the err\_eof flag).
    \item reserved (DW0) [27:24]: Reserved.
    \item length (DW0) [23:12]: Specifies the number of valid bytes in the buffer that this descriptor points to. This field in a transmit descriptor is written by software and indicates how many bytes can be read from the buffer; this field in a receive descriptor is written by hardware automatically and indicates how many valid bytes have been stored in the buffer.
    \item size (DW0) [11:0]: Specifies the size of the buffer that this descriptor points to.
    \item buffer address pointer (DW1): Address of the buffer.
    \item next descriptor address (DW2): Address of the next descriptor. If the current descriptor is the last one, this value is 0. This field can only point to the memory space.
        \begin{tiplisting}
            Please note that addresses of GDMA descriptors should be 4-byte aligned.
        \end{tiplisting}
\end{itemize}

If the length of data received is smaller than the size of the buffer, the GDMA controller will not use the available space of the buffer in the next transaction. If the length of data received is larger than the size of the buffer, the GDMA controller will use new descriptors to complete the data transfer after the current transactions.

\subsection{Peripheral-to-Memory and Memory-to-Peripheral Data Transfer}

The GDMA controller can transfer data from memory to peripheral (transmit) and from peripheral to memory (receive). A transmit channel transfers data in the specified memory location to a peripheral's transmitter via an outlink\textcolor{red}{\it{n}}, whereas a receive channel transfers data received by a peripheral to the specified memory location via an inlink\textcolor{red}{\it{n}}.

Every transmit and receive channel can be connected to any peripheral with the GDMA feature. Table \ref{tab:gdma-ahb-peri-sel} illustrates how to select the peripheral to be connected via registers. ``Dummy-\regindex{n}" corresponds to register values for memory-to-memory data transfer. When a channel is connected to a peripheral, the rest of the channels cannot be connected to that peripheral.

\subsection{Memory-to-Memory Data Transfer}

The GDMA controller also allows memory-to-memory data transfer. Such data transfer can be enabled by setting \hyperref[fielddesc:AHBDMAMEMTRANSENCHN]{AHB\_DMA\_MEM\_TRANS\_EN\_CH\regindex{n}}, which connects the output of transmit channel \textcolor{red}{\it{n}} to the input of receive channel \textcolor{red}{\it{n}}. Note that a transmit channel is only connected to the receive channel with the same number (\textcolor{red}{\it{n}}), and \hyperref[fielddesc:AHBDMAPERIINSELCHN]{AHB\_DMA\_PERI\_IN\_SEL\_CH\regindex{n}} and \hyperref[fielddesc:AHBDMAPERIOUTSELCHN]{AHB\_DMA\_PERI\_OUT\_SEL\_CH\regindex{n}} should be configured to the same value corresponding to ``Dummy".

\subsection{Enabling GDMA}

The software uses the GDMA controller through linked lists. When the GDMA controller receives data, software loads an inlink, configures the \hyperref[fielddesc:AHBDMAINLINKADDRCHN]{AHB\_DMA\_INLINK\_ADDR\_CH\regindex{n}} field with the address of the first receive descriptor, and sets the \hyperref[fielddesc:AHBDMAINLINKSTARTCHN]{AHB\_DMA\_INLINK\_START\_CH\regindex{n}} bit to enable GDMA. When the GDMA controller transmits data, software loads an outlink, prepares data to be transmitted, configures the \hyperref[fielddesc:AHBDMAOUTLINKADDRCHN]{AHB\_DMA\_OUTLINK\_ADDR\_CH\regindex{n}} field with the address of the first transmit descriptor, and sets the \hyperref[fielddesc:AHBDMAOUTLINKSTARTCHN]{AHB\_DMA\_OUTLINK\_START\_CH\regindex{n}} bit to enable GDMA. The \hyperref[fielddesc:AHBDMAINLINKSTARTCHN]{AHB\_DMA\_INLINK\_START\_CH\regindex{n}} bit and \hyperref[fielddesc:AHBDMAOUTLINKSTARTCHN]{AHB\_DMA\_OUTLINK\_START\_CH\regindex{n}} bit are cleared automatically by hardware.

In some cases, you may want to append more descriptors to a DMA transfer that is already started. Naively, it would seem to be possible to do this by clearing the EOF bit of the final descriptor in the existing list and setting its next descriptor address pointer field (DW2) to the first descriptor of the to-be-added list. However, this strategy fails if the existing DMA transfer is almost or entirely finished. Instead, the GDMA controller has specialized logic to make sure a DMA transfer can be continued or restarted: if the transfer is ongoing, the controller will make sure to take the appended descriptors into account; if the transfer has already finished, the controller will restart with the new descriptors. This is implemented by the Restart function.

When using the Restart function, software needs to rewrite the address of the first descriptor in the new list to DW2 of the last descriptor in the loaded list, and set \hyperref[fielddesc:AHBDMAINLINKRESTARTCHN]{AHB\_DMA\_INLINK\_RESTART\_CH\regindex{n}} bit or \hyperref[fielddesc:AHBDMAOUTLINKRESTARTCHN]{AHB\_DMA\_OUTLINK\_RESTART\_CH\regindex{n}} bit (these two bits are cleared automatically by hardware). As shown in Figure \ref{fig:gdma-link-restart}, by doing so hardware can obtain the address of the first descriptor in the new list when reading the last descriptor in the loaded list, and then read the new list.

\begin{figure}[H]
\includegraphics[width=1\textwidth]{\modulefiles/figures/link_restart}
\centering
\caption{Relationship among Linked Lists}
\label{fig:gdma-link-restart}
\end{figure}


\subsection{Linked List Reading Process}

Once configured and enabled by software, the GDMA controller starts to read the linked list from memory. The GDMA performs checks on descriptors in the linked list. Only if descriptors pass the checks, the corresponding GDMA channel will start data transfer. If the descriptors fail any of the checks, hardware will trigger descriptor error interrupt (either AHB\_DMA\_IN\_DSCR\_ERR\_CH\regindex{n}\_INT or AHB\_DMA\_OUT\_DSCR\_ERR\_CH\regindex{n}\_INT), and the channel will halt.

The checks performed on descriptors are:
\begin{itemize}
    \item Owner bit check when \hyperref[fielddesc:AHBDMAINCHECKOWNERCHN]{AHB\_DMA\_IN\_CHECK\_OWNER\_CH\regindex{n}} or \hyperref[fielddesc:AHBDMAOUTCHECKOWNERCHN]{AHB\_DMA\_OUT\_CHECK\_OWNER\_CH\regindex{n}} is set to 1. If the owner bit is 0, the buffer is accessed by the CPU. In this case, the owner bit fails the check. The owner bit will not be checked if \hyperref[fielddesc:AHBDMAINCHECKOWNERCHN]{AHB\_DMA\_IN\_CHECK\_OWNER\_CH\regindex{n}} or \hyperref[fielddesc:AHBDMAOUTCHECKOWNERCHN]{AHB\_DMA\_OUT\_CHECK\_OWNER\_CH\regindex{n}} is 0.
    \item Descriptor address check, which checks if the descriptor address is located in configured memory space for GDMA. If a GDMA descriptor points to \hyperref[fielddesc:AHBDMAACCESSINTRMEMSTARTADDR]{AHB\_DMA\_ACCESS\_INTR\_MEM\_START\_ADDR} \verb+~+ \hyperref[fielddesc:AHBDMAACCESSINTRMEMENDADDR]{AHB\_DMA\_ACCESS\_INTR\_MEM\_END\_ADDR}, it passes the check. For details, please refer to Section \ref{sec:dma-access-inter-ram}.
\end{itemize}

After the software detects a descriptor error interrupt, it must load new descriptors, and enable GDMA by setting \hyperref[fielddesc:AHBDMAOUTLINKSTARTCHN]{AHB\_DMA\_OUTLINK\_START\_CH\regindex{n}} or \hyperref[fielddesc:AHBDMAINLINKSTARTCHN]{AHB\_DMA\_INLINK\_START\_CH\regindex{n}} bit.


\subsection{EOF}

{\bfseries Note: }In this chapter, EOF of transmit descriptors refers to suc\_eof (i.e., bit 30 of DW0), while EOF of receive descriptors refers to both suc\_eof and err\_eof (i.e., bit 28 of DW0).

The GDMA controller uses EOF (end of frame) flags to indicate the end of data segment transfer corresponding to a specific descriptor.

For data transmission, the GDMA generates two types of EOF interrupts:
\begin{itemize}
    \item AHB\_DMA\_OUT\_EOF\_CH\regindex{n}\_INT, generated when the suc\_eof bit of any descriptor in the linked list is set, and the data corresponding to this descriptor has been transmitted. This interrupt is enabled by setting the \hyperref[fielddesc:AHBDMAOUTEOFCHNINTENA]{AHB\_DMA\_OUT\_EOF\_CH\regindex{n}\_INT\_ENA} bit.
    \item AHB\_DMA\_OUT\_TOTAL\_EOF\_CH\regindex{n}\_INT, generated when the suc\_eof bit of the last descriptor in the linked list is set, and the data corresponding to the last descriptor has been transmitted. This interrupt is enabled by setting the \hyperref[fielddesc:AHBDMAOUTTOTALEOFCHNINTENA]{AHB\_DMA\_OUT\_TOTAL\_EOF\_CH\regindex{n}\_INT\_ENA} bit.
\end{itemize}

For data reception, the GDMA generates one type of EOF interrupts:
\begin{itemize}
    \item AHB\_DMA\_IN\_SUC\_EOF\_CH\regindex{n}\_INT, generated when a data segment with an EOF flag has been received. This interrupt is enabled by setting the \hyperref[fielddesc:AHBDMAINSUCEOFCHNINTENA]{AHB\_DMA\_IN\_SUC\_EOF\_CH\regindex{n}\_INT\_ENA} bit.
\end{itemize}

When detecting an \hyperref[int: GDMAOUTTOTALEOFINT]{AHB\_DMA\_OUT\_TOTAL\_EOF\_CH\regindex{n}\_INT} or \hyperref[int:GDMAINSUCEOFINT]{AHB\_DMA\_IN\_SUC\_EOF\_CH\regindex{n}\_INT} interrupt, software can read the value of the \hyperref[fielddesc:AHBDMAOUTEOFDESADDRCHN]{AHB\_DMA\_OUT\_EOF\_DES\_ADDR\_CH\regindex{n}} or \hyperref[fielddesc:AHBDMAINSUCEOFDESADDRCHN]{AHB\_DMA\_IN\_SUC\_EOF\_DES\_ADDR\_CH\regindex{n}} field, which stores the address of the finished descriptor. Therefore, the software can tell which descriptors have been used and reclaim them as needed.


\subsection{Accessing Memory}\label{sec:dma-access-inter-ram}

Any transmit and receive channels of GDMA can access the memory address space configured by \hyperref[fielddesc:AHBDMAACCESSINTRMEMSTARTADDR]{AHB\_DMA\_ACCESS\_INTR\_MEM\_START\_ADDR} and \hyperref[fielddesc:AHBDMAACCESSINTRMEMENDADDR]{AHB\_DMA\_ACCESS\_INTR\_MEM\_END\_ADDR}.

To improve data transfer efficiency, GDMA can send data in burst mode. Burst mode is disabled by default, and the burst length can be configured to SINGLE, INCR4, or INCR8 respectively by setting \hyperref[fielddesc:AHBDMAINDATABURSTMODESELCHN]{AHB\_DMA\_IN\_DATA\_BURST\_MODE\_SEL\_CH\regindex{n}} and \hyperref[fielddesc:AHBDMAOUTDATABURSTMODESELCHN]{AHB\_DMA\_OUT\_DATA\_BURST\_MODE\_SEL\_CH\regindex{n}} to 0, 1, or 2.

When GDMA accesses the configured memory space, there are no requirements for descriptor field alignment.


\subsection{Arbitration}

To ensure timely response to peripherals running at a high speed with low latency (such as general-purpose SPI), the GDMA controller implements two arbitration schemes, based on channel priority and channel weight respectively.

\begin{itemize}
    \item Priority arbitration: Each channel can be assigned a priority from 0 \verb+~+ 5 (in total 6 levels). The larger the number, the higher the priority. When several channels perform data transfers at the same time, the GDMA would respond to the transfer requests according to priority levels, and the channel with higher priority would get a response more timely.
    \item Weight arbitration:
        \begin{itemize}
            \item Each channel is assigned a weight (i.e., the number of tokens) from 0 \verb+~+ 15. The GDMA divides the AHB bus clock period into multiple time slots, and in each time slot, the number of transfers performed by a channel is determined by the number of its tokens. Every time a channel performs a data transfer, one of its tokens is spent. If all of its tokens have been spent in a time slot, then this channel's transfer request will no longer be responded to until the time slot expires, or until tokens of all channels have been spent and the new time slot starts.
            \item If the number of tokens assigned to a channel is not zero, the GDMA waits for this channel's tokens to be spent even if it does not have data transfer requests, and will not exit from this time slot ahead of expiration. This leads to a waste of bandwidth. Therefore, the GDMA provides weight arbitration optimization. When this feature is enabled, a channel without data transfer requests will not be involved in the arbitration and its tokens will be ignored. When all channels no longer have transfer requests, the GDMA arbiter exits from the current time slot before expiration. This way, the arbitration response is accelerated and the transfer efficiency is improved.
        \end{itemize}
\end{itemize}


\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
    \item If channels have the same weight but different priorities, on condition that the number of bytes to be transferred is the same and not large, channels with higher priorities would finish data transfers first.
    \item If channels have different priorities and weights, on condition that the number of bytes to be transferred is the same, channels with higher weights would be allocated more bandwidth and finish data transfers first.
\end{itemize}
\end{tiplisting}




\section{Event Task Matrix Feature}\label{sec:dma-etm-func}

The GDMA controller on \chipname{} supports the Event Task Matrix (ETM) function, which allows GDMA's ETM tasks to be triggered by any peripherals' ETM events, or GDMA's ETM events to trigger any peripherals' ETM tasks. This section introduces the ETM tasks and events related to GDMA. For more information, please refer to Chapter \ref{mod:evntaskmatrix} \nameref{mod:evntaskmatrix}.

GDMA can receive the following ETM tasks:

\begin{itemize}
    \item GDMA\_TASK\_IN\_START\_CH\regindex{n}: Enables the corresponding RX channel \regindex{n} for data transfer.
    \item GDMA\_TASK\_OUT\_START\_CH\regindex{n}: Enables the corresponding TX channel \regindex{n} for data transfer.
\end{itemize}

\begin{tiplisting}
Above ETM tasks can achieve the same functions as CPU configuring \hyperref[fielddesc:AHBDMAINLINKSTARTCHN]{AHB\_DMA\_INLINK\_START\_CH\regindex{n}} and \\\hyperref[fielddesc:AHBDMAOUTLINKSTARTCHN]{AHB\_DMA\_OUTLINK\_START\_CH\regindex{n}}. When \hyperref[fielddesc:AHBDMAINETMENCHN]{AHB\_DMA\_IN\_ETM\_EN\_CH\regindex{n}}  or \hyperref[fielddesc:AHBDMAOUTETMENCHN]{AHB\_DMA\_OUT\_ETM\_EN\_CH\regindex{n}} is 1, only ETM tasks can be used to configure the transfer direction and enable the corresponding GDMA channel. When \hyperref[fielddesc:AHBDMAINETMENCHN]{AHB\_DMA\_IN\_ETM\_EN\_CH\regindex{n}} or \hyperref[fielddesc:AHBDMAOUTETMENCHN]{AHB\_DMA\_OUT\_ETM\_EN\_CH\regindex{n}} is 0, only CPU can be used to enable the corresponding GDMA channel.
\end{tiplisting}

GDMA can generate the following ETM events:

\begin{itemize}
    \item GDMA\_EVT\_IN\_DONE\_CH\regindex{n}: Indicates that the data has been received according to the receive descriptor via channel \regindex{n}.
    \item GDMA\_EVT\_IN\_SUC\_EOF\_CH\regindex{n}: Indicates that the data corresponding to a receive descriptor has been received via channel \regindex{n} and the EOF bit of this descriptor is 1.
    \item GDMA\_EVT\_IN\_FIFO\_EMPTY\_CH\regindex{n}: Indicates that the RX FIFO has become empty.
    \item GDMA\_EVT\_IN\_FIFO\_FULL\_CH\regindex{n}: Indicates that the RX FIFO has become full.
    \item GDMA\_EVT\_OUT\_DONE\_CH\regindex{n}: Indicates that the data has been transmitted according to the transmit descriptor via channel \regindex{n}.
    \item GDMA\_EVT\_OUT\_EOF\_CH\regindex{n}: Indicates that the data corresponding to a transmit descriptor has been transmitted or received via channel \regindex{n} and the EOF bit of this descriptor is 1.
    \item GDMA\_EVT\_OUT\_TOTAL\_EOF\_CH\regindex{n}: Indicates that the data corresponding to the last transmit descriptors has been sent via transmit channel \regindex{n} and the EOF bit of this descriptor is 1.
    \item GDMA\_EVT\_OUT\_FIFO\_EMPTY\_CH\regindex{n}: Indicates that the TX FIFO has become empty.
    \item GDMA\_EVT\_OUT\_FIFO\_FULL\_CH\regindex{n}: Indicates that the TX FIFO has become full.
\end{itemize}

In practical applications, GDMA's ETM events can trigger its own ETM tasks. For example, the GDMA\_EVT\_OUT\_TOTAL\_EOF\_CH0 event can trigger the GDMA\_TASK\_IN\_START\_CH1 task, and in this way trigger a new round of GDMA operations.

\section{Interrupts}

\chipname{}'s GDMA module can generate the following interrupt signals that will be sent to the \hyperref[mod:intmtrx]{Interrupt Matrix}.

    \begin{itemize}
        \item GDMA\_IN\_CH0\_INTR
        \item GDMA\_IN\_CH1\_INTR
        \item GDMA\_OUT\_CH0\_INTR
        \item GDMA\_OUT\_CH1\_INTR
    \end{itemize}

There are several internal interrupt sources from GDMA that can generate the above interrupt signals.

The GDMA\_IN\_CH\regindex{n}\_INTR (\regindex{n} is 0 \verb+~+ 1) signals are generated by the following interrupt sources:

\begin{itemize}
    \item \label{int:GDMAINDONEINT}AHB\_DMA\_IN\_DONE\_CH\regindex{n}\_INT: Triggered when all data corresponding to a receive descriptor has been received via receive channel \textcolor{red}{\it{n}}.
    \item \label{int:GDMAINSUCEOFINT}AHB\_DMA\_IN\_SUC\_EOF\_CH\regindex{n}\_INT: Triggered when the suc\_eof bit in a receive descriptor is 1 and the data corresponding to this receive descriptor has been received via receive channel \regindex{n}.
    \item \label{int:GDMAINDSCREMPTYINT}AHB\_DMA\_IN\_DSCR\_EMPTY\_CH\regindex{n}\_INT: Triggered when the size of the buffer pointed by receive descriptors is smaller than the length of data to be received via receive channel \textcolor{red}{\it{n}}.
    \item \label{int:GDMAINDSCRERRINT}AHB\_DMA\_IN\_DSCR\_ERR\_CH\regindex{n}\_INT: Triggered when a receive descriptor on receive channel \textcolor{red}{\it{n}} fails any of the two descriptor checks.
    \item \label{int:GDMAINFIFOOVFINT}AHB\_DMA\_INFIFO\_OVF\_CH\regindex{n}\_INT: Triggered when the RX FIFO of GDMA overflows.
    \item \label{int:GDMAINFIFOUDFINT}AHB\_DMA\_INFIFO\_UDF\_CH\regindex{n}\_INT: Triggered when the  RX FIFO of GDMA underflows.
\end{itemize}

The GDMA\_OUT\_CH\regindex{n}\_INTR (\regindex{n} is 0 \verb+~+ 1) signals are generated by the following interrupt sources:

\begin{itemize}
    \item \label{int:GDMAOUTDONEINT}AHB\_DMA\_OUT\_DONE\_CH\regindex{n}\_INT: Triggered when all data corresponding to a transmit descriptor has been sent via transmit channel \textcolor{red}{\it{n}}.
    \item \label{int:GDMAOUTEOFINT}AHB\_DMA\_OUT\_EOF\_CH\regindex{n}\_INT: Triggered when the suc\_eof bit in a transmit descriptor is 1 and data corresponding to this descriptor has been sent via transmit channel \textcolor{red}{\it{n}}. If \hyperref[fielddesc:AHBDMAOUTEOFMODECHN]{AHB\_DMA\_OUT\_EOF\_MODE\_CH\regindex{n}} is 0, this interrupt will be triggered when the last byte of data corresponding to this descriptor enters GDMA's transmit channel; if \hyperref[fielddesc:AHBDMAOUTEOFMODECHN]{AHB\_DMA\_OUT\_EOF\_MODE\_CH\regindex{n}} is 1, this interrupt is triggered when the last byte of data is taken from GDMA's transmit channel.
    \item \label{int:GDMAOUTDSCRERRINT}AHB\_DMA\_OUT\_DSCR\_ERR\_CH\regindex{n}\_INT: Triggered when a transmit descriptor on transmit channel \textcolor{red}{\it{n}} fails any of the two descriptor checks.
    \item \label{int: GDMAOUTTOTALEOFINT}AHB\_DMA\_OUT\_TOTAL\_EOF\_CH\regindex{n}\_INT: Triggered when all data corresponding to a linked list (including multiple descriptors) has been sent via transmit channel \textcolor{red}{\it{n}}.
    \item \label{int:GDMAOUTFIFOOVFINT}AHB\_DMA\_OUTFIFO\_OVF\_CH\regindex{n}\_INT: Triggered when the TX FIFO of GDMA overflows.
    \item \label{int:GDMAOUTFIFOUDFINT}AHB\_DMA\_OUTFIFO\_UDF\_CH\regindex{n}\_INT: Triggered when the TX FIFO of GDMA underflows.
\end{itemize}


\section{Programming Procedures}

The clock gating for GDMA can be configured via \hyperref[fielddesc:PCRGDMACLKEN]{PCR\_GDMA\_CLK\_EN}, and is enabled by default. GDMA can be reset by configuring \hyperref[fielddesc:PCRGDMARSTEN]{PCR\_GDMA\_RST\_EN}.

\subsection{Programming Procedures for GDMA's Transmit Channel}

To transmit data, GDMA's transmit channel should be configured by software as follows:
\begin{enumerate}
    \item Set \hyperref[fielddesc:AHBDMAOUTRSTCHN]{AHB\_DMA\_OUT\_RST\_CH\regindex{n}} first to 1 and then to 0, to reset the state machine of GDMA's transmit channel and FIFO pointer.
    \item Load an outlink, and configure \hyperref[fielddesc:AHBDMAOUTLINKADDRCHN]{AHB\_DMA\_OUTLINK\_ADDR\_CH\regindex{n}}  with address of the first transmit descriptor.
    \item Configure \hyperref[fielddesc:AHBDMAPERIOUTSELCHN]{AHB\_DMA\_PERI\_OUT\_SEL\_CH\regindex{n}} with the value corresponding to the peripheral to be connected, as shown in Table \ref{tab:gdma-ahb-peri-sel}.
    \item Set \hyperref[fielddesc:AHBDMAOUTLINKSTARTCHN]{AHB\_DMA\_OUTLINK\_START\_CH\regindex{n}} to enable GDMA's transmit channel for data transfer.
    \item Configure and enable the corresponding peripheral. See details in the individual chapter for the corresponding peripheral.
    \item Wait for the AHB\_DMA\_OUT\_TOTAL\_EOF\_CH\regindex{n}\_INT interrupt, which indicates the completion of data transfer.
\end{enumerate}

\subsection{Programming Procedures for GDMA's Receive Channel}
To receive data, GDMA's receive channel should be configured by software as follows:
\begin{enumerate}
    \item Set \hyperref[fielddesc:AHBDMAINRSTCHN]{AHB\_DMA\_IN\_RST\_CH\regindex{n}} first to 1 and then to 0, to reset the state machine of GDMA's receive channel and FIFO pointer.
    \item Load an inlink, and configure \hyperref[fielddesc:AHBDMAINLINKADDRCHN]{AHB\_DMA\_INLINK\_ADDR\_CH\regindex{n}} with address of the first receive descriptor.
    \item Configure \hyperref[fielddesc:AHBDMAPERIINSELCHN]{AHB\_DMA\_PERI\_IN\_SEL\_CH\regindex{n}} with the value corresponding to the peripheral to be connected, as shown in Table \ref{tab:gdma-ahb-peri-sel}.
    \item Set \hyperref[fielddesc:AHBDMAINLINKSTARTCHN]{AHB\_DMA\_INLINK\_START\_CH\regindex{n}} to enable GDMA's receive channel for data transfer.
    \item Configure and enable the corresponding peripheral. See details in the individual chapter for the corresponding peripheral.
\end{enumerate}

\subsection{Programming Procedures for Memory-to-Memory Transfer}
To transfer data from one memory location to another, GDMA should be configured by software as follows:
\begin{enumerate}
    \item Set \hyperref[fielddesc:AHBDMAOUTRSTCHN]{AHB\_DMA\_OUT\_RST\_CH\regindex{n}} first to 1 and then to 0, to reset the state machine of GDMA's transmit channel and FIFO pointer.
    \item Set \hyperref[fielddesc:AHBDMAINRSTCHN]{AHB\_DMA\_IN\_RST\_CH\regindex{n}} first to 1 and then to 0, to reset the state machine of GDMA's receive channel and FIFO pointer.
    \item Load an outlink, and configure \hyperref[fielddesc:AHBDMAOUTLINKADDRCHN]{AHB\_DMA\_OUTLINK\_ADDR\_CH\regindex{n}} with address of the first transmit descriptor.
    \item Load an inlink, and configure \hyperref[fielddesc:AHBDMAINLINKADDRCHN]{AHB\_DMA\_INLINK\_ADDR\_CH\regindex{n}} with address of the first receive descriptor.
    \item Set \hyperref[fielddesc:AHBDMAMEMTRANSENCHN]{AHB\_DMA\_MEM\_TRANS\_EN\_CH\regindex{n}} to enable memory-to-memory transfer.
    \item Configure \hyperref[fielddesc:AHBDMAPERIINSELCHN]{AHB\_DMA\_PERI\_IN\_SEL\_CH\regindex{n}} and \hyperref[fielddesc:AHBDMAPERIOUTSELCHN]{AHB\_DMA\_PERI\_OUT\_SEL\_CH\regindex{n}} to the same ``Dummy" value.
    \item Set \hyperref[fielddesc:AHBDMAOUTLINKSTARTCHN]{AHB\_DMA\_OUTLINK\_START\_CH\regindex{n}} to enable GDMA's transmit channel for data transfer.
    \item Set \hyperref[fielddesc:AHBDMAINLINKSTARTCHN]{AHB\_DMA\_INLINK\_START\_CH\regindex{n}} to enable GDMA's receive channel for data transfer.
    \item If the suc\_eof bit is set in a transmit descriptor, an AHB\_DMA\_IN\_SUC\_EOF\_CH\regindex{n}\_INT interrupt will be triggered when the data segment corresponding to this descriptor has been transmitted.
\end{enumerate}

\subsection{Programming Procedures for Channel Priority and Weight}

The priority arbitration can be configured as follows:
\begin{enumerate}
    \item Configure the channel priority for TX and RX respectively via \hyperref[fielddesc:AHBDMATXPRICHN]{AHB\_DMA\_TX\_PRI\_CH\regindex{n}} and \hyperref[fielddesc:AHBDMARXPRICHN]{AHB\_DMA\_RX\_PRI\_CH\regindex{n}}.
\end{enumerate}

The weight arbitration can be configured as follows:
\begin{enumerate}
    \item Configure the time slot for TX and RX respectively via \hyperref[fielddesc:AHBDMAARBTIMEOUTTX]{AHB\_DMA\_ARB\_TIMEOUT\_TX} and \hyperref[fielddesc:AHBDMAARBTIMEOUTRX]{AHB\_DMA\_ARB\_TIMEOUT\_RX}.
    \item Configure the number of tokens for TX and RX respectively via \hyperref[fielddesc:AHBDMATXCHARBWEIGHCHN]{AHB\_DMA\_TX\_CH\_ARB\_WEIGH\_CH\regindex{n}} and \hyperref[fielddesc:AHBDMARXCHARBWEIGHCHN]{AHB\_DMA\_RX\_CH\_ARB\_WEIGH\_CH\regindex{n}}.
    \item Enable weight arbitration optimization for TX and RX respectively by clearing \hyperref[fielddesc:AHBDMATXARBWEIGHOPTDIRCHN]{AHB\_DMA\_TX\_ARB\_WEIGH\_OPT\_DIR\_CH\regindex{n}} and \hyperref[fielddesc:AHBDMARXARBWEIGHOPTDIRCHN]{AHB\_DMA\_RX\_ARB\_WEIGH\_OPT\_DIR\_CH\regindex{n}}.
    \item Enable the arbitration for TX and RX respectively by setting \hyperref[fielddesc:AHBDMAWEIGHTENTX]{AHB\_DMA\_WEIGHT\_EN\_TX} and \hyperref[fielddesc:AHBDMAWEIGHTENRX]{AHB\_DMA\_WEIGHT\_EN\_RX}.
\end{enumerate}
