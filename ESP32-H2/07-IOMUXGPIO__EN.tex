\hypertarget{iomuxgpio}{}
\chapter{IO MUX and GPIO Matrix (GPIO, IO MUX)}
\label{mod:iomuxgpio}

\section{Overview}

The \chipname{} chip features 19 GPIO pins. Each pin can be used as a general-purpose I/O, or be connected to an internal peripheral signal. Through GPIO matrix and IO MUX, peripheral input signals can be from any IO pins, and peripheral output signals can be routed to any IO pins. Together these modules provide highly configurable I/O.

\textbf{Note that the GPIO pins are numbered from GPIO0} \verb+~+ \textbf{GPIO5, GPIO8} \verb+~+ \textbf{GPIO14, and GPIO22} \verb+~+ \textbf{GPIO27.}


\section{Features}

\textbf{GPIO matrix has the following features:}

\begin{itemize}
\item A full-switching matrix between the peripheral input/output signals and the GPIO pins.
\item 78 peripheral input signals sourced from the input of any GPIO pins.
\item 99 peripheral output signals routed to the output of any GPIO pins.

\item Signal synchronization for peripheral inputs based on \textbf{IO MUX operating clock}. For more information about the operating clock of IO MUX, please refer to Section \ref{mod:resclk} \textit{\nameref{mod:resclk}}.

\item GPIO Filter hardware for input signal filtering.
\item Glitch Filter hardware for second time filtering on the input signal.
\item Sigma delta modulated (SDM) output.
\item GPIO simple input and output.
\end{itemize}

\textbf{IO MUX has the following features:}

\begin{itemize}
\item Better high-frequency digital performance achieved by some digital signals (SPI, JTAG, UART) bypassing GPIO matrix. In this case, IO MUX is used to connect these pins directly to peripherals.
\item A configuration register \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO\regindex{n}\_REG} provided for each GPIO pin. The pin can be configured to
    \begin{itemize}
        \item perform GPIO function routed by GPIO matrix;
        \item or perform direct connection bypassing GPIO matrix.
    \end{itemize}

\end{itemize}



\section{Architectural Overview}
Figure \ref{fig:iomux-gpio-matrix-overview-detail} shows in details how GPIO matrix and IO MUX route signals from pins to peripherals, and from peripherals to pins.

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/figure2-over-detailed-20230220-h2.eps}
    \caption{Architecture of IO MUX and GPIO Matrix}
    \label{fig:iomux-gpio-matrix-overview-detail}
\end{figure}


\begin{enumerate}
    \item Only part of peripheral input signals (marked ``yes" in column ``Direct input through IO MUX" in Table \ref{tab:iomuxgpio-periph-signals-via-gpio-matrix}) can bypass GPIO matrix. The other input signals can only be routed to peripherals via GPIO matrix.
    \item There are only 19 inputs from GPIO SYNC to GPIO matrix, since  \chipname{} provides 19 GPIO pins in total. 

    \item GPIO pins are controlled by the signals: IE, OE, WPU, and WPD.
    \item Only part of peripheral outputs (marked ``yes" in column ``Direct output through IO MUX" in Table \ref{tab:iomuxgpio-periph-signals-via-gpio-matrix}) can be routed to pins bypassing GPIO matrix. The other output signals can only be routed to pins via GPIO matrix.
    \item There are 19 outputs (corresponding to GPIO pin  \regindex{X}: 0 \verb+~+ 5, 8 \verb+~+ 14, 22 \verb+~+ 27) from GPIO matrix to IO MUX. 
\end{enumerate}

Figure \ref{fig:pad-overview} shows the internal structure of a pad, which is an electrical interface between the chip logic and the GPIO pin. The structure is applicable to all 19 GPIO pins and can be controlled using IE, OE, WPU, and WPD signals.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.6\textwidth]{\modulefiles/figures/pad-overview_20210815.eps}    \caption{Internal Structure of a Pad}
    \label{fig:pad-overview}
\end{figure}

\begin{itemize}
    \item IE: input enable
    \item OE: output enable
    \item WPU: internal weak pull-up resistor
    \item WPD: internal weak pull-down resistor
    \item Bonding pad: a terminal point of the chip logic used to make a physical connection from the chip die to GPIO pin in the chip package
\end{itemize}

\section{Peripheral Input via GPIO Matrix}
\label{sec:peripheral-input-via-gpio-matrix}

\subsection{Overview}

To receive a peripheral input signal via GPIO matrix, the matrix is configured to source the peripheral input signal from one of the 19 GPIOs (0 \verb+~+ 5, 8 \verb+~+ 14, 22 \verb+~+ 27), see Table  \ref{tab:iomuxgpio-periph-signals-via-gpio-matrix}. Meanwhile, the register corresponding to the peripheral signal should be set to receive input signal via GPIO matrix.

As shown in Figure  \ref{fig:iomux-gpio-matrix-overview-detail}, when GPIO matrix is used to input a signal from the pin, all external input signals are sourced from the GPIO pins and then filtered by the GPIO Filter, as shown in Step  \ref{step:2} in Section \ref{sec:functional-description}.

The Glitch Filter hardware can filter eight of the output signals from the GPIO Filter, and the other unselected signals go directly to the GPIO SYNC hardware, as shown in Step \ref{step:3} in Section \ref{sec:functional-description}.

All signals filtered by the GPIO Filter hardware or the Glitch Filter hardware are synchronized by the GPIO SYNC hardware to IO MUX operating clock and then enter the GPIO matrix, see Section \ref{sec:inputconfig}. Such signal filtering and synchronization features apply to all GPIO matrix signals but do not apply when using the IO MUX.


\subsection{Signal Synchronization}
\label{sec:inputconfig}

\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=1.0\textwidth]{\modulefiles/figures/gpio_input_sync_20220309}}
    \caption{GPIO Input Synchronized on Rising Edge or on Falling Edge of IO MUX Operating Clock}
    \label{Figure:gpio_input_sync}
\end{figure}


Figure \ref{Figure:gpio_input_sync} shows the functionality of GPIO SYNC. In the figure, negative sync and positive sync mean GPIO input is synchronized on falling edge and on rising edge of IO MUX operating clock respectively.

The synchronization function is disabled by default by the synchronizer, i.e., \hyperref[fielddesc:GPIOPINNSYNC1BYPASS]{GPIO\_PIN\regindex{x}\_SYNC1/2\_BYPASS [1:0]} = 0. But when an asynchronous peripheral signal is connected to the pin, the signal should be synchronized by the two-level synchronizer (i.e., the first-level synchronizer and the second-level synchronizer as shown in Figure \ref{Figure:gpio_input_sync}) to lower the probability of causing metastability. For more information, see Step  \ref{step:4} in the following section.

\subsection{Functional Description}
\label{sec:functional-description}

To read GPIO pin \regindex{X}\hyperref[input-note-1]{$^1$} into peripheral signal \regindex{Y}, follow the steps below:

\begin{enumerate}
    \item Configure register \hyperref[regdesc:GPIOFUNCNINSELCFGREG]{GPIO\_FUNC\regindex{y}\_IN\_SEL\_CFG\_REG} corresponding to peripheral signal \regindex{Y} in GPIO matrix:
    \begin{itemize}
        \item Set \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_SIG\regindex{y}\_IN\_SEL} to enable peripheral signal input via GPIO matrix.
        \item Set \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} to the desired GPIO pin, i.e., \regindex{X} here.
    \end{itemize}
    {\bfseries Note that} some peripheral signals have no valid \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_SIG\regindex{y}\_IN\_SEL} bit, namely, these peripherals can only receive input signals via GPIO matrix.
    \item \label{step:2} Optionally enable the GPIO Filter for pin input signals by setting \hyperref[fielddesc:IOMUXGPIONFILTEREN]{IO\_MUX\_GPIO\regindex{x}\_FILTER\_EN}. Only the signals with a valid width of more than two clock cycles can be sampled, see Figure  \ref{Figure:gpio_filter}.
\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/gpio_filter}}
    \caption{GPIO Filter Timing of GPIO Input Signals}
    \label{Figure:gpio_filter}
\end{figure}
\item \label{step:3} Glitch filter hardware supports eight channels, each of which selects one signal from the 19 (0 \verb+~+ 5, 8 \verb+~+ 14, 22 \verb+~+ 27) output signals from the GPIO Filter hardware and conducts the second-time filtering on the selected signal. This Glitch Filter hardware can be used to filter slow-speed signals. To enable this feature, follow the steps below:
\begin{itemize}
    \item Configure \hyperref[fielddesc:GPIOSDFILTERCH0INPUTIONUM]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_INPUT\_IO\_NUM} to \regindex{m}. \regindex{n} (0 \verb+~+ 7) represents the channel number. \regindex{m} (0 \verb+~+ 5, 8 \verb+~+ 14, 22 \verb+~+ 27) represents the GPIO pin number.
    \item Configure \hyperref[fielddesc:GPIOSDFILTERCH0WINDOWWIDTH]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_WINDOW\_WIDTH} to \textbf{VALUE1} and \hyperref[fielddesc:GPIOSDFILTERCH0WINDOWTHRES]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_WINDOW\_THRES} to \textbf{VALUE2}. During \textbf{VALUE1} + 1 cycles, if there are \textbf{VALUE2} + 1 input signals that do not match the current output signal value, the Glitch Filter hardware inverts the output signal. \hyperref[fielddesc:GPIOSDFILTERCH0WINDOWWIDTH]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_WINDOW\_WIDTH} and  \hyperref[fielddesc:GPIOSDFILTERCH0WINDOWTHRES]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_WINDOW\_THRES} can be configured to the same value \textbf{VALUE3}, then only signals with a width greater than \textbf{VALUE3} + 1 clock cycles will be sampled.
    \item Set  \hyperref[fielddesc:GPIOSDFILTERCH0EN]{GPSD\_FILTER\_CH\regindex{n}\_EN} to enable channel \regindex{n}.
\end{itemize}

An example is shown in Figure \ref{Figure:glitch_filter}, where \hyperref[fielddesc:GPIOSDFILTERCH0WINDOWWIDTH]{GPIO\_EXT\_FILTER\_CH\regindex{x}\_WINDOW\_WIDTH} is configured to 3 and \hyperref[fielddesc:GPIOSDFILTERCH0WINDOWTHRES]{GPIO\_EXT\_FILTER\_CH\regindex{x}\_WINDOW\_THRES} to 2. The output signal value (signal\_out) keeps as ``0" in the four clock cycles before T1. The input signal value (signal\_in) has been "1" for three clock cycles in the same period, then the output signal is inverted to ``1" after T1.
\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/figure-glitch-filter-20220630-c6.eps}}
    \caption{Glitch Filter Timing Example}
    \label{Figure:glitch_filter}
\end{figure}
    \item \label{step:4} Synchronize GPIO input signals. To do so, please set \hyperref[regdesc:GPIOPINNREG]{GPIO\_PIN\regindex{x}\_REG} corresponding to GPIO pin \regindex{X} as follows:
    \begin{itemize}
        \item Set \hyperref[fielddesc:GPIOPINNSYNC1BYPASS]{GPIO\_PIN\regindex{x}\_SYNC1\_BYPASS} to enable input signal synchronized on rising edge or on falling edge in the first-level synchronization, see Figure \ref{Figure:gpio_input_sync}.
        \item Set \hyperref[fielddesc:GPIOPINNSYNC2BYPASS]{GPIO\_PIN\regindex{x}\_SYNC2\_BYPASS} to enable input signal synchronized on rising edge or on falling edge in the second-level synchronization, see Figure \ref{Figure:gpio_input_sync}.
    \end{itemize}
    \item Configure IO MUX register to enable pin input. For this end, please set \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO\regindex{x}\_REG} corresponding to GPIO pin \regindex{X} as follows:
    \begin{itemize}
        \item Set \hyperref[fielddesc:IOMUXGPIONFUNIE]{IO\_MUX\_GPIO\regindex{x}\_FUN\_IE} to enable input\hyperref[input-note-2]{$^2$}.
        \item Set or clear \hyperref[fielddesc:IOMUXGPIONFUNWPU]{IO\_MUX\_GPIO\regindex{x}\_FUN\_WPU} and \hyperref[fielddesc:IOMUXGPIONFUNWPD]{IO\_MUX\_GPIO\regindex{x}\_FUN\_WPD} as desired to enable or disable pull-up and pull-down resistors.
    \end{itemize}
\end{enumerate}

For example, to connect I2S MSCK input signal  \hyperref[input-note-3]{$^3$} (I2S\_MCLK\_in, signal index 12) to GPIO3, please follow the steps below. Note that GPIO3 is also named as MTDO pin.

\begin{enumerate}
\item Set \hyperref[regdesc:GPIOFUNCNINSELCFGREG]{GPIO\_SIG12\_IN\_SEL} in register  \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_FUNC12\_IN\_SEL\_CFG\_REG} to enable peripheral signal input via GPIO matrix.
\item Set \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC12\_IN\_SEL} in register \hyperref[regdesc:GPIOFUNCNINSELCFGREG]{GPIO\_FUNC12\_IN\_SEL\_CFG\_REG} to 3, i.e., select GPIO3.
\item Set \hyperref[fielddesc:IOMUXGPIONFUNIE]{IO\_MUX\_GPIO3\_FUN\_IE} in register \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO3\_REG} to enable pin input.
\end{enumerate}

\begin{tiplisting}
\vspace{-2em}
\begin{enumerate}
\item \label{input-note-1} One input pin can be connected to multiple peripheral input signals.
\item \label{input-note-2} The input signal can be inverted by configuring \hyperref[fielddesc:GPIOFUNCNININVSEL]{GPIO\_FUNC\regindex{y}\_IN\_INV\_SEL}.
\item \label{input-note-3} It is possible to have a peripheral read a constantly low or constantly high input value without connecting this input to a pin. This can be done by selecting a special \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} input, instead of a GPIO number:
\begin {itemize}
\item When \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} is set to 0x3C, input signal is always 0.
\item When \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} is set to 0x38, input signal is always 1.
\end{itemize}
\end{enumerate}
\end{tiplisting}

\subsection{Simple GPIO Input}\label{IOMUXSimpleGPIOInput}

GPIO matrix can also be used for simple GPIO input. For this case, the input value of one GPIO pin can be read at any time without routing the GPIO input to any peripherals. \hyperref[regdesc:GPIOINREG]{GPIO\_IN\_REG} holds the input values of each GPIO pin.

To implement simple GPIO input, follow the steps below:
\begin{itemize}
    \item Set  \hyperref[fielddesc:IOMUXGPIONFUNIE]{IO\_MUX\_GPIO\regindex{x}\_FUN\_IE} in register  \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO\regindex{x}\_REG}, to enable pin input.
    \item Read the GPIO input from  \hyperref[regdesc:GPIOINREG]{GPIO\_IN\_REG[\regindex{x}]}.
\end{itemize}

\section{Peripheral Output via GPIO Matrix}
\label{sec:peripheral-output-via-gpio-matrix}

\subsection{Overview}

To output a signal from a peripheral via GPIO matrix, the matrix is configured to route peripheral output signals (only signals with a name assigned in the column ``Output signal" in Table \ref{tab:iomuxgpio-periph-signals-via-gpio-matrix}) to one of the 19 GPIOs (0 \verb+~+ 5, 8 \verb+~+ 14, 22 \verb+~+ 27).


The output signal is routed from the peripheral into GPIO matrix and then into IO MUX. IO MUX must be configured to set the chosen pin to GPIO function. This enables the GPIO output signal to be connected to the pin.

\begin{tiplisting}
There is a range of peripheral output signals (97 \verb+~+ 100 in Table  \ref{tab:iomuxgpio-periph-signals-via-gpio-matrix}) which are not connected to any peripheral, but to the input signals (97 \verb+~+ 100) directly.
\end{tiplisting}
\vspace{-2em}


\subsection{Functional Description}\label{IOMUXPeripheralOutput}

The 99 output signals (signals with a name assigned in the column ``Output signal" in Table  \ref{tab:iomuxgpio-periph-signals-via-gpio-matrix}) can be set to go through GPIO matrix into IO MUX and then to a pin. Figure  \ref{fig:iomux-gpio-matrix-overview-detail} illustrates the configuration.

To output peripheral signal \regindex{Y} to a particular GPIO pin \regindex{X}\hyperref[output-note-1]{$^1$}, follow the steps below:

\begin{enumerate}
    \item Configure registers \hyperref[regdesc:GPIOFUNCNOUTSELCFGREG]{GPIO\_FUNC\regindex{x}\_OUT\_SEL\_CFG\_REG} and  \hyperref[regdesc:GPIOENABLEREG]{GPIO\_ENABLE\_REG}[\regindex{x}] corresponding to GPIO pin \regindex{X} in GPIO matrix. Recommended operation: use corresponding \hyperref[regdesc:GPIOENABLEW1TSREG]{W1TS} (write 1 to set) and \hyperref[regdesc:GPIOENABLEW1TCREG]{W1TC} (write 1 to clear) registers to set or clear \hyperref[regdesc:GPIOENABLEREG]{GPIO\_ENABLE\_REG}.


    \begin{itemize}
        \item Set the \hyperref[fielddesc:GPIOFUNCNOUTSEL]{GPIO\_FUNC\regindex{x}\_OUT\_SEL} field in register \hyperref[regdesc:GPIOFUNCNOUTSELCFGREG]{GPIO\_FUNC\regindex{x}\_OUT\_SEL\_CFG\_REG} to the index of the desired peripheral output signal \regindex{Y}.
        \item If the signal should always be enabled as an output, set the \hyperref[fielddesc:GPIOFUNCNOENSEL]{GPIO\_FUNC\regindex{x}\_OEN\_SEL} bit in register \hyperref[regdesc:GPIOFUNCNOUTSELCFGREG]{GPIO\_FUNC\regindex{x}\_OUT\_SEL\_CFG\_REG} and the bit in register \hyperref[regdesc:GPIOENABLEW1TSREG]{GPIO\_ENABLE\_W1TS\_REG}, corresponding to GPIO pin \regindex{X}. To have the output enable signal decided by internal logic (for example, the SPIQ\_oe in column ``Output enable signal when \hyperref[fielddesc:GPIOFUNCNOENSEL]{GPIO\_FUNC\regindex{n}\_OEN\_SEL} = 0" in Table \ref{tab:iomuxgpio-periph-signals-via-gpio-matrix}), clear the \hyperref[fielddesc:GPIOFUNCNOENSEL]{GPIO\_FUNC\regindex{x}\_OEN\_SEL} bit instead.
       \item Set the corresponding bit in register \hyperref[regdesc:GPIOENABLEW1TCREG]{GPIO\_ENABLE\_W1TC\_REG} to disable the output from the GPIO pin.
    \end{itemize}
    \item For an open drain output, set the \hyperref[fielddesc:GPIOPINNPADDRIVER]{GPIO\_PIN\regindex{x}\_PAD\_DRIVER} bit in register \hyperref[regdesc:GPIOPINNREG]{GPIO\_PIN\regindex{x}\_REG} corresponding to GPIO pin \regindex{X}.
    \item Configure IO MUX register to enable output via GPIO matrix. Set \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO\regindex{x}\_REG} corresponding to GPIO pin \regindex{X} as follows:
    \begin{itemize}
        \item Set the field \hyperref[fielddesc:IOMUXGPIONMCUSEL]{IO\_MUX\_GPIO\regindex{x}\_MCU\_SEL} to desired IO MUX function corresponding to GPIO pin \regindex {X}. This is Function 1 (GPIO function), numeric value 1, for all pins.
        \item Set the \hyperref[fielddesc:IOMUXGPIONFUNDRV]{IO\_MUX\_GPIO\regindex{x}\_FUN\_DRV} field to the desired value for output strength (0 \verb+~+ 3). The higher the drive strength, the more current can be sourced/sunk from the pin.
        \begin{itemize}
            \item 0: \verb+~+5 mA
            \item 1: \verb+~+10 mA
            \item 2: \verb+~+20 mA (default)
            \item 3: \verb+~+40 mA
        \end{itemize}

        \item If using open drain mode, set/clear the \hyperref[fielddesc:IOMUXGPIONFUNWPU]{IO\_MUX\_GPIO\regindex{x}\_FUN\_WPU} and \hyperref[fielddesc:IOMUXGPIONFUNWPD]{IO\_MUX\_GPIO\regindex{x}\_FUN\_WPD} bits to enable/disable the internal pull-up/pull-down resistors.
    \end{itemize}
\end{enumerate}

\begin{tiplisting}
\vspace{-2em}
\begin{enumerate}
\item \label{output-note-1} The output signal from a single peripheral can be sent to multiple pins simultaneously.
\item \label{output-note-2} The output signal can be inverted by setting \hyperref[fielddesc:GPIOFUNCNOUTINVSEL]{GPIO\_FUNC\regindex{x}\_OUT\_INV\_SEL}.
\end{enumerate}
\end{tiplisting}

\subsection{Simple GPIO Output}

GPIO matrix can also be used for simple GPIO output. For this case, one GPIO pin can be configured to directly output the desired value, without routing any peripheral output to this pin. This can be done as below:

\begin{itemize}
    \item Set GPIO matrix \hyperref[fielddesc:GPIOFUNCNOUTSEL]{GPIO\_FUNC\regindex{n}\_OUT\_SEL} with a special peripheral index 128 (0x80);
    \item Set the corresponding bit in \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG} register to the desired GPIO output value.
\end{itemize}

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
\item \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}[0] \verb+~+ \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}[27] correspond to GPIO0 \verb+~+ GPIO27 respectively.  \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}[28] \verb+~+ \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}[31] are invalid.
\item Recommended operation: use \hyperref[fielddesc:GPIOOUTW1TS]{GPIO\_OUT\_W1TS}/\hyperref[fielddesc:GPIOOUTW1TC]{GPIO\_OUT\_W1TC} to set or clear the register \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}.
\end{itemize}
\end{tiplisting}

\subsection{Sigma Delta Modulated Output (SDM)}

\subsubsection{Functional Description}

Four out of the 99 peripheral output signals (index: 83 \verb+~+ 86 in Table \ref{tab:iomuxgpio-periph-signals-via-gpio-matrix} support 1-bit second-order sigma delta modulation. By default the output is enabled for these four channels. This Sigma Delta modulator can also output PDM (pulse density modulation) signal with configurable duty cycle. The transfer function is:

\[
    H(z)=\textrm{X(z)z$^{-1}$} + \textrm{E(z)(1-z$^{-1}$)$^{2}$}
    \]

E(z) is quantization error and X(z) is the input.\\
This modulator supports scaling down of IO MUX operating clock by divider 1 \verb+~+ 256:

\begin{itemize}
    \item Set  \hyperref[fielddesc:GPIOSDFUNCTIONCLKEN]{GPIO\_EXT\_FUNCTION\_CLK\_EN} to enable the modulator clock.
    \item Configure \hyperref[fielddesc:GPIOSDSD0PRESCALE]{GPIO\_EXT\_SD\regindex{n}\_PRESCALE} (\regindex{n} = 0 \verb+~+ 3 for the four channels).
\end{itemize}

After scaling, the clock cycle is equal to one pulse output cycle from the modulator.

\hyperref[fielddesc:GPIOSDSD0IN]{GPIO\_EXT\_SD\regindex{n}\_IN} is a signed number with a range of [-128, 127] and is used to control the duty cycle $^{1}$ of PDM output signal.

\begin{itemize}
    \item \hyperref[fielddesc:GPIOSDSD0IN]{GPIO\_EXT\_SD\regindex{n}\_IN} = -128, the duty cycle of the output signal is \begin{math}0\% \end{math}.
    \item \hyperref[fielddesc:GPIOSDSD0IN]{GPIO\_EXT\_SD\regindex{n}\_IN} = 0, the duty cycle of the output signal is near \begin{math}50\% \end{math}.
    \item \hyperref[fielddesc:GPIOSDSD0IN]{GPIO\_EXT\_SD\regindex{n}\_IN} = 127, the duty cycle of the output signal is near \begin{math}100\% \end{math}.
\end{itemize}

The formula for calculating PDM signal duty cycle is shown as below:

    \[
    Duty\_Cycle=\frac{\hyperref[fielddesc:GPIOSDSD0IN]{GPIO\_EXT\_SD\regindex{n}\_IN} + 128} {256}
    \]

\begin{tiplisting}
For PDM signals, duty cycle refers to the percentage of high level cycles to the whole statistical period (several pulse cycles, for example, 256 pulse cycles).
\end{tiplisting}

\subsubsection{SDM Configuration}

The configuration of SDM is shown below:

\begin{itemize}
    \item Route one of SDM outputs to a pin via GPIO matrix, see Section  \ref{IOMUXPeripheralOutput}.
    \item Enable the modulator clock by setting  \hyperref[fielddesc:GPIOSDFUNCTIONCLKEN]{GPIO\_EXT\_FUNCTION\_CLK\_EN}.
    \item Configure the divider value by setting  \hyperref[fielddesc:GPIOSDSD0PRESCALE]{GPIO\_EXT\_SD\regindex{n}\_PRESCALE}.
    \item Configure the duty cycle of SDM output signal by setting  \hyperref[fielddesc:GPIOSDSD0IN]{GPIO\_EXT\_SD\regindex{n}\_IN}.
\end{itemize}

\section{Direct Input and Output via IO MUX}

\subsection{Overview}

Some digital signals (SPI and JTAG) can bypass GPIO matrix for better high-frequency digital performance. In this case, IO MUX is used to connect these pins directly to peripherals.

This option is less flexible than routing signals via GPIO matrix, as the IO MUX register for each GPIO pin can only select from a limited number of functions, but high-frequency digital performance can be improved.

\subsection{Functional Description}

Two fields must be configured in order to bypass GPIO matrix for peripheral input signals:

\begin{enumerate}
\item \hyperref[fielddesc:IOMUXGPIONMCUSEL]{IO\_MUX\_GPIO\regindex{n}\_MCU\_SEL} for the GPIO pin must be set to the required pin function. For the list of pin functions, please refer to Section  \ref{sec:iomux_pads}.
\item Clear \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_SIG\regindex{n}\_IN\_SEL} to route the input directly to the peripheral.
\end{enumerate}

To bypass GPIO matrix for peripheral output signals, \hyperref[fielddesc:IOMUXGPIONMCUSEL]{IO\_MUX\_GPIO\regindex{n}\_MCU\_SEL} for the GPIO pin must be set to the required pin function.

\begin{tiplisting}
Not all signals can be directly connected to peripheral via IO MUX. Some input/output signals can only be connected to the peripheral via GPIO matrix.
\end{tiplisting}

\section{Pin Hold Feature}\label{PadHold}

Each GPIO pin (including the LP pins: GPIO8 \verb+~+ GPIO14) has an individual hold function controlled by an LP register. When the pin is set to hold, the state is latched at that moment and will not change no matter how the internal signals change or how the IO MUX/GPIO configuration is modified. Users can use the hold function for the pins to retain the pin state through a core reset triggered by watchdog time-out or Deep-sleep events. 

To use this feature, follow the steps below:

\begin{itemize}
\item Digital pins (GPIO0 \verb+~+ GPIO5, GPIO22 \verb+~+ GPIO27):
    \begin{itemize}
        \item To maintain pin input/output status in Deep-sleep mode, users can set LP\_AON\_GPIO\_HOLD0\_REG[\regindex{n}] to 1 before powering down. To disable the hold function after the chip is woken up, users can set LP\_AON\_GPIO\_HOLD0\_REG[\regindex{n}] to 0.
        \item Or users can set PMU\_TIE\_HIGH\_HP\_PAD\_HOLD\_ALL to maintain the input/output status of all digital pins, and set PMU\_TIE\_LOW\_HP\_PAD\_HOLD\_ALL to disable the hold function of all digital pins.
    \end{itemize}
\item LP pins (GPIO8 \verb+~+ GPIO14):
    \begin{itemize}
        \item The input and output values of LP GPIO pins are controlled by LP\_AON\_GPIO\_HOLD0\_REG[\regindex{n}], PMU\_TIE\_HIGH\_LP\_PAD\_HOLD\_ALL, and PMU\_TIE\_LOW\_LP\_PAD\_HOLD\_ALL. Users can set LP\_AON\_GPIO\_HOLD0\_REG[\regindex{n}] to 1 to hold the value of GPIO\regindex{n}, or set LP\_AON\_GPIO\_HOLD0\_REG[\regindex{n}] to 0 to disable the hold function of GPIO\regindex{n}.
        \item Or users can set PMU\_TIE\_HIGH\_LP\_PAD\_HOLD\_ALL to hold the values of all LP pins, and set PMU\_TIE\_LOW\_LP\_PAD\_HOLD\_ALL to disable the hold function of all LP pins.
        \item When the LP pin is held in the input state, it can serve as a wake-up source to wake up the chip from Deep-sleep mode.
    \end{itemize}
\end{itemize}

\section{Hysteresis Characteristics of GPIO Pins}\label{PadHysEn}




Each GPIO pin has hysteresis functionality. When hysteresis is not enabled, as shown in Figure \ref{Figure:hysteresis_fig1}, the level flip of the signal (C) input to the chip from the PAD has only one threshold (Vt, about 1.7 V). When the voltage on the PAD is higher than Vt, the level on the C is high. Otherwise, it is low. However, if the signal on the PAD has noise, it may affect the signal on C.

\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.8\textwidth]{\modulefiles/figures/hysteresis_figure1_20230220.eps}}
    \caption{Example of level flip on the chip pad when the hysteresis function is not enabled}
    \label{Figure:hysteresis_fig1}
\end{figure}

When hysteresis is enabled, as shown in Figure \ref{Figure:hysteresis_fig2}, the level flip of C has two thresholds, high-level threshold (Vth, about 1.7 V) and low-level threshold (Vtl, about 1.4 V). When the voltage of pad goes from low to high, if the voltage is higher than Vth, the level of C is high. When the voltage of PAD goes from high to low, if the voltage is lower than Vtl, the level of C is low. When the voltage of PAD is between Vth and Vtl, the level of C does not change. The hysteresis function can reduce the impact of noise, play an anti-interference role, and reduce the level flip times of C.

To enable the hysteresis function, follow the steps below:




\begin{itemize}
    \item When \hyperref[fielddesc:IOMUXGPIONHYSSEL]{IO\_MUX\_GPIO\regindex{n}\_HYS\_SEL} is 0 (\regindex{n} ranges from 0 \verb+~+ 5, 8 \verb+~+ 14 and 22 \verb+~+ 27, corresponding to GPIO0 \verb+~+ GPIO5, GPIO8 \verb+~+ GPIO14 and GPIO22 \verb+~+ GPIO27):
    \begin{itemize}
        \item The hysteresis function for GPIO0  \verb +~+ GPIO5 is enabled when the corresponding bit in  \hyperref[fielddesc:EFUSEHYSENPAD0]{EFUSE\_HYS\_EN\_PAD0[0:5]} is 1. The hysteresis function for GPIO0  \verb +~+ GPIO5 is disabled when the corresponding bit in  \hyperref[fielddesc:EFUSEHYSENPAD0]{EFUSE\_HYS\_EN\_PAD0[0:5]} is 0.
        \item The hysteresis function for GPIO8 \verb+~+ GPIO14 and GPIO22 \verb+~+ GPIO27 is enabled when the corresponding bit in   \hyperref[fielddesc:EFUSEHYSENPAD1]{EFUSE\_HYS\_EN\_PAD1[2:8]} and \hyperref[fielddesc:EFUSEHYSENPAD1]{EFUSE\_HYS\_EN\_PAD1[16:21]} is 1. The hysteresis function for GPIO8 \verb+~+ GPIO14 and GPIO22 \verb+~+ GPIO27 is disabled when the corresponding bit in   \hyperref[fielddesc:EFUSEHYSENPAD1]{EFUSE\_HYS\_EN\_PAD1[2:8]} and \hyperref[fielddesc:EFUSEHYSENPAD1]{EFUSE\_HYS\_EN\_PAD1[16:21]} is 0.
    \end{itemize}
    \item When \hyperref[fielddesc:IOMUXGPIONHYSSEL]{IO\_MUX\_GPIO\regindex{n}\_HYS\_SEL} is 1 (\regindex{n} ranges from 0 \verb+~+ 5, 8 \verb+~+ 14 and 22 \verb+~+ 27,corresponding to GPIO0 \verb+~+ GPIO5, GPIO8 \verb+~+ GPIO14 and GPIO22 \verb+~+ GPIO27):
    \begin{itemize}
        \item Set \hyperref[fielddesc:IOMUXGPIONHYSEN]{IO\_MUX\_GPIO\regindex{n}\_HYS\_EN} to 1 to enable the hysteresis function for GPIO\regindex{n}.
        \item Set \hyperref[fielddesc:IOMUXGPIONHYSEN]{IO\_MUX\_GPIO\regindex{n}\_HYS\_EN} to 0 to disable the hysteresis function for GPIO\regindex{n}.
    \end{itemize}
\end{itemize}

It is recommended to set \hyperref[fielddesc:IOMUXGPIONHYSSEL]{IO\_MUX\_GPIO\regindex{n}\_HYS\_SEL} to 1, and use \hyperref[fielddesc:IOMUXGPIONHYSEN]{IO\_MUX\_GPIO\regindex{n}\_HYS\_EN} to enable or disable the hysteresis function of GPIO\regindex{n}.

\section{Function of Analog PAD Voltage Comparator}
\label{sec:zcd_pads}




GPIO10 and GPIO11 pads have the function of analog PAD voltage comparator, which can be enabled by setting \hyperref[fielddesc:GPIOSDXPDCOMP]{GPIO\_EXT\_XPD\_COMP} to 1. After enabling the function of analog PAD voltage comparator, when the voltage on GPIO11 pad is higher than the reference voltage, the PAD\_COMP\_OUT signal indicating the comparison result will be high, otherwise it will be low.

Set the value of \hyperref[fielddesc:GPIOSDMODECOMP]{GPIO\_EXT\_MODE\_COMP} as follows:
\begin{itemize}
    \item 0: the reference voltage is (\hyperref[fielddesc:GPIOSDDREFCOMP]{GPIO\_EXT\_DREF\_COMP}*VDDPST2)/10.
    \item 1: the reference voltage is the voltage on GPIO10 PAD.
\end{itemize}

The PAD\_COMP\_OUT signal will be synchronized to the operating clock of the IO MUX to generate the PAD\_COMP\_OUT\_sync signal, which will be used as the interrupt source GPIO\_EXT\_PAD\_COMP\_INT.

Set the value of \hyperref[fielddesc:GPIOSDZERODETMODE]{GPIO\_EXT\_ZERO\_DET\_MODE} as follows:

\begin{itemize}
    \item 0: disable interrupt source generation.
    \item 1, 2: reserved
    \item 3: enable interrupt source and set any edge of PAD\_COMP\_OUT\_sync signal as interrupt source.
\end{itemize}

Meanwhile, after generating an interrupt source, new interrupt sources will be masked within \hyperref[fielddesc:GPIOSDZERODETFILTERCNT]{GPIO\_EXT\_ZERO\_DET\_FILTER\_CNT} IO MUX operating clock cycles.

\section{Event Task Matrix Function}



In \chipname{} , GPIO supports ETM function, that is, the ETM task of GPIO can be triggered by the ETM event of any peripheral, or the ETM task of any peripheral can be triggered by the ETM event of GPIO. For more details about ETM, please refer to Chapter \ref{mod:evntaskmatrix} \textit{\nameref{mod:evntaskmatrix}}. Only ETM tasks and ETM events related to GPIO are introduced here.

The GPIO ETM provides eight task channels \regindex{x} (0 \verb+~+ 7). The ETM tasks that each task channel can receive are:

\begin{itemize}
    \item GPIO\_TASK\_CH\regindex{x}\_SET: GPIO goes high when triggered;
    \item GPIO\_TASK\_CH\regindex{x}\_CLEAR: GPIO goes low when triggered;
    \item GPIO\_TASK\_CH\regindex{x}\_TOGGLE: GPIO toggle level when triggered.
\end{itemize}

Below is an example to configure task channel  \regindex{x} to control GPIO\regindex{y}:

\begin{itemize}
    \item Configure \hyperref[fielddesc:IOMUXGPIONMCUSEL]{IO\_MUX\_GPIO\regindex{y}\_MCU\_SEL} to 1, to select Function 1 listed in Table \ref{table:iomux_pads};
    \item Configure \hyperref[regdesc:GPIOENABLEREG]{GPIO\_ENABLE\_REG[\regindex{y}]} to 1;
    \item Configure \hyperref[fielddesc:GPIOSDETMTASKGPIO0SEL]{GPIO\_EXT\_ETM\_TASK\_GPIO\regindex{y}\_SEL} to \regindex{x};
    \item Set \hyperref[fielddesc:GPIOSDETMTASKGPIO0EN]{GPIO\_EXT\_ETM\_TASK\_GPIO\regindex{y}\_EN}, to enable ETM task channel \regindex{x} to control GPIO\regindex{y}.
\end{itemize}

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
\item One task channel can be selected by one or more GPIOs.

\item When two or three of the signals GPIO\_TASK\_CH\regindex{x}\_SET, GPIO\_TASK\_CH\regindex{x}\_CLEAR, and GPIO\_TASK\_CH\regindex{x}\_\\TOGGLE of the task channel  \regindex{x} selected by GPIO\regindex{y} are valid at the same time, then GPIO\_TASK\_CH\regindex{x}\_SET has the highest priority, GPIO\_TASK\_CH\regindex{x}\_CLEAR takes the second higher priority, and GPIO\_TASK\_CH\regindex{x}\_TOGGLE has the lowest priority.

\item When GPIO\regindex{y} is controlled by ETM task channel, the values of \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}, \hyperref[fielddesc:GPIOFUNCNOUTINVSEL]{GPIO\_FUNC\regindex{n}\_OUT\_INV\_SEL}, and \hyperref[fielddesc:GPIOFUNCNOUTSEL]{GPIO\_FUNC\regindex{n}\_OUT\_SEL} may be modified by the hardware. For such reason, it's recommended to reconfigure these registers when the GPIO is free from the control of ETM task channel.
\end{itemize}
\end{tiplisting}

GPIO has eight event channels, and the ETM events that each event channel can generate are:

\begin{itemize}
    \item GPIO\_EVT\_CH\regindex{x}\_RISE\_EDGE: Indicates that the output signal of the corresponding GPIO filter (see Figure \ref{fig:iomux-gpio-matrix-overview-detail}) has a rising edge;
    \item GPIO\_EVT\_CH\regindex{x}\_FALL\_EDGE: Indicates that the output signal of the corresponding GPIO filter (see Figure \ref{fig:iomux-gpio-matrix-overview-detail}) has a falling edge;
    \item GPIO\_EVT\_CH\regindex{x}\_ANY\_EDGE: Indicates that the output signal of the corresponding GPIO filter (see Figure \ref{fig:iomux-gpio-matrix-overview-detail}) is reversed.
\end{itemize}

The specific configuration of the event channel is as follows:

\begin{itemize}
    \item Set  \hyperref[fielddesc:GPIOSDETMCH0EVENTEN]{GPIO\_EXT\_ETM\_CH\regindex{x}\_EVENT\_EN} to enable event channel \regindex{x} (0 \verb+~+ 7).
    \item Configure \hyperref[fielddesc:GPIOSDETMCH0EVENTSEL]{GPIO\_EXT\_ETM\_CH\regindex{x}\_EVENT\_SEL} to \regindex{y} (0 \verb+~+ 5, 8 \verb+~+ 14 and 22 \verb+~+ 27), i.e., select one from the 19 GPIOs.
\end{itemize}

\begin{tiplisting}
One GPIO can be selected by one or more event channels.
\end{tiplisting}

In specific applications, GPIO ETM events can be used to trigger GPIO ETM tasks. For example, event channel 0 selects GPIO0, GPIO1 selects task channel 0, and the GPIO\_EVT\_CH\regindex{0}\_RISE\_EDGE event is used to trigger the GPIO\_TASK\_CH\regindex{0}\_TOGGLE task. When a square wave signal is input to the chip through GPIO0, the chip outputs a square wave signal with a frequency divided by 2 through GPIO1.
