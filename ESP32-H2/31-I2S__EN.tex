\chapter{I2S Controller (I2S)}
\label{mod:i2s}

\section{Overview}

\chipname{} has a built-in I2S interface, which provide flexible communication interfaces for streaming digital data in multimedia applications, especially digital audio applications.

The I2S standard bus defines three signals, namely, a bit clock signal (BCK), a channel/word select signal (WS), and a serial data signal (SD). A basic I2S data bus has one master and one slave. The roles remain unchanged throughout the communication. The I2S module on \chipname{} provides separate transmit (TX) and receive (RX) units for high performance.

\section{Terminology}

To better illustrate the functionality of I2S, the following terms are used in this chapter. 

\begin{longtable}[c]{ R{6cm} p{10cm} }
    \textbf{Master mode}     & As a master, I2S drives BCK/WS signals and transmits data to or receives data from a slave. \\ 
    \textbf{Slave mode}      & As a slave, I2S is driven by BCK/WS signals and receives data from or transmits data to a master. \\
    \textbf{Full-duplex} & There are two separate data lines. The transmitted and received data is carried simultaneously.\\
    \textbf{Half-duplex} & Only one side, the master or the slave, transmits data first, and the other side receives data. Data transmission and reception cannot occur simultaneously.\\
    \textbf{A-law and $\mu$-law} & A-law and \textrm{$\mu$}-law are compression/decompression algorithms in digital pulse code modulated (PCM) non-uniform quantization, which can effectively improve the signal-to-quantization noise ratio. \\
    \textbf{TDM RX mode} & In this mode, pulse code modulated (PCM) data is received utilizing time division multiplexing (TDM). The signal lines include BCK, WS, and SD. Data from 16 channels at most can be received. TDM Philips standard, TDM MSB alignment standard, and TDM PCM standard are supported in this mode, depending on user configuration.\\
    \textbf{PDM RX mode} & In this mode, pulse density modulation (PDM) data is received. Used signals include WS and DATA. PDM standard is supported in this mode by user configuration.\\
    \textbf{TDM TX mode} & In this mode, pulse code modulated (PCM) data is sent in a way of time division multiplexing (TDM). The signal lines include BCK, WS, and DATA. Data up to 16 channels can be sent. TDM Philips standard, TDM MSB alignment standard, and TDM PCM standard are supported in this mode, depending on user configuration.\\
    \textbf{PDM TX mode} & In this mode, pulse density modulation (PDM) data is sent. The signal lines include WS and DATA. PDM standard is supported in this mode by user configuration.\\
    \textbf{PCM-to-PDM Data Format Converter} & A data format converter that converts PCM data into PDM data (hereinafter referred to as PCM-to-PDM converter), which can be enabled in PDM TX master mode. When the converter is enabled, I2S fetches pulse code modulated (PCM) data from storage via GDMA, converts it to pulse density modulation (PDM) data, and then transmits it.\\
\end{longtable}

\section{Features}

The I2S module has the following features:

\begin{itemize}
    \item Master mode and slave mode
    \item Full-duplex and half-duplex communications
    \item Separate TX and RX units that can work independently or simultaneously
    \item A variety of audio standards supported: 
        \begin{itemize}
            \item TDM Philips standard
            \item TDM MSB alignment standard
            \item TDM PCM standard
            \item PDM standard
        \end{itemize}
        \item Various TX/RX modes supported:
        \begin{itemize}
        \item TDM TX mode, up to 16 channels supported
        \item TDM RX mode, up to 16 channels supported
        \item PDM TX mode
        \begin{itemize}
            \item Raw PDM data transmission
            \item PCM-to-PDM data format conversion, up to 2 channels supported
        \end{itemize}
        \item PDM RX mode
        \begin{itemize}
            \item Raw PDM data reception
        \end{itemize}
        \end{itemize}
    \item Configurable clock source with frequencies up to \iftagged{ESP32-H2, ESP32-H21}{96}{160} MHz
    \item Configurable high-precision sample clock with a variety of sampling frequencies supported (refer to the note below for more details)
    \item 8/16/24/32-bit data width
    \item ETM feature
    \item Direct Memory Access
    \item Standard I2S interface interrupts 
\end{itemize}

\begin{tiplisting}
In slave mode, due to the frequency limitation of the clock source, the maximum sampling frequency of the \chipname{} I2S is limited by the data bit width and the number of channels. For example, sampling frequencies up to \iftagged{ESP32-H2, ESP32-H21}{187.5}{312.5} kHz (such as 8 kHz, 16 kHz, 32 kHz, 44.1 kHz, 48 kHz, 88.2 kHz, 96 kHz, 128 kHz\iftagged{ESP32-H2, ESP32-H21}{}{, and 256 kHz}) are supported in 32-bit two-channel sampling. Please refer to Section \ref{sec:i2s-tx-rx-clk} for detailed information.
\end{tiplisting}

\section{System Architecture}

Figure \ref{fig:i2s-sys-diagram} shows the structure of the \chipname{} I2S module, consisting of:

\begin{itemize}
    \item Transmit control unit (TX Unit)
    \item Receive control unit (RX Unit)
    \item Input and output timing unit (I/O Sync)
    \item Clock divider (Clock Generator)
    \item 64 x 32-bit TX FIFO
    \item 64 x 32-bit RX FIFO
    \item Compress/Decompress units
\end{itemize}

\begin{figure}[H]
    \centering
    \iftagged{ESP32-H2, ESP32-H21}{\fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/i2s_graph_20230614__EN}}}{\fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/i2s_graph_20241230__EN}}}
    \caption{\chipname{} I2S System Diagram}
    \label{fig:i2s-sys-diagram}
\end{figure}

The I2S module supports direct memory access (DMA) to internal memory. For more information, see Chapter \ref{mod:dma} \textit{\nameref{mod:dma}}. 

Both the TX unit and the RX unit have a three-line interface that uses a bit clock line (BCK), a word select line (WS), and a serial data line (SD). The SD line of the TX unit is used for data output and the SD line of the RX unit for data input. The BCK and WS signal lines of the TX and RX units are used for data output in master mode, and the BCK and WS signal lines of the TX and RX units are used for data input in slave mode.

The signal bus of the I2S module is shown at the right part of Figure \ref{fig:i2s-sys-diagram}. The naming of these signals in RX and TX units follows the pattern of I2S\textcolor{red}{A}\_\textcolor{red}{B}\_\textcolor{red}{C}, such as I2S\textcolor{red}{I}\_\textcolor{red}{BCK}\_\textcolor{red}{in}.

\begin{itemize}
    \item ``A" indicates that the signal belongs to the I2S TX or RX unit, which includes:
    \begin{itemize}
        \item ``I": Signal input to/output from the RX unit
        \item ``O": Signal input to/output from the TX unit
    \end{itemize}
    \item ``B" represents the signal function, which includes:
    \begin{itemize}
        \item BCK
        \item WS
        \item SD
    \end{itemize}
    \item ``C" represents the signal direction, which includes:
    \begin{itemize}
        \item ``in": Input signal into the I2S module
        \item ``out": Output signal from the I2S module
    \end{itemize}
\end{itemize}

\section{Supported Audio Standards} \label{sec:i2s-supported-audio-standards}

\chipname{} I2S supports multiple audio standards, including TDM Philips standard, TDM MSB alignment standard, TDM PCM standard, and PDM standard.

Select the needed standard by configuring the following bits:

\begin{itemize}
    \item \hyperref[fielddesc:I2SRXMSBSHIFT]{I2S\_TX/RX\_TDM\_EN}
    \begin{itemize}
        \item 0: Disable TDM mode
        \item 1: Enable TDM mode
    \end{itemize}
    \item \hyperref[fielddesc:I2SRXMSBSHIFT]{I2S\_TX/RX\_PDM\_EN}
    \begin{itemize}
        \item 0: Disable PDM mode
        \item 1: Enable PDM mode
    \end{itemize}    
    \item \hyperref[fielddesc:I2SRXMSBSHIFT]{I2S\_TX/RX\_MSB\_SHIFT}
    \begin{itemize}
        \item 0: WS and SD signals change simultaneously, i.e., enable MSB alignment standard
        \item 1: WS signal changes one BCK clock cycle earlier than SD signal, i.e., enable Philips standard or select PCM standard
    \end{itemize}
\end{itemize}

\begin{tiplisting}
\hyperref[fielddesc:I2SRXMSBSHIFT]{I2S\_TX/RX\_TDM\_EN} and \hyperref[fielddesc:I2SRXMSBSHIFT]{I2S\_TX/RX\_PDM\_EN} must not be configured to 1 or 0 at the same time, otherwise \chipname{} I2S will transmit data incorrectly in a mode that is neither TDM nor PDM.
\end{tiplisting}

\subsection{TDM Philips Standard}

Philips standards require that WS signal changes one BCK clock cycle earlier than SD signal on BCK falling edge, which means that WS signal is valid from one clock cycle before transmitting the first bit of channel data and changes one clock before the end of channel data transfer. SD signal line transmits the most significant bit of audio data first. 

Compared with the basic Philips standard, TDM Philips standard supports multiple channels. See Figure \ref{fig:i2s-tdm-philips-std-timing-diagram}.

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/i2s_tdm_phillips_mode.png}
    \caption{TDM Philips Standard Timing Diagram}
    \label{fig:i2s-tdm-philips-std-timing-diagram}
\end{figure}

\subsection{TDM MSB Alignment Standard}

MSB alignment standards require that WS and SD signals change simultaneously on the falling edge of BCK. The WS signal is valid until the end of the channel data transfer. The SD signal line transmits the most significant bit of audio data first.

Compared with the basic MSB alignment standard, TDM MSB alignment standard supports multiple channels. See Figure \ref{fig:i2s-tdm-msb-align-std-timing-diagram}.

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/i2s_tdm_msb_mode.png}
    \caption{TDM MSB Alignment Standard Timing Diagram}
    \label{fig:i2s-tdm-msb-align-std-timing-diagram}
\end{figure}

\subsection{TDM PCM Standard}

Short frame synchronization under the PCM standards requires that the WS signal changes one BCK clock cycle earlier than the SD signal on the falling edge of BCK, which means that the WS signal becomes valid one clock cycle before transferring the first bit of channel data and remains unchanged in this BCK clock cycle. SD signal line first transmits the most significant bit of audio data. 

Compared with the basic PCM standard, TDM PCM standard supports multiple channels. See Figure \ref{fig:i2s-tdm-pcm-std-timing-diagram}.

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/i2s_tdm_pcm_mode.png}
    \caption{TDM PCM Standard Timing Diagram}
    \label{fig:i2s-tdm-pcm-std-timing-diagram}
\end{figure}

\subsection{PDM Standard}

Under PDM standard, WS signal changes continuously during data transmission. The low-level and high-level of this signal indicates the left channel and right channel respectively. WS and SD signals change simultaneously. See Figure \ref{fig:i2s-pdm-std-timing-diagram}. 

\begin{figure}[H]

\section{I2S TX/RX Clock}\label{sec:i2s-tx-rx-clk}

I2S\_TX/RX\_CLK is the master clock of I2S TX/RX unit, divided from:

\tagged{ESP32-H2, ESP32-H21}
{%
\begin{itemize}
    \item 40 MHz XTAL\_CLK 
    \item 96 MHz PLL\_F96M\_CLK
    \item 64 MHz PLL\_F64M\_CLK
    \item I2S\_MCLK\_in (external input clock)
\end{itemize}
}
\tagged{ESP32-C61}
{%
\begin{itemize}
    \item 40 MHz XTAL\_CLK 
    \item 120 MHz PLL\_F120M\_CLK
    \item 160 MHz PLL\_F160M\_CLK
    \item I2S\_MCLK\_in (external input clock)
\end{itemize}
}

The serial clock (BCK) of the I2S TX/RX unit is divided from I2S\_TX/RX\_CLK, as shown in Figure \ref{fig:i2s-clk}. \hyperref[fielddesc:PCRI2SRXCLKMSEL]{PCR\_I2S\_TX/RX\_CLKM\_SEL} is used to select clock source for TX/RX unit, and \hyperref[fielddesc:PCRI2SRXCLKMEN]{PCR\_I2S\_TX/RX\_CLKM\_EN} to enable or disable the clock source.

\begin{figure}[H]
    \centering
    \iftagged{ESP32-H2, ESP32-H21}{\fbox{\includegraphics[width=0.6\textwidth]{\modulefiles/figures/i2s_clk_20230427}}}{\fbox{\includegraphics[width=0.6\textwidth]{\modulefiles/figures/i2s_clk_20241230}}}
    \caption{I2S Clock Generator}
    \label{fig:i2s-clk}
\end{figure}


The following formula shows the relation between I2S\_TX/RX\_CLK frequency $f_{\textrm{I2S\_TX/RX\_CLK}}$ and the divider clock source frequency $f_{\textrm{I2S\_CLK\_S}}$:
$$f_{\textrm{I2S\_TX/RX\_CLK}}=\frac{f_{\textrm{I2S\_CLK\_S}}}{\textrm{N} + \frac{\textrm{b}}{\textrm{a}}}$$

N is an integer value between 2 and 256. The value of N is mapped to that of \hyperref[fielddesc:PCRI2STXCLKMDIVNUM]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_NUM} as follows:

\begin{itemize}
    \item When \hyperref[fielddesc:PCRI2STXCLKMDIVNUM]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_NUM} = 0, N = 256;
    \item When \hyperref[fielddesc:PCRI2STXCLKMDIVNUM]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_NUM} = 1, N = 2;
    \item When \hyperref[fielddesc:PCRI2STXCLKMDIVNUM]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_NUM} has any other value, N =  \hyperref[fielddesc:PCRI2STXCLKMDIVNUM]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_NUM}.
\end{itemize}

The values of ``a" and ``b" in fractional divider depend only on x, y, z, and yn1. The corresponding formulas are as follows:

\begin{itemize}
    \item When $\textrm{b} <= \frac{\textrm{a}}{2}$, $\textrm{yn1} = 0$, $\textrm{x} = floor([\frac{\textrm{a}}{\textrm{b}}]) - 1$, $\textrm{y} = \textrm{a}\%\textrm{b}$, $\textrm{z} = \textrm{b}$; 
    \item When $\textrm{b} > \frac{\textrm{a}}{2}$, $\textrm{yn1}=1$, $\textrm{x} = floor([\frac{\textrm{a}}{\textrm{a - b}}]) - 1$, $\textrm{y} = \textrm{a}\%\textrm{(a - b)}$, $\textrm{z}=\textrm{a - b}$.
\end{itemize}

The values of x, y, z, and yn1 are configured in \hyperref[fielddesc:PCRI2STXCLKMDIVX]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_X}, \hyperref[fielddesc:PCRI2STXCLKMDIVY]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_Y}, \hyperref[fielddesc:PCRI2STXCLKMDIVZ]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_Z}, and \hyperref[fielddesc:PCRI2STXCLKMDIVYN1]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_YN1}.

To configure the integer divider, clear \hyperref[fielddesc:PCRI2STXCLKMDIVX]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_X} and \hyperref[fielddesc:PCRI2STXCLKMDIVZ]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_Z}, then set \hyperref[fielddesc:PCRI2STXCLKMDIVY]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_Y} to 1.

\begin{tiplisting}
Using fractional divider may introduce some clock jitter.
\end{tiplisting}

In master TX mode, the serial clock BCK for I2S TX unit is I2SO\_BCK\_out divided from I2S\_TX\_CLK, which is:

$$f_{\textrm{I2SO\_BCK\_out}} = \frac{f_{\textrm{I2S\_TX\_CLK}}}{\textrm{MO}}$$

``MO" is an integer value:
$$ \textrm{MO} = \textrm{\hyperref[fielddesc:I2STXBCKDIVNUM]{I2S\_TX\_BCK\_DIV\_NUM}} + 1 $$

\vspace{-1em}
\begin{tiplisting}
Note that \hyperref[fielddesc:I2STXBCKDIVNUM]{I2S\_TX\_BCK\_DIV\_NUM} must not be configured as 1.
\end{tiplisting}

In master RX mode, the serial clock BCK for I2S RX unit is I2SI\_BCK\_out divided from I2S\_RX\_CLK, which is:

$$f_{\textrm{I2SI\_BCK\_out}} = \frac{f_{\textrm{I2S\_RX\_CLK}}}{\textrm{MI}}$$

``MI" is an integer value:
$$ \textrm{MI} = \textrm{\hyperref[fielddesc:I2STXBCKDIVNUM]{I2S\_RX\_BCK\_DIV\_NUM}} + 1 $$

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
    \item \hyperref[fielddesc:I2SRXBCKDIVNUM]{I2S\_RX\_BCK\_DIV\_NUM} must not be configured as 1.
    \item In I2S slave mode, make sure $f_{\textrm{I2S\_TX/RX\_CLK}}$ >= 8 * $f_{\textrm{BCK}}$. The I2S module can output I2S\_MCLK\_out as the master clock for peripherals.
\end{itemize}
\end{tiplisting}

\section{I2S Reset}\label{sec:i2s-reset}

The units and FIFOs in the I2S module are reset by the following bits.

\begin{itemize}
\item I2S TX/RX units: Reset by \hyperref[fielddesc:I2STXRESET]{I2S\_TX\_RESET} and  \hyperref[fielddesc:I2SRXRESET]{I2S\_RX\_RESET};
\item I2S TX/RX FIFO: Reset by \hyperref[fielddesc:I2STXFIFORESET]{I2S\_TX\_FIFO\_RESET} and \hyperref[fielddesc:I2SRXFIFORESET]{I2S\_RX\_FIFO\_RESET}.
\end{itemize}

\begin{tiplisting}
The I2S module clock must be configured first before the module and FIFO are reset.
\end{tiplisting} 

\section{I2S Master/Slave Mode}

The \chipname{} I2S module can operate as a master or a slave in half-duplex and full-duplex communications, depending on the configuration of  \hyperref[fielddesc:I2SRXSLAVEMOD]{I2S\_RX\_SLAVE\_MOD} and  \hyperref[fielddesc:I2STXSLAVEMOD]{I2S\_TX\_SLAVE\_MOD}.

\begin{itemize}
    \item \hyperref[fielddesc:I2STXSLAVEMOD]{I2S\_TX\_SLAVE\_MOD} 
    \begin{itemize}
        \item 0: Master TX mode
        \item 1: Slave TX mode
    \end{itemize}
    \item \hyperref[fielddesc:I2SRXSLAVEMOD]{I2S\_RX\_SLAVE\_MOD}
    \begin{itemize}
        \item 0: Master RX mode
        \item 1: Slave RX mode
    \end{itemize}
\end{itemize}

\subsection{Master/Slave TX Mode} \label{sec:i2s-master-slave-tx-mode}

\begin{itemize}
    \item I2S works as a master transmitter:
    \begin{itemize}
        \item Set \hyperref[fielddesc:I2STXSTART]{I2S\_TX\_START} to start transmitting data. When this bit is set, the TX unit keeps driving the clock signal and serial data.
        \item If \hyperref[fielddesc:I2STXSTOPEN]{I2S\_TX\_STOP\_EN} is set and all the data in FIFO is transmitted, the master stops transmitting data and clock signals.
        \item If \hyperref[fielddesc:I2STXSTOPEN]{I2S\_TX\_STOP\_EN} is cleared and all the data in FIFO is transmitted, meanwhile no new data is filled into FIFO, the TX unit keeps transmitting the last data frame and clock signal.
        \item Master stops transmitting data when \hyperref[fielddesc:I2STXSTART]{I2S\_TX\_START} is cleared.
    \end{itemize}
    \item I2S works as a slave transmitter:
    \begin{itemize}
        \item Set \hyperref[fielddesc:I2STXSTART]{I2S\_TX\_START}. Wait for the master BCK clock to enable a transmit operation.
        \item If \hyperref[fielddesc:I2STXSTOPEN]{I2S\_TX\_STOP\_EN} is set and all the data in FIFO is transmitted, then the slave keeps transmitting zeros, till the master stops providing BCK signal.
        \item If \hyperref[fielddesc:I2STXSTOPEN]{I2S\_TX\_STOP\_EN} is cleared and all the data in FIFO is transmitted, meanwhile no new data is filled into FIFO, the TX unit keeps transmitting the last data frame. 
        \item If \hyperref[fielddesc:I2STXSTART]{I2S\_TX\_START} is cleared, slave keeps transmitting zeros till the master stops providing BCK clock signal.
    \end{itemize}
\end{itemize}

\subsection{Master/Slave RX Mode}\label{sec:i2s-master-slave-rx-mode}

\begin{itemize}
    \item I2S works as a master receiver:
    \begin{itemize}
        \item Set \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} to start receiving data. When this bit is set, the RX unit keeps outputting clock signal and sampling input data.
        \item Configure \hyperref[fielddesc:I2SRXSTOPMODE]{I2S\_RX\_STOP\_MODE} to control the suspension of data reception:
        \begin{itemize}
            \item 0: The RX unit only suspends data reception when \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} is cleared.
            \item 1: The RX unit suspends data reception when \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} is cleared or the number of received bytes is greater than the value configured in \hyperref[regdesc:I2SRXEOFNUMREG]{I2S\_RX\_EOF\_NUM\_REG}. When \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} is not cleared, data reception can be restarted by setting \hyperref[fielddesc:I2SRXRESET]{I2S\_RX\_RESET} to 1.
            \item 2: The RX unit suspends data reception when \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} is cleared or GDMA RX FIFO is full. When \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} is not cleared and the GDMA RX FIFO is no longer full, data reception can be restarted by setting \hyperref[fielddesc:I2SRXRESET]{I2S\_RX\_RESET} to 1.
        \end{itemize}
        \item RX unit stops receiving data when \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} is cleared.
    \end{itemize}
    \item I2S works as a slave receiver:
    \begin{itemize}
        \item Set \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START}. Wait for the master BCK signal to start receiving data.
        \item Configure \hyperref[fielddesc:I2SRXSTOPMODE]{I2S\_RX\_STOP\_MODE} to control data reception suspension:
        \begin{itemize}
            \item 0: The RX unit only suspends data reception when \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} is cleared.
            \item 1: The RX unit suspends data reception when the number of received bytes is greater than the value configured in \hyperref[regdesc:I2SRXEOFNUMREG]{I2S\_RX\_EOF\_NUM\_REG}. When \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} is not cleared, data reception can be restarted by setting \hyperref[fielddesc:I2SRXRESET]{I2S\_RX\_RESET} to 1.
            \item 2: The RX unit suspends data reception when \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} is cleared or GDMA RX FIFO is full. When \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} is not cleared and the GDMA RX FIFO is no longer full, data reception can be restarted by setting \hyperref[fielddesc:I2SRXRESET]{I2S\_RX\_RESET} to 1.
        \end{itemize}
        \item The RX unit stops receiving data when \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} is cleared.
    \end{itemize}
\end{itemize}

\section{Transmitting Data}\label{sec:i2s-tx-data}

\vspace{1em}
\begin{tiplisting}
Updating the configuration described in this and subsequent sections requires to set \hyperref[fielddesc:I2STXUPDATE]{I2S\_TX\_UPDATE} accordingly to synchronize registers from APB clock domain to TX clock domain. For more detailed configurations, see Section \ref{sec:i2s-configure-i2s-tx-mode}.
\end{tiplisting}

\vspace{-1em}
In TX mode, I2S first reads data through GDMA and transmits these data out via output signals according to the configured data mode and channel mode.
\subsection{Data Format Control}\label{sec:i2s-data-control}
Data format is controlled in the following phases:
\begin{itemize}
    \item Phase I: Read data from memory and write it to TX FIFO;
    \item Phase II: Read the TX data from TX FIFO and convert the data according to the output data mode;
    \item Phase III: Clock out the TX data serially.
\end{itemize}

\subsubsection{Bit Width Control of Channel Valid Data}

The bit width of the valid data in each channel is determined by \hyperref[fielddesc:I2STXBITSMOD]{I2S\_TX\_BITS\_MOD} and \hyperref[fielddesc:I2STX24FILLEN]{I2S\_TX\_24\_FILL\_EN}. For details, see the table below.

\end{tiplisting}

\subsubsection{A-law/$\mu$-law Compression and Decompression}

\chipname{} I2S compresses/decompresses the valid data into 32-bit by A-law or by $\mu$-law. If the bit width of valid data is smaller than 32, zeros are filled to the extra high bits of the data to be compressed/decompressed by default.

\begin{tiplisting}
Extra high bits here mean the bits[31: channel valid data width] of the data to be compressed/decompressed.
\end{tiplisting}

Configure \hyperref[fielddesc:I2STXPCMBYPASS]{I2S\_TX\_PCM\_BYPASS}:
\begin{itemize}
\item 0: Compress or decompress the data
\item 1: Do not compress or decompress the data
\end{itemize}
Configure \hyperref[fielddesc:I2STXPCMCONF]{I2S\_TX\_PCM\_CONF}: 
\begin{itemize}
\item 0: Decompress the data using A-law
\item 1: Compress the data using A-law
\item 2: Decompress the data using $\mu$-law
\item 3: Compress the data using $\mu$-law
\end{itemize}

At this point, the first phase of data format control is completed.

\subsubsection{Bit Width Control of Channel TX Data}\label{sec:i2s-tdm-chan-bits}

The TX data width in each channel is determined by \hyperref[fielddesc:I2STXTDMCHANBITS]{I2S\_TX\_TDM\_CHAN\_BITS}.

\begin{itemize}
\item If TX data width in each channel is larger than the valid data width, zeros will be filled to these extra bits. Configure  \hyperref[fielddesc:I2STXLEFTALIGN]{I2S\_TX\_LEFT\_ALIGN}:
\begin{itemize}
\item 0: The valid data is at the lower bits of TX data. Zeros are filled into higher bits of TX data;
\item 1: The valid data is at the higher bits of TX data. Zeros are filled into lower bits of TX data.
\end{itemize}
\item If the TX data width in each channel is smaller than the valid data width, only the lower bits of valid data are sent out, and the higher bits are discarded.
\end{itemize}
At this point, the second phase of data format control is completed.

\subsubsection{Bit Order Control of Channel Data}

The data bit order in each channel is controlled by \hyperref[fielddesc:I2STXBITORDER]{I2S\_TX\_BIT\_ORDER}:
\begin{itemize}
\item 0: Not reverse the valid data bit order;
\item 1: Reverse the valid data bit order.
\end{itemize}

At this point, the data format control is completed. The data after format control will be sent sequentially from high to low. Figure \ref{fig:i2s-tx-data-format-control} shows the complete process of TX data format control.

\subsection{Channel Mode Control}

\chipname{} I2S supports both TDM TX mode and PDM TX mode. Set \hyperref[fielddesc:I2STXTDMEN]{I2S\_TX\_TDM\_EN} to enable TDM TX mode, or set \hyperref[fielddesc:I2STXPDMEN]{I2S\_TX\_PDM\_EN} to enable PDM TX mode.

\vspace{-1em}
\begin{tiplisting}
\hyperref[fielddesc:I2STXTDMEN]{I2S\_TX\_TDM\_EN} and \hyperref[fielddesc:I2STXPDMEN]{I2S\_TX\_PDM\_EN} must not be cleared or set simultaneously. 
\end{tiplisting}
\vspace{-1em}

\subsubsection{TDM TX Mode}\label{sec:i2s-chan-control-tdm-mode} 

In TDM TX mode, I2S supports up to 16 channels to transmit data. The total number of TX channels in use is controlled by \hyperref[fielddesc:I2STXTDMTOTCHANNUM]{I2S\_TX\_TDM\_TOT\_CHAN\_NUM}. For example, If \hyperref[fielddesc:I2STXTDMTOTCHANNUM]{I2S\_TX\_TDM\_TOT\_CHAN\_NUM} is set to 5, six channels in total (channel 0-5) will be used to transmit data. See Figure \ref{fig:i2s-tdm-chan-control}.

\vspace{-1em}
\begin{tiplisting}
Most stereo I2S codecs can be controlled by setting the I2S module into 2-channel mode under TDM standard.
\end{tiplisting}

In these TX channels, if \hyperref[fielddesc:I2STXTDMCHAN0EN]{I2S\_TX\_TDM\_CHAN{\textcolor{red}{n}}\_EN} is set to:

\begin{itemize}
    \item 1: This channel transmits the channel data out;
    \item 0: The TX data to be sent by this channel is controlled by \hyperref[fielddesc:I2STXCHANEQUAL]{I2S\_TX\_CHAN\_EQUAL}:
    \begin{itemize}
        \item 1: The data of the previous channel is sent out;
        \item 0: The data stored in \hyperref[fielddesc:I2SSINGLEDATA]{I2S\_SINGLE\_DATA} is sent out.
    \end{itemize}
\end{itemize}

In TDM TX master mode, WS signal is controlled by \hyperref[fielddesc:I2STXWSIDLEPOL]{I2S\_TX\_WS\_IDLE\_POL} and \hyperref[fielddesc:I2STXTDMWSWIDTH]{I2S\_TX\_TDM\_WS\_WIDTH}:
\begin{itemize}
    \item \hyperref[fielddesc:I2STXWSIDLEPOL]{I2S\_TX\_WS\_IDLE\_POL}: The default level of WS signal;
    \item \hyperref[fielddesc:I2STXTDMWSWIDTH]{I2S\_TX\_TDM\_WS\_WIDTH}: The cycles the WS default level lasts for when transmitting all channel data.
\end{itemize}

\hyperref[fielddesc:I2STXHALFSAMPLEBITS]{I2S\_TX\_HALF\_SAMPLE\_BITS} x 2 is equal to the BCK cycles in one WS period.

\textbf{TDM Channel Configuration Example}

In this example, the register configuration is as follows:
\begin{itemize}
    \item I2S\_TX\_TDM\_CHAN\_NUM = 5, i.e., channel 0-5 are used to transmit data.
    \item I2S\_TX\_CHAN\_EQUAL = 1, i.e., data of the previous channel will be transmitted if the I2S\_TX\_TDM\_CHAN\regindex{n}\_EN (\regindex{n} = 0-5) is cleared.
    \item I2S\_TX\_TDM\_CHAN0/2/5\_EN = 1, i.e., these channels transmit their channel data out.
    \item I2S\_TX\_TDM\_CHAN1/3/4\_EN = 0, i.e., these channels transmit the previous channel's data out.
\end{itemize}

\section{Receiving Data}\label{sec:i2s-rx-data}


In RX mode, I2S first reads data from the peripheral interface and then stores the data in memory via GDMA according to the configured channel mode and data mode.

\subsection{Channel Mode Control}

\chipname{} I2S supports both TDM RX mode and PDM RX mode. Set \hyperref[fielddesc:I2SRXTDMEN]{I2S\_RX\_TDM\_EN} to enable TDM RX mode, or set \hyperref[fielddesc:I2SRXPDMEN]{I2S\_RX\_PDM\_EN} to enable PDM RX mode.

\begin{tiplisting}
\hyperref[fielddesc:I2SRXTDMEN]{I2S\_RX\_TDM\_EN} and \hyperref[fielddesc:I2SRXPDMEN]{I2S\_RX\_PDM\_EN} must not be cleared or set simultaneously.
\end{tiplisting}
\vspace{-1em}

\subsubsection{TDM RX Mode}

In TDM RX mode, I2S supports up to 16 channels to input data. The total number of RX channels in use is controlled by \hyperref[fielddesc:I2SRXTDMTOTCHANNUM]{I2S\_RX\_TDM\_TOT\_CHAN\_NUM}. For example, if \hyperref[fielddesc:I2SRXTDMTOTCHANNUM]{I2S\_RX\_TDM\_TOT\_CHAN\_NUM} is set to 5, channel 0-5 will be used to receive data.

In these RX channels, if \hyperref[fielddesc:I2STXTDMCHAN0EN]{I2S\_RX\_TDM\_CHAN{\textcolor{red}{n}}\_EN} is set to:
\begin{itemize}
    \item 1: The channel data is valid and will be stored into RX FIFO;
    \item 0: The channel data is invalid and will not be stored into RX FIFO.
\end{itemize}
In TDM master mode, WS signal is controlled by \hyperref[fielddesc:I2SRXWSIDLEPOL]{I2S\_RX\_WS\_IDLE\_POL} and \hyperref[fielddesc:I2SRXTDMWSWIDTH]{I2S\_RX\_TDM\_WS\_WIDTH}.
\begin{itemize}
    \item \hyperref[fielddesc:I2SRXWSIDLEPOL]{I2S\_RX\_WS\_IDLE\_POL}: The default level of WS signal;
    \item \hyperref[fielddesc:I2SRXTDMWSWIDTH]{I2S\_RX\_TDM\_WS\_WIDTH}: The cycles the WS default level lasts for when receiving all channel data.
\end{itemize}

\hyperref[fielddesc:I2SRXHALFSAMPLEBITS]{I2S\_RX\_HALF\_SAMPLE\_BITS} x 2 is equal to the BCK cycles in one WS period.

\subsubsection{PDM RX Mode} 

In PDM RX mode, I2S supports PDM raw data reception. It converts the serial data from channels to the data to be entered into memory.

In PDM RX master mode, the default level of the WS signal is controlled by \hyperref[fielddesc:I2SRXWSIDLEPOL]{I2S\_RX\_WS\_IDLE\_POL}. WS frequency is half of the BCK frequency. The configuration of the BCK signal is similar to that of WS signal as described in Section \ref{sec:i2s-tx-rx-clk}. Note, in PDM RX mode, the value of \hyperref[fielddesc:I2SRXHALFSAMPLEBITS]{I2S\_RX\_HALF\_SAMPLE\_BITS} must be same as that of \hyperref[fielddesc:I2SRXBITSMOD]{I2S\_RX\_BITS\_MOD}.

\subsection{Data Format Control}

The data format is controlled in the following phases:

\begin{itemize}
    \item Phase I: Serial input data is converted into the data to be saved to RX FIFO;
    \item Phase II: The data is read from RX FIFO and converted according to the input data mode.
\end{itemize}

\subsubsection{Bit Order Control of Channel Data}

The channel data will be stored as the data to be input in order from high to low. The data bit order in each channel is controlled by  \hyperref[fielddesc:I2SRXBITORDER]{I2S\_RX\_BIT\_ORDER}:
\begin{itemize}
\item 0: The bit order of the data to be input is not reversed;
\item 1: The bit order of the data to be input is reversed.
\end{itemize}
At this point, the first phase of data format control is completed. The data to be input after bit order control is stored in the RX FIFO.

\subsubsection{Bit Width Control of Channel Storage (Valid) Data}

The storage data width in each channel is controlled by \hyperref[fielddesc:I2SRXBITSMOD]{I2S\_RX\_BITS\_MOD} and \hyperref[fielddesc:I2SRX24FILLEN]{I2S\_RX\_24\_FILL\_EN}. See the table below.

\begin{table}[H]
    \centering
    \caption{Channel Storage Data Width}
    \label{tab:i2s-chan-storage-data-width}
    \begin{tabular}{|p{3cm}|R{3.3cm}|R{4cm}|}

\subsubsection{Bit Width Control of Channel RX Data}

The RX data width in each channel is determined by \hyperref[fielddesc:I2SRXTDMCHANBITS]{I2S\_RX\_TDM\_CHAN\_BITS}.
\begin{itemize}
\item If the storage data width in each channel is smaller than the received (RX) data width, then only the bits within the storage data width is saved into memory. Configure \hyperref[fielddesc:I2SRXLEFTALIGN]{I2S\_RX\_LEFT\_ALIGN} to:
\begin{itemize}
\item 0: Only the lower bits of the received data within the storage data width is stored to memory;
\item 1: Only the higher bits of the received data within the storage data width is stored to memory.
\end{itemize}
\item If the received data width is smaller than the storage data width in each channel, the higher bits of the received data will be filled with zeros and then the data is saved to memory.
\end{itemize}

The received data is then converted into storage data (to be stored to memory) after some processing, such as discarding extra bits or filling zeros in missing bits. \subsubsection{A-law/$\mu$-law Compression and Decompression} 

\chipname{} I2S compresses/decompresses the storage data in 32-bit by A-law or by $\mu$-law. By default, zeros are filled into high bits.

Configure \hyperref[fielddesc:I2SRXPCMBYPASS]{I2S\_RX\_PCM\_BYPASS}:
\begin{itemize}
\item 0: Compress or decompress the data
\item 1: Do not compress or decompress the data
\end{itemize}
Configure \hyperref[fielddesc:I2SRXPCMCONF]{I2S\_RX\_PCM\_CONF}:
\begin{itemize}
\item 0: Decompress the data using A-law
\item 1: Compress the data using A-law
\item 2: Decompress the data using $\mu$-law
\item 3: Compress the data using $\mu$-law
\end{itemize}
At this point, the data format control is completed. Data then is stored into memory via GDMA.

\section{Event Task Matrix Feature}\label{sec:i2s-etm-func}

\chipname{} I2S supports the Event Task Matrix (ETM) function, which allows I2S's ETM tasks to be triggered by any peripherals' ETM events, or I2S's ETM events to trigger any peripherals' ETM tasks. This section introduces the ETM tasks and events related to I2S. For more information, please refer to Chapter \ref{mod:evntaskmatrix} \nameref{mod:evntaskmatrix}.

I2S can receive the following ETM tasks:

\begin{itemize}
    \item \label{task:I2STASKSTARTTX}\iftagged{ESP32-C61}{I2S0\_TASK\_START\_TX}{I2S\_TASK\_START\_TX}: Enables I2S TX for data transfer.
    \item \label{task:I2STASKSTARTRX}\iftagged{ESP32-C61}{I2S0\_TASK\_START\_RX}{I2S\_TASK\_START\_RX}: Enables I2S RX for data transfer.
    \item \label{task:I2STASKSTOPTX}\iftagged{ESP32-C61}{I2S0\_TASK\_STOP\_TX}{I2S\_TASK\_STOP\_TX}: Stops I2S TX data transfer.
    \item \label{task:I2STASKSTOPRX}\iftagged{ESP32-C61}{I2S0\_TASK\_STOP\_RX}{I2S\_TASK\_STOP\_RX}: Stops I2S RX data transfer.
\end{itemize}

I2S can generate the following ETM events:

\begin{itemize}
    \item \label{evt:I2SEVTTXDONE}\iftagged{ESP32-C61}{I2S0\_EVT\_TX\_DONE}{I2S\_EVT\_TX\_DONE}: Indicates that I2S TX has completed data transmission. Triggered when all data in the TX FIFO has been sent.
    \item \label{evt:I2SEVTRXDONE}\iftagged{ESP32-C61}{I2S0\_EVT\_RX\_DONE}{I2S\_EVT\_RX\_DONE}: Can be triggered in different ways depending on the configured value of \hyperref[fielddesc:I2SRXSTOPMODE]{I2S\_RX\_STOP\_MODE}:
    \begin{itemize}
        \item 0: Will not be triggered;
        \item 1: When triggered, indicates that the number of bytes received by I2S RX is greater than the receive length value configured by \hyperref[regdesc:I2SRXEOFNUMREG]{I2S\_RX\_EOF\_NUM\_REG};
        \item 2: When triggered, indicates that the GDMA RX FIFO is full.
    \end{itemize}
    \item \label{evt:I2SEVTXWORDSSENT}\iftagged{ESP32-C61}{I2S0\_EVT\_X\_WORDS\_SENT}{I2S\_EVT\_X\_WORDS\_SENT}: Indicates that the word number sent by I2S TX is equal to or larger than the value set by \hyperref[fielddesc:I2SETMTXSENDWORDNUM]{I2S\_ETM\_TX\_SEND\_WORD\_NUM}.
    \item \label{evt:I2SEVTXWORDSRECEIVED}\iftagged{ESP32-C61}{I2S0\_EVT\_X\_WORDS\_RECEIVED}{I2S\_EVT\_X\_WORDS\_RECEIVED}: Indicates that the word number received by I2S RX is equal to or larger than the value set by \hyperref[fielddesc:I2SETMRXRECEIVEWORDNUM]{I2S\_ETM\_RX\_RECEIVE\_WORD\_NUM}.
\end{itemize}

In practical applications, I2S's ETM events can trigger its own ETM tasks. For example, the \iftagged{ESP32-C61}{I2S0\_EVT\_X\_WORDS\_SENT}{I2S\_EVT\_X\_WORDS\_SENT} event can trigger the \iftagged{ESP32-C61}{I2S0\_TASK\_STOP\_TX}{I2S\_TASK\_STOP\_TX} task, and in this way stop the I2S operation through ETM.


\section{I2S Interrupts} \label{sec:i2s-interrupts}

\chipname{}'s I2S can generate the I2S\_INTR interrupt signal that will be sent to the \hyperref[mod:intmtrx]{Interrupt Matrix}. There are several internal interrupt sources from I2S  that can generate the above interrupt signal(s) as follows:

\begin{itemize}
\item \label{int:I2STXHUNGINT}I2S\_TX\_HUNG\_INT: Triggered when data transmission is timed out. For example, if the I2S module is configured as TX slave mode, but the master does not provide BCK or WS signals for a long time (specified in \hyperref[regdesc:I2SLCHUNGCONFREG]{I2S\_LC\_HUNG\_CONF\_REG}), this interrupt will be triggered.
\item \label{int:I2SRXHUNGINT}I2S\_RX\_HUNG\_INT: Triggered when the data reception is timed out. For example, if the I2S module is configured as RX slave mode, but the master does not transmit data for a long time (specified in \hyperref[regdesc:I2SLCHUNGCONFREG]{I2S\_LC\_HUNG\_CONF\_REG}), this interrupt will be triggered.
\item \label{int:I2STXDONEINT}I2S\_TX\_DONE\_INT: Triggered when sending data is complete, i.e., all data in I2S TX FIFO has been sent.
\item \label{int:I2SRXDONEINT}I2S\_RX\_DONE\_INT: Triggered when the data receiving is completed.
\end{itemize}

\begin{tiplisting}
For definitions of \textit{interrupt}, \textit{interrupt signal}, \textit{interrupt source}, and their correlations, please refer to Chapter \ref{mod:intmtrx} \textit{\nameref{mod:intmtrx}} > Section \ref{sec:intmtx-int-terminology} \textit{\nameref{sec:intmtx-int-terminology}}.
\end{tiplisting}

Each interrupt source can be configured by a common set of registers that are described in Section \hyperref[interrupt-config-registers]{\textit{Interrupt Configuration Registers}}. The specific registers can be found in Section \ref{sec:i2s-reg-summ} \textit{\nameref{sec:i2s-reg-summ}}.

\section{Software Configuration Process}
\subsection{Configure I2S as TX Mode}\label{sec:i2s-configure-i2s-tx-mode}
Follow the steps below to configure I2S as TX mode via software:

\begin{enumerate}
    \item Configure the clock as described in Section \ref{sec:i2s-tx-rx-clk}.
    \item Configure signal pins according to Table \ref{tab:i2s-signal-descr}.
    \item Select the mode needed by configuring \hyperref[fielddesc:I2STXSLAVEMOD]{I2S\_TX\_SLAVE\_MOD}.
    \begin{itemize}
        \item 0: Master TX mode
        \item 1: Slave TX mode
    \end{itemize}
    \item Set needed TX data mode and TX channel mode as described in Section \ref{sec:i2s-tx-data}, and then set \hyperref[fielddesc:I2STXUPDATE]{I2S\_TX\_UPDATE}.
    \item Reset TX unit and TX FIFO as described in Section \ref{sec:i2s-reset}.
    \item Enable corresponding interrupts. See Section \ref{sec:i2s-interrupts}.\label{other:i2s-item-tx-enable-corresponding-interrupts}
    \item Configure DMA outlink. 
    \item Set \hyperref[fielddesc:I2STXSTOPEN]{I2S\_TX\_STOP\_EN} if needed. For more information, please refer to Section \ref{sec:i2s-master-slave-tx-mode}. 
    \item Start transmitting data:
    \begin{itemize}
        \item In master mode, wait till I2S slave gets ready, then set \hyperref[fielddesc:I2STXSTART]{I2S\_TX\_START} to start transmitting data;
        \item In slave mode, set \hyperref[fielddesc:I2STXSTART]{I2S\_TX\_START}. When the I2S master supplies BCK and WS signals, I2S slave starts transmitting data.
    \end{itemize}
    \item Wait for the interrupt signals set in Step \ref{other:i2s-item-tx-enable-corresponding-interrupts}, or check whether the transfer is completed by querying \hyperref[fielddesc:I2STXIDLE]{I2S\_TX\_IDLE}:
    \begin{itemize}
        \item 0: Transmitter is working;
        \item 1: Transmitter is in idle state.
    \end{itemize}
    \item Clear \hyperref[fielddesc:I2STXSTART]{I2S\_TX\_START} to stop data transfer.
\end{enumerate}
\subsection{Configure I2S as RX Mode}\label{sec:i2s-configure-i2s-rx-mode}
Follow the steps below to configure I2S as RX mode via software:

\begin{enumerate}
    \item Configure the clock as described in Section \ref{sec:i2s-tx-rx-clk}.
    \item Configure signal pins according to Table \ref{tab:i2s-signal-descr}.
    \item Select the mode needed by configuring \hyperref[fielddesc:I2SRXSLAVEMOD]{I2S\_RX\_SLAVE\_MOD}.
    \begin{itemize}
        \item 0: Master RX mode
        \item 1: Slave RX mode
    \end{itemize}
    \item Set needed RX data mode and RX channel mode as described in Section \ref{sec:i2s-rx-data}, and then set \hyperref[fielddesc:I2SRXUPDATE]{I2S\_RX\_UPDATE}.
    \item Reset the RX unit and its FIFO according to Section \ref{sec:i2s-reset}.
    \item Enable the corresponding interrupts. See Section \ref{sec:i2s-interrupts}.\label{other:i2s-item-rx-enable-corresponding-interrupts}
    \item Configure GDMA inlink and set the length of RX data by configuring \hyperref[regdesc:I2SRXEOFNUMREG]{I2S\_RX\_EOF\_NUM\_REG}.
    \item Start receiving data:
    \begin{itemize}
        \item In master mode, when the slave is ready, set \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} to start receiving data.
        \item In slave mode, set  \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} to start receiving data when BCK and WS signals are received from the master.
    \end{itemize}
    \item The received data is then stored to the specified memory address according to the configuration of GDMA. Then the corresponding interrupt set in step \ref{other:i2s-item-rx-enable-corresponding-interrupts} is generated.
\end{enumerate}
