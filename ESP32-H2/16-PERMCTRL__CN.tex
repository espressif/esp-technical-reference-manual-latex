\chapter{权限控制 (PMS)}
\label{mod:permctrl}

\section{概述}


\chipname{} 整个系统的权限管理可以分为两部分：PMP（Physical Memory Protection，物理存储器保护）和 APM（Access Permission Management，访问权限管理）。

PMP 和 APM 管控的区域分布如表 \ref{tab:pmp-apm} 所示。

对于主机 CPU，PMP 与 APM 的管控关系如图 \ref{fig:pmp_apm} 所示。PMP 能够管控 CPU 访问所有的地址空间。APM 不能管控 CPU 访问 ROM、HP SRAM 和 EX\_MEM。主机 CPU 如果需要访问 ROM、HP SRAM 和 EX\_MEM 数据通路，只需要受 PMP 管控；如果要访问 LP SRAM 等其他数据通路，则需要先经过 PMP 的权限管控再经过 APM 的权限管控，如果 PMP 检查不通过，则不会触发 APM 权限管控。

PMP 相关的寄存器位于 HP CPU 内部，需要通过特殊指令进行获取或配置。关于如何配置 PMP，请参考章节 \textit{\nameref{mod:riscvcpu}} > \textit{\nameref{sec:riscvcpu-pmp}}。以下章节将详细介绍 APM 模块的功能和配置。


APM 权限管控模块包含两部分：TEE（Trusted Execution Environment，可信执行环境）控制器和 APM 控制器。他们分别包含自己的寄存器模块：TEE 寄存器和 APM 寄存器。


\begin{itemize}
    \item TEE 控制器负责配置 \chipname{} 中主机（如 GDMA）访问内存或外设寄存器的安全模式，共支持四种安全模式：TEE、REE0 (Rich Execution Environment)、REE1 和 REE2。
    \item APM 控制器负责管理主机访问内存和外设寄存器的权限。通过将预先配置好的地址范围及每个地址范围对应的读、写以及执行权限，与总线上携带的信息，如 ID 号（具体可参考表 \ref{tab:master_source})、安全模式、访问地址、读写执行等信息做对比，从而判断是否允许访问。
\end{itemize}

TEE 寄存器用于配置每个主机的安全模式，APM 寄存器用于指定每一个安全模式的访问权限和访问地址范围。通过 TEE 控制器和 APM 控制器的配合，可以精确控制所有主机对存储器和外设寄存器的访问权限。


\section{主要特性}\label{sec:permctrl-features}

\chipname{} 的 TEE 控制器具有以下特性：

\begin{itemize}
    \item 支持 4 种安全模式
    \item 支持 32 个主机安全模式配置

\end{itemize}

\chipname{} 的 APM 控制器具有以下特性：

\begin{itemize}
    \item 最多支持 16 组地址范围的配置
    \item 支持内部存储器以及外设寄存器的访问权限管理
    \item 支持中断功能
    \item 支持异常信息记录功能
\end{itemize}

\section{功能描述}

\subsection{TEE 控制器功能描述}

\chipname{} 芯片支持四种安全模式：TEE、REE0、REE1、REE2。

当 CPU 作为主机访问内存或外设寄存器时，可先配置 CPU 的用户模式或机器模式，然后配置其安全模式。关于 CPU 机器模式及用户模式的配置方法请参考 RISC-V 指令集手册 v1.10 第二卷“特权架构” (RISC\-/V Instruction Set Manual, Volume II: Privileged Architecture, Version 1.10)。
\begin{itemize}
    \item 当 CPU 为机器模式时，其安全模式为 TEE 模式。
    \item 当 CPU 为用户模式时，其安全模式为 REE 模式，具体还需设置 \hyperref[regdesc:TEEM0MODECTRLREG]{TEE\_M0\_MODE\_CTRL\_REG} 寄存器中的 \hyperref[fielddesc:TEEM0MODE]{TEE\_M0\_MODE} 指定 REE0，REE1 或 REE2 模式：
    \begin{itemize}
        \item 若 \hyperref[fielddesc:TEEM0MODE]{TEE\_M0\_MODE} 为 0，即选择 TEE 模式，但是在 CPU 的用户模式下，其实际的安全模式为 REE0 模式。
        \item 若 \hyperref[fielddesc:TEEM0MODE]{TEE\_M0\_MODE} 为 1， 2 或 3，则安全模式分别为 REE0，REE1，REE2。
    \end{itemize}
\end{itemize}

对于其他主机访问内存或外设寄存器，可通过配置 TEE 寄存器模块的 \hyperref[fielddesc:TEEM0MODE]{TEE\_M\regindex{n}\_MODE} 来配置其安全模式，其中 \regindex{n} 与表 \ref{tab:master_source} 中的主机 ID 号相对应。

\subsubsection{地址范围}

\hyperref[sec:permctrl-reg-hp]{HP\_APM\_REG} 寄存器模块可配置 16 组地址范围。\hyperref[regdesc:HPAPMREGION0ADDRSTARTREG]{HP\_APM\_REGION\regindex{n}\_ADDR\_START} 和 \hyperref[regdesc:HPAPMREGION0ADDRENDREG]{HP\_APM\_REGION\regindex{n}\_ADDR\_END} 分别控制高性能寄存器的第 \regindex{n+1} 组地址范围的起始和结束地址。可通过配置 \hyperref[fielddesc:HPAPMREGIONFILTEREN]{HP\_APM\_REGION\_FILTER\_EN\_REG} 寄存器的 bit \regindex{n} 使能第 \regindex{n+1} 组地址范围，默认使能第 1 组地址范围。

\hyperref[sec:permctrl-reg-lp]{LP\_APM\_REG} 寄存器模块可配置 4 组地址范围。\hyperref[fielddesc:LPAPMREGION0ADDRSTART]{LP\_APM\_REGION\regindex{n}\_ADDR\_START} 和 \hyperref[fielddesc:LPAPMREGION0ADDREND]{LP\_APM\_REGION\regindex{n}\_ADDR\_END} 分别控制低功耗寄存器的第 \regindex{n+1} 组地址范围的起始和结束地址。可通过配置 \hyperref[fielddesc:LPAPMREGIONFILTEREN]{LP\_APM\_REGION\_FILTER\_EN\_REG} 寄存器的 bit \regindex{n} 使能第 \regindex{n+1} 组地址范围，默认使能第 1 组地址范围。

配置地址范围时，地址的最小对齐单位是 4 字节，即地址的低两位为 0。 例如，地址范围可以设置为 0x4080000C \verb+~+ 0x40808774 或 0x600C0008 \verb+~+ 0X600CFF70。

\subsubsection{地址范围的访问权限}

每个地址范围内，均可以为不同的安全模式单独配置访问权限：

\begin{itemize}

    \item TEE 模式下的主机始终拥有地址范围内的读、写、执行权限。
    \item REE0、REE1 和 REE2 模式下的主机，根据不同的访问路径，其读写执行权限由 \hyperref[regdesc:HPAPMREGION0ATTRREG]{HP\_APM\_REGION\regindex{n}\_ATTR\_REG} 或 \hyperref[regdesc:LPAPMREGION0ATTRREG]{LP\_APM\_REGION\regindex{n}\_ATTR\_REG} 配置。

\end{itemize}

由同一寄存器模块控制的所有访问路径共享该寄存器模块的配置，即配置的地址范围及其权限。例如图 \ref{fig:APM_Ctrl_Location} 中 HP\_APM\_CTRL M0-M3 在进行访问管控时，都依照 HP\_APM\_REG 寄存器模块中配置的 16 组地址范围及其对应的读、写、执行权限。LP\_APM\_CTRL M0 在进行访问管控时，依照 LP\_APM\_REG 寄存器模块中配置的 4 组地址范围及其对应的读、写、执行权限。

如图 \ref{fig:APM_Ctrl_Location}，所有主机访问 LP SRAM 时都会通过 HP\_APM\_CTRL M1。假设 \hyperref[fielddesc:HPAPMM1FUNCEN]{HP\_APM\_M1\_FUNC\_EN} 使能，REE1 安全模式的主机需要读取 LP SRAM 的数据，则具体流程如下：


\begin{enumerate}
    \item HP\_APM\_CTRL M1 会先判断请求访问的地址是否在 \hyperref[sec:permctrl-reg-hp]{HP\_APM\_REG} 寄存器模块中配置的 16 组地址范围内。
    \item 假设该访问地址在配置的第 2 组地址范围内，则HP\_APM\_CTRL M1 判断该组地址范围是否使能，即 \hyperref[fielddesc:HPAPMREGIONFILTEREN]{HP\_APM\_REGION\_FILTER\_EN} 的 bit 1 是否为 1。
    \item 若地址范围已使能，HP\_APM\_CTRL M1 判断主机在 REE1 模式下对第 2 组地址范围是否具有读的权限，即  \hyperref[regdesc:HPAPMREGION0ATTRREG]{HP\_APM\_REGION1\_ATTR\_REG}  中的 \hyperref[fielddesc:HPAPMREGION0R1R]{HP\_APM\_REGION1\_R1\_R} 是否有效（即为 1），若有效，则此次读请求会被正确响应，否则读到的数据恒为 0。
\end{enumerate}

在 APM 寄存器配置的多个区域的地址空间重叠的情况下，例如一个区域设置为不可读，一个区域设置为可读，两个区域的地址重叠，这部分重叠的地址空间的属性是可读的。写以及执行权限适用同样的规则。

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
    \item 上电时，只有 CPU 默认处于 TEE 模式，其他主机均处于 REE2 模式。并且默认状态下，APM 控制器会阻止所有处于 REE0、REE1 和 REE2 模式的主机的访问请求。
    \item 章节 \ref{sec:permctrl-reg-summ} \nameref{sec:permctrl-reg-summ} 中列出的所有寄存器均只有处于 TEE 安全模式的主机才可以配置。
\end{itemize}
\end{tiplisting}

\section{配置流程}

主机访问内存或外设寄存器路径中 APM 管控的配置流程为：
\begin{enumerate}

    \item 配置 CPU 到机器模式（即 TEE 模式）。
    \item 根据表 \ref{tab:master_source} 中的主机 ID 号, 配置 \hyperref[fielddesc:TEEM0MODE]{TEE\_M\regindex{n}\_MODE} 寄存器，配置 \chipname{} 中主机的安全模式。
    \item 配置 \hyperref[regdesc:HPAPMREGION0ADDRSTARTREG]{HP\_APM\_REGION\regindex{n}\_ADDR\_START}、\hyperref[regdesc:HPAPMREGION0ADDRENDREG]{HP\_APM\_REGION\regindex{n}\_ADDR\_END} 寄存器，或者 \hyperref[regdesc:LPAPMREGION0ADDRSTARTREG]{LP\_APM\_REGION\regindex{n}\_ADDR\_START}、\hyperref[regdesc:LPAPMREGION0ADDRENDREG]{LP\_APM\_REGION\regindex{n}\_ADDR\_END} 寄存器，配置 APM 寄存器中的各组地址范围。

    \item 配置 \hyperref[regdesc:HPAPMREGION0ATTRREG]{HP\_APM\_REGION\regindex{n}\_ATTR\_REG} 寄存器，或者 \hyperref[regdesc:LPAPMREGION0ATTRREG]{LP\_APM\_REGION\regindex{n}\_ATTR\_REG} 寄存器，配置 APM 寄存器中的各组地址范围的访问权限。

    \item 配置 \hyperref[fielddesc:HPAPMREGIONFILTEREN]{HP\_APM\_REGION\_FILTER\_ EN\_REG} 寄存器，或者 \hyperref[fielddesc:LPAPMREGIONFILTEREN]{LP\_APM\_REGION\_FILTER\_ EN\_REG} 寄存器，使能对应地址范围。

    \item 配置 \hyperref[regdesc:HPAPMFUNCCTRLREG]{HP\_APM\_ FUNC\_CTRL\_REG} 或 \hyperref[regdesc:LPAPMFUNCCTRLREG]{LP\_APM \_FUNC\_CTRL\_REG} 寄存器使能对应路径的权限检查，默认使能。
\end{enumerate}

以 I2S 通过 GDMA 访问 HP SRAM 为例，假设仅允许其在第 4 组地址范围 0x40805000 \verb+~+ 0x4080F000 地址范围内读写，则配置流程为：

\begin{enumerate}
    \item 配置 CPU 为机器模式（即 TEE 模式）。

    \item 根据表 \ref{tab:master_source} 中的主机 ID 号, 配置 \hyperref[fielddesc:TEEM0MODE]{TEE\_M19\_MODE} 寄存器为 1，将 I2S 通过 GDMA 访问的安全模式设置为 REE0 模式。

    \item 配置 \hyperref[regdesc:HPAPMREGION0ADDRSTARTREG]{HP\_APM\_REGION3\_ADDR\_START} 为 0x40805000， 配置 \hyperref[regdesc:HPAPMREGION0ADDRSTARTREG]{HP\_APM\_REGION3\_ADDR\_END} 为 0x4080F000。
    \item 配置 \hyperref[fielddesc:HPAPMREGION0R0W]{HP\_APM\_REGION3\_R0\_W} 和 \hyperref[fielddesc:HPAPMREGION0R0R]{HP\_APM\_REGION3\_R0\_R} 为 1。

    \item 配置 \hyperref[fielddesc:HPAPMREGIONFILTEREN]{HP\_APM\_REGION\_FILTER\_EN} 的 bit3 为 1。

    \item 配置 \hyperref[fielddesc:HPAPMM2FUNCEN]{HP\_APM\_M2\_FUNC\_EN} 为1。

\end{enumerate}



\section{非法访问与中断}


若总线上携带的信息与配置不一致时，\chipname{} 将视其为非法访问，并进行如下处理：


\begin{itemize}
    \item 拒绝该访问请求，并返回默认值，具体表现为：
    \begin{itemize}
        \item 读取和执行操作时返回 0
        \item 写入操作时无效
    \end{itemize}
    \item 触发中断
\end{itemize}


APM 控制器会自动记录非法访问的相关信息，包括主机的 ID 、安全模式、访问地址、非法访问原因（地址越限／权限不足）以及各访问路径的权限判定状态。所有这些信息都可以从章节 \ref{sec:permctrl-reg-summ} \nameref{sec:permctrl-reg-summ} 列出的相关寄存器中获取。

以 HP\_APM\_CTRL M0 数据访问路径为例，当出现非法访问时：
\begin{itemize}
    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONID]{HP\_APM\_M0\_EXCEPTION\_ID} 记录访问主机的 ID。
    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONMODE]{HP\_APM\_M0\_EXCEPTION\_MODE} 记录访问安全模式。
    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONADDR]{HP\_APM\_M0\_EXCEPTION\_ADDR} 记录访问地址。
    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONSTATUS]{HP\_APM\_M0\_EXCEPTION\_STATUS} 记录非法访问原因。
    \begin{itemize}
        \item  当访问的地址不在 16 组地址范围中配置的区域时，bit1 会被置 1，表示地址越限。
        \item  当访问的地址在某一组或多组地址范围内，但访问主机在这一组或多组地址范围内没有对应的读/写/执行操作权限时，bit0 会被置 1，表示权限不足。
    \end{itemize}
    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONREGION]{HP\_APM\_M0\_EXCEPTION\_REGION} 记录各地址范围的权限判定状态。该寄存器总共 16 个 bit，对应 16 组地址范围， bit0 对应第 1 组地址范围。若访问的地址在配置使能的某一组地址范围内，但在这组地址范围内没有要求的读/写/执行操作权限时，本寄存器对应于这组地址范围的 bit 位会被置1。


\end{itemize}

\chipname{} 中 APM 控制器可产生 5 个中断信号，并将其发送给 \textit{\nameref{mod:intmtrx}}：

    \begin{itemize}
    \item HP\_APM\_M0\_INTR
    \item HP\_APM\_M1\_INTR
    \item HP\_APM\_M2\_INTR
    \item HP\_APM\_M3\_INTR
    \item LP\_APM\_M0\_INTR
    \end{itemize}

这 5 个中断信号与图 \ref{fig:APM_Ctrl_Location} 中受管控的访问路径一一对应，当一个受管控的访问路径有非法访问，则对应的中断会被产生。
