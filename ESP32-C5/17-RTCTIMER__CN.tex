\chapter{实时时钟定时器}
\label{mod:rtctimer}

%%% Introduction %%%

\section{概述}

实时时钟定时器（RTC 定时器）是一个 48 位的可读计时器，能够在芯片的任意功耗模式下工作，当 HP 系统中的定时器外设不可用时，继续为系统提供定时器服务。同时还支持配置定时器中断、记录系统中特定事件发生的时刻。

%%%%%%%%%%%%%%%%%%%%
%%% Feature List %%%
%%%%%%%%%%%%%%%%%%%%

\section{主要特性}
\label{sec:rtctimer-features}
\begin{itemize}
    \item 48 位的计时器
    \item 触发特定事件时可记录事件发生的时刻，支持的特定事件有：
    \begin{itemize}
        \item HP 系统复位
        \item CPU 进入 stall 状态
        \item CPU 退出 stall 状态
        \item 晶振时钟开启
        \item 晶振时钟关闭
    \end{itemize}
    \item 通过配置寄存器触发 RTC 定时器记录当前时间
    \item 支持缓存最近两次特定事件的发生的时间
    \item 支持在目标时刻产生中断，目标时刻可配置\tagged{ESP32-C5}{，可同时配置两个目标时刻}
    \item 除 LP 系统上电复位外的其余任何复位/睡眠均不会使 RTC 定时器停止或复位
    \item 可用作唤醒源（详见章节 \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}）
\end{itemize}

\section{功能描述}
\label{sec:rtctimer-func-descr}

\begin{itemize}
    \item 48 位计时器
    \begin{itemize}
        \item RTC 定时器的实现基于一个 48 位的计时器，其驱动时钟是由位于 Always-On 电源域的 RTC\_SLOW\_CLK提供，采用不间断的循环计时（LP 系统复位除外），当 48 位计时器溢出时会产生溢出中断。 关于电源域的更多信息，请查看章节 \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}})。
    \end{itemize}

    \item 记录特定事件的发生时刻
    \begin{itemize}
        \item RTC 定时器支持记录三类特定事件的发生时刻：
        \begin{itemize}
            \item HP 系统复位
            \item CPU 进入或退出 stall 状态
            \item 晶振时钟开启或关闭
        \end{itemize}
        \item 三类事件都有各自独立的配置寄存器，当配置寄存器使能后，若发生对应的事件，产生的硬件脉冲就会触发 RTC 定时器记录当前时间。对应的配置寄存器见下表：
        \begin{longtable}[c]{|p{6cm}|p{9cm}|}
        \caption{配置 RTC 定时器记录特定事件的发生时刻} \label{tab:configure-rtc-timer-log-time-of-specific-events}
        \\\hline
        \rowcolor{lightgray}
        \textbf{配置寄存器} & \textbf{描述} \\
        \hline
        \hyperref[fielddesc:RTCTIMERMAINTIMERSYSRST]{RTC\_TIMER\_MAIN\_TIMER\_SYS\_RST} &  使能 RTC 定时器 记录系统复位时刻。 \\
        \hline
        \hyperref[fielddesc:RTCTIMERMAINTIMERSYSSTALL]{RTC\_TIMER\_MAIN\_TIMER\_SYS\_STALL} &  使能 RTC 定时器 记录 CPU 进入或退出 stall 状态时刻。\\
        \hline
        \hyperref[fielddesc:RTCTIMERMAINTIMERXTALOFF]{RTC\_TIMER\_MAIN\_TIMER\_XTAL\_OFF} &  使能 RTC 定时器 记录晶振时钟开启或关闭时刻。\\
        \hline
        \end{longtable}
    \end{itemize}


    \item 通过配置寄存器触发 RTC 定时器记录当前时间
    \begin{itemize}
        \item 除了上述三类特定事件发生时可以触发 RTC 定时器记录当前时间，还可以通过配置寄存器 \hyperref[fielddesc:RTCTIMERMAINTIMERUPDATE]{RTC\_TIMER\_MAIN\_TIMER\_UPDATE} 触发 RTC 定时器记录当前时间。
        \item 配置 \hyperref[fielddesc:RTCTIMERMAINTIMERUPDATE]{RTC\_TIMER\_MAIN\_TIMER\_UPDATE} 寄存器会产生一个脉冲信号，触发 RTC 定时器记录当前时间。
    \end{itemize}

    \item 目标时刻产生计数中断
    \begin{itemize}
        \item RTC 定时器支持\iftagged{ESP32-C5}{同时配置两个目标时刻，分别为目标时刻 0 和目标时刻 1，}{配置目标时刻0，}当计时器到达目标时刻 0 时会触发 RTC\_TIMER\_CMP0\_INT 中断\tagged{ESP32-C5}{，当计时器到达目标时刻 1 时会触发 RTC\_TIMER\_CMP1\_INT 中断}。
        \item 目标时刻的配置寄存器为：
        \begin{longtable}[c]{|p{6cm}|p{9cm}|}
        \caption{RTC 定时器目标时刻配置} \label{tab:rtc-timer-target-time-configuration}
        \\\hline
        \rowcolor{lightgray}
        \textbf{配置寄存器} & \textbf{描述} \\
        \hline
        \iftagged{ESP32-C5}{\multirow{1}{*}{\hyperref[fielddesc:RTCTIMERMAINTIMERTAREN0]{RTC\_TIMER\_MAIN\_TIMER\_TAR\_EN\regindex{n}}} & \regindex{n} = 0 或 1，}{\multirow{1}{*}{\hyperref[fielddesc:RTCTIMERMAINTIMERTAREN0]{RTC\_TIMER\_MAIN\_TIMER\_TAR\_EN0}} & }配置目标时刻使能。 \\
        \hline
        \iftagged{ESP32-C5}{\multirow{1}{*}{\hyperref[fielddesc:RTCTIMERMAINTIMERTARLOW0]{RTC\_TIMER\_MAIN\_TIMER\_TAR\_LOW\regindex{n}}} & \regindex{n} = 0 或 1，}{\multirow{1}{*}{\hyperref[fielddesc:RTCTIMERMAINTIMERTARLOW0]{RTC\_TIMER\_MAIN\_TIMER\_TAR\_LOW0}} &}配置目标时刻的低 32 位。\\
        \hline
        \iftagged{ESP32-C5}{\multirow{1}{*}{\hyperref[fielddesc:RTCTIMERMAINTIMERTARHIGH0]{RTC\_TIMER\_MAIN\_TIMER\_TAR\_HIGH\regindex{n}}} & \regindex{n} = 0 或 1，}{\multirow{1}{*}{\hyperref[fielddesc:RTCTIMERMAINTIMERTARHIGH0]{RTC\_TIMER\_MAIN\_TIMER\_TAR\_HIGH0}} &}配置目标时刻的高 16 位。\\
        \hline
        \end{longtable}
    \end{itemize}


    \item 读取缓存时刻
    \begin{itemize}
        \item RTC 定时器有两组寄存器用于缓存特定事件的发生时刻，分别为寄存器组 0 和寄存器组 1；
        \item 寄存器组 0 包括寄存器 \hyperref[regdesc:RTCTIMERMAINBUF0LOWREG]{RTC\_TIMER\_MAIN\_BUF0\_LOW\_REG} 和 \hyperref[regdesc:RTCTIMERMAINBUF0LOWREG]{RTC\_TIMER\_MAIN\_BUF0\_HIGH\_REG}，用于记录 RTC 定时器在当前触发下的计数值；
        \item 寄存器组 1 包括寄存器 \hyperref[regdesc:RTCTIMERMAINBUF1LOWREG]{RTC\_TIMER\_MAIN\_BUF1\_LOW\_REG} 和 \hyperref[regdesc:RTCTIMERMAINBUF1HIGHREG]{RTC\_TIMER\_MAIN\_BUF1\_HIGH\_REG}，用于记录 RTC 定时器在上一次触发下的计数值；
        \item 每次有新的触发，上一次触发时的记录将从寄存器组 0 移至寄存器组 1（寄存器组 1 中之前的记录将被覆盖），而本次触发的记录将存储在寄存器组 0。因此， RTC 定时器最多可同时记录两次触发的值。
    \end{itemize}
\end{itemize}

\section{事件任务矩阵功能}
\label{sec:rtctimer-etm-func}

在 \chipname{} 中，RTC 定时器支持 ETM 功能，即可以通过 RTC 定时器的 ETM 事件触发任意外设的 ETM 任务。RTC 定时器不支持 ETM 任务。关于 ETM 更多详细信息，请参考章节 \ref{mod:evntaskmatrix} \nameref{mod:evntaskmatrix}。这里仅介绍与 RTC 定时器相关的 ETM 事件。

RTC 定时器可产生的 ETM 事件有：

\begin{itemize}
    \item RTC\_EVT\_CMP：表示 RTC 定时器计时到达寄存器 \hyperref[fielddesc:RTCTIMERMAINTIMERTARLOW0]{RTC\_TIMER\_MAIN\_TIMER\_TAR\_LOW0} 和 \hyperref[fielddesc:RTCTIMERMAINTIMERTARHIGH0]{RTC\_TIMER\_MAIN\_TIMER\_TAR\_HIGH0} 所配置的目标时刻 0。
    \item RTC\_EVT\_OVF：表示 48 位计时器计数溢出。
    \item RTC\_EVT\_TICK：表示 RTC 计时器递增 1 时触发的事件。
\end{itemize}

\section{中断}
\chipname{} 的 RTC 定时器可以生成以下中断信号并发送给\hyperref[mod:intmtrx]{中断矩阵}。

    \begin{itemize}
        \item LP\_TIMER\_REG\_0\_INTR（该中断供 HP CPU 使用）
        \tagged{ESP32-C5}{\item LP\_TIMER\_REG\_1\_INTR（该中断供 LP CPU 使用）}
    \end{itemize}

这些中断信号由 RTC 定时器的内部中断源生成。表 \ref{tab:rtctimer-int-sources} 列出了 RTC 定时器的内部中断源、中断触发条件、以及生成的中断信号。

\\ \hline
\rowcolor{lightgray}
\textbf{内部中断源} & \textbf{触发条件}  &  \textbf{中断信号}\\ \hline
\label{int:RTCTIMERCMP0INT}RTC\_TIMER\_CMP0\_INT &  RTC 定时器计时到达目标时刻0 & LP\_TIMER\_REG\_0\_INTR  \\ \hline
\tagged{ESP32-C5}{\label{int:RTCTIMERCMP1INT}RTC\_TIMER\_CMP1\_INT &  RTC 定时器计时到达目标时刻1 & LP\_TIMER\_REG\_1\_INTR \\ \hline}
\label{int:RTCTIMEROVERFLOWINT}RTC\_TIMER\_OVERFLOW\_INT & 48 位计时器计数溢出 & LP\_TIMER\_REG\_0\_INTR  \tagged{ESP32-C5}{LP\_TIMER\_REG\_1\_INTR}\\ \hline
