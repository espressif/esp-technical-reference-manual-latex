\hypertarget{tmpsen}{}
\chapter{Temperature Sensor}
\label{mod:tmpsen}


\section{Overview}

\chipname{} provides a temperature sensor to monitor temperature changes inside the chip in real time. The sensor converts analog voltage to digital values and supports compensation for the temperature offset.

\section{Features}

The temperature sensor has the following features:

\begin{itemize}
    \item Software-triggered temperature measurement. Once triggered, the sensor continuously measures temperature. Software can read the data any time.
    \item Hardware-triggered automatic temperature monitoring
    \item Two modes for automatic monitoring of temperature and support for triggering interrupts
    \item Configurable temperature offset based on the application scenario for improved accuracy
    \item Configurable temperature measurement range
\item Support for several Event Task Matrix (ETM) related events and tasks
\end{itemize}

\section{Architecture}

Figure \ref{fig:tsens_struct} shows the internal structure of the temperature sensor.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{\modulefiles/figures/tsens_struct_20221209}
    \caption{Temperature Sensor Architecture}
    \label{fig:tsens_struct}
\end{figure}

As Figure \ref{fig:tsens_struct} shows, the temperature sensor module contains the following major blocks:

\begin{itemize}
\item Tsens Ctrl \& Detect: temperature sensor
    \item HW Timer: triggers automatic temperature monitoring
    \item Tsens\_data Monitor: monitors whether the temperature is outside the threshold range
    \item sync\_module: Synchronization block between APB clock domain and temperature sensor clock domain
\end{itemize}

\section{Functional Description}

\subsection{Temperature Sensor Power Up}

The temperature sensor can be powered up by setting the \hyperref[fielddesc:APBSARADCTSENSPU]{APB\_SARADC\_TSENS\_PU} register.

\subsection{Temperature Sensor Clock}

The temperature sensor has two clock sources: RC\_FAST\_CLK and XTAL\_CLK, selected by \hyperref[fielddesc:PCRTSENSCLKSEL]{PCR\_TSENS\_CLK\_SEL}. The clock can be divided with \hyperref[fielddesc:APBSARADCTSENSCLKDIV]{APB\_SARADC\_TSENS\_CLK\_DIV}.

\subsection{Automatic Temperature Monitoring Modes}

Two modes for the automatic temperature monitoring are available for selection by configuring \hyperref[fielddesc:APBSARADCWAKEUPMODE]{APB\_SARADC\_WAKEUP\_MODE}:

   \begin{itemize}
       \item Absolute value mode:
           \begin{itemize}
                \item Monitors the absolute value of the current temperature. Configure \hyperref[fielddesc:APBSARADCWAKEUPTHLOW]{APB\_SARADC\_WAKEUP\_TH\_LOW} and \hyperref[fielddesc:APBSARADCWAKEUPTHHIGH]{APB\_SARADC\_WAKEUP\_TH\_HIGH} to set the temperature thresholds. An interrupt will be triggered if the sampled value is above the high threshold or below the low threshold.
           \end{itemize}
       \item Change value mode:
            \begin{itemize}
                \item Monitors the temperature changes inside the chip. If the temperature increment of two consecutive samplings exceeds the high threshold configured in \hyperref[fielddesc:APBSARADCWAKEUPTHHIGH]{APB\_SARADC\_WAKEUP\_TH\_HIGH}, or the temperature decrement of two consecutive samplings exceeds the low threshold configured in \hyperref[fielddesc:APBSARADCWAKEUPTHLOW]{APB\_SARADC\_WAKEUP\_TH\_LOW}, an interrupt will be triggered. For example, when \hyperref[fielddesc:APBSARADCWAKEUPTHLOW]{APB\_SARADC\_WAKEUP\_TH\_LOW} is configured as 8, if two consecutive sampling values are 28 and 19 respectively, i.e., the temperature decrement is 9, then an interrupt will be triggered.
            \end{itemize}
    \end{itemize}

\subsection{Temperature Measurement Range and Offset}

To improve the temperature measurement accuracy, the temperature sensor is designed with five measurement ranges as shown in Table \ref{tab:tsensor-dac}. Each measurement range comes with specific measurement errors. Users do not need to worry about the errors, as they can choose specific offsets to get calibrated data.

\begin{longtable}[c]{|l|r|}
    \caption{Temperature Measurement Range and Offset}
    \label{tab:tsensor-dac}
    \\\hline
    \rowcolor{lightgray}
    Temperature Measurement Range (°C)   & Temperature Offset \\\hline
    50 \verb+~+ 125 &  –2 \\\hline
    20 \verb+~+ 100 &  –1 \\\hline
    –10 \verb+~+ 80 &  0 \\\hline
    –30 \verb+~+ 50 &  1 \\\hline
    –40 \verb+~+ 20 &  2 \\\hline
\end{longtable}

\subsection{Data Conversion}

The sensor output value is stored in \hyperref[fielddesc:APBSARADCTSENSOUT]{APB\_SARADC\_TSENS\_OUT}. To calculate the actual temperature \(T\) (°C) based on \(VALUE\), use the following equation:

\[
        T = 0.4386 * VALUE – 27.88 * offset – 20.52
\]

where \(offset\) is the temperature offset shown in Table \ref{tab:tsensor-dac}.

\section{Programming Procedure}

The temperature sensor can be started by software as follows:
    \begin{enumerate}
        \item Set \hyperref[fielddesc:APBSARADCTSENSPU]{APB\_SARADC\_TSENS\_PU} to power up the temperature sensor.
        \item Set \hyperref[fielddesc:PCRTSENSCLKEN]{PCR\_TSENS\_CLK\_EN} to enable the temperature sensor clock.
        \item Configure \hyperref[fielddesc:APBSARADCTSENSCLKSEL]{APB\_SARADC\_TSENS\_CLK\_SEL} to select the temperature sensor clock.
        \item Wait for \hyperref[fielddesc:APBSARADCTSENSXPDWAIT]{APB\_SARADC\_TSENS\_XPD\_WAIT} clock cycles till the temperature sensor releases from reset and starts measuring the temperature.
        \item Wait for 100 $\mu$s (so that the output value gradually approaches the actual temperature) and then read the data from \hyperref[fielddesc:APBSARADCTSENSOUT]{APB\_SARADC\_TSENS\_OUT}.
    \end{enumerate}

To enable hardware-triggered automatic temperature monitoring, add the following programming steps before powering up the sensor:
    \begin{enumerate}
        \item Configure \hyperref[fielddesc:APBSARADCTSENSSAMPLERATE]{APB\_SARADC\_TSENS\_SAMPLE\_RATE} to set sampling rate.
        \item Configure \hyperref[fielddesc:APBSARADCWAKEUPMODE]{APB\_SARADC\_WAKEUP\_MODE} to select an automatic temperature monitoring mode.
        \item Configure \hyperref[fielddesc:APBSARADCWAKEUPTHHIGH]{APB\_SARADC\_WAKEUP\_TH\_HIGH/LOW} to set the high and low temperature monitoring thresholds.
        \item Set \hyperref[fielddesc:APBSARADCWAKEUPEN]{APB\_SARADC\_WAKEUP\_EN} to start temperature monitoring.
        \item Set \hyperref[fielddesc:APBSARADCTSENSSAMPLEEN]{APB\_SARADC\_TSENS\_SAMPLE\_EN} to enable automatic temperature monitoring.
    \end{enumerate}

In automatic temperature monitoring mode, the output value will not be stored. However, users can get the value anytime from \hyperref[fielddesc:APBSARADCTSENSOUT]{APB\_SARADC\_TSENS\_OUT}.

\section{Interrupts}

\chipname{}'s Temperature Sensor can generate the following interrupt signal that will be sent to the \hyperref[mod:intmtrx]{Interrupt Matrix}.

    \begin{itemize}
        \item LP\_TSENS\_INTR
    \end{itemize}

There is one internal interrupt source from the Temperature Sensor that can generate the above interrupt signal:

    \begin{itemize}
        \item \label{interreupt:APBSARADCTSENSINT} APB\_SARADC\_TSENS\_INT: Triggered when the temperature sample value exceeds the threshold.
    \end{itemize}

\begin{tiplisting}
For definitions of \textit{interrupt}, \textit{interrupt signal}, \textit{interrupt source}, and their correlations, please refer to Chapter \ref{mod:intmtrx} \textit{\nameref{mod:intmtrx}} > Section \ref{sec:intmtx-int-terminology} \textit{\nameref{sec:intmtx-int-terminology}}.
\end{tiplisting}

Each interrupt source can be configured by a common set of registers that are described in Section \hyperref[interrupt-config-registers]{\textit{Interrupt Configuration Registers}}. The specific registers can be found in Section \ref{sec:tmpsen-reg-summ} \textit{\nameref{sec:tmpsen-reg-summ}}.

\section{Event Task Matrix Feature}
\label{sec:tmpsen-etm-func}

The temperature sensor on \chipname{} supports the Event Task Matrix (ETM) function, which allows the temperature sensor's ETM tasks to be triggered by any peripherals' ETM events, or temperature sensor's ETM events to trigger any peripherals' ETM tasks. This section introduces the ETM tasks and events related to the temperature sensor. For more information, please refer to Chapter \ref{mod:evntaskmatrix} \textit{\nameref{mod:evntaskmatrix}}.

The temperature sensor can receive the following ETM tasks:

\begin{itemize}
    \item TMPSNSR\_TASK\_START\_SAMPLE: The temperature sensor starts sampling when this task is triggered.
    \item TMPSNSR\_TASK\_STOP\_SAMPLE: The temperature sensor stops sampling when this task is triggered.
\end{itemize}

The temperature sensor can generate the following ETM events:

\begin{itemize}
    \item TMPSNSR\_EVT\_OVER\_LIMIT: Generated when the temperature is beyond the threshold.
\end{itemize}

In practical applications, temperature sensor's ETM events can trigger its own ETM tasks.

For example, the TMPSNSR\_EVT\_OVER\_LIMIT event can trigger the TMPSNSR\_TASK\_STOP\_SAMPLE task.
