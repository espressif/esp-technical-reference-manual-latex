\chapter{System and Memory}
\label{mod:sysmem}

\section{Overview}

\chipname{} has an ultra-low power and highly-integrated system with two processors:
\begin{itemize}
    \item a high-performance 32-bit RISC-V processor (HP CPU), five-stage pipeline, clock frequency up to 240 MHz.
    \item a low-power 32-bit RISC-V processor (LP CPU), two-stage pipeline, clock frequency up to 48 MHz.
\end{itemize}
All internal memory, external memory, and peripherals are located on the HP CPU and LP CPU buses.
\section{Features}

\begin{itemize}
    \item {\bfseries Address Space}
        \begin{itemize}
    	    \item 720 KB of internal memory address space, accessible by the instruction bus or data bus
    	    \item 832 KB of peripheral address space
    	    \item 32 MB of virtual address space for external memory, accessible by the instruction bus or data bus
    	    \item 384 KB of internal DMA address space
                \item 32 MB of external DMA address space
        \end{itemize}
    \item {\bfseries Internal Memory}
        \begin{itemize}
            \item 320 KB of ROM 
	        \item 384 KB of HP SRAM
	        \item 16 KB of LP SRAM
	    \end{itemize}
    \item {\bfseries External Memory}
        \begin{itemize}
            \item Up to 32 MB of external flash
            \item Up to 32 MB of external RAM
	    \end{itemize}
    \item \textbf{Peripheral Space}
        \begin{itemize}
           \item 61 modules/peripherals in total
        \end{itemize}
    \item {\bfseries GDMA}
        \begin{itemize}
            \item 7 GDMA-supported modules/peripherals
        \end{itemize}
\end{itemize}

\section{Functional Description}

\subsection{Address Mapping}

Figure \ref{fig:sysmem-address-mapping-structure} illustrates the system structure and address mapping. All the non-reserved addresses are accessible via both the instruction bus and the data bus, meaning that the instruction bus and the data bus share the same address space. 

Both the data bus and instruction bus of the HP CPU and LP CPU are little-endian. All buses have a 32-bit data width. 
HP CPU and LP CPU can access data via the data bus using single-byte, double-byte, and 4-byte alignment.

The HP CPU has the following access capabilities:
\begin{itemize}
\item Direct access to internal memory via both the data bus and instruction bus
  \begin{itemize}
        \item 384 KB HP SRAM (0x4080\_0000 \verb+~+ 0x4085\_FFFF)
        \item 16 KB LP SRAM  (0x5000\_0000 \verb+~+ 0x5000\_3FFF)
    \end{itemize}
\item Access to internal memory via \hyperref[par:rom]{ROM-Cache}
    \begin{itemize}
        \item 320 KB ROM (0x4000\_0000 \verb+~+ 0x4004\_FFFF)
    \end{itemize}
\begin{tiplisting}
Software can access the ROM addresses in two ways: cache-able access or noncache-able access, depending on the control of CPU Physical Memory Attribution (PMA). Cache-able access is done via cache. Noncache-able access is done without cache.
\end{tiplisting}
\item Access to external memory through the cache

The virtual address is mapped to the physical address space of the external memory through the MMU.
    \begin{itemize}
        \item Up to 32 MB external flash (0×4200\_0000 \verb+~+ 0×43FF\_FFFF)
        \item Up to 32 MB external RAM (0×4200\_0000 \verb+~+ 0×43FF\_FFFF)
    \end{itemize}
\item Direct access to modules/peripherals via the data bus, including:
    \begin{itemize}
        \item HP CPU Peripherals
        \item HP Peripherals
        \item LP Peripherals
    \end{itemize}
\end{itemize}
The LP CPU has the following access capabilities:
\begin{itemize}
    \item Direct access to HP SRAM and LP SRAM via both the data bus and instruction bus
     \begin{itemize}
        \item 384 KB HP SRAM (0x4080\_0000 \verb+~+ 0x4085\_FFFF)
        \item 16 KB LP SRAM  (0x5000\_0000 \verb+~+ 0x5000\_3FFF)
    \end{itemize}
    \item Direct access to modules/peripherals via the data bus, including:
    \begin{itemize}
        \item HP Peripherals
        \item LP Peripherals
    \end{itemize}
\end{itemize}

\subsection{ Internal Memory}
\label{para:sysmem-internal-memory}

\chipname{} has various types of internal memory:

\begin{itemize}
    \item ROM (320 KB): The ROM is a read-only memory and can not be programmable. It contains the ROM code of some low-level system software and read-only data. Note that this memory can only be accessed by HP CPU.
    \item HP SRAM (384 KB): The HP SRAM is a volatile memory that can be quickly accessed by the HP CPU or LP CPU (generally within a single HP CPU clock cycle for HP CPU).
    \item LP SRAM (16 KB): The LP SRAM is also a volatile memory, however, in Deep-sleep mode, data stored in the LP SRAM will not be lost. The LP SRAM can be accessed by the HP CPU or LP CPU and is usually used to store program instructions and data that need to be kept in sleep mode.
\end{itemize}

\subsubsection{ROM}
\label{par:rom}

This 320 KB ROM is a read-only memory, accessed by the HP CPU directly through the instruction bus or through the data bus via  0x4000\_0000 \verb+~+ 0x4004\_FFFF or via ROM-Cache, see Table  \ref{tab:sysmem-memory-cn}.

As shown in Figure  \ref{fig:sysmem-rom-cache-arch},  \chipname{} is able to access ROM via ROM-Cache. This 4 KB ROM-Cache is two-way set associative, with a block size of 64 bytes.
Both the instruction bus and the data bus can access the ROM-Cache at the same time, and the ROM-Cache responds to one or the other through arbitration. If data missing happens, ROM-Cache controller initiates a request to the ROM.

\subsubsection{HP SRAM}
\label{other:sysmem-internal-sram-0}

This 384 KB HP SRAM is a read-and-write memory, accessed by the HP CPU and LP CPU through the instruction bus or data bus via their shared addresses 0×4080\_0000 $\sim$ 0×4085\_FFFF, see Table  \ref{tab:sysmem-memory-cn}. 


\subsubsection{LP SRAM}

This 16 KB LP SRAM is a read-and-write memory, accessed by the HP CPU or LP CPU through the instruction bus or data bus via their shared addresses 0x5000\_0000 \verb+~+ 0x5000\_3FFF, see Table  \ref{tab:sysmem-memory-cn}.

LP SRAM can be accessed by the following modes:
\begin{itemize}
    \item high-speed mode, i.e., the LP SRAM is accessed in HP CPU clock frequency. In this case, HP CPU can access the LP SRAM without any latency. But the latency of LP CPU accessing LP SRAM ranges from a few dozen to dozens of LP CPU cycles.
    \item low-speed mode, i.e., the LP SRAM is accessed in LP CPU clock frequency. In this case, LP CPU can access the LP SRAM without any latency. But the latency of HP CPU accessing LP SRAM ranges from a few dozen to dozens of HP CPU cycles.
\end{itemize}

Switch the modes based on application scenarios.
\begin{itemize}
    \item If the LP CPU is not working, switch to high-speed mode to improve the access speed of the HP CPU.
    \item If the LP CPU is executing code in the LP SRAM, switch to the low-speed mode.
    \item If the HP CPU is in sleep mode, it is a must to switch to the low-speed mode.
\end{itemize}

\textbf{Mode Switch Configuration}
\begin{itemize}
    \item Configure  \hyperref[fielddesc:LPAONFASTMEMMUXSEL]{LP\_AON\_FAST\_MEM\_MUX\_SEL} to select the mode needed:
    \begin{itemize}
        \item 0: low-speed mode
        \item 1: high-speed mode
    \end{itemize}
    \item Set  \hyperref[fielddesc:LPAONFASTMEMMUXSELUPDATE]{LP\_AON\_FAST\_MEM\_MUX\_SEL\_UPDATE} to start mode switch.
    \item Read  \hyperref[fielddesc:LPAONFASTMEMMUXSELSTATUS]{LP\_AON\_FAST\_MEM\_MUX\_SEL\_STATUS} to check if mode switch is done:
    \begin{itemize}
        \item 0: mode is switched
        \item 1: mode is not switched
    \end{itemize}
\end{itemize}

\subsection{External Memory} 
\label{para:sysmem-external-memory}

\chipname{} supports SPI, Dual SPI, Quad SPI, and QPI interfaces that allow connection to external flash and RAM. 
\chipname{} also supports hardware manual encryption and automatic decryption based on XTS-AES algorithm to protect users’ programs and data in the external flash and RAM.


\subsubsection{External Memory Address Mapping}

The external memory can be accessed by HP CPU via the cache or accessed by the GDMA. According to information inside the MMU (Memory Management Unit), the cache maps the HP CPU and GDMA’s address (0x4200\_0000 \verb+~+ 0x43FF\_FFFF) into a physical address of the external memory. 
With this address mapping, \chipname{} can address up to 32 MB external flash and 32 MB external RAM.\\
Note that the instruction bus shares the same address space (32 MB) with the data bus to access the external memory. 

\subsubsection{Cache}
As shown in Figure  \ref{fig:sysmem-cache-arch},  \chipname{} has a shared four-way set-associative cache. Its size is 32 KB and its block size is 32 bytes.
The instruction bus and data bus can access the cache simultaneously, but the cache can only respond to one of them at a time through arbitration. If a data missing happens, cache controller initiates a request to the external memory.

\subsubsection{Cache Operations}

\chipname{} cache supports the following operations:

\begin{enumerate}
\item \textbf{Write-back}
\begin{itemize}
    \item Clears dirty bits in the tag memory and updates new data to external memory.
    \item After the write-back operation, the new data is updated to external memory.
    \item Users can choose whether to invalidate the data in the cache.
    \item If the data is not invalidated, HP CPU will read/write data directly from/to the cache where data missing will not occur.
\end{itemize}

\item \textbf{Clean}
\begin{itemize}
    \item Clears dirty bits in the tag memory without updating data to external memory.
    \item After the clean operation, old data remains in external memory, while the cache holds the new one (unaware of it).
    \item HP CPU can then read/write the data directly from/to the cache, where data missing will not occur.
\end{itemize}

\item \textbf{Invalidate}
\begin{itemize}
    \item Clean the valid bits in tag memory. This is to remove valid data from the cache.
    \item Once this operation is done, the deleted data is stored only in external memory.
    \item HP CPU needs to access external memory if it wants to access the data again.
    \item Two types of invalidate operation:
    \begin{itemize}
    \item Manual-Invalidate: is performed only on data in the specified cache area.
    \item Invalidate-All: is performed on all data in the cache.
    \end{itemize}
\end{itemize}

\item \textbf{Preload}
\begin{itemize}
    \item Loads instructions and data into the cache in advance.
    \item Minimum unit of preload operation is one block.
    \item Two types of preload operation:
    \begin{itemize}
    \item Manual-Preload: involves the hardware prefetching continuous data based on the virtual address specified by the software.
    \item Auto-Preload: involves the hardware prefetching continuous data based on the current address where the cache hits or misses (configuration-dependent).
    \end{itemize}
\end{itemize}

\item \textbf{Lock/Unlock}
\begin{itemize}
    \item Lock operation prevents easily replacing data in the cache.
    \item Two types of lock:
    \begin{itemize}
    \item Prelock: locks data in the specified area when filling missing data to cache memory, leaving the data outside the specified area unlocked.
    \item Manual Lock: checks data in the cache memory and locks data only if it falls in the specified area, leaving data outside the specified area unlocked.
    \end{itemize}
    \item When there is missing data, the cache replaces the data in the unlocked way first, ensuring that the data in the locked way remains in the cache and will not be replaced.
    \item When all ways within the cache are locked, the cache will replace data, as if it was not locked.
    \item Unlocking is the reverse of locking and can only be done manually.
    \item Manual-Invalidate operation only works on the unlocked data. If you plan to perform this operation on the locked data, please unlock them first.
    
\end{itemize}
\end{enumerate}

\subsection{GDMA Address Space}

The General Direct Memory Access (GDMA) peripheral in \chipname{} consisting of three TX channels and three RX channels provides Direct Memory Access (DMA) service, including:

\begin{itemize}
    \item data transfers between different locations of internal memory
    \item data transfers between modules/peripherals and internal memory
    \item data transfers between different locations of external memory
    \item data transfers between modules/peripherals and external memory
    \item data transfers between internal memory and external memory
\end{itemize}
GDMA uses the same addresses as the data bus to access HP SRAM and external RAM, i.e., GDMA uses address range  0x4080\_0000 \verb+~+ 0x4085\_FFFF to access HP SRAM, and  0x4200\_0000 \verb+~+ 0x43FF\_FFFF to external RAM. Seven modules/peripherals in \chipname{} work together with GDMA. As shown in Figure  \ref{fig:sysmem-gdma-peripheral}, seven vertical lines correspond to these seven modules/peripherals with GDMA function. The horizontal line represents a certain channel of GDMA (can be any channel), and the intersection of the vertical line and the horizontal line indicates that a module/peripheral has the ability to access the corresponding channel of GDMA.
