\chapter{Power Supply Detector}
\label{mod:psdet}

\section{Overview}

\chipname{} has a power supply detector that can monitor the voltage on the power pins related to the on-chip clock and digital circuits. The power supply detector includes a brown-out detector and four voltage glitch detectors. The brown-out detector prevents the SoC from brown-out resets, while the voltage glitch detectors protect the SoC against voltage glitch attacks. The power detector operates in the always-on power domain, allowing it to monitor voltage at any time.

%%%%%%%%%%%%%%%%%%%%
%%% Feature List %%%
%%%%%%%%%%%%%%%%%%%%

\section{Features}
\label{sec:psdet-features}

\begin{itemize}
    \item Brown-out detector supports two brown-out detection modes (Mode 0 and Mode 1)
    \begin{itemize}
        \item Mode 0 supports:
            \begin{itemize}
            \item interrupt generation
            \item RF circuits power-down
            \item flash suspend triggering
            \item system reset
            \end{itemize}
        \item Mode 1 supports:
            \begin{itemize}
            \item system reset until the voltage returns to normal
            \end{itemize}
    \end{itemize}
    \item Voltage glitch detector can detect voltage glitches lasting 50 ns or longer and trigger a system reset.
\end{itemize}

\subsection{Brown-out Detector}

The brown-out detector checks the voltage of pins VDDA3, VDDPST1, VDDPST2, and VDDPST3, about every 280 $\mu$s. When the voltage on these pins falls below a predefined threshold of 2.7 V (default setting), the detector activates a signal to power down the power-hungry RF circuits. This action provides additional time for the digital system to save and transfer important data. The brown-out detector consumes very little power and remains active as long as the chip is powered on.

\hyperref[fielddesc:LPANABODMODE0INTRAW]{LP\_ANA\_BOD\_MODE0\_INT\_RAW} indicates the detection result from the brown-out detector. By default, this register reads a value of 0. It changes to 1 when the voltage on the monitored pin falls below a predefined threshold.

When a brown-out signal is detected, the brown-out detector handles it in one of the following two modes (Mode 1 is the default):

\begin{itemize}
    \item Mode 0:
    \begin{itemize}
        \item Mode 0 is enabled by setting the \texttt{bod\_mode0\_en} (\hyperref[fielddesc:LPANABODMODE0INTRENA]{LP\_ANA\_BOD\_MODE0\_INTR\_ENA}) signal.
        \item The brown-out detector triggers an interrupt when the counter counts to the thresholds predefined in Int Comparer (\hyperref[fielddesc:LPANABODMODE0INTRWAIT]{LP\_ANA\_BOD\_MODE0\_INTR\_WAIT}). If \hyperref[fielddesc:LPANABODMODE0CLOSEFLASHENA]{LP\_ANA\_BOD\_MODE0\_CLOSE\_FLASH\_ENA} is set, flash suspend will be triggered; if \hyperref[fielddesc:LPANABODMODE0PDRFENA]{LP\_ANA\_BOD\_MODE0\_PD\_RF\_ENA} is set, the RF module will be powered down.
        \item The brown-out detector resets the chip based on the configuration of \texttt{bod\_mode0\_rst\_sel} (\hyperref[fielddesc:LPANABODMODE0RESETSEL]{LP\_ANA\_BOD\_MODE0\_RESET\_SEL}) when the brown-out counter counts to the thresholds predefined in Rst Comparer (\hyperref[fielddesc:LPANABODMODE0RESETWAIT]{LP\_ANA\_BOD\_MODE0\_RESET\_WAIT}). The reset is enabled by \texttt{bod\_mode0\_rst\_en} (\hyperref[fielddesc:LPANABODMODE0RESETENA]{LP\_ANA\_BOD\_MODE0\_RESET\_ENA}).
    \end{itemize}

    \item Mode 1: Resets the system directly.
\end{itemize}

The brown-out reset workflow is illustrated in the diagram below.

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/brownout-reset-workflow}
   \caption{Brown-out Reset Workflow}
    \label{fig:lowpowm-Brown-out-detector-mode}
\end{figure}

Registers for controlling related signals are described below.

\begin{itemize}
    \item \texttt{bod\_mode0\_en}: \hyperref[fielddesc:LPANABODMODE0INTRENA]{LP\_ANA\_BOD\_MODE0\_INTR\_ENA}
    \item \texttt{bod\_mode0\_rst\_en}:  \hyperref[fielddesc:LPANABODMODE0RESETENA]{LP\_ANA\_BOD\_MODE0\_RESET\_ENA}
    \item \texttt{bod\_mode0\_rst\_sel}:  \hyperref[fielddesc:LPANABODMODE0RESETSEL]{LP\_ANA\_BOD\_MODE0\_RESET\_SEL} configures the reset type:
            \begin{itemize}
            \item 0: chip reset
            \item 1: system reset
            \end{itemize}
    For more information regarding chip reset and system reset, please refer to Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}}.
    \item \texttt{bod\_mode1\_sel}: The first bit of \hyperref[fielddesc:LPANAANAFIBENA]{LP\_ANA\_ANA\_FIB\_ENA}.
    \item \texttt{bod\_mode1\_rst\_en}:  \hyperref[fielddesc:LPANABODMODE1RESETENA]{LP\_ANA\_BOD\_MODE1\_RESET\_ENA}
\end{itemize}

\subsection{Voltage Glitch Detectors}

Figure \ref{fig:psdet-power-glitch-detector-mode} shows the structure of the voltage glitch detectors.

\begin{figure}[H]
    \centering   \includegraphics[width=0.7\textwidth]{\modulefiles/figures/power_glitch.png}
   \caption{Structure of Voltage Glitch Detectors}\label{fig:psdet-power-glitch-detector-mode}
\end{figure}

The \chipname{} has four voltage glitch detectors, each of which can detect glitches longer than 50 ns on the corresponding power supply pins, including VDDPST1, VDDPST2, VDDPST3, VDDA8, and VDDA3. VDDPST2 and VDDPST3 are connected and thus share one voltage glitch detector. All resets generated by the detectors are combined into one voltage glitch reset.

The voltage glitch detectors' enable control is managed by SW\_RESET\_ENA\_SEL. When set to 1, it is forcibly enabled by hardware, and when set to 0, it is controlled by the software register SW\_RESET\_ENA. The configuration for detecting different power pins is shown in the table below:

\begin{longtable}{ | l | l | l | }
\caption{Configuration for Detecting Power Pins} \label{tab:psdet-power-detec-pins}
\\ \hline
\rowcolor{lightgray}
\textbf{VDD Pins} & \textbf{SW\_RESET\_ENA}  &  \textbf{SW\_RESET\_ENA\_SEL}\\ \hline
VDDPST2/3 & \hyperref[fielddesc:LPANAPWRGLITCHRESETENA]{LP\_ANA\_PWR\_GLITCH\_RESET\_ENA[0]} & \hyperref[fielddesc:LPANAANAFIBENA]{LP\_ANA\_ANA\_FIB\_ENA[2]} \\\hline
VDDPST1 & \hyperref[fielddesc:LPANAPWRGLITCHRESETENA]{LP\_ANA\_PWR\_GLITCH\_RESET\_ENA[1]} & \hyperref[fielddesc:LPANAANAFIBENA]{LP\_ANA\_ANA\_FIB\_ENA[3]} \\\hline
VDDA3 & \hyperref[fielddesc:LPANAPWRGLITCHRESETENA]{LP\_ANA\_PWR\_GLITCH\_RESET\_ENA[2]} & \hyperref[fielddesc:LPANAANAFIBENA]{LP\_ANA\_ANA\_FIB\_ENA[4]} \\\hline
VDDA8 & \hyperref[fielddesc:LPANAPWRGLITCHRESETENA]{LP\_ANA\_PWR\_GLITCH\_RESET\_ENA[3]} & \hyperref[fielddesc:LPANAANAFIBENA]{LP\_ANA\_ANA\_FIB\_ENA[5]} \\\hline

\section{Interrupts}

\chipname{}'s power supply detector can generate the following interrupt signals that will be sent to the \hyperref[mod:intmtrx]{Interrupt Matrix}.

    \begin{itemize}
        \item LP\_ANA\_INTR (only for HP CPU)
        \item LP\_ANA\_LP\_INTR (only for LP CPU)
    \end{itemize}

The interrupt signals are generated by the power supply detector's internal interrupt sources which are listed in Table \ref{tab:psdet-int-sources} with their trigger conditions.

\begin{longtable}{ |l | L{7.3cm} | l | }
\caption{Power Supply Detector's Internal Interrupt Source} \label{tab:psdet-int-sources}
\\ \hline
\rowcolor{lightgray}
\textbf{Internal Interrupt Source} & \textbf{Trigger Condition}  &  \textbf{Interrupt Signal}\\ \hline
\label{int:LPANABODMODE0INT}LP\_ANA\_BOD\_MODE0\_INT  & Triggered when the brown-out detector detects that the voltage is below
the threshold.  & LP\_ANA\_INTR  \\ \hline
\label{int:LPANABODMODE0LPINT}LP\_ANA\_BOD\_MODE0\_LP\_INT & Triggered when the brown-out detector detects that the voltage is below

\end{longtable}
