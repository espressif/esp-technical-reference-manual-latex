\chapter{低功耗处理器}
\label{mod:ulp}


\chipname{} 低功耗处理器 (Low-Power CPU, LP CPU) 是基于 RISC-V ISA 的 32 位处理器，包括整数 (I)、乘法/除法 (M)、原子 (A) 和压缩 (C) 标准扩展。LP CPU 内核具有 2 级有序标量流水线，具有极低的功耗。CPU 内核架构包含中断控制器 (INTC)、调试模块 (DM) 和用于访问存储器和外设的系统总线 (SYS BUS) 接口。

LP CPU 默认处于睡眠状态（详见章节 \ref{sec:sleep}）, 在芯片进入 Deep-sleep 时可保持上电状态（详见章节 \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}），能够访问绝大部分外设和存储器，详情请参考章节 \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}。LP CPU 有两种应用场景：
\begin{itemize}
    \item 功耗不敏感场景：高性能处理器 (HP CPU) 处于工作状态，LP CPU 可协助 HP CPU 完成一些对速度和效率不敏感的控制或计算
    \item 功耗敏感场景：HP CPU 处于掉电状态来节省功耗时，可以唤醒 LP CPU 处理一些外部唤醒事件
\end{itemize}

HP CPU 和 LP CPU 可访问的资源如图 \ref{figure:ulp-overview} 所示：

    \centering
    \includegraphics[width=0.7\textwidth]{./03-ULP/figures/03-ULP-lp-core-overview.pdf}
    \caption{LP CPU 概览}
    \label{figure:ulp-overview}
\end{figure}

在上图中，HP 电源域 (HP Power Domain) 和 LP 电源域 (LP Power Domain) 可分别上下电，当其中一个电源域处于掉电状态时，另一个电源域的 CPU 无法访问该域的资源，只有当该域处于上电状态时才可访问其资源，更多关于电源的信息，请参考章节 \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}。

\section{特性}
\label{sec:ulp-features}

LP CPU 具有以下特性：

\begin{itemize}
    \item 时钟工作频率高达 20 MHz
    \item 支持 1 个向量中断入口
    \item 调试模块 (DM) 符合 RISC-V 调试规范 v0.13，支持通过行业标准的 JTAG/USB 端口连接外部调试器
    \item 硬件触发器符合 RISC-V 调试规范 v0.13，具有 2 个断点/观察点
    \item 32 位 AHB 系统总线，用于访问存储器和外设
    \item 支持核心性能指标事件
    \item 可唤醒 HP CPU 或向 HP CPU 发送中断
    \item 可访问 HP 存储器和 LP 存储器
    \item 可访问所有外设空间

\end{itemize}

\section{中断和异常}

低功耗 CPU 根据 RISC\-/V 指令集手册 V1.10 第二卷“特权架构” 实现了中断和异常的处理。当 CPU 进入中断/异常处理程序时，内核将执行以下操作：
\begin{itemize}
    \item 将当前 PC 保存到 \hyperref[regdesc:LPMEPC]{mepc} CSR 中
    \item 将 \hyperref[regdesc:LPMSTATUS]{mstatus} 的 \hyperref[fielddesc:LPMSTATUSMIE]{MIE} 保存到 \hyperref[regdesc:LPMSTATUS]{mstatus} 的 \hyperref[fielddesc:LPMSTATUSMPIE]{MPIE}
    \item 将当前特权模式保存到 \hyperref[regdesc:LPMSTATUS]{mstatus} 的 \hyperref[fielddesc:LPMSTATUSMPP]{MPP}
    \item 将 \hyperref[regdesc:LPMSTATUS]{mstatus} 的 \hyperref[fielddesc:LPMSTATUSMIE]{MIE}  清 0
    \item 特权模式切换到机器模式
    \item 跳转到处理程序地址
        \begin{itemize}
            \item 对于所有异常原因，跳转到 \hyperref[regdesc:LPMTVEC]{mtvec} CSR 中向量表的基地址
            \item 对于中断，跳转到 $mtvec + 4*30$
        \end{itemize}
    \item 执行 mret 指令后，内核跳转到之前保存在 \hyperref[regdesc:LPMEPC]{mepc} CSR 中的程序计数器，并将 \hyperref[regdesc:LPMSTATUS]{mstatus} 的 \hyperref[fielddesc:LPMSTATUSMIE]{MIE} 恢复为 \hyperref[regdesc:LPMSTATUS]{mstatus} 的 \hyperref[fielddesc:LPMSTATUSMPIE]{MPIE}，特权模式切换到 \hyperref[fielddesc:LPMSTATUSMPP]{MPP} 对应的特权模式
\end{itemize}


当内核启动时，向量表的基地址被初始化为启动地址 0x50000000。启动后可通过写入 \hyperref[regdesc:LPMTVEC]{mtvec} CSR 更改基地址。有关配置和状态寄存器的详细信息，请参阅章节 \ref{sec:ulp-reg-summ}。

内核在复位后从 0x50000080 的地址开始取指。

\subsection{中断}\label{sec:ulp-intr}



\chipname{} 低功耗 CPU 仅支持 1 个中断入口，所有的中断事件都会跳转到该入口。\chipname{} 低功耗 CPU 支持中断的外设源有：
\begin{itemize}
    \item \hyperref[mod:lowpowm]{电源管理单元 (PMU)}
    \item \hyperref[mod:lowpowm]{低功耗定时器 (RTC\_TIMER)}
    \item \hyperref[mod:uart]{低功耗 UART (LP\_UART)}
    \item \hyperref[mod:i2c]{低功耗 I2C (LP\_I2C)}
    \item \hyperref[mod:iomuxgpio]{低功耗 IO MUX (LP IO MUX)}
\end{itemize}

各外设的中断请参考各外设章节。

\subsection{中断处理}

默认情况下，全局禁用中断，因为 \hyperref[regdesc:LPMSTATUS]{mstatus} 的 \hyperref[fielddesc:LPMSTATUSMIE]{MIE} 的复位值为 0。软件必须设置该位以启用全局中断。

\begin{enumerate}
    \item 中断使能
        \begin{itemize}
            \item 置位 \hyperref[regdesc:LPMSTATUS]{mstatus} 的 \hyperref[fielddesc:LPMSTATUSMIE]{MIE} 域，启用全局中断使能
            \item 置位 \hyperref[regdesc:LPMIE]{mie} CSR 的第 30 位，使能中断 30
        \end{itemize}
    \item 中断使能后，LP CPU 即可响应中断，同时去配置相应外设的中断，使其能够触发中断信号给 LP CPU。
    \item 触发中断后 LP CPU 会跳转到 $mtvec + 4*30$。
    \item 进入中断服务程序 (ISR) 后，用户需要查询 \hyperref[regdesc:LPPERIINTERRUPTSOURCEREG]{LPPERI\_INTERRUPT\_SOURCE\_REG} 获取触发中断的外设，并进行处理，注意如果有多个外设触发，依次处理每一个中断，直到所有的外设都处理完成，如果未处理完所有中断，会再次进入中断服务程序。
    \item 中断清除，只需要清除各外设的中断信号即可。
\end{enumerate}

\section{调试}

本节介绍如何调试和测试低功耗 CPU。调试功能由标准 JTAG 管脚提供，并符合 RISC-V 外部调试支持规范 v0.13 (RISC-V External Debug Support, Version 0.13)。

\chipname{} 系统调试结构请参考章节 \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}} 中的图 \ref{fig:riscvcpu-dbg-sys-overview}。

用户与运行调试器（Debugger，例如 GDB）的调试主机（DEBUG HOST，例如笔记本电脑）进行交互。调试器通过调试转换器（Debug Translator，可能包含硬件驱动，例如 OpenOCD）与调试传输硬件（DEBUG TRANSPORT，例如 ESP-Prog 适配器）进行通信。调试传输硬件通过标准 JTAG 接口将调试主机连接到处理器内核的调试传输模块 (JTAG DTM)。JTAG DTM 使用调试模块接口 (DMI) 提供对调试模块 (DM) 的访问。

\chipname{} 有两个 DTM 通过菊花链结构互联，其中 TAP0 用于连接 HP CPU，TAP1 用于连接 LP CPU。

HP 调试模块 (DM) 调试单核 HP CPU。LP 调试模块 (DM) 调试单核 LP CPU。







低功耗 CPU 实现了四个内核调试寄存器：\hyperref[regdesc:LPDCSR]{dcsr}、\hyperref[regdesc:LPDPC]{dpc}、\hyperref[regdesc:LPDSCRATCH0]{dscratch0} 和 \hyperref[regdesc:LPDSCRATCH1]{dscratch1}。所有这些寄存器只能从调试模式访问，如果软件试图在内核未处于调试模式的情况下进行访问，则会触发非法指令异常。

\subsection{特性}
低功耗 CPU 支持以下调试特性：
\begin{itemize}
    \item {向调试器提供有关实现的必要信息}
    \item {支持暂停和恢复 CPU 内核}
    \item {CPU 内核寄存器（包括 CSR）可以由调试器读取/写入}
    \item {可以通过调试器复位 CPU 内核}
    \item {可以在软件断点（植入的断点指令）上暂停 CPU}
    \item {硬件单步调试}
    \item {支持两个硬件触发器（可用作断点/观察点），具体见章节 \ref{sec:ulp-trigger}}
\end{itemize}

\subsection{功能描述}
调试机制遵守 RISC-V 外部调试支持规范 v0.13。有关调试功能的详细介绍，请参考 RISC-V 外部调试支持规范 v0.13 (RISC-V External Debug Support, Version 0.13)。

根据 RISC-V 外部调试支持规范，hart 存在 nonexistent、unavail、 running、halted 等状态，低功耗 CPU 默认处于 unavail 状态，需要用户配置 \hyperref[regdesc:LPPERICPUREG]{LPPERI\_CPU\_REG} 寄存器清除 unavail 状态，从而能够连接低功耗 CPU 进行调试。

\section{硬件触发器}\label{sec:ulp-trigger}

\subsection{特性}
硬件触发器模块提供了断点和观察点功能，供调试使用。硬件触发器具有以下特性：

\begin{itemize}
    \item 2 个独立触发单元
    \item 每个单元都可以配置为匹配程序计数器的地址
    \item 可以暂停执行并将控制权转移给调试器

\end{itemize}

\subsection{功能描述}
\label{sec:ulp-hwt-func-descr}

硬件触发器模块提供了 3 个 CSR，详情见章节 \ref{sec:ulp-hwt-reg}。其中，\hyperref[regdesc:LPTDATA1]{tdata1} 和 \hyperref[regdesc:LPTDATA2]{tdata2} 是抽象 CSR，也就是说它们是用于访问某个触发单元中的内部寄存器的影子寄存器，一次访问一个触发单元。

要选择特定的触发单元，需要将相应的编号 (0-1) 写入 \hyperref[regdesc:LPTSELECT]{tselect} CSR。当写入有效数值时，抽象 CSR \hyperref[regdesc:LPTDATA1]{tdata1} 和 \hyperref[regdesc:LPTDATA2]{tdata2} 将自动匹配该触发单元的内部寄存器。每个触发单元都有两个内部寄存器，即 \hyperref[regdesc:LPMCONTROL]{mcontrol} 和 \hyperref[regdesc:LPMADDRESS]{maddress}，它们分别与 \hyperref[regdesc:LPTDATA1]{tdata1} 和 \hyperref[regdesc:LPTDATA2]{tdata2} 匹配。

向 \hyperref[regdesc:LPTSELECT]{tselect} 写入超过 1 的数值时会导致 \hyperref[regdesc:LPTSELECT]{tselect} 写入 1。

由于软件或调试器可能需要知道所选触发器的类型以便正确解读 \hyperref[regdesc:LPTDATA1]{tdata1} 和 \hyperref[regdesc:LPTDATA2]{tdata2}，因此 \hyperref[regdesc:LPTDATA1]{tdata1} 的 4 个位 (31-28) 对所选触发器的类型进行了编码。此域为只读访问属性，并且值始终为 0x2，代表触发器仅支持地址/数据匹配。因此，可以推断 \hyperref[regdesc:LPTDATA1]{tdata1} 和 \hyperref[regdesc:LPTDATA2]{tdata2} 会通过 \hyperref[regdesc:LPMCONTROL]{mcontrol} 和 \hyperref[regdesc:LPMADDRESS]{maddress} 被解读。RISC-V 外部调试支持规范 v0.13 提供了其他可能值的信息，但是该触发模块仅支持 0x2 类型。

一旦选定了触发单元，就可以通过置位  \hyperref[regdesc:LPMCONTROL]{mcontrol} CSR (\hyperref[regdesc:LPTDATA1]{tdata1}) 中相应的域并将目标地址写入 \hyperref[regdesc:LPMADDRESS]{maddress} CSR (\hyperref[regdesc:LPTDATA2]{tdata2}) 来对该触发单元进行配置。


\subsection{触发执行流程}
\label{sec:ulp-hwt-suggest-op}

当触发器触发引起硬件线程暂停并进入调试模式时 (\hyperref[fielddesc:LPMCONTROLACTION]{action} = 1)：
\begin{itemize}
    \item \hyperref[regdesc:LPDPC]{dpc} 被设置为当前 PC（在解码阶段）
    \item \hyperref[regdesc:LPDCSR]{dcsr} 的 cause 域被设置为 2，表示暂停是由于触发器触发引起
\end{itemize}

\subsection{存储器}

\chipname{} 低功耗 CPU 可以访问的存储器资源有 LP SRAM 和 HP SRAM，详情参考章节 \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}。其中：
\begin{itemize}
    \item LP SRAM：16 KB, 地址范围为 0x5000\_0000 \verb+~+ 0x5000\_3FFF，可进行取指、读数据、写数据等操作
    \item HP SRAM：384 KB, 地址范围为 0x4080\_0000 \verb+~+ 0x4085\_FFFF，可进行取指、读数据、写数据等操作
\end{itemize}
\begin{tiplisting}
LP CPU 访问 HP SRAM 的延迟很大，但可以无延迟访问 LP SRAM。
\end{tiplisting}

LP CPU 支持原子指令集，LP CPU 和 HP CPU 可通过原子指令访问存储器，从而实现内存访问的原子性。关于原子指令集的细节请参考 RISC-V 指令集手册 v2.2 第一卷“非特权架构” (RISC-V Instruction Set Manual Volume I: Unprivileged ISA, Version 2.2)。

注意：只有 HP SRAM 可以支持 HP CPU 和 LP CPU 的原子访问。

章节 \ref{mod:sysmem} \textit{\nameref{mod:sysmem}} 表\ref{tab:sysmem-base-address} 详细列出了低功耗 CPU 可访问的外设及基地址。

\section{事件任务矩阵功能}
在 \chipname{} 中，LP CPU 支持事件任务矩阵 (ETM) 功能，即可以通过任意外设的 ETM 事件触发 LP CPU 的 ETM 任务，或者通过 LP CPU 的 ETM 事件触发任意外设的 ETM 任务。关于 ETM 更多详细信息，请参考章节 \ref{mod:evntaskmatrix} \nameref{mod:evntaskmatrix}。这里仅介绍与 LP CPU 相关的 ETM 任务和 ETM 事件。

LP CPU 可接收的 ETM 任务有：

\begin{itemize}
    \item ULP\_TASK\_WAKEUP\_CPU：唤醒 LP CPU
\end{itemize}

LP CPU 可产生的 ETM 事件有：

\begin{itemize}
    \item ULP\_EVT\_ERR\_INTR：表示 LP CPU 发生异常
    \item ULP\_EVT\_START\_INTR：表示 LP CPU 时钟打开
\end{itemize}


\section{睡眠和唤醒流程}\label{sec:sleep}
\subsection{特性}
\begin{itemize}
    \item 低功耗 CPU 工作在低功耗系统中，可以在高性能系统工作或进入睡眠的时候，独立睡眠、唤醒及工作
    \item 低功耗 CPU 可以根据软件运行状态，主动配置寄存器进入睡眠状态
    \item 低功耗 CPU 可以被如下事件唤醒：
    \begin{itemize}
        \item 主核置位寄存器 \hyperref[fielddesc:PMUHPTRIGGERLP]{PMU\_HP\_TRIGGER\_LP}
        \item LP IO 的中断状态
        \item ETM 事件
        \item RTC 定时器超时
        \item LP UART 接收一定数量的 RX 脉冲，需要使能 \hyperref[fielddesc:LPPERILPUARTWAKEUPEN]{LPPERI\_LP\_UART\_WAKEUP\_EN}
   \end{itemize}
\end{itemize}

\subsection{流程}
低功耗 CPU 默认处于睡眠状态，低功耗 CPU 的睡眠唤醒模块将按照如下流程唤醒开始工作以及进入睡眠。

LP CPU 的唤醒源配置如表 \ref{tab:ulp-wake-sources} 所示。

\begin{figure}[!ht ]
    \centering
    \includegraphics[width=1.0\textwidth]{./03-ULP/figures/03-ULP-lp-core-sleep-wake-flow.pdf}
    \caption{LP CPU 唤醒和睡眠流程}
    \label{figure:ulp-overvew}
\end{figure}

LP CPU 上电后的首次启动取决于 HP CPU 对能唤醒使能信号和唤醒源的配置。
\begin{itemize}
    \item LP CPU 的初始化
        \begin{itemize}
            \item 初始化 LP 存储器。
            \item 启动 LP CPU。由于 LP CPU 的启动依赖于唤醒过程，所以推荐使用 \hyperref[fielddesc:PMUHPTRIGGERLP]{PMU\_HP\_TRIGGER\_LP} 寄存器来开始 LP CPU 的初始化，方法如下：
            \begin{itemize}
               \item 将 \hyperref[fielddesc:PMULPCPUWAKEUPEN]{PMU\_LP\_CPU\_WAKEUP\_EN} 设置为 0x1
               \item 将 \hyperref[fielddesc:PMUHPTRIGGERLP]{PMU\_HP\_TRIGGER\_LP} 设置为 0x1
               \item LP CPU 被唤醒后开始运行
            \end{itemize}
        \end{itemize}
    \item 唤醒启动流程：
    \begin{itemize}
     \item 睡眠唤醒模块接收到唤醒信号向 PMU 发送上电请求。
     \item 如果当前功耗状态（时钟，电源等）已经满足低功耗 CPU 需求，则 PMU 立刻回复完成信号，否则将调整功耗状态再回复完成信号。
     \item 睡眠唤醒模块关闭低功耗 CPU 的 STALL 状态，并打开中断接收使能。
     \item 睡眠唤醒模块开启时钟并释放复位（如果睡眠没有使能复位则忽略该步骤），开始工作。
   \end{itemize}

    \item 睡眠流程：
    \begin{itemize}
    \item 低功耗 CPU 配置寄存器 \hyperref[fielddesc:PMULPCPUSLEEPREQ]{PMU\_LP\_CPU\_SLEEP\_REQ}，睡眠唤醒模块启动睡眠流程。
    \item 如果 \hyperref[fielddesc:PMULPCPUSLPSTALLEN]{PMU\_LP\_CPU\_SLP\_STALL\_EN} 为 1，睡眠唤醒模块则使能低功耗 CPU 的 STALL 状态，如果为 0，则不使能低功耗 CPU 的 STALL 状态。
    \item 睡眠唤醒模块等待 \hyperref[fielddesc:PMULPCPUSLPSTALLWAIT]{PMU\_LP\_CPU\_SLP\_STALL\_WAIT} 个低功耗 CPU 时钟周期后，关闭低功耗 CPU 的时钟，如果 \hyperref[fielddesc:PMULPCPUSLPRESETEN]{PMU\_LP\_CPU\_SLP\_RESET\_EN} 为 1，则使能低功耗 CPU 的复位。
   \end{itemize}
   \end{itemize}
