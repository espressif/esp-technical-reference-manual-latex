\chapter{GPIO Matrix and IO MUX}
\label{mod:iomuxgpio}


\section{Overview}
GPIO matrix is a hardware structure that allows dynamic remapping between the GPIO pins and peripheral input/output signals to provide greater flexibility. IO MUX (Input/Output Multiplexing) is a technology that allows the same pin to perform different functions at different times, allowing dynamic switching of pin functions through configuration.

\chipname{} provides two groups of GPIO matrix and IO MUX:
\begin{itemize}
    \item HP GPIO matrix and HP IO MUX
    \item LP GPIO matrix and LP IO MUX
\end{itemize}

The \chipname{} chip features 29 GPIO pins, numbered from GPIO0 to GPIO28, including
\begin{itemize}
    \item 7 LP GPIO pins (GPIO0$\sim$GPIO6) for either HP or LP peripherals.
    \item 14 HP GPIO pins (GPIO7$\sim$GPIO14, GPIO23$\sim$GPIO28) for HP peripherals only.
    \item 8 dedicated GPIO pins (GPIO15$\sim$GPIO22) for external flash and PSRAM. \textbf{These pins can not be used for other purpose, and therefore are not described in subsequent sections}. For more information regarding these pins, please refer to {\textit{\href{https://www.espressif.com/sites/default/files/documentation/esp32-c5_datasheet_en.pdf}{\chipname{} Datasheet}}} > Section \textit{Pin Mapping Between Chip and Flash/PSRAM}.
\end{itemize}
Each pin can be used as a general-purpose I/O, or be connected to an internal peripheral signal. Together these modules provide highly configurable I/O.
\begin{itemize}
\item Through HP GPIO matrix and HP IO MUX, HP peripheral input signals can be from any GPIO pins, and HP peripheral output signals can be routed to any GPIO pins.
\item Through LP GPIO matrix and LP IO MUX, LP peripheral input signals can be from any LP GPIO pins, and LP peripheral output signals can be routed to any LP GPIO pins.
\end{itemize}

\textbf{Unless otherwise specified, GPIO matrix refers to both LP GPIO matrix and HP GPIO matrix, so as to IO MUX.}

\section{Features}
\subsection{HP GPIO Matrix and HP IO MUX}
\textbf{HP GPIO matrix has the following features:}

\begin{itemize}
\item A full-switching matrix between HP peripheral input/output signals and the GPIO pins
\item 76 HP peripheral input signals sourced from the input of any GPIO pins
\item 77 HP peripheral output signals routed to the output of any GPIO pins
\item Signal synchronization for HP peripheral inputs based on \textbf{HP IO MUX operating clock}
\item GPIO Filter hardware for input signal filtering
\item Glitch Filter hardware for second-time filtering on input signal
\item Sigma delta modulated (SDM) output
\item GPIO simple input and output
\item HP GPIO Wakeup
\end{itemize}

\textbf{HP IO MUX has the following features:}

\begin{itemize}
\item Control of 21 GPIOs (GPIO0$\sim$GPIO14, GPIO23$\sim$GPIO28) for HP peripherals.
\item A configuration register \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO\regindex{n}\_REG} provided for each GPIO pin, to control the pin's input/output, pull-up/pull-down, drive strength, and function selection.
\item Better high-frequency digital performance achieved by routing some digital signals (such as SPI) directly from HP IO MUX to peripherals.

\end{itemize}
\subsection{LP GPIO Matrix and LP IO MUX}
\textbf{LP GPIO matrix has the following features:}

\begin{itemize}
\item GPIO simple input and output
\item LP GPIO Wakeup
\end{itemize}

\textbf{LP IO MUX has the following feature:}

\begin{itemize}
    \item Control of 7 LP GPIO pins (GPIO0$\sim$GPIO6) for LP peripherals.
    \item A configuration register \hyperref[regdesc:LPIOMUXGPIONREG]{LP\_IO\_MUX\_GPIO\regindex{n}\_REG} provided for each LP GPIO pin, to control the pin's input/output, pull-up/pull-down, drive strength, and function selection.
    \item A field \hyperref[fielddesc:LPAONGPIOMUXSEL] {LP\_AON\_GPIO\_MUX\_SEL} provided to select the source of the pin control signals, from HP IO MUX or LP IO MUX.
\end{itemize}

\section{Peripheral Input via GPIO Matrix}
\label{sec:peripheral-input-via-gpio-matrix}

\subsection{Overview}

To receive a peripheral input signal via HP GPIO matrix,
\begin{itemize}
    \item Configure the matrix to source the peripheral input signal from one of the 21 GPIOs (0$\sim$14, 23$\sim$28), see Table \ref{tab:iomuxgpio-periph-signals-via-hp-gpio-matrix-en}.
    \item Configure the peripheral signal to receive input signal via HP GPIO matrix.
    \item Configure the GPIO pin to be controlled by HP IO MUX.
\end{itemize}

For detailed configuration, see Figure \ref{fig:iomux-gpio-matrix-overview-detail} and Section \ref{sec:functional-description}.

As shown in Figure \ref{fig:iomux-gpio-matrix-overview-detail}, when GPIO matrix is used to input a signal from the pin, all external input signals are sourced from the GPIO pins and then filtered by the GPIO Filter, as shown in Step  \ref{step:2} in Section \ref{sec:functional-description}.

The Glitch Filter is only available in HP GPIO matrix. The Glitch Filter hardware can filter eight of the output signals from the GPIO Filter, and the other unselected signals go directly to the GPIO SYNC hardware, as shown in Step \ref{step:3} in Section \ref{sec:functional-description}.

All signals filtered by the GPIO Filter hardware or the Glitch Filter hardware are synchronized by the GPIO SYNC hardware to IO MUX operating clock and then enter the GPIO matrix, see Section \ref{sec:inputconfig}. Such signal filtering and synchronization features apply to all GPIO matrix signals but do not apply when using the IO MUX.

\subsection{Signal Synchronization}
\label{sec:inputconfig}

Only HP GPIO matrix supports this signal synchronization function. Figure \ref{Figure:gpio_input_sync} shows the functionality of GPIO SYNC. In the figure, negative sync and positive sync mean GPIO input is synchronized on falling edge and on rising edge of HP IO MUX operating clock respectively.

\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=1.0\textwidth]{\modulefiles/figures/gpio_input_sync_20220309}}
    \caption{GPIO Input Synchronized on Rising Edge or on Falling Edge of HP IO MUX Operating Clock}
    \label{Figure:gpio_input_sync}
\end{figure}

The synchronization function is disabled by default by the synchronizer. But when an asynchronous peripheral signal is connected to the pin, the signal should be synchronized by the two-level synchronizer (i.e., the first-level synchronizer and the second-level synchronizer as shown in Figure \ref{Figure:gpio_input_sync}) to lower the probability of causing metastability.

\subsection{GPIO Filter}
\label{IOMUXGPIOFilter}
Only HP GPIO matrix supports this GPIO Filter function. When this function is enabled, only signals with a valid width of more than two clock cycles can be sampled, as illustrated in Figure  \ref{Figure:gpio_filter}.
\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/gpio_filter}}
    \caption{GPIO Filter Timing of GPIO Input Signals}
    \label{Figure:gpio_filter}
\end{figure}

\subsection{Glitch Filter}
\label{IOMUXGlitchFilter}
The Glitch Filter function is exclusive to the HP GPIO matrix. The Glitch Filter hardware supports eight channels, each of which selects one signal from the 21 (0$\sim$14, 23$\sim$28) output signals generated by the GPIO Filter hardware. It then conducts a second round of filtering on the selected signal. This Glitch Filter hardware can be used to filter slow-speed signals. For more information, see Step \ref{step:3} in Section \ref{sec:prgramming-procedure-for-hp}.

\subsection{Simple GPIO Input}
\label{IOMUXSimpleGPIOInput}
Both the HP GPIO matrix and LP GPIO matrix support the Simple GPIO Input function. Enabling this function allows the direct reading of the input value of a GPIO pin at any time, without the need to route the GPIO input to any peripherals.

For HP GPIO matrix, to implement simple GPIO input, follow the steps below:
\begin{itemize}
    \item Set \hyperref[fielddesc:IOMUXGPIONMCUIE] {IO\_MUX\_GPIO\regindex{x}\_MCU\_IE} in \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO\regindex{x}\_REG}, to enable pin input.
    \item Read the GPIO input from  \hyperref[regdesc:GPIOINREG]{GPIO\_IN\_REG[\regindex{x}]}.
\end{itemize}

For LP GPIO matrix, to implement simple GPIO input, follow the steps below:
\begin{itemize}
    \item Set \hyperref[fielddesc:LPIOMUXGPIONFUNIE] {LP\_IOMUX\_PAD\regindex{x}\_FUN\_IE} in \hyperref[regdesc:LPIOMUXGPIONREG]{LP\_IO\_MUX\_GPIO\regindex{x}\_REG}, to enable pin input.
    \item Read the GPIO input from  \hyperref[regdesc:LPGPIOINREG]{LP\_GPIO\_IN\_REG[\regindex{x}]}.
\end{itemize}

\subsection{Programming Procedure}
\label{sec:functional-description}
\subsubsection{HP GPIO Matrix}
\label{sec:prgramming-procedure-for-hp}
To read GPIO pin \regindex{X}\hyperref[input-note-1]{$^1$} into HP peripheral signal \regindex{Y}, follow the steps below:

\begin{enumerate}
    \item Configure \hyperref[regdesc:GPIOFUNCMINSELCFGREG]{GPIO\_FUNC\regindex{y}\_IN\_SEL\_CFG\_REG} corresponding to HP peripheral signal \regindex{Y} in HP GPIO matrix:
    \begin{itemize}
        \item Set \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_SIG\regindex{y}\_IN\_SEL} to enable peripheral signal input via HP GPIO matrix.
        \item Set \hyperref[fielddesc:GPIOFUNCMINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} to the desired GPIO pin, i.e., \regindex{X} here.
    \end{itemize}
    {\bfseries Note:} some peripheral signals have no valid \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_SIG\regindex{y}\_IN\_SEL} bit, namely, these peripherals can only receive input signals via HP GPIO matrix.
    \item \label{step:2} Optionally enable the GPIO Filter for pin input signals by setting \hyperref[fielddesc:IOMUXGPIONFILTEREN] {IO\_MUX\_GPIO\regindex{x}\_FILTER\_EN}.
\item \label{step:3}  Enable Glitch Filter feature as follows:
\begin{itemize}
    \item Configure \hyperref[fielddesc:GPIOEXTFILTERCH0INPUTIONUM]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_INPUT\_IO\_NUM} to \regindex{m}. \regindex{n} (0$\sim$7) represents the channel number. \regindex{m} (0$\sim$14, 23$\sim$28) represents the GPIO pin number.
    \item Configure \hyperref[fielddesc:GPIOEXTFILTERCH0WINDOWWIDTH]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_WINDOW\_WIDTH} to \textbf{VALUE1} and \hyperref[fielddesc:GPIOEXTFILTERCH0WINDOWTHRES]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_WINDOW\_THRES} to \textbf{VALUE2}. During \textbf{VALUE1} + 1 cycles, if there are \textbf{VALUE2} + 1 input signals that do not match the current output signal value, the Glitch Filter hardware inverts the output signal. \hyperref[fielddesc:GPIOEXTFILTERCH0WINDOWWIDTH]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_WINDOW\_WIDTH} and  \hyperref[fielddesc:GPIOEXTFILTERCH0WINDOWTHRES]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_WINDOW\_THRES} can be configured to the same value \textbf{VALUE3}, then only signals with a width greater than \textbf{VALUE3} + 1 clock cycles will be sampled.
    \item Set  \hyperref[fielddesc:GPIOEXTFILTERCH0EN]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_EN} to enable channel \regindex{n}.
\end{itemize}

An example is shown in Figure \ref{Figure:glitch_filter}, where \hyperref[fielddesc:GPIOEXTFILTERCH0WINDOWWIDTH]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_WINDOW\_WIDTH} is configured to 3 and \hyperref[fielddesc:GPIOEXTFILTERCH0WINDOWTHRES]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_WINDOW\_THRES} to 2. The output signal value (signal\_out) keeps as ``0" in the four clock cycles before T1. The input signal value (signal\_in) has been ``1" for three clock cycles in the same period, then the output signal is inverted to ``1" after T1.
\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/figure-glitch-filter-20220630}}
    \caption{Glitch Filter Timing Example}
    \label{Figure:glitch_filter}
\end{figure}
    \item \label{step:4} Synchronize GPIO input signals. To do so, please set \hyperref[regdesc:GPIOPINNREG]{GPIO\_PIN\regindex{x}\_REG} corresponding to GPIO pin \regindex{X} as follows:
    \begin{itemize}
        \item Set \hyperref[fielddesc:GPIOPINNSYNC1BYPASS]{GPIO\_PIN\regindex{x}\_SYNC1\_BYPASS} to enable input signal synchronized on rising edge or on falling edge in the first-level synchronization, see Figure \ref{Figure:gpio_input_sync}.
        \item Set \hyperref[fielddesc:GPIOPINNSYNC2BYPASS]{GPIO\_PIN\regindex{x}\_SYNC2\_BYPASS} to enable input signal synchronized on rising edge or on falling edge in the second-level synchronization, see Figure \ref{Figure:gpio_input_sync}.
    \end{itemize}
    \item Configure HP IO MUX register to enable pin input. For this end, please set \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO\regindex{x}\_REG} corresponding to GPIO pin \regindex{X} as follows:
    \begin{itemize}
        \item Set \hyperref[fielddesc:IOMUXGPIONFUNIE]{IO\_MUX\_GPIO\regindex{x}\_FUN\_IE} to enable input\hyperref[input-note-2]{$^2$}$^,$\hyperref[input-note-3]{$^3$}.
        \item Set or clear \hyperref[fielddesc:IOMUXGPIONFUNWPU]{IO\_MUX\_GPIO\regindex{x}\_FUN\_WPU} and \hyperref[fielddesc:IOMUXGPIONFUNWPD]{IO\_MUX\_GPIO\regindex{x}\_FUN\_WPD} as desired to enable or disable pull-up and pull-down resistors.
    \end{itemize}
\end{enumerate}

\begin{tiplisting}
\vspace{-2em}
\begin{enumerate}
\item \label{input-note-1} One input pin can be connected to multiple peripheral input signals.
\item \label{input-note-2} The input signal can be inverted by configuring \hyperref[fielddesc:GPIOFUNCNININVSEL]{GPIO\_FUNC\regindex{y}\_IN\_INV\_SEL}.
\item \label{input-note-3} It is possible to have an HP peripheral read a constantly low or constantly high input value without connecting this input to a pin. This can be done by selecting a special \hyperref[fielddesc:GPIOFUNCMINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} input, instead of a GPIO number:
\begin {itemize}
\item When \hyperref[fielddesc:GPIOFUNCMINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} is set to 0x40, input signal is always 0.
\item When \hyperref[fielddesc:GPIOFUNCMINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} is set to 0x60, input signal is always 1.
\end{itemize}
\end{enumerate}
\end{tiplisting}

\textbf{Programming Example}\\

To connect UART0 RXD input signal (UORXD\_in, signal index 6) to GPIO7, please follow the steps below.

\begin{enumerate}
\item Set \hyperref[regdesc:GPIOFUNCMINSELCFGREG]{GPIO\_SIG6\_IN\_SEL} in \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_FUNC6\_IN\_SEL\_CFG\_REG} to enable peripheral signal input via HP GPIO matrix.
\item Set \hyperref[fielddesc:GPIOFUNCMINSEL]{GPIO\_FUNC6\_IN\_SEL} in \hyperref[regdesc:GPIOFUNCMINSELCFGREG]{GPIO\_FUNC6\_IN\_SEL\_CFG\_REG} to 7, i.e., select GPIO7.
\item Set \hyperref[fielddesc:IOMUXGPIONFUNIE]{IO\_MUX\_GPIO7\_FUN\_IE} in \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO7\_REG} to enable pin input.
\end{enumerate}

\subsubsection{LP GPIO Matrix}

For inputs, LP GPIO matrix only supports the GPIO wakeup function. For LP GPIO input wakeup programming, please refer to Section \ref{sec:lpgpio_wakeup}.


\section{Peripheral Output via GPIO Matrix}
\label{sec:peripheral-output-via-gpio-matrix}

\subsection{Overview}

To output a signal from a peripheral via GPIO matrix:
\begin{itemize}
    \item Configure the HP GPIO matrix to route HP peripheral output signals (only signals with a name assigned in the column ``Output signal" in Table \ref{tab:iomuxgpio-periph-signals-via-hp-gpio-matrix-en}) to one of the 21 GPIOs (0$\sim$14, 23$\sim$28).
    \item Then the output signal is routed from the peripheral into GPIO matrix and then into IO MUX. IO MUX must be configured to set the chosen pin to GPIO function. This enables the GPIO output signal to be connected to the pin.
\end{itemize}

For the detailed programming procedure, see Section \ref{sec:programming-procedure-hp-gpio-matrix-output}.

\begin{tiplisting}
There is a range of peripheral output signals (97$\sim$100 in Table \ref{tab:iomuxgpio-periph-signals-via-hp-gpio-matrix-en}) which are not connected to any peripheral, but to the input signals (97$\sim$100) directly.
\end{tiplisting}
\vspace{-2em}


\subsection{Simple GPIO Output}\label{sec:simple-gpio-output}

GPIO matrix can also be used for simple GPIO output. For this case, one GPIO pin can be configured to directly output the desired value, without routing any peripheral output to this pin.

Follow the steps below to configure HP GPIO matrix for simple GPIO output:

\begin{itemize}
    \item Set \hyperref[fielddesc:GPIOFUNCNOUTSEL]{GPIO\_FUNC\regindex{x}\_OUT\_SEL} with a special peripheral index 256 (0x100).
    \item Configure the corresponding bit in \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG} to the desired GPIO output value.
\end{itemize}

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
\item The bits \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}[0$\sim$14, 23$\sim$28] correspond to GPIO0$\sim$GPIO14, GPIO23$\sim$GPIO28.
\item Recommended operation: use \hyperref[fielddesc:GPIOOUTW1TS]{GPIO\_OUT\_W1TS} and \hyperref[fielddesc:GPIOOUTW1TC]{GPIO\_OUT\_W1TC} to set or clear \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}.
\end{itemize}
\end{tiplisting}

LP GPIO matrix supports only the Simple GPIO Output function. There is no need to configure function selection registers. Just configure the corresponding bit in \hyperref[regdesc:LPGPIOOUTREG]{LP\_GPIO\_OUT\_REG} to the desired GPIO output value.
\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
    \item The bits \hyperref[regdesc:LPGPIOOUTREG]{LP\_GPIO\_OUT\_REG}[0$\sim$6] correspond to GPIO0$\sim$GPIO6.
    \item Recommended operation: use \hyperref[fielddesc:LPGPIOOUTW1TS]{LP\_GPIO\_OUT\_DATA\_W1TS} and \hyperref[fielddesc:LPGPIOOUTW1TC]{LP\_GPIO\_OUT\_DATA\_W1TC} to set or clear \hyperref[regdesc:LPGPIOOUTREG]{LP\_GPIO\_\\OUT\_REG}.
\end{itemize}
\end{tiplisting}

\subsection{Sigma Delta Modulated Output (SDM)}

\subsubsection{Functional Description}

Four HP peripheral output signals (index: 76$\sim$79 in Table \ref{tab:iomuxgpio-periph-signals-via-hp-gpio-matrix-en}) support 1-bit second-order sigma delta modulation. By default the output is enabled for these four channels. This Sigma Delta modulator can also output PDM (pulse density modulation) signal with configurable duty cycle. The function is:

\[
    H(z)=\textrm{X(z)z$^{-1}$} + \textrm{E(z)(1-z$^{-1}$)$^{2}$}
    \]

E(z) is quantization error and X(z) is the input.\\
This modulator supports scaling down of IO MUX operating clock by divider 1$\sim$256:
\begin{itemize}
    \item Set  \hyperref[fielddesc:GPIOEXTSIGMADELTACLKEN]{GPIO\_EXT\_SIGMADELTA\_CLK\_EN} to enable the modulator clock.
    \item Configure \hyperref[fielddesc:GPIOEXTSDNPRESCALE]{GPIO\_EXT\_SD\regindex{n}\_PRESCALE} (\regindex{n} = 0$\sim$3 for the four channels).
\end{itemize}

After scaling, the clock cycle is equal to one pulse output cycle from the modulator.

\hyperref[fielddesc:GPIOEXTSDNIN]{GPIO\_EXT\_SD\regindex{n}\_IN} is a signed number with a range of [-128, 127] and is used to control the duty cycle$^{1}$ of PDM output signal.

\begin{itemize}
    \item \hyperref[fielddesc:GPIOEXTSDNIN]{GPIO\_EXT\_SD\regindex{n}\_IN} = -128, the duty cycle of the output signal is \begin{math}0\% \end{math}.
    \item \hyperref[fielddesc:GPIOEXTSDNIN]{GPIO\_EXT\_SD\regindex{n}\_IN} = 0, the duty cycle of the output signal is near \begin{math}50\% \end{math}.
    \item \hyperref[fielddesc:GPIOEXTSDNIN]{GPIO\_EXT\_SD\regindex{n}\_IN} = 127, the duty cycle of the output signal is near \begin{math}100\% \end{math}.
\end{itemize}

The formula for calculating PDM signal duty cycle is shown as below:

    \[
    Duty\_Cycle=\frac{\hyperref[fielddesc:GPIOEXTSDNIN]{GPIO\_EXT\_SD\regindex{n}\_IN} + 128} {256}
    \]

\begin{tiplisting}
For PDM signals, duty cycle refers to the percentage of high level cycles to the whole statistical period (several pulse cycles, for example, 256 pulse cycles).
\end{tiplisting}

\subsubsection{SDM Configuration}

The configuration of SDM is shown below:

\begin{itemize}
    \item Route one of SDM outputs to a pin via HP GPIO matrix, see Section  \ref{sec:programming-procedure-hp-gpio-matrix-output}.
    \item Enable the modulator clock by setting \hyperref[fielddesc:GPIOEXTSIGMADELTACLKEN]{GPIO\_EXT\_SIGMADELTA\_CLK\_EN}.
    \item Configure the divider value by setting \hyperref[fielddesc:GPIOEXTSDNPRESCALE]{GPIO\_EXT\_SD\regindex{n}\_PRESCALE}.
    \item Configure the duty cycle of SDM output signal by setting  \hyperref[fielddesc:GPIOEXTSDNIN]{GPIO\_EXT\_SD\regindex{n}\_IN}.
\end{itemize}

\subsection{Programming Procedure}

\subsubsection{HP GPIO Matrix}
\label{sec:programming-procedure-hp-gpio-matrix-output}

The output signals with a name assigned in the column ``Output signal" in Table \ref{tab:iomuxgpio-periph-signals-via-hp-gpio-matrix-en} can be set to go through the HP GPIO matrix into HP IO MUX and then to a pin. Figure \ref{fig:iomux-gpio-matrix-overview-detail} illustrates the configuration.

To output peripheral signal \regindex{Y} to a particular GPIO pin \regindex{X}, follow the steps below:

\begin{enumerate}
    \item Configure \hyperref[regdesc:GPIOFUNCNOUTSELCFGREG]{GPIO\_FUNC\regindex{x}\_OUT\_SEL\_CFG\_REG} and  \hyperref[regdesc:GPIOENABLEREG]{GPIO\_ENABLE\_REG}[\regindex{x}] corresponding to GPIO pin \regindex{X} in HP GPIO matrix. Recommended operation: use corresponding \hyperref[regdesc:GPIOENABLEW1TSREG]{W1TS} (write 1 to set) and \hyperref[regdesc:GPIOENABLEW1TCREG]{W1TC} (write 1 to clear) registers to set or clear \hyperref[regdesc:GPIOENABLEREG]{GPIO\_ENABLE\_REG}.


    \begin{itemize}
        \item Set the \hyperref[fielddesc:GPIOFUNCNOUTSEL]{GPIO\_FUNC\regindex{x}\_OUT\_SEL} field in register \hyperref[regdesc:GPIOFUNCNOUTSELCFGREG]{GPIO\_FUNC\regindex{x}\_OUT\_SEL\_CFG\_REG} to the index of the desired peripheral output signal \regindex{Y}.
        \item If the signal should always be enabled as an output, set the bit \hyperref[fielddesc:GPIOFUNCNOESEL]{GPIO\_FUNC\regindex{x}\_OE\_SEL} and the corresponding bit in \hyperref[regdesc:GPIOENABLEW1TSREG]{GPIO\_ENABLE\_W1TS\_REG}. To have the output enable signal decided by internal logic (for example, the SPIQ\_oe in column ``Output enable signal when \hyperref[fielddesc:GPIOFUNCNOESEL]{GPIO\_FUNC\regindex{n}\_OE\_SEL} = 0" in Table \ref{tab:iomuxgpio-periph-signals-via-hp-gpio-matrix-en}), clear the \hyperref[fielddesc:GPIOFUNCNOESEL]{GPIO\_FUNC\regindex{x}\_OE\_SEL} bit instead.
       \item Set the corresponding bit in register \hyperref[regdesc:GPIOENABLEW1TCREG]{GPIO\_ENABLE\_W1TC\_REG} to disable the output from the GPIO pin.
    \end{itemize}
    \item For an open drain output, set the field \hyperref[fielddesc:GPIOPINNPADDRIVER]{GPIO\_PIN\regindex{x}\_PAD\_DRIVER}.
    \item Configure HP IO MUX register to enable output via HP GPIO matrix. Set \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO\regindex{x}\_REG} corresponding to GPIO pin \regindex{X} as follows:
    \begin{itemize}
        \item Set the field \hyperref[fielddesc:IOMUXGPIONMCUSEL]{IO\_MUX\_GPIO\regindex{x}\_MCU\_SEL} to desired HP IO MUX function corresponding to GPIO pin \regindex {X}. This is Function 1 (GPIO function), numeric value 1, for all pins.
        \item Set the \hyperref[fielddesc:IOMUXGPIONFUNDRV]{IO\_MUX\_GPIO\regindex{x}\_FUN\_DRV} field to the desired value for output strength (0$\sim$3). The higher the drive strength, the more current can be sourced/sunk from the pin.
        \item If using open drain mode, set or clear the \hyperref[fielddesc:IOMUXGPIONFUNWPU]{IO\_MUX\_GPIO\regindex{x}\_FUN\_WPU} and \hyperref[fielddesc:IOMUXGPIONFUNWPD]{IO\_MUX\_GPIO\regindex{x}\_FUN\_WPD} bits to enable or disable the internal pull-up and pull-down resistors.
    \end{itemize}
    \item Enable hysteresis function:
    \begin{itemize}
        \item See Section \ref{PadHysEn}.
    \end{itemize}
\end{enumerate}

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
\item The output signal from a single peripheral can be sent to multiple pins simultaneously.
\item The output signal can be inverted by setting \hyperref[fielddesc:GPIOFUNCNOUTINVSEL]{GPIO\_FUNC\regindex{x}\_OUT\_INV\_SEL}.
\end{itemize}
\end{tiplisting}

\section{Direct Input and Output via IO MUX}

\subsection{Overview}

Some digital signals such as SPI and JTAG can bypass GPIO matrix for better high-frequency digital performance. In this case, IO MUX is used to connect these pins directly to peripherals. This option is less flexible than routing signals via GPIO matrix, as the IO MUX register for each GPIO pin can only select from a limited number of functions, but high-frequency digital performance can be improved.

\chipname{} provides 7 GPIO pins (GPIO0$\sim$GPIO6) with low power (LP) capabilities. These pins can be controlled by either HP IO MUX or LP IO MUX. If controlled by LP IO MUX, these pins will bypass HP IO MUX and HP GPIO matrix for the use by peripherals in LP system.

When configured as LP GPIOs, the pins can still be controlled by the peripherals in LP system during chip Deep-sleep, and wake up the chip from Deep-sleep.

\subsection{Functional Description}

The pins with LP functions (GPIO0$\sim$GPIO6) are controlled by bit[\regindex{n}] of LP\_AON\_GPIO\_MUX\_SEL in LP\_AON\_GPIO\_MUX\_REG (\regindex{n} = 0$\sim$6). By default, all these bits are set to 0, routing all input/output signals via HP IO MUX.

If bit[\regindex{n}] is set to 1 in LP\_AON\_GPIO\_MUX\_SEL, then input/output signals are controlled by LP IO MUX. In this mode, \hyperref[regdesc:LPIOMUXGPIONREG]{LP\_IO\_MUX\_GPIO\regindex{n}\_REG} is used to control the LP GPIO pins. See \ref{tab:LP-IO-MUX-EN} for the LP functions of each LP GPIO pin. Note that \hyperref[regdesc:LPIOMUXGPIONREG]{LP\_IO\_MUX\_GPIO\regindex{n}\_REG} applies the LP GPIO pin numbering. In \chipname{}, the numbering of LP GPIO pins is the same as that of the HP GPIO pins.

\subsubsection{HP IO MUX}
Two fields must be configured in order to bypass HP GPIO matrix for HP peripheral input signals:

\begin{enumerate}
\item \hyperref[fielddesc:IOMUXGPIONMCUSEL]{IO\_MUX\_GPIO\regindex{n}\_MCU\_SEL} for the GPIO pin must be set to the required pin function. For the list of pin functions, please refer to Table \ref{tab:HP-IO-MUX-EN}.
\item Clear \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_SIG\regindex{n}\_IN\_SEL} to route the input directly to the peripheral.
\end{enumerate}

To bypass HP GPIO matrix for HP peripheral output signals, \hyperref[fielddesc:IOMUXGPIONMCUSEL]{IO\_MUX\_GPIO\regindex{n}\_MCU\_SEL} for the GPIO pin must be set to the required pin function.

\subsubsection{LP IO MUX}

To bypass LP GPIO matrix for LP peripheral input/output signals, \hyperref[fielddesc:LPIOMUXGPIONMCUSEL] {LP\_IO\_MUX\_GPIO\regindex{n}\_MCU\_SEL} for the GPIO pin must be set to the required pin function. For the list of pin functions, please refer to Table \ref{tab:LP-IO-MUX-EN}.

\begin{tiplisting}
Not all signals can be directly connected to peripheral via IO MUX. Some input/output signals can only be connected to peripheral via GPIO matrix. See Table \ref{tab:iomuxgpio-periph-signals-via-hp-gpio-matrix-en}.
\end{tiplisting}

\section{Pin Hold Feature}\label{PadHold}

Each GPIO pin has an individual hold function controlled by Power Management Unit (PMU) or registers. When the pin is set to hold, the state is latched at that moment and will not change no matter how the internal signals change or how the IO MUX/GPIO configuration is modified. Users can use the Hold function for the pins to retain the pin state through a core reset triggered by watchdog time-out or Deep-sleep events.

The Hold state of each GPIO pin is controlled by the result of OR operation of the pin's Hold enable signal and the global Hold enable signal.
\begin{itemize}
    \item Hold signal of each individual pin:
    \begin{itemize}
        \item \hyperref[regdesc:LPAONGPIOHOLD0REG]{LP\_AON\_GPIO\_HOLD0\_REG[\regindex{n}]} (\regindex{n} = 0$\sim$14, 23$\sim$28) controls the Hold signal of each pin.
        \end{itemize}
    \item Global Hold signal:
        \begin{itemize}
            \item Global Hold signal of HP GPIO pins:
            \begin{itemize}
                \item PMU\_TIE\_HIGH\_HP\_PAD\_HOLD\_ALL: enables the global Hold signal of all HP GPIO pins.
                \item PMU\_TIE\_LOW\_HP\_PAD\_HOLD\_ALL: disables the global Hold signal of all HP GPIO pins.
            \end{itemize}
            \item Global Hold signal of LP GPIO pins:
            \begin{itemize}
                \item PMU\_TIE\_HIGH\_LP\_PAD\_HOLD\_ALL: enables the global Hold signal of all LP GPIO pins.
                \item PMU\_TIE\_LOW\_LP\_PAD\_HOLD\_ALL: disables the global Hold signal of all LP GPIO pins.
            \end{itemize}
        \end{itemize}
    \end{itemize}
Enable or disable the Hold feature of the pins by one of the following ways:
\begin{itemize}
    \item \textbf{Method 1: enable or disable the Hold feature of individual pins:}\\
    To maintain the pin's input/output status in Deep-sleep, set  \hyperref[regdesc:LPAONGPIOHOLD0REG]{LP\_AON\_GPIO\_HOLD0\_REG[\regindex{n}]} (where \regindex{n} = 0$\sim$14, 23$\sim$28 corresponds to GPIO0$\sim$GPIO14, GPIO23$\sim$GPIO28). To disable the hold function after waking up, clear the bit[\regindex{n}].
    \item \textbf{Method 2: enable or disable the Hold feature of all HP/LP pins:}\\
    \begin{itemize}
        \item HP Pins\\
        Set \hyperref[fielddesc:PMUTIEHIGHHPPADHOLDALL]{PMU\_TIE\_HIGH\_HP\_PAD\_HOLD\_ALL} to maintain the input/output status of all HP pins and set \hyperref[fielddesc:PMUTIELOWHPPADHOLDALL]{PMU\_TIE\_LOW\_HP\_PAD\_HOLD\_ALL} to disable the hold function for all HP pins.
        \item LP Pins\\
        Set \hyperref[fielddesc:PMUTIEHIGHLPPADHOLDALL]{PMU\_TIE\_HIGH\_LP\_PAD\_HOLD\_ALL} to maintain the input/output status of all LP pins and set \hyperref[fielddesc:PMUTIELOWLPPADHOLDALL]{PMU\_TIE\_LOW\_LP\_PAD\_HOLD\_ALL} to disable the hold function for all LP pins.
    \end{itemize}
    \item \textbf{Method 3: Use PMU Task}\\
    Configure the PMU task to hold the HP pins before Deep-sleep. In Deep-sleep, the HP pins would be hold automatically by the PMU.
\end{itemize}


\section{Hysteresis Characteristics}\label{PadHysEn}

Each GPIO pin has hysteresis functionality. When hysteresis is not enabled, as shown in Figure \ref{Figure:hysteresis_fig1}, the level flip of the signal (C) input to the chip from the PAD has only one threshold (Vt, about 1.7 V). When the voltage on the PAD is higher than Vt, the level on the C is high. Otherwise, it is low. However, noise on the PAD may affect the signal on C.

\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.8\textwidth]{\modulefiles/figures/hysteresis_figure1_20230220}}
    \caption{Example of Level Flip on the Chip Pad — Hysteresis Function Not Enabled}
    \label{Figure:hysteresis_fig1}
\end{figure}

When hysteresis is enabled, as shown in Figure \ref{Figure:hysteresis_fig2}, the level flip of C has two thresholds, high-level threshold (Vth, about 1.7 V) and low-level threshold (Vtl, about 1.4 V). When the voltage of pad goes from low to high, if the voltage is higher than Vth, the level of C is high. When the voltage of PAD goes from high to low, if the voltage is lower than Vtl, the level of C is low. When the voltage of PAD is between Vth and Vtl, the level of C does not change. The hysteresis function plays an anti-interference role by mitigating the impact of noise, consequently reducing the level flip time of C.

\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.8\textwidth]{\modulefiles/figures/hysteresis_figure2_20230220}}
    \caption{Example of Level Flip on the Chip Pad — Hysteresis Function Enabled}
    \label{Figure:hysteresis_fig2}
\end{figure}

To enable the hysteresis function, follow the steps below:

\begin{itemize}
    \item \hyperref[fielddesc:IOMUXGPIONHYSSEL]{IO\_MUX\_GPIO\regindex{n}\_HYS\_SEL} = 0 (\regindex{n} = 0$\sim$14, 23$\sim$28, corresponding to GPIO0$\sim$GPIO14, GPIO23$\sim$GPIO28)
    \begin{itemize}
        \item Set \hyperref[fielddesc:EFUSEHYSENPAD0]{EFUSE\_HYS\_EN\_PAD} to enable the hysteresis function for GPIO0$\sim$GPIO14 and GPIO23$\sim$GPIO28.
        \item Or clear \hyperref[fielddesc:EFUSEHYSENPAD1]{EFUSE\_HYS\_EN\_PAD} to disable the hysteresis function for GPIO0$\sim$GPIO14 and GPIO23$\sim$GPIO28.
    \end{itemize}
    \item \hyperref[fielddesc:IOMUXGPIONHYSSEL]{IO\_MUX\_GPIO\regindex{n}\_HYS\_SEL} = 1 (\regindex{n} = 0$\sim$14, 23$\sim$28, corresponding to GPIO0$\sim$GPIO14, GPIO23$\sim$GPIO28)
    \begin{itemize}
        \item Set \hyperref[fielddesc:IOMUXGPIONHYSEN]{IO\_MUX\_GPIO\regindex{n}\_HYS\_EN} to enable the hysteresis function for GPIO\regindex{n}.
        \item Clear \hyperref[fielddesc:IOMUXGPIONHYSEN]{IO\_MUX\_GPIO\regindex{n}\_HYS\_EN} to disable the  hysteresis function for GPIO\regindex{n}.
    \end{itemize}
\end{itemize}

\textbf{Recommended Operation}:\\
\begin{itemize}
    \item Set \hyperref[fielddesc:IOMUXGPIONHYSSEL]{IO\_MUX\_GPIO\regindex{n}\_HYS\_SEL}.
    \item Then enable or disable the the hysteresis function for GPIO\regindex{n} using \hyperref[fielddesc:IOMUXGPIONHYSEN]{IO\_MUX\_GPIO\regindex{n}\_HYS\_EN}.
\end{itemize}

    The default configuration of each pin after reset:

\begin{itemize}
\item \textbf{0} - IE = 0 (input disabled)
\item \textbf{1} - IE = 1 (input enabled)
\item \textbf{3} - IE = 1, WPU = 1 (input enabled, pull-up resistor enabled)
\item \textbf{4} - IE = 1, OE = 1 (input enabled, output enabled)
\item \textbf{1*} - If \hyperref[fielddesc:EFUSEDISPADJTAG]{EFUSE\_DIS\_PAD\_JTAG} = 1, the pin MTCK is left floating after reset, i.e., IE = 1. If \hyperref[fielddesc:EFUSEDISPADJTAG]{EFUSE\_DIS\_PAD\_JTAG} = 0, the pin MTCK is connected to internal pull-up resistor, i.e., IE = 1, WPU = 1.

\section{Event Task Matrix Function}

In \chipname{}, GPIO supports ETM function, that is, the ETM task of GPIO can be triggered by the ETM event of any peripheral, or the ETM task of any peripheral can be triggered by the ETM event of GPIO. For more details about ETM, please refer to Chapter \ref{mod:evntaskmatrix} \textit{\nameref{mod:evntaskmatrix}}. Only ETM tasks and ETM events related to GPIO are introduced here.

The GPIO ETM provides eight task channels \regindex{x} (0$\sim$7). The ETM tasks that each task channel can receive are:

\begin{itemize}
    \item GPIO\_TASK\_CH\regindex{x}\_SET: GPIO goes high when triggered.
    \item GPIO\_TASK\_CH\regindex{x}\_CLEAR: GPIO goes low when triggered.
    \item GPIO\_TASK\_CH\regindex{x}\_TOGGLE: GPIO toggles level when triggered.
\end{itemize}

Below is an example to configure task channel  \regindex{x} to control GPIO\regindex{y}:

\begin{itemize}
    \item Configure \hyperref[fielddesc:IOMUXGPIONMCUSEL]{IO\_MUX\_GPIO\regindex{y}\_MCU\_SEL} to 1, to select Function 1 listed in Table \ref{tab:HP-IO-MUX-EN}.
    \item Configure \hyperref[regdesc:GPIOENABLEREG]{GPIO\_ENABLE\_REG[\regindex{y}]} to 1.
    \item Configure \hyperref[fielddesc:GPIOEXTETMTASKGPIO0SEL]{GPIO\_EXT\_ETM\_TASK\_GPIO\regindex{y}\_SEL} to \regindex{x}.
    \item Set \hyperref[fielddesc:GPIOEXTETMTASKGPIO0EN]{GPIO\_EXT\_ETM\_TASK\_GPIO\regindex{y}\_EN}, to enable ETM task channel \regindex{x} to control GPIO\regindex{y}.
\end{itemize}

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
\item One task channel can be selected by one or more GPIOs.

\item When two or three of the signals GPIO\_TASK\_CH\regindex{x}\_SET, GPIO\_TASK\_CH\regindex{x}\_CLEAR, and GPIO\_TASK\_CH\regindex{x}\_TOGGLE of the task channel \regindex{x} selected by GPIO\regindex{y} are valid at the same time, then GPIO\_TASK\_CH\regindex{x}\_SET has the highest priority, GPIO\_TASK\_CH\regindex{x}\_CLEAR takes the second higher priority, and GPIO\_TASK\_CH\regindex{x}\_TOGGLE has the lowest priority.

\item When GPIO\regindex{y} is controlled by ETM task channel, the values of \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}, \hyperref[fielddesc:GPIOFUNCNOUTINVSEL]{GPIO\_FUNC\regindex{n}\_OUT\_INV\_SEL}, and \hyperref[fielddesc:GPIOFUNCNOUTSEL]{GPIO\_FUNC\regindex{n}\_OUT\_SEL} may be modified by the hardware. For such reason, it's recommended to reconfigure these registers when the GPIO is free from the control of ETM task channel.
\end{itemize}
\end{tiplisting}

GPIO has eight event channels, and the ETM events that each event channel can generate are:

\begin{itemize}
    \item GPIO\_EVT\_CH\regindex{x}\_RISE\_EDGE: Indicates that the output signal of the corresponding GPIO Filter (see Figure \ref{fig:iomux-gpio-matrix-overview-detail}) has a rising edge.
    \item GPIO\_EVT\_CH\regindex{x}\_FALL\_EDGE: Indicates that the output signal of the corresponding GPIO Filter (see Figure \ref{fig:iomux-gpio-matrix-overview-detail}) has a falling edge.
    \item GPIO\_EVT\_CH\regindex{x}\_ANY\_EDGE: Indicates that the output signal of the corresponding GPIO Filter (see Figure \ref{fig:iomux-gpio-matrix-overview-detail}) is reversed.
\end{itemize}

The specific configuration of the event channel is as follows:

\begin{itemize}
    \item Set  \hyperref[fielddesc:GPIOEXTETMCH0EVENTEN]{GPIO\_EXT\_ETM\_CH\regindex{x}\_EVENT\_EN} to enable event channel \regindex{x} (0$\sim$7).
    \item Configure \hyperref[fielddesc:GPIOEXTETMCH0EVENTSEL]{GPIO\_EXT\_ETM\_CH\regindex{x}\_EVENT\_SEL} to \regindex{y} (0$\sim$14, 23$\sim$28), i.e., select one from the 21 GPIOs.
\end{itemize}

\begin{tiplisting}
One GPIO can be selected by one or more event channels.
\end{tiplisting}

In some applications, GPIO ETM events can be used to trigger GPIO ETM tasks. For example, event channel 0 selects GPIO0, GPIO1 selects task channel 0, and the GPIO\_EVT\_CH\regindex{0}\_RISE\_EDGE event is used to trigger the GPIO\_TASK\_CH\regindex{0}\_TOGGLE task. When a square wave signal is input to the chip through GPIO0, the chip outputs a square wave signal with a frequency divided by 2 through GPIO1.
