\chapter{Interrupt Matrix}
\label{mod:intmtrx}
\section{Overview}
The interrupt matrix embedded in \chipname{} routes one or multiple peripheral interrupt sources to any of the ESP-RISC-V CPU's peripheral interrupts.

The \chipname{} has 84 peripheral interrupt sources that can be routed to any of the 32 CPU peripheral interrupts using the interrupt matrix.

\begin{tiplisting}
This chapter focuses on how to map peripheral interrupt sources to the CPU interrupts. For information about interrupt configuration, vector, and interrupt handling operations recommended by the ISA, please refer to Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}  > Section \ref{sec:riscvcpu-int-ctrl} \textit{\nameref{sec:riscvcpu-int-ctrl}}.
\end{tiplisting}
\vspace{-1em}

\section{Terminology}\label{sec:intmtx-int-terminology}

The following terms related to interrupts are defined in the context of the \textit{\chipname{} Technical Reference Manual} to help readers better understand this document:

\subsection{Interrupt}

An interrupt refers to the event or condition that occurs, causing the CPU to temporarily suspend its current execution and handle a higher-priority task. It is a mechanism that allows the CPU to respond to specific events promptly.

The \textit{\chipname{} Technical Reference Manual} may use the term ``interrupt" in a broader sense to refer to both interrupt signal and interrupt source.

\subsection{Interrupt Signal/Interrupt Source}

Interrupt signal and interrupt source are only defined from different perspectives and mean the same thing.

From the perspective of peripherals, interrupt signals are generated by the peripheral's internal interrupt sources and are sent to the interrupt matrix.

From the perspective of the interrupt matrix, it receives the interrupt signals sent from the peripheral and considers them interrupt sources. The interrupt matrix then outputs CPU peripheral interrupt signals to the CPU.

From the perspective of the CPU, the interrupt signals from the interrupt matrix become sources and are sent to the CPU core together with the core local interrupt sources.

\section{Features}

The interrupt matrix embedded in the \chipname{} has the following features:

\begin{itemize}
    \item 84 peripheral interrupt sources accepted as input
    \item 32 HP CPU peripheral interrupts generated to the HP CPU as output
    \item Current interrupt status query of peripheral interrupt sources
    \item Multiple interrupt sources mapping to a single HP CPU interrupt (i.e., shared interrupts)
    \item Delegation of CPU User Mode interrupts to Machine Mode interrupts
\end{itemize}

\section{Architecture}

Figure \ref{fig:interrupt-matrix-structure} shows the structure of the interrupt matrix.

You need to configure the \hyperref[sec:intmtrx-reg-summ]{interrupt matrix registers} to map the peripheral interrupt sources to the HP CPU interrupts. The Interrupt Matrix Controller in Figure \ref{fig:interrupt-matrix-structure} manages the mapping and sends the interrupt status of each interrupt source to the interrupt status registers which belong to the interrupt matrix registers.

\subsection{HP CPU Interrupts}

The HP CPU of the \chipname{} implements its interrupt mechanism using standard RISC-V (v0.9) Core-Local Interrupt Controller (CLIC) interrupt. The HP CPU supports 2 core local interrupts (CLINT) and 32 peripheral interrupts, which are numbered from 16 to 47. The peripheral interrupt sources are mapped to the 32 HP CPU peripheral interrupts through the interrupt matrix. Each HP CPU peripheral interrupt has the following properties:

\begin{itemize}
    \item Priority levels configurable, see Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}} > Section \ref{sec:riscvcpu-clic-int-level-prio} \textit{\nameref{sec:riscvcpu-clic-int-level-prio}} for details. 
    \item Configurable as level-triggered or edge-triggered.
    \item Lower-priority interrupts maskable by setting interrupt threshold.
\end{itemize}

\begin{tiplisting}
For detailed information about the functions and configuration procedures of CPU interrupts, see Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}} > Section \ref{sec:riscvcpu-int-ctrl}\textit{\nameref{sec:riscvcpu-int-ctrl}}. The configuration registers of CPU interrupts are listed in Section \ref{sec:intpri-reg-summ} \textit{\nameref{sec:intpri-reg-summ}}.
\end{tiplisting}
\vspace{-1em}

\subsection{Assign Peripheral Interrupt Source to HP CPU Peripheral Interrupt}

In this section, the following terms are used to describe the operation of the interrupt matrix.

\begin{itemize}

\item \regindex{SOURCE}: stands for a peripheral interrupt source in Table \ref{tab:peripheral-interrupt-config-status-source}.

\item \hyperref[INTMTXCORE0SOURCEINTRMAPREG]{INTMTX\_CORE0\_\regindex{SOURCE}\_INTR\_MAP\_REG}: stands for the interrupt source mapping register for a \regindex{SOURCE}.

\item Num\_P: stands for the index of HP CPU interrupts which can be 16 \verb+~+ 47.

\item Interrupt\_P: stands for the HP CPU interrupt numbered as Num\_P.
\end{itemize}

\subsubsection{Assign One Peripheral Interrupt Source to HP CPU Peripheral Interrupt}

Setting the corresponding source mapping register \hyperref[INTMTXCORE0SOURCEINTRMAPREG]{INTMTX\_CORE0\_\regindex{SOURCE}\_INTR\_MAP\_REG} of \regindex{SOURCE} to Num\_P assigns this interrupt source to Interrupt\_P.

\subsubsection{Assign Multiple Peripheral Interrupt Sources to HP CPU Peripheral Interrupt}

Setting the corresponding source mapping register \hyperref[INTMTXCORE0SOURCEINTRMAPREG]{INTMTX\_CORE0\_\regindex{SOURCE}\_INTR\_MAP\_REG} of each \regindex{SOURCE} to the same Num\_P assigns multiple sources to the same Interrupt\_P. Any of these sources can trigger CPU Interrupt\_P. When an interrupt signal is generated, the CPU should check the interrupt status registers to figure out which peripheral generated the interrupt. For more information, see Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}} > Section \ref{sec:riscvcpu-int-ctrl}\textit{\nameref{sec:riscvcpu-int-ctrl}}.

\subsubsection{Unassign SOURCE}

Writing 0 to the \hyperref[INTMTXCORE0SOURCEINTRMAPREG]{INTMTX\_CORE0\_\regindex{SOURCE}\_INTR\_MAP\_REG} register disables the corresponding interrupt source.


\subsection{Delegated Interrupts}

The ESP32-C5 RISC-V CPU supports Machine Mode and User Mode. The 32 external HP CPU interrupts can be configured as either Machine Mode interrupts or User Mode interrupts. When the HP CPU is in Machine Mode, it only responds to Machine Mode interrupts and does not respond to User Mode interrupts. When the HP CPU is in User Mode, it can respond to all interrupts.

Under normal conditions, User Mode interrupts are only handled when the CPU is operating in User Mode. However, with the interrupt delegation feature, Machine Mode can be configured to handle certain User Mode interrupts, while the CPU itself remains in Machine Mode. This is achieved by configuring registers to delegate User Mode interrupts to Machine Mode interrupts.

\begin{itemize}
\item Each interrupt source of the interrupt matrix can independently enable or disable the interrupt delegation feature.
\item Multiple interrupt sources from the interrupt matrix can enable the interrupt delegation feature at the same time.
\item Delegation is allowed to only one interrupt signal from the interrupt matrix.
\end{itemize}

Software can configure the \hyperref[regdesc:INTMTXCORE0INTSIGIDXASSERTINSECREG]{INTMTX\_CORE0\_INT\_SIG\_IDX\_ASSERT\_IN\_SEC\_REG} register to n to delegate a User Mode interrupt to a Machine Mode interrupt with interrupt number n.

To enable the delegated interrupt function for interrupt source \regindex{SOURCE} in the interrupt matrix, software can set the corresponding bit in the \hyperref[INTMTXCORE0SOURCEINTRMAPREG]{INTMTX\_CORE0\_\regindex{SOURCE}\_INTR\_PASS\_IN\_SEC\_REG} register to 1.

\subsection{Query Current Interrupt Status of SOURCE}

After enabling a \regindex{SOURCE}, you can query its current interrupt status by reading the corresponding bit value in \hyperref[regdesc:INTMTXCORE0INTSTATUS0REG]{INTMTX\_CORE0\_INT\_STATUS\_\regindex{n}\_REG} (read only). For the mapping between \hyperref[regdesc:INTMTXCORE0INTSTATUS0REG]{INTMTX\_CORE0\_INT\_STATUS\_\regindex{n}\_REG} and the \regindex{SOURCE}, please refer to Table \ref{tab:peripheral-interrupt-config-status-source}.
