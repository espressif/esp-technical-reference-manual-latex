\chapter{Low-Power Management}\label{mod:lowpowm}

\section{Overview}

\chipname{} features an advanced low-power management system that optimizes the chip's power consumption while maintaining its high performance.

The low-power management system employs various power-saving techniques such as sleep modes, dynamic voltage and frequency scaling, and peripheral power gating to minimize the chip's power consumption.

At the core of this low-power management system is the power management unit (PMU) hardware component. The PMU powers up and down different power domains of the chip to achieve the best balance between chip performance, power consumption, and wake-up latency.

\section{Terminology}

The following terms related to low-power management are defined in the context of the \textit{\chipname{} Technical Reference Manual}
to help readers better understand this document:

\begin{longtable}[c]{ R{6cm} p{10cm} }
\textbf{Low-power management} & Refers to the whole system that manages the chip's power consumption. \\
\textbf{Power management unit (PMU)} & Refers to the specific hardware module that controls power up and down for power domains, clocks, and power-related logic. \\
\textbf{Power domain} & Refers to the smallest unit that can be independently powered up or down. A power domain contains one or multiple modules within the chip. \\
\textbf{PMU states} & Refers to the states of the PMU's state machine. Users can configure the clock gating and power gating of a power domain in each of the states.\\
\textbf{Power modes} & Refers to the preset power modes that power up different domains for typical application scenarios.\\
\end{longtable}
\section{Features}

The PMU has the following features:

\begin{itemize}
    \item Supports four configurable PMU states. Software can flexibly configure them according to the needs.
        \begin{itemize}
        \item HP\_ACTIVE
        \item HP\_MODEM
        \item HP\_SLEEP
        \item LP\_SLEEP
        \end{itemize}
    \item Supports five preset power modes that suit varying typical usage scenarios:
    \begin{itemize}
        \item Active
        \item Modem-sleep
        \item Light-sleep0
        \item Light-sleep1
        \item Deep-sleep
    \end{itemize}
    \item 16 KB SRAM (LP SRAM)
    \item 10 always-on (AON) registers
    \item RTC fast boot
    \item Supports a programmable retention DMA that backs up and restores the CPU and peripherals status when the chip switches between PMU states.
    \item Supports a power controller that controls the power and clocks depending on the power modes
\end{itemize}

\section{Functional Description}

\chipname{}'s low-power management involves the following components:

\begin{itemize}
    \item \textbf{Power scheme}: The power scheme of \chipname{} includes power regulators, digital power domains, analog power domains, etc.

    \item \textbf{PMU controller}: It is the core part of PMU that controls the power up and down of the power domains, clocks, etc.

    \item \textbf{One RTC timer}: For more information, please refer to Chapter \ref{mod:rtctimer} \textit{\nameref{mod:rtctimer}}.

    \item \textbf{10 always-on registers (\hyperref[regdesc:LPAONSTORENREG]{LP\_AON\_STORE0\_REG} $\sim$ \hyperref[regdesc:LPAONSTORENREG]{LP\_AON\_STORE9\_REG})}: These registers are always powered up and are not affected by any low-power modes, thus can be used for storing data that should not be lost.

    \item \textbf{Seven LP GPIO pins (GPIO0 $\sim$ GPIO6)}: These pins are always powered up and are not affected by any low-power modes, which makes them suitable for working as wake-up sources when the chip is in low-power modes. These pins can also work as regular GPIOs. For more information about the LP GPIOs, please refer to Chapter \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}}.

    \item \textbf{LP SRAM}: The 16 KB SRAM is accessible to both HP CPU and LP CPU. It works under the HP CPU clock when accessed by the HP CPU and under the LP CPU clock when accessed by the LP CPU.

\end{itemize}

The following sections provide a detailed description of the components mentioned above.

Figure \ref{fig:lowpowm-power-structure} shows the power scheme of \chipname{} that mainly includes:

\begin{itemize}
    \item Two regulators
    \item Analog power domains
    \item Digital power domains
\end{itemize}

\subsubsection{Regulators}\label{sec:lowpowm-regulators}

As shown in Figure \ref{fig:lowpowm-power-structure}, the analog part of \chipname{} contains two regulators that regulate the power supply to different power domains. The two regulators are:

\begin{itemize}
    \item \textbf{HP sys regulator}, used for regulating the power supply to the HP system. It features strong drive capability, higher power consumption, and adjustable output voltage.
    \item \textbf{LP sys regulator}, used for regulating the power supply to the LP system. Its output voltage is also adjustable.
\end{itemize}

\subsubsection{Digital Power Domains}\label{sec:lowpowm_digital_domain}

\begin{itemize}
    \item The HP system contains the following digital power domains:
    \begin{itemize}
        \item \textbf{CPU}: It mainly includes the CPU and its supporting peripherals (such as TRACE).
        \item \textbf{Modem}: It consists of wireless MAC and baseband.
        \item \textbf{Peripherals + ROM}: It mainly includes bus, HP peripherals.
        \item \textbf{Internal SRAMx}: It is divided into four sub-power domains, as shown in Figure \ref{fig:lowpowm-power-structure}, where each of SRAM0/1/2 has an independent power switch, while SRAM3 is directly connected to the regulator without a power switch.
        \item \textbf{Modem Power}: It mainly includes modules that control the operating of the wireless section.
    \end{itemize}

There is an independent power switch between the regulator and each power domain, enabling
up/down control of the digital power domain.

    \item The LP system contains the following digital power domains:
    \begin{itemize}
        \item \textbf{LP PD peripherals}: It mainly includes the LP CPU and LP peripherals.
        \item \textbf{LP always-on}: It mainly includes LP always-on peripherals (e.g., RTC timer) and PMU controller. This power domain keeps powered up all the time.
    \end{itemize}
\end{itemize}

\subsubsection{Analog Power Domains}

\begin{itemize}
    \item The HP system contains the following analog power domains:
    \begin{itemize}
    \item PLL
    \end{itemize}

    \item The LP system contains the following analog power domains:
    \begin{itemize}
    \item External Main Clock
    \item Fast RC Oscillator
    \item RF circuit
    \end{itemize}
\end{itemize}

\subsection{PMU}

The PMU of \chipname{} controls power consumption-related components of each power domain, such as power and clock. PMU consists of the following major parts:

\begin{itemize}
    \item \textbf{PMU main state machine}: It records and switches the PMU states.
    \item \textbf{Sleep/wake-up controller}: It sends sleep or wake-up requests to the PMU main state machine.
    \item \textbf{Power controllers}: They control the power and clock signals depending on the power modes of the chip. The power controllers include:
        \begin{itemize}
            \item \textbf{Digital power controller}: It powers up or down the digital power domains.
            \item \textbf{Analog power controller}: It enables the analog modules, such as the regulators, analog clocks, etc.
            \item \textbf{Clock controller}: It manages the clock gating of peripherals and selects analog clock sources for digital clocks.
            \item \textbf{Data backup controller}: It controls the data backup and restore process when the chip switches between PMU states.

            \item \textbf{System controller}: It controls some system-level modules, such as suspending watchdog functionality in sleep mode (when the CPU is unavailable).
        \end{itemize}
\end{itemize}

The PMU workflow involves the following steps:

\begin{itemize}
\item The sleep/wake-up controller sends sleep or wake-up requests to the PMU main state machine.
\item The PMU main state machine generates power gating, clock gating, and reset signals.
\item The power controllers and clock controllers power up or down different power domains and clocks based on the signals generated by the PMU main state machine, allowing the chip to enter or exit different low-power modes.
\end{itemize}

The PMU workflow is shown in Figure \ref{fig:lowpowm-workflow}. The definitions of HP\_SWITCH and LP\_SWITCH are as follows:

\begin{itemize}
    \item HP\_SWITCH: The intermediate state for the HP system transitioning between HP\_ACTIVE and HP\_SLEEP. The hardware completes the control switching of various controllers in this state.
    \item LP\_SWITCH: The intermediate state for the LP system transitioning between HP\_SLEEP and LP\_SLEEP. The hardware completes the control switching of various controllers in this state.
\end{itemize}

\begin{figure}[H]
    \centering    \includegraphics[width=1.1\textwidth]{\modulefiles/figures/13-LOWPOWM_Workflow.pdf}        \caption{PMU Workflow}\label{fig:lowpowm-workflow}
\end{figure}


The following sections describe the main parts of PMU.

\subsubsection{PMU Main State Machine}

The PMU main state machine receives sleep and wake-up signals, changes the state of power and clock
through the power controllers, thereby switching PMU states, and achieving a balance between performance and power consumption of the chip.

The PMU main state machine supports four PMU states, each controlled by different sleep and wake-up signals, supporting software customization of power and clocks. These PMU states allow the software to expand power modes for various application scenarios. The four PMU states are:

\begin{itemize}
    \item \textbf{HP\_ACTIVE}: The circuits on the chip are powered up to a maximum, supporting the HP system and LP system operation.
    \item \textbf{HP\_MODEM}: Modem (wireless MAC and baseband) can operate independently of the CPU.
    \item \textbf{HP\_SLEEP}: The HP system is in sleep, supporting LP peripherals operation.
    \item \textbf{LP\_SLEEP}: The HP system and LP peripherals are in sleep, while the always-on circuits remain operational.
\end{itemize}

\begin{tiplisting}
    The division of HP and LP system is as follows:
    \begin{itemize}
    \item LP system (peripherals): LP RISC-V 32-bit Microprocessor, LP GPIO, LP UART, LP I2C
    \item LP system (always-on circuits): LP Memory (LP SRAM), PMU, RTC Watchdog Timer, RTC Super Watchdog Timer, RTC Timer, eFuse Controller, Power Glitch Detector
    \item HP system: All peripherals (including Modem) except those belonging to the LP system.
    \end{itemize}

For more details, please refer to \href{https://www.espressif.com/sites/default/files/documentation/esp32-c5_datasheet_en.pdf}{\textit{\chipname{} Datasheet}} > \textit{Functional Block Diagram}.


\end{tiplisting}

HP\_ACTIVE, HP\_MODEM, HP\_SLEEP are the states of the HP system, while LP\_SLEEP is the state of the LP system.

\begin{itemize}
    \item If a module belongs to the HP system, its power up/down is configured in the HP\_ACTIVE/HP\_MODEM/HP\_SLEEP states, but not in the LP\_SLEEP state. It reuses the HP\_SLEEP configuration in the LP\_SLEEP state.
    \item If a module belongs to the LP system, its power up/down is configured in the LP\_SLEEP state and reuses the HP\_SLEEP configuration in the HP\_ACTIVE/HP\_MODEM/HP\_SLEEP states.
\end{itemize}

Take the HP CPU as an example. Since the HP CPU belongs to the HP system, it can be powered up or down in HP\_ACTIVE/HP\_MODEM/HP\_SLEEP states through the following registers and reuse the HP\_SLEEP configuration in LP\_SLEEP.

\begin{itemize}
    \item HP\_ACTIVE: \hyperref[fielddesc:PMUHPACTIVEPDHPCPUPDEN]{PMU\_HP\_ACTIVE\_PD\_HP\_CPU\_PD\_EN}
    \item HP\_MODEM: \hyperref[fielddesc:PMUHPMODEMPDHPCPUPDEN]{PMU\_HP\_MODEM\_PD\_HP\_CPU\_PD\_EN}
    \item HP\_SLEEP: \hyperref[fielddesc:PMUHPSLEEPPDHPCPUPDEN]{PMU\_HP\_SLEEP\_PD\_HP\_CPU\_PD\_EN}
\end{itemize}

Similarly, users can define other power domains' power up and down in different PMU states. For the configuration registers, please refer to Section \ref{sec:lowpowm-reg-summ}.

\begin{tiplisting}
In the following text, all such registers will be collectively referred to as PMU\_\regindex{PMUSTATE}\_PD\_\regindex{POWERDOMAIN}\_PD\_EN, where \regindex{PMUSTATE} represents the four PMU states.
\end{tiplisting}



Once the configuration is done, PMU uses various controllers to make these configurations effective, as described in the sections below.

\subsubsection{Sleep/Wake-up Controller}

The sleep/wake-up controller initiates sleep and wake-up requests to the PMU main state machine. \chipname{} supports multiple wake sources to wake the CPU from different power modes. Table \ref{tab:lowpowm-wakeup-source} lists all the
wake-up sources.

\begin{table}[H]
    \centering
    \caption{Wake-up Sources}
    \label{tab:lowpowm-wakeup-source}
    \begin{threeparttable}
    \begin{tabular}{| r | l | c | c |}
    \hline
    \rowcolor{lightgray}
    \textbf{\hyperref[fielddesc:PMUWAKEUPENA]{PMU\_WAKEUP\_ENA}} & \textbf{Wake-up Sources} &  \textbf{Light-sleep} & \textbf{Deep-sleep}  \\\hline
    0x2 & EXT\_IO\tnote{1} & Y & Y \\\hline
    0x4 & GPIO\tnote{1} & Y & Y \\\hline
    0x8 & Wi-Fi beacon  & Y & - \\\hline
    0x10 & RTC Timer  & Y & Y \\\hline
    0x20 & Wi-Fi\tnote{2} & Y & – \\\hline
    0x40 & UART0\tnote{3} & Y & – \\\hline
    0x80 & UART1\tnote{3} & Y & – \\\hline
    0x100 & SDIO slave controller\tnote{4} & Y & – \\\hline
    0x400 & Bluetooth & Y & – \\\hline
    0x800 & LP CPU  & Y & Y \\\hline
    \end{tabular}

\begin{tablenotes}
    \item[1] In Deep-sleep mode, only the LP GPIOs can work as wake-up sources.
    \item[2] To wake up the chip with Wi-Fi, the chip switches between Active, Modem-sleep, and Light-sleep modes. The RF module is woken up at predetermined intervals to keep Wi-Fi connectivity and communication.
    \item[3] A wake-up is triggered when the number of RX pulses received exceeds the threshold set in \hyperref[fielddesc:UARTACTIVETHRESHOLD]{UART\_ACTIVE\_THRESHOLD}. For details, please refer to Chapter \ref{mod:uart} \textit{\nameref{mod:uart}}.
    \item[4] When the SDIO lave controller receives CMD52 CMD53, it triggers wake-up. This wake-up feature requires the sleep voltage to be higher than 0.9 V, and the voltage configuration varies across different chips due to manufacturing differences.
\end{tablenotes}
\end{threeparttable}
\end{table}

\chipname{} provides a hardware mechanism that can reject sleep, meaning if some peripherals are in an uninterruptible working state and the CPU tries to sleep, the peripherals will send a wake-up signal to prevent the CPU from sleeping, thus ensuring the peripherals work normally.

The wake-up sources in Table \ref{tab:lowpowm-wakeup-source} can all be configured as events to reject sleep. Users can configure the following registers to implement sleep rejection. The configuration values of \hyperref[fielddesc:PMUSLEEPREJECTENA]{PMU\_SLEEP\_REJECT\_ENA} and \hyperref[fielddesc:PMUREJECTCAUSE]{PMU\_SLP\_REJECT\_CAUSE\_REG} and the corresponding wake-up sources are the same as shown in Table \ref{tab:lowpowm-wakeup-source}.

\begin{itemize}
    \item Enable sleep rejection feature:
    \begin{itemize}
        \item Set \hyperref[fielddesc:PMUSLPREJECTEN]{PMU\_SLP\_REJECT\_EN} to 1 to enable the sleep rejection feature.
        \item Configure \hyperref[fielddesc:PMUSLEEPREJECTENA]{PMU\_SLEEP\_REJECT\_ENA} to enable the sleep rejection signal source.
    \end{itemize}
    \item Read \hyperref[fielddesc:PMUREJECTCAUSE]{PMU\_SLP\_REJECT\_CAUSE\_REG} for the source of sleep rejection event.
\end{itemize}

\subsubsection{Analog Power Controller}\label{sec:lowpowm_analog_power_controller}

The analog power controller controls the power up and down of the analog circuits (including voltage regulators, high-speed clocks, and slow-speed clocks) in PMU states.

The configuration of the regulators is as follows:

\begin{itemize}
    \item Configure \hyperref[fielddesc:PMUHPACTIVEHPREGULATORXPD]{PMU\_\regindex{PMUSTATE}\_HP\_REGULATOR\_XPD} or \hyperref[fielddesc:PMUHPSLEEPLPREGULATORXPD]{PMU\_\regindex{PMUSTATE}\_LP\_REGULATOR\_XPD} to enable or disable the output voltage of the HP/LP sys regulator in the target PMU state. Turning off the LP sys regulator is not recommended, as it may cause the PMU itself to power down, resulting in chip malfunction.
\end{itemize}

The configuration of the high-speed clocks (XTAL\_CLK and PLL\_CLK) is as follows:

\begin{itemize}
    \item XTAL\_CLK: Configure \hyperref[fielddesc:PMUHPSLEEPXPDXTAL]{PMU\_HP\_SLEEP\_XPD\_XTAL} to 1 to enable XTAL\_CLK when the chip switches PMU state to HP\_SLEEP.

    Note: To avoid the instability in XTAL\_CLK during startup, users have the option to configure \hyperref[fielddesc:PMUWAITXTALSTABLE]{PMU\_WAIT\_XTAL\_STABLE} to delay the gate opening for XTAL\_CLK. This delay ensures that the gate opening is enabled after \hyperref[fielddesc:PMUWAITXTALSTABLE]{PMU\_WAIT\_XTAL\_STABLE} CLK\_DYN\_FAST\_CLK cycles following the power-up of XTAL\_CLK.

    \item PLL\_CLK: PMU can enable PLL\_CLK in different PMU states by configuring \hyperref[fielddesc:PMUHPACTIVEXPDBBPLL]{PMU\_HP\_ACTIVE\_XPD\_BBPLL} to 1. For example, configuring \hyperref[fielddesc:PMUHPACTIVEXPDBBPLL]{PMU\_HP\_ACTIVE\_XPD\_BBPLL} to 1 will enable the PLL\_CLK clock when the chip is in HP\_ACTIVE state.

    Note:
    \begin{itemize}
        \item Before enabling PLL\_CLK please ensure that XTAL\_CLK is stable.
        \item To avoid the instability in PLL\_CLK during startup, users have the option to configure \hyperref[fielddesc:PMUWAITPLLSTABLE]{PMU\_WAIT\_PLL\_STABLE} to delay the gate opening for PLL\_CLK. This delay ensures that the gate opening is enabled after \hyperref[fielddesc:PMUWAITPLLSTABLE]{PMU\_WAIT\_PLL\_STABLE} number of CLK\_DYN\_FAST\_CLK cycles following the power-up of PLL\_CLK.
    \end{itemize}
\end{itemize}

Slow-speed clocks (RC\_FAST\_CLK and XTAL32K\_CLK) operate with low power. In HP\_ACTIVE, HP\_MODEM, and HP\_SLEEP states, \hyperref[regdesc:PMUHPSLEEPLPCKPOWERREG]{PMU\_HP\_SLEEP\_LP\_CK\_POWER\_REG} controls the power up and down of the slow-speed clocks. In LP\_SLEEP state, \hyperref[regdesc:PMULPSLEEPLPCKPOWERREG]{PMU\_LP\_SLEEP\_LP\_CK\_POWER\_REG} controls the power up and down of the slow-speed clocks.

\subsubsection{Digital Power Controller}

The digital power controller controls the power up and down of digital power domains in different PMU states. Unlike the analog power controller, the digital power controller does not directly control the regulator but instead controls the power switch connected to the regulator to power up and down the digital power domains.

The LP PD Peripherals domain can only be powered up and down while the PMU state switches between HP\_SLEEP and LP\_SLEEP PMU. The other power domains can be powered up and down while the PMU states switch between HP\_SLEEP, HP\_ACTIVE, and HP\_MODEM.

When the chip switches between PMU states, if the power configuration of a power domain in the current PMU state does not match the power configuration it is about to switch to, then the power up-down process will be activated. Take the power up-to-down process of the CPU power domain as an example. Configure \hyperref[fielddesc:PMUHPMODEMPDHPCPUPDEN]{PMU\_HP\_MODEM\_PD\_HP\_CPU\_PD\_EN} to 1 or 0 to indicate that the CPU power domain is powered down or up in the HP\_MODEM state. PMU will perform the following configurations:

\begin{itemize}

\item Enable the digital isolation unit to ensure that the powered-down modules do not output unstable voltage levels to the powered-up modules. When a power domain loses power, the output of this module will be clamped to a fixed value.

\item Enable reset. When the CPU power domain loses power, its global reset signal is set to a reset state, which persists for a period after the CPU power domain is re-powered. This mechanism guarantees a reset-to-release process for the CPU power domain during power-up, effectively mitigating any instability caused by power up-down transitions.
\end{itemize}

The following explains the power up and down of each digital power domain:

\begin{itemize}
    \item Internal SRAMx

    The power up and down of the Internal SRAMx domain is not controlled by a dedicated register. The Internal SRAMx domain shares the \hyperref[fielddesc:PMUHPACTIVEPDTOPPDEN]{PMU\_\regindex{PMUSTATE}\_PD\_TOP\_PD\_EN} register with the Peripherals domain. If \hyperref[fielddesc:PMUPDHPMEM0PDMASK]{PMU\_PD\_HP\_MEM\regindex{n}\_PD\_MASK} (\regindex{n}=0,1,2) is 0, both the Internal SRAMx and Peripherals power domains are powered up or down simultaneously. If \hyperref[fielddesc:PMUPDHPMEM0PDMASK]{PMU\_PD\_HP\_MEM\regindex{n}\_PD\_MASK} is 1, the Internal SRAMx can remain powered up when the Peripherals domain is powered down.

    \item Modem Power

    Modem Power is connected to the HP sys regulator through a power switch. \hyperref[fielddesc:PMUHPACTIVEPDHPAONPDEN]{PMU\_HP\_ACTIVE\_PD\_HP\_AON\_PD\_EN} controls its power up and down in HP\_ACTIVE state. From Figure \ref{fig:lowpowm-power-structure}, it can be seen that if any of the CPU, Modem, or Peripherals + ROM power domains needs to be powered up, the Modem Power domain must also be powered up. Such a design meets functional requirements.

    \item Peripherals + ROM/Modem/CPU:

    Each of the three power domains can be powered up and down in different PMU states using the PMU\_\regindex{PMUSTATE}\_PD\_\regindex{n}\_PD\_EN register (\regindex{n}=TOP/HP\_WIFI/HP\_CPU). ``WIFI" in the register represents the Modem Power domain.

    When the Peripherals domain is powered down, the following features are configurable:
        \begin{itemize}
        \item Powering down the Peripherals domain may cause instability in GPIOs. This can be addressed by maintaining the state of the GPIOs (excluding eight LP GPIOs) through PMU. For example, configuring \hyperref[fielddesc:PMUHPSLEEPHPPADHOLDALL]{PMU\_HP\_SLEEP\_HP\_PAD\_HOLD\_ALL} keeps GPIOs in the same state as before Peripherals was powered down in HP\_SLEEP.

        \item In HP\_ACTIVE and HP\_MODEM states, the Internal SRAMx domain can be powered down or put into Deep-sleep mode. In Deep-sleep, the memory cannot be read or written to, but data can be retained. Configure \hyperref[fielddesc:PMUHPACTIVEHPMEMDSLP]{PMU\_\regindex{PMUSTATE}\_HP\_MEM\_DSLP} to enable Memory Deep-sleep mode.
        \end{itemize}

    \item LP PD Peripherals

    The LP PD Peripherals power domain remains powered up when the chip is in HP\_ACTIVE and HP\_MODEM states. The power state of LP PD Peripherals is configurable only in HP\_SLEEP and LP\_SLEEP states. The LP PD Peripherals power domain has an independent digital power switch, and its power control is not dependent on the power state of other digital power domains.

\end{itemize}

\subsubsection{Clock Controller}

The clock controller mainly controls the high-performance system clocks and LP system clocks when the chip switches between PMU states.

The high-performance system clocks include HP\_ROOT\_CLK and high-performance system peripherals clocks. When the chip switches between HP\_ACTIVE, HP\_MODEM, and HP\_SLEEP states, PMU can switch, power up/down, and divide the frequency of HP\_ROOT\_CLK, as well as power up/down high-performance system peripherals clocks.

\begin{itemize}

\item HP\_ROOT\_CLK can be controlled as follows:

    \begin{itemize}
    \item Configure \hyperref[fielddesc:PMUHPACTIVESYSCLKSLPSEL]{PMU\_\regindex{PMUSTATE}\_SYS\_CLK\_SLP\_SEL} to select PMU to control the clock source of HP\_ROOT\_CLK in corresponding PMU state.

    \item Configure \hyperref[fielddesc:PMUHPACTIVEICGSYSCLOCKEN]{PMU\_\regindex{PMUSTATE}\_ICG\_SYS\_CLOCK\_EN} to 0 to disable HP\_ROOT\_CLK in the corresponding PMU state.

    \item Configure \hyperref[fielddesc:PMUHPACTIVEDIGSYSCLKSEL]{PMU\_\regindex{PMUSTATE}\_DIG\_SYS\_CLK\_SEL} to select the clock source of HP\_ROOT\_CLK in corresponding PMU state. For details, please refer to Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}} > Table \ref{tab:resclk-cpu-clk-src}.
    \end{itemize}

\item High-performance system peripheral clocks can be controlled as follows:

    \begin{itemize}
    \item Configure \hyperref[fielddesc:PMUHPACTIVEICGSLPSEL]{PMU\_\regindex{PMUSTATE}\_ICG\_SLP\_SEL} to 1 so that the clock gating in the target state is controlled by PMU. Configure this register to 0 so that the clock gating is controlled by PCR registers.

    \item Configure \hyperref[fielddesc:PMUHPACTIVEDIGICGFUNCEN]{PMU\_\regindex{PMUSTATE}\_DIG\_ICG\_FUNC\_EN} to power up or down the function clock in the target PMU state. For detailed configuration please see Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}} > Table \ref{tab:clk_pmu_func}.

    \item Configure \hyperref[fielddesc:PMUHPACTIVEDIGICGAPBEN]{PMU\_\regindex{PMUSTATE}\_DIG\_ICG\_APB\_EN} to power up/down the APB clock in the target PMU state. For detailed configuration please see Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}} > Table \ref{tab:clk_pmu_apb}.
    \end{itemize}

\end{itemize}

The LP system clocks are mainly used in the low-power system and include the following clocks:

    \begin{itemize}
    \item LP\_SLOW\_CLK
    \item LP\_FAST\_CLK
    \item LP\_DYN\_SLOW\_CLK
    \item LP\_DYN\_FAST\_CLK
    \end{itemize}

The clock frequency of LP\_DYN\_FAST\_CLK is controlled by hardware, depending on the PMU state (and cannot be changed by the user):

    \begin{itemize}
    \item In LP\_SLEEP state: The LP\_DYN\_FAST\_CLK frequency is the same as LP\_SLOW\_CLK.
    \item In HP\_ACTIVE/HP\_MODEM/HP\_SLEEP state: The LP\_DYN\_FAST\_CLK frequency is the same as LP\_FAST\_CLK.
    \end{itemize}

\subsubsection{Backup Controller}

\chipname{} has a Retention DMA module that can transfer data between memory and peripherals when the chip switches between PMU states, so that the data is backed up when the power domain is powered down and restored when the power domain is powered up again.

Data transfer is implemented in the Peripherals power domain. PMU only generates relevant control signals. It is important to note that the data transfer control registers are directional, as unlike other control registers, these control behaviors are determined by both the original PMU state and the target PMU state.

For example, the control registers for transitioning from HP\_ACTIVE to HP\_SLEEP and from HP\_MODEM to HP\_SLEEP are different. The possible PMU state switches are listed below, collectively represented by \regindex{PMUSTATE\_TRANS} in the register names:

\begin{itemize}
    \item HP\_SLEEP2ACTIVE
    \item HP\_SLEEP2MODEM
    \item HP\_MODEM2ACTIVE
    \item HP\_MODEM2SLEEP
    \item HP\_ACTIVE2SLEEP
\end{itemize}

The following introduces how PMU controls the Retention DMA:

\begin{itemize}
    \item Enable data transfer: Configure \hyperref[fielddesc:PMUHPSLEEP2ACTIVEBACKUPEN]{PMU\_\regindex{PMUSTATE\_TRANS}\_BACKUP\_EN} to 1 to enable data transfer when the corresponding PMU state switch is performed.
    \item Enable corresponding clocks: Before data transfer starts, configure \hyperref[fielddesc:PMUHPMODEM2SLEEPBACKUPCLKSEL]{PMU\_\regindex{PMUSTATE\_TRANS}\_BACKUP\_CLK\_SEL} to select the clock source of the Retention DMA, and configure \hyperref[fielddesc:PMUHPACTIVEBACKUPICGFUNCEN]{PMU\_\regindex{PMUSTATE}\_BACKUP\_ICG\_FUNC\_EN} to enable the clock.

    After the data transfer is completed, the value of \hyperref[fielddesc:PMUHPACTIVEBACKUPICGFUNCEN]{PMU\_\regindex{PMUSTATE}\_BACKUP\_ICG\_FUNC\_EN} is determined by the configuration of \hyperref[fielddesc:PMUHPACTIVEDIGICGFUNCEN]{PMU\_\regindex{PMUSTATE}\_DIG\_ICG\_FUNC\_EN} in the target PMU state.

    \item Configure data transfer direction: Configure the highest bit of \hyperref[fielddesc:PMUHPMODEM2SLEEPBACKUPMODE]{PMU\_\regindex{PMUSTATE\_TRANS}\_BACKUP\_MODE}:
    \begin{itemize}
        \item 1: From peripheral to memory
        \item 0: From memory to peripheral
    \end{itemize}
    \item Select linked list pointer: Configure the lower four bits of \hyperref[fielddesc:PMUHPMODEM2SLEEPBACKUPMODE]{PMU\_\regindex{PMUSTATE\_TRANS}\_BACKUP\_MODE} to select the linked list pointer. The pointer is set within the linked list.\\
\end{itemize}

\subsubsection{System Controller}

The system controller controls some functional modules when the chip switches PMU states, to achieve stable and low-power chip performance. Specifically, the system controller supports:

\begin{itemize}
    \item \textbf{Pausing the watchdog function}: Configuring \hyperref[fielddesc:PMUHPACTIVEDIGPAUSEWDT]{PMU\_\regindex{PMUSTATE}\_DIG\_PAUSE\_WDT} to 1 disables the RTC watchdog timer (RWDT) function when the chip switches to the corresponding target PMU state. Note that if this register is configured to 0 in any sleep mode, the watchdog function is not disabled, and RWDT will reset as the CPU does not feed the watchdog.

    \item \textbf{Switching GPIO to sleep mode}, where the configuration of the GPIO holds. Consider a GPIO pin working in low-drive mode as an input and a wake-up pin. Setting \hyperref[fielddesc:PMUHPACTIVEHPPADHOLDALL]{PMU\_\regindex{PMUSTATE}\_HP\_PAD\_HOLD\_ALL} to 1 latches the current configuration of the GPIO pin as the sleep configuration when the chip transitions to the corresponding PMU state. For more information about the GPIO's hold function, please refer to Chapter \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}} > Section \ref{PadHold}.

    \item \textbf{Disabling UART wake-up function}: Configuring \hyperref[fielddesc:PMUHPACTIVEUARTWAKEUPEN]{PMU\_\regindex{PMUSTATE}\_UART\_WAKEUP\_EN} to 0 disables the four UART wake-up modes in the corresponding PMU state. For more information on UART wake-up modes, please refer to Chapter \ref{mod:uart} \textit{\nameref{mod:uart}}.

    \item \textbf{Pausing the CPU}: Setting \hyperref[fielddesc:PMUHPACTIVEDIGCPUSTALL]{PMU\_\regindex{PMUSTATE}\_DIG\_CPU\_STALL} to 1 suspends the CPU in the corresponding PMU state.
\end{itemize}

\section{RTC Boot}

In Deep-sleep mode, both the ROM and RAM of the chip are powered down, so the time required for SPI Boot (copying data from flash) upon waking up is long. Therefore, compared to Light-sleep and Modem-sleep modes, the wake-up from Deep-sleep mode takes longer time. However, in Deep-sleep mode, the 16 KB SRAM can remain powered up. Therefore, users can put small-sized code (i.e., ``deep sleep wake stubs") into the 16 KB SRAM to avoid the delay caused by SPI boot, thus speeding up the chip wake-up process.



To enable RTC boot, follow the steps below:

\begin{enumerate}
    \item Set \hyperref[fielddesc:LPAONCORE0STATVECTORSEL]{LP\_AON\_CORE0\_STAT\_VECTOR\_SEL} to 0 to start up the chip from the LP SRAM.

    \item Calculate CRC for of "deep sleep wakeup stubs" in the LP SRAM and save the result in \hyperref[regdesc:LPAONSTORENREG]{LP\_AON\_STORE7\_REG}.

    \item Set \hyperref[regdesc:LPAONSTORENREG]{LP\_AON\_STORE6\_REG} to the entry address of the "deep sleep wakeup stubs" in LP SRAM.

    \item Configure sleep mode for the chip.

    \item When the CPU is powered up, it begins unpacking the ROM and performing partial initialization. Then, the CPU recalculates the CRC code of the RTC fast memory. If the result matches what is stored in \hyperref[regdesc:LPAONSTORENREG]{LP\_AON\_STORE7\_REG}, the CPU jumps to the "deep sleep wakeup stubs". If the CRC does not match, the CPU directly enters the SPI boot process.

\end{enumerate}

\section{Event Task Matrix Feature}
\label{sec:lowpowm-etm-func}

The low-power management system on \chipname{} supports the Event Task Matrix (ETM) function, which allows the low-power management system's ETM tasks to be triggered by any peripherals' ETM events, or the low-power management system's ETM events to trigger any peripherals' ETM tasks. This section introduces the ETM tasks and events related to the low-power management system. For more information, please refer to Chapter \ref{mod:evntaskmatrix} \nameref{mod:evntaskmatrix}.

The low-power management system can receive the following ETM task:

\begin{itemize}
\item PMU\_TASK\_SLEEP\_REQ: Triggers PMU's sleep process.
\end{itemize}

The low-power management system can generate the following ETM event:

\begin{itemize}
\item PMU\_EVT\_SLEEP\_WAKEUP: Indicates that PMU is woken up to the HP\_ACTIVE state.
\end{itemize}

\section{Interrupts}

\chipname{}'s low-power management system can generate the following interrupt signals:

\begin{itemize}
    \item PMU\_INTR
    \item PMU\_LP\_INT
\end{itemize}

PMU\_INTR is sent to the \hyperref[mod:intmtrx]{Interrupt Matrix}, while PMU\_LP\_INT is sent to the LP CPU.

The interrupt signals are generated by the internal interrupt sources of each module, specifically:

PMU\_INTR:

\begin{itemize}
\item \label{int:PMUSOCWAKEUPINT}PMU\_SOC\_WAKEUP\_INT: Triggered when the chip is woken up to the HP\_ACTIVE state.
\item \label{int:PMUSOCSLEEPREJECTINT}PMU\_SOC\_SLEEP\_REJECT\_INT: Triggered when a sleep-rejection source rejects a sleep request.
\item \label{int:PMUSWINT}PMU\_SW\_INT: Triggered when LP CPU is used as a wake-up source and wakes up the chip to HP\_ACTIVE state.

\item \label{int:PMUSDIOIDLEINT}PMU\_SDIO\_IDLE\_INT: Triggered when SDIO is in idle state.
\item \label{int:PMULPCPUEXCINT}PMU\_LP\_CPU\_EXC\_INT: Triggered when there are LP CPU exceptions.
\end{itemize}

PMU\_LP\_INT:

\begin{itemize}
\item \label{int:PMUHPSWTRIGGERINT}PMU\_HP\_SW\_TRIGGER\_INT: Triggered when HP CPU wakes up LP CPU.

\item \label{int:PMUACTIVESWITCHSLEEPSTARTINT}PMU\_ACTIVE\_SWITCH\_SLEEP\_START\_INT: Triggered when the PMU state starts to switch from HP\_ACTIVE to HP\_SLEEP.
\item \label{int:PMUMODEMSWITCHSLEEPSTARTINT}PMU\_MODEM\_SWITCH\_SLEEP\_START\_INT: Triggered when the PMU state starts to switch from HP\_MODEM to HP\_SLEEP.
\item \label{int:PMUSLEEPSWITCHMODEMSTARTINT}PMU\_SLEEP\_SWITCH\_MODEM\_START\_INT: Triggered when the PMU state starts to switch from HP\_SLEEP to HP\_MODEM.
\item \label{int:PMUSLEEPSWITCHACTIVESTARTINT}PMU\_SLEEP\_SWITCH\_ACTIVE\_START\_INT: Triggered when the PMU state starts to switch from HP\_SLEEP to HP\_ACTIVE.
\item \label{int:PMUMODEMSWITCHACTIVESTARTINT}PMU\_MODEM\_SWITCH\_ACTIVE\_START\_INT: Triggered when the PMU state starts to switch from HP\_MODEM to HP\_ACTIVE.
\item \label{int:PMUACTIVESWITCHSLEEPENDINT}PMU\_ACTIVE\_SWITCH\_SLEEP\_END\_INT: Triggered when the PMU state has switched from HP\_ACTIVE to HP\_SLEEP.
\item \label{int:PMUMODEMSWITCHSLEEPENDINT}PMU\_MODEM\_SWITCH\_SLEEP\_END\_INT: Triggered when the PMU state has switched from  HP\_MODEM to HP\_SLEEP.
\item \label{int:PMUSLEEPSWITCHMODEMENDINT}PMU\_SLEEP\_SWITCH\_MODEM\_END\_INT: Triggered when the PMU state has switched from  HP\_SLEEP to HP\_MODEM.
\item \label{int:PMUSLEEPSWITCHACTIVEENDINT}PMU\_SLEEP\_SWITCH\_ACTIVE\_END\_INT: Triggered when the PMU state has switched from HP\_SLEEP to HP\_ACTIVE.
\item \label{int:PMUMODEMSWITCHACTIVEENDINT}PMU\_MODEM\_SWITCH\_ACTIVE\_END\_INT: Triggered when the PMU state has switched from HP\_MODEM to HP\_ACTIVE.
\item \label{int:PMULPCPUWAKEUPINT}PMU\_LP\_CPU\_WAKEUP\_INT: Triggered when LP CPU is woken up.
\end{itemize}


Each interrupt source can be configured by a common set of registers that are described in Section \hyperref[interrupt-config-registers]{\textit{Interrupt Configuration Registers}}. The specific registers can be found in Section \ref{sec:lowpowm-reg-summ} \textit{\nameref{sec:lowpowm-reg-summ}}.
