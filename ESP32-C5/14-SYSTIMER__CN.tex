\chapter{系统定时器}
\label{mod:systimer}

\section{概述}
\chipname{} 芯片内置 52 位系统定时器。该定时器可用于生成操作系统所需的滴答定时中断，也可以用作普通的定时器生成周期或单次延时中断。


\section{主要特性}

系统定时器具有如下特性：

\begin{itemize}
    \item 集成两个 52 位计数器和三个 52 位比较器
    \item 支持软件访问由 APB\_CLK 时钟驱动的寄存器
    \item 支持 CNT\_CLK 时钟计数，两次计数周期的平均频率为 16 MHz
    \item 配置 XTAL\_CLK (40 MHz/48 MHz) 作为 CNT\_CLK 时钟源
    \item 支持 52 位报警值 (t) 和 26 位报警周期 ($\delta$t)
    \item 支持两种报警模式：
    \begin{itemize}
        \item 单次报警模式：根据设定的目标报警值 (t)，生成一次性报警
        \item 周期报警模式：根据设定的报警周期 ($\delta$t)，生成周期性报警
    \end{itemize}
    \item 支持根据设置的报警值 (t) 或报警周期 ($\delta$t)，通过三个比较器生成三个独立中断
    \item 支持软件配置基准计数值。例如，支持从 Light-sleep 唤醒之后，系统定时器通过软件加载 RTC 定时器记录的睡眠时间，并进行补偿
    \item CPU 处于停止状态或处于在线调试状态时，系统定时器可选择停止运行或继续运行
    \item 支持事件任务矩阵 (ETM) 事件报警
\end{itemize}

\section{系统定时器架构}
系统定时器内置两个计数器（UNIT0 和 UNIT1）以及三个比较器（COMP0、COMP1 和 COMP2）。比较器用于监控计数器的计数值是否达到报警值。定时器的功能块图见图 \ref{fig:system-timer-structure}。

\section{时钟源选择}
计数器和比较器使用 XTAL\_CLK 或者 RC\_FAST\_CLK 作为时钟源。可通过配置寄存器 \hyperref[regdesc:PCRSYSTIMERFUNCCLKCONFREG]{PCR\_SYSTIMER\_FUNC\_CLK\_CONF\_REG} 的 \hyperref[fielddesc:PCRSYSTIMERFUNCCLKSEL]{PCR\_SYSTIMER\_FUNC\_CLK\_SEL} 字段来选择时钟源。XTAL\_CLK 经分频后，生成平均频率为 16 MHz 的计数器时钟信号 CNT\_CLK，如图 \ref{fig:system-timer-Alarm-Generation} 所示。每个 CNT\_CLK 时钟周期，计数递增 1/16 $\mu s$，即 16 个周期递增 1 $\mu s$。

APB\_CLK 为配置寄存器等软件操作提供时钟信号。更多有关 APB\_CLK 的信息，见章节 \ref{mod:resclk} \textit{\nameref{mod:resclk}}。

用户可使用系统寄存器的两个相关位来控制系统定时器，具体如下：

\begin{itemize}
    \item 置位寄存器 \hyperref[regdesc:PCRSYSTIMERCONFREG]{PCR\_SYSTIMER\_CONF\_REG} 中 \hyperref[fielddesc:PCRSYSTIMERCLKEN]{PCR\_SYSTIMER\_CLK\_EN} 位，使能系统定时器的 APB\\\_CLK 信号。
    \item 置位寄存器 \hyperref[regdesc:PCRSYSTIMERCONFREG]{PCR\_SYSTIMER\_CONF\_REG} 中 \hyperref[fielddesc:PCRSYSTIMERRSTEN]{PCR\_SYSTIMER\_RST\_EN} 位，复位系统定时器。
\end{itemize}
注意，复位后，系统定时器的寄存器将恢复到默认值。更多信息可参考章节 \ref{mod:resclk} \textit{\nameref{mod:resclk}}。

\subsection{计数器}
系统定时器集成两个 52 位计数器，用 UNIT\regindex{n} 表示，\regindex{n} 可以取 0 或 1。计数器使用 16 MHz CNT\_CLK 作为计数时钟。用户可通过配置寄存器 \hyperref[regdesc:SYSTIMERCONFREG]{SYSTIMER\_CONF\_REG} 中的下面三个位来控制计数器 UNIT\regindex{n}，具体如下：
\begin{itemize}
    \item \hyperref[fielddesc:SYSTIMERTIMERUNIT1WORKEN]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_WORK\_EN}：置位此位，使能计数器 UNIT\regindex{n}。
    \item \hyperref[fielddesc:SYSTIMERTIMERUNIT1CORE0STALLEN]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_CORE0\_STALL\_EN}：置位此位，CPU0 停止运行后，计数器 UNIT\regindex{n} 也将停止工作。CPU 恢复运行后，计数器重新开始工作。
    \item \hyperref[fielddesc:SYSTIMERTIMERUNIT1CORE0STALLEN]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_CORE1\_STALL\_EN}：置位此位，CPU1 停止运行后，计数器 UNIT\regindex{n} 也将停止工作。CPU 恢复运行后，计数器重新开始工作。
\end{itemize}

计数器 UNIT\regindex{n} 处于工作状态时，计数值按计数周期递增。UNIT\regindex{n} 停止工作时，计数值将保持不变，不再递增。

计数起始值的低 32 位和高 20 位分别从 \hyperref[fielddesc:SYSTIMERTIMERUNIT0LOADLO]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_LOAD\_LO} 和 \hyperref[fielddesc:SYSTIMERTIMERUNIT0LOADHI]{SYSTIMER\_TIMER\_\\UNIT\regindex{n}\_LOAD\_HI} 中装载。置位 \hyperref[fielddesc:SYSTIMERTIMERUNIT0LOAD]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_LOAD} 将触发重装载事件，当前计数起始值立即更新。如果计数器 UNIT\regindex{n} 处于工作状态，则从新装载的计数值开始计数。

置位 \hyperref[fielddesc:SYSTIMERTIMERUNIT0UPDATE]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_UPDATE} 将触发更新事件。当前计数值的低 32 位和高 20 位被锁存至寄存器 \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUELO]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_LO} 和 \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEHI]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_HI} 后，\hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEVALID]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_VALID} 会被置起。在下次更新事件发生前，\hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUELO]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_LO} 和 \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEHI]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_HI} 寄存器中的值保持不变。




\subsection{比较器和报警}
系统定时器有三个 52 位比较器，用 COMP\regindex{x} 表示，其中 \regindex{x} 可以取 0、1、2。比较器可根据设置的不同报警值 (t) 或报警周期 ($\delta$t)，触发不同的中断。

用户可配置寄存器 \hyperref[fielddesc:SYSTIMERTARGET0PERIODMODE]{SYSTIMER\_TARGET\regindex{x}\_PERIOD\_MODE}，选择比较器 COMP\regindex{x} 生成报警的模式：
\begin{itemize}
    \item 1：周期报警模式
    \item 0：单次报警模式
\end{itemize}

选择周期报警模式时，寄存器 \hyperref[fielddesc:SYSTIMERTARGET2PERIOD]{SYSTIMER\_TARGET\regindex{x}\_PERIOD} 中的值为报警周期 ($\delta$t)。假设当前计数值为 t1，经过一段时间，当计数值达到 t1 + $\delta$t 时，将触发一次报警中断。再经过一段时间，当计数值达到 t1 + 2×$\delta$t 时，将再一次触发报警中断，以此类推。通过上述方式即可实现周期性报警。

选择单次报警模式时，\hyperref[fielddesc:SYSTIMERTIMERTARGET0LO]{SYSTIMER\_TIMER\_TARGET\regindex{x}\_LO} 和 \hyperref[fielddesc:SYSTIMERTIMERTARGET0HI]{SYSTIMER\_TIMER\_TARGET\regindex{x}\_HI} 分别提供报警值 (t) 的低 32 位和高 20 位。假设当前计数值为 t2 (t2 <= t)，经过一段时间，当计数到报警值 (t) 时，触发一次报警。与周期报警模式不同，单次报警模式仅生成一次报警中断。

用户可通过寄存器 \hyperref[fielddesc:SYSTIMERTARGET0TIMERUNITSEL]{SYSTIMER\_TARGET\regindex{x}\_TIMER\_UNIT\_SEL} 选择用于与 COMP\regindex{x} 进行比较的计数器值，然后生成报警：

\begin{itemize}
        \item 1：与计数器 UNIT\regindex{1} 的计数值进行比较
        \item 0：与计数器 UNIT\regindex{0} 的计数值进行比较
\end{itemize}

置位 \hyperref[fielddesc:SYSTIMERTARGET0WORKEN]{SYSTIMER\_TARGET\regindex{x}\_WORK\_EN}，COMP\regindex{x} 开始进行比较：
\begin{itemize}
    \item 在单次报警模式下，COMP\regindex{x} 将比较计数器中的实际计数值与寄存器中设置的报警值 (t)；
    \item 在周期报警模式下，COMP\regindex{x} 将比较计数器中的实际计数值与 t1 + n×$\delta$t (n = 1,2,3...)。
\end{itemize}

在单次报警模式下，当实际计数值等于报警值 (t)，或在周期报警模式下，实际计数值等于 t1 + n×$\delta$t (n = 1,2,3...)时，将触发一次报警中断。但如果设定的报警值 (t) 小于当前计数值，即报警值 (t) 已成为过去，在当前计数值超过设定的报警值 (t) 一定范围 (0 \verb+~+ $2^{51}$ –1)时，也将立即触发中断。无论是在单次报警模式还是在周期报警模式下，都可以从 SYSTIMER\_TARGET\regindex{x}\_LO\_RO 和 SYSTIMER\_TARGET\regindex{x}\_HI\_RO 中读取实际报警值的低 32 位和高 20 位。当前计数值 $t_{c}$、报警值 $t_{t}$ 和触发报警的关系如下表所示：

\begin{longtable}[c]{ | p{4cm} | p{6.5cm} |}
\caption{报警触发条件}
\\\hline
\rowcolor{lightgray}
\textbf{$t_{c}$ 与 $t_{t}$ 的关系}  & \textbf{触发条件} \\ \hline
$t_{c}$- $t_{t}$ <= 0 & 当 $t_{c}$ = $t_{t}$ 时，触发报警\\ \hline

\subsection{事件任务矩阵}
在 \chipname{} 中，系统定时器支持 ETM 功能，即可以通过任意外设的 ETM 事件触发系统定时器的 ETM 任务，或者通过系统定时器的 ETM 事件触发任意外设的 ETM 任务。关于 ETM 更多详细信息，请参考章节 \ref{mod:evntaskmatrix} \nameref{mod:evntaskmatrix}。这里仅介绍与系统定时器相关的 ETM 任务和 ETM 事件。

系统定时器可产生的 ETM 事件有：

\begin{itemize}
\item SYSTIMER\_EVT\_CNT\_CMP\regindex{x}：表示 COMP\regindex{x} 产生的报警事件。
\end{itemize}

当 \hyperref[fielddesc:SYSTIMERETMEN]{SYSTIMER\_ETM\_EN} 置 1 时，对应的报警事件会传到 ETM 模块。

\subsection{同步操作}

    \item 通过软件向配置寄存器字段写入合适的值，见表 \ref{synchronization-configuration-table} 第一列。

\end{enumerate}

\begin{enumerate}
    \item 软件将 1 写入更新寄存器 \hyperref[fielddesc:SYSTIMERTIMERUNIT0UPDATE]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_UPDATE} 中。
    \item 软件读取相应位 \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEVALID]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_VALID} 为有效，即表示同步完成。

\end{enumerate}

\section{编程示例}\label{sec:instructions-of-usage}
注意，在配置 COMP\regindex{x} 和 UNIT\regindex{n} 过程中，需保证对应的 COMP 和 UNIT 处于工作状态。
\subsection{读取当前计数器的值}\label{sec:to-read-system-timer-value}
\begin{enumerate}
    \item 置位 \hyperref[fielddesc:SYSTIMERTIMERUNIT0UPDATE]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_UPDATE}，将计数器 UNIT\regindex{n} 的值更新至寄存器 \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEHI]{SYSTIMER\_TIMER\_UNIT\regindex{n}\\\_VALUE\_HI} 和 \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUELO]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_LO}；
    \item 轮询 \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEVALID]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_VALID}，直至其值为 1。
    之后，用户可从寄存器 \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEHI]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_HI} 和 \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUELO]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_LO} 中读取计数器的值；
    \item 读取寄存器 \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUELO]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_LO}（低 32 位）和 \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEHI]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_HI}（高 20 位）。
\end{enumerate}

\subsection{在单次报警模式下配置一次性报警}
\begin{enumerate}
    \item 设置 \hyperref[fielddesc:SYSTIMERTARGET0TIMERUNITSEL]{SYSTIMER\_TARGET\regindex{x}\_TIMER\_UNIT\_SEL} 选择与 COMP\regindex{x} 进行比较的计数器（UNIT\regindex{0} 或 UNIT\regindex{1}）。
    \item 读取当前计数器的值，步骤见章节  \ref{sec:to-read-system-timer-value}。读取的当前值可用于计算步骤 4 中的报警值 (t)。
    \item 清除 \hyperref[fielddesc:SYSTIMERTARGET0PERIODMODE]{SYSTIMER\_TARGET\regindex{x}\_PERIOD\_MODE}，使能单次报警模式。
    \item 设置报警值 (t)，并将报警值 (t) 的低 32 位和高 20 位分别写入   \hyperref[fielddesc:SYSTIMERTIMERTARGET0LO]{SYSTIMER\_TIMER\_TARGET\regindex{x}\_LO} 和 \hyperref[fielddesc:SYSTIMERTIMERTARGET0HI]{SYSTIMER\_TIMER\_TARGET\regindex{x}\_HI}。
    \item 置位 \hyperref[fielddesc:SYSTIMERTIMERCOMP0LOAD]{SYSTIMER\_TIMER\_COMP\regindex{x}\_LOAD}，同步报警值 (t)，即将报警值 (t) 装载至比较器 COMP\regindex{x}。
    \item 置位 \hyperref[fielddesc:SYSTIMERTARGET0WORKEN]{SYSTIMER\_TARGET\regindex{x}\_WORK\_EN} 使能选择的比较器 COMP\regindex{x}；比较器 COMP\regindex{x} 开始比较计数值与报警值 (t)。
    \item 置位 \hyperref[fielddesc:SYSTIMERTARGET0INTENA]{SYSTIMER\_TARGET\regindex{x}\_INT\_ENA}，使能中断。UNIT\regindex{n} 达到报警值 (t) 时，触发一次报警中断 SYSTIMER\_\\TARGET\regindex{x}\_INT。
\end{enumerate}

\subsection{在周期报警模式下配置周期性报警}
\begin{enumerate}
    \item 设置 \hyperref[fielddesc:SYSTIMERTARGET0TIMERUNITSEL]{SYSTIMER\_TARGET\regindex{x}\_TIMER\_UNIT\_SEL} 选择与 COMP\regindex{x} 进行比较的计数器。
    \item 将报警周期 ($\delta$t) 写入 \hyperref[fielddesc:SYSTIMERTARGET0PERIOD]{SYSTIMER\_TARGET\regindex{x}\_PERIOD}。
    \item 置位 \hyperref[fielddesc:SYSTIMERTIMERCOMP0LOAD]{SYSTIMER\_TIMER\_COMP\regindex{x}\_LOAD} 同步报警周期值，即将 ($\delta$t) 装载至比较器 COMP\regindex{x}。
    \item 先清除再置位 \hyperref[fielddesc:SYSTIMERTARGET0PERIODMODE]{SYSTIMER\_TARGET\regindex{x}\_PERIOD\_MODE} 将 COMP\regindex{x} 配置为周期报警模式。
    \item 置位 \hyperref[fielddesc:SYSTIMERTARGET0WORKEN]{SYSTIMER\_TARGET\regindex{x}\_WORK\_EN} 使能选择的比较器 COMP\regindex{x}；比较器 COMP\regindex{x} 开始将计数值与计数初始值 + n×$\delta$t (n = 1, 2, 3...) 进行比较。
    \item 置位 \hyperref[fielddesc:SYSTIMERTARGET0INTENA]{SYSTIMER\_TARGET\regindex{x}\_INT\_ENA}，使能中断。UNIT\regindex{n} 计数达到计数初始值 + n×$\delta$t (n = 1, 2, 3...) 时，触发一次 SYSTIMER\_TARGET\regindex{x}\_INT 中断。
\end{enumerate}


\subsection{唤醒后时间补偿}
\begin{enumerate}
    \item 在芯片进入 Light-sleep 模式之前，用户需配置 RTC 定时器，精确记录睡眠时间，见章节 \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}。
    \item 从 Light-sleep 模式唤醒后，读取 RTC 定时器记录的睡眠时间。
    \item 读取当前系统定时器的计数值，见章节  \ref{sec:to-read-system-timer-value}。
    \item 将 RTC 记录的以 RTC\_SLOW\_CLK 周期为单位的睡眠时间，转换成以 CNT\_CLK (16 MHz) 周期为单位的睡眠时间。例如，如果 RTC\_SLOW\_CLK 频率为 32 kHz，则 RTC 定时器记录的时间乘以 500。
    \item 将 RTC 定时器转换后的值加到系统定时器的当前计数值：
    \begin{itemize}
        \item 将计算所得值的低 32 位写入 \hyperref[fielddesc:SYSTIMERTIMERUNIT0LOADLO]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_LOAD\_LO}，高 20 位写入 \hyperref[fielddesc:SYSTIMERTIMERUNIT0LOADHI]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_LOAD\_HI}。
        \item 置位 \hyperref[fielddesc:SYSTIMERTIMERUNIT0LOAD]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_LOAD}，将新的定时器值装载到系统定时器。至此，系统定时器更新完成。
        \end{itemize}
\end{enumerate}
