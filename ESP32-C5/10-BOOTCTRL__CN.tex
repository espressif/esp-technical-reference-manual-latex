\chapter{芯片 Boot 控制}
\label{mod:bootctrl}


\section{概述}

芯片在上电或硬件复位时，可以通过 strapping 管脚和 eFuse 参数配置如下启动参数，无需微处理器的参与：

\begin{itemize}
    \item \textbf{芯片启动模式}
    \begin{itemize}
        \item Strapping 管脚：GPIO26、GPIO27 和 GPIO28
    \end{itemize}
    \item \textbf{SDIO 输入采样沿和输出驱动沿控制}
        \begin{itemize}
            \item Strapping 管脚：GPIO25 和 MTDI
        \end{itemize}
    \item \textbf{ROM 代码日志打印}
        \begin{itemize}
            \item Strapping 管脚：GPIO27
            \item eFuse 参数：\hyperref[fielddesc:EFUSEUARTPRINTCONTROL]{EFUSE\_UART\_PRINT\_CONTROL} 和 \hyperref[fielddesc:EFUSEDISUSBSERIALJTAGROMPRINT]{EFUSE\_DIS\_USB\_SERIAL\_JTAG\_ROM\_PRINT}
        \end{itemize}
    \item \textbf{JTAG 信号源}
    \begin{itemize}
        \item Strapping 管脚：GPIO7
        \item eFuse 参数：\hyperref[fielddesc:EFUSEDISPADJTAG]{EFUSE\_DIS\_PAD\_JTAG}、\hyperref[fielddesc:EFUSEDISUSBJTAG]{EFUSE\_DIS\_USB\_JTAG} 和 \hyperref[fielddesc:EFUSEJTAGSELENABLE]{EFUSE\_JTAG\_SEL\_ENABLE}
    \end{itemize}
\end{itemize}

上述 eFuse 位的默认值均为 0，也就是说没有烧写过。eFuse 只能烧写一次，一旦烧写为 1，便不能恢复为 0。有关烧写 eFuse 的信息，请参考章节 \ref{mod:efuse} \textit{\nameref{mod:efuse}}。

在芯片复位过程中（请参考章节 \ref{mod:resclk} \textit{\nameref{mod:resclk}}），硬件将采样 strapping 管脚电平存储到锁存器中，并一直保持到芯片掉电或下一次芯片复位。strapping 管脚的锁存值可以通过软件从寄存器 \hyperref[fielddesc:GPIOSTRAPPING]{GPIO\_STRAPPING} 中读取。

\section{功能描述}

本小节主要介绍芯片复位时的功能以及控制该功能使用到的 strapping 管脚和 eFuse 组合模式。

\begin{importantlisting}
请使用本章节所介绍的组合，其它组合可能会导致不可控结果。
\end{importantlisting}

\subsection{芯片启动模式控制}
\label{sec:bootModeControl}

复位释放后，GPIO26、GPIO28 和 GPIO27 共同决定启动模式，详见表 \ref{tab:bootctrl-boot-mode-control} \textit{\nameref{tab:bootctrl-boot-mode-control}}。

\begin{table}[H]
    \centering
    \caption{芯片启动模式控制}
    \label{tab:bootctrl-boot-mode-control}

    \begin{threeparttable}
    \begin{tabular}{ | l | c | c | c | }
    \hline
        \rowcolor{lightgray}
        \textbf{启动模式} & \textbf{GPIO26} & \textbf{GPIO27} & \textbf{GPIO28} \\\hline
        \textbf{SPI Boot}\tnote{1} & 任意值 & 任意值 & \textbf{1}\tnote{1} \\\hline
        Joint Download Boot 0\tnote{2} & 任意值 & 1 & 0 \\\hline
        Joint Download Boot 1\tnote{3} & 0 & 0 & 0 \\\hline
        \end{tabular}

        \begin{tablenotes}
            \item[1] \textbf{加粗}表示默认值和默认配置。
            \item[2] Joint Download Boot 0 模式下支持以下下载方式：
                \begin{itemize}
                    \item USB-Serial-JTAG Download Boot
                    \item UART Download Boot
                \end{itemize}
            \item[3] Joint Download Boot 1 模式下支持以下下载方式：
                \begin{itemize}
                    \item UART Download Boot
                    \item SDIO Download Boot
                \end{itemize}
        \end{tablenotes}
     \end{threeparttable}
\end{table}

在 SPI Boot 模式下，ROM 引导加载程序通过从 SPI flash 中读取程序来启动系统。

在 Joint Download Boot 0 模式下，用户可通过 UART0、USB 或 SPI Slave 接口将二进制文件下载至 flash，或将二进制文件下载至 SRAM 并运行 SRAM 中的程序。

在 Joint Download Boot 1 模式下，用户可通过 UART0、SDIO 接口将二进制文件下载至 flash，或将二进制文件下载至 SRAM 并运行 SRAM 中的程序。

芯片启动的具体流程见图 \ref{fig:boot_flow} \textit{\nameref{fig:boot_flow}}。

\begin{figure}[H]
    \centering
    {\includegraphics[width=1.0\textwidth]{\modulefiles/figures/boot_flow.eps}}\\
    \caption{芯片启动流程}
    \label{fig:boot_flow}
\end{figure}

下面几个 eFuse 可用于控制启动模式的具体行为：

\begin{itemize}
    \item \hyperref[fielddesc:EFUSEDISFORCEDOWNLOAD]{\textbf{EFUSE\_DIS\_FORCE\_DOWNLOAD}}
    \begin{itemize}
        \item 如果此 eFuse 设置为 0（默认），软件可通过设置 \hyperref[fielddesc:LPAONFORCEDOWNLOADBOOT]{LP\_AON\_FORCE\_DOWNLOAD\_BOOT}，触发 CPU 复位，将芯片启动模式强制从 SPI Boot 模式切换至 Joint Download Boot 模式。这种情况下，硬件会将寄存器 \hyperref[fielddesc:GPIOSTRAPPING]{GPIO\_STRAPPING}[4:3] 的值从 "1x" 覆盖为 "01"；

        \item 如果此 eFuse 设置为 1，则禁用 \hyperref[fielddesc:LPAONFORCEDOWNLOADBOOT]{LP\_AON\_FORCE\_DOWNLOAD\_BOOT}, 寄存器 \hyperref[fielddesc:GPIOSTRAPPING]{GPIO\_STRAPPING} 的值不受影响 。
    \end{itemize}

    \item \hyperref[fielddesc:EFUSEDISDOWNLOADMODE]{\textbf{EFUSE\_DIS\_DOWNLOAD\_MODE}}

    如果此 eFuse 设置为 1，则禁用 Joint Download Boot 模式，寄存器 \hyperref[fielddesc:GPIOSTRAPPING]{GPIO\_STRAPPING} 的值则不受 \hyperref[fielddesc:LPAONFORCEDOWNLOADBOOT]{LP\_AON\_FORCE\_DOWNLOAD\_BOOT} 的影响。

    \item \hyperref[fielddesc:EFUSEENABLESECURITYDOWNLOAD]{\textbf{EFUSE\_ENABLE\_SECURITY\_DOWNLOAD}}

    如果此 eFuse 设置为 1，则在 Joint Download Boot 模式下，只允许读取、写入和擦除明文 flash，不支持 SRAM 或寄存器操作。如已禁用 Joint Download Boot 模式，请忽略此 eFuse。

    \item \hyperref[fielddesc:EFUSEDISDIRECTBOOT]{\textbf{EFUSE\_DIS\_DIRECT\_BOOT}}

    如果此 eFuse 设置为 1，则禁用 Direct Boot 模式。
\end{itemize}

USB 串口/JTAG 控制器可将芯片从 SPI Boot 模式强制切换到 Joint Download Boot 模式，或从 Joint Download Boot 模式强制切换到 SPI Boot 模式。更多信息，请参考章节 \ref{mod:usbserialjtag} \textit{\nameref{mod:usbserialjtag}}。

    \item USB 串口/JTAG 控制器

\subsection{JTAG 信号源控制}
