\chapter{低功耗管理}\label{mod:lowpowm}

\section{概述}

\chipname{} 拥有先进的低功耗管理系统，可以优化设备的功耗，并同时保持高性能。

该系统通过采用多种节能技术，例如睡眠模式、动态电压和频率调节以及外设电源门控等，可以有效地将设备功耗降至最低。

在这一管理系统中，电源管理单元 (PMU) 扮演着核心角色，它可以灵活地控制各个电源域的上下电，以此实现芯片性能、功耗和唤醒时间之间的最佳平衡。

\section{术语}

下列有关低功耗管理的术语是在《\chipname{} 技术参考手册》的语境下定义的，旨在帮助读者更好地理解本文档。

\begin{longtable}[c]{ R{4cm} p{12cm} }
\textbf{低功耗管理系统} & 指的是管理芯片功耗的整个系统。\\
\textbf{电源管理单元 (PMU)} & 指的是负责芯片功耗管理的核心控制电路，它包括对芯片电源、时钟以及功耗相关逻辑的控制。\\
\textbf{电源域} & 指的是可独立控制上下电的最小单元。一个电源域可能包含一个或多个模块。 \\
\textbf{PMU 状态} & 指的是 PMU 状态机的状态。用户可以配置不同电源域在每一种状态下的时钟开关和电源开关。\\
\textbf{功耗模式} & 指的是预设的典型功耗模式，可满足一些常见场景的功耗需求。\\
\end{longtable}

\section{主要特性}

PMU 具有以下特性：

\begin{itemize}
    \item 支持 4 种可配置 PMU 状态，软件可根据需求灵活配置
    \begin{itemize}
        \item HP\_ACTIVE
        \item HP\_MODEM
        \item HP\_SLEEP
        \item LP\_SLEEP
    \end{itemize}
    \item 支持 5 种预设功耗模式，可满足应用场景中的多种典型功耗需求
    \begin{itemize}
        \item Active
        \item Modem-sleep
        \item Light-sleep0
        \item Light-sleep1
        \item Deep-sleep
    \end{itemize}
    \item 16 KB SRAM (LP SRAM)
    \item 10 个 ``always-on" (AON) 寄存器
    \item RTC 快速启动
    \item 可灵活配置的备份 DMA，可在芯片切换 PMU 状态时，备份与恢复 CPU 及外设的关键数据
    \item 功耗控制器，可根据芯片所处的功耗模式切换状态，以控制电源和时钟等
\end{itemize}

\section{功能描述}

\chipname{} 的低功耗管理系统包括如下组成部分:

\begin{itemize}
    \item \textbf{电源结构}：即供电管理网络，包括调压器、数字电源域、模拟电源域等部分。

    \item \textbf{PMU 控制器}：PMU 的核心部分，用于实现模拟电源、数字电源、时钟等系统控制。

    \item \textbf{1 个 RTC 定时器}：更多信息，请参考章节 \ref{mod:rtctimer} \textit{\nameref{mod:rtctimer}}。

    \item \textbf{10 个 ``always-on" 寄存器 (\hyperref[regdesc:LPAONSTORENREG]{LP\_AON\_STORE0\_REG} $\sim$ \hyperref[regdesc:LPAONSTORENREG]{LP\_AON\_STORE9\_REG})}：这 10 个寄存器永远处于工作状态，不受功耗模式的影响，可用于存储一些关键数据。

    \item \textbf{7 个 LP GPIO 管脚 (GPIO0 $\sim$ GPIO6)}：这 7 个管脚永远处于工作状态，不受功耗模式的影响，可用作睡眠模式下的唤醒源，也可以作为标准的 GPIO 管脚使用。更多细节，请见章节 \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}}。

    \item \textbf{16 KB LP SRAM}：HP CPU 与 LP CPU 均可访问。HP CPU 访问时，该存储器使用 HP CPU 时钟，而在 LP CPU 访问时，使用 LP CPU 时钟。

\end{itemize}

以下详细介绍这几个组成部分。

\begin{itemize}
    \item 2 个调压器
    \item 模拟电源域
    \item 数字电源域
\end{itemize}

\subsubsection{调压器}\label{sec:lowpowm-regulators}

如图 \ref{fig:lowpowm-power-structure} 所示，\chipname{} 的模拟部分包含 2 个调压器，负责调节不同电源域的供电：

\begin{itemize}
    \item 1 个 HP（高性能）系统调压器，负责调节高性能模块的供电，驱动能力较强，功耗较大，输出电压可调。
    \item 1 个 LP（低功耗）系统调压器，负责调节低功耗模块的供电，输出电压同样可调。
\end{itemize}


\subsubsection{数字电源域}\label{sec:lowpowm_digital_domain}

\chipname{} 的数字部分主要分为如下几个数字电源域，其中 HP 系统由 HP 系统调压器供电，调压器与每个电源域之间均有一个独立的电源开关，可实现数字电源域的上下电控制；LP 系统由 LP 系统调压器供电。

\begin{itemize}
    \item HP 系统的电源域包括：
    \begin{itemize}
        \item \textbf{CPU}：主要包括 CPU 及支持其工作的外设（追踪模块等）。
        \item \textbf{Modem}：包括无线 MAC 和基带
        \item \textbf{Peripherals + ROM}：主要包括总线、HP 外设等功能模块。

        \item \textbf{Internal SRAMx}：细分了 4 个子电源域，如图 \ref{fig:lowpowm-power-structure} 所示，SRAM0/1/2 每个都有独立的开关控制供电，SRAM3 与调压器直接连接，无开关控制上下电。
        \item \textbf{Modem Power}：主要包括控制无线部分工作状态的各个模块。
    \end{itemize}
    \item LP 系统的电源域包括：
    \begin{itemize}
        \item \textbf{LP PD peripherals}：主要包括 LP CPU、LP 外设。
        \item \textbf{LP always-on}：主要包括 LP always on 外设（如 RTC 定时器）和 PMU 控制模块，所有模式下均不掉电。
    \end{itemize}
\end{itemize}

\subsubsection{模拟电源域}

HP 系统包括以下模拟电源域：

\begin{itemize}
    \item PLL
\end{itemize}

LP 系统包括以下模拟电源域：

\begin{itemize}
    \item External Main Clock（外部主时钟）
    \item Fast RC Oscillator（快速 RC 振荡器）
    \item RF circuit（射频电路）
\end{itemize}

\subsection{PMU}

\chipname{} 的 PMU 控制着每个电源域功耗相关的组件，如电源、时钟，它主要由如下几个部分构成：

\begin{itemize}
    \item \textbf{PMU 主状态机}：记录并切换 PMU 状态
    \item \textbf{睡眠与唤醒控制器}：用于向 PMU 主状态机发起睡眠与唤醒请求
    \item \textbf{功耗控制器}：可根据芯片所处的功耗模式切换状态，以控制电源和时钟等。功耗控制器包括：
        \begin{itemize}
            \item \textbf{数字电源控制器}：用于控制数字电源域的上下电
            \item \textbf{模拟电源控制器}：用于控制模拟模块的使能，比如调压器、模拟时钟等
            \item \textbf{时钟控制器}：用于管理外设时钟的门控，以及选择数字时钟的模拟时钟源
            \item \textbf{数据备份控制器}：用于控制 PMU 状态切换时的数据备份与恢复过程
            \item \textbf{系统控制器}：用于控制部分系统级模块，如睡眠模式下（此时 CPU 不可用）看门狗功能的挂起等
        \end{itemize}
\end{itemize}

PMU 的工作流程为：
\begin{itemize}
    \item 睡眠与唤醒控制器向 PMU 主状态机发送睡眠或唤醒请求。
    \item PMU 主状态机接着产生电源门控、时钟门控和复位信号。
    \item 电源控制器和时钟控制器会根据 PMU 主状态机产生的信号，打开或关闭不同的电源域和时钟信号，从而让芯片进入或退出各种低功耗模式。
\end{itemize}

PMU 的工作流程如图 \ref{fig:lowpowm-workflow} 所示。其中，HP\_SWITCH 和 LP\_SWITCH 的定义如下：

\begin{itemize}
\item HP\_SWITCH：HP 系统在 HP\_ACTIVE 和 HP\_SLEEP 之间切换的中间状态。硬件将在这个状态完成各个控制器的控制切换。
\item LP\_SWITCH：LP 系统在 HP\_SLEEP 和 LP\_SLEEP 之间切换的中间状态。硬件将在这个状态完成各个控制器的控制切换。
\end{itemize}

    \caption{PMU 的主要工作流程}\label{fig:lowpowm-workflow}
\end{figure}

以下章节分别介绍 PMU 的各个主要组成部分。

\subsubsection{PMU 主状态机}

PMU 主状态机可以接收睡眠与唤醒信号，并通过功耗控制器完成电源上下电和时钟开关，从而实现 PMU 状态的切换，使芯片在目标 PMU 状态下达到性能与功耗的平衡。

PMU 主状态机包括 4 个 PMU 状态，每个状态由不同的睡眠与唤醒信号控制切换，支持软件对每个状态进行个性化的配置，如配置电源、时钟等。软件可基于这 4 个 PMU 状态，拓展出不同的应用层面的功耗模式。4 个 PMU 状态为：

\begin{itemize}
    \item HP\_ACTIVE：所有电路最大程度开启的 PMU 状态，该状态支持 HP 系统和 LP 系统工作。
    \item HP\_MODEM：该状态支持 Modem 系统（无线 MAC 和基带）独立于 CPU 工作。
    \item HP\_SLEEP：HP 系统睡眠状态，支持 LP 外设工作。
    \item LP\_SLEEP：HP 系统与 LP 外设均处于睡眠状态，always-on 电路保持工作。
\end{itemize}
\newpage
\begin{tiplisting}
    HP 和 LP 系统的划分为：
    \begin{itemize}
    \item LP 系统（外设）：LP RISC-V 32-bit Microprocessor、LP GPIO、LP UART、LP I2C
    \item LP 系统（always-on 电路）：LP Memory (LP SRAM)、PMU、RTC 看门狗定时器、RTC 超级看门狗、RTC 定时器、eFuse 控制器、电源毛刺监测器
    \item HP 系统：除 LP 系统以外的所有外设，包括 Modem
    \end{itemize}
    更多内容可参考 \href{https://www.espressif.com/sites/default/files/documentation/esp32-c5_datasheet_cn.pdf}{《\chipname{} 技术规格书》} > 功能框图。
\end{tiplisting}

HP\_ACTIVE，HP\_MODEM，HP\_SLEEP 为 HP 系统的状态，LP\_SLEEP 为 LP 系统的状态。
\begin{itemize}
    \item 某个模块如果属于 HP 系统，则只能配置在 HP\_ACTIVE/HP\_MODEM/HP\_SLEEP 三种状态下的上下电，在 LP\_SLEEP 的状态下会复用 HP\_SLEEP 的状态。
    \item 某个模块如果属于 LP 系统，则只能配置 LP\_SLEEP 状态下的上下电，HP\_ACTIVE/HP\_MODEM/HP\_SLEEP 三种状态复用 HP\_SLEEP 的状态。
\end{itemize}

例如，HP CPU 属于 HP 系统，则可以通过以下寄存器配置 HP CPU 在 HP\_ACTIVE/HP\_MODEM/HP\_SLEEP 三种状态下的上下电，LP\_SLEEP 复用 HP\_SLEEP 的配置。

\begin{itemize}
    \item HP\_ACTIVE：\hyperref[fielddesc:PMUHPACTIVEPDHPCPUPDEN]{PMU\_HP\_ACTIVE\_PD\_HP\_CPU\_PD\_EN}
    \item HP\_MODEM：\hyperref[fielddesc:PMUHPMODEMPDHPCPUPDEN]{PMU\_HP\_MODEM\_PD\_HP\_CPU\_PD\_EN}
    \item HP\_SLEEP：\hyperref[fielddesc:PMUHPSLEEPPDHPCPUPDEN]{PMU\_HP\_SLEEP\_PD\_HP\_CPU\_PD\_EN}
\end{itemize}

类似地，用户可以用同样的方法，定义其他电源域在不同 PMU 状态下的上下电，具体寄存器请见第 \ref{sec:lowpowm-reg-summ} 小节。

\begin{tiplisting}
为方便描述，后文中此类寄存器全部统称为 PMU\_\regindex{PMUSTATE}\_PD\_\regindex{POWERDOMAIN}\_PD\_EN，\regindex{PMUSTATE} 代表 PMU 的 4 种状态。
\end{tiplisting}

配置完成后，PMU 将借助各控制器使这些配置生效，详见下方各章节的描述。

\subsubsection{睡眠与唤醒控制器}

睡眠与唤醒控制器负责向 PMU 主状态机发起睡眠与唤醒请求。\chipname{} 支持多种唤醒源将 CPU 从不同功耗模式下唤醒，用户可通过 \hyperref[fielddesc:PMUWAKEUPENA]{PMU\_WAKEUP\_ENA} 按照下表使能对应的唤醒源。

\begin{table}[H]
    \centering
    \caption{唤醒源}
    \label{tab:lowpowm-wakeup-source}
    \begin{threeparttable}
    \begin{tabular}{| r | l | c | c |}
    \hline
    \rowcolor{lightgray}
    \textbf{\hyperref[fielddesc:PMUWAKEUPENA]{PMU\_WAKEUP\_ENA}} & \textbf{唤醒源} &  \textbf{Light-sleep} & \textbf{Deep-sleep}  \\\hline
    0x2 & EXT\_IO\tnote{1} & Y & Y \\\hline
    0x4 & GPIO\tnote{1} & Y & Y \\\hline
    0x8 & Wi-Fi beacon  & Y & - \\\hline
    0x10 & RTC Timer  & Y & Y \\\hline
    0x20 & Wi-Fi\tnote{2} & Y & – \\\hline
    0x40 & UART0\tnote{3} & Y & – \\\hline
    0x80 & UART1\tnote{3} & Y & – \\\hline
    0x100 & SDIO 从机控制器\tnote{4} & Y & – \\\hline
    0x400 & Bluetooth & Y & – \\\hline
    0x800 & LP CPU  & Y & Y \\\hline
    \end{tabular}

\begin{tablenotes}
    \item[1] Deep-sleep模式下，只有 LP GPIO 可作为唤醒源。

    \item[2] 为了通过 Wi-Fi 唤醒芯片，芯片将在 Active、Modem-sleep 和Light-sleep 之间进行切换， RF 模块将在预设间隔中唤醒，保证 Wi-Fi 的正常连接和数据通信。

    \item[3] 当接收到的 RX 脉冲数量超过阈值寄存器 \hyperref[fielddesc:UARTACTIVETHRESHOLD]{UART\_ACTIVE\_THRESHOLD} 的设置时，即触发唤醒。详情请见章节 \ref{mod:uart} \textit{\nameref{mod:uart}}。
    \item[4] 当 SDIO 从机控制器接收到 CMD52/CMD53 的时候，触发唤醒。该唤醒模式需要休眠的电压高于 0.9 V。电压配置存在芯片体质差异。
    \end{tablenotes}
    \end{threeparttable}
\end{table}

\chipname{} 提供了硬件拒绝睡眠的机制，即当某些外设处于不可中断的工作状态而 CPU 试图进入睡眠状态时，外设将会发出唤醒信号，以此来拒绝 CPU 的睡眠命令，从而保证外设正常工作。

表 \ref{tab:lowpowm-wakeup-source} 中的唤醒源均可配置成作为拒绝睡眠的事件，用户可配置如下寄存器实现拒绝睡眠。\hyperref[fielddesc:PMUSLEEPREJECTENA]{PMU\_SLEEP\_REJECT\_ENA} 和 \hyperref[fielddesc:PMUREJECTCAUSE]{PMU\_SLP\_REJECT\_CAUSE\_REG} 的配置值与唤醒源的对应关系均与表中一致。

\begin{itemize}
    \item  使能拒绝睡眠功能
        \begin{itemize}
            \item 置位 \hyperref[fielddesc:PMUSLPREJECTEN]{PMU\_SLP\_REJECT\_EN} 使能拒绝睡眠功能
            \item 配置 \hyperref[fielddesc:PMUSLEEPREJECTENA]{PMU\_SLEEP\_REJECT\_ENA} 使能拒绝睡眠信号源
        \end{itemize}
    \item 读取寄存器 \hyperref[fielddesc:PMUREJECTCAUSE]{PMU\_SLP\_REJECT\_CAUSE\_REG} 确认拒绝睡眠的事件源
\end{itemize}

\subsubsection{模拟电源控制器}\label{sec:pmu_analog_power_controller}

模拟电源控制器用于控制模拟电路部分（包括调压器、高速时钟、低速时钟）在某种 PMU 状态下的上下电情况。

调压器的具体配置如下：

\begin{itemize}
    \item 配置寄存器 \hyperref[fielddesc:PMUHPACTIVEHPREGULATORXPD]{PMU\_\regindex{PMUSTATE}\_HP\_REGULATOR\_XPD} 或 \hyperref[fielddesc:PMUHPSLEEPLPREGULATORXPD]{PMU\_\regindex{PMUSTATE}\_LP\_REGULATOR\_XPD} 可使能或关闭目标 PMU 状态下 HP 系统或 LP 系统调压器的输出电压。LP 系统调压器不建议关闭，否则会导致 PMU 本身掉电，从而导致芯片异常。
\end{itemize}

高性能时钟（XTAL\_CLK 和 PLL\_CLK）的具体配置如下：

\begin{itemize}
    \item XTAL\_CLK：配置 \hyperref[fielddesc:PMUHPSLEEPXPDXTAL]{PMU\_HP\_SLEEP\_XPD\_XTAL} 为 1，在芯片 PMU 状态切换至 HP\_SLEEP 的过程中，开启 XTAL\_CLK。

    注意：为避免 XTAL\_CLK 起振阶段的不稳定时钟影响电路，可配置 \hyperref[fielddesc:PMUWAITXTALSTABLE]{PMU\_WAIT\_XTAL\_STABLE} 实现 XTAL\_CLK 全局门控的延迟打开，即该门控将在 XTAL\_CLK 上电后的 \hyperref[fielddesc:PMUWAITXTALSTABLE]{PMU\_WAIT\_XTAL\_STABLE} 个 CLK\_DYN\_FAST\_CLK 周期时间后打开。

    \item PLL\_CLK：PMU 可在芯片处于不同 PMU 状态下打开 PLL\_CLK，仅需配置不同 PMU 状态下的 \hyperref[fielddesc:PMUHPACTIVEXPDBBPLL]{PMU\_HP\_ACTIVE\_XPD\_BBPLL} 寄存器为 1。例如，配置 \hyperref[fielddesc:PMUHPACTIVEXPDBBPLL]{PMU\_HP\_ACTIVE\_XPD\_BBPLL} 为 1，则在芯片处于 HP\_ACTIVE 状态下，打开 PLL\_CLK 时钟。

    注意：
    \begin{itemize}
        \item 在打开 PLL\_CLK 前还需确保 XTAL\_CLK 已经稳定开启。
        \item 为避免 PLL\_CLK 起振阶段的不稳定时钟影响电路，可配置 \hyperref[fielddesc:PMUWAITPLLSTABLE]{PMU\_WAIT\_PLL\_STABLE} 实现 PLL\_CLK 全局门控的延迟打开，即该门控将在 PLL\_CLK 上电后的 \hyperref[fielddesc:PMUWAITPLLSTABLE]{PMU\_WAIT\_PLL\_STABLE} 个CLK\_DYN\_FAST\_CLK 周期时间后打开。
    \end{itemize}
\end{itemize}

低速时钟功耗较低（RC\_FAST\_CLK 和 XTAL32K\_CLK），在 HP\_ACTIVE、HP\_MODEM 和 HP\_SLEEP 状态下的开关情况均由 \hyperref[regdesc:PMUHPSLEEPLPCKPOWERREG]{PMU\_HP\_SLEEP\_LP\_CK\_POWER\_REG} 决定。在 LP\_SLEEP 状态下由 \hyperref[regdesc:PMULPSLEEPLPCKPOWERREG]{PMU\_LP\_SLEEP\_LP\_CK\_POWER\_REG} 配置。

\subsubsection{数字电源控制器}

数字电源控制器用于控制数字电源域在某种 PMU 状态下的上下电情况。与模拟电源控制器不同的是，数字电源控制器不直接控制调压器，而是通过控制与调压器相连的电源开关，实现各个数字电源域的上下电。

数字电源域中 LP PD Peripherals 只支持在 HP\_SLEEP 与 LP\_SLEEP 两个 PMU 状态之间切换时上下电，其余电源域只支持在 HP\_SLEEP、HP\_ACTIVE，HP\_MODEM 三个 PMU 状态间切换时上下电。

当芯片在不同 PMU 状态之间切换时，如果某电源域在当下 PMU 状态下的上下电配置与即将切换至的上下电配置不符，则将激活上下电流程。以 CPU 电源域从上电到下电的流程为例。配置寄存器 \hyperref[fielddesc:PMUHPMODEMPDHPCPUPDEN]{PMU\_HP\_MODEM\_PD\_HP\_CPU\_PD\_EN} 为 1，表示在 HP\_MODEM 状态下关闭 CPU 电源域的供电；或配置为 0，表示在 HP\_MODEM 状态下对 CPU 电源域供电。PMU 将做出如下控制：

\begin{itemize}

\item 使能数字隔离单元，确保下电模块不往上电模块输出不稳定的电平值。当有电源域掉电时，该模块的输出将被钳位至固定值。

\item 使能复位，CPU 电源域掉电后，该电源域的全局复位信号处于复位状态，在重新上电后的一段时间内该电源域仍然保持复位，这将确保该电源域在上电的过程中存在一个复位到释放的过程，消除上下电引入的寄存器不定态。

\end{itemize}

下面说明各个数字电源域的开关方式：

\begin{itemize}
\item Internal SRAMx

    Internal SRAMx 电源域的上下电状态并无专门寄存器控制，它与 Peripherals 共用一套 \hyperref[fielddesc:PMUHPACTIVEPDTOPPDEN]{PMU\_\regindex{PMUSTATE}\_PD\_TOP\_PD\_EN} 寄存器。若未配置 \hyperref[fielddesc:PMUPDHPMEM0PDMASK]{PMU\_PD\_HP\_MEM\regindex{n}\_PD\_MASK} (\regindex{n}=0,1,2) 寄存器，则 Internal SRAMx 与 Peripherals 电源域将同时上下电；若配置 \hyperref[fielddesc:PMUPDHPMEM0PDMASK]{PMU\_PD\_HP\_MEMn\_PD\_MASK} 寄存器为 1，则 Internal SRAMx 在 Peripherals 掉电时可保持上电。

\item Modem Power

    Modem Power 与 HP 系统调压器通过一个开关连接，可通过配置寄存器 \hyperref[fielddesc:PMUHPACTIVEPDHPAONPDEN]{PMU\_HP\_ACTIVE\_PD\_HP\_AON\_PD\_EN} 决定 HP\_ACTIVE 状态下 Modem Power 的开关。由图 \ref{fig:lowpowm-power-structure} 可知，CPU、Modem、Peripherals + ROM 中若要一个电源域上电，则需同时保证 Modem Power 电源域上电，这种设计也与功能需求保持一致。

\item Peripherals + ROM/Modem/CPU

    可通过寄存器 PMU\_\regindex{PMUSTATE}\_PD\_\regindex{n}\_PD\_EN (\regindex{n}=TOP/HP\_WIFI/HP\_CPU) 配置各电源域在对应 PMU 状态下的上下电（寄存器中的 WIFI 即为 Modem电源域）。

    Peripherals 掉电时还存在以下几个可配功能：
        \begin{itemize}
        \item Peripherals 掉电后会导致 GPIO 管脚状态不稳定，可通过 PMU 控制 GPIO 管脚（8 个 LP GPIO 除外）进入保持状态。例如配置 \hyperref[fielddesc:PMUHPSLEEPHPPADHOLDALL]{PMU\_HP\_SLEEP\_HP\_PAD\_HOLD\_ALL} 使 GPIO 管脚在 HP\_SLEEP 下保持 Peripherals 掉电前的相关配置。

        \item 在 HP\_ACTIVE 和 HP\_MODEM 状态下，除了支持 Internal SRAMx 电源域掉电外，还支持标准供电下的 Memory Deep-sleep 模式，该模式下存储器将无法读写，但可以保持数据。配置寄存器 \hyperref[fielddesc:PMUHPACTIVEHPMEMDSLP]{PMU\_\regindex{PMUSTATE}\_HP\_MEM\_DSLP} 可实现 Memory Deep-sleep 模式。
        \end{itemize}

\item LP PD Peripherals

    LP PD Peripherals 电源域在芯片处于 HP\_ACTIVE 和 HP\_MODEM 状态下始终保持上电，无法配置，仅允许配置 HP\_SLEEP 与 LP\_SLEEP 状态下的上下电。此电源域拥有独立的数字电源开关，且它的上下电控制不依赖于其他数字电源域的上下电。
\end{itemize}

\subsubsection{时钟控制器}

时钟控制器主要用于芯片 PMU 状态切换时，对时钟源的选择及部分时钟的使能，以此实现对高性能系统时钟与低功耗系统时钟的控制。

高性能系统时钟包括 HP\_ROOT\_CLK 和高性能系统外设时钟。当芯片的 PMU 状态在 HP\_ACTIVE、 HP\_MODEM 与 HP\_SLEEP 之间切换时，PMU 可实现对 HP\_ROOT\_CLK 时钟源的切换、开关、分频，以及高性能系统外设时钟的开关。


\begin{itemize}

\item HP\_ROOT\_CLK 控制的相关寄存器及功能介绍如下：

    \begin{itemize}
    \item 配置不同 PMU 状态对应的 \hyperref[fielddesc:PMUHPACTIVESYSCLKSLPSEL]{PMU\_\regindex{PMUSTATE}\_SYS\_CLK\_SLP\_SEL} 寄存器为 1，表示芯片进入该 PMU 状态后，时钟源受 PMU 控制。

    \item 配置不同 PMU 状态对应的 \hyperref[fielddesc:PMUHPACTIVEICGSYSCLOCKEN]{PMU\_\regindex{PMUSTATE}\_ICG\_SYS\_CLOCK\_EN} 寄存器为 0，表示芯片进入该 PMU 状态后，关闭 HP\_ROOT\_CLK 的时钟门控，即该时钟将不可用。

    \item 配置不同 PMU 状态对应的 \hyperref[fielddesc:PMUHPACTIVEDIGSYSCLKSEL]{PMU\_\regindex{PMUSTATE}\_DIG\_SYS\_CLK\_SEL}，选择芯片进入对应 PMU 状态后使用的时钟源，具体请见章节 \ref{mod:resclk} \textit{\nameref{mod:resclk}} > 表 \ref{tab:resclk-cpu-clk-src}。
    \end{itemize}

\item 高性能系统外设时钟控制的相关寄存器及功能介绍如下：

    \begin{itemize}
    \item 配置 \hyperref[fielddesc:PMUHPACTIVEICGSLPSEL]{PMU\_\regindex{PMUSTATE}\_ICG\_SLP\_SEL} 寄存器为 1，目标状态下的各时钟门控的开关将受 PMU 的控制，该寄存器为 0 时，门控开关由 PCR 寄存器决定。

    \item 配置寄存器 \hyperref[fielddesc:PMUHPACTIVEDIGICGFUNCEN]{PMU\_\regindex{PMUSTATE}\_DIG\_ICG\_FUNC\_EN} 决定目标 PMU 状态的功能时钟门控开关。具体配置请见章节 \ref{mod:resclk} \textit{\nameref{mod:resclk}} > 表 \ref{tab:clk_pmu_func}。

    \item 配置寄存器 \hyperref[fielddesc:PMUHPACTIVEDIGICGAPBEN]{PMU\_\regindex{PMUSTATE}\_DIG\_ICG\_APB\_EN} 决定目标 PMU 状态的 APB 时钟门控开关。具体配置请见章节 \ref{mod:resclk} \textit{\nameref{mod:resclk}} > 表 \ref{tab:clk_pmu_apb}。

    \end{itemize}

\end{itemize}

低功耗系统时钟主要用于低功耗系统，包括如下四种时钟：

    \begin{itemize}
    \item LP\_SLOW\_CLK
    \item LP\_FAST\_CLK
    \item LP\_DYN\_SLOW\_CLK
    \item LP\_DYN\_FAST\_CLK
    \end{itemize}

其中 LP\_DYN\_FAST\_CLK 的时钟频率根据 PMU 状态由硬件完成如下固定切换（用户无法更改）：

    \begin{itemize}
    \item LP\_SLEEP：频率与 LP\_SLOW\_CLK 一致。
    \item HP\_ACTIVE、HP\_MODEM、HP\_SLEEP：频率与 LP\_FAST\_CLK 一致。
    \end{itemize}

\subsubsection{数据备份控制器}

\chipname{} 支持在切换 PMU 状态时，驱动一个 Retention DMA 模块，进行存储器与外设间的数据搬移，从而在电源域掉电时备份数据并在再次上电时恢复数据。

数据搬移操作由 Peripherals 电源域中的模块实现，PMU 只产生相关控制信号。需要注意的是，决定数据搬移的控制寄存器存在方向性，因为与 PMU 的其他控制寄存器不同是，这些控制行为由原来的 PMU 状态和目标 PMU 状态共同决定。

以目标 PMU 状态为 HP\_SLEEP 为例，从 HP\_ACTIVE 睡眠至 HP\_SLEEP 与从 HP\_MODEM 睡眠至 HP\_SLEEP 所需要备份的寄存器显然是不同的，因此下文的寄存器名中的 \regindex{PMUSTATE\_TRANS} 存在如下可能：

\begin{itemize}
    \item HP\_SLEEP2ACTIVE
    \item HP\_SLEEP2MODEM
    \item HP\_MODEM2ACTIVE
    \item HP\_MODEM2SLEEP
    \item HP\_ACTIVE2SLEEP
\end{itemize}

下面将介绍 PMU 支持的 Retention DMA 控制：

\begin{itemize}
    \item 使能数据搬移：配置寄存器 \hyperref[fielddesc:PMUHPSLEEP2ACTIVEBACKUPEN]{PMU\_\regindex{PMUSTATE\_TRANS}\_BACKUP\_EN} 为 1，表示在进行对应的 PMU 状态切换时使能数据搬移。
    \item 打开对应时钟：在数据搬移开始前，配置 \hyperref[fielddesc:PMUHPMODEM2SLEEPBACKUPCLKSEL]{PMU\_\regindex{PMUSTATE\_TRANS}\_BACKUP\_CLK\_SEL} 选择 Retention DMA 工作时钟源，配置 \hyperref[fielddesc:PMUHPACTIVEBACKUPICGFUNCEN]{PMU\_\regindex{PMUSTATE}\_BACKUP\_ICG\_FUNC\_EN} 为 1 打开时钟。

    数据搬移结束后，门控时钟寄存器 \hyperref[fielddesc:PMUHPACTIVEBACKUPICGFUNCEN]{PMU\_\regindex{PMUSTATE}\_BACKUP\_ICG\_FUNC\_EN} 的值将由目标状态下的 \hyperref[fielddesc:PMUHPACTIVEDIGICGFUNCEN]{PMU\_\regindex{PMUSTATE}\_DIG\_ICG\_FUNC\_EN} 决定。

    \item 配置数据搬移方向：配置 \hyperref[fielddesc:PMUHPMODEM2SLEEPBACKUPMODE]{PMU\_\regindex{PMUSTATE\_TRANS}\_BACKUP\_MODE} 寄存器的最高位：
        \begin{itemize}
            \item 1：外设到存储器
            \item 0：存储器到外设
        \end{itemize}

    \item 选择链表指针：配置 \hyperref[fielddesc:PMUHPMODEM2SLEEPBACKUPMODE]{PMU\_\regindex{PMUSTATE\_TRANS}\_BACKUP\_MODE} 寄存器的低四位，选择链表指针，指针在链表中设置。
\end{itemize}

\subsubsection{系统控制器}

系统控制器用于在芯片切换 PMU 状态时控制部分功能模块，以实现芯片在各 PMU 状态下的稳定、节能工作，主要可控制的功能模块包括：

\begin{itemize}
    \item \textbf{暂停看门狗功能}：配置 \hyperref[fielddesc:PMUHPACTIVEDIGPAUSEWDT]{PMU\_\regindex{PMUSTATE}\_DIG\_PAUSE\_WDT} 寄存器为 1，可在 PMU 状态切换至对应 PMU 状态时禁用 RTC 看门狗定时器 (RWDT) 功能。注意，任一睡眠模式下若配置该寄存器为 0，表示不禁用该功能，则 RWDT 将因为没有 CPU 参与喂狗而产生复位。

    \item \textbf{切换 GPIO 至管脚睡眠模式}，该模式下将固定管脚的配置。例如将某个 GPIO 管脚固定为输入，低驱动模式以降低其功耗，并作为一个唤醒管脚使用，配置 \hyperref[fielddesc:PMUHPACTIVEHPPADHOLDALL]{PMU\_\regindex{PMUSTATE}\_HP\_PAD\_HOLD\_ALL} 寄存器为 1，可在 PMU 状态切换至对应 PMU 状态后锁存当前的配置作为 GPIO 的睡眠配置。GPIO 管脚保持功能的详细介绍请参考章节 \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}} > 小节 \ref{PadHold} \textit{\nameref{PadHold}}。

    \item \textbf{禁用 UART 唤醒功能}：配置 \hyperref[fielddesc:PMUHPACTIVEUARTWAKEUPEN]{PMU\_\regindex{PMUSTATE}\_UART\_WAKEUP\_EN} 寄存器为 0，禁用对应 PMU 状态下的 4 种 UART 唤醒模式。有关 UART 唤醒模式的具体介绍，请见章节 \ref{mod:uart} \textit{\nameref{mod:uart}}。

    \item \textbf{暂停 CPU 运行}：配置 \hyperref[fielddesc:PMUHPACTIVEDIGCPUSTALL]{PMU\_\regindex{PMUSTATE}\_DIG\_CPU\_STALL} 寄存器为 1，在对应 PMU 状态下将 CPU 挂起。
\end{itemize}

\section{RTC 启动}

在 Deep-sleep 下，芯片的 ROM 和 RAM 均断电，因此在唤醒时 SPI Boot（从 flash 复制数据）所需时间更长。因此，相较于 Light-sleep 和 Modem-sleep 模式，Deep-sleep 模式的唤醒时间要长的多。不过，值得注意的是，在 Deep-sleep 模式下， 16 KB SRAM 可以处于上电状态。因此，用户可以将一些代码规模不大（如 “deep sleep wake stub”）写入低功耗管理系统内的 16 KB SRAM，避免 SPI 启动带来的延迟，从而加速芯片唤醒过程。

使用过程如下所示：

\begin{enumerate}
    \item 配置 \hyperref[fielddesc:LPAONCORE0STATVECTORSEL]{LP\_AON\_CORE0\_STAT\_VECTOR\_SEL} 为 0，使 CPU 从 LP SRAM 启动。

    \item 计算  LP SRAM的 CRC 码，并将结果保存在寄存器 \hyperref[regdesc:LPAONSTORENREG]{LP\_AON\_STORE7\_REG} 中。

    \item 配置寄存器 \hyperref[regdesc:LPAONSTORENREG]{LP\_AON\_STORE6\_REG} 为 “deep sleep wake stub” 在 LP SRAM 的入口地址。
    \item 配置芯片进入睡眠模式。

    \item 当 CPU 开启时，开始进行 ROM 解包和部分初始化工作。此后，再次计算 LP SRAM 的 CRC 码。如果与寄存器 \hyperref[regdesc:LPAONSTORENREG]{LP\_AON\_STORE7\_REG} 中保存的结果一致，则 CPU 跳转至 “deep sleep wake stub”；如果 CRC 不一致，则直接进入 SPI boot 流程。
\end{enumerate}

\section{事件任务矩阵功能} \label{sec:lowpowm-etm-func}

在 \chipname{} 中，低功耗管理模块支持 ETM 功能，即可以通过任意外设的 ETM 事件触发 低功耗管理模块的 ETM 任务，或者通过 低功耗管理模块的 ETM 事件触发任意外设的 ETM 任务。关于 ETM 更多详细信息，请参考章节 \ref{mod:evntaskmatrix} \textit{\nameref{mod:evntaskmatrix}}。这里仅介绍与低功耗管理模块相关的 ETM 任务和 ETM 事件。

\end{itemize}

低功耗管理模块可产生的 ETM 事件有：

\begin{itemize}
\item PMU\_EVT\_SLEEP\_WAKEUP：该事件表示 PMU 被唤醒至 HP\_ACTIVE。
\end{itemize}

\section{中断}

\chipname{} 的低功耗管理模块可以生成以下中断信号：

\begin{itemize}
    \item PMU\_INTR
    \item PMU\_LP\_INT
\end{itemize}

PMU\_INTR 发送给 \hyperref[mod:intmtrx]{中断矩阵}，PMU\_LP\_INT 输出到 LP CPU。

这些中断信号由各自模块的内部中断源生成。具体为：

PMU\_INTR：

\begin{itemize}
\item \label{int:PMUSOCWAKEUPINT}PMU\_SOC\_WAKEUP\_INT：当芯片被唤醒至 HP\_ACTIVE 时触发。

\item \label{int:PMUSOCSLEEPREJECTINT}PMU\_SOC\_SLEEP\_REJECT\_INT：当拒绝睡眠源拒绝了一个睡眠请求时触发。

\item \label{int:PMUSWINT}PMU\_SW\_INT：当用 LP CPU 作为唤醒源将芯片唤醒至 HP\_ACTIVE 时触发。
\item \label{int:PMUSDIOIDLEINT}PMU\_SDIO\_IDLE\_INT：当 SDIO 处于 IDLE 状态时触发。
\item \label{int:PMULPCPUEXCINT}PMU\_LP\_CPU\_EXC\_INT：当 LP CPU 异常时触发。
\end{itemize}

PMU\_LP\_INT：

\begin{itemize}
\item \label{int:PMUHPSWTRIGGERINT}PMU\_HP\_SW\_TRIGGER\_INT：当用 HP CPU 作为唤醒源唤醒 LP CPU 时触发。

\item \label{int:PMUACTIVESWITCHSLEEPSTARTINT}PMU\_ACTIVE\_SWITCH\_SLEEP\_START\_INT：当 PMU 状态开始从 HP\_ACTIVE 切换至 HP\_SLEEP 时触发。
\item \label{int:PMUMODEMSWITCHSLEEPSTARTINT}PMU\_MODEM\_SWITCH\_SLEEP\_START\_INT：当 PMU 状态开始从 HP\_MODEM 切换至 HP\_SLEEP 时触发。
\item \label{int:PMUSLEEPSWITCHMODEMSTARTINT}PMU\_SLEEP\_SWITCH\_MODEM\_START\_INT：当 PMU 状态开始从 HP\_SLEEP 切换至 HP\_MODEM 时触发。
\item \label{int:PMUSLEEPSWITCHACTIVESTARTINT}PMU\_SLEEP\_SWITCH\_ACTIVE\_START\_INT：当 PMU 状态开始从 HP\_SLEEP 切换至 HP\_ACTIVE 时触发。
\item \label{int:PMUMODEMSWITCHACTIVESTARTINT}PMU\_MODEM\_SWITCH\_ACTIVE\_START\_INT：当 PMU 状态开始从 HP\_MODEM 切换至 HP\_ACTIVE 时触发。
\item \label{int:PMUACTIVESWITCHSLEEPENDINT}PMU\_ACTIVE\_SWITCH\_SLEEP\_END\_INT：当 PMU 状态完成从 HP\_ACTIVE 切换至 HP\_SLEEP 时触发。
\item \label{int:PMUMODEMSWITCHSLEEPENDINT}PMU\_MODEM\_SWITCH\_SLEEP\_END\_INT：当 PMU 状态完成从 HP\_MODEM 切换至 HP\_SLEEP 时触发。
\item \label{int:PMUSLEEPSWITCHMODEMENDINT}PMU\_SLEEP\_SWITCH\_MODEM\_END\_INT：当 PMU 状态完成从 HP\_SLEEP 切换至 HP\_MODEM 时触发。
\item \label{int:PMUSLEEPSWITCHACTIVEENDINT}PMU\_SLEEP\_SWITCH\_ACTIVE\_END\_INT：当 PMU 状态完成从 HP\_SLEEP 切换至 HP\_ACTIVE 时触发。
\item \label{int:PMUMODEMSWITCHACTIVEENDINT}PMU\_MODEM\_SWITCH\_ACTIVE\_END\_INT：当 PMU 状态完成从 HP\_MODEM 切换至 HP\_ACTIVE 时触发。
\item \label{int:PMULPCPUWAKEUPINT}PMU\_LP\_CPU\_WAKEUP\_INT：当 LP CPU 被唤醒时触发。
\end{itemize}

每个中断源都有一组通用的配置寄存器，详见章节 \hyperref[interrupt-config-registers]{\textit{中断配置寄存器}}。具体寄存器名称请查看 \ref{sec:lowpowm-reg-summ} \textit{\nameref{sec:lowpowm-reg-summ}}。
