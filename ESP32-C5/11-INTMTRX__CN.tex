\chapter{中断矩阵}
\label{mod:intmtrx}
\section{概述}

\chipname{} 中断矩阵将任一或多个外部中断源映射到 ESP-RISC-V CPU 的任一外部中断上。

\chipname{} 有 84 个外部中断源，但 CPU 只支持 32 个外部中断。因此，将这些外部中断源映射至 CPU 中断必须使用中断矩阵。

\begin{tiplisting}
本章节只涉及将外部中断源映射到 CPU 中断，关于中断配置、中断向量表、中断服务程序推荐处理机制请参考章节 \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}} >  \ref{sec:riscvcpu-int-ctrl}\textit{\nameref{sec:riscvcpu-int-ctrl}}。
\end{tiplisting}
\vspace{-1em}

\section{中断术语}\label{sec:intmtx-int-terminology}

下列有关中断的术语是在《\chipname{} 技术参考手册》的语境下定义的，旨在帮助读者更好地理解本文档。

\subsection{中断}

中断指的是在特定事件或条件下，CPU 中止当前执行去处理优先级更高的任务。中断机制使 CPU 能够迅速响应特定的事件。

《\chipname{} 技术参考手册》里涉及的“中断”是个广义的概念，可以泛指中断信号或中断源。

\subsection{中断信号/中断源}

中断信号和中断源只是定义的角度不同，实际意思相同。

从外设的角度来看，中断信号是由外设的内部中断源产生的，并发送到中断矩阵。

从中断矩阵的角度来看，它接收来自外设发送的中断信号，并将其视为中断源。然后，中断矩阵将 CPU 外部中断信号发送到 CPU。

从 CPU 的角度来看，来自中断矩阵的中断信号又变成中断源，与核心本地中断源一起发送给 CPU 中断控制器。

\section{特性}

\chipname{} 的中断矩阵模块具有如下特性：
\begin{itemize}
    \item 接收 84 个外部中断源作为输入
    \item 生成 32 个 HP CPU 的外部中断作为输出
    \item 支持查询外部中断源当前的中断状态
    \item 支持将多个中断源映射到单个 HP CPU 中断（即共享中断）
    \item 支持 CPU 用户模式中断委托到机器模式中断
\end{itemize}

    \end{itemize}

\subsection{HP CPU 中断}

\chipname{} HP CPU 支持 RISC-V Core-Local Interrupt Controller (CLIC) 标准 (v0.9) 中断。包含 2 个内部中断和 32 个外部中断 (中断号 16 \verb+~+ 47)，通过中断矩阵将外部中断源映射到 32 个 HP CPU 外部中断。每个中断：
\begin{itemize}
    \item 优先级可配置，具体编码方式见章节 \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}} > \ref{sec:riscvcpu-clic-int-level-prio} \textit{\nameref{sec:riscvcpu-clic-int-level-prio}}；
    \item 可配置为电平触发或者边沿触发；
    \item 可通过设置中断阈值屏蔽低优先级的中断。
\end{itemize}

\begin{tiplisting}
CPU 中断的具体功能及配置流程见章节 \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}} >  \ref{sec:riscvcpu-int-ctrl}\textit{\nameref{sec:riscvcpu-int-ctrl}}。CPU 中断的配置寄存器请见本章节的 \ref{sec:intpri-reg-summ} \textit{\nameref{sec:intpri-reg-summ}}。
\end{tiplisting}
\vspace{-1em}


\subsection{分配外部中断源至 HP CPU 外部中断}

在本小节中，我们将使用以下术语描述中断矩阵相关操作：

\begin{itemize}

\item \regindex{SOURCE}：代表某个外部中断中断源，详见表 \ref{tab:peripheral-interrupt-config-status-source}。

\item \hyperref[INTMTXCORE0SOURCEINTRMAPREG]{INTMTX\_CORE0\_\regindex{SOURCE}\_INTR\_MAP\_REG}：\regindex{SOURCE} 的中断源映射寄存器。
\item Num\_P：HP CPU 外部中断编号，范围：16 \verb+~+ 47。

\item Interrupt\_P：表示中断号为 Num\_P 的 HP CPU 中断。
\end{itemize}

\subsubsection{分配一个外部中断源 SOURCE 至 HP CPU 外部中断}

将 \regindex{SOURCE} 对应的寄存器 INTMTX\_CORE0\_\regindex{SOURCE}\_INTR\_MAP\_REG 配成 Num\_P，即可将该中断源分配至序号为 Num\_P 的 CPU 中断 (Interrupt\_P)。

\subsubsection{分配多个外部中断源 SOURCE 至 HP CPU 外部中断}

将各个中断源对应的寄存器 INTMTX\_CORE0\_\regindex{SOURCE}\_INTR\_MAP\_REG 均配置成相同的 Num\_P，即可将多个 \regindex{SOURCE} 分配至同一 CPU 外部中断 Interrupt\_P。上述任一外部中断均会触发 CPU 外部中断 Interrupt\_P。待中断触发后，CPU 需查询中断状态寄存器，判断产生中断的外设。 更多信息，见章节 \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}} >  \ref{sec:riscvcpu-int-ctrl}\textit{\nameref{sec:riscvcpu-int-ctrl}}。
\subsubsection{关闭 HP CPU 外部中断源 SOURCE}

将中断源对应的寄存器 INTMTX\_CORE0\_\regindex{SOURCE}\_INTR\_MAP\_REG 配置成 0，即可关闭外部中断源。



\subsection{委托中断}
\chipname{} RISC-V CPU 支持机器模式和用户模式。 32 个 HP CPU 外部中断可配制为机器模式中断或用户模式中断。 当 HP CPU 处于机器模式时，只会响应机器模式中断，但不会相应用户模式中断。当 HP CPU 处于用户模式时，可以响应所有中断。


委托中断功能支持 CPU 在机器模式下响应用户模式中断。实现方式为通过寄存器配置，将用户模式中断委托到机器模式中断。

\begin{itemize}

    \item 每个中断矩阵的中断源都可以独立开启中断委托功能。
    \item 可同时有多个中断矩阵的中断源开启中断委托功能。
    \item 只可以委托到一个中断矩阵的中断信号。

\end{itemize}

软件可以通过配置 \hyperref[regdesc:INTMTXCORE0INTSIGIDXASSERTINSECREG]{INTMTX\_CORE0\_INT\_SIG\_IDX\_ASSERT\_IN\_SEC\_REG} 为 n 选择将用户模式中断委托到中断号为 n 的机器模式中断。

软件可配置 \hyperref[INTMTXCORE0SOURCEINTRMAPREG]{INTMTX\_CORE0\_\regindex{SOURCE}\_INTR\_PASS\_IN\_SEC\_REG} 为 1 使能中断矩阵中断源 \regindex{SOURCE} 的委托中断功能。


\subsection{查询外部中断源 SOURCE 当前的中断状态}

配置开启 \regindex{SOURCE} 后，读取寄存器 \hyperref[regdesc:INTMTXCORE0INTSTATUS0REG] {INTMTX\_CORE0\_INT\_STATUS\_\regindex{n}\_REG}（只读）中特定位的值可以获取 \regindex{SOURCE} 当前的中断状态。寄存器 \hyperref[regdesc:INTMTXCORE0INTSTATUS0REG]{INTMTX\_CORE0\_INT\_STATUS\_\regindex{n}\_REG} 与 \regindex{SOURCE} 的对应关系如表 \ref{tab:peripheral-interrupt-config-status-source} 所示。
