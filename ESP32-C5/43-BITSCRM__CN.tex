\hypertarget{bitscrm}{}
\chapter{比特调节器}
\label{mod:bitscrm}

\chipname{} 中有大量支持 DMA（直接存储器访问）的外设（详情请参考章节 \ref{mod:dma} \textit{\nameref{mod:dma}}），它们可以在 CPU 不参与的情况下将数据从存储器传输至外设或从外设传输至存储器，但这需要外设传输的数据格式与软件支持的数据格式相同，如果格式不同，则需要 CPU 重写数据格式，如交换字节、反转字节和左右移位数据。

由于位操作通常相当耗费 CPU 资源，而设计 DMA 的初衷是在传输过程中避免使用 CPU，因此 \chipname{} 集成了一个比特调节器 (BitScrambler)，专门用于修改存储器和外设之间数据传输的格式，支持的传输方向包括存储器到外设、存储器到存储器、外设到存储器。除此之外，比特调节器还是一个灵活的可编程状态机，能够执行更高级的操作。

\section{概述}

在 \chipname{} 中，支持 DMA 的外设是指能够通过 DMA 通道直接从存储器读取数据和/或直接向存储器写入数据的外设。这些数据的读取/写入格式取决于外设，灵活性较低。对于 \chipname{} 中大多数支持 DMA 的外设来说，可以在这些路径中挂载比特调节器来提高灵活性。

当某个外设挂载比特调节器后，比特调节器就可以修改与该外设连接的 DMA 通道上的任何数据。写入 DMA 通道的数据会出现在比特调节器的输入端，而比特调节器输出端的数据将沿着 DMA 通道继续传输。比特调节器内置指令存储器，存储的指令用于控制输入和输出数据格式的映射关系。因此，通过编写合适的比特调节器程序，可以让比特调节器改写源数据。

如图 \ref{fig:bitscrm-blockdia-sys} 所示，比特调节器可以放置在 DMA 的 TX 路径中，修改从存储器流向外设的数据；也可以放置在 DMA 的 RX 路径中，修改从外设流向存储器的数据；还可以配置为环回模式，用于修改从存储器流向存储器的数据。

\section{特性}
\label{sec:bitscrm-features}

比特调节器有以下特性：

\begin{itemize}
    \item 一个比特调节器核心，可用于 RX 单元（外设到存储器）或 TX 单元（存储器到外设）
    \item 支持存储器到存储器的传输
    \item 每个 DMA 时钟周期最多可处理 32 位数据
    \item 数据路径由存储在指令存储器中的比特调节器程序控制
    \item 输入寄存器每个时钟周期可读取 0、8、16 或 32 位
    \item 输出寄存器：
    \begin{itemize}
        \item 每个时钟周期可写入 0、8、16 或 32 位
        \item 输出寄存器位的数据源：64 位输入数据、两个计数器、LUT RAM 数据、上个周期的数据输出、比较器
        \item 32 位输出寄存器位中的每一位可以来自数据源的任意位
    \end{itemize}
    \item 8 x 257 位指令存储器，用于存储八条指令，配置控制流和数据路径
    \item 2048 字节查找表（LUT）存储器，可配置为不同的字宽
\end{itemize}

\section{应用示例}
\label{sec:bitscrm-app-examples}

\begin{itemize}
    \item 数据中字节、半字节或位顺序的交换
    \item 将基于调色板的位图转换为全 RGB 位图
    \item 数据的游程编码 (run-length encoding) 或解码
    \item 生成复杂波形，例如用于 WS2811 芯片
\end{itemize}


\section{架构}
\label{sec:bitscrm-arch-overview}

比特调节器可以分为数据路径、控制路径以及寄存器，CPU 可通过这些寄存器配置数据路径和控制路径。数据路径在 DMA 输入流、内部寄存器和 DMA 输出流之间传输比特。控制路径解析指令存储器中的指令，以控制数据路径，并处理程序流程。

\subsection{数据路径}

\begin{figure}[ht]
\centering
\includegraphics[scale=1.0,width=\textwidth/2]{43-BITSCRM/figures/blockdia_data}
\caption{比特调节器数据路径图}\label{fig:bitscrm-blockdia-data}
\end{figure}

比特调节器的数据路径从 DMA 输入流 (DMA incoming stream) 中获取数据，根据存储在指令存储器中的程序代码处理数据，然后将其写入 DMA 输出流 (DMA outgoing stream) 中。

\begin{enumerate}
    \item 数据从 DMA 输入流保存到一个 64 位寄存器中。在每个指令周期结束时，根据指令的配置从 DMA 输入流中读取 N 位（N 可以是 0、8、16 或 32）。
    \item 寄存器中的数据向寄存器的最低有效位（LSB）方向移动，最低有效位的 N 位消失。
    \item 将最新读取的数据保存在最高的 N 位。
\end{enumerate}

或者在比特调节器启动时，可以选择预先自动读取前 64 位到寄存器中。

在比特调节器的另一端，数据存入一个 32 位输出寄存器。在每个指令周期结束时，根据指令的配置将最低的 0、8、16 或 32 位写入 DMA 输出流。

比特调节器的名称来源于其可以将输入比特放置在输出中的任意位置，这是通过 32 个 128 选 1 多路复用器 (128-to-1 multiplexer) 来实现的。对于输出寄存器中的 32 位的每一位，都有一个多路复用器根据比特调节器控制路径的输出，从相应的数据源中选择输入信号。这些数据源包括：

\begin{itemize}

\item 从输入 FIFO 移入的 64 位数据（即图 \ref{fig:bitscrm-blockdia-data} 中左侧的 DMA FIFO 模块）。如前所述，来自 DMA 输入流的数据被存储在此处。为了便于使用循环来遍历输入的 64 位数据，指令可以启用相对寻址。在这种寻址模式下，任何从输入 FIFO 寄存器获取比特的多路复用器实际上会获取偏移计数器 A 寄存器值的比特（即图 \ref{fig:bitscrm-blockdia-data} 中的 CTR A）。

\item 从上一个周期发送到输出的 32 位数据（即图 \ref{fig:bitscrm-blockdia-data} 中的 PREV OUTPUT 模块）。
发送到输出寄存器的数据将在一个时钟周期后出现在这个寄存器中。这使得比特调节器可以在多个时钟周期中“记住”数据：即使某个比特没有发送到输出 FIFO，只要是输出到了输出寄存器，下一个时钟周期仍然可以通过选择这个寄存器来访问该比特。通过再次从这个寄存器中选择比特进行输出，可以实现比特的长期记忆，使它们在随后的周期中出现在此寄存器中。

\item 由 LUT RAM 输出的 8、16 或 32 位数据。当 LUT 宽度为 32 位、16 位、8 位时，则数据地址分别是上一周期输出数据的第 16 到 24、25、26 位。LUT RAM 是比特调节器的一部分，它首先接收到某个地址，然后在 LUT RAM 中查找该地址对应的数据，最后输出该数据。比特调节器初始化时可填充 LUT RAM，但比特调节器不能修改 LUT RAM 。可根据需求将 LUT RAM 配置成以下模式进行初始化：

\begin{itemize}
    \item 512 x 32 位字
    \item 1024 x 16 位字
    \item 2048 x 8 位字

\end{itemize}

LUT RAM 可以用于数据的转换，例如用于某传感器的校准曲线，也可用于实现数学函数：将地址输入拆分成两个或两个以上的输入变量，数据输出作为函数的输出；存储器应加载每个输入值对应的函数输出。

\item 2 x 16 位计数器提供的 32 位数据（即图 \ref{fig:bitscrm-blockdia-data} 中的 CTR A/B 模块）。
该寄存器包含两个 16 位计数器，分别命名为计数器 A 和 B，可通过控制路径指令对它们进行加载、递增和递减等操作。

\item 计数器 B 和之前输出数据（对应图 \ref{fig:bitscrm-blockdia-data} 中的 PREV OUT）之间的比较器中包含的 30 位数据，属于 AUX 模块的一部分
比较结果可用于输出或程序流程，例如在解码游程编码流 (run-length encoded stream) 或生成 PWM 信号时非常有用。

\item 2 位数据（属于图 \ref{fig:bitscrm-blockdia-data} 中的 AUX 模块），一位固定为高电平，一位固定为低电平。使用场景：某协议需要输出一个始终为高电平或低电平的位，不受输入数据影响。

\end{itemize}

请注意，某些数据源无法同时访问。具体而言，指令无法同时从输入 FIFO 源寄存器的高 32 位以及计数器寄存器中获取数据。当从 LUT 读取数据时，对于宽度为 N 的 LUT，输入 FIFO 寄存器的最高 N 位和计数器寄存器的最高 N 位不可用。

\subsection{控制路径}

\begin{figure}[H]
\centering
\includegraphics[scale=1.0,width=\textwidth/2]{43-BITSCRM/figures/blockdia_control}
\caption{比特调节器控制路径图}
\label{fig:bitscrm-blockdia-control}
\end{figure}

比特调节器的行为由存储在指令 RAM 中的程序控制。指令 RAM 最多可以存储 8 条指令，每条指令为 257 位。这些指令可配置该指令运行时钟周期内的数据路径，以及控制程序流程的操作码。

比特调节器的程序和微处理器的程序类似，每个周期都会执行指令存储器中的下一条指令，除非操作码明确指示需要跳转到另一个位置。具体来说，这种行为是使用指令指针（IP，参见图 \ref{fig:bitscrm-blockdia-control}）来实现的，指向当前正在执行的指令。除此之外，操作码还可以影响计数器寄存器。

在配置数据路径的指令位中，一些位用于配置 32 个多路复用器，另一些位用于配置计数器位或 LUT 输出是否可用。指令还可以将多路复用器设置为相对模式，在这种模式下，从 DMA FIFO 获取的比特的位置将被偏移计数器 B 位。

下文介绍指令各域的含义。

\begin{itemize}
\item CTL\_MUX\_SEL\_n：包含 7 位，选择输出寄存器中第 n 位的来源。详情参见表 \ref{tab:bitscrm-ctrmuxsrc}。
\item OPCODE：操作码，该域的位格式请参见表 \ref{tab:bitscrm-instruction-opcode}。
\item CTL\_READ\_IN：在指令结束时选择从输入 FIFO 读取的位数。

\begin{itemize}
\item 0：不读取任何数据
\item 1：读取 8 位数据
\item 2：读取 16 位数据
\item 3：读取 32 位数据
\end{itemize}

\item CTL\_WR\_OUT：选择从输出寄存器写入到输出 FIFO 的位数：

\begin{itemize}
\item 0：不写入任何数据
\item 1：写入 8 位数据
\item 2：写入 16 位数据
\item 3：写入 32 位数据
\end{itemize}

\item CTL\_MUX\_REL、CTL\_CTR\_SRC\_SEL 和 CTR\_LUT\_SRC\_SEL：配置哪些位可以通过 CTL\_MUX\_SEL 域来选择。

\begin{itemize}
    \item CTRL\_CTR\_SRC\_SEL（控制计数器源选择）配置计数器 A 和 B 的位是否可用。
    \item CTL\_LUT\_SRC\_SEL（控制 LUT 源选择）配置 LUT RAM 的输出值是否可用。详见表 \ref{tab:bitscrm-ctrmuxsrc}。
    \item 当 CTL\_MUX\_REL（控制多路复用器相对寻址）为 1 时，且 CTL\_MUX\_SEL\_n 小于等于 63 时， 比特调节器将使用计数器 A 重新计算有效的 CTL\_MUX\_SEL\_n。具体行为取决于 CTR\_SRC\_SEL，详见表 \ref{tab:bitscrm-ctrmuxrel}。
\end{itemize}

\end{itemize}

\subsection{配置寄存器}

比特调节器的配置是通过一组寄存器实现的。这些寄存器可向比特调节器写入程序、写入 LUT RAM、配置控制器行为、以及启动和停止控制器的操作，还可将控制器挂载到外设的 DMA 路径。

\chipname{} 只有一个比特调节器核心，可放置在 TX 路径（数据从存储器发送到外设）或 RX 路径（数据从外设发送到存储器）。为了兼容其他 ESP 系列芯片，当比特调节器核心处于 TX 路径时，则由 BITSCRAMBLER\_TX\_ 为前缀的寄存器控制；处于 RX 路径时，则由 BITSCRAMBLER\_RX\_ 为前缀的寄存器控制。哪个路径处于激活状态由 \hyperref[fielddesc:BITSCRAMBLERRXENA]{BITSCRAMBLER\_RX\_ENA} 和 \hyperref[fielddesc:BITSCRAMBLERTXENA]{BITSCRAMBLER\_TX\_ENA} 决定，且在任意时刻只允许其中一个处于激活状态。


\begin{tiplisting}
为方便阐述，本节仅使用 TX 路径寄存器进行描述，若需 RX 路径寄存器的说明，只需替换描述中的前缀即可。
\end{tiplisting}

当比特调节器修改某外设的输入或输出时，需要挂载到该外设的 DMA 路径。为此，需将该外设的对应值写入 HP\_SYSTEM\_BITSCRAMBLER\_PERI\_SEL\_REG 寄存器的HP\_SYSTEM\_BITSCRAMBLER\_PERI\_TX\_SEL 域或 HP\_SYSTEM\_BITSCRAMBLER\_PERI\_RX\_SEL 域。

比特调节器也可以在存储器到存储器模式下运行，但需要将 \hyperref[regdesc:BITSCRAMBLERSYSREG]{BITSCRAMBLER\_SYS\_REG} 寄存器中的 \hyperref[fielddesc:BITSCRAMBLERLOOPMODE]{BITSCRAMBLER\_LOOP\_MODE} 域置为 1。该模式下，TX 比特调节器进入环回模式，而 RX 比特调节器 则未使用且无法使用。并且比特调节器仍然需要连接到外设，但该外设无需配置，而且对应的外设 DMA 路径将无法使用。

主处理器无法直接访问比特调节器的程序存储器和 LUT RAM，但可以间接访问，即在一个寄存器中设置地址，然后在另一个寄存器中写入数据。

具体而言，对于 LUT RAM，地址写入 \hyperref[regdesc:BITSCRAMBLERTXLUTCFG0REG]{BITSCRAMBLER\_TX\_LUT\_CFG0\_REG} 的 \hyperref[fielddesc:BITSCRAMBLERTXLUTIDX]{BITSCRAMBLER\_TX\_LUT\_IDX} 域，数据写入 \hyperref[regdesc:BITSCRAMBLERTXLUTCFG1REG]{BITSCRAMBLER\_TX\_LUT\_CFG1\_REG} 寄存器，或从该寄存器读取数据。LUT 的位宽对比特调节器可见的，可以使用 \hyperref[fielddesc:BITSCRAMBLERTXLUTMODE]{BITSCRAMBLER\_TX\_LUT\_MODE} 域进行配置。需要注意的是，当主处理器以上述方式写入 LUT 时，地址和数据大小需要符合配置的 LUT 位宽。


同样，对于指令存储器，地址写入 \hyperref[regdesc:BITSCRAMBLERTXINSTCFG0REG]{BITSCRAMBLER\_TX\_INST\_CFG0\_REG}，数据写入或从 \hyperref[regdesc:BITSCRAMBLERTXINSTCFG1REG]{BITSCRAMBLER\_TX\_INST\_CFG1\_REG} 中读取。由于比特调节器指令为 257 位，因此一个指令需要写入 9 个 32 位字，第 257 位位于第 9 个字的最低有效位。为此，\hyperref[regdesc:BITSCRAMBLERTXINSTCFG0REG]{BITSCRAMBLER\_TX\_INST\_CFG0\_REG} 寄存器被分为两个域，指令的位置写入 \hyperref[fielddesc:BITSCRAMBLERTXINSTIDX]{BITSCRAMBLER\_TX\_INST\_IDX} 域，而指令内的字偏移写入 \hyperref[fielddesc:BITSCRAMBLERTXINSTPOS]{BITSCRAMBLER\_TX\_INST\_POS} 域。


由于比特调节器生成的数据可以比输入多也可以比输入少，所以它控制的 DMA 流可以以一种或两种方式结束。第一种情况是接收器（在 RX 比特调节器的情况下是存储器，而在 TX 比特调节器的情况下是外设）停止接收数据，因为它的配置是只接收一定数量的字节。这种情况比较简单，比特调节器因为无法再写入数据而停止。另一种方式是发射器（在 RX 比特调节器的情况下是外设，在 TX 比特调节器的情况下是存储器）停止发送数据。这种情况更加复杂，因为比特调节器可能正在处理需要写入输出的一些数据，也可能不在处理。

\chipname{} 比特调节器设计了两种方法来处理输入数据流结束后的一定数量的数据，目的是为了完整地处理这些数据。这两种方法依赖于 \hyperref[regdesc:BITSCRAMBLERRXTAILINGBITSREG]{BITSCRAMBLER\_RX\_TAILING\_BITS\_REG} 寄存器设置的具体值 N：


\begin{itemize}
\item EOF-on-N-reads（读取 N 字节后结束）：在输入数据流结束后，比特调节器从输入读取 N 个虚拟字节。这些字节的值为零。在读取完第 N 个字节后，比特调节器停止，DMA 流结束。

\item EOF-on-N-writes（写入 N 字节后结束）：在输入数据流结束后，比特调节器允许向输出写入 N 个字节。在此期间，任何输入流上的读取都会返回值为零的虚拟字节。在写入完第 N 个字节后，比特调节器停止，DMA 流结束。
\end{itemize}

\section{配置流程}
\label{sec:bitscrm-prog-procedures}

请注意，下文描述的是TX 路径使能时比特调节器的配置流程，路径方向为存储器到外设方向或者处于环回模式下存储器到存储器方向。对于外设到存储器方向的路径，则使用比特调节器的 RX 路径。除非另作说明，否则将下文描述中的 TX 替换为 RX 即为比特调节器 RX 路径的配置流程。

\begin{enumerate}
    \item 配置 HP\_SYSTEM\_BITSCRAMBLER\_PERI\_SEL\_REG 寄存器中的 HP\_SYSTEM\_BITSCRAMBLER\_PERI\_TX\_SEL 域将比特调节器挂载到外设。请注意，即使在环回模式下，此步骤也是必须的。
    \item 配置 \hyperref[regdesc:BITSCRAMBLERTXCTRLREG]{BITSCRAMBLER\_TX\_CTRL\_REG} 寄存器中的 \hyperref[fielddesc:BITSCRAMBLERTXENA]{BITSCRAMBLER\_TX\_ENA} 域使能比特调节器。请确保任意时刻 \hyperref[fielddesc:BITSCRAMBLERTXENA]{BITSCRAMBLER\_TX\_ENA} 和 \hyperref[fielddesc:BITSCRAMBLERRXENA]{BITSCRAMBLER\_RX\_ENA} 中只能有一个寄存器的值为 1。
    \item 根据需要决定是否配置环回模式。如果需要，将 \hyperref[regdesc:BITSCRAMBLERSYSREG]{BITSCRAMBLER\_SYS\_REG} 寄存器的 \hyperref[fielddesc:BITSCRAMBLERLOOPMODE] {BITSCRAMBLER\_LOOP\_MODE} 域置 1，否则清零。
    \item 将程序加载到比特调节器中。对于每条指令和指令内的每个 32 位字，设置 \hyperref[regdesc:BITSCRAMBLERTXINSTCFG0REG]{BITSCRAMBLER\_TX\_INST\_CFG0\_REG} 寄存器中的 \hyperref[fielddesc:BITSCRAMBLERTXINSTIDX]{BITSCRAMBLER\_TX\_INST\_IDX} 和 \hyperref[fielddesc:BITSCRAMBLERTXINSTPOS]{BITSCRAMBLER\_TX\_INST\_POS} 域，然后将字写入 \hyperref[regdesc:BITSCRAMBLERTXINSTCFG1REG]{BITSCRAMBLER\_TX\_INST\_CFG1\_REG} 寄存器。注意，第 257 位写入到 \hyperref[regdesc:BITSCRAMBLERTXINSTCFG1REG]{BITSCRAMBLER\_TX\_INST\_CFG1\_REG} 的第 0 位。
    \item 如果需要，将 LUT 挂载到比特调节器中：
    \begin{enumerate}
        \item 配置 \hyperref[regdesc:BITSCRAMBLERTXLUTCFG0REG]{BITSCRAMBLER\_TX\_LUT\_CFG0\_REG} 寄存器中 \hyperref[fielddesc:BITSCRAMBLERTXLUTMODE] {BITSCRAMBLER\_TX\_LUT\_MODE} 域来设置 LUT 宽度
        \item 将地址写入 \hyperref[regdesc:BITSCRAMBLERTXLUTCFG0REG]{BITSCRAMBLER\_TX\_LUT\_CFG0\_REG}
        \item 将数据字或字节写入 \hyperref[regdesc:BITSCRAMBLERTXLUTCFG1REG]{BITSCRAMBLER\_TX\_LUT\_CFG1\_REG}
    \end{enumerate}
        请注意，在写入数据之后和启动比特调节器之前更改 LUT 宽度都是可以的。
    \item 配置比特调节器。根据比特调节器程序的需求，设置或清除 \hyperref[regdesc:BITSCRAMBLERTXCTRLREG]{BITSCRAMBLER\_TX\_CTRL\_REG} 寄存器中的相应位，也可将 \hyperref[regdesc:BITSCRAMBLERTXTAILINGBITSREG]{BITSCRAMBLER\_TX\_TAILING\_BITS\_REG} 设置为适当的值。
    \item 清除 \hyperref[regdesc:BITSCRAMBLERTXCTRLREG]{BITSCRAMBLER\_TX\_CTRL\_REG} 寄存器的\hyperref[fielddesc:BITSCRAMBLERTXHALT]{BITSCRAMBLER\_TX\_HALT} 域，以启动比特调节器。
    \item 启动 DMA 传输。有关 DMA 子系统的更多信息，请参阅章节 \ref{mod:dma} \textit{\nameref{mod:dma}}，如果未使用环回模式，请参阅各外设章节。
    \item 等待 DMA 传输完成。
    \item 设置 \hyperref[regdesc:BITSCRAMBLERTXCTRLREG]{BITSCRAMBLER\_TX\_CTRL\_REG} 寄存器中的 \hyperref[fielddesc:BITSCRAMBLERTXHALT]{BITSCRAMBLER\_TX\_HALT} 域停止比特调节器。
    \item 等待比特调节器处于停止状态。当 \hyperref[regdesc:BITSCRAMBLERTXSTATEREG]{BITSCRAMBLER\_TX\_STATE\_REG} 寄存器中的 \hyperref[fielddesc:BITSCRAMBLERTXINIDLE]{BITSCRAMBLER\_TX\_IN\_IDLE} 位被置位时，比特调节器停止。
    \item 在 \hyperref[regdesc:BITSCRAMBLERTXCTRLREG]{BITSCRAMBLER\_TX\_CTRL\_REG} 寄存器中的 \hyperref[fielddesc:BITSCRAMBLERTXFIFORST]{BITSCRAMBLER\_TX\_FIFO\_RST} 位先写入 1 然后再写入 0 来复位 FIFO。
    \item 比特调节器可以使用当前的程序和 LUT 配置开始进行新的传输，从步骤 7 开始即可。
\end{enumerate}
