\chapter{System Registers}
\label{mod:sysreg}
\hypertarget{sysreg}{}


\section{Overview}

\chipname{} supports a set of auxiliary chip features listed in subsection \ref{sec:sysreg-features} \textit{\nameref{sec:sysreg-features}} below. This chapter describes the registers used to configure these features.

\section{Features}
\label{sec:sysreg-features}

\chipname{} system registers control the following peripheral blocks and core modules:

\begin{itemize}
    \item External Memory Encryption/Decryption
    \item Anti-DPA attack security
    \item HP CPU/LP CPU debug
    \item Bus timeout protection
\end{itemize}


\section{Function Description}

\subsection{External Memory Encryption/Decryption Configuration}

Register \hyperref[regdesc:HPSYSTEMEXTERNALDEVICEENCRYPTDECRYPTCONTROLREG]{HP\_SYSTEM\_EXTERNAL\_DEVICE\_ENCRYPT\_DECRYPT\_CONTROL\_REG} configures encryption and decryption options of the external memory. For details, please refer to Chapter \ref{mod:extmemencr}
\textit{\nameref{mod:extmemencr}}.


\subsection{Anti-DPA Attack Security Control}
\label{sec:sysreg-anti-dpa-attack-security-control}

\chipname{} has a dual protection mechanism against Differential Power Analysis (DPA) attacks at the hardware level.

\begin{itemize}
    \item First, a mask mechanism is introduced in the symmetric encryption operation, which interferes with the power consumption trajectory by masking the real data in the operation process. This security mechanism cannot be turned off.
    \item Second, the clock selected for the operation will change dynamically in real time, blurring the power consumption trajectory during the operation. For this security mechanism, \chipname{} provides 4 security levels for users to choose to adapt to different applications.
\end{itemize}

\begin{table}[H]
    \centering
    \caption{Security Level}
    \label{tab:sysreg-security-level}
    \begin{threeparttable}
    \begin{tabular}[c]{| l | c | c | c |}
    \hline
    \rowcolor{lightgray}
        \textbf{Security-Level Name} & \textbf{Security-Level Value} & \textbf{PLL\_CLK (MHz)} & \textbf{XTAL\_CLK (MHz)} \\ \hline
        SEC\_DPA\_OFF       & 0    & 160                   & 48          \\\hline
        SEC\_DPA\_LOW       & 1    & (120 ,160]\tnote{A}   & (24,48]\tnote{A}   \\\hline
        SEC\_DPA\_MIDDLE    & 2    & (96,160]\tnote{A}     & (16,48]\tnote{A} \\\hline
        SEC\_DPA\_HIGH      & 3    & (80,160]\tnote{A}     & (9.6,48]\tnote{A}   \\\hline
    \end{tabular}
    \begin{tablenotes}
        \item[A] (x,y] means the operating frequency is greater than x Hz, and equal to or less than y Hz.
    \end{tablenotes}
    \end{threeparttable}
\end{table}

By default, the field \hyperref[fielddesc:HPSYSTEMSECDPALEVEL]{HP\_SYSTEM\_SEC\_DPA\_CFG\_SEL} in register \hyperref[regdesc:HPSYSTEMSECDPACONFREG]{HP\_SYSTEM\_SEC\_DPA\_CONF\_REG} is 0. In this case, the security-level is decided by the eFuse field \hyperref[fielddesc:EFUSESECDPALEVEL]{EFUSE\_SEC\_DPA\_LEVEL}. If the field \hyperref[fielddesc:HPSYSTEMSECDPALEVEL]{HP\_SYSTEM\_SEC\_DPA\_CFG\_SEL} is set to 1, the security-level is decided by \hyperref[fielddesc:HPSYSTEMSECDPALEVEL]{HP\_SYSTEM\_SEC\_DPA\_CFG\_LEVEL} in register \hyperref[regdesc:HPSYSTEMSECDPACONFREG]{HP\_SYSTEM\_SEC\_DPA\_CONF\_REG}.


\subsection{HP CPU/LP CPU Debug Control}

Register \hyperref[fielddesc:HPSYSTEMCOREDEBUGRUNSTALLENABLE]{HP\_SYSTEM\_CORE\_DEBUG\_RUNSTALL\_ENABLE} controls the RunStall feature, which facilitates the debugging of HP CPU and LP CPU. If enabled, when any of the HP CPUs is in debug mode, the other one is stalled automatically.

For details, please refer to the Subsection \ref{sec:riscvcpu-dbg} \textit{\nameref{sec:riscvcpu-dbg}} in Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}.

\subsection{Bus Timeout Protection}

\chipname{} supports bus timeout protection and allows configurable timeout threshold. When a transfer is initiated, the internal counter of the timeout protection module increments by 1 on each clock cycle.
\begin{itemize}
    \item If the accumulated value remains below the timeout threshold and a response is received from the slave device, the counter is cleared.
    \item If the accumulated value exceeds the threshold and no response has been received, the module forces the bus return signal high to terminate the transfer. At the same time, an interrupt is triggered, and the module logs the exception address and the master ID associated with the access.
\end{itemize}


\subsubsection{CPU Peripheral Timeout Protection Register}

Register \hyperref[regdesc:HPSYSTEMCPUPERITIMEOUTCONFREG]{HP\_SYSTEM\_CPU\_PERI\_TIMEOUT\_CONF\_REG} configures the timeout protection for accessing CPU peripherals, which refer to the peripherals or modules whose addresses are in the range of 0x600C\_0000–0x600C\_FFFF. For details, please refer to Subsection \ref{para:sysmem-mod-peri-add-mapping} \textit{\nameref{para:sysmem-mod-peri-add-mapping}} in Chapter \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}.


When a timeout occurs, the \hyperref[tab:peripheral-interrupt-config-status-source]{CPU\_PERI\_TIMEOUT\_INTR} interrupt will be triggered.

\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEMCPUPERITIMEOUTCONFREG]{HP\_SYSTEM\_CPU\_PERI\_TIMEOUT\_CONF\_REG}: Enables timeout protection and configures the timeout threshold.
    \item \hyperref[regdesc:HPSYSTEMCPUPERITIMEOUTADDRREG]{HP\_SYSTEM\_CPU\_PERI\_TIMEOUT\_ADDR\_REG}: When a timeout occurs, this register will record the address of the timeout.
    \item \hyperref[regdesc:HPSYSTEMCPUPERITIMEOUTUIDREG]{HP\_SYSTEM\_CPU\_PERI\_TIMEOUT\_UID\_REG}: When a timeout occurs, this register will record the master ID of the timeout.
\end{itemize}


\subsubsection{HP Peripheral Timeout Protection Register}

\hyperref[regdesc:HPSYSTEMHPPERITIMEOUTCONFREG]{HP\_SYSTEM\_HP\_PERI\_TIMEOUT\_CONF\_REG} configures the timeout protection for accessing HP peripherals, which refer to the peripherals or modules whose addresses are in the range of 0x6000\_0000–0x6009\_FFFF. For details, please refer to Subsection \ref{para:sysmem-mod-peri-add-mapping} \textit{\nameref{para:sysmem-mod-peri-add-mapping}} in Chapter \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}.

When a timeout occurs, the \hyperref[tab:peripheral-interrupt-config-status-source]{HP\_PERI\_TIMEOUT\_INTR} interrupt will be triggered.

\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEMHPPERITIMEOUTCONFREG]{HP\_SYSTEM\_HP\_PERI\_TIMEOUT\_CONF\_REG}: Enables timeout protection and configures the timeout threshold.
    \item \hyperref[regdesc:HPSYSTEMHPPERITIMEOUTADDRREG]{HP\_SYSTEM\_HP\_PERI\_TIMEOUT\_ADDR\_REG}: When a timeout occurs, this register will record the address of the timeout.
    \item \hyperref[regdesc:HPSYSTEMHPPERITIMEOUTUIDREG]{HP\_SYSTEM\_HP\_PERI\_TIMEOUT\_UID\_REG}: When a timeout occurs, this register will record the master ID of the timeout.
\end{itemize}

\subsubsection{LP Peripheral Timeout Protection Register}

Register \hyperref[regdesc:LPPERIBUSTIMEOUTCONFREG]{LP\_PERI\_BUS\_TIMEOUT\_CONF\_REG} configures the timeout protection for accessing LP peripherals, which refer to the peripherals or modules whose addresses are in the range of 0x600B\_0000–0x600B\_FFFF. For details, please refer to Subsection \ref{para:sysmem-mod-peri-add-mapping} \textit{\nameref{para:sysmem-mod-peri-add-mapping}} in Chapter \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}.


When a timeout occurs, the \hyperref[tab:peripheral-interrupt-config-status-source]{LP\_PERI\_TIMEOUT\_INTR} interrupt will be triggered.

\begin{itemize}
    \item \hyperref[regdesc:LPPERIBUSTIMEOUTCONFREG]{LP\_PERI\_BUS\_TIMEOUT\_CONF\_REG}: Enables timeout protection and configures the timeout threshold.
    \item \hyperref[regdesc:LPPERIBUSTIMEOUTADDRREG]{LP\_PERI\_BUS\_TIMEOUT\_ADDR\_REG}: When a timeout occurs, this register will record the address of the timeout.
    \item \hyperref[regdesc:LPPERIBUSTIMEOUTUIDREG]{LP\_PERI\_BUS\_TIMEOUT\_UID\_REG}: When a timeout occurs, this register will record the master ID of the timeout.
\end{itemize}
