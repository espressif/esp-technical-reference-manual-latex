\chapter{Permission Control (PMS)}
\label{mod:permctrl}

\section{Overview}

The permission control of \chipname{} can be divided into two parts: PMP (Physical Memory Protection) and APM (Access Permission Management).

PMP is located inside the HP CPU and can control the HP CPU's access to all address spaces. APM is an access permission management module located at the bus port. APM, together with PMP, manages permissions for the HP CPU to access parts of the memory and the peripheral registers. APM can manage the LP CPU independently to access parts of the internal and external memory. Additionally, APM manages DMA masters such as GDMA and TCM\_MEM\_MONITOR to access parts of internal and external memory.

The distribution of management areas by PMP and APM is shown in Table \ref{tab:pmp-apm}. For the HP CPU, the control relationship between PMP and APM is shown in Figure \ref{fig:pmp_apm}.

The diagram illustrates that when the HP CPU accesses ROM and EXT\_MEM, the access paths are solely managed by PMP. On the other hand, when the HP CPU accesses the other address spaces, the access paths are controlled by both PMP and APM. If the PMP check fails, the APM permission control will not be triggered. For peripheral registers, there are two levels of APM permission control: the first level manages access permissions of address regions, and the second level manages access permissions of peripherals.

PMP-related registers are located inside the HP CPU and can be read or configured with special instructions. For how to configure PMP, please refer to Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}} > \textit{\nameref{sec:riscvcpu-pmp}}.

The following sections will provide a detailed introduction to the APM module including its features, functions, and configurations.

\section{Introduction to APM}

The APM module contains two parts: the TEE (Trusted Execution Environment) controller and the APM controller. Each module contains its own registers: TEE registers and APM registers.

\begin{itemize}
\item The TEE controller configures the security mode of a particular master in \chipname{} (such as GDMA) to access memory or peripheral registers. It supports four security modes: TEE, REE0 (Rich Execution Environment), REE1, and REE2.

\item The APM controller has two types: SYS\_APM controller and PERI\_APM controller. The registers of SYS\_APM controller are in the APM registers, and the registers of PERI\_APM controller are in the TEE registers.

    \begin{itemize}
    \item The SYS\_APM controller manages a master's access permissions when accessing memory and peripheral registers. By comparing the pre-configured address ranges and corresponding access permissions with the information carried on the bus, such as ID number (please refer to Table \ref{tab:master_source}), security mode, access address, access permissions, etc, the SYS\_APM controller determines whether the access is allowed.

    \item The PERI\_APM controller manages the master's access permissions to peripheral registers. It compares the pre-configured read and write permission registers for each peripheral with the information carried on the bus, such as secure mode, the peripheral to which the access address belongs, and access permissions, to determine whether access is allowed.
\end{itemize}
\end{itemize}

The TEE registers configure the security mode of each master. The APM registers specify the access permission and access address range of each master in the configured security mode. With the TEE controller and APM controller, \chipname{} can precisely control the access permission of all masters to the memory and peripheral registers.

\section{Features}\label{sec:permctrl-features}

\chipname{} has two TEE controllers: HP\_TEE and LP\_TEE.

\begin{itemize}
    \item HP\_TEE has the following features:
        \begin{itemize}
        \item Supports four security modes for the masters
        \item Supports configurable security mode for 32 masters
        \end{itemize}
    \item LP\_TEE has the following features:
        \begin{itemize}
        \item Supports four security modes for the masters
        \item Supports configurable security mode for one master (i.e., the LP CPU)
        \end{itemize}
\end{itemize}

\chipname{} has four SYS\_APM controllers: HP\_APM\_CTRL, LP\_APM\_CTRL, LP\_APM0\_CTRL, and CPU\_APM\_CTRL.

\begin{itemize}
    \item HP\_APM\_CTRL has the following features:
        \begin{itemize}
        \item Supports access permission configuration for up to 16 address ranges
        \item Supports access management to HP SRAM, HP CPU peripheral registers, HP system peripheral registers, and external memory
        \item Interrupt function on illegal access
        \item Exception information record
        \end{itemize}
    \item LP\_APM\_CTRL has the following features:
        \begin{itemize}
        \item Supports access permission configuration for up to eight address ranges
        \item Supports access management to LP SRAM (in low-speed mode) and LP peripheral registers
        \item Interrupt function on illegal access
        \item Exception information record
        \end{itemize}
    \item LP\_APM0\_CTRL has the following features:
        \begin{itemize}
        \item Supports access permission configuration for up to eight address ranges
        \item Supports access management to LP SRAM (in high-speed mode)
        \item Interrupt function on illegal access
        \item Exception information record
        \end{itemize}
    \item CPU\_APM\_CTRL has the following features:
        \begin{itemize}
        \item Supports access permission configuration for up to eight address ranges
        \item Supports HP CPU's access management to HP SRAM
        \item Interrupt function on illegal access
        \item Exception information record
        \end{itemize}
\end{itemize}

\chipname{} has two PERI\_APM controllers: HP\_PERI\_APM\_CTRL and LP\_PERI\_APM\_CTRL.
\begin{itemize}
    \item HP\_PERI\_APM\_CTRL and LP\_PERI\_APM\_CTRL share the following features:
        \begin{itemize}
        \item Support separate permission configuration for each peripheral register
        \item Support returning error status to the master via the system bus
        \item Exception information record
        \end{itemize}
\end{itemize}

\section{Functional Description}

\subsection{TEE Controller Functional Description}

\chipname{} supports four security modes: TEE, REE0, REE1, and REE2.

For the HP CPU to access memory or peripheral registers, first select the machine mode or user mode for it, then configure its security mode with HP\_TEE registers. For the configuration of machine mode and user mode, please refer to RISC\-/V Instruction Set Manual, Volume II: Privileged Architecture, Version 1.10.

\begin{itemize}
    \item When the HP CPU is in machine mode, its security mode is TEE mode.
    \item When the HP CPU is in user mode, its security mode is REE mode, defined by \hyperref[fielddesc:TEEMNMODE]{TEE\_M0\_MODE} as follows:
    \begin{itemize}
        \item If \hyperref[fielddesc:TEEMNMODE]{TEE\_M0\_MODE} is 0, which is TEE mode, the valid mode that actually takes effect in the HP CPU user mode is REE0.
        \item if \hyperref[fielddesc:TEEMNMODE]{TEE\_M0\_MODE} is 1, 2, or 3, the security mode is REE0, REE1, and REE2 respectively.
    \end{itemize}
\end{itemize}

For the LP CPU to access memory or peripheral registers, set its secure mode with \hyperref[fielddesc:LPTEEM0MODE]{LP\_TEE\_M0\_MODE}.

As for other masters, configure their security mode with \hyperref[fielddesc:TEEMNMODE]{TEE\_M\regindex{n}\_MODE}. \regindex{n} represents the ID number of master as listed in Table \ref{tab:master_source}.

\subsubsection{SYS\_APM Controller}

\paragraph{Address Ranges}

\begin{itemize}
    \item \hyperref[sec:permctrl-reg-hp]{HP\_APM\_REG} register module can configure up to 16 address ranges (i.e., region \regindex{n}, \regindex{n}=0-15) for HP\_APM\_CTRL. The start and end addresses of each region (address range) are configured by  \hyperref[regdesc:HPAPMREGIONNADDRSTARTREG]{HP\_APM\_REGION\regindex{n}\_ADDR\_START} and \hyperref[regdesc:HPAPMREGIONNADDRENDREG]{HP\_APM\_REGION\regindex{n}\_ADDR\_END}, respectively. Configure the bit \regindex{n} of \hyperref[fielddesc:HPAPMREGIONFILTEREN]{HP\_APM\_REGION\_FILTER\_EN\_REG} to enable region \regindex{n}. Region 0 (i.e., the first address range) is enabled by default.

    \item \hyperref[sec:permctrl-reg-lp0]{LP\_APM0\_REG} register module can configure up to eight address ranges (i.e., region \regindex{n}, \regindex{n}=0-7) for LP\_APM0\_CTRL. The start and end addresses of each region are configured by \hyperref[fielddesc:LPAPM0REGIONNADDRSTART]{LP\_APM0\_REGION\regindex{n}\_ADDR\_START} and \hyperref[fielddesc:LPAPM0REGIONNADDREND]{LP\_APM0\_REGION\regindex{n}\_ADDR\_END}. Configure the bit \regindex{n} of \hyperref[fielddesc:LPAPM0REGIONFILTEREN]{LP\_APM0\_REGION\_FILTER\_EN\_REG} to enable region \regindex{n}. Region 0 (i.e., the first address range) is enabled by default.

    \item \hyperref[sec:permctrl-reg-lp]{LP\_APM\_REG} register module can configure up to eight address ranges (i.e., region \regindex{n}, \regindex{n}=0-7) for LP\_APM\_CTRL. The start and end addresses of each region are configured by \hyperref[fielddesc:LPAPMREGIONNADDRSTART]{LP\_APM\_REGION\regindex{n}\_ADDR\_START} and \hyperref[fielddesc:LPAPMREGIONNADDREND]{LP\_APM\_REGION\regindex{n}\_ADDR\_END}. Configure the bit \regindex{n} of \hyperref[fielddesc:LPAPMREGIONFILTEREN]{LP\_APM\_REGION\_FILTER\_EN\_REG} to enable region \regindex{n}. Region 0 (i.e., the first address range) is enabled by default.

    \item \hyperref[sec:permctrl-reg-cpu]{CPU\_APM\_REG} register module can configure up to eight address ranges (i.e., region \regindex{n}, \regindex{n}=0-7) for CPU\_APM\_CTRL. The start and end addresses of each region are configured by \hyperref[fielddesc:CPUAPMREGIONNADDRSTART]{CPU\_APM\_REGION\regindex{n}\_ADDR\_START} and \hyperref[fielddesc:CPUAPMREGIONNADDREND]{CPU\_APM\_REGION\regindex{n}\_ADDR\_END}. Configure the bit \regindex{n} of \hyperref[fielddesc:CPUAPMREGIONFILTEREN]{CPU\_APM\_REGION\_FILTER\_EN\_REG} to enable region \regindex{n}. Region 0 (i.e., the first address range) is enabled by default.
\end{itemize}

When configuring the address ranges, the address requires a 4-byte alignment (the lower two bits of the address are 0). For example, the address range could be set to 0x4080000C \verb+~+ 0x40808774 or 0x600C0008 \verb+~+ 0X600CFF70.

\paragraph{Access Permissions of Address Ranges}

For each address range, the access permissions (read/write/execute) are configurable in different security modes:

\begin{itemize}

    \item Masters in TEE mode always have read, write, and execute permissions in the address range.

    \item For masters in REE0, REE1 or REE2 mode, access permissions can be configured with \hyperref[regdesc:HPAPMREGIONNATTRREG]{HP\_APM\_REGION\regindex{n}\_ATTR\_REG}, \hyperref[regdesc:LPAPMREGIONNATTRREG]{LP\_APM\_REGION\regindex{n}\_ATTR\_REG}, or \hyperref[regdesc:LPAPM0REGIONNATTRREG]{LP\_APM0\_REGION\regindex{n}\_ATTR\_REG} based on the access path.

\end{itemize}

Different access paths managed by the same register module share the configuration of address ranges and access permissions. For example, the permission management of data paths HP\_APM\_CTRL M0-M4 shown in Figure \ref{fig:APM_Ctrl_Location} should follow the address ranges and access permissions of the 16 address ranges configured in the register module \hyperref[sec:permctrl-reg-hp]{HP\_APM\_REG}. Likewise, the permission management of data path LP\_APM\_CTRL M0-M1 shown in Figure \ref{fig:APM_Ctrl_Location} should follow the address ranges and access permissions of the four address ranges configured in the register module \hyperref[sec:permctrl-reg-lp]{LP\_APM\_REG}.

Take HP SRAM as an example. All masters access it through HP\_APM\_CTRL M1, except that the HP CPU accesses it through PMP and CPU\_APM\_CTRL, and the LP CPU accesses it through HP\_APM M2. Suppose that \hyperref[fielddesc:HPAPMM1FUNCEN]{HP\_APM\_M1\_FUNC\_EN} is enabled and a master in REE1 mode needs to access HP SRAM. The whole process is as follows:

\begin{enumerate}
    \item HP\_APM\_CTRL M1 will first determine whether the address requested to access is within the 16 address ranges configured in the \hyperref[sec:permctrl-reg-hp]{HP\_APM\_REG} register module.

    \item Assuming that the address requested to access is within the second address range, then HP\_APM\_CTRL M1 determines whether the address range is enabled, that is, whether bit 1 of \hyperref[fielddesc:HPAPMREGIONFILTEREN]{HP\_APM\_REGION\_FILTER\_EN} is 1.

    \item If the address range is enabled, HP\_APM\_CTRL M1 checks whether the master has read permission for the second address range, that is, whether \hyperref[fielddesc:HPAPMREGIONNR1R]{HP\_APM\_REGION1\_R1\_R} in \hyperref[regdesc:HPAPMREGIONNATTRREG]{HP\_APM\_REGION1\_ATTR\_REG} is 1. If valid, the read request will be allowed, otherwise, 0 will be returned.
\end{enumerate}

When the HP power domain (please refer to Chapter \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}) restarts after power-down, the LP CPU does not have access to HP SRAM by default. In TEE mode, you can configure \hyperref[fielddesc:LPTEEFORCEACCHPMEMEN]{LP\_TEE\_FORCE\_ACC\_HPMEM\_EN} to 1 for the LP CPU to access HP SRAM without requiring permission checks from the SYS\_APM controller.

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
    \item When the chip is powered up, only the HP CPU is in TEE mode by default, and other masters are in REE2 mode. By default, the SYS\_APM controller blocks access requests from all masters in REE0, REE1, and REE2 modes.

    \item All registers listed in \ref{sec:permctrl-reg-summ} \textit{\nameref{sec:permctrl-reg-summ}} can only be configured by the masters that are in TEE security mode.

    \item The address ranges configured may overlap. For example, if one region is set to be unreadable and another region is readable, then the overlapping area of the two regions is readable. The same rules apply for write and execute permissions.

    \item You can configure \hyperref[fielddesc:HPAPMREGIONNLOCK]{HP\_APM\_REGION\regindex{n}\_LOCK}, \hyperref[fielddesc:LPAPMREGIONNLOCK]{LP\_APM\_REGION\regindex{n}\_LOCK}, and \hyperref[fielddesc:LPAPM0REGIONNLOCK]{LP\_APM0\_REGION\regindex{n}\_LOCK} to lock the configuration of the start and end address of REGION\regindex{n} and its access permission. Only chip reset, system reset, and core reset can unlock the configurations and restore all configurations in APM registers to their default values.
\end{itemize}
\end{tiplisting}

\subsubsection{PERI\_APM Controller}

PERI\_APM does not perform permission control by address ranges, but on a per-peripheral register basis. For each peripheral register \regindex{PERI}, there are the following eight configuration fields:

\begin{itemize}
    \item HP/LP\_TEE\_WRITE\_TEE\_\regindex{PERI}
    \item HP/LP\_TEE\_WRITE\_REE0\_\regindex{PERI}
    \item HP/LP\_TEE\_WRITE\_REE1\_\regindex{PERI}
    \item HP/LP\_TEE\_WRITE\_REE2\_\regindex{PERI}
    \item HP/LP\_TEE\_READ\_TEE\_\regindex{PERI}
    \item HP/LP\_TEE\_READ\_REE0\_\regindex{PERI}
    \item HP/LP\_TEE\_READ\_REE1\_\regindex{PERI}
    \item HP/LP\_TEE\_READ\_REE2\_\regindex{PERI}
\end{itemize}

Through the above eight fields, the read/write permissions of slave \regindex{PERI} in the four security modes TEE, REE0, REE1, and REE2 are respectively controlled. A value of 1 indicates having the corresponding read/write permission under that security mode.

After reset, by default, for each peripheral register, only HP/LP\_TEE\_WRITE\_TEE\_\regindex{PERI} and HP/LP\_TEE\_READ\_TEE\_\regindex{PERI} are set to 1; other fields are all 0, indicating that only the master in TEE security mode can configure peripheral registers.

The six register modules mentioned in this chapter are also peripherals whose access permissions are controlled by PERI\_APM. The six register modules and their corresponding permission control registers in PERI\_APM are as follows:

\begin{itemize}
    \item \hyperref[sec:permctrl-reg-hp]{HP\_APM\_REG}, with the permission control register \hyperref[regdesc:TEEHPAPMCTRLREG]{TEE\_HP\_APM\_CTRL\_REG}

    \item \hyperref[sec:permctrl-reg-lp0]{LP\_APM0\_REG}, with the permission control register \hyperref[regdesc:TEEHPAPMCTRLREG]{TEE\_HP\_APM\_CTRL\_REG}

    \item \hyperref[sec:permctrl-reg-lp]{LP\_APM\_REG}, with the permission control register \hyperref[regdesc:LPTEELPAPMCTRLREG]{LP\_TEE\_LP\_APM\_CTRL\_REG}

    \item \hyperref[sec:permctrl-reg-cpu]{CPU\_APM\_REG}, with the permission control register \hyperref[regdesc:TEECPUAPMCTRLREG]{TEE\_CPU\_APM\_CTRL\_REG}

    \item \hyperref[sec:permctrl-reg-hp-tee]{TEE\_REG}, with the permission control register \hyperref[regdesc:TEETEECTRLREG]{TEE\_TEE\_CTRL\_REG}

    \item \hyperref[sec:permctrl-reg-lp-tee]{LP\_TEE\_REG}, with the permission control register \hyperref[regdesc:LPTEELPTEECTRLREG]{LP\_TEE\_LP\_TEE\_CTRL\_REG}
\end{itemize}

In these permission control registers, only the HP/LP\_TEE\_{WRITE/READ}\_TEE\_\regindex{PERI} fields are configurable; the HP/LP\_TEE\_{WRITE/READ}\_REE{0/1/2}\_\regindex{PERI} fields are not configurable and fixed to 0, indicating that only masters in TEE security mode can access these registers.

\section{Illegal Access and Interrupts}

\subsection{SYS\_APM Controller}

When the information carried on the bus does not match the configuration, \chipname{} treats it as an illegal access and handles it as follows:

\begin{itemize}
    \item Rejects the access request and returns a default value, specifically:
    \begin{itemize}
        \item Returns 0 for read and execute operations
        \item Ignores write operations
    \end{itemize}
    \item Triggers interrupt
\end{itemize}

The SYS\_APM controller will automatically record information related to the illegal access, including the master ID, security mode, access address, reasons for illegal access (address out of bounds or permission restrictions), and permission management result of each access path. All this information can be obtained from relevant registers listed in Section \ref{sec:permctrl-reg-summ} \textit{\nameref{sec:permctrl-reg-summ}}.

Take the access path HP\_APM\_CTRL M0 as an example. When an illegal access occurs:

\begin{itemize}
    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONID]{HP\_APM\_M0\_EXCEPTION\_ID} records the master ID.
    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONMODE]{HP\_APM\_M0\_EXCEPTION\_MODE} records the security mode.
    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONADDR]{HP\_APM\_M0\_EXCEPTION\_ADDR} records the access address.
    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONSTATUS]{HP\_APM\_M0\_EXCEPTION\_STATUS} records the reason for illegal access.
    \begin{itemize}
        \item If the address requested to access is not within the enabled address ranges, bit 1 will be set to 1, indicating the address is out of bounds.
        \item If the address requested to access is within the enabled address ranges, but the master does not have the read/write/execution permission within this region, then bit 0 will be set to 1, indicating permission restrictions.
    \end{itemize}

    \item \hyperref[fielddesc:HPAPMM0EXCEPTIONREGION]{HP\_APM\_M0\_EXCEPTION\_REGION} records the permission management result of each address range. This register has a total of 16 bits, corresponding to 16 address ranges, and bit 0 corresponds to the first address range. When the address to access is within a particular enabled address range, but the master does not have the corresponding read/write/execute permission within this address range, the corresponding bit of this register will be set to 1.

\end{itemize}

The SYS\_APM controller can generate 12 interrupt signals, which will be sent to \hyperref[mod:intmtrx]{Interrupt Matrix}:

\begin{itemize}
    \item HP\_APM\_M0\_INTR
    \item HP\_APM\_M1\_INTR
    \item HP\_APM\_M2\_INTR
    \item HP\_APM\_M3\_INTR
    \item HP\_APM\_M4\_INTR
    \item LP\_APM\_M0\_INTR
    \item LP\_APM\_M1\_INTR
    \item LP\_APM0\_M0\_INTR
    \item CPU\_APM\_M0\_INTR
    \item CPU\_APM\_M1\_INTR
    \item CPU\_APM\_M2\_INTR
    \item CPU\_APM\_M3\_INTR
\end{itemize}

These interrupt signals correspond to the controlled access paths shown in Figure \ref{fig:APM_Ctrl_Location}.

Each interrupt is configured with an interrupt enable register APM\_INT\_EN and an interrupt clear register EXCEPTION\_STATUS\_CLR. When the interrupt enable register is set to 1, if an illegal access occurs on a controlled access path, the corresponding interrupt will be generated. Configuring the interrupt clear register clears the interrupt signal until the next illegal access occurs.

\subsection{PERI\_APM Controller}

When the information carried on the bus does not match the configuration, \chipname{} treats it as an illegal access and handles it as follows:

\begin{itemize}
    \item Rejects the access request and returns a default value, specifically:
    \begin{itemize}
        \item Returns 0 for read and execute operations
        \item Ignores write operations
    \end{itemize}
    \item Returns an error message to the master via the bus, causing the master to enter an exception state
\end{itemize}

Setting HP/LP\_BUS\_ERR\_RESP\_EN to 1 enables returning an error message on illegal access.

The PERI\_APM controller does not log information related to illegal access.

\section{Programming Procedure}

For a master to access memory or peripheral registers, follow the programming procedures below:

\begin{enumerate}

    \item Set the HP CPU to the machine mode (i.e., TEE mode).
    \item Configure \hyperref[fielddesc:TEEMNMODE]{TEE\_M\regindex{n}\_MODE} or \hyperref[fielddesc:LPTEEM0MODE]{LP\_TEE\_M\regindex{n}\_MODE} to choose the security mode for the master. The master ID \regindex{n} is defined in Table \ref{tab:master_source}.

    \item Configure \hyperref[regdesc:HPAPMREGIONNADDRSTARTREG]{HP\_APM\_REGION\regindex{n}\_ADDR\_START} and \hyperref[regdesc:HPAPMREGIONNADDRENDREG]{HP\_APM\_REGION\regindex{n}\_ADDR\_END}, or \hyperref[regdesc:LPAPMREGIONNADDRSTARTREG]{LP\_APM\_REGION\regindex{n}\_ADDR\_START} and \hyperref[regdesc:LPAPMREGIONNADDRENDREG]{LP\_APM\_REGION\regindex{n}\_ADDR\_END}, or \hyperref[regdesc:LPAPM0REGIONNADDRSTARTREG]{LP\_APM0\_REGION\regindex{n}\_ADDR\_START} and \hyperref[regdesc:LPAPM0REGIONNADDRENDREG]{LP\_APM0\_REGION\regindex{n}\_ADDR\_END}, or \hyperref[regdesc:CPUAPMREGIONNADDRSTARTREG]{CPU\_APM\_REGION\regindex{n}\_ADDR\_START} and \hyperref[regdesc:CPUAPMREGIONNADDRENDREG]{CPU\_APM\_REGION\regindex{n}\_ADDR\_END} to define the start and end address of the access address ranges.

    \item Configure \hyperref[regdesc:HPAPMREGIONNATTRREG]{HP\_APM\_REGION\regindex{n}\_ATTR\_REG}, or \hyperref[regdesc:LPAPMREGIONNATTRREG]{LP\_APM\_REGION\regindex{n}\_ATTR\_REG}, or \hyperref[regdesc:LPAPM0REGIONNATTRREG]{LP\_APM0\_REGION\regindex{n}\_ATTR\_REG}, or \hyperref[regdesc:CPUAPMREGIONNATTRREG]{CPU\_APM\_REGION\regindex{n}\_ATTR\_REG} to set the access permissions of each region.


    \item Configure \hyperref[fielddesc:HPAPMREGIONFILTEREN]{HP\_APM\_REGION\_FILTER\_ EN\_REG}, or \hyperref[fielddesc:LPAPMREGIONFILTEREN]{LP\_APM\_REGION\_FILTER\_ EN\_REG}, or \hyperref[fielddesc:LPAPM0REGIONFILTEREN]{LP\_APM0\_REGION\_FILTER\_ EN\_REG}, or \hyperref[fielddesc:CPUAPMREGIONFILTEREN]{CPU\_APM\_REGION\_FILTER\_ EN\_REG} to enable region \regindex{n}.

    \item Configure \hyperref[regdesc:HPAPMFUNCCTRLREG]{HP\_APM\_ FUNC\_CTRL\_REG}, or \hyperref[regdesc:LPAPMFUNCCTRLREG]{LP\_APM \_FUNC\_CTRL\_REG}, or \hyperref[regdesc:LPAPM0FUNCCTRLREG]{LP\_APM0 \_FUNC\_CTRL\_REG}, or \hyperref[regdesc:CPUAPMFUNCCTRLREG]{CPU\_APM \_FUNC\_CTRL\_REG} to enable the permission management for different access paths (enabled by default).

    \item (Required only when accessing peripheral registers) Configure \hyperref[regdesc:TEEPERICTRLREG]{TEE\_\regindex{PERI}\_CTRL\_REG} or \hyperref[regdesc:LPTEEPERICTRLREG]{LP\_TEE\_\regindex{PERI}\_CTRL\_REG} to set the access permissions of the \regindex{PERI} register.

\end{enumerate}

Take I2S accessing HP SRAM via GDMA as an example, assuming that it is only allowed to read and write in the fourth address range 0x40805000 \verb+~+ 0x4080FFFF:

\begin{enumerate}

    \item Configure the HP CPU to machine mode (i.e., TEE mode).

    \item According to the master ID number in Table \ref{tab:master_source}, set \hyperref[fielddesc:TEEMNMODE]{TEE\_M19\_MODE} to 1, so that the security mode for I2S access via GDMA is REE0.

    \item Configure \hyperref[regdesc:HPAPMREGIONNADDRSTARTREG]{HP\_APM\_REGION3\_ADDR\_START} to 0x40805000 and \hyperref[regdesc:HPAPMREGIONNADDRSTARTREG]{HP\_APM\_REGION3\_ADDR\_END} to 0x4080EFFC, respectively.

    \item Set \hyperref[fielddesc:HPAPMREGIONNR0W]{HP\_APM\_REGION3\_R0\_W} and \hyperref[fielddesc:HPAPMREGIONNR0R]{HP\_APM\_REGION3\_R0\_R} to 1 to enable the access permissions to the address range.

    \item Set bit 3 of \hyperref[fielddesc:HPAPMREGIONFILTEREN]{HP\_APM\_REGION\_FILTER\_EN} to 1 to enable region 3 (i.e., the fourth address range).

    \item Set \hyperref[fielddesc:HPAPMM1FUNCEN]{HP\_APM\_M1\_FUNC\_EN} to 1 to enable the permission management for HP\_APM\_CTRL M1.

\end{enumerate}

Through this configuration, I2S can read and write in the address range of 0x40805000 to 0x4080FFFF in HP SRAM using GDMA.
