\begin{register}{H}{mtimectl}{0x4010}\label{regdesc:CLINTMTIMECTLREG}
\regfield{reserved}{26}{6}{{0x0000000}}%
\regfield{MTIME\_SAM}{2}{4}{{0x0}}%
\regfield{MTIME\_OVF}{1}{3}{{0x0}}%
\regfield{MTIME\_EN}{1}{0}{{0x1}}%
\reglabel{Reset}\regnewline%
\begin{regdesc}
\begin{reglist}
\label{fielddesc:CLINTMTIMECTLEN}\item [MTIME\_EN] 配置是否启用系统计数器。\\
1：启用\\
0：禁用\\
仅核心 0 的 MTIMERCTL 寄存器实现了该位。写入核心 1 的 MTIMERCTL 的相应位对系统计时器无效。由于核心 1 可以访问核心 0 的 CLINT 寄存器，所以核心 1 可以通过核心 0 的 MTIMERCTL 寄存器运行或暂停计时器。\\
(R/W)
\label{fielddesc:CLINTMTIMECTLOVF}\item [MTIME\_OVF] 配置系统计数器的溢出位。\\
1：表示系统计数器值为最大值，即 0xFFFFFFFFFFFFFFFF\\
0：其他情况\\
当计数器值为 0xFFFFFFFFFFFFFFFF，即最大值时，此位由硬件设置为 1。软件可以通过写入 0 来清除此位。写入 1 没有影响。\\
(R/W)
\label{fielddesc:CLINTMTIMECTLSAM}\item [MTIME\_SAM] 配置 MTIMELO 和 MTIMEHI 寄存器的采样模式，以准确读取 64 位系统计数。在 64 位系统计数中，只有一半的数据可原子读取；因此，采样模式可以采样另一半的值，并将其存储在缓冲区中，并在下一次读取时，直接从缓冲区读取，而不是读取计数器中实际存储的数据。\\
0x0：无采样（默认）\\
0x1：读取 MTIMELO 时采样系统计数的高 32 位\\
0x2：读取 MTIMEHI 时采样系统计数的低 32 位\\
0x3：读取 MTIMELO 或 MTIMEHI 时，采样系统计数的另一半\\
(R/W)
\end{reglist}
\end{regdesc}
\end{register}
