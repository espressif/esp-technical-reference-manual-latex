\hypertarget{efuse}{}
\chapter{eFuse 控制器 (EFUSE)}
\label{mod:efuse}

\section{概述}

\chipname{} 系统中有一块 1024 位的 eFuse 存储器用于存储参数内容和用户数据，参数内容包括一些硬件模块的控制参数、系统数据参数以及加解密模块使用的密钥等。eFuse 存储器的各个位一旦被烧写为 1，则不能再恢复为 0。eFuse 控制器按照用户配置完成对 eFuse 存储器 中各参数中的各个位的烧写。从芯片外部，eFuse 数据只能通过 eFuse 控制器读取。对于某些 数据，如果未启用读保护，则可以从芯片外部读取该数据;如果启用了读保护，则无法从芯片外部读取该数据。不过，存储在 eFuse 中的某些密钥始终可以供硬件加密模块(例如数字签名、HMAC 等)在内部使用，芯片外部无法获得这些数据。

\section{主要特性}

eFuse 控制器具有以下特性：

\begin{itemize}
    \item 1024 位一次性可编程存储，有 256 个保留位供用户使用
    \item 烧写保护可配置
    \item 读取保护可配置
    \item 使用多种硬件编码方式保护 eFuse 存储器内的参数内容
\end{itemize}

\section{功能描述}

\subsection{结构}

eFuse存储器 从结构上分成 4 个块 (BLOCK0 \verb+~+ BLOCK3)。

BLOCK0 存储了大部分硬件模块使用的控制参数。

表 \ref{tab:efuse-block0-para} 列出了用户可访问（可读并可用）的所有 BLOCK0 中的参数名称、偏移地址、位宽、是否可供硬件使用、烧写保护，以及功能描述。

在这些参数中，{\bfseries \hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS}} 用于控制其他参数的烧写，{\bfseries \hyperref[fielddesc:EFUSERDDIS]{EFUSE\_RD\_DIS}} 用于控制用户读取 BLOCK3 的参数。更多关于这两个参数的信息请见章节\ref{sec:efuse-wr-dis}、\ref{sec:efuse-rd-dis}。

\subsubsection{\hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS}}
\label{sec:efuse-wr-dis}

参数 \hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS} 决定了 eFuse 存储器中所有的参数是否处于烧写保护状态。烧写完 \hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS} 参数后，需要更新 eFuse 控制器的读寄存器以保证烧写保护状态生效。

表 \ref{tab:efuse-block0-para} 以及表 \ref{tab:efuse-block-1-3-para} 中的“\hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS} 烧写保护位”列描述了各参数的烧写保护状态具体由 \hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS} 的哪个位决定。

当某个参数对应的烧写保护位为 0 时，表示此参数未处于烧写保护状态，可以烧写该参数，但已经被烧写的参数不能被重复烧写。

当某个参数对应的烧写保护位为 1 时，表示此参数处于烧写保护状态，此参数的每一个位都无法被更改，未被烧写的位永远为 0，已经被烧写的位永远为 1。因此如果某个参数已经处于烧写保护状态了，则会一直处在该状态，无法再更改。

\subsubsection{\hyperref[fielddesc:EFUSERDDIS]{EFUSE\_RD\_DIS}}
\label{sec:efuse-rd-dis}

所有参数中，只有 BLCOK3 的参数受用户读取保护状态的约束，即表 \ref{tab:efuse-block-1-3-para} 中“\hyperref[fielddesc:EFUSERDDIS]{EFUSE\_RD\_DIS} 读取保护”列非“N/A”的参数。烧写完 \hyperref[fielddesc:EFUSERDDIS]{EFUSE\_RD\_DIS} 参数后，需要更新 eFuse 控制器的读寄存器以保证读取保护状态生效。

如果对应的 \hyperref[fielddesc:EFUSERDDIS]{EFUSE\_RD\_DIS} 位为 0，则表示用户可以读取该数据；若对应的 \hyperref[fielddesc:EFUSERDDIS]{EFUSE\_RD\_DIS} 位为 1，则表示此位管理的参数处于用户读取保护状态。

除 BLOCK3 之外，其他参数不受读取保护状态的约束，均可被用户读取。

\subsubsection{数据存储方式}
\label{sec:efuse-data-storage}

eFuse控制器 使用硬件编码机制保护数据，对用户不可见。

BLOCK0 使用 4 备份方式存储参数，即 BLOCK0 中的所有参数（除了 \hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS}）均在 eFuse 存储器中存储了 4 份。4 备份机制对用户不可见。

BLOCK0中 \hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS} 为 8 比特，其余参数为32 比特，因此BLOCK0在eFuse 存储器中共占据了8 + 32 * 4 = 136 比特的存储空间。

BLOCK1 \verb+~+ BLOCK3 使用 RS (44, 32) 编码方式，最多支持自动校正 6 个字节。本文 RS (44, 32) 使用的本源多项式为 \(p(x)=x^8+x^4+x^3+x^2+1\)。

如图 \ref{fig:efuse-lfsr-circuit-1} 和 \ref{fig:efuse-lfsr-circuit-2} 所示，移位寄存器电路对 32 字节参数进行 RS (44, 32) 编码处理，将 32 字节数据处理为 44 字节，其中：

\begin{itemize}
    \item 字节 0 \verb+~+ 31 为数据本身
    \item 字节 32 \verb+~+ 43 为储存在 8 位触发器 DFF1, DFF2, ..., DFF12 中的奇偶校验字节（gf\_mul\_n 为 \(GF(2^8)\) 域中某一字节数据与元素 \(\alpha^n\) 相乘的结果（n 为整数）
\end{itemize}

然后，硬件将这 44 字节数据一起烧入 eFuse 存储器。eFuse 控制器会在读 eFuse 存储器 的过程中自动完成解码和自动校正。

由于 RS 校验码是在整个 256 位的 eFuse block 上生成的，因此每个 block 只能写入一次。

BLOCK1由于数据比特不足256 比特，因此不足256比特的部分在RS（44，32）编码时硬件会将其视为0，不会影响最终的编码结果。

使用RS（44，32）编码方式的BLOCK中，BLOCK1 数据参数为 88 比特，RS校验码为 96 比特，因此BLOCK1在eFuse存储器中共占据了 88 + 96 = 184 比特的存储空间。

BLOCK2 和BLOCK3的数据参数均为 256 比特，RS校验码为 96 比特，因此在eFuse存储器中共占据了（ 256 + 96 ）* 2 = 704 比特的存储空间。





\subsection{烧写参数}
\label{sec:efuse-software-programming-para}


烧写 eFuse 参数时，需要按块烧写。BLOCK0 \verb+~+ BLOCK3 共用同一段地址来存储即将烧写的参数并通过配置 \hyperref[fielddesc:EFUSEBLKNUM]{EFUSE\_BLK\_NUM} 参数表明当前需要烧写的是哪一个块。

{\bfseries 烧写 BLOCK0}

将寄存器域 \hyperref[fielddesc:EFUSEBLKNUM]{EFUSE\_BLK\_NUM} 配置为0。

\hyperref[regdesc:EFUSEPGMDATA0REG]{EFUSE\_PGM\_DATA0\_REG} \verb+~+ \hyperref[regdesc:EFUSEPGMDATA1REG]{EFUSE\_PGM\_DATA1\_REG} 存储着BLOCK0即将烧写的参数。

\hyperref[regdesc:EFUSEPGMDATA6REG]{EFUSE\_PGM\_DATA2\_REG} \verb+~+ \hyperref[regdesc:EFUSEPGMDATA7REG]{EFUSE\_PGM\_DATA7\_REG} 以及\\ \hyperref[regdesc:EFUSEPGMCHECKVALUE0REG]{EFUSE\_PGM\_CHECK\_VALUE0\_REG} \verb+~+ \hyperref[regdesc:EFUSEPGMCHECKVALUE2REG]{EFUSE\_PGM\_CHECK\_VALUE2\_REG} 中的数据不影响 BLOCK0 的烧写。

{\bfseries 烧写 BLOCK1}

将寄存器域 \hyperref[fielddesc:EFUSEBLKNUM]{EFUSE\_BLK\_NUM} 配置为1。\hyperref[regdesc:EFUSEPGMDATA0REG]{EFUSE\_PGM\_DATA0\_REG} \verb+~+ \hyperref[regdesc:EFUSEPGMDATA5REG]{EFUSE\_PGM\_DATA2\_REG} 存储着 BLOCK1 即将烧写的参数，\hyperref[regdesc:EFUSEPGMCHECKVALUE0REG]{EFUSE\_PGM\_CHECK\_VALUE0\_REG} \verb+~+ \hyperref[regdesc:EFUSEPGMCHECKVALUE2REG]{EFUSE\_PGM\_CHECK\_VALUE2\_REG} 中存储着对应的 RS 校验码。\hyperref[regdesc:EFUSEPGMDATA6REG]{EFUSE\_PGM\_DATA3\_REG} \verb+~+ \hyperref[regdesc:EFUSEPGMDATA7REG]{EFUSE\_PGM\_DATA7\_REG} 中的数据不影响 BLOCK1 的烧写。

{\bfseries 烧写 BLOCK2 \verb+~+ 3}

将寄存器域 \hyperref[fielddesc:EFUSEBLKNUM]{EFUSE\_BLK\_NUM} 配置为 2 或 3 。\hyperref[regdesc:EFUSEPGMDATA0REG]{EFUSE\_PGM\_DATA0\_REG} \verb+~+ \hyperref[regdesc:EFUSEPGMDATA7REG]{EFUSE\_PGM\_DATA7\_REG} 存储着即将烧写的参数，\hyperref[regdesc:EFUSEPGMCHECKVALUE0REG]{EFUSE\_PGM\_CHECK\_VALUE0\_REG}  \verb+~+ \hyperref[regdesc:EFUSEPGMCHECKVALUE2REG]{EFUSE\_PGM\_CHECK\_VALUE2\_REG} 中存储着对应的 RS 校验码。

{\bfseries 烧写流程}

烧写参数的流程如下：
\begin{enumerate}
\item 配置 \hyperref[fielddesc:EFUSEBLKNUM]{EFUSE\_BLK\_NUM} 参数，决定烧写哪一个块。

\item 将需要烧写的参数填写到寄存器 \hyperref[regdesc:EFUSEPGMDATA0REG]{EFUSE\_PGM\_DATA0\_REG} \verb+~+ \hyperref[regdesc:EFUSEPGMDATA7REG]{EFUSE\_PGM\_DATA7\_REG} 和\\ \hyperref[regdesc:EFUSEPGMCHECKVALUE0REG]{EFUSE\_PGM\_CHECK\_VALUE0\_REG}  \verb+~+  \hyperref[regdesc:EFUSEPGMCHECKVALUE2REG]{EFUSE\_PGM\_CHECK\_VALUE2\_REG} 中。


\item 确保 eFuse 烧写电压 VDDQ 的配置正确，具体请参考章节 \ref{sec:efuse-vddq-timing}。

\item 配置寄存器 \hyperref[regdesc:EFUSECONFREG]{EFUSE\_CONF\_REG} 的 \hyperref[fielddesc:EFUSEOPCODE]{EFUSE\_OP\_CODE} 位域为 0x5A5A。

\item 配置寄存器 \hyperref[regdesc:EFUSECMDREG]{EFUSE\_CMD\_REG} 的 \hyperref[fielddesc:EFUSEPGMCMD]{EFUSE\_PGM\_CMD} 位域为 1。

\item 轮询寄存器 \hyperref[regdesc:EFUSECMDREG]{EFUSE\_CMD\_REG} 直到其为 0x0，或者等待烧写完成中断产生。识别烧写/读取完成中断产生的方法详见章节 \ref{sec:efuse-software-read-para} 最后的说明。

\item 将 \hyperref[regdesc:EFUSEPGMDATA0REG]{EFUSE\_PGM\_DATA0\_REG} \verb+~+ \hyperref[regdesc:EFUSEPGMDATA7REG]{EFUSE\_PGM\_DATA7\_REG} 和\\ \hyperref[regdesc:EFUSEPGMCHECKVALUE0REG]{EFUSE\_PGM\_CHECK\_VALUE0\_REG}  \verb+~+  \hyperref[regdesc:EFUSEPGMCHECKVALUE2REG]{EFUSE\_PGM\_CHECK\_VALUE2\_REG} 中写入的参数清零。

\item 执行更新 eFuse 控制器的读寄存器操作使写入的新值生效，具体请参考章节 \ref{sec:efuse-software-read-para}。
\item 检查错误寄存器内容。若读取错误寄存器内数值不为 0，需要再次执行上述步骤 1 \verb+~+ 7 重新烧写一次，通过该方式可以解决由于烧写不充分导致错误寄存器内数值不为 0 的问题。对于不同的 eFuse 块，需要检查的错误寄存器如下。
\begin{itemize}
    \item BLOCK0: \hyperref[regdesc:EFUSERDREPEATERRREG]{EFUSE\_RD\_REPEAT\_ERR\_REG}
    \item BLOCK1: \hyperref[regdesc:EFUSERDRSERRREG]{EFUSE\_RD\_RS\_ERR\_REG}[3:0]
    \item BLOCK2: \hyperref[regdesc:EFUSERDRSERRREG]{EFUSE\_RD\_RS\_ERR\_REG}[7:4]
    \item BLOCK3: \hyperref[regdesc:EFUSERDRSERRREG]{EFUSE\_RD\_RS\_ERR\_REG}[11:8]
\end{itemize}
\end{enumerate}


{\bfseries 限制}

BLOCK0 中不同的参数，甚至对于同一个参数中的不同位可以在多次烧写中分别完成。
但是由于eFuse 存储器本身的烧写使用寿命限制，我们并不推荐这样做，而是建议尽量减少烧写次数。
我们建议对于某个参数中的所有需要烧写的位都在一次烧写中完成。
并且当 \hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS} 的某个位管理的所有参数都烧写之后，
就立即烧写 \hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS} 的这个位。
甚至可以在同一次烧写中既烧写 \hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS} 的某个位管理的所有参数，
同时也烧写 \hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS} 的这个位。通过该方式可以有效保证烧写保护功能，防止\hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS}的烧写和其保护的参数烧写出现混乱。另外严禁对已经烧写了的位重复烧写，否则将发生烧写错误。

BLOCK2 中数据信息在出厂时已经烧写完毕，不允许再次烧写。

BLOCK1 和 BLOCK3 中每一个 BLOCK 都只能烧写一次，不允许重复烧写或分多次烧写。


\subsection{用户读取参数}
\label{sec:efuse-software-read-para}

用户不能直接读取 eFuse存储器 中烧写的信息内容。eFuse 控制器能够将烧写的数据信息读取到对应的地址段的寄存器内，用户再通过读取以 EFUSE\_RD\_ 开始的寄存器来获取 eFuse  存储器内的信息。下表 \ref{tab:efuse-softw-read-para} 列出了读取数据的寄存器名称以及对应烧写时的烧写寄存器名称。

{\bfseries 更新 eFuse 控制器的读寄存器}
\label{sec:efuse-update-reg}

eFuse 控制器通过读取 eFuse 存储器 来更新相应寄存器的数据。读取操作在系统复位时进行，也可以根据需要由用户主动发出（例如在需要读取新烧写 eFuse 存储器中的数据内容时）。用户触发 eFuse 控制器读取操作的流程如下：
\begin{enumerate}
    \item 配置寄存器 \hyperref[regdesc:EFUSECONFREG]{EFUSE\_CONF\_REG} 的 \hyperref[fielddesc:EFUSEOPCODE]{EFUSE\_OP\_CODE} 位域为 0x5AA5。
    \item 配置寄存器 \hyperref[regdesc:EFUSECMDREG]{EFUSE\_CMD\_REG} 的 \hyperref[fielddesc:EFUSEREADCMD]{EFUSE\_READ\_CMD} 位域为 1。
    \item 轮询寄存器 \hyperref[regdesc:EFUSECMDREG]{EFUSE\_CMD\_REG} 直到其为 0x0，或者等待 read\_done interrupt（读取完成中断）产生，识别烧写/读取完成中断产生的方法详见下方说明。
    \item 用户从 eFuse 存储器中读取参数的值。
\end{enumerate}
eFuse 控制器的读寄存器中的数值将一直保持到下一次执行更新 eFuse 控制器读操作。

{\bfseries 烧写错误检测}

烧写错误记录寄存器允许用户检测 eFuse 存储器中的参数和 eFuse 控制器读取的参数是否存在不一致的错误。

\hyperref[regdesc:EFUSERDREPEATERRREG]{EFUSE\_RD\_REPEAT\_ERR\_REG} 寄存器用于指示 BLOCK0 中除了 \hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS} 外的其他参数的烧写是否出错（对应位为 1 代表烧写出错，此位作废；为 0 代表烧写正确）。

\hyperref[regdesc:EFUSERDRSERRREG]{EFUSE\_RD\_RS\_ERR\_REG} 寄存器记录 eFuse 控制器读 BLOCK1 \verb+~+ BLOCK3 过程中，纠错的字节数目以及 RS 解码是否失败的信息。

每次更新 eFuse 控制器的读寄存器操作完成之后，上述寄存器内的数值都会被更新。

{\bfseries 识别烧写/读取操作完成}

识别烧写/读取操作完成的方法如下。位 1 对应烧写操作，位 0 对应读取操作。
\begin{itemize}

    \item 方法 1：
        \begin{enumerate}
            \item 轮询寄存器 \hyperref[regdesc:EFUSEINTRAWREG]{EFUSE\_INT\_RAW\_REG} 的位 1/0，直到位 1/0 为 1，表示烧写/读取操作完成。
        \end{enumerate}
    \item 方法 2：
        \begin{enumerate}
            \item 将寄存器 \hyperref[regdesc:EFUSEINTENAREG]{EFUSE\_INT\_ENA\_REG} 的位 1/0 置 1，使 eFuse 控制器能够产生烧写/读取完成中断。
            \item 配置中断矩阵使 CPU 能够响应 eFuse 控制器的中断信号，可参见 \ref{mod:intmtrx} \textit{\nameref{mod:intmtrx}}。

            \item 等待烧写/读取完成中断产生。

            \item 对寄存器 \hyperref[regdesc:EFUSEINTCLRREG]{EFUSE\_INT\_CLR\_REG} 的位 1/0 置 1 以清除烧写/读取完成中断。
    \end{enumerate}
\end{itemize}

{\bfseries 注意事项}

在 eFuse 控制器执行寄存器更新操作过程中，会复用 \hyperref[regdesc:EFUSEPGMDATA0REG]{EFUSE\_PGM\_DATA\textcolor{red}{\it{n}}\_REG} (n=0，1，.., 7) 寄存器的存储空间，所以在启动 eFuse 控制器更新寄存器之前，不要将有意义的数据写入上述寄存器中。\\
芯片启动过程中，eFuse 控制器会自动更新 eFuse 存储器数据到用户可访问的寄存器。用户可以通过读取相应的寄存器获取 eFuse 存储器内烧写的数据。因此，用户无需再驱动 eFuse 控制器执行读更新操作。

\subsection{eFuse VDDQ 时序}
\label{sec:efuse-vddq-timing}

eFuse 控制器工作在 20 MHz 时钟频率下，其烧写电压 VDDQ 的配置参数需要满足以下条件：
\begin{itemize}
    \item \hyperref[fielddesc:EFUSEDACNUM]{EFUSE\_DAC\_NUM}（烧写电压上升周期数），默认烧写电压为 2.5 V，每个上升周期增加 0.01 V，该参数对应的默认值为 255；
    \item \hyperref[fielddesc:EFUSEDACCLKDIV]{EFUSE\_DAC\_CLK\_DIV}（烧写电压时钟分频系数），要求烧写电压时钟周期大于 1 $\mu$s。
    \item \hyperref[fielddesc:EFUSEPWRONNUM]{EFUSE\_PWR\_ON\_NUM} （eFuse 烧写电压上电等待时间），要求该等待时间结束后烧写电压已稳定，即要求配置数值大于 (\hyperref[fielddesc:EFUSEDACCLKDIV]{EFUSE\_DAC\_CLK\_DIV} \begin{math}x\end{math} \hyperref[fielddesc:EFUSEDACNUM]{EFUSE\_DAC\_NUM})。
    \item \hyperref[fielddesc:EFUSEPWROFFNUM]{EFUSE\_PWR\_OFF\_NUM}（烧写电压掉电等待时间），要求该时间大于 10 $\mu$s。
\end{itemize}

\subsection{中断}
\begin{itemize}

\item 读取完成中断：当 eFuse 控制器读取完成后，此中断被触发。如果要启动该中断信号，需将寄存器 \hyperref[regdesc:EFUSEINTENAREG]{EFUSE\_INT\_ENA\_REG} 的 \hyperref[fielddesc:EFUSEREADDONEINTENA]{EFUSE\_READ\_DONE\_INT\_ENA} 域置 1。
\end{itemize}
