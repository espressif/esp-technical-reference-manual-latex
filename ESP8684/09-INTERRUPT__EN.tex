\chapter{Interrupt Matrix (INTMTRX)}
\label{mod:intmtrx}


\section{Overview}

The interrupt matrix embedded in \chipname{} independently routes peripheral interrupt sources to the ESP-RISC-V CPU's peripheral interrupts, to timely inform CPU to process the coming interrupts. 

The \chipname{} has 43 peripheral interrupt sources. To map them to 31 CPU interrupts, this interrupt matrix is needed.

\begin{tiplisting}
This chapter focuses on how to map peripheral interrupt sources to CPU interrupts. For more details about interrupt configuration, vector, and ISA suggested operations, please refer to Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}.
\end{tiplisting}


\section{Features}
Interrupt matrix has the following features:

\begin{itemize}
    \item 43 peripheral interrupt sources as input
    \item 31 CPU peripheral interrupts as output
    \item Able to query current status of peripheral interrupt sources
    \item Configurable priority, type, threshold, and enable signal of CPU interrupts
\end{itemize}

\ref{fig:interrupt-matrix-structure} shows the structure of the interrupt matrix.

\subsection{Peripheral Interrupt Sources}

The \chipname{} has 43 peripheral interrupt sources in total. Table \ref{tab:peripheral-interrupt-config-status-source} lists all these sources and their configuration/status registers. 

\begin{itemize}
    \item Column ``Index": Peripheral interrupt source index, can be 0 \verb+~+ 42.
    \item Column ``Chapter": In which chapter the interrupt source is described in details.
    \item Column ``Source": Name of the peripheral interrupt source.
    \item Column ``Configuration Register": Registers used to configure routing of the peripheral interrupt sources to CPU peripheral interrupts
    \item Column ``Status Register": Registers used for indicating the interrupt status of peripheral interrupt sources.
    \begin{itemize}
        \item Column ``Status Register - Bit": Bit position in status register, indicating the interrupt status.
        \item Column ``Status Register - Name": Name of status registers.
    \end{itemize}
     
    
\end{itemize}

\subsection{CPU Interrupts}

The \chipname{} implements its interrupt mechanism using an interrupt controller instead of RISC-V Privileged ISA specification. The ESP-RISC-V CPU has 31 interrupts, with unique IDs (1 \verb+~+ 31). Each CPU interrupt has the following properties. 

\begin{itemize}
    \item Priority levels from 1 (lowest) to 15 (highest).
    \item Configurable as high-level triggered or rising-edge triggered.
    \item Programmable threshold for masking interrupts with lower priority. 
\end{itemize}


\begin{tiplisting}
For detailed information about how to configure CPU interrupts, see Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}. 
\end{tiplisting}

\subsection{Allocate Peripheral Interrupt Source to CPU Interrupt}

In this section, the following terms are used to describe the operation of the interrupt matrix.

\begin{itemize}

\item Source\_\regindex{X}: stands for a peripheral interrupt source, wherein \regindex{X} means the index of this interrupt source in Table \ref{tab:peripheral-interrupt-config-status-source}.

\item \hyperref[regdesc:INTERRUPTCORE0PIFPMSMONITORVIOLATESIZEINTRMAPREG]{INTERRUPT\_CORE0\_SOURCE\_\regindex{X}\_MAP\_REG}: stands for a configuration register, mapping Source\_\regindex{X} to CPU interrupt.

\item Num\_P: the ID of CPU interrupts, can be 1 \verb+~+ 31.

\item Interrupt\_P: stands for the CPU interrupt with ID = Num\_P.

\end{itemize}

\subsubsection{Allocate one peripheral interrupt source (Source\_\regindex{X}) to CPU}
Setting the corresponding configuration register \hyperref[regdesc:INTERRUPTCORE0PIFPMSMONITORVIOLATESIZEINTRMAPREG]{INTERRUPT\_CORE0\_SOURCE\_\regindex{X}\_MAP\_REG} of Source\_\regindex{X} to Num\_P allocates this interrupt source to Interrupt\_P. 

\subsubsection{Allocate multiple peripheral interrupt sources (Source\_X\regindex{n}) to CPU}
Setting the corresponding configuration register \hyperref[regdesc:INTERRUPTCORE0PIFPMSMONITORVIOLATESIZEINTRMAPREG]{INTERRUPT\_CORE0\_SOURCE\_X\regindex{n}\_MAP\_REG} of each interrupt source to the same Num\_P allocates multiple sources to the same Interrupt\_P. Any of these sources can trigger CPU Interrupt\_P. When an interrupt signal is generated, interrupt service routine (ISR) should check the interrupt status registers to figure out which peripheral generated the interrupt. For more information, see Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}.

\subsubsection{Disable CPU peripheral interrupt source (Source\_\regindex{X})}
Clearing the configuration register \hyperref[regdesc:INTERRUPTCORE0PIFPMSMONITORVIOLATESIZEINTRMAPREG]{INTERRUPT\_CORE0\_SOURCE\_\regindex{X}\_MAP\_REG} disables the corresponding interrupt source.

\subsection{Query Current Interrupt Status of Peripheral Interrupt Source}

Users can query current interrupt status of a peripheral interrupt source by reading the bit value in \hyperref[regdesc:INTERRUPTCORE0INTRSTATUS0REG]{INTERRUPT\_CORE0\_INTR\_STATUS\_\regindex{n}\_REG} (read only). For the mapping between \hyperref[regdesc:INTERRUPTCORE0INTRSTATUS0REG]{INTERRUPT\_CORE0\_INTR\_STATUS\_\regindex{n}\_REG} and peripheral interrupt sources, please refer to Table \ref{tab:peripheral-interrupt-config-status-source}.
