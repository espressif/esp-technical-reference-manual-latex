\hypertarget{dma}{}
\chapter{通用 DMA 控制器 (GDMA)}
\label{mod:dma}



\section{概述}

通用直接存储访问 (General Direct Memory Access, GDMA) 用于在外设与存储器之间以及存储器与存储器之间提供高速数据传输。软件可以在无需 CPU 干预的情况下通过 GDMA 快速搬移数据，从而降低了 CPU 的工作负载，提高了效率。

\chipname{} GDMA 共有 2 个独立的通道，其中包括 1 个发送通道（即发送通道 0）和 1 个接收通道（即接收通道 0）。这 2 个通道被支持 GDMA 功能的外设所共享，用户可以将通道分配给任何支持 DMA 功能的外设。这些外设包括：SPI2 和 SHA。

GDMA 支持通道间固定优先级及轮询仲裁以管理外设不同的带宽需求。

\section{特性}

GDMA 控制器具有以下几个特点：

\begin{itemize}
    \item 数据传输以字节为单位，传输数据量可软件编程
    \item 支持链表
    \item 访问内部 RAM 时，支持 INCR burst 传输
    \item GDMA 能够访问的内部 RAM 最大地址空间为 256 KB
    \item 包含 1 个 TX、1 个 RX 通道
    \item 任一通道支持可配置的外设选择
    \item 通道间固定优先级及轮询仲裁\item AHB 总线架构
\end{itemize}

\section{架构}

\chipname{} 中所有需要进行高速数据传输的模块都具有 GDMA 功能。GDMA 控制器与 CPU 的数据总线使用相同的地址空间访问内部 RAM。图 \ref{fig:gdma-engine} 为 GDMA 引擎基本架构图。

\begin{figure}[H]
\includegraphics[width=0.9\textwidth]{\modulefiles/figures/gdma_engine}
\centering
\caption{GDMA 引擎 的架构}
\label{fig:gdma-engine}
\end{figure}

GDMA 引擎共有 2 个独立的通道，其中包括 1 个发送通道和 1 个接收通道。每个通道可选择与不同的外设相连，从而实现通道资源被外设共享。

GDMA 引擎通过 AHB\_BUS 将数据存入内部 RAM 或者将数据从内部 RAM 取出。在通过 AHB\_BUS 传输数据之前，GDMA 采用固定优先级的仲裁机制对每个通道的读写请求进行仲裁。内部 RAM 的具体使用范围详见章节 \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}。

软件可以通过挂载链表的方式来使用 GDMA 引擎。链表本身须存储在片内 RAM 中，包括 outlink（发送链表）与 inlink（接收链表）。GDMA 从片内 RAM 中取得链表，然后根据 outlink 中的内容将相应 RAM 中的数据发送出去，或者根据 inlink 中的内容将接收的数据存入指定 RAM 地址空间。


\section{功能描述}

\subsection{外设和存储间的数据传输}
GDMA 支持存储到外设及外设到存储的数据传输，分别对应 TX 及 RX 功能。TX 通道通过 outlink 实现将指定存储区域中的数据搬运到外设的发送端；RX 通道通过 inlink 实现将外设接收到的数据搬运到指定的存储区域。

\subsection{存储到存储的数据传输}
GDMA 支持存储到存储的数据传输。置位 \hyperref[fielddesc:GDMAMEMTRANSENCH0]{GDMA\_MEM\_TRANS\_EN\_CH0}，发送通道 0 的输出将与接收通道 0 的输入相连，从而使能存储到存储的数据传输功能。

\subsection{链表}

\begin{figure}[H]
\includegraphics[width=1\textwidth]{\modulefiles/figures/link_descriptor}
\centering
\caption{链表结构图}
\label{fig:gdma-link-descriptor}
\end{figure}


图 \ref{fig:gdma-link-descriptor} 所示为链表的结构图。发送链表与接收链表结构相同。每个链表由一个或者若干个描述符构成，一个描述符由 3 个字组成。链表应存放在内部 RAM 中，供 GDMA 引擎使用。描述符每一字段的意义如下：

\begin{itemize}
    \item owner (DW0) [31]：表示当前描述符对应的 buffer 允许的操作者。\\
    1'b0：允许的操作者为 CPU；\\
    1'b1：允许的操作者为 GDMA 控制器。\\
    在 GDMA 使用完该描述符对应的 buffer 后，对于 接收描述符，硬件默认会自动将该位清零；对于发送描述符，需要将 \hyperref[fielddesc:GDMAOUTAUTOWRBACKCH0]{GDMA\_OUT\_AUTO\_WRBACK\_CH0} 置 1，硬件才会自动将该位清零。 软件在挂载链表前需要将该位置 1。

    {\bfseries 注意：} 本文以 GDMA\_OUT 开头的寄存器对应 TX 通道寄存器，以 GDMA\_IN 开头的寄存器对应 RX 通道寄存器。
    \item suc\_eof (DW0) [30]：表示一个描述符对应的数据成功传输后是否触发 GDMA\_IN\_SUC\_EOF\_CH0\_INT 或 GDMA\_OUT\_EOF\_CH0\_INT 中断。\\
    1'b0：当前描述符成功传输后不会触发中断；\\
    1'b1：当前描述符成功传输后触发中断。\\
    对于接收描述符，需要软件将该 位 写 0，硬件会在接收到包含 EOF 标志的数据后将该位置 1。\\对于发送描述符，需要软件按需要将描述符中的该位置 1。
    如果软件将在某个描述符中将该位配置为 1，则 GDMA 在处理完该描述符时，会在发送给外设的数据中加入 EOF 标志，告知外设该段数据是一个阶段性结束。
    \item Reserved (DW0) [29]：保留。此位为无关项。
    \item err\_eof (DW0) [28]：表示接收结束错误标志。\\
    对于接收描述符，外设在收完描述符对应的数据段并检测到接收数据错误会将该位置 1。
    \item Reserved (DW0) [27:24]：保留。
    \item length (DW0) [23:12]：表示当前描述符对应的 buffer 中的有效字节数。对于发送描述符，该段由软件填写，表示从 buffer 中读取数据时需要读取的字节数；对于接收描述符，该段由硬件使用完该 buffer 后或者接收到最后一个数据时自动填写，表示 buffer 中存储的有效字节数。
    \item size (DW0) [11:0]：表示当前描述符对应的 buffer 容量的字节数，size 需要大于或者等于 length。
    \item buffer address pointer (DW1)：buffer 的地址。
    \item next descriptor address (DW2)：下一个描述符的地址。该地址必须指向片内 RAM 的地址空间。如果当前描述符为链表中最后一个描述符时，该值填 0。
\end{itemize}



用 GDMA 接收数据时，如果数据帧或者包结束时，当前描述符的 suc\_eof 位将被置 1，GDMA 停止继续向该描述符指定的 buffer 接收数据。即使此时已接收数据的长度小于当前描述符指定的 buffer 长度，后续接收的数据也不会继续占用该 buffer 的剩余空间，而会进入下一个描述符指定的 buffer。



\subsection{启动 DMA}

软件通过挂载链表的方式来使用 GDMA。
\begin{itemize}
    \item 对于接收数据，软件挂载好接收链表并准备好接收数据，配置 \\\hyperref[fielddesc:GDMAINLINKADDRCH0]{GDMA\_INLINK\_ADDR\_CH0} 字段指向第一个接收链表描述符，然后置位 \hyperref[fielddesc:GDMAINLINKSTARTCH0]{GDMA\_INLINK\_START\_CH0} 位启动 GDMA。
    \item 对于发送数据，软件挂载好发送链表并准备好发送数据，配置 \hyperref[fielddesc:GDMAOUTLINKADDRCH0]{GDMA\_OUTLINK\_ADDR\_CH0} 字段指向第一个发送链表描述符，然后置位 \hyperref[fielddesc:GDMAOUTLINKSTARTCH0]{GDMA\_OUTLINK\_START\_CH0} 位启动 GDMA。
\end{itemize}
\hyperref[fielddesc:GDMAINLINKSTARTCH0]{GDMA\_INLINK\_START\_CH0} 与 \hyperref[fielddesc:GDMAOUTLINKSTARTCH0]{GDMA\_OUTLINK\_START\_CH0} 位由硬件自动清零。

有时您可能想要在 DMA 数据传输已经开始后追加更多描述符。要挂载更多描述符，原本看似只需将链表最末尾描述符的 next descriptor address (DW2) 字段配置为新链表第一个描述符对应的地址。但如果 DMA 数据传输已经或马上就要结束，这个方法便行不通了。GDMA 引擎有专门的逻辑来确保数据传输继续或重启：如果数据传输仍在进行，GDMA 引擎会确保顾及到新追加的描述符；如果数据传输已经结束，GDMA 引擎会重启数据传输，传输新追加的描述符。这个逻辑由 Restart 功能实现。

软件使用 Restart 功能时，需要重写已挂载链表的最后一个描述符，使其第三个字中的内容（即 DW2）指向新链表的首地址；然后置位 \hyperref[fielddesc:GDMAINLINKRESTARTCH0]{GDMA\_INLINK\_RESTART\_CH0} 或者 \hyperref[fielddesc:GDMAOUTLINKRESTARTCH0]{GDMA\_OUTLINK\_RESTART\_CH0}（这两个位由硬件自动清零），如图 \ref{fig:gdma-link-restart} 所示，硬件会在读取已挂载链表的最后一个描述符时，获取新挂载链表的地址，从而继续处理新挂载的链表。

\subsection{读链表}

软件在配置并启动 GDMA 后，GDMA会从内部 RAM 读取链表。GDMA 会检查读入的链表描述符是否正确。只有当链表描述符通过检查时，GDMA 对应的通道 才会开始搬运数据。当链表描述符没有通过检查，硬件将触发描述符错误中断（GDMA\_IN\_DSCR\_ERR\_CH0\_INT 或者 GDMA\_OUT\_DSCR\_ERR\_CH0\_INT），同时该通道将会处于阻塞状态，停止工作。

描述符检查项包括：
\begin{itemize}
    \item \hyperref[fielddesc:GDMAINCHECKOWNERCH0]{GDMA\_IN\_CHECK\_OWNER\_CH0} 或者 \hyperref[fielddesc:GDMAOUTCHECKOWNERCH0]{GDMA\_OUT\_CHECK\_OWNER\_CH0} 置 1 时，检查描述符的 owner 位。如果该位为 0，表示当前操作者应为 CPU，检查失败。将\hyperref[fielddesc:GDMAINCHECKOWNERCH0]{GDMA\_IN\_CHECK\_OWNER\_CH0} 或者 \hyperref[fielddesc:GDMAOUTCHECKOWNERCH0]{GDMA\_OUT\_CHECK\_OWNER\_CH0} 置 0 可以跳过检查；\\

    \item 检查描述符中第二个字指示的地址是否在 0x3FCA0000 \verb+~+ 0x3FCDFFFF 范围内（请参见本节 \ref{sec:dma-access-inter-ram}）。如果不在该范围内，则检查失败。\\
\end{itemize}

软件在检查到通道描述符错误中断后，需要复位对应的通道，重新配置该 DMA 通道并启动。具体流程详见章节 \ref{tx-program}、章节 \ref{rx-program} 和章节 \ref{tx-rx-program}。

{\bfseries 注意：} 描述符的第三个字指示的地址只能在片内，指向下一个可用描述符；所有描述符都需存在内存中。


\subsection{数据传输结束标志}

GDMA 通过 EOF 来指示对应描述符所需传输的数据段数据传输结束。

发送数据时，置位 \hyperref[fielddesc:GDMAOUTTOTALEOFCH0INTENA]{GDMA\_OUT\_EOF\_CH0\_INT\_ENA} 位使能\\ GDMA\_OUT\_EOF\_CH0\_INT 中断， 当带有 EOF 标志的描述符对应 buffer 的数据传输完成后， GDMA 会产生该中断。

接收数据时， 置位 \hyperref[fielddesc:GDMAINSUCEOFCH0INTENA]{GDMA\_IN\_SUC\_EOF\_CH0\_INT\_ENA} 位使能 GDMA\_IN\_SUC\_EOF\_CH0\_INT 中断，表示带有 EOF 标志的数据段数据接收完成。GDMA 还支持 GDMA\_IN\_ERR\_CH0\_EOF\_INT 中断，置位 \hyperref[fielddesc:GDMAINERREOFCH0INTENA]{GDMA\_IN\_ERR\_EOF\_CH0\_INT\_ENA} 使能该中断，表示对应描述符所需传输的数据段数据接收完成，但该帧或包接收数据有错误。

软件在检测到 GDMA\_OUT\_TOTAL\_EOF\_CH0\_INT 或 GDMA\_IN\_SUC\_EOF\_CH0\_INT 中断时，可以记录 \\\hyperref[fielddesc:GDMAOUTEOFDESADDRCH0]{GDMA\_OUT\_EOF\_DES\_ADDR\_CH0} 或 \hyperref[fielddesc:GDMAINSUCEOFDESADDRCH0]{GDMA\_IN\_SUC\_EOF\_DES\_ADDR\_CH0} 字段的值，即最后一个描述符的地址。这样，软件可以知道哪些描述符已经被使用并根据需要回收描述符。

{\bfseries 注意：} 本章中提到发送链表描述符的 EOF 为 suc\_eof，接收链表描述符的 EOF 可以为 suc\_eof 和 err\_eof。

\subsection{访问片内 RAM}\label{sec:dma-access-inter-ram}
GDMA 任意 RX/TX 通道均可以访问片内 RAM，其可访问的片内地址空间为 0x3FCA0000 \verb+~+ 0x3FCDFFFF。为加速数据传输速率，支持突发传输模式，置位 \hyperref[fielddesc:GDMAINDATABURSTENCH0]{GDMA\_IN\_DATA\_BURST\_EN\_CH0} 使能 RX 通道突发传输模式；置位 \hyperref[fielddesc:GDMAOUTDATABURSTENCH0]{GDMA\_OUT\_DATA\_BURST\_EN\_CH0} 使能 TX 通道突发传输模式。默认情况下，突发传输没有使能。

\subsection{仲裁}

为了确保及时响应高速低延迟的外设请求，比如 SPI 等，GDMA 在通道仲裁机制中引入固定优先级，即每个通道的优先级可配置。GDMA 支持 10（0 \verb+~+ 9）个等级的优先级。其数值越大，对应的优先级越高，请求响应越及时。当若干个通道配置为相同的优先级时，这几个通道间对请求的响应将采用轮询仲裁机制。

需要注意的是，所有外设总的吞吐率之和不能超过 GDMA 能支持的最大有效带宽，否则低优先级的外设请求可能无法获得及时响应。



\section{GDMA 中断}
\begin{itemize}
\item \label{int:GDMAINFIFOOVFINT} GDMA\_INFIFO\_OVF\_CH0\_INT：当 GDMA 接收数据方向缓存溢出时触发此中断。
\item \label{int:GDMAINFIFOUDFINT}GDMA\_INFIFO\_UDF\_CH0\_INT：当 GDMA 接收数据方向缓存无数据时触发此中断。
\item \label{int:GDMAINDSCREMPTYINT}GDMA\_IN\_DSCR\_EMPTY\_CH0\_INT：对于接收通道 0，当接收链表描述符指向的 buffer 大小小于待接收数据长度时触发此中断。
\item \label{int:GDMAINDSCRERRINT}GDMA\_IN\_DSCR\_ERR\_CH0\_INT：对于接收通道 0，当接收链表描述符里有错误时触发此中断。
\item \label{int:GDMAINERREOFINT}GDMA\_IN\_ERR\_EOF\_CH0\_INT：对于接收通道 0，当接收的描述符对应数据段中有错误发生时触发此中断。
\item \label{int:GDMAINSUCEOFINT}GDMA\_IN\_SUC\_EOF\_CH0\_INT：对于接收通道 0，当一个接收链表描述符对应的数据接收完成，并且描述符的 suc\_eof 为 1 时触发此中断。
\item \label{int:GDMAINDONEINT}GDMA\_IN\_DONE\_CH0\_INT：对于接收通道 0，当一个接收链表描述符对应的数据接收完成时触发此中断。
\item \label{int:GDMAOUTFIFOOVFINT} GDMA\_OUTFIFO\_OVF\_CH0\_INT：当 GDMA 发送数据方向缓存溢出时触发此中断。
\item \label{int:GDMAOUTFIFOUDFINT}GDMA\_OUTFIFO\_UDF\_CH0\_INT：当 GDMA 发送数据方向缓存无数据时触发此中断。
\item \label{int:GDMAOUTTOTALEOFINT}GDMA\_OUT\_TOTAL\_EOF\_CH0\_INT：对于发送通道 0，当一个链表（可包含多个链表描述符）对应的所有数据都已发送完成时触发此中断。
\item \label{int:GDMAOUTDSCRERRINT}GDMA\_OUT\_DSCR\_ERR\_CH0\_INT：对于发送通道 0，当发送链表描述符里有错误时触发此中断。
\item \label{int:GDMAOUTEOFINT}GDMA\_OUT\_EOF\_CH0\_INT：对于发送通道 0，当发送描述符的 EOF 位为 1，并且该描述符对应的数据发送完成时触发此中断。当 \hyperref[fielddesc:GDMAOUTEOFMODECH0]{GDMA\_OUT\_EOF\_MODE\_CH0} 为 0 时，该描述符对应的最后一个数据进入到 GDMA TX 通道时，该中断触发；当 \hyperref[fielddesc:GDMAOUTEOFMODECH0]{GDMA\_OUT\_EOF\_MODE\_CH0} 为 1 时，该描述符对应的最后一个数据从 GDMA TX 通道取出时，该中断触发。
\item \label{int:GDMAOUTDONEINT}GDMA\_OUT\_DONE\_CH0\_INT：对于发送通道 0，当一个发送链表描述符对应的数据发送完成时触发此中断。
\end{itemize}


\section{编程流程}

\subsection{GDMA 时钟与复位配置流程}
GDMA 的时钟与复位配置流程如下：
\begin{enumerate}
    \item 置位系统寄存器 \hyperref[fielddesc:SYSTEMDMACLKEN]{SYSTEM\_DMA\_CLK\_EN}，使能 GDMA 模块时钟；
    \item 对系统寄存器 \hyperref[fielddesc:SYSTEMDMARST]{SYSTEM\_DMA\_RST} 置 0，释放 GDMA 的复位信号。
\end{enumerate}

\subsection{GDMA TX 通道配置流程}\label{tx-program}
利用 GDMA 发送数据时，GDMA TX 通道的软件配置流程如下：
\begin{enumerate}
    \item 对寄存器 \hyperref[fielddesc:GDMAOUTRSTCH0]{GDMA\_OUT\_RST\_CH0} 置 1 然后置 0，复位 GDMA TX 通道状态机和 FIFO 指针；
    \item 挂载好发送链表，配置寄存器 \hyperref[fielddesc:GDMAOUTLINKADDRCH0]{GDMA\_OUTLINK\_ADDR\_CH0} 指向第一个发送链表描述符；
    \item 配置 \hyperref[fielddesc:GDMAPERIOUTSELCH0]{GDMA\_PERI\_OUT\_SEL\_CH0} 为对应的外设号，见表\ref{tab:gdma-peri-sel}；
    \item 置位 \hyperref[fielddesc:GDMAOUTLINKSTARTCH0]{GDMA\_OUTLINK\_START\_CH0} 启动 GDMA TX 通道发送数据；
    \item 配置对应的外设（SPI2 或 SHA），并启动该外设，具体配置请参考对应的外设章节；
    \item 等待 GDMA\_OUT\_TOTAL\_EOF\_CH0\_INT 中断，即数据传输完成。
\end{enumerate}

\subsection{GDMA RX 通道配置流程}\label{rx-program}
利用 GDMA 接收数据时，GDMA RX 通道的软件配置流程如下：
\begin{enumerate}
    \item 对寄存器 \hyperref[fielddesc:GDMAINRSTCH0]{GDMA\_IN\_RST\_CH0} 置 1 然后置 0，复位 GDMA RX 通道状态机和 FIFO 指针；
    \item 挂载好接收链表，配置寄存器 \hyperref[fielddesc:GDMAINLINKADDRCH0]{GDMA\_INLINK\_ADDR\_CH0} 指向第一个接收链表描述符；
    \item 配置 \hyperref[fielddesc:GDMAPERIINSELCH0]{GDMA\_PERI\_IN\_SEL\_CH0} 为对应的外设号，见表\ref{tab:gdma-peri-sel}；
    \item 置位 \hyperref[fielddesc:GDMAINLINKSTARTCH0]{GDMA\_INLINK\_START\_CH0} 启动 GDMA RX 通道发送数据；
    \item 配置对应的外设 (SPI2)，并启动该外设，具体配置请参考对应的外设章节；
\end{enumerate}

\subsection{GDMA 存储器到存储器配置流程}\label{tx-rx-program}
利用 GDMA 从存储到存储搬运数据时配置流程如下：
\begin{enumerate}
    \item 对寄存器 \hyperref[fielddesc:GDMAOUTRSTCH0]{GDMA\_OUT\_RST\_CH0} 置 1 然后置 0，复位 GDMA TX 通道状态机和 FIFO 指针；
    \item 对寄存器 \hyperref[fielddesc:GDMAINRSTCH0]{GDMA\_IN\_RST\_CH0} 置 1 然后置 0，复位 GDMA RX 通道状态机和 FIFO 指针；
    \item 挂载好发送链表，配置寄存器 \hyperref[fielddesc:GDMAOUTLINKADDRCH0]{GDMA\_OUTLINK\_ADDR\_CH0} 指向第一个发送链表描述符；
    \item 挂载好接收链表，配置寄存器 \hyperref[fielddesc:GDMAINLINKADDRCH0]{GDMA\_INLINK\_ADDR\_CH0} 指向第一个接收链表描述符；
    \item 置位 \hyperref[fielddesc:GDMAMEMTRANSENCH0]{GDMA\_MEM\_TRANS\_EN\_CH0} 使能 memory-to-memory 传输功能；
    \item 置位 \hyperref[fielddesc:GDMAOUTLINKSTARTCH0]{GDMA\_OUTLINK\_START\_CH0} 启动 GDMA TX 通道发送数据；
    \item 置位 \hyperref[fielddesc:GDMAINLINKSTARTCH0]{GDMA\_INLINK\_START\_CH0} 启动 GDMA RX 通道发送数据；
    \item 如果某个发送链表描述符的 suc\_eof 位配置为 1，则该描述符对应的数据段传输结束后会触发 GDMA\_IN\_SUC\_EOF\_CH0\_INT 中断。
\end{enumerate}
