\chapter{eFuse Controller (eFuse)}
\label{mod:efuse}


\section{ Overview}

\chipname{} contains a 1024-bit eFuse memory to store parameters. Once an eFuse bit is programmed to 1, it can never be reverted to 0. The eFuse controller programs individual bits of parameters in eFuse memory according to user configurations.

\chipname{} contains a 1024-bit eFuse \colorbox{yellow}{memory} to store parameters. Once an eFuse bit is programmed to 1, it can never be reverted to 0. The eFuse controller programs individual bits of parameters in eFuse \colorbox{yellow}{memory} according to user configurations.
}

 \section{Features}

eFuse Controller has the following features:

\begin{itemize}
    \item 1024-bit One-time programmable storage with 256 reserved bits available for users.
    \item Configurable write protection
    \item Configurable read protection
    \item Various hardware encoding schemes against data corruption
\end{itemize}

\section{Functional Description}

\subsection{Structure}

eFuse data is organized in 4 blocks (BLOCK0 \verb+~+ BLOCK3).

BLOCK0 holds most parameters. BLOCK1 and BLOCK2 contains system parameters, and BLOCK3 can store a 256-bit key or two 128-bit keys.

\subsubsection{\hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS}}
\label{sec:efuse-wr-dis}

Parameter \hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS} determines whether individual eFuse parameters are write-protected. After \hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS} has been programmed, execute an eFuse read operation to make write protection take effect.



\subsubsection{\hyperref[fielddesc:EFUSERDDIS]{EFUSE\_RD\_DIS}}
\label{sec:efuse-rd-dis}

Only the eFuse BLOCK3 can be individually read protected to prevent any access from outside the chip. After \hyperref[fielddesc:EFUSERDDIS]{EFUSE\_RD\_DIS} has been programmed, execute an eFuse read operation to make read protection take effect.


If the corresponding \hyperref[fielddesc:EFUSERDDIS]{EFUSE\_RD\_DIS} bit is 0, then the eFuse block can be read by users; if the corresponding \hyperref[fielddesc:EFUSERDDIS]{EFUSE\_RD\_DIS} bit is 1, then the parameter controlled by this bit is user protected.



\subsubsection{Data Storage}
\label{sec:efuse-data-storage}

Internally, eFuses use hardware encoding schemes to protect data from corruption, which are invisible for users.

All BLOCK0 parameters except for \hyperref[fielddesc:EFUSEWRDIS]{EFUSE\_WR\_DIS} are stored with four backups, meaning each bit is stored four times. This backup scheme is not visible to users.

BLOCK1 \verb+~+ BLOCK3 use RS (44, 32) coding scheme that supports up to 6 bytes of automatic error correction. The primitive polynomial of RS (44, 32) is \(p(x)=x^8+x^4+x^3+x^2+1\).

The eFuse controller also use generated random read cycles and randomization of data reading sequence to prevent DPA (Differential Power Analysis) attacks.



\subsection{Programming of Parameters}
\label{sec:efuse-software-programming-para}

The eFuse controller can only program eFuse parameters in one block at a time. BLOCK0 \verb+~+ BLOCK3 share the same programming data register(EFUSE\_PGM\_DATA0\_REG $\sim$ EFUSE\_PGM\_DATA6\_REG) to store the parameters to be programmed. Configure parameter \hyperref[fielddesc:EFUSEBLKNUM]{EFUSE\_BLK\_NUM} to indicate which block should be programmed.



\subsection{User Read of Parameters}
\label{sec:efuse-software-read-para}

Users cannot read eFuse bits in efuse memory directly. The eFuse Controller hardware reads all eFuse bits and stores the results to their corresponding repeat registers in its memory space. The repeat register for BLOCK0 is EFUSE\_RD\_REPEAT\_DATA0\_REG, and the repeat register for BLOCK1 to BLOCK3 is EFUSE\_RD\_BLK1\_DATAx\_REG  $\sim$ EFUSE\_RD\_BLK3\_DATAx\_REG.


Users cannot read eFuse bits \colorbox{yellow}{in eFuse memory} directly. The eFuse Controller hardware reads all eFuse bits and stores the results to their corresponding \colorbox{yellow}{repeat} registers in its memory space.

The repeat register for block0 is EFUSE\_RD\_REPEAT\_DATA0\_REG, and the repeat register for BLOCK1 to BLOCK3 is EFUSE\_RD\_BLK1\_DATAx\_REG  $\sim$ EFUSE\_RD\_BLK3\_DATAx\_REG.


{\bfseries Updating eFuse read registers}
\label{sec:efuse-update-reg}

The eFuse Controller reads internal eFuses to update corresponding registers. This read operation happens on system reset and can also be triggered manually by users as needed (e.g., if new eFuse values have been programmed).

{\bfseries Error detection}

Error record registers allow users to detect if there are any inconsistencies in the stored backup eFuse parameters.
