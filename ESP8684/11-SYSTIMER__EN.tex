\chapter{System Timer (SYSTIMER)}
\label{mod:systimer}

\section{Overview}
\chipname{} provides a 52-bit timer, which can be used to generate tick interrupts for the operating system, or be used as a general timer to generate periodic interrupts or one-time interrupts.

The timer consists of two counters UNIT0 and UNIT1. The count values can be monitored by three comparators COMP0, COMP1, and COMP2. See the timer block diagram on Figure \ref{fig:system-timer-structure}.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.6\textwidth]{11-SYSTIMER/figures/systimer_components_20220718.pdf}
    \caption{System Timer Structure}
    \label{fig:system-timer-structure}
\end{figure}

\section{Features}

The system timer has the following features:

\begin{itemize}
    \item Two 52-bit counters and three 52-bit comparators 
    \item Software accessing registers clocked by APB\_CLK
    \item Use CNT\_CLK for counting
    \item 52-bit alarm values (t) and 26-bit alarm periods ($\delta$t)
    \item Two modes to generate alarms:
    \begin{itemize}
        \item Target mode: only a one-time alarm is generated based on the alarm value (t)
        \item Period mode: periodic alarms are generated based on the alarm period ($\delta$t)
    \end{itemize}
    \item Three comparators generating three independent interrupts based on configured alarm value (t) or alarm period ($\delta$t)
    \item Software configuring the reference count value. For example, the system timer is able to load back the sleep time recorded by RTC timer via software after Light-sleep
    \item Able to stall or continue running when CPU stalls or enters on-chip-debugging mode
\end{itemize}


\section{Clock Source Selection}
The counters and comparators are driven using XTAL\_CLK. After scaled by a fractional divider, a $f_{XTAL\_CLK}/3$ clock is generated in one count cycle and a $f_{XTAL\_CLK}/2$ clock in another count cycle. The average clock frequency is $f_{XTAL\_CLK}/2.5$, i.e. the CNT\_CLK in Figure \ref{fig:system-timer-Alarm-Generation}.

Software operation such as configuring registers is clocked by APB\_CLK. For more information about APB\_CLK, see Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}}. 

The following two bits of system registers are also used to control the system timer:

\begin{itemize}
    \item SYSTEM\_SYSTIMER\_CLK\_EN in register SYSTEM\_PERIP\_CLK\_EN0\_REG: enable APB\_CLK signal to system timer.
    \item SYSTEM\_SYSTIMER\_RST in register SYSTEM\_PERIP\_RST\_EN0\_REG: reset system timer. 
\end{itemize}

Note that if the timer is reset, its registers will be restored to their default values. For more information, please refer to Table Peripheral Clock Gating and Reset in Chapter \ref{mod:sysreg} \textit{\nameref{mod:sysreg}}.

\subsection{Counter}
The system timer has two 52-bit timer counters, shown as UNIT\regindex{n} (\regindex{n} = 0 or 1). Their counting clock source is CNT\_CLK. Whether UNIT\regindex{n} works or not is controlled by two bits in register \hyperref[regdesc:SYSTIMERCONFREG]{SYSTIMER\_CONF\_REG}:
\begin{itemize}
    \item \hyperref[fielddesc:SYSTIMERTIMERUNIT1WORKEN]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_WORK\_EN}: set this bit to enable the counter UNIT\regindex{n} in system timer.
    \item \hyperref[fielddesc:SYSTIMERTIMERUNIT1CORE0STALLEN]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_CORE0\_STALL\_EN}: if this bit is set, the counter UNIT\regindex{n} stops when CPU is stalled. The counter continues its counting after the CPU resumes.
\end{itemize}
The configuration of the two bits to control the counter UNIT\regindex{n} is shown below, assuming that CPU is stalled.

When the counter UNIT\regindex{n} is at work, the count value is incremented on each counting cycle. When the counter UNIT\regindex{n} is stopped, the count value stops increasing and keeps unchanged.

The lower 32 and higher 20 bits of initial count value are loaded from the registers \hyperref[fielddesc:SYSTIMERTIMERUNIT0LOADLO]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_LOAD\_LO} and \hyperref[fielddesc:SYSTIMERTIMERUNIT0LOADHI]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_LOAD\_HI}. Writing 1 to the bit \hyperref[fielddesc:SYSTIMERTIMERUNIT0LOAD]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_LOAD} will trigger a reload event, and the current count value will be changed immediately. If UNIT\regindex{n} is at work, the counter will continue to count up from the new reloaded value.

Writing 1 to \hyperref[fielddesc:SYSTIMERTIMERUNIT0UPDATE]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_UPDATE} will trigger an update event. The lower 32 and higher 20 bits of current count value will be locked into the registers \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUELO]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_LO} and \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEHI]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_HI}, and then \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEVALID]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_VALID} is asserted. Before the next update event, the values of \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUELO]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_LO} and \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEHI]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_HI} remain unchanged.


\subsection{Comparator and Alarm}
The system timer has three 52-bit comparators, shown as COMP\regindex{x} (\regindex{x} = 0, 1, or 2). The comparators can generate independent interrupts based on different alarm values (t) or alarm periods ($\delta$t).

Configure \hyperref[fielddesc:SYSTIMERTARGET0PERIODMODE]{SYSTIMER\_TARGET\regindex{x}\_PERIOD\_MODE} to choose from the two alarm modes for each COMP\regindex{x}:
\begin{itemize}
    \item 1: period mode
    \item 0: target mode
\end{itemize}

In period mode, the alarm period ($\delta$t) is provided by the register \hyperref[fielddesc:SYSTIMERTARGET2PERIOD]{SYSTIMER\_TARGET\regindex{x}\_PERIOD}. Assuming that current count value is t1, when it reaches (t1 + $\delta$t), an alarm interrupt will be generated. Another alarm interrupt also will be generated when the count value reaches (t1 + 2*$\delta$t). By such way, periodic alarms are generated.

In target mode, the lower 32 bits and higher 20 bits of the alarm value (t) are provided by  \hyperref[fielddesc:SYSTIMERTIMERTARGET0LO]{SYSTIMER\_TIMER\_TARGET\\\regindex{x}\_LO} and \hyperref[fielddesc:SYSTIMERTIMERTARGET0HI]{SYSTIMER\_TIMER\_TARGET\regindex{x}\_HI}. Assuming that current count value is t2 (t2 <= t), an alarm interrupt will be generated when the count value reaches the alarm value (t). Unlike in period mode, only one alarm interrupt is generated in target mode.

\hyperref[fielddesc:SYSTIMERTARGET0TIMERUNITSEL]{SYSTIMER\_TARGET\regindex{x}\_TIMER\_UNIT\_SEL} is used to choose the count value from which timer counter to be compared for alarm:

\begin{itemize}
        \item 1: use the count value from UNIT\regindex{1}
        \item 0: use the count value from UNIT\regindex{0}
\end{itemize}

Finally, set \hyperref[fielddesc:SYSTIMERTARGET0WORKEN]{SYSTIMER\_TARGET\regindex{x}\_WORK\_EN} and COMP\regindex{x} starts to compare the count value with the alarm value (t) in target mode or with the alarm period (t1 + n*$\delta$t) in period mode.

An alarm is generated when the count value equals to the alarm value (t) in target mode or to the start value + n*alarm period $\delta$t (n = 1,2,3...) in period mode. But if the alarm value (t) set in registers is less than current count value, i.e. the target has already passed, or current count value is larger than the target value (t) within a range (0 \verb+~+ $2^{51}$ -1), an alarm interrupt also is generated immediately. \subsection{Interrupt}
Each comparator has one level-type alarm interrupt, named as SYSTIMER\_TARGET\regindex{x}\_INT. Interrupts signal is asserted high when the comparator starts to alarm. Until the interrupt is cleared by software, it remains high. To enable interrupts, set the bit \hyperref[fielddesc:SYSTIMERTARGET0INTENA]{SYSTIMER\_TARGET\regindex{x}\_INT\_ENA}.


\section{Programming Procedure}\label{sec:instructions-of-usage}
When configuring COMP\regindex{x} and UNIT\regindex{n}, please ensure the corresponding COMP and UNIT are at work.
\subsection{Read Current Count Value}\label{sec:to-read-system-timer-value}
\begin{enumerate}
    \item Set \hyperref[fielddesc:SYSTIMERTIMERUNIT0UPDATE]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_UPDATE} to update the current count value into \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEHI]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_\\VALUE\_HI} and \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUELO]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_LO}.
    \item Poll the reading of \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEVALID]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_VALID}, till it's 1, which means user now can read the count value from \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEHI]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_HI} and \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUELO]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_LO}.
    \item Read the lower 32 bits and higher 20 bits from \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUELO]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_LO} and \hyperref[fielddesc:SYSTIMERTIMERUNIT0VALUEHI]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_VALUE\_HI}. 
\end{enumerate}

\subsection{Configure One-Time Alarm in Target Mode}
\begin{enumerate}
    \item Set \hyperref[fielddesc:SYSTIMERTARGET0TIMERUNITSEL]{SYSTIMER\_TARGET\regindex{x}\_TIMER\_UNIT\_SEL} to select the counter (UNIT\regindex{0} or UNIT\regindex{1}) used for COMP\regindex{x}.
    \item Read current count value, see Section \ref{sec:to-read-system-timer-value}. This value will be used to calculate the alarm value (t) in Step 4.
    \item Clear \hyperref[fielddesc:SYSTIMERTARGET0PERIODMODE]{SYSTIMER\_TARGET\regindex{x}\_PERIOD\_MODE} to enable target mode.
    \item Set an alarm value (t), and fill its lower 32 bits to \hyperref[fielddesc:SYSTIMERTIMERTARGET0LO]{SYSTIMER\_TIMER\_TARGET\regindex{x}\_LO}, and the higher 20 bits to \hyperref[fielddesc:SYSTIMERTIMERTARGET0HI]{SYSTIMER\_TIMER\_TARGET\regindex{x}\_HI}.
    \item Set \hyperref[fielddesc:SYSTIMERTIMERCOMP0LOAD]{SYSTIMER\_TIMER\_COMP\regindex{x}\_LOAD} to synchronize the alarm value (t) to COMP\regindex{x}, i.e. load the alarm value (t) to the COMP\regindex{x}.   
    \item Set \hyperref[fielddesc:SYSTIMERTARGET0WORKEN]{SYSTIMER\_TARGET\regindex{x}\_WORK\_EN} to enable the selected COMP\regindex{x}. COMP\regindex{x} starts comparing the count value with the alarm value (t).
    \item Set \hyperref[fielddesc:SYSTIMERTARGET0INTENA]{SYSTIMER\_TARGET\regindex{x}\_INT\_ENA} to enable timer interrupt. When Unit\regindex{n} counts to the alarm value (t), a SYSTIMER\_TARGET\regindex{x}\_INT interrupt is triggered.
\end{enumerate}

\subsection{Configure Periodic Alarms in Period Mode}
\begin{enumerate}
    \item Set \hyperref[fielddesc:SYSTIMERTARGET0TIMERUNITSEL]{SYSTIMER\_TARGET\regindex{x}\_TIMER\_UNIT\_SEL} to select the counter (UNIT\regindex{0} or UNIT\regindex{1}) used for COMP\regindex{x}.
    \item Set an alarm period ($\delta$t), and fill it to \hyperref[fielddesc:SYSTIMERTARGET0PERIOD]{SYSTIMER\_TARGET\regindex{x}\_PERIOD}.
    \item Set \hyperref[fielddesc:SYSTIMERTIMERCOMP0LOAD]{SYSTIMER\_TIMER\_COMP\regindex{x}\_LOAD} to synchronize the alarm period ($\delta$t) to COMP\regindex{x}, i.e. load the alarm period ($\delta$t) to COMP\regindex{x}.
    \item Clear and then set \hyperref[fielddesc:SYSTIMERTARGET0PERIODMODE]{SYSTIMER\_TARGET\regindex{x}\_PERIOD\_MODE} to configure COMP\regindex{x} into period mode.
    \item Set \hyperref[fielddesc:SYSTIMERTARGET0WORKEN]{SYSTIMER\_TARGET\regindex{x}\_WORK\_EN} to enable the selected COMP\regindex{x}. COMP\regindex{x} starts comparing the count value with the sum of start value + n*$\delta$t (n = 1, 2, 3...).
    \item Set \hyperref[fielddesc:SYSTIMERTARGET0INTENA]{SYSTIMER\_TARGET\regindex{x}\_INT\_ENA} to enable timer interrupt. A SYSTIMER\_TARGET\regindex{x}\_INT interrupt is triggered when Unit\regindex{n} counts to start value + n*$\delta$t (n = 1, 2, 3...) set in Step 2.
\end{enumerate}

\subsection{Update After Light-sleep}
\begin{enumerate}
    \item Configure the RTC timer before the chip goes to Light-sleep, to record the exact sleep time. For more information, see Chapter \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}.
    \item Read the sleep time from the RTC timer when the chip is woken up from Light-sleep. 
    \item Read current count value of system timer, see Section \ref{sec:to-read-system-timer-value}.
    \item Convert the time value recorded by the RTC timer from the clock cycles based on RTC\_SLOW\_CLK to that based on CNT\_CLK. For example, if the frequency of RTC\_SLOW\_CLK is 32 KHz, the recorded RTC timer value should be converted by multiplying by 500. 
    \item Add the converted RTC value to the current count value of the system timer:
    \begin{itemize}
        \item Fill the new value into \hyperref[fielddesc:SYSTIMERTIMERUNIT0LOADLO]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_LOAD\_LO} (low 32 bits) and \hyperref[fielddesc:SYSTIMERTIMERUNIT0LOADHI]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_LOAD\_HI} (high 20 bits).
        \item Set \hyperref[fielddesc:SYSTIMERTIMERUNIT0LOAD]{SYSTIMER\_TIMER\_UNIT\regindex{n}\_LOAD} to load new timer value into system timer. In such a way, the system timer is updated.    \end{itemize}
\end{enumerate}
