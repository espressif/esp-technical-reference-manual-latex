\chapter{系统和存储器}
\label{mod:sysmem}

\section{概述}

\chipname{} 是一个超低功耗和高度集成的片上系统，它集成了一颗 RISC-V 32 位单核处理器，四级流水线架构，主频高达 120 MHz。所有的内部存储器、外部存储器以及外设都分布在 CPU 的总线上。

\section{主要特性}

\chipname{} 的系统与存储器具有如下特性：

\begin{itemize}
    \item {\bfseries 地址空间}
        \begin{itemize}
    	    \item 848 KB 内部存储器指令地址空间
    	    \item 576 KB 内部存储器数据地址空间
    	    \item 828 KB 外设地址空间
    	    \item 4 MB 外部存储器指令虚地址空间
    	    \item 4 MB 外部存储器数据虚地址空间
    	    \item 576 KB 内部 DMA 地址空间
        \end{itemize}
    \item {\bfseries 内部存储器}
        \begin{itemize}
            \item 576 KB 内部 ROM
	        \item 272 KB 内部 SRAM
	    \end{itemize}
    \item {\bfseries 外部存储器}
        \begin{itemize}
            \item 最大支持 16 MB 片外 flash
	    \end{itemize}
    \item {\bfseries 外设空间}
        \begin{itemize}
           \item 总计 22 个模块/外设
        \end{itemize}
    \item {\bfseries GDMA}
        \begin{itemize}
            \item 2 个具有 GDMA 功能的模块/外设
        \end{itemize}
\end{itemize}

图 \ref{fig:sysmem-address-mapping-structure} 描述了系统结构与地址映射结构。

\section{功能描述}

\subsection{地址映射}

地址 0x4000\_0000 以下的部分属于数据总线的地址范围，
地址 0x4000\_0000 \verb+~+ 0x4FFF\_FFFF 部分为指令总线的地址范围，
地址 0x5000\_0000 及以上的部分是外设总线的地址范围。

CPU 的数据总线与指令总线都为小端序。
CPU 可以通过数据总线进行单字节、双字节、4 字节的数据访问。
CPU 也可以通过指令总线进行数据访问，但只能是 4 字节对齐的访问。

CPU 能够：
\begin{itemize}
\item 通过数据总线与指令总线直接访问内部存储器；
\item 通过 cache 访问映射到虚地址空间的外部存储器；
\item 通过外设总线直接访问模块/外设。
\end{itemize}

图 \ref{fig:sysmem-address-mapping-structure}    描述了数据总线、指令总线与外设总线中的各段地址所能访问的目标。

系统中部分内部存储器与部分外部存储器既可以被数据总线访问也可以被指令总线访问，
这种情况下，CPU 可以通过多个地址访问到同一目标。


\subsection{内部存储器}
\label{para:sysmem-internal-memory}

\chipname{} 的内部存储器包含如下两种类型：

\begin{itemize}
    \item 内部 ROM (576 KB)：内部 ROM 是只读存储器，不可编程。其中存放有一些系统底层软件的 ROM 代码（软件指令和一些只读数据）。
    \item 内部 SRAM (272 KB)：内部静态存储器（SRAM）是易失性存储器，可以快速响应 CPU 的访问请求（通常一个 CPU 时钟周期）。
    \begin{itemize}
        \item SRAM 中的一部分可以被配置成外部存储器访问的缓存。
        \item SRAM 中的某些部分只可以被 CPU 的指令总线访问。
        \item SRAM 中的某些部分既可以被 CPU 的指令总线访问，又可以被 CPU 的数据总线访问。
    \end{itemize}
\end{itemize}

基于上述对两种类型的内部存储器的描述，\chipname{} 的内部存储器可以被分为两个部分：内部 ROM (576 KB)、内部 SRAM (272 KB)。


CPU 通过不同的总线访问这几部分内部存储器时会有些许限制（如某些部分只允许 CPU 通过数据总线访问），据此内部存储器可以被区分的更加细致。表 \ref{tab:sysmem-internal-memory}  列出了所有内部存储器以及可以访问内部存储器的数据总线与指令总线地址段。

\input{\modulefiles/tables/03-SYSMEM-internal-memory__CN}

{\bfseries 1. 内部 ROM 0}

内部 ROM 0 的容量为 256 KB，只读。如表 \ref{tab:sysmem-internal-memory} 所示，CPU 只可以通过指令总线地址段 0x4000\_0000 \verb+~+ 0x4003\_FFFF 访问这部分存储器。

{\bfseries 2. 内部 ROM 1}

内部 ROM 1 的容量为 320 KB，只读。如表 \ref{tab:sysmem-internal-memory} 所示，CPU 可以通过指令总线地址段 0x4004\_0000 \verb+~+ 0x4008\_FFFF 或数据总线地址段 0x3FF0\_0000 \verb+~+ 0x3FF4\_FFFF 同序访问这部分存储器。

这两段地址同序访问内部 ROM 1 是指： 地址 04004\_0000 与 0x3FF0\_0000 访问到相同的字，0x4004\_0004 与 0x3FF0\_0004 访问到相同的字，0x4004\_0008 与 0x3FF0\_0008 访问到相同的字，以此类推（下文的“同序访问”也参照此描述）。

{\bfseries 3. 内部 SRAM 0}
\label{other:sysmem-internal-sram-0}

内部 SRAM 0 的容量为 16 KB，可读可写。如表 \ref{tab:sysmem-internal-memory} 所示，CPU 只可以通过指令总线访问这部分存储器。

这部分存储器可以被配置为指令缓存，用来缓存外部存储器的指令或只读数据。此时，已被配置为指令缓存的部分不可以被 CPU 访问。



{\bfseries 4. 内部 SRAM 1}

内部 SRAM 1 容量为 256 KB，可读可写。如表 \ref{tab:sysmem-internal-memory} 所示，CPU 可以通过数据或指令总线同序访问。




\subsection{外部存储器}
\label{para:sysmem-external-memory}

\chipname{} 支持以 SPI、Dual SPI、Quad SPI、QPI 等接口形式连接片外 flash。
\chipname{} 还支持基于 XTS-AES 算法的硬件手动加密和自动解密功能，从而保护开发者片外 flash 中的程序和数据。


\subsubsection{外部存储器地址映射}

CPU 借助缓存（Cache）来访问外部存储器。Cache 将根据内存管理单元 (Memory Management Unit, MMU) 中的信息把 CPU 的地址映射为访问片外存储的实地址。经过地址映射，\chipname{} 最大支持 16 MB 的片外 flash。

通过高速缓存，\chipname{} 可支持以下地址空间映射。请注意，指令总线地址空间（4 MB）和数据总线地址空间 (4 MB) 是共用的。

\begin{itemize}
    \item 4 MB 的指令总线地址空间以 64 KB 为单位映射到片外 flash。
    \item 4 MB 的数据总线（只读）地址空间以 64 KB 为单位映射到片外 flash。
\end{itemize}

表 \ref{tab:sysmem-external-memory} 列出了在访问外部存储器时 CPU 的数据总线与指令总线与 Cache 的对应关系。

\subsubsection{高速缓存}

如图 \ref{fig:sysmem-cache-arch} 所示， \chipname{} 采用一个只读的统一 cache，为四路组相联，容量为 16 KB，块大小为 32 字节。当 cache 处于工作状态时，将占用部分内部存储空间（参见第 \ref{other:sysmem-internal-sram-0} 节关于内部 SRAM 0 的描述）。

指令总线和数据总线可以同时访问该 cache，但此时 cache 只能对其中一个作出相应。当 cache 缺失时，cache 控制器会向外部存储器发起请求。

\subsubsection{Cache 操作}

\chipname{} cache 支持如下操作：

\begin{enumerate}
\item \textbf{失效 (Invalidate)}：该操作用于删除 cache 中的有效数据。该操作完成后，删除的数据将仅存于外部存储器中。如果 CPU 接着去访问该数据，那么需要访问外部存储器。该操作包括两种类型：自动失效 (Auto-Invalidate) 和 手动失效 (Manual-Invalidate)。手动失效仅对 cache 中落入指定区域的地址对应的数据做失效处理，而自动失效会对 cache 中的所有数据做失效处理。

\end{enumerate}

\subsection{GDMA 地址空间}

\chipname{} 中的 GDMA (General Direct Memory Access) 外设可提供 直接内存访问（Direct Memory Access, DMA)服务，包括：

\begin{itemize}
    \item 内部存储器中不同位置的数据搬运；
    \item 模块/外设和内部存储器之间的数据搬运。
\end{itemize}

GDMA 可以通过与数据总线完全相同的地址读写内部 SRAM 1，即 GDMA 通过地址 0x3FCA\_0000 \verb+~+ 0x3FCD\_FFFF 访问内部 SRAM 1。请注意，GDMA 无法访问被 cache 占用的内部存储器。


\chipname{} 中共有 2 个外设/模块可以和 GDMA 联合工作，分别是 SPI2 和 SHA Accelerator ，两个外设共用GDMA的一个通道，不可以同时开启 GDMA 功能。



具有 GDMA 功能的模块/外设通过 GDMA 可以访问任何 GDMA 可以访问到的存储器。
更多关于 GDMA 的信息，请参考章节 \ref{mod:dma} \textit{\nameref{mod:dma}}。



\subsection{模块/外设}

CPU 可通过外设总线的地址段 0x6000\_0000 \verb+~+ 0x600C\_EFFF 访问模块/外设。
