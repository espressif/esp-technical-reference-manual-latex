\chapter{权限管理 (PMS)}
\label{mod:permctrl}
\hypertarget{permctrl}{}


%%%%%%%%%%%%%%%%
%%% Overview %%%
%%%%%%%%%%%%%%%%


\section{概述}

\chipname{} 芯片内置了一个权限控制模块 (Permission Controller, PMS)， 可以将硬件资源（内存和外设）分配给不同环境，从而实现特权环境和非特权环境的分离。

\begin{itemize}
    \item 特权环境：
    \begin{itemize}
        \item 可以访问所有的外设和存储空间。
        \item 主要执行所有需要保密的操作，如用户认证、安全通信和数据加解密等。
    \end{itemize}
    \item 非特权环境：
    \begin{itemize}
        \item 只能访问部分的外设和存储空间。
        \item 执行其他操作，如用户操作系统、各种应用程序等。
    \end{itemize}
\end{itemize}


此外，\chipname{} 芯片的 RISC-V CPU 内核还包含一个物理存储器保护 (PMP) 单元，可以供软件设置存储器访问特权（读、写、执行权限），但存在以下限制：
    \begin{itemize}
        \item 最多可配置 16 个区域，在某些场景下无法完全支持 \chipname{} 丰富的外设和不同类型的存储器。
        \item 仅能够限制 CPU 的访问，无法管控 GDMA 的访问。
    \end{itemize}

为了解决这些问题，\chipname{} 专门增加了权限管理模块 (PMS) ，作为对 CPU 物理存储器保护 (PMP) 单元的补充，为用户提供完善的权限管理选项。

芯片在进行权限管理时的主要流程如下（可参考图 \ref{fig:pms_overview}）：
\begin{enumerate}
    \item 检查 PMP 权限
    \begin{itemize}
        \item 如通过，则继续检查 PMS 权限
        \item 如不通过，则直接触发访问错误异常，不继续检查 PMS 权限
    \end{itemize}
    \item 检查 PMS 权限
    \begin{itemize}
        \item 如通过，则允许访问
        \item 如不通过，则触发中断
    \end{itemize}
\end{enumerate}

\begin{figure}[H]
        \centering
        \includegraphics[width=0.5\textwidth]{./08-PERMCTRL/figures/c3_pms_pmp_new.pdf}
        \caption{Permission Control Overview}
        \label{fig:pms_overview}
\end{figure}

有关 PMP 物理存储器保护单元的介绍，请前往章节 \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}} 中的第 \ref{sec:riscvcpu-pmp-overview} 小节。有关世界控制器的介绍，请前往章节 \ref{mod:wctrl} \textit{\nameref{mod:wctrl}}。本章节主要介绍 \chipname{} 的 PMS 权限管理模块。


\section{主要特性}\label{sec:}

\chipname{} 的权限控制模块具有以下特性：

\begin{itemize}
    \item 支持特权环境和非特权环境下的独立权限管理
    \item 支持片内存储器的权限管理，包括：
    \begin{itemize}
        \item CPU 对片内存储器的访问权限控制
        \item GDMA 对片内存储器的访问权限控制
    \end{itemize}
    \item 支持片外存储器的权限管理
    \begin{itemize}
        \item CPU 通过 SPI1 访问外部存储器的权限控制
        \item CPU 通过 Cache 访问外部存储器的权限控制
    \end{itemize}
    \item 支持外设空间的权限管理
    \begin{itemize}
        \item 各外设空间均支持独立的权限控制
        \item 支持非对齐访问的监测
    \end{itemize}
    \item 支持自定义地址段权限管理
    \item 内置权限寄存器锁保护机制
    \item 内置权限监测中断机制
\end{itemize}

\section{特权环境与非特权环境}\label{sec:permctrl-privileged-mode}

\chipname{} 芯片在检查 PMS 权限时：
\begin{itemize}
    \item 当芯片处于特权环境时，则检查特权环境对应的权限寄存器
    \item 当芯片处于非特权环境时，则检查非特权环境对应的权限寄存器
\end{itemize}

用户使芯片进入特权环境有两种方法：

\begin{itemize}
    \item 配置世界控制器的世界切换
    \begin{itemize}
        \item 切换至安全世界：即进入特权环境
        \item 切换至非安全世界：即退出特权环境
    \end{itemize}
    \item 配置 CPU 的特权模式
    \begin{itemize}
        \item 切换至机器模式：即进入特权环境
        \item 切换至用户模式：即退出特权环境
    \end{itemize}
\end{itemize}

具体通过配置 \hyperref[fielddesc:PMSPRIVILEGEMODESEL]{PMS\_PRIVILEGE\_MODE\_SEL} 进行选择：
\begin{itemize}
    \item 0（默认）：选择通过配置世界控制器，具体步骤详见 \ref{mod:wctrl} \textit{\nameref{mod:wctrl}}
    \item 1：选择通过配置 CPU 的特权模式，具体步骤详见 \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}
\end{itemize}

下面将详细介绍如何配置芯片在不同特权模式下对不同区域的权限。


\section{片内存储器的权限管理}

\chipname{} 片内存储器的主要包括：
\begin{itemize}
    \item ROM：共 384 KB，分为 256 KB Internal ROM0 和 128 KB Internal ROM1。
    \item SRAM：共 400 KB，分为 16 KB Internal SRAM0 和 384 KB Internal SRAM1。
    \item RTC 快速内存：共 8 KB，可由分割线分为高、低两个片区，各片区可独立配置权限。
\end{itemize}

本节将依次介绍 \chipname{} 中有关片内存储资源访问权限管理。

\subsubsection{Internal SRAM1 权限配置 }\label{sec:permctrl_sram1}

\chipname{} 的 Internal SRAM1 分为 Block0 \verb+~+ Block2（详见表 \ref{tab:permctrl-SRAM_block_address}）：
\begin{itemize}
    \item CPU 的数据总线 DBUS、指令总线 IBUS 和 GDMA 均可访问，且允许同时进行访问。
    \item 支持自定义地址段，可实现灵活的权限控制。
\end{itemize}
\chipname{} 允许用户通过 5 条分割线将 SRAM1 分割为 6 个自定义地址段，每个区域可以分别配置权限。

具体来说，Internal SRAM1 可由 IRam0\_DRam0\_split\_line 划分为指令空间和数据空间。
\begin{itemize}
    \item 指令空间 (Instruction Region)：
    \begin{itemize}
    \item 指令空间应配置为仅可由 IBUS 访问
    \item 且可以由 IRam0\_split\_line\_0 和 IRam0\_split\_line\_1 进一步划分为 3 个分割区域
    \end{itemize}
    \item 数据空间 (Data Region)：
    \begin{itemize}
        \item 数据空间应配置为仅可由 DBUS 访问
        \item 且可由 DRam0\_split\_line\_0 和 DRam0\_split\_line\_1 进一步划分为 3 个分割区域
    \end{itemize}
\end{itemize}

\subparagraph{划分区域}\label{sec:permctrl-split-line}

\chipname{} 允许用户自行配置分割线的位置，每条分割线均有自己对应的配置寄存器。
\begin{itemize}
    \item 指令和数据空间分割线 IRam0\_DRam0\_split\_line 配置寄存器：
    \begin{itemize}
        \item \hyperref[regdesc:PMSCOREXIRAM0DRAM0DMASPLITLINECONSTRAIN1REG]{PMS\_CORE\_X\_IRAM0\_DRAM0\_DMA\_SPLIT\_LINE\_CONSTRAIN\_1\_REG}
    \end{itemize}
    \item 指令空间内部分割线 IRam0\_split\_line\_0  配置寄存器：
    \begin{itemize}
        \item \hyperref[regdesc:PMSCOREXIRAM0DRAM0DMASPLITLINECONSTRAIN2REG]{PMS\_CORE\_X\_IRAM0\_DRAM0\_DMA\_SPLIT\_LINE\_CONSTRAIN\_2\_REG}
    \end{itemize}
    \item 指令空间内部分割线 IRam0\_split\_line\_1  配置寄存器：
    \begin{itemize}
        \item \hyperref[regdesc:PMSCOREXIRAM0DRAM0DMASPLITLINECONSTRAIN3REG]{PMS\_CORE\_X\_IRAM0\_DRAM0\_DMA\_SPLIT\_LINE\_CONSTRAIN\_3\_REG}
    \end{itemize}
    \item 数据空间内部分割线 DRam0\_split\_line\_0  配置寄存器：
    \begin{itemize}
        \item \hyperref[regdesc:PMSCOREXIRAM0DRAM0DMASPLITLINECONSTRAIN4REG]{PMS\_CORE\_X\_IRAM0\_DRAM0\_DMA\_SPLIT\_LINE\_CONSTRAIN\_4\_REG}
    \end{itemize}
    \item 数据空间内部分割线 DRam0\_split\_line\_1  配置寄存器：
    \begin{itemize}
        \item \hyperref[regdesc:PMSCOREXIRAM0DRAM0DMASPLITLINECONSTRAIN5REG]{PMS\_CORE\_X\_IRAM0\_DRAM0\_DMA\_SPLIT\_LINE\_CONSTRAIN\_5\_REG}
    \end{itemize}
\end{itemize}

\begin{figure}[H]
        \centering
        \includegraphics[width=0.5\textwidth]{./08-PERMCTRL/figures/split_line_category.eps}
        \caption{分割线 Category 分配示意图}
        \label{fig:PERMCTRL_category_config}
\end{figure}
每条分割线在配置时应：
\begin{enumerate}
    \item 首先配置分割线所处的 Block：
    \begin{itemize}
        \item 将分割线所处 Block 对应的 Category\_\regindex{x} 域配置为 0x1 或者 0x2（没有区别）；
        \item 之前所有 Block 对应的 Category\_0 \verb+~+ Category\_\regindex{x-1} 域全被配置为 0x0；
        \item 之后所有 Block 对应的 Category\_\regindex{x+1} \verb+~+ Category\_2 域全部配置为 0x3。
    \end{itemize}
        举例而言，假设分割线位于 Block1，则应将 Block1 对应的 Category\_1 域配置为 0x1 或者 0x2；将 Block0 对应的 Category\_0 域全部配置为 0x0；将 Block2 对应的 Category\_2 域全部配置为 0x3。可参考图 \ref{fig:PERMCTRL_category_config}。反过来，查询到 Category\_1 域为 0x1 或者 0x2 也可以了解分割线处于 Block1。
    \item 配置分割线在所处 Block 中的位置：
    \begin{itemize}
        \item 将需要分割的 IBUS 或 DBUS 访问地址的 [16:9] 位写入分割线对应配置寄存器的 \hyperref[fielddesc:PMSCOREXIRAM0SRAMLINE0SPLITADDR]{SPLITADDR} 域中。
        \item 注意分割地址必须配置为与 512 Byte 对齐，也就是说必须是 0x200 的整数倍。
    \end{itemize}
        举例而言，假设需要分割的 DBUS 地址为 0x3fc88000，则应将该地址的 [16:9] 位，即 0b01000000 写入该分割线对应配置寄存器的 \hyperref[fielddesc:PMSCOREXIRAM0SRAMLINE0SPLITADDR]{SPLITADDR} 域。
        \item 分割线可基于 IBUS 地址段或者 DBUS 地址段，其地址是等价的。比如 DBUS 地址 0x3fc88000 和 IBUS 地址 0x40388000 处于相同位置，其 [16:9] 都是 0b01000000。
\end{enumerate}




值得注意的是，用户在配置 5 条分割线时还应遵循以下原则：

 \begin{itemize}
     \item 位置要求
     \begin{itemize}
         \item 指令和数据空间分割线可以配置为 SRAM1 的任何位置
         \item 指令空间中的 2 条内部分割线不能超出指令空间，相对位置无要求。
         \item 数据空间中的 2 条内部分割线不能超出数据空间，相对位置无要求。
     \end{itemize}
     \item 各分割线可以相互重合，相当于不存在。举例而言：
     \begin{itemize}
         \item 数据空间中的 2 条内部分隔线如果不重合，则相当于将数据空间分为 3 个细分区域；
         \item 数据空间中的 2 条内部分隔线如果重合，则相当于仅将数据空间分为 2 个细分区域；
         \item 数据空间中的 2 条内部分隔线如果本身 两两重合，且与指令和数据空间分割线重合，则相当于对数据空间内部不做细分，仅有一个数据空间区域。
     \end{itemize}
     \end{itemize}

\subsection{自定义地址段权限管理}

\begin{itemize}
    \item 针对每个接收通道，共 3 套；

    \item 共用配置寄存器，共 1 套。
\end{itemize}

\begin{itemize}

    \begin{itemize}

        \item \hyperref[regdesc:PMSREGIONPMSCONSTRAIN3REG]{PMS\_REGION\_PMS\_CONSTRAN\_\regindex{n}\_REG}（\regindex{n} = 3 \verb+~+ 10），用于配置每个地址段的起始地址。注意，每个地址段的起始地址，也是上一个地址段的结束地址。

\section{非法访问与中断}

当访问类型与配置允许访问的类型不一致时，\chipname{} 将视其为\textbf{非法访问}，并进行如下处理：

\begin{itemize}
    \item 过滤该访问请求，并返回默认值，具体表现为：
    \begin{itemize}
        \item 执行和读取操作返回 0（非法访问片内存储器和外设存储空间）或 0xdeadbeaf （非法访问片外存储器的默认值）
        \item 写入操作无效
    \end{itemize}
    \item 触发相应中断（如相应中断已使能），详见下文介绍。
\end{itemize}

本章节介绍的所有权限管理中断仅记录第一次发生中断的信息，因此建议用户检查到该中断信号后及时处理，并在处理后及时置位中断清除信号，从而保证下一次中断的顺利记录。
