\chapter{Timer Group (TIMG)}
\label{mod:timg}



\section{Overview}
General purpose timers can be used to precisely time an interval, trigger an interrupt after a particular interval (periodically and aperiodically), or act as a hardware clock. As shown in Figure \ref{fig:timg}, the ESP32-C3 chip contains two timer groups, namely timer group 0 and timer group 1. Each timer group consists of one general purpose timer referred to as T0 and one Main System Watchdog Timer. All general purpose timers are based on 16-bit prescalers and 54-bit auto-reload-capable up-down counters.

\begin{figure}[h! ]
    \centering
    \includegraphics[width=0.3\textwidth]{\modulefiles/figures/timergroup_arch}
    \caption{Timer Units within Groups}
    \label{fig:timg}
\end{figure}

Note that while the Main System Watchdog Timer registers are described in this chapter, their functional description is included in the Chapter \ref{mod:wdt} \textit{\nameref{mod:wdt}}. Therefore, the term `timers' within this chapter refers
to the general purpose timers.

The timers' features are summarized as follows:
\begin{itemize}
    \item A 16-bit clock prescaler, from 2 to 65536
    \item A 54-bit time-base counter programmable to incrementing or decrementing
    \item Able to read real-time value of the time-base counter  
    \item Halting and resuming the time-base counter
    \item Programmable alarm generation    
    \item Timer value reload (Auto-reload at alarm or software-controlled instant reload)
    \item Level interrupt generation
\end{itemize}

\section{Functional Description}


\begin{figure}[h! ]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/726timergroup_architecture}}
    \caption{Timer Group Architecture}
    \label{fig:timg-architecture}

Figure\ref{fig:timg-architecture} is a diagram of timer T0 in a timer group. T0 contains a clock selector, a 16-bit integer divider as a prescaler, a timer-based counter and a comparator for alarm generation.

\subsection{16-bit Prescaler and Clock Selection}

The timer can select between the APB clock (APB\_CLK) or external clock (XTAL\_CLK) as its clock source by setting the \hyperref[fielddesc:TIMGTXUSEXTAL]{TIMG\_T0\_USE\_XTAL} field of the \hyperref[regdesc:TIMGTXCONFIGREG]{TIMG\_T0CONFIG\_REG} register. The selected clock is switched on by setting \hyperref[fielddesc:TIMGTIMERCLKISACTIVE]{TIMG\_TIMER\_CLK\_IS\_ACTIVE} field of the \hyperref[regdesc:TIMGREGCLKREG]{TIMG\_REGCLK\_REG} register to 1 and switched off by setting it to 0. The clock is then divided by a 16-bit prescaler to generate the time-base counter clock (TB\_CLK) used by the time-base counter. When the \hyperref[fielddesc:TIMGTXDIVIDER]{TIMG\_T0\_DIVIDER} field 
is configured as 2 \verb+~+ 65536, the divisor of the prescaler would be 2 \verb+~+ 65536. Note that programming value 0 to \hyperref[fielddesc:TIMGTXDIVIDER]{TIMG\_T0\_DIVIDER} will result in the divisor being 65536. When the \hyperref[fielddesc:TIMGTXDIVIDER]{TIMG\_T0\_DIVIDER} is set to 1, the actual divisor is 2 so the timer counter value represents the half of real time.

To modify the 16-bit prescaler, please first configure the \hyperref[fielddesc:TIMGTXDIVIDER]{TIMG\_T0\_DIVIDER} field, and then set \hyperref[fielddesc:TIMGTXDIVIDERRST]{TIMG\_T0\_DIVIDER\_RST} to 1. Meanwhile, the timer must be disabled (i.e. \hyperref[fielddesc:TIMGTXEN]{TIMG\_T0\_EN} should be cleared). Otherwise, the result can be unpredictable.

\subsection{54-bit Time-base Counter}
The 54-bit time-base counters are based on TB\_CLK and can be configured to increment or decrement via the \hyperref[fielddesc:TIMGTXINCREASE]{TIMG\_T0\_INCREASE} field. The time-base counter can be enabled or disabled by setting or clearing the \hyperref[fielddesc:TIMGTXEN]{TIMG\_T0\_EN} field, respectively. When enabled, the time-base counter increments or decrements on each cycle of TB\_CLK. When disabled, the time-base counter is essentially frozen. Note that the \hyperref[fielddesc:TIMGTXINCREASE]{TIMG\_T0\_INCREASE} field can be changed while \hyperref[fielddesc:TIMGTXEN]{TIMG\_T0\_EN} is set and this will cause the time-base counter to change direction instantly.

To read the 54-bit value of the time-base counter, the timer value must be latched to two registers before being read by the CPU (due to the CPU being 32-bit). By writing any value to the \hyperref[regdesc:TIMGTXUPDATEREG]{TIMG\_T0UPDATE\_REG}, the current value of the 54-bit timer starts to be latched into the \hyperref[regdesc:TIMGTXLOREG]{TIMG\_T0LO\_REG} and \hyperref[regdesc:TIMGTXHIREG]{TIMG\_T0HI\_REG} registers containing the lower 32-bits and 
higher 22-bits, respectively. When \hyperref[regdesc:TIMGTXUPDATEREG]{TIMG\_T0UPDATE\_REG} is cleared by hardware, it indicates the latch operation has been completed and current timer value can be read from the \hyperref[regdesc:TIMGTXLOREG]{TIMG\_T0LO\_REG} and \hyperref[regdesc:TIMGTXHIREG]{TIMG\_T0HI\_REG} registers. \hyperref[regdesc:TIMGTXLOREG]{TIMG\_T0LO\_REG} and \hyperref[regdesc:TIMGTXHIREG]{TIMG\_T0HI\_REG} registers will remain unchanged for the CPU to read in its own time until \hyperref[regdesc:TIMGTXUPDATEREG]{TIMG\_T0UPDATE\_REG} is written to again.

\subsection{Alarm Generation}
A timer can be configured to trigger an alarm when the timer's current value matches the alarm value. An alarm will cause an interrupt to occur and (optionally) an automatic reload of the timer's current value (see Section \ref{sec:timg-func-descr-reload}). 

The 54-bit alarm value is configured using \hyperref[regdesc:TIMGTXALARMLOREG]{TIMG\_T0ALARMLO\_REG} and \hyperref[regdesc:TIMGTXALARMHIREG]{TIMG\_T0ALARMHI\_REG}, which 
represent the lower 32-bits and higher 22-bits of the alarm value, respectively. However, the configured alarm value is ineffective until the alarm is enabled by setting the \hyperref[fielddesc:TIMGTXALARMEN]{TIMG\_T0\_ALARM\_EN} field. To avoid alarm being enabled `too late' (i.e. the timer value has already passed the alarm value when the alarm is enabled), the hardware will trigger the alarm immediately if the current timer value is higher than the alarm value (within a defined range) when the up-down counter increments, or lower than the alarm value (within a defined range) when the up-down counter decrements. Table \ref{tab:timg-alarm-generation-up} and Table \ref{tab:timg-alarm-generation-down} show the relationship between the current value of the timer, the alarm value, and when an alarm is triggered.The current time value and the alarm value are defined as follows:

\begin{itemize}
    \item TIMG\_VALUE = \{\hyperref[regdesc:TIMGTXHIREG]{TIMG\_T0HI\_REG}, \hyperref[regdesc:TIMGTXLOREG]{TIMG\_T0LO\_REG}\}
    \item ALARM\_VALUE = \{\hyperref[regdesc:TIMGTXALARMHIREG]{TIMG\_T0ALARMHI\_REG}, \hyperref[regdesc:TIMGTXALARMLOREG]{TIMG\_T0ALARMLO\_REG}\}
\end{itemize}

When an alarm occurs, the \hyperref[fielddesc:TIMGTXALARMEN]{TIMG\_T0\_ALARM\_EN} field is automatically cleared and no alarm will occur again until the \hyperref[fielddesc:TIMGTXALARMEN]{TIMG\_T0\_ALARM\_EN} is set next time.

\subsection{Timer Reload} \label{sec:timg-func-descr-reload}
A timer is reloaded when a timer's current value is overwritten with a reload value stored in the \hyperref[fielddesc:TIMGTXLOADLO]{TIMG\_T0\_LOAD\_LO} and \hyperref[fielddesc:TIMGTXLOADHI]{TIMG\_T0\_LOAD\_HI} fields that correspond to the lower 32-bits and higher 22-bits of the timer's new value, respectively. However, writing a reload value to \hyperref[fielddesc:TIMGTXLOADLO]{TIMG\_T0\_LOAD\_LO} and \hyperref[fielddesc:TIMGTXLOADHI]{TIMG\_T0\_LOAD\_HI} will not cause the timer's current value to change. Instead, the reload value is ignored by the timer until a reload event occurs. A reload event can be triggered either by a software instant reload or an auto-reload at alarm.

A software instant reload is triggered by the CPU writing any value to \hyperref[regdesc:TIMGTXLOADREG]{TIMG\_T0LOAD\_REG}, which causes the timer's current value to be instantly reloaded. If \hyperref[fielddesc:TIMGTXEN]{TIMG\_T0\_EN} is set, the timer will continue incrementing or decrementing from the new value. If \hyperref[fielddesc:TIMGTXEN]{TIMG\_T0\_EN} is cleared, the timer will remain frozen at the new value until counting is re-enabled. 

An auto-reload at alarm will cause a timer reload when an alarm occurs, thus allowing the timer to continue incrementing or decrementing from the reload value. This is generally useful for resetting the timer's value when using periodic alarms. To enable auto-reload at alarm, the \hyperref[fielddesc:TIMGTXAUTORELOAD]{TIMG\_T0\_AUTORELOAD} field should be set. If not enabled, the timer's value will continue to increment or decrement past the alarm value after an alarm.

\subsection{RTC\_SLOW\_CLK Frequency Calculation}

Via XTAL\_CLK, a timer could calculate the frequency of clock sources for RTC\_SLOW\_CLK (i.e. RC\_RTC\_SLOW\_CLK, RC\_FAST\_DIV\_CLK, and XTAL32K\_CLK) as follows:

\begin{enumerate}
    \item Start periodic or one-shot frequency calculation;
    \item Once receiving the signal to start calculation, the counter of XTAL\_CLK and the counter of RTC\_SLOW\_CLK begin to work at the same time. When the counter of RTC\_SLOW\_CLK counts to C0, the two counters stop counting simultaneously;
    \item Assume the value of XTAL\_CLK's counter is C1, and the frequency of RTC\_SLOW\_CLK would be calculated as:
    $f\_rtc = \frac{C0 \times f\_XTAL\_CLK}{C1}$
\end{enumerate}
   
\subsection{Interrupts}\label{sec:timg-interrupt}

Each timer has its own interrupt line that can be routed to the CPU, and thus each timer group has a total of two interrupt lines. Timers generate level interrupts that must be explicitly cleared by the CPU on each triggering.

Interrupts are triggered after an alarm (or stage timeout for watchdog timers) occurs. Level interrupts will be held high after an alarm (or stage timeout) occurs, and will remain so until manually cleared. To enable a timer's interrupt, the \hyperref[fielddesc:TIMGT0INTENA]{TIMG\_T0\_INT\_ENA} bit should be set.

The interrupts of each timer group are governed by a set of registers. Each timer within the group has a corresponding bit in each of these registers:

\begin{itemize}

\item \hyperref[fielddesc:TIMGT0INTRAW]{TIMG\_T0\_INT\_RAW} :
An alarm event sets it to 1. The bit will remain set until the timer's corresponding bit in \hyperref[fielddesc:TIMGT0INTCLR]{TIMG\_T0\_INT\_CLR} is written.
\item \hyperref[fielddesc:TIMGWDTINTRAW]{TIMG\_WDT\_INT\_RAW} : A stage time out will set the timer's bit to 1. The bit will remain set until the timer's corresponding bit in \hyperref[fielddesc:TIMGWDTINTCLR]{TIMG\_WDT\_INT\_CLR} is written.
\item \hyperref[fielddesc:TIMGT0INTST]{TIMG\_T0\_INT\_ST} : Reflects the status of each timer's interrupt and is generated by masking the bits of \hyperref[fielddesc:TIMGT0INTRAW]{TIMG\_T0\_INT\_RAW} with \hyperref[fielddesc:TIMGT0INTENA]{TIMG\_T0\_INT\_ENA}. 
\item \hyperref[fielddesc:TIMGWDTINTST]{TIMG\_WDT\_INT\_ST} : Reflects the status of each watchdog timer's interrupt and is generated by masking the bits of \hyperref[fielddesc:TIMGWDTINTRAW]{TIMG\_WDT\_INT\_RAW} with \hyperref[fielddesc:TIMGWDTINTENA]{TIMG\_WDT\_INT\_ENA}. 
\item \hyperref[fielddesc:TIMGT0INTENA]{TIMG\_T0\_INT\_ENA} : Used to enable or mask the interrupt status bits of timers within the group.
\item \hyperref[fielddesc:TIMGWDTINTENA]{TIMG\_WDT\_INT\_ENA} : Used to enable or mask the interrupt status bits of watchdog timer within the group.
\item \hyperref[fielddesc:TIMGT0INTCLR]{TIMG\_T0\_INT\_CLR} : Used to clear a timer's interrupt by setting its corresponding bit to 1. The timer's corresponding bit in \hyperref[fielddesc:TIMGT0INTRAW]{TIMG\_T0\_INT\_RAW} and \hyperref[fielddesc:TIMGT0INTST]{TIMG\_T0\_INT\_ST} will be cleared as a result. Note that a timer's interrupt must be cleared before the next interrupt occurs.
\item \hyperref[fielddesc:TIMGWDTINTCLR]{TIMG\_WDT\_INT\_CLR} : Used to clear a timer's interrupt by setting its corresponding bit to 1. The watchdog timer's corresponding bit in \hyperref[fielddesc:TIMGWDTINTRAW]{TIMG\_WDT\_INT\_RAW} and \hyperref[fielddesc:TIMGWDTINTST]{TIMG\_WDT\_INT\_ST} will be cleared as a result. Note that a watchdog timer's interrupt must be cleared before the next interrupt occurs.
\end{itemize}

\section{Configuration and Usage}

\subsection{Timer as a Simple Clock}\label{sec:timg-func-descr-simple-clock}
\begin{enumerate}
    \item Configure the time-base counter
    \begin{itemize}
        \item Select clock source by setting or clearing \hyperref[fielddesc:TIMGTXUSEXTAL]{TIMG\_T0\_USE\_XTAL} field.
        \item Configure the 16-bit prescaler by setting \hyperref[fielddesc:TIMGTXDIVIDER]{TIMG\_T0\_DIVIDER}.
        \item Configure the timer direction by setting or clearing \hyperref[fielddesc:TIMGTXINCREASE]{TIMG\_T0\_INCREASE}.
        \item Set the timer's starting value by writing the starting value to \hyperref[fielddesc:TIMGTXLOADLO]{TIMG\_T0\_LOAD\_LO} and \hyperref[fielddesc:TIMGTXLOADHI]{TIMG\_T0\_LOAD\_HI}, then reloading it into the timer by writing any value to \hyperref[regdesc:TIMGTXLOADREG]{TIMG\_T0LOAD\_REG}.
    \end{itemize}
    \item Start the timer by setting \hyperref[fielddesc:TIMGTXEN]{TIMG\_T0\_EN}.
    \item Get the timer's current value.
    \begin{itemize}
        \item  Write any value to \hyperref[regdesc:TIMGTXUPDATEREG]{TIMG\_T0UPDATE\_REG} to latch the timer's current value.
        \item Wait until \hyperref[regdesc:TIMGTXUPDATEREG]{TIMG\_T0UPDATE\_REG} is cleared by hardware.
        \item  Read the latched timer value from \hyperref[regdesc:TIMGTXLOREG]{TIMG\_T0LO\_REG} and \hyperref[regdesc:TIMGTXHIREG]{TIMG\_T0HI\_REG}.   
    \end{itemize}   
\end{enumerate}

\subsection{Timer as One-shot Alarm}\label{sec:timg-func-descr-timer-alarm}
\begin{enumerate}
    \item Configure the time-base counter following step 1 of Section \ref{sec:timg-func-descr-simple-clock}.
    \item Configure the alarm.
    \begin{itemize}
        \item Configure the alarm value by setting \hyperref[regdesc:TIMGTXALARMLOREG]{TIMG\_T0ALARMLO\_REG} and \hyperref[regdesc:TIMGTXALARMHIREG]{TIMG\_T0ALARMHI\_REG}.
        \item Enable interrupt by setting  \hyperref[fielddesc:TIMGT0INTENA]{TIMG\_T0\_INT\_ENA}.
    \end{itemize}
    \item Disable auto reload by clearing \hyperref[fielddesc:TIMGTXAUTORELOAD]{TIMG\_T0\_AUTORELOAD}.
    \item Start the alarm by setting \hyperref[fielddesc:TIMGTXALARMEN]{TIMG\_T0\_ALARM\_EN}.
    \item Handle the alarm interrupt.
    \begin{itemize}
        \item Clear the interrupt by setting the timer's corresponding bit in \hyperref[fielddesc:TIMGT0INTCLR]{TIMG\_T0\_INT\_CLR}.
        \item Disable the timer by clearing \hyperref[fielddesc:TIMGTXEN]{TIMG\_T0\_EN}.
    \end{itemize}
\end{enumerate}

\subsection{Timer as Periodic Alarm}

\begin{enumerate}
    \item Configure the time-base counter following step 1 in Section \ref{sec:timg-func-descr-simple-clock}.
    \item Configure the alarm following step 2 in Section \ref{sec:timg-func-descr-timer-alarm}.
    \item Enable auto reload by setting \hyperref[fielddesc:TIMGTXAUTORELOAD]{TIMG\_T0\_AUTORELOAD} and configure the reload value via \hyperref[fielddesc:TIMGTXLOADLO]{TIMG\_T0\_LOAD\_LO} and \hyperref[fielddesc:TIMGTXLOADHI]{TIMG\_T0\_LOAD\_HI}.
    \item Start the alarm by setting \hyperref[fielddesc:TIMGTXALARMEN]{TIMG\_T0\_ALARM\_EN}.
    \item Handle the alarm interrupt (repeat on each alarm iteration).
    \begin{itemize}
        \item Clear the interrupt by setting the timer's corresponding bit in \hyperref[fielddesc:TIMGT0INTCLR]{TIMG\_T0\_INT\_CLR}.
        \item If the next alarm requires a new alarm value and reload value (i.e. different alarm interval per iteration), then \hyperref[regdesc:TIMGTXALARMLOREG]{TIMG\_T0ALARMLO\_REG}, \hyperref[regdesc:TIMGTXALARMHIREG]{TIMG\_T0ALARMHI\_REG}, \hyperref[fielddesc:TIMGTXLOADLO]{TIMG\_T0\_LOAD\_LO}, and \hyperref[fielddesc:TIMGTXLOADHI]{TIMG\_T0\_LOAD\_HI} should be reconfigured as needed. Otherwise, the aforementioned registers should remain unchanged.
        \item Re-enable the alarm by setting \hyperref[fielddesc:TIMGTXALARMEN]{TIMG\_T0\_ALARM\_EN}.
    \end{itemize}
    \item Stop the timer (on final alarm iteration).
    \begin{itemize}
        \item Clear the interrupt by setting the timer's corresponding bit in \hyperref[fielddesc:TIMGT0INTCLR]{TIMG\_T0\_INT\_CLR}.
        \item Disable the timer by clearing \hyperref[fielddesc:TIMGTXEN]{TIMG\_T0\_EN}.  
    \end{itemize}
\end{enumerate}

\subsection{RTC\_SLOW\_CLK Frequency Calculation}
\begin{enumerate}
    \item One-shot frequency calculation
    \begin{itemize}
        \item Select the clock whose frequency is to be calculated (clock source of RTC\_SLOW\_CLK) via \hyperref[fielddesc:TIMGRTCCALICLKSEL]{TIMG\_RTC\_CALI\_CLK\_SEL}, and configure the time of calculation via \hyperref[fielddesc:TIMGRTCCALIMAX]{TIMG\_RTC\_CALI\_MAX}.
        \item Select one-shot frequency calculation by clearing \hyperref[fielddesc:TIMGRTCCALISTARTCYCLING]{TIMG\_RTC\_CALI\_START\_CYCLING}, and enable the two counters via \hyperref[fielddesc:TIMGRTCCALISTART]{TIMG\_RTC\_CALI\_START}.
        \item Once \hyperref[fielddesc:TIMGRTCCALIRDY]{TIMG\_RTC\_CALI\_RDY} becomes 1, read \hyperref[fielddesc:TIMGRTCCALIVALUE]{TIMG\_RTC\_CALI\_VALUE} to get the value of XTAL\_CLK's counter, and calculate the frequency of  RTC\_SLOW\_CLK.
    \end{itemize}
    \item Periodic frequency calculation
    \begin{itemize}
        \item Select the clock whose frequency is to be calculated (clock source of RTC\_SLOW\_CLK) via \hyperref[fielddesc:TIMGRTCCALICLKSEL]{TIMG\_RTC\_CALI\_CLK\_SEL}, and configure the time of calculation via \hyperref[fielddesc:TIMGRTCCALIMAX]{TIMG\_RTC\_CALI\_MAX}.
        \item Select periodic frequency calculation by enabling \hyperref[fielddesc:TIMGRTCCALISTARTCYCLING]{TIMG\_RTC\_CALI\_START\_CYCLING}.
        \item When \hyperref[fielddesc:TIMGRTCCALICYCLINGDATAVLD]{TIMG\_RTC\_CALI\_CYCLING\_DATA\_VLD} is 1, \hyperref[fielddesc:TIMGRTCCALIVALUE]{TIMG\_RTC\_CALI\_VALUE} is valid.
    \end{itemize}
    \item Timeout\\
    If the counter of RTC\_SLOW\_CLK cannot finish counting in \hyperref[fielddesc:TIMGRTCCALITIMEOUTRSTCNT]{TIMG\_RTC\_CALI\_TIMEOUT\_RST\_CNT} cycles,  \\\hyperref[fielddesc:TIMGRTCCALITIMEOUT]{TIMG\_RTC\_CALI\_TIMEOUT} will be set to indicate a timeout.
\end{enumerate}
