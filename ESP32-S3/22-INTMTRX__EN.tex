\hypertarget{intmtrx}{}
\chapter{Interrupt Matrix (INTERRUPT)}
\label{mod:intmtrx}

\section{Overview}

The interrupt matrix embedded in ESP32-S3 independently allocates peripheral interrupt sources to the two CPUs' peripheral interrupts, to timely inform CPU0 or CPU1 to process the interrupts once the interrupt signals are generated.

Peripheral interrupt sources must be routed to CPU0/CPU1 peripheral interrupts via this interrupt matrix due to the following considerations:
\begin{itemize}
    \item ESP32-S3 has 99 peripheral interrupt sources. To map them to 32 CPU0 interrupts or 32 CPU1 interrupts, this matrix is needed.
    \item Through this matrix, one peripheral interrupt source can be mapped to multiple CPU0 interrupts or CPU1 interrupts according to application requirements.
\end{itemize}

\section{Features}

\begin{itemize}
    \item Accept 99 peripheral interrupt sources as input
    \item Generate 26 peripheral interrupts to CPU0 and 26 peripheral interrupts to CPU1 as output. Note that the remaining six CPU0 interrupts and six CPU1 interrupts are internal interrupts.
    \item Support to disable CPU non-maskable interrupt (NMI) sources
    \item Support to query current interrupt status of peripheral interrupt sources
\end{itemize}

Figure \ref{fig:interrupt-matrix-structure} shows the structure of the interrupt matrix.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.9\textwidth]{22-INTMTRX/figures/22-INTMTRX-struct.eps}
    \caption{Interrupt Matrix Structure}
    \label{fig:interrupt-matrix-structure}
\end{figure}
All the interrupts generated by the peripheral interrupt sources can be handled by CPU0 or CPU1. Users can configure \hyperref[CPU0RegisterSummary]{CPU0 interrupt registers} (``Core0 Interrupt Reg" module in Figure \ref{fig:interrupt-matrix-structure}) to allocate peripheral interrupt sources to CPU0, or configure \hyperref[CPU1RegisterSummary]{CPU1 interrupt registers} (``Core1 Interrupt Reg" module in Figure \ref{fig:interrupt-matrix-structure}) to allocate peripheral interrupt sources to CPU1. Peripheral interrupt sources can be allocated both to CPU0 and CPU1 simultaneously, if so, CPU0 and CPU1 will accept the interrupts.

\section{Functional Description}

\subsection{Peripheral Interrupt Sources}

ESP32-S3 has 99 peripheral interrupt sources in total. For the peripheral interrupt sources and their configuration/status registers, please refer to Table \ref{tab:peripheral-interrupt-config-status-source}.

\begin{itemize}
    \item Column ``No.": the peripheral interrupt source number, can be 0 \verb+~+ 98
    \item Column ``Source": all peripheral interrupt sources available
    \item Column ``Configuration Register": the registers used for routing the peripheral interrupt sources to CPU0/CPU1 peripheral interrupts
    \item Column ``Status Register": the registers used for indicating the interrupt status of peripheral interrupt sources

    \begin{itemize}
    \item Column ``Status Register - Bit": the bit position in status registers
    \item Column ``Status Register - Name": the name of status registers
    \end{itemize}
\end{itemize}

The register in column ``Configuration Register" and the bit in column ``Bit" correspond to the peripheral interrupt source in column ``Source". For example, the configuration register for interrupt source MAC\_INTR is \hyperref[regdesc:INTERRUPTCORE0MACINTRMAPREG]{INTERRUPT\_CORE\regindex{x}\_MAC\_INTR\_MAP\_REG}, and its status bit in \hyperref[regdesc:INTERRUPTCORE0INTRSTATUS0REG]{INTERRUPT\_CORE\regindex{x}\_INTR\_STATUS\_0\_REG} is bit0.

Note that CORE\regindex{x} in the table can be CORE0 (CPU0) or CORE1 (CPU1).

\subsection{CPU Interrupts}

Each CPU has 32 interrupts, numbered from 0 \verb+~+ 31, including 26 peripheral interrupts and six internal interrupts.
\begin{itemize}
    \item Peripheral interrupts: triggered by peripheral interrupt sources, include the following types:
    \begin{itemize}
        \item Level-triggered interrupts: triggered by a high level signal. The interrupt sources should hold the level till the CPU\regindex{x} handles the interrupts.
        \item Edge-triggered interrupts: triggered on a rising edge. CPU\regindex{x} responds to this kind of interrupts immediately.
        \item NMI interrupt: once triggered, the NMI interrupt can not be masked by software using the CPU\regindex{x} internal registers. World Controller provides a way to mask this kind of interrupt. For more information, see Chapter \ref{mod:wctrl} \textit{\nameref{mod:wctrl}}.
    \end{itemize}
    \item Internal interrupts: generated inside CPU\regindex{x}, include the following types:
    \begin{itemize}
       \item Timer interrupts: triggered by internal timers and are used to generate periodic interrupts.
        \item Software interrupts: triggered when software writes to special registers.
        \item Profiling interrupt: triggered for performance monitoring and analysis.
    \end{itemize}
\end{itemize}

Level-triggered and edge-triggered both describe the ways of CPU\regindex{x} to accept interrupt signals. For level-triggered interrupts, the level of interrupt signal should be kept till the CPU handles the interrupt, otherwise the interrupt may be lost. For edge-triggered interrupts, when a rising edge is detected, this edge will be recorded by CPU\regindex{x}, which then allows the interrupt signal to be released.

Interrupt matrix routes the peripheral interrupt sources to any of the CPU\regindex{x} peripheral interrupts. By such way, CPU\regindex{x} can receive the interrupt signals from peripheral interrupt sources. Table \ref{tab:CPU-interrupt} lists all the interrupts and their types as well as priorities.

ESP32-S3 supports the above-mentioned 32 interrupts at six levels as shown in the table below. A higher level corresponds to a higher priority. NMI has the highest interrupt priority and once triggered, the CPU\regindex{x} must handle such interrupt. Nested interrupts are also supported, i.e., low-level interrupts can be stopped by high-level interrupts.

\subsection{Allocate Peripheral Interrupt Source to CPU\regindex{x} Interrupt}

In this section, the following terms are used to describe the operation of the interrupt matrix.

\begin{itemize}

\item Source\_\regindex{Y}: stands for a peripheral interrupt source, wherein, \regindex{Y} means the number of this interrupt source in Table \ref{tab:peripheral-interrupt-config-status-source}.

\item \hyperref[regdesc:INTERRUPTCORE0CACHECORE1ACSINTMAPREG]{INTERRUPT\_CORE\regindex{x}\_SOURCE\_\regindex{Y}\_MAP\_REG}: stands for a configuration register for the peripheral interrupt source (Source\_\regindex{Y}) of CPU\regindex{x}.

\item Interrupt\_P: stands for the CPU\regindex{x} peripheral interrupt numbered as Num\_P. The value of Num\_P can be 0 \verb+~+ 5, 8 \verb+~+ 10, 12 \verb+~+ 14, 17 \verb+~+ 28, and 30 \verb+~+ 31. See Table \ref{tab:CPU-interrupt}.

\item Interrupt\_I: stands for the CPU\regindex{x} internal interrupt numbered as Num\_I. The value of Num\_I can be 6, 7, 11, 15, 16, and 29. See Table \ref{tab:CPU-interrupt}.
\end{itemize}

\subsubsection{Allocate one peripheral interrupt source (Source\_\regindex{Y}) to CPU\regindex{x}}
Setting the corresponding configuration register \hyperref[regdesc:INTERRUPTCORE0CACHECORE1ACSINTMAPREG]{INTERRUPT\_CORE\regindex{x}\_SOURCE\_\regindex{Y}\_MAP\_REG} of Source\_\regindex{Y} to Num\\\_P allocates this interrupt source to Interrupt\_P. Num\_P here can be any value from 0 \verb+~+ 5, 8 \verb+~+ 10, 12 \verb+~+ 14, 17 \verb+~+ 28, and 30 \verb+~+ 31. Note that one CPU\regindex{x} interrupt can be shared by multiple peripherals.

\subsubsection{Allocate multiple peripheral interrupt sources (Source\_Y\regindex{n}) to CPU\regindex{x}}
Setting the corresponding configuration register \hyperref[regdesc:INTERRUPTCORE0CACHECORE1ACSINTMAPREG]{INTERRUPT\_CORE\regindex{x}\_SOURCE\_Y\regindex{n}\_MAP\_REG} of each interrupt source to the same Num\_P allocates multiple sources to the same Interrupt\_P. Any of these sources can trigger CPU\regindex{x} Interrupt\_P. When an interrupt signal is generated, CPU\regindex{x} checks the interrupt status registers to figure out which peripheral the signal comes from.

\subsubsection{Disable CPU\regindex{x} peripheral interrupt source (Source\_\regindex{Y})}
Setting the corresponding configuration register \hyperref[regdesc:INTERRUPTCORE0CACHECORE1ACSINTMAPREG]{INTERRUPT\_CORE\regindex{x}\_SOURCE\_\regindex{Y}\_MAP\_REG} of the source to any Num\_I disables this interrupt Source\_\regindex{Y}. The choice of Num\_I (6, 7, 11, 15, 16, 29) does not matter, as none of peripheral interrupt sources allocated to Num\_I is connected to the CPU\regindex{x}. Therefore this functionality can be used to disable peripheral interrupt sources.

\subsection{Disable CPU\regindex{x} NMI Interrupt}

All CPU\regindex{x} interrupts, except for NMI interrupt (No.14 in Table \ref{tab:CPU-interrupt}), can be masked and enabled by software using CPU special register (INTENABLE). NMI interrupt can not be masked by the way above, but ESP32-S3 provides two ways to mask NMI interrupt:
\begin{itemize}
    \item Disconnect peripheral interrupt sources from NMI interrupt, i.e., the sources routed to NMI interrupt before are now routed to other interrupts. By such way, the previous NMI interrupt is maskable.
    \item Connect peripheral interrupt sources with NMI interrupt, but use World Controller module to mask NMI interrupt. For more information, see Chapter Chapter \ref{mod:wctrl} \textit{\nameref{mod:wctrl}}.
\end{itemize}


\subsection{Query Current Interrupt Status of Peripheral Interrupt Source}

Users can query current interrupt status of a CPU\regindex{x} peripheral interrupt source by reading the bit value in \hyperref[regdesc:INTERRUPTCORE0INTRSTATUS0REG]{INTERRUPT\_CORE\regindex{x}\_INTR\_STATUS\_\regindex{n}\_REG} (read only). For the mapping between \hyperref[regdesc:INTERRUPTCORE0INTRSTATUS0REG]{INTERRUPT\_CORE\regindex{x}\_INTR\_STATUS\_\\\regindex{n}\_REG} and peripheral interrupt sources, please refer to Table \ref{tab:peripheral-interrupt-config-status-source}.
