\hypertarget{iomuxgpio}{}
\chapter{IO MUX and GPIO Matrix (GPIO, IO MUX)}
\label{mod:iomuxgpio}

\section{Overview}

The \chipname{} chip features 45 physical GPIO pins. Each pin can be used as a general-purpose I/O, or be connected to an internal peripheral signal. Through GPIO matrix, IO MUX, and RTC IO MUX, peripheral input signals can be from any GPIO pin, and peripheral output signals can be routed to any GPIO pin. Together these modules provide highly configurable I/O.

{\bfseries Note that the 45 GPIO pins are numbered from 0 \verb+~+ 21 and 26 \verb+~+ 48. All these pins can be configured either as input or output.}


\section{Features}

\textbf{GPIO Matrix Features}

\begin{itemize}
\item A full-switching matrix between the peripheral input/output signals and the GPIO pins. 
\item 175 digital peripheral input signals can be sourced from the input of any GPIO pins. 
\item The output of any GPIO pins can be from any of the 184 digital peripheral output signals. 
\item Supports signal synchronization for peripheral inputs based on APB clock bus.
\item Provides input signal filter.
\item Supports sigma delta modulated output.
\item Supports GPIO simple input and output.
\end{itemize}

\textbf{IO MUX Features}

\begin{itemize}
\item Provides one configuration register \hyperref[regdesc:IOMUXnREG]{IO\_MUX\_GPIO\regindex{n}\_REG} for each GPIO pin. The pin can be configured to
    \begin{itemize}
        \item perform GPIO function routed by GPIO matrix;
        \item or perform direct connection bypassing GPIO matrix.
    \end{itemize}
\item Supports some high-speed digital signals (SPI, JTAG, UART) bypassing GPIO matrix for better high-frequency digital performance. In this case, IO MUX is used to connect these pins directly to peripherals.
\end{itemize}

\textbf{RTC IO MUX Features}

\begin{itemize}
    \item Controls low power feature of 22 RTC GPIO pins.
    \item Controls analog functions of 22 RTC GPIO pins.
    \item Redirects 22 RTC input/output signals to RTC system.
\end{itemize}

\section{Peripheral Input via GPIO Matrix}

\subsection{Overview}

To receive a peripheral input signal via GPIO matrix, the matrix is configured to source the peripheral input signal from one of the 45 GPIOs (0 \verb+~+ 21, 26 \verb+~+ 48), see Table \ref{tab:iomuxgpio-periph-signals-via-gpio-matrix}. Meanwhile, register corresponding to the peripheral signal should be set to receive input signal via GPIO matrix.

\subsection{Signal Synchronization}
\label{sec:inputconfig}
When signals are directed from pins using the GPIO matrix, the signals will be synchronized to the APB bus clock by the GPIO SYNC hardware, then go to GPIO matrix. This synchronization applies to all GPIO matrix signals but does not apply when using the IO MUX, see Figure \ref{Figure:iomux_gpio_matrix_all}.

\clearpage

\begin{figure}[!ht]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{21-IOMUXGPIO/figures/gpio_input_sync}}
    \caption{GPIO Input Synchronized on APB Clock Rising Edge or on Falling Edge}
    \label{Figure:gpio_input_sync}
\end{figure}

Figure \ref{Figure:gpio_input_sync} shows the functionality of GPIO SYNC. In the figure, negative sync and positive sync mean GPIO input is synchronized on APB clock falling edge and on APB clock rising edge, respectively.

\subsection{Functional Description}

To read GPIO pin \regindex{X}\hyperref[input-note-1]{$^1$} into peripheral signal \regindex{Y}, follow the steps below:

\begin{enumerate}
    \item Configure register \hyperref[regdesc:GPIOFUNCNINSELCFGREG]{GPIO\_FUNC\regindex{y}\_IN\_SEL\_CFG\_REG} corresponding to peripheral signal \regindex{Y} in GPIO matrix:
    \begin{itemize}
        \item Set \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_SIG\regindex{y}\_IN\_SEL} to enable peripheral signal input via GPIO matrix.
        \item Set \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} to the desired GPIO pin, i.e., \regindex{X} here.
    \end{itemize}
    {\bfseries Note that} some peripheral signals have no valid \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_SIG\regindex{y}\_IN\_SEL} bit, namely, these peripherals can only receive input signals via GPIO matrix.
    \item Optionally enable the filter for pin input signals by setting the register \hyperref[fielddesc:IOMUXFILTEREN]{IO\_MUX\_FILTER\_EN}. Only the signals with a valid width of more than two APB clock cycles can be sampled, see Figure \ref{Figure:gpio_filter}.
\begin{figure}[h!]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{21-IOMUXGPIO/figures/gpio_filter}}
    \caption{Filter Timing of GPIO Input Signals}
    \label{Figure:gpio_filter}
\end{figure}

    \item Synchronize GPIO input. To do so, please set \hyperref[regdesc:GPIOPINNREG]{GPIO\_PIN\regindex{x}\_REG} corresponding to GPIO pin \regindex{X} as follows:
    \begin{itemize}
        \item Set \hyperref[fielddesc:GPIOPINNSYNC1BYPASS]{GPIO\_PIN\regindex{x}\_SYNC1\_BYPASS} to enable input signal synchronized on rising edge or on falling edge in the first clock, see Figure \ref{Figure:gpio_input_sync}.
        \item Set \hyperref[fielddesc:GPIOPINNSYNC2BYPASS]{GPIO\_PIN\regindex{x}\_SYNC2\_BYPASS} to enable input signal synchronized on rising edge or on falling edge in the second clock, see Figure \ref{Figure:gpio_input_sync}.
    \end{itemize}
    \item Configure IO MUX register to enable pin input. For this end, please set \hyperref[regdesc:IOMUXnREG]{IO\_MUX\_\regindex{X}\_REG} corresponding to GPIO pin \regindex{x} as follows:
    \begin{itemize}
        \item Set \hyperref[fielddesc:IOMUXFUNIE]{IO\_MUX\_FUN\_IE} to enable input\hyperref[input-note-2]{$^2$}.
        \item Set or clear \hyperref[fielddesc:IOMUXFUNWPU]{IO\_MUX\_FUN\_WPU} and \hyperref[fielddesc:IOMUXFUNWPD]{IO\_MUX\_FUN\_WPD}, as desired, to enable or disable pull-up and pull-down resistors.
    \end{itemize}
\end{enumerate}

For example, to connect RMT channel 0 input signal\hyperref[input-note-3]{$^3$} (rmt\_sig\_in0, signal index 81) to GPIO40, please follow the steps below. Note that GPIO40 is also named as MTDO pin.

\begin{enumerate}
\item Set \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_SIG81\_IN\_SEL} in register \hyperref[regdesc:GPIOFUNCNINSELCFGREG]{GPIO\_FUNC81\_IN\_SEL\_CFG\_REG} to enable peripheral signal input via GPIO matrix.
\item Set \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC81\_IN\_SEL} in register \hyperref[regdesc:GPIOFUNCNINSELCFGREG]{GPIO\_FUNC81\_IN\_SEL\_CFG\_REG} to 40, i.e., select GPIO40.
\item Set \hyperref[fielddesc:IOMUXFUNIE]{IO\_MUX\_FUN\_IE} in register \hyperref[regdesc:IOMUXnREG]{IO\_MUX\_GPIO40\_REG} to enable pin input.
\end{enumerate}

\begin{tiplisting}
\vspace{-2em}
\begin{enumerate}
\item \label{input-note-1}  One pin input can be connected to multiple peripheral input signals.
\item \label{input-note-2} The input signal can be inverted by configuring \hyperref[fielddesc:GPIOFUNCNININVSEL]{GPIO\_FUNC\regindex{y}\_IN\_INV\_SEL}.
\item \label{input-note-3} It is possible to have a peripheral read a constantly low or constantly high input value without connecting this input to a pin. This can be done by selecting a special \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} input, instead of a GPIO number:
\begin {itemize}
\item When \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} is set to 0x{}3C, input signal is always 0.
\item When \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} is set to 0x{}38, input signal is always 1.
\end{itemize}
\end{enumerate}
\end{tiplisting}

\subsection{Simple GPIO Input}\label{IOMUXSimpleGPIOInput}

\hyperref[regdesc:GPIOINREG]{GPIO\_IN\_REG}/\hyperref[regdesc:GPIOIN1REG]{GPIO\_IN1\_REG} holds the input values of each GPIO pin. The input value of any GPIO pin can be read at any time without configuring GPIO matrix for a particular peripheral signal. However, it is necessary to enable pin input by setting \hyperref[fielddesc:IOMUXFUNIE]{IO\_MUX\_FUN\_IE} in register \hyperref[regdesc:IOMUXnREG]{IO\_MUX\_\regindex{x}\_REG} corresponding to pin \regindex{X}, as described in Section \ref{sec:inputconfig}.

\section{Peripheral Output via GPIO Matrix}

\subsection{Overview}

To output a signal from a peripheral via GPIO matrix, the matrix is configured to route peripheral output signals (only signals with a name assigned in the column "Output signal" in Table \ref{tab:iomuxgpio-periph-signals-via-gpio-matrix}) to one of the 45 GPIOs (0 \verb+~+ 21, 26 \verb+~+ 48).

The output signal is routed from the peripheral into GPIO matrix and then into IO MUX. IO MUX must be configured to set the chosen pin to GPIO function. This enables the output GPIO signal to be connected to the pin.

\begin{tiplisting}
There is a range of peripheral output signals (208 \verb+~+ 212 in Table \ref{tab:iomuxgpio-periph-signals-via-gpio-matrix}) which are not connected to any peripheral, but to the input signals (208 \verb+~+ 212) directly. These can be used to input a signal from one GPIO pin and output directly to another GPIO pin.
\end{tiplisting}


\vspace{-2em}
\subsection{Functional Description}\label{IOMUXPeripheralOutput}

Some of the 256 output signals (signals with a name assigned in the column "Output signal" in Table \ref{tab:iomuxgpio-periph-signals-via-gpio-matrix}) can be set to go through GPIO matrix into IO MUX and then to a pin. Figure \ref{Figure:iomux_gpio_matrix_all} illustrates the configuration.

To output peripheral signal \regindex{Y} to a particular GPIO pin \regindex{X}\hyperref[output-note-1]{$^1$ $^,$ $^2$}, follow these steps:

\begin{enumerate}
    \item Configure \hyperref[regdesc:GPIOFUNCNOUTSELCFGREG]{GPIO\_FUNC\regindex{x}\_OUT\_SEL\_CFG\_REG} and \hyperref[regdesc:GPIOENABLEREG]{GPIO\_ENABLE\_REG}[\regindex{x}] corresponding to GPIO pin \regindex{X} in GPIO matrix. Recommended operation: use corresponding \hyperref[regdesc:GPIOENABLEW1TSREG]{W1TS} (write 1 to set) and \hyperref[regdesc:GPIOENABLEW1TCREG]{W1TC} (write 1 to clear) registers to set or clear \hyperref[regdesc:GPIOENABLEREG]{GPIO\_ENABLE\_REG}.
    \begin{itemize}
        \item Set the \hyperref[fielddesc:GPIOFUNCNOUTSEL]{GPIO\_FUNC\regindex{x}\_OUT\_SEL} field in register \hyperref[regdesc:GPIOFUNCNOUTSELCFGREG]{GPIO\_FUNC\regindex{x}\_OUT\_SEL\_CFG\_REG} to the index of the desired peripheral output signal \regindex{Y}.
        \item If the signal should always be enabled as an output, set the bit \hyperref[fielddesc:GPIOFUNCNOENSEL]{GPIO\_FUNC\regindex{x}\_OEN\_SEL} in register \hyperref[regdesc:GPIOFUNCNOUTSELCFGREG]{GPIO\_FUNC\regindex{x}\_OUT\_SEL\_CFG\_REG} and the bit in register \hyperref[regdesc:GPIOENABLEW1TSREG]{GPIO\_ENABLE/ENABLE1\_W1TS\_REG}, corresponding to GPIO pin \regindex{X}. To have the output enable signal decided by internal logic (for example, the SPIQ\_oe in column ``Output enable signal when \hyperref[fielddesc:GPIOFUNCNOENSEL]{GPIO\_FUNC\regindex{n}\_OEN\_SEL} = 0" in Table \ref{tab:iomuxgpio-periph-signals-via-gpio-matrix}), clear the bit \hyperref[fielddesc:GPIOFUNCNOENSEL]{GPIO\_FUNC\regindex{x}\_OEN\_SEL} instead.
       \item Set the corresponding bit in register \hyperref[regdesc:GPIOENABLEW1TCREG]{GPIO\_ENABLE/ENABLE1\_W1TC\_REG} to disable the output from the GPIO pin.
    \end{itemize}
    \item For an open drain output, set the bit \hyperref[fielddesc:GPIOPINNPADDRIVER]{GPIO\_PIN\regindex{x}\_PAD\_DRIVER} in register \hyperref[regdesc:GPIOPINNREG]{GPIO\_PIN\regindex{x}\_REG} corresponding to GPIO pin \regindex{X}.
    \item Configure IO MUX register to enable output via GPIO matrix. Set the \hyperref[regdesc:IOMUXnREG]{IO\_MUX\_\regindex{x}\_REG} corresponding to GPIO pin \regindex{X} as follows:
    \begin{itemize}
        \item Set the field \hyperref[fielddesc:IOMUXMCUSEL]{IO\_MUX\_MCU\_SEL} to desired IO MUX function corresponding to GPIO pin \regindex {X}. This is Function 1 (GPIO function), numeric value 1, for all pins.
        \item Set the field \hyperref[fielddesc:IOMUXFUNDRV]{IO\_MUX\_FUN\_DRV} to the desired value for output strength (0 \verb+~+ 3).
        \item If using open drain mode, set/clear \hyperref[fielddesc:IOMUXFUNWPU]{IO\_MUX\_FUN\_WPU} and \hyperref[fielddesc:IOMUXFUNWPD]{IO\_MUX\_FUN\_WPD} to enable/disable the internal pull-up/pull-down resistors.
    \end{itemize}
\end{enumerate}

\begin{tiplisting}
\vspace{-2em}
\begin{enumerate}
\item \label{output-note-1} The output signal from a single peripheral can be sent to multiple pins simultaneously.
\item \label{output-note-3} The output signal can be inverted by setting \hyperref[fielddesc:GPIOFUNCNOUTINVSEL]{GPIO\_FUNC\regindex{x}\_OUT\_INV\_SEL}.
\end{enumerate}
\end{tiplisting}

\subsection{Simple GPIO Output}

GPIO matrix can also be used for simple GPIO output. This can be done as below:

\begin{itemize}
    \item Set GPIO matrix \hyperref[fielddesc:GPIOFUNCNOUTSEL]{GPIO\_FUNC\regindex{n}\_OUT\_SEL} with a special peripheral index 256 (0x{}100);
    \item Set the corresponding bit in \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}[31:0] or \hyperref[regdesc:GPIOOUT1REG]{GPIO\_OUT1\_REG}[21:0] to the desired GPIO output value.
\end{itemize}

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
\item \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}[21:0] and \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}[31:26] correspond to GPIO0 \verb+~+ 21 and GPIO26 \verb+~+ 31, respectively. \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}[25:22] are invalid.
\item \hyperref[regdesc:GPIOOUT1REG]{GPIO\_OUT1\_REG}[16:0] correspond to GPIO32 \verb+~+ 48, and \hyperref[regdesc:GPIOOUT1REG]{GPIO\_OUT1\_REG}[21:17] are invalid.
\item Recommended operation: use corresponding W1TS and W1TC registers, such as \hyperref[fielddesc:GPIOOUTW1TS]{GPIO\_OUT\_W1TS}/\hyperref[fielddesc:GPIOOUTW1TC]{GPIO\_OUT\_\\W1TC} to set or clear the registers \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}/\hyperref[regdesc:GPIOOUT1REG]{GPIO\_OUT1\_REG}.
\end{itemize}
\end{tiplisting}

\subsection{Sigma Delta Modulated Output}

\subsubsection{Functional Description}

Eight out of the 256 peripheral outputs (index: 93 \verb+~+ 100 in Table \ref{tab:iomuxgpio-periph-signals-via-gpio-matrix}) support 1-bit second-order sigma delta modulation. By default output is enabled for these eight channels. This Sigma Delta modulator can also output PDM (pulse density modulation) signal with configurable duty cycle. The transfer function is:

\[
    H(z)=\textrm{X(z)z$^{-1}$} + \textrm{E(z)(1-z$^{-1}$)$^{2}$}
    \]

E(z) is quantization error and X(z) is the input.\\
This modulator supports scaling down of APB\_CLK by divider 1 \verb+~+ 256:
\begin{itemize}
    \item Set \hyperref[fielddesc:GPIOSDFUNCTIONCLKEN]{GPIO\_FUNCTION\_CLK\_EN} to enable the modulator clock.
    \item Configure \hyperref[fielddesc:GPIOSDSDNPRESCALE]{GPIO\_SD\regindex{n}\_PRESCALE} (\regindex{n} is 0 \verb+~+ 7 for eight channels).
\end{itemize}

After scaling, the clock cycle is equal to one pulse output cycle from the modulator.

\hyperref[fielddesc:GPIOSDSDNIN]{GPIO\_SD\regindex{n}\_IN} is a signed number with a range of [-128, 127] and is used to control the duty cycle $^{1}$ of PDM output signal.

\begin{itemize}
    \item \hyperref[fielddesc:GPIOSDSDNIN]{GPIO\_SD\regindex{n}\_IN} = -128, the duty cycle of the output signal is \begin{math}0\% \end{math}.
    \item \hyperref[fielddesc:GPIOSDSDNIN]{GPIO\_SD\regindex{n}\_IN} = 0, the duty cycle of the output signal is near \begin{math}50\% \end{math}.
    \item \hyperref[fielddesc:GPIOSDSDNIN]{GPIO\_SD\regindex{n}\_IN} = 127, the duty cycle of the output signal is close to \begin{math}100\% \end{math}.
\end{itemize}

The formula for calculating PDM signal duty cycle is shown as below:

    \[
    Duty\_Cycle=\frac{\hyperref[fielddesc:GPIOSDSDNIN]{GPIO\_SD\regindex{n}\_IN} + 128} {256}
    \]

\begin{tiplisting}
For PDM signals, duty cycle refers to the percentage of high level cycles to the whole statistical period (several pulse cycles, for example 256 pulse cycles).
\end{tiplisting}

\subsubsection{SDM Configuration}

The configuration of SDM is shown below:

\begin{itemize}
    \item Route one of SDM outputs to a pin via GPIO matrix, see Section \ref{IOMUXPeripheralOutput}.
    \item Enable the modulator clock by setting \hyperref[fielddesc:GPIOSDFUNCTIONCLKEN]{GPIO\_FUNCTION\_CLK\_EN}.
    \item Configure the divider value by setting \hyperref[fielddesc:GPIOSDSDNPRESCALE]{GPIO\_SD\regindex{n}\_PRESCALE}.
    \item Configure the duty cycle of SDM output signal by setting \hyperref[fielddesc:GPIOSDSDNIN]{GPIO\_SD\regindex{n}\_IN}.
\end{itemize}


\section{Direct Input and Output via IO MUX}

\subsection{Overview}

Some high-speed signals (SPI and JTAG) can bypass GPIO matrix for better high-frequency digital performance. In this case, IO MUX is used to connect these pins directly to the peripherals.

This option is less flexible than routing signals via GPIO matrix, as the IO MUX register for each GPIO pin can only select from a limited number of functions, but high-frequency digital performance can be improved.

\subsection{Functional Description}

Two registers must be configured in order to bypass GPIO matrix for peripheral input signals:

\begin{enumerate}
\item \hyperref[fielddesc:IOMUXMCUSEL]{IO\_MUX\_MCU\_SEL} for the GPIO pin must be set to the required pin function. For the list of pin functions, please refer to Section \ref{sec:iomux_pads}.
\item Clear \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_SIG\regindex{n}\_IN\_SEL} to route the input directly to the peripheral.
\end{enumerate}

To bypass GPIO matrix for peripheral output signals, \hyperref[fielddesc:IOMUXMCUSEL]{IO\_MUX\_MCU\_SEL} for the GPIO pin must be set to the required pin function. For the list of pin functions, please refer to Section \ref{sec:iomux_pads}.

\begin{tiplisting}
Not all signals can be connected to peripheral via IO MUX. Some input/output signals can only be connected to peripheral via GPIO matrix.
\end{tiplisting}

\section{RTC IO MUX for Low Power and Analog Input/Output}

\subsection{Overview}

\chipname{} provides 22 GPIO pins with low power capabilities (RTC) and analog functions, which are handled by the RTC subsystem of \chipname{}. IO MUX and GPIO matrix are not used for these functions, rather, RTC IO MUX is used to redirect 22 RTC input/output signals to the RTC subsystem.

When configured as RTC GPIOs, the output pins can still retain the output level value when the chip is in Deep-sleep mode, and the input pins can wake up the chip from Deep-sleep.

\subsection{Low Power Capabilities}

The pins with RTC functions are controlled by \hyperref[fielddesc:RTCIOTOUCHPADNMUXSEL]{RTC\_IO\_TOUCH/RTC\_PAD\regindex{n}\_MUX\_SEL} bit in register \hyperref[regdesc:RTCIOTOUCHPADNREG]{RTC\_IO\_\\TOUCH/RTC\_PAD\regindex{n}\_REG}. By default all bits in these registers are set to 0, routing all input/output signals via IO MUX.

If \hyperref[fielddesc:RTCIOTOUCHPADNMUXSEL]{RTC\_IO\_TOUCH/RTC\_PAD\regindex{n}\_MUX\_SEL} is set to 1, then input/output signals to and from that pin is routed to the RTC subsystem. In this mode, \hyperref[regdesc:RTCIOTOUCHPADNREG]{RTC\_IO\_TOUCH/RTC\_PAD\regindex{n}\_REG} is used to control RTC low power pins. Note that \hyperref[regdesc:RTCIOTOUCHPADNREG]{RTC\_IO\_TOUCH/RTC\_PAD\regindex{n}\_REG} applies the RTC GPIO pin numbering, not the GPIO pin numbering. See Table \ref{tab:iomuxgpio-rtc-iomax-pads-functions} for RTC functions of RTC IO MUX pins.

\subsection{Analog Functions}

When the pin is used for analog purpose, make sure this pin is left floating by configuring the register \hyperref[regdesc:RTCIOTOUCHPADNREG]{RTC\_IO\_TOUCH\\/RTC\_PAD\regindex{n}\_REG}. By such way, external analog signal is connected to internal analog signal via GPIO pin. The configuration is as follows:

\begin{itemize}
    \item Set \hyperref[fielddesc:RTCIOTOUCHPADNMUXSEL]{RTC\_IO\_TOUCH/RTC\_PAD\regindex{n}\_MUX\_SEL}, to select RTC IO MUX to route input and output signals.
    \item Clear \hyperref[regdesc:RTCIOTOUCHPADNREG]{RTC\_IO\_TOUCH/RTC\_PAD\regindex{n}\_FUN\_IE}, \hyperref[regdesc:RTCIOTOUCHPADNREG]{RTC\_IO\_TOUCH/RTC\_PAD\regindex{n}\_\\FUN\_RUE}, and \hyperref[regdesc:RTCIOTOUCHPADNREG]{RTC\_IO\_TOUCH/RTC\_PAD\regindex{n}\_FUN\_RDE}, to set this pin floating.
    \item Configure \hyperref[regdesc:RTCIOTOUCHPADNREG]{RTC\_IO\_TOUCH/RTC\_PAD\regindex{n}\_FUN\_SEL} to 0, to enable analog function 0.
    \item Write 1 to \hyperref[regdesc:RTCIOTOUCHPADNREG]{RTC\_GPIO\_ENABLE\_W1TC}, to clear output enable.
\end{itemize}
See Table \ref{tab:iomuxgpio-iomax-pins-analog-functions} for analog functions of RTC IO MUX pins.

\section{Pin Functions in Light-sleep}

Pins may provide different functions when \chipname{} is in Light-sleep mode. If \hyperref[fielddesc:IOMUXSLPSEL]{IO\_MUX\_SLP\_SEL} in register \hyperref[regdesc:IOMUXnREG]{IO\_MUX\_\regindex{n}\_REG} for a GPIO pin is set to 1, a different set of bits will be used to control the pin when the chip is in Light-sleep mode.

\begin{longtable}[c]{|p{3.5cm}|p{4.5cm}|p{4.5cm}|}
\caption{Bits Used to Control IO MUX Functions in Light-sleep Mode}

\section{Pin Hold Feature}\label{PadHold}

Each GPIO pin (including the RTC pins) has an individual hold function controlled by an RTC register. When the pin is set to hold, the state is latched at that moment and will not change no matter how the internal signals change or how the IO MUX/GPIO configuration is modified. Users can use the hold function for the pins to retain the pin state through a core reset triggered by watchdog time-out or Deep-sleep events.

\begin{itemize}
    \item \textbf{Digital Pins} (GPIO26 \verb+~+ GPIO48)\\
    The Hold state of each digital pin is controlled by the result of OR operation of the pin's Hold enable signal and the global Hold enable signal.
    \begin{itemize}
    \item \hyperref[regdesc:RTCCNTLDIGPADHOLDREG] {RTC\_CNTL\_DIG\_PAD\_HOLD\_REG[\regindex{n}]}, controls the Hold signal of each pin of GPIO26 \verb+~+ GPIO48.
    \item \hyperref[fielddesc:RTCCNTLDGPADFORCEHOLD]{RTC\_CNTL\_DG\_PAD\_FORCE\_HOLD}, controls the global Hold signal of all digital pins.
\end{itemize}
To use this feature, follow the steps below:
\begin{itemize}
        \item To maintain the pin's input/output status in Deep-sleep, set \hyperref[regdesc:RTCCNTLDIGPADHOLDREG] {RTC\_CNTL\_DIG\_PAD\_HOLD\_REG[\regindex{n}]} to hold the value of each digital pin, \regindex{n} = 5 \verb+~+ 27, corresponding to GPIO26 \verb+~+ GPIO48. Or clear \hyperref[regdesc:RTCCNTLDIGPADHOLDREG]{RTC\_CNTL\_DIG\_PAD\_HOLD\_REG[\regindex{n}]} to disable the Hold function of the pin.
        \item Alternatively, set \hyperref[fielddesc:RTCCNTLDGPADFORCEHOLD]{RTC\_CNTL\_DG\_PAD\_FORCE\_HOLD} to hold the values of all digital pins, or set \hyperref[fielddesc:RTCCNTLDGPADFORCEUNHOLD]{RTC\_CNTL\_DG\_PAD\_FORCE\_UNHOLD} to disable the hold function of all digital pins.
    \end{itemize}
\item \textbf{RTC Pins} (GPIO0 \verb+~+ GPIO21)\\
The Hold state of each RTC pin is controlled by the result of OR operation of the pin's Hold enable signal and the global Hold enable signal.
    \begin{itemize}
    \item \hyperref[regdesc:RTCCNTLRTCPADHOLDREG]{RTC\_CNTL\_RTC\_PAD\_HOLD\_REG[\regindex{n}]} (\regindex{n} = 0 \verb+~+ 21), controls the Hold signal of each pin of GPIO0 \verb+~+ GPIO21.
    \item \hyperref[fielddesc:RTCCNTLRTCPADFORCEHOLD]{RTC\_CNTL\_RTC\_PAD\_FORCE\_HOLD}, controls the global Hold signal of all RTC pins.
\end{itemize}

To use this feature, follow the steps below:

\begin{itemize}
    \item To maintain the pin's input/output status in Deep-sleep, set \hyperref[regdesc:RTCCNTLRTCPADHOLDREG]{RTC\_CNTL\_RTC\_PAD\_HOLD\_REG[\regindex{n}]} (\regindex{n} = 0 \verb+~+ 21, corresponding to GPIO0 \verb+~+ GPIO21). Or clear the bits above to disable the Hold function of the pin.
    \item Alternatively, set \hyperref[fielddesc:RTCCNTLRTCPADFORCEHOLD]{RTC\_CNTL\_RTC\_PAD\_FORCE\_HOLD} to hold the values of all RTC pins, or clear \hyperref[fielddesc:RTCCNTLRTCPADFORCEHOLD]{RTC\_CNTL\_RTC\_PAD\_FORCE\_HOLD} to disable the hold function of all RTC pins.
\end{itemize}

\end{itemize}

\section{Power Supply and Management of GPIO Pins}\label{I/O Pad Power Supplies}

\subsection{Power Supply of GPIO Pins}
For more information on the power supply for GPIO pins, please refer to Pin Definition in  {\it{\href{https://www.espressif.com/sites/default/files/documentation/esp32-s3_datasheet_en.pdf}{\chipname{} Datasheet}}}.

\subsection{Power Supply Management}

Each \chipname{} pin is connected to one of the three different power domains.

\begin{itemize}
    \item VDD3P3\_RTC: the input power supply for both RTC and CPU
    \item VDD3P3\_CPU: the input power supply for CPU
    \item VDD\_SPI: configurable input/output power supply
\end{itemize}

VDD\_SPI can be configured to use an internal LDO. The LDO input and output both are 1.8 V. If the LDO is not enabled, VDD\_SPI is connected directly to the same power supply as VDD3P3\_RTC.

The VDD\_SPI configuration is determined by the value of strapping pin GPIO45, or can be overridden by eFuse and/or register settings. See {\it{\href{https://www.espressif.com/sites/default/files/documentation/esp32-s3_datasheet_en.pdf}{\chipname{} Datasheet}}} sections Power Scheme and Strapping Pins for more details.

Note that GPIO33 \verb+~+ GPIO37 and GPIO47 \verb+~+ GPIO48 can be powered either by VDD\_SPI or VDD3P3\_CPU.
