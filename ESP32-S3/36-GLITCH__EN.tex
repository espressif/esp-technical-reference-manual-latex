\hypertarget{glitch}{}
\chapter{Clock Glitch Detection}
\label{mod:glitch}

\section{Overview}

The Clock Glitch Detection module on \chipname{} detects glitches in external crystal XTAL\_CLK signals, and generates a system reset signal (see Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}}) when detecting glitches to reset the whole digital circuit including RTC. By doing so, it prevents attackers from injecting glitches on external crystal XTAL\_CLK clock to compromise \chipname{} and thus strengthens chip security.

\section{Functional Description}

\subsection{Clock Glitch Detection}

The Clock Glitch Detection module on \chipname{} monitors input clock signals from XTAL\_CLK. If it detects a glitch, namely a clock pulse (a or b in the figure below) with a width shorter than 3 ns, input clock signals from XTAL\_CLK are blocked.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.5\textwidth]{\modulefiles/figures/glitch-clock-width}
    \caption{XTAL\_CLK Pulse Width}
    \label{fig:glitch-pulse-width}
\end{figure}

\subsection{Reset}

Once detecting a glitch on XTAL\_CLK that affects the circuit's normal operation, the Clock Glitch Detection module triggers a system reset if \hyperref[fielddesc:RTCCNTLGLITCHRSTEN]{RTC\_CNTL\_GLITCH\_RST\_EN} bit is enabled. By default, this bit is set to enable a reset.
