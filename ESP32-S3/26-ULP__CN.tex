\chapter{超低功耗协处理器 (ULP-FSM, ULP-RISC-V)}
\label{mod:ulp}
\definecolor{istcolor}{HTML}{7e0f12}
\section{概述}

超低功耗协处理器 (ULP, Ultra-Low-Power coprocessor) 是一种功耗极低的处理器设备，可在芯片进入 Deep-sleep 时保持上电（详见章节 \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}），允许开发者通过存储在 RTC 存储器中的专用程序，访问 RTC 外设、内部传感器及 RTC 寄存器。在对功耗敏感的场景下，主 CPU 处于睡眠状态以降低功耗，协处理器可以由协处理器定时器唤醒，通过控制 RTC GPIO、RTC I2C、SAR ADC、温度传感器 (TSENS) 等外设监测外部环境或与外部电路进行交互，并在达到唤醒条件时主动唤醒主 CPU。

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{./26-ULP/figures/26-ULP-ulp-overview.png}
    \caption{超低功耗协处理器概图}
    \label{fig:coprocessor-overview}
\end{figure}

ESP32-S3 搭载了基于有限状态机 (FSM) 实现的 ULP 协处理器（以下简称 ULP-FSM）和基于 RISC-V 指令集的 ULP 协处理器（以下简称 ULP-RISC-V），用户可根据需求灵活选择。

\section{特性}

\begin{itemize}
    \item 可访问最多 8 KB SRAM RTC 慢速内存，用于储存指令和数据
    \item 时钟采用 17.5 MHz RTC\_FAST\_CLK
    \item 支持正常模式和 Monitor 模式
    \item 可唤醒 CPU 或向 CPU 发送中断
    \item 可访问外设、内部传感器及 RTC 寄存器
\end{itemize}

ULP-FSM 和 ULP-RISC-V 不能同时工作，用户只能选择其中一个作为 ESP32-S3 的超低功耗协处理器。ULP-FSM 与 ULP-RISC-V 特性比较如下：

\section{协处理器的睡眠和唤醒流程}


ESP32-S3 的协处理器经过专门设计，无论 CPU 是否处于休眠状态，均可独立于 CPU 运行。

在典型场景中，为了降低功耗，系统可进入 Deep-sleep 模式。系统进入睡眠模式前需完成以下操作：

\begin{enumerate}
    \item 将协处理器需要执行的程序载入到 RTC 慢速内存；
    \item 配置 \hyperref[fielddesc:RTCCNTLCOCPUSEL]{RTC\_CNTL\_COCPU\_SEL} 寄存器，选择协处理器；
    \begin{itemize}
        \item 0：选择使用 ULP-RISC-V
        \item 1：选择使用 ULP-FSM
    \end{itemize}
    \item 如果选择使用 ULP-RISC-V，则还需要：        \begin{itemize}
        \item 置位再复位 \hyperref[fielddesc:RTCCNTLCOCPUCLKFO]{RTC\_CNTL\_COCPU\_CLK\_FO}；
        \item 置位 \hyperref[fielddesc:RTCCNTLCOCPUCLKGATEEN]{RTC\_CNTL\_COCPU\_CLKGATE\_EN}。
    \end{itemize}

    \item 配置 \hyperref[regdesc:RTCCNTLULPCPTIMER1REG]{RTC\_CNTL\_ULP\_CP\_TIMER\_1\_REG} 寄存器来设置硬件定时器的唤醒间隔时间；
    \item 选择定时器使能方式：
        \begin{itemize}
            \item 软件使能：软件置位 \hyperref[fielddesc:RTCCNTLULPCPSLPTIMEREN]{RTC\_CNTL\_ULP\_CP\_SLP\_TIMER\_EN}；
        \item RTC GPIO 使能：软件配置 \hyperref[fielddesc:RTCCNTLULPCPGPIOWAKEUPENA]{RTC\_CNTL\_ULP\_CP\_GPIO\_WAKEUP\_ENA} 开启通过 RTC GPIO 使能硬件定时器选项。更多信息见章节 \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}。
        \end{itemize}
    \item 配置系统进入睡眠状态，主 CPU 休眠。
\end{enumerate}

在 Deep-sleep 模式下：

\begin{enumerate}
   \item 硬件定时器周期性地将低功耗控制器置于 Monitor 状态，然后唤醒协处理器。更多信息见章节 \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}；
   \item 协处理器唤醒后执行一些必要操作，例如通过低功耗传感器监控芯片外部环境等操作；
   \item 操作完成后，系统返回 Deep-sleep 模式；
   \item 协处理器进入休眠，等待下一次唤醒。
\end{enumerate}

进入 Monitor 模式后，协处理器的唤醒和睡眠流程见图 \ref{fig:ULP-program-control-flow}，包括：

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{./26-ULP/figures/26-ULP-ulp-exec-control-flow.eps}
    \caption{协处理器睡眠和唤醒流程}
    \label{fig:ULP-program-control-flow}
\end{figure}

\begin{enumerate}
    \item 使能硬件定时器，定时器开始计数；
    \item 硬件定时器过期，唤醒协处理器。协处理器进入 Run 状态，执行预先烧写的程序；
    \item 协处理器执行 HALT 相关操作进入 HALT 状态；协处理器程序停止运行开始休眠。定时器再次启动，重复以上流程；
    \begin{itemize}
        \item ULP-RISC-V 的 HALT 操作：置位寄存器 \hyperref[fielddesc:RTCCNTLCOCPUDONE]{RTC\_CNTL\_COCPU\_DONE}；
        \item ULP-FSM 的 HALT 操作：执行 HALT 指令。
    \end{itemize}
    \item 通过协处理器程序或软件关闭硬件定时器，协处理器将不再进入 Monitor 状态。
        \begin{itemize}
            \item 软件关闭：软件清零 \hyperref[fielddesc:RTCCNTLULPCPSLPTIMEREN]{RTC\_CNTL\_ULP\_CP\_SLP\_TIMER\_EN}；
            \item RTC GPIO 关闭：软件清零 \hyperref[fielddesc:RTCCNTLULPCPGPIOWAKEUPENA]{RTC\_CNTL\_ULP\_CP\_GPIO\_WAKEUP\_ENA}，并置位 \hyperref[fielddesc:RTCCNTLULPCPGPIOWAKEUPCLR]{RTC\_CNTL\_ULP\_CP\_\\GPIO\_WAKEUP\_CLR}。

        注：硬件定时器的关闭方法需要与使能方法一致。
        \end{itemize}
\end{enumerate}

\textbf{注意：}\\
    由于 ULP-RISC-V 进入 HALT 状态需要配置 \hyperref[fielddesc:RTCCNTLCOCPUDONE]{RTC\_CNTL\_COCPU\_DONE}，所以建议预先烧写的程序用下列代码结尾：
    \begin{itemize}
        \item 置位 \hyperref[fielddesc:RTCCNTLCOCPUDONE]{RTC\_CNTL\_COCPU\_DONE}，结束本次 ULP-RISC-V 的运行，并使 ULP-RISC-V 进入休眠模式；
        \item 置位 \hyperref[fielddesc:RTCCNTLCOCPUSHUTRESETEN]{RTC\_CNTL\_COCPU\_SHUT\_RESET\_EN}，复位 ULP-RISC-V。
        注：硬件已经预留了足够的时间支持 ULP-RISC-V 在进入休眠之前完成该操作。
    \end{itemize}

\section{ULP-FSM}
\subsection{特性}

ULP-FSM 协处理器是一种可编程有限状态机，可在 CPU 进入 Deep-sleep 状态时工作。协处理器支持部分通用 CPU 指令，可进行一些复杂的逻辑与算术运算。此外，ULP-FSM 还支持一些特殊的 RTC 控制与外设控制指令。ULP-FSM 的运行代码和数据存储在 8 KB SRAM RTC 慢速内存中（CPU 也可访问该区域）。因此，这块内存经常用于存储一些协处理器和 CPU 的通用指令。ULP-FSM 可通过执行 HALT 指令停止运行。

ULP-FSM 具有以下特性：
\begin{itemize}
    \item 采用 4 个 16 位通用寄存器 (R0 \verb+~+ R3)，进行数据操作和内存访问；
    \item 采用 1 个 8 位阶段计数器寄存器 Stage\_cnt，可通过 ALU 指令进行操作并用于 JUMP 指令；
    \item 内置专用指令，可直接控制低功耗外设，如 SAR ADC，温度传感器等。
\end{itemize}

\subsection{指令集} \label{sec:instruction-set}

ULP-FSM 可支持下列指令：

\begin{itemize}
    \item ALU - 算术与逻辑
    \item LD、ST、REG\_RD 及 REG\_WR - 加载与数据存储
    \item JUMP - 跳转至某地址
    \item WAIT 和 HALT - 管理程序执行
    \item WAKE - 唤醒 CPU 及与 CPU 通信
    \item TSENS 和 ADC - 测量
\end{itemize}

ULP-FSM 指令的格式见图 \ref{fig:ULP-instruction}。

根据 \textit{Operands} 的设置不同，同一个 \textit{OpCode} 可对应多种不同操作。例如，ALU 能够执行 10 种不同的算术和逻辑运算，JUMP 也可执行有条件跳转、无条件跳转、绝对跳转及相对跳转等多种形式的跳转。

ULP-FSM 协处理器的所有指令均固定为 32 位。通过这一系列指令，协处理器程序即可得到执行。程序内部的执行均采用 32 位寻址。该程序具体存储在 1 块专用的慢速内存区，地址范围为 0x{}5000\_0000 到 0x{}5000\_1FFF (8 KB)，对主 CPU 可见。


\subsubsection{ALU - 算术与逻辑运算} \label{para:ALU}

算术逻辑单元 (ALU) 可以进行算术和逻辑运算，操作对象为协处理器寄存器中存储的数值或指令中存储的立即值。具体可以支持的运算类型如下：
\begin{itemize}
    \item 算术 - 加 (ADD) 和减 (SUB)
    \item 逻辑 - 按位与 (AND) 和按位或 (OR)
    \item 移位 - 左移 (LSH) 和右移 (RSH)
    \item 寄存器赋值 - 移动 (MOVE)
    \item 计数器寄存器操作 - STAGE\_RST、STAGE\_INC 和 STAGE\_DEC
\end{itemize}

尽管此处 \textit{OpCode} 相同，均为 7，但可通过设置协处理器指令 [27:21] 位，选择特定的算术和逻辑运算。

\textbf{描述}

该指令可以从外设寄存器中读取最高 16 位的内容，并存入通用寄存器。

~
\begin{centering}
    R0 = REG[Addr][High:Low] \\
\end{centering}
~\\
如需读取的内容超过 16 位，即 \textit{High} - \textit{Low} + 1 > 16，则该指令将返回 [Low+15:Low] 的内容。

\textbf{注意：}
\begin{itemize}
    \item 该指令可访问 RTC\_CNTL、RTC\_IO、SENS 及 RTC\_I2C 外设中的寄存器。ULP 协处理器可通过相同寄存器在外设总线上的地址 (addr\_bus)，计算外设寄存器的地址，具体方式见下：

    ~
    \begin{centering}
        addr\_ulp = (addr\_bus - DR\_REG\_RTCCNTL\_BASE)/4 \\
    \end{centering}
    ~

    \item \textit{addr\_ulp} 以 32 位字（而非字节）为单位，0 可投射至 DR\_REG\_RTCCNTL\_BASE（从主 CPU 的角度）。因此，10 位 ULP 协处理器的地址可覆盖外设寄存器空间的 4096 字节，包括 DR\_REG\_RTCCNTL\_BASE（0x{}60008000）、DR\_REG\_RTCIO\_BASE（0x{}60008400）、DR\_REG\_SENS\_BASE（0x{}60008800）及 DR\_REG\_\\RTC\_I2C\_BASE（0x{}60008800）区域。更多地址映射信息，见章节 \ref{sec:address-mapping}。
\end{itemize}

\subsubsection{中断处理}

当中断发生时，ULP-RISC-V 将执行以下操作：

\begin{enumerate}
    \item 将当前 PC 保存到 Q0
    \item 将被响应的中断保存到 Q1
    \item 跳转到中断入口 (0x{}10)
\end{enumerate}

进入中断入口后，需要保存上下文，可使用 Q2 和 Q3 作为暂存备份寄存器。当需要退出中断时，恢复上下文，执行 retirq 指令即可从中断返回，跳转到 Q0。ULP 不支持中断嵌套，在执行 retirq 指令之前不会被任何中断打断。

\section{RTC I2C 控制器} \label{sec:RTC-I2C-Controller}

ULP 协处理器可通过 RTC I2C 控制器与外部 I2C 从机进行基本的读写操作。

\subsection{连接 RTC I2C 信号}

SDA 和 SCL 时钟信号可通过 \hyperref[regdesc:RTCIOSARI2CIOREG]{RTCIO\_SAR\_I2C\_IO\_REG} 寄存器，连接至 2 个 GPIO 管脚（4 个可选），详细定义请见章节 IO MUX 和 GPIO 矩阵中的 RTC\_MUX 管脚清单。

\subsection{配置 RTC I2C 控制器}

ULP 协处理器在正常使用 I2C 指令之前必须配置 RTC I2C 控制器中的特定参数，具体即向 RTC I2C 寄存器写入特定计时参数。这一步可通过主 CPU 或 ULP 自身运行程序完成。

\begin{tiplisting}
计时参数均以 RTC\_FAST\_CLK (17.5 MHz) 为单位。
\end{tiplisting}

\begin{enumerate}
    \item 通过 \hyperref[fielddesc:RTCI2CSCLLOWPERIODREG]{RTC\_I2C\_SCL\_LOW\_PERIOD\_REG} 和 \hyperref[fielddesc:RTCI2CSCLHIGHPERIODREG]{RTC\_I2C\_SCL\_HIGH\_PERIOD\_REG} 设置 RTC\_FAST\_CLK 周期中 SCL 时钟的高低电平宽度和周期（例，频率为 100 kHz 时，设置 \hyperref[fielddesc:RTCI2CSCLLOWPERIODREG]{RTC\_I2C\_SCL\_LOW\_PERIOD\_REG} = 40、 \hyperref[fielddesc:RTCI2CSCLHIGHPERIODREG]{RTC\_I2C\_SCL\_HIGH\_PERIOD\_REG} = 40）。

    \item 通过 RTC\_FAST\_CLK 中的 \hyperref[regdesc:RTCI2CSDADUTYREG]{RTC\_I2C\_SDA\_DUTY\_REG} 设置 SDA 切换前等待的周期数（例，\hyperref[regdesc:RTCI2CSDADUTYREG]{RTC\_I2C\_SDA\_\\DUTY\_REG} = 16）。

    \item 通过 \hyperref[regdesc:RTCI2CSCLSTARTPERIODREG]{RTC\_I2C\_SCL\_START\_PERIOD\_REG} 设置启动信号后的等待时间（例，\hyperref[regdesc:RTCI2CSCLSTARTPERIODREG]{RTC\_I2C\_SCL\_START\_PERIOD\_REG} = 30）。

    \item 通过 \hyperref[regdesc:RTCI2CSCLSTOPPERIODREG]{RTC\_I2C\_SCL\_STOP\_PERIOD\_REG} 设置停止信号前的等待时间（例，\hyperref[regdesc:RTCI2CSCLSTOPPERIODREG]{RTC\_I2C\_SCL\_STOP\_PERIOD\_REG} = 44）。

    \item 通过 \hyperref[fielddesc:RTCI2CTIMEOUTREG]{RTC\_I2C\_TIME\_OUT\_REG} 设置通信超时参数（例，\hyperref[fielddesc:RTCI2CTIMEOUTREG]{RTC\_I2C\_TIME\_OUT\_REG} = 200）。

    \item 通过 \hyperref[regdesc:RTCI2CCTRLREG]{RTC\_I2C\_CTRL\_REG} 中的 \hyperref[fielddesc:RTCI2CMSMODE]{RTC\_I2C\_MS\_MODE} 位启动主机模式。

    \item 配置外部从机地址：
    \begin{itemize}
        \item 如果选择使用主 CPU 或 ULP-RISC-V，则需要将外部从机的地址写入 \hyperref[fielddesc:SENSSARI2CSLAVEADDR]{SENS\_SAR\_I2C\_SLAVE\_ADDR}。
        \item 如果选择使用 ULP-FSM，则需要将外部从机的地址写入 SENS\_I2C\_SLAVE\_ADDR\regindex{{\it{n}}} (\regindex{{\it{n}}}: 0-7)。
    \end{itemize}
    最多可通过这种方式预编程 8 个从机地址。此后，可为每次通信选择 1 个上述地址，共同组成协处理器 I2C 指令。
\end{enumerate}

完成上述 RTC I2C 初始化配置后，即可由主 CPU 或者协处理器开始与外部 I2C 发起通信。

\subsection{使用 RTC I2C}

\subsubsection{I2C 指令编码格式}

RTC I2C 的指令编码与 I2C0/I2C1 的编码方式基本一致（见 I2C 控制器章节的 CMD\_Controller）。不同的地方在于：
\begin{itemize}
    \item RTC I2C 的 op\_code 对应的命令不同：
    \begin{itemize}
        \item RSTART: op\_code 等于 0 时为 RSTART 命令
        \item WRITE: op\_code 等于 1 时为 WRITE 命令
        \item READ: op\_code 等于 2 时为 READ 命令
        \item STOP: op\_code 等于 3 时为 STOP 命令
        \item END: op\_code 等于 4 时为 END 命令
    \end{itemize}
    \item RTC I2C 为不同的操作设置了固定的指令段，具体如下
    \begin{itemize}
        \item 命令 0 \verb+~+ 命令 1：I2C 写操作
        \item 命令 2 \verb+~+ 命令 6：I2C 读操作
    \end{itemize}
\end{itemize}
\textbf{注意：}
所有从机地址均为 7 位。


\subsubsection{I2C\_RD - I2C 读流程}

执行 I2C 读取操作之前，需配置以下信息：

\begin{itemize}
    \item 根据需求配置 I2C 的指令列表，包括指令顺序、指令码和读取数据个数 (byte\_num) 等信息。配置方法见 I2C 控制器章节中 I2C0/I2C1 的配置方法。
    \item 使用寄存器  \hyperref[fielddesc:SENSSARI2CREGADDR]{SENS\_SAR\_I2C\_REG\_ADDR} 配置从机寄存器地址。
    \item 置位 \hyperref[fielddesc:SENSSARI2CSTARTFORCE]{SENS\_SAR\_I2C\_START\_FORCE}，以及 \hyperref[fielddesc:SENSSARI2CSTART]{SENS\_SAR\_I2C\_START} 开始 I2C 的传输。
    \item 每接收到 RTC\_I2C\_RX\_DATA\_INT 中断，将读取的数据 (RTC\_I2C\_RDATA) 转存至 SRAM RTC 慢速内存中或直接使用。
\end{itemize}

I2C\_RD 指令的执行步骤如下，见图 \ref{fig:I2C-read-operation}：

\begin{enumerate}
    \item 主机发送启动 (START) 信号；
    \item 主机发送命令字节，包括从机地址和读/写控制位（此时，读/写控制位置为 0，代表“写”）。从机地址可从 SENS\_I2C\_SLAVE\_ADDR\regindex{{\it{n}}} 中获取；
    \item 从机发送应答 (ACK) 信号；
    \item 主机发送从机寄存器地址；
    \item 从机发送应答信号；
    \item 主机发送重复启动 (RSTART) 信号；
    \item 主机发送从机地址，其中读/写控制位置为 1，代表“读”；
    \item 从机发送 1 个字节的数据；
    \item 主机判断传输字节数是否达到当前指令设定的传输字节数。如果达到规定字数，则从读指令中跳出，主机发送非应答信号。否则重复步骤 8，继续等待从机发送下一个字节。
    \item 进入停止指令，主机发送停止 (STOP) 信号，结束读取。
\end{enumerate}

\subsubsection{I2C\_WR - I2C 写流程}

执行 I2C 写操作之前，需配置以下信息：

\begin{itemize}
    \item 根据需求配置 I2C 的指令列表，包括指令顺序、指令码和待写的数据个数 (byte\_num) 等信息。配置方法见 I2C 控制器章节中 I2C0/I2C1 的配置方法；
    \item 使用寄存器  \hyperref[fielddesc:SENSSARI2CREGADDR]{SENS\_SAR\_I2C\_REG\_ADDR} 配置从机寄存器地址；
    \item 使用寄存器 hyperref[fielddesc:SENSSARI2CWDATA]{SENS\_SAR\_I2C\_WDATA} 配置传输数据；
    \item 置位 \hyperref[fielddesc:SENSSARI2CSTARTFORCE]{SENS\_SAR\_I2C\_START\_FORCE}，以及 \hyperref[fielddesc:SENSSARI2CSTART]{SENS\_SAR\_I2C\_START} 开始 I2C 的传输；
    \item 每次当接收到 RTC\_I2C\_TX\_DATA\_INT 中断，更新下一个需要传输的数据 (hyperref[fielddesc:SENSSARI2CWDATA]{SENS\_SAR\_I2C\_WDATA})。
\end{itemize}


I2C\_WR 指令的执行步骤如下，见图 \ref{fig:I2C-write-operation}：

\begin{enumerate}
    \item 主机发送开始信号；
    \item 主机发送命令字节，包括从机地址和读/写控制位（此时，读/写控制位置为 0，代表“写”）。从机地址可从 SENS\_I2C\_SLAVE\_ADDR\regindex{{\it{n}}} 中获取；
    \item 从机发送应答信号；
    \item 进入下一条指令，主机发送从机寄存器地址；
    \item 从机发送应答信号；
    \item 主机发送重复启动信号；
    \item 主机发送从机地址，其中读/写位置为 0，代表“写”；
    \item 主机发送 1 个字节的数据；
    \item 从机发送应答信号；主机判断传输字节数是否达到当前指令设定的传输字节数，如果达到规定字数，则结束写指令跳转至下一条指令。否则重复步骤 8，继续发送下一个字节；
    \item 进入停止指令，主机发送停止指令，结束本次传输。
\end{enumerate}

\subsubsection{检测错误条件} \label{para:ULP-detecting-errors}
应用程序可以通过查询 \hyperref[regdesc:RTCI2CINTSTREG]{RTC\_I2C\_INT\_ST\_REG} 寄存器中的特定位，判断指令是否成功执行。为了检查特定的通信活动，应首先设置 \hyperref[regdesc:RTCI2CINTENAREG]{RTC\_I2C\_INT\_ENA\_REG} 寄存器中的相应位。注意，系统位图将移 1。如果检测到特定通信活动，且设置了 \hyperref[regdesc:RTCI2CINTSTREG]{RTC\_I2C\_INT\_ST\_REG} 寄存器，则可通过 \hyperref[regdesc:RTCI2CINTCLRREG]{RTC\_I2C\_INT\_CLR\_REG} 寄存器清零。

\subsection{RTC I2C 中断}\label{sec:ulp-interrupt}

\begin{itemize}
    \item \label{interrupt:RTCI2CSLAVETRANCOMPINT}RTC\_I2C\_SLAVE\_TRAN\_COMP\_INT：从机完成传输后则触发此中断。
    \item \label{interrupt:RTCI2CARBITRATIONLOSTINT}RTC\_I2C\_ARBITRATION\_LOST\_INT：主机失去总线控制权后则触发此中断。
    \item \label{interrupt:RTCI2CMASTERTRANCOMPINT}RTC\_I2C\_MASTER\_TRAN\_COMP\_INT：主机传输完成后则触发此中断。
    \item \label{interrupt:RTCI2CTRANSCOMPLETEINT}RTC\_I2C\_TRANS\_COMPLETE\_INT：检测到 STOP 位时则触发此中断。
    \item \label{interrupt:RTCI2CTIMEOUTINT}RTC\_I2C\_TIME\_OUT\_INT：出现超时事件则触发此中断。
    \item \label{interrupt:RTCI2CACKERRINT}RTC\_I2C\_ACK\_ERR\_INT：出现 ACK 错误则触发此中断。
    \item \label{interrupt:RTCI2CRXDATAINT}RTC\_I2C\_RX\_DATA\_INT：接收数据则触发此中断。
    \item \label{interrupt:RTCI2CTXDATAINT}RTC\_I2C\_TX\_DATA\_INT：发送数据则触发此中断。
    \item \label{interrupt:RTCI2CDETECTSTARTINT}RTC\_I2C\_DETECT\_START\_INT：检测到开始信号则触发此中断。
\end{itemize}
