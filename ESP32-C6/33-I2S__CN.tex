\chapter{I2S 控制器 (I2S)}
\label{mod:i2s}


\section{概述}
\chipname{} 内置一个 I2S 接口，为多媒体应用，尤其是为数字音频应用提供了灵活的数据通信接口。

I2S 标准总线定义了三种信号：串行时钟信号 BCK、字选择信号 WS 和串行数据信号 SD。一个基础的 I2S 数据总线包含一个主机和一个从机。主机和从机的角色在通信过程中保持不变。\chipname{} 的 I2S 模块包含独立的发送单元和接收单元，能够保证优良的通信性能。

\section{术语}

为了更好地说明 I2S 的功能，本章使用了以下术语。 


\begin{longtable}[c]{ R{4cm} p{10cm} }

\textbf{主机模式}     & I2S 作为主机驱动 BCK/WS，向从机发送或从其接收数据。 \\ 
\textbf{从机模式}      & I2S 作为从机由 BCK/WS 驱动，从主机接收或向其发送数据。 \\
\textbf{全双工} & 主机与从机之间的发送线和接收线各自独立，发送数据和接收数据同时进行。\\
\textbf{半双工} & 主机和从机只能有一方先发送数据，另一方接收数据。发送数据和接收数据不能同时进行。 \\
\textbf{A 律和 $\mu$ 律} & A 律和 \textrm{$\mu$} 律是数字脉冲编码调制 (PCM) 非均匀量化中的压缩/解压缩算法，可以有效地改善信号的量化信噪比。 \\
\textbf{TDM RX 模式} & 即 TDM 输入模式，利用时分复用方式接收脉冲编码调制 (PCM) 数据，并将其通过 DMA 存入储存器的模式。信号线包括 BCK、WS 和 DATA。可以接收最多 16 个通道的数据。通过用户配置，可支持 TDM Philips 格式、TDM MSB 对齐格式、TDM PCM 格式等。\\
\textbf{普通 PDM RX 模式} & 即普通 PDM 输入模式，接收脉冲密度调制 (PDM) 数据，并将其通过 DMA 存入储存器的模式。信号线包括 WS 和 DATA。通过用户配置，可支持 PDM 标准格式等。\\
\textbf{TDM TX 模式} & 即 TDM 输出模式，通过 DMA 从储存器中取得脉冲编码调制 (PCM) 数据，并利用时分复用方式将其发送的模式。信号线包括 BCK、WS 和 DATA，可以发送最多 16 个通道的数据。通过用户配置，可支持 TDM Philips 格式、TDM MSB 对齐格式、TDM PCM 格式等。\\
\textbf{普通 PDM TX 模式} & 即普通 PDM 输出模式，通过 DMA 从储存器中取得脉冲密度调制 (PDM) 数据，并将其发送的模式。信号线包括 WS 和 DATA。通过用户配置，可支持 PDM 标准格式等。\\
\textbf{PCM-to-PDM TX 模式} & 即 PCM-to-PDM 输出模式，通过 DMA 从储存器中取得脉冲编码调制 (PCM) 数据，将其转换为脉冲密度调制 (PDM) 数据，并将其发送的\textbf{主机}模式。信号线包括 WS 和 DATA。通过用户配置，可支持 PDM 标准格式等。\\
\end{longtable}

\section{特性}

I2S 模块具有以下特性：

\begin{itemize}
    \item 支持主机模式和从机模式
    \item 支持全双工和半双工通信
    \item 支持 TX 模块和 RX 模块独立工作或同时工作
    \item 支持多种音频标准：
        \begin{itemize}    
            \item TDM Philips 标准
            \item TDM MSB 对齐标准
            \item TDM PCM 标准
            \item PDM 标准
        \end{itemize}
    \item 可配置高精度采样时钟：
        \begin{itemize}
            \item 支持如下采样频率：8 kHz、16 kHz、32 kHz、44.1 kHz、48 kHz、88.2 kHz、96 kHz、128 kHz 和 192 kHz（注意：从机模式下，由于时钟源的频率限制，支持的最高采样频率受到数据位宽和声道数的限制，具体请参考章节 \ref{sec:i2s-tx-rx-clk}）
        \end{itemize}
    \item 支持 8/16/24/32 位数据通信
    \item 支持 DMA
    \item 支持 I2S 接口中断
\end{itemize}

图 \ref{fig:i2s-sys-diagram} 为 \chipname{} I2S 模块的结构框图。\chipname{} I2S 模块包含：
\begin{itemize}
    \item 独立的发送单元 (TX Unit)
    \item 独立的接收单元 (RX Unit)
    \item 输入输出时序调节单元 (I/O Sync)
    \item 时钟分频器 (Clock Generator)
    \item 64 x 32-bit TX FIFO
    \item 64 x 32-bit RX FIFO
    \item 压缩/解压缩模块 (Compress/Decompress) 
\end{itemize}
\chipname{} I2S 模块支持 DMA，可访问内部存储器，更多信息见章节 \ref{mod:dma} \textit{\nameref{mod:dma}}。

发送单元和接收单元各自有一组三线接口，分别为串行时钟线 BCK、字选择线 WS 和串行数据线 SD。其中，发送单元的 SD 线固定为输出，接收单元的 SD 线固定为接收。发送单元和接收单元的 BCK 和 WS 信号线均可配置为主机输出模式或从机输入模式。

图 \ref{fig:i2s-sys-diagram} 中的右半部分展示了 I2S 模块的信号总线。RX 和 TX 模块的信号命名规则为：I2S\textcolor{red}{A}\_\textcolor{red}{B}\_\textcolor{red}{C}，例如 I2S\textcolor{red}{I}\_\textcolor{red}{BCK}\_\textcolor{red}{in}。其中：

\begin{itemize}
    \item ``A" 表示 I2S 模块的数据总线的方向，包括：
    \begin{itemize}
        \item ``I" 表示输入
        \item ``O" 表示输出
    \end{itemize}
    \item ``B" 表示信号功能，包括：
    \begin{itemize}
        \item BCK
        \item WS
        \item SD
    \end{itemize}
    \item ``C" 表示该信号的方向，包括：
    \begin{itemize}
        \item ``in" 表示该信号输入 I2S 模块
        \item ``out" 表示该信号自 I2S 模块输出
    \end{itemize}
\end{itemize}

\section{I2S 模块支持的音频协议} \label{sec:i2s-supported-audio-standards}

\chipname{} I2S 模块支持多种音频标准，包括 TDM Philips 标准、TDM MSB 对齐标准、TDM PCM 标准以及 PDM 标准。

用户可通过配置以下寄存器，选择所需的音频标准：
\begin{itemize}
    \item \hyperref[fielddesc:I2SRXMSBSHIFT]{I2S\_TX/RX\_TDM\_EN}
    \begin{itemize}
        \item 0：禁用 TDM 模式
        \item 1：选择 TDM 模式
    \end{itemize}
    \item \hyperref[fielddesc:I2SRXMSBSHIFT]{I2S\_TX/RX\_PDM\_EN}
    \begin{itemize}
        \item 0：禁用 PDM 模式
        \item 1：选择 PDM 模式
    \end{itemize}    
    \item \hyperref[fielddesc:I2SRXMSBSHIFT]{I2S\_TX/RX\_MSB\_SHIFT}
    \begin{itemize}
        \item 0：配置 WS 信号和 SD 信号同时开始变化，即选择 MSB 对齐标准
        \item 1：配置 WS 信号先于 SD 信号一个 BCK 时钟周期开始变化，即选择 Philips 标准或 PCM 标准
    \end{itemize}
    \item \hyperref[fielddesc:I2SRXPCMBYPASS]{I2S\_TX/RX\_PCM\_BYPASS}
    \begin{itemize}
        \item 0：选择 PCM 标准
        \item 1：禁用 PCM 标准
    \end{itemize}
\end{itemize}

\subsection{TDM Philips 标准模式}

在 Philips 标准下，在 BCK 的下降沿，WS 信号先于 SD 信号一个 BCK 时钟周期开始变化，即 WS 信号从当前通道数据的第一个位之前的一个时钟开始有效，并在当前通道数据发送结束前一个 BCK 时钟周期开始变化。SD 信号线上首先传输音频数据的最高位。

与 Philips 标准相比，TDM Philips 标准支持的通道更多，见图 \ref{fig:i2s-tdm-philips-std-timing-diagram}。
\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/i2s_tdm_phillips_mode.png}
    \caption{时序图 -- TDM Philips 标准}
    \label{fig:i2s-tdm-philips-std-timing-diagram}
\end{figure}


\subsection{TDM MSB 对齐标准模式}

MSB 对齐标准下，在 BCK 下降沿，WS 信号和 SD 信号同时变化。WS 持续到当前通道数据发送结束，SD 信号线上首先传输音频数据的最高位。

与 MSB 对齐标准相比，TDM MSB 对齐标准支持更多通道，见图 \ref{fig:i2s-tdm-msb-align-std-timing-diagram}。

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/i2s_tdm_msb_mode.png}
    \caption{时序图 -- TDM MSB 对齐标准}
    \label{fig:i2s-tdm-msb-align-std-timing-diagram}
\end{figure}


\subsection{TDM PCM 标准模式}

在 PCM 标准的短帧同步模式下，在 BCK 的下降沿，WS 信号先于 SD 信号一个 BCK 时钟周期开始变化，即 WS 信号从当前通道数据的第一个位之前的一个时钟开始有效，并持续一个 BCK 时钟周期。SD 信号线上首先传输音频数据的最高位。

与 PCM 标准相比，TDM PCM 标准支持更多通道，见图 \ref{fig:i2s-tdm-pcm-std-timing-diagram} 所示。

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/i2s_tdm_pcm_mode.png}
    \caption{时序图 -- TDM PCM 标准}
    \label{fig:i2s-tdm-pcm-std-timing-diagram}
\end{figure}


\subsection{PDM 标准模式}
如图 \ref{fig:i2s-pdm-std-timing-diagram} 所示，在 PDM 标准下，WS 代表左/右声道，在 BCK 的下降沿，WS 与 SD 同时变化。WS 在数据发送过程中持续变化，WS 的高低对应两个声道。

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/i2s_pdm_mode.png}
    \caption{时序图 -- PDM 标准}
    \label{fig:i2s-pdm-std-timing-diagram}
\end{figure}

\section{I2S TX/RX 模块时钟}\label{sec:i2s-tx-rx-clk}


I2S\_TX/RX\_CLK 作为 I2S TX/RX 模块的主时钟，可由以下时钟分频所得：
\begin{itemize}
    \item 40 MHz XTAL\_CLK 
    \item 160 MHz PLL\_F160M\_CLK
    \item 240 MHz PLL\_F240M\_CLK
    \item 或外部输入时钟：I2S\_MCLK\_in 
\end{itemize}
I2S TX/RX 模块的串行时钟 BCK 再由 I2S\_TX/RX\_CLK 分频获得，如图 \ref{fig:i2s-clk} 所示。\hyperref[fielddesc:PCRI2SRXCLKMSEL]{PCR\_I2S\_TX/RX\_CLKM\_SEL} 用于选择 I2S TX/RX 的时钟源，\hyperref[fielddesc:PCRI2SRXCLKMEN]{PCR\_I2S\_TX/RX\_CLKM\_EN} 用于使能或者关闭 I2S TX/RX 模块的时钟源。

\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.6\textwidth]{\modulefiles/figures/i2s_clk_20221122}}
    \caption{I2S 时钟分频器}
    \label{fig:i2s-clk}
\end{figure}

I2S\_TX/RX\_CLK 的频率 $f_{\textrm{I2S\_TX/RX\_CLK}}$ 与分频器时钟源频率 $f_{\textrm{I2S\_CLK\_S}}$ 间的关系如下：
$$f_{\textrm{I2S\_TX/RX\_CLK}}=\frac{f_{\textrm{I2S\_CLK\_S}}}{\textrm{N} + \frac{\textrm{b}}{\textrm{a}}}$$

其中，2 =< N <= 256（N 为整数），N 对应为 \hyperref[regdesc:PCRI2STXCLKMCONFREG]{PCR\_I2S\_TX/RX\_CLKM\_CONF\_REG} 寄存器中 \hyperref[fielddesc:PCRI2STXCLKMDIVNUM]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_NUM} 的值，具体为：

\begin{itemize}
    \item \hyperref[fielddesc:PCRI2STXCLKMDIVNUM]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_NUM} = 0 时，N = 256；
    \item \hyperref[fielddesc:PCRI2STXCLKMDIVNUM]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_NUM} = 1 时，N = 2；
    \item \hyperref[fielddesc:PCRI2STXCLKMDIVNUM]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_NUM} 为其他值时，N = \hyperref[fielddesc:PCRI2STXCLKMDIVNUM]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_NUM} 的值。

\end{itemize}

分数部分分频系数 a、b 唯一对应系数 x、y、z、yn1。系数对应公式为：
\begin{itemize}
    \item 当 $\textrm{b} <= \frac{\textrm{a}}{2}$ 时，$\textrm{yn1} = 0$，$\textrm{x} = floor([\frac{\textrm{a}}{\textrm{b}}]) - 1$，$\textrm{y} = \textrm{a}\%\textrm{b}$，$\textrm{z} = \textrm{b}$；
    \item 当 $\textrm{b} > \frac{\textrm{a}}{2}$ 时，$\textrm{yn1}=1$，$\textrm{x} = floor([\frac{\textrm{a}}{\textrm{a - b}}]) - 1$，$\textrm{y} = \textrm{a}\%\textrm{(a - b)}$，$\textrm{z}=\textrm{a - b}$。
\end{itemize}

系数 x、y、z、yn1 在 \hyperref[fielddesc:PCRI2STXCLKMDIVX]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_X}、\hyperref[fielddesc:PCRI2STXCLKMDIVY]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_Y}、\hyperref[fielddesc:PCRI2STXCLKMDIVZ]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_Z}、\hyperref[fielddesc:PCRI2STXCLKMDIVYN1]{PCR\_I2S\_TX/RXCLKM\_DIV\_YN1} 中配置。 \\
对于整数分频，\hyperref[fielddesc:PCRI2STXCLKMDIVX]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_X} 和 \hyperref[fielddesc:PCRI2STXCLKMDIVZ]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_Z} 清零，\hyperref[fielddesc:PCRI2STXCLKMDIVY]{PCR\_I2S\_TX/RX\_CLKM\_DIV\_Y} 置为 1。

\begin{tiplisting}
使用小数分频功能可能会产生时钟抖动。
\end{tiplisting}

在主机发送模式下，I2S TX 模块的串行时钟 BCK 为 I2SO\_BCK\_out 信号，由 I2S\_TX\_CLK 分频获得。即：

$$f_{\textrm{I2SO\_BCK\_out}} = \frac{f_{\textrm{I2S\_TX\_CLK}}}{\textrm{MO}}$$


其中，MO 的值为 \hyperref[fielddesc:I2STXBCKDIVNUM]{I2S\_TX\_BCK\_DIV\_NUM} 的值 + 1，即：
$$ \textrm{MO} = \textrm{\hyperref[fielddesc:I2STXBCKDIVNUM]{I2S\_TX\_BCK\_DIV\_NUM}} + 1 $$

\vspace{-1em}
\begin{tiplisting}
注意，\hyperref[fielddesc:I2STXBCKDIVNUM]{I2S\_TX\_BCK\_DIV\_NUM} 不可配置为 1。
\end{tiplisting}


在主机接收模式下，I2S RX 模块的串行时钟 BCK 为 I2SI\_BCK\_out 信号，由 I2S\_RX\_CLK 分频获得。即：

$$f_{\textrm{I2SI\_BCK\_out}} = \frac{f_{\textrm{I2X\_RX\_CLK}}}{\textrm{MI}}$$

其中 MI 对应 \hyperref[fielddesc:I2SRXBCKDIVNUM]{I2S\_RX\_BCK\_DIV\_NUM} 的值 + 1，即：
$$ \textrm{MI} = \textrm{\hyperref[fielddesc:I2STXBCKDIVNUM]{I2S\_RX\_BCK\_DIV\_NUM}} + 1 $$

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
    \item \hyperref[fielddesc:I2SRXBCKDIVNUM]{I2S\_RX\_BCK\_DIV\_NUM} 不可配置为 1；
    \item 当模块处于 I2S 从机模式时，必须保证 $f_{\textrm{I2S\_TX/RX\_CLK}}$ >= 8 * $f_{\textrm{BCK}}$。另外模块可以输出 I2S\_MCLK\_out 作为外部设备的主时钟。
\end{itemize}
\end{tiplisting}

\section{I2S 模块复位} \label{sec:i2s-reset}

I2S 模块中各个单元以及 FIFO 可通过配置相关位进行复位：
\begin{itemize}
\item I2S TX/RX 单元：可配置 \hyperref[fielddesc:I2STXRESET]{I2S\_TX\_RESET} 和 \hyperref[fielddesc:I2SRXRESET]{I2S\_RX\_RESET} 进行复位；
\item I2S TX/RX FIFO：可配置 \hyperref[fielddesc:I2STXFIFORESET]{I2S\_TX\_FIFO\_RESET} 和 \hyperref[fielddesc:I2SRXFIFORESET]{I2S\_RX\_FIFO\_RESET} 进行复位。
\end{itemize}


\begin{tiplisting}
在模块和 FIFO 复位之前，需要先配置 I2S 模块时钟。
\end{tiplisting} 
 

\section{I2S 主/从机模式}

\chipname{} I2S 模块可作为主机或从机，两种模式均支持半双工通信或全双工通信。用户可配置  \hyperref[fielddesc:I2SRXSLAVEMOD]{I2S\_RX\_SLAVE\_MOD} 和 \hyperref[fielddesc:I2STXSLAVEMOD]{I2S\_TX\_SLAVE\_MOD} 选择需要的模式。

\begin{itemize}
    \item \hyperref[fielddesc:I2STXSLAVEMOD]{I2S\_TX\_SLAVE\_MOD} 
    \begin{itemize}
        \item 0：主机发送模式
        \item 1：从机发送模式
    \end{itemize}
    \item \hyperref[fielddesc:I2SRXSLAVEMOD]{I2S\_RX\_SLAVE\_MOD}
    \begin{itemize}
        \item 0：主机接收模式
        \item 1：从机接收模式
    \end{itemize}
\end{itemize}

\subsection{主/从机发送模式} \label{sec:i2s-master-slave-tx-mode}
\begin{itemize}
    \item 主机发送模式
    \begin{itemize}
        \item 置位 \hyperref[fielddesc:I2STXSTART]{I2S\_TX\_START} 启动一次发送操作。
        \item 置位该位，发送单元会一直输出时钟信号和串行数据。
        \item 置位 \hyperref[fielddesc:I2STXSTOPEN]{I2S\_TX\_STOP\_EN} 时，如果 FIFO 中的数据全部发送完毕，则主机停止发送数据和时钟信号。
        \item 清零 \hyperref[fielddesc:I2STXSTOPEN]{I2S\_TX\_STOP\_EN} 时，如果 FIFO 中的数据全部发送完毕，并且没有新数据填入，发送模块将一直发送最后一帧数据和时钟信号。
        \item 当 \hyperref[fielddesc:I2STXSTART]{I2S\_TX\_START} 被清零时，主机停止发送数据。
    \end{itemize}
    \item 从机发送模式
     \begin{itemize}
        \item 置位 \hyperref[fielddesc:I2STXSTART]{I2S\_TX\_START}。
        \item 发送单元等待主机 BCK 时钟，来启动发送操作。
        \item 置位 \hyperref[fielddesc:I2STXSTOPEN]{I2S\_TX\_STOP\_EN} 时，如果 FIFO 中的数据全部发送完毕，则从机发送数据一直为零，直到主机停止发送 BCK 时钟为止。
        \item 清零 \hyperref[fielddesc:I2STXSTOPEN]{I2S\_TX\_STOP\_EN} 时，如果 FIFO 中的数据全部发送完毕，并且没有新数据填入，发送单元将一直发送最后一帧数据。
        \item 当 \hyperref[fielddesc:I2STXSTART]{I2S\_TX\_START} 被清零时，从机发送数据一直为零，直到主机停止发送 BCK 时钟为止。
    \end{itemize}
\end{itemize}

\subsection{主/从机接收模式}
\begin{itemize}
    \item 主机接收模式
    \begin{itemize}
        \item 置位 \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} 启动一次接收操作。
        \item 接收单元会一直输出时钟信号，并对输入数据进行采样。
        \item 清零 \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START}，接收单元停止接收数据。
    \end{itemize}
    \item 从机接收模式
     \begin{itemize}
        \item 置位 \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START}。
        \item 等待主机 BCK 时钟，来启动接收操作。
        \item 清零 \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START}，接收单元停止接收数据。
    \end{itemize}
\end{itemize}

\section{发送数据}\label{sec:i2s-tx-data}

\vspace{1em}
\begin{tiplisting}
本小节以及后续小节所述的配置，均需要通过置位 \hyperref[fielddesc:I2STXUPDATE]{I2S\_TX\_UPDATE} 的方式来进行更新，从而将 I2S TX 寄存器数据从 APB 时钟域同步到 I2S TX 时钟域。详细配置见第 \ref{sec:i2s-configure-i2s-tx-mode} 小节。
\end{tiplisting}
\vspace{-1em}
\chipname{} I2S 发送数据时，从 DMA 读取数据，经过数据格式控制和通道模式控制，从外设输出信号输出对应数据。
\subsection{数据格式控制}\label{sec:i2s-data-control}
数据格式控制分为三个阶段：
\begin{itemize}
    \item 第一阶段：从内存中读出有效数据并写入 TX FIFO；
    \item 第二阶段：将待发送数据从 TX FIFO 中读出，并进行输出数据模式转换；
    \item 第三阶段：将待发送数据转换为串行数据流输出。
\end{itemize}

\hyperref[fielddesc:I2STXBITSMOD]{I2S\_TX\_BITS\_MOD} 和 \hyperref[fielddesc:I2STX24FILLEN]{I2S\_TX\_24\_FILL\_EN} 决定了每个通道的有效数据位宽，其可取值和对应的有效数据位宽如下表：

\end{tiplisting}

\subsubsection{A 率/$\mu$ 率压缩/解压缩}
\chipname{} I2S 按照 32-bit（缺省高位补 0$^*$）的方式对排列好字节序的有效数据进行 A 率/$\mu$ 率压缩/解压缩。
\vspace{-1em}
\begin{tiplisting}
缺省高位补 0，即如果有效数据位宽小于 32，则待压缩/解压缩数据的 [31: 有效位宽] 位自动填充 0。
\end{tiplisting}

配置 \hyperref[fielddesc:I2STXPCMBYPASS]{I2S\_TX\_PCM\_BYPASS} 为：
\begin{itemize}
\item 0：进行压缩/解压缩
\item 1：不进行压缩/解压缩
\end{itemize}
配置 \hyperref[fielddesc:I2STXPCMCONF]{I2S\_TX\_PCM\_CONF} 为：
\begin{itemize}
\item 0：A 律解压缩
\item 1：A 律压缩
\item 2：$\mu$ 律解压缩
\item 3：$\mu$ 律压缩
\end{itemize}
至此，数据格式控制的第一阶段完成。

\subsubsection{通道发送数据位宽} 
\chipname{} I2S 中，\hyperref[fielddesc:I2STXTDMCHANBITS]{I2S\_TX\_TDM\_CHAN\_BITS} 决定了每个通道发送数据的位宽。

\begin{itemize}
\item 当每个通道发送数据的位宽大于有效数据位宽时，会在剩余的位置补充 0。此时，配置 \hyperref[fielddesc:I2STXLEFTALIGN]{I2S\_TX\_LEFT\_ALIGN} 为：
\begin{itemize}
\item 0：有效数据位于发送数据低位，在数据高位补零；
\item 1：有效数据位于发送数据高位，在数据低位补零。
\end{itemize}
\item 当每个通道发送数据的位宽小于有效数据位宽时，每次发送数据仅发送低位有效数据部分，高位部分将被舍弃。
\end{itemize}
至此，数据格式控制的第二阶段完成。

\subsubsection{通道数据比特顺序}
\chipname{} I2S 通道数据比特顺序由 \hyperref[fielddesc:I2STXBITORDER]{I2S\_TX\_BIT\_ORDER} 控制：
\begin{itemize}
\item 0：不调换有效数据比特顺序；
\item 1：调换有效数据比特顺序。
\end{itemize}
至此，数据格式控制部分全部完成。经过数据格式控制的数据将从高位向低位依次发送。图 \ref{fig:i2s-tx-data-format-control} 所示即为一次完整的 TX 数据格式控制过程。

\end{figure}


\subsection{通道模式控制}
\chipname{} I2S 支持 TDM 和 PDM 两种发送模式。置位 \hyperref[fielddesc:I2STXTDMEN]{I2S\_TX\_TDM\_EN} 使能 TDM 发送模式，置位 \hyperref[fielddesc:I2STXPDMEN]{I2S\_TX\_PDM\_EN} 使能 PDM 发送模式。

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
    \item \hyperref[fielddesc:I2STXTDMEN]{I2S\_TX\_TDM\_EN} 和 \hyperref[fielddesc:I2STXPDMEN]{I2S\_TX\_PDM\_EN} 不能同时置位或同时清零。 
    \item 将 I2S 模块设置为 TDM 双通道模式，可实现控制大多数 I2S 双声道编解码器。
    \end{itemize}
\end{tiplisting}

\subsubsection{TDM 输出模式下 I2S 通道模式} \label{sec:i2s-chan-control-tdm-mode}
在 TDM 输出模式下，I2S 支持的发送通道数与通道有效数据位宽有如下关系：

\begin{table}[H]
    \centering
    \caption{支持通道数与通道有效数据位宽关系}
    \label{tab:i2s-tx-support-channel}
    \begin{tabular}{|l|r|}
    \hline
    \rowcolor{lightgray}
    \textbf{通道有效数据位宽} & \textbf{最多支持的通道数} \\ \hline
        32  & 4     \\\hline
        24  & 5     \\\hline
        16  & 8     \\\hline
        8   & 16    \\\hline

发送通道数由 \hyperref[fielddesc:I2STXTDMTOTCHANNUM]{I2S\_TX\_TDM\_TOT\_CHAN\_NUM} 控制。例如，配置 \hyperref[fielddesc:I2STXTDMTOTCHANNUM]{I2S\_TX\_TDM\_TOT\_CHAN\_NUM} 为 5，则六个通道（通道 0 \verb+~+ 5）将用于发送数据，见图
\ref{fig:i2s-tdm-chan-control}。 

在发送数据的通道中，如果其对应的 \hyperref[fielddesc:I2STXTDMCHAN0EN]{I2S\_TX\_TDM\_CHAN{\textcolor{red}{n}}\_EN} 为：

\begin{itemize}
    \item 1：则该通道发送通道数据；
    \item 0：则该通道发送数据由 \hyperref[fielddesc:I2Ssec:i2s-tx-dataEQUAL]{I2S\_TX\_CHAN\_EQUAL} 控制，当该值为：
    \begin{itemize}
        \item 1：则发送上个通道的数据；
        \item 0：则发送 \hyperref[fielddesc:I2SSINGLEDATA]{I2S\_SINGLE\_DATA} 的值。
    \end{itemize}
\end{itemize}
当 I2S 处于主机 TDM 输出模式下，WS 信号由 \hyperref[fielddesc:I2STXWSIDLEPOL]{I2S\_TX\_WS\_IDLE\_POL} 和 \hyperref[fielddesc:I2STXTDMWSWIDTH]{I2S\_TX\_TDM\_WS\_WIDTH} 控制：
    \begin{itemize}
        \item \hyperref[fielddesc:I2STXWSIDLEPOL]{I2S\_TX\_WS\_IDLE\_POL} 的值为 WS 信号的默认电平；
        \item \hyperref[fielddesc:I2STXTDMWSWIDTH]{I2S\_TX\_TDM\_WS\_WIDTH} 的值为在发送所有通道数据的过程中，WS 为默认电平的周期数。
    \end{itemize}

另外，\hyperref[fielddesc:I2STXHALFSAMPLEBITS]{I2S\_TX\_HALF\_SAMPLE\_BITS} 的值乘以 2，即为一个 WS 周期对应的 BCK 周期数。

    \item 配置 I2S\_TX\_CHAN\_EQUAL 为 1，即如果相应通道的 I2S\_TX\_TDM\_CHAN\regindex{n}\_EN 被置位，则该通道将发送上一通道的数据。\regindex{n} = 0 \verb+~+ 5。
    \item 配置 I2S\_TX\_TDM\_CHAN0/2/5\_EN 为 1，即这些通道将用于发送通道数据。
    \item 配置 I2S\_TX\_TDM\_CHAN1/3/4\_EN 为 0，则这些通道将发送上一个通道的数据。
\end{itemize}

\begin{itemize}
    \item 配置 \hyperref[fielddesc:I2SPCM2PDMCONVEN]{I2S\_PCM2PDM\_CONV\_EN} 为 0，即选择普通 PDM 输出模式。
    \item 配置 \hyperref[fielddesc:I2STXMONO]{I2S\_TX\_MONO} 为 0，即 I2S 在 WS 的高低电平均通过 DMA 从存储器取数。
    \item 配置 \hyperref[fielddesc:I2Ssec:i2s-tx-dataMOD]{I2S\_TX\_CHAN\_MOD} 为 2，即选择单声道模式，右声道数据将被舍弃。
    \item 配置 \hyperref[fielddesc:I2STXWSIDLEPOL]{I2S\_TX\_WS\_IDLE\_POL} 为 1，则左声道和右声道均发送左通道数据。
\end{itemize}

\section{接收数据}\label{sec:i2s-rx-data}


I2S 接收数据时，从外设接口读取数据，经过通道模式控制和数据格式控制，通过 DMA 输入数据至内存。

\subsection{通道模式控制}
\chipname{} I2S 支持 TDM 和 PDM 两种输入模式。置位 \hyperref[fielddesc:I2SRXTDMEN]{I2S\_RX\_TDM\_EN} 使能 TDM 输入模式，置位 \hyperref[fielddesc:I2SRXPDMEN]{I2S\_RX\_PDM\_EN} 使能 PDM 输入模式。

\begin{tiplisting}
\hyperref[fielddesc:I2SRXTDMEN]{I2S\_RX\_TDM\_EN} 和 \hyperref[fielddesc:I2SRXPDMEN]{I2S\_RX\_PDM\_EN} 不能同时置位或同时清零。
\end{tiplisting}

\subsubsection{TDM 输入模式下 I2S 通道模式}
在 TDM 输入模式下，I2S 支持的接收通道数与通道有效数据位宽有如下关系：

\begin{table}[H]
    \centering
    \caption{支持接收通道数与通道有效数据位宽关系}
    \label{tab:i2s-rx-support-channel}
    \begin{tabular}{|l|r|}
    \hline
    \rowcolor{lightgray}
    \textbf{通道有效数据位宽} & \textbf{最多支持的通道数} \\ \hline
        32  & 4     \\\hline
        24  & 5     \\\hline
        16  & 8     \\\hline
        8   & 16    \\\hline
    \end{tabular}
\end{table}

在 TDM 输入模式下，I2S 最多支持 16 个接收通道。接收通道数由 \hyperref[fielddesc:I2SRXTDMTOTCHANNUM]{I2S\_RX\_TDM\_TOT\_CHAN\_NUM} 控制。例如，配置 \hyperref[fielddesc:I2SRXTDMTOTCHANNUM]{I2S\_RX\_TDM\_TOT\_CHAN\_NUM} 为 5，则通道 0 \verb+~+ 5 接收数据。

在接收数据的通道中，如果其对应的 \hyperref[fielddesc:I2STXTDMCHAN0EN]{I2S\_RX\_TDM\_CHAN{\textcolor{red}{n}}\_EN} 为：
\begin{itemize}
    \item 0：则该通道数据无效，不会存入 RX FIFO；
    \item 1：则该通道数据有效，会被存入 RX FIFO。
\end{itemize}
当 I2S 处于主机 TDM 输入模式下，WS 信号由 \hyperref[fielddesc:I2SRXWSIDLEPOL]{I2S\_RX\_WS\_IDLE\_POL} 和 \hyperref[fielddesc:I2SRXTDMWSWIDTH]{I2S\_RX\_TDM\_WS\_WIDTH} 控制：
\begin{itemize}
    \item \hyperref[fielddesc:I2SRXWSIDLEPOL]{I2S\_RX\_WS\_IDLE\_POL} 的值为 WS 信号的默认电平；
    \item \hyperref[fielddesc:I2SRXTDMWSWIDTH]{I2S\_RX\_TDM\_WS\_WIDTH} 的值为在接收所有通道的过程中，WS 为默认电平的周期数。
\end{itemize}
另外，\hyperref[fielddesc:I2SRXHALFSAMPLEBITS]{I2S\_RX\_HALF\_SAMPLE\_BITS} 的值乘以 2，即为一个 WS 周期对应的 BCK 周期数。

\subsubsection{PDM 输入模式下 I2S 通道模式} 
在 PDM 输入模式下，I2S 将通道中的串行数据转换为待输入数据。

当 I2S 处于主机 PDM 输入模式下，WS 信号的默认电平由 \hyperref[fielddesc:I2SRXWSIDLEPOL]{I2S\_RX\_WS\_IDLE\_POL} 控制，WS 信号频率为BCK信号频率的一半，请参考 \ref{sec:i2s-tx-rx-clk} 小节配置 BCK 信号的方式配置 WS 信号。注意，在 PDM 输入模式下，请配置 \hyperref[fielddesc:I2SRXHALFSAMPLEBITS]{I2S\_RX\_HALF\_SAMPLE\_BITS} 的值与 \hyperref[fielddesc:I2SRXBITSMOD]{I2S\_RX\_BITS\_MOD} 的值相同。

\subsection{数据格式控制}
数据格式控制分为两个阶段：
\begin{itemize}
    \item 第一阶段：从串行数据流输入转换为待输入数据，并存入 RX FIFO；
    \item 第二阶段：将待输入数据从 RX FIFO 中读出，并进行输入数据模式转换。
\end{itemize}

\subsubsection{通道数据比特顺序} 
\chipname{} I2S 通道数据将从高位到低位依次存入待输入数据，并由 \hyperref[fielddesc:I2SRXBITORDER]{I2S\_RX\_BIT\_ORDER} 控制是否调换比特序，当该值为：
\begin{itemize}
\item 0：不调换待输入数据的比特序；
\item 1：调换待输入数据的比特序。
\end{itemize}
至此，数据格式控制的第一阶段完成。

\subsubsection{通道储存数据位宽}
\hyperref[fielddesc:I2SRXBITSMOD]{I2S\_RX\_BITS\_MOD} 和 \hyperref[fielddesc:I2SRX24FILLEN]{I2S\_RX\_24\_FILL\_EN} 决定了每个通道的储存数据位宽，其可取值和对应的储存数据位宽如下表：

    \hline
    \rowcolor{lightgray}
    \textbf{通道储存数据位宽} & \textbf{\hyperref[fielddesc:I2SRXBITSMOD]{I2S\_RX\_BITS\_MOD}} & \textbf{\hyperref[fielddesc:I2SRX24FILLEN]{I2S\_RX\_24\_FILL\_EN}}  \\ \hline
    \multirow{2}*{32} & 31 & x \\\cline{2-3}

    8     & 7 & x \\\hline

\subsubsection{通道接收数据位宽} 
\chipname{} I2S 中，\hyperref[fielddesc:I2SRXTDMCHANBITS]{I2S\_RX\_TDM\_CHAN\_BITS} 决定了每个通道接收数据的位宽。
\begin{itemize}
\item 当每个通道储存数据位宽小于接收数据的位宽时，接收时仅会在接收数据中取储存位宽作为储存数据。此时，配置 \hyperref[fielddesc:I2SRXLEFTALIGN]{I2S\_RX\_LEFT\_ALIGN} 为：
\begin{itemize}
\item 0：储存数据位于接收数据低位；
\item 1：储存数据位于接收数据高位。
\end{itemize}
\item 当每个通道接收数据的位宽小于储存数据位宽时，会将接收数据高位补 0，作为储存数据。
\end{itemize}

\subsubsection{通道储存数据字节序} 
通道接收数据经过截取/补全后，成为了待储存数据，\hyperref[fielddesc:I2SRXBIGENDIAN]{I2S\_RX\_BIG\_ENDIAN} 用于控制待储存数据的字节序。下表描述了不同通道储存数据位宽下，该寄存器对待储存数据的控制。

\subsubsection{A 率/$\mu$ 率压缩/解压缩} 

\chipname{} I2S 对排列好字节序的待储存数据会按照 32-bit（缺省高位补 0）的方式进行 A 率/$\mu$ 率压缩/解压缩。

配置 \hyperref[fielddesc:I2SRXPCMBYPASS]{I2S\_RX\_PCM\_BYPASS} 为：
\begin{itemize}
\item 0：进行压缩/解压缩
\item 1：不进行压缩/解压缩
\end{itemize}
配置 \hyperref[fielddesc:I2SRXPCMCONF]{I2S\_RX\_PCM\_CONF} 为：
\begin{itemize}
\item 0：A 律解压缩
\item 1：A 律压缩
\item 2，$\mu$ 律解压缩
\item 3，$\mu$ 律压缩
\end{itemize}
至此，数据格式控制部分全部完成，数据通过 DMA 存入内存。

\section{软件配置流程}
\subsection{软件配置 I2S 发送流程}\label{sec:i2s-configure-i2s-tx-mode}
软件配置 I2S 发送的流程如下：

\begin{enumerate}
    \item 根据 \ref{sec:i2s-tx-rx-clk} 小节的描述，配置时钟。

    \item 根据表 \ref{tab:i2s-signal-descr} 的描述，配置信号管脚。
    
    \item 根据主从机模式配置 \hyperref[fielddesc:I2STXSLAVEMOD]{I2S\_TX\_SLAVE\_MOD}：
    \begin{itemize}
        \item 0：主机发送模式
        \item 1：从机发送模式
    \end{itemize}
   
    \item 根据 \ref{sec:i2s-tx-data} 小节的描述，配置正确的发送数据模式和发送通道模式，置位 \hyperref[fielddesc:I2STXUPDATE]{I2S\_TX\_UPDATE}。

    \item 根据 \ref{sec:i2s-reset} 小节的描述，复位发送单元和发送 FIFO。
    \item 根据 \ref{sec:i2s-interrupts} 小节的描述使能相应的中断。\label{other:i2s-item-tx-enable-corresponding-interrupts}
    
    \item 配置 DMA 发送链表。 
    
    \item 根据需要置位 \hyperref[fielddesc:I2STXSTOPEN]{I2S\_TX\_STOP\_EN}，更多信息见章节 \ref{sec:i2s-master-slave-tx-mode}。 

    \item 开始发送数据：
    \begin{itemize}
        \item 主机模式下，等待 I2S 从设备配置完成后，置位 \hyperref[fielddesc:I2STXSTART]{I2S\_TX\_START} 开始发送数据；
        \item 从机模式下，置位 \hyperref[fielddesc:I2STXSTART]{I2S\_TX\_START}。I2S 主设备提供 BCK 和 WS 信号后，开始发送数据。
    \end{itemize}

    \item 等待步骤 \ref{other:i2s-item-tx-enable-corresponding-interrupts} 设置的中断信号，或查询 \hyperref[fielddesc:I2STXIDLE]{I2S\_TX\_IDLE} 检查传输是否结束：
    \begin{itemize}
        \item 0：发送设备为工作状态；
        \item 1：发送设备为空闲状态。
    \end{itemize}
    
    \item 清零 \hyperref[fielddesc:I2STXSTART]{I2S\_TX\_START}。
\end{enumerate}
\subsection{软件配置 I2S 接收流程}\label{sec:i2s-configure-i2s-rx-mode}
软件配置 I2S 接收模式的流程如下：

\begin{enumerate}
    \item 根据 \ref{sec:i2s-tx-rx-clk} 小节的描述，配置时钟。

    \item 根据表 \ref{tab:i2s-signal-descr} 的描述，配置信号管脚。

    \item 配置 \hyperref[fielddesc:I2SRXSLAVEMOD]{I2S\_RX\_SLAVE\_MOD} 选择需要的模式：
    \begin{itemize}
        \item 0：主机接收模式
        \item 1：从机接收模式
    \end{itemize}
    
    \item 根据 \ref{sec:i2s-rx-data} 小节的描述，配置正确的接收通道模式和接收数据模式，置位 \hyperref[fielddesc:I2SRXUPDATE]{I2S\_RX\_UPDATE}。

    \item 根据 \ref{sec:i2s-reset} 小节的描述，复位接收单元和接收 FIFO。
    \item 根据 \ref{sec:i2s-interrupts} 小节的描述使能相应的中断。\label{other:i2s-item-rx-enable-corresponding-interrupts}
    
    \item 配置 DMA 接收链表，并在 \hyperref[regdesc:I2SRXEOFNUMREG]{I2S\_RX\_EOF\_NUM\_REG} 中配置接收数据长度。

    \item 开始接收数据：
    \begin{itemize}
        \item 在主机模式下，等待从机准备好后，置位 \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START} 开始接收数据；
        \item 在从机模式下，置位 \hyperref[fielddesc:I2SRXSTART]{I2S\_RX\_START}，等待主机提供 BCK 和 WS 信号后开始接收数据。
    \end{itemize}
    \item 接收的数据由 DMA 根据配置，存到 \chipname{} 存储器的指定地址。最终产生步骤 \ref{other:i2s-item-rx-enable-corresponding-interrupts} 中设置的中断。
\end{enumerate}


\section{I2S 中断} \label{sec:i2s-interrupts}

\begin{itemize}
\item \label{int:I2STXHUNGINT}I2S\_TX\_HUNG\_INT：当发送数据超时即触发此中断。例如，I2S 配置为从机发送模式，但主机长时间未提供 BCK 或 WS 信号，则将触发该中断。超时配置见寄存器 \hyperref[regdesc:I2SLCHUNGCONFREG]{I2S\_LC\_HUNG\_CONF\_REG}。
\item \label{int:I2SRXHUNGINT}I2S\_RX\_HUNG\_INT：当接收数据超时即触发此中断。例如，I2S 配置为从机接收模式，但主机长时间未发送数据，则将触发该中断。超时配置见寄存器 \hyperref[regdesc:I2SLCHUNGCONFREG]{I2S\_LC\_HUNG\_CONF\_REG}。
\item \label{int:I2STXDONEINT}I2S\_TX\_DONE\_INT：当发送数据完成即触发此中断。
\item \label{int:I2SRXDONEINT}I2S\_RX\_DONE\_INT：当接收数据完成即触发此中断。
\end{itemize}

\section{事件任务矩阵功能} \label{sec:i2s-etm-func}

在 \chipname{} 中，I2S 支持 ETM 功能，即可以通过任意外设的 ETM 事件触发 I2S 的 ETM 任务，或者通过 I2S 的 ETM 事件触发任意外设的 ETM 任务。关于 ETM 更多详细信息，请参考章节 \ref{mod:evntaskmatrix} \nameref{mod:evntaskmatrix}。这里仅介绍与 I2S 相关的 ETM 任务和 ETM 事件。

I2S 可接收的 ETM 任务有：

\begin{itemize}
    \item \label{task:I2STASKSTARTTX}I2S\_TASK\_START\_TX：触发时启动 I2S TX 数据传输。
    \item \label{task:I2STASKSTARTRX}I2S\_TASK\_START\_RX：触发时启动 I2S RX 数据传输。
    \item \label{task:I2STASKSTOPTX}I2S\_TASK\_STOP\_TX：触发时停止 I2S TX 数据传输。
    \item \label{task:I2STASKSTOPRX}I2S\_TASK\_STOP\_RX：触发时停止 I2S RX 数据传输。
\end{itemize}

I2S 可产生的 ETM 事件有：

\begin{itemize}
    \item \label{evt:I2SEVTTXDONE}I2S\_EVT\_TX\_DONE：表示 I2S TX 发送完成。
    \item \label{evt:I2SEVTRXDONE}I2S\_EVT\_RX\_DONE：表示 I2S RX 接收完成。
    \item \label{evt:I2SEVTXWORDSSENT}I2S\_EVT\_X\_WORDS\_SENT：表示 I2S TX 发送字数大于等于 \hyperref[fielddesc:I2SETMTXSENDWORDNUM]{I2S\_ETM\_TX\_SEND\_WORD\_NUM} 的值。
    \item \label{evt:I2SEVTXWORDSRECEIVED}I2S\_EVT\_X\_WORDS\_RECEIVED：表示 I2S RX 接收字数大于等于 \hyperref[fielddesc:I2SETMRXRECEIVEWORDNUM]{I2S\_ETM\_RX\_RECEIVE\_WORD\_NUM} 的值。
\end{itemize}

在具体应用中，可以配置 I2S 的 ETM 事件来触发 I2S 的 ETM 任务，例如配置 I2S\_EVT\_X\_WORDS\_SENT 事件触发 I2S\_TASK\_STOP\_TX 任务，从而以 ETM 的方式完成停止 I2S 的操作。
