\section{Reset}
\subsection{Overview}
\chipname{} provides four types of reset that occur at different levels, namely CPU Reset, Core Reset, System Reset, and Chip Reset. All reset types mentioned above (except Chip Reset) preserve the data stored in internal memory. Figure  \ref{figure:Reset} shows the scopes of affected subsystems by each type of reset.

\subsection{Architectural Overview}
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{./09-RESCLK/figures/reset-overview-20220821.eps}
    \caption{Reset Types}
    \label{figure:Reset}
\end{figure}
\chipname{}'s Digital System consists of \hyperref[mod:riscvcpu]{High Performance System (HP system)} that includes Digital Core and Wireless Circuit, and \hyperref[mod:ulp]{Low Power System (LP system)}. See Figure \ref{figure:Reset} for details.

\subsection{Features}
\label{sec:resclk-features}
\begin{itemize}
    \item Four reset types:
    \begin{itemize}
    \item CPU Reset: resets CPU core. Once such reset is released, the instructions from the CPU reset vector will be executed.
    \item Core Reset: resets the whole digital system except LP system, including CPU, peripherals, Wi-Fi, Bluetooth$^®$ LE, and digital GPIOs.
    \item System Reset: resets the whole digital system, including LP system.
    \item Chip Reset: resets the whole chip.
    \end{itemize}
    \item Software reset and hardware reset:
    \begin{itemize}
        \item Software Reset: triggered via software by configuring the corresponding registers of CPU, see Chapter  \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}.
        \item Hardware Reset: triggered directly by the hardware.
    \end{itemize}
\end{itemize}

\subsection{Functional Description}
CPU will be reset immediately when any type of reset above occurs. Users can retrieve reset source codes by reading  \hyperref[fielddesc:RTCCLKRSTRESETCAUSE]{RTC\_CLKRST\_RESET\_CAUSE} after the reset is released.

\subsection{Peripheral Reset}

Peripherals can be reset individually by configuring corresponding registers, or globally by Core Reset, System Reset, or Chip Reset. The reset registers of \chipname{} peripherals are merged to Power/Clock/Reset (PCR) module. See Section  \ref{sec:resclk-reg-summ} \textit{\nameref{sec:resclk-reg-summ}} for detailed information.

\section{Clock}

\subsection{Overview}

\chipname{} clocks are mainly sourced from oscillator (OSC), RC, and PLL circuit, and then processed by the dividers or selectors, which allows most functional modules to select their working clock according to their power consumption and performance requirements. Figure  \ref{figure:Clock} shows the system clock structure.

\subsection{Architectural Overview}



\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{./09-RESCLK/figures/sys_clk_struct.eps}
    \caption{System Clock}
    \label{figure:Clock}
\end{figure}

\begin{tiplisting}
The AUTODIV in the figure will divide 480 MHz PLL\_CLK into 160 MHz clock by hardware control only when the MUX before selects PLL\_CLK. If the MUX before selects RC\_FAST\_CLK or XTAL\_CLK, AUTODIV will not divide the clock frequency.
\end{tiplisting}

\subsection{ Features }

\chipname{} clock sources as shown on the left side of Figure \ref{figure:Clock} can be classified into two types depending on their frequencies:
\begin{itemize}
    \item High-speed clock sources for devices working at a higher frequency, such as CPU and digital peripherals
    \begin{itemize}
        \item PLL\_CLK (480 MHz): internal PLL clock. Its reference clock is XTAL\_CLK.
        \item XTAL\_CLK (40 MHz): external crystal clock
    \end{itemize}
    \item Slow-speed clock sources for LP system and some peripherals working in low-power mode
    \begin{itemize}
        \item XTAL32K\_CLK (32 kHz): external crystal clock
	    \item RC\_FAST\_CLK (17.5 MHz by default): internal fast RC oscillator with adjustable frequency
	    \item RC\_SLOW\_CLK (136 kHz by default): internal slow RC oscillator
	    \item OSC\_SLOW\_CLK (32 kHz by default): external slow clock input through \hyperref[table:rtcmux_analog_pads]{XTAL\_32K\_P}. After configuring this GPIO, also configure the Hold function (see Chapter \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}} >  \ref{PadHold} \textit{\nameref{PadHold}})
    \end{itemize}
\end{itemize}

\subsection{Functional Description}

\subsubsection{HP System Clock}

As Figure \ref{figure:Clock} shows, CPU\_CLK is the master clock for CPU and it can be as high as 160 MHz when CPU works in high performance mode. Alternatively, CPU can run at lower frequencies, such as at 2 MHz, to achieve lower power consumption. CPU\_CLK shares the same clock sources with AHB\_CLK, CRYPTO\_CLK, and MSPI\_CLK. Users can select from XTAL\_CLK, PLL\_CLK, or RC\_FAST\_CLK as the clock source of CPU\_CLK by configuring  \hyperref[fielddesc:PCRSOCCLKSEL]{PCR\_SOC\_CLK\_SEL}. See Table  \ref{tab:resclk-cpu-clk-src} and Table  \ref{tab:resclk-cpu-clk-freq}. When PLL\_CLK is selected as the clock source, CPU\_CLK will be divided into 160 MHz clock by hardware control before the configurable divider, please refer to AUTODIV in figure \ref{figure:Clock}. By default, the CPU clock is sourced from XTAL\_CLK with a divider of 1, i.e., the CPU clock is 40 MHz.

The available divider values for CPU\_CLK and AHB\_CLK are as follows:

\begin{itemize}
     \item \hyperref[fielddesc:PCRCPUHSDIVNUM]{PCR\_CPU\_HS\_DIV\_NUM}: 0, 1, 3
     \item \hyperref[fielddesc:PCRCPULSDIVNUM]{PCR\_CPU\_LS\_DIV\_NUM}: 0, 1, 3, 7, 15, 31
     \item \hyperref[fielddesc:PCRAHBHSDIVNUM]{PCR\_AHB\_HS\_DIV\_NUM}: 3, 7, 15
     \item \hyperref[fielddesc:PCRAHBLSDIVNUM]{PCR\_AHB\_LS\_DIV\_NUM}: 0, 1, 3, 7, 15, 31
\end{itemize}

As shown in \ref{figure:Clock}, to generate APB\_CLK, AHB\_CLK might be divided twice. The first division is compulsory. That is, AHB\_CLK is always divided by the divisor (\hyperref[fielddesc:PCRAPBDIVNUM]{PCR\_APB\_DIV\_NUM} + 1). The second division (also called automatic frequency reduction) is optional. When there is no request from the host in the chip to access peripheral registers, AHB\_CLK will be further divided by \hyperref[fielddesc:PCRAPBDECREASEDIVNUM]{APB\_DECREASE\_DIV\_NUM} + 1. If the host initiates a request to access peripheral registers, APB\_CLK will be restored to the frequency after the first division.

Note that the chip's performance will degrade due to the automatic frequency reduction. This function can be disabled (already disabled by default) by configuring \hyperref[fielddesc:PCRAPBDECREASEDIVNUM]{APB\_DECREASE\_DIV\_NUM} to 0.

\subsubsection{LP System Clock}

The LP system can operate when most other clocks are disabled. LP system clocks include LP\_SLOW\_CLK and LP\_FAST\_CLK.

The clock sources for LP\_SLOW\_CLK and LP\_FAST\_CLK are low-frequency clocks:
\begin{itemize}
    \item LP\_SLOW\_CLK can be derived from:
    \begin{itemize}
        \item RC\_SLOW\_CLK
        \item XTAL32K\_CLK
        \item OSC\_SLOW\_CLK
    \end{itemize}
    \item LP\_FAST\_CLK can be derived from:
    \begin{itemize}
        \item 20 MHz XTAL\_D2\_CLK, which is XTAL\_CLK divided by 2
        \item RC\_FAST\_CLK
    \end{itemize}
\end{itemize}

The clock source of LP\_DYN\_SLOW\_CLK is LP\_SLOW\_CLK.

The clock source of LP\_DYN\_FAST\_CLK depends on the chip's power mode (see Chapter \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}).
\begin{itemize}
    \item Select LP\_FAST\_CLK as its clock source in Active and Modem-sleep mode
    \item Select LP\_SLOW\_CLK as its clock source in Light-sleep and Deep-sleep mode
\end{itemize}

\textbf{PLL\_CLK}

PLL\_F480M\_CLK is the source clock of PLL，which is 480 MHz. PLL\_D2\_CLK (240 MHz), PLL\_F160M\_CLK, PLL\_F80M\_CLK, and PLL\_F48M\_CLK are divided from PLL\_F480M\_CLK.


\textbf{CRYPTO\_CLK}

As shown in Figure \ref{figure:Clock}, CRYPTO\_CLK shares the same clock sources with CPU\_CLK, and its frequency is up to 160 MHz.


To protect encryption and decryption peripherals from DPA (Differential Power Analysis) attacks, a random divider strategy is implemented for the function clock of encryption and decryption peripherals. Four security levels are available, depending on the range of random divider. Users can select the security level by configuring  \hyperref[regdesc:HPSYSTEMSECDPACONFREG]{HP\_SYSTEM\_SEC\_DPA\_CONF\_REG}. If  \hyperref[fielddesc:HPSYSTEMSECDPACFGSEL]{HP\_SYSTEM\_SEC\_DPA\_CFG\_SEL} is set to 1, the security level is determined by configuration of \hyperref[fielddesc:EFUSESECDPALEVEL]{EFUSE\_SEC\_DPA\_LEVEL}, otherwise, by the value of  \hyperref[fielddesc:HPSYSTEMSECDPALEVEL]{HP\_SYSTEM\_SEC\_DPA\_LEVEL}.


\textbf{LED\_PWM}

LEDC module uses PLL\_F80M\_CLK, RC\_FAST\_CLK and XTAL\_CLK  as clock source when APB\_CLK is disabled. In other words, when the system is in low-power mode, most peripherals will be halted (APB\_CLK is turned off), but LEDC can work normally via RC\_FAST\_CLK.

\subsubsection{Wi-Fi and Bluetooth LE Clock}
Wi-Fi and Bluetooth LE can work only when CPU\_CLK uses PLL\_CLK as its clock source. Suspending PLL\_CLK requires that Wi-Fi and  Bluetooth LE have entered low-power mode first.

\subsection{HP System Clock Gating Controlled by PMU}

\subsection{HP System Clock Configuration}

The clock source of HP\_ROOT\_CLK can be configured via \hyperref[fielddesc:PCRSOCCLKSEL]{PCR\_SOC\_CLK\_SEL}.

\begin{itemize}
    \item If PLL\_CLK is selected as the clock source of HP\_ROOT\_CLK,
    \begin{itemize}
        \item the clock divisor for CPU\_CLK can be configured via \hyperref[fielddesc:PCRCPUHSDIVNUM]{PCR\_CPU\_HS\_DIV\_NUM}.
        \item the clock divisor for AHB\_CLK can be configured via \hyperref[fielddesc:PCRAHBHSDIVNUM]{PCR\_AHB\_HS\_DIV\_NUM}.
    \end{itemize}

    \item If XTAL\_CLK or RC\_FAST\_CLK is selected as the clock source of HP\_ROOT\_CLK,
    \begin{itemize}
        \item the clock divisor for CPU\_CLK can be configured via \hyperref[fielddesc:PCRCPULSDIVNUM]{PCR\_CPU\_LS\_DIV\_NUM}.
        \item the clock divisor for AHB\_CLK can be configured via \hyperref[fielddesc:PCRAHBLSDIVNUM]{PCR\_AHB\_LS\_DIV\_NUM}.
    \end{itemize}
\end{itemize}

\subsection{LP System Clock Configuration}

The clock source of LP\_SLOW\_CLK can be configured via \hyperref[fielddesc:LPCLKRSTSLOWCLKSEL]{LP\_CLKRST\_SLOW\_CLK\_SEL}.

The clock source of LP\_FAST\_CLK can be configured via \hyperref[fielddesc:LPCLKRSTFASTCLKSEL]{LP\_CLKRST\_FAST\_CLK\_SEL}.

\subsection{Peripheral Clock Reset and Configuration} \label{sec:periperal-clock-configuration}

\begin{importantlisting}
\vspace{-0.5em}

\chipname{} features low power consumption. This is why some peripheral clocks are gated (disabled) by default. Before using any of these peripherals, it is mandatory to enable the clock for the given peripheral by setting the corresponding CLK\_EN bit to 1, and release the peripheral from reset state to make it operational by setting the RST\_EN bit to 0.
\end{importantlisting}

The clocks of most peripherals can be classified into two types:
\begin{itemize}
    \item Bus clock: used to configure peripheral registers.
    \item Function clock: such as UART's reference clock, used by peripherals to operate.
\end{itemize}
The operating clock (function clock) of most peripherals can be selected from multiple clock sources. In the description of the gating registers, it will be stated whether the register belongs to the bus clock (AHB\_CLK, APB\_CLK) gating register or the function clock gating register.

Bus clock switches, function clock switches, and the configuration registers for clock source selection and clock frequency division are grouped into the PCR module. For more information, see Section \ref{sec:resclk-reg-summ}：\textit{Register Summary}.

When a peripheral is not working, users can turn off its function clock by configuring related registers. Turning off the peripheral’s function clock does not affect the rest of the system.

Take I2C clock configuration as an example.


\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{./09-RESCLK/figures/sys_clk_struct_pcr.eps}
    \caption{Clock Configuration Example}
    \label{figure:Clock Config}
\end{figure}


Figure \ref{figure:Clock Config} shows the clock structure of I2C. The clock structure of other peripherals is similar to this one. CLK\_SWITCH is used to select a clock output and CLK\_GATE to turn on/off the clock.

In scenarios that require low power consumption, when the peripheral is not in use, in addition to turning off the function clock, the bus clock of the peripheral can also be turned off to further lower power consumption.

Note that if you turn off the bus clock first, the function clock may continue working. It is recommended to turn off the function clock first and then the bus clock when turning off the clocks. It is also recommended to turn on the bus clock first and then the function clock when turning on the clocks.
