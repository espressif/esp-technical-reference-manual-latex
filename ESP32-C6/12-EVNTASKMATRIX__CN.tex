\chapter{事件任务矩阵 (SOC\_ETM)}
\label{mod:evntaskmatrix}

%%%%%%%%%%%%%%%%%%%%
%%% Overview %%%
%%%%%%%%%%%%%%%%%%%%

\section{概述}
事件任务矩阵 (ETM) 外设包含50个可配置通道。每个通道可以将任意指定外设的事件映射到任意指定外设的任务，从而触发外设执行指定任务，无需CPU干预。


%%%%%%%%%%%%%%%%%%%%
%%% Features %%%
%%%%%%%%%%%%%%%%%%%%

\section{特性}
\label{sec:evntaskmatrix-features}

事件任务矩阵具有以下特性：
\begin{itemize}
    \item 支持从多个外设接收多种事件
    \item 支持为多个外设生成多种任务
    \item 拥有 50 个可独立配置的 ETM 通道
    \item 每个通道接收到的事件可以是所有事件中的任意一个，每个通道接收到的事件可以映射到任意的任务上输出
    \item 每个 ETM 通道都可以独立使能。当通道未使能时，它不会响应所配置的事件，也不会生成要映射到的任务
    \item 能够产生事件、接收任务的外设有：GPIO、LED PWM、通用定时器、RTC 定时器、系统定时器、MCPWM、温度传感器、ADC、I2S、LP CPU、GDMA 和 PMU
\end{itemize}

由于 50 个 ETM 通道的功能和操作都是相同的。 以下部分将 ETM 通道统称为通道\regindex{n}（其中 \regindex{n} 范围从 0 到 49）。

\subsection{架构}

\begin{figure}[H]

图 \ref{fig:etm-architecture} 展示了事件任务矩阵的架构。

事件任务矩阵有50个独立通道，每个通道可以选择所有事件中的任意一个作为该通道的输入（有关配置方法，参考 \ref{sec:evntaskmatrix-events} 章节），并且可以选择映射事件到所有任务中的任何一个（有关配置方法，参考 \ref{sec:evntaskmatrix-tasks} 章节）。每个通道都有独立的使能配置位（有关配置方法，参考 \ref{sec:evntaskmatrix-channel-enable} 章节）。

图 \ref{fig:etm-channel} 展示了单个事件任务矩阵通道的结构，其中 \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID} 字段配置 MUX 选择所有事件之一作为通道\regindex{n} 的输入，\hyperref[fielddesc:SOCETMCH0TASKID]{SOC\_ETM\_CH\regindex{n}\_TASK\_ID} 字段配置 DEMUX 以映射通道\regindex{n} 所选择的事件到所有任务之一。字段 \hyperref[fielddesc:SOCETMCHENABLE0]{SOC\_ETM\_CH\_ENABLE\regindex{n}} 和 \hyperref[fielddesc:SOCETMCHDISABLE0]{SOC\_ETM\_CH\_DISABLE\regindex{n}} 用于使能或关闭通道\regindex{n}，字段 \hyperref[fielddesc:SOCETMCHENABLED0]{SOC\_ETM\_CH\_ENABLED\regindex{n}} 用于指示通道\regindex{n} 的开启或关闭状态。

\subsection{时序考虑因素}

ETM外设接收的事件、发送的任务和ETM通道之间驱动时钟的结构如图 \ref{fig:etm-clk-archi} 所示。

\begin{figure}[H]
    \centering
    \includegraphics[width=1\textwidth]{\modulefiles/figures/etm_clk_archi}
    \caption{事件任务矩阵时钟结构}
    \label{fig:etm-clk-archi}
\end{figure}

ETM 运行在 AHB\_CLK 时钟域下（详见章节 \ref{mod:resclk} \textit{\nameref{mod:resclk}}），每个事件对应一个由相应外设在其时钟域中产生的脉冲信号，每个任务被 ETM 映射为对应外设时钟域下的脉冲信号。由于产生事件的外设、事件任务矩阵和接收任务的外设不一定由同个时钟驱动，需要同步处理，所以对两个连续事件之间的时间间隔有要求，否则会丢失事件。为了让事件任务矩阵成功接收到每个事件，对于每个可以产生事件的外设，其产生的两个连续事件脉冲之间的时间间隔必须大于一个事件任务矩阵的时钟周期，即
$ceil(\frac{peripheral\_clock\_frequenty }{ETM\_clock\_frequenty})$ 个外设时钟周期。

例如，假设外设 A 在 80 MHz 时钟域（PLL\_F80M\_CLK）生成事件 1，ETM 在 40 MHz 时钟域（AHB\_CLK）运行。要成功接收每个事件 1，两个连续事件 1 之间的时间间隔必须大于两个外设 A 的时钟周期（即一个 ETM 时钟周期）。

同样的，为了使事件任务矩阵成功地将接收到的事件（即同步到事件任务矩阵时钟域的事件）映射为外设的任务，事件任务矩阵时钟域中的两个连续事件脉冲之间的时间间隔必须大于一个外设的时钟周期，即 $ceil(\frac{ETM\_clock\_frequenty}{peripheral\_clock\_frequenty})$ 个事件任务矩阵时钟周期。

例如，假设外设 B 在 20 MHz 时钟域（RC\_FAST\_CLK）接收任务 1，ETM 在 40 MHz 时钟域（AHB\_CLK）运行。要成功将每个接收到的事件映射到任务 1，两个连续事件之间的时间间隔必须大于两个 ETM 时钟周期（即一个外设 B 时钟周期）。

因此，要将外设 A 产生的两个连续事件完整映射到外设 B 的两个任务中，这两个事件之间的时间间隔必须大于$ceil(\frac{peripheral\_A\_clock\_frequenty}{ETM\_clock\_frequenty })*ceil(\frac{ETM\_clock\_frequenty}{peripheral\_B\_clock\_frequenty})$ 个外设 A 时钟周期。

例如，假设外设 A 在 80 MHz 时钟域（PLL\_F80M\_CLK）生成事件 1，外设 B 在 20 MHz 时钟域（RC\_FAST\_CLK）接收任务 1，ETM 在 40 MHz 时钟域（AHB\_CLK）运行。要成功将每个事件 1（外设 A 生成）映射到任务 1（外设 B 接收），两个连续事件 1 之间的时间间隔必须大于 $2*2=4$ 个外设 A 时钟周期。

\subsection{通道控制}
\label{sec:evntaskmatrix-channel-enable}

事件任务矩阵的每个通道都可以独立配置为使能或关闭。 当 通道\regindex{n} 配置为使能时，如果该通道接收到所配置的事件（字段 \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID}），该事件将映射到所配置的任务上（字段 \hyperref[fielddesc:SOCETMCH0TASKID]{SOC\_ETM\_CH\regindex{n}\_TASK\_ID}）。 当通道\regindex{n} 配置为不使能时，即使该通道接收到所配置的事件（字段\hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID}）， 也不会产生任何任务。

使能事件任务矩阵通道\regindex{n} 的配置过程如下：
\begin{enumerate}
    \item 向字段 \hyperref[fielddesc:SOCETMCHENABLE0]{SOC\_ETM\_CH\_ENABLE\regindex{n}} 写1
    \item 读取字段 \hyperref[fielddesc:SOCETMCHENABLED0]{SOC\_ETM\_CH\_ENABLED\regindex{n}} 的值，如果该字段的值为 1，则表示通道\regindex{n} 已使能，否则通道\regindex{n} 未使能
\end{enumerate}

关闭事件任务矩阵通道\regindex{n} 的配置过程如下：
\begin{enumerate}
    \item 向字段 \hyperref[fielddesc:SOCETMCHDISABLE0]{SOC\_ETM\_CH\_DISABLE\regindex{n}} 写1
    \item 读取字段 \hyperref[fielddesc:SOCETMCHENABLED0]{SOC\_ETM\_CH\_ENABLED\regindex{n}} 的值，如果该字段的值为0，则表示通道\regindex{n} 已经关闭，否则通道\regindex{n} 使能
\end{enumerate}

( 如果将字段 \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID} 的值配置为 0 或将字段 \hyperref[fielddesc:SOCETMCH0TASKID]{SOC\_ETM\_CH\regindex{n}\_TASK\_ID} 的值配置为 0，事件任务矩阵通道\regindex{n} 也将被关闭。）

事件任务矩阵通道\regindex{n} 的完整配置过程如下：
\begin{enumerate}
    \item 向字段 \hyperref[fielddesc:PCRETMCLKEN]{PCR\_ETM\_CLK\_EN} 写 1，开启 ETM 的时钟
    \item 配置字段 \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID} 的值，选择通道\regindex{n} 所要接收的事件
    \item 配置字段 \hyperref[fielddesc:SOCETMCH0TASKID]{SOC\_ETM\_CH\regindex{n}\_TASK\_ID} 的值，选择通道\regindex{n} 所接收到的事件要映射到的任务
    \item 向字段 \hyperref[fielddesc:SOCETMCHENABLE0]{SOC\_ETM\_CH\_ENABLE\regindex{n}} 写1
    \item 当通道\regindex{n} 不再需要将配置的事件映射到配置的任务时，向字段\hyperref[fielddesc:SOCETMCHDISABLE0]{SOC\_ETM\_CH\_DISABLE\regindex{n}}写1来关闭通道\regindex{n}。 之后，如果要为通道\regindex{n} 配置新的事件和任务的映射关系，则重复步骤1-3，如果不做任何配置，则通道\regindex{n} 保持停止运行状态
    \item 若需复位 ETM，可通过向 \hyperref[fielddesc:PCRETMRSTEN]{PCR\_ETM\_RST\_EN} 字段先写 1 再写 0 来实现
\end{enumerate}
