\chapter{SHA 加速器 (SHA)} \label{mod:sha}

\section{概述}

\chipname{}  内置 SHA（安全哈希算法）硬件加速器可完成 SHA 运算，具有 \hyperref[sec:sha-typical-sha]{Typical SHA} 和 \hyperref[sec:sha-dma-sha]{DMA-SHA} 两种工作模式。整体而言，相比基于纯软件的 SHA 运算，SHA 硬件加速器能够极大地提高运算速度。

\section{主要特性}

\chipname{} 的 SHA 硬件加速器：

\begin{itemize}
    \item 支持 \href{https://doi.org/10.6028/NIST.FIPS.180-4}{FIPS PUB 180-4 规范}中的以下运算标准
    \begin{itemize}
        \item SHA-1 运算
        \item SHA-224 运算
        \item SHA-256 运算
    \end{itemize}
    \item 提供两种工作模式
    \begin{itemize}
        \item Typical SHA 工作模式 
        \item DMA-SHA 工作模式  
    \end{itemize}
    \item 允许插入 (interleaved) 功能（仅限 Typical SHA 工作模式）
    \item 允许中断功能（仅限 DMA-SHA 工作模式）
\end{itemize}

\section{工作模式简介}
\chipname{} 内置的 SHA 加速器支持两种工作模式。

\begin{itemize}
    \item \hyperref[sec:sha-typical-sha]{Typical SHA 工作模式}：所有数据读写统一通过 CPU 访问完成。
    \item \hyperref[sec:sha-dma-sha]{DMA-SHA 工作模式}：所有读数据通过硬件上的 DMA 完成。具体来说，用户可配置 DMA 控制器，由 DMA 控制器提供 SHA 运算过程中所需的数据信息。因此，可以释放 CPU 执行其他任务。
\end{itemize}    

SHA 加速器的激活仅需使能 \hyperref[regdesc:PCRSHACONFREG]{PCR\_SHA\_CONF\_REG} 外围时钟的 \hyperref[fielddesc:PCRRSACLKEN]{PCR\_SHA\_CLK\_EN} 位，并同时清零 \hyperref[fielddesc:PCRRSARSTEN]{PCR\_SHA\_RST\_EN} 位。此外，还需要清零 \hyperref[fielddesc:PCRDSRSTEN]{PCR\_DS\_RST\_EN} 和 \hyperref[fielddesc:PCRHMACRSTEN]{PCR\_HMAC\_RST\_EN} 复位 \nameref{mod:digsig} 和 \nameref{mod:hmac}。

用户可通过配置 \hyperref[regdesc:SHASTARTREG]{SHA\_START\_REG} 或 \hyperref[regdesc:SHADMASTARTREG]{SHA\_DMA\_START\_REG} 选择 SHA 加速器的工作模式，先配置的工作模式生效，具体请见表 \ref{tab:sha-hash-algo-select}。

\section{功能描述}

SHA 加速器可以提取信息摘要 (message digest)，其主要工作流程分为两步：\hyperref[sec:sha-preprocessing]{信息预处理}和\hyperref[sec:sha-hash-operation]{哈希运算}。

\subsection{信息预处理}\label{sec:sha-preprocessing}

信息预处理分为三个主要步骤：\hyperref[sec:sha-textpadding]{附加填充比特}、\hyperref[sec:sha-parsingmessage]{信息解析}和\hyperref[sec:sha-initialhash]{设置初始哈希值}。

\subsubsection{附加填充比特}\label{sec:sha-textpadding}

SHA 加速器仅能处理长度为 512 位及其整倍数的信息。因此，在将信息送至 SHA 加
速器进行运算前，应先通过软件操作将信息填充为符合要求的长度。

假设待处理信息 \begin{math} M \end{math} 的长度为 \begin{math} m \end{math} 位，则填充步骤如下：

\begin{enumerate}
    \item 首先，在待处理信息后填充 1 个 ``1"；
    \item 随后，再填充 {\it{k}} 个 ``0"。其中，{\it{k}} 为满足 \begin{math}m + 1 + k \ {\equiv} \ 448 \ mod \end{math} 512 的最小非负数解；
    \item 最后，在末尾填充一个 64 位的信息块。该信息块的内容为用二进制表示的待处理信息的长度，即 {\it{m}} 的值。
\end{enumerate}

更多详情，请参考 \href{https://doi.org/10.6028/NIST.FIPS.180-4}{《FIPS PUB 180-4 规范》} 〉章节 “Padding the Message”。

\subsubsection{信息解析}\label{sec:sha-parsingmessage}

在完成信息填充后，我们还需将待处理信息（及其填充）解析为 \begin{math}N\end{math} 个 512 位的信息块，即 \begin{math}M^{(1)}\end{math}、\begin{math}M^{(2)}\end{math}、 \dots、\begin{math}M^{(N)}\end{math}。一个 512 位信息块包括 16 个 32 位的字 (word)，则第 \begin{math}i\end{math} 个信息块的第一个 32 位字表示为 M$^{(i)}_{0}$，第二个 32 位字表示为 M$^{(i)}_{1}$，\dots，第 16 个 32 位字表示为 M$^{(i)}_{15}$。

SHA 加速器在工作时，每次处理的信息块数据均将按照如下规则写入相应的寄存器中：将 M$^{(i)}_{0}$ 存放在 \hyperref[regdesc:SHAMNREG]{SHA\_M\_0\_REG} 中，M$^{(i)}_1$ 存放在 \hyperref[regdesc:SHAMNREG]{SHA\_M\_1\_REG}，\dots，M$^{(i)}_{15}$ 存放在 \hyperref[regdesc:SHAMNREG]{SHA\_M\_15\_REG} 中。

\begin{tiplisting}
有关“信息块”及相关概念的描述，请参考 \href{https://doi.org/10.6028/NIST.FIPS.180-4}{《FIPS PUB 180-4 规范》} > 章节 ``Glossary of Terms and Acronyms"。
\end{tiplisting}

\subsubsection{哈希初始值 (Initial Hash Value)}\label{sec:sha-initialhash}
在进行哈希运算前，首先必须设置哈希初始值 H$^{(0)}$，其中 SHA-1、SHA-224 和 SHA-256 运算的哈希初始值为常量 C，且已经固定在硬件中，无需额外配置。

\subsection{哈希运算流程}\label{sec:sha-hash-operation}

在完成信息预处理后，\chipname{} SHA 加速器将正式开始哈希运算，最终根据不同运算标准得到不同长度的信息摘要。
正如上文所述，\chipname{} SHA 加速器支持 \hyperref[sec:sha-typical-sha]{Typical SHA} 和 \hyperref[sec:sha-dma-sha]{DMA-SHA} 两种工作模式，下面将对这两种工作模式的具体流程进行介绍。

\subsubsection{Typical SHA 模式下的运算流程}\label{sec:sha-typical-sha}

通常情况下，\chipname{} 的 SHA 会先处理完当前信息的所有信息块并生成该信息的信息摘要，之后再开始计算新的信息摘要。

不过，\chipname{} SHA 加速器还支持 “interleaved” 运算（仅 Typical SHA 支持），即在完成当前信息的所有运算前，用户在每个信息块计算完成后均可插入新的运算。

具体来说，用户可以将存储在 \hyperref[regdesc:SHAHNREG]{SHA\_H\_\regindex{n}\_REG} 寄存器中的信息摘要暂时保存到其他地方，然后让 SHA 加速器来完成其他优先级更高的运算任务。当插入的运算结束后，用户再将之前暂存的信息摘要重新写入 \hyperref[regdesc:SHAHNREG]{SHA\_H\_\regindex{n}\_REG} 中，并继续完成之前中断的计算。

\subparagraph{Typical SHA 的具体运算流程}

\begin{enumerate}
    \item \label{others:sha-sel-algorithm} 选择运算标准。
    \begin{itemize}
        \item 配置 \hyperref[regdesc:SHAMODEREG]{SHA\_MODE\_REG} 寄存器，设置运算标准。具体配置，请参考表 \ref{tab:sha-mode-summary}。
    \end{itemize}
    \item \label{others:sha-current-block} 处理当前信息块。
    \begin{itemize}
        \item 将当前信息块写入 \hyperref[regdesc:SHAMNREG]{SHA\_M\_\regindex{n}\_REG} 寄存器。
    \end{itemize}
    \item 启动 SHA 加速器 \hyperref[others:sha-note-typical-sha]{$^1$}。
            \begin{itemize}
                \item 如果为首次运算，则对 \hyperref[regdesc:SHASTARTREG]{SHA\_START\_REG} 寄存器置 1，启动 SHA 加速器的运算。此时，SHA 加速器按照步骤 \ref{others:sha-sel-algorithm} 中选定的运算标准，使用硬件中固定的哈希初始值进行运算；
                \item 如果非首次运算\hyperref[others:sha-note-typical-sha]{$^2$}，则对 \hyperref[regdesc:SHACONTINUEREG]{SHA\_CONTINUE\_REG} 寄存器置 1，启动 SHA 加速器的运算。此时，SHA 加速器使用 \hyperref[regdesc:SHAHNREG]{SHA\_H\_\regindex{n}\_REG} 寄存器中的值作为哈希初始值进行运算。
            \end{itemize}
    \item 查询当前信息块的处理进度。
    \begin{itemize}
        \item 轮询寄存器 \hyperref[regdesc:SHABUSYREG]{SHA\_BUSY\_REG} 一直到读回的值为 0，代表 SHA 硬件加速器已完成对当前信息块的计算，进入“空闲”状态\hyperref[others:sha-note-typical-sha]{$^3$}。
    \end{itemize}
        
    \item 选择是否有后续的待处理信息块。
    \begin{itemize}
        \item 如果存在后续待处理信息块，则跳回执行步骤 \ref{others:sha-current-block}。
        \item 否则，继续执行。
    \end{itemize}
    \item 获取信息摘要：
    \begin{itemize}
        \item 从寄存器堆 \hyperref[regdesc:SHAHNREG]{SHA\_H\_\regindex{n}\_REG} 取出信息摘要。
    \end{itemize}
\end{enumerate}

\begin{tiplisting}\label{others:sha-note-typical-sha}
\vspace{-2.5em}
\begin{enumerate}
\item 这里，在 SHA 加速器进行硬件运算时，如果存在后续待处理信息块，软件还可以同时将后续信息块写入 \\ \hyperref[regdesc:SHAMNREG]{SHA\_M\_\textcolor{red}{\it{n}}\_REG} 寄存器，以节省时间。
\item 比如重新启动 SHA 加速器完成之前暂停任务的情况。
\item 这里，你可以选择是否需要插入其他任务。如需插入，请前往 \hyperref[others:sha-interleaved]{插入任务工作流程} 具体查看。
\end{enumerate} 
\end{tiplisting}

\label{others:sha-interleaved}如上文所述，\chipname{} SHA 加速器\textbf{支持在 Typical SHA 模式下 ``插入" 任务}。 

具体工作流程如下。

\begin{enumerate}
    \item 保存插入前任务的以下数据，准备将 SHA 加速器的使用权移交给插入的任务。
        \begin{itemize}
            \item 读取并保存寄存器 \hyperref[regdesc:SHAMODEREG]{SHA\_MODE\_REG} 中的运算标准类型。
            \item 读取并保存寄存器堆 \hyperref[regdesc:SHAHNREG]{SHA\_H\_\regindex{n}\_REG} 中的信息摘要。
        \end{itemize}
    \item 执行插入的任务。具体按照插入运行类型的不同，请见 \hyperref[sec:sha-typical-sha]{Typical SHA} 或 \hyperref[sec:sha-dma-sha]{DMA-SHA 工作流程}。
    \item 恢复插入前任务的以下数据，准备将 SHA 加速器的使用权交还给插入前的任务。
        \begin{itemize}
            \item 将获得使用权前保存的运算标准类型重新写入寄存器 \hyperref[regdesc:SHAMODEREG]{SHA\_MODE\_REG};
            \item 将获得使用权前保存的信息摘要写入寄存器堆 \hyperref[regdesc:SHAHNREG]{SHA\_H\_\regindex{n}\_REG}。
        \end{itemize}
    \item 将之前任务的下一个待处理信息块写入 \hyperref[regdesc:SHAMNREG]{SHA\_M\_\regindex{n}\_REG} 寄存器，并对 \hyperref[regdesc:SHACONTINUEREG]{SHA\_CONTINUE\_REG} 寄存器置 1，重新启动 SHA 加速器，完成之前暂停的任务。
\end{enumerate}



\subsubsection{DMA-SHA 模式下的运算流程}\label{sec:sha-dma-sha}

\chipname{} SHA 加速器在 DMA-SHA 工作模式下不支持 “interleaved” 运算，即用户必须在每次 DMA 运算（可能包括 1 个或多个信息块）全部结束后才能插入新的运算。这种情况下，用户如有插入运算需求，可将较大信息块进行拆分，并进行多次 DMA 运算。每次 DMA 运算之间允许插入其他运算标准的计算任务。

单次 DMA 运算最多可以处理 63 个数据块。

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

与 Typical SHA 不同，SHA 在 DMA-SHA 工作模式下，运算过程中的数据搬运过程均由硬件完成。具体配置可见章节 \ref{mod:dma} \textit{\nameref{mod:dma}}。

\textbf{DMA-SHA 的具体工作流程}

\begin{enumerate}
    \item 选择运算标准。
    \begin{itemize}
        \item 配置 \hyperref[regdesc:SHAMODEREG]{SHA\_MODE\_REG} 寄存器，设置运算标准。具体配置，请参考表 \ref{tab:sha-mode-summary}。
    \end{itemize}
    \item 选择是否启用中断。请将 \hyperref[regdesc:SHAIRQENAREG]{SHA\_IRQ\_ENA\_REG} 寄存器配置为 1 以启动中断。
    \item 配置块个数。
    \begin{itemize}
        \item  将待加密数据的总块数 {\it{M}} 写入  \hyperref[regdesc:SHADMABLOCKNUMREG]{SHA\_DMA\_BLOCK\_NUM\_REG} 寄存器。
    \end{itemize}
    \item 开始 DMA-SHA 运算。
        \begin{itemize}
            \item 如果当前 DMA-SHA 运算为接着另一次 DMA-SHA 的运算，需要提前将另一次计算得到的信息摘要写入寄存器堆 \hyperref[regdesc:SHAHNREG]{SHA\_H\_\regindex{n}\_REG} 中，随后将 1 写入寄存器 \hyperref[regdesc:SHADMACONTINUEREG]{SHA\_DMA\_CONTINUE\_REG}；
            \item 否则，只需要将 1 写入寄存器 \hyperref[regdesc:SHADMASTARTREG]{SHA\_DMA\_START\_REG}。
        \end{itemize}
    \item 等待 DMA-SHA 运算结束。判断 DMA-SHA 运算结束有以下两种方法：
        \begin{itemize}
            \item 轮询寄存器 \hyperref[regdesc:SHABUSYREG]{SHA\_BUSY\_REG} 结果为 0。
            \item 等待中断信号产生。此时，应及时通过软件将 \hyperref[regdesc:SHACLEARIRQREG]{SHA\_CLEAR\_IRQ\_REG} 寄存器置为 1 以清除中断。
        \end{itemize}
    
    \item 获取信息摘要
    \begin{itemize}
        \item 从寄存器堆 \hyperref[regdesc:SHAHNREG]{SHA\_H\_\regindex{n}\_REG} 取出信息摘要。
    \end{itemize}
    
\end{enumerate}

\subsection{中断}\label{sec:sha-Interrupt}

在 DMA-SHA 工作模式下，SHA 加速器允许中断发生。
\begin{itemize}
    \item 用户可通过将 \hyperref[regdesc:SHAIRQENAREG]{SHA\_IRQ\_ENA\_REG} 寄存器配置为 1 开启中断。如开启中断功能，SHA 加速器在完成运算时，中断发生。
    \item 注意，该中断必须由软件将 \hyperref[regdesc:SHACLEARIRQREG]{SHA\_CLEAR\_IRQ\_REG} 寄存器置为 1 进行清除。
\end{itemize} 

在 Typical SHA 工作模式下，SHA 加速器的时间开销较小，因此不支持中断功能。
