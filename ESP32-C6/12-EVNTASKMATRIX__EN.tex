\chapter{Event Task Matrix (SOC\_ETM)}
\label{mod:evntaskmatrix}

%%%%%%%%%%%%%%%%%%%%
%%% Overview %%%
%%%%%%%%%%%%%%%%%%%%


\section{Overview}
The Event Task Matrix (ETM) peripheral contains 50 configurable channels. Each channel can map an event of any specified peripheral to a task of any specified peripheral. In this way, peripherals can be triggered to execute specified tasks without CPU intervention.


%%%%%%%%%%%%%%%%%%%%
%%% Features %%%
%%%%%%%%%%%%%%%%%%%%

\section{Features}
\label{sec:evntaskmatrix-features}

The Event Task Matrix has the following features:
\begin{itemize}
    \item Receive various events from multiple peripherals
    \item Generate various tasks for multiple peripherals
    \item 50 independently configurable ETM channels
    \item An ETM channel can be set up to receive any event, and map it to any task
    \item Each ETM channel can be enabled independently. If not enabled, the channel will not respond to the configured event and generate the task mapped to that event
    \item Peripherals supporting ETM include GPIO, LED PWM, general-purpose timers, RTC Timer, system timer, MCPWM, temperature sensor, ADC, I2S, LP CPU, GDMA, and PMU
\end{itemize}

Note that the 50 ETM channels are identical regarding their features and operations. Thus, in the following sections ETM channels are collectively referred to as channel\regindex{n} (where \regindex{n} ranges from 0 to 49).

\section{Functional Description}

\subsection{Architecture}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.6\textwidth]{\modulefiles/figures/etm_archi}
    \caption{Event Task Matrix Architecture}
    \label{fig:etm-architecture}
\end{figure}

Figure \ref{fig:etm-architecture} shows the architecture of the Event Task Matrix.

The Event Task Matrix has 50 independent channels. A channel can choose any event as  input, and map the event to any task as output (For configuration procedures, refer to Section \ref{sec:evntaskmatrix-events} and Section \ref{sec:evntaskmatrix-tasks} respectively). Each channel has an individual enable bit (For configuration procedures, refer to Section \ref{sec:evntaskmatrix-channel-enable}).

\begin{figure}[H]
    \centering
    \includegraphics[width=0.7\textwidth]{\modulefiles/figures/etm_channel}
    \caption{ETM Channel\regindex{n} Architecture}
    \label{fig:etm-channel}
\end{figure}

Figure \ref{fig:etm-channel} illustrates the structure of an ETM channel. The \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID} field configures the MUX (multiplexer) to select one of the events as the input of channel\regindex{n}. The \hyperref[fielddesc:SOCETMCH0TASKID]{SOC\_ETM\_CH\regindex{n}\_TASK\_ID} field configures the DEMUX (demultiplexer) to map the event selected by channel\regindex{n} to one of the tasks. \hyperref[fielddesc:SOCETMCHENABLE0]{SOC\_ETM\_CH\_ENABLE\regindex{n}} and \hyperref[fielddesc:SOCETMCHDISABLE0]{SOC\_ETM\_CH\_DISABLE\regindex{n}} are used to enable or disable channel\regindex{n}. \hyperref[fielddesc:SOCETMCHENABLED0]{SOC\_ETM\_CH\_ENABLED\regindex{n}} is used to indicate the status of the channel\regindex{n}.

Events from different channels can be optionally mapped to the same task (For example, field \hyperref[fielddesc:SOCETMCH0TASKID]{SOC\_ETM\_CH\regindex{n}\_TASK\_ID} of multiple channels can be configured with the same value, and field \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID} can be configured with the same or different values). In this case, when the event received by any of the channels is valid, the task is generated. If events received by multiple channels are valid at the same time, the task will be generated only once.

\subsection{Timing Considerations}

Figure \ref{fig:etm-clk-archi} shows the structure of clocks that drive received events, sent tasks, and ETM channels.

\begin{figure}[H]
    \centering
    \includegraphics[width=1\textwidth]{\modulefiles/figures/etm_clk_archi}
    \caption{Event Task Matrix Clock Architecture}
    \label{fig:etm-clk-archi}
\end{figure}

ETM is running at the AHB\_CLK domain (see Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}}). Each event corresponds to a pulse signal generated by the corresponding peripheral in its clock domain, while each task is mapped by the ETM to a pulse signal under its corresponding peripheral clock domain. The peripherals generating events, the Event Task Matrix, and peripherals receiving tasks are not necessarily running off the same clock and as such need to be synchronized. Therefore, there must be a minimum interval between two consecutive events to avoid event loss: to make sure the Event Task Matrix receives every event successfully, for peripherals generating event pulses, the interval between two consecutive pulses must be greater than one ETM clock cycle, namely $ceil(\frac{peripheral\_clock\_frequency}{ETM\_clock\_frequency})$ in the unit of peripheral clock cycles.

For example, assuming that event 1 generated by peripheral A is in the 80 MHz clock domain (PLL\_F80M\_CLK), and the ETM runs in the 40 MHz clock domain (AHB\_CLK). To receive each event 1 successfully, the interval between two consecutive event 1 must be greater than two peripheral A clock cycles (i.e. one ETM clock cycle).

Likewise, to make sure the Event Task Matrix maps the received event (i.e. event synchronized to the ETM's clock domain) successfully to a task, the interval between two consecutive event pulses in the ETM clock domain must be greater than one peripheral clock cycle, namely $ceil(\frac{ETM\_clock\_frequency}{peripheral\_clock\_frequency})$ in the unit of ETM clock cycles.

For example, assuming that task 1 received by peripheral B is in the 20 MHz clock domain (RC\_FAST\_CLK), and the ETM runs in the 40 MHz clock domain (AHB\_CLK). To map each received event successfully to task 1, the interval between two consecutive events must be greater than two ETM clock cycles (i.e. one peripheral B clock cycle).

As a result, to map two consecutive events generated by peripheral A to peripheral B, the interval between these two events must be $ceil(\frac{peripheral\_A\_clock\_frequency}{ETM\_clock\_frequency})*ceil(\frac{ETM\_clock\_frequency}{peripheral\_B\_clock\_frequency})$ in the unit of peripheral A clock cycles.

For example, assuming that event 1 generated by peripheral A is in the 80 MHz clock domain (PLL\_F80M\_CLK), task 1 received by peripheral B is in the 20 MHz clock domain (RC\_FAST\_CLK), and the ETM runs in the 40 MHz clock domain (AHB\_CLK). To successfully map each event 1 (generated by peripheral A) to task 1 (received by peripheral B), the interval between two consecutive event 1 must be greater than $2*2=4$ peripheral A clock cycles.



\subsection{Channel Control}
\label{sec:evntaskmatrix-channel-enable}
Each ETM channel can be independently configured to be enabled or disabled. When channel\regindex{n} is enabled and receives the event configured via \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID}, it maps the event to the task configured via \hyperref[fielddesc:SOCETMCH0TASKID]{SOC\_ETM\_CH\regindex{n}\_TASK\_ID}. When channel\regindex{n} is disabled, even if it receives the configured via \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID}, no task will be generated.

To enable ETM channel\regindex{n}:
\begin{enumerate}
    \item Write 1 to \hyperref[fielddesc:SOCETMCHENABLE0]{SOC\_ETM\_CH\_ENABLE\regindex{n}}
    \item Read \hyperref[fielddesc:SOCETMCHENABLED0]{SOC\_ETM\_CH\_ENABLED\regindex{n}}. 1 indicates that channel\regindex{n} has been enabled, and 0 indicates disabled
\end{enumerate}


To disable ETM channel\regindex{n}:
\begin{enumerate}
    \item Write 1 to \hyperref[fielddesc:SOCETMCHDISABLE0]{SOC\_ETM\_CH\_DISABLE\regindex{n}}
    \item Read \hyperref[fielddesc:SOCETMCHENABLED0]{SOC\_ETM\_CH\_ENABLED\regindex{n}}. 0 indicates that channel\regindex{n} is disabled, and 1 indicates enabled
\end{enumerate}

If \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID}  or  \hyperref[fielddesc:SOCETMCH0TASKID]{SOC\_ETM\_CH\regindex{n}\_TASK\_ID} is configured to 0, ETM channel\regindex{n} will also be disabled.

The complete procedure to configure ETM channel\regindex{n} is as follows:
\begin{enumerate}
    \item Enable the ETM's clock by writing 1 to \hyperref[fielddesc:PCRETMCLKEN]{PCR\_ETM\_CLK\_EN}
    \item Select the event to be received by channel\regindex{n} via \hyperref[fielddesc:SOCETMCH0EVTID]{SOC\_ETM\_CH\regindex{n}\_EVT\_ID}
    \item Select the task mapped to the received event via  \hyperref[fielddesc:SOCETMCH0TASKID]{SOC\_ETM\_CH\regindex{n}\_TASK\_ID}
    \item Write 1 to field \hyperref[fielddesc:SOCETMCHENABLE0]{SOC\_ETM\_CH\_ENABLE\regindex{n}}
    \item When channel\regindex{n} no longer needs to map the selected event to the selected task, disable channel\regindex{n} by setting \hyperref[fielddesc:SOCETMCHDISABLE0]{SOC\_ETM\_CH\_DISABLE\regindex{n}}. To configure a new event and task mapping, repeat Steps 1 to 3. If no configurations, channel\regindex{n} will remain disabled
    \item The ETM module can be reset by writing 1 and then 0 to the \hyperref[fielddesc:PCRETMRSTEN]{PCR\_ETM\_RST\_EN} field
\end{enumerate}
