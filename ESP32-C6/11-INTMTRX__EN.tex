\chapter{Interrupt Matrix (INTMTX)}
\label{mod:intmtrx}


\section{Overview}

The interrupt matrix embedded in \chipname{} independently routes peripheral interrupt sources to the ESP-RISC-V CPU's peripheral interrupts to timely inform CPU to process the coming interrupts. 

The \chipname{} has 77 peripheral interrupt sources that can be routed to any of the 28 CPU interrupts using the interrupt matrix.

\begin{tiplisting}
This chapter focuses on how to map peripheral interrupt sources to CPU interrupts. For more details about interrupt configuration, vector, and interrupt handling operations recommended by the ISA, please refer to Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}.
\end{tiplisting}
\vspace{-1em}


\section{Features}

The interrupt matrix embedded in \chipname{} has the following features:

\begin{itemize}
    \item 77 peripheral interrupt sources accepted as input
    \item 28 CPU peripheral interrupts generated to CPU as output
    \item Current interrupt status query of peripheral interrupt sources
    \item Multiple interrupt sources mapping to a single CPU interrupt (i.e., shared interrupts)
\end{itemize}

Figure \ref{fig:interrupt-matrix-structure} shows the structure of the interrupt matrix.

\section{Functional Description}

\subsection{Peripheral Interrupt Sources}

The \chipname{} has 77 peripheral interrupt sources in total. Table \ref{tab:peripheral-interrupt-config-status-source} lists all these sources and their mapping/status registers. 


\begin{itemize}
    \item Column ``No.": Peripheral interrupt source number, can be 0 \verb+~+ 76. 
    \item Column ``Chapter": in which chapter the interrupt source is described in detail.
    \item Column ``Interrupt Source": Name of the peripheral interrupt source.
    \item Column ``Interrupt Source Mapping Register": Registers used for routing the peripheral interrupt sources to CPU peripheral interrupts.
    \item Column ``Interrupt Status Register": Registers used for indicating the interrupt status of peripheral interrupt sources.
    \begin{itemize}
        \item Column ``Interrupt Status Register - Bit": Bit position in status register, indicating the interrupt status.
        \item Column ``Interrupt Status Register - Name": Name of status registers.
    \end{itemize}
\end{itemize}

\subsection{CPU Interrupts}

The \chipname{} implements its interrupt mechanism using an interrupt controller instead of RISC-V Privileged ISA specification. The CPU has 32 interrupts, numbered from 0 \verb+~+ 31. The interrupts numbered 0, 3, 4, and 7 are used by the CPU for core-local interrupts (CLINT), while the remaining 28 interrupts (numbered 1, 2, 5, 6, and 8 \verb+~+ 31) are available for use in the interrupt matrix.

Each CPU interrupt has the following properties:
\begin{itemize}
    \item Priority levels from 1 (lowest) to 15 (highest).
    \item Configurable as level-triggered or edge-triggered.
    \item Lower-priority interrupts mask-able by setting interrupt threshold.
\end{itemize}

\begin{tiplisting}
For detailed information about the function and configuration of CPU interrupts, see Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}. 
\end{tiplisting}
\vspace{-1em}



\subsection{Assign Peripheral Interrupt Source to CPU Interrupt}

In this section, the following terms are used to describe the operation of the interrupt matrix.


\begin{itemize}

\item Source\_\regindex{X}: stands for a peripheral interrupt source, wherein \regindex{X} means the number of this interrupt source in Table \ref{tab:peripheral-interrupt-config-status-source}.

\item INTMTX\_CORE0\_SOURCE\_\regindex{X}\_INTR\_MAP\_REG: stands for an interrupt source mapping register for the peripheral interrupt source (Source\_\regindex{X}).

\item Num\_P: the index of CPU interrupts which can be 1, 2, 5, 6, 8 \verb+~+ 31.
\item Interrupt\_P: stands for the CPU interrupt numbered as Num\_P.
\end{itemize}

\subsubsection{Assign One Peripheral Interrupt Source (Source\_\regindex{X}) to CPU}
Setting the corresponding source mapping  register INTMTX\_CORE0\_SOURCE\_\regindex{X}\_INTR\_MAP\_REG of Source\_\regindex{X} to Num\_P assigns this interrupt source to Interrupt\_P. 

\subsubsection{Assign Multiple Peripheral Interrupt Sources (Source\_\regindex{X}) to CPU}
Setting the corresponding source mapping register INTMTX\_CORE0\_SOURCE\_\regindex{X}\_INTR\_MAP\_REG of each interrupt source to the same Num\_P assigns multiple sources to the same Interrupt\_P. Any of these sources can trigger CPU Interrupt\_P. When an interrupt signal is generated, CPU should check the interrupt status registers to figure out which peripheral generated the interrupt. For more information, see Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}.

\subsubsection{Disable CPU Peripheral Interrupt Source (Source\_\regindex{X})}
Writing 0 to the INTMTX\_CORE0\_SOURCE\_\regindex{X}\_INTR\_MAP\_REG register disables the corresponding interrupt source.

\subsection{Query Current Interrupt Status of Peripheral Interrupt Source}

After enabling peripheral interrupt sources, users can query current interrupt status of a peripheral interrupt source by reading the bit value in \hyperref[regdesc:INTMTXCORE0INTSTATUS0REG]{INTMTX\_CORE0\_INT\_STATUS\_\regindex{n}\_REG} (read only). For the mapping between \hyperref[regdesc:INTMTXCORE0INTSTATUS0REG]{INTMTX\_CORE0\_INT\_STATUS\_\regindex{n}\_REG} and peripheral interrupt sources, please refer to Table \ref{tab:peripheral-interrupt-config-status-source}.
