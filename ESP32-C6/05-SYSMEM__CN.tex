\chapter{系统和存储器}
\label{mod:sysmem}

\section{概述}

\chipname{} 是一个超低功耗和高度集成的系统，它集成了：
\begin{itemize}
    \item 一颗高性能 RISC-V 32 位单核处理器 (HP CPU)，四级流水线架构，主频高达 160 MHz
    \item 一颗低功耗 RISC-V 32 位单核处理器 (LP CPU)，二级流水线架构，主频高达 20 MHz
\end{itemize}
所有的内部存储器、外部存储器以及外设都分布在 HP CPU 和 LP CPU 的总线上。


\section{主要特性}

\begin{itemize}
    \item {\bfseries 地址空间}
        \begin{itemize}
    	    \item 832 KB 内部存储器空间，可被指令总线或数据总线访问
    	    \item 832 KB 外设地址空间
    	    \item 16 MB 外部存储器虚拟地址空间，可被指令总线或数据总线访问
    	    \item 512 KB 内部 DMA 地址空间
        \end{itemize}
    \item {\bfseries 内部存储器}
        \begin{itemize}
            \item 320 KB 的 ROM 
	        \item 512 KB 的 HP 存储器 (HP SRAM)
	        \item 16 KB 的 LP 存储器 (LP SRAM)


	    \end{itemize}
    \item {\bfseries 外部存储器}
        \begin{itemize}
            \item 最大支持 16 MB 片外 flash
	    \end{itemize}
    \item \textbf{外设空间}
        \begin{itemize}
           \item 总计 51 个模块/外设 
        \end{itemize}
    \item {\bfseries GDMA}
        \begin{itemize}
            \item 8 个具有 GDMA 功能的模块/外设
        \end{itemize}
\end{itemize}

\section{功能描述}

\subsection{地址映射}

所有非保留地址均可通过指令总线和数据总线进行访问，即指令总线和数据总线访问的地址空间一样。

HP CPU 和 LP CPU 的数据总线与指令总线均为小端序。
HP CPU 和 LP CPU 可以通过数据总线进行单字节、双字节、四字节的数据访问。

CPU 能够：
\begin{itemize}
\item 通过数据总线与指令总线直接访问内部存储器。
\item（仅 HP CPU 可以）通过 Cache 访问映射到虚地址空间的外部存储器。
\item 通过数据总线直接访问模块/外设。
\end{itemize}

表 \ref{tab:sysmem-memory-cn} 描述了数据总线与指令总线中的各段地址所能访问的目标。

\subsection{内部存储器}
\label{para:sysmem-internal-memory}

\chipname{} 的内部存储器包含如下三种类型：

\begin{itemize}
    \item ROM (320 KB)：ROM 是只读存储器，不可编程。其中存放有一些系统底层软件 ROM 的 代码和只读数据。
    \item HP SRAM (512 KB)：HP SRAM 是易失性存储器，可以快速响应 HP CPU 和 LP CPU 的访问请求（HP CPU 通常是一个 HP CPU 时钟周期）。
    \item LP SRAM (16 KB)：LP SRAM 以静态 RAM (SRAM) 方式 实现，因此也是易失性存储器。但是，在 Deep-sleep 模式下，存放在 LP SRAM 中的数据不会丢失。LP SRAM 可以被 HP CPU 和 LP CPU 访问，通常用来存放一些在睡眠模式下仍需保持的程序指令和数据。

\end{itemize}





{\bfseries 1. ROM}

ROM 的容量为 320 KB，只读。如表 \ref{tab:sysmem-memory-cn} 所示，HP CPU 可以通过指令总线或数据总线地址段 0x4000\_0000 \verb+~+ 0x4004\_FFFF 访问这部分存储器。


{\bfseries 2. HP SRAM}
\label{other:sysmem-internal-sram-0}

HP SRAM 的容量为 512 KB，可读可写。如表 \ref{tab:sysmem-memory-cn} 所示，HP CPU 和 LP CPU 可以通过数据总线或指令总线访问这部分存储器。



{\bfseries 3. LP SRAM}

LP SRAM 容量为 16 KB，为可读可写的 SRAM。如表 \ref{tab:sysmem-memory-cn} 所示，HP CPU 和 LP CPU 可以通过数据总线和指令总线的共用地址段 0x5000\_0000 \verb+~+ 0x5000\_3FFF 访问这部分存储器。

LP SRAM 存在两种访问模式：
\begin{itemize}
    \item 高速访问模式：以 HP CPU 时钟频率进行访问，此时 HP CPU 能够无延迟访问 LP SRAM，但是 LP CPU 访问 LP SRAM 的延迟在十几个到几十个 LP CPU 周期不等。
    \item 低速访问模式：以 LP CPU 时钟频率进行访问，此时 LP CPU 能够无延迟访问 LP SRAM，但是 HP CPU 访问 LP SRAM 的延迟在十几个到几十个 HP CPU 周期不等。
\end{itemize}

用户可根据应用场景进行模式的切换，比如：
\begin{itemize}
    \item 如果 LP CPU 未启动，可以切换到高速模式，从而提高 HP CPU 访问速度。
    \item 如果 LP CPU 正在 LP SRAM 执行代码，可以切换到低速访问模式。
\end{itemize}
注意当 HP CPU 休眠时必须切换到低速模式。

切换配置流程：
\begin{itemize}
    \item 配置 \hyperref[fielddesc:LPAONFASTMEMMUXSEL]{LP\_AON\_FAST\_MEM\_MUX\_SEL} 选择模式：
    \begin{itemize}
        \item 1：高速模式
        \item 0：低速模式
    \end{itemize}
    \item 配置 \hyperref[fielddesc:LPAONFASTMEMMUXSELUPDATE]{LP\_AON\_FAST\_MEM\_MUX\_SEL\_UPDATE} 开始切换
    \item 查询 \hyperref[fielddesc:LPAONFASTMEMMUXSELSTATUS]{LP\_AON\_FAST\_MEM\_MUX\_SEL\_STATUS} 确认是否切换完成：
    \begin{itemize}
        \item 0：表示切换完成。
        \item 1：表示切换未完成。
    \end{itemize}
\end{itemize}


\subsection{外部存储器} 
\label{para:sysmem-external-memory}


\chipname{} 支持以 SPI、Dual SPI、Quad SPI、QPI 等接口形式连接片外 flash。 
\chipname{} 还支持基于 XTS-AES 算法的硬件手动加密和自动解密功能，从而保护开发者片外 flash 中的程序和数据。


\subsubsection{外部存储器地址映射}

HP CPU 借助高速缓存 (Cache) 来访问外部存储器。Cache 将根据内存管理单元 (Memory Management Unit, MMU) 中的信息把 HP CPU 的地址 (0x4200\_0000 \verb+~+ 0x42FF\_FFFF) 映射为访问片外存储的实地址。经过地址映射，\chipname{} 最大支持 16 MB 的片外 flash。请注意，指令总线地址空间 (16 MB) 和数据总线地址空间 (16 MB) 是共用的。 





\subsubsection{高速缓存}
如图 \ref{fig:sysmem-cache-arch} 所示， \chipname{} 采用一个只读的统一 Cache，为 4 路组相联，容量为 32 KB，块大小为 32 字节。
指令总线和数据总线可以同时访问该 Cache，Cache 经过仲裁对其中一个做出响应。当 Cache 缺失时，Cache 控制器会向外部存储器发起请求。

\subsubsection{Cache 操作}

\chipname{} Cache 支持如下几种操作：

\begin{enumerate}
\item \textbf{失效 (Invalidate)}：该操作用于删除 Cache 中的有效数据。该操作完成后，删除的数据将仅存于外部存储器中。如果 HP CPU 接着去访问该数据，那么需要访问外部存储器。该操作包括两种类型：整体失效 (Invalidate-All) 和手动失效 (Manual-Invalidate)。手动失效仅对 Cache 中落入指定区域的地址对应的数据做失效处理，而整体失效会对 Cache 中的所有数据做失效处理。

\item \textbf{预取 (Preload)}：该功能用于将指令和数据提前加载到 Cache 中。预取操作的最小单位为 1 个块。预取分为手动预取 (Manual-Preload) 和自动预取 (Auto-Preload)，手动预取是指硬件按软件指定的虚地址预取一段连续的数据；自动预取是指硬件根据当前命中/缺失（取决于配置）的地址，自动地预取一段连续的数据。


\item \textbf{锁定/解锁 (Lock/Unlock)}：该操作用于保护 Cache 中的数据不被替换掉。锁定分为预锁定和手动锁定。预锁定开启时，Cache 在填充缺失数据到 Cache 时，如果该数据落在指定区域，则该数据将被锁定，未落入指定区域的数据不会被锁定。 手动锁定开启时，Cache 检查 Cache 中的数据，并将落在指定区域的数据锁定，未落入指定区域的数据不会被锁定。
当缺失发生时，Cache 会优先替换掉未被锁定的那一路的数据，因此锁定区域的数据会一直保存在 Cache 中。但当所有路都被锁定时, Cache 将进行正常替换，就像所有路都没有被锁定一样。解锁是锁定的逆操作，但解锁只有手动解锁。

请注意，手动失效操作只对未被锁定的数据起作用。如果想对已锁定的数据执行手动失效操作，请先解锁这些数据。

\end{enumerate}

\subsection{GDMA 地址空间}

\chipname{} 中的 GDMA (General Direct Memory Access) 外设包含三个 TX 通道和三个 RX 通道，可提供 直接内存访问 (Direct Memory Access, DMA) 服务，包括：

\begin{itemize}
    \item 内部存储器中不同位置的数据搬运
    \item 模块/外设和内部存储器之间的数据搬运
\end{itemize}

GDMA 可以通过与数据总线完全相同的地址读写 HP SRAM，即 GDMA 通过地址 0x4080\_0000 \verb+~+ 0x4087\_FFFF 访问 HP SRAM。

\chipname{} 中共有八个外设/模块可以和 GDMA 联合工作。如图 \ref{fig:sysmem-gdma-peripheral} 所示，其中的八根竖线依次对应这八个具有 GDMA 功能的外设/模块，横线表示 GDMA 的某一通道（可为任意通道），竖线与横线的交点表示对应外设/模块可以访问 GDMA 的某一通道。同一行上有多个交点则表示这几个外设/模块不可以同时开启 GDMA 功能。


\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/05-SYSMEM-GDMA-peripheral-0818.eps}}
     \caption{具有 GDMA 功能的外设/模块}
    \label{fig:sysmem-gdma-peripheral}
\end{figure}

具有 GDMA 功能的模块/外设通过 GDMA 可以访问任何 GDMA 可以访问到的存储器。
更多关于 GDMA 的信息，请参考章节 \ref{mod:dma} \textit{\nameref{mod:dma}}。

\begin{tiplisting}
当使用 GDMA 访问任何存储器时，都需要获取对应的访问权限，否则访问将会失败。关于权限管理的更多信息，请参考章节 \ref{mod:permctrl} \textit{\nameref{mod:permctrl}}。
\end{tiplisting}
