\hypertarget{lpmailbox}{}
\chapter{LP Mailbox}
\label{mod:lpmailbox}

\section{Overview}

\chipname{} integrates an LP Mailbox module which provides an efficient inter-core communication mechanism between the LP CPU and HP CPU0/1. The LP Mailbox module comprises of sixteen 32-bit message registers that the LP CPU and HP CPU0/1 can use to store and exchange message. Inter-core communication between LP CPU and HP CPU0/1 is achieved through an interrupt mechanism implemented within the LP Mailbox module.

\section{Features}
\chipname{} LP Mailbox has the following features:
\begin{itemize}
    \item Sixteen 32-bit message registers for inter-core communication
    \item LP CPU external interrupt signal: \hyperref[tab:lpmailbox-int-sources]{MB\_LP\_INTR}
    \item HP CPU\regindex{x} (\regindex{x}: 0 \verb+~+ 1) external interrupt signal: \hyperref[tab:lpmailbox-int-sources]{MB\_HP\_INTR}
\end{itemize}

\subsection{Message Registers}
The LP Mailbox provides sixteen 32-bit message registers in total, denoted as \hyperref[regdesc:MBMESSAGE0REG]{MB\_MESSAGE\_0\_REG} \verb+~+ \hyperref[regdesc:MBMESSAGE15REG]{MB\_MESSAGE\_15\_REG}, for caching inter-core communication messages. Both LP CPU and HP CPU0/1 can read and write these message registers to pass or retrieve message.

Read and write message registers is done through the APB bus, using APB\_CLK, which is the only one clock in LP Mailbox. For more information about the clock, please refer to Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}}.

\subsection{Interrupts}
\label{sec:lpmailbox-interrupt}


\chipname{} LP Mailbox can generate the following external interrupt signals:
    \begin{itemize}
        \item MB\_LP\_INTR
        \item MB\_HP\_INTR
    \end{itemize}
MB\_LP\_INTR is directly mapped to LP CPU (see \textit{\nameref{mod:ulp}}), and MB\_HP\_INTR can be mapped to HP CPU0/1 through the \hyperref[mod:intmtrx]{Interrupt Matrix}.
These interrupt signals are generated by internal interrupt sources of the LP Mailbox module. Table \ref{tab:lpmailbox-int-sources} lists the internal interrupt sources, interrupt trigger conditions, and generated interrupt signals of the LP Mailbox.

\begin{longtable}{ | L{5cm} | L{6.5cm} | L{4cm} | }
\caption{LP Mailbox's Internal Interrupt Sources} \label{tab:lpmailbox-int-sources}
\\ \hline
\rowcolor{lightgray}
\textbf{Internal Interrupt Source} & \textbf{Trigger Condition}  &  \textbf{Interrupt Signal}\\ \hline
\label{int:MBHPmINT} MB\_HP\_\regindex{m}\_INT &  Write to MB\_MESSAGE\_\regindex{m}\_REG &  MB\_HP\_INTR  \\ \hline
\label{int:MBLPnINT} MB\_LP\_\regindex{n}\_INT &  Write to MB\_MESSAGE\_\regindex{n}\_REG &  MB\_LP\_INTR  \\ \hline
\end{longtable}
\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
    \item \regindex{m} and \regindex{n} can be 0 \verb+~+ 15.
    \item\hyperref[tab:lpmailbox-int-sources]{MB\_HP\_INTR} can also serve as an external interrupt source for LP CPU and be mapped to LP CPU by LP\_INTR\_MAP as illustrated in Figure \ref{fig:lpmbsysdigram}. For more information about LP\_INTR\_MAP, please refer to Chapter \ref{mod:ulp} \textit{\nameref{mod:ulp}}.
    \item\hyperref[tab:lpmailbox-int-sources]{MB\_LP\_INTR} can also serve as an external interrupt source for HP CPU\regindex{x} and can be mapped to HP CPU\regindex{x} through the \hyperref[mod:intmtrx]{Interrupt Matrix}.
    \item When the LP Mailbox is required to assign different interrupt sources for HP CPU0 and HP CPU1, it can map \hyperref[tab:lpmailbox-int-sources]{MB\_HP\_INTR} and  \hyperref[tab:lpmailbox-int-sources]{MB\_LP\_INTR} independently to HP CPU0 and HP CPU1 using the \hyperref[mod:intmtrx]{Interrupt Matrix}.
\end{itemize}
\end{tiplisting}

\subsection{Inter-Core Communication}
When communication is required between LP CPU and HP CPU\regindex{x}, subset of or all message registers in the LP Mailbox can be utilized to share message as needed. The synchronization of message can be achieved through the corresponding interrupts. During communication, software must ensure that only the sender is allowed to write to the message registers to prevent data corruption.

\begin{itemize}
    \item Example of initiating communication from HP CPU0 to LP CPU using MB\_MESSAGE\_\regindex{n}\_REG:
    \begin{itemize}
        \item Set \hyperref[fielddesc:MBLP0INTENA]{MB\_LP\_\regindex{n}\_INT\_ENA} to enable the MB\_LP\_\regindex{n}\_INT interrupt.
        \item HP CPU0 writes to \hyperref[regdesc:MBMESSAGE0REG]{MB\_MASSEGE\_\regindex{n}\_REG}, caching shared message.
        \item LP CPU checks \hyperref[regdesc:MBLPINTSTREG]{MB\_LP\_INT\_ST\_REG} interrupt status register, finding the  \hyperref[fielddesc:MBLP0INTST]{MB\_LP\_\regindex{n}\_ST} status bit set.
        \item LP CPU reads \hyperref[regdesc:MBMESSAGE0REG]{MB\_MASSEGE\_\regindex{n}\_REG} to retrieve shared message.
    \end{itemize}
    \item Example of initiating communication from LP CPU to HP CPU0:
    \begin{itemize}
        \item Set \hyperref[fielddesc:MBHP0INTENA]{MB\_HP\_\regindex{m}\_INT\_ENA} to enable the MB\_HP\_\regindex{m}\_INT interrupt.
        \item Configure the \hyperref[mod:intmtrx]{Interrupt Matrix} to map MB\_HP\_INTR to HP CPU0. See Chapter \ref{mod:intmtrx} \textit{\nameref{mod:intmtrx}}.
        \item LP CPU writes to \hyperref[regdesc:MBMESSAGE0REG]{MB\_MASSEGE\_\regindex{m}\_REG}, caching shared messages.
        \item HP CPU0 checks the \hyperref[regdesc:MBHPINTSTREG]{MB\_HP\_INT\_ST\_REG} interrupt status register, finding the  \hyperref[fielddesc:MBHP0INTST]{MB\_HP\_\regindex{m}\_ST} status bit set.
        \item HP CPU0 reads \hyperref[regdesc:MBMESSAGE0REG]{MB\_MASSEGE\_\regindex{m}\_REG} to retrieve shared messages.
    \end{itemize}
\end{itemize}
