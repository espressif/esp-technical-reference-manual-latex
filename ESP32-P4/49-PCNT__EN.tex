\chapter{Pulse Count Controller (PCNT)}
\label{mod:pcnt}

%%%%%%%%%%%%%%%%%%%%
%%% Introduction %%%
%%%%%%%%%%%%%%%%%%%%

The pulse count controller (PCNT) is designed to count input pulses.

\section{Introduction}

\begin{figure}[ht]
\includegraphics[width=0.4\textwidth]{\modulefiles/figures/pcnt-block-diagram.eps}
\centering
\caption{PCNT Overview}
\label{fig:pcnt-block-diagram}
\end{figure}




As shown in Figure \ref{fig:pcnt-block-diagram}, PCNT has four independent pulse counters called units, which have their groups of registers. Each unit includes two channels (ch0 and ch1) and a 16-bit signed counter. Each channel has an input pulse signal and an input signal filtering module. The 16-bit signed counter can be configured for incremental or decremental counting. The clock for PCNT module is APB\_CLK.


\section{Feature List}
\label{sec:pcnt-features}

A PCNT has the following features:

\begin{itemize}
    \item Four independent pulse counters (units) that count from 1 to 65535
    \item Each unit consists of two independent channels sharing one pulse counter
    \item All channels have input pulse signals (e.g., sig\_ch0\_u\regindex{n}) with their corresponding control signals (e.g., ctrl\_ch0\_u\regindex{n})
    \item Independently filter glitches of input pulse signals (sig\_ch0\_u\regindex{n} and sig\_ch1\_u\regindex{n}) and control signals (ctrl\_ch0\_u\regindex{n} and ctrl\_ch1\_u\regindex{n}) on each unit
    \item Each channel has the following parameters:
        \begin{enumerate}
            \item Selection between counting on rising or falling edges of the input pulse signal
            \item Configuration to Increment, Decrement, or Disable counter mode for control signal's high and low states
        \end{enumerate}
    \item Maximum frequency of pulses: $\frac{f_{APB\_CLK}}{2}$
\end{itemize}

\section{Architectural Overview}
\label{sec:pcnt-arch-overview}

Each unit of PCNT includes two channels (ch0 and ch1) and the functionality of the two channels is identical. The remainder of the chapter will take channel 0 (ch0) as example and \regindex{n} denotes the number of a unit from 0 \verb+~+ 3. Each channel of PCNT has two input signals:

\begin{enumerate}
    \item One input pulse signal (e.g., sig\_ch0\_u\regindex{n}, the input pulse signal for ch0 of unit \regindex{n} ch0)
    \item One control signal (e.g., ctrl\_ch0\_u\regindex{n}, the control signal for ch0 of unit \regindex{n} ch0)
\end{enumerate}

\begin{figure}[ht]
\includegraphics[width=1\textwidth]{\modulefiles/figures/pcnt_arch_updated_20221026}
\centering
\caption{PCNT Architectural Overview}
\label{fig:pcnt-arch}
\end{figure}



Figure \ref{fig:pcnt-arch} shows PCNT's architecture. As stated above, ctrl\_ch0\_u\regindex{n} is the control signal for ch0 of unit \regindex{n}. Its high and low states can be assigned in different counter modes and used for pulse counting of the channel's input pulse signal sig\_ch0\_u\regindex{n} on falling or rising edges.


\section{Functional Description}
\label{sec:pcnt-func-descr}

The available counter modes of PCNT are as follows:

\begin{itemize}
\item Increment mode: When a channel detects an active edge of sig\_ch0\_u\regindex{n} (can be configured by software), the counter value pulse\_cnt increases by 1. Upon reaching \hyperref[fielddesc:PCNTCNTHLIMUN]{PCNT\_CNT\_H\_LIM\_U\regindex{n}}, pulse\_cnt is cleared. If \hyperref[fielddesc:PCNTCNTPAUSEUN]{PCNT\_CNT\_PAUSE\_U\regindex{n}} is set to 1, then pulse\_cnt freezes.

\item Decrement mode: When a channel detects an active edge of sig\_ch0\_u\regindex{n} (can be configured by software), the counter value pulse\_cnt decreases by 1. Upon reaching \hyperref[fielddesc:PCNTCNTLLIMUN]{PCNT\_CNT\_L\_LIM\_U\regindex{n}}, pulse\_cnt is cleared. If \hyperref[fielddesc:PCNTCNTPAUSEUN]{PCNT\_CNT\_PAUSE\_U\regindex{n}} is set to 1, then pulse\_cnt freezes.

\item Disable mode: Counting is disabled, and the counter value pulse\_cnt freezes.
\end{itemize}

Each unit has one filter for all its control and input pulse signals. A filter can be enabled by setting the bit \hyperref[fielddesc:PCNTFILTERENUN]{PCNT\_FILTER\_EN\_U\regindex{n}}. The filter monitors the signals and ignores all the noise, i.e., the glitches with pulse widths shorter than \hyperref[fielddesc:PCNTFILTERTHRESUN]{PCNT\_FILTER\_THRES\_U\regindex{n}} APB clock cycles in length.

As shown on Figure \ref{fig:pcnt-arch}, each unit has two channels which process different input pulse signals and increase or decrease values via their respective inc\_dec modules, then the two channels send these values to the counter module which has a 16-bit wide signed register. This counter can be suspended by setting \hyperref[fielddesc:PCNTCNTPAUSEUN]{PCNT\_CNT\_PAUSE\_U\regindex{n}}, and cleared by setting \hyperref[fielddesc:PCNTPULSECNTRSTUN]{PCNT\_PULSE\_CNT\_RST\_U\regindex{n}}.

The PCNT has five watchpoints that share one interrupt. The interrupt can be enabled or disabled by interrupt enable signals of each individual watchpoint.

\begin{itemize}
    \item Maximum count value: When pulse\_cnt reaches \hyperref[fielddesc:PCNTCNTHLIMUN]{PCNT\_CNT\_H\_LIM\_U\regindex{n}}, a high limit interrupt is triggered and \hyperref[fielddesc:PCNTCNTTHRHLIMLATUN]{PCNT\_CNT\_THR\_H\_LIM\_LAT\_U\regindex{n}} is high.
    \item Minimum count value: When pulse\_cnt reaches \hyperref[fielddesc:PCNTCNTLLIMUN]{PCNT\_CNT\_L\_LIM\_U\regindex{n}}, a low limit interrupt is triggered and \\\hyperref[fielddesc:PCNTCNTTHRLLIMLATUN]{PCNT\_CNT\_THR\_L\_LIM\_LAT\_U\regindex{n}} is high.
    \item Two threshold values: When pulse\_cnt equals either \hyperref[fielddesc:PCNTCNTTHRES0UN]{PCNT\_CNT\_THRES0\_U\regindex{n}} or \hyperref[fielddesc:PCNTCNTTHRES1UN]{PCNT\_CNT\_THRES1\_U\regindex{n}}, an interrupt is triggered and either \hyperref[fielddesc:PCNTCNTTHRTHRES0LATUN]{PCNT\_CNT\_THR\_THRES0\_LAT\_U\regindex{n}} or \hyperref[fielddesc:PCNTCNTTHRTHRES1LATUN]{PCNT\_CNT\_THR\_THRES1\_LAT\_U\regindex{n}} is high respectively.
    \item Zero: When pulse\_cnt is 0, an interrupt is triggered and \hyperref[fielddesc:PCNTCNTTHRZEROLATUN]{PCNT\_CNT\_THR\_ZERO\_LAT\_U\regindex{n}} is valid.
\end{itemize}

If \hyperref[fielddesc:PCNTCNTHLIMUN]{PCNT\_CNT\_H\_LIM\_U\regindex{n}} and/or \hyperref[fielddesc:PCNTCNTLLIMUN]{PCNT\_CNT\_L\_LIM\_U\regindex{n}} are reconfigured by software when PCNT is working, the new configuration will take effect after pulse\_cnt counts to any of the above five watchpoints; If \hyperref[fielddesc:PCNTCNTTHRES0UN]{PCNT\_CNT\_THRES0\_U\regindex{n}} and/or \hyperref[fielddesc:PCNTCNTTHRES1UN]{PCNT\_CNT\_THRES1\_U\regindex{n}} are reconfigured by software, the new configuration will take effect immediately.

\section{Programming Procedures}
\label{sec:pcnt-prog-procedures}

In each unit, channel 0 and channel 1 can be configured to work independently or together. The three subsections below provide details of channel 0 incrementing independently, channel 0 decrementing independently, and channel 0 and channel 1 incrementing together. For other working modes not elaborated in this section (e.g., channel 1 incrementing/decremeting independently, or one channel incrementing while the other decrementing), reference can be made to these three subsections.

\subsection{Channel 0 Incrementing Independently} \label{sec:pcnt-func-descr-one-channel-inc}

\begin{figure}[H]
\fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/pulse_inc}}
\centering
\caption{Channel 0 Up Counting Diagram}
\label{fig:pcnt-inc}
\end{figure}

Figure \ref{fig:pcnt-inc} illustrates how channel 0 is configured to increment independently on the rising edge of sig\_ch0\_u\regindex{n} while channel 1 is disabled (see Subsection \ref{sec:pcnt-func-descr} for how to disable channel 1). The configuration of channel 0 is shown below.

\begin{itemize}
    \item \hyperref[fielddesc:PCNTCH0LCTRLMODEUN]{PCNT\_CH0\_LCTRL\_MODE\_U\regindex{n}}=0: When ctrl\_ch0\_u\regindex{n} is low, the counter mode specified for the low state turns on, in this case it is Increment mode.
    \item \hyperref[fielddesc:PCNTCH0HCTRLMODEUN]{PCNT\_CH0\_HCTRL\_MODE\_U\regindex{n}}=2: When ctrl\_ch0\_u\regindex{n} is high, the counter mode specified for the low state turns on, in this case it is Disable mode.
    \item \hyperref[fielddesc:PCNTCH0POSMODEUN]{PCNT\_CH0\_POS\_MODE\_U\regindex{n}}=1: The counter increments on the rising edge of sig\_ch0\_u\regindex{n}.
    \item \hyperref[fielddesc:PCNTCH0NEGMODEUN]{PCNT\_CH0\_NEG\_MODE\_U\regindex{n}}=0: The counter idles on the falling edge of sig\_ch0\_u\regindex{n}.
    \item \hyperref[fielddesc:PCNTCNTHLIMUN]{PCNT\_CNT\_H\_LIM\_U\regindex{n}}=5: When pulse\_cnt counts up to \hyperref[fielddesc:PCNTCNTHLIMUN]{PCNT\_CNT\_H\_LIM\_U\regindex{n}}, it is cleared.
\end{itemize}

\subsection{Channel 0 Decrementing Independently} \label{sec:pcnt-one-channel-dec}

\begin{figure}[H]
\fbox{\includegraphics[width=0.8\textwidth]{\modulefiles/figures/pulse_dec}}
\centering
\caption{Channel 0 Down Counting Diagram}
\label{fig:pcnt-dec}
\end{figure}

Figure \ref{fig:pcnt-dec} illustrates how channel 0 is configured to decrement independently on the rising edge of sig\_ch0\_u\regindex{n} while channel 1 is disabled. The configuration of channel 0 in this case differs from that in Figure \ref{fig:pcnt-inc} in the following aspects:

\begin{itemize}
    \item \hyperref[fielddesc:PCNTCH0POSMODEUN]{PCNT\_CH0\_POS\_MODE\_U\regindex{n}}=2: the counter decrements on the rising edge of sig\_ch0\_u\regindex{n}.
    \item \hyperref[fielddesc:PCNTCNTLLIMUN]{PCNT\_CNT\_L\_LIM\_U\regindex{n}}=-5: when pulse\_cnt counts down to \hyperref[fielddesc:PCNTCNTLLIMUN]{PCNT\_CNT\_L\_LIM\_U\regindex{n}}, it is cleared.
\end{itemize}

\subsection{Channel 0 and Channel 1 Incrementing Together}\label{sec:pcnt-two-channels-inc}

\begin{figure}[H]
\fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/pulse_add_2}}
\centering
\caption{Two Channels Up Counting Diagram}
\label{fig:pcnt-two-inc}
\end{figure}

Figure \ref{fig:pcnt-two-inc} illustrates how channel 0 and channel 1 are configured to increment on the rising edge of sig\_ch0\_u\regindex{n} and sig\_ch1\_u\regindex{n} respectively at the same time. It can be seen in Figure \ref{fig:pcnt-two-inc} that control signal ctrl\_ch0\_u\regindex{n} and ctrl\_ch1\_u\regindex{n} have the same waveform, so as input pulse signal sig\_ch0\_u\regindex{n} and sig\_ch1\_u\regindex{n}. The configuration procedure is shown below.

\begin{itemize}
    \item For channel 0:
    \begin{itemize}
        \item \hyperref[fielddesc:PCNTCH0LCTRLMODEUN]{PCNT\_CH0\_LCTRL\_MODE\_U\regindex{n}}=0: When ctrl\_ch0\_u\regindex{n} is low, the counter mode specified for the low state turns on, in this case it is Increment mode.

        \item \hyperref[fielddesc:PCNTCH0HCTRLMODEUN]{PCNT\_CH0\_HCTRL\_MODE\_U\regindex{n}}=2: When ctrl\_ch0\_u\regindex{n} is high, the counter mode specified for the low state turns on, in this case it is Disable mode.

        \item \hyperref[fielddesc:PCNTCH0POSMODEUN]{PCNT\_CH0\_POS\_MODE\_U\regindex{n}}=1: The counter increments on the rising edge of sig\_ch0\_u\regindex{n}.

        \item \hyperref[fielddesc:PCNTCH0NEGMODEUN]{PCNT\_CH0\_NEG\_MODE\_U\regindex{n}}=0: The counter idles on the falling edge of sig\_ch0\_u\regindex{n}.

    \end{itemize}

    \item For channel 1:
    \begin{itemize}
        \item \hyperref[fielddesc:PCNTCH1LCTRLMODEUN]{PCNT\_CH1\_LCTRL\_MODE\_U\regindex{n}}=0: When ctrl\_ch1\_u\regindex{n} is low, the counter mode specified for the low state turns on, in this case it is Increment mode.

        \item \hyperref[fielddesc:PCNTCH1HCTRLMODEUN]{PCNT\_CH1\_HCTRL\_MODE\_U\regindex{n}}=2: When ctrl\_ch1\_u\regindex{n} is high, the counter mode specified for the low state turns on, in this case it is Disable mode.

        \item \hyperref[fielddesc:PCNTCH1POSMODEUN]{PCNT\_CH1\_POS\_MODE\_U\regindex{n}}=1: The counter increments on the rising edge of sig\_ch1\_u\regindex{n}.

        \item \hyperref[fielddesc:PCNTCH1NEGMODEUN]{PCNT\_CH1\_NEG\_MODE\_U\regindex{n}}=0: The counter idles on the falling edge of sig\_ch1\_u\regindex{n}.

    \end{itemize}

    \item \hyperref[fielddesc:PCNTCNTHLIMUN]{PCNT\_CNT\_H\_LIM\_U\regindex{n}}=10: When pulse\_cnt counts up to \hyperref[fielddesc:PCNTCNTHLIMUN]{PCNT\_CNT\_H\_LIM\_U\regindex{n}}, it is cleared.

\end{itemize}
