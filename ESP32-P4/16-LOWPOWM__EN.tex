\hypertarget{lowpowm}{}
\chapter{Low-Power Management}\label{mod:lowpowm}

\section{Overview}

\chipname{} features an advanced low-power management system that optimizes the chip's power consumption while maintaining its high performance.

The low-power management system employs various power-saving techniques such as sleep modes, dynamic voltage and frequency scaling, and peripheral power gating to minimize the chip's power consumption.

At the core of this low-power management system is the power management unit (PMU) hardware component. The PMU powers up and down different power domains of the chip to achieve the best balance between chip performance, power consumption, and wake-up latency.

\section{Terminology}

The following terms related to low-power management are defined in the context of the \textit{\chipname{} Technical Reference Manual}
to help readers better understand this document:

\begin{longtable}[c]{ R{6cm} p{10cm} }
\textbf{Low-power management} & Refers to the whole system that manages the chip's power consumption. \\
\textbf{Power management unit (PMU)} & Refers to the specific hardware module that controls power up and down for power domains, clocks, and power-related logic. \\
\textbf{Power domain} & Refers to the smallest unit that can be independently powered up or down. A power domain can contain one or multiple modules within the chip. \\
\textbf{PMU states} & Refers to the states of the PMU's state machine. Users can configure the clock gating and power gating of a power domain in each of the states.\\
\textbf{Power modes} & Refers to the preset power modes that power up different domains for typical application scenarios.\\
\end{longtable}

\section{Features}

The PMU has the following features:

\begin{itemize}
    \item Supports three configurable PMU states. Software can flexibly configure them according to the needs.
        \begin{itemize}
        \item HP\_ACTIVE
        \item HP\_SLEEP
        \item LP\_SLEEP
        \end{itemize}
    \item Supports \iftagged{ESP32-P4-latest}{five}{four} preset power modes that suit various typical usage scenarios:
    \begin{itemize}
        \item Active
        \iftagged{ESP32-P4-latest}{
            \item Light-sleep0 (This mode does not power down the peripherals, but only the CPU. The clocks are not turned off. The power consumption is high, and the wake-up latency is short.)
            \item Light-sleep1 (This mode does not power down the peripherals, but only turns off the clocks. The power consumption is relatively higher and the wake-up latency shorter.)
            \item Light-sleep2 (This mode powers down the peripherals and needs data backup and restoration, resulting in lower power consumption and a longer wake-up latency.)
        }{
            \item Light-sleep0 (This mode does not power down the peripherals, but only turns off the clocks, resulting in higher power consumption and a shorter wake-up latency.)
            \item Light-sleep1 (This mode powers down the peripherals and needs data backup and restoration, resulting in lower power consumption and a longer wake-up latency.)
        }
        \item Deep-sleep
    \end{itemize}
    \item LP SRAM (32 KB): Keep power up in all modes, can store HP CPU's non-volatile data, and LP CPU's instructions and data.
    \item Supports a programmable retention DMA that backs up and restores the CPU and peripherals status when the chip switches between PMU states.
    \item Supports a power controller that controls the power and clocks depending on the power modes.
\end{itemize}

\section{Functional Description}

\chipname{}'s low-power management involves the following components:

\begin{itemize}
    \item \textbf{Power scheme}: The power scheme of \chipname{} includes power regulators, digital power domains, analog power domains, etc.

    \item \textbf{PMU controller}: It is the core part of the PMU, controlling the power up and down of power domains, clocks, etc.

    \item \textbf{One RTC timer}: For more information, please refer to Chapter \ref{mod:rtctimer} \textit{\nameref{mod:rtctimer}}.

    \item \textbf{16 LP GPIO pins (GPIO0–GPIO15)}: These pins are always powered up and are not affected by any low-power modes, which makes them suitable for working as wake-up sources when the chip is in low-power modes. These pins can also work as regular GPIOs. For more information on LP GPIOs, refer to Chapter \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}}.

    \item \textbf{32 KB SRAM}: The 32 KB SRAM is accessible to both HP CPU and LP CPU. It works under the HP CPU clock when accessed by the HP CPU and under the LP CPU clock when accessed by the LP CPU.

    \item \textbf{Brownout detector}: Monitors the power of the voltage supply pins, ensuring stable chip operation and preventing the SoC from potential malfunction if it is subject to glitches or under-voltage. For more information, refer to Chapter \ref{mod:brownout} \textit{\nameref{mod:brownout}}.
\end{itemize}

\subsubsection{Regulators}\label{sec:lowpowm-regulators}

As shown in Figure \ref{fig:lowpowm-power-structure}, the analog section of \chipname{} includes one external DCDC with feedback regulation and two internal HP/LP system regulators, responsible for regulating the power supply for different power domains.

\begin{itemize}
    \item The \textbf{DCDC voltage regulation feedback system} adjusts the voltage of the external DCDC.
    \item The \textbf{HP (high-performance) system regulator} has strong drive capability, higher power consumption, and adjustable output voltage. It is only suitable for power-up and sleep modes.
\item The \textbf{LP (low-power) system regulator} regulates the power supply of low-power modules. Its output voltage is also adjustable.
\end{itemize}

The DCDC voltage regulation feedback system and the HP system regulator are responsible for powering the HP system. The external DCDC can adjust its output voltage through the internal voltage feedback system. By default, the chip is powered by the HP system regulator when powered up. After power-up is complete, it is recommended to switch to the DCDC power supply for better efficiency and load capacity.

\subsubsection{Digital Power Domains}\label{sec:lowpowm_digital_domain}

\begin{itemize}
    \item The HP system contains the following digital power domains:
    \begin{itemize}
        \iftagged{ESP32-P4-latest}{
            \item \textbf{CPU (PD\_HP\_CPU)}: Mainly includes the CPU and peripherals that support its operation.
            \item \textbf{Peripherals + ROM (PD\_TOP)}: Mainly includes the bus and HP peripherals.
        }{
            \item \textbf{Peripherals + ROM + CPU (PD\_TOP)}: Mainly includes the bus and HP peripherals.
        }

        \item \textbf{L2MEM\_G0–G5}: It is divided into six sub-power domains, as shown in Figure \ref{fig:lowpowm-power-structure}.

        \item \textbf{HP\_CNNT (PD\_HP\_CNNT)}: Mainly includes USB and SDIO modules.
    \end{itemize}

There is an independent power switch between the regulator and each power domain, enabling up/down control of the digital power domain.

    \item The LP system contains the following digital power domains:
    \begin{itemize}
        \item \textbf{LP PD peripherals (PD\_LP\_PERI)}: Mainly includes LP CPU and LP peripherals.
        \item \textbf{LP always-on (PD\_AON)}: It mainly includes LP always-on peripherals (e.g., RTC timer) and PMU controller. This power domain keeps powered up all the time.
    \end{itemize}
\end{itemize}

\subsubsection{Analog Power Domains}

\begin{itemize}
    \item The HP system contains the following analog power domains:
    \begin{itemize}
    \item CPLL\_CLK
    \item SPLL\_CLK
    \item Audio PLL
    \item SDIO PLL
    \end{itemize}

    \item The LP system contains the following analog power domains:
    \begin{itemize}
    \item External Main Clock
    \item Fast RC Oscillator
    \end{itemize}
\end{itemize}

\subsubsection{Battery Power Domain}

The battery power domain (VDD\_BAT) serves as a backup power domain. When all external power supplies (VDD\_ANA) are turned off, the low-power system regulators, LP\_SLOW\_CLK, and LP\_FAST\_CLK can all be powered by the VDD\_BAT supply.

\subsection{PMU}

The PMU of \chipname{} controls power consumption-related components of each power domain, such as power and clock. PMU consists of the following major parts:

\begin{itemize}
\item \textbf{PMU main state machine}: It records and switches the PMU states.
\item \textbf{Sleep/wake-up controller}: It sends sleep or wake-up requests to the PMU main state machine.
\item \textbf{Power controllers}: They control the power and clock signals depending on the power modes of the chip. The power controllers include:
    \begin{itemize}
    \item \textbf{Digital power controller}: It powers up or down the digital power domains.
    \item \textbf{Analog power controller}: It enables the analog modules, such as the regulators, analog clocks, etc.
    \item \textbf{Clock controller}: It manages the clock gating of peripherals and selects analog clock sources for digital clocks.
    \item \textbf{Data backup controller}: It controls the data backup and restore process when the chip switches between PMU states.
    \item \textbf{System controller}: It controls some system-level modules, such as suspending watchdog functionality in sleep mode (when the CPU is unavailable).
    \end{itemize}
\end{itemize}

The PMU workflow involves the following steps:

\begin{itemize}
\item The sleep/wake-up controller sends sleep or wake-up requests to the PMU main state machine.
\item The PMU main state machine generates power gating, clock gating, and reset signals.
\item The power controllers and clock controllers power up or down different power domains and clocks based on the signals generated by the PMU main state machine, allowing the chip to enter or exit different low-power modes.
\end{itemize}

The PMU workflow is shown in Figure \ref{fig:lowpowm-workflow}. The definitions of HP\_SWITCH and LP\_SWITCH are as follows:

\begin{itemize}
    \item HP\_SWITCH: The intermediate state for the HP system transitioning between HP\_ACTIVE and HP\_SLEEP. The hardware completes the control switching of various controllers in this state.
    \item LP\_SWITCH: The intermediate state for the LP system transitioning between HP\_SLEEP and LP\_SLEEP. The hardware completes the control switching of various controllers in this state.
\end{itemize}

The following sections describe the main parts of PMU.

\subsubsection{PMU Main State Machine}

The PMU main state machine receives sleep and wake-up signals, changes the state of power and clock through the power controllers, thereby switching PMU states, and achieving a balance between performance and power consumption of the chip.

The PMU main state machine supports three PMU states, each controlled by different sleep and wake-up signals, supporting software customization of power and clocks. These PMU states allow the software to expand power modes for various application scenarios. The three PMU states are:

\begin{itemize}
    \item \textbf{HP\_ACTIVE}: The circuits on the chip are powered up to a maximum, supporting the HP system and LP system operation.
    \item \textbf{HP\_SLEEP}: The HP system is in sleep, supporting LP peripherals operation.
    \item \textbf{LP\_SLEEP}: The HP system and LP peripherals are in sleep, while the always-on circuits remain operational.
\end{itemize}

\begin{tiplisting}
    The division of the HP and LP systems is as follows:
    \begin{itemize}
    \item LP system (peripherals): LP RISC-V 32-bit Microprocessor, LP GPIO, LP UART, LP I2C, LP I2S, touch sensor, temperature sensor, LP ADC, LP SPI
    \item LP system (always-on circuits): PMU, RTC Watchdog Timer, Super Watchdog
    \item HP system: All peripherals except those belonging to the LP system.
    \end{itemize}

For more details, please refer to \href{https://www.espressif.com/sites/default/files/documentation/esp32-p4_datasheet_en.pdf}{\textit{\chipname{} Datasheet}} > \textit{Functional Block Diagram}.
\end{tiplisting}

HP\_ACTIVE is the operating state of the HP system, HP\_SLEEP is the sleep state of the HP system, LP\_SLEEP is the sleep state of the LP system. The operating state of the LP system reuses the HP\_SLEEP state and is therefore also referred to as HP\_SLEEP.

\begin{itemize}
    \item If a module belongs to the HP system, its power up/down can be configured in the HP\_ACTIVE/HP\_SLEEP states. It reuses the HP\_SLEEP state in the LP\_SLEEP state.
    \item If a module belongs to the LP system, its power up/down can be configured in the HP\_SLEEP/LP\_SLEEP state. It reuses the HP\_SLEEP state in the HP\_ACTIVE state.
\end{itemize}

Take the HP CPU as an example. The HP CPU belongs to the HP system, so it can be configured to power up/down in the HP\_ACTIVE/HP\_SLEEP states through the following registers and reuse the HP\_SLEEP configurations in LP\_SLEEP.

\begin{itemize}
    \iftagged{ESP32-P4-latest}{
        \item HP\_ACTIVE: \hyperref[fielddesc:PMUHPACTIVEPDHPCPUPDEN]{PMU\_HP\_ACTIVE\_PD\_HP\_CPU\_PD\_EN}
        \item HP\_SLEEP: \hyperref[fielddesc:PMUHPSLEEPPDHPCPUPDEN]{PMU\_HP\_SLEEP\_PD\_HP\_CPU\_PD\_EN}
    }{
        \item HP\_ACTIVE: \hyperref[fielddesc:PMUHPACTIVEPDTOPPDEN]{PMU\_HP\_ACTIVE\_PD\_TOP\_PD\_EN}
        \item HP\_SLEEP: \hyperref[fielddesc:PMUHPSLEEPPDTOPPDEN]{PMU\_HP\_SLEEP\_PD\_TOP\_PD\_EN}
    }
\end{itemize}

Similarly, users can configure the power up and down for other power domains in different PMU states. For the configuration registers, please refer to Section \ref{sec:lowpowm-reg-summ}.

\begin{tiplisting}
In the following text, all such configuration registers are collectively referred to as PMU\_\regindex{n1}\_PD\_\regindex{POWERDOMAIN}\_PD\_EN, where \regindex{n1} represents the three PMU states.
\end{tiplisting}

Once the configuration is done, the PMU uses various controllers to make these configurations effective, as described in the following sections.

\subsubsection{Analog Power Controller}\label{sec:lowpowm_analog_power_controller}

The analog power controller controls the power up and down of the analog circuits (including voltage regulators, high-speed clocks, and slow-speed clocks) in the PMU states.

The configuration of the regulators is as follows:

\begin{itemize}
    \item Configure PMU\_\regindex{n1}\_HP\_REGULATOR\_XPD or PMU\_\regindex{n1}\_LP\_REGULATOR\_XPD to enable or disable the output voltage of the HP/LP sys regulator in the target PMU state. Turning off the LP sys regulator is not recommended, as it may cause the PMU itself to power down, resulting in chip malfunction.
\end{itemize}

The configuration of the high-speed clocks (XTAL\_CLK and PLL clocks) is as follows:

\begin{itemize}
    \item XTAL\_CLK: Configure \hyperref[fielddesc:PMUHPSLEEPXPDXTAL]{PMU\_HP\_SLEEP\_XPD\_XTAL} to 1 to enable XTAL\_CLK when the chip switches the PMU state to HP\_SLEEP.

    Note: To avoid the instability in XTAL\_CLK during startup, users have the option to configure \hyperref[fielddesc:PMUWAITXTLSTABLE]{PMU\_WAIT\_XTL\_STABLE} to delay the gate opening for XTAL\_CLK. This delay ensures that the gate opening is enabled after \hyperref[fielddesc:PMUWAITXTLSTABLE]{PMU\_WAIT\_XTL\_STABLE} CLK\_DYN\_FAST\_CLK cycles following the power-up of XTAL\_CLK.

    \item CPLL\_CLK: PMU can enable CPLL\_CLK in different PMU states by configuring PMU\_\regindex{n1}\_XPD\_PLL[0] to 1. For example, configuring \hyperref[fielddesc:PMUHPACTIVEXPDPLL]{PMU\_HP\_ACTIVE\_XPD\_PLL}[0] to 1 enables the CPLL\_CLK clock when the chip is in HP\_ACTIVE state.

    \item SPLL\_CLK: PMU can enable SPLL\_CLK in different PMU states by configuring PMU\_\regindex{n1}\_XPD\_PLL[1] to 1. For example, configuring \hyperref[fielddesc:PMUHPACTIVEXPDPLL]{PMU\_HP\_ACTIVE\_XPD\_PLL}[1] to 1 enables the SPLL\_CLK clock when the chip is in HP\_ACTIVE state.

    \item Audio PLL: PMU can enable Audio PLL in different PMU states by configuring PMU\_\regindex{n1}\_XPD\_PLL[2] to 1. For example, configuring \hyperref[fielddesc:PMUHPACTIVEXPDPLL]{PMU\_HP\_ACTIVE\_XPD\_PLL}[2] to 1 enables the Audio PLL clock when the chip is in HP\_ACTIVE state.

    \item SDIO PLL: PMU can enable SDIO PLL in different PMU states by configuring PMU\_\regindex{n1}\_XPD\_PLL[3] to 1. For example, configuring \hyperref[fielddesc:PMUHPACTIVEXPDPLL]{PMU\_HP\_ACTIVE\_XPD\_PLL}[3] to 1 enables the SDIO PLL clock when the chip is in HP\_ACTIVE state.

    Note:
    \begin{itemize}
        \item Before enabling the PLL clocks please ensure that XTAL\_CLK is stable.

        \item To avoid the instability in the PLL clocks during startup, users have the option to configure \hyperref[fielddesc:PMUWAITPLLSTABLE]{PMU\_WAIT\_PLL\_STABLE} to delay the gate opening for the PLL clocks. This delay ensures that the gate opening is enabled after \hyperref[fielddesc:PMUWAITPLLSTABLE]{PMU\_WAIT\_PLL\_STABLE} CLK\_DYN\_FAST\_CLK cycles following the power-up of the PLL clocks.
    \end{itemize}
\end{itemize}

Slow-speed clocks (RC\_FAST\_CLK and XTAL32K\_CLK) operate with low power. The power up and down of the slow-speed clocks in HP\_ACTIVE and HP\_SLEEP states are controlled by \hyperref[fielddesc:PMUHPSLEEPXPDFOSCCLK]{PMU\_HP\_SLEEP\_XPD\_FOSC\_CLK}. In LP\_SLEEP state, the power up and down of the slow-speed clocks are controlled by \hyperref[fielddesc:PMULPSLEEPXPDFOSCCLK]{PMU\_LP\_SLEEP\_XPD\_FOSC\_CLK}.

\subsubsection{Digital Power Controller}

The digital power controller controls the power up and down of digital power domains in different PMU states. Unlike the analog power controller, the digital power controller does not directly control the regulator but instead controls the power switch connected to the regulator to power up and down the digital power domains.

The LP PD Peripherals domain can only be powered up and down while the PMU state switches between HP\_SLEEP and LP\_SLEEP. The other digital power domains can only be powered up and down while the PMU state switches between HP\_SLEEP and HP\_ACTIVE.

When the chip switches between PMU states, if the power configuration of a power domain in the current PMU state does not match the power configuration it is about to switch to, then the power up-down process will be activated. Take the power up-to-down process of the CPU power domain as an example. Configure \hyperref[fielddesc:PMUHPSLEEPPDTOPPDEN]{PMU\_HP\_SLEEP\_PD\_TOP\_PD\_EN} to 1 or 0 to power down or up the CPU power domain in HP\_SLEEP. PMU will perform the following configurations:

\begin{itemize}

\item Enable the digital isolation unit to ensure that the powered-down modules do not output unstable voltage levels to the powered-up modules. When a power domain loses power, the output of this module will be clamped to a fixed value.

\item Enable reset. When the CPU power domain loses power, its global reset signal is set to a reset state, which persists for a period after the CPU power domain is re-powered. This mechanism guarantees a reset-to-release process for the CPU power domain during power-up, effectively mitigating any instability caused by power up-down transitions.
\end{itemize}

The following explains the power up and down of each digital power domain:

\begin{itemize}
\item L2MEM\_G0–G5

Configure PMU\_\regindex{n1}\_PD\_HP\_MEM\_PD\_EN to power up or down L2MEM\_G0–G5.

\item HP\_CNNT

Configure PMU\_\regindex{n1}\_PD\_CNNT\_PD\_EN to power up or down HP\_CNNT.

\tagged{ESP32-P4-latest}{
    \item PD\_HP\_CPU

Configure PMU\_\regindex{n1}\_PD\_TOP\_PD\_EN to power or down PD\_TOP.

When the Peripherals domain is powered down, the following features are configurable:

    \begin{itemize}
    \item Powering down the Peripherals domain may cause instability in GPIOs. This can be addressed by maintaining the state of the GPIOs (excluding the LP GPIOs) through PMU. For example, configuring \hyperref[fielddesc:PMUHPSLEEPHPPADHOLDALL]{PMU\_HP\_SLEEP\_HP\_PAD\_HOLD\_ALL} can keep GPIOs in the same state as before Peripherals was powered down in HP\_SLEEP.

    \item In the HP\_ACTIVE state, Memory Deep-sleep mode is supported under a standard power supply. The memory cannot be read or written in this mode, but data can be retained. Configure PMU\_\regindex{n1}\_HP\_MEM\_DSLP to enable this mode.
    \end{itemize}

\item LP PD Peripherals

The LP PD Peripherals power domain remains powered up when the chip is in HP\_ACTIVE state. The power state of LP PD Peripherals is configurable only in HP\_SLEEP and LP\_SLEEP states. The LP PD Peripherals power domain has an independent digital power switch, and its power control is not dependent on the power state of other digital power domains.

\end{itemize}

\subsubsection{Clock Controller}

The clock controller mainly controls the high-performance system clocks and LP system clocks when the chip switches between PMU states.

The high-performance system clocks include ROOT\_CLK and high-performance system peripherals clocks. When the chip switches between HP\_ACTIVE and HP\_SLEEP states, PMU can switch, power up/down, and divide the frequency of ROOT\_CLK, as well as power up/down high-performance system peripherals clocks.

\begin{itemize}

\item ROOT\_CLK can be controlled as follows:

    \begin{itemize}
    \item Configure PMU\_\regindex{n1}\_SYS\_CLK\_SLP\_SEL to 1 to indicate that when the chip enters the corresponding PMU state, the clock source is controlled by PMU.

    \item Configure PMU\_\regindex{n1}\_ICG\_SYS\_CLOCK\_EN to 0 to disable ROOT\_CLK in the corresponding PMU state.

    \item Configure PMU\_\regindex{n1}\_DIG\_SYS\_CLK\_SEL to select the clock source after the chip enters the corresponding PMU state. For details, please refer to Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}} > Table \ref{tab:resclk-cpu-clk-src}.
    \end{itemize}

\item High-performance system peripheral clocks can be controlled as follows:

\subsubsection{Backup Controller}

\chipname{} has a Retention DMA module that can transfer data between memory and peripherals when the chip switches between PMU states. In this way, the data is backed up when the power domain is powered down and restored when the power domain is powered up again.

Data transfer is implemented in the Peripherals power domain. PMU only generates relevant control signals. It is important to note that the data transfer control registers are directional, as unlike other control registers, these control behaviors are determined by both the original PMU state and the target PMU state.

For example, the control registers for transitioning from HP\_ACTIVE to HP\_SLEEP and from HP\_SLEEP to HP\_ACTIVE are different. The possible PMU state switches are listed below, collectively represented by \regindex{n2} in the register names:

\begin{itemize}
\item HP\_SLEEP2ACTIVE
\item HP\_ACTIVE2SLEEP
\end{itemize}

The following section introduces how PMU controls the Retention DMA:

\begin{itemize}
\item Enable data transfer: Configure PMU\_\regindex{n2}\_BACKUP\_EN to 1 to enable data transfer when the corresponding PMU state switch is performed.

\item Enable corresponding clocks: Before data transfer starts, configure PMU\_\regindex{n2}\_BACKUP\_CLK\_SEL to select the clock source of the Retention DMA, and configure PMU\_\regindex{n1}\_BACKUP\_ICG\_FUNC\_EN to enable the clock.

After the data transfer is completed, the value of PMU\_\regindex{n1}\_BACKUP\_ICG\_FUNC\_EN is determined by the configuration of PMU\_\regindex{n1}\_DIG\_ICG\_FUNC\_EN in the target PMU state.

\item Configure data transfer direction: Configure the highest bit of PMU\_\regindex{n2}\_BACKUP\_MODE:
    \begin{itemize}
        \item 1: From peripheral to memory
        \item 0: From memory to peripheral
    \end{itemize}

\item Select linked list pointer: Configure the lower two bits of PMU\_\regindex{n2}\_BACKUP\_MODE to select the linked list pointer, specifically:
    \begin{itemize}
    \item 0: PAU\_LINK\_ADDR\_0
    \item 1: PAU\_LINK\_ADDR\_1
    \item 2: PAU\_LINK\_ADDR\_2
    \item 3: PAU\_LINK\_ADDR\_3
    \end{itemize}
\end{itemize}

\subsubsection{System Controller}

The system controller controls some functional modules when the chip switches PMU states, to achieve stable and low-power chip performance. Specifically, the system controller supports:

\begin{itemize}
\item \textbf{Pausing the watchdog function}: Configuring PMU\_\regindex{n1}\_DIG\_PAUSE\_WDT to 1 disables the RTC watchdog timer (RWDT) function when the chip switches to the corresponding target PMU state. Note that if this register is configured to 0 in any sleep mode, the watchdog function is not disabled, and RWDT will reset as the CPU does not feed the watchdog.

\item \textbf{Switching GPIO to sleep mode}, where the configuration of the GPIO holds. Consider a GPIO pin working in low-drive mode as an input and a wake-up pin. Setting PMU\_\regindex{n1}\_HP\_PAD\_HOLD\_ALL to 1 latches the current configuration of the GPIO pin as the sleep configuration when the chip transitions to the corresponding PMU state. For more information on the hold function of GPIO, please refer to Chapter \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}} > Section \ref{PadHold}.

\item \textbf{Disabling UART wake-up function}: Configuring PMU\_\regindex{n1}\_UART\_WAKEUP\_EN to 0 disables the four UART wake-up modes in the corresponding PMU state. For more information on UART wake-up modes, please refer to Chapter \ref{mod:uart} \textit{\nameref{mod:uart}}.

\item \textbf{Pausing CPU}: Setting PMU\_\regindex{n1}\_DIG\_CPU\_STALL to 1 suspends the CPU in the corresponding PMU state.
\end{itemize}

\subsubsection{Battery Power Controller}

As shown in Figure \ref{fig:lowpowm-power-structure}, the low-power system regulator can be powered by VDD\_BAT (battery) or VDD\_ANA (analog). \hyperref[fielddesc:PMUHPSLEEPVDDBATMODE]{PMU\_\regindex{n1}\_VDDBAT\_MODE} controls whether to enable the battery power. When \hyperref[fielddesc:PMUHPSLEEPVDDBATMODE]{PMU\_\regindex{n1}\_VDDBAT\_MODE} is 0, the battery power is not enabled, and the low-power system regulator is powered by VDD\_ANA and can be used in Active mode or Light-sleep0/1 modes. When \hyperref[fielddesc:PMUHPSLEEPVDDBATMODE]{PMU\_\regindex{n1}\_VDDBAT\_MODE} is 1, the battery power is enabled and can be used in Deep-sleep mode, where the main power, including VDD\_ANA, is cut off, and only the LP AON \& PMU power domains are powered up.

\subsubsection{Output Regulator Control}

As shown in Figure \ref{fig:lowpowm-power-structure}, \chipname{} has four output regulators:

\begin{itemize}
\item VDDO\_FLASH (VO1), typically used for on-chip flash voltage regulation
\item VDDO\_PSRAM (VO2), typically used for on-chip PSRAM voltage regulation
\item VDDO\_3 (VO3) and VDDO\_4 (VO4), which convert the input VDD\_LDO (typically 3.3 V) into the voltages needed by peripherals such as flash, PSRAM, MIPI, and SDIO.

    \end{itemize}

    \paragraph{Software Configuration Steps}

    The recommended software configuration steps (example) are as follows:

    \begin{enumerate}
    \item Configure output mode: set \texttt{PMU\_0PxA\_FORCE\_TIEH\_SEL\_y} so the register controls mode selection, then choose LDO or bypass mode via \texttt{PMU\_0PxA\_TIEH\_SEL\_y}/\texttt{PMU\_0PxA\_TIEH\_y} (see Table \ref{tab:VOX-registers}).
    \item If using LDO mode, configure output voltage: configure \texttt{PMU\_ANA\_0PxA\_DREF\_y} and \texttt{PMU\_ANA\_0PxA\_MUL\_y} to satisfy \(V_{OUT}=\mathrm{MUL}\times V_{REF}\).
    \item Configure waiting counter: set stage 1 (\texttt{TARGET0}) to the desired over-current protection duration and stage 2 (\texttt{TARGET1}) to the voltage stabilization waiting time (see Table \ref{tab:lowpowm-target}).
    \item Enable over-current protection before power-up: set \texttt{PMU\_ANA\_0PxA\_EN\_CUR\_LIM\_y}=1 (see Table \ref{tab:lowpowm-vo-lim-prot}).
    \item Enable the regulator: set \texttt{PMU\_0PxA\_XPD\_y}=1 (see Table \ref{tab:lowpowm-xpdctrl}), then wait for the stage 1 timeout interrupt (e.g., VO3 uses \texttt{PMU\_0P2A\_CNT\_TARGET0\_REACH\_0\_INT}, VO4 uses \texttt{PMU\_0P2A\_CNT\_TARGET0\_REACH\_1\_INT}).

    \end{enumerate}

\end{itemize}

\section{Event Task Matrix Feature}
\label{sec:lowpowm-etm-func}

The low-power management system on \chipname{} supports the Event Task Matrix (ETM) function, which allows the low-power management system's ETM tasks to be triggered by any peripherals' ETM events, or the low-power management system's ETM events to trigger any peripherals' ETM tasks. This section introduces the ETM tasks and events related to the low-power management system. For more information, please refer to Chapter \ref{mod:evntaskmatrix} \nameref{mod:evntaskmatrix}.

The low-power management system can receive the following ETM tasks:

\begin{itemize}
\item PMU\_TASK\_SLEEP\_REQ: Triggers PMU's sleep process.
\end{itemize}

The low-power management system can generate the following ETM events:

\begin{itemize}
\item PMU\_EVT\_SLEEP\_WAKEUP: Indicates that PMU is woken up to the HP\_ACTIVE state.
\end{itemize}

\section{Interrupts}

\chipname{}'s low-power management system can generate the following interrupt signals:

\begin{itemize}
\item PMU\_INTR
\item PMU\_LP\_INT
\end{itemize}

Among these interrupt signals, PMU\_INTR is sent to the \hyperref[mod:intmtrx]{Interrupt Matrix}, while PMU\_LP\_INT is sent to the LP CPU.

The interrupt signals are generated by the internal interrupt sources of each module, specifically:

PMU\_INTR:

\begin{itemize}
\item \label{int:PMUSOCWAKEUPINT}PMU\_SOC\_WAKEUP\_INT: Triggered when the chip is woken up to the HP\_ACTIVE state.
\item \label{int:PMUSOCSLEEPREJECTINT}PMU\_SOC\_SLEEP\_REJECT\_INT: Triggered when a sleep-rejection source rejects a sleep request.
\item \label{int:PMUSWINT}PMU\_SW\_INT: Triggered when LP CPU is used as a wake-up source and wakes up the chip to HP\_ACTIVE state.
\item \label{int:PMUSDIOIDLEINT}PMU\_SDIO\_IDLE\_INT: Triggered when SDIO is in idle state.
\item \label{int:PMULPCPUEXCINT}PMU\_LP\_CPU\_EXC\_INT: Triggered when there are LP CPU exceptions.
\item \label{int:PMU0P2ACNTTARGET1REACH1INT}PMU\_0P2A\_CNT\_TARGET1\_REACH\_1\_INT: Triggered when VO4 regulator's waiting counter target 1 reaches timeout.
\item \label{int:PMU0P2ACNTTARGET0REACH1INT}PMU\_0P2A\_CNT\_TARGET0\_REACH\_1\_INT: Triggered when VO4 regulator's waiting counter target 0 reaches timeout.
\item \label{int:PMU0P2ACNTTARGET1REACH0INT}PMU\_0P2A\_CNT\_TARGET1\_REACH\_0\_INT: Triggered when VO3 regulator's waiting counter target 1 reaches timeout.
\item \label{int:PMU0P2ACNTTARGET0REACH0INT}PMU\_0P2A\_CNT\_TARGET0\_REACH\_0\_INT: Triggered when VO3 regulator's waiting counter target 0 reaches timeout.
\item \label{int:PMU0P1ACNTTARGET1REACH1INT}PMU\_0P1A\_CNT\_TARGET1\_REACH\_1\_INT: Triggered when VO2 regulator's waiting counter target 1 reaches timeout.
\item \label{int:PMU0P1ACNTTARGET0REACH1INT}PMU\_0P1A\_CNT\_TARGET0\_REACH\_1\_INT: Triggered when VO2 regulator's waiting counter target 0 reaches timeout.
\item \label{int:PMU0P1ACNTTARGET1REACH0INT}PMU\_0P1A\_CNT\_TARGET1\_REACH\_0\_INT: Triggered when VO1 regulator's waiting counter target 1 reaches timeout.
\item \label{int:PMU0P1ACNTTARGET0REACH0INT}PMU\_0P1A\_CNT\_TARGET0\_REACH\_0\_INT: Triggered when VO1 regulator's waiting counter target 0 reaches timeout.
\end{itemize}

PMU\_LP\_INT:

\begin{itemize}
\item \label{int:PMULPCPUSLEEPREJECTINT}PMU\_LP\_CPU\_SLEEP\_REJECT\_INT: Triggered when the LP sleep request is rejected.

\item \label{int:PMUHPSWTRIGGERINT}PMU\_HP\_SW\_TRIGGER\_INT: Triggered when HP CPU wakes up LP CPU.

\item \label{int:PMUACTIVESWITCHSLEEPSTARTINT}PMU\_ACTIVE\_SWITCH\_SLEEP\_START\_INT: Triggered when the PMU state starts to switch from HP\_ACTIVE to HP\_SLEEP.

\item \label{int:PMUSLEEPSWITCHACTIVESTARTINT}PMU\_SLEEP\_SWITCH\_ACTIVE\_START\_INT: Triggered when the PMU state starts to switch from HP\_SLEEP to HP\_ACTIVE.

\item \label{int:PMUACTIVESWITCHSLEEPENDINT}PMU\_ACTIVE\_SWITCH\_SLEEP\_END\_INT: Triggered when the PMU state finishes switching from HP\_ACTIVE to HP\_SLEEP.

\item \label{int:PMUSLEEPSWITCHACTIVEENDINT}PMU\_SLEEP\_SWITCH\_ACTIVE\_END\_INT: Triggered when the PMU state finishes switching from HP\_SLEEP to HP\_ACTIVE.

\item \label{int:PMULPCPUWAKEUPINT}PMU\_LP\_CPU\_WAKEUP\_INT: Triggered when LP CPU is woken up.

\item \label{int:PMU0P2ACNTTARGET1REACH1LPINT}PMU\_0P2A\_CNT\_TARGET1\_REACH\_1\_LP\_INT: Triggered when VO4 regulator's waiting counter target 1 reaches timeout.
\item \label{int:PMU0P2ACNTTARGET0REACH1LPINT}PMU\_0P2A\_CNT\_TARGET0\_REACH\_1\_LP\_INT: Triggered when VO4 regulator's waiting counter target 0 reaches timeout.
\item \label{int:PMU0P2ACNTTARGET1REACH0LPINT}PMU\_0P2A\_CNT\_TARGET1\_REACH\_0\_LP\_INT: Triggered when VO3 regulator's waiting counter target 1 reaches timeout.
\item \label{int:PMU0P2ACNTTARGET0REACH0LPINT}PMU\_0P2A\_CNT\_TARGET0\_REACH\_0\_LP\_INT: Triggered when VO3 regulator's waiting counter target 0 reaches timeout.
\item \label{int:PMU0P1ACNTTARGET1REACH1LPINT}PMU\_0P1A\_CNT\_TARGET1\_REACH\_1\_LP\_INT: Triggered when VO2 regulator's waiting counter target 1 reaches timeout.
\item \label{int:PMU0P1ACNTTARGET0REACH1LPINT}PMU\_0P1A\_CNT\_TARGET0\_REACH\_1\_LP\_INT: Triggered when VO2 regulator's waiting counter target 0 reaches timeout.
\item \label{int:PMU0P1ACNTTARGET1REACH0LPINT}PMU\_0P1A\_CNT\_TARGET1\_REACH\_0\_LP\_INT: Triggered when VO1 regulator's waiting counter target 1 reaches timeout.
\item \label{int:PMU0P1ACNTTARGET0REACH0LPINT}PMU\_0P1A\_CNT\_TARGET0\_REACH\_0\_LP\_INT: Triggered when VO1 regulator's waiting counter target 0 reaches timeout.
\end{itemize}

Each interrupt source can be configured by a common set of registers that are described in Section \hyperref[interrupt-config-registers]{\textit{Interrupt Configuration Registers}}. The specific registers can be found in Section \ref{sec:lowpowm-reg-summ} \textit{\nameref{sec:lowpowm-reg-summ}}.
