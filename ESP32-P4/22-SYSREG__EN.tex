\chapter{System Registers (SYSREG)}
\label{mod:sysreg}
\hypertarget{sysreg}{}

\section{Overview}

\chipname{} includes various features that can be configured through dedicated registers. This chapter mainly enumerates such registers. For detailed introduction about a specific feature, please go to respective chapters.

\section{Function Description}

\subsection{HP System Registers}

\subsubsection{External Memory Encryption/Decryption Configuration}

Register \hyperref[regdesc:HPSYSTEMCRYPTOCTRLREG]{HP\_SYSTEM\_CRYPTO\_CTRL\_REG} configures encryption and decryption options of the external memory (external Flash and RAM). For details, please refer to Chapter \ref{mod:extmemencr} \textit{\nameref{mod:extmemencr}}.

\subsubsection{HP Peripherals Clock Configuration and Power Control}

The following registers are used to control the power signals of memory in corresponding peripherals.
\begin{itemize}
    \item RSA: \hyperref[regdesc:HPSYSTEMHPRSAPDCTRLREG]{HP\_SYSTEM\_HP\_RSA\_PD\_CTRL\_REG}
    \item ECC: \hyperref[regdesc:HPSYSTEMHPECCPDCTRLREG]{HP\_SYSTEM\_HP\_ECC\_PD\_CTRL\_REG}
    \item RSA: \hyperref[regdesc:HPSYSTEMHPUARTPDCTRLREG]{HP\_SYSTEM\_HP\_UART\_PD\_CTRL\_REG}
\end{itemize}

The following fields in register \hyperref[regdesc:HPSYSTEMHPPERIMEMCLKFORCEONREG]{HP\_SYSTEM\_HP\_PERI\_MEM\_CLK\_FORCE\_ON\_REG} control the clock gating signals of memory in corresponding peripherals.
\begin{itemize}
    \item GDMA: \hyperref[fielddesc:HPSYSTEMGDMAMEMCLKFORCEON]{HP\_SYSTEM\_GDMA\_MEM\_CLK\_FORCE\_ON}
    \item BitScrambler
    \begin{itemize}
        \item TX memory: \hyperref[fielddesc:HPSYSTEMBITSCRAMBLERTXMEMCLKFORCEON]{HP\_SYSTEM\_BITSCRAMBLER\_TX\_MEM\_CLK\_FORCE\_ON}
        \item RX memory: \hyperref[fielddesc:HPSYSTEMBITSCRAMBLERRXMEMCLKFORCEON]{HP\_SYSTEM\_BITSCRAMBLER\_RX\_MEM\_CLK\_FORCE\_ON}
    \end{itemize}
    \item RMT: \hyperref[fielddesc:HPSYSTEMRMTMEMCLKFORCEON]{HP\_SYSTEM\_RMT\_MEM\_CLK\_FORCE\_ON}
\end{itemize}

\subsubsection{HP Cache Clock and Reset Configuration}

\chipname{} allows software to directly control the clock and reset signals of HP cache.

The following fields in register \hyperref[regdesc:HPSYSTEMHPCACHECLKCONFIGREG]{HP\_SYSTEM\_HP\_CACHE\_CLK\_CONFIG\_REG} control the clock signals of different HP cache. Writing 1 to respective fields enables the clock for respective cache.
\begin{itemize}
    \item L1 ichache 0: \hyperref[fielddesc:HPSYSTEML1I0CACHECLKON]{HP\_SYSTEM\_L1\_I0\_CACHE\_CLK\_ON}
    \item L1 ichache 1: \hyperref[fielddesc:HPSYSTEML1I1CACHECLKON]{HP\_SYSTEM\_L1\_I1\_CACHE\_CLK\_ON}
    \item L1 dcache: \hyperref[fielddesc:HPSYSTEML1DCACHECLKON]{HP\_SYSTEM\_L1\_D\_CACHE\_CLK\_ON}
    \item L2 cache: \hyperref[fielddesc:HPSYSTEML2CACHECLKON]{HP\_SYSTEM\_L2\_CACHE\_CLK\_ON}
\end{itemize}

The following fields in register \hyperref[regdesc:HPSYSTEMCACHERESETCONFIGREG]{HP\_SYSTEM\_CACHE\_RESET\_CONFIG\_REG} control the reset signals of different HP cache. Writing 1 and then 0 to respective fields resets respective cache.
\begin{itemize}
    \item L1 ichache 0: \hyperref[fielddesc:HPSYSTEML1I0CACHERESET]{HP\_SYSTEM\_L1\_I0\_CACHE\_RESET}
    \item L1 ichache 1: \hyperref[fielddesc:HPSYSTEML1I1CACHERESET]{HP\_SYSTEM\_L1\_I1\_CACHE\_RESET}
    \item L1 dcache: \hyperref[fielddesc:HPSYSTEML1DCACHERESET]{HP\_SYSTEM\_L1\_D\_CACHE\_RESET}
\end{itemize}

\subsubsection{HP SPM and HP L2MEM Clock Configuration}

The following register fields control the clock signals of HP \hyperref[mod:sysmem]{SPM} (Scratchpad Memory) and HP \hyperref[mod:sysmem]{L2MEM}. Writing 1 to respective register fields forcibly enable the clock of respective memory.
\begin{itemize}
    \item HP SPM: \hyperref[fielddesc:HPSYSTEMHPSPMCLKFORCEON]{HP\_SYSTEM\_HP\_SPM\_CLK\_FORCE\_ON}
    \item L2MEM: \hyperref[fielddesc:HPSYSTEML2MEMCLKFORCEON]{HP\_SYSTEM\_L2\_MEM\_CLK\_FORCE\_ON}
    \item L2ROM: \hyperref[fielddesc:HPSYSTEML2ROMCLKFORCEON]{HP\_SYSTEM\_L2\_ROM\_CLK\_FORCE\_ON}
\end{itemize}


\subsubsection{HP SPM Parity Check Configuration}

\chipname{}'s has implemented Parity Check in HP SPM to detect one-bit memory flip error.

This feature can be configured and controlled via the following registers:
\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEMHPSPMPARITYCHECKCTRLREG]{HP\_SYSTEM\_HP\_SPM\_PARITY\_CHECK\_CTRL\_REG}: write 1 to enable the parity check in HP SPM.
    \item \hyperref[regdesc:HPSYSTEMHPSPMINITREG]{HP\_SYSTEM\_HP\_SPM\_INIT\_REG}: configures the initialization of HP SPM before enabling the parity check.
    \item Interrupt (\hyperref[tab:sysreg-int-sources]{HP\_SPM\_PARITY\_ERR\_INT}) related registers:
    \begin{itemize}
        \item \hyperref[regdesc:HPSYSTEMHPSPMINTRAWREG]{HP\_SYSTEM\_HP\_SPM\_INT\_RAW\_REG}: the raw interrupt status of \hyperref[tab:sysreg-int-sources]{HP\_SPM\_PARITY\_ERR\_INT}
        \item \hyperref[regdesc:HPSYSTEMHPSPMINTSTREG]{HP\_SYSTEM\_HP\_SPM\_INT\_ST\_REG}: the masked interrupt status of \hyperref[tab:sysreg-int-sources]{HP\_SPM\_PARITY\_ERR\_INT}
        \item \hyperref[regdesc:HPSYSTEMHPSPMINTENAREG]{HP\_SYSTEM\_HP\_SPM\_INT\_ENA\_REG}: write 1 to enable \hyperref[tab:sysreg-int-sources]{HP\_SPM\_PARITY\_ERR\_INT}
        \item \hyperref[regdesc:HPSYSTEMHPSPMINTCLRREG]{HP\_SYSTEM\_HP\_SPM\_INT\_CLR\_REG}: write 1 to clear \hyperref[tab:sysreg-int-sources]{HP\_SPM\_PARITY\_ERR\_INT}
    \end{itemize}
    \item \hyperref[regdesc:HPSYSTEMHPSPMPARITYINTRECORDREG]{HP\_SYSTEM\_HP\_SPM\_PARITY\_INT\_RECORD\_REG}: records the address when \hyperref[tab:sysreg-int-sources]{HP\_SPM\_PARITY\_ERR\_INT} occurs.
    \item \hyperref[regdesc:HPSYSTEMHPSPMERRRESPCTRLREG]{HP\_SYSTEM\_HP\_SPM\_ERR\_RESP\_CTRL\_REG}: write 1 so HP SPM reports parity error to the HP CPU and triggers exception.
\end{itemize}

For detailed configuration steps, please refer to Chapter \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}.


\subsubsection{HP L2MEM ECC Check Configuration}

\chipname{}'s has implemented the Error Correction Code (ECC) check in HP L2MEM (including L2RAM and cache) to correct one-bit memory flip error.

This feature can be configured and controlled via the following registers:
\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEML2MEML2RAMECCREG]{HP\_SYSTEM\_L2\_MEM\_L2\_RAM\_ECC\_REG} and \hyperref[regdesc:HPSYSTEML2MEML2CACHEECCREG]{HP\_SYSTEM\_L2\_MEM\_L2\_CACHE\_ECC\_REG}: write 1 to enable the ECC check for L2MEM and cache respectively.
    \item \hyperref[regdesc:HPSYSTEML2MEMREFRESHREG]{HP\_SYSTEM\_L2\_MEM\_REFRESH\_REG}: configures the initialization of HP L2MEM before enabling the ECC check.
    \item Interrupt (\hyperref[tab:sysreg-int-sources]{L2\_MEM\_ECC\_ERR\_INT}) related registers:
    \begin{itemize}
        \item \hyperref[regdesc:HPSYSTEML2MEMINTRAWREG]{HP\_SYSTEM\_L2\_MEM\_INT\_RAW\_REG}: the raw interrupt status of \hyperref[tab:sysreg-int-sources]{L2\_MEM\_ECC\_ERR\_INT}
        \item \hyperref[regdesc:HPSYSTEML2MEMINTSTREG]{HP\_SYSTEM\_L2\_MEM\_INT\_ST\_REG}: the masked interrupt status of \hyperref[tab:sysreg-int-sources]{L2\_MEM\_ECC\_ERR\_INT}
        \item \hyperref[regdesc:HPSYSTEML2MEMINTENAREG]{HP\_SYSTEM\_L2\_MEM\_INT\_ENA\_REG}: write 1 to enable \hyperref[tab:sysreg-int-sources]{L2\_MEM\_ECC\_ERR\_INT}
        \item \hyperref[regdesc:HPSYSTEML2MEMINTCLRREG]{HP\_SYSTEM\_L2\_MEM\_INT\_CLR\_REG}: write 1 to clear \hyperref[tab:sysreg-int-sources]{L2\_MEM\_ECC\_ERR\_INT}
    \end{itemize}
    \item \hyperref[regdesc:HPSYSTEML2MEMINTRECORD1REG]{HP\_SYSTEM\_L2\_MEM\_INT\_RECORD1\_REG}: records information when \hyperref[tab:sysreg-int-sources]{L2\_MEM\_ECC\_ERR\_INT} occurs.
    \item \hyperref[regdesc:HPSYSTEML2MEMERRRESPCTRLREG]{HP\_SYSTEM\_L2\_MEM\_ERR\_RESP\_CTRL\_REG}: write 1 so HP L2MEM reports ECC error to HP CPU and triggers exception.
\end{itemize}

For detailed configuration steps, please refer to Chapter \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}.

\subsubsection{BitScrambler Configuration}

\chipname{} has some peripherals that support DMA. These peripherals sometimes need BitScramblers to change the format of data before communicating with DMA.

Configure \hyperref[regdesc:HPSYSTEMBITSCRAMBLERPERISELREG]{HP\_SYSTEM\_BITSCRAMBLER\_PERI\_SEL\_REG} to attach such a peripheral to the RX and TX channels of bitscamblers respectively.

For detailed configuration steps, please refer to Chapter \ref{mod:bitscrm} \textit{\nameref{mod:bitscrm}}.

For a list of DMA-capable peripherals, please refer to Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}}.

\subsubsection{Ethernet MAC Control}

\chipname{} can send and receive data via Ethernet MAC (Media Access Controller) according to the IEEE 802.3 standard.

The control and configuration registers are:

\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEMGMACCTRL0REG]{HP\_SYSTEM\_GMAC\_CTRL0\_REG}
    \item \hyperref[regdesc:HPSYSTEMGMACCTRL1REG]{HP\_SYSTEM\_GMAC\_CTRL1\_REG}
    \item \hyperref[regdesc:HPSYSTEMGMACCTRL2REG]{HP\_SYSTEM\_GMAC\_CTRL2\_REG}
\end{itemize}

For details, please refer to Chapter \ref{mod:ethernetmac} \textit{\nameref{mod:ethernetmac}}.

\subsubsection{USB OTG 2.0 Control}

\chipname{} features a USB 2.0 High-Speed On-The-Go peripheral (OTG\_HS). To control it, use \hyperref[regdesc:HPSYSTEMUSBOTG20CTRLREG]{HP\_SYSTEM\_USBOTG20\_CTRL\_REG}.

For details, please refer to Chapter \ref{mod:usbotghs} \textit{\nameref{mod:usbotghs}}.

\subsubsection{CPU Control and Record}
\label{sec:sysreg-cpu-registers}

\chipname{} configures CPU debug features and stores its status information via the following system registers.

\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEMCPUCORESTALLEDSTREG]{HP\_SYSTEM\_CPU\_CORESTALLED\_ST\_REG}: records the core stalled status of CPU0 and CPU1.
    \item \hyperref[regdesc:HPSYSTEMCOREDEBUGRUNSTALLCONFREG]{HP\_SYSTEM\_CORE\_DEBUG\_RUNSTALL\_CONF\_REG}: write 1 to enable debug runstall feature between HP CPU and LP CPU.
    \item \hyperref[regdesc:HPSYSTEMHPCOREDMACTIVELPCOREREG]{HP\_SYSTEM\_HP\_CORE\_DMACTIVE\_LPCORE\_REG}: represents the dmactive value of LP CPU's debug module.
\end{itemize}

For details, please refer to Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}.

\subsubsection{HP GPIO Hold Control}

Each GPIO pin of \chipname{} has an individual hold function that is controlled by the following registers:

\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEMGPIOOHOLDCTRL0REG]{HP\_SYSTEM\_GPIO\_O\_HOLD\_CTRL0\_REG}: controls the hold signal of HP GPIO16 \verb+~+ HP GPIO47
    \item \hyperref[regdesc:HPSYSTEMGPIOOHOLDCTRL1REG]{HP\_SYSTEM\_GPIO\_O\_HOLD\_CTRL1\_REG}: controls the hold signal of GPIO48 \verb+~+ HP GPIO56
    \item \hyperref[regdesc:HPSYSTEMGPIODEDHOLDCTRLREG]{HP\_SYSTEM\_GPIO\_DED\_HOLD\_CTRL\_REG}: the lower 6 bits control the hold signal of flash pads, and the higher 20 bits control the hold signal of 16 line PSRAM pad
\end{itemize}

For details, please refer to Chapter \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}}.

\subsubsection{HP GPIO Output Control}

Each GPIO pin of \chipname{} can be configured to directly output the desired value via the following registers (hysteresis):

\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEMGPIOOHYSCTRL0REG]{HP\_SYSTEM\_GPIO\_O\_HYS\_CTRL0\_REG}: configures whether or not to enable the hysteresis feature of HP GPIO16 \verb+~+ HP GPIO47.
    \item \hyperref[regdesc:HPSYSTEMGPIOOHYSCTRL1REG]{HP\_SYSTEM\_GPIO\_O\_HYS\_CTRL1\_REG}: configures whether or not to enable the hysteresis feature of HP GPIO48 \verb+~+ HP GPIO56.
\end{itemize}

For details, please refer to Section \ref{PadHysEn} in Chapter \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}}.

\subsubsection{Illegal Access and Unauthorized Access}

\chipname{}'s buses return error response and record transfer address, transfer direction, and master id when illegal access and unauthorized access occur.

\begin{itemize}
    \item Illegal access: access to unmapped address space
    \item Unauthorized access: access to mapped address space without permission
\end{itemize}

Such information will be logged in the following registers:

\begin{itemize}
    \item HP CPU and other AHB masters (\hyperref[tab:sysreg-int-sources]{ICM\_CPU\_ADDRHOLE\_INT})
    \begin{itemize}
        \item \hyperref[regdesc:HPSYSTEMICMCPUADDRHOLEADDRREG]{HP\_SYSTEM\_ICM\_CPU\_ADDRHOLE\_ADDR\_REG}
        \item \hyperref[regdesc:HPSYSTEMICMCPUADDRHOLEINFOREG]{HP\_SYSTEM\_ICM\_CPU\_ADDRHOLE\_INFO\_REG}
    \end{itemize}
    \item AXI masters (\hyperref[tab:sysreg-int-sources]{ICM\_SYS\_ADDRHOLE\_INT} )
    \begin{itemize}
        \item \hyperref[regdesc:HPSYSTEMICMSYSADDRHOLEADDRREG]{HP\_SYSTEM\_ICM\_SYS\_ADDRHOLE\_ADDR\_REG}
        \item \hyperref[regdesc:HPSYSTEMICMSYSADDRHOLEINFOREG]{HP\_SYSTEM\_ICM\_SYS\_ADDRHOLE\_INFO\_REG}
\end{itemize}
\end{itemize}

For detailed information on address space mapping, please refer to Chapter \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}.

For detailed information on access permission management, please refer to Chapter \ref{mod:permctrl} \textit{\nameref{mod:permctrl}}.


\subsubsection{HP CPU Bus Timeout Protection}

\chipname{} supports Timeout Protection on HP CPU's IBUS, DBUS and AHB bus.

When a transfer is initiated, the counter inside the Timeout Protection module increments by one on each clock cycle.

\begin{itemize}
    \item If the accumulated value remains below the timeout threshold and the slave device responds to the transfer, the internal counter is cleared.
    \item If the accumulated value exceeds the timeout threshold and the slave device has not responded, the Timeout Protection module takes control of the bus to return an error response to the HP CPU. At the same time, an interrupt is generated.
\end{itemize}

Related control and configuration registers are described below.


\subparagraph{HP CPU IBUS Timeout Protection}

Configure \hyperref[regdesc:HPSYSTEMHPCOREIBUSTIMEOUTREG]{HP\_SYSTEM\_HP\_CORE\_IBUS\_TIMEOUT\_REG} to enable timeout protection and set the timeout threshold for HP CPU accessing HP SPM, L2MEM and EXT MEM by IBUS.

When a timeout occurs, the \hyperref[tab:sysreg-int-sources]{HP\_CORE\regindex{x}\_IBUS\_TIMEOUT\_INT} interrupt will be asserted.


\subparagraph{HP CPU DBUS Timeout Protection}

Configure \hyperref[regdesc:HPSYSTEMHPCOREDBUSTIMEOUTREG]{HP\_SYSTEM\_HP\_CORE\_DBUS\_TIMEOUT\_REG} to enable timeout protection and set the timeout threshold for HP CPU accessing HP SPM, L2MEM and EXT MEM by DBUS.

When a timeout occurs, the \hyperref[tab:sysreg-int-sources]{HP\_CORE\regindex{x}\_DBUS\_TIMOUT\_INT} interrupt will be asserted.

\subparagraph{HP CPU AHB BUS Timeout Protection}

Configure \hyperref[regdesc:HPSYSTEMHPCOREAHBTIMEOUTREG]{HP\_SYSTEM\_HP\_CORE\_AHB\_TIMEOUT\_REG} to enable timeout protection and set the timeout threshold for HP CPU accessing LP peripherals, HP peripherals registers by AHB bus.

When a timeout occurs, the \hyperref[tab:sysreg-int-sources]{HP\_CORE\regindex{x}\_AHB\_TIMEOUT\_INT} interrupt will be asserted.

\subsubsection{AXI Matrix}

\chipname{}'s AXI Matrix connects multiple AXI-compliant master and slave devices, enabling efficient communication.

\begin{itemize}
    \item Masters such as all AHB masters (master port1 through AHB2AXI bridge), HP CPU cache (master port2), DW GDMA\_0 (master port6), DW GDMA\_1 (master port7), AXI GDMA (master port9), 2DDMA (master port11), H264\_0 (master port12), and H264\_1 (master port13)
    \item Slaves such as L2MEM (slave port2), Flash (slave port3), PSRAM (slave port4), DSI Host (slave port6) and CSI Host(slave port7).
\end{itemize}

All AXI master ports except master port 2 in the AXI matrix support the QoS (Quality of Service) bandwidth regulator. The AXI matrix also includes a deadlock timeout protection mechanism, slave arbiter priority configuration for slave ports, and QoS arbiter priority configuration for master ports.


\subparagraph{Deadlock Timeout Protection}

\chipname{}'s Deadlock Timeout Protection is always enabled. Configure the timeout threshold for all AXI masters in the AXI matrix that access AXI Slaves such as Flash, PSRAM, DSI HOST, and CSI HOST via \hyperref[regdesc:HPSYSTEMICMDLOCKTIMEOUTREG]{HP\_SYSTEM\_ICM\_DLOCK\_TIMEOUT\_REG}.

When a timeout occurs, AXI Matrix will record transfer direction, AXI ID, master ID and slave ID in \hyperref[regdesc:HPSYSTEMICMDLOCKSTATUSREG]{HP\_SYSTEM\_ICM\_DLOCK\_STATUS\_REG}, and assert \hyperref[tab:sysreg-int-sources]{ICM\_DLOCK\_INTR} interrupt at the same time.


\subparagraph{Slave Arbiter Priority}

Configure the arbitration priority for slaves in the AXI Matrix to arbitrate reading data and responses via \hyperref[regdesc:HPSYSTEMICMSLVARBPRIORITYREG]{HP\_SYSTEM\_ICM\_SLV\_ARB\_PRIORITY\_REG}.

The higher the value, the higher the priority.

\subparagraph{Master QoS Arbiter Priority}

Configure the QoS (Quality of Service) values for masters in the AXI matrix to arbitrate the read address channel and write address channel via \hyperref[regdesc:HPSYSTEMICMMSTARQOSREG0REG]{HP\_SYSTEM\_ICM\_MST\_ARQOS\_REG0\_REG} and \hyperref[regdesc:HPSYSTEMICMMSTAWQOSREG0REG]{HP\_SYSTEM\_ICM\_MST\_AWQOS\_REG0\_REG}, respectively.

A higher QoS value indicates a higher priority during arbitration.

\subsubsection{Post Write}

The following registers controls the Post Write mode of different components to speed up the write access.
\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEMPERI1APBPOSTWENREG]{HP\_SYSTEM\_PERI1\_APB\_POSTW\_EN\_REG}: enables the Post Write mode for APB2APB bridge between peripherals.
    \item \hyperref[regdesc:HPSYSTEMAPBSYNCPOSTWENREG]{HP\_SYSTEM\_APB\_SYNC\_POSTW\_EN\_REG}: enables the Post Write mode for APB2APB bridge in DSI, CSI and ISP.
    \item \hyperref[fielddesc:HPSYSTEMCPUICMH2XPOSTWREN]{HP\_SYSTEM\_CPU\_ICM\_H2X\_POST\_WR\_EN}: enables the Post Write mode for AHB2AXI bridge from AHB masters to AXI slaves.
\end{itemize}

\subsection{LP System Registers}

\subsubsection{LP Timer Stall Control}

The \hyperref[fielddesc:LPSYSTEMSYSTIMERSTALLSEL]{LP\_SYSTEM\_SYSTIMER\_STALL\_SEL} field allows selection of the systimer\_stall signal source for the LP timer, choosing between HP CPU0 and HP CPU1:

\begin{itemize}
    \item 0: from CPU0
    \item 1: from CPU1
\end{itemize}

\subsubsection{Focused Ion Beam (FIB) Control}

The \hyperref[fielddesc:LPSYSTEMLPFIBSEL]{LP\_SYSTEM\_LP\_FIB\_SEL} field allows selection of the control source for the brownout detector and the super watchdog, choosing between the analog system or software register.

\begin{itemize}
    \item Bit-1: selects the control source for the brownout detector.
    \begin{itemize}
        \item 0: controlled by software register
        \item 1: controlled by the analog system.
    \end{itemize}
    \item Bit-2: selects the control source for the super watchdog.
        \begin{itemize}
            \item 0: controlled by software register
            \item 1: controlled by the analog system.
        \end{itemize}
\end{itemize}

Configure \hyperref[fielddesc:LPSYSTEMDIGFIB]{LP\_SYSTEM\_DIG\_FIB} to enable the brownout detector or the super watchdog via software register.
\begin{itemize}
    \item Bit-1: write 1 to enable the brownout detector.
    \item Bit-2: write 1 to enable the super watchdog.
\end{itemize}

Read \hyperref[fielddesc:LPSYSTEMANAFIB]{LP\_SYSTEM\_ANA\_FIB} to check the FIB state of brownout detector or the super watchdog via analog system (FIB state).
\begin{itemize}
    \item Bit-1: brownout detector
    \item Bit-2: super watchdog
\end{itemize}

\subsubsection{Boot Mode}

Configure \hyperref[fielddesc:LPSYSTEMFORCEDOWNLOADBOOT]{LP\_SYSTEM\_FORCE\_DOWNLOAD\_BOOT} to forcibly switch the chip from SPI Boot mode to Joint Download Boot mode.

\subsubsection{LP CPU Control}

Configure the LP CPU via \hyperref[regdesc:LPSYSTEMSYSCTRLREG]{LP\_SYSTEM\_SYS\_CTRL\_REG}. In particular:

\begin{itemize}
    \item \hyperref[fielddesc:LPSYSTEMLPCPUBOOTADDR]{LP\_SYSTEM\_LP\_CPU\_BOOT\_ADDR}: configures the boot address of the LP CPU.
    \item \hyperref[regdesc:LPSYSTEMLPCPUEXCPCREG]{LP\_SYSTEM\_LP\_CPU\_EXC\_PC\_REG}: represents the exception PC address of the LP CPU.
    \item \hyperref[regdesc:LPSYSTEMLPCPUDBGPCREG]{LP\_SYSTEM\_LP\_CPU\_DBG\_PC\_REG}: represents the current PC address of the LP CPU.
    \item \hyperref[fielddesc:LPSYSTEMLPCOREERRRESPDIS]{LP\_SYSTEM\_LP\_CORE\_ERR\_RESP\_DIS}: configures whether or not to disable error response for respective LP CPU buses.
        \begin{itemize}
            \item Bit-0: IBUS
            \item Bit-1: DBUS
            \item Bit-2: AHB bus
        \end{itemize}
    \item \hyperref[fielddesc:LPSYSTEMLPCOREETMWAKEUPFLAG]{LP\_SYSTEM\_LP\_CORE\_ETM\_WAKEUP\_FLAG}: indicates the status of an ETM task triggered by a wakeup event of the LP CPU.
    \item \hyperref[fielddesc:LPSYSTEMLPCOREETMWAKEUPFLAGCLR]{LP\_SYSTEM\_LP\_CORE\_ETM\_WAKEUP\_FLAG\_CLR}: write 1 to clear \hyperref[fielddesc:LPSYSTEMLPCOREETMWAKEUPFLAG]{LP\_SYSTEM\_LP\_CORE\_ETM\_WAKEUP\_FLAG}.
\end{itemize}


\subsubsection{Reset Control}

\chipname{} supports for types of resets that occur at different levels. Such reset can be controlled by some system registers introduced in the following sections.

For a full list of all the reset types and reset sources, please refer to Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}}.

\subparagraph{Analog and System Reset Source Bypass Control}

\hyperref[fielddesc:LPSYSTEMANARSTBYPASS]{LP\_SYSTEM\_ANA\_RST\_BYPASS}: configures whether to bypass the following reset sources:
\begin{itemize}
    \item Brown-out detector
    \item Super watchdog
    \item Power glitch detector
\end{itemize}

\hyperref[fielddesc:LPSYSTEMSYSRSTBYPASS]{LP\_SYSTEM\_SYS\_RST\_BYPASS}: configures whether to bypass the following reset sources:
\begin{itemize}
    \item Software
    \item HP Watchdog timer
    \item LP Watchdog timer
    \item eFuse
\end{itemize}

\subparagraph{Power-On Reset Bypass Control} \label{sec:power-on-by-pass}

\hyperref[fielddesc:LPSYSTEMHPPORSTNBYPASSCTRL]{LP\_SYSTEM\_HP\_PO\_RSTN\_BYPASS\_CTRL}: configures whether or not to bypass the following reset sources for the HP power domain during the Power-On reset.
\begin{itemize}
    \item Bit-0: eFuse
    \item Bit-1: LP watchdog timer
    \item Bit-2: USB (JTAG)
    \item Bit-3: USB (UART)
    \item Bit-4: Unused
    \item Bit-5: Unused
    \item Bit-6: HP watchdog timer
    \item Bit-7: Software
\end{itemize}

\hyperref[fielddesc:LPSYSTEMHPPOCNNTRSTNBYPASSCTRL]{LP\_SYSTEM\_HP\_PO\_CNNT\_RSTN\_BYPASS\_CTRL}: configures whether or not to bypass the following reset sources for the HP CNNT power domain during the Power-On reset.
\begin{itemize}
    \item Bit-0: eFuse
    \item Bit-1: LP watchdog timer
    \item Bit-2: USB (JTAG)
    \item Bit-3: USB (UART)
    \item Bit-4: Unused
    \item Bit-5: Unused
    \item Bit-6: HP watchdog timer
    \item Bit-7: Software
\end{itemize}

For details about \chipname{}'s power domains, please refer to Chapter \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}.

\subparagraph{Software System Reset}

Configure \hyperref[fielddesc:LPSYSTEMSYSSWRST]{LP\_SYSTEM\_SYS\_SW\_RST} to trigger a software system reset.

\subparagraph{IOMUX Reset}

Configure \hyperref[fielddesc:LPSYSTEMIOMUXRESETDISABLE]{LP\_SYSTEM\_IO\_MUX\_RESET\_DISABLE} to disable the HP IOMUX reset.

\subsubsection{Wakeup Control}

These registers control the behavior of some PMU wakeup sources.

\hyperref[fielddesc:LPSYSTEMUSBOTG20INSUSPEND]{LP\_SYSTEM\_USBOTG20\_IN\_SUSPEND}: configures whether or not to enable wakeup signals from USB OTG2.0.

\hyperref[fielddesc:LPSYSTEMUSBOTG20WAKEUPCLR]{LP\_SYSTEM\_USBOTG20\_WAKEUP\_CLR}: write 1 to clear wakeup signals from USB OTG2.0 to PMU.

\subsubsection{LPROM and LP SPM Clock Control}

\hyperref[fielddesc:LPSYSTEMLPSPMROMCLKFORCEON]{LP\_SYSTEM\_LP\_SPM\_ROM\_CLK\_FORCE\_ON}: set to force on LP SPM\_ROM clock if LP CPU clock is on.

\hyperref[fielddesc:LPSYSTEMLPSPMRAMCLKFORCEON]{LP\_SYSTEM\_LP\_SPM\_RAM\_CLK\_FORCE\_ON}: set to force on LP SPM\_RAM clock.

\subsubsection{Analog Voltage Comparator}

Analog voltage comparator\regindex{x} (\regindex{x} = 0, 1) are configured by the following register fields:
\begin{itemize}
    \item \hyperref[fielddesc:LPSYSTEMXPDCOMP0]{LP\_SYSTEM\_XPD\_COMP\regindex{x}} (\regindex{x} = 0, 1): configures whether or not to enable analog voltage comparator\regindex{x}.
    \item \hyperref[fielddesc:LPSYSTEMMODECOMP0]{LP\_SYSTEM\_MODE\_COMP\regindex{x}} (\regindex{x} = 0, 1): configures the comparison mode:
    \begin{itemize}
        \item 0: comparing the main voltage with the external reference voltage
        \item 1: comparing the main voltage with the internal reference voltage\\
    \end{itemize}
    \item \hyperref[fielddesc:LPSYSTEMDREFCOMP0]{LP\_SYSTEM\_DREF\_COMP\regindex{x}} (\regindex{x} = 0, 1): configures the internal reference voltage.
    \begin{itemize}
        \item The adjustable voltage range is 0 to (0.7 * VDDPST\_6) V.
        \item The adjystable step is 0.1 * VDDPST\_6 V.
    \end{itemize}
\end{itemize}

For details, please refer to Chapter \ref{mod:anacom} \textit{\nameref{mod:anacom}}.

\subsubsection{Illegal Access and Unauthorized Access}

\chipname{}'s buses return error response and record transfer address, transfer direction, and master id when illegal access and unauthorized access occur.

\begin{itemize}
    \item Illegal access: access to unmapped address space
    \item Unauthorized access: access to mapped address space without permission
\end{itemize}

Such information will be logged in the following registers:

\begin{itemize}
    \item For LP CPU's access by IBUS and DBUS (\hyperref[tab:sysreg-int-sources]{IDBUS\_ADDRHOLE\_INTR})
        \begin{itemize}
            \item \hyperref[regdesc:LPSYSTEMIDBUSADDRHOLEADDRREG]{LP\_SYSTEM\_IDBUS\_ADDRHOLE\_ADDR\_REG}\\
            \item \hyperref[regdesc:LPSYSTEMIDBUSADDRHOLEINFOREG]{LP\_SYSTEM\_IDBUS\_ADDRHOLE\_INFO\_REG}\\
        \end{itemize}
    \item For LP CPU's access by AHB or an access from HP CPU (\hyperref[tab:sysreg-int-sources]{LP\_ADDRHOLE\_INTR}):
        \begin{itemize}
            \item \hyperref[regdesc:LPSYSTEMLPADDRHOLEADDRREG]{LP\_SYSTEM\_LP\_ADDRHOLE\_ADDR\_REG}\\
            \item \hyperref[regdesc:LPSYSTEMLPADDRHOLEINFOREG]{LP\_SYSTEM\_LP\_ADDRHOLE\_INFO\_REG}\\
        \end{itemize}
\end{itemize}

For detailed information on address space mapping, please refer to Chapter \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}.

For detailed information on access permission management, please refer to Chapter \ref{mod:permctrl} \textit{\nameref{mod:permctrl}}.


\subsubsection{LP Bus Timeout Protection}

\chipname{} supports Timeout Protection on LP CPU's IBUS, DBUS and AHB bus.

When a transfer is initiated, the counter inside the Timeout Protection module increments by one on each clock cycle.

\begin{itemize}
    \item If the accumulated value remains below the timeout threshold and the slave device responds to the transfer, the internal counter is cleared.
    \item If the accumulated value exceeds the timeout threshold and the slave device has not responded, the Timeout Protection module takes control of the bus to return an error response to the LP CPU. At the same time, an interrupt is generated.
\end{itemize}

Related control and configuration registers are described below.


\subparagraph{LP CPU IBUS Timeout Protection}

Configure \hyperref[regdesc:LPSYSTEMLPCOREIBUSTIMEOUTREG]{LP\_SYSTEM\_LP\_CORE\_IBUS\_TIMEOUT\_REG} to enable timeout protection and set the timeout threshold for LP CPU accessing LPROM and LP SPM by IBUS.
When a timeout occurs, the \hyperref[tab:sysreg-int-sources]{LP\_CORE\_IBUS\_TIMEOUT\_INTR} interrupt will be asserted.

\subparagraph{LP CPU DBUS Timeout Protection}

Configure \hyperref[regdesc:LPSYSTEMLPCOREDBUSTIMEOUTREG]{LP\_SYSTEM\_LP\_CORE\_DBUS\_TIMEOUT\_REG} to enable timeout protection and set the timeout threshold for LP CPU accessing LPROM and LP SPM by DBUS.

When a timeout occurs, the \hyperref[tab:sysreg-int-sources]{LP\_CORE\_DBUS\_TIMEOUT\_INTR} interrupt will be asserted.

\subparagraph{LP CPU AHB Bus Timeout Protection}

Configure \hyperref[regdesc:LPSYSTEMLPCOREAHBTIMEOUTREG]{LP\_SYSTEM\_LP\_CORE\_AHB\_TIMEOUT\_REG} to enable timeout protection and set the timeout threshold for LP CPU accessing LP peripherals, HP peripherals and CPU peripherals by AHB bus.

When a timeout occurs, the \hyperref[tab:sysreg-int-sources]{LP\_CORE\_AHB\_TIMEOUT\_INTR} interrupt will be asserted.


\subsubsection{RNG Control}

Registers \hyperref[regdesc:LPSYSTEMRNGDATAREG]{LP\_SYSTEM\_RNG\_DATA\_REG} and \hyperref[regdesc:LPSYSTEMRNGCFGREG]{LP\_SYSTEM\_RNG\_CFG\_REG} are used to configure and control the random number generator.
