\chapter{RISC-V 追踪编码器 (TRACE)}
\label{mod:riscvtracenc}


\chipname{} 的 高性能 CPU (HP CPU) 通过追踪编码器 (Trace Encoder) 实现了对指令追踪接口的支持。追踪编码器连接 CPU 的指令追踪接口，将信息压缩成更小的数据包，存入内部 SRAM（详见章节 \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}）中。


\begin{figure}[H]
        \centering
        \includegraphics[width=0.8\textwidth]{\modulefiles/figures/trace.pdf}
        \caption{追踪编码器概述图}
        \label{fig:encoder}
\end{figure}

\section{术语}

为了更好地说明 RISC-V 追踪编码器的功能，本章使用了以下术语。

\begin{longtable}[c]{ R{5cm} p{10cm} }
\textbf{hart} &  RISC-V 硬件线程 \\
\textbf{分支 (branch)} & 在一定条件下改变执行流程的指令 \\
\textbf{无法推测的不连续地址 (uninferable discontinuity, or updiscon)} & 无法仅通过程序二进制文件推测出的程序计数器地址偏移\\
\textbf{增量 (delta)} & 指令在存储器中的存储地址不连续时，程序计数器的地址偏移\\
\textbf{陷阱 (trap)} & 异常或中断，会触发陷阱处理程序 \\
\textbf{筛选 (qualification)} & 指令符合筛选条件，即称为通过筛选，才会被追踪 \\
\textbf{te\_inst} & 编码器发送的数据包的名称 \\
\textbf{退役 (retire)} & 机器状态更新时，执行指令的最后一个阶段 \\
\textbf{EPC} & 异常程序计数器 \\

\end{longtable}

\section{介绍}

对于复杂的系统来说，了解程序执行流程并非易事。造成软件不按预期工作的可能有很多种因素，比如与其他核、外设、实时事件的交互，执行不畅，或以上所有因素的结合。

调试器很难实时监测系统运行中的程序执行流程，因为调试器是侵入性的，会破坏系统的运行状态。但是，监测程序执行又很重要。

此时，可使用指令追踪来追踪程序执行。


\begin{figure}[H]
        \centering
        \includegraphics[width=1\textwidth]{\modulefiles/figures/trace_over.pdf}
        \caption{指令追踪流程概览}
        \label{fig:trace}
\end{figure}

图 \ref{fig:trace}  为指令增量追踪的流程。每个 HP 核都有一个单独的编码器，可以独立工作，追踪流程相同：
\begin{itemize}
    \item HP CPU 核有一个指令追踪接口，可以输出 HP CPU 执行指令的信息，包括指令地址、指令类型等。更多关于 \chipname{} HP CPU 指令追踪接口的信息，可参考章节 \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}。
    \item 追踪编码器通过 HP CPU 的指令追踪接口获取相关指令追踪信息，将信息压缩成带宽更小的数据包，存入系统存储器中。
    \item 调试器（即图中的 Debug Host）可以将系统存储器中的追踪数据包通过 JTAG 或 USB Serial/JTAG 导出，然后用解码器解压缩、重建程序的执行流。追踪解码器通常是外部 PC 上的软件，在获取追踪数据包后，根据在 hart 上运行的程序二进制数据，重建程序的指令流。解码可以离线进行，也可以在 hart 执行时实时进行。
\end{itemize}

\begin{itemize}
    \item 兼容 Efficient Trace for RISC-V Version 2.0（RISC-V 高效追踪规范 v2.0），实现的参数详见表 \ref{tab:encoder_parameters}

            \item 包的大小超过配置的存储器空间时触发中断

\section{架构概览}
\label{sec:riscvtracenc-arch-overview}


本章节主要介绍 \chipname{} 追踪编码器的具体实现。

如图 \ref{fig:encoder} 所示，追踪编码器包含一个编码器、一个 FIFO、一个寄存器配置模块和一个发送控制模块。

编码器通过指令追踪接口接收 HP CPU 的指令信息，将指令信息压缩成不同的数据包，写入内部 FIFO。

发送控制模块通过 AHB 总线将 FIFO 中的数据写入内部 SRAM。

FIFO 的深度为 128，宽度为 8 位。存储器写入带宽不足时，FIFO 可能会溢出，发生丢包。如果发生丢包，编码器会发送一个包通知丢包情况，在 FIFO 变空前一直停止工作，如果中断已使能还会触发中断。

\section{功能描述}
\label{sec:riscvtracenc-func-desc}

\subsection{同步}

为保障追踪的有效性，追踪过程中必须定期同步。同步可以通过发送完整的指令地址实现。同步计数器的值达到 \hyperref[regdesc:TRACERESYNCPROLONGEDREG]{TRACE\_RESYNC\_PROLONGED\_REG} 寄存器的 \hyperref[fielddesc:TRACERESYNCPROLONGED]{TRACE\_RESYNC\_PROLONGED} 字段的值时，编码器会发送一个同步数据包（格式 3 子格式 0，见章节 \ref{sec:riscvtracenc-format-3-sub-0}）。

追踪编码器支持两种同步模式，可通过 \hyperref[fielddesc:TRACERESYNCMODE]{TRACE\_RESYNC\_MODE} 配置：
\begin{itemize}
    \item 0：关闭同步计数器
    \item 1：无效值，没有作用
    \item 2：同步计数器按数据包计数
    \item 3：同步计数器按周期计数
\end{itemize}

可通过增加 \hyperref[fielddesc:TRACERESYNCPROLONGED]{TRACE\_RESYNC\_PROLONGED} 的值，降低发送同步数据包的频率，从而降低同步数据包对带宽的占用。

\subsection{地址模式}

\chipname{} 支持两种地址模式：增量地址模式和完整地址模式。
\begin{itemize}
    \item 增量地址模式（默认）：在 增量地址模式下，地址会编码为当前指令的实际地址与上一个包含地址的数据包报告的指令实际地址的差值。这种差值编码所需的位比完整地址少，可以令追踪压缩更高效。
    \item 完整地址模式：在完整地址模式下，追踪的所有地址会编码为绝对地址，而不是差值。这种编码方式相对低效，但是可以辅助软件解码器开发者进行调试。该模式可以通过置位 \hyperref[fielddesc:TRACEFULLADDRESS]{TRACE\_FULL\_ADDRESS} 使能。
\end{itemize}

\subsection{过滤}\label{sec:riscvtracenc-filter}

过滤机制控制编码器追踪的标准。比如，过滤机制可以让编码器在出现下列情况时追踪：
\begin{itemize}
    \item 指令地址在特定范围内时追踪
    \item 从一个指令地址开始追踪，到出现第二个指令地址时结束追踪
    \item 针对一个或多个特定特权等级追踪
    \item 出现异常及/或特定异常原因出发中断处理程序时追踪
\end{itemize}

每个过滤器单元包含一个过滤器和一个比较器单元。

每个比较器单元实际包含一对比较器（主比较器和次比较器，简称 P、S），可根据要求设置与信号单元匹配的范围，并支持：

\begin{itemize}
    \item 从 iaddress 和 tval 中选择输入
    \item 每个比较器可以独立选择运算方式（$<$、$>$、$=$、$!=$ 等）
    \item 次匹配值可以是主比较器的屏蔽值
    \item 两个比较器有多种一起使用的方式：$P$、$P\&\&S$、$!(P\&\&S)$、锁存（P 匹配时置 1，S 匹配时清零）
    \item 每个比较器可以报告特定指令地址
\end{itemize}

每个过滤器可以指定要过滤的 hart 指令，并具有以下功能：
\begin{itemize}
    \item 选择比较器单元进行匹配
    \item 可以选择多个特权等级和异常原因作为输入进行匹配
    \item 选择匹配中断
\end{itemize}

\subsection{锚定}

由于数据包的长度不定，为在写入存储器时分清数据包之间的边界，\chipname{} 编码器在数据包之间插入 0：
\begin{itemize}
    \item 数据包的长度最大为 13 字节，因此正常数据包中不能出现连续 14 个值为 0 的字节。连续 14 个及以上值为 0 的字节后，第一个不为 0 的字节肯定是数据包的第一个字节。
    \item 每发送 128 个数据包，编码器会向存储器写入 14 个值为 0 的字节作为锚定符。
\end{itemize}


\subsection{写存储器模式}

写追踪存储器时，追踪数据包的大小可能会超过追踪存储器的容量。此时可配置写存储器的模式，选择是否循环写入数据包：

\begin{itemize}
    \item 循环模式：追踪数据包的大小超过追踪存储器的容量（即 \hyperref[regdesc:TRACEMEMCURRENTADDRREG]{TRACE\_MEM\_CURRENT\_ADDR\_REG} 的值达到 \hyperref[regdesc:TRACEMEMENDADDRREG]{TRACE\_MEM\_END\_ADDR\_REG} 的值）时，存储器循环写入数据，回到存储器的起始地址 \hyperref[regdesc:TRACEMEMSTARTADDRREG]{TRACE\_MEM\_START\_ADDR\_REG}，此前写入的数据会被新数据覆盖。
    \item 非循环模式：追踪数据包的大小超过追踪存储器的容量时，存储器不循环，停在 \hyperref[regdesc:TRACEMEMENDADDRREG]{TRACE\_MEM\_END\_ADDR\_REG}，此前写入的数据保留。
\end{itemize}

\subsection{自动重启}

因 FIFO 溢出而丢包时，编码器将停止工作，需要软件手动恢复或硬件自动重启。如果 \hyperref[regdesc:TRACETRIGGERREG]{TRACE\_TRIGGER\_REG} 寄存器的 \hyperref[fielddesc:TRACERESTARTENA]{TRACE\_RESTART\_ENA} 位为 1，FIFO 清空后编码器会自动重启。

如果使能自动重启功能，编码器无论出现什么情况都会自动重启。因此，要关闭编码器，必须先清除 \hyperref[regdesc:TRACETRIGGERREG]{TRACE\_TRIGGER\_REG} 寄存器的 \hyperref[fielddesc:TRACERESTARTENA]{TRACE\_RESTART\_ENA} 位从而关闭自动重启。

\subsection{配置编码器选项}

编码器具有一些可选的配置选项。

\begin{itemize}
    \item 完整地址模式
        \begin{itemize}
            \item 编码器默认使用 增量地址模式。可以置位 \hyperref[fielddesc:TRACEFULLADDRESS]{TRACE\_FULL\_ADDRESS} 以启用完整地址模式用来调试。
        \end{itemize}

        \item 丢包后重启
            \begin{itemize}
                \item 因 FIFO 溢出而丢失追踪数据包时，如果 \hyperref[fielddesc:TRACERESTARTENA]{TRACE\_RESTART\_ENA} 为 1，编码器将自动停止。FIFO 为空时，编码器将重新开始报告数据包。
            \end{itemize}

        \item FIFO 将满时暂停 CPU
            \begin{itemize}
                \item 如果 \hyperref[fielddesc:TRACESTALLENA]{TRACE\_STALL\_ENA} 为 1，FIFO 将满时会向 CPU 发出暂停请求，CPU 会一直暂停，直到请求失效。
            \end{itemize}

        \item CPU 暂停时停止追踪

            如果 \hyperref[fielddesc:TRACEHALTENA]{TRACE\_HALT\_ENA} 为 1，编码器会在 CPU 暂停时先输出一个数据包，报告暂停前最后一条指令的地址，然后再输出一个支持数据包指示追踪已停止。halted 信号失效时，编码器重新开始追踪，从同步数据包开始。

        \item CPU 复位时停止追踪

            如果 \hyperref[fielddesc:TRACERESETENA]{TRACE\_RESET\_ENA} 为 1，编码器会单独复位。编码器会在 CPU 复位时先输出一个数据包，报告复位前最后一条指令的地址，然后再输出一个支持数据包指示追踪已停止。CPU 退出复位状态时，编码器重新开始追踪，从同步数据包开始。

        \item 使用调试模块的触发输出

            RISC-V hart 的调试模块有一个触发单元，其匹配控制寄存器（\hyperref[regdesc:MCONTROL]{mcontrol}）有一个 4 位的 \hyperref[fielddesc:MCONTROLACTION]{action} 字段，该字段值为 2 \verb+~+ 4 时用于追踪。\chipname{} 已实现此功能。\hyperref[fielddesc:MCONTROLACTION]{action} 字段的值详见表 \ref{tab:debug_mcontrol}。

\subsection{配置过滤器}

章节 \ref{sec:riscvtracenc-filter} 中描述的过滤单元可以按以下方式配置：

\begin{enumerate}
    \item 选择以下任一匹配模式：
        \begin{itemize}
            \item 置位 \hyperref[fielddesc:TRACEMATCHPRIVILEGE]{TRACE\_MATCH\_PRIVILEGE} 启用特权匹配模式：
                \begin{itemize}
                    \item 通过 \hyperref[fielddesc:TRACEMATCHCHOICEPRIVILEGE]{TRACE\_MATCH\_CHOICE\_PRIVILEGE} 配置特权级别
                \end{itemize}
            \item 置位 \hyperref[fielddesc:TRACEMATCHECAUSE]{TRACE\_MATCH\_ECAUSE} 启用 ecause 匹配模式：
                \begin{itemize}
                    \item 通过 \hyperref[fielddesc:TRACEMATCHCHOICEECAUSE]{TRACE\_MATCH\_CHOICE\_ECAUSE} 配置 ecause 值
                \end{itemize}
            \item 置位 \hyperref[fielddesc:TRACEMATCHINTERRUPT]{TRACE\_MATCH\_INTERRUPT} 启用中断匹配模式：
                \begin{itemize}
                    \item 通过 \hyperref[fielddesc:TRACEMATCHVALUEINTERRUPT]{TRACE\_MATCH\_VALUE\_INTERRUPT} 配置中断级别
                \end{itemize}
            \item 置位 \hyperref[fielddesc:TRACEMATCHCOMP]{TRACE\_MATCH\_COMP} 启用比较器匹配模式：
                \begin{itemize}
                    \item 配置主比较器：
                        \begin{itemize}
                            \item 通过 \hyperref[fielddesc:TRACEPINPUT]{TRACE\_P\_INPUT} 选择输入
                            \item 通过 \hyperref[fielddesc:TRACEPFUNCTION]{TRACE\_P\_FUNCTION} 选择比较器运算符
                            \item 通过 \hyperref[regdesc:TRACEFILTERPCOMPARATORMATCHREG]{TRACE\_FILTER\_P\_COMPARATOR\_MATCH\_REG} 配置匹配值
                        \end{itemize}
                    \item 必要时配置次比较器：
                        \begin{itemize}
                            \item 通过 \hyperref[fielddesc:TRACESINPUT]{TRACE\_S\_INPUT} 选择输入
                            \item 通过 \hyperref[fielddesc:TRACESFUNCTION]{TRACE\_S\_FUNCTION} 选择比较器运算符
                            \item 通过 \hyperref[regdesc:TRACEFILTERSCOMPARATORMATCHREG]{TRACE\_FILTER\_S\_COMPARATOR\_MATCH\_REG} 配置匹配值
                        \end{itemize}
                    \item 通过 \hyperref[fielddesc:TRACEMATCHMODE]{TRACE\_MATCH\_MODE} 选择用于匹配的比较器：
                        \begin{itemize}
                            \item 0：仅主比较器用于匹配，次比较器无效
                            \item 1：主比较器和次比较器都用于匹配 ($P\&\&S$)
                            \item 2：主比较器或次比较器都不用于匹配 $!(P\&\&S)$
                            \item 3：主比较器匹配时开始过滤，次比较器匹配时结束过滤
                        \end{itemize}
                \end{itemize}
                如果置位 \hyperref[fielddesc:TRACEPNOTIFY]{TRACE\_P\_NOTIFY} 或 \hyperref[fielddesc:TRACESNOTIFY]{TRACE\_S\_NOTIFY}，比较器会以通知的形式报告匹配地址，指令不会过滤。例如：
                \begin{itemize}
                    \item 如果 \hyperref[fielddesc:TRACEMATCHMODE]{TRACE\_MATCH\_MODE} 为 0，并且 \hyperref[fielddesc:TRACEPNOTIFY]{TRACE\_P\_NOTIFY} 为 1，则所有指令都会追踪，指令与主比较器匹配时会发出通知。
                    \item 如果 \hyperref[fielddesc:TRACEMATCHMODE]{TRACE\_MATCH\_MODE} 为 0，并且 \hyperref[fielddesc:TRACESNOTIFY]{TRACE\_S\_NOTIFY} 为 1，则指令将通过主比较器过滤，指令与次比较器匹配时会发出通知。
                \end{itemize}
        \end{itemize}
    \item 置位 \hyperref[fielddesc:TRACEFILTEREN]{TRACE\_FILTER\_EN} 以启用过滤单元。
\end{enumerate}

\subsection{使能编码器}


\begin{itemize}
    \item 通过 \hyperref[regdesc:TRACEMEMSTARTADDRREG]{TRACE\_MEM\_START\_ADDR\_REG} 和 \hyperref[regdesc:TRACEMEMENDADDRREG]{TRACE\_MEM\_END\_ADDR\_REG} 配置追踪存储器的地址空间。编码器可以访问 L2 MEM 和 PSRAM：
        \begin{itemize}
            \item L2MEM: 0x4FF00000 \verb+~+ 0x4FFBFFFF
            \item PSRAM: 0x48000000 \verb+~+ 0x4BFFFFFF
        \end{itemize}
    \item 置位 \hyperref[fielddesc:TRACEMEMCURRENTADDRUPDATE]{TRACE\_MEM\_CURRENT\_ADDR\_UPDATE}，将 \hyperref[regdesc:TRACEMEMCURRENTADDRREG]{TRACE\_MEM\_CURRENT\_ADDR\_REG} 的值更新为 \hyperref[regdesc:TRACEMEMSTARTADDRREG]{TRACE\_MEM\_START\_ADDR\_REG} 的值

    \item（可选）通过 \hyperref[regdesc:TRACETRIGGERREG]{TRACE\_TRIGGER\_REG} 寄存器的 \hyperref[fielddesc:TRACEMEMLOOP]{TRACE\_MEM\_LOOP} 位配置写存储器的模式
        \begin{itemize}
            \item 0：非循环模式
            \item 1：循环模式（默认）
        \end{itemize}


    \item 通过 \hyperref[regdesc:TRACERESYNCPROLONGEDREG]{TRACE\_RESYNC\_PROLONGED\_REG} 寄存器的 \hyperref[fielddesc:TRACERESYNCMODE]{TRACE\_RESYNC\_MODE} 位配置同步模式
        \begin{itemize}
            \item 0：关闭同步计数器（默认）
            \item 1：无效值，没有作用
            \item 2：同步计数器按数据包计数
            \item 3：同步计数器按周期计数
                \end{itemize}

    \item（可选）通过 \hyperref[regdesc:TRACERESYNCPROLONGEDREG]{TRACE\_RESYNC\_PROLONGED\_REG} 配置同步计数器的阈值，默认为 128

    \item（可选）使能中断
        \begin{itemize}
            \item 向 \hyperref[regdesc:TRACEINTRENAREG]{TRACE\_INTR\_ENA\_REG} 寄存器的相应位写 1 开启对应中断
            \item 向\hyperref[regdesc:TRACEINTRCLRREG]{TRACE\_INTR\_CLR\_REG} 寄存器的相应位写 1 清除对应中断
            \item 读 \hyperref[regdesc:TRACEINTRRAWREG]{TRACE\_INTR\_RAW\_REG} 寄存器的值，获取中断状态
        \end{itemize}

    \item（可选）向 \hyperref[regdesc:TRACETRIGGERREG]{TRACE\_TRIGGER\_REG} 寄存器的 \hyperref[fielddesc:TRACERESTARTENA]{TRACE\_RESTART\_ENA} 位写 1，使能自动重启。该功能默认开启

    \item 向 \hyperref[regdesc:TRACETRIGGERREG]{TRACE\_TRIGGER\_REG} 寄存器的 \hyperref[fielddesc:TRACETRIGGERON]{TRACE\_TRIGGER\_ON} 字段写 1，使能编码器

    \item 使能追踪编码器，有以下两种方式：
        \begin{itemize}
            \item 向 \hyperref[regdesc:TRACETRIGGERREG]{TRACE\_TRIGGER\_REG} 寄存器的 \hyperref[fielddesc:TRACETRIGGERON]{TRACE\_TRIGGER\_ON} 位写 1
            \item 通过调试模块的 \hyperref[fielddesc:MCONTROLACTION]{action} 字段使能编码器。触发器地址匹配且 \hyperref[fielddesc:MCONTROLACTION]{action} 字段为 2 时，编码器使能。触发器的配置，请参考 RISC-V Debug Specification。
    \end{itemize}
\end{itemize}

编码器使能后，会一直追踪 HP CPU 的指令接口，向追踪存储器写入数据包。

\subsection{关闭编码器}

停止输出追踪数据包，有以下两种方式：
\begin{enumerate}
    \item 向 \hyperref[regdesc:TRACETRIGGERREG]{TRACE\_TRIGGER\_REG} 寄存器的 \hyperref[fielddesc:TRACETRIGGEROFF]{TRACE\_TRIGGER\_OFF} 位写 1 关闭编码器
    \item 通过调试模块的 \hyperref[fielddesc:MCONTROLACTION]{action} 字段关闭编码器。触发器地址匹配且 \hyperref[fielddesc:MCONTROLACTION]{action} 字段为 3 时，编码器关闭。触发器的配置，请参考 RISC-V Debug Specification。
\end{enumerate}

由于内部 FIFO 的存在，数据包在追踪结束后不会立即写入存储器。建议查询\hyperref[regdesc:TRACEFIFOSTATUSREG]{TRACE\_FIFO\_STATUS\_REG} 确认数据是否全部写入存储器。

\subsection{通知}

如果一个目标地址很特殊，即使不是无法推测的不连续地址，用户也可能想要报告。

通知和报告特殊地址有以下两种方式：
\begin{enumerate}
    \item 调试模块的触发器地址匹配，并且 \hyperref[fielddesc:MCONTROLACTION]{action} 为 4
    \item 过滤器比较器匹配
\end{enumerate}

收到通知请求时，编码器会发出格式 2 或格式 1 的数据包报告地址，具体取决于 branch\_map 字段。如果 branch\_map 为 0，则发送格式 2 数据包，否则发送格式 1 数据包。notify 字段表示数据包是否报告的是通知地址。如果地址是无法推测的不连续地址，即使是通知用的数据包，notify 字段也不会变为高电平，否则解码器无法正确重建指令流。

\subsection{解码数据包}

\begin{itemize}
    \item 找到解码的第一个地址
        \begin{itemize}
            \item 读 \hyperref[regdesc:TRACEINTRRAWREG]{TRACE\_INTR\_RAW\_REG} 寄存器的 \hyperref[fielddesc:TRACEMEMFULLINTRRAW]{TRACE\_MEM\_FULL\_INTR\_RAW} 位，查看追踪存储器是否已满
            \begin{itemize}
                \item 如果值为 0，从 \hyperref[regdesc:TRACEMEMSTARTADDRREG]{TRACE\_MEM\_START\_ADDR\_REG} 寄存器读取追踪数据包
                \item 如果值为 1，且开启了存储器循环模式，则之前的数据包已被覆盖。此时，读取 \hyperref[regdesc:TRACEMEMCURRENTADDRREG]{TRACE\_MEM\_CURRENT\_ADDR\_REG} 的值，获取最后一个写入的地址，该地址即为解码器处理的第一个地址
            \end{itemize}
        \end{itemize}

    \item 用解码器解码数据包
        \begin{itemize}
            \item 解码器从第一个地址开始读取所有数据包，结合二进制文件重建数组
            \item 如章节 \ref{sec:riscvtracenc-packetes} 所述，编码器向存储器分区边界写 14 个值为 0 的字节。因此出现 14 个值为 0 的字节时，则表示下一个非 0 字节为新数据包的头部
        \end{itemize}
\end{itemize}

\subsection{AHB 配置}

\chipname{} 编码器支持 AHB 总线，可选配置项可以控制编码器的传输带宽。

如果有多个 DMA 主机，可以通过 \hyperref[regdesc:TRACEAHBCONFIGREG]{TRACE\_AHB\_CONFIG\_REG} 配置 AHB 突发传输，增加编码器的带宽。
\begin{itemize}
    \item \hyperref[fielddesc:TRACEHBURST]{TRACE\_HBURST} 支持以下配置：
        \begin{itemize}
            \item 0x0：SINGLE
            \item 0x1：INCR（长度未定义）
            \item 0x2：INCR4
            \item 0x4：INCR8
            \item 其他值：保留
        \end{itemize}
        突发传输意味着仲裁成功后，可以连续传输多个字。SINGLE 传输为 1 拍，INCR 长度未定义，INCR4 为 4 拍，INCR  为 8 拍。突发长度越长，带宽越大。

    \item 传输配置为 INCR 时，由于传输长度不确定，为了避免编码器长时间占用总线，传输长度达到 \hyperref[fielddesc:TRACEMAXINCR]{TRACE\_MAX\_INCR} 后 INCR 传输结束，重新开始总线仲裁。
\end{itemize}

该配置用于调整带宽。带宽足够时，不需要配置突发传输，可使用默认配置。

\subsection{软件保留}

\hyperref[fielddesc:TRACEWORKSTATUS]{TRACE\_WORK\_STATUS} 表示编码器的状态。编码器不处于空闲状态时，必须备份配置，并在芯片进入睡眠状态时还原配置。
