\section{概述}

\chipname{} 的 MIPI CSI 模块为智能手机、平板电脑及其他便携式电子设备提供与摄像头模块连接的标准接口。该接口连接摄像头与移动设备应用处理器，旨在实现高速数据传输，确保摄像头与主机系统之间的高效通信。

\chipname{} MIPI CSI 由 MIPI RX D-PHY 接口和 MIPI CSI-2 主机控制器组成，提供一个时钟通道和两个数据通道，每个通道支持高达 1.5 Gbps 的数据传输速率，可与符合 \hyperlink{https://www.mipi.org/specifications/csi-2}{MIPI CSI-2 规范} 的摄像头传感器通信。

\begin{tiplisting}
\hyperlink{https://www.mipi.org/specifications/csi-2}{MIPI CSI-2 规范} 定义了图像应用处理器与摄像头之间的通信。它是 MIPI 联盟标准定义的通信协议的一部分，用于移动系统中的芯片间通信。
\end{tiplisting}

\section{术语}

本小节介绍描述 MIPI CSI 功能的术语。

\begin{longtable}[c]{ R{3cm} p{12cm} }
\textbf{MIPI CSI-2}& MIPI 联盟摄像头串行接口 2 (CSI-2) 规范。该接口通过 PHY 协议接口 (PPI) 从 MIPI RX D-PHY 接收摄像头数据，并通过 32 位图像接口将数据输出到图像信号处理器（ISP）。\\
\textbf{MIPI D-PHY}& MIPI 联盟规范的源同步物理层。该接口实现了 MIPI D-PHY 接口的通用通道物理层，通过一条时钟通道和两条数据通道从 MIPI 摄像头传感器接收时钟和数据。\\
\textbf{PPI}& MIPI 联盟规范中定义的 PHY 协议接口。该接口连接 CSI-2 主机控制器和 D-PHY。\\
\textbf{ISP}& 图像信号处理器，用于处理从摄像头接收的图像。有关 ISP 的详细信息，请参阅章节 \ref{mod:isp} \textit{\nameref{mod:isp}}。\\
\textbf{发射器}& 发射数据的设备，通常为 MIPI 摄像头。\\
\textbf{接收器}& 接收数据的设备，在本章中为 \chipname{} 芯片。\\
\textbf{时钟通道}& MIPI D-PHY 接口的时钟通道，用于传输高速差分时钟信号。\\
\textbf{数据通道}& MIPI D-PHY 接口的数据通道，用于传输数据和命令。\\
\textbf{线路}& MIPI D-PHY 接口数据通道的线路，每一条数据通道都包含 P、N 两条线路。\\
\textbf{序列} & 通过数据通道接收的一串状态码，控制 MIPI RX D-PHY 从控制模式进入其他模式。\\ 
\textbf{控制模式} & MIPI RX D-PHY 的工作模式之一，此模式用于接收来自发射器的请求。所有其他模式结束后都会回到该模式\\
\textbf{高速数据接收模式} & MIPI RX D-PHY 的工作模式之一，此模式下以突发传输的方式接收数据。\\
\textbf{Escape 模式} & MIPI RX D-PHY 的工作模式之一，此模式支持数据通道进行低速异步通信，以保持较低功耗状态。\\
\textbf{超低功耗状态 (ULPS)} & MIPI RX D-PHY 的工作模式之一，在此模式下，时钟通道和数据通道均处于空闲状态，处于除关断外的最低功耗。\\
\textbf{Test Code} & 用于管理和测试 MIPI D-PHY 的特定控制或配置代码，例如可用于设置工作模式。\\
\end{longtable}

\section{主要特性}
\label{sec:mipicsi-features}

\chipname{} MIPI CSI 支持以下特性： 

\begin{itemize}
    \item MIPI RX D-PHY 符合 MIPI D-PHY 接口规范修订版 1.1
    \item MIPI CSI-2 主机符合 MIPI CSI-2 规范
    \item 具有两个数据通道，每个通道支持 80 Mbps \verb+~+ 1.5 Gbps 的速率范围
    \item Escape 模式下支持超低功耗状态 (ULPS)
    \item 支持与 ISP 连接的 32 位图像接口
    \item 支持多种输入图像格式：
        \begin{itemize}
            \item RGB888/RGB666/RGB565
            \item YUV422/YUV420
            \item RAW8/RAW10/RAW12
        \end{itemize}
    \item 支持 PHY 层、数据包层、线路层和帧层的错误检测与纠正
    \item 支持通过数据加扰以减少电磁干扰 (EMI)
\end{itemize}

\begin{tiplisting}
\chipname 不支持 MIPI 规范中定义的不同虚拟通道，因此数据包头中的虚拟通道号将被忽略。
\end{tiplisting}

\section{结构概览}
\label{sec:mipicsi-arch-overview}

图 \ref{fig:mipicsi-arch} 展示了 MIPI CSI 的架构及其与其他系统组件的连接。MIPI CSI 包括以下组件：

\begin{itemize}
    \item \textbf{MIPI RX D-PHY}：实现物理链路层，包括一条时钟通道 (CP/CN) 和两条数据通道 (D0P/D0N, D1P/D1N) 用于连接摄像头。
    \item \textbf{MIPI CSI-2 主机}：实现 CSI-2 协议，从 D-PHY 接收摄像头数据并通过图像接口输出，包括：
        \begin{itemize}
            \item \textbf{流水线}：寄存 PPI 信号。
            \item \textbf{解扰器}：启用后可将 PPI 加扰数据转换为原始值。
            \item \textbf{PHY 适配层}：管理 PHY 接口，包括 PHY 错误处理。
            \item \textbf{数据包分析器}：处理从通道接收的数据，包括数据通道合并、头部解码、各种错误检测和纠正等。
            \item \textbf{图像接口}：将像素重新排序为 32 位图像接口数据，并生成时序精确的视频同步信号。
            \item \textbf{错误管理}：监测并通知 CSI-2 链路上的错误状态。
            \item \textbf{寄存器库}：提供对配置和控制寄存器的访问。
        \end{itemize}
    \item \textbf{PPI 接口}：连接 MIPI RX D-PHY 和 MIPI CSI-2 主机控制器。
    \item \textbf{Test Code 接口}：将配置细节从 MIPI CSI-2 主机控制器发送到 MIPI RX D-PHY。
\end{itemize}

\section{功能描述}
\label{sec:mipicsi-func-descr}

本小节介绍 MIPI RX D-PHY 的操作状态和模式，以及支持 MIPI CSI 使用的主要功能。有关详细配置，请参阅小节 \ref{sec:mipicsi-prog-procedures} \nameref{sec:mipicsi-prog-procedures}。

\subsection{通道状态}

发射器通过驱动特定的线电平来决定通道状态。在正常操作中，通道由 HS-TX 或 LP-TX 驱动。

\begin{itemize}
    \item HS-TX：高速传输。HS-TX 以差分方式驱动通道，具有两种状态：HS-0 和 HS-1。当 HS-TX 驱动通道时，通道始终处于高速数据接收模式。
    \begin{itemize}
        \item HS-0：高速数据接收模式下传输差分信号 0。
        \item HS-1：高速数据接收模式下传输差分信号 1。
    \end{itemize}
    \item LP-TX：低功耗传输。两个 LP-TX 以单端方式分别独立驱动通道的两条线路，具有四种状态：LP-00、LP-01、LP-10 和 LP-11。当 LP-TX 驱动通道时，通道处于控制模式或 Escape 模式。
    \begin{itemize}
        \item LP-00：在控制模式和 Escape 模式下，其他 LP 状态 (LP-01, LP-10, LP-11) 之间的过渡状态。在超低功耗状态 (ULPS) 下，通道也是该状态。
        \item LP-01：在 Escape 模式下传输信号 0；或从控制模式进入高速数据接收模式的序列的开始信号。\\
        进入高速数据接收模式的完整序列为：LP-11 $\rightarrow$ LP-01 $\rightarrow$ LP-00
        \item LP-10：在 Escape 模式下传输信号 1；或从控制模式进入 Escape 模式或 Turnaround 模式的序列的开始信号。\\
        进入 Escape 模式的完整序列为：LP-11 $\rightarrow$ LP-10 $\rightarrow$ LP-00 $\rightarrow$ LP-01 $\rightarrow$ LP-00。\\
        进入 Turnaround 模式的完整序列为：LP-11 $\rightarrow$ LP-10 $\rightarrow$ LP-00 $\rightarrow$ LP-10 $\rightarrow$ LP-00。
        \item LP-11：停止状态，该状态为通道的空闲状态。
    \end{itemize}
\end{itemize}

\begin{tiplisting}
Turnaround 模式用于交换数据传输方向，此模式在 \chipname{} MIPI CSI 中未被使用，因此本章节不做进一步说明。
\end{tiplisting}

\subsubsection{断电状态}

在断电状态 (No-Power state) 下，PHY 无供电电压。

当 VDD\_HP\_\regindex{n} (n：0-3) 和 VDD\_MIPI\_DPHY 电源关闭时，MIPI CSI 的 MIPI RX D-PHY 处于断电状态。

\subsubsection{关断状态}

关断状态 (Shut-Down state) 时，PHY 处于空闲状态。

在关断状态下，PHY 禁用所有模拟模块，重置所有数字逻辑，并将 D0P/D0N、D1P/D1N 和 CP/CN 通道设置为高阻抗 (Hi-Z)。激活关断状态可确保最低功耗。将 \hyperref[fielddesc:CSIHOSTDPHYRSTZ]{CSI\_HOST\_DPHY\_RSTZ} 和 \hyperref[fielddesc:CSIHOSTPHYSHUTDOWNZ]{CSI\_HOST\_PHY\_SHUTDOWNZ} 设置为 0 可启用 PHY 的关断状态。

在退出关断状态之前，需要配置一些参数，包括要使用的通道数量和通道工作频率范围。

MIPI RX D-PHY 由两条数据通道组成。配置 \hyperref[fielddesc:CSIHOSTNLANES]{CSI\_HOST\_N\_LANES}，选择使用一条或两条数据通道。在退出关断状态之前，这些配置应保持静态或至少稳定。

根据 MIPI RX D-PHY 的具体要求，可能需要执行额外的配置步骤。默认情况下，PHY 在 80 \verb+~+ 110 Mbps 的较低范围内工作。为了适配更高的比特率，用户可以在 Test Code \hyperref[sec:mipicsi-test-code-0x44]{通道 0 的 HS RX 控制} 中使用适当的代码调整 HS 频率范围（hsfreqrange 字段）。控制接口与 MIPI RX D-PHY 的其余部分相互独立，因此建议在关断状态下进行频率调整。

\subsubsection{AFE 初始化}

在 \hyperref[fielddesc:CSIHOSTDPHYRSTZ]{CSI\_HOST\_DPHY\_RSTZ} 和 \hyperref[fielddesc:CSIHOSTPHYSHUTDOWNZ]{CSI\_HOST\_PHY\_SHUTDOWNZ} 释放后，PHY 开始初始化序列，进入正常运行。建议先释放 \hyperref[fielddesc:CSIHOSTDPHYRSTZ]{CSI\_HOST\_DPHY\_RSTZ}，再释放 \hyperref[fielddesc:CSIHOSTPHYSHUTDOWNZ]{CSI\_HOST\_PHY\_SHUTDOWNZ}。此外，请确保此时 CSI\_DPHY\_CFG\_CLK（详细信息请参考章节 \ref{mod:resclk} \textit{\nameref{mod:resclk}} > 表 \ref{tab:resclk-periph-clks-hp2}）可用且稳定。

初始化序列包含启用内部模块和执行内部校准等多个步骤。完成这些步骤后，控制权转交给通道，通道通过启用或禁用相应的接收器，处理来自发射器的 LP/HS 请求的功耗管理。

将 \hyperref[fielddesc:CSIHOSTPHYSTOPSTATEDATA0]{CSI\_HOST\_PHY\_STOPSTATEDATA\_\regindex{n}} 和 \hyperref[fielddesc:CSIHOSTPHYSTOPSTATECLK]{CSI\_HOST\_PHY\_STOPSTATECLK} 设置为 1，执行所有初始化步骤。在此状态下，时钟和数据通道均处于停止状态 (LP-11)。

初始化周期 (TINIT) 是一个依赖于协议的参数，根据规范定义，其最短持续时间为 100 $\mu$s。设置 \hyperref[fielddesc:CSIHOSTPHYSTOPSTATEDATA0]{CSI\_HOST\_PHY\_STOPSTATEDATA\_\regindex{n}} 和 \hyperref[fielddesc:CSIHOSTPHYSTOPSTATECLK]{CSI\_HOST\_PHY\_STOPSTATECLK} 为 1 后，软件应等待超过 100 $\mu$s，然后开始摄像头数据接收。

\subsubsection{控制模式}

AFE 初始化完成后，MIPI RX D-PHY 默认保持在控制模式 (Control Mode)，同时在线路上接收 LP-11 状态码（停止状态），直到接收到新请求。在接收请求后，PHY 进入高速数据接收模式、Escape 模式或超低功耗状态。所有请求开始和结束时，线路必须处于停止状态。

\subsubsection{高速数据接收模式}

在高速数据接收模式 (High-Speed Data Reception mode) 下，PHY 以突发传输的方式接收数据。通道仅在突发传输期间以高速模式运行。

如图 \ref{fig:mipicsi-hs} 所示，一次突发传输包括三部分：低功耗初始化序列、高速数据有效载荷和接收结束序列。

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\linewidth]{\modulefiles/figures/hs-trans.pdf}
    \caption{MIPI CSI HS 数据接收序列}
    \label{fig:mipicsi-hs}
\end{figure}

接收器在接收到 [LP-11 $\rightarrow$ LP-01 $\rightarrow$ LP-00] 的低功耗序列状态码后，进入高速模式，然后通过 ``00011101" 同步序列实现同步。同步完成后，PHY 接收高速数据，直到在通道上检测到从 LP-00 到 LP-11 的转换。

\subsubsection{Escape 模式}

Escape 模式支持数据通道进行低速异步通信。在 Escape 模式下，通道保持在低功耗状态。在收到 Escape 模式请求序列 [LP-11 $\rightarrow$ LP-10 $\rightarrow$ LP-00 $\rightarrow$ LP-01 $\rightarrow$ LP-00] 后，数据通道进入此模式。如果通道在达到 LP-00 之前检测到 LP-11 状态码，请求将被中止，通道将返回到停止状态。进入 Escape 模式后，发射器会发送一个 8 位命令以指示请求的操作。

MIPI CSI 仅支持 Escape 模式下的超低功耗状态 (ULPS)。进入 ULPS 的 8 位命令为 ``b00011110"。无效的进入命令会被忽略，接收器将等待，直到发射器返回到停止状态。

\subparagraph{超低功耗状态}

超低功耗状态 (Ultra Low Power State) 是除关断状态外功耗最低的状态。

发送超低功耗状态进入命令 ``00011110"，可在数据通道上激活此模式。此后，时钟通道和数据通道均处于空闲状态 (LP-00)。

\subsubsection{MIPI RX D-PHY 编码接口}
\label{sec:mipicsi-rx-dphy-program-interface}

MIPI RX D-PHY 的 Test Code 编程包括两个步骤。首先，将所需的 Test Code 编程到寄存器中，然后将相应的 Test Data 提供给测试仪的输入端。

在配置 Test Code 之前，执行以下步骤：

\begin{itemize}
    \item 将 \hyperref[fielddesc:CSIHOSTPHYSHUTDOWNZ]{CSI\_HOST\_PHY\_SHUTDOWNZ} 和 \hyperref[fielddesc:CSIHOSTDPHYRSTZ]{CSI\_HOST\_DPHY\_RSTZ} 置 0 以关闭和复位 PHY。
    \item（推荐）将 \hyperref[fielddesc:CSIHOSTPHYTESTCLR]{CSI\_HOST\_PHY\_TESTCLR} 置 1，以在首次配置任何 Test Code 之前应用重置脉冲。
\end{itemize}

然后，按照以下步骤配置 Test Code：

\begin{itemize}
    \item 设置所需的 Test Code。
    
    当 \hyperref[fielddesc:CSIHOSTPHYTESTEN]{CSI\_HOST\_PHY\_TESTEN} 为 1 且 \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} 处于下降沿时，Test Code 被采样锁存。
    \begin{itemize}
        \item 确保 \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} 为 1。
        \item 将 \hyperref[fielddesc:CSIHOSTPHYTESTDIN]{CSI\_HOST\_PHY\_TESTDIN} 设置为 8 位 Test Code。
        \item 将 \hyperref[fielddesc:CSIHOSTPHYTESTEN]{CSI\_HOST\_PHY\_TESTEN} 设置为 1。
        \item 将 \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} 设置为 0。
        在 \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} 处于下降沿时，\hyperref[fielddesc:CSIHOSTPHYTESTDIN]{CSI\_HOST\_PHY\_TESTDIN}[7:0] 的值会被内部锁存为当前的 Test Code。
        \item 将 \hyperref[fielddesc:CSIHOSTPHYTESTEN]{CSI\_HOST\_PHY\_TESTEN} 设置为 0。
    \end{itemize}
    \item 输入 Test Data。
    
    当 \hyperref[fielddesc:CSIHOSTPHYTESTEN]{CSI\_HOST\_PHY\_TESTEN} 为 0 且 \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} 处于上升沿时，Test Data 将被编程到内部。
    \begin{itemize}
        \item 设置 \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} 为 0（如果尚未完成）。
        \item 设置 \hyperref[fielddesc:CSIHOSTPHYTESTDIN]{CSI\_HOST\_PHY\_TESTDIN} 为 8 位 Test Data。
        \item 设置 \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} 为 1。当 \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} 为上升沿时，Test Data 将被编程到内部。
        \item 重复上述步骤，添加更多 Test Data。
    \end{itemize}
\end{itemize}

将 \hyperref[fielddesc:CSIHOSTPHYTESTCLR]{CSI\_HOST\_PHY\_TESTCLR} 设置为 1 可复位 Test Code。仅在第一次编程操作之前，或希望将 PHY 的配置重置为默认值时需要执行此步骤。

图 \ref{fig:mipicsi-test} 展示了 MIPI RX D-PHY 控制接口的时序图。Test Code 成功编程后，\hyperref[fielddesc:CSIHOSTPHYTESTDOUT]{CSI\_HOST\_PHY\_TESTDOUT} 将异步输出与特定 Test Code 相关的数据。这些数据可能包括回读信息或其他有意义的信号。

    \caption{控制接口时序图}

\end{figure}

\subsection{解扰器}

数据加扰使用随机化技术来减轻电磁干扰 (EMI) 和射频 (RF) 自干扰，可以将链路的传输能量扩展到更宽的频率范围。

传输数据通过伪随机比特流 (PRBS) 加扰，以减少通道级别上出现重复模式的可能性。PRBS 由线性反馈移位寄存器 (LFSR) 生成，生成多项式为：

\begin{equation}
    G(x) = x^{16} + x^5 + x^4 + x^3 + 1
\end{equation}

每条数据通道都有一个解扰模块。将 \hyperref[fielddesc:CSIHOSTSCRAMBLEENABLE]{CSI\_HOST\_SCRAMBLE\_ENABLE} 设置为 1 可启用此模块。每条通道可以通过 \hyperref[fielddesc:CSIHOSTSCRAMBLESEEDLANEN]{CSI\_HOST\_SCRAMBLE\_SEED\_LANE\regindex{n}} 配置不同的 16 位 LFSR 种子值。

表 \ref{tab:mipicsi-prbs-seed} 列出了控制器复位后通道的默认种子值。

\section{中断}
\label{sec:mipicsi-int}

\chipname{} 的 MIPI CSI 模块可以生成以下中断信号并发送给\hyperref[mod:intmtrx]{中断矩阵}。

CSI\_HOST\_INT\_ST\_<group> 寄存器组会报告错误条件，并触发 CSI\_INTR 中断信号。

\section{配置流程}
\label{sec:mipicsi-prog-procedures}

本小节介绍如何配置 MIPI RX D-PHY 和 MIPI CSI-2 主机以接收图像数据。MIPI CSI 仅在高速数据接收模式下接收图像数据。

\subsection{开始高速数据接收}

\subsubsection{时钟与复位}
\begin{itemize}
    \item 设置 \hyperref[fielddesc:HPSYSCLKRSTCSIHOSTSYSCLKEN]{HP\_SYS\_CLKRST\_CSI\_HOST\_SYS\_CLK\_EN} 为 1，启用 MIPI CSI 时钟。
    \item 设置 \hyperref[fielddesc:HPSYSCLKRSTCSIHOSTSYSRSTEN]{HP\_SYS\_CLKRST\_CSI\_HOST\_SYS\_RST\_EN} 为 0，释放 MIPI CSI 复位。
    \item 配置 \hyperref[fielddesc:HPSYSCLKRSTMIPICSIDPHYCLKSRCSEL]{HP\_SYS\_CLKRST\_MIPI\_CSI\_DPHY\_CLK\_SRC\_SEL}，为 MIPI RX D-PHY 选择合适的时钟源。
    \item 设置 \hyperref[fielddesc:HPSYSCLKRSTMIPICSIDPHYCFGCLKEN]{HP\_SYS\_CLKRST\_MIPI\_CSI\_DPHY\_CFG\_CLK\_EN} 为 1，启用 MIPI RX D-PHY 的配置时钟。
    \item 设置 \hyperref[fielddesc:HPSYSCLKRSTRSTENCSIBRG]{HP\_SYS\_CLKRST\_RST\_EN\_CSI\_BRG} 为 1，释放 ISP 的 CSI\_Bridge 模块并释放 ISP 复位。
\end{itemize}

\subsubsection{MIPI RX D-PHY 和 MIPI 主机初始化}

\begin{itemize}
    \item 清除 \hyperref[fielddesc:CSIHOSTPHYTESTCLR]{CSI\_HOST\_PHY\_TESTCLR}，复位 CSI\_HOST 主机控制器逻辑。
    \item 设置 \hyperref[fielddesc:CSIHOSTPHYTESTCLR]{CSI\_HOST\_PHY\_TESTCLR} 为 1，然后清除 \hyperref[fielddesc:CSIHOSTPHYTESTCLR]{CSI\_HOST\_PHY\_TESTCLR} 以复位 Test Code 逻辑。
    \item 在 MIPI RX D-PHY 编程接口中用 Test Code 0x44 配置 MIPI RX D-PHY 频率范围。
    \item 配置 \hyperref[fielddesc:CSIHOSTNLANES]{CSI\_HOST\_N\_LANES}，选择系统中的活跃通道数。
    \item 设置 \hyperref[fielddesc:CSIHOSTPHYSHUTDOWNZ]{CSI\_HOST\_PHY\_SHUTDOWNZ} 和 \hyperref[fielddesc:CSIHOSTDPHYRSTZ]{CSI\_HOST\_DPHY\_RSTZ} 为 1，释放 MIPI RX D-PHY 复位。
    \item 将 CSI\_HOST\_INT\_MSK\_*\_REG 寄存器的对应位设置为 1，启用错误中断。
    \item（如需）配置解扰功能。
    \item 设置 \hyperref[fielddesc:CSIHOSTCSI2RESETN]{CSI\_HOST\_CSI2\_RESETN} 为 1，释放 CSI\_HOST 复位。
    \item 等待 \hyperref[fielddesc:CSIHOSTPHYSTOPSTATEDATA0]{CSI\_HOST\_PHY\_STOPSTATEDATA\_\regindex{n}} 和 \hyperref[fielddesc:CSIHOSTPHYSTOPSTATECLK]{CSI\_HOST\_PHY\_STOPSTATECLK} 变为 1。
    \item 启动摄像头图像捕获。
\end{itemize}

\subsection{停止高速数据接收}

\begin{itemize}
    \item 禁用 VDMA 中的相关通道并等待 VDMA 被禁用。请参阅章节 \ref{mod:vdma} \textit{\nameref{mod:vdma}}。
    \item 复位 ISP。请参阅章节 \ref{mod:isp} \textit{\nameref{mod:isp}}。
    \item 设置 \hyperref[fielddesc:HPSYSCLKRSTRSTENCSIBRG]{HP\_SYS\_CLKRST\_RST\_EN\_CSI\_BRG} 为 0，复位 MIPI CSI 模块。
\end{itemize}
