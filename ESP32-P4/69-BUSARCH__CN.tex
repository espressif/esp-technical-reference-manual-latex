\hypertarget{busarch}{}
\chapter{总线架构}
\label{mod:busarch}


\chipname{} SoC 采用多层总线矩阵作为通信架构的核心，通过高效互连的结构连接多个主设备和从设备。该架构支持并发访问，有效缓解访问冲突，使高性能和低功耗的 RISC-V 核心、DMA 控制器、调试接口及各类外设能够在竞争最小的情况下同时运行。通过优化数据流与系统性能，即使多个高速外设同时工作，总线矩阵也能够确保系统高效运转。

\chipname{} SoC 的主系统由以下部分组成：

\begin{itemize}
    \item 主设备 (master)：
        \begin{itemize}
            \item 高性能 (HP) 32 位 RISC-V 处理器 0 的 I-bus、D-bus 和 AHB 总线
            \item 高性能 (HP) 32 位 RISC-V 处理器 1 的 I-bus、D-bus 和 AHB 总线
            \item 低功耗 (LP) 32 位 RISC-V 处理器
            \item TRACE0（Trace 接口 0）
            \item TRACE1（Trace 接口 1）
            \item HP L2MEM MONITOR （HP L2MEM 监测器）
            \item HP SPM MONITOR （HP SPM 监测器）
            \item EMAC
            \item USB OTG\_HS（USB OTG 高速）
            \item USB OTG\_FS（USB OTG 全速）
            \item SD/MMC
            \item GDMA-AHB
            \item VDMA
            \item GDMA-AXI
            \item 2D-DMA
            \item H264 ENC
        \end{itemize}
    \item 从设备 (slave)：
        \begin{itemize}
            \item LP ROM
            \item LP SRAM
            \item HP ROM
            \item HP SPM
            \item HP L2MEM
            \item 外部flash
            \item 外部 RAM
            \item 外设（LP \& HP \& HP CPU 外设）
            \item CSI Bridge
            \item DSI Bridge
        \end{itemize}
\end{itemize}

图 \ref{fig:bus-arch} \textit{\nameref{fig:bus-arch}} 中的总线说明如下：


\begin{itemize}
    \item \textbf{HP CPU I-bus 总线}：连接高性能 (HP) 32 位 RISC-V 处理器的指令总线和总线矩阵，用于处理器取指。直接访问 HP SPM 或者通过 cache 间接访问其他包含代码的存储器，包括HP ROM、 HP L2MEM 以及外部 flash 和 外部RAM；通常情况下，cache 对用户而言是透明的。
    \item \textbf{HP CPU D-bus 总线}：连接高性能 (HP) 32 位 RISC-V 处理器的数据总线和总线矩阵，用于字面量加载和调试访问。直接访问 HP SPM 或者通过 cache 访问其它存储代码或数据的存储器（如 HP ROM、或外部 flash 和 外部RAM），还可以直接访问外设寄存器，但仅限于HP CPU 外设；通常情况下，cache 对用户而言是透明的。
    \item \textbf{HP CPU AHB 总线}：连接高性能 (HP) 32 位 RISC-V 处理器的 AHB 总线和总线矩阵，用于访问外设或存储器中的数据。通过 AHB 接口可访问 LP 与 HP 外设，并可通过地址映射配置直接访问 HP L2MEM、HP ROM、LP SRAM 以及外部 flash 和 外部RAM （地址映射的详细配置请参阅章节 \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}）。
    \item \textbf{LP CPU 总线}：连接低功耗 (LP) 32 位 RISC-V 处理器和总线矩阵，供 LP CPU 访问外设或存储器。可访问对象包括 LP SRAM, LP ROM, HP L2MEM、HP ROM 和 外部 flash， 外部 RAM 以及 LP \& HP \& HP CPU 外设。
    \item \textbf{TRACE 总线}：连接 TRACE0 和 TRACE1 模块和总线矩阵，用于执行调试操作。可访问对象包括 HP L2MEM 以及外部RAM。
    \item \textbf{MEM MONITOR 总线}：连接 HP L2MEM MONITOR 和 HP SPM MONITOR 与总线矩阵，用于将各自监控的内存区域的监控数据写入存储器。两个 MEM MONITOR 接口访问通路相同，可访问对象包括 HP L2MEM 以及外部 RAM。
    \item \textbf{EMAC 总线}：连接以太网 MAC 控制器和总线矩阵，用于高速网络数据传输。可访问对象包括 HP L2MEM 以及外部RAM。
    \item \textbf{USB OTG\_HS 总线}：连接高速 USB OTG 控制器和总线矩阵，用于 USB 2.0 高速数据传输。可访问对象包括 HP L2MEM 以及外部 RAM。
    \item \textbf{USB OTG\_FS 总线}：连接全速 USB OTG 控制器和总线矩阵，用于 USB 2.0 全速数据传输。可访问对象包括 HP L2MEM 以及外部 RAM。
    \item \textbf{SD/MMC 总线}：连接 SD/MMC 控制器和总线矩阵，用于 SD/MMC 卡的数据读写。可访问对象包括 HP L2MEM 以及外部 RAM，支持通过 DMA 进行高速数据传输。
    \item \textbf{GDMA 总线}：芯片包含2种 GDMA 控制器和 1个 VDMA控制器用于高速数据搬运。
        \begin{itemize}
            \item \textbf{GDMA-AHB}：基于 AHB 总线协议的 DMA 控制器
            \item \textbf{GDMA-AXI}：基于 AXI 总线协议的 DMA 控制器
            \item \textbf{VDMA}： 基于 AXI 总线协议的 Video DMA 控制器
        \end{itemize}
        DMA 均提供一条连接至总线矩阵的线路和一条连接外部存储器的线路。GDMA 通过这些总线进行高速数据传输，包括外设到内存、内存到外设、外设到外设以及内存到内存的传输。可访问对象包括 HP L2MEM,以及外部 flash 和 外部 RAM。针对 DMA 访问外部存储器的场景，为确保数据吞吐率，建议尽可能设置较长的 burst 长度。
    \item \textbf{2D-DMA 总线}：连接 DMA2D 图形加速器和总线矩阵，用于 2D 图形数据搬运和处理。可访问对象包括 HP L2MEM 以及 外部 RAM。
    \item \textbf{H264 ENC 总线}：连接 H.264 视频编码器和总线矩阵，包含两个 AXI 接口，访问通路相同。用于视频编码过程中的数据访问。可访问对象包括 HP L2MEM 以及 外部 RAM。
\end{itemize}
