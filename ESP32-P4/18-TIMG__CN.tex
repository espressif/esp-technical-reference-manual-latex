\hypertarget{timg}{}
\chapter{定时器组 (TIMG)}
\label{mod:timg}

\section{概述}
通用定时器可用于准确设定时间间隔、在一定间隔后触发（周期或非周期的）中断或充当硬件时钟。如图 \ref{fig:timg} 所示，\chipname{} 包含两个定时器组，即定时器组 0 和定时器组 1（下文用 TIMG\regindex{n} 表示，\regindex{n} 为 0 或 1）。每个定时器组有两个通用定时器（下文用 T\regindex{x} 表示，\regindex{x} 为 0 或 1）和一个主系统看门狗定时器 (MWDT)。通用定时器基于 16 位预分频器和 54 位可自动重新加载的可逆计数器。

\begin{figure}[h! ]
    \centering
    \includegraphics[width=0.35\textwidth]{\modulefiles/figures/timg_units}
    \caption{定时器组概览}
    \label{fig:timg}
\end{figure}


本章虽然包含主系统看门狗定时器的寄存器描述，但其功能描述请参阅章节 \ref{mod:wdt} \textit{\nameref{mod:wdt}}。本章中“定时器”指代通用定时器。

\section{主要特性}

定时器具有如下功能：
\begin{itemize}
    \item 54 位时基计数器，可配置成递增或递减
    \item 三个时钟源：PLL\_F80M\_CLK 或 XTAL\_CLK 或 RC\_FAST\_CLK 时钟
    \item 16 位时钟预分频器，分频系数为 2 到 65536
    \item 可读取时基计数器的实时值
    \item 暂停和恢复时基计数器
    \item 可配置的警报生成机制
    \item 计数器值重新加载 —— 报警时自动重新加载或软件控制的即时重新加载
    \item 时钟频率计算 —— 基于晶振时钟计算导入 TIMG0 的被测时钟频率，该时钟可称之为 TIMG0\_CALI\_CLK
    \item 电平触发中断
    \item 支持多个 ETM 任务和事件
\end{itemize}

\section{功能描述}
图 \ref{fig:timg-architecture} 为定时器组 TIMG\regindex{n} 的 T\regindex{x} 定时器。T\regindex{x} 包含一个 16 位整数预分频器、一个时基计数器和一个用于产生警报的比较器。

\subsection{16 位预分频器与时钟选择器}

以定时器组 TIMG\regindex{n} 中的 T\regindex{x} 定时器为例：

\begin{itemize}

\item 该定时器可通过配置 \hyperref[regdesc:PCRTIMERGROUP0TIMERCLKCONFREG]{HP\_SYS\_CLKRST\_PERI\_CLK\_CTRL20\_REG} 寄存器或 \hyperref[regdesc:PCRTIMERGROUP0TIMERCLKCONFREG]{HP\_SYS\_CLKRST\_PERI\_CLK\_CTRL21\_REG} 寄存器的 \hyperref[fielddesc:PCRTG0TIMERCLKSEL]{HP\_SYS\_CLKRST\_TIMERGRP\regindex{n}\_T\regindex{x}\_SRC\_SEL} 字段来选择时钟源。该字段为 0 时，选择 XTAL\_CLK 为时钟源；为 1 时，选择 RC\_FAST\_CLK 为时钟源；为 2 时，选择 PLL\_F80M\_CLK 为时钟源。


\item 要开启所选时钟，需配置 \hyperref[regdesc:PCRTIMERGROUP0TIMERCLKCONFREG]{HP\_SYS\_CLKRST\_PERI\_CLK\_CTRL20\_REG} 寄存器或 \hyperref[regdesc:PCRTIMERGROUP0TIMERCLKCONFREG]{HP\_SYS\_CLKRST\_PERI\_CLK\_CTRL21\_REG} 寄存器的 \hyperref[fielddesc:PCRTG0TIMERCLKEN]{HP\_SYS\_CLKRST\_TIMERGRP\regindex{n}\_T\regindex{x}\_CLK\_EN} 字段为 1，要关闭时钟需清零该字段。所选时钟经 16 位预分频器分频，产生时基计数器使用的时基计数器时钟 (TB\_CLK)。16 位预分频器的分频系数可通过 \hyperref[fielddesc:TIMGTXDIVIDER]{TIMG\_T\regindex{x}\_DIVIDER} 字段配置。


\end{itemize}

\hyperref[fielddesc:TIMGTXDIVIDER]{TIMG\_T\regindex{x}\_DIVIDER} 字段可配置为 0 \verb+~+ 65535 之间的任意值，分频系数的范围为 2 \verb+~+ 65536。具体来说：
\begin{itemize}
    \item 当 \hyperref[fielddesc:TIMGTXDIVIDER]{TIMG\_T\regindex{x}\_DIVIDER} 字段置 0 时：分频系数为 65536。
    \item 当 \hyperref[fielddesc:TIMGTXDIVIDER]{TIMG\_T\regindex{x}\_DIVIDER} 字段置 1 时：分频系数为 2。
    \item 当 \hyperref[fielddesc:TIMGTXDIVIDER]{TIMG\_T\regindex{x}\_DIVIDER} 字段置 2 时：分频系数仍为 2。
    \item 当 \hyperref[fielddesc:TIMGTXDIVIDER]{TIMG\_T\regindex{x}\_DIVIDER} 字段置 3 \verb+~+ 65535 时：分频系数为 3 \verb+~+ 65535。
\end{itemize}


要更改 16 位预分频器，需先重新配置 \hyperref[fielddesc:TIMGTXDIVIDER]{TIMG\_T\regindex{x}\_DIVIDER} 字段，再将 \hyperref[fielddesc:TIMGTXDIVCNTRST]{TIMG\_T\regindex{x}\_DIVCNT\_RST} 置 1，同时需要关闭定时器（即 \hyperref[fielddesc:TIMGTXEN]{TIMG\_T\regindex{x}\_EN} 必须清零），否则会造成不可预知的结果。

\subsection{54 位时基计数器}
54 位时基计数器基于 TB\_CLK，可通过 \hyperref[fielddesc:TIMGTXINCREASE]{TIMG\_T\regindex{x}\_INCREASE} 字段配置为递增或递减。时基计数器可通过置位或清零 \hyperref[fielddesc:TIMGTXEN]{TIMG\_T\regindex{x}\_EN} 字段使能或关闭。使能时，时基计数器的值会在每个 TB\_CLK 周期递增或递减。关闭时，时基计数器暂停计数。注意，无论\hyperref[fielddesc:TIMGTXEN]{TIMG\_T\regindex{x}\_EN} 是否置位，\hyperref[fielddesc:TIMGTXINCREASE]{TIMG\_T\regindex{x}\_INCREASE} 字段都可以更改，时基计数器可立即改变计数方向。

时基计数器 54 位定时器的当前值必须被锁入两个寄存器，才能被 CPU 读取（因为 CPU 为 32 位）。向 \\\hyperref[regdesc:TIMGTXUPDATEREG]{TIMG\_T\regindex{x}UPDATE\_REG} 写入任意值时，54 位定时器的值开始被锁入寄存器 \hyperref[regdesc:TIMGTXLOREG]{TIMG\_T\regindex{x}LO\_REG} 和 \hyperref[regdesc:TIMGTXHIREG]{TIMG\_T\regindex{x}HI\_REG}，两个寄存器分别锁存低 32 位和高 22 位。当 \hyperref[regdesc:TIMGTXUPDATEREG]{TIMG\_T\regindex{x}UPDATE\_REG} 被硬件清零，表明锁存操作已经完成，可以从这两个寄存器中读取当前计数值。在 \hyperref[regdesc:TIMGTXUPDATEREG]{TIMG\_T\regindex{x}UPDATE\_REG} 被写入新值之前，保持寄存器 \hyperref[regdesc:TIMGTXLOREG]{TIMG\_T\regindex{x}LO\_REG} 和 \hyperref[regdesc:TIMGTXHIREG]{TIMG\_T\regindex{x}HI\_REG} 的值不变，以供 32 位的 CPU 读值。

\subsection{警报生成}
定时器可配置为在当前值与报警值相同时触发报警。报警会产生中断，同时（可选择）让定时器的当前值自动重新加载（详见第 \ref{sec:timg-func-descr-reload} 节）。

54 位报警值可在 \hyperref[regdesc:TIMGTXALARMLOREG]{TIMG\_T\regindex{x}ALARMLO\_REG} 和 \hyperref[regdesc:TIMGTXALARMHIREG]{TIMG\_T\regindex{x}ALARMHI\_REG} 配置，两者分别代表报警值的低 32 位和高 22 位。但是，只有置位 \hyperref[fielddesc:TIMGTXALARMEN]{TIMG\_T\regindex{x}\_ALARM\_EN} 字段使能报警功能后，配置的报警值才会生效。为解决报警使能“过晚”（即报警使能时，定时器的值已过报警值），出现以下情况时硬件会立即触发报警：
\begin{itemize}
    \item 可逆计数器向上计数时，定时器的当前值高于报警值（在一定范围内）
    \item 可逆计数器向下计数时，定时器的当前值低于报警值（在一定范围内）
\end{itemize}

表 \ref{tab:timg-alarm-generation-up} 和表 \ref{tab:timg-alarm-generation-down} 说明了定时器当前值、报警值与报警触发的关系。假设定时器当前值和报警值如下：

\begin{itemize}
    \item TIMG\_VALUE = \{\hyperref[regdesc:TIMGTXHIREG]{TIMG\_T\regindex{x}HI\_REG}, \hyperref[regdesc:TIMGTXLOREG]{TIMG\_T\regindex{x}LO\_REG}\}
    \item ALARM\_VALUE = \{\hyperref[regdesc:TIMGTXALARMHIREG]{TIMG\_T\regindex{x}ALARMHI\_REG}, \hyperref[regdesc:TIMGTXALARMLOREG]{TIMG\_T\regindex{x}ALARMLO\_REG}\}
\end{itemize}

\subsection{定时器重新加载} \label{sec:timg-func-descr-reload}
定时器重新加载指将定时器的低 32 位和高 22 位分别更新为 \hyperref[fielddesc:TIMGTXLOADLO]{TIMG\_T\regindex{x}\_LOAD\_LO} 和 \hyperref[fielddesc:TIMGTXLOADHI]{TIMG\_T\regindex{x}\_LOAD\_HI} 字段存储的重新加载值。但是，把重新加载值写入 \hyperref[fielddesc:TIMGTXLOADLO]{TIMG\_T\regindex{x}\_LOAD\_LO} 和 \hyperref[fielddesc:TIMGTXLOADHI]{TIMG\_T\regindex{x}\_LOAD\_HI} 字段不会改变定时器的当前值。写入的重新加载值会被定时器忽视，直到重新加载事件被触发。重新加载事件可由软件即时重新加载或报警时自动重新加载触发。

CPU 在寄存器 \hyperref[regdesc:TIMGTXLOADREG]{TIMG\_T\regindex{x}LOAD\_REG} 写任意值会触发软件即时重新加载，定时器的当前值会立即改变。若此时 \hyperref[fielddesc:TIMGTXEN]{TIMG\_T\regindex{x}\_EN} 是置位状态，定时器会继续从新数值开始递增或递减计数。若此时 \hyperref[fielddesc:TIMGTXALARMEN]{TIMG\_T\regindex{x}\_ALARM\_EN} 置位, 仍会根据表 \ref{tab:timg-alarm-generation-up} 或表 \ref{tab:timg-alarm-generation-down} 中的关系在对应时刻产生报警。若 \hyperref[fielddesc:TIMGTXEN]{TIMG\_T\regindex{x}\_EN} 是清零状态，定时器将保持当前值，直至计数重新使能。

报警时，自动重新加载功能可让定时器在报警时重新加载，从重新加载值开始继续递增或递减计数。该功能通常用于周期性报警时重置定时器的值。要使能报警时自动重新加载，需将 \hyperref[fielddesc:TIMGTXAUTORELOAD]{TIMG\_T\regindex{x}\_AUTORELOAD} 字段置 1。如未使能该功能，报警后定时器的值会在过报警值后继续递增或递减。

\subsection{事件任务矩阵功能}
\label{sec:timg-etm-func}

在 \chipname{} 中，TIMG\regindex{n} 支持 ETM 功能，即可以通过任意外设的 ETM 事件触发 TIMG\regindex{n} 的 ETM 任务，或者通过 TIMG\regindex{n} 的 ETM 事件触发任意外设的 ETM 任务。关于 ETM 更多详细信息，请参考章节 \ref{mod:evntaskmatrix} \nameref{mod:evntaskmatrix}。这里仅介绍与 TIMG\regindex{n} 相关的 ETM 任务和 ETM 事件。

TIMG\regindex{n} 可接收的 ETM 任务有：

\begin{itemize}
    \item TG0\_TASK\_CNT\_START\_TIMER0：触发时启用时基计数器。
    \item TG1\_TASK\_CNT\_START\_TIMER0：触发时启用时基计数器。
    \item TG0\_TASK\_CNT\_STOP\_TIMER0：触发时关闭时基计数器。
    \item TG1\_TASK\_CNT\_STOP\_TIMER0：触发时关闭时基计数器。

\begin{tiplisting}
以上两个 ETM 任务与 APB 配置 \hyperref[fielddesc:TIMGTXEN]{TIMG\_T0\_EN} 实现的功能相同。 当这些操作同时发生时, 各操作优先级由高到低如下：

\begin{enumerate}
    \item TG0\_TASK\_CNT\_START\_TIMER0 与 TG1\_TASK\_CNT\_START\_TIMER0：触发时启用时基计数器
    \item TG0\_TASK\_CNT\_STOP\_TIMER0 与 TG1\_TASK\_CNT\_STOP\_TIMER0：触发时关闭时基计数器
    \item APB 方式配置 \hyperref[fielddesc:TIMGTXEN]{TIMG\_T0\_EN}：启用或关闭时基计数器
\end{enumerate}
\end{tiplisting}

\item TG0\_TASK\_ALARM\_START\_TIMER0：触发时启用警报生成。
\item TG1\_TASK\_ALARM\_START\_TIMER0：触发时启用警报生成。

\begin{tiplisting}
通过 APB 方式配置 \hyperref[regdesc:TIMGTXCONFIGREG]{TIMG\_T0\_ALARM\_EN} 以及硬件事件触发也可以启用警报生成功能。当这些操作同时发生时，各个操作的优先级从高到低依次为：

\begin{enumerate}
    \item TG0\_TASK\_ALARM\_START\_TIMER0 与 TG1\_TASK\_ALARM\_START\_TIMER0：触发时启用警报生成
    \item 警报事件：触发时关闭警报生成
    \item APB 方式配置 \hyperref[regdesc:TIMGTXCONFIGREG]{TIMG\_T0\_ALARM\_EN}：启用/关闭警报生成
\end{enumerate}
\end{tiplisting}


\item TG0\_TASK\_CNT\_CAP\_TIMER0：触发时会将当前计数器的值更新至 \hyperref[regdesc:TIMGTXLOREG]{TIMG\_T0LO\_REG} 和 \hyperref[regdesc:TIMGTXHIREG]{TIMG\_T0HI\_REG} 寄存器。
\item TG1\_TASK\_CNT\_CAP\_TIMER0：触发时会将当前计数器的值更新至 \hyperref[regdesc:TIMGTXLOREG]{TIMG\_T0LO\_REG} 和 \hyperref[regdesc:TIMGTXHIREG]{TIMG\_T0HI\_REG} 寄存器。
\item TG0\_TASK\_CNT\_RELOAD\_TIMER0：触发时会用存储在 \hyperref[fielddesc:TIMGTXLOADLO]{TIMG\_T0\_LOAD\_LO} 和 \hyperref[fielddesc:TIMGTXLOADHI]{TIMG\_T0\_LOAD\_HI} 中的重新加载值覆盖当前计数器值。
\item TG1\_TASK\_CNT\_RELOAD\_TIMER0：触发时会用存储在 \hyperref[fielddesc:TIMGTXLOADLO]{TIMG\_T0\_LOAD\_LO} 和 \hyperref[fielddesc:TIMGTXLOADHI]{TIMG\_T0\_LOAD\_HI} 中的重新加载值覆盖当前计数器值。
\end{itemize}

TIMG\regindex{n} 可产生的 ETM 事件有：

\begin{itemize}
    \item TG0\_EVT\_CNT\_CMP\_TIMER0：表示 TIMG0 的 T0 定时器中断事件。
    \item TG1\_EVT\_CNT\_CMP\_TIMER0：表示 TIMG1 的 T0 定时器中断事件。

\begin{tiplisting}
只有当 \hyperref[fielddesc:TIMGETMEN]{TIMG\regindex{n}\_ETM\_EN} 设置为 1 时，TIMG\regindex{n} 的 ETM 任务和事件才会生效。
\end{tiplisting}
\end{itemize}

在具体应用中，TIMG\regindex{n} 的 ETM 事件可以用来触发 TIMG\regindex{n} 的 ETM 任务。

例如，TG0\_EVT\_CNT\_CMP\_TIMER0 与 TG1\_EVT\_CNT\_CMP\_TIMER0 事件分别可以触发 TG0\_TASK\_ALARM\_START\_TIMER0 与 TG1\_TASK\_ALARM\_START\_TIMER0 任务，从而实现周期性报警。具体配置步骤请参考 \ref{sec:timg-conf-etm} \nameref{sec:timg-conf-etm}。


\subsection{TIMG0\_CALI\_CLK 频率计算}\label{sec:rtc-calibration-desc}
定时器组可以以 XTAL\_CLK 为基准时钟，计算 TIMG0\_CALI\_CLK 的实际频率。计算方式如下：
\begin{enumerate}
    \item 通过周期性或单次计算的方式启动频率计算模块（详见章节 \ref{sec:rtc-calibration-procedure}）；
    \item 在接收到计算开始的信号后，两个分别工作在 XTAL\_CLK 以及 TIMG0\_CALI\_CLK 的计数器同时开始计数，当 TIMG0\_CALI\_CLK 的计数器达到设定的计算周期 C0 时，同时停止两个计数器；
    \item 通过 XTAL\_CLK 的计数器值 C1 即可计算 TIMG0\_CALI\_CLK 的时钟频率：$f\_cali = \frac{C0 \times f\_XTAL\_CLK}{C1}$
\end{enumerate}

\subsection{中断}

\section{配置与使用}
\subsection{定时器用作简单时钟}\label{sec:timg-func-descr-simple-clock}
\begin{enumerate}
    \item 配置时基计数器。
        \begin{itemize}
        \item 配置 \hyperref[regdesc:PCRTIMERGROUP0TIMERCLKCONFREG]{HP\_SYS\_CLKRST\_PERI\_CLK\_CTRL21\_REG} 寄存器的 \hyperref[fielddesc:PCRTG0TIMERCLKSEL]{HP\_SYS\_CLKRST\_TIMERGRP0\_TGRT\_CLK\_SRC\_SEL} 字段选择时钟源。
        \item 置位 \hyperref[fielddesc:TIMGTXDIVIDER]{TIMG\_T\regindex{x}\_DIVIDER} 配置 16 位预分频器。
        \item 置位或清除 \hyperref[fielddesc:TIMGTXINCREASE]{TIMG\_T\regindex{x}\_INCREASE} 配置定时器方向。
        \item 在 \hyperref[fielddesc:TIMGTXLOADLO]{TIMG\_T\regindex{x}\_LOAD\_LO} 和 \hyperref[fielddesc:TIMGTXLOADHI]{TIMG\_T\regindex{x}\_LOAD\_HI} 上写初始值设置定时器的初始值，然后在 \\\hyperref[regdesc:TIMGTXLOADREG]{TIMG\_T\regindex{x}LOAD\_REG} 上写任意值将初始值重新加载进定时器。
    \end{itemize}
    \item 置位 \hyperref[fielddesc:TIMGTXEN]{TIMG\_T\regindex{x}\_EN} 开启定时器。
    \item 获得定时器的当前值。
    \begin{itemize}
        \item 在 \hyperref[regdesc:TIMGTXUPDATEREG]{TIMG\_T\regindex{x}UPDATE\_REG} 上写任意值锁存定时器的当前值。
        \item 等待硬件将 \hyperref[regdesc:TIMGTXUPDATEREG]{TIMG\_T\regindex{x}UPDATE\_REG} 清 0。
        \item 从 \hyperref[regdesc:TIMGTXLOREG]{TIMG\_T\regindex{x}LO\_REG} 和 \hyperref[regdesc:TIMGTXHIREG]{TIMG\_T\regindex{x}HI\_REG} 读取锁存的定时器值。
    \end{itemize}
\end{enumerate}

\subsection{定时器用于单次报警}\label{sec:timg-func-descr-timer-alarm}
\begin{enumerate}
    \item 按照第 \ref{sec:timg-func-descr-simple-clock} 节的第 1 步配置时基计数器。
    \item 配置报警。
    \begin{itemize}
        \item 置位 \hyperref[regdesc:TIMGTXALARMLOREG]{TIMG\_T\regindex{x}ALARMLO\_REG} 和 \hyperref[regdesc:TIMGTXALARMHIREG]{TIMG\_T\regindex{x}ALARMHI\_REG} 配置报警值。
        \item 置位 \hyperref[fielddesc:TIMGTXINTENA]{TIMG\_T\regindex{x}\_INT\_ENA} 使能中断。
    \end{itemize}
    \item 清零 \hyperref[fielddesc:TIMGTXAUTORELOAD]{TIMG\_T\regindex{x}\_AUTORELOAD} 关闭自动重新加载。
    \item 置位 \hyperref[fielddesc:TIMGTXALARMEN]{TIMG\_T\regindex{x}\_ALARM\_EN} 开启报警。
    \item 处理报警中断。
    \begin{itemize}
        \item 置位定时器在 \hyperref[fielddesc:TIMGTXINTCLR]{TIMG\_T\regindex{x}\_INT\_CLR} 的对应位清除中断。
        \item 清零 \hyperref[fielddesc:TIMGTXEN]{TIMG\_T\regindex{x}\_EN} 关闭定时器。
    \end{itemize}
\end{enumerate}

\subsection{通过 APB 设置定时器用于周期性报警}

\begin{enumerate}
    \item 按照第 \ref{sec:timg-func-descr-simple-clock} 节的第 1 步配置时基计数器。
    \item 按照第 \ref{sec:timg-func-descr-timer-alarm} 节的第 2 步配置报警。
    \item 置位 \hyperref[fielddesc:TIMGTXAUTORELOAD]{TIMG\_T\regindex{x}\_AUTORELOAD} 使能自动重新加载，将重新加载值写入 \hyperref[fielddesc:TIMGTXLOADLO]{TIMG\_T\regindex{x}\_LOAD\_LO} 和 \hyperref[fielddesc:TIMGTXLOADHI]{TIMG\_T\regindex{x}\_LOAD\_HI}。
    \item 置位 \hyperref[fielddesc:TIMGTXALARMEN]{TIMG\_T\regindex{x}\_ALARM\_EN} 开启报警。
    \item 处理报警中断（每次报警时重复）。
    \begin{itemize}
        \item 置位定时器在 \hyperref[fielddesc:TIMGTXINTCLR]{TIMG\_T\regindex{x}\_INT\_CLR} 的对应位清除中断。
        \item 如下一次报警需要新的报警值和重新加载值（即每次都有不同的报警间隔），则应根据需要重新配置 \hyperref[regdesc:TIMGTXALARMLOREG]{TIMG\_T\regindex{x}ALARMLO\_REG}、\hyperref[regdesc:TIMGTXALARMHIREG]{TIMG\_T\regindex{x}ALARMHI\_REG}、\hyperref[fielddesc:TIMGTXLOADLO]{TIMG\_T\regindex{x}\_LOAD\_LO} 和 \hyperref[fielddesc:TIMGTXLOADHI]{TIMG\_T\regindex{x}\_LOAD\_HI} 。否则，上述寄存器应保持不变。
        \item 置位 \hyperref[fielddesc:TIMGTXALARMEN]{TIMG\_T\regindex{x}\_ALARM\_EN} 重新使能报警。
    \end{itemize}
    \item（最后一次报警时）关闭定时器。
    \begin{itemize}
        \item 置位定时器在 \hyperref[fielddesc:TIMGTXINTCLR]{TIMG\_T\regindex{x}\_INT\_CLR} 的对应位清除中断。
        \item 清零 \hyperref[fielddesc:TIMGTXEN]{TIMG\_T\regindex{x}\_EN} 关闭定时器。
    \end{itemize}
\end{enumerate}

\subsection{通过 ETM 设置定时器用于周期性报警} \label{sec:timg-conf-etm}

\begin{enumerate}
    \item 使能 ETM 模块的时钟
    \item 将 ETM 事件映射到相应的 ETM 任务（使用 ETM 事件触发对应的 ETM 任务）
    \begin{itemize}
        \item 如果 \hyperref[fielddesc:TIMGTXAUTORELOAD]{TIMG\_T0\_AUTORELOAD} 置 1，通过一个 ETM 通道映射 TG0\_EVT\_CNT\_CMP\_TIMER0 与 TG1\_EVT\_CNT\_CMP\_TIMER0 事件到 TG0\_TASK\_ALARM\_START\_TIMER0 与 TG1\_TASK\_ALARM\_START\_TIMER0 任务。
        \item 如果 \hyperref[fielddesc:TIMGTXAUTORELOAD]{TIMG\_T0\_AUTORELOAD} 置 0，除了映射 TG0\_EVT\_CNT\_CMP\_TIMER0 与 TG1\_EVT\_CNT\_CMP\_TIMER0 事件到TG0\_TASK\_ALARM\_START\_TIMER0 与 TG1\_TASK\_ALARM\_START\_TIMER0 任务外，还需要用另一个 ETM 通道映射 TG0\_EVT\_CNT\_CMP\_TIMER0 与 TG1\_EVT\_CNT\_CMP\_TIMER0 事件到 TG0\_TASK\_CNT\_RELOAD\_TIMER0 与 TG1\_TASK\_CNT\_RELOAD\_TIMER0 任务。
    \end{itemize}
    \item 选择使能一个或两个 ETM 通道。
    \item 将 TIMER\_ETM\_EN 置 1 从而使能定时器组的 ETM 事件和任务。
    \item 按照第 \ref{sec:timg-func-descr-simple-clock} 节的第 1 步配置时基计数器。
    \item 按照第 \ref{sec:timg-func-descr-timer-alarm} 节的第 2 步配置报警。
    \item 通过 \hyperref[fielddesc:TIMGTXLOADLO]{TIMG\_T0\_LOAD\_LO} 和 \hyperref[fielddesc:TIMGTXLOADHI]{TIMG\_T0\_LOAD\_HI} 配置重新加载值。
    \item 处理 TIMER\regindex{n}\_EVT\_CNT\_CMP\_TIMER0 (\regindex{n}:0-1)
    \begin{itemize}
        \item 当警报生成时，TG0\_EVT\_CNT\_CMP\_TIMER0 与 TG1\_EVT\_CNT\_CMP\_TIMER0 事件也会产生，此时报警器关闭警报生成。
        \item 如果 \hyperref[fielddesc:TIMGTXAUTORELOAD]{TIMG\_T0\_AUTORELOAD} 为 1，重新加载值会覆盖当前计数器的值。TG0\_TASK\_ALARM\_START\_TIMER0 与 TG1\_TASK\_ALARM\_START\_TIMER0 会重新使能警报生成。
        \item 如果 \hyperref[fielddesc:TIMGTXAUTORELOAD]{TIMG\_T0\_AUTORELOAD} 为 0，由于 TG0\_TASK\_CNT\_RELOAD\_TIMER0 与 TG1\_TASK\_CNT\_RELOAD\_TIMER0 任务，重新加载值会覆盖当前计数器的值。TG0\_TASK\_ALARM\_START\_TIMER0 与 TG1\_TASK\_ALARM\_START\_TIMER0 任务会重新使能警报生成。
    \end{itemize}
    \item（最后一次报警时）关闭定时器。
    \begin{itemize}
        \item 关闭用于映射定时器组事件和任务的 ETM 通道。
        \item 设置 TIMER\_ETM\_EN 为 0。
        \item 置位定时器在 \hyperref[fielddesc:TIMGTXINTCLR]{TIMG\_T0\_INT\_CLR} 的对应位清除中断。
        \item 清零 \hyperref[fielddesc:TIMGTXEN]{TIMG\_T0\_EN} 关闭定时器。
    \end{itemize}
\end{enumerate}



\subsection{RTC\_SLOW\_CLK 频率计算}\label{sec:rtc-calibration-procedure}
\begin{enumerate}
    \item 单次计算
    \begin{itemize}
         \item 设置 \hyperref[fielddesc:TIMGRTCCALICLKSEL]{HP\_SYS\_CLKRST\_TIMERGRP0\_TGRT\_CLK\_EN} 开启 TIMG0\_CALI\_CLK 时钟门控。
         \item 设置 \hyperref[fielddesc:TIMGRTCCALICLKSEL]{HP\_SYS\_CLKRST\_TIMERGRP0\_TGRT\_CLK\_SRC\_SEL} 择一路时钟作为 TIMG0\_CALI\_CLK  的时钟源。
         \item 设置 \hyperref[fielddesc:TIMGRTCCALICLKSEL]{HP\_SYS\_CLKRST\_TIMERGRP0\_TGRT\_CLK\_DIV\_NUM} 输出整数分频后的 TIMG0\_CALI\_CLK。
        \item 设置 \hyperref[fielddesc:TIMGRTCCALIMAX]{TIMG\_RTC\_CALI\_MAX} 配置频率计算时间。
        \item 清空 \hyperref[fielddesc:TIMGRTCCALISTARTCYCLING]{TIMG\_RTC\_CALI\_START\_CYCLING} 选择单次校准模式，然后配置 \hyperref[fielddesc:TIMGRTCCALISTART]{TIMG\_RTC\_CALI\_START} 开启两个计数器。
        \item 等待  \hyperref[fielddesc:TIMGRTCCALIRDY]{TIMG\_RTC\_CALI\_RDY} 的值变为 1，读取 \hyperref[fielddesc:TIMGRTCCALIVALUE]{TIMG\_RTC\_CALI\_VALUE} 获取 XTAL\_CLK 计数器值，根据章节 \ref{sec:rtc-calibration-desc} 的公式计算 RTC\_SLOW\_CLK 频率。
    \end{itemize}
    \item 周期性计算
    \begin{itemize}
        \item 设置 \hyperref[fielddesc:TIMGRTCCALICLKSEL]{HP\_SYS\_CLKRST\_TIMERGRP0\_TGRT\_CLK\_SRC\_SEL} 选择需要计算频率的时钟（RTC\_SLOW\_CLK 的时钟源），设置 \hyperref[fielddesc:TIMGRTCCALIMAX]{TIMG\_RTC\_CALI\_MAX} 配置频率计算时间。
        \item 使能 \hyperref[fielddesc:TIMGRTCCALISTARTCYCLING]{TIMG\_RTC\_CALI\_START\_CYCLING}，硬件将不间断进行频率计算过程。
        \item 只要 \hyperref[fielddesc:TIMGRTCCALICYCLINGDATAVLD]{TIMG\_RTC\_CALI\_CYCLING\_DATA\_VLD} 为 1，即表示 \hyperref[fielddesc:TIMGRTCCALIVALUE]{TIMG\_RTC\_CALI\_VALUE} 有效。
    \end{itemize}
    \item 超时\\
    如果 RTC\_SLOW\_CLK 的计数器没有在 \hyperref[fielddesc:TIMGRTCCALITIMEOUTRSTCNT]{TIMG\_RTC\_CALI\_TIMEOUT\_RST\_CNT} 的 XTAL\_CLK 计数器内完成计数，将置位 \hyperref[fielddesc:TIMGRTCCALITIMEOUT]{TIMG\_RTC\_CALI\_TIMEOUT} 标记计算超时。
\end{enumerate}
