\section{Introduction}


The Analog I2C Controller contains two I2C masters dedicated to configuring and communicating with certain analog circuit modules. Those modules can adjust their operating states by configuring their internal registers to better suit the current application scenario of the chip. For example, the Analog I2C Controller can configure an analog sensor to add a uniform offset to the ADC sampling values, thereby extending the sampling voltage range. Each configurable analog module has an I2C slave internally, which is assigned with an independent address. The Analog I2C Controller supports the configuration of the following analog modules:

\begin{itemize}
    \item CPU\_PLL: This mainly includes the configuration register for CPLL\_CLK, with a slave address of 0x67. For more information on CPLL\_CLK, refer to Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}}.
    \item SYS\_PLL: This mainly includes the configuration register for SPLL\_CLK, with a slave address of 0x66. For more information on SPLL\_CLK, refer to Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}}.
    \item MSPI: This mainly includes the configuration register for MPLL\_CLK , with a slave address of 0x63. For more information on MPLL\_CLK, refer to Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}}.
    \item PLLA: This mainly includes the configuration register for APLL\_CLK , with a slave address of 0x6f. For more information on APLL\_CLK, refer to Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}}.
    \item ANA\_SENSOR: This mainly includes the configuration register for analog sensors (such as \textit{\nameref{mod:tmpsen}} and  \textit{\nameref{mod:adc}}), with a slave address of 0x69.
    \item BIAS: This mainly includes the voltage detection configuration register, with a slave address of 0x6a. For more information, refer to Chapter \ref{mod:brownout} \textit{\nameref{mod:brownout}}.
\end{itemize}

\section{Feature List}
\label{sec:anai2c-features}

The Analog I2C Controller has the following features:

\begin{itemize}
    \item Master mode only
    \item 7-bit addressing
    \item Adjustable transmission rate
    \item Communication in the sleep modes supported by the Low-Power CPU
    \item Dual master operation mode
\end{itemize}

\section{Architectural Overview}
\label{sec:anai2c-arch-overview}

The figure below shows the architecture of the Analog I2C Controller.

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/ana_i2c_arch.pdf}
    \caption{Analog I2C Controller Architecture}
    \label{fig:anai2c-functional-overview}
\end{figure}

\begin{itemize}
    \item Configuration Registers: A register module interfacing with the software, providing relevant configurations for the operation of the I2C master. This module controls the transmission address, data, etc., and monitors this transmission process by reading the status registers of I2C0/1\_MST.
    \item I2C0/1\_MST: Two I2C masters that can independently configure analog modules in parallel. The I2C signals they generate will be selected by a subsequent MUX and output to the analog modules.
    \item I2C signal MUX: A module that determines which analog master, I2C0\_MST or I2C1\_MST, configures a particular analog module.
\end{itemize}

\section{Functional Description}
\label{sec:anai2c-func-descr}

\textbf{Transmission Rate Adjustment}

The transmission rate and signal phase of the I2C signal can be configured by \hyperref[fielddesc:ANAI2CMSTI2C0SDASIDEGUARD]{ANA\_I2C\_MST\_I2C\regindex{x}\_SDA\_SIDE\_GUARD} and \hyperref[fielddesc:ANAI2CMSTI2C0SCLPULSEDUR]{ANA\_I2C\_MST\_I2C\regindex{x}\_SCL\_PULSE\_DUR}. The relationship between the register values and the waveform is shown in the following figure.

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/i2c_phase.pdf}
    \caption{Signal Phase}
    \label{fig:anai2c-signal-phase}
\end{figure}

\textbf{Operation in Sleep Modes}

In chip sleep modes, the operation of the Analog I2C Controller can be driven by the Low-Power CPU (see Chapter \ref{mod:ulp} \textit{\nameref{mod:ulp}}).

\textbf{Dual Master Operation Mode}

The module integrates two independent master modules, each with its own set of configurations. The two masters can operate independently, configuring different analog modules, thereby improving operational efficiency.

\section{Programming Procedures}
\label{sec:anai2c-prog-procedures}

The following is the process for configuring analog modules using the Analog I2C Controller.

\begin{itemize}
    \item Write 1 to \hyperref[fielddesc:LPPERICKENLPI2CMST]{LPPERI\_CK\_EN\_LP\_I2CMST} to enable the clock of the Controller.
    \item Configure \hyperref[fielddesc:ANAI2CMSTANACONF2]{ANA\_I2C\_MST\_ANA\_CONF2} to select the module for communication.
    \item Configure \hyperref[fielddesc:ANAI2CMSTI2C0SDASIDEGUARD]{ANA\_I2C\_MST\_I2C\regindex{x}\_SDA\_SIDE\_GUARD} and \hyperref[fielddesc:ANAI2CMSTI2C0SCLPULSEDUR]{ANA\_I2C\_MST\_I2C\regindex{x}\_SCL\_PULSE\_DUR} to determine the transmission rate and signal phase of the I2C signal.
    \item Configure \hyperref[fielddesc:ANAI2CMSTI2C0CTRL]{ANA\_I2C\_MST\_I2C\regindex{x}\_CTRL} to select relevant information for the current communication.
    \item Read \hyperref[fielddesc:ANAI2CMSTI2C0BUSY]{ANA\_I2C\_MST\_I2C\regindex{x}\_BUSY}. A low state indicates the end of the current communication for I2C\regindex{x}.
\end{itemize}
