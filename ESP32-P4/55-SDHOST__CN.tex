\chapter{SD/MMC 主机控制器 (SDHOST)}
\label{mod:sdhost}


\section{概述}

\chipname{} 存储卡接口控制器提供了一个访问安全数字输入输出卡 (SDIO)、MMC 卡和 CE-ATA 设备的硬件接口，用于连接高级外围设备总线 (APB) 和外部存储设备。该控制器支持两个外部卡（卡 0 和卡 1）。所有 SD/MMC 模块接口信号都须通过 GPIO 矩阵传输至 GPIO 管脚。

\section{主要特性}
该模块支持以下特性：

\begin{itemize}
\item 支持两个外部卡
\item 支持 3.0、3.01 版本 SD 存储卡标准，最高支持传输速率为 DDR50
\item 支持 4.41、4.5、4.51 版本 MMC，最高支持传输速率为 DDR50
\item 支持 1.1 版本 CE-ATA
\item 支持 1-bit、4-bit 和 8-bit 位宽模式
\end{itemize}

SD/MMC 控制器连接的拓扑结构如图 \ref{fig:sdhost-card-sys-topo} 所示。该控制器支持两组外设工作，但不支持同时工作。

SD/MMC 的外部接口信号主要为时钟信号 (sdhost\_cclk\_out\_1.eg:card1)、命令信号 (sdhost\_ccmd\_out\_1)、数据信号 (sdhost\_cdata\_in\_1[7:0]/sdhost\_cdata\_out\_1[7:0])，SD/MMC 控制器可通过这些外部接口信号与外部设备通信。其它信号还包括卡中断信号、卡检测信号和写保护信号等。各个信号的方向如图 \ref{fig:sdhost-ext-sign} 所示。每个管脚的方向和描述如表 \ref{tab:sdhost-signal-description} 所示。

\section{功能描述}
\subsection{SD/MMC 主机控制器结构}
SD/MMC 主机控制器主要由两大功能块组成，如图 \ref{fig:sdhost-block-diag} 所示，分别为：

\begin{itemize}
    \item 总线接口单元 (BIU)：提供 APB 总线访问寄存器的接口、RAM 数据访问方式 和 DMA 数据读写操作
    \item 卡接口单元 (CIU)：处理外部存储卡的接口协议，控制时钟。
\end{itemize}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.9\textwidth]{\modulefiles/figures/55-SDHOST-block-diagram}
    \caption{SDIO 主机结构框图}
    \label{fig:sdhost-block-diag}
\end{figure}

\subsubsection{总线接口单元 (BIU)}
该模块提供了通过主机接口单元 (HIU) 访问寄存器和 RAM 数据的方式。除此之外，它通过 DMA 接口提供独立的 RAM 数据访问。它通过 DMA 接口访问存储器中的数据。图 \ref{fig:sdhost-block-diag} 描述了该模块的内部结构。

BIU 支持以下功能：

\begin{itemize}
	\item 主机接口
	\item DMA 接口
	\item 中断控制
	\item 寄存器访问
	\item FIFO 访问
	\item 上电/上拉控制和卡检测
\end{itemize}

\subsubsection{卡接口单元 (CIU)}
该模块实现卡专用接口协议。在 CIU 中，命令通路 (Cmd Path) 控制单元和数据通路 (Data Path) 控制单元将控制器分别连接到 SD/MMC/CE-ATA 卡的命令接口和数据接口。CIU 还提供时钟控制。图 \ref{fig:sdhost-block-diag} 描述了 CIU 的内部结构。

CIU 由以下主要功能模块组成：

\begin{itemize}
	\item 命令通路
	\item 数据通路
	\item SDIO 中断控制
	\item 时钟控制
	\item Mux/De-Mux 单元
\end{itemize}

\subsection{命令通路}

命令通路具有以下功能：
\begin{itemize}
	\item 配置时钟参数
	\item 配置卡命令参数
	\item 向卡总线发送命令 (sdhost\_ccmd\_out)
	\item 从卡总线接收响应 (sdhost\_ccmd\_in)
	\item 向 BIU 发送响应
	\item 在命令线上发送 P-bit
\end{itemize}

命令通路状态机如图 \ref{fig:sdhost-cmd-state-machine} 所示。

\begin{figure}[H]
    \centering
    \includegraphics[width=0.55\textwidth]{\modulefiles/figures/55-SDHOST-cmd-state-machine.eps}
    \caption{命令通路状态机}
    \label{fig:sdhost-cmd-state-machine}
\end{figure}

\subsection{数据通路}

发送时，数据通路模块会取出 RAM 中的数据，并将其发送至 sdhost\_cdata\_out；接收时，数据通路会接收 sdhost\_cdata\_in 上的数据，并将其导入 RAM 中。数据发送命令未运行时，数据通路将加载新的数据参数，即希望发送的数据、读/写数据发送、流/块发送、块大小、字节数、卡类型、超时寄存器等。

如果在命令寄存器 (\hyperref[regdesc:SDHOSTCMDREG]{SDHOST\_CMD\_REG}) 中置了 \hyperref[fielddesc:SDHOSTDATAEXPECTED]{SDHOST\_DATA\_EXPECTED} 位，则新命令为数据传输命令，数据通路将执行以下操作之一：

\begin{itemize}
\item 若 \hyperref[fielddesc:SDHOSTREADWRITE]{SDHOST\_READ\_WRITE} 位为 1，发送数据
\item 若 \hyperref[fielddesc:SDHOSTREADWRITE]{SDHOST\_READ\_WRITE} 位为 0，接收数据
\end{itemize}

\subsubsection{数据发送}
接收到数据写入命令的响应后，该模块将在两个时钟周期开始发送数据。即使命令通路在响应信息中检测到响应错误或循环冗余检查 (CRC) 错误，数据发送也会正常进行。但是，如果直到响应超时仍没有从卡接收到响应，则不发送数据。根据 \hyperref[regdesc:SDHOSTCMDREG]{SDHOST\_CMD\_REG} 寄存器中 \hyperref[fielddesc:SDHOSTTRANSFERMODE]{SDHOST\_TRANSFER\_MODE} 位的值，数据发送状态机将数据以流或块的形式加至卡数据总线上。数据发送状态机如图 \ref{fig:sdhost-tx-state-machine} 所示。

\begin{figure}[H]
    \centering
    \includegraphics[width=0.7\textwidth]{\modulefiles/figures/55-SDHOST-tx-state-machine.eps}
    \caption{数据发送状态机}
    \label{fig:sdhost-tx-state-machine}
\end{figure}

\subsubsection{数据接收}

数据读取命令完成两个时钟周期后，该模块将开始接收数据。即使命令通路检测到响应错误或 CRC 错误，数据接收也会正常进行。如果一直未从卡接收到响应而发生了响应超时，则 BIU 无法接收到 CIU 数据发送完成的信号。如果 CIU 发送的命令是卡所禁止的非法操作，那么将无法开始读取从卡发送的数，且 BIU 也不会接收到 CIU 数据发送完成的信号。

若在数据超时前未接收到数据，数据通路将向 BIU 发出数据超时信号并结束数据传输。根据 \hyperref[regdesc:SDHOSTCMDREG]{SDHOST\_CMD\_REG} 寄存器中的 \hyperref[fielddesc:SDHOSTTRANSFERMODE]{SDHOST\_TRANSFER\_MODE} 位的值，数据接收状态机以流或块的形式从卡数据总线获取数据。数据接收状态机如图 \ref{fig:sdhost-rx-state-machine} 所示。

\end{figure}

\section{CIU 操作的软件限制}
\begin{itemize}
	\item 一次只能选择一个卡执行命令或发送数据。例如，当向某张卡发送数据或从这张卡接收数据时，不可以给另一张卡发送其它新的命令。但是，可以将新的命令发送到同一张卡上，使其读取设备状态或停止数据传输。
	\item 一次只能发出一个数据传输命令。
	\item 在开放式写卡操作期间，如果由于 RAM 为空导致卡时钟停止，那么软件必须先将数据填充到 RAM 中并启动卡时钟，然后才可以向卡发出一个停止/中止命令。
	\item 在 SDIO/Combo 卡数据传输期间，如果卡功能暂停，并且软件想要恢复所暂停的数据传输，则其必须先重置 RAM（置位并释放 \hyperref[fielddesc:SDHOSTFIFORESET]{SDHOST\_FIFO\_RESET}）并启动恢复命令，这和启动一个新的数据传输命令相似。
	\item 要在进行卡数据传输中发出卡复位命令（CMD0、CMD15 或 CMD52\_reset），软件必须在 \hyperref[regdesc:SDHOSTCMDREG]{SDHOST\_CMD\_REG} 寄存器上置位 \hyperref[fielddesc:SDHOSTSTOPABORTCMD]{SDHOST\_STOP\_ABORT\_CMD}，保证 CIU 可以在发出卡复位命令后停止数据传输。
	\item 当在 \hyperref[regdesc:SDHOSTRINTSTSREG]{SDHOST\_RINTSTS\_REG} 寄存器中设置数据结束位错误时，CIU 无法控制 SDIO 中断。因此在该情况下，软件应忽略 SDIO 中断，并向卡发出停止/中止命令使其停止发送数据。
    \item 在一次读卡过程中，如果由于 RAM 已满而导致卡时钟停止，那么软件将至少读取两个 RAM 地址来重启卡时钟。
	\item 在一次命令/数据传输中只能选取一个 CE-ATA 设备。例如，当一个 CE-ATA 设备已经在传输数据，则其它 CE-ATA 设备不可传输新命令。
	\item 使能 CE-ATA 设备的中断 (nIEN=0) 后，如果该设备已经有正在执行的 SDHOST\_RW\_BLK 命令，则不可以再向这一设备发送新的 SDHOST\_RW\_BLK 命令。只有在等待 CCS (Command Completion Signal) 时可以发送 CCSD (Command Completion Signal Disable)。
	\item 但是，如果一个 CE-ATA (nlEN=1) 设备未使能，则可以向同一设备发送新的命令来读取状态信息。
	\item CE-ATA 设备不支持开放式数据传输。
	\item CE-ATA 数据传输不支持 sdhost\_send\_auto\_stop 信号（禁止软件置位 \hyperref[fielddesc:SDHOSTSENDAUTOSTOP]{SDHOST\_SEND\_AUTO\_STOP}）。
\end{itemize}

置位命令起始位后，在发送出命令之前以下寄存器的值不可改变：

\begin{itemize}
	\item CMD --- 命令
	\item CMDARG --- 命令参数
	\item BYTCNT --- 字节计数
	\item BLKSIZ --- 块大小
	\item CLKDIV --- 时钟分频器
	\item CKLENA --- 时钟使能
	\item CLKSRC --- 时钟源
	\item TMOUT --- 超时
	\item CTYPE --- 卡类型
\end{itemize}

\section{收发数据 RAM}
RAM 子模块是一个收发数据缓存，分为接收和发送两个单元。可通过 CPU 和 DMA 实现数据的收发，从而进行读写操作，可参阅章节 \ref{sec:sdhost-linked-list}。

\subsection{TX RAM 模块}
可通过两种方式使能写入操作：CPU 和 DMA 写入。

使用 CPU 将数据写入 TX RAM 时，可将数据通过 APB 接口直接写入到 \hyperref[fielddesc:SDHOSTBUFFIFOREG]{\hyperref[regdesc:SDHOSTBUFFIFOREG]{SDHOST\_BUFFIFO\_REG}} 寄存器中，该寄存器对应的 FIFO 位宽为 32 位，深度为 512。

使用 DMA 将数据写入 TX RAM 时，可通过构造 DMA 链表描述符实现，参阅章节 \ref{sec:sdhost-linked-list}。

\subsection{RX RAM 模块}
可通过两种方式使能读取操作：CPU 和 DMA 读取。

使用 CPU 从 RX RAM 读取数据时，可通过 APB 接口读取 \hyperref[fielddesc:SDHOSTBUFFIFOREG]{\hyperref[regdesc:SDHOSTBUFFIFOREG]{SDHOST\_BUFFIFO\_REG}} 寄存器获取数据，该寄存器对应的 FIFO 位宽为 32 位，深度为 512。

使用 DMA 从 RX RAM 读取数据时，可通过构造 DMA 链表描述符实现，参阅章节 \ref{sec:sdhost-linked-list}。

\section{操作流程}
\subsection{寄存器初始化流程}

寄存器初始化过程如下：

\begin{enumerate}
\item 写控制寄存器 (\hyperref[regdesc:SDHOSTCTRLREG]{SDHOST\_CTRL\_REG})，原始中断寄存器 (\hyperref[regdesc:SDHOSTRINTSTSREG]{SDHOST\_RINTSTS\_REG}) 和 SDIO 中断屏蔽寄存器 (\hyperref[regdesc:SDHOSTINTMASKREG]{SDHOST\_INTMASK\_REG})，清除残留的中断信息，设置控制和中断使能参数。
\item 写时钟分频器配置寄存器 (\hyperref[regdesc:SDHOSTCLKDIVREG]{SDHOST\_CLKDIV\_REG})，时钟源选择寄存器 (\hyperref[fielddesc:SDHOSTCLKSRCREG]{\hyperref[regdesc:SDHOSTCLKSRCREG]{SDHOST\_CLKSRC\_REG}}) 和时钟使能寄存器 (\hyperref[regdesc:SDHOSTCLKENAREG]{SDHOST\_CLKENA\_REG})，设置输出给卡的时钟。
\item 根据卡的具体参数写其他配置寄存器。例如，可以设置
    \begin{itemize}
    \item 卡位宽 (\hyperref[regdesc:SDHOSTCTYPEREG]{SDHOST\_CTYPE\_REG})
    \item 用户身份标识 (\hyperref[fielddesc:SDHOSTUSRIDREG]{\hyperref[regdesc:SDHOSTUSRIDREG]{SDHOST\_USRID\_REG}})
    \item UHS 模式电压和 DDR (\hyperref[regdesc:SDHOSTUHSREG]{SDHOST\_UHS\_REG})
    \item EMMC 起始位模式 (\hyperref[regdesc:SDHOSTEMMCDDRREG]{SDHOST\_EMMCDDR\_REG})
    \item SDIO 模式 (\hyperref[regdesc:SDHOSTCLKEDGESELREG]{SDHOST\_CLK\_EDGE\_SEL\_REG})
    \item 调整超时阈值 （\hyperref[regdesc:SDHOSTTMOUTREG]{SDHOST\_TMOUT\_REG}）
    \item 防抖滤波参数 （\hyperref[regdesc:SDHOSTDEBNCEREG]{SDHOST\_DEBNCE\_REG}）
    \item 内部时序 （\hyperref[regdesc:SDHOSTENSHIFTREG]{SDHOST\_ENSHIFT\_REG}）
    \end{itemize}
    配置 \hyperref[regdesc:SDHOSTRSTNREG]{SDHOST\_RST\_N\_REG} 可进行卡复位。
\end{enumerate}

\subsection{命令发送流程}

命令发送过程如下：

\begin{enumerate}
\item 写命令参数数据寄存器 (\hyperref[fielddesc:SDHOSTCMDARGREG]{\hyperref[regdesc:SDHOSTCMDARGREG]{SDHOST\_CMDARG\_REG}})，设置命令参数数据。
\item 如果需要传输数据，写数据传输长度配置寄存器 (\hyperref[regdesc:SDHOSTBYTCNTREG]{SDHOST\_BYTCNT\_REG})，卡数据块大小配置寄存器 (\hyperref[regdesc:SDHOSTBLKSIZREG]{SDHOST\_BLKSIZ\_REG})，卡数据阈值控制寄存器 (\hyperref[regdesc:SDHOSTCARDTHRCTLREG]{SDHOST\_CARDTHRCTL\_REG})，设置数据传输参数。
\item 如果需要传输数据，可以通过 DMA 或者 FIFO 实现，建议使用 DMA 进行数据传输。
\begin{itemize}
    \item 如果选择通过 DMA 传输数据，参考章节 \ref{sec:sdhost-dma-init}、章节 \ref{sec:sdhost-dma-tx}、及章节 \ref{sec:sdhost-dma-rx}。
    \item 如果选择通过 FIFO 传输数据，需写 FIFO 配置寄存器 (\hyperref[regdesc:SDHOSTFIFOTHREG]{SDHOST\_FIFOTH\_REG}) 设置所需的数据收发阈值，若为向卡写数据还需将数据写入 FIFO （ \hyperref[fielddesc:SDHOSTBUFFIFOREG]{\hyperref[regdesc:SDHOSTBUFFIFOREG]{SDHOST\_BUFFIFO\_REG}}）并在命令执行过程中持续将数据补充写入 FIFO 。
\end{itemize}
\item 写命令和启动配置寄存器 (\hyperref[regdesc:SDHOSTCMDREG]{SDHOST\_CMD\_REG})，设置命令信息并启动命令。
\item 硬件接收命令后会将寄存器 \hyperref[regdesc:SDHOSTCMDREG]{SDHOST\_CMD\_REG} 中的 \hyperref[fielddesc:SDHOSTSTARTCMD]{SDHOST\_START\_CMD} 字段清 0，硬件无法接收命令时会将寄存器 \hyperref[regdesc:SDHOSTRINTSTSREG]{SDHOST\_RINTSTS\_REG} 中的第 12 位 HLE 置 1。
\item 读寄存器 \hyperref[regdesc:SDHOSTRINTSTSREG]{SDHOST\_RINTSTS\_REG} 或使能中断来获知命令执行情况。如有需要，可以读寄存器 \hyperref[regdesc:SDHOSTRESP0REG]{SDHOST\_RESP\regindex{n}\_REG}（\regindex{n} 为 0 \verb+~+ 3）获知卡返回的命令响应信息。
\item 如果选择通过 FIFO 从卡读取数据，读 FIFO (\hyperref[fielddesc:SDHOSTBUFFIFOREG]{\hyperref[regdesc:SDHOSTBUFFIFOREG]{SDHOST\_BUFFIFO\_REG}}) 可以获取卡返回的数据。
\end{enumerate}

具体命令格式请参考卡规范协议，并根据命令格式确定各寄存器字段的配置值。

\subsection{DMA 控制器初始化流程}\label{sec:sdhost-dma-init}

DMA 控制器初始化过程如下：

\begin{enumerate}
\item 写 DMA 控制器的总线模式寄存器 (\hyperref[regdesc:SDHOSTBMODREG]{SDHOST\_BMOD\_REG})，设置主机总线访问参数。
\item 写 DMA 控制器的中断使能寄存器 (\hyperref[regdesc:SDHOSTIDINTENREG]{SDHOST\_IDINTEN\_REG})，屏蔽不必要的中断类型。
\item 软件驱动器创建发送链表或接收链表。然后向 DMA 控制器链表基地址寄存器 (\hyperref[fielddesc:SDHOSTDBADDRREG]{\hyperref[regdesc:SDHOSTDBADDRREG]{SDHOST\_DBADDR\_REG}}) 中写入链表的起始地址。
\item DMA 控制器引擎尝试从链表中获取描述符。
\end{enumerate}

\subsection{DMA 控制器数据发送流程}\label{sec:sdhost-dma-tx}

DMA 控制器发送数据的过程如下：

\begin{enumerate}
\item 主机设置发送数据的描述符单元 (DES0 \verb+~+ DES3)，置位 OWNER 位 (DES0[31])，同时准备数据缓存。
\item 主机在命令寄存器 (\hyperref[regdesc:SDHOSTCMDREG]{SDHOST\_CMD\_REG}) 中写入数据命令。
\item 主机设置所需的数据发送阈值（在 \hyperref[regdesc:SDHOSTFIFOTHREG]{SDHOST\_FIFOTH\_REG} 寄存器中的 \hyperref[fielddesc:SDHOSTTXWMARK]{SDHOST\_TX\_WMARK} 字段进行）。
\item DMA 控制器引擎读取描述符并检查 OWNER 位。如果 OWNER 位未置位，表明该描述符所允许的操作者为主机。此时，DMA 控制器进入挂起状态，并在 \hyperref[regdesc:SDHOSTIDSTSREG]{SDHOST\_IDSTS\_REG} 寄存器中产生禁能描述符中断。然后，主机需在 \hyperref[regdesc:SDHOSTPLDMNDREG]{SDHOST\_PLDMND\_REG} 中写入任意值来释放 DMA 控制器资源。
\item DMA 控制器等待 \hyperref[regdesc:SDHOSTRINTSTSREG]{SDHOST\_RINTSTS\_REG} 寄存器中的命令完成 (CMDD) 位变成 1，如果 BIU 无报错，则表明数据发送已完成。
\item 然后，DMA 控制器引擎等待 BIU 发送的 DMA 总线请求，该请求将基于配置的数据发送阈值生成。对于使用突发传输不能访问的最后一个字节，则在 AHB 总线上执行单次发送。
\item DMA 控制器从主机存储器的数据缓存获取发送数据，并通过 RAM 将其发送至卡中。
\item 当数据跨越多个描述符时，DMA 控制器将获取下一个描述符，并继续使用后面的描述符进行后续操作。最后一个描述符位将显示该数据是否跨越多个描述符。
\item 当数据发送完成且 \hyperref[fielddesc:SDHOSTIDSTSTI]{SDHOST\_IDSTS\_TI} 位已使能，将通过置位 \hyperref[fielddesc:SDHOSTIDSTSTI]{SDHOST\_IDSTS\_TI} 将状态信息更新至寄存器 \hyperref[regdesc:SDHOSTIDSTSREG]{SDHOST\_IDSTS\_REG}。同时，DMA 控制器将通过对 DES0 执行写操作来清除 OWNER 位。
\end{enumerate}

\subsection{DMA 控制器数据接收流程}\label{sec:sdhost-dma-rx}

DMA 控制器接收数据的过程如下：

\begin{enumerate}
\item 主机设置接收数据的描述符单元 (DES0 \verb+~+ DES3)，置位 OWNER 位 (DES0[31])。
\item 主机在命令寄存器 (\hyperref[regdesc:SDHOSTCMDREG]{SDHOST\_CMD\_REG}) 中写入读数据命令。
\item 主机设置所需的数据接收阈值（在 \hyperref[regdesc:SDHOSTFIFOTHREG]{SDHOST\_FIFOTH\_REG} 寄存器中的 \hyperref[fielddesc:SDHOSTRXWMARK]{SDHOST\_RX\_WMARK} 字段进行）。
\item DMA 控制器引擎读取描述符并检查 OWNER 位。如果 OWNER 位未置位，表明该描述符所允许的操作者为主机。此时，DMA 控制器进入挂起状态，并在 \hyperref[regdesc:SDHOSTIDSTSREG]{SDHOST\_IDSTS\_REG} 寄存器中产生禁能描述符中断。然后，软件需在 \hyperref[regdesc:SDHOSTPLDMNDREG]{SDHOST\_PLDMND\_REG} 中写入任意值释放 DMA 控制器资源。
\item DMA 控制器等待 \hyperref[regdesc:SDHOSTRINTSTSREG]{SDHOST\_RINTSTS\_REG} 寄存器中的命令完成位 (CMDD) 变成 1，如果 BIU 无报错，则表明数据接收已完成。
\item DMA 控制器引擎等待 BIU 发送的 DMA 总线请求。该请求将基于配置的数据接收阈值生成。对于使用突发传输不能访问的最后一个字节，则在 AHB 总线上执行单次传输。
\item DMA 控制器从 RAM 获取数据，并将其发送至主机存储器。
\item 当数据跨越多个描述符时，DMA 控制器将获取下一个描述符，并继续使用后面的描述符进行后续操作。最后一个描述符位将显示该数据是否跨越多个描述符。
\item 当数据接收完成且 \hyperref[fielddesc:SDHOSTIDSTSRI]{SDHOST\_IDSTS\_RI} 位已使能，将通过置位 \hyperref[fielddesc:SDHOSTIDSTSRI]{SDHOST\_IDSTS\_RI} 将状态信息更新至寄存器 \hyperref[regdesc:SDHOSTIDSTSREG]{SDHOST\_IDSTS\_REG}。同时，DMA 控制器将通过对 DES0 执行写操作来清除 OWNER 位。
\end{enumerate}

\section{时钟相位选择}
\label{sec:sdhost-phase-selection}

SD/MMC 主机控制器的源时钟相位可以选择，用来调整输入数据信号或输出数据信号的建立时间的时序。

SD/MMC 主机控制器的源时钟可以选择高频高性能时钟或者低频低功耗时钟。选择高频高性能时钟时，可通过设置寄存器 \hyperref[regdesc:SDHOSTDLLCLKCONFREG]{SDHOST\_DLL\_CLK\_CONF\_REG}，来调整内部信号时钟相位、输出信号驱动时钟相位和输入信号采样时钟相位，调整单位为时钟源周期的 1/64。选择低频低功耗时钟时，可通过设置系统时钟相关寄存器，来调整内部信号时钟相位、输出信号驱动时钟相位和输入信号采样时钟相位，可选择的相位有 0 度、90 度、180 度和 270 度，即调整单位为时钟源周期的 1/4。

\section{中断}

SD/MMC 主机控制器的下列内部中断源可以生成中断信号 SDIO\_HOST\_INTR，并发送给\hyperref[mod:intmtrx]{中断矩阵}。

\begin{itemize}
\item SDIO\_INT：来自卡的中断。
\item EBE：结束位错误/无 CRC 错误时触发。
\item ACD：自动命令结束时触发。
\item SBE/BCI：接收起始位错误时触发。
\item HLE：硬件锁定写入错误时触发。
\item FRUN：FIFO 空/满错误时触发。
\item HTO：主机填充数据超时时触发。
\item DRTO：数据读取超时时触发。
\item RTO：卡响应超时时触发。
\item DCRC：数据 CRC 错误时触发。
\item RCRC：卡响应 CRC 错误时触发。
\item RXDR：从卡读取数据，FIFO 内数据量超过阈值时触发。
\item TXDR：向卡发送数据，FIFO 内数据量少于阈值时触发。
\item DTO：数据传输结束时触发。
\item CMDD：命令执行完毕时触发。
\item RE：卡响应错误时触发。
\item CD：检测到卡时触发。
\item IDSTS\_RI：DMA 链表的数据接收完成时触发。
\item IDSTS\_TI：DMA 链表的数据发送完成时触发。
\item IDSTS\_FBE：致命总线错误发生时触发，发送/接收期间主机中止。此时，DMA 禁用所有的总线访问。
\item IDSTS\_DU：DMA 链表由于 OWNER 位 = 0 (DES0 [31] = 0) 而不可用时触发。
\end{itemize}

寄存器 \hyperref[regdesc:SDHOSTRINTSTSREG]{SDHOST\_RINTSTS\_REG} 和 \hyperref[regdesc:SDHOSTIDSTSREG]{SDHOST\_IDSTS\_REG} 中包含所有可能导致中断的位。寄存器 \hyperref[regdesc:SDHOSTINTMASKREG]{SDHOST\_INTMASK\_REG} 和 \hyperref[regdesc:SDHOSTIDINTENREG]{SDHOST\_IDINTEN\_REG} 中包含所有可导致中断的事件的使能位。

寄存器 \hyperref[regdesc:SDHOSTIDSTSREG]{SDHOST\_IDSTS\_REG} 中有三组中断汇总：卡错误汇总 (第 5 位：\hyperref[fielddesc:SDHOSTIDSTSCES]{SDHOST\_IDSTS\_CES})，正常中断汇总 (第 8 位：\hyperref[fielddesc:SDHOSTIDSTSNIS]{SDHOST\_IDSTS\_NIS}) 和异常中断汇总 (第 9 位：\hyperref[fielddesc:SDHOSTIDSTSAIS]{SDHOST\_IDSTS\_AIS})。对相应的位写 1，可以清除中断。当某组中的所有使能中断都被清除，相应的汇总位将被清零。

中断不排序，如果中断在驱动程序响应之前发生，则不会产生其他中断。 例如，IDSTS\_RI 表示一个或多个数据已发送至主机缓存。

并发事件只会产生一个中断。驱动程序须扫描寄存器 \hyperref[regdesc:SDHOSTRINTSTSREG]{SDHOST\_RINTSTS\_REG} 和 \hyperref[regdesc:SDHOSTIDSTSREG]{SDHOST\_IDSTS\_REG} 来查找导致中断的原因。

\begin{tiplisting}
中断、中断源、中断信号等术语的解释以及它们之间的联系请参考章节 \ref{mod:intmtrx} \textit{\nameref{mod:intmtrx}} > \ref{sec:intmtx-int-terminology} \textit{\nameref{sec:intmtx-int-terminology}}。
\end{tiplisting}
