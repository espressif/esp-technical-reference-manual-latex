\section{Introduction}

The MIPI CSI module of \chipname{} provides a standard interface with camera modules for mobile devices, such as smartphones, tablets, and other portable electronics. It connects cameras to application processors in mobile devices, enabling high-speed data transfer and efficient communication between the camera and the host system.

\chipname{} MIPI CSI implements a MIPI RX D-PHY and a MIPI CSI-2 host controller. It provides one clock lane and two data lanes, supporting a data transmission of up to 1.5 Gbps per lane to communicate with camera sensors compliant with the \hyperlink{https://www.mipi.org/specifications/csi-2}{MIPI CSI-2 specification}.

\begin{tiplisting}
The \hyperlink{https://www.mipi.org/specifications/csi-2}{MIPI CSI-2 specification} defines the communication between image application processors and cameras. It is part of the communication protocols defined by MIPI Alliance standards, designed for chip-to-chip communication in mobile systems.
\end{tiplisting}

\section{Terminology}

This section covers terminology used to describe the functionality of MIPI CSI.

\begin{longtable}[c]{ R{3cm} p{12cm} }
\textbf{MIPI CSI-2}& MIPI Alliance Specification for Camera Serial Interface 2. It receives camera data through the PHY Protocol Interface (PPI) from the MIPI RX D-PHY and outputs data through a 32-bit image interface to the Image Signal Processor (ISP).\\
\textbf{MIPI D-PHY}& MIPI Alliance Specification for the source synchronous physical layer. It implements the physical layer of universal lanes for the MIPI D-PHY interface, which receives clock and data from MIPI camera sensors through one clock lane and two data lanes.\\ 
\textbf{PPI}& PHY Protocol Interface defined in MIPI Alliance Specification. It connects the CSI-2 host controller and D-PHY. \\
\textbf{ISP}& Image Signal Processor, used to process images received from the camera. For more details about ISP, please refer to Chapter \ref{mod:isp} \textit{\nameref{mod:isp}}. \\
\textbf{Transmitter}& The device that transmits data, usually a MIPI camera. \\
\textbf{Receiver}& The device that receives data, in this chapter, the \chipname{} chip. \\
\textbf{Clock lane}& The clock lane of the MIPI D-PHY interface, used to transmit high-speed differential clock signals.\\
\textbf{Data lane}& The data lane of the MIPI D-PHY interface, used to transmit data and commands.\\
\textbf{Line}& The line of the data lane of the MIPI D-PHY interface; each data lane consists of two lines: P and N.\\
\textbf{Sequence} & A string of state codes received through the data lane, controlling the MIPI RX D-PHY to enter other modes from Control mode.\\
\textbf{Control mode} & One of the operating modes of the MIPI RX D-PHY, used to receive requests from the transmitter. All other modes return to this mode.\\
\textbf{High-Speed Data Reception mode} & One of the operating modes of the MIPI RX D-PHY, used to receive data in burst mode.\\
\textbf{Escape mode} & One of the operating modes of the MIPI RX D-PHY, used to support low-speed asynchronous communication on the data lane to maintain a low-power state.\\
\textbf{Ultra Low Power State (ULPS)} & One of the operating modes of the MIPI RX D-PHY, in which the clock lane and data lane are in idle state, maintaining the lowest power consumption except for the Shut-Down state.\\
\textbf{Test code} & Specific control or configuration codes used to manage and test the MIPI D-PHY, such as setting operational mode.\\ 
\end{longtable}

\section{Feature List}
\label{sec:mipicsi-features}

MIPI CSI in \chipname{} supports the following features:

\begin{itemize}
    \item MIPI RX D-PHY compliant with MIPI D-PHY interface specification revision 1.1
    \item MIPI CSI-2 host compliant with MIPI CSI-2 specification
    \item Two data lanes, each with a rate ranging from 80 Mbps to 1.5 Gbps
    \item Ultra Low Power State in Escape mode
    \item 32-bit image interface with ISP
    \item Various input image formats:
        \begin{itemize}
            \item RGB888/RGB666/RGB565
            \item YUV422/YUV420
            \item RAW8/RAW10/RAW12
        \end{itemize}
    \item Error detection and correction at PHY level, packet level, line level, and frame level
    \item Electromagnetic Interference (EMI) mitigation by data scrambling
\end{itemize}

\begin{tiplisting}
\chipname{} does not support different virtual channels defined in the MIPI specification, so virtual channel numbers in the packet header are ignored.
\end{tiplisting}

\section{Architectural Overview}
\label{sec:mipicsi-arch-overview}

Figure \ref{fig:mipicsi-arch} shows the architecture of MIPI CSI and its connection to other system components. The MIPI CSI consists of the following components:

\begin{itemize}
    \item \textbf{MIPI RX D-PHY}: Implements the physical link layer, including one clock lane (CP/CN) and two data lanes (D0P/D0N and D1P/D1N) to connect with cameras.
    \item \textbf{MIPI CSI-2 host}: Implements the CSI-2 protocol, receives camera data from D-PHY, and outputs through the Image Interface, which includes:
        \begin{itemize}
            \item \textbf{Pipeline}: Registers PPI signals.
            \item \textbf{Descrambler}: Converts PPI scrambled data to its original values when enabled.
            \item \textbf{PHY adaptation layer}: Manages the PHY interface, including PHY error handling.
            \item \textbf{Packet analyzer}: Processes received data from lanes, including data lane merging, header decoding, and various error detection and correction.
            \item \textbf{Image interface}: Reorders pixels into 32-bit Image Interface data and generates timing-accurate video synchronization signals.
            \item \textbf{Error management}: Monitors and notifies about error conditions on the CSI-2 link.
            \item \textbf{Register bank}: Provides access to configuration and control registers.
        \end{itemize}
    \item \textbf{PPI interface}: Connects the MIPI RX D-PHY and the MIPI CSI-2 host controller.
    \item \textbf{Test code interface}: Sends configuration details from the MIPI CSI-2 host controller to the MIPI RX D-PHY.
\end{itemize}

\section{Functional Description}
\label{sec:mipicsi-func-descr}

This section introduces the operating states and modes of the MIPI RX D-PHY and the major functions that facilitate the use of MIPI CSI. For detailed configurations, refer to Section \ref{sec:mipicsi-prog-procedures} \nameref{sec:mipicsi-prog-procedures}.

\subsection{Data Lane State}

The functions of the transmitter determine the data lane state by driving certain line levels. During normal operation, either an HS-TX or an LP-TX is driving a lane. 

\begin{itemize}
    \item HS-TX: High-Speed transaction. An HS-TX always drives the lane differentially. It operates in two states: HS-0 and HS-1. When HS-TX drives the lane, the lane remains in High-Speed Data Reception mode.
    \begin{itemize}
        \item HS-0: Transmits differential data 0 in High-Speed Data Reception mode.
        \item HS-1: Transmits differential data 1 in High-Speed Data Reception mode.
    \end{itemize}
    \item LP-TX: Low-Power transaction. The two LP-TXs drive the two lines of a lane independently and single-ended. It operates in four states: LP-00, LP-01, LP-10, and LP-11. When LP-TX drives the lane, the lane is in Control mode or Escape mode.
    \begin{itemize}
        \item LP-00: A bridge state between other LP states (LP-01, LP-10, LP-11) in Control Mode or Escape Mode. When in Ultra Low Power State (ULPS), lanes are also in the LP-00 state.
        \item LP-01: Transmits data 0 in Escape mode, or indicates entry into High-Speed Data Reception mode from Control mode.\\
        The complete sequence for entering High-Speed Data Reception mode is: LP-11 $\rightarrow$ LP-01 $\rightarrow$ LP-00
        \item LP-10: Transmits data 1 in Escape mode, or indicates entry into Escape mode or Turnaround mode from Control mode.\\
        The complete sequence for entering Escape mode is: LP-11 $\rightarrow$ LP-10 $\rightarrow$ LP-00 $\rightarrow$ LP-01 $\rightarrow$ LP-00.\\
        The complete sequence for entering the Turnaround mode is: LP-11 $\rightarrow$ LP-10 $\rightarrow$ LP-00 $\rightarrow$ LP-10 $\rightarrow$ LP-00
        \item LP-11: The stop state, i.e., the default idle state of a lane.
    \end{itemize}
\end{itemize}

\begin{tiplisting}
The Turnaround mode is used for switching the data transmission direction. This mode is not used in the \chipname{} MIPI CSI, and will not be further explained in this chapter.
\end{tiplisting}

\subsection{MIPI RX D-PHY Operation}

Figure \ref{fig:mipicsi-phy-modes} illustrates the various states and modes of the MIPI RX D-PHY during initialization and active state. Detailed descriptions of each states and modes are presented in the following sections.

\begin{figure}[H]

    \label{fig:mipicsi-phy-modes}

\subsubsection{No-Power State}

In the No-Power state, no supply voltage is applied to the PHY.

The MIPI RX D-PHY of MIPI CSI is in No-Power state when the power supply VDD\_HP\_\regindex{n} (n: 0-3) and VDD\_MIPI\_DPHY are off.

\subsubsection{Shut-Down State}

Shut-Down state refers to the idle state.

In Shut-Down state, the PHY disables all analog blocks, resets all digital logic, and sets the D0P/D0N, D1P/D1N, and CP/CN lanes to high impedance (Hi-Z). Activating the Shut-Down state ensures the lowest power consumption. To enable the Shut-Down state of the PHY, set \hyperref[fielddesc:CSIHOSTDPHYRSTZ]{CSI\_HOST\_DPHY\_RSTZ} and \hyperref[fielddesc:CSIHOSTPHYSHUTDOWNZ]{CSI\_HOST\_PHY\_SHUTDOWNZ} to 0.

Before leaving the Shut-Down state, configure several parameters, including the lane number to be used and the lane operation frequency range.

The MIPI RX D-PHY consists of two data lanes. Configure \hyperref[fielddesc:CSIHOSTNLANES]{CSI\_HOST\_N\_LANES} to select either one or both data lanes. These configurations should remain static or stable before exiting the Shut-Down state.

Depending on the specific requirements for the MIPI RX D-PHY, additional configuration steps might be necessary. By default, the PHY operates within a lower range of 80-110 Mbps. To accommodate higher bit rates, adjust the HS frequency ranges (the hsfreqrange field) in test code \hyperref[sec:mipicsi-test-code-0x44]{HS RX Control of lane 0} with the appropriate code. It is recommended to perform this adjustment in the Shut-Down state, as the control interface is independent of the rest of the MIPI RX D-PHY.

The different ranges supported are shown in Table \ref{tab:mipicsi-hsfreq}. The hsfreqrange field indicates the HS operating frequency range and is accessible through test code 0x44. For detailed configuration, please refer to Section \ref{sec:mipicsi-rx-dphy-config} \textit{\nameref{sec:mipicsi-rx-dphy-config}}.

When \hyperref[fielddesc:CSIHOSTDPHYRSTZ]{CSI\_HOST\_DPHY\_RSTZ} and \hyperref[fielddesc:CSIHOSTPHYSHUTDOWNZ] {CSI\_HOST\_PHY\_SHUTDOWNZ} are set to 1, the MIPI RX D-PHY leaves the Shut-Down state and starts the initialization procedure.

\subsubsection{AFE Initialization}

After releasing \hyperref[fielddesc:CSIHOSTDPHYRSTZ]{CSI\_HOST\_DPHY\_RSTZ} and \hyperref[fielddesc:CSIHOSTPHYSHUTDOWNZ]{CSI\_HOST\_PHY\_SHUTDOWNZ}, the PHY begins an initialization sequence for normal operation. It is recommended to release \hyperref[fielddesc:CSIHOSTPHYSHUTDOWNZ] {CSI\_HOST\_PHY\_SHUTDOWNZ} before \hyperref[fielddesc:CSIHOSTDPHYRSTZ]{CSI\_HOST\_DPHY\_RSTZ}. Additionally, ensure that CSI\_DPHY\_CFG\_CLK (refer to Table \ref{tab:resclk-periph-clks-hp2} in Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}} for details) is available and stable at this point.

The initialization sequence includes several steps, such as enabling internal blocks and performing internal calibrations. Once completed, control transfers to the lanes, which manage power for LP/HS requests from the transmitter by enabling or disabling the corresponding receivers.

Set \hyperref[fielddesc:CSIHOSTPHYSTOPSTATEDATA0]{CSI\_HOST\_PHY\_STOPSTATEDATA\_\regindex{n}} and \hyperref[fielddesc:CSIHOSTPHYSTOPSTATECLK]{CSI\_HOST\_PHY\_STOPSTATECLK} to 1 to perform all initialization steps. In this state, both clock and data lanes are in stop state (LP-11).

The initialization period (TINIT) is a protocol-dependent parameter with a minimum duration of 100 $\mu$s, as defined by the specification. After setting \hyperref[fielddesc:CSIHOSTPHYSTOPSTATEDATA0]{CSI\_HOST\_PHY\_STOPSTATEDATA\_\regindex{n}} and \hyperref[fielddesc:CSIHOSTPHYSTOPSTATECLK]{CSI\_HOST\_PHY\_STOPSTATECLK} to 1, the software should wait more than 100 $\mu$s before starting camera data reception.

\subsubsection{Control Mode}

After completing AFE initialization, the MIPI RX D-PHY remains in Control mode by default. It receives the LP-11 state code (stop state) on the lines until a request is made. Upon receiving a request, the PHY enters either High-Speed Data Reception mode, Escape mode, or Ultra Low Power State. All requests must begin and end with the lanes in the stop state.

\subsubsection{High-Speed Data Reception Mode}

In High-Speed Data Reception mode, the PHY receives data in bursts. The lane operates in high-speed mode only during these bursts.

As shown in Figure \ref{fig:mipicsi-hs}, a burst consists of three parts: the low-power initialization sequence, the high-speed data payload, and the end of reception sequence.

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\linewidth]{\modulefiles/figures/hs-trans.pdf}
    \caption{MIPI CSI HS Data Reception Sequence}
    \label{fig:mipicsi-hs}
\end{figure}

The receiver enters high-speed mode following the low-power sequence of [LP-11 > LP-01 > LP-00]. Synchronization is achieved through a sync sequence of ``00011101". After synchronization, the PHY receives high-speed data until a transition from LP-00 to LP-11 is detected on the lane.

\subsubsection{Escape Mode}

Escape mode allows asynchronous communication using data lanes at low speed. During Escape mode, the lane remains in low-power state. A data lane enters this mode after Escape mode request sequence [LP-11 > LP-10 > LP-00 > LP-01 > LP-00]. If an LP-11 state code is detected before the lane reaches LP-00, the request will be aborted, and the lane will return to stop state. Upon entering Escape mode, the transmitter sends an 8-bit command to indicate a requested action.

MIPI CSI only supports Ultra Low Power State (ULPS) in escape mode. The 8-bit command to enter ULPS is ``b00011110". If the entry command is not valid, it will be ignored, and the receiver will wait until the transmitter returns to the stop state.

\subparagraph{Ultra Low Power State}

Ultra Low Power State involves the lowest power consumption, except for the Shut-Down state.

For data lanes, this mode is activated by sending an Ultra Low Power State entry command ``00011110". After that, the clock lane and data lanes enter the space state (LP-00).

Figure \ref{fig:mipicsi-ulps} shows the ULPS sequences for clock and data lanes.

\subsection{MIPI RX D-PHY Configuration}
\label{sec:mipicsi-rx-dphy-config}

The MIPI RX D-PHY includes test codes for configuring MIPI RX D-PHY operation. Refer to Section \ref{sec:mipi-csi-test-code} for all supported test codes.

\subsubsection{MIPI RX D-PHY Programming Interface}
\label{sec:mipicsi-rx-dphy-program-interface}

The MIPI RX D-PHY test code programming involves two steps: first, program the test code into the registers; second, provide the test data to the tester's inputs.

Before configuring a test code, perform the following steps:

\begin{itemize}
    \item Write 0 to \hyperref[fielddesc:CSIHOSTPHYSHUTDOWNZ]{CSI\_HOST\_PHY\_SHUTDOWNZ} and \hyperref[fielddesc:CSIHOSTDPHYRSTZ]{CSI\_HOST\_DPHY\_RSTZ} to shut down and reset the PHY.
    \item It is also recommended to set \hyperref[fielddesc:CSIHOSTPHYTESTCLR]{CSI\_HOST\_PHY\_TESTCLR} to 1 to apply a reset pulse before the first configuration of any test code.
\end{itemize}

Next, configure the test code by following these steps:

\begin{itemize}
    \item Set the test code.
    
    The test code loads when \hyperref[fielddesc:CSIHOSTPHYTESTEN]{CSI\_HOST\_PHY\_TESTEN} is set to 1 and with the falling edge on \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK}.
    \begin{itemize}
        \item Ensure that \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} is set to 1.
        \item Set \hyperref[fielddesc:CSIHOSTPHYTESTDIN]{CSI\_HOST\_PHY\_TESTDIN} to the 8-bit test code. 
        \item Set \hyperref[fielddesc:CSIHOSTPHYTESTEN]{CSI\_HOST\_PHY\_TESTEN} to 1.
        \item Set \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} to 0.
        The falling edge on \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} latches the value of \hyperref[fielddesc:CSIHOSTPHYTESTDIN]{CSI\_HOST\_PHY\_TESTDIN}[7:0] as the current test code.
        \item Set \hyperref[fielddesc:CSIHOSTPHYTESTEN]{CSI\_HOST\_PHY\_TESTEN} to 0.
    \end{itemize}
    \item Enter the test data.
    
    The test data programs to the latest loaded test code when \hyperref[fielddesc:CSIHOSTPHYTESTEN]{CSI\_HOST\_PHY\_TESTEN} is 0 and with the rising edge on \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK}.
    \begin{itemize}
        \item Set \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} to 0, if not done already.
        \item Set \hyperref[fielddesc:CSIHOSTPHYTESTDIN]{CSI\_HOST\_PHY\_TESTDIN} to the 8-bit test data.
        \item Set \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} to 1. The rising edge on \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} programs the test data internally.
        \item Repeat the above steps to add more test data for the same test code.
    \end{itemize}
\end{itemize}

Set \hyperref[fielddesc:CSIHOSTPHYTESTCLR]{CSI\_HOST\_PHY\_TESTCLR} to 1 to reset a test code. This is necessary only before the first programming operation or if you wish to reset the PHY's configuration to its default value.

Figure \ref{fig:mipicsi-test} shows a timing diagram for the MIPI RX D-PHY control interface. After programming a test code, \hyperref[fielddesc:CSIHOSTPHYTESTDOUT]{CSI\_HOST\_PHY\_TESTDOUT} asynchronously outputs the relevant data associated with the specific test code. This data may include read-back information or other meaningful signals.

\begin{figure}[H]
    \centering
    \includegraphics[width=1.1\linewidth]{\modulefiles/figures/control-inf.pdf}
    \caption{Control Interface Timing Diagram}
    \label{fig:mipicsi-test}
\end{figure}

\begin{tiplisting}
Some test codes require two write data operations. In the first operation, the value of \hyperref[fielddesc:CSIHOSTPHYTESTCLK]{CSI\_HOST\_PHY\_TESTCLK} changes from 1 to 0. Ensure that the falling edge in the clock does not occur when \hyperref[fielddesc:CSIHOSTPHYTESTEN]{CSI\_HOST\_PHY\_TESTEN} is set to 1. Otherwise, the current test data will be latched as an erroneous test code.

It is highly recommended to set the test interface to an inactive state by programming the test code 0x00 when not used, as shown in the final sequence in Figure \ref{fig:mipicsi-test}.
\end{tiplisting}

\subsubsection{MIPI RX D-PHY Test Code}\label{sec:mipi-csi-test-code}
The MIPI RX D-PHY is configured through test codes via \hyperref[regdesc:CSIHOSTPHYTESTCTRL0REG]{CSI\_HOST\_PHY\_TEST\_CTRL0\_REG} and \hyperref[regdesc:CSIHOSTPHYTESTCTRL1REG]{CSI\_HOST\_PHY\_TEST\_CTRL1\_REG}. Refer to Section \ref{sec:mipicsi-rx-dphy-program-interface} \textit{\nameref{sec:mipicsi-rx-dphy-program-interface}} for more details.

\subparagraph{Normal Operation}

Use the following test code to enable normal operation of the PHY and the wake-up state of the test interface. In this mode, the test interface is inactive.

\begin{itemize}
    \item \textbf{Test Code}: 0x00
    \item \textbf{Test Data}: None
\end{itemize}

\subparagraph{HS RX Control of Lane 0}
\label{sec:mipicsi-test-code-0x44}

Use the following test code to select the HS operating frequency range (hsfreqrange).

\subsection{Descrambler}

Data scrambling uses randomization techniques to mitigate electromagnetic interference (EMI) and radio frequency (RF) self-interference. This spreads the transmission energy of the link across a wider frequency band.

The transmitted data is scrambled with a Pseudo-Random-Bit-Stream (PRBS) to reduce the likelihood of repetitive patterns on a lane basis. The PRBS is generated using a Linear Feedback Shift Register (LFSR) that implements the generator polynomial:

\begin{equation}
    G(x) = x^{16} + x^5 + x^4 + x^3 + 1
\end{equation}

Each data lane has an instance of the de-scrambling block. Set \hyperref[fielddesc:CSIHOSTSCRAMBLEENABLE]{CSI\_HOST\_SCRAMBLE\_ENABLE} to 1 to enable this block. Each lane can be configured with a different 16-bit LFSR seed value via \hyperref[fielddesc:CSIHOSTSCRAMBLESEEDLANEN]{CSI\_HOST\_SCRAMBLE\_SEED\_LANE\regindex{n}}.

The default seed values after resetting the controller are listed in \ref{tab:mipicsi-prbs-seed}.

\section{Interrupts}
\label{sec:mipicsi-int}


\chipname{}'s MIPI CSI can generate the CSI\_INTR interrupt signal that will be sent to the \hyperref[mod:intmtrx]{Interrupt Matrix}.

The CSI\_HOST\_INT\_ST\_<group> registers report error conditions and trigger the CSI\_INTR interrupt signal.

You can mask the interrupt pin by using the CSI\_HOST\_INT\_MSK\_<group> registers. All errors are masked by default. Set any bit of these registers to 1 to enable the specific error interrupt. The error asserts the respective bit in the CSI\_HOST\_INT\_ST\_<group> register. The CSI\_HOST\_INT\_ST\_<group> registers clear after a read operation.

Setting any bit of the Interrupt Force registers (CSI\_HOST\_INT\_FORCE\_<group>) to 1 triggers the corresponding interrupt without needing to activate the error conditions. These registers are for test purposes.

Figure\ref{fig:mipicsi-intr} shows the interrupt mechanism:
\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\linewidth]{\modulefiles/figures/mipicsi_intr_20240808.pdf}
    \caption{MIPI CSI Interrupt Mechanism}
    \label{fig:mipicsi-intr}
\end{figure}

The interrupt signal CSI\_INTR can be generated by the following internal interrupt sources (see Table \ref{tab:mipicsi-error} \textit{\nameref{tab:mipicsi-error}} for details interrupt source description):
\begin{itemize}
    \item PHY Fatal Interrupt:
    \begin{itemize}
        \item PHY\_ERRSOTSYNCHS\_\regindex{n} (\regindex{n}: 0-1)
    \end{itemize}
    \item Packet Fatal Interrupt:
    \begin{itemize}
        \item ERR\_ECC\_DOUBLE
        \item SHORTER\_PAYLOAD
    \end{itemize}    
    \item Boundary Frame Fatal Interrupt:
    \begin{itemize}
        \item ERR\_F\_BNDRY\_MATCH\_VC\regindex{n} (\regindex{n}: 0-15)
    \end{itemize}        
    \item Sequence Frame Fatal Interrupt:
    \begin{itemize}
        \item ERR\_F\_SEQ\_VC\regindex{n} (\regindex{n}: 0-15)
    \end{itemize}   
    \item CRC Frame Fatal Interrupt:
    \begin{itemize}
        \item ERR\_FRAME\_DATA\_VC\regindex{n} (\regindex{n}: 0-15)
    \end{itemize}      
    \item Payload CRC Fatal Interrupt:
    \begin{itemize}
        \item ERR\_CRC\_VC\regindex{n} (\regindex{n}: 0-15)
    \end{itemize}          
    \item Data ID Interrupt:
    \begin{itemize}
        \item ERR\_ID\_VC\regindex{n} (\regindex{n}: 0-15)
    \end{itemize}         
    \item ECC Corrected Interrupt:
    \begin{itemize}
        \item ERR\_ECC\_CORRECTED\_VC\regindex{n} (\regindex{n}: 0-15)
    \end{itemize}

\section{Programming Procedures}
\label{sec:mipicsi-prog-procedures}

This section describes how to configure the MIPI RX D-PHY and the MIPI CSI-2 host for image data reception. MIPI CSI receives image data only in high-speed data reception mode.

\subsection{Start High-Speed Data Reception}

\subsubsection{Clock and Reset}
\begin{itemize}
    \item Set \hyperref[fielddesc:HPSYSCLKRSTCSIHOSTSYSCLKEN]{HP\_SYS\_CLKRST\_CSI\_HOST\_SYS\_CLK\_EN} to 1 to enable the MIPI CSI clock.
    \item Set \hyperref[fielddesc:HPSYSCLKRSTRSTENCSIHOST]{HP\_SYS\_CLKRST\_RST\_EN\_CSI\_HOST} to 0 to release the MIPI CSI reset.
    \item Configure \hyperref[fielddesc:HPSYSCLKRSTMIPICSIDPHYCLKSRCSEL]{HP\_SYS\_CLKRST\_MIPI\_CSI\_DPHY\_CLK\_SRC\_SEL} to select a proper clock source for the MIPI RX D-PHY configuration clock.
    \item Set \hyperref[fielddesc:HPSYSCLKRSTMIPICSIDPHYCFGCLKEN]{HP\_SYS\_CLKRST\_MIPI\_CSI\_DPHY\_CFG\_CLK\_EN} to 1 to enable the MIPI RX D-PHY configuration clock.
    \item Set \hyperref[fielddesc:HPSYSCLKRSTRSTENCSIBRG]{HP\_SYS\_CLKRST\_RST\_EN\_CSI\_BRG} to 1 to release the CSI\_Bridge module of the ISP and reset the ISP.
\end{itemize}
\subsubsection{MIPI RX D-PHY and MIPI Host Initialization}
\begin{itemize}
    \item Clear \hyperref[fielddesc:CSIHOSTCSI2RESETN]{CSI\_HOST\_CSI2\_RESETN} to reset the CSI\_HOST controller logic.
    \item Set \hyperref[fielddesc:CSIHOSTPHYTESTCLR]{CSI\_HOST\_PHY\_TESTCLR} to 1, then clear \hyperref[fielddesc:CSIHOSTPHYTESTCLR]{CSI\_HOST\_PHY\_TESTCLR} to reset the test code logic.
    \item Configure the MIPI RX D-PHY frequency range in test code 0x44 using the MIPI RX D-PHY programming interface.
    \item Configure \hyperref[fielddesc:CSIHOSTNLANES]{CSI\_HOST\_N\_LANES} to define the number of active lanes in the system.
    \item Set \hyperref[fielddesc:CSIHOSTPHYSHUTDOWNZ] {CSI\_HOST\_PHY\_SHUTDOWNZ} and \hyperref[fielddesc:CSIHOSTDPHYRSTZ]{CSI\_HOST\_DPHY\_RSTZ} to 1 to release the MIPI RX D-PHY from the reset state.
    \item Set the corresponding bit in the CSI\_HOST\_INT\_MSK\_*\_REG register to 1 to enable the error interrupt.
    \item Configure the descrambler function if needed.
    \item Set \hyperref[fielddesc:CSIHOSTCSI2RESETN]{CSI\_HOST\_CSI2\_RESETN} to 1 to release the CSI\_HOST reset.
    \item Wait until \hyperref[fielddesc:CSIHOSTPHYSTOPSTATEDATA0]{CSI\_HOST\_PHY\_STOPSTATEDATA\_\regindex{n}} and \hyperref[fielddesc:CSIHOSTPHYSTOPSTATECLK]{CSI\_HOST\_PHY\_STOPSTATECLK} are 1.
    \item Start the camera image capture.
\end{itemize}

\subsection{Stop High-Speed Data Reception}

\begin{itemize}
    \item Disable related channels in . Wait until  is disabled. Refer to Chapter \ref{mod:vdma} \textit{\nameref{mod:vdma}}.
    \item Reset the ISP. Refer to Chapter \ref{mod:isp} \textit{\nameref{mod:isp}}.
    \item Set \hyperref[fielddesc:HPSYSCLKRSTRSTENCSIHOST]{HP\_SYS\_CLKRST\_RST\_EN\_CSI\_HOST} to 1 to reset the MIPI CSI.
\end{itemize}
