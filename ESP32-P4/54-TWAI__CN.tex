\chapter{双线汽车接口 (TWAI)}
\label{mod:twai}

双线汽车接口 (Two-wire Automotive Interface, TWAI$^\circledR$) 协议是一种多主机、多播的通信协议，具有错误检测、发送错误信号以及内置报文优先仲裁等功能。TWAI 协议适用于汽车和工业应用（参见章节 \ref{sec:twai-functional-protocol}）。

\chipname{} 包含三个 TWAI 控制器，即 TWAI 0、TWAI 1 和 TWAI 2，任意控制器都可通过外部收发器连接到 TWAI 总线。TWAI 控制器包含一系列先进的功能，用途广泛，可用于如汽车产品、工业自动化控制、楼宇自动化等。

\section{主要特性}
\chipname{} 单个 TWAI 控制器具有以下特性：
\begin{itemize}
    \item 兼容 ISO 11898-1 协议（CAN 规范 2.0）
    \item 支持标准格式（11 位标识符）和扩展格式（29 位标识符）两种帧格式
    \item 支持 1 Kbit/s \verb+~+ 1 Mbit/s 位速率
    \item 支持多种操作模式：
    \begin{itemize}
        \item 正常模式
        \item 只听模式（不影响总线）
        \item 自测模式（发送数据时无需应答）
    \end{itemize}
    \item 配置 64 字节接收 FIFO
    \item 支持特殊发送：
    \begin{itemize}
        \item 单次发送（发生错误时不会自动重新发送）
        \item 自发自收（TWAI 控制器同时发送和接收报文）
    \end{itemize}
    \item 配置数据接收过滤器（支持单过滤器和双过滤器模式）
    \item 支持错误检测与处理
    \begin{itemize}
        \item 配置错误计数器
        \item 可配置错误报警阈值
        \item 内置错误代码记录
        \item 内置仲裁丢失记录
    \end{itemize}
    \begin{itemize}
        \item 支持收发器自动待机功能
    \end{itemize}
\end{itemize}

\section{协议概述}
\label{sec:twai-functional-protocol}
\subsection{TWAI 特性}
TWAI 协议连接网络中的两个或多个节点，并允许各节点以确定性延迟的方式进行报文交互。TWAI 总线具有以下特性：

{\bfseries 单通道通信与不归零编码：} TWAI 总线只有一根传输线进行单通道通信，为半双工通信。同步调整也在单通道中进行，无需其他通道（如时钟通道和使能通道）。TWAI 上报文的位流采用不归零编码 (NRZ) 方式。

{\bfseries 位值：} 单通道可处于显性状态或隐性状态，显性状态的逻辑值为 0，隐性状态的逻辑值为 1。发送显性状态数据的节点优先级高于发送隐性状态数据的节点。总线的物理实现（如使用差分对或者单端信号）由其应用决定。

{\bfseries 位填充：} TWAI 报文的某些域已经过位填充。发送器在发送连续五个位的相同值（如显性数值或隐性数值）后，会自动插入一个互补位。同理，在接收五个相同值的连续位后，接收器应将下一个位视为填充位。位填充应用于以下域：SOF、仲裁域、控制域、数据域和 CRC 序列（参见章节 \ref{sec:twai-messages}）。

{\bfseries 多播：} 当各节点连接到同个总线上时，这些节点都将接收到相同的位。如无总线错误（参见章节 \ref{sec:twai-errors}），各节点上的数据将保持一致。

{\bfseries 多主机：} 任意节点都可发起数据传输。如果当前已有正在进行的数据传输，则节点将在当前传输结束后再发起新的数据传输。

{\bfseries 报文优先级与仲裁：} 若两个或多个节点同时发起数据传输，TWAI 协议将确保其中一个节点获得总线的优先仲裁权。优先仲裁权由各节点所发送报文的仲裁域决定。

{\bfseries 错误检测：} 各节点将积极检测总线上的错误，并向 TWAI 总线发送错误帧来广播检测到的错误。

{\bfseries 故障限制：} 各节点都配置有错误计数器，计数器数量依据 TWAI 协议规则增加或减少。当错误计数超过一定阈值时，对应节点将自动关闭并退出网络。

{\bfseries 可配置位速率：} 一个 TWAI 总线的位速率是可配置的。但是，同一总线中的所有节点须以相同位速率工作。

{\bfseries 发送器与接收器：} 不论何时，任意 TWAI 节点都可作为发送器和接收器。
\begin{itemize}
    \item 产生报文的节点即为发送器。在总线空闲或该节点失去仲裁优先权之前，该节点将一直作为发送器。请注意，仲裁期间存在多个作为发送器的节点。
    \item 所有不是发送器的节点都可作为接收器。
\end{itemize}

\subsection{TWAI 报文}
\label{sec:twai-messages}
TWAI 节点使用报文发送数据，并在监测到总线上存在错误时，向其他节点发送错误信号。报文有多种帧类型，不同的帧类型具有不同的帧格式。

TWAI 协议支持以下帧类型：

\begin{itemize}
    \item 数据帧
    \item 远程帧
    \item 错误帧
    \item 过载帧
    \item 帧间距
\end{itemize}

TWAI 协议支持以下帧格式：

\begin{itemize}
    \item 标准格式 (SFF)，由 11 位标识符组成
    \item 扩展格式 (EFF)，由 29 位标识符组成
\end{itemize}

\subsubsection{数据帧和远程帧}
节点使用数据帧向其他节点发送数据，负载可为 0 \verb+~+ 8 字节数据。节点使用远程帧向其他节点请求具有相同标识符的数据帧，因此，远程帧中不包含任何数据字节。但是，数据帧和远程帧中包含许多相同域。下图 \ref{fig:twai-bit-fields} 所示为不同帧类型和不同帧格式中包含的域和子域。

{\bfseries 仲裁域}\\
当两个或多个节点同时发送数据帧和远程帧时，将根据仲裁域的位信息来决定总线上获得仲裁优先权的节点。在发送仲裁域位信息时，如果一个节点在发送隐性位的同时检测到了一个显性位，表示有其他节点覆盖了这个隐性位。那么，这个发送隐性位的节点将丢失总线仲裁，应立即转为接收器。

仲裁域主要由帧标识符组成，并且由最高位开始发送。根据显性位代表的逻辑值为 0，隐性位代表的逻辑值为 1，有以下规律:

\begin{itemize}
    \item ID 值最小的帧将总是获得仲裁（逻辑 0 是显性位值）。
    \item 如果 ID 数值相同，由于数据帧的 \hyperref[table_item:canframe_rtr]{RTR} 位为显性位，数据帧将优先于远程帧。
    \item 如果 ID 的前 11 位相同，由于扩展帧的 \hyperref[table_item:canframe_srr]{SRR} 位为隐形位，标准格式帧将优先于扩展格式帧。
\end{itemize}

{\bfseries 控制域}\\
控制域主要由数据长度代码 (DLC) 组成。DLC 表示一个数据帧中负载的数据字节长度，或一个远程帧请求的数据字节长度。DLC 从最高位开始发送。

{\bfseries 数据域}\\
数据域中包含一个数据帧实际负载的数据字节。远程帧中不包含数据域。

{\bfseries CRC 域}\\
CRC 域主要由 CRC 序列组成。CRC 序列是一个 15 位的循环冗余校验编码，根据数据帧或远程帧中位填充前的内容（从 SOF 到数据域末尾的所有部分）计算而来。

{\bfseries 确认域}\\
确认 (ACK) 域由确认槽和确认分界符组成，主要功能在于接收器向发送器报告已正确接收到有效报文。

\subsubsection{错误帧和过载帧}

{\bfseries 错误帧}

当某节点检测到总线错误时，将发送一个错误帧。错误帧由特殊的错误标志构成，该标志由某相同值的六个连续位组成，因而违反了位填充的规则。所以，当某节点检测到总线错误并发送错误帧时，其余节点也将相应地检测到一个填充错误并各自发送错误帧。也就是说，当发生总线错误时，通过上述过程，可将该报文传递至总线上的所有节点。


当某节点检测到总线错误时，该节点将于下一个位发送错误帧。不过，当总线错误类型为 CRC 错误时，错误帧将从确认分界符的下一个位开始（参见章节 \ref{sec:twai-errors}）。下图 \ref{fig:twai-fields-of-an-error-frame} 所示为一个错误帧所包含的不同域：

下列情况将触发发送过载帧：
\begin{enumerate}
    \item 接收器内部要求延迟发送下一个数据帧或远程帧。
    \item 在\hyperref[table_item:ifg]{间歇域}中的首个和第二个位上检测到显性位。
    \item 如果在错误分界符的第八个（最后一个）位上检测到显性位。请注意，在这种情况下，TEC 和 REC 的值不会增加（参见章节 \ref{sec:twai-errors}）。

\begin{itemize}

        \item 第 2、3 条情况下，发送的过载帧须从检测到显性位的后一个位开始。
\end{itemize}

\subsection{TWAI 错误}
\label{sec:twai-errors}
\subsubsection{错误类型}

TWAI 中的总线错误包括以下类型：

{\bfseries 位错误}\\
当节点发送一个位值（显性位或隐性位）但检测到相反的位时（如发送显性位时检测到了隐性位），就会发生位错误。但是，如果发送的位是隐性位，且位于仲裁域、确认槽或被动错误标志中，那么此时检测到显性位则不会认定为位错误。

{\bfseries 填充错误}\\
当检测到相同值的六个连续位时（违反位填充的编码规则），发生填充错误。

{\bfseries CRC 错误}\\
接收器将根据接收到的数据帧和远程帧的有效位（位填充前）计算 CRC 值。当接收器计算的值与接收到的数据帧和远程帧中的 CRC 序列不匹配时，会发生 CRC 错误。


{\bfseries 格式错误}\\
当某个报文中的固定格式位中包含非法位时，可检测到格式错误。比如，r1 和 r0 域必须固定为显性。

{\bfseries 确认错误}\\
当发送器无法在确认槽中检测到显性位时，会发生确认错误。

\subsubsection{错误状态}
每个节点 TWAI 控制器通过维护两个错误计数器来实现故障界定，计数数值决定错误状态。这两个错误计数器分别为：发送错误计数 (TEC) 和接收错误计数 (REC)。TWAI 包含以下错误状态：

{\bfseries 主动错误}\label{table_item:active_err}\\
处于主动错误状态的节点可参与到总线交互中，且在检测到错误时发送主动错误标志。

{\bfseries 被动错误}\label{table_item:passive_err}\\
处于被动错误状态的节点可参与到总线交互中，且在检测到错误时发送被动错误标志。被动错误节点发送数据帧或远程帧后，需在后续的帧间距中增加挂起传送域。

{\bfseries 离线}\\
禁止处于离线状态的节点以任意方式干扰总线（如不允许离线节点进行数据传输）。

\subsubsection{错误状态跳转}

\begin{enumerate}
    \item 当一个节点的 TEC 和/或 REC 数值大于等于 128 时，该节点变为被动错误节点。对于导致节点发生上述状态切换的错误，该节点仍发送主动错误标志。请注意，一旦 REC 数值到达 128，后续任何增加该值的动作都是无效的，直到 REC 数值返回到 128 以下。
    \item 当某节点的 TEC 数值大于等于 256 时，该节点进入离线状态。
    \item 当某被动错误节点的 TEC 和 REC 数值都小于等于 127 时，该节点变为主动错误节点。
    \item 当离线节点在总线上检测到 128 次 11 个连续隐性位后，该节点可变为主动错误节点（TEC 和 REC 数值都重设为 0）。
\end{enumerate}

\subsubsection{错误计数规则}

TEC 和 REC 根据以下规则递增/递减。\textbf{请注意，一条报文传输中可应用多个规则。}


\textbf{TEC 的递增/递减规则：}
\begin{enumerate}
    \item 当发送器发送错误标志时，TEC 数值增加 8。但是，以下情况不适用于该规则：
    \begin{itemize}
        \item 发送器处于被动错误状态，在检测到应答错误且发送被动错误标志时检测不到显性位时，TEC 数值不应增加。
        \item 发送器在仲裁期间因填充错误而发送错误标志，且填充位本该是隐性位但是检测到显性位，TEC 数值不应增加。
    \end{itemize}
    \item 若发送器在发送主动错误标志和过载标志时检测到位错误，TEC 数值增加 8。
    \item 任意节点在发送主动/被动错误标志或过载标志后，节点仅能承载最多 7 个连续显性位。在发送主动错误标志或过载标志时检测到第 14 个连续显性位，或在被动错误标志后检测到第 8 个连续显性位后，发送器的 TEC 数值增加 8，而接收器的 REC 数值增加 8。每一附加的 8 个连续显性位都会使发送器的 TEC 和接收器的 REC 数值增加 8。
    \item 每当发送器成功发送报文后，即接收到 ACK，且直到 EOF 完成都未发生错误时，TEC 数值减小 1，除非 TEC 的数值已经为 0。
\end{enumerate}

\textbf{REC 的递增/递减规则：}
\begin{enumerate}
    \item 发送错误标志后，当接收器第一个检测到的位是显性位时，REC 数值增加 8。
    \item 当接收器检测到错误时，REC 数值增加 1，检测到的错误为发送主动错误标志或过载标志期间的位错误除外。
    \item 若接收器在发送主动错误标志和过载标志时检测到位错误，REC 数值增加 8。
    \item 当接收器成功接收报文后，即确认槽前未检测到错误，且已成功发送 ACK 时，REC 数值相应减小。
    \begin{itemize}
        \item 若 REC 数值位于 1 \verb+~+ 127 之间，则其值减小 1。
        \item 若 REC 数值大于 127，则其值减小到 127。
        \item 若 REC 数值为 0，则仍保持为 0。
    \end{itemize}
\end{enumerate}

\subsection{TWAI 位时序}
\subsubsection{标称位}

TWAI 协议允许 TWAI 总线以特定的位速率运行。但是，总线内的所有节点必须以统一位速率运行。

\begin{itemize}
    \item \textbf{标称位速率} 为每秒发送比特数量。
    \item \textbf{标称位时间} 为 \textbf{1/标称位速率}。
\end{itemize}

每个标称位时间中含多个段，每段由多个时间定额 (Time Quanta) 组成。\textbf{时间定额} 为最小时间单位，作为一种预分频时钟信号应用于各个节点中。下图 \ref{fig:twai-bit-timing} 所示为一个标称位时间内所包含的段。

TWAI 控制器将在以一个时间定额的步长内进行操作，每个时间定额中都会分析 TWAI 的总线状态。如果两个连续的时间定额中总线状态不同（隐性-显性，或反之），意味着有边沿产生。PBS1 和 PBS2 的交点将被视为采样点，采样的总线状态即为这个位的数值。

\subsubsection{硬同步与再同步}


由于时钟偏移和抖动，同一总线上节点的位时序可能会脱离相位段。因而，位边沿可能会偏移到同步段的前后。针对上述位边沿偏移的问题，TWAI 提供多种同步方式。设发生相位误差时位边沿偏移的时间定额数量为\textbf{“e”}，该值与 SS 相关。

\begin{itemize}
    \item 主动相位误差 (e > 0)：位边沿位于同步段之后采样点之前，即边沿向后偏移。
    \item 被动相位误差 (e < 0)：位边沿位于前个位的采样点之后同步段之前，即边沿向前偏移。
\end{itemize}

为解决相位误差，有两种同步方式，\textbf{硬同步}与\textbf{再同步}。\textbf{硬同步}与\textbf{再同步}遵守以下规则：

\begin{itemize}
    \item 单个位时序中仅可执行一次同步。
    \item 同步仅可发生在隐性位到显性位的边沿上。
\end{itemize}

{\bfseries 硬同步}\\
总线空闲期间，硬同步发生在隐性位到显性位的变化边沿上（如总线空闲后的第一个 SOF 位）。此时，所有节点都将重启其内部位时序，从而使该变化边沿位于重启位时序的同步段内。

{\bfseries 再同步}\\
非总线空闲期间，再同步发生在隐性位到显性位的变化边沿上。如果边沿上有主动相位误差 (e > 0)，则增加 PBS1 段的长度。如果边沿上有被动相位误差 (e < 0)，则减小 PBS2 段的长度。

PBS1/PBS2 具体增加和减小的时间定额取决于相位误差的绝对值，同时也受可配置的同步跳宽 (SJW) 数值限制。

\begin{itemize}
    \item 当相位误差的绝对值小于等于 SJW 数值时，PBS1/PBS2 将增加/减小 \textbf{e} 个时间定额。该过程与硬同步具有相同效果。
    \item 当相位误差的绝对值大于 SJW 数值时，PBS1/PBS2 将增加/减小与 SJW 相同数值的时间定额。这意味着，在完全解决相位误差之前，可能需要执行多次位的再同步。
\end{itemize}

\subsection{位时序逻辑}
位时序逻辑 (BTL) 模块负责以预先配置的位速率发送和接受报文。BTL 模块还负责同步位时序，确保数据传输的稳定性。位速率由多个可编程的段组成，且用户可设置每个段的时间定额长度，来调整传播延迟、控制器处理时间等因素。

\subsection{位流处理器}
位流处理 (BSP) 模块负责对发送缓冲器的数据进行帧处理（如位填充和附加 CRC 域），并为位时序逻辑 (BTL) 模块生成位流。同时，BSP 模块还负责处理从 BTL 模块中接收的位流（如去填充和验证 CRC），并将处理后报文写入接收 FIFO。BSP 还负责检测 TWAI 总线上的错误并将此类错误报告给错误管理逻辑 (EML)。

\subsection{接收过滤器}
接收过滤器是一个可编程的报文过滤单元，允许 TWAI 控制器根据报文的标识符域接收或拒绝该报文。通过接
收过滤器的报文才能被存储到接收 FIFO 中。用户可配置接收过滤器的模式：单过滤器、双过滤器。

\subsection{接收 FIFO}
接收 FIFO 位于 TWAI 控制器内部，是大小为 64 字节的缓冲器，负责存储通过接收过滤器的接收报文。接收 FIFO 中存储的报文大小可以不同（范围在 3 \verb+~+ 13 字节之间）。当接收 FIFO 为满或剩余空间不足以完全存储下当前的接收报文时，将触发溢出中断，后续的接收报文将丢失，直到接收 FIFO 中清除出足够的存储空间。接收 FIFO 中的第一条报文将被映射到 13 字节的接收缓冲器中，直到该报文被清除（通过释放接收缓冲器指令）。清除后，接收 FIFO 将释放上一条已清除报文的空间，同时将窗口映射到接收 FIFO 中的下一条报文。


\subsection{错误管理逻辑}
错误管理逻辑 (EML) 模块负责更新 TEC 和 REC 数值、记录错误信息（如错误类型和错误位置）、更新控制器的错误状态，确保 BSP 模块发送正确的错误标志。此外，该模块还负责记录 TWAI 控制器丢失仲裁时的 bit 位置。

\subsection{寄存器模块}
\chipname{} 的 CPU 使用 32 位字对齐地址访问外设。但对于 TWAI 控制器中的大部分寄存器，仅最低有效字节 (bits [7:0]) 数据有效。在这些寄存器中，bits [31:8] 数值在写入时被忽略，在读取时返回 0。

{\bfseries 配置寄存器}\\
配置寄存器存储 TWAI 控制器的各配置项，如位速率、操作模式、接收过滤器等。只有在 TWAI 控制器处于复位模式时，才可修改配置寄存器（参见章节 \ref{sec:twai-modes}）。

{\bfseries 指令寄存器}\\
CPU 通过指令寄存器驱动 TWAI 控制器执行任务，如发送报文或清除接收缓冲器。只有在 TWAI 控制器处于操作模式时，才可修改指令寄存器（参见章节 \ref{sec:twai-modes}）。

{\bfseries 中断 \verb+&+ 状态寄存器}\\
中断寄存器显示 TWAI 控制器中发生的事件（每个事件由一个单独的位表示）。状态寄存器显示 TWAI 控制器的当前状态。

{\bfseries 错误管理寄存器}\\
错误管理寄存器包括错误计数和捕捉寄存器。错误计数寄存器表示 TEC 和 REC 的数值。捕捉寄存器负责记录相关信息，如 TWAI 控制器在何处检测到总线错误或何时丢失仲裁。

{\bfseries 发送缓冲寄存器}\\
发送缓冲器大小为 13 字节，用于存储 TWAI 的待发送报文。

{\bfseries 接收缓冲寄存器}\\
接收缓冲器大小为 13 字节，用于存储单个报文。接收缓冲器是读取接收 FIFO 的窗口，接收 FIFO 中的第一个报文将被映射到接收缓冲器中。

请注意，发送缓冲寄存器、接收缓冲寄存器和接收过滤配置寄存器共享同一段地址偏移 0x0040 \verb+~+ 0x0070。CPU 访问上述地址范围内寄存器遵循以下规则：

\begin{itemize}
    \item 当 TWAI 控制器处于复位模式时，该地址范围被映射到接收过滤配置寄存器。
    \item 当 TWAI 控制器处于操作模式时：
    \begin{itemize}
        \item 对地址范围的所有读取都映射到接收缓冲寄存器中。
        \item 对地址范围的所有写入都映射到发送缓冲寄存器中。
    \end{itemize}
\end{itemize}




\section{功能描述}

\subsection{模式}
\label{sec:twai-modes}
\chipname{} TWAI 控制器有两种工作模式：复位模式和操作模式。将 \hyperref[fielddesc:TWAIRESETMODE]{TWAI\_RESET\_MODE} 位置 1，进入复位模式；置 0，进入操作模式。

\subsubsection{复位模式}
要修改 TWAI 控制器的各种配置寄存器，需进入复位模式。进入复位模式时，TWAI 控制器彻底与 TWAI 总线断开连接。复位模式下，TWAI 控制器将无法发送任何报文（包括错误信号）。任何正在进行的报文传输将立即被终止。同样的，TWAI 控制器在该模式下也将无法接收任何报文。

\subsubsection{操作模式}
进入操作模式后，TWAI 控制器与总线相连，并且写保护各配置寄存器，以确保控制器的配置在运行期间保持一致。操作模式下，TWAI 控制器可以发送和接收报文（包括错误信号），但具体取决于 TWAI 控制器配置于哪种运行子模式。TWAI 控制器支持以下三种子模式：

\begin{itemize}
    \item {\bfseries 正常模式：} TWAI 控制器可以发送和接收包含错误信号在内的报文（如，错误帧和过载帧）。
    \item {\bfseries 自测模式：} 与正常模式相同，但在该模式下，TWAI 控制器发送报文时，即使在 CRC 域之后没有接收到应答信号，也不会产生应答错误。通常在 单个 TWAI 控制器自测时使用该模式。
    \item {\bfseries 只听模式：} TWAI 控制器可以接收报文，但在 TWAI 总线上保持完全被动。因此，TWAI 控制器将无法发送任何报文、应答或错误信号。错误计数寄存器将保持冻结状态。该模式用于 TWAI 总线监控。
\end{itemize}

请注意，退出复位模式后（如，进入操作模式时），TWAI 控制器需等待 11 个连续隐性位出现，才能完全连接上 TWAI 总线（即，可以发送或接收报文）。

\subsubsection{帧数据}
帧数据域包含发送或接收的数据帧，范围为 0 \verb+~+ 8 bytes。其中的有效字节数应与 DLC 相同。但是，如果 DLC 数值大于 8，则帧数据域的有效字节数仍为 8。远程帧中不包含数据载荷，因此不存在帧数据域。

比如，当发送 5 个字节的数据帧时，CPU 应在 DLC 域中写入数值 5，并将数据写入数据域 1 \verb+~+ 5 字节对应的寄存器。同样，当接收 DLC 为 5 的数据帧时，只有 1 \verb+~+ 5 数据字节中包含 CPU 可以读取的有效载荷数据。

\subsection{接收 FIFO 和数据溢出}

接收 FIFO 是一个 64-byte 的内部缓冲器，用于以先进先出的原则存储接收到的报文。一条接收报文可在接收 FIFO 中占 3 \verb+~+ 13 bytes 空间，且其中字节序与接收缓冲器的寄存器地址顺序相同。接收缓冲寄存器将被映射到接收 FIFO 中第一条报文。


当 TWAI 控制器接收到第一条报文时，\hyperref[fielddesc:TWAIRXMESSAGECOUNTER]{TWAI\_RX\_MESSAGE\_COUNTER} 的值将增至 1，RXI 中断开始有效，第一条报文会自动更新到缓冲器中。后续每当 TWAI 控制器接收到新的报文，\hyperref[fielddesc:TWAIRXMESSAGECOUNTER]{TWAI\_RX\_MESSAGE\_COUNTER} 的值会增加 1，直至最大值 64，此时如果接收 FIFO 中有足够的剩余空间，报文内容将被写入到接收 FIFO 中。

软件读取接收缓冲器中的消息后，通过将 \hyperref[fielddesc:TWAIRELEASEBUFFER]{TWAI\_RELEASE\_BUFFER} 的位置 1，释放接收 FIFO 中此报文所占的空间，\hyperref[fielddesc:TWAIRXMESSAGECOUNTER]{TWAI\_RX\_MESSAGE\_COUNTER} 的值也将减小 1。然后，接收缓冲器将映射接收 FIFO 中的下一条报文。软件应重复以上操作直至 \hyperref[fielddesc:TWAIRXMESSAGECOUNTER]{TWAI\_RX\_MESSAGE\_COUNTER} 为 0。此时， RXI 中断停止触发，接收缓冲器中的数据是无效的，\hyperref[fielddesc:TWAISTATUSRECEIVEBUFFER]{TWAI\_STATUS\_RECEIVE\_BUFFER} 为 0。

当 TWAI 控制器接收到一条报文，但接收 FIFO 没有足够空间完整地存储这条接收报文时（不论是因为报文内容大小大于接收 FIFO 中的空闲空间，还是因为接收 FIFO 已满），便会发生数据溢出。

数据溢出发生时：
\begin{itemize}
    \item 接收 FIFO 中剩余的空间将填满溢出报文的内容。如果接收 FIFO 已满，则无法存储溢出报文的任何内容。
    \item 接收 FIFO 首次发生数据溢出时，将触发数据溢出中断。
    \item 溢出报文仍将增加 \hyperref[fielddesc:TWAIRXMESSAGECOUNTER]{TWAI\_RX\_MESSAGE\_COUNTER} 的值，直到 \hyperref[fielddesc:TWAIRXMESSAGECOUNTER]{TWAI\_RX\_MESSAGE\_COUNTER} 达到最大值 64。
    \item 接收 FIFO 将在内部将溢出报文标记为无效。可使用 \hyperref[fielddesc:TWAISTATUSMISS]{TWAI\_STATUS\_MISS} 位，确认目前接收缓冲器映射的报文是有效报文还是溢出报文。
\end{itemize}

为了清除接收 FIFO 中的溢出报文，应重复调用 \hyperref[fielddesc:TWAIRELEASEBUFFER]{TWAI\_RELEASE\_BUFFER}，直到 \hyperref[fielddesc:TWAIRXMESSAGECOUNTER]{TWAI\_RX\_MESSAGE\_COUNTER} 为 0。这就要求用户读取接收 FIFO 中的所有有效报文，并清除所有溢出报文。


\subsection{接收过滤器}

接收过滤器允许 TWAI 控制器根据报文 ID 过滤接收报文（甚至可以过滤报文的第一个数据字节和帧类型）。只有通过过滤的报文才能存储到接收 FIFO 中。接收过滤器的使用可以一定程度地减轻 TWAI 控制器的运行负荷（如，可减少使用接收 FIFO 和发生接收中断的次数），因为 TWAI 控制器将只需要操作过滤后的一小部分报文。

接收过滤的配置寄存器和发送/接收缓冲寄存器使用相同的访问地址空间，只有当 TWAI 控制器处于复位模式时，该地址段才被映射到接收过滤器的配置寄存器。

接收过滤器的配置寄存器由 32 位的 Code 值和 32 位的 Mask 值组成。Code 值将指定一种位排列模式，每条过滤报文中的位都必须匹配该模式，才能使该报文通过过滤。Mask 值可屏蔽 Code 值中的某些位（屏蔽位将被设置为“不相关”位）。如图 \ref{fig:twai-acceptance-filter} 所示，为了使报文通过过滤，每条过滤报文的 ID 位都必须匹配 Code 值所设模式或者被 Mask 值屏蔽。

\subsection{错误管理}

TWAI 协议要求每个 TWAI 节点中都包含发送错误计数器 (TEC) 和接收错误计数器 (REC)。这两个错误计数的数值决定了 TWAI 控制器当前的错误状态（如，主动错误、被动错误、离线）。TWAI 控制器将 TEC 和 REC 的数值分别存储在 \hyperref[regdesc:TWAITXERRCNTREG]{TWAI\_TX\_ERR\_CNT\_REG} 和 \hyperref[regdesc:TWAIRXERRCNTREG]{TWAI\_RX\_ERR\_CNT\_REG} 中，CPU 可随时进行读取。除了错误状态之外，TWAI 控制器还提供错误报警限制 (EWL) 的功能，这个功能可在 TWAI 控制器进入被动错误状态之前，提醒用户当前发生的严重总线错误。

TWAI 控制器的当前错误状态通过以下各数值和状态位体现，即：TEC、REC、\hyperref[fielddesc:TWAISTATUSERR]{TWAI\_STATUS\_ERR} 和 \hyperref[fielddesc:TWAISTATUSNODEBUSOFF]{TWAI\_STATUS\_NODE\_BUS\_OFF}。这些数值和状态位的变化也将触发中断，从而提醒用户当前的错误状态变化（可参见第 \ref{sec:twai-interrupt-management} 章）。下图 \ref{fig:twai-error-state-transition} 所示为错误状态、上述数值和状态位以及错误状态相关中断之间的关系。

\subsubsection{错误报警限制}

错误报警限制 (EWL) 为 TEC 和 REC 的可配置阈值，若错误计数数值超过该阈值，将触发 EWI 中断。EWL 将作为一个报警功能提示当前发生的严重 TWAI 总线错误，且在 TWAI 控制器进入被动错误状态之前被触发。EWL 数值应在寄存器 \hyperref[regdesc:TWAIERRWARNINGLIMITREG]{TWAI\_ERR\_WARNING\_LIMIT\_REG} 中进行配置，配置同时 TWAI 控制器必须处于复位模式下。\hyperref[regdesc:TWAIERRWARNINGLIMITREG]{TWAI\_ERR\_WARNING\_LIMIT\_REG} 默认数值为 96。

当 TEC 和/或 REC 数值大于等于 EWL 数值时，\hyperref[fielddesc:TWAISTATUSERR]{TWAI\_STATUS\_ERR} 位将立即被置 1。同理，当 TEC 和 REC 数值都小于 EWL 数值时，\hyperref[fielddesc:TWAISTATUSERR]{TWAI\_STATUS\_ERR} 位将立即复位为 0。只要 \hyperref[fielddesc:TWAISTATUSERR]{TWAI\_STATUS\_ERR}（或 \hyperref[fielddesc:TWAISTATUSNODEBUSOFF]{TWAI\_STATUS\_NODE\_BUS\_OFF}）位值发生变化，便会触发错误报警中断。

\subsubsection{被动错误}

当 TEC 或 REC 数值大于 127 时，TWAI 控制器处于被动错误状态。同理，当 TEC 和 REC 数值都小于等于 127 时，TWAI 控制器进入主动错误状态。每当 TWAI 控制器从主动错误状态变为被动错误状态，或反之时，都将触发被动错误中断。

\subsubsection{离线状态与离线恢复}

当 TEC 数值大于 255 时，TWAI 控制器将进入离线状态。进入离线状态后，TWAI 控制器将自动进行以下动作：

\begin{itemize}
    \item REC 数值置为 0
    \item TEC 数值置为 127
    \item \hyperref[fielddesc:TWAISTATUSNODEBUSOFF]{TWAI\_STATUS\_NODE\_BUS\_OFF} 位置 1
    \item 进入复位模式
\end{itemize}

每当 \hyperref[fielddesc:TWAISTATUSNODEBUSOFF]{TWAI\_STATUS\_NODE\_BUS\_OFF} 位（或 \hyperref[fielddesc:TWAISTATUSERR]{TWAI\_STATUS\_ERR} 位）数值发生变化时，都将触发错误报警中断。

为了返回主动错误状态，TWAI 控制器必须进行离线恢复。要启动离线恢复，首先需要退出复位模式，进入操作模式。然后要求 TWAI 控制器在总线上检测到 128 次 11 个连续隐性位。

每一次 TWAI 控制器检测到 11 个连续隐性位时，TEC 数值都将减小，以追踪离线恢复进程。当离线恢复完成后（TEC 数值从 127 减小到 0），\hyperref[fielddesc:TWAISTATUSNODEBUSOFF]{TWAI\_STATUS\_NODE\_BUS\_OFF} 位将自动复位为 0，从而触发错误报警中断。

\subsection{收发器自动待机}
\label{sec:twai-low-power}
为降低功耗，TWAI 收发器通常支持自动待机模式。TWAI 控制器可以在适当的时候（例如，当总线将长时间空闲时）产生待机信号，将收发器置于待机状态。如果控制器停止发出待机信号，或在收发器检测到总线活动（也称为唤醒功能）时，收发器将退出待机模式。

\chipname{} 的 TWAI 控制器支持通过硬件（自动）和软件（手动）两种方式输出待机信号，来控制芯片外部连接的 TWAI 收发器的开关。在硬件控制下，当总线保持空闲的时间超过可配置的时间时，TWAI 控制器将自动输出待机信号。在软件控制下，待机信号可由软件直接手动产生/关闭。

\begin{itemize}
    \item 硬件方式：
    \begin{enumerate}
        \item 通过将 \hyperref[regdesc:TWAIHWCFGREG]{TWAI\_HW\_CFG\_REG} 寄存器中的 \hyperref[fielddesc:TWAIHWSTANDBYEN]{TWAI\_HW\_STANDBY\_EN} 域置位，使能硬件待机功能。
        \item 配置 \hyperref[regdesc:TWAIHWSTANDBYCNTREG]{TWAI\_HW\_STANDB\_CNT\_REG} 寄存器。该寄存器表明 TWAI 控制器进入空闲状态后多久会产生待机信号，数值为 TWAI 控制器工作时钟（默认为 40 MHz）的周期数。
    \end{enumerate}
    \item 软件方式：
    \begin{enumerate}
        \item 直接置位 \hyperref[regdesc:TWAISWSTANDBYCFGREG]{TWAI\_SW\_STANDBY\_CFG\_REG} 寄存器中的 \hyperref[fielddesc:TWAISWSTANDBYEN]{TWAI\_SW\_STANDBY\_EN} 域，便可使 TWAI 控制器产生待机信号。
    \end{enumerate}
\end{itemize}

上述两种方式产生的待机信号，在满足以下任一条件后，会被拉低（清零）。
\begin{enumerate}
    \item TWAI 控制器接收端退出空闲状态时（例如接收端收到新报文），待机信号将被自动清零。
    \item 用户通过置位 \hyperref[regdesc:TWAISWSTANDBYCFGREG]{TWAI\_SW\_STANDBY\_CFG\_REG} 寄存器中的 \hyperref[fielddesc:TWAISWSTANDBYCLR]{TWAI\_SW\_STANDBY\_CLR} 域，也可以将待机信号拉低。
\end{enumerate}

\label{int:TWAITRANSMITINT}TWAI\_TRANSMIT\_INT (TXI) &
当发送缓冲器由锁定状态变为为空闲状态，将会触发发送中断 (TXI)。此时软件可以将其他报文加载到发送缓冲器中等待发送。以下几种情况都会触发该中断：
\begin{itemize}
    \item 报文发送已成功完成（如应答未发现错误）。任何发送失败将自动重发。
    \item 单次发送已完成（\hyperref[fielddesc:TWAISTATUSTRANSMISSIONCOMPLETE]{TWAI\_STATUS\_TRANSMISSION\_COMPLETE} 位指示发送成功与否）。

\end{itemize}

\label{int:TWAIERRWARNINGINT}TWAI\_ERR\_WARNING\_INT (EWI) &
每当寄存器 \hyperref[regdesc:TWAISTATUSREG]{TWAI\_STATUS\_REG} 中 \hyperref[fielddesc:TWAISTATUSERR]{TWAI\_STATUS\_ERR} 和 \hyperref[fielddesc:TWAISTATUSNODEBUSOFF]{TWAI\_STATUS\_NODE\_BUS\_OFF} 的位值改变时（如，从 0 变为 1 或反之），都会触发错误报警中断 (EWI)。根据 EWI 触发时 \hyperref[fielddesc:TWAISTATUSERR]{TWAI\_STATUS\_ERR} 和 \hyperref[fielddesc:TWAISTATUSNODEBUSOFF]{TWAI\_STATUS\_NODE\_BUS\_OFF} 的值分成以下几种情况：
\begin{itemize}
    \item 如果 \hyperref[fielddesc:TWAISTATUSERR]{TWAI\_STATUS\_ERR} = 0 且 \hyperref[fielddesc:TWAISTATUSNODEBUSOFF]{TWAI\_STATUS\_NODE\_BUS\_OFF} = 0：

        \item 如果 TWAI 控制器处于主动错误状态，则表示 TEC 和 REC 的值都返回到了 TWAI\_ERR\_WARNING\_LIMIT\_REG 所设的阈值之下。
        \item 如果 TWAI 控制器此前正处于总线恢复状态，则表示此时总线恢复已成功完成。
    \end{itemize}
    \item 如果 \hyperref[fielddesc:TWAISTATUSERR]{TWAI\_STATUS\_ERR} = 1 且 \hyperref[fielddesc:TWAISTATUSNODEBUSOFF]{TWAI\_STATUS\_NODE\_BUS\_OFF} = 0：表示 TEC 或 REC 数值已超过 TWAI\_ERR\_WARNING\_LIMIT\_REG 所设的阈值。
    \item 如果 \hyperref[fielddesc:TWAISTATUSERR]{TWAI\_STATUS\_ERR} = 1 且 \hyperref[fielddesc:TWAISTATUSNODEBUSOFF]{TWAI\_STATUS\_NODE\_BUS\_OFF} = 1：表示 TWAI 控制器已进入 BUS\_OFF 状态（因 TEC >= 256）。

\end{itemize}

\label{int:TWAIDATAOVERRUNINT}TWAI\_DATA\_OVERRUN\_INT (DOI) &
每当接收 FIFO 中有溢出发生时，都会触发数据溢出中断 (DOI)。DOI 表示接收 FIFO 已满且应立即进行读取，以防出现更多溢出报文。
只有导致接收 FIFO 溢出的第一条报文可触发 DOI（如，当接收 FIFO 从未满变为开始溢出时）。任意后续的溢出报文将不会再次重复触发 DOI。只有当所有接收的（有效报文或溢出）报文都被读取后，才能再次触发 DOI。

& TWAI\_INT \\ \hline
\label{int:TWAIERRPASSIVEINT}TWAI\_ERR\_PASSIVE\_INT (EPI) &
每当 TWAI 控制器从主动错误变为被动错误，或反之时，都会触发被动错误中断 (EPI)。
& TWAI\_INT \\ \hline
\label{int:TWAIARBITRATIONLOSTINT}TWAI\_ARBITRATION\_LOST\_INT (ALI) &
每当 TWAI 控制器尝试发送报文且丢失仲裁时，都会触发仲裁丢失中断 (ALI)。TWAI 控制器丢失仲裁的 bit 位置将自动记录在仲裁丢失捕捉寄存器 (\hyperref[regdesc:TWAIARBLOSTCAPREG]{TWAI\_ARB\_LOST\_CAP\_REG}) 中。仲裁丢失捕捉寄存器被清除（通过 CPU 读取该寄存器）之前，将不会再记录新发生的仲裁失败时的 bit 位置。
& TWAI\_INT \\ \hline
\label{int:TWAIBUSERRINT}TWAI\_BUS\_ERR\_INT (BEI) &
每当 TWAI 控制器在 TWAI 总线上检测到错误时，都会触发总线错误中断 (BEI)。发生总线错误时，总线错误的类型和发生错误时的 bit 位置都将自动记录在错误捕捉寄存器 (\hyperref[regdesc:TWAIERRCODECAPREG]{TWAI\_ERR\_CODE\_CAP\_REG}) 中。错误捕捉寄存器被清除（通过 CPU 的读取）之前，将不会再记录新的总线错误信息。
& TWAI\_INT \\ \hline
\label{int:TWAIIDLEINT}TWAI\_IDLE\_INT (BISI) &
当 TWAI 控制器进入空闲状态的时钟周期数超过预先设配置的 \hyperref[regdesc:TWAIIDLEINTRCNTREG]{TWAI\_IDLE\_INTR\_CNT\_REG} 寄存器内的数值时，将产生总线空闲状态中断 (BISI)。用户可以通过配置该中断来获知 TWAI 控制器空闲状态，进一步决定是否关闭外部 TWAI 接收器来降低整体功耗（参见\ref{sec:twai-low-power}）。
& TWAI\_INT \\ \hline
\end{longtable}



只要在 \hyperref[regdesc:TWAIINTERRUPTREG]{TWAI\_INTERRUPT\_REG} 一个或多个中断位为 1，TWAI 控制器中的中断信号即为有效，当 \hyperref[regdesc:TWAIINTERRUPTREG]{TWAI\_INTERRUPT\_REG} 中的所有位都被清除时，TWAI 控制器中的中断信号则失效。读操作访问寄存器 \hyperref[regdesc:TWAIINTERRUPTREG]{TWAI\_INTERRUPT\_REG} 后，其中的大多数中断位将自动清除。但是，只有通过 \hyperref[fielddesc:TWAIRELEASEBUFFER]{TWAI\_RELEASE\_BUFFER} 指令位清除所有接收报文后，接收中断位才能被清除。
