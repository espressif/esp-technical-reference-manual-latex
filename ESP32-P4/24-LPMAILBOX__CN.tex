\hypertarget{lpmailbox}{}
\chapter{LP 信箱控制器}
\label{mod:lpmailbox}

\section{概述}

\chipname{} 中包含一个 LP 信箱控制器模块，旨在通过硬件机制以实现 LP CPU 和 HP CUP0/1 之间高效的核间通信。LP 信箱控制器模块中包含 16 个 32 位信息寄存器可供 LP CPU 和 HP CPU0/1 存储并传递信息，并通过中断机制实现 LP CPU 和 HP CPU0/1 之间的核间通信。

\section{特性}
\chipname{} 的 LP 信箱控制器模块具有以下特性：
\begin{itemize}
    \item 支持多达 16 个 32 位信息寄存器用于核间通信
    \item 支持 LP CPU 外部中断信号 \hyperref[tab:lpmailbox-int-sources]{MB\_LP\_INTR}
    \item 支持 HP CPU\regindex{x} (\regindex{x}: 0 \verb+~+ 1) 外部中断信号 \hyperref[tab:lpmailbox-int-sources]{MB\_HP\_INTR}
\end{itemize}

\section{功能描述}
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\linewidth]{\modulefiles/figures/LP_MB}
    \caption{LP 信箱控制器系统框图}
    \label{fig:lpmbsysdigram}

\subsection{信息寄存器}
LP 信箱控制器中包含了 \hyperref[regdesc:MBMESSAGE0REG]{MB\_MESSAGE\_0\_REG} \verb+~+ \hyperref[regdesc:MBMESSAGE15REG]{MB\_MESSAGE\_15\_REG} 共计 16 组 32 位信息寄存器，用于缓存核间通信信息。LP CPU 和 HP CPU0/1 均可读写这些信息寄存器以传递或获取信息。

读写信息寄存器通过 APB 总线来完成，使用 APB\_CLK 时钟。该时钟为 LP 信箱控制器使用的唯一时钟，更多时钟信息，见章节 \ref{mod:resclk} \textit{\nameref{mod:resclk}}。

\subsection{中断}
\label{sec:lpmailbox-interrupt}

\chipname{} 的 LP 信箱控制器可以生成如下外部中断信号：
    \begin{itemize}
        \item MB\_LP\_INTR
        \item MB\_HP\_INTR
    \end{itemize}
MB\_LP\_INTR 直接映射至 LP CPU（见章节 \ref{mod:ulp} \textit{\textcolor{red}{\nameref{mod:ulp}}}），MB\_HP\_INTR 可通过\hyperref[mod:intmtrx]{中断矩阵}映射至 HP CPU0/1。
这些中断信号由 LP 信箱控制器模块的内部中断源生成。表 \ref{tab:lpmailbox-int-sources} 列出了 LP 信箱控制器的内部中断源、中断触发条件、以及生成的中断信号。
\begin{longtable}{ | L{5cm} | L{6.5cm} | L{4cm} | }
\caption{LP 信箱控制器的内部中断源} \label{tab:lpmailbox-int-sources}
\\ \hline
\rowcolor{lightgray}
\textbf{内部中断源} & \textbf{触发条件}  &  \textbf{中断信号}\\ \hline
\label{int:MBHPmINT} MB\_HP\_\regindex{m}\_INT &  写 MB\_MESSAGE\_\regindex{m}\_REG &  MB\_HP\_INTR  \\ \hline
\label{int:MBLPnINT} MB\_LP\_\regindex{n}\_INT &  写 MB\_MESSAGE\_\regindex{n}\_REG &  MB\_LP\_INTR  \\ \hline
\end{longtable}
\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
    \item \regindex{m} 和 \regindex{n} 可以是 0 \verb+~+ 15。
    \item\hyperref[tab:lpmailbox-int-sources]{MB\_HP\_INTR} 同样可作为 LP CPU 的外部中断源，通过 LP 中断映射模块映射（LP\_INTR\_MAP，见图 \ref{fig:lpmbsysdigram}）至 LP CPU。有关 LP\_INTR\_MAP 的更多信息，见章节 \ref{mod:ulp} \textit{\nameref{mod:ulp}}。
    \item\hyperref[tab:lpmailbox-int-sources]{MB\_LP\_INTR} 同样可以作为 HP CPU\regindex{x} 的外部中断源，可通过\hyperref[mod:intmtrx]{中断矩阵}将其映射至 HP CPU\regindex{x}。
    \item 当 LP 信箱控制器需要为 HP CPU0 和 HP CPU1 分配不同的中断源时，可通过\hyperref[mod:intmtrx]{中断矩阵}将 \hyperref[tab:lpmailbox-int-sources]{MB\_HP\_INTR} 和 \hyperref[tab:lpmailbox-int-sources]{MB\_LP\_INTR} 分别映射至 HP CPU0 和 HP CPU1。
    \end{itemize}
\end{tiplisting}

\subsection{核间通信}
当 LP CPU 与 HP CPU\regindex{x} 之间需要通信时，可以根据需求利用 LP 信箱控制器中部分或者全部信息寄存器来分享信息，并通过相应的中断来同步信息。在通信期间，需要软件保证仅发送方可写 LP 信箱控制器中的信息寄存器，防止信息寄存器中的数据被篡改。
\begin{itemize}
    \item 以 HP CPU0 通过 MB\_MESSAGE\_\regindex{n}\_REG 向 LP CPU 发起通信为例：
    \begin{itemize}
        \item 置位 \hyperref[fielddesc:MBLP0INTENA]{MB\_LP\_\regindex{n}\_INT\_ENA} 使能 MB\_LP\_\regindex{n}\_INT 中断。
        \item HP CPU0 写 \hyperref[regdesc:MBMESSAGE0REG]{MB\_MESSAGE\_\regindex{n}\_REG}，缓存共享信息。
        \item LP CPU 检测 \hyperref[regdesc:MBLPINTSTREG]{MB\_LP\_INT\_ST\_REG} 中断状态寄存器发现 \hyperref[fielddesc:MBLP0INTST]{MB\_LP\_\regindex{n}\_ST} 状态位置位。
        \item LP CPU 读取 \hyperref[regdesc:MBMESSAGE0REG]{MB\_MESSAGE\_\regindex{n}\_REG} 获取共享信息。
    \end{itemize}
    \item 以 LP CPU 向 HP CPU0 发起通信为例：
    \begin{itemize}
        \item 置位 \hyperref[fielddesc:MBHP0INTENA]{MB\_HP\_\regindex{m}\_INT\_ENA} 使能 MB\_HP\_\regindex{m}\_INT 中断。
        \item 配置\hyperref[mod:intmtrx]{中断矩阵}，将 MB\_HP\_INTR 映射至 HP CPU0（见章节 \ref{mod:intmtrx} \textit{\nameref{mod:intmtrx}}）。
        \item LP CPU 写 \hyperref[regdesc:MBMESSAGE0REG]{MB\_MESSAGE\_\regindex{m}\_REG}，缓存共享信息。
        \item HP CPU0 检测 \hyperref[regdesc:MBHPINTSTREG]{MB\_HP\_INT\_ST\_REG} 中断状态寄存器，发现 \hyperref[fielddesc:MBHP0INTST]{MB\_HP\_\regindex{m}\_ST} 状态位置位。
        \item HP CPU0 读取 \hyperref[regdesc:MBMESSAGE0REG]{MB\_MESSAGE\_\regindex{m}\_REG} 获取共享信息。
    \end{itemize}
\end{itemize}
