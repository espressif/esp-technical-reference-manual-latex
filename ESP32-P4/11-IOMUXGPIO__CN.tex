\hypertarget{iomuxgpio}{}
\chapter{GPIO 交换矩阵和 IO MUX}
\label{mod:iomuxgpio}

\section{概述}
GPIO 交换矩阵是一种硬件结构，允许芯片上的 GPIO 管脚与外设输入/输出信号之间进行动态的重新映射和交换，以提供更大的灵活性。IO MUX（输入/输出多路复用）是一种技术，允许同一管脚在不同时间承担不同的功能，通过配置实现对管脚功能的动态切换。\chipname{} 提供两组 GPIO 交换矩阵和 IO MUX：
\begin{itemize}
    \item HP GPIO 交换矩阵和 HP IO MUX
    \item LP GPIO 交换矩阵和 LP IO MUX
\end{itemize}

\chipname{} 共有 55 个 GPIO 管脚，其中包括 16 个低功耗 (LP) GPIO 管脚和 39 个高性能 (HP) GPIO 管脚。每个管脚都可用作一个通用 IO，或连接一个内部的外设信号。
\begin{itemize}
\item 利用 HP GPIO 交换矩阵和 HP IO MUX，可配置 HP 外设模块的输入信号来源于任何的 GPIO 管脚，并且 HP 外设模块的输出信号也可连接到任意 GPIO 管脚。
\item 利用 LP GPIO 交换矩阵和 LP IO MUX，可配置 LP 外设模块的输入信号来源于任何的 LP GPIO 管脚，并且 LP 外设模块的输出信号也可连接到任意 LP GPIO 管脚。

\end{itemize}
这些模块共同组成了芯片的 GPIO 控制。上述 55 个 GPIO 管脚的编号为：GPIO0 \verb+~+ GPIO54。
    \begin{itemize}
        \item GPIO 管脚 0 \verb+~+ 15 为 LP GPIO 管脚，可由 HP 或 LP 外设使用。
        \item GPIO 管脚 16 \verb+~+ 54 为 HP GPIO 管脚，只能由 HP 外设使用。
    \end{itemize}
该 \chipname{} 芯片有专用于连接外部 flash 和封装内 PSRAM 的管脚，此类管脚不可用于其他用途。更多信息见\textit{\href{https://www.espressif.com/sites/default/files/documentation/esp32-p4_datasheet_cn.pdf}{《\chipname{} 技术规格书》}} > 章节\textit{芯片与 flash 的管脚对应关系}。

除非另有说明，本章节所述的 GPIO 交换矩阵同时包括 HP GPIO 交换矩阵和 LP GPIO 交换矩阵；IO MUX 同时包括 HP IO MUX 和 LP IO MUX。

\section{主要特性}
\subsection{HP GPIO 交换矩阵和 HP IO MUX}
\textbf{HP GPIO 交换矩阵具有以下特性：}
\begin{itemize}
\item HP 外设输入输出信号和 GPIO 管脚之间的全交换矩阵
\item 222 个 HP 外设输入信号可以选择任意一个 GPIO 管脚的输入信号
\item 每个 GPIO 管脚的输出信号可以来自 232 个 HP 外设输出信号的任意一个
\item HP 外设输入信号经 GPIO SYNC 模块同步至 HP IO MUX 运行时钟
\item 支持 GPIO 滤波器对输入信号进行滤波
\item 支持毛刺滤波器对输入信号进行二次滤波
\item 支持 Sigma Delta 调制输出 (SDM)
\item 支持 GPIO 简单输入输出
\item 支持 HP GPIO 唤醒
\end{itemize}

\textbf{HP IO MUX 具有以下特性：}
\begin{itemize}
\item 控制 55 个 GPIO (GPIO0 \verb+~+ GPIO54) 供 HP 外设使用
\item 为每个 GPIO 管脚提供一个寄存器 \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO\regindex{n}\_REG}，用于控制管脚的输入/输出、上拉/下拉、驱动强度、功能选择等配置
\item 支持高频信号如 SPI、EMAC 等直接通过 HP IO MUX 输入和输出外设，实现更好的高频数字特性
\end{itemize}

\subsection{LP GPIO 交换矩阵和 LP IO MUX}

\textbf{LP GPIO 交换矩阵具有如下特性：}
\begin{itemize}
\item LP 外设输入输出信号和 LP GPIO 管脚之间的全交换矩阵
\item 14 个 LP 外设输入信号可以选择任意一个 LP GPIO 管脚的输入信号
\item 每个 LP GPIO 管脚的输出信号可以来自 14 个 LP 外设输出信号的任意一个
\item 支持 GPIO 滤波器对输入信号进行滤波
\item 支持 GPIO 简单输入输出
\item 支持 LP GPIO 唤醒
\end{itemize}

\textbf{LP IO MUX 具有如下特性：}
\begin{itemize}
    \item 16 个 LP GPIO 管脚 (GPIO0 \verb+~+ GPIO15) 供 LP 外设使用
    \item 为每个 LP GPIO 管脚提供一个 \hyperref[regdesc:LPIOMUXPADNREG]{LP\_IOMUX\_PAD\regindex{x}\_REG} 寄存器，用于控制管脚的输入/输出、上拉/下拉、驱动强度、功能选择、IO MUX 选择等配置
\end{itemize}

\section{通过 GPIO 交换矩阵的外设输入}
\label{sec:peripheral-input-via-gpio-matrix}

\subsection{概述}

通过 HP GPIO 交换矩阵接收外部输入信号，一般步骤如下：
\begin{itemize}
    \item 配置 HP GPIO 交换矩阵从 55 个 GPIO (0 \verb+~+ 54) 中获取外部输入信号，见表 \ref{tab:iomuxgpio-periph-signals-via-hp-gpio-matrix-cn}。
    \item 配置外设输入选择通过 HP GPIO 交换矩阵接收输入信号。
    \item 配置 GPIO 管脚由 HP IO MUX 控制。
\end{itemize}
具体配置流程，可参考图 \ref{fig:iomux-gpio-matrix-overview-detail} 和章节 \ref{sec:functional-description}。

通过 LP GPIO 交换矩阵接收外部输入信号，一般步骤如下：
\begin{itemize}
    \item 配置 LP GPIO 交换矩阵从 16 个 GPIO (0 \verb+~+ 15) 中获取外部输入信号，见表 \ref{tab:iomuxgpio-periph-signals-via-lp-gpio-matrix-cn}。
    \item 配置外设输入选择通过 LP GPIO 交换矩阵接收输入信号。
    \item 配置 LP GPIO 管脚由 LP IO MUX 控制。
\end{itemize}
具体配置流程，可参考图 \ref{fig:iomux-gpio-matrix-overview-detail} 和章节 \ref{sec:functional-description}。

如图 \ref{fig:iomux-gpio-matrix-overview-detail} 所示，使用 GPIO 交换矩阵将信号从管脚输入时，所有外部输入信号均来源于 GPIO 管脚，然后经 GPIO 滤波器进行滤波，见章节 \ref{sec:functional-description} 中的步骤 \ref{step:2}。

毛刺滤波器仅在 HP GPIO 交换矩阵中可用。毛刺滤波器硬件可从 GPIO 滤波器输出的信号中选择八个信号进行滤波，其他没被选中的信号则直接进入 GPIO SYNC 硬件，见章节 \ref{sec:functional-description} 中的步骤 \ref{step:3}。

所有信号经 GPIO 滤波器硬件或毛刺滤波器硬件滤波后，均将被 GPIO SYNC 硬件同步至 IO MUX 运行时钟，然后进入 GPIO 交换矩阵，见章节 \ref{sec:inputconfig}。外部输入信号也可经 IO MUX 直接输入至外设，但信号无法进行滤波及时钟同步操作。

\subsection{信号同步}
\label{sec:inputconfig}

HP GPIO 交换矩阵支持信号同步功能。GPIO SYNC 模块的功能如图 \ref{Figure:gpio_input_sync} 所示。其中，negative sync 为 GPIO 输入经过 HP IO MUX 运行时钟的下降沿同步，positive sync 为 GPIO 输入经过 HP IO MUX 运行时钟的上升沿同步。

\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=1.0\textwidth]{\modulefiles/figures/gpio_input_sync_20220309}}
    \caption{GPIO 输入经 HP IO MUX 运行时钟上升沿或下降沿同步}
    \label{Figure:gpio_input_sync}
\end{figure}

同步器 (synchronizer) 默认关闭同步功能。但如果一个异步外设信号连接到管脚时，该信号应通过两级同步器（即图  \ref{Figure:gpio_input_sync} 中的 first-level synchronizer 和 second-level synchronizer）进行同步，以减小亚稳态产生的概率。


\subsection{GPIO 滤波器}
\label{IOMUXGPIOFilter}
HP GPIO 交换矩阵和 LP GPIO 交换矩阵均支持 GPIO 滤波功能。使能 GPIO 滤波器功能后，只有当输入信号的有效宽度大于两个时钟周期时，输入信号才会被采样。否则，输入信号将会被滤掉。见图 \ref{Figure:gpio_filter}。
\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/gpio_filter}}
    \caption{GPIO 输入信号滤波时序图}
    \label{Figure:gpio_filter}
\end{figure}

\subsection{毛刺滤波器}
\label{IOMUXGlitchFilter}
毛刺滤波功能仅在 HP GPIO 交换矩阵中可用。毛刺滤波器硬件支持八个通道，每个通道可从 GPIO 滤波器硬件输出的 55 个信号 (0 \verb+~+ 54) 中选择一个信号，进行二次滤波。该毛刺滤波器硬件可用于对慢速信号进行滤波。见章节 \ref{sec:prgramming-procedure-for-hp} 中的步骤  \ref{step:3}。

\subsection{简单 GPIO 输入}
\label{IOMUXSimpleGPIOInput}
HP GPIO 交换矩阵和 LP GPIO 交换矩阵均支持简单 GPIO 输入。使用简单 GPIO 输入模式时，任意 GPIO 管脚的值均可随时读取，而无需将 GPIO 管脚输入绑定到某个外设信号。

配置 HP GPIO 交换矩阵实现简单 GPIO 输入，具体过程如下：
\begin{itemize}
    \item 置位 \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO\regindex{x}\_REG} 的 \hyperref[fielddesc:IOMUXGPIONMCUIE] {IO\_MUX\_GPIO\regindex{x}\_MCU\_IE} 位，使能管脚输入。
    \item 读取 \hyperref[regdesc:GPIOINREG]{GPIO\_IN\_REG[\regindex{x}]}，即可实现简单 GPIO 输入。
\end{itemize}

配置 LP GPIO 交换矩阵实现简单 GPIO 输入，具体过程如下：
\begin{itemize}
    \item 置位  \hyperref[regdesc:LPIOMUXPADNREG]{LP\_IO\_MUX\_PAD\regindex{x}\_REG} 的  \hyperref[fielddesc:LPIOMUXPADNFUNIE] {LP\_IOMUX\_PAD\regindex{x}\_FUN\_IE} 位，使能管脚输入。
    \item 读取 \hyperref[regdesc:LPGPIOINREG]{LP\_GPIO\_IN\_REG[\regindex{x}]}，即可实现简单 GPIO 输入。
\end{itemize}

\subsection{编程流程}
\label{sec:functional-description}
\subsubsection{HP GPIO 交换矩阵}
\label{sec:prgramming-procedure-for-hp}
把某个 HP 外设输入信号 \regindex{Y} 绑定到某个 GPIO 管脚  \regindex{X} 的配置过程如下：

\begin{enumerate}
    \item 在 HP GPIO 交换矩阵中配置外设信号  \regindex{Y} 的 \hyperref[regdesc:GPIOFUNCNINSELCFGREG]{GPIO\_FUNC\regindex{y}\_IN\_SEL\_CFG\_REG} 寄存器：
    \begin{itemize}
        \item 置位 \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_SIG\regindex{y}\_IN\_SEL} 选择通过 HP GPIO 交换矩阵接收外部输入信号。
        \item 设置 \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} 为需要的 GPIO 管脚编号，此处应为 \regindex{X}。
    \end{itemize}
    {\bfseries 注意：}并不是所有的 HP 外设信号都有有效的 \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_SIG\regindex{y}\_IN\_SEL} 位，即有些外设信号只能通过 HP GPIO 交换矩阵接收外部输入信号。
    \item \label{step:2} 可选：置位 \hyperref[fielddesc:IOMUXGPIONFILTEREN]{IO\_MUX\_GPIO\regindex{n}\_FILTER\_EN} 使能 GPIO 管脚的输入信号滤波功能。
\item \label{step:3} 启用毛刺滤波器功能，步骤如下：
\begin{itemize}
    \item 配置  \hyperref[fielddesc:GPIOSDFILTERCH0INPUTIONUM]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_INPUT\_IO\_NUM} 为 \regindex{m}。此处 \regindex{n} 为通道编号，取值范围为：0 \verb+~+ 7。\regindex{m} 为 GPIO 管脚编号，取值范围为：0 \verb+~+ 54。
    \item 配置  \hyperref[fielddesc:GPIOSDFILTERCH0WINDOWWIDTH]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_WINDOW\_WIDTH} 为 \textbf{VALUE1}，配置 \hyperref[fielddesc:GPIOSDFILTERCH0WINDOWTHRES]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_WINDOW\_THRES} 为 \textbf{VALUE2}。在 \textbf{VALUE1} + 1 个周期内，如果存在 \textbf{VALUE2} + 1 个输入信号与当前输出信号值不匹配，毛刺滤波器硬件将反转输出信号。\hyperref[fielddesc:GPIOSDFILTERCH0WINDOWWIDTH]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_WINDOW\_WIDTH} 和 \hyperref[fielddesc:GPIOSDFILTERCH0WINDOWTHRES]{GPIO\_EXT\_FILTER\_CH\regindex{n}\_WINDOW\_THRES} 可以配置为相同的值 \textbf{VALUE3}，那么只有宽度大于 \textbf{VALUE3} + 1 个时钟周期的信号才会被采样。
    \item 置位 \hyperref[fielddesc:GPIOSDFILTERCH0EN]{GPIOSD\_FILTER\_CH\regindex{n}\_EN} 使能通道 \regindex{n}。
\end{itemize}

相关示例见图 \ref{Figure:glitch_filter}，其中 \hyperref[fielddesc:GPIOSDFILTERCH0WINDOWWIDTH]{GPIO\_EXT\_FILTER\_CH\regindex{x}\_WINDOW\_WIDTH} 配置为 3 \hyperref[fielddesc:GPIOSDFILTERCH0WINDOWTHRES]{GPIO\_EXT\_FILTER\_CH\regindex{x}\_WINDOW\_THRES} 配置为 2。在 T1 之前的四个时钟周期内，输出信号值 (signal\_out) 一直为 “0”。输入信号值 (signal\_in) 有三个时钟周期为“1”，则输出信号在 T1 之后反转为 “1”。
\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/figure-glitch-filter-20220630-p4}}
    \caption{毛刺滤波器时序示例}
    \label{Figure:glitch_filter}
\end{figure}
    \item \label{step:4} 同步 GPIO 输入信号。配置 GPIO 管脚 \regindex{X} 的 \hyperref[regdesc:GPIOPINNREG]{GPIO\_PIN\regindex{x}\_REG} 来同步 GPIO 输入信号，过程如下：
    \begin{itemize}
        \item 如图 \ref{Figure:gpio_input_sync} 所示，配置 \hyperref[fielddesc:GPIOPINNSYNC1BYPASS]{GPIO\_PIN\regindex{x}\_SYNC1\_BYPASS} 使能输入信号在第一级同步中为上升沿或下降沿同步。
        \item 如图 \ref{Figure:gpio_input_sync} 所示，配置 \hyperref[fielddesc:GPIOPINNSYNC2BYPASS]{GPIO\_PIN\regindex{x}\_SYNC2\_BYPASS} 使能输入信号在第二级同步中为上升沿或下降沿同步。
    \end{itemize}
    \item 配置 HP IO MUX 寄存器使能 GPIO 管脚的输入功能。配置 GPIO 管脚 \regindex{X} 的 \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO\regindex{x}\_REG}，过程如下：
    \begin{itemize}
        \item 置位 \hyperref[fielddesc:IOMUXGPIONFUNIE]{IO\_MUX\_GPIO\regindex{x}\_FUN\_IE} 使能输入\hyperref[input-note-2]{$^2$}。
        \item 置位或清零 \hyperref[fielddesc:IOMUXGPIONFUNWPU]{IO\_MUX\_GPIO\regindex{x}\_FUN\_WPU} 和 \hyperref[fielddesc:IOMUXGPIONFUNWPD]{IO\_MUX\_GPIO\regindex{x}\_FUN\_WPD}，使能或关闭内部上拉/下拉电阻。
    \end{itemize}
\end{enumerate}
\begin{tiplisting}
\vspace{-2em}
\begin{enumerate}
\item \label{input-note-1} 同一个输入管脚可以同时绑定多个输入信号。
\item \label{input-note-2} 置位 \hyperref[fielddesc:GPIOFUNCNININVSEL]{GPIO\_FUNC\regindex{y}\_IN\_INV\_SEL} 可以把输入信号取反。
\item \label{input-note-3} 无需将输入信号绑定到一个 GPIO 管脚也可以使 HP 外设读取恒低或恒高电平的输入值。实现方式为选择特定的 \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} 输入值而不是一个 GPIO 序号：
\begin {itemize}
\item 设置 \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} 为 62，则输入信号恒为 0。
\item 设置 \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC\regindex{y}\_IN\_SEL} 为 63，则输入信号恒为 1。
\end{itemize}
\end{enumerate}
\end{tiplisting}

\textbf{编程示例}

要把 UART0 TXD 输入信号\hyperref[input-note-3]{$^3$}（uart0\_rxd\_pad\_in，信号索引号 10）绑定到 GPIO7，请按照以下步骤操作：
\begin{enumerate}
\item 置位 \hyperref[regdesc:GPIOFUNCNINSELCFGREG]{GPIO\_FUNC10\_IN\_SEL\_CFG\_REG} 中的 \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_SIG10\_IN\_SEL} 位，使能通过 HP GPIO 交换矩阵接收外部输入信号。
\item 配置 \hyperref[regdesc:GPIOFUNCNINSELCFGREG]{GPIO\_FUNC10\_IN\_SEL\_CFG\_REG} 中的 \hyperref[fielddesc:GPIOFUNCNINSEL]{GPIO\_FUNC10\_IN\_SEL} 为 7，即选择管脚 GPIO7。
\item 置位 \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO7\_REG} 中 \hyperref[fielddesc:IOMUXGPIONFUNIE]{IO\_MUX\_GPIO7\_FUN\_IE} 位使能管脚输入。
\end{enumerate}

\subsubsection{LP GPIO 交换矩阵}

通过 LP GPIO 交换矩阵进行外设输入的编程过程与章节 \ref{sec:prgramming-procedure-for-hp} 中的描述相似，区别之处在于：
\begin{itemize}
    \item LP GPIO 交换矩阵没有毛刺滤波功能。
    \item 相关的控制寄存器和状态寄存器需要使用 LP GPIO 寄存器，见章节 \ref{sec:lp-gpio-register-summ} 和章节 \ref{sec:lp-iomux-register-summ}。
\end{itemize}
\begin{tiplisting}
无需将输入信号绑定到一个 LP GPIO 管脚也可以使 LP 外设读取恒低或恒高电平的输入值。实现方式为选择特定的 \hyperref[fielddesc:LPGPIOSIGNINSEL]{LP\_GPIO\_SIG\regindex{y}\_IN\_SEL} 输入值而不是一个 GPIO 序号：
\begin {itemize}
\item 设置 \hyperref[fielddesc:LPGPIOSIGNINSEL]{LP\_GPIO\_SIG\regindex{y}\_IN\_SEL} 为 16，则输入信号恒为 0。
\item 设置 \hyperref[fielddesc:LPGPIOSIGNINSEL]{LP\_GPIO\_SIG\regindex{y}\_IN\_SEL} 为 24，则输入信号恒为 1。
\end{itemize}
\end{tiplisting}

\section{通过 GPIO 交换矩阵的外设输出}
\label{sec:peripheral-output-via-gpio-matrix}

\subsection{概述}

通过 GPIO 交换矩阵输出外设信号，一般步骤如下：
\begin{itemize}
    \item 配置 HP GPIO 交换矩阵将 HP 外设信号（即在表 \ref{tab:iomuxgpio-periph-signals-via-hp-gpio-matrix-cn} 中``输出信号”一栏所列出的信号）输出到 55 个 GPIO (0 \verb+~+ 54) 管脚
    \item 或配置 LP GPIO 交换矩阵将 LP 外设信号（即在表 \ref{tab:iomuxgpio-periph-signals-via-lp-gpio-matrix-cn} 中``输出信号”一栏所列出的信号）输出到 16 个 LP GPIO (0 \verb+~+ 15) 管脚
    \item 输出信号从外设输出到 GPIO 交换矩阵，然后到达 IO MUX。IO MUX 必须设置相应管脚为 GPIO 功能，这样输出 GPIO 信号就能连接到相应管脚。
\end{itemize}
\begin{tiplisting}
表 \ref{tab:iomuxgpio-periph-signals-via-hp-gpio-matrix-cn} 中输出索引号为 250 \verb+~+ 255 的外设信号，没有连接至外设，可配置为从一个 GPIO 管脚输出后，直接由另一个 GPIO 管脚输入（索引号：250 \verb+~+ 255）。
\end{tiplisting}
\vspace{-2em}

\subsection{简单 GPIO 输出}

GPIO 交换矩阵也可用于简单 GPIO 输出，即 GPIO 管脚可直接输出所期望的输出值，而无需将某个外设信号绑定到该 GPIO 管脚。

对于 HP GPIO 交换矩阵，具体配置如下：

\begin{itemize}
    \item 设置 \hyperref[fielddesc:GPIOFUNCNOUTSEL]{GPIO\_FUNC\regindex{x}\_OUT\_SEL} 为特定的外设索引值 256 (0x100)。
    \item 设置 \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}/\hyperref[regdesc:GPIOOUT1REG]{GPIO\_OUT1\_REG} 中相应位的值为期望 GPIO 输出的值。
\end{itemize}
\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
\item \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}[0] \verb+~+ \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}[31] 对应 GPIO0 \verb+~+ GPIO31，\hyperref[regdesc:GPIOOUT1REG]{GPIO\_OUT1\_REG}[0] \verb+~+ \hyperref[regdesc:GPIOOUT1REG]{GPIO\_OUT1\_REG}[22] 对应 GPIO32 \verb+~+ GPIO54。
\item 推荐使用相应的 W1TS 和 W1TC 寄存器，例如 \hyperref[fielddesc:GPIOOUTW1TS]{GPIO\_OUT/OUT1\_W1TS} 或 \hyperref[fielddesc:GPIOOUTW1TC]{GPIO\_OUT/OUT1\_W1TC} 来置位/清除 \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT/OUT1\_REG}。
\end{itemize}
\end{tiplisting}
对于 LP GPIO 交换矩阵，具体配置如下：

\begin{itemize}
    \item 设置 \hyperref[fielddesc:LPGPIOFUNCNOUTSEL]{LP\_GPIO\_FUNC\regindex{x}\_OUT\_SEL} 为特定的外设索引值 32 (0x20)。
    \item 设置 \hyperref[regdesc:LPGPIOOUTREG]{LP\_GPIO\_OUT\_REG} 中相应位的值为期望 GPIO 输出的值。
\end{itemize}
\begin{tiplisting}
推荐使用相应的 W1TS 和 W1TC 寄存器，例如 \hyperref[fielddesc:LPGPIOOUTDATAW1TS]{LP\_GPIO\_OUT\_DATA\_W1TS} 或 \hyperref[fielddesc:LPGPIOOUTDATAW1TC]{LP\_GPIO\_OUT\_DATA\_W1TC} 来置位/清除 \hyperref[regdesc:LPGPIOOUTREG]{LP\_GPIO\_OUT\_REG}。
\end{tiplisting}

\subsection{Sigma Delta 调制输出 (SDM)}

\subsubsection{功能描述}

HP 外设输出信号中有八个信号（在表 \ref{tab:iomuxgpio-periph-signals-via-hp-gpio-matrix-cn} 中索引为：72 \verb+~+ 79）支持 1-bit 二阶 SDM 调制输出。上述八个信号通道默认输出使能。Sigma Delta 调制器可实现输出可配占空比的 PDM（脉冲密度调制）信号。二阶 Sigma Delta 调制器函数为：

\[
    H(z)=\textrm{X(z)z$^{-1}$} + \textrm{E(z)(1-z$^{-1}$)$^{2}$}
    \]

E(z) 为量化误差，X(z) 为输入。\\
Sigma Delta 调制器内部支持对 IO MUX 运行时钟 的 1 \verb+~+ 256 倍分频：
\begin{itemize}
    \item 置位 \hyperref[fielddesc:GPIOSDFUNCTIONCLKEN]{GPIO\_EXT\_SD\_FUNCTION\_CLK\_EN} 使能调制器时钟。
    \item 配置 \hyperref[fielddesc:GPIOSDSDNPRESCALE]{GPIO\_EXT\_SD\regindex{n}\_PRESCALE} 实现分频。\regindex{n} 取值范围为 0 \verb+~+ 7，对应八个信号通道。
\end{itemize}

分频后的时钟周期为调制器输出单位脉冲的周期。

\hyperref[fielddesc:GPIOSDSDNIN]{GPIO\_EXT\_SD\regindex{n}\_IN} 为有符号数，范围为 [-128, 127]，配置此寄存器控制输出 PDM 信号的占空比$^{1}$。

\begin{itemize}
    \item \hyperref[fielddesc:GPIOSDSDNIN]{GPIO\_EXT\_SD\regindex{n}\_IN} = -128，调制器输出信号占空比为 \begin{math}0\% \end{math}。
    \item \hyperref[fielddesc:GPIOSDSDNIN]{GPIO\_EXT\_SD\regindex{n}\_IN} = 0，调制器输出信号占空比接近 \begin{math}50\% \end{math}。
    \item \hyperref[fielddesc:GPIOSDSDNIN]{GPIO\_EXT\_SD\regindex{n}\_IN} = 127，调制器输出信号占空比接近 \begin{math}100\% \end{math}。
\end{itemize}

PDM 信号占空比计算公式为：

    \[
    Duty\_Cycle=\frac{\hyperref[fielddesc:GPIOSDSDNIN]{GPIO\_EXT\_SD\regindex{n}\_IN} + 128} {256}
    \]

\begin{tiplisting}
对 PDM 信号来说，占空比是指在若干脉冲周期内（比如 256 个脉冲周期），高电平占整个统计周期的比值。
\end{tiplisting}

\subsubsection{配置方法}

SDM 的配置方法如下：

\begin{itemize}
    \item 将 SDM 输出经 HP GPIO 交换矩阵连接至相应管脚，见章节 \ref{IOMUXPeripheralOutput}。
    \item 置位 \hyperref[fielddesc:GPIOSDFUNCTIONCLKEN]{GPIO\_EXT\_SD\_FUNCTION\_CLK\_EN}，使能 SDM 时钟
    \item 配置 \hyperref[fielddesc:GPIOSDSDNPRESCALE]{GPIO\_EXT\_SD\regindex{n}\_PRESCALE} 设置时钟分频系数。
    \item 配置 \hyperref[fielddesc:GPIOSDSDNIN]{GPIO\_EXT\_SD\regindex{n}\_IN} 设置 SDM 输出信号的占空比。
\end{itemize}

\subsection{编程流程}\label{IOMUXPeripheralOutput}

\subsubsection{HP GPIO 交换矩阵}
\label{sec:prgramming-procedure-for-hp-output}
如图 \ref{fig:iomux-gpio-matrix-overview-detail} 所示，表 \ref{tab:iomuxgpio-periph-signals-via-hp-gpio-matrix-cn} 中“输出信号”列的某一个信号通过 HP GPIO 交换矩阵到达 HP IO MUX，然后连接到某个 GPIO 管脚。

输出外设信号 \regindex{Y} 到某一 GPIO 管脚 \regindex{X}\hyperref[output-note-1]{$^1$} 的步骤如下：

\begin{enumerate}
    \item 在 HP GPIO 交换矩阵中配置 GPIO 管脚 \regindex{X} 的 \hyperref[regdesc:GPIOFUNCNOUTSELCFGREG]{GPIO\_FUNC\regindex{x}\_OUT\_SEL\_CFG\_REG} 和  \hyperref[regdesc:GPIOENABLEREG]{GPIO\_ENABLE\_REG}[\regindex{x}]。推荐使用相应 \hyperref[regdesc:GPIOENABLEW1TSREG]{W1TS}（写 1 置位）和 \hyperref[regdesc:GPIOENABLEW1TCREG]{W1TC}（写 1 清零）来更新 \hyperref[regdesc:GPIOENABLEREG]{GPIO\_ENABLE\_REG} 中的值。


    \begin{itemize}
        \item 设置 \hyperref[regdesc:GPIOFUNCNOUTSELCFGREG]{GPIO\_FUNC\regindex{x}\_OUT\_SEL\_CFG\_REG} 中的 \hyperref[fielddesc:GPIOFUNCNOUTSEL]{GPIO\_FUNC\regindex{x}\_OUT\_SEL} 字段为外设输出信号 Y 的索引号 (Y)。
        \item 要将信号强制使能为输出模式，需要将 GPIO 管脚 \regindex{X} 对应的 \hyperref[regdesc:GPIOFUNCNOUTSELCFGREG]{GPIO\_FUNC\regindex{x}\_OUT\_SEL\_CFG\_REG} 寄存器中 \hyperref[fielddesc:GPIOFUNCNOESEL]{GPIO\_FUNC\regindex{x}\_OE\_SEL} 字段置位；同时需要将 \hyperref[regdesc:GPIOENABLEW1TSREG]{GPIO\_ENABLE\_W1TS\_REG} 中的相应位置位。或者，将 \hyperref[fielddesc:GPIOFUNCNOESEL]{GPIO\_FUNC\regindex{x}\_OE\_SEL} 清零，即选择采用外设的输出使能信号，此时输出使能信号由内部逻辑功能决定。比如，表 \ref{tab:iomuxgpio-periph-signals-via-hp-gpio-matrix-cn} 中 “\hyperref[fielddesc:GPIOFUNCNOESEL]{GPIO\_FUNC\regindex{n}\_OE\_SEL} = 0 时输出信号的输出使能信号” 一栏的 spi2\_dqs\_pad\_oe 信号。
       \item 置位 \hyperref[regdesc:GPIOENABLEW1TCREG]{GPIO\_ENABLE\_W1TC\_REG} 中相应位可以关闭 GPIO 管脚的输出。
    \end{itemize}
    \item 要选择以开漏方式输出，可以设置 GPIO 管脚 \regindex{X} 的 \hyperref[regdesc:GPIOPINNREG]{GPIO\_PIN\regindex{x}\_REG} 寄存器中 \hyperref[fielddesc:GPIOPINNPADDRIVER]{GPIO\_PIN\regindex{x}\_PAD\_DRIVER} 位。
    \item 配置 HP IO MUX 寄存器来选择经由 HP GPIO 交换矩阵输出信号。配置 GPIO 管脚 \regindex{X} 的 \hyperref[regdesc:IOMUXGPIONREG]{IO\_MUX\_GPIO\regindex{x}\_REG} 的过程如下：
    \begin{itemize}
        \item 设置 GPIO 管脚 \regindex {X} 的  \hyperref[fielddesc:IOMUXGPIONMCUSEL]{IO\_MUX\_GPIO\regindex{x}\_MCU\_SEL} 为所需的管脚功能。此处选择数值 1，即功能 1（GPIO 功能），适用于所有管脚。
        \item 设置 \hyperref[fielddesc:IOMUXGPIONFUNDRV]{IO\_MUX\_GPIO\regindex{x}\_FUN\_DRV} 字段为特定的输出强度值 (0 \verb+~+ 3)，值越大，输出驱动能力越强。
        \item 在开漏模式下，通过置位/清零 \hyperref[fielddesc:IOMUXGPIONFUNWPU]{IO\_MUX\_GPIO\regindex{x}\_FUN\_WPU} 和 \hyperref[fielddesc:IOMUXGPIONFUNWPD]{IO\_MUX\_GPIO\regindex{x}\_FUN\_WPD} 使能或关闭上拉/下拉电阻。
    \end{itemize}
    \item 使能迟滞功能：
    \begin{itemize}
        \item GPIO0 \verb+~+ GPIO15：置位 \hyperref[fielddesc:LPIOMUXLPGPIOHYS]{LP\_IOMUX\_LP\_GPIO\_HYS}[\regindex{x}]（x: 0 \verb+~+ 15，对应 GPIO0 \verb+~+ GPIO15）。
        \item GPIO16 \verb+~+ GPIO47：置位 \hyperref[fielddesc:HPREGGPIO0HYSLOW]{HP\_SYSTEM\_GPIO\_0\_HYS\_LOW}[\regindex{x}]（x: 0 \verb+~+ 31，对应 GPIO16 \verb+~+ GPIO47）。
        \item GPIO48 \verb+~+ GPIO54：置位 \hyperref[fielddesc:HPREGGPIO0HYSHIGH]{HP\_SYSTEM\_GPIO\_0\_HYS\_HIGH}[\regindex{x}]（x: 0 \verb+~+ 6，对应 GPIO48 \verb+~+ GPIO54）。
    \end{itemize}
\end{enumerate}

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
\item \label{output-note-1} 某一个外设的输出信号可以同时从多个管脚输出。
\item \label{output-note-2} 置位 \hyperref[fielddesc:GPIOFUNCNOUTINVSEL]{GPIO\_FUNC\regindex{x}\_OUT\_INV\_SEL} 可以把输出的信号取反。
\end{itemize}
\end{tiplisting}

\subsubsection{LP GPIO 交换矩阵}

通过 LP GPIO 交换矩阵进行外设输出的编程过程与章节  \ref{sec:prgramming-procedure-for-hp-output} 中的描述相似，区别之处在于：相关的控制寄存器和状态寄存器需要使用 LP GPIO 寄存器，见章节 \ref{sec:lp-gpio-register-summ} 和章节 \ref{sec:lp-iomux-register-summ}。

\section{IO MUX 的直接输入输出功能}

\subsection{概述}

SDMMC、EMAC 等数字信号可以旁路 GPIO 交换矩阵以实现更好的高频数字特性。所以此类信号可以直接通过 IO MUX 输入和输出。这样比使用 GPIO 交换矩阵的灵活度要低，即每个 GPIO 管脚的 IO MUX 寄存器只有较少的功能选择，但可以实现更好的高频数字特性。

\chipname{} 中有 16 个 GPIO 管脚具有低功耗 (LP) 性能。这些管脚可由 HP IO MUX 或 LP IO MUX 控制。如果由 LP IO MUX 控制，这些管脚将旁路 HP IO MUX 和 HP GPIO 交换矩阵，直接供 LP 系统中的外设使用。

当这些管脚被配置为 LP GPIO 管脚，仍然能够在芯片处于 Deep-sleep 模式下由 LP 系统的外设控制，也可以作为唤醒源将芯片从 Deep-sleep 中唤醒。

\subsection{功能描述}

LP 管脚 (GPIO0 \verb+~+ GPIO15) 由 \hyperref[regdesc:LPIOMUXPADNREG]{LP\_IOMUX\_PAD\regindex{n}\_REG} 中的 \hyperref[fielddesc:LPIOMUXPADNMUXSEL] {LP\_IOMUX\_PAD\regindex{n}\_MUX\_SEL} (\regindex{n} = 0 \verb+~+ 15) 位控制。此位默认置为 0，通过 HP IO MUX 输入输出信号。

如果置位 \hyperref[fielddesc:LPIOMUXPADNMUXSEL]{LP\_IOMUX\_PAD\regindex{n}\_MUX\_SEL}，则控制权交由 LP IO MUX 接管。在这种模式下，\hyperref[regdesc:LPIOMUXPADNREG]{LP\_IOMUX\_PAD\regindex{n}\_REG} 寄存器用于控制 LP GPIO 管脚。表 \ref{tab:LP-IO-MUX-CN} 列出了 GPIO 管脚的 LP 功能。请注意 \hyperref[regdesc:LPIOMUXPADNREG]{LP\_IOMUX\_PAD\regindex{n}\_REG} 寄存器使用的是 LP GPIO 管脚的序号，不是 HP GPIO 管脚的序号。

\subsubsection{HP IO MUX}
对于 HP 外设输入信号，旁路 HP GPIO 交换矩阵必须配置以下两个字段：

\begin{enumerate}
\item GPIO 管脚的 \hyperref[fielddesc:IOMUXGPIONMCUSEL]{IO\_MUX\_GPIO\regindex{n}\_MCU\_SEL} 必须配置为相应的管脚功能。表 \ref{tab:HP-IO-MUX-CN} 列出了管脚功能。
\item 清除 \hyperref[fielddesc:GPIOSIGNINSEL]{GPIO\_SIG\regindex{n}\_IN\_SEL}，直接将输入信号连接到外设。
\end{enumerate}

对于 HP 外设输出信号，旁路 HP GPIO 交换矩阵只需将 GPIO 管脚的 \hyperref[fielddesc:IOMUXGPIONMCUSEL]{IO\_MUX\_GPIO\regindex{n}\_MCU\_SEL} 配置为相应的管脚功能即可。

\subsubsection{LP IO MUX}
对于 LP 外设输入信号，旁路 LP GPIO 交换矩阵必须配置以下两个字段：

\begin{enumerate}
\item GPIO 管脚的 \hyperref[fielddesc:LPIOMUXPADNMUXSEL] {LP\_IOMUX\_PAD\regindex{n}\_MUX\_SEL} 必须配置为相应的管脚功能。表 \ref{tab:HP-IO-MUX-CN} 列出了管脚功能。
\item 清零 \hyperref[fielddesc:LPGPIOSIGNINSEL] {LP\_GPIO\_SIG\regindex{n}\_IN\_SEL}，直接将输入信号连接到外设。
\end{enumerate}

\section{GPIO 管脚的迟滞特性}\label{PadHysEn}

每个 GPIO 管脚都具有迟滞功能。当未使能迟滞功能时，如图  \ref{Figure:hysteresis_fig1} 所示，芯片焊盘 (PAD) 上输入给芯片内部的信号 (C) 的电平翻转只有一个阈值（Vt，约 1.7 V）。当 PAD 的电压高于 Vt 时，C 的电平为高，否则为低。但如果 PAD 上的信号有噪声，可能会影响 C 上的信号。

\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.8\textwidth]{\modulefiles/figures/hysteresis_figure1_20230220}}
    \caption{未使能迟滞功能时 PAD 上的电平翻转示例}
    \label{Figure:hysteresis_fig1}
\end{figure}

当使能迟滞功能时，如图 \ref{Figure:hysteresis_fig2} 所示，C 的电平翻转有两个阈值，分别为高电平阈值（Vth，约 1.7 V）和低电平阈值（Vtl，约 1.4 V）。当 PAD 的电压由低变高时，如果电压高于 Vth，则 C 的电平为高。当 PAD 的电压由高变低时，若电压低于 Vtl，则 C 的电平为低。当 PAD 的电压在 Vth 和 Vtl 之间时，C 的电平不变。迟滞功能可以减少噪声的影响，起到抗干扰作用，减少 C 的电平翻转次数。

要使能迟滞功能，请按照以下步骤操作：

\begin{itemize}
        \item 置位 \hyperref[fielddesc:LPIOMUXLPGPIOHYS] {LP\_IOMUX\_LP\_GPIO\_HYS[\regindex{n}]}（\regindex{n} = 0 \verb+~+ 15，对应 GPIO0 \verb+~+ GPIO15），并置位  \hyperref[fielddesc:HPSYSGPIO0HYSLOW]{HP\_SYS\_GPIO0\_HYS\_LOW[\regindex{n}]}（\regindex{n} = 0 \verb+~+ 31 对应 GPIO16 \verb+~+ GPIO47）或置位  \hyperref[fielddesc:HPSYSGPIO0HYSHIGH]{HP\_SYS\_GPIO0\_HYS\_HIGH[\regindex{n}]}（\regindex{n} = 0 \verb+~+ 6，对应 GPIO48 \verb+~+ GPIO54），使能 GPIO\regindex{n} 的迟滞功能。
        \item 清零  \hyperref[fielddesc:LPIOMUXLPGPIOHYS] {LP\_IOMUX\_LP\_GPIO\_HYS[\regindex{n}]}（\regindex{n} = 0 \verb+~+ 15，对应 GPIO0 \verb+~+ GPIO15），并清零  \hyperref[fielddesc:HPSYSGPIO0HYSLOW]{HP\_SYS\_GPIO0\_HYS\_LOW[\regindex{n}]}（\regindex{n} = 0 \verb+~+ 31，对应 GPIO16 \verb+~+ GPIO47），或清零  \hyperref[fielddesc:HPSYSGPIO0HYSHIGH]{HP\_SYS\_GPIO0\_HYS\_HIGH[\regindex{n}]}（\regindex{n} = 0 \verb+~+ 6，对应 GPIO48 \verb+~+ GPIO54），关闭 GPIO\regindex{n} 的迟滞功能。
\end{itemize}

\section{事件任务矩阵 (ETM) 功能}

在 \chipname{} 中，GPIO 支持 ETM 功能，即可以通过任意外设的 ETM 事件触发 GPIO 的 ETM 任务，或者通过 GPIO 的 ETM 事件触发任意外设的 ETM 任务。关于 ETM 更多详细信息，请参考章节 \ref{mod:evntaskmatrix} \textit{\nameref{mod:evntaskmatrix}}。这里仅介绍与 GPIO 相关的 ETM 任务和 ETM 事件。

GPIO ETM 有八个任务通道 \regindex{x} (0 \verb+~+ 7)。每个任务通道可接收的 ETM 任务有：

\begin{itemize}
    \item GPIO\_TASK\_CH\regindex{x}\_SET：触发时 GPIO 变为高电平。
    \item GPIO\_TASK\_CH\regindex{x}\_CLEAR：触发时 GPIO 变为低电平。
    \item GPIO\_TASK\_CH\regindex{x}\_TOGGLE：触发时 GPIO 电平翻转。
\end{itemize}

将任务通道 \regindex{x} 配置到 GPIO\regindex{y} 的示例如下：

\begin{itemize}
    \item 配置 \hyperref[fielddesc:IOMUXGPIONMCUSEL]{IO\_MUX\_GPIO\regindex{y}\_MCU\_SEL} 的值为 1，使得 GPIO\regindex{y} 选择表  \ref{tab:HP-IO-MUX-CN} 中的功能 1。
    \item 配置 \hyperref[regdesc:GPIOENABLEREG]{GPIO\_ENABLE\_REG[\regindex{y}]} 的值为 1。
    \item 配置  \hyperref[fielddesc:GPIOSDETMTASKGPIO0SEL]{GPIO\_EXT\_ETM\_TASK\_GPIO\regindex{y}\_SEL} 为  \regindex{x}。
    \item 置位  \hyperref[fielddesc:GPIOSDETMTASKGPIO0EN]{GPIO\_EXT\_ETM\_TASK\_GPIO\regindex{y}\_EN}，使能 GPIO\regindex{y} 受到任务通道  \regindex{x} 的控制。
\end{itemize}

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
\item 一个任务通道可被一个或多个 GPIO 选择。

\item 如果 GPIO\regindex{y} 选择的任务通道 \regindex{x} 其信号（GPIO\_TASK\_CH\regindex{x}\_SET、GPIO\_TASK\_CH\regindex{x}\_CLEAR 和 GPIO\_TASK\_CH\regindex{x}\_\\TOGGLE）中有两个或三个同时有效，则 GPIO\_TASK\_CH\regindex{x}\_SET 优先级最高，GPIO\_TASK\_CH\regindex{x}\_CLEAR 优先级次之，GPIO\_TASK\_CH\regindex{x}\_TOGGLE 优先级最低。

\item GPIO\regindex{y} 受到 ETM 任务通道控制时， \hyperref[regdesc:GPIOOUTREG]{GPIO\_OUT\_REG}、 \hyperref[fielddesc:GPIOFUNCNOUTINVSEL]{GPIO\_FUNC\regindex{n}\_OUT\_INV\_SEL} 和  \hyperref[fielddesc:GPIOFUNCNOUTSEL]{GPIO\_FUNC\regindex{n}\_OUT\_SEL} 的值可能会被硬件修改。在 GPIO 退出 ETM 任务通道控制后，建议对上述寄存器重新进行配置。
\end{itemize}
\end{tiplisting}

GPIO 有八个事件通道，每个事件通道可以生成的 ETM 事件有：

\begin{itemize}
    \item GPIO\_EVT\_CH\regindex{x}\_RISE\_EDGE：表示对应 GPIO 滤波器（见图  \ref{fig:iomux-gpio-matrix-overview-detail}）输出信号出现上升沿。
    \item GPIO\_EVT\_CH\regindex{x}\_FALL\_EDGE：表示对应 GPIO 滤波器（见图  \ref{fig:iomux-gpio-matrix-overview-detail}）输出信号出现下降沿。
    \item GPIO\_EVT\_CH\regindex{x}\_ANY\_EDGE：表示对应 GPIO 滤波器（见图  \ref{fig:iomux-gpio-matrix-overview-detail}）输出信号发生翻转。
\end{itemize}

事件通道具体配置如下：

\begin{itemize}
    \item 置位 \hyperref[fielddesc:GPIOSDETMCH0EVENTEN]{GPIO\_EXT\_ETM\_CH\regindex{x}\_EVENT\_EN} 使能事件通道 \regindex{x} (0 \verb+~+ 7)。
    \item 配置 \hyperref[fielddesc:GPIOSDETMCH0EVENTSEL]{GPIO\_EXT\_ETM\_CH\regindex{x}\_EVENT\_SEL} 为 \regindex{y} (0 \verb+~+ 54)，即选择 55 个 GPIO 中的一个 GPIO。
\end{itemize}

\begin{tiplisting}
 一个 GPIO 可被一个或多个事件通道选择。
\end{tiplisting}

GPIO 的 ETM 事件可以用来触发 GPIO 的 ETM 任务。例如，事件通道 0 选择 GPIO0，GPIO1 选择任务通道 0，并且 GPIO\_EVT\_CH\regindex{0}\_RISE\_EDGE 事件用于触发 GPIO\_TASK\_CH\regindex{0}\_TOGGLE 任务。当通过 GPIO0 向芯片输入一个方波信号时，芯片通过 GPIO1 输出一个二分频的方波信号。
