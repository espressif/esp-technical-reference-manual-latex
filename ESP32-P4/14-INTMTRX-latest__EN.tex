\chapter{Interrupt Matrix}
\label{mod:intmtrx}

\section{Overview}

The interrupt matrix embedded in \chipname{} independently routes peripheral interrupt sources to the ESP-RISC-V CPU's peripheral interrupts to timely inform HP CPU0 or HP CPU1 to process the coming interrupts.

Peripheral interrupt sources must be routed to HP CPU0 or HP CPU1 through the interrupt matrix for the following reasons:
\begin{itemize}
    \item Each HP CPU has 32 interrupt inputs, but the \chipname{} provides 128 peripheral interrupt sources. To connect all sources to the CPUs, the interrupt matrix is required.
    \item The interrupt matrix allows a single peripheral interrupt source to be mapped to any HP CPU0 or HP CPU1 interrupt input, depending on application requirements.
\end{itemize}

\begin{tiplisting}
This chapter focuses on how to map peripheral interrupt sources to HP CPU0/HP CPU1 interrupts. For more details about interrupt configuration, interrupt vectors, and interrupt handling operations recommended by the ISA, please refer to Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}.
\end{tiplisting}
\vspace{-1em}



\section{Interrupt Terminology in \chipname{}}\label{sec:intmtx-int-terminology}

The following terms related to interrupts are defined in the context of the \textit{\chipname{} Technical Reference Manual} to help readers better understand this document:

\subsection{Interrupt}

An interrupt is an event or condition that temporarily suspends the CPU's current execution to execute an interrupt service routine (ISR). This mechanism enables the CPU to respond immediately to important events.

The \textit{\chipname{} Technical Reference Manual} uses the term ``interrupt" in a broader sense to refer to both interrupt signal and interrupt source.

\subsection{Interrupt Signal/Interrupt Source}

Both interrupt signal and interrupt source describe the same signal from different points in the system.

From the perspective of peripherals, interrupt signals are generated by the peripheral's internal interrupt sources and are sent to the interrupt matrix.

From the perspective of the interrupt matrix, it receives the interrupt signals sent from peripherals and considers them interrupt sources. The interrupt matrix then outputs CPU peripheral interrupt signals to the CPU.

From the perspective of the CPU, the interrupt signals from the interrupt matrix become interrupt sources and are sent to the CPU core together with the core local interrupt sources.

\section{Features}

The interrupt matrix embedded in the \chipname{} has the following features:

\begin{itemize}
    \item 128 peripheral interrupt sources accepted as input
    \item 32 HP CPU0 peripheral interrupts and 32 HP CPU1 peripheral interrupts generated to HP CPU as output
    \item Current interrupt status query for peripheral interrupt sources
    \item Multiple interrupt sources mapped to a single HP CPU0 or HP CPU1 interrupt (i.e., shared interrupts)
    \item Remapping of HP CPU0 or HP CPU1 User Mode interrupts to Machine Mode interrupts
\end{itemize}

Figure \ref{fig:interrupt-matrix-structure} shows the structure of the interrupt matrix.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{\modulefiles/figures/14-INTMTRX-struct.png}
    \caption{Interrupt Matrix Structure}
    \label{fig:interrupt-matrix-structure}
\end{figure}

All interrupts generated by the peripheral interrupt sources can be handled by HP CPU0 or HP CPU1. Users can configure \hyperref[sec: cpu0-intmtrx-reg]{HP CPU0 interrupt registers} (``Core0 Interrupt Reg" module in Figure \ref{fig:interrupt-matrix-structure}) to assign peripheral interrupt sources to HP CPU0, or configure \hyperref[sec: cpu1-intmtrx-reg]{HP CPU1 interrupt registers} (``Core1 Interrupt Reg" module in Figure \ref{fig:interrupt-matrix-structure}) to assign peripheral interrupt sources to HP CPU1. Peripheral interrupt sources can be mapped to both HP CPU0 and HP CPU1 at the same time, allowing both to receive the interrupts.


\section{Functional Description}

\subsection{Peripheral Interrupt Sources}

\chipname{} has 128 peripheral interrupt sources in total. Table \ref{tab:peripheral-interrupt-config-status-source} lists all these sources and their mapping/status registers.


\begin{itemize}
    \item Column ``No.": Peripheral interrupt source number, which can be 0 \verb+~+ 82, 84 \verb+~+ 98, 100 \verb+~+ 111, 113 \verb+~+ 130 (83, 99 and 112 are reserved).
    \item Column ``Chapter": The chapter in which the interrupt source is described in detail.
    \item Column ``Interrupt Source": Name of the peripheral interrupt source.
    \item Column ``Interrupt Source Mapping Register": Registers used for routing the peripheral interrupt sources to HP CPU peripheral interrupts.
    \item Column ``Interrupt Status Register": Registers used for indicating the interrupt status of peripheral interrupt sources.
    \begin{itemize}
        \item Column ``Interrupt Status Register - Bit": Bit position in status register, indicating the interrupt status.
        \item Column ``Interrupt Status Register - Name": Name of status registers.
    \end{itemize}
\end{itemize}

The registers in column ``Interrupt Source Mapping Register" and column ``Interrupt Status Register" correspond to the peripheral interrupt source in column ``Interrupt Source". For example, the interrupt source mapping register for interrupt source RMT\_INTR is \hyperref[regdesc:CORE0RMTINTMAPREG]{CORE\regindex{x}\_RMT\_INT\_MAP\_REG}, and its status register is \hyperref[regdesc:CORE0INTRSTATUSREG1REG]{CORE\regindex{x}\_INTR\_STATUS\_REG\_1\_REG}.

Note that CORE\regindex{x} in the table can be CORE0 (HP CPU0) or CORE1 (HP CPU1).

\subsection{Assign Peripheral Interrupt Source to HP CPU Interrupt}

In this section, the following terms are used to describe the operation of the interrupt matrix.

\begin{itemize}

\item \regindex{SOURCE}: Stands for a peripheral interrupt source in Table \ref{tab:peripheral-interrupt-config-status-source}.
\item \hyperref[fielddesc:CORE0SOURCEMAP]{CORE\regindex{x}\_\regindex{SOURCE}\_MAP}: Stands for an interrupt source mapping register for the peripheral interrupt source (\regindex{SOURCE}) of HP CPU\regindex{x}.
\item Num\_P: The index of HP CPU interrupts, which can be 16 \verb+~+ 47.
\item Interrupt\_P: Stands for the HP CPU interrupt numbered as Num\_P.

\end{itemize}

\subsubsection{Assign One Peripheral Interrupt Source (\regindex{SOURCE}) to HP CPU\regindex{x}}

Setting the corresponding source mapping register \hyperref[fielddesc:CORE0SOURCEMAP]{CORE\regindex{x}\_\regindex{SOURCE}\_MAP} of \regindex{SOURCE} to Num\_P assigns this interrupt source to Interrupt\_P of HP CPU\regindex{x}.

\subsubsection{Assign Multiple Peripheral Interrupt Sources (\regindex{SOURCE}) to HP CPU\regindex{x}}

Setting the corresponding source mapping register \hyperref[fielddesc:CORE0SOURCEMAP]{CORE\regindex{x}\_\regindex{SOURCE}\_MAP} of each interrupt source to the same Num\_P assigns multiple sources to the same Interrupt\_P of HP CPU\regindex{x}. Any of these sources can trigger CPU Interrupt\_P of HP CPU\regindex{x}. In other words, the CPU Interrupt\_P will be shared between these interrupt sources. When an interrupt signal is generated, HP CPU\regindex{x} should check the interrupt status registers to determine which peripheral generated the interrupt. For more information, see Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}.

\subsubsection{Disable HP CPU\regindex{x} Peripheral Interrupt Source (\regindex{SOURCE})}

Writing 0 to the \hyperref[fielddesc:CORE0SOURCEMAP]{CORE\regindex{x}\_\regindex{SOURCE}\_MAP} register disables the corresponding interrupt source.

\subsection{Query Current Interrupt Status of HP CPU\regindex{x} Peripheral Interrupt Source}

After enabling peripheral interrupt sources, users can query the current interrupt status of HP CPU\regindex{x} peripheral interrupt sources by reading the bit value in \hyperref[fielddesc:CORE0INTRSTATUS0]{CORE\regindex{x}\_INTR\_STATUS\_\regindex{n}} (read only). For the mapping between \hyperref[fielddesc:CORE0INTRSTATUS0]{CORE\regindex{x}\_INTR\_STATUS\_\regindex{n}} and peripheral interrupt sources, please refer to Table \ref{tab:peripheral-interrupt-config-status-source}.

\subsection{Interrupt Remapping}

The RISC-V HP CPU\regindex{x} (x can be 0 or 1) of \chipname{} supports Machine Mode and User Mode.

HP CPU\regindex{x} provides a total of 32 peripheral interrupts. Each peripheral interrupt can be delegated by HP CPU\regindex{x} as either a Machine-mode interrupt or a User-mode interrupt.

Under normal conditions, when HP CPU\regindex{x} is running in Machine Mode, it can respond only to Machine-mode interrupts and cannot respond to User-mode interrupts. When HP CPU\regindex{x} is running in User Mode, it can respond to both Machine-mode and User-mode interrupts.

To allow HP CPU\regindex{x} to indirectly respond to User-mode interrupts while operating in Machine Mode, the system introduces the Interrupt Remap mechanism.


Interrupt remapping allows software to redirect HP CPU\regindex{x} interrupts originally assigned to User Mode to a single specified Machine-mode interrupt. In this way, HP CPU\regindex{x} can indirectly respond to User-mode interrupts while running in Machine Mode.



Interrupt remapping features:
\begin{itemize}
    \item Enable interrupt remapping: Each peripheral interrupt source can be independently configured to participate in interrupt remapping. For example, setting \hyperref[fielddesc:CORE0SOURCESRCPASSINSEC]{CORE\regindex{x}\_\regindex{SOURCE}\_SRC\_PASS\_IN\_SEC} to 1 enables interrupt remapping for interrupt matrix source \regindex{SOURCE}.
    \item Specify the target Machine-mode interrupt: All remapped User-mode interrupt signals are uniformly mapped to the same specified Machine-mode interrupt. For example, writing a value n to \hyperref[regdesc:CORE0INTRSIGIDXASSERTINSECREG]{CORE\regindex{x}\_INTR\_SIG\_IDX\_ASSERT\_IN\_SEC\_REG} selects the Machine-mode interrupt with interrupt number n as the remapping target.
    \item Access restrictions for remapping-related registers: the following registers and fields related to interrupt remapping are writable only when HP CPU\regindex{x} is operating in Machine Mode:
  \begin{itemize}
    \item \hyperref[regdesc:CORE0INTRSIGIDXASSERTINSECREG]{CORE\regindex{x}\_INTR\_SIG\_IDX\_ASSERT\_IN\_SEC\_REG}
    \item \hyperref[fielddesc:CORE0SOURCESRCPASSINSEC]{CORE\regindex{x}\_\regindex{SOURCE}\_SRC\_PASS\_IN\_SEC}
    \item \hyperref[fielddesc:CORE0SOURCESRCINSECFLAG]{CORE\regindex{x}\_\regindex{SOURCE}\_SRC\_IN\_SEC\_FLAG}
    \item When \hyperref[fielddesc:CORE0SOURCESRCINSECFLAG]{CORE\regindex{x}\_\regindex{SOURCE}\_SRC\_IN\_SEC\_FLAG} is set to 1,
          \hyperref[fielddesc:CORE0SOURCEMAP]{CORE\regindex{x}\_\regindex{SOURCE}\_MAP} is also writable only in Machine Mode.
  \end{itemize}
\end{itemize}

When interrupt source \regindex{SOURCE} has been mapped to the peripheral interrupt signal \regindex{n} of CORE\regindex{x}, interrupt remapping for source \regindex{SOURCE} occurs only when all of the following conditions are met:
\begin{itemize}
  \item \hyperref[fielddesc:CORE0SOURCESRCPASSINSEC]{CORE\regindex{x}\_\regindex{SOURCE}\_SRC\_PASS\_IN\_SEC} is set to 1
  \item CORE\regindex{x} is currently operating in Machine Mode
  \item The peripheral interrupt signal \regindex{n} of CORE\regindex{x} is internally delegated to User Mode by the CPU
  \item The target interrupt signal specified by \hyperref[regdesc:CORE0INTRSIGIDXASSERTINSECREG]{CORE\regindex{x}\_INTR\_SIG\_IDX\_ASSERT\_IN\_SEC\_REG} is internally delegated to Machine Mode by the CPU
\end{itemize}
