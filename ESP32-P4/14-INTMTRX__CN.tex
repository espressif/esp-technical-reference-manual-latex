\hypertarget{intmtrx}{}
\chapter{中断矩阵}
\label{mod:intmtrx}

\section{概述}


\chipname{} 中断矩阵将任一外部中断源单独映射到 ESP-RISC-V CPU 的任一外部中断上，以便在外设中断信号产生后，及时通知 HP CPU0 或 HP CPU1 进行处理。

外部中断源必须经中断矩阵分配至 HP CPU0/HP CPU1 外部中断，主要是因为：
\begin{itemize}
    \item \chipname{} 有 126 个外部中断源，但 HP CPU0/HP CPU1 只支持 32 个中断。因此，将这些外部中断源映射至 HP CPU0 中断或 HP CPU1 必须使用中断矩阵。
    \item 通过中断矩阵，可以根据应用需要，将一个外部中断源映射至多个 HP CPU0 中断或 HP CPU1 中断。
\end{itemize}

\begin{tiplisting}
本章节只涉及将外部中断源映射到 CPU0 或 CPU1 中断，关于中断配置、中断向量表、中断服务程序推荐处理机制请参考章节 \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}。
\end{tiplisting}
\vspace{-1em}

\section{\chipname{} 中断术语}\label{sec:intmtx-int-terminology}

下列有关中断的术语是在《\chipname{} 技术参考手册》的语境下定义的，旨在帮助读者更好地理解本文档。

\subsection{中断}

中断指的是在特定事件或条件下，CPU 中止当前执行去处理优先级更高的任务。中断机制使 CPU 能够迅速响应特定的事件。

《\chipname{} 技术参考手册》里涉及的“中断”是个广义的概念，可以泛指中断信号或中断源。

\subsection{中断信号/中断源}

中断信号和中断源只是定义的角度不同，实际意思相同。

从外设的角度来看，中断信号是由外设的内部中断源产生的，并发送到中断矩阵。

从中断矩阵的角度来看，它接收来自外设发送的中断信号，并将其视为中断源。然后，中断矩阵将 CPU 外部中断信号发送到 CPU。

从 CPU 的角度来看，来自中断矩阵的中断信号又变成中断源，与核心本地中断源一起发送给 CPU 中断控制器。

\section{特性}

\chipname{} 的中断矩阵模块具有如下特性：

\begin{itemize}
    \item 接收 126 个外部中断源作为输入
    \item 生成 32 个 HP CPU0 的外部中断和 32 个 HP CPU1 的外部中断作为输出
    \item 支持查询外部中断源当前的中断状态
    \item 支持将多个中断源映射到单个 HP CPU0 中断或 HP CPU1 中断（即共享中断）
\end{itemize}

中断矩阵的结构如图 \ref{fig:interrupt-matrix-structure} 所示。
\begin{figure}[H]
    \centering
    {\includegraphics[width=0.8\textwidth]{\modulefiles/figures/14-INTMTRX-struct.png}}
    \caption{中断矩阵结构图}
    \label{fig:interrupt-matrix-structure}
\end{figure}

所有外部中断源产生的中断信号均可由 HP CPU0 或 HP CPU1 进行处理。配置 \hyperref[sec: cpu0-intmtrx-reg]{HP CPU0 中断寄存器}（图 \ref{fig:interrupt-matrix-structure} Core0 Interrupt Reg 模块）可将外部中断源分配给 HP CPU0 的外部中断，此时外部中断源产生的中断信号会由 HP CPU0 响应处理。同理，配置 \hyperref[sec: cpu1-intmtrx-reg]{HP CPU1 中断寄存器}（图 \ref{fig:interrupt-matrix-structure} Core1 Interrupt Reg 模块）可将中断信号交由 HP CPU1 响应处理。也可将外部中断源同时分配给 HP CPU0 和 HP CPU1, 此时 HP CPU0 和 HP CPU1 都会接收到中断信号。


\section{功能描述}

\subsection{外部中断源}

\chipname{} 共有 126 个有效外部中断源。表 \ref{tab:peripheral-interrupt-config-status-source} 列出了所有外部中断源及其对应的中断源映射寄存器与中断状态寄存器。


\begin{itemize}
    \item ``No."：表示外部中断源序号，范围：0 \verb+~+ 98， 100 \verb+~+ 111， 113 \verb+~+ 127（99 和 112 保留）
    \item ``章节"：详细描述外部中断源的章节
    \item ``中断源"：外部中断源名称
    \item ``中断源映射寄存器"：用于将外部中断源分配至 HP CPU 外部中断
    \item ``中断状态寄存器"：用于读取中断源的中断状态
    \begin{itemize}
        \item ``中断状态寄存器 - 位"：表示在中断状态寄存器中的比特位置，用于记录相应中断源的状态
        \item ``中断状态寄存器 - 名称"：表示中断状态寄存器的名称
    \end{itemize}
\end{itemize}

中断源与同一行中断源映射寄存器和中断状态寄存器一一对应，例如：中断源 RMT\_INTR 对应的中断源映射寄存器为 \hyperref[regdesc:CORE0RMTINTMAPREG]{CORE\regindex{x}\_RMT\_INT\_MAP\_REG}，对应的中断状态寄存器为 \hyperref[regdesc:CORE0INTRSTATUSREG1REG]{CORE\regindex{x}\_INTR\_STATUS\_REG\_1\_REG}。

注意，表格中的 CORE\regindex{x} 可以是 CORE0 (HP CPU0) 或 CORE1 (HP CPU1)。

\subsection{分配外部中断源至 HP CPU 外部中断}

在本小节中，我们将使用以下术语描述中断矩阵相关操作：

\begin{itemize}

\item Source\_\regindex{Y}：代表某个外部中断源，其中 \regindex{Y} 为中断源编号，详见表 \ref{tab:peripheral-interrupt-config-status-source}。
\item \hyperref[regdesc:CORE0SOURCEregindexYMAPREG]{CORE\regindex{x}\_SOURCE\_\regindex{Y}\_MAP\_REG}：HP CPU\regindex{x} 外部中断源 (Source\_\regindex{Y}) 的中断源映射寄存器。
\item Num\_P： HP CPU 中断编号，范围： 16 \verb+~+ 47。
\item Interrupt\_P：表示中断号为 Num\_P 的 HP CPU 中断。

\end{itemize}
\subsubsection{分配一个外部中断源 Source\_\regindex{Y} 至 HP CPU\regindex{x} 外部中断}

将外部中断源 Source\_\regindex{Y} 对应的寄存器 \hyperref[regdesc:CORE0SOURCEregindexYMAPREG]{CORE\regindex{x}\_SOURCE\_\regindex{Y}\_MAP\_REG} 配成 Num\_P，即可将该中断源分配至序号为 Num\_P 的 HP CPU\regindex{x} 中断 (Interrupt\_P)。

\subsubsection{分配多个外部中断源 SOURCE\_\regindex{Y} 至 HP CPU\regindex{x} 外部中断}
将各个中断源对应的寄存器 \hyperref[regdesc:CORE0SOURCEregindexYMAPREG]{CORE\regindex{x}\_SOURCE\_\regindex{Y}\_MAP\_REG} 均配置成相同的 Num\_P，即可将多个中断源 SOURCE\_\regindex{Y} 分配至同一 HP CPU\regindex{x} 外部中断 Interrupt\_P。上述任一外设中断均会触发 HP CPU\regindex{x} 外部中断 Interrupt\_P。待中断触发后， HP CPU\regindex{x} 需查询中断状态寄存器，判断产生中断的外设。 更多信息，见章节 \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}。

\subsubsection{关闭 HP CPU\regindex{x} 外部中断源 Source\_\regindex{Y}}

将中断源对应的寄存器 \hyperref[regdesc:CORE0SOURCEregindexYMAPREG]{CORE\regindex{x}\_SOURCE\_\regindex{Y}\_MAP\_REG} 配置成 0，即可关闭外部中断源。

\subsubsection{查询 HP CPU\regindex{x} 外部中断源当前的中断状态}
配置开启外设中断源后，读取寄存器 \hyperref[fielddesc:CORE0INTRSTATUS0]{CORE\regindex{x}\_INTR\_STATUS\_\regindex{n}\_REG}（只读）中特定位的值可以获取 HP CPU\regindex{x} 外部中断源当前的中断状态。寄存器 \hyperref[fielddesc:CORE0INTRSTATUS0]{CORE\regindex{x}\_INTR\_STATUS\_\regindex{n}\_REG} 与外部中断源的对应关系如表 \ref{tab:peripheral-interrupt-config-status-source} 所示。
