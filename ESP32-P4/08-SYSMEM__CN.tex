\chapter{系统和存储器}
\label{mod:sysmem}

\section{概述}

\chipname{} 集成了两个处理器，包括：
\begin{itemize}
    \item 一个高性能 32 位 RISC-V 双核处理器 (HP CPU)，五级流水线，主频达 \iftagged{ESP32-P4-latest}{400}{360} MHz。
    \item 一个低功耗 32 位 RISC-V 单核处理器 (LP CPU)，两级流水线，主频达 40 MHz。
\end{itemize}

所有的内部存储器、外部存储器以及外设均分布在 HP CPU 和 LP CPU 的总线上。

\section{主要特性}

\begin{itemize}
    \item {\bfseries 地址空间}
    \begin{itemize}
	    \item 896 KB HP 内部存储器空间，可被 HP CPU 的指令总线或数据总线通过 cache 进行访问
        \item 8 KB HP 内部存储器空间，可被 HP CPU 的指令总线或数据总线访问
        \item 48 KB LP 内部存储器空间，可被 LP CPU 的指令总线或数据总线进行零等待访问
    	\item 1256 KB 外设地址空间
    	\item 64 MB 外部 flash 虚拟地址空间，可被指令总线或数据总线访问
        \item 64 MB 外部 RAM 虚拟地址空间，可被指令总线或数据总线访问
    	\item 768 KB 内部 DMA 地址空间
    	\item 128 MB 外部 DMA 地址空间
    \end{itemize}
    \item {\bfseries 内部存储器}
    \begin{itemize}
        \item 128 KB HP ROM
	    \item 768 KB HP L2MEM
        \item 8 KB HP SPM（暂存内存）
        \item 16 KB LP ROM
        \item 32 KB LP SRAM
	\end{itemize}
    \item {\bfseries 外部存储器}
    \begin{itemize}
        \item 最大支持 64 MB 外部 flash 和 RAM
	\end{itemize}
    \item \textbf{外设空间}
    \begin{itemize}
        \item 总计 96 个模块/外设
    \end{itemize}
    \item {\bfseries GDMA}
    \begin{itemize}
        \item 7 个模块/外设支持 GDMA-AHB
        \item 6 个模块/外设支持 GDMA-AXI
    \end{itemize}
\end{itemize}

\section{功能描述}
\subsection{ 地址映射}

图 \ref{fig:sysmem-address-mapping-structure} 展示了系统结构与地址映射结构。所有非保留地址均可通过指令总线和数据总线进行访问，即指令总线和数据总线访问的地址空间一样。
HP CPU 和 LP CPU 的数据总线与指令总线均为小端序。除 HP CPU DBUS 为 128 位数据宽度外，其他总线均为 32 位数据宽度。HP CPU 可以通过数据总线进行单字节、双字节、四字节的数据访问，也可以在 AI 指令 (AI Instruction) 中使用最多 16 字节对齐。
LP CPU 可以通过数据总线进行单字节、双字节、四字节的数据访问。

HP CPU 支持：
\begin{itemize}
    \item 直接通过数据总线和指令总线访问 HP SPM。
    \item 通过 cache 访问映射到地址空间的内部存储器和外部存储器。
    \begin{itemize}
        \item 128 KB HP ROM (0x4FC0\_0000 \verb+~+ 0x4FC1\_FFFF)
        \item 768 KB HP L2MEM (0x4FF0\_0000 \verb+~+ 0x4FFB\_FFFF)
        \item 64 MB 外部 flash (0x4000\_0000 \verb+~+ 0x43FF\_FFFF)
        \item 64 MB 外部 RAM  (0x4800\_0000 \verb+~+ 0x4BFF\_FFFF)
    \end{itemize}
    \begin{tiplisting}
    软件可以以 cache-able 或 noncache-able 的方式使用以上 ``0x4" 开头的地址。其中 cache-able 访问方式是指通过 cache 进行访问；noncache-able 访问方式是不通过 cache 进行直接访问，与下文直接访问以 ``0x8" 开头的地址类似，但直接访问 ``0x8" 开头的地址多用于调试。cache-able 访问还是 noncache-able 访问则是由 CPU PMA (Physical Memory Attribution) 控制。
    \end{tiplisting}
    \item 直接访问映射到地址空间的内部存储器和外部存储器，带宽相对通过 cache 访问较慢。
    \begin{itemize}
        \item 128 KB HP ROM (0x8FC0\_0000 \verb+~+ 0x8FC1\_FFFF)
        \item 768 KB HP L2MEM (0x8FF0\_0000 \verb+~+ 0x8FFB\_FFFF)
        \item 32 KB LP SRAM (0x5010\_8000 \verb+~+ 0x5010\_FFFF)
        \item 64 MB 外部 flash (0x8000\_0000 \verb+~+ 0x83FF\_FFFF)
        \item 64 MB 外部 RAM (0x8800\_0000 \verb+~+ 0x8BFF\_FFFF)
    \end{itemize}
    \item 直接通过数据总线访问模块/外设。
    \begin{itemize}
        \item HP CPU 外设
        \item HP 外设
        \item LP 外设
    \end{itemize}
\end{itemize}

LP CPU 支持：
\begin{itemize}
    \item 直接通过数据总线和指令总线访问 LP SRAM 和 LP ROM。
    \item 支持访问内部存储器和外部存储器。
    \begin{itemize}
        \item 128 KB HP ROM (0x4FC0\_0000 \verb+~+ 0x4FC1\_FFFF)
        \item 768 KB HP L2MEM (0x4FF0\_0000 \verb+~+ 0x4FFB\_FFFF)
        \item 64 MB 外部 flash (0x4000\_0000 \verb+~+ 0x43FF\_FFFF)
        \item 64 MB 外部 RAM  (0x4800\_0000 \verb+~+ 0x4BFF\_FFFF)
    \end{itemize}
    \item 直接通过数据总线访问模块/外设。
    \begin{itemize}
        \item HP CPU 外设
        \item HP 外设
        \item LP 外设
    \end{itemize}
\end{itemize}

\subsection{ 内部存储器}
\label{para:sysmem-internal-memory}

\chipname{} 的内部存储器包括如下类型：
\begin{itemize}
    \item \textbf{HP ROM (128 KB)}：HP 系统的只读存储器，不可编程。用于存放系统底层软件的 ROM 代码和只读数据。HP CPU 通过 l2 cache 访问 HP ROM，访问频率为 HP CPU 频率的一半。
    \item \textbf{HP L2MEM (768 KB)}：易失性存储器，可被 HP CPU、LP CPU 或 DMA 外设访问。访问频率为 HP CPU 频率的一半。HP L2MEM 可配置为在 Light-sleep 模式下保持供电，因此可用于数据保留或寄存器备份。
    \item \textbf{LP ROM (16 KB)}：LP 系统的只读存储器。用于存放 LP CPU 启动代码以及一些系统基础函数。LP ROM 可被 LP CPU 零等待访问。
    \item \textbf{LP SRAM (32 KB)}：易失性存储器，可被 HP CPU 和 LP CPU 访问。访问频率与 LP CPU 频率相同。
    \item \textbf{HP SPM (8 KB)}：易失性存储器，可被 HP CPU 访问，每两个周期完成一次访问。
\end{itemize}

{\bfseries 1. HP ROM}

HP ROM 的容量为 128 KB，属于只读存储器，由 HP CPU 使用指令总线或数据总线通过以下地址进行访问，见表 \ref{tab:sysmem-memory}：
\begin{itemize}
    \item  0x4FC0\_0000（可配置为 cache-able 或 noncache-able）\verb+~+ 0x4FC1\_FFFF（可配置为 cache-able 或 noncache-able）。
    \item 0x8FC0\_0000（CPU 直接访问）\verb+~+ 0x8FC1\_FFFF（CPU 直接访问）。
\end{itemize}

{\bfseries  2. HP L2MEM}
\label{other:sysmem-internal-sram-0}

HP L2MEM 的容量为 768 KB，属于可读可写存储器，由 HP CPU，LP CPU 使用指令总线或数据总线或 DMA 主机通过 AHB 矩阵或 AXI 矩阵通过以下地址进行访问，见表 \ref{tab:sysmem-memory}：
\begin{itemize}
    \item 0x4FF0\_0000（可配置为 cache-able 或 noncache-able） \verb+~+ 0x4FFB\_FFFF（可配置为 cache-able 或 noncache-able）。
    \item 0x8FF0\_0000（CPU 直接访问）\verb+~+ 0x8FFB\_FFFF（CPU 直接访问）。
\end{itemize}

\tagged{ESP32-P4-latest}{%
L2MEM 支持多种容量划分配置：L2 RAM 与 L2 cache 共享 768 KB 总容量，可在以下四种配置中选择其一：
\begin{itemize}
    \item Config 0：L2 RAM 256 KB，L2 cache 512 KB
    \item Config 1：L2 RAM 512 KB，L2 cache 256 KB
    \item Config 2：L2 RAM 640 KB，L2 cache 128 KB
    \item Config 3：L2 RAM 768 KB，L2 cache 0 KB
\end{itemize}
上述四种配置下 HP L2MEM 的地址布局如图 \ref{fig:sysmem_l2mem_address_eco5} 所示。

    \caption{HP L2MEM 地址布局}

HP L2MEM 可被以下外设访问：
\begin{itemize}
    \item 全速 USB2.0、高速 USB2.0、GMAC、SDMMC、TRACE、GDMA-AHB 和 SYSMON 通过 AHB 矩阵访问。
    \item GDMA、GDMA-AXI、2D-DMA 和 H264 编码器通过 AXI 矩阵访问。
\end{itemize}

HP L2MEM 支持纠错码校验 (Error Correction Code check)，即 ECC 校验，可以纠正 1 位内存翻转错误。如果发生 1 位翻转错误，HP L2MEM 中的数据仍然有效。在 ECC 检查之前，需要初始化 HP L2MEM。

HP L2MEM 由 6 个存储单元组成，每个为 128 KB。初始化过程可以为每个单元单独配置，以优化功耗。建议按顺序逐一初始化存储单元，不建议同时初始化所有单元。初始化整个 768 KB HP L2MEM 会导致过多的功耗，触发欠压。

\textbf{配置步骤}

以单元 0 为例进行说明，其他单元的操作与单元 0 相同。
\begin{itemize}
    \item 置位 HP\_SYSTEM\_L2\_MEM\_UNIT0\_REFRESH\_EN，并清零 HP\_SYSTEM\_L2\_MEM\_REFRESH\_CNT\_RESET，开始初始化 HP L2MEM。
    \item 读取 HP\_SYSTEM\_L2\_MEM\_UNIT0\_REFRESH\_DONE 的值。
    \item HP\_SYSTEM\_L2\_MEM\_UNIT0\_REFRESH\_DONE 的值变为 1 后，置位 HP\_SYSTEM\_L2\_RAM\_UNIT0\_ECC\_EN 开始 ECC 校验。
    \item  HP\_SYSTEM\_L2\_MEM\_UNIT0\_REFRESH\_DONE 的值变为 1 之后，建议清零 HP\_SYSTEM\_L2\_MEM\_UNIT0\_REFRESH\_EN 并置位 HP\_SYSTEM\_L2\_MEM\_REFRESH\_CNT\_RESET，结束初始化。
\end{itemize}
在初始化过程中，避免访问初始化中的 HP L2MEM 单元。

{\bfseries 3. LP ROM}

LP ROM 的容量为 16 KB，属于只读存储器，由 LP CPU 使用指令总线或数据总线通过共用地址 (0x5010\_0000 \verb+~+ 0x5010\_3FFF) 进行访问，如表  \ref{tab:sysmem-memory} 所示。

{\bfseries 4. LP SRAM}

LP SRAM 的容量为 32 KB，属于可读可写存储器，由 HP CPU 使用 AHB 矩阵或 LP CPU 使用指令总线或数据总线通过共用地址 (0x5010\_8000 \verb+~+ 0x5010\_FFFF) 进行访问，如表  \ref{tab:sysmem-memory} 所示。LP SRAM 支持原子操作。

{\bfseries 5. HP SPM}

HP SPM 的容量为 8 KB，属于可读可写存储器，由 HP CPU 使用指令总线或数据总线通过共用地址 (0x3010\_0000 \verb+~+ 0x3010\_1FFF) 进行访问，如表  \ref{tab:sysmem-memory} 所示。HP SPM 支持 HP CPU 原子操作。

HP SPM 支持奇偶校验，检测 1 位内存翻转错误。在奇偶校验之前，需要先初始化 HP SPM。

\textbf{配置步骤}
\begin{itemize}
    \item 置位 HP\_SYSTEM\_SPM\_INIT\_EN，并清零 HP\_SYSTEM\_SPM\_INIT\_CNT\_RESET，开始初始化 HP SPM。
    \item 读取 HP\_SYSTEM\_SPM\_INIT\_DONE 的值。
    \item HP\_SYSTEM\_SPM\_INIT\_DONE 的值变为 1 后，置位 HP\_SYSTEM\_HP\_SPM\_PARITY\_CHECK\_EN 开始奇偶校验。
    \item HP\_SYSTEM\_SPM\_INIT\_CNT\_RESET 的值变为 1 之后，建议清零 HP\_SYSTEM\_SPM\_INIT\_EN 并置位 HP\_SYSTEM\_SPM\_INIT\_CNT\_RESET，结束初始化。
\end{itemize}
在初始化过程中，避免访问 HP SPM。

\subsection{ 外部存储器}
\label{para:sysmem-external-memory}

\chipname{} 支持以 SPI、Dual SPI、Quad SPI、QPI 等接口形式连接外部 flash，支持以 OPI、HPI 等接口形式连接外部 RAM。\chipname{} 还支持以下安全机制，用以保护用户存储在外部 flash 和 RAM 中的数据：
\begin{itemize}
    \item 支持基于 XTS-AES 的硬件手动和自动加解密功能，从而保护开发者外部 flash 中的程序和数据。
    \item 支持基于 XTS-AES 的自动加解密功能，从而保护开发者外部 RAM 中的数据。
\end{itemize}

\subsubsection{外部存储器地址映射}

HP CPU 可使用 cache-able 或 noncache-able 的方式通过地址段 0x4000\_0000 \verb+~+ 0x43FF\_FFFF 或 0x4800\_0000 \verb+~+ 0x4BFF\_FFFF 访问外部 flash 或外部 RAM。HP CPU 也可以通过地址段 0x8000\_0000 \verb+~+ 0x83FF\_FFFF 或 0x8800\_0000 \verb+~+ 0x8BFF\_FFFF 直接访问外部 flash 或外部 RAM，该方式相对通过 cache 的访问较慢，因此通常用于调试。

HP CPU 访问外部存储器时，将根据内存管理单元 (Memory Management Unit, MMU) 中的信息，把 HP CPU 的地址映射为具体的实地址：
\begin{itemize}
    \item (0x4000\_0000 \verb+~+ 0x43FF\_FFFF) 映射为 HP CPU 通过 cache 访问外部 flash 的实地址。
    \item (0x4800\_0000 \verb+~+ 0x4BFF\_FFFF) 映射为 HP CPU 通过 cache 访问外部 RAM 的实地址。
    \item (0x8000\_0000 \verb+~+ 0x83FF\_FFFF) 映射为 HP CPU 直接访问外部 flash 的实地址。
    \item (0x8800\_0000 \verb+~+ 0x8BFF\_FFFF) 映射为 HP CPU 直接访问外部 RAM 的实地址。
\end{itemize}
经过地址映射，\chipname{} 最大支持 64 MB 的外部 flash 和 64 MB 的外部 RAM。
请注意，指令总线地址空间 (64 MB) 和数据总线地址空间 (64 MB) 是共用的。

\subsubsection{cache}

如图 \ref{fig:sysmem-cache-arch} 所示，\chipname{} 采用两级 cache 系统，具有如下特性：
\begin{itemize}
    \item 一级 cache (l1 cache) 包括：
    \begin{itemize}
        \item 指令 cache (icache)：大小为 16 KB，块大小为 64 B，四路组相联
        \item 数据 cache (dcache)：大小为 64 KB，块大小为 64 B，两路组相联，支持透写 (write-through) 和回写 (write-back) 两种写策略
    \end{itemize}
    \item 二级 cache (l2 cache)：大小为 128 KB/256 KB/512 KB，块大小为 64 B/128 B，八路组相联
    \item 支持 cache-able 访问和 noncache-able 访问，可通过 CPU PMA 配置访问属性
    \item 支持预取 (preload) 功能
    \item 支持锁定 (lock) 功能
    \item 支持关键字优先 (critical word first) 和提前重启 (early restart)
\end{itemize}

\subsubsection{cache 操作}

\chipname{} l1 cache 和 l2 cache 支持以下操作：
\begin{enumerate}
    \item \textbf{回写 (Write-back)}（仅 dcache 和 l2 cache 支持该功能）：
    \begin{itemize}
        \item 清除标记存储器中的脏位，并将新数据更新到外部存储器。
        \item 回写操作完成后，新数据将被更新到外部存储器中。
        \item 可以选择是否对 cache 中的数据作失效处理。
        \item 如果没有选择失效处理，那么 CPU 可以直接从 cache 读取/写入数据，而不会发生缓存缺失。
    \end{itemize}

    \item \textbf{清除 (Clean)}（仅 dcache 和 l2 cache 支持该功能）：
    \begin{itemize}
        \item 清除标记存储器中的脏位，但不把新数据更新到外部存储器。
        \item 清理操作完成后，外部存储器中存储的仍然是旧数据，而 cache 会保留新的数据，但 cache 并不知道其存储的为新数据。
        \item 然后 CPU 可以直接从 cache 读取/写入数据，而不会发生缓存缺失。
    \end{itemize}

    \item \textbf{失效 (Invalidate)}：
    \begin{itemize}
        \item 删除 cache 中的有效数据。
        \item 该操作完成后，删除的数据将仅存于外部存储器中。
        \item 如果 HP CPU 接着去访问该数据，那么需要访问外部存储器。
        \item 该操作包括两种类型：
        \begin{itemize}
            \item 手动失效 (Manual-Invalidate)：仅对 cache 中落入指定区域的地址对应的数据做失效处理。
            \item 整体失效 (Invalidate-All)：会对 cache 中的所有数据做失效处理。
        \end{itemize}
    \end{itemize}

    \item \textbf{预取 (Preload)}：
    \begin{itemize}
        \item 将指令和数据提前加载到 cache 中。
        \item 预取操作的最小单位为 1 个缓存块。
        \item 该操作包括两种类型：
        \begin{itemize}
            \item 手动预取 (Manual-Preload)：硬件按软件指定的虚地址预取一段连续的数据。
            \item 自动预取 (Auto-Preload)：硬件根据当前命中/缺失（取决于配置）的地址，自动预取一段连续的数据。
        \end{itemize}
    \end{itemize}

    \item \textbf{锁定/解锁 (Lock/Unlock)}：
    \begin{itemize}
        \item 保护 cache 中的数据不被替换掉。
        \item 该锁定操作包括两种类型：
        \begin{itemize}
            \item 预锁定：在填充缺失数据到 cache 时，如果该数据落在指定区域，cache 则将该数据锁定，未落入指定区域的数据不会被锁定。
            \item 手动锁定：检查 cache 中的数据，并将落在指定区域的数据锁定，未落入指定区域的数据不会被锁定。
        \end{itemize}
        \item 当缺失发生时，cache 会优先替换掉未被锁定的那一路的数据，因此锁定区域的数据会一直保存在 cache 中。
        \item 但当所有路都被锁定时, cache 将进行正常替换，就像所有路都没有被锁定一样。
        \item 解锁是锁定的逆操作，但解锁只有手动解锁。
        \item 请注意，手动失效操作只对未被锁定的数据起作用。如果想对已锁定的数据执行手动失效操作，请先解锁这些数据。
    \end{itemize}
\end{enumerate}

\subsection{ DMA 地址空间}

\chipname{} 支持 AHB 通用直接内存访问 (GDMA-AHB)、AXI 通用直接内存访问 (GDMA-AXI) 和 VDMA。均可提供直接内存访问服务。在  \chipname{} 中，高速 USB 2.0、全速 USB2.0、GDMAC、Trace0/1、VDMA、2D-DMA、H264 编码器等主机模块也都内置了 DMA 功能模块，也可以提供直接内存访问服务。

上述所有支持 DMA 的主机模块都可以访问：
\begin{itemize}
    \item HP L2MEM (0x4FF0\_0000\verb+~+0x4FFB\_FFFF)。
    \item 外部 flash (0x4000\_0000 \verb+~+ 0x43FF\_FFFF)。
    \item 外部 RAM (0x4800\_0000 \verb+~+ 0x4BFF\_FFFF)。
    \end{itemize}
此外，VDMA 主机 1 还可以访问：
\begin{itemize}
    \item MIPI CSI 内存 (0x5010\_4000 \verb+~+ 0x5010\_4FFF)。
    \item MIPI DSI 内存 (0x5010\_5000 \verb+~+ 0x5010\_5FFF)。
\end{itemize}

\begin{tiplisting}
当使用 DMA 访问任何存储器时，都需要获取对应的访问权限，否则访问将会失败。关于权限管理的更多信息，请参考章节  \ref{mod:permctrl} \textit{\nameref{mod:permctrl}}。
\end{tiplisting}
