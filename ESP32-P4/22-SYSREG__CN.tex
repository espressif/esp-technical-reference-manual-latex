\chapter{系统寄存器 (SYSREG)}
\label{mod:sysreg}
\hypertarget{sysreg}{}

\section{概述}

\chipname{} 支持丰富的辅助功能，可由专用寄存器进行配置。本章将主要介绍这些寄存器。有关具体功能的介绍，请前往对应章节。

\section{功能描述}

\subsection{HP 系统寄存器}

\subsubsection{外部存储器加密/解密配置}

寄存器 \hyperref[regdesc:HPSYSTEMCRYPTOCTRLREG]{HP\_SYSTEM\_CRYPTO\_CTRL\_REG} 用于配置外部存储器（外部 Flash 和 RAM）的加解密选项。详细信息请参见章节 \ref{mod:extmemencr} \textit{\nameref{mod:extmemencr}}。

\subsubsection{HP 外设时钟配置与电源控制}

以下寄存器用于控制对应外设中存储器的电源信号：
\begin{itemize}
    \item RSA：\hyperref[regdesc:HPSYSTEMHPRSAPDCTRLREG]{HP\_SYSTEM\_HP\_RSA\_PD\_CTRL\_REG}
    \item ECC：\hyperref[regdesc:HPSYSTEMHPECCPDCTRLREG]{HP\_SYSTEM\_HP\_ECC\_PD\_CTRL\_REG}
    \item UART：\hyperref[regdesc:HPSYSTEMHPUARTPDCTRLREG]{HP\_SYSTEM\_HP\_UART\_PD\_CTRL\_REG}
\end{itemize}

寄存器 \hyperref[regdesc:HPSYSTEMHPPERIMEMCLKFORCEONREG]{HP\_SYSTEM\_HP\_PERI\_MEM\_CLK\_FORCE\_ON\_REG} 中的以下字段用于控制对应外设中存储器的时钟门控信号：
\begin{itemize}
    \item GDMA：\hyperref[fielddesc:HPSYSTEMGDMAMEMCLKFORCEON]{HP\_SYSTEM\_GDMA\_MEM\_CLK\_FORCE\_ON}
    \item 比特调节器 (BitScrambler)：
    \begin{itemize}
        \item TX 存储器：\hyperref[fielddesc:HPSYSTEMBITSCRAMBLERTXMEMCLKFORCEON]{HP\_SYSTEM\_BITSCRAMBLER\_TX\_MEM\_CLK\_FORCE\_ON}
        \item RX 存储器：\hyperref[fielddesc:HPSYSTEMBITSCRAMBLERRXMEMCLKFORCEON]{HP\_SYSTEM\_BITSCRAMBLER\_RX\_MEM\_CLK\_FORCE\_ON}
    \end{itemize}
    \item RMT：\hyperref[fielddesc:HPSYSTEMRMTMEMCLKFORCEON]{HP\_SYSTEM\_RMT\_MEM\_CLK\_FORCE\_ON}
\end{itemize}

\subsubsection{HP Cache 时钟与复位配置}

\chipname{} 允许软件直接控制 HP cache 的时钟与复位信号。

寄存器 \hyperref[regdesc:HPSYSTEMHPCACHECLKCONFIGREG]{HP\_SYSTEM\_HP\_CACHE\_CLK\_CONFIG\_REG} 中的以下字段用于控制不同 HP cache 的时钟信号，将相应字段写为 1 可使能对应 cache 的时钟。
\begin{itemize}
    \item L1 ichache 0：\hyperref[fielddesc:HPSYSTEML1I0CACHECLKON]{HP\_SYSTEM\_L1\_I0\_CACHE\_CLK\_ON}
    \item L1 ichache 1：\hyperref[fielddesc:HPSYSTEML1I1CACHECLKON]{HP\_SYSTEM\_L1\_I1\_CACHE\_CLK\_ON}
    \item L1 dcache：\hyperref[fielddesc:HPSYSTEML1DCACHECLKON]{HP\_SYSTEM\_L1\_D\_CACHE\_CLK\_ON}
    \item L2 cache：\hyperref[fielddesc:HPSYSTEML2CACHECLKON]{HP\_SYSTEM\_L2\_CACHE\_CLK\_ON}
\end{itemize}

寄存器 \hyperref[regdesc:HPSYSTEMCACHERESETCONFIGREG]{HP\_SYSTEM\_CACHE\_RESET\_CONFIG\_REG} 中的以下字段用于控制不同 HP cache 的复位信号，可通过向相应字段先写 1 后写 0 实现复位。
\begin{itemize}
    \item L1 ichache 0：\hyperref[fielddesc:HPSYSTEML1I0CACHERESET]{HP\_SYSTEM\_L1\_I0\_CACHE\_RESET}
    \item L1 ichache 1：\hyperref[fielddesc:HPSYSTEML1I1CACHERESET]{HP\_SYSTEM\_L1\_I1\_CACHE\_RESET}
    \item L1 dcache：\hyperref[fielddesc:HPSYSTEML1DCACHERESET]{HP\_SYSTEM\_L1\_D\_CACHE\_RESET}
\end{itemize}

\subsubsection{HP SPM 与 HP L2MEM 时钟配置}

以下寄存器字段用于控制 HP \hyperref[mod:sysmem]{SPM}（暂存内存）与 HP \hyperref[mod:sysmem]{L2MEM} 的时钟信号，将相应字段写为 1 可强制使能对应存储器的时钟。
\begin{itemize}
    \item HP SPM：\hyperref[fielddesc:HPSYSTEMHPSPMCLKFORCEON]{HP\_SYSTEM\_HP\_SPM\_CLK\_FORCE\_ON}
    \item L2MEM：\hyperref[fielddesc:HPSYSTEML2MEMCLKFORCEON]{HP\_SYSTEM\_L2\_MEM\_CLK\_FORCE\_ON}
    \item L2ROM：\hyperref[fielddesc:HPSYSTEML2ROMCLKFORCEON]{HP\_SYSTEM\_L2\_ROM\_CLK\_FORCE\_ON}
\end{itemize}


\subsubsection{HP SPM 奇偶校验配置}

\chipname{} 在 HP SPM 中实现了奇偶校验功能，用于检测单比特内存翻转错误。

该功能可通过以下寄存器进行配置与控制：
\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEMHPSPMPARITYCHECKCTRLREG]{HP\_SYSTEM\_HP\_SPM\_PARITY\_CHECK\_CTRL\_REG}：写 1 可使能 HP SPM 的奇偶校验。
    \item \hyperref[regdesc:HPSYSTEMHPSPMINITREG]{HP\_SYSTEM\_HP\_SPM\_INIT\_REG}：在使能奇偶校验前配置 HP SPM 的初始化。
    \item 中断（\hyperref[tab:sysreg-int-sources]{HP\_SPM\_PARITY\_ERR\_INT}）相关寄存器：
    \begin{itemize}
        \item \hyperref[regdesc:HPSYSTEMHPSPMINTRAWREG]{HP\_SYSTEM\_HP\_SPM\_INT\_RAW\_REG}：\hyperref[tab:sysreg-int-sources]{HP\_SPM\_PARITY\_ERR\_INT} 的原始中断状态
        \item \hyperref[regdesc:HPSYSTEMHPSPMINTSTREG]{HP\_SYSTEM\_HP\_SPM\_INT\_ST\_REG}：\hyperref[tab:sysreg-int-sources]{HP\_SPM\_PARITY\_ERR\_INT} 的屏蔽中断状态
        \item \hyperref[regdesc:HPSYSTEMHPSPMINTENAREG]{HP\_SYSTEM\_HP\_SPM\_INT\_ENA\_REG}：写 1 可使能 \hyperref[tab:sysreg-int-sources]{HP\_SPM\_PARITY\_ERR\_INT}
        \item \hyperref[regdesc:HPSYSTEMHPSPMINTCLRREG]{HP\_SYSTEM\_HP\_SPM\_INT\_CLR\_REG}：写 1 可清除 \hyperref[tab:sysreg-int-sources]{HP\_SPM\_PARITY\_ERR\_INT}
    \end{itemize}
    \item \hyperref[regdesc:HPSYSTEMHPSPMPARITYINTRECORDREG]{HP\_SYSTEM\_HP\_SPM\_PARITY\_INT\_RECORD\_REG}：记录 \hyperref[tab:sysreg-int-sources]{HP\_SPM\_PARITY\_ERR\_INT} 发生时的地址。
    \item \hyperref[regdesc:HPSYSTEMHPSPMERRRESPCTRLREG]{HP\_SYSTEM\_HP\_SPM\_ERR\_RESP\_CTRL\_REG}：写 1 后，HP SPM 会将奇偶校验错误报告给 HP CPU 并触发异常。
\end{itemize}

详细配置步骤请参考章节 \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}。


\subsubsection{HP L2MEM ECC 校验配置}

\chipname{} 在 HP L2MEM（包括 L2RAM 和缓存）中实现了 ECC（Error Correction Code，纠错码）校验功能，用于修正单比特内存翻转错误。

该功能可通过以下寄存器进行配置与控制：
\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEML2MEML2RAMECCREG]{HP\_SYSTEM\_L2\_MEM\_L2\_RAM\_ECC\_REG} 和 \hyperref[regdesc:HPSYSTEML2MEML2CACHEECCREG]{HP\_SYSTEM\_L2\_MEM\_L2\_CACHE\_ECC\_REG}：分别写 1 以使能 L2MEM 和缓存的 ECC 校验。
    \item \hyperref[regdesc:HPSYSTEML2MEMREFRESHREG]{HP\_SYSTEM\_L2\_MEM\_REFRESH\_REG}：在使能 ECC 校验前配置 HP L2MEM 的初始化。
    \item 中断（\hyperref[tab:sysreg-int-sources]{L2\_MEM\_ECC\_ERR\_INT}）相关寄存器：
    \begin{itemize}
        \item \hyperref[regdesc:HPSYSTEML2MEMINTRAWREG]{HP\_SYSTEM\_L2\_MEM\_INT\_RAW\_REG}：\hyperref[tab:sysreg-int-sources]{L2\_MEM\_ECC\_ERR\_INT} 的原始中断状态
        \item \hyperref[regdesc:HPSYSTEML2MEMINTSTREG]{HP\_SYSTEM\_L2\_MEM\_INT\_ST\_REG}：\hyperref[tab:sysreg-int-sources]{L2\_MEM\_ECC\_ERR\_INT} 的屏蔽中断状态
        \item \hyperref[regdesc:HPSYSTEML2MEMINTENAREG]{HP\_SYSTEM\_L2\_MEM\_INT\_ENA\_REG}：写 1 可使能 \hyperref[tab:sysreg-int-sources]{L2\_MEM\_ECC\_ERR\_INT}
        \item \hyperref[regdesc:HPSYSTEML2MEMINTCLRREG]{HP\_SYSTEM\_L2\_MEM\_INT\_CLR\_REG}：写 1 可清除 \hyperref[tab:sysreg-int-sources]{L2\_MEM\_ECC\_ERR\_INT}
    \end{itemize}
    \item \hyperref[regdesc:HPSYSTEML2MEMINTRECORD1REG]{HP\_SYSTEM\_L2\_MEM\_INT\_RECORD1\_REG}：记录 \hyperref[tab:sysreg-int-sources]{L2\_MEM\_ECC\_ERR\_INT} 发生时的信息。
    \item \hyperref[regdesc:HPSYSTEML2MEMERRRESPCTRLREG]{HP\_SYSTEM\_L2\_MEM\_ERR\_RESP\_CTRL\_REG}：写 1 后，HP L2MEM 会将 ECC 错误报告给 HP CPU 并触发异常。
\end{itemize}

详细配置步骤请参考章节 \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}。

\subsubsection{比特调节器 (BitScrambler) 配置}

\chipname{} 有部分外设支持 DMA，但有时需要比特调节器以修改传输数据的格式，否则可能出现问题。

通过配置 \hyperref[regdesc:HPSYSTEMBITSCRAMBLERPERISELREG]{HP\_SYSTEM\_BITSCRAMBLER\_PERI\_SEL\_REG}，可将此类外设分别连接到比特调节器的 RX 和 TX 通道。

详细配置步骤请参考章节 \ref{mod:bitscrm} \textit{\nameref{mod:bitscrm}}。

有关支持 DMA 的外设列表，请参考章节 \ref{mod:resclk} \textit{\nameref{mod:resclk}}。

\subsubsection{以太网 MAC 控制}

\chipname{} 可通过以太网 MAC（媒体访问控制器）按照 IEEE 802.3 标准进行数据收发。

相关控制与配置寄存器如下：

\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEMGMACCTRL0REG]{HP\_SYSTEM\_GMAC\_CTRL0\_REG}
    \item \hyperref[regdesc:HPSYSTEMGMACCTRL1REG]{HP\_SYSTEM\_GMAC\_CTRL1\_REG}
    \item \hyperref[regdesc:HPSYSTEMGMACCTRL2REG]{HP\_SYSTEM\_GMAC\_CTRL2\_REG}
\end{itemize}

详细信息请参考章节 \ref{mod:ethernetmac} \textit{\nameref{mod:ethernetmac}}。

\subsubsection{USB OTG 2.0 控制}

\chipname{} 搭载了 USB 2.0 高速 On-The-Go 外设（OTG\_HS），可通过 \hyperref[regdesc:HPSYSTEMUSBOTG20CTRLREG]{HP\_SYSTEM\_USBOTG20\_CTRL\_REG} 进行控制。

详细信息请参考章节 \ref{mod:usbotghs} \textit{\nameref{mod:usbotghs}}。

\subsubsection{CPU 控制与记录}
\label{sec:sysreg-cpu-registers}

\chipname{} 可通过以下系统寄存器配置 CPU 调试功能并记录其状态信息：

\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEMCPUCORESTALLEDSTREG]{HP\_SYSTEM\_CPU\_CORESTALLED\_ST\_REG}：记录 CPU0 和 CPU1 的 core stalled 状态。
    \item \hyperref[regdesc:HPSYSTEMCOREDEBUGRUNSTALLCONFREG]{HP\_SYSTEM\_CORE\_DEBUG\_RUNSTALL\_CONF\_REG}：写 1 以启用 HP CPU 与 LP CPU 之间的调试 runstall 功能。
    \item \hyperref[regdesc:HPSYSTEMHPCOREDMACTIVELPCOREREG]{HP\_SYSTEM\_HP\_CORE\_DMACTIVE\_LPCORE\_REG}：代表 LP CPU 调试模块的 dmactive 值。
\end{itemize}

详细信息请参考章节 \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}。

\subsubsection{HP GPIO 保持控制}

\chipname{} 的每个 GPIO 管脚都具备独立的保持功能，可通过以下寄存器进行控制：

\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEMGPIOOHOLDCTRL0REG]{HP\_SYSTEM\_GPIO\_O\_HOLD\_CTRL0\_REG}：控制 HP GPIO16 \verb+~+ HP GPIO47 的保持信号。
    \item \hyperref[regdesc:HPSYSTEMGPIOOHOLDCTRL1REG]{HP\_SYSTEM\_GPIO\_O\_HOLD\_CTRL1\_REG}：控制 GPIO48 \verb+~+ HP GPIO56 的保持信号。
    \item \hyperref[regdesc:HPSYSTEMGPIODEDHOLDCTRLREG]{HP\_SYSTEM\_GPIO\_DED\_HOLD\_CTRL\_REG}：低 6 位控制 Flash 管脚的保持信号，高 20 位控制 16 线 PSRAM 管脚的保持信号。
\end{itemize}

详细信息请参考章节 \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}}。

\subsubsection{HP GPIO 输出控制}

\chipname{} 的每个 GPIO 管脚均可通过以下寄存器配置为直接输出所需数值（即迟滞功能）：

\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEMGPIOOHYSCTRL0REG]{HP\_SYSTEM\_GPIO\_O\_HYS\_CTRL0\_REG}：配置是否使能 HP GPIO48 \verb+~+ HP GPIO56 的的迟滞功能。
    \item \hyperref[regdesc:HPSYSTEMGPIOOHYSCTRL1REG]{HP\_SYSTEM\_GPIO\_O\_HYS\_CTRL1\_REG}：配置是否使能 HP GPIO48 \verb+~+ HP GPIO56 的迟滞功能。
\end{itemize}

详细信息请参考章节 \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}} 中的 \ref{PadHysEn} 小节。

\subsubsection{非法访问与未授权访问}

当发生非法访问或未授权访问时，\chipname{} 的总线将返回错误响应，并记录访问地址、访问方向以及主机 ID 等信息。

\begin{itemize}
    \item 非法访问：访问未映射的地址空间。
    \item 未授权访问：访问已映射但无访问权限的地址空间。
\end{itemize}

相关信息将记录在以下寄存器中：

\begin{itemize}
    \item HP CPU 和其他 AHB 主机（\hyperref[tab:sysreg-int-sources]{ICM\_CPU\_ADDRHOLE\_INT}）：
    \begin{itemize}
        \item \hyperref[regdesc:HPSYSTEMICMCPUADDRHOLEADDRREG]{HP\_SYSTEM\_ICM\_CPU\_ADDRHOLE\_ADDR\_REG}
        \item \hyperref[regdesc:HPSYSTEMICMCPUADDRHOLEINFOREG]{HP\_SYSTEM\_ICM\_CPU\_ADDRHOLE\_INFO\_REG}
    \end{itemize}
    \item AXI 主机（\hyperref[tab:sysreg-int-sources]{ICM\_SYS\_ADDRHOLE\_INT}）：
    \begin{itemize}
        \item \hyperref[regdesc:HPSYSTEMICMSYSADDRHOLEADDRREG]{HP\_SYSTEM\_ICM\_SYS\_ADDRHOLE\_ADDR\_REG}
        \item \hyperref[regdesc:HPSYSTEMICMSYSADDRHOLEINFOREG]{HP\_SYSTEM\_ICM\_SYS\_ADDRHOLE\_INFO\_REG}
    \end{itemize}
\end{itemize}

有关地址空间映射的详细信息，请参考章节 \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}。

有关访问权限管理的详细信息，请参考章节 \ref{mod:permctrl} \textit{\nameref{mod:permctrl}}。


\subsubsection{HP CPU 总线超时保护}

\chipname{} 支持对 HP CPU 的 IBUS、DBUS 和 AHB 总线进行超时保护。

当传输开始时，超时保护模块内部的计数器会在每个时钟周期内加一。

\begin{itemize}
    \item 如果累计值未超过配置的超时阈值，并且从机已响应此次传输，则计数器将被清零。
    \item 如果累计值超过了配置的超时阈值，且从机仍未响应传输请求，则超时保护模块将接管总线，向 HP CPU 返回错误响应，同时触发中断。
\end{itemize}

相关的控制与配置寄存器说明如下。


\subparagraph{HP CPU IBUS 超时保护}

配置 \hyperref[regdesc:HPSYSTEMHPCOREIBUSTIMEOUTREG]{HP\_SYSTEM\_HP\_CORE\_IBUS\_TIMEOUT\_REG} 可使能超时保护功能，并设置 HP CPU 通过 IBUS 访问 HP SPM、L2MEM 和外部内存（EXT MEM）的超时阈值。

当超时发生时，将触发 \hyperref[tab:sysreg-int-sources]{HP\_CORE\regindex{x}\_IBUS\_TIMEOUT\_INT} 中断。


\subparagraph{HP CPU DBUS 超时保护}

配置 \hyperref[regdesc:HPSYSTEMHPCOREDBUSTIMEOUTREG]{HP\_SYSTEM\_HP\_CORE\_DBUS\_TIMEOUT\_REG} 可使能超时保护功能，并设置 HP CPU 通过 DBUS 访问 HP SPM、L2MEM 和外部内存（EXT MEM）的超时阈值。

当超时发生时，将触发 \hyperref[tab:sysreg-int-sources]{HP\_CORE\regindex{x}\_DBUS\_TIMEOUT\_INT} 中断。

\subparagraph{HP CPU AHB 总线超时保护}

配置 \hyperref[regdesc:HPSYSTEMHPCOREAHBTIMEOUTREG]{HP\_SYSTEM\_HP\_CORE\_AHB\_TIMEOUT\_REG} 可使能超时保护功能，并设置 HP CPU 通过 AHB 总线访问 LP 外设和 HP 外设寄存器的超时阈值。

当超时发生时，将触发 \hyperref[tab:sysreg-int-sources]{HP\_CORE\regindex{x}\_AHB\_TIMEOUT\_INT} 中断。

\subsubsection{AXI 总线矩阵}

\chipname{} 的 AXI 总线矩阵连接多个符合 AXI 协议的主机和从机，可实现高效通信。

\begin{itemize}
    \item 主机包括所有 AHB 主机（通过 AHB2AXI 桥接器连接的 master port1）、HP CPU 缓存（master port2）、DW GDMA\_0（master port6）、DW GDMA\_1（master port7）、AXI GDMA（master port9）、2DDMA（master port11）、H264\_0（master port12）以及 H264\_1（master port13）；
    \item 从机包括 L2MEM（slave port2）、Flash（slave port3）、PSRAM（slave port4）、DSI Host（slave port6）以及 CSI Host（slave port7）。
\end{itemize}

除 master port2 外，AXI 总线矩阵中所有 AXI 主端口均支持 QoS（服务质量）带宽调节功能。此外，AXI 总线矩阵还具备死锁超时保护机制，用于 slave 端口的仲裁优先级配置，以及用于 master 端口的 QoS 仲裁优先级配置。


\subparagraph{死锁超时保护}

\chipname{} 的死锁超时保护功能始终启用。可通过 \hyperref[regdesc:HPSYSTEMICMDLOCKTIMEOUTREG]{HP\_SYSTEM\_ICM\_DLOCK\_TIMEOUT\_REG} 配置 AXI 总线矩阵中所有主机访问 flash、PSRAM、DSI HOST 和 CSI HOST 等 AXI 从机时的超时阈值。

当发生超时时，AXI 总线矩阵会将传输方向、AXI ID、主机 ID 和从机 ID 记录在 \hyperref[regdesc:HPSYSTEMICMDLOCKSTATUSREG]{HP\_SYSTEM\_ICM\_DLOCK\_STATUS\_REG} 中，并同时触发 \hyperref[tab:sysreg-int-sources]{ICM\_DLOCK\_INTR} 中断。


\subparagraph{从机仲裁优先级}

通过 \hyperref[regdesc:HPSYSTEMICMSLVARBPRIORITYREG]{HP\_SYSTEM\_ICM\_SLV\_ARB\_PRIORITY\_REG} 配置 AXI Matrix 中从机的读取数据和响应仲裁优先级。

数值越高，优先级越高。

\subparagraph{主机 QoS 仲裁优先级}

通过 \hyperref[regdesc:HPSYSTEMICMMSTARQOSREG0REG]{HP\_SYSTEM\_ICM\_MST\_ARQOS\_REG0\_REG} 和 \hyperref[regdesc:HPSYSTEMICMMSTAWQOSREG0REG]{HP\_SYSTEM\_ICM\_MST\_AWQOS\_REG0\_REG} 配置 AXI Matrix 中主机的读取地址通道和写入地址通道的 QoS（服务质量）值。

QoS 值越高，仲裁时的优先级越高。

\subsubsection{加速写}

以下寄存器用于配置加速写 (Post Write) 模式：
\begin{itemize}
    \item \hyperref[regdesc:HPSYSTEMPERI1APBPOSTWENREG]{HP\_SYSTEM\_PERI1\_APB\_POSTW\_EN\_REG}：使能外设之间 APB2APB 桥的加速写模式。
    \item \hyperref[regdesc:HPSYSTEMAPBSYNCPOSTWENREG]{HP\_SYSTEM\_APB\_SYNC\_POSTW\_EN\_REG}：使能 DSI、CSI 和 ISP 中 APB2APB 桥的加速写模式。
    \item \hyperref[fielddesc:HPSYSTEMCPUICMH2XPOSTWREN]{HP\_SYSTEM\_CPU\_ICM\_H2X\_POST\_WR\_EN}：使能 AHB2AXI 桥从 AHB 主机到 AXI 从机的加速写模式。
\end{itemize}

\subsection{LP 系统寄存器}

\subsubsection{LP 定时器 Stall 控制}

\hyperref[fielddesc:LPSYSTEMSYSTIMERSTALLSEL]{LP\_SYSTEM\_SYSTIMER\_STALL\_SEL} 字段可选择 LP 定时器的 systimer\_stall 信号源是来自 HP CPU0 或 HP CPU1：

\begin{itemize}
    \item 0：来自 CPU0
    \item 1：来自 CPU1
\end{itemize}

\subsubsection{聚焦离子束 (FIB) 控制}

\hyperref[fielddesc:LPSYSTEMLPFIBSEL]{LP\_SYSTEM\_LP\_FIB\_SEL} 字段可选择欠压检测器 (brownout detector) 和超级看门狗 (super watchdog) 的控制源是来自模拟系统或软件寄存器。

\begin{itemize}
    \item Bit-1：选择欠压检测器的控制源。
        \begin{itemize}
            \item 0：由软件寄存器控制
            \item 1：由模拟系统控制
        \end{itemize}
    \item Bit-2：选择超级看门狗的控制源。
        \begin{itemize}
            \item 0：由软件寄存器控制
            \item 1：由模拟系统控制
        \end{itemize}
\end{itemize}

通过软件寄存器配置 \hyperref[fielddesc:LPSYSTEMDIGFIB]{LP\_SYSTEM\_DIG\_FIB} 来启用欠压检测器或超级看门狗。
\begin{itemize}
    \item Bit-1：写 1 以启用欠压检测器。
    \item Bit-2：写 1 以启用超级看门狗。
\end{itemize}

读取 \hyperref[fielddesc:LPSYSTEMANAFIB]{LP\_SYSTEM\_ANA\_FIB} 来查看欠压检测器或超级看门狗的 FIB 状态。
\begin{itemize}
    \item Bit-1：欠压检测器的 FIB 状态
    \item Bit-2：超级看门狗的 FIB 状态
\end{itemize}

\subsubsection{启动模式}

配置 \hyperref[fielddesc:LPSYSTEMFORCEDOWNLOADBOOT]{LP\_SYSTEM\_FORCE\_DOWNLOAD\_BOOT}，可以强制将芯片从 SPI Boot 模式切换到 Joint Download Boot 模式。

\subsubsection{LP CPU 控制}

通过 \hyperref[regdesc:LPSYSTEMSYSCTRLREG]{LP\_SYSTEM\_SYS\_CTRL\_REG} 配置 LP CPU。具体如下：

\begin{itemize}
    \item \hyperref[fielddesc:LPSYSTEMLPCPUBOOTADDR]{LP\_SYSTEM\_LP\_CPU\_BOOT\_ADDR}：配置 LP CPU 的启动地址。
    \item \hyperref[regdesc:LPSYSTEMLPCPUEXCPCREG]{LP\_SYSTEM\_LP\_CPU\_EXC\_PC\_REG}：表示 LP CPU 的异常 PC 地址。
    \item \hyperref[regdesc:LPSYSTEMLPCPUDBGPCREG]{LP\_SYSTEM\_LP\_CPU\_DBG\_PC\_REG}：表示 LP CPU 的当前 PC 地址。
    \item \hyperref[fielddesc:LPSYSTEMLPCOREERRRESPDIS]{LP\_SYSTEM\_LP\_CORE\_ERR\_RESP\_DIS}：配置是否禁用 LP CPU 各总线的错误响应。
        \begin{itemize}
            \item Bit-0：IBUS
            \item Bit-1：DBUS
            \item Bit-2：AHB 总线
        \end{itemize}
    \item \hyperref[fielddesc:LPSYSTEMLPCOREETMWAKEUPFLAG]{LP\_SYSTEM\_LP\_CORE\_ETM\_WAKEUP\_FLAG}：指示由 LP CPU 唤醒事件触发的 ETM 任务的状态。
    \item \hyperref[fielddesc:LPSYSTEMLPCOREETMWAKEUPFLAGCLR]{LP\_SYSTEM\_LP\_CORE\_ETM\_WAKEUP\_FLAG\_CLR}：写 1 以清除 \hyperref[fielddesc:LPSYSTEMLPCOREETMWAKEUPFLAG]{LP\_SYSTEM\_LP\_CORE\_ETM\_WAKEUP\_FLAG}。
\end{itemize}


\subsubsection{复位控制}

\chipname{} 支持多种不同级别的复位类型，具体可通过系统寄存器进行控制，具体见下方各小节。

有关所有复位类型和复位源的完整列表，请参阅第 \ref{mod:resclk} 章 \textit{\nameref{mod:resclk}}。

\subparagraph{模拟和系统复位源旁路控制}

\hyperref[fielddesc:LPSYSTEMANARSTBYPASS]{LP\_SYSTEM\_ANA\_RST\_BYPASS}：配置是否绕过以下复位源：
\begin{itemize}
    \item 欠压检测器（Brown-out detector）
    \item 超级看门狗（Super watchdog）
    \item 电源毛刺检测器（Power glitch detector）
\end{itemize}

\hyperref[fielddesc:LPSYSTEMSYSRSTBYPASS]{LP\_SYSTEM\_SYS\_RST\_BYPASS}：配置是否绕过以下复位源：
\begin{itemize}
    \item 软件复位
    \item HP 看门狗定时器（HP Watchdog timer）
    \item LP 看门狗定时器（LP Watchdog timer）
    \item eFuse
\end{itemize}

\subparagraph{上电复位旁路控制} \label{sec:power-on-by-pass}

\hyperref[fielddesc:LPSYSTEMHPPORSTNBYPASSCTRL]{LP\_SYSTEM\_HP\_PO\_RSTN\_BYPASS\_CTRL}：配置在上电复位期间是否绕过 HP 电源域的以下复位源。
\begin{itemize}
    \item Bit-0：eFuse
    \item Bit-1：LP 看门狗定时器
    \item Bit-2：USB (JTAG)
    \item Bit-3：USB (UART)
    \item Bit-4：未使用
    \item Bit-5：未使用
    \item Bit-6：HP 看门狗定时器
    \item Bit-7：软件复位
\end{itemize}

\hyperref[fielddesc:LPSYSTEMHPPOCNNTRSTNBYPASSCTRL]{LP\_SYSTEM\_HP\_PO\_CNNT\_RSTN\_BYPASS\_CTRL}：配置在上电复位期间是否绕过 HP CNNT 电源域的以下复位源。
\begin{itemize}
    \item Bit-0：eFuse
    \item Bit-1：LP 看门狗定时器
    \item Bit-2：USB (JTAG)
    \item Bit-3：USB (UART)
    \item Bit-4：未使用
    \item Bit-5：未使用
    \item Bit-6：HP 看门狗定时器
    \item Bit-7：软件复位
\end{itemize}

有关 \chipname{} 电源域的详细信息，请参阅章节 \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}。

\subparagraph{软件系统复位}

通过配置 \hyperref[fielddesc:LPSYSTEMSYSSWRST]{LP\_SYSTEM\_SYS\_SW\_RST} 触发软件系统复位。

\subparagraph{IOMUX 复位}

配置 \hyperref[fielddesc:LPSYSTEMIOMUXRESETDISABLE]{LP\_SYSTEM\_IO\_MUX\_RESET\_DISABLE} 禁用 HP IOMUX 复位。

\subsubsection{唤醒控制}

以下寄存器可控制一些 PMU 唤醒源的行为。

\hyperref[fielddesc:LPSYSTEMUSBOTG20INSUSPEND]{LP\_SYSTEM\_USBOTG20\_IN\_SUSPEND}：配置是否启用来自 USB OTG2.0 的唤醒信号。

\hyperref[fielddesc:LPSYSTEMUSBOTG20WAKEUPCLR]{LP\_SYSTEM\_USBOTG20\_WAKEUP\_CLR}：写 1 以清除来自 USB OTG2.0 的唤醒信号至 PMU。

\subsubsection{LPROM 和 LP SPM 时钟控制}

\hyperref[fielddesc:LPSYSTEMLPSPMROMCLKFORCEON]{LP\_SYSTEM\_LP\_SPM\_ROM\_CLK\_FORCE\_ON}：如果 LP CPU 时钟已开启，则设置此位以强制开启 LP SPM\_ROM 时钟。

\hyperref[fielddesc:LPSYSTEMLPSPMRAMCLKFORCEON]{LP\_SYSTEM\_LP\_SPM\_RAM\_CLK\_FORCE\_ON}：设置此位以强制开启 LP SPM\_RAM 时钟。

\subsubsection{模拟电压比较器控制}
模拟电压比较器\regindex{x} (\regindex{x} = 0, 1) 通过以下寄存器字段进行配置：

\begin{itemize}
    \item \hyperref[fielddesc:LPSYSTEMXPDCOMP0]{LP\_SYSTEM\_XPD\_COMP\regindex{x}} (\regindex{x} = 0, 1)：配置是否启用模拟电压比较器\regindex{x}。
    \item \hyperref[fielddesc:LPSYSTEMMODECOMP0]{LP\_SYSTEM\_MODE\_COMP\regindex{x}} (\regindex{x} = 0, 1)：配置比较模式：
        \begin{itemize}
            \item 0：将主电压与外部参考电压进行比较
            \item 1：将主电压与内部参考电压进行比较
        \end{itemize}
    \item \hyperref[fielddesc:LPSYSTEMDREFCOMP0]{LP\_SYSTEM\_DREF\_COMP\regindex{x}} (\regindex{x} = 0, 1)：配置内部参考电压。
    \begin{itemize}
        \item 可调节电压范围为 0 到 (0.7 * VDDPST\_6) V。
        \item 可调节步长为 0.1 * VDDPST\_6 V。
    \end{itemize}
\end{itemize}

详细信息请参阅章节 \ref{mod:anacom} \textit{\nameref{mod:anacom}}。

\subsubsection{非法访问与未授权访问}

当发生非法访问或未授权访问时，\chipname{} 的总线将返回错误响应，并记录访问地址、访问方向以及主机 ID 等信息。

\begin{itemize}
    \item 非法访问：访问未映射的地址空间。
    \item 未授权访问：访问已映射但无访问权限的地址空间。
\end{itemize}

相关信息将记录在以下寄存器中：

\begin{itemize}
    \item LP CPU 的 IBUS 和 DBUS 访问（\hyperref[tab:sysreg-int-sources]{IDBUS\_ADDRHOLE\_INTR}）：
    \begin{itemize}
            \item \hyperref[regdesc:LPSYSTEMIDBUSADDRHOLEADDRREG]{LP\_SYSTEM\_IDBUS\_ADDRHOLE\_ADDR\_REG}\\
            \item \hyperref[regdesc:LPSYSTEMIDBUSADDRHOLEINFOREG]{LP\_SYSTEM\_IDBUS\_ADDRHOLE\_INFO\_REG}\\
    \end{itemize}
    \item LP CPU 的 AHB 总线或 HP CPU 访问（\hyperref[tab:sysreg-int-sources]{LP\_ADDRHOLE\_INTR}）：
    \begin{itemize}
            \item \hyperref[regdesc:LPSYSTEMLPADDRHOLEADDRREG]{LP\_SYSTEM\_LP\_ADDRHOLE\_ADDR\_REG}\\
            \item \hyperref[regdesc:LPSYSTEMLPADDRHOLEINFOREG]{LP\_SYSTEM\_LP\_ADDRHOLE\_INFO\_REG}\\
    \end{itemize}
\end{itemize}

有关地址空间映射的详细信息，请参考章节 \ref{mod:sysmem} \textit{\nameref{mod:sysmem}}。

有关访问权限管理的详细信息，请参考章节 \ref{mod:permctrl} \textit{\nameref{mod:permctrl}}。


\subsubsection{LP CPU 总线超时保护}

\chipname{} 支持对 LP CPU 的 IBUS、DBUS 和 AHB 总线进行超时保护。

当传输开始时，超时保护模块内部的计数器会在每个时钟周期内加一。

\begin{itemize}
    \item 如果累计值未超过配置的超时阈值，并且从机已响应此次传输，则计数器将被清零。
    \item 如果累计值超过了配置的超时阈值，且从机仍未响应传输请求，则超时保护模块将接管总线，向 LP CPU 返回错误响应，同时触发中断。
\end{itemize}

相关的控制与配置寄存器说明如下。


\subparagraph{LP CPU IBUS 超时保护}

配置 \hyperref[regdesc:LPSYSTEMLPCOREIBUSTIMEOUTREG]{LP\_SYSTEM\_LP\_CORE\_IBUS\_TIMEOUT\_REG} 可使能超时保护功能，并设置 LP CPU 通过 IBUS 访问 LPROM 和 LP SPM 的超时阈值。
当超时发生时，将触发 \hyperref[tab:sysreg-int-sources]{LP\_CORE\_IBUS\_TIMEOUT\_INTR} 中断。

\subparagraph{LP CPU DBUS 超时保护}

配置 \hyperref[regdesc:LPSYSTEMLPCOREDBUSTIMEOUTREG]{LP\_SYSTEM\_LP\_CORE\_DBUS\_TIMEOUT\_REG} 可使能超时保护功能，并设置 LP CPU 通过 DBUS 访问 LPROM 和 LP SPM 的超时阈值。

当超时发生时，将触发 \hyperref[tab:sysreg-int-sources]{LP\_CORE\_DBUS\_TIMEOUT\_INTR} 中断。

\subparagraph{LP CPU AHB 总线超时保护}

配置 \hyperref[regdesc:LPSYSTEMLPCOREAHBTIMEOUTREG]{LP\_SYSTEM\_LP\_CORE\_AHB\_TIMEOUT\_REG} 可使能超时保护功能，并设置 LP CPU 通过 AHB 总线访问 LP 外设、HP 外设和 CPU 外设的超时阈值。

当超时发生时，将触发 \hyperref[tab:sysreg-int-sources]{LP\_CORE\_AHB\_TIMEOUT\_INTR} 中断。


\subsubsection{随机数生成器控制}

寄存器 \hyperref[regdesc:LPSYSTEMRNGDATAREG]{LP\_SYSTEM\_RNG\_DATA\_REG} 和 \hyperref[regdesc:LPSYSTEMRNGCFGREG]{LP\_SYSTEM\_RNG\_CFG\_REG} 用于配置和控制随机数生成器。
