\chapter{Voice Activity Detection (VAD)}
\label{mod:vad}
%%%%%%%%%%%%%%%%%%%%
%%% Introduction %%%
%%%%%%%%%%%%%%%%%%%%

\section{Introduction}

\chipname{} integrates a Voice Activity Detection (VAD) module. This module facilitates the hardware implementation of the first-stage algorithm for voice wake-up and other multimedia functions. Additionally, it provides hardware support for low-power voice wake-up solutions.



%%%%%%%%%%%%%%%%%%%%
%%% Feature List %%%
%%%%%%%%%%%%%%%%%%%%

\section{Feature List}
\label{sec:vad-features}

The VAD module has the following features:

\begin{itemize}
    \item VAD algorithm processes voice data frame by frame, with each frame containing 256 data points. The data sampling rate is 8 kHz, and the bit width is 16 bits
    \item 2 KB buffer that stores up to 4 frames of data
    \item Independent system wake-up source
    \item Configurable interrupt sources
    \item Flexible configuration of algorithm parameters
\end{itemize}

%%% Architectural Overview %%%
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\section{Architectural Overview}
\label{sec:vad-arch-overview}

Figure \ref{fig:vad-sys-diagram} is the block diagram of the \chipname{} VAD module. The VAD module includes:

\begin{itemize}
    \item Energy threshold check
    \item FFT (Fast Fourier Transform) calculation
    \item LTSD (Long-Term Spectral Divergence) calculation
\end{itemize}

\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.7\textwidth]{\modulefiles/figures/vad_arch.pdf}}
    \caption{\chipname{} VAD Architecture}
    \label{fig:vad-sys-diagram}
\end{figure}

The VAD module reads 8 kHz, 16-bit wide audio data from LP I2S memory. It processes the data through three stages: energy detection, FFT, and LTSD. Then the module detects the voice activity status and sends the VAD status signal to the LP I2S module, which is used to generate the corresponding interrupt and wake-up signal.

%%% Functional Description %%%

\section{Functional Description}
\label{sec:vad-func-descr}
The \chipname{}'s VAD module offers flexible and configurable functionality. Users can adjust algorithm parameters and wake-up sources to meet their specific needs.


\subsection{Algorithm Parameter Configuration}
\label{sec:vad-param-config}


\textbf{Number of Initial Frames}

When the VAD module starts running, it first enters an initialization phase before normal operation. During this phase, it automatically learns noise parameters. The duration of the initialization phase depends on the number of initial frames configured by the user. Proper setting of initial frames can effectively suppress noise and significantly reduce false positives. However, if the initialization phase is extended over too many frames, voice activity within these frames may be missed. Therefore, please configure an appropriate number of initial frames based on actual conditions.

It is recommended to place the device in the target environment noise during the initialization, and set the number of initial frames to 100-200 frames.

Users can configure the number of initial frames via \hyperref[fielddesc:LPI2SPARAMINITFRAMENUM]{LP\_I2S\_PARAM\_INIT\_FRAME\_NUM}.

\begin{tiplisting}
It is recommended to let the VAD module learn noise parameters during the initialization. However, the VAD module will also continuously learn and update noise parameters during normal operation. Therefore, in scenarios where the false positive rate during initial frames is not a concern, the number of initial frames can be reduced, allowing the VAD module to enter the normal operation more quickly.
\end{tiplisting}


\textbf{Energy Threshold}

The VAD module sets an energy threshold before the voice activity detection. Setting the threshold can filter out low-energy voice activities (e.g., distant voices) from triggering a wake-up. Appropriate energy threshold setting can effectively reduce false positives and conserve power.

It defines the energy value of a speech frame as the mean square of one-frame data. The VAD module calculates the sum of squares of one-frame data in real-time. If this sum is less than the energy threshold multiplied by 256, subsequent calculations will not be performed. Users can configure the energy threshold via \hyperref[fielddesc:LPI2SPARAMMINENERGY]{LP\_I2S\_PARAM\_MIN\_ENERGY}.


\textbf{Band Energy Check}

The band energy check assesses the proportion of band energy within the overall frequency spectrum. Enabling the band energy check in varying environments can help reduce false positives; however, it may also reduce true positives. Users should determine whether to enable or disable the band energy check based on the environment.

Users can enable or disable the band energy check via \hyperref[fielddesc:LPI2SPARAMSKIPBANDENERGY]{LP\_I2S\_PARAM\_SKIP\_BAND\_ENERGY}.

\begin{itemize}
    \item 0: Enable band energy check
    \item 1: Disable band energy check
\end{itemize}


\textbf{Voice Activity Detection}

The VAD module has two voice activity status: voice activity status and non-voice activity status, managed by voice frame counter speech\_count and non-voice frame counter silent\_count. These counters control transitions between the status as follows:

\begin{itemize}
    \item \textbf{Non-voice activity status:}
    \begin{itemize}
        \item When voice activities are detected in the current frame, the count value of speech\_count increases by 1.
        \item When voice activities are not detected in the current frame, the count value of speech\_count decreases by 1 (down to a minimum of 0).
        \item If the count value of speech\_count is greater than \hyperref[fielddesc:LPI2SPARAMHANGOVERSPEECH]{LP\_I2S\_PARAM\_HANGOVER\_SPEECH}, the VAD module enters the voice activity status and resets silent\_count to 0.
        \item Under the non-voice activity status, the VAD module continuously learns and updates noise information.
    \end{itemize}
    
    \item \textbf{Voice activity status:}
    \begin{itemize}
        \item When voice activities are detected in the current frame, the count value of speech\_count increases by 1, and the count value of silent\_count decreases by 1 (down to a minimum of 0).
        \item When voice activities are not detected in the current frame, the count value of silent\_count increases by 1:
        \begin{itemize}
            \item If the count value of speech\_count is greater than \hyperref[fielddesc:LPI2SPARAMMINSPEECHCOUNT]{LP\_I2S\_PARAM\_MIN\_SPEECH\_COUNT}, and
            
            \begin{itemize}
                \item The count value of silent\_count is greater than \hyperref[fielddesc:LPI2SPARAMHANGOVERSILENT]{LP\_I2S\_PARAM\_HANGOVER\_SILENT}, the VAD module transitions to the non-voice activity status and resets both speech\_count and silent\_count to 0;
                \item The count value of silent\_count is less than \hyperref[fielddesc:LPI2SPARAMHANGOVERSILENT]{LP\_I2S\_PARAM\_HANGOVER\_SILENT}, the VAD module maintains the voice activity status.
            \end{itemize}
            
            \item If the count value of speech\_count is less than or equal to \hyperref[fielddesc:LPI2SPARAMMINSPEECHCOUNT]{LP\_I2S\_PARAM\_MIN\_SPEECH\_COUNT}, the VAD module transitions directly to the non-voice activity status and resets speech\_count to 0.
        \end{itemize}
        \item Under the voice activity status, the VAD module pauses updating noise information. When the count value of speech\_count exceeds \hyperref[fielddesc:LPI2SPARAMMAXSPEECHCOUNT]{LP\_I2S\_PARAM\_MAX\_SPEECH\_COUNT}, indicating stability in the new environment, it resumes learning and updating noise information.
    \end{itemize}
\end{itemize}


\textbf{Status Registers}

The VAD module provides several key operational variables as status registers (read-only) to facilitate user debugging. The status registers include:
\begin{itemize}
    \item \hyperref[fielddesc:LPI2SVADFLAG]{LP\_I2S\_VAD\_FLAG}: Represents the current voice activity detection status. 1 represents the VAD module is in the voice activity status, while 0 represents it is in the non-voice activity status.
    \item \hyperref[fielddesc:LPI2SENERGYENOUGH]{LP\_I2S\_ENERGY\_ENOUGH}: Represents whether the current frame passes the energy threshold check. 1 represents the current frame passes the energy threshold check, while 0 represents not pass.
    \item \hyperref[fielddesc:LPI2SSPEECHCOUNTOB]{LP\_I2S\_SPEECH\_COUNT\_OB}: Represents the count value of the voice frame counter, speech\_count.
    \item \hyperref[fielddesc:LPI2SSILENTCOUNTOB]{LP\_I2S\_SILENT\_COUNT\_OB}: Represents the count value of the non-voice frame counter, silent\_count.
\end{itemize}


\subsection{Wake-up Source Configuration}

When the VAD module is in the voice activity status, it triggers the wake-up source LP\_I2S\_WAKEUP. Users can configure the wake-up source (please refer to Chapter \ref{mod:lpi2s} \textit{\nameref{mod:lpi2s}}), enabling the VAD to become one of the wake-up sources for \chipname{}.

\section{Interrupts}

\chipname{}'s VAD module shares the same interrupt signal LP\_I2S\_INTR with the LP I2S module. The following interrupt sources can generate the interrupt signal:
\begin{itemize}
\item \label{int:LPI2SVADDONEINT}LP\_I2S\_VAD\_DONE\_INT: Triggered when the VAD completes processing one frame of data.
\item \label{int:LPI2SVADRESETDONEINT}LP\_I2S\_VAD\_RESET\_DONE\_INT: Triggered when the VAD is reset.
\end{itemize}

Each interrupt source can be configured by a common set of registers that are described in Section \hyperref[interrupt-config-registers]{\textit{Interrupt Configuration Registers}}. The specific registers can be found in Section \ref{sec:vad-reg-summ} \textit{\nameref{sec:vad-reg-summ}}.

\section{Programming Procedure}
\label{sec:vad-prog-procedures}

\subsection{Automatic Operation Mode}

\begin{enumerate}
    \item Configure the VAD parameters as described in Section \ref{sec:vad-param-config}.
    \item Enable the VAD automatic detection mode by writing 1 to \hyperref[fielddesc:LPI2SVADEN]{LP\_I2S\_VAD\_EN}.
    \item Configure the LP I2S to operate in 16-bit receive mode and start receiving data in 16-bit format. The received data will be stored in the LP I2S memory.
    \item The VAD module will automatically perform operations on each frame of data after the LP I2S receives the complete frame.
    \item (Optional) When the VAD module triggers a wake-up source/interrupt signal, the LP I2S memory already contains the voice frame that triggered the VAD wake-up, along with the previous three voice frames. Users can read these four frames of voice data from the LP I2S memory via the LP core for subsequent processing. For details on the size of the LP I2S memory and data reading methods, please refer to Chapter \ref{mod:lpi2s} \textit{\nameref{mod:lpi2s}} > Section \ref{sec:lpi2s-memory} \textit{\nameref{sec:lpi2s-memory}}.
\end{enumerate}


\subsection{Manual Operation Mode}

\begin{enumerate}
    \item Configure the VAD parameters as described in Section \ref{sec:vad-param-config}.
    \item Manually store the voice data in the LP I2S memory. For details, refer to Chapter \ref{mod:lpi2s} \textit{\nameref{mod:lpi2s}} > Section \ref{sec:lpi2s-memory} \textit{\nameref{sec:lpi2s-memory}}.
    \item With all the voice data stored in the memory, write 1 to \hyperref[fielddesc:LPI2SVADFORCESTART]{LP\_I2S\_VAD\_FORCE\_START}, then the VAD will perform operations on one frame of data.
    \item When the VAD module completes processing one frame of data, \hyperref[int:LPI2SVADDONEINT]{LP\_I2S\_VAD\_DONE\_INT} interrupt will be generated.
\end{enumerate}
