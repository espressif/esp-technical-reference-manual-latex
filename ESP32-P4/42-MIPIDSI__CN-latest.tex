\section{介绍}
MIPI DSI (Mobile Industry Processor Interface Display Serial Interface) 代表移动行业处理器接口显示串行接口，是一个标准接口协议，用于将显示器连接到移动设备，如智能手机、平板电脑和其他嵌入式系统。在 \chipseries{} 中，MIPI DSI 由三个部分组成：一个 DSI 桥接器，一个 MIPI DSI 主控制器，以及一个 MIPI 双向 D-PHY。它提供一个时钟通道和两条数据通道，并支持每个通道最高 1.5 Gbps 的通信速率，支持与符合 DSI 标准的显示器通信。DSI 属于 MIPI 联盟定义的一组通信协议。

DSI 桥接器提供系统内存与 MIPI DSI 主控制器之间的接口。它通过 VDMA 从系统内存接收图像数据，并通过显示像素接口 (DPI) 将图像数据输出到 MIPI DSI 主控制器中。
MIPI DSI 主控制器实现了 MIPI DSI 规范中定义的协议功能。它从 DSI 桥接器接收图像数据，并通过 PHY 协议接口 (PPI) 将图像数据输出到 MIPI 双向 D-PHY。
MIPI 双向 D-PHY 实现了 MIPI D-PHY 接口的双向通用通道的物理层，提供一个时钟通道和两个数据通道。PHY 仅用作主设备。

\section{功能列表}
\label{sec:mipidsi-features}

MIPI DSI 在 \chipname{} 中支持以下功能：
\begin{itemize}
    \item 在从内存读取图像数据时，将 DSI 桥或 VDMA 配置为流控制器
    \item 支持在 DSI 桥中配置 DPI 视频时序
    \item 支持多种输入图像格式：RGB888、RGB666、RGB565、YUV422、YUV420、GRAY
    \item 支持各种 DPI 图像输出格式：RGB888、RGB666、RGB565
    \item 最大 DPI 频率：240 MHz
    \item MIPI DSI 主机符合 MIPI DSI 规范
    \item 符合 MIPI D-PHY 接口规范，修订版 1.1 的 PHY
    \item PHY 包含 PLL
    \item 连续和非连续时钟通道模式
    \item 支持通过数据通道 0 进入逃逸模式
    \item 在 HS TX 中，通道操作范围为 80 Mbps 到 1.5 Gbps
    \item 支持突发和非突发视频模式
    \item 支持通过 APB 接口的命令模式访问显示寄存器
    \item 支持 MIPI 联盟规范中定义的显示命令集 (DCS)
    \item 视频图案生成器
    \item 支持超低功耗模式
\end{itemize}

\section{架构概述}
\label{sec:mipidsi-arch-overview}
图 \ref{figure:mipidsi_arch} 显示了 MIPI DSI 的架构。

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefigures/mipidsi_arch_CN.pdf}
    \caption{MIPI DSI 架构}
    \label{figure:mipidsi_arch}
\end{figure}
\begin{itemize}
    \item \textbf{DSI 桥接器}：DSI 桥接器通过 VDMA 从系统内存接收图像数据，将其转换为 DPI 数据格式，然后输出到 MIPI DSI 主机。
    \item \textbf{MIPI DSI 主机}：MIPI DSI 主机从两个来源接收图像数据：DSI 桥接器或视频图案生成器。然后，接收到的图像数据由数据包处理器根据 MIPI DSI 规范进行打包，并发送到 PHY 接口控制器。最后，图像数据被转换为 PPI 数据格式并输出到 MIPI DSI 双向 D-PHY。MIPI DSI 主机包括：

    \begin{itemize}
        \item \textbf{DPI 接口}：从 DSI 桥接器接收像素数据并将其缓冲到 FIFO。
        \item \textbf{寄存器组}：提供对配置和控制寄存器的访问。
        \item \textbf{APB 到通用接口}：将 APB 操作转换为通用命令。
        \item \textbf{视频图案生成器}：生成视频测试图案。
        \item \textbf{数据包处理器}：安排链路内部的活动，包括数据包构建、生成 ECC CRC 代码、数据包接收等。
        \item \textbf{PHY 接口控制器}：管理 PHY PPI 接口，包括启用低功耗 (LP) 传输/接收、高速 (HS) 传输等。
        \item \textbf{错误管理}：监控并通知 DSI 链路上的错误状态。
    \end{itemize}
    \item \textbf{MIPI DSI D-PHY}：MIPI DSI 双向 D-PHY 实现了具有一个时钟通道和两个数据通道的物理层，连接到外部显示驱动器。MIPI DSI 双向 D-PHY 还包括一个 PLL 以生成 HS TX 时钟。
\end{itemize}

\subsection{DSI 桥接器}
\label{sec:dsi-bridge}

DSI Bridge 通过 VDMA 从系统内存接收图像数据。数据传输流控制器可以通过 \hyperref[fielddesc:DSIBRGDSIDMAFLOWCONTROLLER]{DSI\_BRG\_DSI\_DMA\_FLOW\_CONTROLLER} 配置为 VDMA 或 DSI Bridge。建议使用 VDMA 作为流控制器以获得更好的性能。




当使用 VDMA 作为流控制器接收图像数据时，可以将图像拆分为多个块传输。要启用此功能，请将 \hyperref[fielddesc:DSIBRGDMAMULTIBLKEN]{DSI\_BRG\_DMA\_MULTIBLK\_EN} 设置为 1，并将 \hyperref[fielddesc:DSIBRGDMAFLOWMULTIBLKNUM]{DSI\_BRG\_DMA\_FLOW\_MULTIBLK\_NUM} 配置为所需值。有关块传输的详细说明，请参见 \ref{mod:vdma} \textit{\nameref{mod:vdma}} 的 \textit{传输层次结构} 小节。

\subsection{DSI 主机}

\subsubsection{视频传输}
DSI Host 使用 DPI 作为输入接口。它用于在视频模式下从 HOST 向外设传输实时像素流。

有两种视频传输模式：

\begin{itemize}
    \item \textbf{突发模式}：在突发模式下，一行像素会被缓冲到FIFO中，并在一个单独的数据包中无中断地传输。这使得像素行能够在短时间内发送出去，然后让通道回到低功耗 (LP) 状态，从而降低功耗，并为其他数据传输节省时间。
    \item \textbf{非突发模式}：在非突发模式下，一行像素被分成几个数据包。这种模式用于匹配像素带宽与 DSI 链路带宽，并减少主机和外设的内存需求。非突发模式有两种：
    \begin{itemize}
        \item 带同步脉冲的非突发模式：允许外设准确重建原始视频时序，包括同步脉冲宽度。
        \item 带同步事件的非突发模式：不需要准确重建同步脉冲。
    \end{itemize}
\end{itemize}

如果主机和外设满足以下要求，建议选择突发模式。否则，像素数据可能会丢失，导致显示外设发生故障。
\begin{itemize}
    \item 显示外设支持在一个单一块中接收一行完整像素，并且接收缓冲区不会溢出。
    \item DSI 输出带宽远高于 DPI 输入带宽，该带宽差异使传输通道能够在每一行数据传输完成后进入低功耗 (LP) 状态。
\end{itemize}

如果无法满足任何要求，则应选择非突发模式。在非突发模式下，DSI 主机应配置为匹配 DSI 输出比率和 DPI 输入比率。以下方程显示了传输参数应如何配置（左侧表示 DSI 输出比率，右侧表示 DPI 输入比率）：
\begin{itemize}
    \item 如果启用了空数据包 (\hyperref[fielddesc:DSIHOSTVIDNULLSIZE]{DSI\_HOST\_VID\_NULL\_SIZE} $\neq$ 0)：
    \begin{align*}
    & lanebyteclk\_period \times \hyperref[fielddesc:DSIHOSTVIDNUMCHUNKS]{DSI\_HOST\_VID\_NUM\_CHUNKS} \\
    & \quad \times (\hyperref[fielddesc:DSIHOSTVIDPKTSIZE]{DSI\_HOST\_VID\_PKT\_SIZE} \times bytes\_per\_pixel + 12 \\
    & \quad + \hyperref[fielddesc:DSIHOSTVIDNULLSIZE]{DSI\_HOST\_VID\_NULL\_SIZE}) / number\_of\_lanes \\
    &= dpiclk\_period \times pixels\_per\_line
    \end{align*}
    \item 如果禁用了空数据包 (\hyperref[fielddesc:DSIHOSTVIDNULLSIZE]{DSI\_HOST\_VID\_NULL\_SIZE} $=$ 0)：
    \begin{align*}
    & lanebyteclk\_period \times \hyperref[fielddesc:DSIHOSTVIDNUMCHUNKS]{DSI\_HOST\_VID\_NUM\_CHUNKS} \\
    & \quad \times (\hyperref[fielddesc:DSIHOSTVIDPKTSIZE]{DSI\_HOST\_VID\_PKT\_SIZE} \times bytes\_per\_pixel + 6) \\
    & \quad / number\_of\_lanes = dpiclk\_period \times pixels\_per\_line
    \end{align*}
\end{itemize}

有关 lanebyteclk 和 dpiclk 的定义，请参阅 \ref{sec:clocks-and-display-resolution}。

\subsubsection{命令传输}
\label{sec:command-transmission}
DSI 主机允许通过 APB 从接口，以命令模式传输 DSI 规范中厂商自定义的通用命令。

命令数据包使用 APB 寄存器构建：
\begin{itemize}
    \item \textbf{\hyperref[regdesc:DSIHOSTGENPLDDATAREG]{DSI\_HOST\_GEN\_PLD\_DATA\_REG}}：向该寄存器写入数据会将数据作为命令数据包的有效载荷发送。读取该寄存器会返回回读数据包的有效载荷。

\paragraph{无电源状态 (No-Power)}
无电源 (No-Power) 状态的特征是没有对 PHY 提供任何供电电压。

对于 \chipname{} 的 MIPI DSI，当电源 VDD\_HP\_\regindex{n} (n: 0-3) 和 VDD\_MIPI\_DPHY 关闭时，MIPI BD D-PHY 处于无电源状态。

\paragraph{关闭状态 (Shutdown)}
在关闭状态下，会禁用所有模拟模块，重置所有数字逻辑，DP/DN 和 CP/CN 通道处于高阻抗 (Hi-Z)。要启用 PHY 的关闭模式，将 \hyperref[fielddesc:DSIHOSTPHYRSTZ]{DSI\_HOST\_PHY\_RSTZ} 和 \hyperref[fielddesc:DSIHOSTPHYSHUTDOWNZ]{DSI\_HOST\_PHY\_SHUTDOWNZ} 设置为 0。

\paragraph{PLL 锁定和 AFE 初始化}
在释放 \hyperref[fielddesc:DSIHOSTPHYRSTZ]{DSI\_HOST\_PHY\_RSTZ} 和 \hyperref[fielddesc:DSIHOSTPHYSHUTDOWNZ]{DSI\_HOST\_PHY\_SHUTDOWNZ} 后，PHY 开始初始化序列以启用其正常操作。建议在释放 \hyperref[fielddesc:DSIHOSTPHYRSTZ]{DSI\_HOST\_PHY\_RSTZ} 之前释放 \hyperref[fielddesc:DSIHOSTPHYSHUTDOWNZ]{DSI\_HOST\_PHY\_SHUTDOWNZ}。此外，确保 DSI\_DPHY\_CFG\_CLK 和 DSI\_DPHY\_PLL\_REFCLK（有关详细信息，请参见第 \ref{mod:resclk} 章中的表 \ref{tab:resclk-periph-clks-hp2} \textit{\nameref{mod:resclk}}）在此时可用且稳定。

之后，PLL 变为活动状态，PHY 等待 PLL 锁定。锁定状态由 \hyperref[fielddesc:DSIHOSTPHYLOCK]{DSI\_HOST\_PHY\_LOCK} 指示。

在 PLL 锁定后，开始 AFE 初始化。当 \hyperref[fielddesc:DSIHOSTPHYSTOPSTATECLKLANE]{DSI\_HOST\_PHY\_STOPSTATECLKLANE} \hyperref[fielddesc:DSIHOSTPHYSTOPSTATE0LANE]{DSI\_HOST\_PHY\_STOPSTATE\textcolor{red}{n}LANE} 被置位时，所有初始化步骤完成，通道处于停止状态 (LP-11)。

\paragraph{控制模式 (Control)}
AFE 初始化完成后，MIPI BD D-PHY 默认保持在控制模式，同时在线路上驱动停止状态 LP-11，直至有请求发出。任何请求都必须从该状态开始，并在该状态结束。请求发出后，通道可以进入高速数据传输模式、逃逸模式、超低功耗状态或转向状态。

\paragraph{高速数据传输}
DSI 主机在收到来自 DSI 桥接器的数据后，会发送高速数据传输请求。在高速数据传输模式下，物理层 (PHY) 以突发方式传输图像数据。通道仅在这些突发传输期间以高速模式运行。

如图 \ref{fig:mipidsi-hs} 所示，通道上的突发由三个部分组成：低功耗初始化序列、高速数据载荷和传输结束序列。

     \centering
     \includegraphics[width=1.0\linewidth]{\modulefigures/mipidsi-hs.pdf}
     \caption{MIPI DSI HS 数据传输序列}
     \label{fig:mipidsi-hs}
 \end{figure}

\paragraph{逃逸模式 (Escape)}
逃逸模式允许在数据通道上以低速进行异步通信。在逃逸模式下，通道保持在低功耗状态。 \chipname{} 中的 MIPI DSI 在逃逸模式下支持低功耗数据传输 (LPDT) 和超低功耗状态 (ULPS)。

如果 \hyperref[fielddesc:DSIHOSTLPCMDEN]{DSI\_HOST\_LP\_CMD\_EN} 设置为 1，命令将在 LPDT 中发送。

当 DSI 空闲时，可以通过 \hyperref[regdesc:DSIHOSTPHYULPSCTRLREG]{DSI\_HOST\_PHY\_ULPS\_CTRL\_REG} 将 PHY 驱动到 ULPS 以节省功耗。通道状态可以通过 \hyperref[regdesc:DSIHOSTPHYSTATUSREG]{DSI\_HOST\_PHY\_STATUS\_REG} 观察得知。

\paragraph{转向}
PHY 可以通过转向交换传输方向。这用于接收来自 DSI 设备的命令响应。

\subsubsection{PLL 参数}
\label{sec:pll-parameters}
PLL 的 VCO 输出频率是输入参考时钟 DSI\_DPHY\_PLL\_REFCLK 的频率以及倍增比率 $M$ 和分频比率 $N$ 的函数：

\[
f_{VCO} = \frac{M}{N} \times f_{REFCLK}
\]

该函数必须满足以下限制：

\[
40\,\mathrm{MHz} \geq \frac{f_{REFCLK}}{N} \geq 5\,\mathrm{MHz}
\]

此外，$M$ 必须是偶数。$M$ 和 $N$ 的值可以通过 \hyperref[sec:test-code-0x17]{测试代码0x17} 和 \hyperref[sec:test-code-0x18]{测试代码0x18} 进行配置，配置值等于实际值减 1，如表 \ref{tab:m-n-ratios} 所示。之后，\hyperref[sec:test-code-0x19]{测试代码0x19} 的第 5 位和第 4 位应设置为 1，使 $M$ 和 $N$ 配置生效。

\subsection{时钟和显示分辨率}
\label{sec:clocks-and-display-resolution}

DSI 的时钟架构如图 \ref{figure:mipidsi-clk-arch} 所示。所有时钟为：
\begin{itemize}
    \item \textbf{PCLK}：APB 总线时钟
    \item \textbf{ACLK}：AXI 总线时钟
    \item \textbf{DSI\_DPICLK}：DPI 接口时钟
    \item \textbf{DSI\_DPHY\_CFG\_CLK}：DPHY 配置时钟
    \item \textbf{DSI\_DPHY\_PLL\_REFCLK}：DPHY PLL 输入参考时钟
    \item \textbf{TX\_HS\_CLK}：DPHY PLL 输出时钟
    \item \textbf{TX\_BYTE\_CLK}：高速 (HS) 传输字节时钟，TX\_HS\_CLK 除以 8。
    \item \textbf{TX\_ESC\_CLK}：低功耗 (LP) 逃逸模式传输时钟，从 TX\_BYTE\_CLK 分出。分频比可以通过 \hyperref[fielddesc:DSIHOSTTOCLKDIVISION]{DSI\_HOST\_TO\_CLK\_DIVISION} 配置，允许的最大频率为 20 MHz。
\end{itemize}

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefigures/mipidsi-clk-arch.pdf}
    \caption{MIPI DSI 时钟架构}
    \label{figure:mipidsi-clk-arch}
\end{figure}

DSI 支持连续和非连续时钟通道。
\begin{itemize}
    \item 对于连续时钟通道，需要把 \hyperref[fielddesc:DSIHOSTAUTOCLKLANECTRL]{DSI\_HOST\_AUTO\_CLKLANE\_CTRL} 配置成 1, 把 \hyperref[fielddesc:DSIHOSTPHYTXREQUESTCLKHS]{DSI\_HOST\_PHY\_TXREQUESTCLKHS} 配置成 1
    \item 对于非连续时钟通道，需要把 \hyperref[fielddesc:DSIHOSTAUTOCLKLANECTRL]{DSI\_HOST\_AUTO\_CLKLANE\_CTRL} 配置成 0, 把 \hyperref[fielddesc:DSIHOSTPHYTXREQUESTCLKHS]{DSI\_HOST\_PHY\_TXREQUESTCLKHS} 配置成 1
\end{itemize}

在视频模式传输中，视频时序参数如图 \ref{figure:mipidsi-video-timing} 所示。
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{\modulefigures/mipidsi-video-timing.pdf}
    \caption{视频时序}
    \label{figure:mipidsi-video-timing}
\end{figure}

对于 DPI，视频时序和分辨率可以通过以下 DSI 桥接寄存器字段进行配置。之后，向 \hyperref[fielddesc:DSIBRGDPICONFIGUPDATE]{DSI\_BRG\_DPI\_CONFIG\_UPDATE} 写入 1 使配置生效。
\begin{itemize}
    \item \hyperref[fielddesc:DSIBRGHTOTAL]{DSI\_BRG\_HTOTAL}：以 DSI\_DPICLK 为单位的总水平长度 (HTOTAL)
    \item \hyperref[fielddesc:DSIBRGHDISP]{DSI\_BRG\_HDISP}：以 DSI\_DPICLK 为单位的有效水平长度 (HACT)
    \item \hyperref[fielddesc:DSIBRGHBANK]{DSI\_BRG\_HBANK}：以 DSI\_DPICLK 为单位的水平后沿长度 (HBP)
    \item \hyperref[fielddesc:DSIBRGHSYNC]{DSI\_BRG\_HSYNC}：以 DSI\_DPICLK 为单位的水平同步长度 (HSA)
    \item \hyperref[fielddesc:DSIBRGVTOTAL]{DSI\_BRG\_VTOTAL}：以行数为单位的总垂直长度 (VTOTAL)
    \item \hyperref[fielddesc:DSIBRGVDISP]{DSI\_BRG\_VDISP}：以行数为单位的有效垂直长度(VACT)
    \item \hyperref[fielddesc:DSIBRGVBANK]{DSI\_BRG\_VBANK}：以行数为单位的垂直后沿长度 (VBP)
    \item \hyperref[fielddesc:DSIBRGVSYNC]{DSI\_BRG\_VSYNC}：以行数为单位的垂直同步长度 (VSA)

\end{itemize}


DSI 也需要类似的视频时序和分辨率配置，相关寄存器字段如下：
\begin{itemize}
    \item \hyperref[fielddesc:DSIHOSTVIDHSATIME]{DSI\_HOST\_VID\_HSA\_TIME}：以 TX\_BYTE\_CLK 为单位的水平同步长度 (HSA)
    \item \hyperref[fielddesc:DSIHOSTVIDHBPTIME]{DSI\_HOST\_VID\_HBP\_TIME}：以 TX\_BYTE\_CLK 为单位的水平后沿长度 (HBP)
    \item \hyperref[fielddesc:DSIHOSTVIDHLINETIME]{DSI\_HOST\_VID\_HLINE\_TIME}：以 TX\_BYTE\_CLK 为单位的总水平长度 (HTOTAL)
    \item \hyperref[fielddesc:DSIHOSTVSALINES]{DSI\_HOST\_VSA\_LINES}：以行数为单位的垂直同步长度 (VSA)
    \item \hyperref[fielddesc:DSIHOSTVBPLINES]{DSI\_HOST\_VBP\_LINES}：以行数为单位的垂直后沿长度 (VBP)
    \item \hyperref[fielddesc:DSIHOSTVFPLINES]{DSI\_HOST\_VFP\_LINES}：以行数为单位的垂直前沿长度 (VFP)
    \item \hyperref[fielddesc:DSIHOSTVACTIVELINES]{DSI\_HOST\_VACTIVE\_LINES}：以行数为单位的有效垂直长度 (VACT)
\end{itemize}

DPI 和 DSI 的视频时序配置应匹配，以避免行缓冲区溢出。DPI 和 DSI 中每行的实际时序应相同。

\section{配置过程}
\label{sec:mipidsi-prog-procedures}

\subsection{DSI 桥接配置}
\label{sec:dsi-bridge-configuration}
\begin{itemize}
    \item 配置 DSI Bridge 时钟并释放复位。
    \item 配置输入和输出颜色模式。详细信息请参阅 \hyperref[sec:dsi-bridge]{DSI Bridge}。
    \item 配置视频时序。有关详细信息，请参阅 \hyperref[sec:clocks-and-display-resolution]{时钟和显示分辨率}。
    \item 将 0 写入 \hyperref[fielddesc:DSIBRGDSIDMAFLOWCONTROLLER]{DSI\_BRG\_DSI\_DMA\_FLOW\_CONTROLLER} 设置 VDMA 为流控制器。
    \item 配置 \hyperref[fielddesc:DSIBRGDMABURSTLEN]{DSI\_BRG\_DMA\_BURST\_LEN} DMA 突发长度以匹配 VDMA 的配置。
    \item 配置 \hyperref[fielddesc:DSIBRGDSIRAWBUFALMOSTEMPTYTHRD]{DSI\_BRG\_DSI\_RAW\_BUF\_ALMOST\_EMPTY\_THRD} 为 $1024 - \hyperref[fielddesc:DSIBRGDMABURSTLEN]{DSI\_BRG\_DMA\_BURST\_LEN}$ 获得更好的性能。
    \item 将 1 写入 \hyperref[fielddesc:DSIBRGDPIEN]{DSI\_BRG\_DPI\_EN} 以启用 DPI。
    \item 将 1 写入 \hyperref[fielddesc:DSIBRGDSIEN]{DSI\_BRG\_DSI\_EN} 以启用 DSI Bridge。
    \item 配置 VDMA 并将 VDMA 设置为流控制器。详细信息请参阅 \ref{mod:vdma}。
\end{itemize}
\label{sec:mipidsi-dsi-bridge-programming-proc}

\subsection{D-PHY 配置}
\label{sec:mipidsi-dsi-dphy-programming-proc}
\begin{itemize}
    \item 将 \hyperref[fielddesc:DSIHOSTPHYSHUTDOWNZ]{DSI\_HOST\_PHY\_SHUTDOWNZ} 和 \hyperref[fielddesc:DSIHOSTPHYRSTZ]{DSI\_HOST\_PHY\_RSTZ} 设置为 0，将 \hyperref[fielddesc:DSIHOSTPHYTESTCLR]{DSI\_HOST\_PHY\_TESTCLR} 设置为 1。
    \item 应用适当的 DSI\_DPHY\_CFG\_CLK 和 DSI\_DPHY\_PLL\_REFCLK。
    \item 通过 \hyperref[sec:test-code-0x44]{测试代码 0x44} 配置 hsfreqrange。详细值请参见表 \ref{tab:mipidsi-hsfreq}。
    \item 配置 PLL 参数。详细信息参见 \hyperref[sec:pll-parameters]{PLL 参数}。
    \item 配置 \hyperref[fielddesc:DSIHOSTTOCLKDIVISION]{DSI\_HOST\_TO\_CLK\_DIVISION} 以应用适当的 TX\_ESC\_CLK 分频比。
    \item 将 \hyperref[fielddesc:DSIHOSTPHYSHUTDOWNZ]{DSI\_HOST\_PHY\_SHUTDOWNZ} 和 \hyperref[fielddesc:DSIHOSTPHYRSTZ]{DSI\_HOST\_PHY\_RSTZ} 设置为 1。
    \item 等待 \hyperref[fielddesc:DSIHOSTPHYSTOPSTATE0LANE]{DSI\_HOST\_PHY\_STOPSTATE0LANE} 和 \hyperref[fielddesc:DSIHOSTPHYSTOPSTATE1LANE]{DSI\_HOST\_PHY\_STOPSTATE1LANE} 信号置位。
\end{itemize}

\subsection{MIPI DSI 通用配置}
\label{sec:mipidsi-dsi-common-programming-proc}
\begin{itemize}
    \item 释放全局复位。
    \item 配置 \hyperref[fielddesc:DSIHOSTNLANES]{DSI\_HOST\_N\_LANES} 以定义要使用的通道数量。
    \item 配置视频时序设置。有关详细信息，请参阅 \hyperref[sec:clocks-and-display-resolution]{时钟和显示分辨率}。
    \item 通过 \hyperref[sec:mipidsi-dsi-dphy-programming-proc]{D-PHY 配置} 初始化 DPHY。
    \item 将 \hyperref[fielddesc:DSIHOSTSHUTDOWNZ]{DSI\_HOST\_SHUTDOWNZ} 写入 1 以释放 DSI 主机的关机状态。
\end{itemize}

\subsection{系统内存数据输出流配置}
\begin{itemize}
    \item 参考 \hyperref[sec:dsi-bridge-configuration]{DSI Bridge 配置} 中的方法配置 DSI Bridge。
    \item 参考 \hyperref[sec:mipidsi-dsi-common-programming-proc]{MIPI DSI 通用配置} 配置 DSI Host。
\end{itemize}

\subsection{视频模式输出流配置}
\begin{itemize}
    \item 通过寄存器字段 \hyperref[fielddesc:DSIHOSTVPGORIENTATION]{DSI\_HOST\_VPG\_ORIENTATION}、\hyperref[fielddesc:DSIHOSTVPGMODE]{DSI\_HOST\_VPG\_MODE} 和 \hyperref[fielddesc:DSIHOSTVPGEN]{DSI\_HOST\_VPG\_EN} 配置模式。有关详细信息，请参阅 \hyperref[sec:video-pattern-generator]{视频图案生成器}。
    \item 通过 \hyperref[sec:mipidsi-dsi-common-programming-proc]{MIPI DSI 通用配置} 配置 DSI 主机。
\end{itemize}


\subsection{使用 DCS 命令操作显示寄存器}
\begin{itemize}
    \item 写入显示
        \begin{itemize}
            \item 将数据包有效载荷写入 \hyperref[regdesc:DSIHOSTGENPLDDATAREG]{DSI\_HOST\_GEN\_PLD\_DATA\_REG}。
            \item 将数据包头写入 \hyperref[regdesc:DSIHOSTGENHDRREG]{DSI\_HOST\_GEN\_HDR\_REG}。
        \end{itemize}
    \item 从显示中读取
        \begin{itemize}

        \end{itemize}
\end{itemize}
