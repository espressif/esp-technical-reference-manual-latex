\chapter{Reset and Clock}
\label{mod:resclk}


\section{Reset}
\subsection{Overview}
\chipname{} provides four types of reset that occur at different levels, namely CPU Reset, Core Reset, System Reset, and Chip Reset. All reset types mentioned above (except Chip Reset) preserve the data stored in internal memory. Figure \ref{figure:Reset} shows the scopes of affected subsystems by each type of reset.

\subsection{Architectural Overview}
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{\modulefiles/figures/reset-overview-new.pdf}
    \caption{Reset Types}
    \label{figure:Reset}
\end{figure}
\chipname{}'s Digital System consists of \hyperref[mod:riscvcpu]{High Performance System (HP system)} that includes Digital Core and HP memory, and \hyperref[mod:lowpowm]{Low Power System (LP system)} that includes the Low Power Always On (LP AON) system, LP Core, and LP memory. See Figure \ref{figure:Reset} for details (note that HP memory and LP memory would not be reset, so they are not shown in the figure).

\subsection{Features}
\label{sec:resclk-features}
\begin{itemize}
    \item Four reset types:
    \begin{itemize}
    \item CPU Reset: resets CPU core. HP CPU0, HP CPU1, and LP CPU can be reset independently:
    \begin{itemize}
            \item HP CPU0 will be automatically released from reset after chip power-up, and start execution from CPU Reset Vector (0x30100000).
            \item HP CPU1 is at reset by default after chip power-up, and needs to be manually released from reset by writing 0 to \hyperref[fielddesc:HPSYSCLKRSTRSTENCORE1GLOBAL]{HP\_SYS\_CLKRST\_RST\_EN\_CORE1\_GLOBAL}. Once released from reset, HP CPU1 will start execution from CPU Reset Vector (0x30100000).
            \item LP CPU is at reset after chip power-up, and needs to be manually released from reset by configuring the power management unit (PMU). Once released from reset, LP CPU will start execution from LP\_SYSTEM\_LP\_CPU\_BOOT\_ADDR (0x50100000 by default). For details, see Chapter \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}.
    \end{itemize}
    \item Core Reset: resets the whole digital system except for LP AON. HP core and LP core can be reset independently: HP Core Reset resets HP CPU0, HP CPI1, HP peripherals, HP GPIO, etc., and LP Core Reset resets LP CPU and LP peripherals.
    \item System Reset: resets the whole digital system, including the LP system.
    \item Chip Reset: resets the whole chip.
    \end{itemize}
    \item Software reset and hardware reset:
    \begin{itemize}
        \item Software Reset: triggered via software by configuring the corresponding registers of CPU, see Chapter \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}.
        \item Hardware Reset: triggered directly by the hardware.
    \end{itemize}
\end{itemize}

\subsection{Functional Description}
CPU will be reset immediately when any type of reset above occurs. After the reset is released, users can retrieve reset source codes by reading the following fields:
\begin{itemize}
    \item HP CPU0: \hyperref[fielddesc:LPCLKRSTHPCORE0RESETCAUSE]{LP\_CLKRST\_HPCORE0\_RESET\_CAUSE}
    \item HP CPU1: \hyperref[fielddesc:LPCLKRSTHPCORE0RESETCAUSE]{LP\_CLKRST\_HPCORE1\_RESET\_CAUSE}
    \item LP CPU: \hyperref[fielddesc:LPCLKRSTLPCORERESETCAUSE]{LP\_CLKRST\_LPCORE\_RESET\_CAUSE}
\end{itemize}

\subsection{Peripheral Reset}

Peripherals can be reset individually by configuring corresponding registers, or globally by Core Reset, System Reset, or Chip Reset.

\chipname{} has three groups of peripheral reset registers, prefixed with HP\_SYS\_CLKRST, LP\_CLKRST, and LPPERI. See Section  \ref{sec:resclk-reg-summ} \textit{\nameref{sec:resclk-reg-summ}} for detailed information.

\section{Clock}

\subsection{Overview}

\chipname{} clocks are mainly sourced from oscillator (OSC, including Resistor-Capacitor circuit), crystal (XTAL), and PLL circuit, and then processed by the dividers or selectors, which allows most functional modules to select their working clock according to their power consumption and performance requirements. Figure \ref{figure:Clock} shows the HP and LP system clock structure, including the main clock path and typical peripheral clock generator circuit. \chipname{} has a large number of peripherals, and limited by space, not all peripheral clocks are shown in the figure.

\subsection{Architectural Overview}

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/clock-diagram-new.pdf}
    \caption{HP and LP System Clock}
    \label{figure:Clock}
\end{figure}


\subsection{Features}

\chipname{} clocks can be classified into two types depending on their frequencies:
\begin{itemize}
    \item High speed clocks for devices working at a higher frequency, such as HP CPU0/1 and digital peripherals
    \begin{itemize}
        \item CPLL\_CLK: internal 360 MHz PLL clock. Its reference clock is XTAL\_CLK
        \item MPLL\_CLK: internal 500 MHz PLL clock. Its reference clock is XTAL\_CLK
        \item SPLL\_CLK: internal 480 MHz PLL clock. Its reference clock is XTAL\_CLK
    \end{itemize}
    \item Slow speed clocks for LP system and some peripherals working in low-power mode
    \begin{itemize}
        \item XTAL32K\_CLK: external 32 kHz crystal clock
        \item RC\_SLOW\_CLK: internal 150 kHz slow RC oscillator
        \item OSC\_SLOW\_CLK: external slow clock input through \hyperref[tab:ANA-IO-MUX-EN]{XTAL\_32K\_N}, with a frequency of 32 kHz by default. After configuring this GPIO, also configure the Hold function (see Chapter \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}} >  \ref{PadHold} \textit{\nameref{PadHold}})
        \item XTAL\_CLK: 40 MHz external crystal clock
        \item RC\_FAST\_CLK: internal fast RC oscillator with adjustable frequency (20 MHz by default)
        \item PLL\_LP\_CLK: internal PLL clock with a frequency of 8 MHz by default. Its reference clock can be XTAL32K\_CLK
    \end{itemize}
\end{itemize}

\subsection{Functional Description}

\subsubsection{HP System Clock}

As Figure \ref{figure:Clock} shows, XTAL\_CLK, CPLL\_CLK, and RC\_FAST\_CLK are the multiplexer inputs to generate ROOT\_CLK. For selecting which input, see Table \ref{tab:resclk-cpu-clk-src}.

\begin{longtable}[c]{|l|l| }
       \caption{ROOT\_CLK Clock Source}
		\label{tab:resclk-cpu-clk-src} \\
	    \hline
        \rowcolor{lightgray}
        \textbf{\hyperref[fielddesc:LPCLKRSTHPROOTCLKSRCSEL]{LP\_CLKRST\_HP\_ROOT\_CLK\_SRC\_SEL}} & \textbf{Clock Source} \\

        \hline
        0 & XTAL\_CLK \\
        \hline
        1 & CPLL\_CLK \\
        \hline
        2 & RC\_FAST\_CLK \\
        \hline
\end{longtable}

ROOT\_CLK is then divided by a series of dividers, and sequentially derives:
\begin{itemize}
    \item CPU\_CLK, which drives HP CPUs and their logic
    \item MEM\_CLK, which drives the internal memories (L2 Cache, L2MEM, ROM) and their logic
    \item SYS\_CLK, which is the HP high-speed bus clock that drives the AXI and AHB bus logic
    \item APB\_CLK, which is the HP low-speed bus clock that drives the APB bus logic
\end{itemize}

The above four derived clocks are synchronous SoC clocks, asynchronous to peripheral clocks described in Section \ref{sec:peri_clk} \textit{\nameref{sec:peri_clk}}. Peripheral clocks are asynchronous with each other unless otherwise stated.

The maximum frequencies allowed for these SoC clocks are as follows. Software should ensure that the frequencies of these clocks do not exceed the maximum frequency.
\begin{itemize}
    \item CPU\_CLK: 360 MHz
    \item MEM\_CLK: 200 MHz
    \item SYS\_CLK: 200 MHz
    \item APB\_CLK: 100 MHz
\end{itemize}




Each module has an individual clock gate for the SoC clock it uses. For example, users can enable the clock for HP CPU0 and disable the clock for HP CPU1 by setting the \hyperref[fielddesc:HPSYSCLKRSTCORE0CPUCLKEN]{HP\_SYS\_CLKRST\_CORE0\_CPU\_CLK\_EN} bit and clearing the  \hyperref[fielddesc:HPSYSCLKRSTCORE1CPUCLKEN]{HP\_SYS\_CLKRST\_CORE1\_CPU\_CLK\_EN} bit. ``CPU\_CLK" in bit names indicates that the corresponding clock belongs to the CPU\_CLK group shown in Figure \ref{figure:Clock}.

The internal PLL clocks of \chipname{}, namely MPLL\_CLK and SPLL\_CLK, generate reference clocks for peripherals through a set of parallel clock dividers.

Besides these two PLL clocks, there are two additional PLL clocks dedicated to specific modules:

\begin{itemize}
    \item SDIO\_PLL, which generates three clocks:
        \begin{itemize}
            \item SDIO\_PLL0\_CLK, used for SDIO's high-speed SDIO\_SLF\_CLK (internal signal clock) and the source of SDIO's low-speed clock
            \item SDIO\_PLL1\_CLK, users for SDIO's high-speed SDIO\_DRV\_CLK (output signal driving clock)
            \item SDIO\_PLL2\_CLK, used for SDIO's high-speed SDIO\_SAM\_CLK (input signal sampling clock)
        \end{itemize}
    \item APLL\_CLK, which is dedicated to I2S
\end{itemize}

A typical peripheral clock generation circuit consists of a clock selector, a clock gating unit, and a divider. In high-performance mode, the peripheral can select a high-speed clock source without division for higher processing speed. In low-power mode, the peripheral can select a low-speed clock source or divide it by a large divisor to lower power consumption. When the peripheral is disabled, its clock can be directly disabled.


\begin{tiplisting}
Updates to \hyperref[regdesc:HPSYSCLKRSTROOTCLKCTRL0REG]{HP\_SYS\_CLKRST\_ROOT\_CLK\_CTRL0/1/2/3\_REG} will take effect only after \\\hyperref[fielddesc:HPSYSCLKRSTSOCCLKDIVUPDATE]{HP\_SYS\_CLKRST\_SOC\_CLK\_DIV\_UPDATE} is set.
\end{tiplisting}

LP\_DYN\_SLOW\_CLK and LP\_DYN\_FAST\_CLK can be derived from LP\_SLOW\_CLK or LP\_FAST\_CLK depending on the chip's power mode (see Chapter \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}), and should be able to operate in all power modes to drive LP AON modules. These two clocks are always synchronous with each other.

\begin{itemize}
    \item In Active mode, select LP\_FAST\_CLK as the clock source. In this case, LP\_DYN\_SLOW\_CLK has the same frequency with LP\_SLOW\_CLK, and the same phase with LP\_FAST\_CLK;
    \item In Deep-sleep mode, select the clock source according to PMU configurations. If the LP CPU is off, LP\_SLOW\_CLK is the clock source, and has the same frequency and phase as LP\_DYN\_FAST\_CLK in this case; if the LP CPU is on, LP\_FAST\_CLK is the clock source.
\end{itemize}

LP\_PERI\_CLK is derived from LP\_DYN\_FAST\_CLK and drives all the buses for LP peripherals.

XTAL\_D2\_CLK is derived from XTAL\_CLK divided by 2.

\begin{tiplisting}
To protect the encryption and decryption peripherals from DPA (Differential Power Analysis) attacks, a random divider strategy is implemented for its function clock. Four security levels are available, depending on the range of the random divider. Users can select the security level by configuring \hyperref[fielddesc:HPSYSCLKRSTSECDPACFGSEL]{HP\_SYS\_CLKRST\_SEC\_DPA\_CFG\_SEL}. \\If  \hyperref[fielddesc:HPSYSCLKRSTSECDPACFGSEL]{HP\_SYS\_CLKRST\_SEC\_DPA\_CFG\_SEL} is set to 1, the security level is determined by the configuration of \\\hyperref[fielddesc:HPSYSCLKRSTSECDPALEVEL]{HP\_SYS\_CLKRST\_SEC\_DPA\_LEVEL}. Otherwise, the security level is determined by the value of  EFUSE\_SEC\_DPA\_LEVEL.
\end{tiplisting}


\section{Programming Procedures}
\label{sec:resclk-prog-procedures}

\subsection{HP System Clock Configuration}

\begin{itemize}
    \item Configure the clock divisor for CPU\_CLK: \hyperref[fielddesc:HPSYSCLKRSTCPUCLKDIVNUM]{HP\_SYS\_CLKRST\_CPU\_CLK\_DIV\_NUM} for the integral part, \hyperref[fielddesc:HPSYSCLKRSTCPUCLKDIVNUMERATOR]{HP\_SYS\_CLKRST\_CPU\_CLK\_DIV\_NUMERATOR} for the numerator of the fractional part, and \hyperref[fielddesc:HPSYSCLKRSTCPUCLKDIVDENOMINATOR]{HP\_SYS\_CLKRST\_CPU\_CLK\_DIV\_DENOMINATOR} for the denominator of the fractional part.\\
    Write 1 to \hyperref[fielddesc:HPSYSCLKRSTSOCCLKDIVUPDATE]{HP\_SYS\_CLKRST\_SOC\_CLK\_DIV\_UPDATE} to apply the frequency divisor configurations for CPU\_CLK.
    \item Configure the clock divisor for MEM\_CLK: \hyperref[fielddesc:HPSYSCLKRSTMEMCLKDIVNUM]{HP\_SYS\_CLKRST\_MEM\_CLK\_DIV\_NUM} for the integral part, \hyperref[fielddesc:HPSYSCLKRSTMEMCLKDIVNUMERATOR]{HP\_SYS\_CLKRST\_MEM\_CLK\_DIV\_NUMERATOR} for the numerator of the fractional part, and \hyperref[fielddesc:HPSYSCLKRSTMEMCLKDIVDENOMINATOR]{HP\_SYS\_CLKRST\_MEM\_CLK\_DIV\_DENOMINATOR} for the denominator of the fractional part.\\
    Write 1 to \hyperref[fielddesc:HPSYSCLKRSTSOCCLKDIVUPDATE]{HP\_SYS\_CLKRST\_SOC\_CLK\_DIV\_UPDATE} to apply the frequency divisor configurations for MEM\_CLK.
    \item Configure the clock divisor for SYS\_CLK: \hyperref[fielddesc:HPSYSCLKRSTSYSCLKDIVNUM]{HP\_SYS\_CLKRST\_SYS\_CLK\_DIV\_NUM} for the integral part, \hyperref[fielddesc:HPSYSCLKRSTSYSCLKDIVNUMERATOR]{HP\_SYS\_CLKRST\_SYS\_CLK\_DIV\_NUMERATOR} for the numerator of the fractional part, and \hyperref[fielddesc:HPSYSCLKRSTSYSCLKDIVDENOMINATOR]{HP\_SYS\_CLKRST\_SYS\_CLK\_DIV\_DENOMINATOR} for the denominator of the fractional part.\\
    Write 1 to \hyperref[fielddesc:HPSYSCLKRSTSOCCLKDIVUPDATE]{HP\_SYS\_CLKRST\_SOC\_CLK\_DIV\_UPDATE} to apply the frequency divisor configurations for SYS\_CLK.
    \item Configure the clock divisor for APB\_CLK: \hyperref[fielddesc:HPSYSCLKRSTAPBCLKDIVNUM]{HP\_SYS\_CLKRST\_APB\_CLK\_DIV\_NUM} for the integral part, \hyperref[fielddesc:HPSYSCLKRSTAPBCLKDIVNUMERATOR]{HP\_SYS\_CLKRST\_APB\_CLK\_DIV\_NUMERATOR} for the numerator of the fractional part, and \hyperref[fielddesc:HPSYSCLKRSTAPBCLKDIVDENOMINATOR]{HP\_SYS\_CLKRST\_APB\_CLK\_DIV\_DENOMINATOR} for the denominator of the fractional part.\\
    Write 1 to \hyperref[fielddesc:HPSYSCLKRSTSOCCLKDIVUPDATE]{HP\_SYS\_CLKRST\_SOC\_CLK\_DIV\_UPDATE} to apply the frequency divisor configurations for APB\_CLK.
    \item Configure the clock source for ROOT\_CLK via \hyperref[fielddesc:LPCLKRSTHPROOTCLKSRCSEL]{LP\_CLKRST\_HP\_ROOT\_CLK\_SRC\_SEL}.\\

     \begin{importantlisting}
      \vspace{-0.5em}
      \begin{enumerate}
        \item It is recommended to first configure the frequency divisors for the SoC clocks and apply the configurations, and then configure the clock source for ROOT\_CLK.
        \item Before changing the clock source for ROOT\_CLK, software needs to ensure that the configured clock divisors do not result in clock frequencies exceeding the allowed maximum frequencies.
        \item Writing 1 to \hyperref[fielddesc:HPSYSCLKRSTSOCCLKDIVUPDATE]{HP\_SYS\_CLKRST\_SOC\_CLK\_DIV\_UPDATE} will apply the frequency divisor configurations for CPU\_CLK, MEM\_CLK, SYS\_CLK, and APB\_CLK at one go, but it is not recommended to do so. It would be better to apply the configuration for each clock individually.
        \item When switching from a high divisor to a low divisor, the recommended configuration order is  APB\_CLK -> SYS\_CLK -> MEM\_CLK -> CPU\_CLK. When switching from a low divisor to a high divisor, the recommended configuration order is CPU\_CLK -> MEM\_CLK -> SYS\_CLK -> APB\_CLK. This helps to prevent high-frequency clock glitches.
        \end{enumerate}
      \end{importantlisting}
\end{itemize}


\subsection{LP System Clock Configuration}

\begin{itemize}
    \item Configure the clock source of LP\_SLOW\_CLK via \hyperref[fielddesc:LPCLKRSTSLOWCLKSEL]{LP\_CLKRST\_SLOW\_CLK\_SEL}
    \item Configure the clock source of LP\_FAST\_CLK via \hyperref[fielddesc:LPCLKRSTFASTCLKSEL]{LP\_CLKRST\_FAST\_CLK\_SEL}
    \item Configure the clock divisor of LP\_PERI\_CLK via \hyperref[fielddesc:LPCLKRSTLPPERIDIVNUM]{LP\_CLKRST\_LP\_PERI\_DIV\_NUM}
\end{itemize}

\subsection{Peripheral Clock Reset and Configuration}

\begin{importantlisting}
\vspace{-0.5em}

\chipname{} features low power consumption. This is why some peripheral clocks are gated (disabled) by default. Before using any of these peripherals, it is mandatory to enable the clock for the given peripheral by setting the corresponding CLK\_EN bit to 1, and release the peripheral from the reset state to make it operational by setting the RST\_EN bit to 0.
\end{importantlisting}

The clocks of most peripherals can be classified into two types:
\begin{itemize}
    \item Bus clock: used to configure peripheral registers.
    \item Function clock: such as UART's reference clock, used by peripherals to operate.
\end{itemize}

The function clock of most peripherals can be selected from multiple clock sources. In the description of the gating registers, it will be stated whether a register is a bus clock (SYS\_CLK, APB\_CLK) gating register or a function clock gating register.

Bus clock enable/disable registers, function clock enable/disable registers,  clock source selection registers, and clock frequency division registers are grouped into the HP\_SYS\_CLKRST, LP\_CLKRST, and LPPERI modules. Except for EMA, SDMMAC, USB Serial/JTAG, Full-Speed USB 2.0 OTG, and High-Speed USB 2.0 OTG, whose reset configuration registers are in the LP\_CLKRST module, all the other HP clock and reset registers are in the HP\_SYS\_CLKRST module. For more information, see Section \ref{sec:resclk-reg-summ} \textit{\nameref{sec:resclk-reg-summ}}.

When changing the clock configuration, it is recommended to turn off the corresponding clock gate first, update the clock divider configurations, clock sources, and other configurations, and then turn it on again.

When a peripheral is not working, users can turn off its function clock by configuring related registers. Turning off the peripheral's function clock does not affect the rest of the system.

Take I2C clock configuration as an example.


\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/sys_clk_struct_pcr_new.pdf}
    \caption{Clock Configuration Example}
    \label{figure:Clock Config}
\end{figure}


Figure \ref{figure:Clock Config} shows the clock structure of I2C. The clock structure of other peripherals is similar to this one. CLK\_SWITCH is used to select a clock output and CLK\_GATE to turn on/off the clock.

In scenarios that require low power consumption, when the peripheral is not in use, in addition to turning off the function clock, the bus clock of the peripheral can also be turned off to further lower power consumption.

Note that if you turn off the bus clock first, the functional clock may continue working. Therefore, when turning off clocks, it is recommended to turn off the functional clock first and then the bus clock; when turning on clocks, it is recommended to  turn on the bus clock first and then the functional clock.

\begin{tiplisting}
In this chapter, all divisor configuration registers are configured with the actual divisor minus 1.
\end{tiplisting}
