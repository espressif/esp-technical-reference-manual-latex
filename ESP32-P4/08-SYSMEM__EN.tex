\chapter{System and Memory}
\label{mod:sysmem}

\section{Overview}
\chipname{} integrates two processors:
\begin{itemize}
    \item a high-performance 32-bit RISC-V dual-core processor (HP CPU), five-stage pipeline, clock frequency up to \iftagged{ESP32-P4-latest}{400}{360} MHz.
    \item a low-power 32-bit RISC-V single-core processor (LP CPU), two-stage pipeline, clock frequency up to 40 MHz.
\end{itemize}

All internal memory, external memory, and peripherals are located on the HP CPU and LP CPU buses.

\section{Features}

\begin{itemize}
    \item {\bfseries Address Space}
        \begin{itemize}
    	    \item 896 KB of HP internal memory address space, accessible by the instruction bus or data bus of HP CPU via cache
            \item 8 KB of HP internal memory address space, accessible by the instruction bus or data bus of HP CPU
            \item 48 KB of LP internal memory address space, accessible by the instruction bus or data bus of LP CPU with zero latency
    	    \item 1256 KB of peripheral address space
    	    \item 64 MB of external flash virtual address space, accessible by the instruction bus or data bus
            \item 64 MB of external RAM virtual address space, accessible by the instruction bus or data bus
    	    \item 768 KB of internal DMA address space
    	    \item 128 MB of external DMA address space
        \end{itemize}
    \item {\bfseries Internal Memory}
        \begin{itemize}
            \item 128 KB of HP ROM
	        \item 768 KB of HP L2MEM
            \item 8 KB of HP SPM (Scratchpad Memory)
            \item 16 KB of LP ROM
            \item 32 KB of LP SRAM
	    \end{itemize}
    \item {\bfseries External Memory}
        \begin{itemize}
            \item Supports up to 64 MB of external flash and RAM
	    \end{itemize}
    \item \textbf{Peripheral Space}
        \begin{itemize}
           \item 96 modules/peripherals in total
        \end{itemize}
    \item {\bfseries GDMA}
        \begin{itemize}
            \item 7 GDMA-AHB supported modules/peripherals
            \item 6 GDMA-AXI supported modules/peripherals
        \end{itemize}
\end{itemize}

\section{Functional Description}

\subsection{Address Mapping}
Figure \ref{fig:sysmem-address-mapping-structure} illustrates the system structure and address mapping. All the non-reserved addresses are accessible via both the instruction bus and the data bus, meaning that the instruction bus and the data bus share the same address space.

Both the data bus and instruction bus of the HP CPU and LP CPU are little-endian. However, the HP CPU's data bus (DBUS) has a 128-bit data width, while other buses maintain a 32-bit data width.

The HP CPU can access data via the data bus using single-byte, double-byte, 4-byte alignment, and, in case of AI Instruction, up to 16-byte alignment. The LP CPU can access data via the data bus using single-byte, double-byte, and 4-byte alignment.

The HP CPU has the following access capabilities:
\begin{itemize}
\item Direct access to HP SPM via both the data bus and instruction bus.
\item Access to internal memory and external memory mapped into the address space via cache, including:
    \begin{itemize}
        \item 128 KB HP ROM (0x4FC0\_0000 \verb+~+ 0x4FC1\_FFFF)
        \item 768 KB HP L2MEM (0x4FF0\_0000 \verb+~+ 0x4FFB\_FFFF)
        \item 64 MB external flash (0x4000\_0000 \verb+~+ 0x43FF\_FFFF)
        \item 64 MB external RAM  (0x4800\_0000 \verb+~+ 0x4BFF\_FFFF)
    \end{itemize}
\begin{tiplisting}
Software can use the addresses starting with ``0x4" in two ways: cache-able access or noncache-able access, depending on the control of CPU Physical Memory Attribution (PMA). Cache-able access is done via cache. Noncache-able access is done without cache, which is similar to the direct access to the addresses starting with ``0x8", however, such direct access to ``0x8" addresses is usually for debugging.
\end{tiplisting}
\item Direct access to internal memory and external memory mapped into the following address space without cache. This access is slower than that via cache.
    \begin{itemize}
        \item 128 KB HP ROM (0x8FC0\_0000 \verb+~+ 0x8FC1\_FFFF)
        \item 768 KB HP L2MEM (0x8FF0\_0000 \verb+~+ 0x8FFB\_FFFF)
        \item 32 KB LP SRAM (0x5010\_8000 \verb+~+ 0x5010\_FFFF)
        \item 64 MB external flash (0x8000\_0000 \verb+~+ 0x83FF\_FFFF)
        \item 64 MB external RAM (0x8800\_0000 \verb+~+ 0x8BFF\_FFFF)
    \end{itemize}
\item Direct access to modules/peripherals via the data bus, including:
    \begin{itemize}
        \item HP CPU Peripherals
        \item HP Peripherals
        \item LP Peripherals
    \end{itemize}
\end{itemize}
The LP CPU's access capabilities include:
\begin{itemize}
    \item Direct access to LP SRAM and LP ROM via both the data bus and instruction bus.
    \item Access to internal memory and external memory, including:
    \begin{itemize}
        \item 128 KB HP ROM (0x4FC0\_0000 \verb+~+ 0x4FC1\_FFFF)
        \item 768 KB HP L2MEM (0x4FF0\_0000 \verb+~+ 0x4FFB\_FFFF)
        \item 64 MB external flash (0x4000\_0000 \verb+~+ 0x43FF\_FFFF)
        \item 64 MB external RAM  (0x4800\_0000 \verb+~+ 0x4BFF\_FFFF)
    \end{itemize}
    \item Direct access to modules/peripherals via the data bus, including:
    \begin{itemize}
        \item HP CPU Peripherals
        \item HP Peripherals
        \item LP Peripherals
    \end{itemize}
\end{itemize}

\subsection{Internal Memory}
\label{para:sysmem-internal-memory}

\chipname{} has various types of internal memory:

\begin{itemize}
    \item \textbf{HP ROM (128 KB):} This read-only memory is dedicated to the HP system and is not programmable. It contains the ROM code and read-only data of some low-level system software. The HP ROM is accessed by the HP CPU through the l2 cache at half the frequency of HP CPU.
    \item \textbf{HP L2MEM (768 KB):} A volatile memory accessible by the HP CPU, LP CPU, or DMA peripherals at half the frequency of the HP CPU. HP L2MEM can be configured to retain power during Light-sleep mode, making it suitable for data retention or register backup.
    \item \textbf{LP ROM (16 KB):} This read-only memory serves the LP system, containing the code for booting the LP CPU and some basic system functions. The LP ROM is accessed by the LP CPU with zero latency.
    \item \textbf{LP SRAM (32 KB):} A volatile memory accessible by both the HP CPU and LP CPU, operating at the same frequency as the LP CPU.
    \item \textbf{HP SPM (8 KB):} A volatile memory accessed by the HP CPU, finishing one access in two cycles.
\end{itemize}

{\bfseries 1. Details for HP ROM}

The 128 KB HP ROM is a read-only memory accessed by the HP CPU through the instruction bus or data bus via the following addresses as shown in Table \ref{tab:sysmem-memory}:
\begin{itemize}
    \item 0x4FC0\_0000 (cache-able or noncache-able, configuration-dependent) \verb+~+ 0x4FC1\_FFFF (cache-able or noncache-able, configuration-dependent).
    \item 0x8FC0\_0000 (direct access by CPU) \verb+~+ 0x8FC1\_FFFF (direct access by CPU).
\end{itemize}

{\bfseries  2. Details for HP L2MEM}
\label{other:sysmem-internal-sram-0}

This 768 KB HP L2MEM is a read-and-write memory accessed by the HP CPU or LP CPU through the instruction bus or data bus, or by DMA master through AHB matrix or AXI matrix via the following addresses as shown in Table \ref{tab:sysmem-memory}:
\begin{itemize}
    \item 0x4FF0\_0000 (cache-able or noncache-able, configuration-dependent) \verb+~+ 0x4FFB\_FFFF (cache-able or noncache-able, configuration-dependent).
    \item 0x8FF0\_0000 (direct access by CPU) \verb+~+ 0x8FFB\_FFFF (direct access by CPU).
\end{itemize}

\tagged{ESP32-P4-latest}{%
L2MEM supports multiple capacity split configurations: L2 RAM and L2 cache share a total capacity of 768 KB, and one of the following four configurations can be selected:
\begin{itemize}
    \item Config 0: L2 RAM 256 KB, L2 cache 512 KB
    \item Config 1: L2 RAM 512 KB, L2 cache 256 KB
    \item Config 2: L2 RAM 640 KB, L2 cache 128 KB
    \item Config 3: L2 RAM 768 KB, L2 cache 0 KB
\end{itemize}
The address layout of HP L2MEM for the above four configurations is shown in Figure \ref{fig:sysmem_l2mem_address_eco5}.

    \caption{HP L2MEM Address Layout}
    \label{fig:sysmem_l2mem_address_eco5}

}

HP L2MEM can be accessed by various peripherals:
\begin{itemize}
    \item by USB 2.0 High-speed, USB 2.0 Full-speed, GMAC, SDMMC, TRACE, GDMA-AHB, and SYSMON via AHB matrix.
    \item by GDMA, GDMA-AXI, 2D-DMA, and H264 Encoder via AXI matrix.
\end{itemize}

The HP L2MEM supports Error Correction Code (ECC) check to correct one-bit memory flip error. In the event of a one-bit flip error, the data in HP L2MEM still remains valid. Before ECC check, HP L2MEM itself must be initialized first.

HP L2MEM consists of six memory units, each occupying 128 KB. The initialization process allows individual configuration of each unit for power considerations. It is recommended to initialize memory units sequentially rather than simultaneously. This sequential approach helps prevent excessive power consumption and avoids triggering a brownout condition when initializing the entire 768 KB HP L2MEM.

\textbf{Detailed Configuration Steps:}\\
Take unit0 as an example, other units follow the same configuration steps.
\begin{itemize}
    \item Set HP\_SYSTEM\_L2\_MEM\_UNIT0\_REFRESH\_EN and clear HP\_SYSTEM\_L2\_MEM\_REFRESH\_CNT\_RESET to start initializing HP L2MEM.
    \item Read the value of HP\_SYSTEM\_L2\_MEM\_UNIT0\_REFRESH\_DONE.
    \item Once HP\_SYSTEM\_L2\_MEM\_UNIT0\_REFRESH\_DONE reaches a value of 1, set HP\_SYSTEM\_L2\_RAM\_UNIT0\_ECC\_EN to start ECC check.
    \item (Recommended) clear HP\_SYSTEM\_L2\_MEM\_UNIT0\_REFRESH\_EN and set HP\_SYSTEM\_L2\_MEM\_REFRESH\_CNT\_RESET after HP\_SYSTEM\_L2\_MEM\_UNIT0\_REFRESH\_DONE reaches 1 to complete the initialization process.
\end{itemize}
During the initialization process, avoid any access to HP L2MEM unit being initialized.

{\bfseries 3. Details for LP ROM}

This 16 KB LP ROM is a read-only memory accessed by LP CPU through the instruction bus or data bus via their shared address 0x5010\_0000 \verb+~+ 0x5010\_3FFF as shown in Table \ref{tab:sysmem-memory}.


{\bfseries 4. Details for LP SRAM}

This 32 KB LP SRAM is a read-and-write memory accessed by the HP CPU via AHB matrix and by the LP CPU through the instruction bus or data bus via their shared address 0x5010\_8000 \verb+~+ 0x5010\_FFFF as shown in Table \ref{tab:sysmem-memory}. LP SRAM supports atomic operation.

{\bfseries 5. Details for HP SPM}

This 8 KB HP SPM is a read-and-write memory accessed by the HP CPU through the instruction bus or data bus via their shared address 0x3010\_0000 \verb+~+ 0x3010\_1FFF as shown in Table \ref{tab:sysmem-memory}. Additionally, HP SPM supports atomic operation of the HP CPU.

HP SPM supports parity check to detect one-bit memory flip error. Before conducting the parity check, HP SPM itself must be initialized first.

\textbf{Detailed Configuration Steps:}

\begin{itemize}
    \item Set HP\_SYSTEM\_SPM\_INIT\_EN and clear HP\_SYSTEM\_SPM\_INIT\_CNT\_RESET to start initializing HP SPM.
    \item Read the value of HP\_SYSTEM\_SPM\_INIT\_DONE.
    \item Once HP\_SYSTEM\_SPM\_INIT\_DONE reaches a value of 1, set HP\_SYSTEM\_HP\_SPM\_PARITY\_CHECK\_EN to start parity check.
    \item (Recommended) clear HP\_SYSTEM\_SPM\_INIT\_EN and set HP\_SYSTEM\_SPM\_INIT\_CNT\_RESET after HP\_SYSTEM\_SPM\_INIT\_CNT\_RESET reaches 1 to complete the initialization process.
\end{itemize}
During the above process, avoid any access to HP SPM.

\subsection{External Memory}
\label{para:sysmem-external-memory}

\chipname{} supports SPI, dual SPI, quad SPI and QPI interfaces for connecting to external flash, as well as OPI and HPI interfaces for connecting to external RAM.

\chipname{} provides the following security features to protect users' data in external flash and RAM:
\begin{itemize}
    \item hardware manual encryption and automatic decryption based on XTS-AES algorithm to protect users' programs and data stored in the external flash.
    \item automatic encryption and decryption based on XTS-AES algorithm to protect users' data in the external RAM.
\end{itemize}

\subsubsection{External Memory Address Mapping}

HP CPU can access external flash or RAM via 0x4000\_0000 \verb+~+ 0x43FF\_FFFF or via 0x4800\_0000 \verb+~+ 0x4BFF\_FFFF using cache-able or noncache-able methods. HP CPU can also access external flash or RAM directly via 0x8000\_0000 \verb+~+ 0x83FF\_FFFF or via 0x8800\_0000 \verb+~+ 0x8BFF\_FFFF, which is slower than the access via cache and is therefore commonly used for debugging.\\

When the HP CPU accesses external memory, it maps the addresses to physical addresses based on the information in the Memory Management Unit (MMU).
\begin{itemize}
    \item Range (0x4000\_0000 \verb+~+ 0x43FF\_FFFF) is mapped to the physical address of the external flash accessed by HP CPU via cache.
    \item Range (0x4800\_0000 \verb+~+ 0x4BFF\_FFFF) is mapped to the physical address of the external RAM accessed by HP CPU via cache.
    \item Range (0x8000\_0000 \verb+~+ 0x83FF\_FFFF) is mapped to the physical address of the external flash accessed by HP CPU directly.
    \item Range (0x8800\_0000 \verb+~+ 0x8BFF\_FFFF) is mapped to the physical address of the external RAM accessed by HP CPU directly.
\end{itemize}

This mapping allows \chipname{} to address up to 64 MB of external flash and 64 MB of external RAM. \textbf{Note that} the instruction bus shares the same address space (64 MB) with the data bus for accessing external memory.

\subsubsection{Cache}

As shown in Figure \ref{fig:sysmem-cache-arch}, \chipname{} has a two-level cache system, with the following features:

\begin{itemize}
    \item L1 cache, including:
    \begin{itemize}
        \item 16 KB of instruction cache (icache) with a 64 B block size, four-way set associative
        \item 64 KB of data cache (dcache) with a 64 B block size, two-way set associative, supporting two writing strategies: write-through and write-back
    \end{itemize}
    \item L2 cache: 128 KB/256 KB/512 KB with 64 B/128 B block sizes, eight-way set associative
    \item Cache-able and noncache-able access, depending on CPU PMA
    \item Preload operation
    \item Lock operation
    \item Critical word first and early restart
\end{itemize}

\subsubsection{Cache Operations}

\chipname{} l1 cache and l2 cache supports the following operations:

\begin{enumerate}
    \item \textbf{Write-back} (for dcache and l2 cache only):
    \begin{itemize}
        \item Clears dirty bits in the tag memory and updates new data to external memory.
        \item After the write-back operation, the new data is updated to external memory.
        \item Users can choose whether to invalidate the data in the cache.
        \item If the data is not invalidated, CPU will read/write data directly from/to the cache where data missing will not occur.
    \end{itemize}

    \item \textbf{Clean} (for dcache and l2 cache only):
    \begin{itemize}
        \item Clears dirty bits in the tag memory without updating data to external memory.
        \item After the clean operation, old data remains in external memory, while the cache holds the new one (unaware of it).
        \item CPU can then read/write the data directly from/to the cache, where data missing will not occur.
    \end{itemize}

    \item \textbf{Invalidate}:
    \begin{itemize}
        \item Removes valid data from the cache.
        \item Once this operation is done, the deleted data is stored only in external memory.
        \item HP CPU needs to access external memory if it wants to access the data again.
        \item Two types of invalidate operation:
        \begin{itemize}
            \item Manual-Invalidate: is performed only on data in the specified cache area.
            \item Invalidate-All: is performed on all data in the cache.
        \end{itemize}
    \end{itemize}

    \item \textbf{Preload}:
    \begin{itemize}
        \item Loads instructions and data into the cache in advance.
        \item Minimum unit of preload-operation is one block.
        \item Two types of preload operation:
        \begin{itemize}
            \item Manual-Preload: involves the hardware prefetching continuous data based on the virtual address specified by the software.
            \item Auto-Preload: involves the hardware prefetching continuous data based on the current address where the cache hits or misses (configuration-dependent).
        \end{itemize}
    \end{itemize}

    \item \textbf{Lock/Unlock}:
    \begin{itemize}
        \item Lock operation prevents easily replacing data in the cache.
        \item Two types of lock:
        \begin{itemize}
            \item Prelock: locks data in the specified area when filling missing data to cache memory, leaving the data outside the specified area unlocked.
            \item Manual Lock: checks data in the cache memory and locks data only if it falls in the specified area, leaving data outside the specified area unlocked.
        \end{itemize}
        \item When there is missing data, the cache replaces the data in the unlocked way first, ensuring that the data in the locked way remains in the cache and will not be replaced.
        \item When all ways within the cache are locked, the cache will replace data, as if it was not locked.
        \item Unlocking is the reverse of locking and can only be done manually.
        \item Note: Manual-Invalidate operation only works on the unlocked data. If you plan to perform this operation on the locked data, please unlock them first.
    \end{itemize}
\end{enumerate}

\subsection{DMA Address Space}
\chipname{} includes AHB General Direct Memory Access (GDMA-AHB), AXI General Direct Memory Access (GDMA-AXI), and VDMA, all providing general direct memory access services.

In \chipname{}, masters like USB 2.0 High-speed, USB 2.0 Full-speed, GDMAC, Trace0/1, VDMA, 2D-DMA, and H264 Encoder have built-in DMA function modules, providing direct memory access services.

All DMA-enabled master devices mentioned above can access:
\begin{itemize}
    \item HP L2MEM (0x4FF0\_0000\verb+~+0x4FFB\_FFFF).
    \item external flash (0x4000\_0000 \verb+~+ 0x43FF\_FFFF).
    \item external RAM (0x4800\_0000 \verb+~+ 0x4BFF\_FFFF).
\end{itemize}
Additionally, VDMA Master 1 can also access:
\begin{itemize}
    \item MIPI CSI internal memory (0x5010\_4000 \verb+~+ 0x5010\_4FFF).
    \item MIPI DSI internal memory (0x5010\_5000 \verb+~+ 0x5010\_5FFF).
\end{itemize}

\begin{tiplisting}
When accessing a memory via DMA, a corresponding access permission is needed, otherwise this access may fail. For more information about permission control, please refer to Chapter  \ref{mod:permctrl} \textit{\nameref{mod:permctrl}}.
\end{tiplisting}
