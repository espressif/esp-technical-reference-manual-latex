\chapter{Interrupt Matrix}
\label{mod:intmtrx}

\section{Overview}

The interrupt matrix embedded in \chipname{} independently routes peripheral interrupt sources to the ESP-RISC-V CPU's peripheral interrupts to timely inform HP CPU0 or HP CPU1 to process the coming interrupts. 

Peripheral interrupt sources must be routed to HP CPU0/HP CPU1 peripheral interrupts via this interrupt matrix due to the following considerations:
\begin{itemize}
    \item \chipname{} has 126 peripheral interrupt sources. To map them to 32 HP CPU0 interrupts or 32 HP CPU1 interrupts, this matrix is needed.
    \item Through this matrix, one peripheral interrupt source can be mapped to multiple HP CPU0 interrupts or HP CPU1 interrupts according to application requirements.
\end{itemize}

\begin{tiplisting}
This chapter focuses on how to map peripheral interrupt sources to HP CPU0/HP CPU1 interrupts. For more details about interrupt configuration, vector, and interrupt handling operations recommended by the ISA, please refer to Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}.
\end{tiplisting}
\vspace{-1em}



\section{Interrupt Terminology in \chipname{}}\label{sec:intmtx-int-terminology}

The following terms related to interrupts are defined in the context of the \textit{\chipname{} Technical Reference Manual} to help readers better understand this document:

\subsection{Interrupt} 
An interrupt refers to the event or condition that occurs, causing the CPU to temporarily suspend its current execution and handle a higher-priority task. It is a mechanism that allows the CPU to respond to specific events promptly.

The \textit{\chipname{} Technical Reference Manual} may use the term ``interrupt" in a broader sense to refer to both interrupt signal and interrupt source.

\subsection{Interrupt Signal/interrupt Source} 

Interrupt signal and interrupt source are only defined from different perspectives and mean the same thing. 

From the perspective of peripherals, interrupt signals are generated by the peripheral's internal interrupt sources and are sent to the interrupt matrix.

From the perspective of the interrupt matrix, it receives the interrupt signals sent from the peripheral and considers them interrupt sources. The interrupt matrix then outputs CPU peripheral interrupt signals to the CPU.

From the perspective of the CPU, the interrupt signals from the interrupt matrix become sources and are sent to the CPU core together with  the core local interrupt sources.

\section{Features}

The interrupt matrix embedded in \chipname{} has the following features:

\begin{itemize}
    \item 126 peripheral interrupt sources accepted as input
    \item 32 HP CPU0 peripheral interrupts and 32 HP CPU1 peripheral interrupts generated to HP CPU as output
    \item Current interrupt status query of peripheral interrupt sources
    \item Multiple interrupt sources mapping to a single HP CPU0 or HP CPU1 interrupt (i.e., shared interrupts)
\end{itemize}

Figure \ref{fig:interrupt-matrix-structure} shows the structure of the interrupt matrix.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{\modulefiles/figures/14-INTMTRX-struct.png}
    \caption{Interrupt Matrix Structure}
    \label{fig:interrupt-matrix-structure}
\end{figure}

All the interrupts generated by the peripheral interrupt sources can be handled by HP CPU0 or HP CPU1. Users can configure \hyperref[sec: cpu0-intmtrx-reg]{HP CPU0 interrupt registers} (``Core0 Interrupt Reg" module in Figure \ref{fig:interrupt-matrix-structure}) to assign peripheral interrupt sources to HP CPU0, or configure \hyperref[sec: cpu1-intmtrx-reg]{HP CPU1 interrupt registers} (``Core1 Interrupt Reg" module in Figure \ref{fig:interrupt-matrix-structure}) to assign peripheral interrupt sources to HP CPU1. Peripheral interrupt sources can be assigned both to HP CPU0 and HP CPU1 simultaneously, and if so, HP CPU0 and HP CPU1 will both accept the interrupts.


\section{Functional Description}

\subsection{Peripheral Interrupt Sources}

\chipname{} has 126 peripheral interrupt sources in total. Table \ref{tab:peripheral-interrupt-config-status-source} lists all these sources and their mapping/status registers. 


\begin{itemize}
    \item Column ``No.": Peripheral interrupt source number, can be 0 \verb+~+ 98, 100 \verb+~+ 111, 113 \verb+~+ 127 (99 and 112 are reserved). 
    \item Column ``Chapter": in which chapter the interrupt source is described in detail.
    \item Column ``Interrupt Source": Name of the peripheral interrupt source.
    \item Column ``Interrupt Source Mapping Register": Registers used for routing the peripheral interrupt sources to HP CPU peripheral interrupts.
    \item Column ``Interrupt Status Register": Registers used for indicating the interrupt status of peripheral interrupt sources.
    \begin{itemize}
        \item Column ``Interrupt Status Register - Bit": Bit position in status register, indicating the interrupt status.
        \item Column ``Interrupt Status Register - Name": Name of status registers.
    \end{itemize}
\end{itemize}

The register in column ``Interrupt Source Mapping Register" and in column ``Interrupt Status Register" correspond to the peripheral interrupt source in column ``Interrupt Source". For example, the interrupt source mapping register for interrupt source RMT\_INTR is \hyperref[regdesc:CORE0RMTINTMAPREG]{CORE\regindex{x}\_RMT\_INT\_MAP\_REG}, and its status register is \hyperref[regdesc:CORE0INTRSTATUSREG1REG]{CORE\regindex{x}\_INTR\_STATUS\_REG\_1\_REG}.

Note that CORE\regindex{x} in the table can be CORE0 (HP CPU0) or CORE1 (HP CPU1).

\subsection{Assign Peripheral Interrupt Source to HP CPU Interrupt}

In this section, the following terms are used to describe the operation of the interrupt matrix.

\begin{itemize}

\item Source\_\regindex{Y}: stands for a peripheral interrupt source, wherein \regindex{Y} means the number of this interrupt source in Table \ref{tab:peripheral-interrupt-config-status-source}.
\item \hyperref[regdesc:CORE0SOURCEregindexYMAPREG]{CORE\regindex{x}\_SOURCE\_\regindex{Y}\_MAP\_REG}: stands for an interrupt source mapping register for the peripheral interrupt source (Source\_\regindex{Y}) of HP CPU\regindex{x}.
\item Num\_P: the index of HP CPU interrupts which can be 16 \verb+~+ 47.
\item Interrupt\_P: stands for the HP CPU interrupt numbered as Num\_P.

\end{itemize}

\subsubsection{Assign One Peripheral Interrupt Source (Source\_\regindex{Y}) to HP CPU\regindex{x}}

Setting the corresponding source mapping  register \hyperref[regdesc:CORE0SOURCEregindexYMAPREG]{CORE\regindex{x}\_SOURCE\_\regindex{Y}\_MAP\_REG} of Source\_\regindex{Y} to Num\_P assigns this interrupt source to Interrupt\_P. 

\subsubsection{Assign Multiple Peripheral Interrupt Sources (Source\_\regindex{Y}) to HP CPU\regindex{x}}

Setting the corresponding source mapping register \hyperref[regdesc:CORE0SOURCEregindexYMAPREG]{CORE\regindex{x}\_SOURCE\_\regindex{Y}\_MAP\_REG} of each interrupt source to the same Num\_P assigns multiple sources to the same Interrupt\_P of HP CPU\regindex{x}. Any of these sources can trigger CPU Interrupt\_P of HP CPU\regindex{x}. In other words, the CPU Interrupt\_p will be shared between these interrupt sources. When an interrupt signal is generated, HP CPU\regindex{x} should check the interrupt status registers to figure out which peripheral generated the interrupt. For more information, see Chapter \ref{mod:riscvcpu} \textit{\nameref{mod:riscvcpu}}.

\subsubsection{Disable HP CPU\regindex{x} Peripheral Interrupt Source (Source\_\regindex{x})}

Writing 0 to the \hyperref[regdesc:CORE0SOURCEregindexYMAPREG]{CORE\regindex{x}\_SOURCE\_\regindex{Y}\_MAP\_REG} register disables the corresponding interrupt source.

\subsection{Query Current Interrupt Status of HP CPU\regindex{x} Peripheral Interrupt Source}

After enabling peripheral interrupt sources, users can query current interrupt status of HP CPU\regindex{x} peripheral interrupt source by reading the bit value in \hyperref[fielddesc:CORE0INTRSTATUS0]{CORE\regindex{x}\_INTR\_STATUS\_\regindex{n}} (read only). For the mapping between \hyperref[fielddesc:CORE0INTRSTATUS0]{CORE\regindex{x}\_INTR\_STATUS\_\regindex{n}} and peripheral interrupt sources, please refer to Table \ref{tab:peripheral-interrupt-config-status-source}.
