\hypertarget{lowpowm}{}
\chapter{低功耗管理}\label{mod:lowpowm}

\section{概述}

\chipname{} 拥有先进的低功耗管理系统，可以优化设备的功耗，并同时保持高性能。

低功耗管理系统通过采用多种节能技术，例如睡眠模式、动态电压和频率调节以及外设电源门控等，可以有效地将设备功耗降至最低。

在这一管理系统中，电源管理单元 (PMU) 扮演着核心角色，它可以灵活地控制各个电源域的上下电，以此实现芯片性能、功耗和唤醒时间之间的最佳平衡。

\section{术语}

下列有关低功耗管理的术语是在《\chipname{} 技术参考手册》的语境下定义的，旨在帮助读者更好地理解本文档。

\begin{longtable}[c]{ R{4cm} p{12cm} }
\textbf{低功耗管理系统} & 指的是管理芯片功耗的整个系统。\\
\textbf{电源管理单元 (PMU)} & 指的是负责芯片功耗管理的核心控制电路，它包括对芯片电源、时钟以及功耗相关逻辑的控制。\\
\textbf{电源域} & 指的是可独立控制上下电的最小单元。一个电源域可能包含一个或多个模块。 \\
\textbf{PMU 状态} & 指的是 PMU 状态机的状态。用户可以配置不同电源域在每一种状态下的时钟开关和电源开关。\\
\textbf{功耗模式} & 指的是预设的典型功耗模式，可满足一些常见场景的功耗需求。\\
\end{longtable}

\section{主要特性}

PMU 具有以下特性：

\begin{itemize}
\item 支持 3 种可配置 PMU 状态，软件可根据需求灵活配置
    \begin{itemize}
    \item HP\_ACTIVE
    \item HP\_SLEEP
    \item LP\_SLEEP
    \end{itemize}

\item 支持 \iftagged{ESP32-P4-latest}{5}{4} 种预设功耗模式，可满足多种应用场景中的典型功耗需求
    \begin{itemize}
    \item Active
    \iftagged{ESP32-P4-latest}{
        \item Light-sleep0（外设不下电，仅 CPU 下电，不关闭时钟，功耗大，唤醒延迟小）
        \item Light-sleep1（外设不下电，仅关闭时钟，功耗较大，唤醒延迟较小）
        \item Light-sleep2（外设下电，需要备份恢复数据，功耗较小，唤醒延迟大）
    }{
        \item Light-sleep0（外设不下电，仅关闭时钟，功耗较大，唤醒延迟较小）
        \item Light-sleep1（外设下电，需要备份恢复数据，功耗较小，唤醒延迟大）
    }
    \item Deep-sleep
    \end{itemize}

\item LP SRAM (32 KB)：任何模式下均保持上电，可存放 HP CPU 不掉电数据，LP CPU 的指令和数据

\item 可灵活配置的备份 DMA，可在芯片切换 PMU 状态时，备份与恢复 CPU 及外设的关键数据

\item 功耗控制器，可根据芯片所处的功耗模式切换状态，以控制电源和时钟等
\end{itemize}

\section{功能描述}

\chipname{} 的低功耗管理系统包括如下组成部分：

\begin{itemize}
\item \textbf{电源结构}：即供电管理网络，包括调压器、数字电源域、模拟电源域等部分。

\item \textbf{PMU 控制器}：PMU 的核心部分，用于实现模拟电源、数字电源、时钟等系统控制。

\item \textbf{1 个 RTC 定时器}：更多信息，请参考章节 \ref{mod:rtctimer} \textit{\nameref{mod:rtctimer}}。

\item \textbf{16 个 LP GPIO 管脚 (GPIO0–GPIO15)}：这 16 个管脚永远处于工作状态，不受功耗模式的影响，可用作睡眠模式下的唤醒源，也可以作为标准的 GPIO 管脚使用。更多细节，请见章节 \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}}。

\item \textbf{LP SRAM}：HP CPU 与 LP CPU 均可访问。HP CPU 访问时，该存储器使用 HP CPU 时钟；在 LP CPU 访问时，使用 LP CPU 时钟。

\item  \textbf{欠压检测器}: 用于实时监测供电管脚的电压，确保芯片不会因为电压毛刺或欠压而工作异常。有关欠压检测器的详细介绍，请参考章节 \ref{mod:brownout} \textit{\nameref{mod:brownout}}。
\end{itemize}

以下详细介绍这几个组成部分。

\item 两个内部调压器
\item 四个输出调压器 (EXT\_LDO)
\item 数字电源域
\item 模拟电源域

\end{itemize}

\subsubsection{调压器}\label{sec:lowpowm-regulators}

如图 \ref{fig:lowpowm-power-structure} 所示，\chipname{} 的模拟部分包含 1个 外置带反馈调节的 DCDC、2 个内部 HP/LP 系统调压器，负责调节不同电源域的供电：

\begin{itemize}
\item DCDC 调压反馈器，可以调节片外 DCDC 的电压。
\item HP（高性能）系统调压器，驱动能力较强，功耗较大，输出电压可调，仅适用于上电和睡眠状态。
\item LP（低功耗）系统调压器，负责调节低功耗模块的供电，输出电压同样可调。
\end{itemize}

DCDC 调压反馈器和 HP 系统调压器负责给 HP 系统供电。片外 DCDC 通过内部的电压反馈器可以调节输出电压。芯片上电默认使用 HP 系统调压器供电，上电完成后，建议切换至 DCDC 供电，以获得更好的效率和负载能力。

\subsubsection{数字电源域}\label{sec:lowpowm_digital_domain}

\begin{itemize}
\item HP 系统的数字电源域包括：
    \begin{itemize}
    \iftagged{ESP32-P4-latest}{
        \item \textbf{CPU (PD\_HP\_CPU)}：主要包括 CPU 及支持其工作的外设。
        \item \textbf{Peripherals + ROM (PD\_TOP)}：主要包括总线、HP 外设等功能模块。
    }{
        \item \textbf{Peripherals + ROM + CPU (PD\_TOP)}：主要包括总线、HP 外设等功能模块。
    }
    \item \textbf{L2MEM\_G0–G5}：细分了 6 个子电源域，如图 \ref{fig:lowpowm-power-structure} 所示。
    \item \textbf{HP\_CNNT (PD\_HP\_CNNT)}：主要包括 USB 以及 SDIO 等模块。
    \end{itemize}

调压器与每个电源域之间均有一个独立的电源开关，可实现独立的上下电控制。

\item LP 系统的数字电源域包括：
    \begin{itemize}
    \item \textbf{LP PD peripherals (PD\_LP\_PERI)}：主要包括 LP CPU、LP 外设。
    \item \textbf{LP always-on (PD\_AON)}：主要包括 LP AON 外设（如 RTC 定时器）和 PMU 控制模块，LP SRAM，所有模式下均不掉电。
    \end{itemize}
\end{itemize}

\subsubsection{模拟电源域}

\begin{itemize}
\item HP 系统的模拟电源域包括：
    \begin{itemize}
    \item CPLL\_CLK
    \item SPLL\_CLK
    \item Audio PLL
    \item SDIO PLL
    \end{itemize}

\item LP 系统的模拟电源域包括：
    \begin{itemize}
    \item External Main Clock（外部主时钟）
    \item Fast RC Oscillator（快速 RC 振荡器）
    \end{itemize}
\end{itemize}

\subsubsection{电池电源域}

电池电源域 (VDD\_BAT) 为备用电源域，当所有外部供电 (VDD\_ANA) 全部关断的时候，低功耗系统调压器、LP\_SLOW\_CLK、LP\_FAST\_CLK 可以全部切换到 VDD\_BAT 电源供电。

\subsection{PMU}

\chipname{} 的 PMU 控制着每个电源域功耗相关的组件，如电源、时钟，它主要由如下几个部分构成：

\begin{itemize}
\item \textbf{PMU 主状态机}：记录并切换 PMU 状态
\item \textbf{睡眠与唤醒控制器}：用于向 PMU 主状态机发起睡眠与唤醒请求
\item \textbf{电源控制器}：可根据芯片所处的功耗模式切换状态，以控制电源和时钟等。电源控制器包括：
    \begin{itemize}
    \item \textbf{数字电源控制器}：用于控制数字电源域的上下电
    \item \textbf{模拟电源控制器}：用于控制模拟模块的使能，比如调压器、模拟时钟等
    \item \textbf{时钟控制器}：用于管理外设时钟的门控，以及选择数字时钟的模拟时钟源
    \item \textbf{数据备份控制器}：用于控制 PMU 状态切换时的数据备份与恢复过程
    \item \textbf{系统控制器}：用于控制部分系统级模块，如睡眠模式下（此时 CPU 不可用）看门狗功能的挂起等
    \end{itemize}
\end{itemize}

PMU 的工作流程为：

\begin{itemize}
\item 睡眠与唤醒控制器向 PMU 主状态机发送睡眠或唤醒请求。
\item PMU 主状态机产生电源门控、时钟门控和复位信号。
\item 电源控制器和时钟控制器会根据 PMU 主状态机产生的信号，打开或关闭不同的电源域和时钟信号，从而让芯片进入或退出各种低功耗模式。
\end{itemize}

PMU 的工作流程如图 \ref{fig:lowpowm-workflow} 所示，其中，HP\_SWITCH 和 LP\_SWITCH 的定义如下：

\begin{itemize}
\item HP\_SWITCH：HP 系统在 HP\_ACTIVE 和 HP\_SLEEP 之间切换的中间状态。硬件将在这个状态完成各个控制器的控制切换。
\item LP\_SWITCH：LP 系统在 HP\_SLEEP 和 LP\_SLEEP 之间切换的中间状态。硬件将在这个状态完成各个控制器的控制切换。
\end{itemize}

以下章节分别介绍 PMU 的各个主要组成部分。

\subsubsection{PMU 主状态机}

PMU 主状态机可以接收睡眠与唤醒信号，并通过功耗控制器完成电源上下电和时钟开关，从而实现 PMU 状态的切换，使芯片在目标 PMU 状态下达到性能与功耗的平衡。

PMU 主状态机包括 3 个 PMU 状态，每个状态由不同的睡眠与唤醒信号控制切换。软件可对每个 PMU 状态进行个性化的配置，如配置电源、时钟等。软件可基于这 3 个 PMU 状态，拓展出不同的应用层面的功耗模式。

3 个 PMU 状态为：

\begin{itemize}
\item \textbf{HP\_ACTIVE}：所有电路最大程度开启的 PMU 状态，该状态支持 HP 系统和 LP 系统工作。
\item \textbf{HP\_SLEEP}：HP 系统睡眠状态，支持 LP 外设工作。
\item \textbf{LP\_SLEEP}：HP 系统与 LP 外设均处于睡眠状态，always-on 电路保持工作。
\end{itemize}

\begin{tiplisting}
\vspace{-0.5em}
HP 和 LP 系统的划分为：

    \begin{itemize}
    \item LP 系统（外设）：LP RISC-V 32-bit Microprocessor、LP GPIO、LP UART、LP I2C、LP I2S、触摸传感器、温度传感器、LP ADC、LP SPI
    \item LP 系统（always-on 电路）：PMU，RTC 看门狗定时器，超级看门狗
    \item HP 系统：除 LP 系统以外的所有外设
    \end{itemize}

    更多内容可参考 \href{https://www.espressif.com/sites/default/files/documentation/esp32-p4_datasheet_cn.pdf}{《\chipname{} 技术规格书》} > 功能框图。
\end{tiplisting}

HP\_ACTIVE 为 HP 系统的工作状态，HP\_SLEEP 为 HP 系统的睡眠状态，LP\_SLEEP 为 LP 系统的睡眠状态。LP 系统的工作状态复用 HP\_SLEEP 状态，因此也被称为 HP\_SLEEP。

\begin{itemize}
\item 某个模块如果属于 HP 系统，则只能配置在 HP\_ACTIVE/HP\_SLEEP 两种状态下的上下电，在 LP\_SLEEP 的状态下会复用 HP\_SLEEP 的状态。
\item 某个模块如果属于 LP 系统，则只能配置 HP\_SLEEP/LP\_SLEEP 状态下的上下电，在 HP\_ACTIVE 状态下会复用 HP\_SLEEP 的状态。
\end{itemize}

例如，HP CPU 属于 HP 系统，则可以通过以下寄存器配置 HP CPU 在 HP\_ACTIVE/HP\_SLEEP 状态下的上下电，LP\_SLEEP 复用 HP\_SLEEP 的配置。

\begin{itemize}
    \iftagged{ESP32-P4-latest}{
        \item HP\_ACTIVE：\hyperref[fielddesc:PMUHPACTIVEPDHPCPUPDEN]{PMU\_HP\_ACTIVE\_PD\_HP\_CPU\_PD\_EN}
        \item HP\_SLEEP：\hyperref[fielddesc:PMUHPSLEEPPDHPCPUPDEN]{PMU\_HP\_SLEEP\_PD\_HP\_CPU\_PD\_EN}
    }{
        \item HP\_ACTIVE：\hyperref[fielddesc:PMUHPACTIVEPDTOPPDEN]{PMU\_HP\_ACTIVE\_PD\_TOP\_PD\_EN}
        \item HP\_SLEEP：\hyperref[fielddesc:PMUHPSLEEPPDTOPPDEN]{PMU\_HP\_SLEEP\_PD\_TOP\_PD\_EN}
    }
\end{itemize}

类似地，用户可以用同样的方法，定义其他电源域在不同 PMU 状态下的上下电，具体寄存器请见第 \ref{sec:lowpowm-reg-summ} 小节。

\begin{tiplisting}
为方便描述，后文中此类寄存器全部统称为 PMU\_\regindex{n1}\_PD\_\regindex{POWERDOMAIN}\_PD\_EN，\regindex{n1} 代表 PMU 的 3 种状态。
\end{tiplisting}

配置完成后，PMU 将借助各控制器使这些配置生效，详见下方各章节的描述。

\subsubsection{模拟电源控制器}\label{sec:pmu_analog_power_controller}

模拟电源控制器用于控制模拟电路部分（包括调压器、高性能时钟、低速时钟）在某种 PMU 状态下的上下电情况。

调压器的具体配置如下：

\begin{itemize}
\item 配置寄存器 PMU\_\regindex{n1}\_HP\_REGULATOR\_XPD 或PMU\_\regindex{n1}\_LP\_REGULATOR\_XPD 可使能或关闭目标 PMU 状态下 HP 系统或 LP 系统调压器的输出电压。LP 系统调压器不建议关闭，否则会导致 PMU 本身掉电，从而导致芯片异常。
\end{itemize}

高性能时钟（XTAL\_CLK 和 PLL 时钟）的具体配置如下：

\begin{itemize}
\item XTAL\_CLK：配置 \hyperref[fielddesc:PMUHPSLEEPXPDXTAL]{PMU\_HP\_SLEEP\_XPD\_XTAL} 为 1，在芯片 PMU 状态切换至 HP\_SLEEP 的过程中，开启 XTAL\_CLK。

注意：为避免 XTAL\_CLK 起振阶段的不稳定时钟影响电路，可配置 \hyperref[fielddesc:PMUWAITXTLSTABLE]{PMU\_WAIT\_XTL\_STABLE} 实现 XTAL\_CLK 全局门控的延迟打开，即该门控将在 XTAL\_CLK 上电后的 \hyperref[fielddesc:PMUWAITXTLSTABLE]{PMU\_WAIT\_XTL\_STABLE} 个 CLK\_DYN\_FAST\_CLK 周期时间后打开。

\item CPLL\_CLK：PMU 可在芯片处于不同 PMU 状态下打开 CPLL\_CLK，仅需配置不同 PMU 状态下的 PMU\_\regindex{n1}\_XPD\_PLL[0] 为 1。例如，配置 \hyperref[fielddesc:PMUHPACTIVEXPDPLL]{PMU\_HP\_ACTIVE\_XPD\_PLL}[0] 为 1，则在芯片处于 HP\_ACTIVE 状态下，打开 CPLL\_CLK 时钟。

\item SPLL\_CLK：PMU 可在芯片处于不同 PMU 状态下打开 SPLL\_CLK，仅需配置不同 PMU 状态下的 PMU\_\regindex{n1}\_XPD\_PLL[1] 为 1。例如，配置 \hyperref[fielddesc:PMUHPACTIVEXPDPLL]{PMU\_HP\_ACTIVE\_XPD\_PLL}[1] 为 1，则在芯片处于 HP\_ACTIVE 状态下，打开 SPLL\_CLK 时钟。

\item Audio PLL：PMU 可在芯片处于不同 PMU 状态下打开 Audio\_PLL\_CLK，仅需配置不同 PMU 状态下的 PMU\_\regindex{n1}\_XPD\_PLL[2] 为 1。例如，配置 \hyperref[fielddesc:PMUHPACTIVEXPDPLL]{PMU\_HP\_ACTIVE\_XPD\_PLL}[2] 为 1，则在芯片处于 HP\_ACTIVE 状态下，打开 Audio PLL 时钟。

\item SDIO PLL：PMU 可在芯片处于不同 PMU 状态下打开 SDIO\_PLL\_CLK，仅需配置不同 PMU 状态下的 PMU\_\regindex{n1}\_XPD\_PLL[3] 寄存器为 1。例如，配置 \hyperref[fielddesc:PMUHPACTIVEXPDPLL]{PMU\_HP\_ACTIVE\_XPD\_PLL}[3] 为 1，则在芯片处于 HP\_ACTIVE 状态下，打开 SDIO PLL 时钟。

注意：
\begin{itemize}
\item 在打开 PLL 时钟前还需确保 XTAL\_CLK 已经稳定开启。
\item 为避免 PLL 时钟起振阶段的不稳定时钟影响电路，可配置 \hyperref[fielddesc:PMUWAITPLLSTABLE]{PMU\_WAIT\_PLL\_STABLE} 实现 PLL 时钟全局门控的延迟打开，即该门控将在 PLL 时钟上电后的 \hyperref[fielddesc:PMUWAITPLLSTABLE]{PMU\_WAIT\_PLL\_STABLE} 个CLK\_DYN\_FAST\_CLK 周期时间后打开。
\end{itemize}
\end{itemize}

低速时钟（RC\_FAST\_CLK 和 XTAL32K\_CLK）功耗较低，在 HP\_ACTIVE、HP\_SLEEP 状态下的开关情况由 \hyperref[fielddesc:PMUHPSLEEPXPDFOSCCLK]{PMU\_HP\_SLEEP\_XPD\_FOSC\_CLK} 控制，在 LP\_SLEEP 状态下由 \hyperref[fielddesc:PMULPSLEEPXPDFOSCCLK]{PMU\_LP\_SLEEP\_XPD\_FOSC\_CLK} 控制。

\subsubsection{数字电源控制器}

数字电源控制器用于控制数字电源域在某种 PMU 状态下的上下电情况。与模拟电源控制器不同的是，数字电源控制器不直接控制调压器，而是通过控制与调压器相连的电源开关，实现各个数字电源域的上下电。

数字电源域中 LP PD Peripherals 只支持在 HP\_SLEEP 与 LP\_SLEEP 两个 PMU 状态之间切换时上下电，其余电源域只支持在 HP\_SLEEP、HP\_ACTIVE 两个 PMU 状态间切换时上下电。

当芯片在不同 PMU 状态之间切换时，如果某电源域在当下 PMU 状态下的上下电配置与即将切换至的上下电配置不符，则将激活上下电流程。以 CPU 电源域从上电到下电的流程为例。配置寄存器 \hyperref[fielddesc:PMUHPSLEEPPDTOPPDEN]{PMU\_HP\_SLEEP\_PD\_TOP\_PD\_EN} 为 1，表示在 HP\_SLEEP 状态下关闭 CPU 电源域的供电；或配置为 0，表示在 HP\_SLEEP 状态下对 CPU 电源域供电。PMU 将做出如下控制：

\begin{itemize}

\item 使能数字隔离单元，确保下电模块不往上电模块输出不稳定的电平值。当有电源域掉电时，该模块的输出将被钳位至固定值。

\item 使能复位，CPU 电源域掉电后，该电源域的全局复位信号处于复位状态，在重新上电后的一段时间内该电源域仍然保持复位，这将确保该电源域在上电的过程中存在一个复位到释放的过程，消除上下电引入的寄存器不定态。

\end{itemize}

下面说明各个数字电源域的开关方式：

\begin{itemize}
\item L2MEM\_G0–G5

L2MEM\_G0– G5 电源域的上下电状态统一由 PMU\_\regindex{n1}\_PD\_HP\_MEM\_PD\_EN 控制。

\item HP\_CNNT

HP\_CNNT 电源域的上下电状态由 PMU\_\regindex{n1}\_PD\_CNNT\_PD\_EN 控制。

\tagged{ESP32-P4-latest}{
    \item PD\_HP\_CPU

    PD\_HP\_CPU 电源域的上下电状态由 PMU\_\regindex{n1}\_PD\_HP\_CPU\_PD\_EN 控制。
}

\item \iftagged{ESP32-P4-latest}{Peripherals + ROM (PD\_TOP)}{Peripherals + ROM + CPU (PD\_TOP)}

可通过寄存器 PMU\_\regindex{n1}\_PD\_TOP\_PD\_EN 配置各电源域在对应 PMU 状态下的上下电。

Peripherals 掉电时还存在以下几个可配功能：
    \begin{itemize}
    \item Peripherals 掉电后会导致 GPIO 管脚状态不稳定，可通过 PMU 控制 GPIO 管脚（LP GPIO 除外）进入保持状态。例如配置 \hyperref[fielddesc:PMUHPSLEEPHPPADHOLDALL]{PMU\_HP\_SLEEP\_HP\_PAD\_HOLD\_ALL} 使 GPIO 管脚在 HP\_SLEEP 下保持 Peripherals 掉电前的相关配置。

    \item 在 HP\_ACTIVE 状态下，支持标准供电下的 Memory Deep-sleep 模式，该模式下存储器将无法读写，但可以保持数据。配置寄存器 PMU\_\regindex{n1}\_HP\_MEM\_DSLP 可实现 Memory Deep-sleep 模式。
    \end{itemize}

\item LP PD Peripherals

LP PD Peripherals 电源域在芯片处于 HP\_ACTIVE 状态下始终保持上电，无法配置，仅允许配置 HP\_SLEEP 与 LP\_SLEEP 状态下的上下电。此电源域拥有独立的数字电源开关，且它的上下电控制不依赖于其他数字电源域的上下电。

\end{itemize}

\subsubsection{时钟控制器}

时钟控制器主要用于芯片 PMU 状态切换时，对时钟源的选择及部分时钟的使能，以此实现对高性能系统时钟与低功耗系统时钟的控制。

高性能系统时钟包括 ROOT\_CLK 和高性能系统外设时钟。当芯片的 PMU 状态在 HP\_ACTIVE、HP\_SLEEP 之间切换时，PMU 可实现对 ROOT\_CLK 时钟源的切换、开关、分频，以及高性能系统外设时钟的开关。

    \begin{itemize}
    \item 配置不同 PMU 状态对应的 PMU\_\regindex{n1}\_SYS\_CLK\_SLP\_SEL 寄存器为 1，表示芯片进入该 PMU 状态后，时钟源受 PMU 控制。

    \item 配置不同 PMU 状态对应的 PMU\_\regindex{n1}\_ICG\_SYS\_CLOCK\_EN 寄存器为 0，表示芯片进入该 PMU 状态后，关闭 ROOT\_CLK 的时钟门控，即该时钟将不可用。

    \item 配置不同 PMU 状态对应的 PMU\_\regindex{n1}\_DIG\_SYS\_CLK\_SEL，选择芯片进入对应 PMU 状态后使用的时钟源，具体请见章节 \ref{mod:resclk} \textit{\nameref{mod:resclk}} > 表 \ref{tab:resclk-cpu-clk-src}。

\subsubsection{数据备份控制器}

\chipname{} 支持在切换 PMU 状态时，驱动一个 Retention DMA 模块，进行存储器与外设间的数据搬移，从而在电源域掉电时备份数据并在再次上电时恢复数据。

数据搬移操作由 Peripherals 电源域中的模块实现，PMU 只产生相关控制信号。需要注意的是，决定数据搬移的控制寄存器存在方向性，因为与 PMU 的其他控制寄存器不同的是，这些控制行为由原来的 PMU 状态和目标 PMU 状态共同决定。

比如，从 HP\_ACTIVE 至 HP\_SLEEP 与从 HP\_SLEEP 至 HP\_ACTIVE 所需要备份的寄存器显然是不同的，因此下文的寄存器名中的 \regindex{n2} 存在如下可能：

\begin{itemize}
\item HP\_SLEEP2ACTIVE
\item HP\_ACTIVE2SLEEP
\end{itemize}

下面将介绍 PMU 支持的 Retention DMA 控制：

\begin{itemize}
\item 使能数据搬移：配置寄存器 PMU\_\regindex{n2}\_BACKUP\_EN 为 1，表示在进行对应的 PMU 状态切换时使能数据搬移。

\item 打开对应时钟：在数据搬移开始前，配置 PMU\_\regindex{n2}\_BACKUP\_CLK\_SEL 选择 Retention DMA 工作时钟源，配置 PMU\_\regindex{n1}\_BACKUP\_ICG\_FUNC\_EN 为 1 打开时钟。

数据搬移结束后，门控时钟寄存器 PMU\_\regindex{n1}\_BACKUP\_ICG\_FUNC\_EN 的值将由目标状态下的 PMU\_\regindex{n1}\_DIG\_ICG\_FUNC\_EN 决定。

\item 配置数据搬移方向：配置 PMU\_\regindex{n2}\_BACKUP\_MODE 寄存器的最高位：
    \begin{itemize}
        \item 1：外设到存储器
        \item 0：存储器到外设
    \end{itemize}

\item 选择链表指针：配置 PMU\_\regindex{n2}\_BACKUP\_MODE 寄存器的低两位，选择链表指针：
    \begin{itemize}
    \item 0：PAU\_LINK\_ADDR\_0
    \item 1：PAU\_LINK\_ADDR\_1
    \item 2：PAU\_LINK\_ADDR\_2
    \item 3：PAU\_LINK\_ADDR\_3
    \end{itemize}
\end{itemize}

\subsubsection{系统控制器}

系统控制器用于在芯片切换 PMU 状态时控制部分功能模块，以实现芯片在各 PMU 状态下稳定、节能地工作。系统控制器支持以下功能：

\begin{itemize}
\item \textbf{暂停看门狗功能}：配置 PMU\_\regindex{n1}\_DIG\_PAUSE\_WDT 寄存器为 1，可在 PMU 状态切换至对应 PMU 状态时禁用 RTC 看门狗定时器 (RWDT) 功能。注意，任一睡眠模式下若配置该寄存器为 0，表示不禁用该功能，则 RWDT 将因为没有 CPU 参与喂狗而产生复位。

\item \textbf{切换 GPIO 至管脚睡眠模式}，该模式下将固定管脚的配置。例如将某个 GPIO 管脚固定为输入，低驱动模式以降低其功耗，并作为一个唤醒管脚使用，配置 PMU\_\regindex{n1}\_HP\_PAD\_HOLD\_ALL 寄存器为 1，可在 PMU 状态切换至对应 PMU 状态后锁存当前的配置作为 GPIO 的睡眠配置。GPIO 管脚保持功能的详细介绍请参考章节 \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}} > 小节 \ref{PadHold} \textit{\nameref{PadHold}}。

\item \textbf{禁用 UART 唤醒功能}：配置 PMU\_\regindex{n1}\_UART\_WAKEUP\_EN 寄存器为 0，禁用对应 PMU 状态下的 4 种 UART 唤醒模式。有关 UART 唤醒模式的具体介绍，请见章节 \ref{mod:uart} \textit{\nameref{mod:uart}}。

\item \textbf{暂停 CPU 运行}：配置 PMU\_\regindex{n1}\_DIG\_CPU\_STALL 寄存器为 1，在对应 PMU 状态下将 CPU 挂起。
\end{itemize}

\subsubsection{电池供电控制器}

如图 \ref{fig:lowpowm-power-structure} 所示，低功耗系统调压器可以由 VDD\_BAT（电池）或 VDD\_ANA（模拟）供电。\hyperref[fielddesc:PMUHPSLEEPVDDBATMODE]{PMU\_\regindex{n1}\_VDDBAT\_MODE} 控制是否使能电池供电，0 是不使能电池供电，此时低功耗系统调压器由 VDD\_ANA 供电，可用于 Active 模式或 Light-sleep0/1 模式。\hyperref[fielddesc:PMUHPSLEEPVDDBATMODE]{PMU\_\regindex{n1}\_VDDBAT\_MODE} 为 1 时，使能电池供电，可用于包括 VDD\_ANA 在内的主供电切断的 Deep-sleep 模式下，此时只有 LP AON \& PMU 电源域有电。

    \paragraph{板级用法示例}

    \begin{itemize}
    \item \textbf{示例 1：VO3 为 MIPI D-PHY 供电（2.5 V，LDO 模式）}\\
    在板级上将 VO3 引脚连接至 \texttt{VDD\_MIPI\_DPHY} 等需要 2.5 V 的电源网络。配置思路：选择 LDO 模式，并设置 \(V_{OUT}=2.5\,\mathrm{V}\)。一种可行组合为 \(V_{REF}=1.0\,\mathrm{V}\)（\texttt{DREF=9}）与 \(\mathrm{MUL}=2.5\)（\texttt{MUL=6}），即 \texttt{PMU\_ANA\_0P2A\_DREF\_0=9}、\texttt{PMU\_ANA\_0P2A\_MUL\_0=6}，并将 \texttt{PMU\_0P2A\_TIEH\_0=0}（LDO）。
    \item \textbf{示例 2：VO4 为 VDD\_IO\_5 供电（SD 卡接口电压切换）}\\
    在参考设计中，VO4 可能连接至 \texttt{VDD\_IO\_5} 以为部分 GPIO/SDIO 供电。\\
    -- 输出 3.3 V：可选择透传模式（例如设置 \texttt{PMU\_0P2A\_TIEH\_1=1}）。\\
    -- 输出 1.8 V：选择 LDO 模式，并设置 \(V_{OUT}=1.8\,\mathrm{V}\)。一种可行组合为 \(V_{REF}=0.9\,\mathrm{V}\)（\texttt{DREF=8}）与 \(\mathrm{MUL}=2.0\)（\texttt{MUL=4}），即 \texttt{PMU\_ANA\_0P2A\_DREF\_1=8}、\texttt{PMU\_ANA\_0P2A\_MUL\_1=4}，并将 \texttt{PMU\_0P2A\_TIEH\_1=0}（LDO）。
    \end{itemize}

    \paragraph{软件配置步骤}

推荐的软件配置步骤（示例）如下：

    \begin{enumerate}
    \item 配置输出模式：设置 \texttt{PMU\_0PxA\_FORCE\_TIEH\_SEL\_y} 使模式控制由寄存器生效，然后通过 \texttt{PMU\_0PxA\_TIEH\_SEL\_y}/\texttt{PMU\_0PxA\_TIEH\_y} 选择 LDO 或透传模式（见表 \ref{tab:VOX-registers}）。
    \item 若使用 LDO 模式，配置输出电压：设置对应的 \texttt{PMU\_ANA\_0PxA\_DREF\_y} 与 \texttt{PMU\_ANA\_0PxA\_MUL\_y} 以满足 \(V_{OUT}=\mathrm{MUL}\times V_{REF}\)。
    \item 配置等待计数器：将 stage 1 (\texttt{TARGET0}) 设置为限流保护持续时间，将 stage 2 (\texttt{TARGET1}) 设置为稳定等待时间（见表 \ref{tab:lowpowm-target}）。
    \item 上电前使能限流保护：设置对应 \texttt{PMU\_ANA\_0PxA\_EN\_CUR\_LIM\_y}=1（见表 \ref{tab:lowpowm-vo-lim-prot}）。
    \item 打开调压器：设置对应 \texttt{PMU\_0PxA\_XPD\_y}=1（见表 \ref{tab:lowpowm-xpdctrl}），并等待 stage 1 超时中断（例如 VO3 对应 \texttt{PMU\_0P2A\_CNT\_TARGET0\_REACH\_0\_INT}，VO4 对应 \texttt{PMU\_0P2A\_CNT\_TARGET0\_REACH\_1\_INT}）。
    \item 收到 stage 1 中断后关闭限流保护：将 \texttt{PMU\_ANA\_0PxA\_EN\_CUR\_LIM\_y} 置 0；如需保证稳定，可继续等待 stage 2 超时中断（\texttt{...CNT\_TARGET1\_REACH...\_INT}）。
    \end{enumerate}

\end{itemize}

\section{事件任务矩阵功能} \label{sec:lowpowm-etm-func}

在 \chipname{} 中，低功耗管理模块支持 ETM 功能，即可以通过任意外设的 ETM 事件触发低功耗管理模块的 ETM 任务，或者通过低功耗管理模块的 ETM 事件触发任意外设的 ETM 任务。关于 ETM 更多详细信息，请参考章节 \ref{mod:evntaskmatrix} \textit{\nameref{mod:evntaskmatrix}}。这里仅介绍与低功耗管理模块相关的 ETM 任务和 ETM 事件。

低功耗管理模块可接收的 ETM 任务有：

\begin{itemize}
\item PMU\_TASK\_SLEEP\_REQ：该任务将触发 PMU 开始睡眠流程。
\end{itemize}

低功耗管理模块可产生的 ETM 事件有：

\begin{itemize}
\item PMU\_EVT\_SLEEP\_WAKEUP：该事件表示 PMU 被唤醒至 HP\_ACTIVE。
\end{itemize}

\section{中断}

\chipname{} 的低功耗管理模块可以生成以下中断信号：

\begin{itemize}
\item PMU\_INTR
\item PMU\_LP\_INT
\end{itemize}

其中，PMU\_INTR 发送给 \hyperref[mod:intmtrx]{中断矩阵}；PMU\_LP\_INT 输出到 LP CPU。

这些中断信号由内部中断源生成。具体为：

PMU\_INTR：

\begin{itemize}
\item \label{int:PMUSOCWAKEUPINT}PMU\_SOC\_WAKEUP\_INT：当芯片被唤醒至 HP\_ACTIVE 时触发。

\item \label{int:PMUSOCSLEEPREJECTINT}PMU\_SOC\_SLEEP\_REJECT\_INT：当拒绝睡眠源拒绝了一个睡眠请求时触发。

\item \label{int:PMUSWINT}PMU\_SW\_INT：当用 LP CPU 作为唤醒源将芯片唤醒至 HP\_ACTIVE 时触发。
\item \label{int:PMUSDIOIDLEINT}PMU\_SDIO\_IDLE\_INT：当 SDIO 处于 IDLE 状态时触发。
\item \label{int:PMULPCPUEXCINT}PMU\_LP\_CPU\_EXC\_INT：当 LP CPU 异常时触发。
\item \label{int:PMU0P2ACNTTARGET1REACH1INT}PMU\_0P2A\_CNT\_TARGET1\_REACH\_1\_INT：VO4 调压器等待计数器目标 1 超时中断。
\item \label{int:PMU0P2ACNTTARGET0REACH1INT}PMU\_0P2A\_CNT\_TARGET0\_REACH\_1\_INT：VO4 调压器等待计数器目标 0 超时中断。
\item \label{int:PMU0P2ACNTTARGET1REACH0INT}PMU\_0P2A\_CNT\_TARGET1\_REACH\_0\_INT：VO3 调压器等待计数器目标 1 超时中断。
\item \label{int:PMU0P2ACNTTARGET0REACH0INT}PMU\_0P2A\_CNT\_TARGET0\_REACH\_0\_INT：VO3 调压器等待计数器目标 0 超时中断。
\item \label{int:PMU0P1ACNTTARGET1REACH1INT}PMU\_0P1A\_CNT\_TARGET1\_REACH\_1\_INT：VO2 调压器等待计数器目标 1 超时中断。
\item \label{int:PMU0P1ACNTTARGET0REACH1INT}PMU\_0P1A\_CNT\_TARGET0\_REACH\_1\_INT：VO2 调压器等待计数器目标 0 超时中断。
\item \label{int:PMU0P1ACNTTARGET1REACH0INT}PMU\_0P1A\_CNT\_TARGET1\_REACH\_0\_INT：VO1 调压器等待计数器目标 1 超时中断。
\item \label{int:PMU0P1ACNTTARGET0REACH0INT}PMU\_0P1A\_CNT\_TARGET0\_REACH\_0\_INT：VO1 调压器等待计数器目标 0 超时中断。
\end{itemize}

PMU\_LP\_INT：
\begin{itemize}
\item \label{int:PMULPCPUSLEEPREJECTINT}PMU\_LP\_CPU\_SLEEP\_REJECT\_INT：当 LP CPU 睡眠被拒绝时触发。

\item \label{int:PMUHPSWTRIGGERINT}PMU\_HP\_SW\_TRIGGER\_INT：当用 HP CPU 作为唤醒源唤醒 LP CPU 时触发。

\item \label{int:PMUACTIVESWITCHSLEEPSTARTINT}PMU\_ACTIVE\_SWITCH\_SLEEP\_START\_INT：当 PMU 状态开始从 HP\_ACTIVE 切换至 HP\_SLEEP 时触发。

\item \label{int:PMULPCPUWAKEUPINT}PMU\_LP\_CPU\_WAKEUP\_INT：当 LP CPU 被唤醒时触发。
\item \label{int:PMU0P2ACNTTARGET1REACH1LPINT}PMU\_0P2A\_CNT\_TARGET1\_REACH\_1\_LP\_INT：VO4 调压器等待计数器目标 1 超时中断。

\item \label{int:PMU0P1ACNTTARGET1REACH1LPINT}PMU\_0P1A\_CNT\_TARGET1\_REACH\_1\_LP\_INT：VO2 调压器等待计数器目标 1 超时中断。
