\section{概况}

\chipname{} 内置一个 LP I2S 接口，为\hyperref[mod:vad]{人声检测模块 (Voice Activity Detection, VAD)} 和一些低功耗模式下的数字音频应用提供了数据接收通信接口。

I2S 标准总线定义了三种信号——串行时钟信号 BCK、字选择信号 WS 和串行数据信号 SD。一个基础的 I2S 数据总线包含一个主机和一个从机。主机和从机的角色在通信过程中保持不变。

\chipname{} 的 LP I2S 模块包含独立的 RX 模块，能够在芯片以最低功耗运行时，完成数据接收的功能。相对于 HP I2S，即 I2S\regindex{0}、I2S\regindex{1} 和 I2S\regindex{2}，LP I2S 不支持 DMA 访问，而是由一块独立的内部储存器来存储数据。

\begin{tiplisting}
如需了解 I2S 相关术语，请参考章节 \ref{mod:i2s} \textit{\nameref{mod:i2s}} 中的 \ref{sec:i2s-term} \textit{\nameref{sec:i2s-term}}  小节。
\end{tiplisting}
\vspace{-1em}

\section{主要特性}
\label{sec:lpi2s-features}

LP I2S 模块具有以下特性：

\begin{itemize}
    \item RX 模块支持主机模式和从机模式
    \item 支持多种音频标准：
        \begin{itemize}    
            \item TDM Philips 标准
            \item TDM MSB 对齐标准
            \item TDM PCM 标准
            \item PDM 标准
        \end{itemize}
    \item 支持多种 RX 模式：
        \begin{itemize}
            \item TDM RX 模式，最多支持 2 通道
            \item PDM RX 模式
            \begin{itemize}
                \item 支持原始 PDM 数据接收
                \item 支持将 PDM 数据转换为 PCM 数据接收，最多支持 2 通道
            \end{itemize}
        \end{itemize}
    \item 可配置采样时钟，支持多种采样频率
    \item 支持 16 位的数据位宽
    \item 支持 LP I2S 接口中断
\end{itemize}

\section{架构概览}
\label{sec:lpi2s-arch-overview}

图 \ref{fig:lpi2s-sys-diagram} 为 \chipname{} LP I2S 模块的结构框图。LP I2S 模块包含：

\begin{itemize}
    \item 独立的 RX 模块
    \item 输入输出时序调节单元（I/O 同步）
    \item 时钟分频器（时钟发生器）
    \item 64 x 32 位 RX FIFO
    \item 256 x 32 位 LP I2S 数据存储空间
    \item \hyperref[mod:vad]{人声检测模块 (Voice Activity Detection, VAD)} 通信接口
\end{itemize}

\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/lpi2s_arch_20240729__CN.pdf}}
    \caption{\chipname{} LP I2S 系统架构}
    \label{fig:lpi2s-sys-diagram}
\end{figure}

RX 模块有一组三线接口，分别为串行时钟线 BCK、字选择线 WS 和串行数据线 SD。其中，RX 模块的 SD 线固定为输入接收。主机模式下，RX 模块的 BCK 和 WS 信号线为输出，从机模式下，RX 模块的 BCK 和 WS 信号线为输入。

图 \ref{fig:lpi2s-sys-diagram} 中的右半部分展示了 LP I2S 模块的信号总线。RX 模块的信号命名规则为：LP\_I2S\textcolor{red}{A}\_\textcolor{red}{B}\_\textcolor{red}{C}，例如 LP\_I2S\textcolor{red}{I}\_\textcolor{red}{BCK}\_\textcolor{red}{in}。其中：

\begin{itemize}
    \item ``A" 固定为``I"，表示信号输入至 RX 模块；
    \item ``B" 表示信号功能，包括：
    \begin{itemize}
        \item BCK
        \item WS
        \item SD
    \end{itemize}
    \item ``C" 表示该信号的方向，包括：
    \begin{itemize}
        \item ``in" 表示该信号输入 LP I2S 模块
        \item ``out" 表示该信号自 LP I2S 模块输出
    \end{itemize}
\end{itemize}

\section{支持的音频协议} \label{sec:lpi2s-supported-audio-standards}

\chipname{} LP I2S 模块支持多种音频标准，包括 TDM Philips 标准、TDM MSB 对齐标准、TDM PCM 标准以及 PDM 标准。

用户可通过配置以下寄存器，选择所需的音频标准：

\begin{itemize}
    \item \hyperref[fielddesc:LPI2SRXTDMEN]{LP\_I2S\_RX\_TDM\_EN}
    \begin{itemize}
        \item 0：禁用 TDM 模式
        \item 1：选择 TDM 模式
    \end{itemize}
    \item \hyperref[fielddesc:LPI2SRXPDMEN]{LP\_I2S\_RX\_PDM\_EN}
    \begin{itemize}
        \item 0：禁用 PDM 模式
        \item 1：选择 PDM 模式
    \end{itemize}    
    \item \hyperref[fielddesc:LPI2SRXMSBSHIFT]{LP\_I2S\_RX\_MSB\_SHIFT}
    \begin{itemize}
        \item 0：配置 WS 信号和 SD 信号同时开始变化，即选择 MSB 对齐标准
        \item 1：配置 WS 信号先于 SD 信号一个 BCK 时钟周期开始变化，即选择 Philips 标准或 PCM 标准
    \end{itemize}
\end{itemize}

\begin{tiplisting}
不可将 \hyperref[fielddesc:LPI2SRXTDMEN]{LP\_I2S\_RX\_TDM\_EN} 和 \hyperref[fielddesc:LPI2SRXPDMEN]{LP\_I2S\_RX\_PDM\_EN} 同时配置为 1 或同时配置为 0，否则会导致 LP I2S 以既非 TDM 模式也非 PDM 模式的状态错误发送数据。
\end{tiplisting}

\subsection{TDM Philips 标准模式}

在 Philips 标准下，在 BCK 的下降沿，WS 信号先于 SD 信号一个 BCK 时钟周期开始变化，即 WS 信号从当前通道数据的第一个位之前的一个时钟开始有效，并在当前通道数据发送结束前一个 BCK 时钟周期开始变化。SD 信号线上首先传输音频数据的最高位。

与普通 Philips 标准相比，TDM Philips 标准支持的通道更多，见图 \ref{fig:lpi2s-tdm-philips-std-timing-diagram}。

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/i2s_tdm_phillips_mode.png}
    \caption{时序图 -- TDM Philips 标准}
    \label{fig:lpi2s-tdm-philips-std-timing-diagram}
\end{figure}

\subsection{TDM MSB 对齐标准模式}

MSB 对齐标准下，在 BCK 下降沿，WS 信号和 SD 信号同时变化。WS 持续到当前通道数据发送结束，SD 信号线上首先传输音频数据的最高位。

与普通 MSB 对齐标准相比，TDM MSB 对齐标准支持更多通道，见图 \ref{fig:lpi2s-tdm-msb-align-std-timing-diagram}。

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/i2s_tdm_msb_mode.png}
    \caption{时序图 -- TDM MSB 对齐标准}
    \label{fig:lpi2s-tdm-msb-align-std-timing-diagram}
\end{figure}

\subsection{TDM PCM 标准模式}

在 PCM 标准的短帧同步模式下，在 BCK 的下降沿，WS 信号先于 SD 信号一个 BCK 时钟周期开始变化，即 WS 信号从当前通道数据的第一个位之前的一个时钟开始有效，并持续一个 BCK 时钟周期。SD 信号线上首先传输音频数据的最高位。

与普通 PCM 标准相比，TDM PCM 标准支持更多通道，见图 \ref{fig:lpi2s-tdm-pcm-std-timing-diagram} 所示。

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/i2s_tdm_pcm_mode.png}
    \caption{时序图 -- TDM PCM 标准}
    \label{fig:lpi2s-tdm-pcm-std-timing-diagram}
\end{figure}

\subsection{PDM 标准模式}

如图 \ref{fig:lpi2s-pdm-std-timing-diagram} 所示，在 PDM 标准下，WS 代表左/右声道，WS 与 SD 同时变化。WS 在数据发送过程中持续变化，WS 的高低对应两个声道。请参考图 \ref{fig:lpi2s-pdm-std-timing-diagram}。

\begin{figure}[H]

    \caption{时序图 -- PDM 标准}
    \label{fig:lpi2s-pdm-std-timing-diagram}
\end{figure}


\section{RX 时钟}\label{sec:lpi2s-rx-clk}

作为 LP I2S RX 模块的主时钟，LP\_I2S\_RX\_CLK 可由以下时钟分频所得：

\begin{itemize}
    \item 最高 40 MHz、可配置时钟源的 LP\_FAST\_CLK 
    \item 20 MHz XTAL\_D2\_CLK
    \item 8 MHz PLL\_LP\_CLK 
\end{itemize}

LP I2S RX 模块的串行时钟 BCK 再由 LP\_I2S\_RX\_CLK 分频获得，如图 \ref{fig:lpi2s-clk} 所示。\hyperref[fielddesc:LPPERILPI2SRXCLKSEL]{LPPERI\_LP\_I2S\_RX\_CLK\_SEL} 用于选择 LP I2S RX 的时钟源，\hyperref[fielddesc:LPPERICKENLPI2SRX]{LPPERI\_CK\_EN\_LP\_I2S\_RX} 用于使能或者关闭 LP I2S RX 模块的时钟源。

\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.6\textwidth]{\modulefiles/figures/lpi2s_clk_20240620.pdf}}
    \caption{LP I2S 时钟分频器}
    \label{fig:lpi2s-clk}
\end{figure}

LP\_I2S\_RX\_CLK 的频率 $f_{\textrm{LP\_I2S\_RX\_CLK}}$ 与分频器时钟源频率 $f_{\textrm{LP\_I2S\_CLK\_S}}$ 间的关系如下：
$$f_{\textrm{LP\_I2S\_RX\_CLK}}=\frac{f_{\textrm{LP\_I2S\_CLK\_S}}}{\textrm{N} + \frac{\textrm{b}}{\textrm{a}}}$$

其中，2 <= N <= 256（N 为整数），N 对应寄存器域 \hyperref[fielddesc:LPPERILPI2SRXCLKMDIVNUM]{LPPERI\_LP\_I2S\_RX\_CLKM\_DIV\_N} 的值，具体为：
\begin{itemize}
    \item \hyperref[fielddesc:LPPERILPI2SRXCLKMDIVNUM]{LPPERI\_LP\_I2S\_RX\_CLKM\_DIV\_NUM} = 0 时，N = 256；
    \item \hyperref[fielddesc:LPPERILPI2SRXCLKMDIVNUM]{LPPERI\_LP\_I2S\_RX\_CLKM\_DIV\_NUM} = 1 时，N = 2；
    \item \hyperref[fielddesc:LPPERILPI2SRXCLKMDIVNUM]{LPPERI\_LP\_I2S\_RX\_CLKM\_DIV\_NUM} 为其他值时，N = \hyperref[fielddesc:LPPERILPI2SRXCLKMDIVNUM]{LPPERI\_LP\_I2S\_RX\_CLKM\_DIV\_NUM} 的值。
\end{itemize}

分数部分分频系数 a、b 唯一对应系数 x、y、z、yn1。系数对应公式为：

\begin{itemize}
    \item 当 $\textrm{b} <= \frac{\textrm{a}}{2}$ 时，$\textrm{yn1} = 0$，$\textrm{x} = floor([\frac{\textrm{a}}{\textrm{b}}]) - 1$，$\textrm{y} = \textrm{a}\%\textrm{b}$，$\textrm{z} = \textrm{b}$；
    \item 当 $\textrm{b} > \frac{\textrm{a}}{2}$ 时，$\textrm{yn1}=1$，$\textrm{x} = floor([\frac{\textrm{a}}{\textrm{a - b}}]) - 1$，$\textrm{y} = \textrm{a}\%\textrm{(a - b)}$，$\textrm{z}=\textrm{a - b}$。
\end{itemize}

系数 x、y、z、yn1 通过 \hyperref[fielddesc:LPPERILPI2SRXCLKMDIVX]{LPPERI\_LP\_I2S\_RX\_CLKM\_DIV\_X}、\hyperref[fielddesc:LPPERILPI2SRXCLKMDIVY]{LPPERI\_LP\_I2S\_RX\_CLKM\_DIV\_Y}、\hyperref[fielddesc:LPPERILPI2SRXCLKMDIVZ]{LPPERI\_LP\_I2S\_RX\_CLKM\_DIV\_Z}、\hyperref[fielddesc:LPPERILPI2SRXCLKMDIVYN1]{LPPERI\_LP\_I2S\_RX\_CLKM\_DIV\_YN1} 配置。 

对于整数分频，\hyperref[fielddesc:LPPERILPI2SRXCLKMDIVX]{LPPERI\_LP\_I2S\_RX\_CLKM\_DIV\_X} 和 \hyperref[fielddesc:LPPERILPI2SRXCLKMDIVZ]{LPPERI\_LP\_I2S\_RX\_CLKM\_DIV\_Z} 清零，\hyperref[fielddesc:LPPERILPI2SRXCLKMDIVY]{LPPERI\_LP\_I2S\_RX\_CLKM\_DIV\_Y} 置为 1。

\begin{tiplisting}
使用小数分频功能可能会产生时钟抖动。
\end{tiplisting}

在主机接收模式下，LP I2S RX 模块的串行时钟 BCK 为 LP\_I2SI\_BCK\_out 信号，由 LP\_I2S\_RX\_CLK 分频获得。即：

$$f_{\textrm{LP\_I2SI\_BCK\_out}} = \frac{f_{\textrm{LP\_I2S\_RX\_CLK}}}{\textrm{MI}}$$

其中 MI 的值为：
$$ \textrm{MI} = \textrm{\hyperref[fielddesc:LPI2SRXBCKDIVNUM]{LP\_I2S\_RX\_BCK\_DIV\_NUM}} + 1 $$

\vspace{-1em}
\begin{tiplisting}
注意，\hyperref[fielddesc:LPI2SRXBCKDIVNUM]{LP\_I2S\_RX\_BCK\_DIV\_NUM} 不可配置为 1。
\end{tiplisting}
\vspace{-1em}

当模块处于 LP I2S 从机模式时，为了保障采样的准确性，必须保证 $f_{\textrm{LP\_I2S\_RX\_CLK}}$ >= 8 x $f_{\textrm{BCK}}$。另外模块可以输出 LP\_I2S\_MCLK\_out 作为外部设备的主时钟。

由于 LP I2S 从机模式下，模块时钟频率必须大于等于 BCK 时钟频率的 8 倍，因此 I2S 支持的采样频率受到数据位宽和声道数的限制。例如，时钟源频率最高为 40 MHz，即模块时钟最高能配置为 20 MHz，能支持的 BCK 时钟频率最高为 5 MHz，因此传输双声道 16 位宽数据时，能够支持 78.125 kHz 以下的采样频率，例如 8 kHz、16 kHz、32 kHz、44.1 kHz、48 kHz。具体请参考章节 \ref{sec:lpi2s-rx-clk} \textit{\nameref{sec:lpi2s-rx-clk}}。

除了时钟频率限制外，LP I2S 使用 LP I2S 内部存储器而非 DMA 来存储数据。因此，用户需要及时从 LP I2S 内部存储器读取数据。读取速率可能会限制 LP I2S 支持的采样频率。详细信息，请参考小节 \ref{sec:lpi2s-memory} \textit{\nameref{sec:lpi2s-memory}}。

\section{复位} \label{sec:lpi2s-reset}

LP I2S 模块中各个单元以及 FIFO 可通过配置相关位进行复位：

\begin{itemize}
    \item LP I2S RX 模块：可配置 \hyperref[fielddesc:LPI2SRXRESET]{LP\_I2S\_RX\_RESET} 进行复位；
    \item LP I2S RX FIFO：可配置 \hyperref[fielddesc:LPI2SRXFIFORESET]{LP\_I2S\_RX\_FIFO\_RESET} 进行复位。
\end{itemize}

\begin{tiplisting}
在模块和 FIFO 复位之前，需要先配置 LP I2S 模块时钟。
\end{tiplisting} 

\section{主/从机接收模式}

在 RX 模式下，LP I2S 可作为主机或从机接收数据。

\begin{itemize}
    \item 主机接收模式
    \begin{itemize}
        \item 置位 \hyperref[fielddesc:LPI2SRXSTART]{LP\_I2S\_RX\_START} 启动一次接收操作。RX 模块会一直输出时钟信号，并对输入数据进行采样。
        \item 配置 \hyperref[fielddesc:LPI2SRXSTOPMODE]{LP\_I2S\_RX\_STOP\_MODE} 来控制接收模式何时暂停接收：
        \begin{itemize}
            \item 0：只有 \hyperref[fielddesc:LPI2SRXSTART]{LP\_I2S\_RX\_START} 被清零，RX 模块才会暂停接收。
            \item 1：RX 模块会在 \hyperref[fielddesc:LPI2SRXSTART]{LP\_I2S\_RX\_START} 被清零，或者接收字节数大于等于 \hyperref[regdesc:LPI2SRXEOFNUMREG]{LP\_I2S\_RX\_EOF\_NUM\_REG} 配置的接收长度值时暂停接收。暂停接收后，当 \hyperref[fielddesc:LPI2SRXSTART]{LP\_I2S\_RX\_START} 未被清零时，可通过置位 \hyperref[fielddesc:LPI2SRXRESET]{LP\_I2S\_RX\_RESET} 来重新启动接收。
        \end{itemize}
        \item 清零 \hyperref[fielddesc:LPI2SRXSTART]{LP\_I2S\_RX\_START}，RX 模块停止接收数据。
    \end{itemize}
    \item 从机接收模式
    \begin{itemize}
        \item 置位 \hyperref[fielddesc:LPI2SRXSTART]{LP\_I2S\_RX\_START}。等待主机 BCK 时钟，来启动接收操作。
        \item 配置 \hyperref[fielddesc:LPI2SRXSTOPMODE]{LP\_I2S\_RX\_STOP\_MODE} 来控制接收模式何时暂停接收：
        \begin{itemize}
            \item 0：只有 \hyperref[fielddesc:LPI2SRXSTART]{LP\_I2S\_RX\_START} 被清零，RX 模块才会暂停接收。
            \item 1：RX 模块会在 \hyperref[fielddesc:LPI2SRXSTART]{LP\_I2S\_RX\_START} 被清零，或者接收字节数大于等于 \hyperref[regdesc:LPI2SRXEOFNUMREG]{LP\_I2S\_RX\_EOF\_NUM\_REG} 配置的接收长度值时暂停接收。暂停接收后，当 \hyperref[fielddesc:LPI2SRXSTART]{LP\_I2S\_RX\_START} 未被清零时，可通过置位 \hyperref[fielddesc:LPI2SRXRESET]{LP\_I2S\_RX\_RESET} 来重新启动接收。
        \end{itemize}
        \item 清零 \hyperref[fielddesc:LPI2SRXSTART]{LP\_I2S\_RX\_START}，RX 模块停止接收数据。
    \end{itemize}
\end{itemize}


\section{接收数据}\label{sec:lpi2s-rx-data}

LP I2S 接收数据时，从外设接口读取数据，经过通道模式控制和数据格式控制，输入数据至 LP I2S 存储器。

\subsection{通道模式控制}

\chipname{} LP I2S 支持 TDM 和 PDM 两种输入模式。置位 \hyperref[fielddesc:LPI2SRXTDMEN]{LP\_I2S\_RX\_TDM\_EN} 使能 TDM 输入模式，置位 \hyperref[fielddesc:LPI2SRXPDMEN]{LP\_I2S\_RX\_PDM\_EN} 使能 PDM 输入模式。

\begin{tiplisting}
\hyperref[fielddesc:LPI2SRXTDMEN]{LP\_I2S\_RX\_TDM\_EN} 和 \hyperref[fielddesc:LPI2SRXPDMEN]{LP\_I2S\_RX\_PDM\_EN} 不能同时置位或同时清零。
\end{tiplisting}
\vspace{-1em}

\subsubsection{TDM 输入模式} 

在 TDM 输入模式下，LP I2S 最多支持 2 个接收通道。接收通道数由 \hyperref[fielddesc:LPI2SRXTDMTOTCHANNUM]{LP\_I2S\_RX\_TDM\_TOT\_CHAN\_NUM} 控制。例如，配置 \hyperref[fielddesc:LPI2SRXTDMTOTCHANNUM]{LP\_I2S\_RX\_TDM\_TOT\_CHAN\_NUM} 为 1，则通道 0 \verb+~+ 1 接收数据。

在接收数据的通道中，如果其对应的 \hyperref[fielddesc:LPI2SRXTDMPDMCHAN0EN]{LP\_I2S\_RX\_TDM\_PDM\_CHAN{\textcolor{red}{n}}\_EN} 为：

\begin{itemize}
    \item 0：则该通道数据无效，不会存入 RX FIFO；
    \item 1：则该通道数据有效，会被存入 RX FIFO。
\end{itemize}

当 LP I2S 处于主机 TDM 输入模式下，WS 信号由 \hyperref[fielddesc:LPI2SRXWSIDLEPOL]{LP\_I2S\_RX\_WS\_IDLE\_POL} 和 \hyperref[fielddesc:LPI2SRXTDMWSWIDTH]{LP\_I2S\_RX\_TDM\_WS\_WIDTH} 控制：
\begin{itemize}
    \item \hyperref[fielddesc:LPI2SRXWSIDLEPOL]{LP\_I2S\_RX\_WS\_IDLE\_POL} 的值为 WS 信号的默认电平；
    \item \hyperref[fielddesc:LPI2SRXTDMWSWIDTH]{LP\_I2S\_RX\_TDM\_WS\_WIDTH} 的值为在接收所有通道的过程中，WS 为默认电平的周期数。
\end{itemize}

另外，\hyperref[fielddesc:LPI2SRXHALFSAMPLEBITS]{LP\_I2S\_RX\_HALF\_SAMPLE\_BITS} 的值乘以 2，即为一个 WS 周期对应的 BCK 周期数。

\subsubsection{PDM 输入模式} 

在 PDM 输入模式下，LP I2S 将通道中的串行数据转换为待输入数据。LP I2S 可以接收原始 PDM 数据，或者将 PDM 数据转换为 PCM 数据后接收。

当 LP I2S 处于主机 PDM 输入模式下，WS 信号的默认电平由 \hyperref[fielddesc:LPI2SRXWSIDLEPOL]{LP\_I2S\_RX\_WS\_IDLE\_POL} 控制，WS 信号频率为 BCK 信号频率的一半，请参考 \ref{sec:lpi2s-rx-clk} 小节配置 BCK 信号的方式配置 WS 信号。注意，在 PDM 输入模式下，请配置 \hyperref[fielddesc:LPI2SRXHALFSAMPLEBITS]{LP\_I2S\_RX\_HALF\_SAMPLE\_BITS} 的值与 \hyperref[fielddesc:LPI2SRXBITSMOD]{LP\_I2S\_RX\_BITS\_MOD} 的值相同。

在 LP I2S 启用 PDM-to-PCM 转换器时，可以将接收的 PDM 数据转为 PCM 数据，再进行数据控制。配置 \hyperref[fielddesc:LPI2SRXPDM2PCMEN]{LP\_I2S\_RX\_PDM2PCM\_EN} 启用该转换器。PDM-to-PCM 转换器的寄存器配置如下：

\begin{itemize}
    \item 配置采样频率和下采样率。\\
    LP I2S 使用 PDM-to-PCM 转换器接收数据时，PDM 时钟频率与主从机模式的关系如下：
    \begin{itemize}
        \item 主机模式：PDM 时钟频率即为 BCK 时钟频率。
        \item 从机模式：PDM 时钟频率由外部提供。
    \end{itemize}
    采样频率 ($f_{\textrm{Sampling}}$) 与 PDM 时钟频率的关系如下：
    $$f_{\textrm{Sampling}} = \frac{f_{\textrm{PDM}}}{\textrm{DSR}}$$
    其中，下采样率 DSR 和 \hyperref[fielddesc:LPI2SRXPDMSINCDSR16EN]{LP\_I2S\_RX\_PDM\_SINC\_DSR\_16\_EN} 的关系为：
    $${\textrm{DSR}} = {\textrm{\hyperref[fielddesc:LPI2SRXPDMSINCDSR16EN]{LP\_I2S\_RX\_PDM\_SINC\_DSR\_16\_EN}}} \times {\textrm{64}}$$
    请根据需要的主从机模式、采样频率以及下采样率进行寄存器配置。
    \item 配置有效通道。\\
    启用 PDM-to-PCM 转换器时，最多支持 2 个通道的输入信号，寄存器配置以及对应通道如下表：
\begin{table}[H]
    \centering
    \caption{PDM-to-PCM 数据输入}
    \label{table:PDM_RX_MODE}
    \begin{tabular}{|p{4.3cm}|p{2.0cm}|p{5.8cm}|}
    \hline
    \rowcolor{lightgray}
    \textbf{输入数据信号}&\textbf{通道} &\textbf{使能寄存器} \\ \hline
    \multirow{2}*{LP\_I2SI\_SD\_in}  & 左声道 & \hyperref[fielddesc:LPI2SRXTDMPDMCHAN0EN]{LP\_I2S\_RX\_TDM\_PDM\_CHAN0\_EN} \\ \cline{2-3}
                                        & 右声道 & \hyperref[fielddesc:LPI2SRXTDMPDMCHAN0EN]{LP\_I2S\_RX\_TDM\_PDM\_CHAN1\_EN} \\ \hline
    \end{tabular}
\end{table} 
\end{itemize}

\subsection{数据格式控制}
数据格式控制分为两个阶段：
\begin{itemize}
    \item 第一阶段：从串行数据流输入转换为待输入数据，并存入 RX FIFO；
    \item 第二阶段：将待输入数据从 RX FIFO 中读出，并进行输入数据模式转换。
\end{itemize}

\subsubsection{通道数据比特顺序} 

\chipname{} LP I2S 通道数据将从高位到低位依次存入待输入数据，并由 \hyperref[fielddesc:LPI2SRXBITORDER]{LP\_I2S\_RX\_BIT\_ORDER} 控制是否调换比特序，当该值为：
\begin{itemize}
\item 0：不调换待输入数据的比特序；
\item 1：调换待输入数据的比特序。
\end{itemize}
至此，数据格式控制的第一阶段完成。

\subsubsection{通道储存数据位宽}

\hyperref[fielddesc:LPI2SRXBITSMOD]{LP\_I2S\_RX\_BITS\_MOD} 决定了每个通道的储存数据位宽。对于 \chipname{} LP I2S，\hyperref[fielddesc:LPI2SRXBITSMOD]{LP\_I2S\_RX\_BITS\_MOD} 的值固定为 15，对应的储存数据位宽为 16。

\subsubsection{通道接收数据位宽} 

\hyperref[fielddesc:LPI2SRXTDMCHANBITS]{LP\_I2S\_RX\_TDM\_CHAN\_BITS} 决定了每个通道接收数据的位宽。

\begin{itemize}
\item 如果通道储存数据位宽小于接收数据宽度，则只有储存数据位宽内的数据位会被保存到内存中。此时，配置 \hyperref[fielddesc:LPI2SRXLEFTALIGN]{LP\_I2S\_RX\_LEFT\_ALIGN} 为：
\begin{itemize}
\item 0：储存数据位于接收数据低位；
\item 1：储存数据位于接收数据高位。
\end{itemize}
\item 当每个通道接收数据的位宽小于储存数据位宽时，会将接收数据高位补 0，作为储存数据。
\end{itemize}

\subsubsection{通道储存数据字节序} 

通道接收数据经过截取/补全后，成为了待储存数据，\hyperref[fielddesc:LPI2SRXBIGENDIAN]{LP\_I2S\_RX\_BIG\_ENDIAN} 用于控制待储存数据的字节序。下表描述了该寄存器对待储存数据的控制（通道储存数据位宽固定为 16）。

\begin{table}[H]
    \centering
    \caption{通道储存数据字节序控制}

    \rowcolor{lightgray}

至此，数据格式控制部分全部完成，数据将存入 LP I2S 内部存储器。

\subsection{内部存储器}
\label{sec:lpi2s-memory}

所有经过数据格式控制的数据，都将存入 LP I2S 内部存储器。LP I2S 内部存储器为 16 位宽的循环 buffer，支持 LP I2S RX 模块写入和系统读取。LP I2S 内部存储器拥有一对由硬件维护的读写指针，写入和读取的操作由读写指针控制。

\begin{enumerate}
    \item 当 LP I2S 写入数据时：
    \begin{itemize}
        \item 存储器已满时：覆写最老的数据，写指针加一，读指针也加一（此时读指针的值为写指针加一）
        \item 存储器未满时：写入数据，写指针加一，读指针不变
    \end{itemize}
    \item 当系统读取 LP I2S 存储器时：
    \begin{itemize}
        \item 存储器已空时：读写指针均不变（此时读指针的值等于写指针）
        \item 存储器非空时：写指针不变，读指针加一
    \end{itemize}
\end{enumerate}

\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
    \item 写指针和读指针的加一操作均为循环加法，即当读写指针到达存储器空间边界时，加一操作使其归零。
    \item 在 LP I2S 内部存储器的结构下，系统永远读取存储器中最老的数据。
\end{itemize}
\end{tiplisting}

\section{中断}
\label{sec:lpi2s-intr}

\chipname{} 的 LP I2S 可以生成中断信号 LP\_I2S\_INTR 并发送给 \hyperref[mod:intmtrx]{中断矩阵}。该中断信号由以下 LP I2S 的内部中断源生成。

\begin{itemize}
\item \label{int:LPI2SRXHUNGINT}LP\_I2S\_RX\_HUNG\_INT：当接收数据超时，触发此中断。例如，LP I2S 配置为从机接收模式，但主机长时间未发送数据，则将触发该中断。超时配置见寄存器 \hyperref[regdesc:LPI2SLCHUNGCONFREG]{LP\_I2S\_LC\_HUNG\_CONF\_REG}。
\item \label{int:LPI2SRXDONEINT}LP\_I2S\_RX\_DONE\_INT：当接收数据完成，触发此中断。
\item 
\label{int:LPI2SRXFIFOMEMUDFINT}LP\_I2S\_RX\_FIFOMEM\_UDF\_INT：当 LP I2S 存储器读空时，触发此中断。
\item 
\label{int:LPI2SRXMEMTHRESHOLDINT}LP\_I2S\_RX\_MEM\_THRESHOLD\_INT：当 LP I2S 存储器中的数据量大于 \hyperref[fielddesc:LPI2SRXMEMTHRESHOLD]{LP\_I2S\_RX\_MEM\_THRESHOLD} 中配置的值时，触发此中断。
\end{itemize}

\begin{tiplisting}
中断、中断源、中断信号等术语的解释以及它们之间的联系请参考章节 \ref{mod:intmtrx} \textit{\nameref{mod:intmtrx}} > \ref{sec:intmtx-int-terminology} \textit{\nameref{sec:intmtx-int-terminology}}。
\end{tiplisting}

每个中断源都有一组通用的配置寄存器，详见章节 \hyperref[interrupt-config-registers]{\textit{中断配置寄存器}}。具体寄存器名称请查看 \ref{sec:lpi2s-reg-summ} \textit{\nameref{sec:lpi2s-reg-summ}}。
