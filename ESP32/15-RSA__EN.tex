\chapter{RSA Accelerator (RSA)}
\label{mod:rsa}

\section{Introduction}

The RSA Accelerator provides hardware support for multiple precision arithmetic operations used in RSA asymmetric cipher algorithms.

Sometimes, multiple precision arithmetic is also called "bignum arithmetic", "bigint arithmetic" or "arbitrary precision arithmetic".

\section{Features}

\begin{itemize}
    \item Support for large-number modular exponentiation
    \item Support for large-number modular multiplication
    \item Support for large-number multiplication
    \item Support for various lengths of operands
\end{itemize}


\section{Functional Description}

\subsection{Initialization}

The RSA Accelerator is activated by enabling the corresponding peripheral clock, and by clearing the DPORT\_RSA\_PD bit in the DPORT\_RSA\_PD\_CTRL\_REG register. This releases the RSA Accelerator from reset.

When the RSA Accelerator is released from reset, the register RSA\_CLEAN\_REG reads 0 and an initialization process begins. Hardware initializes the four memory blocks by setting them to 0. After initialization is complete, RSA\_CLEAN\_REG reads 1. For this reason, software should query RSA\_CLEAN\_REG after being released from reset, and before writing to any RSA Accelerator memory blocks or registers for the first time.

\subsection{Large Number Modular Exponentiation}

Large-number modular exponentiation performs \begin{math}Z = X^{Y} \bmod M\end{math}. The operation is based on Montgomery multiplication. Aside from the arguments
\begin{math}X\end{math},
\begin{math}Y\end{math}, and
\begin{math}M\end{math}, two additional ones are needed --- \begin{math}\overline{r}\end{math} and
\begin{math}M^{\prime}\end{math}. These arguments are calculated in advance by software.

The RSA Accelerator supports operand lengths of
\begin{math}N \in \{512,1024,1536,2048,2560,3072,3584,4096\}\end{math} bits. The bit length of arguments
\begin{math}Z\end{math},
\begin{math}X\end{math},
\begin{math}Y\end{math},
\begin{math}M\end{math}, and 
\begin{math}\overline{r}\end{math}
can be any one from the N set, but all numbers in a calculation must be of the same length. The bit length of \begin{math}M^{\prime}\end{math} is always 32.

To represent the numbers used as operands, define a base-\begin{math}b\end{math} positional notation, as follows:
\begin{gather*}
    b = 2^{32}
\end{gather*}

In this notation, each number is represented by a sequence of base-\begin{math}b\end{math} digits, where each base-\begin{math}b\end{math} digit is a 32-bit word. Representing an \begin{math}N\end{math}-bit number requires \begin{math}n\end{math} base-\begin{math}b\end{math} digits (all of the possible \begin{math}N\end{math} lengths are multiples of 32).
\begin{gather*}
    n = \frac{N}{32} \\
    Z = ( Z_{n-1} Z_{n-2} \cdots Z_{0} )_{b} \\
    X = ( X_{n-1} X_{n-2} \cdots X_{0} )_{b} \\
    Y = ( Y_{n-1} Y_{n-2} \cdots Y_{0} )_{b} \\
    M = ( M_{n-1} M_{n-2} \cdots M_{0} )_{b} \\
    \overline{r} = ( \overline{r}_{n-1} \overline{r}_{n-2} \cdots \overline{r}_{0} )_{b}
\end{gather*}

Each of the \begin{math}n\end{math} values in 
\begin{math}Z_{n-1}\end{math} \verb+~+ \begin{math}Z_{0}\end{math},
\begin{math}X_{n-1}\end{math} \verb+~+ \begin{math}X_{0}\end{math},
\begin{math}Y_{n-1}\end{math} \verb+~+ \begin{math}Y_{0}\end{math},
\begin{math}M_{n-1}\end{math} \verb+~+ \begin{math}M_{0}\end{math},
\begin{math}\overline{r}_{n-1}\end{math} \verb+~+ \begin{math}\overline{r}_{0}\end{math} represents one base-\begin{math}b\end{math} digit (a 32-bit word).

\begin{math}Z_{n-1}\end{math},
\begin{math}X_{n-1}\end{math},
\begin{math}Y_{n-1}\end{math},
\begin{math}M_{n-1}\end{math} and
\begin{math}\overline{r}_{n-1}\end{math} are the most significant bits of
\begin{math}Z\end{math},
\begin{math}X\end{math},
\begin{math}Y\end{math},
\begin{math}M\end{math}, while
\begin{math}Z_{0}\end{math},
\begin{math}X_{0}\end{math},
\begin{math}Y_{0}\end{math},
\begin{math}M_{0}\end{math} and
\begin{math}\overline{r}_{0}\end{math} are the least significant bits.

If we define
\begin{gather*}
    R = b^{n}
\end{gather*}

then, we can calculate the additional arguments, as follows:
\begin{gather}
    \overline{r} = R^{2} \bmod M \label{equation:r_bar} \\
    \begin{cases}
        \displaystyle M^{\prime\prime} \times M + 1 = R \times R^{-1} \\
        \displaystyle M^{\prime} = M^{\prime\prime} \bmod b
    \end{cases} \label{equation:M_prime}
\end{gather}


(Equation \ref{equation:M_prime} is written in a form suitable for calculations using the extended binary GCD algorithm.)

Software can implement large-number modular exponentiations in the following order:

\begin{enumerate}

    \item Write \begin{math}( \frac{N}{512} - 1 )\end{math} to RSA\_MODEXP\_MODE\_REG.

    \item Write
        \begin{math}X_{i}\end{math},
        \begin{math}Y_{i}\end{math},
        \begin{math}M_{i}\end{math} and
        \begin{math}\overline{r}_{i}\end{math}
        (\begin{math}i \in [0,n) \cap \mathbb{N}\end{math})
        to memory blocks RSA\_X\_MEM, RSA\_Y\_MEM, RSA\_M\_MEM and RSA\_Z\_MEM. The capacity of each memory block is 128 words. Each word of each memory block can store one base-\begin{math}b\end{math} digit. The memory blocks use the little endian format for storage, i.e. the least significant digit of each number is in the lowest address.
        
       Users need to write data to each memory block only according to the length of the number; data beyond this length are ignored.
        
    \item Write \begin{math}M^{\prime}\end{math} to RSA\_M\_PRIME\_REG.

    \item Write 1 to RSA\_MODEXP\_START\_REG.

    \item Wait for the operation to be completed. Poll RSA\_INTERRUPT\_REG until it reads 1, or until the RSA\_INTR interrupt is generated.
        
    \item Read the result \begin{math}Z_{i}\end{math}
        (\begin{math}i \in [0,n) \cap \mathbb{N}\end{math})
        from RSA\_Z\_MEM.

    \item Write 1 to RSA\_INTERRUPT\_REG to clear the interrupt.

\end{enumerate}

After the operation, the RSA\_MODEXP\_MODE\_REG register, memory blocks RSA\_Y\_MEM and RSA\_M\_MEM, as well as the RSA\_M\_PRIME\_REG will not have changed. However, \begin{math}X_{i}\end{math} in RSA\_X\_MEM and \begin{math}\overline{r}_{i}\end{math} in RSA\_Z\_MEM will have been overwritten. In order to perform another operation, refresh the registers and memory blocks, as required.

\subsection{Large Number Modular Multiplication}

Large-number modular multiplication performs \begin{math}Z = X \times Y \bmod M\end{math}. This operation is based on Montgomery multiplication. The same values \begin{math}\overline{r}\end{math} and
\begin{math}M^{\prime}\end{math} are derived by software using the formulas \ref{equation:r_bar} and \ref{equation:M_prime} shown above.

The RSA Accelerator supports large-number modular multiplication with eight different operand lengths, which are the same as in the large-number modular exponentiation. The operation is performed by a combination of software and hardware. The software performs two hardware operations in sequence.

The software process is as follows:

\begin{enumerate}

    \item Write \begin{math}( \frac{N}{512} - 1 )\end{math} to RSA\_MULT\_MODE\_REG.

    \item Write
        \begin{math}X_{i}\end{math}, 
        \begin{math}M_{i}\end{math} and
        \begin{math}\overline{r}_{i}\end{math}
        (\begin{math}i \in [0,n) \cap \mathbb{N}\end{math}) to registers RSA\_X\_MEM, RSA\_M\_MEM and RSA\_Z\_MEM. Write data to each memory block only according to the length of the number. Data beyond this length are ignored.
    \item Write \begin{math}M^{\prime}\end{math} to RSA\_M\_PRIME\_REG.

    \item Write 1 to RSA\_MULT\_START\_REG.

    \item Wait for the first round of the operation to be completed. Poll RSA\_INTERRUPT\_REG until it reads 1, or until the RSA\_INTR interrupt is generated.
        
    \item Write 1 to RSA\_INTERRUPT\_REG to clear the interrupt.

    \item Write 
        \begin{math}Y_{i}\end{math}
        (\begin{math}i \in [0,n) \cap \mathbb{N}\end{math})
  to RSA\_X\_MEM.
  
Users need to write to the memory block only according to the length of the number. Data beyond this length are ignored.

    \item Write 1 to RSA\_MULT\_START\_REG.

    \item Wait for the second round of the operation to be completed. Poll RSA\_INTERRUPT\_REG until it reads 1, or until the RSA\_INTR interrupt is generated.
    
    \item Read the result \begin{math}Z_{i}\end{math}
        (\begin{math}i \in [0,n) \cap \mathbb{N}\end{math}) from RSA\_Z\_MEM.

    \item Write 1 to RSA\_INTERRUPT\_REG to clear the interrupt.

\end{enumerate}

After the operation, the RSA\_MULT\_MODE\_REG register, and memory blocks RSA\_M\_MEM and RSA\_M\_PRIME\_REG remain unchanged. Users do not need to refresh these registers or memory blocks if the values remain the same.

\subsection{Large Number Multiplication}

Large-number multiplication performs \begin{math}Z = X \times Y\end{math}.
The length of \begin{math}Z\end{math} is twice that of \begin{math}X\end{math} and \begin{math}Y\end{math}. Therefore, the RSA Accelerator supports large-number multiplication with only four operand lengths of \begin{math}N \in \{512,1024,1536,2048\}\end{math} bits. The length \begin{math}\hat{N}\end{math} of the result \begin{math}Z\end{math} is \begin{math}2 \times N\end{math} bits.

Operands \begin{math}X\end{math} and \begin{math}Y\end{math} need to be extended to form arguments \begin{math}\hat{X}\end{math} and \begin{math}\hat{Y}\end{math} which have the same length (\begin{math}\hat{N}\end{math} bits) as the result \begin{math}Z\end{math}. \begin{math}X\end{math} is left-extended and \begin{math}Y\end{math} is right-extended, and defined as follows:
\begin{gather*}
    n = \frac{N}{32} \\
    \hat{N} = 2 \times N\\
    \hat{n} = \frac{\hat{N}}{32} = 2n\\
    \hat{X}
        = ( \hat{X}_{\hat{n}-1} \hat{X}_{\hat{n}-2} \cdots \hat{X}_{0} )_{b}
        = ( \underbrace{0 0 \cdots 0}_{n}
            X )_{b}
        = ( \underbrace{0 0 \cdots 0}_{n}
            X_{n-1} X_{n-2} \cdots X_{0} )_{b} \\
    \hat{Y}
        = ( \hat{Y}_{\hat{n}-1} \hat{Y}_{\hat{n}-2} \cdots \hat{Y}_{0} )_{b}
        = ( Y
            \underbrace{0 0 \cdots 0}_{n} )_{b}
        = ( Y_{n-1} Y_{n-2} \cdots Y_{0}
            \underbrace{0 0 \cdots 0}_{n} )_{b}
\end{gather*}

Software performs the operation in the following order:

\begin{enumerate}

    \item Write \begin{math}( \frac{\hat{N}}{512} - 1 + 8 )\end{math} to RSA\_MULT\_MODE\_REG.

    \item Write
        \begin{math}\hat{X}_{i}\end{math} and
        \begin{math}\hat{Y}_{i}\end{math}
        (\begin{math}i \in [0,\hat{n}) \cap \mathbb{N}\end{math})
to RSA\_X\_MEM and RSA\_Z\_MEM, respectively.

     Write the valid data into each number's memory block, according to their lengths. Values beyond this length are ignored. Half of the base-\begin{math}b\end{math} positional notations written to the memory are zero (using the derivations shown above). These zero values are indispensable.

    \item Write 1 to RSA\_MULT\_START\_REG.

    \item Wait for the operation to be completed. Poll RSA\_INTERRUPT\_REG until it reads 1, or until the RSA\_INTR interrupt is generated.

    \item Read the result \begin{math}Z_{i}\end{math}
        (\begin{math}i \in [0,\hat{n}) \cap \mathbb{N}\end{math}) from  RSA\_Z\_MEM.

    \item Write 1 to RSA\_INTERRUPT\_REG to clear the interrupt.

\end{enumerate}

After the operation, only the RSA\_MULT\_MODE\_REG register remains unmodified.
