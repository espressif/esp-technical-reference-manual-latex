\chapter{AES Accelerator (AES)}
\label{mod:aes}

\section{Introduction}

The AES Accelerator speeds up AES operations significantly, compared to AES algorithms implemented solely in software. The AES Accelerator supports six algorithms of FIPS PUB 197, specifically AES-128, AES-192 and AES-256 encryption and decryption.

\section{Features}

\begin{itemize}
    \item Supports AES-128 encryption and decryption
    \item Supports AES-192 encryption and decryption
    \item Supports AES-256 encryption and decryption
    \item Supports four variations of key endianness and four variations of text endianness
\end{itemize}

\section{Functional Description}

\subsection{AES Algorithm Operations}

The AES Accelerator supports six algorithms of FIPS PUB 197, specifically AES-128, AES-192 and AES-256 encryption and decryption. The AES\_MODE\_REG register can be configured to different values to enable different algorithm operations, as shown in Table \ref{table:AESOperation}.
\begin{longtable}[c]{ |p{7cm}|p{8cm}| }
\caption{Operation Mode} \label{table:AESOperation} \\
\hline \rowcolor{lightgray}
AES\_MODE\_REG[2:0]     & Operation\\\hline
    0                       & AES-128 Encryption    \\ \hline
    1                       & AES-192 Encryption    \\ \hline
    2                       & AES-256 Encryption    \\ \hline
    4                       & AES-128 Decryption    \\ \hline
    5                       & AES-192 Decryption    \\ \hline
    6                       & AES-256 Decryption    \\ \hline
\end{longtable}


\subsection{Key, Plaintext and Ciphertext}

The encryption or decryption key is stored in AES\_KEY\_\textcolor{red}{\it{n}}\_REG, which is a set of eight 32-bit registers.
For AES-128 encryption/decryption, the 128-bit key is stored in AES\_KEY\_0\_REG \verb+~+ AES\_KEY\_3\_REG.
For AES-192 encryption/decryption, the 192-bit key is stored in AES\_KEY\_0\_REG \verb+~+ AES\_KEY\_5\_REG.
For AES-256 encryption/decryption, the 256-bit key is stored in AES\_KEY\_0\_REG \verb+~+ AES\_KEY\_7\_REG.

Plaintext and ciphertext is stored in the AES\_TEXT\_\textcolor{red}{\it{m}}\_REG registers. There are four 32-bit registers.
To enable AES-128/192/256 encryption, initialize the AES\_TEXT\_\textcolor{red}{\it{m}}\_REG registers with plaintext before encryption. When encryption is finished, the AES Accelerator will store back the resulting ciphertext in the AES\_TEXT\_\textcolor{red}{\it{m}}\_REG registers. To enable AES-128/192/256 decryption, initialize the AES\_TEXT\_\textcolor{red}{\it{m}}\_REG registers with ciphertext before decryption. When decryption is finished, the AES Accelerator will store back the resulting plaintext in the AES\_TEXT\_\textcolor{red}{\it{m}}\_REG registers.


\subsection{Endianness}

\textbf{Key Endianness}

Bit 0 and bit 1 in AES\_ENDIAN\_REG define the key endianness. For detailed information, please see Table \ref{table:AES128KeyEndian}, Table \ref{table:AES192KeyEndian} and Table \ref{table:AES256KeyEndian}.
w[0] \verb+~+ w[3] in Table \ref{table:AES128KeyEndian},
w[0] \verb+~+ w[5] in Table \ref{table:AES192KeyEndian} and w[0] \verb+~+ w[7] in Table \ref{table:AES256KeyEndian} are “the first Nk words of the expanded key” as specified in “5.2: Key Expansion” of FIPS PUB 197. “Column Bit” specifies the bytes in the word from w[0] to w[7]. The bytes of AES\_KEY\_\textcolor{red}{\it{n}}\_REG comprise “the first Nk words of the expanded key”.

\textbf{Text Endianness}

Bit 2 and bit 3 in AES\_ENDIAN\_REG define the endianness of input text, while Bit 4 and Bit 5 define the endianness of output text.
The input text refers to the plaintext in AES-128/192/256 encryption and the ciphertext in decryption.
The output text refers to the ciphertext in AES-128/192/256 encryption and the plaintext in decryption.
For details, please see Table \ref{table:AESPlaintextCiphertextEndian}.
“State” in Table \ref{table:AESPlaintextCiphertextEndian} is defined as that in “3.4: The State” of FIPS PUB 197: “The AES algorithm operations are performed on a two-dimensional array of bytes called the State”.
The ciphertext or plaintexts stored in each byte of AES\_TEXT\_\textcolor{red}{\it{m}}\_REG comprise the State.

\subsection{Encryption and Decryption Operations}

\textbf{Single Operation}

\begin{enumerate}
    \item Initialize AES\_MODE\_REG, AES\_KEY\_\textcolor{red}{\it{n}}\_REG, AES\_TEXT\_\textcolor{red}{\it{m}}\_REG and AES\_ENDIAN\_REG.
    \item Write 1 to AES\_START\_REG.
    \item Wait until AES\_IDLE\_REG reads 1.
    \item Read results from AES\_TEXT\_\textcolor{red}{\it{m}}\_REG.
\end{enumerate}

\textbf{Consecutive Operations}

Every time an operation is completed, only AES\_TEXT\_\textcolor{red}{\it{m}}\_REG is modified by the AES Accelerator. Initialization can, therefore, be simplified in a series of consecutive operations.

\begin{enumerate}
    \item Update contents of AES\_MODE\_REG, AES\_KEY\_\textcolor{red}{\it{n}}\_REG and AES\_ENDIAN\_REG, if required.
    \item Load AES\_TEXT\_\textcolor{red}{\it{m}}\_REG.
    \item Write 1 to AES\_START\_REG.
    \item Wait until AES\_IDLE\_REG reads 1.
    \item Read results from AES\_TEXT\_\textcolor{red}{\it{m}}\_REG.
\end{enumerate}

\subsection{Speed}

The AES Accelerator requires 11 to 15 clock cycles to encrypt a message block, and 21 or 22 clock cycles to decrypt a message block.
