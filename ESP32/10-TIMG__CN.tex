\chapter{定时器组 (TIMG)}
\label{mod:timg}

\section{概述}
ESP32 内置 4 个 64-bit 通用定时器。每个定时器包含一个 16-bit 预分频器和一个 64-bit 可自动重新加载向上／向下计数器。

ESP32 的定时器分为 2 组，每组 2 个。 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}} 的 \textcolor{red}{\it{n}} 代表组别，\textcolor{red}{\it{x}} 代表定时器编号。 

定时器特性：
\begin{itemize}
    \item 16-bit 时钟预分频器，分频系数为 2-65536
    \item 64-bit 时基计数器
    \item 可配置的向上／向下时基计数器：增加或减少
    \item 暂停和恢复时基计数器
    \item 报警时自动重新加载
    \item 当报警值溢出/低于保护值时报警
    \item 软件控制的即时重新加载
    \item 电平触发中断和边沿触发中断
\end{itemize}

\section{功能描述}

\subsection{16-bit 预分频器}
每个定时器都以 APB 时钟（缩写 APB\_CLK，频率通常为 80 MHz）作为基础时钟。16-bit 预分频器对 APB 时钟进行分频，产生时基计数器时钟（TB\_clk）。 TB\_clk 每过一个周期，时基计数器会向上数一或者向下数一。
在使用寄存器 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_DIVIDER 配置分频器除数前，必须关闭定时器（清零TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_DIVIDER）。定时器使能时配置预分频器会导致不可预知的结果。预分频器可以对 APB 时钟进行 2 到 65536 的分频。具体来说， TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_DIVIDER 为 1 或 2 时，时钟分频器是 2； TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_DIVIDER 为 0 时，时钟分频器是 65536。如 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_DIVIDER 为其他任意值，时钟会被相同数值分频。

\subsection{64-bit 时基计数器}
TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_INCREASE 置 1 或清零可以将 64-bit 时基计数器分别配置为向上计数或向下计数。同时，64-bit 时基计数器支持自动重新加载和软件即时重新加载，计数器达到软件设定值时会触发报警事件。

TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_EN 置 1 或清零可以使能或关闭计数。 清零后计数器暂停计数，并会在 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_
EN 重新置 1 前保持其值不变。清零 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_EN 会重新加载计数器并改变计数器的值，
但在设置 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_EN 前计数不会恢复。

软件可以通过寄存器 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_LOAD\_LO 和 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_LOAD\_HI 重置计数器的值。重新加载时，寄存器 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_LOAD\_LO 和 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_LOAD\_HI 的值才会被更新到 64-bit 时基计数器内。报警（报警时自动重新加载）或软件（软件即时重新加载）会触发重新加载。寄存器 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_AUTORELOAD 置 1 可以使能报警时自动重新加载。如果报警时自动重新加载未被使能，时基计数器会在报警后继续向上计数或向下计数。在寄存器 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_LOAD\_REG 上写任意值可以触发软件即时重新加载，写值时计数器值会立刻改变。软件也能通过改变 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_INCREASE 的值立刻改变时基计数器计数方向。

软件也能读取时基计数器。但由于计数器是 64-bit，CPU 只能以两个 32-bit 值的形式读值。计数器值首先需要被锁入 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}LO\_REG 和 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}HI\_REG。在 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}UPDATE\_REG 上写任意值可以马上将 64-bit 定时器值锁入两个寄存器。之后，软件可以在任何时间读取寄存器。这样可以防止读取计时器低字和高字时出现读值错误。

\subsection{报警产生}

定时器可以触发报警，报警则会引发重新加载和／或触发中断。如报警寄存器 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_ALARMLO\_REG 和 TIMG\textcolor{red}{\it{n}}\_T\textcolor{red}{\it{x}}\_ALARMHI\_REG 的值等于当前定时器的值，则报警触发。为解决寄存器设置过晚，计数器值超过报警值的问题，当前定时器值高于（适用于向上定时器）或低于（适用于向下定时器）当前报警值也会触发报警。在这种情况下，使能报警功能会马上触发报警。报警使能后，报警使能位自动清零。

\subsection{MWDT}
每个定时器模块另包含一个主系统看门狗定时器（缩写为 MWDT）和有关寄存器。本章列出了寄存器相关信息，功能描述请参阅\hyperref[mod:wdt]{看门狗定时器}章节 。

\subsection{中断}
\begin{itemize}
\item \label{int:TIMERSINTWDTINT}TIMG\textcolor{red}{\it{n}}\_INT\_WDT\_INT 该中断在看门狗定时器中断阶段超时后产生。
\item \label{int:TIMERSINTT1INT}TIMG\textcolor{red}{\it{n}}\_INT\_T1\_INT 该中断由定时器 1 上的报警事件产生。
\item \label{int:TIMERSINTT0INT}TIMG\textcolor{red}{\it{n}}\_INT\_T0\_INT 该中断由定时器 0 上的报警事件产生。
\end{itemize}
