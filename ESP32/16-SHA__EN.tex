\hypertarget{sha}{}
\chapter{SHA Accelerator (SHA)}
\label{mod:sha}

\section{Introduction}

The SHA Accelerator is included to speed up SHA hashing operations significantly, compared to SHA hashing algorithms implemented solely in software. The SHA Accelerator supports four algorithms of FIPS PUB 180-4, specifically SHA-1, SHA-256, SHA-384 and SHA-512.


\section{Features}
Hardware support for popular secure hashing algorithms:
\begin{itemize}
    \item SHA-1
    \item SHA-256
    \item SHA-384
    \item SHA-512
\end{itemize}


\section{Functional Description}


\subsection{Padding and Parsing the Message}

The SHA Accelerator can only accept one message block at a time. Software divides the message into blocks according to “5.2 Parsing the Message” in FIPS PUB 180-4 and writes one block to the SHA\_TEXT\_\textcolor{red}{\it{n}}\_REG registers each time. For SHA-1 and SHA-256, software writes a 512-bit message block to SHA\_TEXT\_0\_REG \verb+ ~ + SHA\_TEXT\_15\_REG each time. For SHA-384 and SHA-512, software writes a 1024-bit message block to SHA\_TEXT\_0\_REG \verb+ ~ + SHA\_TEXT\_31\_REG each time.

The SHA Accelerator is unable to perform the padding operation of “5.1 Padding the Message” in FIPS PUB 180-4; Note that the user software is expected to pad the message before feeding it into the accelerator.


As described in “2.2.1: Parameters” in FIPS PUB 180-4, “$M_{0}^{(i)}$ is the leftmost word of message block i”. $M_{0}^{(i)}$ is stored in SHA\_TEXT\_0\_REG. In the same fashion, the SHA\_TEXT\_1\_REG register stores the second left-most word of a message block $M_{1}^{(N)}$, etc.

\subsection{Message Digest}

When the hashing operation is finished, the message digest will be refreshed by SHA Accelerator and will be stored in SHA\_TEXT\_\textcolor{red}{\it{n}}\_REG. SHA-1 produces a 160-bit message digest and stores it in SHA\_TEXT\_0\_REG \verb+~+ SHA\_TEXT\_4\_REG. SHA-256 produces a 256-bit message digest and stores it in SHA\_TEXT\_0\_REG \verb+~+ SHA\_TEXT\_7\_REG. SHA-384 produces a 384-bit message digest and stores it in SHA\_TEXT\_0\_REG \verb+~+ SHA\_TEXT\_11\_REG. SHA-512 produces a 512-bit message digest and stores it in SHA\_TEXT\_0\_REG \verb+~+ SHA\_TEXT\_15\_REG.

As described in “2.2.1 Parameters” in FIPS PUB 180-4, “$H^{(N)}$ is the final hash value, and is used to determine the message digest”, while “$H_{0}^{(i)}$ is the leftmost word of hash value i”, so the leftmost word $H_{0}^{(N)}$ in the message digest is stored in SHA\_TEXT\_0\_REG. In the same fashion, the second leftmost word $H_{1}^{(N)}$ in the message digest is stored in SHA\_TEXT\_1\_REG, etc.



\subsection{Hash Operation}

There is a set of control registers for SHA-1, SHA-256, SHA-384 and SHA-512, respectively; different hashing algorithms use different control registers.\\
SHA-1 uses SHA\_SHA1\_START\_REG, SHA\_SHA1\_CONTINUE\_REG, SHA\_SHA1\_LOAD\_REG and\\
SHA\_SHA1\_BUSY\_REG.\\ SHA-256 uses SHA\_SHA256\_START\_REG, SHA\_SHA256\_CONTINUE\_REG, \\
SHA\_SHA256\_LOAD\_REG and SHA\_SHA256\_BUSY\_REG. SHA-384 uses SHA\_SHA384\_START\_REG,\\
SHA\_SHA384\_CONTINUE\_REG, SHA\_SHA384\_LOAD\_REG and SHA\_SHA384\_BUSY\_REG.\\
SHA-512 uses SHA\_SHA512\_START\_REG, SHA\_SHA512\_CONTINUE\_REG, SHA\_SHA512\_LOAD\_REG\\ and SHA\_SHA512\_BUSY\_REG. The following steps describe the operation in a detailed manner.


\begin{enumerate}
    \item Feed the accelerator with the first message block:
    \begin{enumerate}
        \item Use the first message block to initialize SHA\_TEXT\_\textcolor{red}{\it{n}}\_REG.
		\item Write 1 to SHA\_\textcolor{red}{\it{X}}\_START\_REG.
		\item Wait for SHA\_\textcolor{red}{\it{X}}\_BUSY\_REG to read 0, indicating that the operation is completed.
    \end{enumerate}

    \item Similarly, feed the accelerator with subsequent message blocks:
    \begin{enumerate}
        \item Initialize SHA\_TEXT\_\textcolor{red}{\it{n}}\_REG using the subsequent message block.
		\item Write 1 to SHA\_\textcolor{red}{\it{X}}\_CONTINUE\_REG.
		\item Wait for SHA\_\textcolor{red}{\it{X}}\_BUSY\_REG to read 0, indicating that the operation is completed.
    \end{enumerate}

    \item Get message digest:
    \begin{enumerate}
        \item Write 1 to SHA\_\textcolor{red}{\it{X}}\_LOAD\_REG.
		\item Wait for SHA\_\textcolor{red}{\it{X}}\_BUSY\_REG to read 0, indicating that operation is completed.
		\item Read message digest from SHA\_TEXT\_\textcolor{red}{\it{n}}\_REG.
	\end{enumerate}
\end{enumerate}

\subsection{Speed}

The SHA Accelerator requires 60 to 100 clock cycles to process a message block and 8 to 20 clock cycles to calculate the final digest.
