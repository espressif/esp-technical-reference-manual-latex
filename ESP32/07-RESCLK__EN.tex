\section{System Reset}
\subsection{Introduction}
The ESP32 has three reset levels: CPU reset, Core reset, and System reset. None of these reset levels clear the RAM. Figure \ref{figure:Reset} shows the subsystems included in each reset level.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.5\textwidth]{./07-RESCLK/figures/reset_20211011.eps}
    \caption{System Reset}
    \label{figure:Reset}
\end{figure}

\begin{itemize}
    \item CPU reset: Only resets the registers of one or both of the CPU cores.
    \item Core reset: Resets all the digital registers, including CPU cores, external GPIO and digital GPIO. The RTC is not reset.
    \item System reset: Resets all the registers on the chip, including those of the RTC.
\end{itemize}

\subsection{Reset Source}

\section{System Clock}

\subsection{Introduction}

The ESP32 integrates multiple clock sources for the CPU cores, the peripherals and the RTC. These clocks can be configured to meet different requirements.
Figure \ref{figure:Clock} shows the system clock structure.

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{./07-RESCLK/figures/clock.png}
    \caption{System Clock}
    \label{figure:Clock}
\end{figure}


\subsection{Clock Source}
The ESP32 can use an external crystal oscillator, an internal PLL or an oscillating circuit as a clock source. Specifically, the clock sources available are:
\begin{itemize}
    \item High Speed Clocks
        \begin{itemize}
            \item PLL\_CLK is an internal PLL clock with a frequency of 320 MHz or 480 MHz.
            \item XTL\_CLK is a clock signal generated using an external crystal with a frequency range of 2 \verb+~+ 40 MHz.
        \end{itemize}
    \item Low Power Clocks
        \begin{itemize}
            \item XTL32K\_CLK is a clock generated using an external crystal with a frequency of 32 KHz.
	        \item RC\_FAST\_CLK is an internal clock with a default frequency of 8 MHz. This frequency is adjustable.
	        \item RC\_FAST\_DIV\_CLK is divided from RC\_FAST\_CLK. Its frequency is (RC\_FAST\_CLK / 256). With the default RC\_FAST\_CLK frequency of  8 MHz, this clock runs at 31.250 KHz.
	        \item RC\_SLOW\_CLK is an internal low power clock with a default frequency of 150 KHz. This frequency is adjustable.
	    \end{itemize}
    \item Audio Clock
        \begin{itemize}
            \item APLL\_CLK is an internal Audio PLL clock with a frequency range of 16 \verb+~+ 128 MHz.
	    \end{itemize}
\end{itemize}

\subsection{CPU Clock}

As Figure \ref{figure:Clock} shows, CPU\_CLK is the master clock for both CPU cores. CPU\_CLK clock can be as high as 240 MHz when the CPU is in high performance mode. Alternatively, the CPU can run at lower frequencies to reduce power consumption.

\subsubsection{APLL\_SCLK Source}

The APLL\_CLK is sourced from PLL\_CLK, with its output frequency configured using the APLL configuration registers.

\subsubsection{PLL\_F160M\_CLK Source}

PLL\_F160M\_CLK is divided from PLL\_CLK by automatically adjusting the clock division and its frequency is always 160 MHz. 

\subsubsection{Clock Source Considerations}

Most peripherals will operate using the APB\_CLK frequency as a reference. When this frequency changes, the peripherals will need to update their clock configuration to operate at the same frequency after the change. Peripherals accessing REF\_TICK can continue operating normally when switching clock sources, without changing clock source. Please see Table \ref{table:peri clk owner} for details.

The LED PWM module can use RC\_FAST\_CLK as a clock source when APB\_CLK is disabled. In other words, when the system is in low-power consumption mode (see Chapter \ref{mod:lowpowm} \textit{\nameref{mod:lowpowm}}), normal peripherals will be halted (APB\_CLK is turned off),  but the LED PWM can work normally via RC\_FAST\_CLK.

\subsection{Wi-Fi BT Clock}

Wi-Fi and BT can only operate if APB\_CLK uses PLL\_CLK as its clock source. Suspending PLL\_CLK requires Wi-Fi and BT to both have entered low-power consumption mode first.

For LOW\_POWER\_CLK, one of XTL32K\_CLK, \hyperlink{SLOW_CLK}{RTC\_SLOW\_CLK}, RC\_FAST\_CLK or XTL\_CLK can be selected as the low-power consumption mode clock source for Wi-Fi and BT.

\subsection{RTC Clock}

The clock sources of \hypertarget{SLOW_CLK}{RTC\_SLOW\_CLK} and RTC\_FAST\_CLK are low-frequency clocks. The RTC module can operate when most other clocks are stopped.

RTC\_SLOW\_CLK is used to clock the Power Management module. It can be sourced from RC\_SLOW\_CLK, XTL32K\_CLK or RC\_FAST\_DIV\_CLK.

RTC\_FAST\_CLK is used to clock the On-chip Sensor module. It can be sourced from a divided XTL\_CLK or from RC\_FAST\_CLK.

\subsection{Audio PLL}\label{audiopll}
The operation of audio and other time-critical data-transfer applications requires highly-configurable, low-jitter, and accurate clock sources. The clock sources derived from system clocks that serve digital peripherals may carry jitter and, therefore, they do not support a high-precision clock frequency setting.

Providing an integrated precision clock source can minimize system cost. To this end, ESP32 integrates an audio PLL.


The Audio PLL formula is as follows:

\[
f_{\rm{out}} = \frac{f_{\rm{xtal}}(\rm{sdm2}+\frac{\rm{sdm1}}{2^8}+\frac{\rm{sdm0}}{2^{16}}+4)}{2(odiv+2)}
\]

The parameters of this formula are defined below:
\begin{itemize}
    \item \begin{math}f_{\rm{xtal}}\end{math}: the frequency of the crystal oscillator, usually 40 MHz;
    \item sdm0: the value is 0 \verb+~+ 255;
    \item sdm1: the value is 0 \verb+~+ 255;
    \item sdm2: the value is 0 \verb+~+ 63;
    \item odiv: the value is 0 \verb+~+ 31;
    \item The operating frequency range of the numerator is 350 MHz \verb+~+ 500 MHz

\[
350 MHz<{f_{\rm{xtal}}(\rm{sdm2}+\frac{\rm{sdm1}}{2^8}+\frac{\rm{sdm0}}{2^{16}}+4)}<500 MHz
\]

\end{itemize}

Please note that sdm1 and sdm0 are not available on revision0 of ESP32. Please consult the silicon revision in \href{\linkprefix\linkchiperrataEN}{\chipseries{} Series SoC Errata} for further details.

Audio PLL can be manually enabled or disabled via registers RTC\_CNTL\_PLLA\_FORCE\_PU and RTC\_CNTL\_PLLA\\\_FORCE\_PD, respectively. Disabling it takes priority over enabling it. When RTC\_CNTL\_PLLA\_FORCE\_PU and RTC\_CNTL\_PLLA\_FORCE\_PD are 0, PLL will follow the state of the system, i.e., when the system enters sleep mode, PLL will be disabled automatically; when the system wakes up, PLL will be enabled automatically.
