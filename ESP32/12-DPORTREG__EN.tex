\chapter{DPort Registers}
\label{mod:dportreg}

\section{Introduction}

The ESP32 integrates a large number of peripherals, and enables the control of individual peripherals to achieve optimal characteristics in performance-vs-power-consumption scenarios. The DPort registers control clock management (clock gating), power management, and the configuration of peripherals and core-system modules.
The system arranges each module with configuration registers contained in the DPort Register.

\section{Features}
DPort registers correspond to different peripheral blocks and core modules:

\begin{itemize}
    \item System and memory
    \item Reset and clock
    \item Interrupt matrix
    \item DMA
    \item MPU/MMU
    \item APP\_CPU controller
    \item Peripheral clock gating and reset
\end{itemize}

\section{Functional Description}

\subsection{System and Memory Register}
System and memory registers are used for system and memory configuration, such as cache configuration and memory remapping. They are listed in Section \ref{sec:dport-reg-summ}, categorized as "System and memory registers". For a detailed description of these registers, please refer to Chapter  \textit{\nameref{mod:sysmem}}.


\subsection{Reset and Clock Registers}
Reset and clock registers are listed in Section \ref{sec:dport-reg-summ}, categorized as "Reset and clock registers". For a detailed description of these registers, please refer to Chapter \textit{\nameref{mod:resclk}}.


\subsection{Interrupt Matrix Register}
\label{sec:interrupt-matrix-register}
The interrupt matrix registers are used for configuring and mapping interrupts through the interrupt matrix. They are listed in Section \ref{sec:dport-reg-summ}, categorized as "Interrupt matrix registers". For a detailed description of these registers, please refer to Chapter \textit{\nameref{mod:intmtrx}}.


\subsection{DMA Registers}
DMA registers are used for the SPI DMA configuration. They are listed in Section \ref{sec:dport-reg-summ}, categorized as "DMA registers". For a detailed description of these registers, please refer to Chapter \textit{\nameref{mod:dma}}.


\subsection{MPU/MMU Registers}
MPU/MMU registers are used for MPU/MMU configuration and operation control. They are listed in Section \ref{sec:dport-reg-summ}, categorized as "MPU/MMU registers". For a detailed description of these registers, please refer to Chapter \textit{\nameref{mod:mmumpu}}.


\subsection{APP\_CPU Controller Registers}

APP\_CPU controller registers are used for some basic configuration of the APP\_CPU, such as performing a stalling execution, and for configuring the ROM boot jump address. The registers are listed in Section \ref{sec:dport-reg-summ}, categorized as "APP\_CPU controller registers". A detailed description of these registers is provided in section \ref{sec:dport-reg}. Note that reset bits are not self-clearing.


\subsection{Peripheral Clock Gating and Reset}\label{sec:peri-clock-gating-reset}

The following registers are used for controlling the clock gating and reset of different peripherals. A detailed description of these registers is provided in section \ref{sec:dport-reg}.

\begin{itemize}
    \item \hyperref[regdesc:PERICLKENREG]{DPORT\_PERI\_CLK\_EN\_REG}
    \item \hyperref[regdesc:PERIRSTENREG]{DPORT\_PERI\_RST\_EN\_REG}
    \item \hyperref[regdesc:PERIPCLKENREG]{DPORT\_PERIP\_CLK\_EN\_REG}
    \item \hyperref[regdesc:PERIPRSTENREG]{DPORT\_PERIP\_RST\_EN\_REG}
    \item \hyperref[regdesc:WIFICLKENREG]{DPORT\_WIFI\_CLK\_EN\_REG}
    \item \hyperref[regdesc:WIFIRSTENREG]{DPORT\_WIFI\_RST\_EN\_REG}
\end{itemize}

\textbf{Notice}:
\begin{itemize}
    \item Clock gating and reset registers are active high.
    \item Reset registers cannot be cleared by hardware. Therefore, SW reset clear is required after setting the reset registers.
    \item ESP32 features low power consumption. This is why some peripheral clocks are gated (disabled) by default. Before using any of these peripherals, it is mandatory to enable the clock for the given peripheral by setting the corresponding CLK\_EN bit to 1, and release the peripheral from reset state to make it operational by setting the RST\_EN bit to 0.

\end{itemize}
