\chapter{Motor Control PWM (MCPWM)}
\label{mod:mcpwm}

\section{Introduction}
The \textbf{M}otor \textbf{C}ontrol \textbf{P}ulse \textbf{W}idth \textbf{M}odulator (MCPWM) peripheral is intended for motor and power control. It provides six PWM outputs that can be set up to operate in several topologies. One common topology uses a pair of PWM outputs driving an H-bridge to control motor rotation speed and rotation direction.

The timing and control resources inside are allocated into two major types of submodules: PWM timers and PWM operators. Each PWM timer provides timing references that can either run freely or be synced to other timers or external sources. Each PWM operator has all necessary control resources to generate waveform pairs for one PWM channel. The MCPWM peripheral also contains a dedicated capture submodule that is used in systems where accurate timing of external events is important.

ESP32 contains two MCPWM peripherals: MCPWM0 and MCPWM1. Their \hyperref[section:register_summary]{control registers} are located in 4-KB memory blocks starting at memory locations 0x3FF5E000 and 0x3FF6C000 respectively.

\section{Features}
Each MCPWM peripheral has one clock divider (prescaler), three PWM timers, three PWM operators, and a capture module. Figure \ref{figure:EPM_overview} shows the submodules inside and the signals on the interface. PWM timers are used for generating timing references. The PWM operators generate desired waveform based on the timing references. Any PWM operator can be configured to use the timing references of any PWM timers. Different PWM operators can use the same PWM timer's timing references to produce related PWM signals. PWM operators can also use different PWM timers' values to produce the PWM signals that work alone. Different PWM timers can also be synced together.

    \centering
    \includegraphics[width=0.9\textwidth]{./29-MCPWM/figures/pwm_outside.eps}
    \caption{MCPWM Module Overview}
    \label{figure:EPM_overview}
\end{figure}

An overview of the submodules' function in Figure \ref{figure:EPM_overview} is shown below:

\begin{itemize}
    \item PWM Timers 0, 1 and 2
        \begin{itemize}
            \item Every PWM timer has a dedicated 8-bit clock prescaler.
            \item The 16-bit counter in the PWM timer can work in count-up mode, count-down mode or count-up-down mode.
            \item A hardware sync can trigger a reload on the PWM timer with a phase register. It will also trigger the prescaler' restart, so that the timer's clock can also be synced. The source of the sync can come from any GPIO or any other PWM timer's sync\_out.
        \end{itemize}
    \item PWM Operators 0, 1 and 2
        \begin{itemize}
            \item Every PWM operator has two PWM outputs: PWM\textcolor{red}{\it{x}}A and PWM\textcolor{red}{\it{x}}B. They can work independently, in symmetric and asymmetric configuration.
            \item Software, asynchronous override control of PWM signals.
            \item Configurable dead-time on rising and falling edges; each set up independently.
            \item All events can trigger CPU interrupts.
            \item Modulating of PWM output by high-frequency carrier signals, useful when gate drives are insulated with a transformer.
            \item Period, time stamps and important control registers have shadow registers with flexible updating methods.
        \end{itemize}
    \item Fault Detection Module
        \begin{itemize}
            \item Programmable fault handling allocated on fault condition in both cycle-by-cycle mode and one-shot mode.
            \item A fault condition can force the PWM output to either high or low logic levels.
        \end{itemize}
    \item Capture Module
        \begin{itemize}
            \item Speed measurement of rotating machinery
            \item Measurement of elapsed time between position sensor pulses
            \item Period and duty-cycle measurement of pulse train signals
            \item Decoding current or voltage amplitude derived from duty-cycle-encoded signals from current/voltage sensors
            \item Three individual capture channels, each of which has a time-stamp register (32 bits)
            \item Selection of edge polarity and prescaling of input capture signal
            \item The capture timer can sync with a PWM timer or external signals.
            \item Interrupt on each of the three capture channels
        \end{itemize}
\end{itemize}

\subsection{PWM Timer Submodule}
\label{section:timer_sub}
Each MCPWM module has three PWM timer submodules. Any of them can determine the necessary event timing for any of the three PWM operator submodules. Built-in synchronization logic allows multiple PWM timer submodules, in one or more MCPWM modules, to work together as a system, when using synchronization signals from the GPIO matrix.

\subsubsection{Configurations of the PWM Timer Submodule}

Users can configure the following functions of the PWM timer submodule:

\begin{itemize}
    \item Control how often events occur by specifying the PWM timer frequency or period.
    \item Configure a particular PWM timer to synchronize with other PWM timers or modules.
    \item Get a PWM timer in phase with other PWM timers or modules.
    \item Set one of the following timer counting modes: count-up, count-down, count-up-down.
    \item Change the rate of the PWM timer clock (PT\_clk) with a prescaler. Each timer has its own prescaler configured with PWM\_TIMER\textcolor{red}{\it{x}}\_PRESCALE of register \hyperref[regdesc:PWMTIMER0CFG0REG]{PWM\_TIMER0\_CFG0\_REG}. The PWM timer increments or decrements at a slower pace, depending on the setting of this register.
\end{itemize}

\subsubsection{PWM Timer's Working Modes and Timing Event Generation} \label{PWM Timer's Working Modes and Timing Event Generation}
The PWM timer has three working modes, selected by the PWM\textcolor{red}{\it{x}} timer mode register:
\begin{itemize}
    \item Count-Up Mode: \\
    In this mode, the PWM timer increments from zero until reaching the value configured in the period register. Once done, the PWM timer returns to zero and starts increasing again. PWM period is equal to the value of period register + 1.\\
    Note: The period register is PWM\_TIMER\textcolor{red}{\it{x}}\_PERIOD (\textcolor{red}{\it{x}} = 0, 1, 2), i.e., \hyperref[regdesc:PWMTIMER0PERIOD]{PWM\_TIMER0\_PERIOD}, \hyperref[regdesc:PWMTIMER1PERIOD]{PWM\_TIMER1\_PERIOD}, \hyperref[regdesc:PWMTIMER2PERIOD]{PWM\_TIMER2\_PERIOD}.
    \item Count-Down Mode: \\
    The PWM timer decrements to zero, starting from the value configured in the period register. After reaching zero, it is set back to the period value. Then it starts to decrement again. In this case, the PWM period is also equal to the value of period register + 1.
    \item Count-Up-Down Mode: \\
    This is a combination of the two modes mentioned above. The PWM timer starts increasing from zero until the period value is reached. Then, the timer decreases back to zero. This pattern is then repeated. The PWM period is the result of (the value of period register × 2 + 1).
\end{itemize}

Please note that in the Count-Up-Down Mode, when the counting direction is increasing, the timer range is [0, period value - 1], and when the counting direction is decreasing, the timer range is [period value, 1]. That is, in this mode, when synchronizing the timer to 0, decreasing counting direction will be illegal, namely, \hyperref[regdesc:PWMTIMER0SYNCREG]{MCPWM\_TIMER\textcolor{red}{\it{n}}\_PHASE\_DIRECTION} cannot be set to 1. Similarly, when synchronizing the timer to period value, increasing counting direction will be illegal, namely, \hyperref[regdesc:PWMTIMER0SYNCREG]{MCPWM\_TIMER\textcolor{red}{\it{n}}\_PHASE\_DIRECTION} cannot be set to 0. Therefore, when the timer is synchronized to 0, the counting direction can only be increasing, and \hyperref[regdesc:PWMTIMER0SYNCREG]{MCPWM\_TIMER\textcolor{red}{\it{n}}\_PHASE\_DIRECTION} will be 0. When the timer is synchronized to the period value, the counting direction can only be decreasing, and \hyperref[regdesc:PWMTIMER0SYNCREG]{MCPWM\_TIMER\textcolor{red}{\it{n}}\_PHASE\_DIRECTION} will be 1.



\subsubsection{PWM Timer Shadow Register}
\label{section:shdw_reg}
The PWM timer's period register and the PWM timer's clock prescaler register have shadow registers. The purpose of a shadow register is to save a copy of the value to be written into the active register at a specific moment synchronized with the hardware. Both register types are defined as follows:
\begin{itemize}
    \item Active Register \\
    This register is directly responsible for controlling all actions performed by hardware.
    \item Shadow Register \\
    It acts as a temporary buffer for a value to be written on the active register. Before this happens, the content of the shadow register has no direct effect on the controlled hardware. At a specific, user-configured point in time, the value saved in the shadow register is copied to the active register. This helps to prevent spurious operation of the hardware, which may happen when a register is asynchronously modified by software. Both the shadow register and the active register have the same memory address. The software always writes into, or reads from the shadow register. The moment of updating the active register is determined by its specific update method register. The update can start when the PWM timer is equal to zero, when the PWM timer is equal to period,at a synchronization moment, or immediately. Software can trigger a globally forced update which will prompt all registers in the module to be updated according to shadow registers.
\end{itemize}

\subsubsection{PWM Timer Synchronization and Phase Locking}
The PWM modules adopt a flexible synchronization method. Each PWM timer has a synchronization input and a synchronization output. The synchronization input can be selected from three synchronization outputs and three synchronization signals from the GPIO matrix. The synchronization output can be generated from the synchronization input signal, or when the PWM timer's value is equal to period or zero. Thus, the PWM timers can be chained together with their phase locked. During synchronization, the PWM timer clock prescaler will reset its counter in order to synchronize the PWM timer clock.

\subsubsection{PWM Generator Submodule}
\label{gen_submodule}
\subparagraph{Purpose of the PWM Generator Submodule}
In this submodule, important timing events are generated or imported. The events are then converted into specific actions to generate the desired waveforms at the PWM\textcolor{red}{\it{x}}A and PWM\textcolor{red}{\it{x}}B outputs.

The PWM generator submodule performs the following actions:
\begin{itemize}
    \item Generation of timing events based on time stamps configured using the A and B registers. Events happen when the following conditions are satisfied:
    \begin{itemize}
        \item UTEA: the PWM timer is counting up and its value is equal to register A.
        \item UTEB: the PWM timer is counting up and its value is equal to register B.
        \item DTEA: the PWM timer is counting down and its value is equal to register A.
        \item DTEB: the PWM timer is counting down and its value is equal to register B.
    \end{itemize}
    \item Generation of U/DT1, U/DT2 timing events based on fault or synchronization events.
    \item Management of priority when these timing events occur concurrently.
    \item Qualification and generation of set, clear and toggle actions, based on the timing events.
    \item Controlling of the PWM duty cycle, depending on configuration of the PWM generator submodule.
    \item Handling of new time stamp values, using shadow, registers to prevent glitches in the PWM cycle.
\end{itemize}

\subparagraph{PWM Operator Shadow Registers}
The time stamp registers A and B, as well as action configuration registers  \hyperref[regdesc:PWMGEN0AREG]{PWM\_GEN\textcolor{red}{\it{x}}\_A\_REG} and \hyperref[regdesc:PWMGEN0BREG]{PWM\_GEN\textcolor{red}{\it{x}}\_B\_REG} are shadowed. Shadowing provides a way of updating registers in sync with the hardware. For a description of the shadow registers, please see \ref{section:shdw_reg}.

\subparagraph{PWM Signal Generation}
The PWM generator submodule controls the behavior of outputs PWM\textcolor{red}{\it{x}}A and PWM\textcolor{red}{\it{x}}B when a particular timing event occurs. The timing events are further qualified by the PWM timer's counting direction (up or down). Knowing the counting direction, the submodule may then perform an independent action at each stage of the PWM timer counting up or down.

The following actions may be configured on outputs PWM\textcolor{red}{\it{x}}A and PWM\textcolor{red}{\it{x}}B:
\begin{itemize}
    \item Set High:\\
    Set the output of PWM\textcolor{red}{\it{x}}A or PWM\textcolor{red}{\it{x}}B to a high level.
    \item Clear Low:\\
    Clear the output of PWM\textcolor{red}{\it{x}}A or PWM\textcolor{red}{\it{x}}B by setting it to a low level.
    \item Toggle:\\
    Change the current output level of PWM\textcolor{red}{\it{x}}A or PWM\textcolor{red}{\it{x}}B to the opposite value. If it is currently pulled high, pull it low, or vice versa.
    \item Do Nothing:\\
    Keep both outputs PWM\textcolor{red}{\it{x}}A and PWM\textcolor{red}{\it{x}}B unchanged. In this state, interrupts can still be triggered.
\end{itemize}

The configuration of actions on outputs is done by using registers \hyperref[regdesc:PWMGEN0AREG]{PWN\_GEN\textcolor{red}{\it{x}}\_A\_REG} and \hyperref[regdesc:PWMGEN0BREG]{PWN\_GEN\textcolor{red}{\it{x}}\_B\_REG}. So, the action to be taken on each output is set independently. Also there is great flexibility in selecting actions to be taken on a given output based on events. More specifically, any event listed in Table \ref{table:gen_in} can operate on either output PWM\textcolor{red}{\it{x}}A or PWM\textcolor{red}{\it{x}}B. To check out registers for particular generator 0, 1 or 2, please refer to register description in Section \ref{section:register_summary}.

\subparagraph{Software-Force Events}
There are two types of software-force events inside the PWM generator:
\begin{itemize}
    \item Non-continuous-immediate (NCI) software-force events\\
    Such types of events are immediately effective on PWM outputs when triggered by software. The forcing is non-continuous, meaning the next active timing events will be able to alter the PWM outputs.
    \item Continuous (CNTU) software-force events\\
    Such types of events are continuous. The forced PWM outputs will continue until they are released by software. The events' triggers are configurable. They can be timing events or immediate events.
\end{itemize}

\subsubsection{Dead Time Generator Submodule}
\label{dead_time_submodule}
\subparagraph{Purpose of the Dead Time Generator Submodule}
Several options to generate signals on PWM\textcolor{red}{\it{x}}A and PWM\textcolor{red}{\it{x}}B outputs, with a specific placement of signal edges, have been discussed in section \ref{gen_submodule}. The required dead time is obtained by altering the edge placement between signals and by setting the signal's duty cycle. Another option is to control the dead time using a specialized submodule – the Dead Time Generator.

The key functions of the dead time generator submodule are as follows:
\begin{itemize}
    \item Generating signal pairs (PWM\textcolor{red}{\it{x}}A and PWM\textcolor{red}{\it{x}}B) with a dead time from a single PWM\textcolor{red}{\it{x}}A input
    \item Creating a dead time by adding delay to signal edges:
    \begin{itemize}
        \item Rising edge delay (RED)
        \item Falling edge delay (FED)
    \end{itemize}
    \item Configuring the signal pairs to be:
    \begin{itemize}
        \item Active high complementary (AHC)
        \item Active low complementary (ALC)
        \item Active high (AH)
        \item Active low (AL)
    \end{itemize}
    \item This submodule may also be bypassed, if the dead time is configured directly in the generator submodule.
\end{itemize}

\subparagraph{Dead Time Generator's Shadow Registers}
Delay registers RED and FED are shadowed with registers \hyperref[regdesc:PWMDT0REDCFGREG]{PWM\_DT\textcolor{red}{\it{x}}\_RED\_CFG\_REG} and \hyperref[regdesc:PWMDT0FEDCFGREG]{PWM\_DT\textcolor{red}{\it{x}}\_FED\_CFG\_REG}. For the description of shadow registers, please see section \ref{section:shdw_reg}.

\subsubsection{PWM Carrier Submodule}
The coupling of PWM output to a motor driver may need isolation with a transformer. Transformers deliver only AC signals, while the duty cycle of a PWM signal may range anywhere from 0\% to 100\%. The PWM carrier submodule passes such a PWM signal through a transformer by using a high frequency carrier to modulate the signal.

\subparagraph{Function Overview}
The following key characteristics of this submodule are configurable:
\begin{itemize}
    \item Carrier frequency
    \item Pulse width of the first pulse
    \item Duty cycle of the second and the subsequent pulses
    \item Enabling/disabling the carrier function
\end{itemize}

\subparagraph{Operational Highlights}
The PWM carrier clock (PC\_clk) is derived from PWM\_clk. The frequency and duty cycle are configured by the PWM\_CARRIER\textcolor{red}{\it{x}}\_PRESCALE and PWM\_CARRIER\textcolor{red}{\it{x}}\_DUTY bits in the \hyperref[regdesc:PWMCARRIER0CFGREG]{PWM\_CARRIER\textcolor{red}{\it{x}}\_CFG\_REG} register. The purpose of one-shot pulses is to provide high-energy impulse to reliably turn on the power switch. Subsequent pulses sustain the power-on status. The width of a one-shot pulse is configurable with the PWM\_CARRIER\textcolor{red}{\it{x}}\_OSHTWTH bits. Enabling/disabling of the carrier submodule is done with the PWM\_CARRIER\textcolor{red}{\it{x}}\_EN bit.

Figure \ref{figure:carrier_0} shows an example of waveforms, where a carrier is superimposed on original PWM pulses. This figure do not show the first one-shot pulse and the duty-cycle control. Related details are covered in the following two sections.

The width of the first pulse is configurable. It may assume one of 16 possible values and is described by the formula below:

\begin{center}
$T_{1stpulse} = T_{PWM\_clk} \times 8 \times (PWM\_CARRIER\textcolor{red}{\it{x}}\_PRESCALE+1) \times (PWM\_CARRIER\textcolor{red}{\it{x}}\_OSHTWTH+1)$ \\
\end{center}

Where:
\begin{itemize}
    \item $T_{PMW\_clk}$ is the period of the PWM clock (PWM\_clk).
    \item $(PWM\_CARRIER\textcolor{red}{\it{x}}\_OSHTWTH+1)$ is the width of the first pulse (whose value ranges from 1 to 16).
    \item $(PWM\_CARRIER\textcolor{red}{\it{x}}\_PRESCALE+1)$ is the PWM carrier clock's (PC\_clk) prescaler value.
\end{itemize}

\subparagraph{Duty Cycle Control}
After issuing the first one-shot pulse, the remaining PWM signal is modulated according to the carrier frequency. Users can configure the duty cycle of this signal. Tuning of duty may be required, so that the signal passes through the isolating transformer and can still operate (turn on/off) the motor drive, changing rotation speed and direction.

The duty cycle may be set to one of seven values, using  PWM\_CARRIER\textcolor{red}{\it{x}}\_DUTY, or bits [7:5] of register\\ \hyperref[regdesc:PWMCARRIER0CFGREG]{PWM\_CARRIER\textcolor{red}{\it{x}}\_CFG\_REG}.

Below is the formula for calculating the duty cycle:

\begin{center}
$Duty =  PWM\_CARRIER\textcolor{red}{\it{x}}\_DUTY \div 8$
\end{center}

\subsubsection{Fault Handler Submodule}
Each MCPWM peripheral is connected to three fault signals (FAULT0, FAULT1 and FAULT2) which are sourced from the GPIO matrix. These signals are intended to indicate external fault conditions, and may be preprocessed by the fault detection submodule to generate fault events. Fault events can then execute the user code to control MCPWM outputs in response to specific faults.

\subparagraph{Function of Fault Handler Submodule}
The key actions performed by the fault handler submodule are:
\begin{itemize}
    \item Forcing outputs PWM\textcolor{red}{\it{x}}A and PWM\textcolor{red}{\it{x}}B, upon detected fault, to one of the following states:
    \begin{itemize}
        \item High
        \item Low
        \item Toggle
        \item No action taken
    \end{itemize}
    \item Execution of one-shot trip (OST) upon detection of over-current conditions/short circuits.
    \item Cycle-by-cycle tripping (CBC) to provide current-limiting operation.
    \item Allocation of either one-shot or cycle-by-cycle operation for each fault signal.
    \item Generation of interrupts for each fault input.
    \item Support for software-force tripping.
    \item Enabling or disabling of submodule function as required.
\end{itemize}

\subparagraph{Operation and Configuration Tips}
This section provides the operational tips and set-up options for the fault handler submodule.

Fault signals coming from pads are sampled and synced in the GPIO matrix. In order to guarantee the successful sampling of fault pulses, each pulse duration must be at least two APB clock cycles. The fault detection submodule will then sample fault signals by using PWM\_clk. So, the duration of fault pulses coming from GPIO matrix must be at least one PWM\_clk cycle. Differently put, regardless of the period relation between APB clock and PWM\_clk, the width of fault signal pulses on pads must be at least equal to the sum of two APB clock cycles and one PWM\_clk cycle.

Each level of fault signals, FAULT0 to FAULT2, can be used by the fault handler submodule to generate fault events (fault\_event0 to fault\_event2). Every fault event can be configured individually to provide CBC action, OST action, or none.

\begin{itemize}
    \item {\bfseries Cycle-by-Cycle (CBC) action:}\\
    When CBC action is triggered, the state of PWM\textcolor{red}{\it{x}}A and PWM\textcolor{red}{\it{x}}B will be changed immediately according to the configuration of registers \hyperref[regdesc:PWMFH0CFG0REG]{PWM\_FH\textcolor{red}{\it{x}}\_A\_CBC\_U/D} and \hyperref[regdesc:PWMFH0CFG0REG]{PWM\_FH\textcolor{red}{\it{x}}\_B\_CBC\_U/D}. Different actions can be indicted when the PWM timer is incrementing or decrementing. Different CBC action interrupts can be triggered for different fault events. Status register \hyperref[regdesc:PWMFH0STATUSREG]{PWM\_FH\textcolor{red}{\it{x}}\_CBC\_ON} indicates whether a CBC action is on or off.
   When the fault event is no longer present, CBC actions on PWM\textcolor{red}{\it{x}}A/B will be cleared at a specified point, which is either a D/UTEP or D/UTEZ event. Register \hyperref[regdesc:PWMFH0CFG1REG]{PWM\_FH\textcolor{red}{\it{x}}\_CBCPULSE} determines at which event PWM\textcolor{red}{\it{x}}A and PWM\textcolor{red}{\it{x}}B will be able to resume normal actions. Therefore, in this mode, the CBC action is cleared or refreshed upon every PWM cycle.
    \item {\bfseries One-Shot (OST) action:}\\
	When OST action is triggered, the state of PWM\textcolor{red}{\it{x}}A and PWM\textcolor{red}{\it{x}}B will be changed immediately, depending on the setting of registers \hyperref[regdesc:PWMFH0CFG0REG]{PWM\_FH\textcolor{red}{\it{x}}\_A\_OST\_U/D} and \hyperref[regdesc:PWMFH0CFG0REG]{PWM\_FH\textcolor{red}{\it{x}}\_B\_OST\_U/D}. Different actions can be configured when PWM timer is incrementing or decrementing. Different OST action interrupts can be triggered form different fault events. Status register \hyperref[regdesc:PWMFH0STATUSREG]{PWM\_FH\textcolor{red}{\it{x}}\_OST\_ON} indicates whether an OST action is on or off.
    The OST actions on PWM\textcolor{red}{\it{x}}A/B are not automatically cleared when the fault event is no longer present. One-shot actions must be cleared manually by negating the value stored in register \hyperref[regdesc:PWMFH0CFG0REG]{PWM\_FH\textcolor{red}{\it{x}}\_CLR\_OST}.
\end{itemize}

\subsection{Capture Submodule}
\subsubsection{Introduction}
The capture submodule contains three complete capture channels. Channel inputs CAP0, CAP1 and CAP2 are sourced from the GPIO matrix. Thanks to the flexibility of the GPIO matrix, CAP0, CAP1 and CAP2 can be configured from any PAD input. Multiple capture channels can be sourced from the same PAD input, while prescaling for each channel can be set differently. Also, capture channels are sourced from different PADs. This provides several options for handling capture signals by hardware in the background, instead of having them processed directly by the CPU.
A capture submodule has the following independent key resources:
\begin{itemize}
    \item One 32-bit timer (counter) which can be synchronized with the PWM timer, another submodule or software.
    \item Three capture channels, each equipped with a 32-bit time-stamp and a capture prescaler.
    \item Independent edge polarity (rising/falling edge) selection for any capture channel.
    \item Input capture signal prescaling (from 1 to 256).
    \item Interrupt capabilities on any of the three capture events.
\end{itemize}

\subsubsection{Capture Timer}
The capture timer is a 32-bit counter incrementing continuously, once enabled. On the input it has an APB clock running typically at 80 MHz. At a sync event the counter is loaded with phase stored in register\\\hyperref[regdesc:PWMCAPTIMERPHASEREG]{PWM\_CAP\_TIMER\_PHASE\_REG}. Sync events can come from PWM timers sync-out, PWM module sync-in or software. The capture timer provides timing references for all three capture channels.

\subsubsection{Capture Channel}
The capture signal coming to a capture channel will be inverted first, if needed, and then prescaled. Finally, specified edges of preprocessed capture signal will trigger capture events. When a capture event occurs, the capture timer's value is stored in time-stamp register \hyperref[regdesc:PWMCAPCH0REG]{PWM\_CAP\_CH\textcolor{red}{\it{x}}\_REG}. Different interrupts can be generated for different capture channels at capture events. The edge that triggers a capture event is recorded in register \hyperref[regdesc:PWMCAPSTATUSREG]{PWM\_CAP\textcolor{red}{\it{x}}\_EDGE}. The capture event can be also forced by software.
