\chapter{Interrupt Matrix (INTERRUPT)}
\label{mod:intmtrx}
\section{Overview}

The Interrupt Matrix embedded in the ESP32 independently allocates peripheral interrupt sources to the two CPUs' peripheral interrupts. This configuration is made to be highly flexible in order to meet many different needs.
\vspace{-2em}

\section{Features}

\begin{itemize}
    \item Accepts 71 peripheral interrupt sources as input.
    \item Generates 26 peripheral interrupt sources per CPU as output (52 total).
    \item CPU NMI Interrupt Mask.
    \item Queries current interrupt status of peripheral interrupt sources.
\end{itemize}

The structure of the Interrupt Matrix is shown in Figure \ref{figure:InterruptMatrixStructure}.

\section{Functional Description}

\subsection{Peripheral Interrupt Source}

ESP32 has 71 peripheral interrupt sources in total. All peripheral interrupt sources are listed in table \ref{table:PeripheralInterruptConfigStatusSource}.
67 of 71 ESP32 peripheral interrupt sources can be allocated to either CPU.

The four remaining peripheral interrupt sources are CPU-specific, two per CPU. GPIO\_INTERRUPT\_PRO and GPIO\_INTERRUPT\_PRO\_NMI can only be allocated to PRO\_CPU. GPIO\_INTERRUPT\_APP and GPIO\_INTERRUPT\_APP\_NMI can only be allocated to APP\_CPU.
As a result, PRO\_CPU and APP\_CPU each have 69 peripheral interrupt sources.

\subsection{Allocate Peripheral Interrupt Sources to Peripheral Interrupt on CPU}

In this section:

\begin{itemize}

\item Source\_X stands for any particular peripheral interrupt source.

\item PRO\_X\_MAP\_REG (or APP\_X\_MAP\_REG) stands for any particular peripheral interrupt configuration register of the PRO\_CPU (or APP\_CPU). The peripheral interrupt configuration register corresponds to the peripheral interrupt source Source\_X. In Table \ref{table:PeripheralInterruptConfigStatusSource} the registers listed under “PRO\_CPU (APP\_CPU) - Peripheral Interrupt Configuration Register” correspond to the peripheral interrupt sources listed in “Peripheral Interrupt Source - Name”.

\item Interrupt\_P stands for CPU peripheral interrupt, numbered as Num\_P. Num\_P can take the ranges 0 \verb+~+ 5, 8 \verb+~+ 10, 12 \verb+~+ 14, 17 \verb+~+ 28, 30 \verb+~+ 31.

\item Interrupt\_I stands for the CPU internal interrupt numbered as Num\_I. Num\_I can take values 6, 7, 11, 15, 16, 29.

\end{itemize}

Using this terminology, the possible operations of the Interrupt Matrix controller can be described as follows:

\begin{itemize}

\item \textbf{Allocate peripheral interrupt source Source\_X to CPU (PRO\_CPU or APP\_CPU)}\\
Set PRO\_X\_MAP\_REG (or APP\_X\_MAP\_REG) to Num\_P. Num\_P can be any CPU peripheral interrupt number. CPU interrupts can be shared between multiple peripherals (see below).

\item \textbf{Disable peripheral interrupt source Source\_X for CPU (PRO\_CPU or APP\_CPU)}\\
Set PRO\_X\_MAP\_REG (or APP\_X \_MAP\_REG) for peripheral interrupt source to any Num\_I. The specific choice of internal interrupt number does not change behaviour, as none of the interrupt numbered as Num\_I is connected to either CPU.

\item \textbf{Allocate multiple peripheral sources Source\_X\textcolor{red}{\it{n}} ORed to PRO\_CPU (APP\_CPU) peripheral interrupt}\\
Set multiple PRO\_X\textcolor{red}{\it{n}}\_MAP\_REG (APP\_X\textcolor{red}{\it{n}}\_MAP\_REG) to the same Num\_P. Any of these peripheral interrupts will trigger CPU Interrupt\_P.

\end{itemize}

\subsection{CPU NMI Interrupt Mask}

The Interrupt Matrix temporarily masks all peripheral interrupt sources allocated to PRO\_CPU's ( or APP\_CPU's ) NMI interrupt, if it receives the signal PRO\_CPU NMI Interrupt Mask ( or APP\_CPU NMI Interrupt Mask ) from the peripheral PID Controller, respectively.

\subsection{Query Current Interrupt Status of Peripheral Interrupt Source}

The current interrupt status of a peripheral interrupt source can be read via the bit value in PRO\_INTR\_STATUS\_REG\_\textcolor{red}{\it{n}} (APP\_INTR\_STATUS\_REG\_\textcolor{red}{\it{n}}), as shown in the mapping in Table \ref{table:PeripheralInterruptConfigStatusSource}.

\section{Registers}
The interrupt matrix registers are part of the DPORT registers and are described in Section \ref{sec:dport-reg-summ} in Chapter \ref{mod:dportreg} \textit{\nameref{mod:dportreg}}.
