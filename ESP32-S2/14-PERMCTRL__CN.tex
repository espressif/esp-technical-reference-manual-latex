\hypertarget{permctrl}{}
\chapter{权限控制 (PMS)}
\label{mod:permctrl}


\section{概述}
ESP32-S2 中不同总线类型、模块对存储器发起的访问权限是不同的。权限控制模块的功能就是通过相应的权限控制寄存器对上述所有的访问权限进行集中管理。

\section{主要特性}

\begin{itemize}
    \item 内部存储器权限管理
    \begin{itemize}
        \item SRAM 部分采用统一管理，部分采用分割管理
        \item RTC FAST SRAM 采用分割管理
        \item RTC SLOW SRAM 采用分割管理
    \end{itemize}
    \item 片外存储器由 MMU 及权限控制模块管理
\end{itemize}

\section{功能描述}

\subsection{内部存储器权限管理}
内部存储器资源包括：ROM，SRAM，RTC FAST，RTC SLOW。各存储器的组成结构如下：
\begin{itemize}
    \item ROM 总大小为 128 KB，由两个 64 KB 的块组成。
    \item SRAM 总大小为 320 KB，分成 4 个 8 KB 和 18 个 16 KB 的块，编号 Block 0 \verb+~+ 21，各 Block 的偏移地址范围如表 \ref{table:SRAM_block_address} 所示，不同总线访问的基地址不同，具体请见章节 \ref{IBUSSRAM}、\ref{DUBS0PermissionControl}、\ref{DMAPermissionControl}。对于 4 个 8 KB 的块 Block 0 \verb+~+ 3，每一个块都配备有独立的权限控制寄存器；而 16 KB 的块 Block 4 \verb+~+ 21 被视作一个大的存储空间，对该存储空间采用\textbf{分割管理}（将存储空间分割成高、低两片区域分别实施权限管理），分割地址不可落在 Block 4 \verb+~+ 5 的地址范围内。
    \item RTC FAST Memory 采用分割管理，高、低两片区域备有独立的权限管理寄存器。
    \item RTC SLOW Memory 采用分割管理，高、低两片区域备有独立的权限管理寄存器。
\end{itemize}

下文分别叙述 ESP32-S2 中能够访问上述内部存储资源的总线类型和模块。

\subsubsection{IBUS 指令总线的访问权限管理}\label{Permission Control for the Instruction Bus (IBUS)}
具备权限管理的 IBUS 指令总线的地址范围为 \hexprefix{}4002\_0000 \verb+~+ \hexprefix{}4007\_1FFF，该地址段包含有 SRAM 存储单元、RTC FAST Memory。访问不同存储单元都有独立的权限控制寄存器，由软件配置完成。

此外本章节中与 IBUS 总线相关的权限控制寄存器还被 \hyperref[fielddesc:PMSPROIRAM0LOCK]{PMS\_PRO\_IRAM0\_LOCK} 信号控制，当该信号配置为 1 时，权限寄存器中配置数值将被锁定，不允许再次更改。同时 \hyperref[fielddesc:PMSPROIRAM0LOCK]{PMS\_PRO\_IRAM0\_LOCK} 信号数值也将保持在 1，不允许更改。当且仅当 CPU 被复位时，\hyperref[fielddesc:PMSPROIRAM0LOCK]{PMS\_PRO\_IRAM0\_LOCK} 信号数值才被重新置 0。

\begin{tiplisting}
\textcolor{red}{*} 分割地址为 32 位宽，其值为基地址+用户配置的域的值×4。比如，将 \hexprefix{}10000 写入 \hyperref[regdesc:PMSPROIRAM02REG]{PMS\_PRO\_IRAM0\_2\_REG} [16:0]，则 IBUS 总线在 SRAM Block 4 \verb+~+ 21 地址范围内的分割地址为 \hexprefix{}4004\_0000。注意 IBUS 和 DBUS0 的分割地址不能落在 SRAM Block 0 \verb+~+ 5 的有效地址范围内。
\end{tiplisting}

若 CPU 通过 IBUS 总线对 SRAM 发起的访问类型与配置允许的访问类型不一致，例如软件配置 IBUS 总线只允许对 SRAM Block5 进行读写访问，但 CPU 通过 IBUS 总线对该 Block 发起了取指访问，权限控制模块将直接拒绝此次访问。

拒绝此次访问，但是 CPU 发起的访问不会被阻塞。具体表现如下：
\begin{itemize}
    \item 写操作，将不会生效，不会改变存储单元内的数据内容；
    \item 读和取指令操作，得到的是无意义的数据。
\end{itemize}

如果 IBUS 总线的违法访问 SRAM 存储单元的中断被使能，权限控制模块还将记录当次访问地址和访问类型，同时给出中断信号。如果连续发生访问不通过情况，硬件只记录第一次出错的信息。访问错误中断为电平信号，需要软件清除。清除访问错误中断标志的同时，相关错误记录也会同时被清除。

\subsubsection{DBUS0 总线的访问权限管理}
CPU 通过 DBUS0 数据总线可以访问的地址范围有两段：\hexprefix{}3FFB\_0000 \verb+~+ \hexprefix{}3FFF\_FFFF 和 \hexprefix{}3FF9\_E000 \verb+~+ \hexprefix{}3FF9\_FFFF，包含 SRAM 存储单元、RTC FAST Memory。访问不同存储单元都有独立的权限控制寄存器，由软件配置完成。

\subsubsection{其它内部存储器权限管理}

\subsubsection{外部存储器权限控制}\label{External Memory Permission Controls}
硬件将外部存储器 (flash + SRAM) 的实地址以 64 KB 为单位分为 8 (4+4) 个区域。每个区域可配置 W/R/X 访问。软件需要预先设定各区域的大小及其访问属性。

8 个区域分别对应 8 组寄存器，flash 和 SRAM 各有 4 组。每组寄存器包含有 3 部分:
\begin{enumerate}
    \item 属性列表: SYSCON\_${X}$\_ACE\_${n}$\_ATTR\_REG 共 3 位， 从高到低分别为W/R/F；
    \item 区域起始地址: SYSCON\_${X}$\_ACE\_${n}$\_ADDR\_REG， 表示实际地址；
    \item 区域长度: SYSCON\_${X}$\_ACE\_${n}$\_SIZE\_REG， 以 64 KB 为单位。
\end{enumerate}
其中“${X}$”表示 flash 或 SRAM，“${n}$”表示 0 \verb+~+ 3。软件配置时需要注意的是 flash 对应的 4 个区域的 总大小必须为 1 GB 且 SRAM 对应的 4 个区域总大小也必须为 1 GB。

CPU 直接访问外部存储器时，权限控制模块只监控 CPU 的写请求，不监控读请求。控制模块会根据访问的物理地址检查 CPU 的访问属性和预先设定的访问属性是否一致。如果事先设定该区域不可写，那么控制模块直接拒绝此次写访问，并记录当前访问的相关信息，并触发访问错误中断。

Cache 缺失、cache 写回和 cache 预取等操作会触发 cache 对外部存储器发起 W/R/X 请求。此时权限控制模块会根据访问的物理地址查询其所在区域的访问属性，并比较 cache 请求和预先设定的访问属性是否一致。只有 cache 请求和设定的访问属性一致时，权限控制模块才会将请求传递到外部存储器。若 cache 发起的是 R/F 请求，控制模块还要将该区域对应的访问属性反馈给 cache， 同时 cache 会将访问属性存起来。当访问请求与当前访问属性不一致时，硬件会记录下当前访问的相关信息，并触发访问错误中断。

CPU 访问 cache 时，权限控制模块均要检查 CPU 的访问属性，检查方法是比较 CPU 的访问属性和存储在本地的访问属性列表，只有两者一致时 CPU 的访问才会成功。当访问请求与当前访问属性不一致时， 硬件会记录下当前访问的相关信息，并触发访问错误中断。

如果连续发生访问属性检查不通过情况，那么硬件只记录第一次出错的信息，后面错误信息将被直接丢弃。另外访问错误中断为电平信号，软件清除访问错误中断标志的同时，错误记录也会同时被清除。
