\chapter{Remote Control Peripheral (RMT)}
\label{mod:rmt}


\section{Introduction}
The RMT (Remote Control) module is designed to send/receive infrared remote control signals that support for a variety of remote control protocols. The RMT module converts pulse codes stored in the module's built-in RAM into output signals, or converts input signals into pulse codes and stores them back in RAM. In addition, the RMT module optionally modulates its output signals with a carrier wave, or optionally filters its input signals.


\section{Functional Description}
\subsection{RMT Architecture}

The RMT module has four channels, numbered from zero to three. Each channel has the same functionality controlled by a dedicated set of registers and is able to transmit or receive data independently. By default, channel 0 transmits data from or receives data to block 0, channel 1 transmit data from or receive data to block 1, and so on, as shown in the figures below. Registers in each channel are indicated by \regindex{n} which is used as a placeholder for the channel number.

Each channel has a clock divider, counter, transmitter, and receiver. In each channel, only its transmitter or receiver can be active at the same time. The four channels share a 256 x 32-bit RAM.

\subsection{RMT RAM}
\begin{figure}[htb! ]
\includegraphics[width=1.0\textwidth]{\modulefiles/figures/rmt_data_structure.eps}
\centering
\caption{Format of Pulse Code in RAM} \label{Figure:RMT2-2}
\end{figure}
The format of pulse code in RAM is shown in Figure \ref{Figure:RMT2-2}. Each pulse code contains a 16-bit entry with two fields, level and period. Level (0 or 1) indicates a high-/low-level value was received or is going to be sent, while "period" points out the clock cycles (see Figure \ref{Figure:RMT2-1} clk\_div) for which the level lasts. A zero period is interpreted as a transmission end-marker.

The RAM is divided into four 64 x 32-bit blocks. By default, each channel uses one block (block zero for channel zero, block one for channel one, and so on). Usually, only one block of 64 x 32-bit worth of data can be sent or received in channel \regindex{n}. If the data size is larger than this block size, users can configure the channel to enable wrap mode or to use more blocks by setting \hyperref[fielddesc:RMTREGMEMSIZECHN]{RMT\_MEM\_SIZE\_CH\regindex{n}}. Setting \hyperref[fielddesc:RMTREGMEMSIZECHN]{RMT\_MEM\_SIZE\_CH\regindex{n}} > 1 will prompt channel \regindex{n} to use the memory of subsequent channels, block (\regindex{n}) \verb+~+ block (\regindex{n} + \hyperref[fielddesc:RMTREGMEMSIZECHN]{RMT\_MEM\_SIZE\_CH\regindex{n}} -1). If so, the subsequent channels \regindex{n} +1 \verb+~+ \regindex{n} + \hyperref[fielddesc:RMTREGMEMSIZECHN]{RMT\_MEM\_SIZE\_CH\regindex{n}} - 1 cannot be used once their RAM blocks are occupied. Note that the RAM used by each channel is mapped from low address to high address. In such mode, channel 0 is able to use the RAM blocks for channels 1, 2 and 3 by setting \hyperref[fielddesc:RMTREGMEMSIZECHN]{RMT\_MEM\_SIZE\_CH\regindex{n}}, but channel 3 cannot use the blocks for channels 0, 1, or 2.

The RMT RAM can be accessed via APB bus, or read by the transmitter and written by the receiver. To protect a receiver from overwriting the blocks a transmitter is about to transmit, \hyperref[fielddesc:RMTREGMEMOWNERCHN]{RMT\_MEM\_OWNER\_CH\regindex{n}} can be configured to designate the block's owner, be it a transmitter or receiver. This way, if this ownership is violated, an \hyperref[fielddesc:RMTREGMEMOWNERERRCHN]{RMT\_MEM\_OWNER\_ERR\_CH\regindex{n}} flag will be generated.

When the RMT module is inactive, the RAM can be put into low-power mode by setting \hyperref[fielddesc:RMTREGMEMFORCEPD]{RMT\_MEM\_FORCE\_PD}.

\subsection{Clock}
The drive clock of a divider is generated by taking either the APB\_CLK or REF\_TICK according to the state of \hyperref[fielddesc:RMTREGREFALWAYSONCHN]{RMT\_REF\_ALWAYS\_ON\_CH\regindex{n}}. (For more information on clock sources, please see Chapter \hyperref[mod:resclk]{Reset and Clock}). Divider value is normally equal to the value of \hyperref[fielddesc:RMTREGDIVCNTCHN]{RMT\_DIV\_CNT\_CH\regindex{n}}, except value 0 that represents divider 256. The clock divider can be reset to zero by clearing \hyperref[fielddesc:RMTREGREFCNTRSTCHN]{RMT\_REF\_CNT\_RST\_CH\regindex{n}}. The clock generated from the divider can be used by the clock counter (see Figure \ref{Figure:RMT2-2}).

\subsection{Transmitter}
When \hyperref[fielddesc:RMTREGTXSTARTCHN]{RMT\_TX\_START\_CH\regindex{n}} is set to 1, the transmitter of channel \regindex{n} will start reading and sending pulse codes from the starting address of its RAM block. The codes are sent starting from low-address entry. The transmitter will stop the transmission, return to idle state and generate an \hyperref[int:RMTCHnTXENDINT]{RMT\_CH\regindex{n}\_TX\_END\_INT} interrupt, when an end-marker (a zero period) is encountered. Also, setting \hyperref[fielddesc:RMTREGTXSTOPCHN]{RMT\_TX\_STOP\_CH\regindex{n}} to 1 stops the transmission and immediately sets the transmitter back to idle. The output level of a transmitter in idle state is determined by the "level" field of the end-marker or by the content of \hyperref[fielddesc:RMTREGIDLEOUTLVCHN]{RMT\_IDLE\_OUT\_LV\_CH\regindex{n}}, depending on the configuration of \hyperref[fielddesc:RMTREGIDLEOUTENCHN]{RMT\_IDLE\_OUT\_EN\_CH\regindex{n}}.

To transmit more pulse codes than can be fitted in the channel's RAM, users can enable wrap mode by configuring \hyperref[fielddesc:RMTREGMEMTXWRAPEN]{RMT\_MEM\_TX\_WRAP\_EN}. In this mode, when the transmitter has reached the end-marker in the channel's memory, it will loop back to the first byte. For example, if \hyperref[fielddesc:RMTREGMEMSIZECHN]{RMT\_MEM\_SIZE\_CH\regindex{n}} is set to 1, the transmitter will start sending data from the address 64 * \regindex{n}, and then the data from higher RAM address. Once the transmitter finishes sending the data from (64 * (\regindex{n} +1) - 1), it will continue sending data from 64 * \regindex{n} till encounters an end-marker. Wrap mode is also applicable for \hyperref[fielddesc:RMTREGMEMSIZECHN]{RMT\_MEM\_SIZE\_CH\regindex{n}} > 1.

An \hyperref[int:RMTCHnTXTHREVENTINT]{RMT\_CH\regindex{n}\_TX\_THR\_EVENT\_INT} interrupt will be generated whenever the size of transmitted pulse codes is larger than or equal to the value set by \hyperref[fielddesc:RMTREGTXLIMCHN]{RMT\_TX\_LIM\_CH\regindex{n}}. In wrap mode, \hyperref[fielddesc:RMTREGTXLIMCHN]{RMT\_TX\_LIM\_CH\regindex{n}} can be set to a half or a fraction of the size of the channel's RAM block. When an \hyperref[int:RMTCHnTXTHREVENTINT]{RMT\_CH\regindex{n}\_TX\_THR\_EVENT\_INT} interrupt is detected, the already used RAM region should be updated by subsequent user defined events. Therefore, when the wrap mode happens the transmitter will seamlessly continue sending the new events.

The output of the transmitter can be modulated using a carrier wave by setting \hyperref[fielddesc:RMTREGCARRIERENCHN]{RMT\_CARRIER\_EN\_CH\regindex{n}}. The carrier waveform is configurable. In a carrier cycle, the high level lasts for (\hyperref[fielddesc:RMTREGCARRIERHIGHCHN]{RMT\_CARRIER\_HIGH\_CH\regindex{n}} +1) clock cycles of APB\_CLK or REF\_TICK, while the low level lasts for (\hyperref[fielddesc:RMTREGCARRIERLOWCHN]{RMT\_CARRIER\_LOW\_CH\regindex{n}} +1) clock cycles of APB\_CLK or REF\_TICK. When \hyperref[fielddesc:RMTREGCARRIEROUTLVCHN]{RMT\_CARRIER\_OUT\_LV\_CH\regindex{n}} is set to 1, carrier wave will be added on high-level output signals; while \hyperref[fielddesc:RMTREGCARRIEROUTLVCHN]{RMT\_CARRIER\_OUT\_LV\_CH\regindex{n}} is set to 0, carrier wave will be added on low-level output signals. Carrier wave can be added on output signals during modulation, or just added on valid pulse codes (the data stored in RAM), which can be set by configuring \hyperref[fielddesc:RMTREGCARRIEREFFENCHN]{RMT\_CARRIER\_EFF\_EN\_CH\regindex{n}}.

The continuous transmission of the transmitter can be enabled by setting \hyperref[fielddesc:RMTREGTXCONTIMODECHN]{RMT\_TX\_CONTI\_MODE\_CH\regindex{n}}. When this register is set, the transmitter will send the pulse codes from RAM in loops. If \hyperref[fielddesc:RMTREGTXLOOPCNTENCHN]{RMT\_TX\_LOOP\_CNT\_EN\_CH\regindex{n}} is set to 1, the transmitter will start counting loop times. Once the counting reaches the value of register \\
\hyperref[fielddesc:RMTREGTXLOOPNUMCHN]{RMT\_TX\_LOOP\_NUM\_CH\regindex{n}}, an \hyperref[int:RMTCHnTXLOOPINT]{RMT\_CH\regindex{n}\_TX\_LOOP\_INT} interrupt will be generated.

Setting \hyperref[fielddesc:RMTREGTXSIMEN]{RMT\_TX\_SIM\_EN} to 1 will enable multiple channels to start sending data simultaneously. \hyperref[fielddesc:RMTREGTXSIMCHN]{RMT\_TX\_SIM\_CH\regindex{n}} will choose which multiple channels are enabled to send data simultaneously.

\subsection{Receiver}
When \hyperref[fielddesc:RMTREGRXENCHN]{RMT\_RX\_EN\_CH\regindex{n}} is set to 1, the receiver in channel \regindex{n} becomes active, detecting signal levels and measuring clock cycles the signals lasts for. These data will be written in RAM in the form of pulse codes. Receiving ends, when the receiver detects no change in a signal level for a number of clock cycles more than the value set by \hyperref[fielddesc:RMTREGIDLETHRESCHN]{RMT\_IDLE\_THRES\_CH\regindex{n}}. The receiver will return to idle state and generate an \hyperref[int:RMTCHnRXENDINT]{RMT\_CH\regindex{n}\_RX\_END\_INT} interrupt.

The receiver has an input signal filter which can be enabled by configuring \hyperref[fielddesc:RMTREGRXFILTERENCHN]{RMT\_RX\_FILTER\_EN\_CH\regindex{n}}. The filter samples input signals continuously, and will detect the signals which remain unchanged for a continuous \hyperref[fielddesc:RMTREGRXFILTERTHRESCHN]{RMT\_RX\_FILTER\_THRES\_CH\regindex{n}} APB clock cycles as valid, otherwise, the signals will be detected as invalid. Only the valid signals can pass through this filter. The filter will remove pulses with a length of less than \hyperref[fielddesc:RMTREGRXFILTERTHRESCHN]{RMT\_RX\_FILTER\_THRES\_CH\regindex{n}} APB clock cycles.

\subsection{Interrupts}
\begin{itemize}
\item \label{int:RMTCHnERRINT}RMT\_CH\regindex{n}\_ERR\_INT: Triggered when channel \regindex{n} does not read or write data correctly. For example, if the transmitter still tries to read data from RAM when the RAM is empty, or the receiver still tries to write data into RAM when the RAM is full, this interrupt will be triggered.
\item \label{int:RMTCHnTXTHREVENTINT}RMT\_CH\regindex{n}\_TX\_THR\_EVENT\_INT: Triggered when the amount of data the transmitter has sent matches the value of \hyperref[fielddesc:RMTREGTXLIMCHN]{RMT\_TX\_LIM\_CH\regindex{n}}.
\item \label{int:RMTCHnTXENDINT}RMT\_CH\regindex{n}\_TX\_END\_INT: Triggered when the transmitter has finished transmitting signals.
\item \label{int:RMTCHnRXENDINT}RMT\_CH\regindex{n}\_RX\_END\_INT: Triggered when the receiver has finished receiving signals.
\item \label{int:RMTCHnTXLOOPINT}RMT\_CH\regindex{n}\_TX\_LOOP\_INT: Triggered when the loop counting reaches the value set by \hyperref[fielddesc:RMTREGTXLOOPNUMCHN]{RMT\_TX\_LOOP\_NUM\_CH\regindex{n}}.
\end{itemize}
