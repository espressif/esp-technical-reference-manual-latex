\hypertarget{aes}{}
\chapter{AES 加速器 (AES)}\label{mod:aes}


\section{概述}

ESP32-S2 内置 AES（高级加密标准）硬件加速器可使用 AES 算法，完成数据的加解密运算，具有 \hyperref[subsection:typicalaes]{Typical AES} 和 \hyperref[subsection:dma-aes]{DMA-AES} 两种工作模式。整体而言，相比基于纯软件的 AES 运算，AES 硬件加速器能够极大地提高运算速度。

\section{主要特性}

ESP32-S2 支持以下特性：
\begin{itemize}
    \item Typical AES 工作模式
        \begin{itemize}
            \item AES-128/AES-192/AES-256 加解密运算
            \item 4 种密钥字节序和 4 种文本字节序
        \end{itemize}
    \end{itemize}

    \begin{itemize}
    \item DMA-AES 工作模式
        \begin{itemize}
            \item 块模式
                \begin{itemize}
                    \item ECB (Electronic Codebook)
                    \item CBC (Cipher Block Chaining)
                    \item OFB (Output Feedback)
                    \item CTR (Counter)
                    \item CFB8 (8-bit Cipher Feedback)
                    \item CFB128 (128-bit Cipher Feedback)
                \end{itemize}
            \item GCM (Galois/Counter Mode)
            \item 中断发生
        \end{itemize}
    \end{itemize}

\section{工作模式简介}

ESP32-S2 内置的 AES 加速器支持 Typical AES 和 DMA-AES 两种工作模式。

\begin{itemize}
    \item Typical AES 工作模式：支持 \href{https://nvlpubs.nist.gov/nistpubs/FIPS/NIST.FIPS.197.pdf}{NIST FIPS 197}，能够实现 AES-128、AES-192、AES-256  加密与解密运算。这种情况下，明文/密文的读/写操作统一通过 CPU 访问完成。
    \item DMA-AES 工作模式：支持 \href{https://nvlpubs.nist.gov/nistpubs/Legacy/SP/nistspecialpublication800-38a.pdf}{NIST SP 800-38A} 标准中的 ECB/CBC/OFB/CTR/CFB8/CFB128 等块加密模式运算以及 \href{https://nvlpubs.nist.gov/nistpubs/Legacy/SP/nistspecialpublication800-38d.pdf}{NIST SP 800-38D} 标准中的 GCM 运算。在这种情况下，明文/密文的传输通过硬件上的 crypto DMA 完成，计算完成时会有中断发生。
\end{itemize}

用户可通过配置 \hyperref[regdesc:AESDMAENABLEREG]{AES\_DMA\_ENABLE\_REG} 选择 AES 加速器的工作模式，具体参考表 \ref{table:AES_DMA_ENABLE}。

\begin{longtable}[c]{ |c|l| }
\caption{工作模式}
\label{table:AES_DMA_ENABLE}
\\\hline
\rowcolor{lightgray}
\hyperref[regdesc:AESDMAENABLEREG]{AES\_DMA\_ENABLE\_REG}       & 工作模式\\
    \hline
    0           & Typical AES    \\
    \hline
    1           & DMA-AES        \\
    \hline
\end{longtable}

有关 Typical AES 和 DMA-AES 两种工作模式的具体介绍，请见下方 \ref{subsection:typicalaes} 章节和 \ref{subsection:dma-aes} 章节。

\begin{importantlisting}
\vspace{-0.5em}

    ESP32-S2 的 \nameref{mod:digsig} 和\hyperref[mod:extmemencr]{片外存储手动加密}模块也会调用 AES 加速器。此时，用户无法正常访问 AES 加速器。
\end{importantlisting}

\section{Typical AES 工作模式}\label{subsection:typicalaes}

在 Typical AES 工作模式下，AES 加速器支持 AES-128/AES-192/AES-256 加解密共 6 种运算类型。
用户可通过配置 \hyperref[regdesc:AESMODEREG]{AES\_MODE\_REG} 寄存器选择具体运算类型，具体可参考表 \ref{table:AESOperation}。

\begin{longtable}[c]{ | c | l | }

\label{table:AESOperation}
\\\hline
\rowcolor{lightgray}

    \hline
    0                   & AES-128 加密    \\
    \hline
    1                   & AES-192 加密    \\
    \hline
    2                   & AES-256 加密    \\
    \hline

    \hline
    5                   & AES-192 解密    \\
    \hline

    \hline

AES 加速器的状态值可查看寄存器 \hyperref[regdesc:AESSTATEREG]{AES\_STATE\_REG}，具体见表 \ref{table:AESStateValue-aes} 所示：

\begin{longtable}[c]{ |c | l | l | }

\label{table:AESStateValue-aes}

    0                   & IDLE      &   加速器空闲或计算完成\\ \hline

在 Typical AES 工作模式下，AES 加速器每加密一个信息块需要 11 \verb+~+ 15 个时钟周期，每解密一个信息块需要 21 或 22 个时钟周期。


\subsection{密钥、明文、密文}

寄存器 \hyperref[regdesc:AESKEYnREG]{AES\_KEY\_\regindex{n}\_REG} 用于存放密钥，由 8 个 32 位寄存器组成。

\begin{itemize}
    \item 如果为 AES-128 加解密运算，则 128 位密钥在寄存器 \hyperref[regdesc:AESKEYnREG]{AES\_KEY\_0\_REG} \verb+~+ \hyperref[regdesc:AESKEYnREG]{AES\_KEY\_3\_REG} 中。
    \item 如果为 AES-192 加解密运算，则 192 位密钥在寄存器 \hyperref[regdesc:AESKEYnREG]{AES\_KEY\_0\_REG} \verb+~+ \hyperref[regdesc:AESKEYnREG]{AES\_KEY\_5\_REG} 中。
    \item 如果为 AES-256 加解密运算，则 256 位密钥在寄存器 \hyperref[regdesc:AESKEYnREG]{AES\_KEY\_0\_REG} \verb+~+ \hyperref[regdesc:AESKEYnREG]{AES\_KEY\_7\_REG} 中。
\end{itemize}

寄存器 \hyperref[regdesc:AESTEXTINnREG]{AES\_TEXT\_IN\_\regindex{m}\_REG} 和 \hyperref[regdesc:AESTEXTOUTnREG]{AES\_TEXT\_OUT\_\regindex{m}\_REG} 用于存放明文或密文，各由 4 个 32 位寄存器组成。

\begin{itemize}
    \item 如果为 AES-128/192/265 加密运算，则运算开始之前用明文初始化寄存器 \hyperref[regdesc:AESTEXTINnREG]{AES\_TEXT\_IN\_\regindex{m}\_REG}。运算完成之后，AES 加速器将把密文更新入寄存器 \hyperref[regdesc:AESTEXTOUTnREG]{AES\_TEXT\_OUT\_\regindex{m}\_REG}。
    \item 如果为 AES-128/192/256 解密运算，则运算开始之前用密文初始化寄存器 \hyperref[regdesc:AESTEXTINnREG]{AES\_TEXT\_IN\_\regindex{m}\_REG}。运算完成之后，AES 加速器将把明文更新入寄存器 \hyperref[regdesc:AESTEXTOUTnREG]{AES\_TEXT\_OUT\_\regindex{m}\_REG}。
\end{itemize}

\subsection{字节序}

\textbf{文本字节序}

在 Typical AES 工作模式下，AES 加速器可以使用密钥对 128 位的 block 进行加解密。其中，输入文本的字节序由寄存器 \hyperref[regdesc:AESENDIANREG]{AES\_ENDIAN\_REG} 的 Bit 2 和 Bit 3 控制，输出文本字节序由 Bit 4 和 Bit 5 控制。具体来说，Bit 2 和 Bit 4 控制每个 word 内 4 个 byte 的顺序，Bit 3 和 Bit 5 控制每个 block 中 4 个 word 的顺序。

不难看出，通过配置 \hyperref[regdesc:AESENDIANREG]{AES\_ENDIAN\_REG} 寄存器，AES 加速器可允许四种文本字节序。表 \ref{table:TypicalAESPlaintextCiphertextEndian} 指定了在四种不同字节序下，寄存器 \hyperref[regdesc:AESTEXTINnREG]{AES\_TEXT\_IN\_\regindex{m}\_REG} 和 \hyperref[regdesc:AESTEXTINnREG]{AES\_TEXT\_OUT\_\regindex{m}\_REG} 中的每个 word 所存放的明文或密文将如何构成 State。

\subsection{Typical AES 工作模式的流程 }

\textbf{单次运算}

\begin{enumerate}
    \item 对寄存器 \hyperref[regdesc:AESDMAENABLEREG]{AES\_DMA\_ENABLE\_REG} 写入 0。
    \item 初始化寄存器 \hyperref[regdesc:AESMODEREG]{AES\_MODE\_REG}、\hyperref[regdesc:AESKEYnREG]{AES\_KEY\_\regindex{n}\_REG}、\hyperref[regdesc:AESTEXTINnREG]{AES\_TEXT\_IN\_\regindex{m}\_REG}、\hyperref[regdesc:AESENDIANREG]{AES\_ENDIAN\_REG}。
    \item 启动运算。对寄存器 \hyperref[regdesc:AESTRIGGERREG]{AES\_TRIGGER\_REG} 写入 1。
    \item 等待运算完成。轮询寄存器 \hyperref[regdesc:AESSTATEREG]{AES\_STATE\_REG}，直到读到 0。
    \item 从寄存器 \hyperref[regdesc:AESTEXTOUTnREG]{AES\_TEXT\_OUT\_\regindex{m}\_REG} 读取结果。
\end{enumerate}

\textbf{连续运算}

在连续运算过程中，每次运算完成之后，只有寄存器 \hyperref[regdesc:AESTEXTOUTnREG]{AES\_TEXT\_OUT\_\regindex{m}\_REG} (\regindex{m}: 0-3) 会被 AES 加速器更新，
而 \hyperref[regdesc:AESDMAENABLEREG]{AES\_DMA\_ENABLE\_REG}、\hyperref[regdesc:AESMODEREG]{AES\_MODE\_REG}、\hyperref[regdesc:AESKEYnREG]{AES\_KEY\_\regindex{n}\_REG}、\hyperref[regdesc:AESENDIANREG]{AES\_ENDIAN\_REG} 等寄存器中的内容不会变化。
所以进行连续运算时可以简化初始化操作。

\begin{enumerate}
    \item 第一次运算之前对寄存器 \hyperref[regdesc:AESDMAENABLEREG]{AES\_DMA\_ENABLE\_REG} 写入 0。
    \item 第一次运算之前初始化寄存器 \hyperref[regdesc:AESMODEREG]{AES\_MODE\_REG}、\hyperref[regdesc:AESKEYnREG]{AES\_KEY\_\regindex{n}\_REG}、\hyperref[regdesc:AESENDIANREG]{AES\_ENDIAN\_REG}。
    \item 更新寄存器 \hyperref[regdesc:AESTEXTINnREG]{AES\_TEXT\_IN\_\regindex{m}\_REG}。 \label{update register in typical AES continuous calculation}
    \item 启动运算。对寄存器 \hyperref[regdesc:AESTRIGGERREG]{AES\_TRIGGER\_REG} 写入 1。
    \item 等待运算完成。轮询寄存器 \hyperref[regdesc:AESSTATEREG]{AES\_STATE\_REG}，直到读到 0。
    \item 从寄存器 \hyperref[regdesc:AESTEXTOUTnREG]{AES\_TEXT\_OUT\_\regindex{m}\_REG} 读取结果 \label{read result in typical AES continuous calculation}。 返回步骤 \ref{update register in typical AES continuous calculation}，进行下一轮运算。
\end{enumerate}

\subsection{密钥、明文、密文}

\textbf{块运算模式}

在块运算模式下，AES 加速器的源数据 (in\_stream) 来自 DMA，结果数据 (out\_stream) 也将被写入 DMA。
\begin{itemize}
    \item 如果为加密运算，则 DMA 从 memory 中读取明文数据流并将其传给 AES。AES 计算出密文后将密文写入 DMA。DMA 再将密文写入 memory。
    \item 如果为解密运算，则 DMA 从 memory 中读取密文数据流并将其传给 AES。AES 计算出明文后将明文写入 DMA。DMA 再将明文写入 memory。
\end{itemize}

AES 加速器在进行块运算时，结果数据与源数据的大小保持一致。此时，DMA 的数据搬运过程和 AES 的计算过程有所交叠，因此总工作时间有所减少。

\label{api:testpadding}值得注意的是，AES 加速器在 DMA-AES 工作模式下要求源数据的大小必须是 128 位的整数倍，否则需要将原始明文封装为 128 位的整数倍，即在原比特串 (bit string) 尾部尽可能少的补 “0”，具体过程见表 \ref{table:DefineFunctionTextPadding} 所示。

\subsection{ 块个数}\label{Label Block Number}

寄存器 \hyperref[regdesc:AESBLOCKNUMREG]{AES\_BLOCK\_NUM\_REG} 存放明文或密文的块个数 (Block Number)，其值等于 length({\bfseries TEXT-PADDING}($P$))/128，也等于 length({\bfseries TEXT-PADDING}($C$))/128。这里的 $P$ 指明文（plaintext）， $C$ 指密文（ciphertext）。该寄存器仅在 DMA-AES 工作模式下有意义。

\subsection{初始向量}\label{Label IV}

\label{memdesc:AESIVMEM}存储器 \hyperref[memdesc:AESIVMEM]{AES\_IV\_MEM} 的空间大小为 16 字节，仅在块运算模式下有效。
对于 CBC/OFB/CFB8/CFB128 等操作，\hyperref[memdesc:AESIVMEM]{AES\_IV\_MEM} 用于存放初始向量 (Initialization Vector, IV) 的值。对于 CTR 操作， \hyperref[memdesc:AESIVMEM]{AES\_IV\_MEM}  存放初始计数器 (Initial Counter Block, ICB) 的值。

IV 和 ICB 都是 128-bit 长的比特串，从左向右被分割成 16 个字节 (Byte0, Byte1, Byte2, \begin{math}\cdots\end{math}, Byte15)， 构成一个字节序列，在 \hyperref[memdesc:AESIVMEM]{AES\_IV\_MEM} 中存放时需要遵循表 \ref{table:DMAAESPlaintextCiphertextEndian} 中的字节序规则，即 Byte0 存放在 \hyperref[memdesc:AESIVMEM]{AES\_IV\_MEM} 中的最低地址中，Byte15 存放在 \hyperref[memdesc:AESIVMEM]{AES\_IV\_MEM} 中的最高地址中。

更多有关 IV 和 ICB 的信息，请参考 \href{https://nvlpubs.nist.gov/nistpubs/Legacy/SP/nistspecialpublication800-38a.pdf}{NIST SP 800-38A} 标准。

\subsection{ 块运算模式的流程 }

\begin{enumerate}
    \item 对寄存器 \hyperref[regdesc:CRYPTODMAAESSHASELECTREG]{CRYPTO\_DMA\_AES\_SHA\_SELECT\_REG} 写入 0。
    \item 配置 Crypto DMA 链表并启动 DMA。请见章节 \ref{mod:dma} \textit{\nameref{mod:dma}}获取更多信息。
    \item 配置 AES：
    \begin{itemize}
        \item 对寄存器 \hyperref[regdesc:AESDMAENABLEREG]{AES\_DMA\_ENABLE\_REG} 写入 1。
        \item 选择是否开启中断。根据需要设置寄存器 \hyperref[regdesc:AESINTENAREG]{AES\_INT\_ENA\_REG} 的值。
        \item 初始化寄存器 \hyperref[regdesc:AESMODEREG]{AES\_MODE\_REG}、\hyperref[regdesc:AESKEYnREG]{AES\_KEY\_\regindex{n}\_REG}、\hyperref[regdesc:AESENDIANREG]{AES\_ENDIAN\_REG}。
        \item 初始化寄存器 \hyperref[regdesc:AESBLOCKMODEREG]{AES\_BLOCK\_MODE\_REG}，请参照表 \ref{table:AESBlockMode}。
        \item 初始化寄存器 \hyperref[regdesc:AESBLOCKNUMREG]{AES\_BLOCK\_NUM\_REG}，请参照章节 \ref{Label Block Number}。
        \item 初始化寄存器 \hyperref[regdesc:AESINCSELREG]{AES\_INC\_SEL\_REG}（                仅在 CTR 块模式下使用）。
        \item 初始化存储器 \hyperref[memdesc:AESIVMEM]{AES\_IV\_MEM}（在 ECB 块模式下不使用）。
    \end{itemize}
    \item 启动运算。对寄存器 \hyperref[regdesc:AESTRIGGERREG]{AES\_TRIGGER\_REG} 写入 1。
    \item \label{wait ECB/CBC/OFB/CTR/CFB8/CFB128 done}
            等待运算完成。轮询寄存器 \hyperref[regdesc:AESSTATEREG]{AES\_STATE\_REG}，直到读到 2。
            如果开启了中断功能，也可以等待 AES\_INT 中断产生。
    \item 确认 DMA 完成从 AES 到内存的数据传输。此时，结果数据已经被 DMA 写入 memory，可以直接从中读取。详情请参考章节 \ref{mod:dma} \textit{\nameref{mod:dma}}。
    \item 如果开启了中断，当处理中断程序完成后，请及时对寄存器 \hyperref[regdesc:AESINTCLRREG]{AES\_INT\_CLR\_REG} 写 1 以清除中断。
    \item 对寄存器 \hyperref[regdesc:AESDMAEXITREG]{AES\_DMA\_EXIT\_REG} 写入 1 释放 AES 加速器。之后如果再读取寄存器 \hyperref[regdesc:AESSTATEREG]{AES\_STATE\_REG} 将读到 0。
    该步操作可以提前完成，但必须在步骤 \ref{wait ECB/CBC/OFB/CTR/CFB8/CFB128 done} 之后。
\end{enumerate}

\subsection{ GCM 运算模式的流程 }

\begin{enumerate}
    \item 对寄存器 \hyperref[regdesc:CRYPTODMAAESSHASELECTREG]{CRYPTO\_DMA\_AES\_SHA\_SELECT\_REG} 写入 0。
    \item 配置 Crypto DMA 链表并启动 DMA。请参照章节 \ref{mod:dma} \textit{\nameref{mod:dma}}获取更多信息。
    \item 配置 AES：
    \begin{itemize}
        \item 对寄存器 \hyperref[regdesc:AESDMAENABLEREG]{AES\_DMA\_ENABLE\_REG} 写入 1。
        \item 选择是否开启中断。根据需要设置寄存器 \hyperref[regdesc:AESINTENAREG]{AES\_INT\_ENA\_REG} 的值。
        \item 初始化寄存器 \hyperref[regdesc:AESMODEREG]{AES\_MODE\_REG}、\hyperref[regdesc:AESKEYnREG]{AES\_KEY\_\regindex{n}\_REG}、\hyperref[regdesc:AESENDIANREG]{AES\_ENDIAN\_REG}。
        \item 初始化寄存器 \hyperref[regdesc:AESBLOCKMODEREG]{AES\_BLOCK\_MODE\_REG} 为 6。
        \item 初始化寄存器 \hyperref[regdesc:AESBLOCKNUMREG]{AES\_BLOCK\_NUM\_REG}，请参照章节 \ref{Label Block Number}。
        \item 初始化寄存器 \hyperref[regdesc:AESAADBLOCKNUMREG]{AES\_AAD\_BLOCK\_NUM\_REG}，请参照章节 \ref{Label AAD Block Number}。
        \item 初始化寄存器 \hyperref[regdesc:AESREMAINDERBITNUMREG]{AES\_REMAINDER\_BIT\_NUM\_REG}，请参照章节 \ref{Label Remainder Bit Number}。解密时不使用。
    \end{itemize}
    \item 启动运算。对寄存器 \hyperref[regdesc:AESTRIGGERREG]{AES\_TRIGGER\_REG} 写入 1。
    \item 轮询寄存器 \hyperref[regdesc:AESSTATEREG]{AES\_STATE\_REG}，直到读到 0。请参照表  \ref{table:AESStateValue-dma}。 此处无中断产生。
    \item 读存储器 \hyperref[memdesc:AESHMEM]{AES\_H\_MEM} 获取 $H$。
    \item 软件计算 $J_0$，然后将其写入存储器 \hyperref[memdesc:AESJ0MEM]{AES\_J0\_MEM}。
    \item 继续运算。对寄存器 \hyperref[regdesc:AESCONTINUEOPREG]{AES\_CONTINUE\_OP\_REG} 写入 1。
    \item \label{wait GCM done}
            等待运算完成。轮询寄存器 \hyperref[regdesc:AESSTATEREG]{AES\_STATE\_REG}，直到读到 2。 请参照表  \ref{table:AESStateValue-dma}。
            如果开启了中断功能，也可以等待 AES\_INT 中断产生。
    \item 此时，$T_0$ 已经准备好。读存储器 \hyperref[memdesc:AEST0MEM]{AES\_T0\_MEM} 获取 $T_0$。
    \item 确认 DMA 完成数据传输。此时，结果数据已经被 DMA 写入 memory，可以直接从中读取。请参照章节 \ref{mod:dma} \textit{\nameref{mod:dma}}获取更多信息。
    \item 如果开启了中断，当处理中断程序完成后，请及时对寄存器 \hyperref[regdesc:AESINTCLRREG]{AES\_INT\_CLR\_REG} 写 1 以清除中断。
    \item 对寄存器 \hyperref[regdesc:AESDMAEXITREG]{AES\_DMA\_EXIT\_REG} 写入 1。之后如果再读取寄存器 \hyperref[regdesc:AESSTATEREG]{AES\_STATE\_REG} 将读到 0。该步操作可以提前完成，但必须在步骤 \ref{wait GCM done} 之后。
\end{enumerate}


\section{GCM 算法}\label{Label GCM algorithm}

ESP32-S2 中 AES 加速器直接支持 GCM 算法（更多信息，请见 \href{https://nvlpubs.nist.gov/nistpubs/Legacy/SP/nistspecialpublication800-38d.pdf}{NIST SP 800-38D} 标准）。值得注意的是，考虑到实际使用中很少会使用长度超过 \begin{math}2^{32}-1\end{math} 比特的 \begin{math}AAD\end{math} 、\begin{math}C\end{math} 和 \begin{math}P\end{math}，我们在算法中将 \begin{math}AAD\end{math} 、\begin{math}C\end{math} 和 \begin{math}P\end{math} 的长度限制在 \begin{math}2^{32}-1\end{math} 比特之内。
图 \ref{figure:GCMEncryptionOperation} 显示了 ESP32-S2 内置 AES 加速器实现 GCM 加密算法的流程。

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{\modulefiles/figures/GCM_encryption.eps}
    \caption{GCM 加密操作流程图}
    \label{figure:GCMEncryptionOperation}
\end{figure}

具体步骤描述见下：

\begin{enumerate}
    \item \label{GCM operation : Hardware Calculate H}
          硬件调用 ECB 算法求解\hyperref[Label Hash subkey]{哈希子密钥 ($H$)} 的值，为哈希计算做准备；
    \item \label{GCM operation : Hardware Calculate AAD hash}
          硬件调用 GHASH 算法对\hyperref[api:testpadding]{封装}后的 $AAD$ 作哈希计算；
    \item \label{GCM operation : Hardware Calculate J0 inc32}
          硬件调用标准增量函数 INC$_{32}$ 对 \hyperref[Label J0]{$J_{0}$} 作运算，为 CTR 加密做准备；
    \item \label{GCM operation : Hardware Calculate C hash}
          硬件调用 GCTR 算法对封装后的明文 $P$ 做加密运算，同时调用 GHASH 算法对封装后的密文 $C$ 作哈希计算；
    \item \label{GCM operation : Hardware Calculate padding block hash}
          硬件调用 GHASH 算法对追加块做哈希计算，获得最终的 128 位哈希结果；
    \item \label{GCM operation : Hardware Calculate T0}
          硬件调用 GCTR 算法对 $J_{0}$ 做加密运算，获得 $T_{0}$ ；
    \item \label{GCM operation : Software Calculate T}
          软件读取硬件结果 $T_{0}$，并调用 MSB$_{t}$ 算法计算最终的认证标签 $T$ 。
\end{enumerate}

GCM 解密运算与 GCM 加密运算基本相同，仅有一处差别：在图 \ref{figure:GCMEncryptionOperation} 中的步骤 \ref{GCM operation : Hardware Calculate C hash} 中，使用 GCTR 算法对封装后的密文做解密运算，得出明文。此处不再赘述。


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\subsection{哈希子密钥（Hash subkey）}\label{Label Hash subkey}

在 GCM 操作中，哈希子密钥 ($H$) 是一个 128-bit 的值，由硬件计算求出，即 图 \ref{figure:GCMEncryptionOperation} 中的步骤 \ref{GCM operation : Hardware Calculate H} ，
对应 \href{https://nvlpubs.nist.gov/nistpubs/Legacy/SP/nistspecialpublication800-38d.pdf}{NIST SP 800-38D} 标准中 “7 GCM Specification” 中的 “ Step 1. Let $H$ = CIPH$_K$({$0^{128}$}) ” 描述。\\
哈希子密钥 ($H$) 的值以字节序的形式存放在存储器 \hyperref[memdesc:AESHMEM]{AES\_H\_MEM} 中\label{memdesc:AESHMEM}，
且需遵循表 \ref{table:DMAAESPlaintextCiphertextEndian} 中的字节序规，
即最左侧字节存放在 \hyperref[memdesc:AESHMEM]{AES\_H\_MEM} 中的最低地址，最右侧字节存放在 \hyperref[memdesc:AESHMEM]{AES\_H\_MEM} 中的最高地址。


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\subsection{\texorpdfstring{$J_0$}{J0}} \label{Label J0}
在 GCM 操作中， $J_0$ 是一个 128-bit 的值，由软件计算求出，将参与到 图 \ref{figure:GCMEncryptionOperation} 中的步骤 \ref{GCM operation : Hardware Calculate J0 inc32} 和步骤 \ref{GCM operation : Hardware Calculate T0}  的计算中。
具体的计算方式在 \href{https://nvlpubs.nist.gov/nistpubs/Legacy/SP/nistspecialpublication800-38d.pdf}{NIST SP 800-38D} 标准中的 “7 GCM Specification”章节部分有明确定义。请参阅 \href{https://nvlpubs.nist.gov/nistpubs/Legacy/SP/nistspecialpublication800-38d.pdf}{NIST SP 800-38D} 获取更多算法标准的相关信息。\\
$J_0$ 值以字节序的形式存放在存储器 \hyperref[memdesc:AESJ0MEM]{AES\_J0\_MEM} 中\label{memdesc:AESJ0MEM}，
且需遵循表 \ref{table:DMAAESPlaintextCiphertextEndian} 中的字节序规，
即最左侧字节存放在 \hyperref[memdesc:AESJ0MEM]{AES\_J0\_MEM} 中的最低地址，最右侧字节存放在 \hyperref[memdesc:AESJ0MEM]{AES\_J0\_MEM} 中的最高地址。


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\subsection{认证标签（Authenticated Tag）}\label{Label Authenticated Tag}

认证标签（Authenticated Tag，简称为Tag）是 GCM 计算的最终结果之一，由软件完成计算，对应图 \ref{figure:GCMEncryptionOperation} 中的步骤 \ref{GCM operation : Software Calculate T} ，具体取值由长度参数 $t$ ($1 <= t <= 128$)决定：
\begin{itemize}
    \item 当 $t = 128$ 时，Tag 的值记为 $T_0$。 $T_0$ 是一个 128-bit 的比特串，以字节序的形式存放在存储器 \hyperref[memdesc:AEST0MEM]{AES\_T0\_MEM} 中\label{memdesc:AEST0MEM}，且需遵循表 \ref{table:DMAAESPlaintextCiphertextEndian} 中的字节序规，即最左侧字节存放在 \hyperref[memdesc:AEST0MEM]{AES\_T0\_MEM} 中的最低地址，最右侧字节存放在 \hyperref[memdesc:AEST0MEM]{AES\_T0\_MEM} 中的最高地址。
    \item 当 $1 <= t < 128$ 时，Tag 的值为 $T_0$ 中最左侧的 $t$ 个比特位，可以用 MSB$_t$($T_0$)  函数表示。例如，MSB$_4$(111011010) = 1110，MSB$_5$(11010011010) = 11010。有关 MSB$_t$(\ ) 函数的具体定义，请见 \href{https://nvlpubs.nist.gov/nistpubs/Legacy/SP/nistspecialpublication800-38d.pdf}{NIST SP 800-38D} 的 “6 Mathematical Components of GCM” 章节。
\end{itemize}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\subsection{ 附加认证消息块个数（ AAD Block Number ）}\label{Label AAD Block Number}

寄存器 \hyperref[regdesc:AESAADBLOCKNUMREG]{AES\_AAD\_BLOCK\_NUM\_REG} 存放附加认证消息（Additional Authenticated Data, AAD）的块个数，其值等于 length({\bfseries TEXT-PADDING}($AAD$))/128。该寄存器仅在 DMA-AES 工作模式下的 GCM 操作中有意义。

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\subsection{ 不完整块的有效比特数 (Remainder Bit Number)}\label{Label Remainder Bit Number}

寄存器 \hyperref[regdesc:AESREMAINDERBITNUMREG]{AES\_REMAINDER\_BIT\_NUM\_REG} 存放明文中不完整块的有效比特数，其值等于 length($P$)\%128。该寄存器仅在 DMA-AES 工作模式下的 GCM 加密操作中有意义。寄存器 \hyperref[regdesc:AESREMAINDERBITNUMREG]{AES\_REMAINDER\_BIT\_NUM\_REG} 的值不会影响密文或明文结果，但会影响 Tag 结果值（包含在 $T_{0}$ 中）。
依据 GCM 算法标准，GCM 运算是 GCTR 运算和 GHASH 运算的结合，GCTR 运算用于执行加解密计算，GHASH 运算用于求解 Tag。

\begin{itemize}
    \item 对于 GCM 加密，硬件首先计算 $C$，而后将其封装为 {\bfseries TEXT-PADDING}($C$)，作为 GHASH 运算的输入。此时，硬件需要根据寄存器 \hyperref[regdesc:AESREMAINDERBITNUMREG]{AES\_REMAINDER\_BIT\_NUM\_REG} 的值决定追加 0 的个数。
    \item 对于 GCM 解密，封装操作 {\bfseries TEXT-PADDING}($C$)交由软件完成，此时寄存器 \hyperref[regdesc:AESREMAINDERBITNUMREG]{AES\_REMAINDER\_BIT\_NUM\_REG} 的值不起作用。
\end{itemize}
