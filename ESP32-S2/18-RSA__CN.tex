\hypertarget{rsa}{}
\chapter{RSA 加速器 (AES)}\label{mod:rsa}


\section{概述}

RSA 加速器可为多种运用于 “RSA 非对称式加密演算法”的高精度计算提供硬件支持，能够极大地降低此类运算的软件复杂度，且支持多种“运算子长度”，具有很高的运算效率。

\section{主要特性}

RSA 加速器支持以下功能：
\begin{itemize}
    \item 大数模幂运算（支持两个加速选项）
    \item 大数模乘运算
    \item 大数乘法运算
    \item 多种运算子长度
    \item 中断功能
\end{itemize}

\section{功能描述}

RSA 加速器的激活仅需使能 \hyperref[regdesc:SYSTEMPERIPCLKEN1REG]{SYSTEM\_PERIP\_CLK\_EN1\_REG} 外围时钟的 \hyperref[fielddesc:SYSTEMCRYPTORSACLKEN]{SYSTEM\_CRYPTO\_RSA\_CLK\_EN} 位，并同时清零 \hyperref[regdesc:SYSTEMRSAPDCTRLREG]{SYSTEM\_RSA\_PD\_CTRL\_REG} 寄存器中的 \hyperref[fielddesc:SYSTEMRSAMEMPD]{SYSTEM\_RSA\_MEM\_PD} 位。

不过，RSA 加速器激活后还须等待\hyperref[Memorylist]{RSA 相关 存储器}初始化完成后才能开始工作。具体来说，寄存器 \hyperref[regdesc:RSACLEANREG]{RSA\_CLEAN\_REG} 读 0 时初始化开始，读 1 时初始化完成。因此，在复位后首次使用 RSA 加速器时，软件需要先查询寄存器 \hyperref[regdesc:RSACLEANREG]{RSA\_CLEAN\_REG} 的值是否为 1，以确保 RSA 加速器可正常工作。

此外，RSA 加速器支持中断功能，可对寄存器 \hyperref[regdesc:RSAINTERRUPTENAREG]{RSA\_INTERRUPT\_ENA\_REG} 写 1 / 0 以开启 / 关闭中断。RSA 加速器的中断功能默认开启。

\begin{importantlisting}
\vspace{-0.5em}

    ESP32-S2 的 \nameref{mod:digsig} 模块也会调用 RSA 加速器。此时，用户无法正常访问 RSA 加速器。
\end{importantlisting}

\subsection{大数模幂运算}\label{subsection:large_modular_Exponentiation}

大数模幂运算的算法是 \begin{math}Z = X^{Y} \bmod M\end{math}，它是基于 Montgomery Multiplication （蒙哥马利乘法）实现的。因此，对于大数模幂运算，除了需要运算子
\begin{math}X\end{math}、
\begin{math}Y\end{math}、
\begin{math}M\end{math}外，
还需要额外两个运算子，即参数
\begin{math}\overline{r}\end{math} 和
\begin{math}M^{\prime}\end{math}。
这两个参数需要通过软件提前运算得到。

RSA 加速器支持长度为
\begin{math}N = 32 \times x\end{math} (\begin{math}x \in \{ 1,2,3, \dots, 128\} \end{math}) 的大数模幂运算。
\begin{math}Z\end{math}、
\begin{math}X\end{math}、
\begin{math}Y\end{math}、
\begin{math}M\end{math} 和
\begin{math}\overline{r}\end{math}
的位宽为这 128 种中的任意一种，要求它们的位宽必须相同，
而 \begin{math}M^{\prime}\end{math} 的位宽始终是 32。

设进制数
\begin{gather*}
    b = 2^{32}
\end{gather*}

则运算子可以由若干个 \begin{math}b\end{math} 进制数来表示：
\begin{gather*}
    n = \frac{N}{32} \\
    Z = ( Z_{n-1} Z_{n-2} \cdots Z_{0} )_{b} \\
    X = ( X_{n-1} X_{n-2} \cdots X_{0} )_{b} \\
    Y = ( Y_{n-1} Y_{n-2} \cdots Y_{0} )_{b} \\
    M = ( M_{n-1} M_{n-2} \cdots M_{0} )_{b} \\
    \overline{r} = ( \overline{r}_{n-1} \overline{r}_{n-2} \cdots \overline{r}_{0} )_{b}
\end{gather*}

其中
\begin{math}Z_{n-1} \cdots Z_{0}\end{math}、
\begin{math}X_{n-1} \cdots X_{0}\end{math}、
\begin{math}Y_{n-1} \cdots Y_{0}\end{math}、
\begin{math}M_{n-1} \cdots M_{0}\end{math}、
\begin{math}\overline{r}_{n-1} \cdots \overline{r}_{0}\end{math}
分别表示一个 \begin{math}b\end{math} 进制数，位宽皆为 32。
且
\begin{math}Z_{n-1}\end{math}、
\begin{math}X_{n-1}\end{math}、
\begin{math}Y_{n-1}\end{math}、
\begin{math}M_{n-1}\end{math}、
\begin{math}\overline{r}_{n-1}\end{math}
分别为
\begin{math}Z\end{math}、
\begin{math}X\end{math}、
\begin{math}Y\end{math}、
\begin{math}M\end{math}、
\begin{math}\overline{r}\end{math}
最高位的 \begin{math}b\end{math} 进制数，
而
\begin{math}Z_{0}\end{math}、
\begin{math}X_{0}\end{math}、
\begin{math}Y_{0}\end{math}、
\begin{math}M_{0}\end{math}、
\begin{math}\overline{r}_{0}\end{math}
分别为
\begin{math}Z\end{math}、
\begin{math}X\end{math}、
\begin{math}Y\end{math}、
\begin{math}M\end{math}、
\begin{math}\overline{r}\end{math}
最低位的 \begin{math}b\end{math} 进制数。

另设 \begin{math}R = b^{n}\end{math}，则计算得参数 \begin{math}\overline{r} = R^{2} \bmod M\end{math}。

\begin{math}M^{\prime}\end{math} 可使用下方公式计算：

\begin{gather*}
        M^{-1} \times M + 1 = R \times R^{-1} \\
        M^{\prime} = M^{-1} \bmod b\\
\end{gather*}

注意，上方公式适用于使用扩展二进制 GCD 算法的运算。

大数模幂运算的软件流程为：

\begin{enumerate}

    \item 对寄存器 \hyperref[regdesc:RSAINTERRUPTENAREG]{RSA\_INTERRUPT\_ENA\_REG} 写 1 / 0 以开启 / 关闭中断。

    \item 配置相关寄存器。

    \begin{enumerate}
        \item 对寄存器 \hyperref[regdesc:RSAMODEREG]{RSA\_MODE\_REG} 写入 \begin{math}( \frac{N}{32} - 1 )\end{math}。

        \item 对寄存器 \hyperref[regdesc:RSAMPRIMEREG]{RSA\_M\_PRIME\_REG} 写入 \begin{math}M^{\prime}\end{math}。

        \item 根据需要配置加速选项相关寄存器。请参照章节 \ref{Label Accelerate options} 获取详细信息。
    \end{enumerate}

    \item 将
        \begin{math}X_{i}\end{math}、
        \begin{math}Y_{i}\end{math}、
        \begin{math}M_{i}\end{math}、
        \begin{math}\overline{r}_{i}\end{math}(\begin{math}i \in \{ 0, 1, \dots, n \end{math}\})
        分别写入存储器 \hyperref[regdesc:RSAmemory]{RSA\_X\_MEM}、\hyperref[regdesc:RSAmemory]{\hyperref[regdesc:RSAmemory]{RSA\_Y\_MEM}}、\hyperref[regdesc:RSAmemory]{RSA\_M\_MEM}、\hyperref[regdesc:RSAmemory]{RSA\_Z\_MEM}。每块存储器的容量都是 128 字 (word)。
        每块存储器的每一个字刚好存放一个 \begin{math}b\end{math} 进制数。
        这些存储器都是低地址存放运算子的低位进制数，高地址存放运算子的高位进制数。

        只需要根据运算子长度，将各个运算子中有效的数据写入存储器，没有使用到的存储器可以是任意值。

    \item 对寄存器 \hyperref[regdesc:RSAMODEXPSTARTREG]{RSA\_MODEXP\_START\_REG} 写入 1 启动计算。

    \item 等待运算结束。轮询寄存器 \hyperref[regdesc:RSAIDLEREG]{RSA\_IDLE\_REG} 直到读到 1，或者等待 RSA 中断产生。

    \item 从存储器 \hyperref[regdesc:RSAmemory]{RSA\_Z\_MEM} 读出运算结果 \begin{math}Z_{i}\end{math}(\begin{math}i \in \{ 0, 1, \dots, n \end{math}\} )。

    \item 若中断功能已开启，对寄存器 \hyperref[regdesc:RSACLEARINTERRUPTREG]{RSA\_CLEAR\_INTERRUPT\_REG} 写入 1 以清除中断。

\end{enumerate}

运算结束后，寄存器 \hyperref[regdesc:RSAMODEREG]{RSA\_MODE\_REG} 中存储的运算子长度信息以及
存储器 \hyperref[regdesc:RSAmemory]{RSA\_Y\_MEM} 中的 \begin{math}Y_{i}\end{math}、
存储器 \hyperref[regdesc:RSAmemory]{RSA\_M\_MEM} 中 的 \begin{math}M_{i}\end{math}、
寄存器 \hyperref[regdesc:RSAMPRIMEREG]{RSA\_M\_PRIME\_REG} 中的 \begin{math}M^{\prime}\end{math} 都不会变化。
但是，存储器 \hyperref[regdesc:RSAmemory]{RSA\_X\_MEM} 中的 \begin{math}X_{i}\end{math} 与
存储器 \hyperref[regdesc:RSAmemory]{RSA\_Z\_MEM} 中的 \begin{math}\overline{r}_{i}\end{math}
都已经被覆盖。所以当需要连续运算时，只需要更新必需的寄存器与存储器即可。


\subsection{大数模乘运算}

大数模乘运算也是基于 Montgomery Multiplication 实现运算
\begin{math}Z = X \times Y \bmod M\end{math}，
所以也需要预先通过软件计算得到
\begin{math}\overline{r}\end{math} 和
\begin{math}M^{\prime}\end{math}。

RSA 加速器也支持 128 种运算子长度的大数模乘运算。

大数模乘运算的软件流程为：

\begin{enumerate}

    \item 对寄存器 \hyperref[regdesc:RSAINTERRUPTENAREG]{RSA\_INTERRUPT\_ENA\_REG} 写 1 / 0 以开启 / 关闭中断。

    \item 配置相关寄存器。

    \begin{enumerate}
        \item 对寄存器 \hyperref[regdesc:RSAMODEREG]{RSA\_MODE\_REG} 写入 \begin{math}( \frac{N}{32} - 1 )\end{math}。

        \item 对寄存器 \hyperref[regdesc:RSAMPRIMEREG]{RSA\_M\_PRIME\_REG} 写入 \begin{math}M^{\prime}\end{math}。

    \end{enumerate}

    \item 将
        \begin{math}X_{i}\end{math}、
        \begin{math}Y_{i}\end{math}、
        \begin{math}M_{i}\end{math}、
        \begin{math}\overline{r}_{i}\end{math}
        (\begin{math}i \in \{0, 1, \dots, n\end{math}\})
        分别写入存储器
        \hyperref[regdesc:RSAmemory]{RSA\_X\_MEM}、
        \hyperref[regdesc:RSAmemory]{RSA\_Y\_MEM}、
        \hyperref[regdesc:RSAmemory]{RSA\_M\_MEM}、
        \hyperref[regdesc:RSAmemory]{RSA\_Z\_MEM}。
        每块存储器的容量都是 128 字 (word)。

        每块存储器的每一个字刚好存放一个 \begin{math}b\end{math} 进制数。这些存储器都是低地址存放运算子的低位进制数，高地址存放运算子的高位进制数。

        只需要根据运算子长度，将各个运算子中有效的数据写入存储器，
        没有使用到的存储器可以是任意值。

    \item 对寄存器 \hyperref[regdesc:RSAMODMULTSTARTREG]{RSA\_MODMULT\_START\_REG} 写入 1。

    \item 等待运算结束。轮询寄存器 \hyperref[regdesc:RSAIDLEREG]{RSA\_IDLE\_REG} 直到读到 1，或者等待 RSA 中断产生。

    \item 从存储器 \hyperref[regdesc:RSAmemory]{RSA\_Z\_MEM} 读出运算结果 \begin{math}Z_{i}\end{math} (\begin{math}i \in \{ 0, 1, \dots, n \end{math}\})。

    \item 若中断功能已开启，对寄存器 \hyperref[regdesc:RSACLEARINTERRUPTREG]{RSA\_CLEAR\_INTERRUPT\_REG} 写入 1 以清除中断。

\end{enumerate}


运算结束后，
寄存器 \hyperref[regdesc:RSAMODEREG]{RSA\_MODE\_REG} 中存储的运算子长度信息以及
存储器 \hyperref[regdesc:RSAmemory]{RSA\_X\_MEM} 中的 \begin{math}X_{i}\end{math}、
存储器 \hyperref[regdesc:RSAmemory]{RSA\_Y\_MEM} 中的 \begin{math}Y_{i}\end{math}、
存储器 \hyperref[regdesc:RSAmemory]{RSA\_M\_MEM} 中 的 \begin{math}M_{i}\end{math}、
寄存器 \hyperref[regdesc:RSAMPRIMEREG]{RSA\_M\_PRIME\_REG} 中的 \begin{math}M^{\prime}\end{math} 都不会变化。
但是，存储器 \hyperref[regdesc:RSAmemory]{RSA\_Z\_MEM} 中的 \begin{math}\overline{r}_{i}\end{math}
已经被覆盖。所以当需要连续运算时，只需要更新必需的寄存器与存储器即可。


\subsection{大数乘法运算}

大数乘法运算实现了 \begin{math}Z = X \times Y\end{math}。其中 \begin{math}Z\end{math} 的长度是
运算子 \begin{math}X\end{math}、\begin{math}Y\end{math} 长度的两倍。
所以 RSA 加速器只支持运算子长度为 \begin{math}N = 32 \times x\end{math} (\begin{math}x \in \{ 1, 2, \dots, 64\} \end{math}) 的大数乘法运算。运算子 \begin{math}Z\end{math} 的长度 \begin{math}\hat{N}\end{math} 为 \begin{math}2 \times N\end{math}。


大数乘法运算的软件流程为：

\begin{enumerate}

    \item 对寄存器 \hyperref[regdesc:RSAINTERRUPTENAREG]{RSA\_INTERRUPT\_ENA\_REG} 写 1 / 0 以开启 / 关闭中断。

    \item 配置相关寄存器。对寄存器 \hyperref[regdesc:RSAMODEREG]{RSA\_MODE\_REG} 写入 \begin{math}( \frac{\hat{N}}{32} - 1 )\end{math}，即 \begin{math}(  \frac{N}{16} - 1 )\end{math}。

    \item 将 \begin{math}{X}_{i}\end{math}、\begin{math}{Y}_{i}\end{math} (\begin{math}i \in \{ 0, 1, \dots, n \end{math}\}) 分别写入存储器 \hyperref[regdesc:RSAmemory]{RSA\_X\_MEM}、\hyperref[regdesc:RSAmemory]{RSA\_Z\_MEM}。每块存储器的容量都是 128 字。
        每块存储器的每一个字刚好存放一个 \begin{math}b\end{math} 进制数。
        这些存储器都是低地址存放运算子的低位进制数，高地址存放运算子的高位进制数。
        \begin{math}n\end{math} 为 \begin{math} \frac{N}{32} \end{math} 。

         \begin{math}{X}_{i}\end{math} (\begin{math}i \in \{ 0, 1, \dots, n \end{math}\}) 要填充到存储器 \hyperref[regdesc:RSAmemory]{RSA\_X\_MEM} 中的第 \begin{math}i\end{math} 个字对应的地址中，
        但需要注意的是，\begin{math}{Y}_{i}\end{math} (\begin{math}i\in\{ 0, 1, \dots, n \end{math}\}) 并不是要填充到存储器 \hyperref[regdesc:RSAmemory]{RSA\_Z\_MEM} 中的第 \begin{math}i\end{math} 个字对应的地址中，
        而是需要填充到存储器 \hyperref[regdesc:RSAmemory]{RSA\_Z\_MEM} 中的第 \begin{math}n+i\end{math} 个字对应的地址中，
        即存储器 \hyperref[regdesc:RSAmemory]{RSA\_Z\_MEM} 的基地址加上偏移量 \begin{math}4 \times (n+i) \end{math}。

        只需要根据运算子长度，将这两个运算子中有效的数据写入存储器，没有使用到的存储器可以是任意值。

    \item 对寄存器 \hyperref[regdesc:RSAMULTSTARTREG]{RSA\_MULT\_START\_REG} 写入 1。

    \item 等待运算结束。轮询寄存器 \hyperref[regdesc:RSAIDLEREG]{RSA\_IDLE\_REG} 直到读到 1，或者等待 RSA 中断产生。

    \item 从存储器 \hyperref[regdesc:RSAmemory]{RSA\_Z\_MEM} 读出运算结果 \begin{math}Z_{i}\end{math} (\begin{math}i \in \{ 0, 1, \dots, n \end{math}\})。
        \begin{math}\hat{n}\end{math} 为  \begin{math} 2 \times n \end{math} 。

    \item 若中断功能已开启，对寄存器 \hyperref[regdesc:RSACLEARINTERRUPTREG]{RSA\_CLEAR\_INTERRUPT\_REG} 写入 1 以清除中断。

\end{enumerate}

运算结束后，
寄存器 \hyperref[regdesc:RSAMODEREG]{RSA\_MODE\_REG} 中存储的运算子长度信息以及
存储器 \hyperref[regdesc:RSAmemory]{RSA\_X\_MEM} 中的 \begin{math}{X}_{i}\end{math} 都不会变化。
但是，存储器 \hyperref[regdesc:RSAmemory]{RSA\_Z\_MEM} 中的 \begin{math}{Y}_{i}\end{math}
已经被覆盖。所以当需要连续运算时，只需要更新必需的寄存器与存储器即可。

\subsection{加速选项} \label{Label Accelerate options}

ESP32-S2 RSA 提供了两种加速选项，分别为 SEARCH 选项和 CONSTANT\_TIME 选项，专用于加速大数模幂运算。这两个加速选项默认关闭，但可以同时使用。

当两个加速选项均关闭时，求解 \begin{math}Z = X^{Y} \bmod M\end{math} 的时间开销完全由运算子长度决定。然后，当任一加速选项开启时，运算的时间开销还与 \begin{math}Y\end{math} 的 0/1 分布有关。

为了更清楚地说明问题，首先假设 \begin{math}Y \end{math} 的二进制表示为：

\begin{gather*}
Y = ( \widetilde{Y}_{N-1} \widetilde{Y}_{N-2} \cdots \widetilde{Y}_{t+1} \widetilde{Y}_{t} \widetilde{Y}_{t-1} \cdots \widetilde{Y}_{0} )_{2} \\
\end{gather*}
其中，
\begin{itemize}
    \item \begin{math} N \end{math} 代表 \begin{math}Y \end{math} 的长度，
%%%%
    \item \begin{math}\widetilde{Y}_{t}\end{math} 的值为 1，
%%%%
    \item \begin{math}\widetilde{Y}_{N-1}\end{math}, \begin{math}\widetilde{Y}_{N-2}\end{math}, \dots, \begin{math}\widetilde{Y}_{t+1}\end{math} 的值均为 0，
%%%%
    \item 且 \begin{math}\widetilde{Y}_{t-1}\end{math}, \begin{math}\widetilde{Y}_{t-2}\end{math}, \dots, \begin{math}\widetilde{Y}_{0}\end{math} 中包括 \begin{math} m
    \end{math} 个 0，其余 \begin{math}t\end{math}-\begin{math}m\end{math} 全部为 1，即 \begin{math}\widetilde{Y}_{t-1} \widetilde{Y}_{t-2}, \cdots, \widetilde{Y}_{0}\end{math} 的汉明重量 (Hamming weight) 为 \begin{math}t -m\end{math}。
\end{itemize}

此时，当启动加速选项时：
\begin{itemize}
\item SEARCH 选项
    \begin{itemize}
        \item RSA 加速器将忽略所有 \begin{math} \widetilde{Y}_{i} \end{math}（\begin{math} i > \alpha \end{math}）位。其中，加速位置 \begin{math} \alpha \end{math} 可通过 \hyperref[regdesc:RSASEARCHPOSREG]{RSA\_SEARCH\_POS\_REG} 寄存器配置。\begin{math} \alpha \end{math} 的最大值不能超过 \begin{math}N\end{math}-1，否则相当于没有加速；且不建议小于 \begin{math} t \end{math}，否则无法正确求解 \begin{math}Z = X^{Y} \bmod M \end{math}。当设置 \begin{math} \alpha \end{math} 为 \begin{math} t \end{math} 时，加速效果最佳。此时，\begin{math}\widetilde{Y}_{N-1}\end{math}, \begin{math}\widetilde{Y}_{N-2}\end{math}, \dots, \begin{math}\widetilde{Y}_{t+1}\end{math} 中的 0 位将在运算中全部被忽略。
    \end{itemize}
\item CONSTANT\_TIME 选项
    \begin{itemize}
        \item RSA 加速器在运算过程中将简化对 \begin{math} Y \end{math} 中 0 位的处理。因此不难想象，\begin{math} Y \end{math} 中的 0 越多，加速效果越明显。
    \end{itemize}
\end{itemize}

为了直观地展示加速选项带来的加速效果，下面通过一个典型实例加以说明。在 \begin{math}Z = X^{Y} \bmod M\end{math} 中，\begin{math}N\end{math} 等于 3072，\begin{math}Y\end{math} 等于 65537。表 \ref{table:accelerationeffect} 展示了 4 种加速选项组合对应的时间开销。
可以看到，相比于不开启任何加速选项，开启加速选项时的时间开销明显大幅度降低。注意，这里 SEARCH 选项开启时设定 \begin{math} \alpha \end{math} 为 16。

\begin{longtable}{ | l | l | r | r | }
    \caption{加速效果}\label{table:accelerationeffect}
    \\\hline\rowcolor{lightgray}
    SEARCH 选项 & CONSTANT\_TIME 选项& 时间开销  & 提速比例 \\ \hline
    关闭 & 关闭 & 376.405 ms & 0\% \\ \hline
    开启 & 关闭 & 2.260 ms  & 99.41\% \\ \hline
    关闭 & 开启 & 1.203 ms  & 99.68\% \\ \hline
    开启 & 开启 & 1.165 ms  & 99.69\% \\ \hline
\end{longtable}
