\hypertarget{spi}{}
\chapter{SPI 控制器 (SPI)}
\label{mod:spi}

\section{概述}

ESP32-S2 芯片集成了四个 SPI 控制器：SPI0、SPI1、通用 SPI2 (GP-SPI2) 和通用 SPI3 (GP-SPI3)，如图  \ref{Figure:ESP_723_SPI_Block_Diagram} 所示。

\begin{figure}[ht]
    \centering
    \fbox{\includegraphics[width=0.8\textwidth]{\modulefiles/figures/SPI_Block_Diagram_in_ESP_723.eps}}
    \caption{SPI 框图}
    \label{Figure:ESP_723_SPI_Block_Diagram}
\end{figure}

上述四个 SPI 控制器均可与外部 SPI 设备通信：

\begin{itemize}
  \item SPI0 有两条片选线 (CS)，CS0 和 CS1。CS0 用于选择 flash，CS1 用于选择外部 RAM。SPI0 仅被 cache 或 EDMA 用于：
  \begin{itemize}
      \item 从外部 RAM 读取数据。
      \item 将数据写入外部 RAM。
      \item 从外部 flash 读取数据。注意，此功能仅适用于 cache。
  \end{itemize}

  \item SPI1 也有两条片选线，CS0 和 CS1。SPI1 可被 CPU 用于访问外部 flash。

  \item GP-SPI2 为通用 SPI 控制器，具有独立的 DMA 通道。GP-SPI2 既可用作主机又可用作从机。用作主机时，GP-SPI2 具有六条 CS 线，CS0 \verb+~+ CS5。

  \item GP-SPI3 也是通用 SPI 控制器，与 ADC 和 DAC 共享一个 DMA 通道。GP-SPI3 既可用作主机又可用作从机。用作主机时，GP-SPI3 具有三条 CS 线，CS0 \verb+~+ CS2。
\end{itemize}

SPI0 和 SPI1 仅供内部使用，通过仲裁器共享 SPI 信号总线。SPI0 和 SPI1 共享的信号总线，其名称前缀为 SPI：包括 SPICLK、SPICS0 \verb+~+ SPICS1、SPID、SPIQ、SPIWP、SPIHD、SPIIO4 \verb+~+ SPIIO7 和 SPIDQS。GP-SPI2 和 GP-SPI3 的信号总线分别以 FSPI (Fast SPI) 和 SPI3 为前缀。前缀 FSPI 表示推荐使用 GP-SPI2 与外部高速 SPI 设备通信。

SPI/FSPI/SPI3 总线 I/O 可通过下述方式映射到物理 GPIO 管脚：

\begin{itemize}
  \item SPI 总线信号可通过 GPIO 矩阵或 IO MUX 连接到 GPIO 管脚。
  \item FSPI 总线信号可通过 GPIO 矩阵或 IO MUX 连接到 GPIO 管脚。
  \item SPI3 总线信号仅可通过 GPIO 矩阵连接到 GPIO 管脚。
\end{itemize}

更多信息，请参考章节 \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}}。

\section{特性}

SPI0 和 SPI1 控制器主要供内部使用。因此，本节及以下章节将主要介绍 GP-SPI2 和 GP-SPI3 控制器。SPI 信号与 ESP32-S2 信号的映射关系见表 \ref{table:SPI_signals}。


\subsection{GP-SPI2 特性}

\subsubsection{GP-SPI2 用作主机}

GP-SPI2用作主机时，具备以下特性：

\begin{itemize}
    \item \hyperref[phase:cmd]{CMD}、\hyperref[phase:addr]{ADDR} 和 \hyperref[phase:data]{DATA} 阶段使用的比特位可软件配置
    \item 可访问工作在不同数据模式下的 SPI 设备，如 flash、外部 RAM 和 LCD 等。
    \begin{itemize}
      \item 支持 1-bit SPI 模式，信号包括：FSPICLK、FSPICS0 \verb+~+ FSPICS5、FSPID 和/或 FSPIQ。全双工通信模式下，支持 1-bit SPI 模式。半双工通信模式下，支持三线半双工 SPI，仅使用一条数据线 FSPID；也支持四线半双工 SPI，分时使用数据线 FSPID 和 FSPIQ。
      \item 支持 2-bit Dual SPI 模式，信号包括：FSPICLK、FSPICS0 \verb+~+ FSPICS5、FSPID 和 FSPIQ。在该模式下，一次使用两条并行数据线，即 FSPID 和 FSPIQ。

      \item 支持 4-bit Quad SPI 模式，信号包括：FSPICLK、FSPICS0 \verb+~+ FSPICS5、FSPID、FSPIQ、FSPIWP 和 FSPIHD。在该模式下，一次使用四条并行数据线，即 FSPID、FSPIQ、FSPIWP 和 FSPIHD。

      \item 支持 QPI 模式，信号包括：FSPICLK、FSPICS0 \verb+~+ FSPICS5、FSPID、FSPIQ、FSPIWP 和 FSPIHD。

      \item 支持 8-bit Octal SPI 模式，信号包括：FSPICLK、FSPICS0 \verb+~+ FSPICS5、FSPID、FSPIQ、FSPIWP、FSPIHD 和 FSPIIO4 \verb+~+ FSPIIO7。在该模式下，一次使用八条并行数据线，即 FSPID、FSPIQ、FSPIWP、FSPIHD 和 FSPIIO4 \verb+~+ FSPIIO7。

      \item 支持 OPI 模式，信号包括：FSPICLK、FSPICS0 \verb+~+ FSPICS5、FSPID、FSPIQ、FSPIWP、FSPIHD 和 FSPIIO4 \verb+~+ FSPIIO7。
    \end{itemize}

    \item 支持 Moto6800/I8080/并行 RGB 接口 8-bit LCD 驱动
    \item 支持 SPI 时钟频率为：APB 时钟 1 \verb+~+ n 分频，见下文\hyperref[division-formula]{分频公式}
    \item SPI\_CD 管脚电平在 CMD 和 ADDR 阶段可配置
    \item 六个 SPI\_CS 管脚连接六个独立 SPI 从机。主机可以在不同的配置模式下（数据模式、全/半双工、CMD、ADDR、DUMMY 等）与上述六个从机进行连续通信。
    \item CS 建立时间 (Setup Time) 和保持时间 (Hold Time) 可配置
    \item 支持 DMA 分段配置传输 (Segmented Configure Transfer, SCT) 模式和单次传输模式，且两种模式使用各自独立的中断
\end{itemize}

\subsubsection{GP-SPI2 用作从机}

GP-SPI2 用作从机时，具备以下特性：
  \begin{itemize}
      \item 支持下方所述的多种数据模式。在不同数据模式下，\hyperref[phase:cmd]{CMD}、\hyperref[phase:addr]{ADDR} 和 \hyperref[phase:data]{DATA} 阶段使用的比特位见表 \ref{tab:data-modes}。
      \begin{itemize}
            \item 支持 1-bit SPI 模式，信号包括：FSPICLK、FSPICS0、FSPID 和/或 FSPIQ。全双工通信模式下，支持 1-bit SPI 模式。半双工通信模式下，支持三线半双工 SPI，仅使用一条数据线 FSPID ；也支持四线半双工 SPI，分时使用数据线 FSPID 和 FSPIQ。

            \item 支持 2-bit Dual SPI 模式，信号包括：FSPICLK、FSPICS0、FSPID 和 FSPIQ。

            \item 支持 4-bit Quad SPI 模式，信号包括：FSPICLK、FSPICS0、FSPID、FSPIQ、FSPIWP 和 FSPIHD。

            \item 支持 QPI 模式，信号包括：FSPICLK、FSPICS0、FSPID、FSPIQ、FSPIWP 和 FSPIHD。
      \end{itemize}
      \item 支持 SPI 时钟频率最高达 40 MHz
      \item 支持章节 \ref{slave communication format} 所述通信格式
      \item 支持从机连续传输 (Segmented Transfer, ST) 模式和单次传输模式，且两种模式使用各自独立的中断
  \end{itemize}

\subsubsection{GP-SPI2 用作主机或从机}

GP-SPI2 用作主机或从机，具备以下共同特性：
  \begin{itemize}
    \item 数据传输长度可配置：
    \begin{itemize}
        \item 在 CPU 控制模式下：1 \verb+~+ 64 字节
        \item 在 DMA 控制模式下：1 \verb+~+ n 字节
    \end{itemize}

    \item 读写数据位的顺序可配置：先读写最高有效位 (MSB)，或先读写最低有效位 (LSB)

    \item 支持全双工和半双工通信
    \item 支持外部 SPI 设备与下方存储空间进行数据交换：
      \begin{itemize}
        \item 由 CPU 控制与 GP-SPI2 数据 buffer 进行数据交换；
        \item 由 DMA 控制与内部 RAM 进行数据交换；
        \item 由 EDMA 控制与外部 RAM 进行数据交换。
      \end{itemize}

    \item 时钟频率、时钟极性和时钟相位可配置
    \item 支持 SPI 时钟模式 0 \verb+~+ 3
  \end{itemize}

\subsection{GP-SPI3 特性}

\subsubsection{GP-SPI3 用作主机}

GP-SPI3 用作主机时，具有以下特性：
  \begin{itemize}
    \item 支持 SPI 时钟频率为：APB 时钟 1 \verb+~+ n 分频，见下文\hyperref[division-formula]{分频公式}
    \item 支持 1-bit LCD 驱动
    \item SPI\_CD 管脚电平在 CMD 和 ADDR 阶段可配置
    \item 三个 SPI\_CS 管脚连接三个独立 SPI 从机。主机可以在不同的配置模式下（数据模式、全/半双工、CMD、ADDR、DUMMY 等）与上述三个从机进行连续通信。
    \item CS 建立时间和保持时间可配置
    \item 支持 DMA 分段配置传输模式和单次传输模式，且两种模式使用各自独立的中断
  \end{itemize}

\subsubsection{GP-SPI3 用作从机}

GP-SPI3 用作从机时，具备以下特性：
  \begin{itemize}
    \item 支持 SPI 时钟频率最高达 40 MHz
    \item 支持章节 \ref{slave communication format} 所述的通信格式
    \item 支持从机连续传输模式和单次传输模式，且两种模式使用各自独立的中断
  \end{itemize}

\subsubsection{GP-SPI3 用作主机或从机}

GP-SPI3 用作主机或从机，具备以下共同特性：
  \begin{itemize}
    \item 支持 1-bit SPI 模式。 全双工通信模式下，支持 1-bit SPI 模式。半双工通信模式下，支持三线半双工 SPI，仅使用一条数据线 SPI3\_D ；也支持四线半双工 SPI，分时使用数据线 SPI3\_D 和 SPI3\_Q。
    \item 数据传输长度可配置：
    \begin{itemize}
        \item 在 CPU 控制模式下：1 \verb+~+ 64 字节
        \item 在 DMA 控制模式下：1 \verb+~+ n 字节
    \end{itemize}
    \item 读写数据位的顺序可配置：先读写最高有效位 (MSB)，或者先读写最低有效位 (LSB)
    \item 支持全双工和半双工通信
    \item 支持外部 SPI 设备与下方存储空间进行数据交换：
      \begin{itemize}
        \item 由 CPU 控制与 GP-SPI3 数据 buffer 进行数据交换
        \item 由 DMA 控制与内部 RAM 进行数据交换
      \end{itemize}

    \item 时钟频率、时钟极性和时钟相位可配置
    \item 支持 SPI 时钟模式 0 \verb+~+ 3
  \end{itemize}

\subsection{SPI 中断特性}

\begin{itemize}
    \item 支持 SPI 中断
    \item 支持 SPI DMA 中断
\end{itemize}

GP-SPI3 与 GP-SPI2 功能基本相同。GP-SPI2 功能描述见第 \ref{GP-SPI2 master mode} 节和第 \ref{GP-SPI2 slave mode} 节。GP-SPI2 和 GP-SPI3 的功能差异见第 \ref{Subsection:Differences Between GP-SPI2 and GP-SPI3} 节。
GP-SPI2 支持以下设置：

\begin{itemize}
    \item 配置寄存器 \hyperref[regdesc:SPIUSERREG]{SPI\_USER\_REG} 中 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN} 位，选择全双工通信模式或半双工通信模式：
    \begin{itemize}
        \item 0：选择半双工模式。
        \item 1：选择全双工模式。
    \end{itemize}
    \item 配置寄存器 \hyperref[regdesc:SPISLAVEREG]{SPI\_SLAVE\_REG} 中 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE} 位，选择主机模式或从机模式。
    \begin{itemize}
        \item 0：选择主机模式。
        \item 1：选择从机模式。
    \end{itemize}
    \item 配置寄存器 \hyperref[regdesc:SPICTRLREG]{SPI\_CTRL\_REG} 中 \hyperref[fielddesc:SPIRDBITORDER]{SPI\_RD\_BIT\_ORDER} 和 \hyperref[fielddesc:SPIWRBITORDER]{SPI\_WR\_BIT\_ORDER} 位，分别设置读/写数据位的顺序。
    \begin{itemize}
        \item 0：先读/写 LSB。
        \item 1：先读/写 MSB。
    \end{itemize}
\end{itemize}

GP-SPI2 主机模式支持单次传输和 DMA 分段配置传输。相关寄存器配置好后，每个单次传输均需要由 ESP32-S2 CPU 触发。而 DMA 分段配置传输仅需 CPU 触发一次，即可完成多次 SPI 传输。DMA 分段配置传输模式详情见章节 \ref{GP-SPI DMA seg-conf-trans}。

GP-SPI2 从机模式同样也支持单次传输和从机连续传输。在 SPI 主机开始分段传输之前，GP-SPI2 应配置为连续传输模式。正确接收 En\_SEG\_TRANS 命令后，连续传输结束。详细信息见章节 \ref{GP-SPI2 slave seg-trans}。

GP-SPI2 用于从机半双工模式时，SPI 主机通过发送从机支持的命令，见表 \ref{table:SPI_slave_CMD_SPI} 和表 \ref{table:SPI_slave_CMD_QPI}，与从机进行通信。

\section{GP-SPI2 主机模式}\label{GP-SPI2 master mode}
GP-SPI2 可配置成主机模式。在主机模式下，GP-SPI2 借助状态机和寄存器，可实现多种功能。更多信息见以下章节。
\begin{itemize}
    \item 章节 \ref{GP-SPI2 state machine}：状态机工作流程
    \item 章节 \ref{GP-SPI2 state and register configuration}：寄存器配置规则
    \item 章节 \ref{GP-SPI2 1 bit} 至章节 \ref{GP-SPI DMA seg-conf-trans application}：具体描述 GP-SPI2 功能及其典型应用
    \item 章节 \ref{GP-SPI2 CS timing}：配置 CS 建立时间和保持时间
    \item 章节 \ref{Subsection:GP-SPI2 CLK}：配置 FSPICLK 的频率、极性和相位
\end{itemize}
\begin{tiplisting}
\vspace{-2em}
\begin{itemize}
    \item 数据以字节为单位进行传输，否则多余的位将丢失。此处多余的位表示总位长对 8 取模的结果。
    \item 如果需要传输非字节比特，推荐使用 CMD 状态或 ADDR 状态来实现。
\end{itemize}
\end{tiplisting}

\subsection{主机模式状态机} \label{GP-SPI2 state machine}
GP-SPI2 主要用于访问 1/2/4/8-bit SPI 设备，如 flash、外部 RAM 和 LCD 等。因此，GP-SPI2 状态的命名规则应与 flash、外部 RAM 和 LCD 的时序名称保持一致。GP-SPI2 状态流程见图  \ref{Figure:SPI_master_mode_flow_chart}，通过图中各个状态，可以实现主机模式下 GP-SPI2 所有功能。


\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{\modulefiles/figures/SPI_master_mode_flow_chart1}
    \caption{GP-SPI2 主机模式流程图}
    \label{Figure:SPI_master_mode_flow_chart}
\end{figure}

GP-SPI2 各个状态描述如下：
\begin{enumerate}
    \item IDLE：GP-SPI2 未处于工作状态或处于从机模式。
    \item CONF：仅用于 DMA 分段配置传输。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和 \hyperref[fielddesc:SPIUSRCONF]{SPI\_USR\_CONF} 使能此状态。不使能 CONF 状态，则可实现单次传输模式。
    \item PREP：准备 SPI 传输，并控制 SPI CS 建立时间。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和 \hyperref[fielddesc:SPICSSETUP]{SPI\_CS\_SETUP} 使能此状态。
    \item \label{phase:cmd} CMD：发送命令序列。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和 \hyperref[fielddesc:SPIUSRCOMMAND]{SPI\_USR\_COMMAND} 使能此状态。
    \item \label{phase:addr} ADDR：发送地址序列。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和 \hyperref[fielddesc:SPIUSRADDR]{SPI\_USR\_ADDR} 使能此状态。
    \item DUMMY（等待周期）：发送 DUMMY 序列。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和 \hyperref[fielddesc:SPIUSRDUMMY]{SPI\_USR\_DUMMY} 使能此状态。
    \item DATA：传输数据阶段，包括：\label{phase:data}
    \begin{itemize}
        \item DOUT：发送数据。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和 \hyperref[fielddesc:SPIUSRMOSI]{SPI\_USR\_MOSI} 使能此状态。
        \item DIN：接收数据。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 和 \hyperref[fielddesc:SPIUSRMISO]{SPI\_USR\_MISO} 使能此状态。
    \end{itemize}
    \item DONE：控制 SPI CS 保持时间。置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 使能此状态。
\end{enumerate}

如图 \ref{Figure:SPI_master_mode_flow_chart} 所示，GP-SPI2 状态机使用计数器 gpc[22:0] 控制每个状态的循环周期。CONF、PREP、CMD、ADDR、DUMMY、DOUT 和 DIN 各状态可单独使能或禁用，也可以单独配置其周期长度。通过使能不同状态，配置各个状态的周期，即可实现对 GP-SPI2 状态机的灵活控制。

例如，当 GP-SPI2 读取数据时，且希望实现：
\begin{itemize}
    \item CMD 为 4-bit 模式；
    \item ADDR 为 2-bit 模式；
    \item DUMMY 为 \textcolor{red}{\it{n}} 个时钟周期；
    \item DIN 为 8-bit 模式。
\end{itemize}
则具体的寄存器配置如下：

\begin{enumerate}
    \item 配置 CMD 状态相关寄存器。
        \begin{itemize}
            \item 配置 \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE} 为需要的命令值；
            \item 配置 \hyperref[fielddesc:SPIUSRCOMMANDBITLEN]{SPI\_USR\_COMMAND\_BITLEN} 为需要的命令位长，其中，\hyperref[fielddesc:SPIUSRCOMMANDBITLEN]{SPI\_USR\_COMMAND\_BITLEN} = 需要的位长 - 1；
            \item 置位 \hyperref[fielddesc:SPIFCMDQUAD]{SPI\_FCMD\_QUAD} 和 \hyperref[fielddesc:SPIUSRCOMMAND]{SPI\_USR\_COMMAND}；
            \item 清零 \hyperref[fielddesc:SPIFCMDDUAL]{SPI\_FCMD\_DUAL} 和 \hyperref[fielddesc:SPIFCMDOCT]{SPI\_FCMD\_OCT}。
        \end{itemize}

    \item 配置 ADDR 状态相关寄存器。
        \begin{itemize}
            \item 配置 \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE} 为需要的地址值；
            \item 配置 \hyperref[fielddesc:SPIUSRADDRBITLEN]{SPI\_USR\_ADDR\_BITLEN} 为需要的地址值位长，其中，\hyperref[fielddesc:SPIUSRADDRBITLEN]{SPI\_USR\_ADDR\_BITLEN} = 需要的位长 - 1；
            \item 置位 \hyperref[fielddesc:SPIUSRADDR]{SPI\_USR\_ADDR} 和 \hyperref[fielddesc:SPIFADDRDUAL]{SPI\_FADDR\_DUAL}；
            \item 清零 \hyperref[fielddesc:SPIFADDRQUAD]{SPI\_FADDR\_QUAD} 和 \hyperref[fielddesc:SPIFADDROCT]{SPI\_FADDR\_OCT}；
        \end{itemize}

    \item 配置 DUMMY 状态相关寄存器。
        \begin{itemize}
            \item 配置 \hyperref[fielddesc:SPIUSRDUMMYCYCLELEN]{SPI\_USR\_DUMMY\_CYCLELEN} 为需要的 DUMMY 周期数，其中，\hyperref[fielddesc:SPIUSRDUMMYCYCLELEN]{SPI\_USR\_DUMMY\_CYCLELEN} = 需要的 DUMMY 周期数 (\textcolor{red}{\it{n}}) - 1；
            \item 置位 \hyperref[fielddesc:SPIUSRDUMMY]{SPI\_USR\_DUMMY}。
        \end{itemize}

    \item 配置 DIN 状态相关寄存器。
        \begin{itemize}
            \item 配置 \hyperref[fielddesc:SPIUSRMISODBITLEN]{SPI\_USR\_MISO\_DBITLEN} 为所需的读数据位长，其中，\hyperref[fielddesc:SPIUSRMISODBITLEN]{SPI\_USR\_MISO\_DBITLEN} = 需要的位长 - 1；
            \item 置位 \hyperref[fielddesc:SPIFREADOCT]{SPI\_FREAD\_OCT} 和 \hyperref[fielddesc:SPIUSRMISO]{SPI\_USR\_MISO}；
            \item 清零 \hyperref[fielddesc:SPIFREADDUAL]{SPI\_FREAD\_DUAL} 和 \hyperref[fielddesc:SPIFREADQUAD]{SPI\_FREAD\_QUAD}；
            \item 在 DMA 控制模式下，配置 GP-SPI2 的 DMA；CPU 控制模式下，不需要任何操作。
        \end{itemize}

    \item 清零 \hyperref[fielddesc:SPIUSRMOSI]{SPI\_USR\_MOSI}；

    \item 设置 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 使能 GP-SPI2 传输。
\end{enumerate}

写数据时（DOUT 状态），则应配置 \hyperref[fielddesc:SPIUSRMOSI]{SPI\_USR\_MOSI} 和 \hyperref[fielddesc:SPIUSRMOSIDBITLEN]{SPI\_USR\_MOSI\_DBITLEN}，同时清零 \hyperref[fielddesc:SPIUSRMISO]{SPI\_USR\_MISO}。
输出数据位长 = \hyperref[fielddesc:SPIUSRMOSIDBITLEN]{SPI\_USR\_MOSI\_DBITLEN} + 1。在 CPU 控制模式下，输出数据应在 GP-SPI2 的 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG} 中配置；在 DMA 控制模式下，则在 DMA TX buffer 中配置。数据字节顺序从 LSB（字节 0）递增到 MSB。

应特别注意 \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE} 中的命令值和 \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE} 中的地址值。

命令值的配置如下：
\begin{itemize}

\item 如果 \hyperref[fielddesc:SPIUSRCOMMANDBITLEN]{SPI\_USR\_COMMAND\_BITLEN} < 8，则命令值写入 \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE}[7:0]，其发送顺序如下：
\begin{itemize}
    \item 如果清零 \hyperref[fielddesc:SPIWRBITORDER]{SPI\_WR\_BIT\_ORDER}，则先发送 \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE}[7:0] 的低位数据，即 \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE}[\hyperref[fielddesc:SPIUSRCOMMANDBITLEN]{SPI\_USR\_COMMAND\_BITLEN}:0]。
    \item 如果置位 \hyperref[fielddesc:SPIWRBITORDER]{SPI\_WR\_BIT\_ORDER}，则先发送  \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE}[7:0] 的高位数据，即 \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE}[7: 7-\hyperref[fielddesc:SPIUSRCOMMANDBITLEN]{SPI\_USR\_COMMAND\_BITLEN}]。
\end{itemize}

\item  如果 7 < \hyperref[fielddesc:SPIUSRCOMMANDBITLEN]{SPI\_USR\_COMMAND\_BITLEN} < 16，则命令值写入 \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE}[15:0]，其发送顺序如下：
\begin{itemize}
    \item 如果清零 \hyperref[fielddesc:SPIWRBITORDER]{SPI\_WR\_BIT\_ORDER}，则先发送 \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE}[7:0]，然后发送 \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE}[15:8] 的低位数据，即 \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE}[\hyperref[fielddesc:SPIUSRCOMMANDBITLEN]{SPI\_USR\_COMMAND\_BITLEN}:8]。
    \item 如果置位 \hyperref[fielddesc:SPIWRBITORDER]{SPI\_WR\_BIT\_ORDER}，则先发送 \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE}[7:0]，然后发送 \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE}[15:8] 的高位数据，即 \hyperref[fielddesc:SPIUSRCOMMANDVALUE]{SPI\_USR\_COMMAND\_VALUE}[15: 15-\hyperref[fielddesc:SPIUSRCOMMANDBITLEN]{SPI\_USR\_COMMAND\_BITLEN}]。
\end{itemize}
\end{itemize}

地址值配置如下：
\begin{itemize}
\item 如果 \hyperref[fielddesc:SPIUSRADDRBITLEN]{SPI\_USR\_ADDR\_BITLEN} < 8，则地址值写入 \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE}[7:0]，其发送顺序如下：
\begin{itemize}
    \item 如果清零 \hyperref[fielddesc:SPIWRBITORDER]{SPI\_WR\_BIT\_ORDER}，则先发送 \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE}[7:0] 的低位数据，即 \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE}[\hyperref[fielddesc:SPIUSRADDRBITLEN]{SPI\_USR\_ADDR\_BITLEN}:0]。
    \item 如果置位 \hyperref[fielddesc:SPIWRBITORDER]{SPI\_WR\_BIT\_ORDER}，则先发送  \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE}[7:0] 的高位数据，即 \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE}[7: 7-\hyperref[fielddesc:SPIUSRADDRBITLEN]{SPI\_USR\_ADDR\_BITLEN}]。
\end{itemize}

\item  如果 7 < \hyperref[fielddesc:SPIUSRADDRBITLEN]{SPI\_USR\_ADDR\_BITLEN} < 16，则地址值写入 \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE}[15:0]，其发送顺序如下：
\begin{itemize}
    \item 如果清零 \hyperref[fielddesc:SPIWRBITORDER]{SPI\_WR\_BIT\_ORDER}，则先发送 \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE}[7:0]，然后发送 \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE}[15:8] 的低位数据，即 \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE}[\hyperref[fielddesc:SPIUSRADDRBITLEN]{SPI\_USR\_ADDR\_BITLEN}:8]。
    \item 如果置位 \hyperref[fielddesc:SPIWRBITORDER]{SPI\_WR\_BIT\_ORDER}，则先发送 \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE}[7:0]，然后发送 \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE}[15:8] 的高位数据，即 \hyperref[fielddesc:SPIUSRADDRVALUE]{SPI\_USR\_ADDR\_VALUE}[15: 15-\hyperref[fielddesc:SPIUSRADDRBITLEN]{SPI\_USR\_ADDR\_BITLEN}]。
\end{itemize}
\end{itemize}

GP-SPI2 的所有状态均可独立配置为 1/2/4/8 位模式。其它类型应用程序的寄存器配置可参考上述示例、表 \ref{table:SPI_USR1} 和表 \ref{table:SPI_USR2}。

\subsection{主机全双工通信（仅支持 1-bit 模式）}\label{GP-SPI2 1 bit}

GP-SPI2 支持全双工通信模式，电子系统中应用最广泛的模式之一。在全双工模式下，SPI 主机提供 CLK 和 CS 信号，然后与从机按照下述方式使用 1-bit 模式同时交换数据：
\begin{itemize}
    \item MOSI（FSPID，发送）
    \item MISO（FSPIQ，接收）
\end{itemize}
全双工通信框图见图  \ref{Figure:GP-SPI2_full_duplex}。

\begin{figure}[ht]
    \centering
    \includegraphics[width=0.4\textwidth]{\modulefiles/figures/GP_SPI2_FD_723}
    \caption{GP-SPI2 主机使用全双工模式与 SPI 从机通信框图}
    \label{Figure:GP-SPI2_full_duplex}
\end{figure}


全双工通信下，CMD、ADDR、DUMMY、DOUT 和 DIN 各个状态的具体行为可配置。通常，全双工模式跳过 CMD、ADDR、DUMMY 状态。传输数据的位长可在 \hyperref[fielddesc:SPIUSRMOSIDBITLEN]{SPI\_USR\_MOSI\_DBITLEN} 和 \hyperref[fielddesc:SPIUSRMISODBITLEN]{SPI\_USR\_MISO\_DBITLEN} 中配置。通信中实际使用的位长 = \hyperref[fielddesc:SPIUSRMOSIDBITLEN]{SPI\_USR\_MOSI\_DBITLEN}/\hyperref[fielddesc:SPIUSRMISODBITLEN]{SPI\_USR\_MISO\_DBITLEN} + 1。

按照以下操作步骤，开始数据传输：
\begin{itemize}
    \item 置位 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN} 并清除 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE} 位，见章节  \ref{section:GP-SPI_Interfaces}。
    \item 置位寄存器 \hyperref[regdesc:SPICMDREG]{SPI\_CMD\_REG} 中 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 位，开始数据传输。
\end{itemize}

CPU 控制模式下的数据传输，其读/写数据字节顺序可在寄存器 \hyperref[regdesc:SPIUSERREG]{SPI\_USER\_REG} 的 \hyperref[fielddesc:SPIRDBYTEORDER]{SPI\_RD\_BYTE\_ORDER} 位和 \hyperref[fielddesc:SPIWRBYTEORDER]{SPI\_WR\_BYTE\_ORDER} 位中配置。DMA 控制模式下的数据传输，其寄存器配置见章节 \ref{Subsection:Usage of GP-SPI2 DMA controlled mode}。

\subsection{主机半双工通信（支持 1/2/4/8-bit 模式）}
\label{GP-SPI2 1, 2, 4, 8 bits}

半双工通信也是 SPI 常用的模式之一。在半双工模式下，SPI 主机发送 CLK 和 CS 信号。在同一时刻，SPI 主机或从机只能有一个可以发送数据，另一个接收数据。SPI 半双工通信的通用格式为 CMD + [ADDR +] [DUMMY +] [DOUT or DIN]，其中，ADDR、DUMMY、DOUT 和 DIN 状态非必选，可单独禁用或启用。

如章节 \ref{GP-SPI2 state and register configuration} 所述，CMD、ADDR、DUMMY、DOUT 和 DIN 各个状态的周期、具体值和并行总线位模式等可独立配置。寄存器配置见表 \ref{table:SPI_USR1} 和表 \ref{table:SPI_USR2}。

半双工 GP-SPI2 的详细属性如下：

\begin{enumerate}
    \item CMD：0 \verb+~+ 16 位，主机发送，从机接收 (MOSI)。
    \item ADDR：0 \verb+~+ 32 位，主机发送，从机接收。
    \item DUMMY：0 \verb+~+ 256 个 FSPICLK 周期，主机发送，从机接收。
    \item DOUT：CPU 控制模式下为 0 \verb+~+ 576 位（72 字节）；DMA 控制模式下为 0 \verb+~+ 8 Mbit，主机发送，从机接收。
    \item DIN：CPU 控制模式下为 0 \verb+~+ 576 位（72 字节）；DMA 控制模式下为 0 \verb+~+ 8 Mbit，从机发送，主机接收 (MISO)。
\end{enumerate}

具体的寄存器配置如下：
\begin{enumerate}
    \item 按照表 \ref{table:SPI_USR1} 和表 \ref{table:SPI_USR2} 所示，配置 GP-SPI2 寄存器；

    \item 按照章节 \ref{GP-SPI2 CS timing} 所述，配置 CS 建立时间和保持时间；

    \item 按照章节 \ref{Subsection:GP-SPI2 CLK} 所述，配置 FSPICLK 属性；

    \item 根据具体使用的模式，准备数据：
    \begin{itemize}
        \item 如果选择 CPU 控制下的 MOSI 模式，则在寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG} 中准备数据；
        \item 如果选择 DMA 控制下的模式，则需要配置 DMA TX/RX 链表，见章节 \ref{Subsection:Usage of GP-SPI2 DMA controlled mode}；
    \end{itemize}
    \item 配置 FSPI 总线信号线路和中断使能信号。\label{enable-interrupt-signal1}

    \item 等待 SPI 从机准备完成；

    \item 置位寄存器 \hyperref[regdesc:SPICMDREG]{SPI\_CMD\_REG} 中 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 位，开始数据传输，然后等待步骤 \ref{enable-interrupt-signal1} 中配置的中断。
\end{enumerate}

GP-SPI2 支持的最高频率为 $f_{\textrm{apb}}$，其中 FSPICLK 可通过分频获得，见章节 \ref{Subsection:GP-SPI2 CLK}。

GP-SPI2 同时提供用于输入输出数据的时序调整模块，实现每条 I/O 线路延迟 $T_{\textrm{apb}}/2$ 的整数倍。时序模块的具体配置见章节 \ref{subsubsection:timing-compensation}。通过在 GPIO 矩阵中设置 DUMMY 周期和延迟，GP-SPI2 可满足时序要求。

支持 1/2/4/8 位双倍数据速率模式 (Double Transfer Rate, DTR)，即在一个 SPI 时钟周期内，在上升沿和下降沿均收发数据：
\begin{itemize}
    \item 置位 \hyperref[fielddesc:SPICMDDTREN]{SPI\_CMD\_DTR\_EN}，则 CMD 值将在 DTR 模式下发送；否则，CMD 阶段将处于单倍数据速率模式 (Single Transfer Rate, STR)。

    \item 置位 \hyperref[fielddesc:SPIADDRDTREN]{SPI\_ADDR\_DTR\_EN}，则 ADDR 值将在 DTR 模式下发送；否则，ADDR 阶段将处于 STR 模式。

    \item 置位 \hyperref[fielddesc:SPIDATADTREN]{SPI\_DATA\_DTR\_EN}，则 DIN 阶段的输入数据和 DOUT 阶段的输出数据将在 DTR 模式下发送；否则，数据将在 STR 模式下发送。

\end{itemize}

\hyperref[fielddesc:SPICMDDTREN]{SPI\_CMD\_DTR\_EN}、\hyperref[fielddesc:SPIADDRDTREN]{SPI\_ADDR\_DTR\_EN} 和 \hyperref[fielddesc:SPIDATADTREN]{SPI\_DATA\_DTR\_EN} 等控制位可分别单独配置，即 CMD 可配置成 STR 模式，而 ADDR 和 DOUT/DIN 可配置成 DTR 模式。

GP-SPI2 只能输出 FSPIDQS 信号，但无法接收该信号。因此，仅支持 DUMMY 固定的 flash 或者外部 RAM，即在一个读操作中，DUMMY 周期固定。

\subsection{DMA 控制下的分段配置传输} \label{GP-SPI DMA seg-conf-trans}

SPI 传输由 GP-SPI2 寄存器控制，需要 CPU 进行配置和触发。为了降低对 CPU 的占用率同时提高 GP-SPI2 的效率，GP-SPI2 在 DMA 控制模式下提供分段配置传输功能。注意 CPU 控制的主机模式不支持分段配置传输。分段配置传输允许 GP-SPI2 在 CPU 仅触发一次的情况下，即可按照配置的模式完成多次数据传输，具体的工作流程见图 \ref{Figure:SPI_master_DMA_seg_trans}。

\begin{figure}[ht]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/seg_trans_scheme}}
    \caption{DMA 控制模式下分段配置传输}
    \label{Figure:SPI_master_DMA_seg_trans}
\end{figure}


如图 \ref{Figure:SPI_master_DMA_seg_trans} 所示，在分段配置传输模式中的某个单次 SPI 传输 (SCT\textcolor{red}{\it{i}}) 开始前，GP-SPI2 硬件可在 CONF 阶段将 GP-SPI2 寄存器重新按照 Conf\_buf\textcolor{red}{\it{i}} 定义的内容进行配置。

建议每个 SCT 的 CONF 阶段均有独立的 DMA CONF 链表和 CONF buffer。串行 DMA TX 链表将所有的 CONF buffer 和 TX data buffer 链接起来，因此可以独立控制每个 SCT 中的 FSPI 总线行为。

同时，每个 SCT 中，GP-SPI2 的状态、相关值和 FSPI 总线周期长、DMA 行为等，均可独立配置。全部的 SCT 传输完成后，触发 GP-SPI2 中断 \hyperref[fielddesc:SPIDMASEGTRANSDONE]{SPI\_DMA\_SEG\_TRANS\_DONE}。

如图 \ref{Figure:SPI_master_DMA_seg_trans} 所示，在 SCT\textcolor{red}{\it i} 中，其传输模式可配置为主机全双工通信模式；在 SCT\textcolor{red}{\it j} 中，其传输模式可配置为主机半双工 MISO 模式；而在 SCT\textcolor{red}{\it k} 中，其传输模式可配置为主机半双工 MOSI 模式。\textcolor{red}{\it i}、\textcolor{red}{\it j}、\textcolor{red}{\it k} 为整数变量，表示任意一个 SCT 编号。

此外，在每个 SCT 中，GP-SPI2 的 CS 建立时间和保持时间可独立编程，更多配置信息见章节 \ref{GP-SPI2 CS timing}。CONF 阶段中，CS 高电平时长等于寄存器 \hyperref[regdesc:SPICMDREG]{SPI\_CMD\_REG} 中 \hyperref[fielddesc:SPICONFBITLEN]{SPI\_CONF\_BITLEN}[22:0] 与寄存器 \hyperref[regdesc:SPISLVWRBUFDLENREG]{SPI\_SLV\_WRBUF\_DLEN\_REG} 中 \hyperref[fielddesc:SPICONFBASEBITLEN]{SPI\_CONF\_BASE\_BITLEN}[6:0] 的和。$f_{\textrm{apb}}$ 为 80 MHz 且 FSPICLK 等于 APB\_CLK 1 分频时，CONF
阶段的 CS 高电平时长可配置为 2 $us$ \verb+~+ 0.2 $s$。如果 FSPICLK 频率较慢（n 分频，n > 1），CS 高电平时长可以持续更久。


在分段配置传输模式下，具体的寄存器配置流程示例如下：
 \begin{enumerate}
   \item 为每个 SCT 传输准备 DMA CONF buffer 描述符和 TX data 描述符（可选）。把 CONF buffer 描述符和几次传输需要的 TX buffer 链接成一个链表。

   \item 同样，为每个 SCT 传输准备 RX buffer 描述符，并链接成一个链表。

   \item 在分段配置传输开始之前，为每个传输配置所有 CONF buffer、TX buffer 和 RX buffer。

   \item 配置 \hyperref[fielddesc:SPIOUTLINKADDR]{SPI\_OUTLINK\_ADDR} 指向 CONF Head 和 TX buffer 描述符链表的地址，之后置位寄存器 \hyperref[regdesc:SPIDMAOUTLINKREG]{SPI\_DMA\_OUT\_LINK\_REG} 中 \hyperref[fielddesc:SPIOUTLINKSTART]{SPI\_OUTLINK\_START} 位，启动 TX DMA。

   \item 清零寄存器 \hyperref[regdesc:SPIDMACONFREG]{SPI\_DMA\_CONF\_REG} 中 \hyperref[fielddesc:SPIRXEOFEN]{SPI\_RX\_EOF\_EN} 位。配置 \hyperref[fielddesc:SPIINLINKSTART]{SPI\_INLINK\_ADDR} 指向 RX buffer 描述符链表起始地址，之后置位寄存器 \hyperref[regdesc:SPIDMAINLINKREG]{SPI\_DMA\_IN\_LINK\_REG} 中 \hyperref[fielddesc:SPIINLINKSTART]{SPI\_INLINK\_START} 位，启动 RX DMA。

   \item 置位寄存器 \hyperref[regdesc:SPISLVRDBYTEREG]{SPI\_SLV\_RD\_BYTE\_REG} 中 \hyperref[fielddesc:SPIUSRCONF]{SPI\_USR\_CONF} 位使能 CONF 阶段。

   \item 置位寄存器 \hyperref[regdesc:SPISLAVEREG]{SPI\_SLAVE\_REG} 中 \hyperref[fielddesc:SPIINTDMASEGTRANSEN]{SPI\_INT\_DMA\_SEG\_TRANS\_EN} 位，使能 \hyperref[fielddesc:SPIDMASEGTRANSDONE]{SPI\_DMA\_SEG\_TRANS\_DONE} 中断。如需配置其它中断，请参考章节 \ref{Section:SPI_Interrupt}。

   \item 等待所有 SCT 从机做好传输准备。

   \item 置位寄存器 \hyperref[regdesc:SPICMDREG]{SPI\_CMD\_REG} 中 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 位，开始 DMA 主机模式分段配置传输。

   \item 等待 \hyperref[fielddesc:SPIDMASEGTRANSDONE]{SPI\_DMA\_SEG\_TRANS\_DONE} 中断，即 DMA 分段配置传输结束，数据已存储至相应内存。
 \end{enumerate}

为节省时间和芯片资源，在 CONF 阶段仅将较上次传输有改动的寄存器配置成新值，其它寄存器配置保持不变。DMA CONF buffer\textcolor{red}{\it i} 中第一个字，即 SPI\_BIT\_MAP\_WORD，记录在 SCT\textcolor{red}{\it i} 传输过程中，每个 SPI 寄存器是否有改动。

        \item 行消隐周期等于 CONF 阶段 CS 高电平时长、CS 建立时间和保持时间的总和，见章节 \ref{GP-SPI DMA seg-conf-trans} 和章节 \ref{GP-SPI2 CS timing}。
    \end{itemize}

GP-SPI2 可自动输出图 \ref{Figure:GP-SPI2_LCD_serial_RGB_frame_structure} 和图 \ref{Figure:GP-SPI2_LCD_serial_RGB_timing} 所示的帧数据，在很大程度上降低了控制 SPI 传输所需的 CPU 时间。具体的配置流程如下：
\begin{enumerate}
    \item 置位寄存器 \hyperref[regdesc:SPILCDCTRLREG]{SPI\_LCD\_CTRL\_REG} 中 \hyperref[fielddesc:SPILCDMODEEN]{SPI\_LCD\_MODE\_EN}，使能并行 RGB LCD 模式；

    \item 根据应用程序，配置帧控制相关的寄存器，见图  \ref{Figure:GP-SPI2_LCD_serial_RGB_frame_structure} 和图  \ref{Figure:GP-SPI2_LCD_serial_RGB_timing}。例如，如果希望设置 \textcolor{red}{\it{n}} 个有效行，则将寄存器 \hyperref[fielddesc:SPILCDVAHEIGHT]{SPI\_LCD\_VA\_HEIGHT}[9:0] 配置成 \textcolor{red}{\it{n}} 即可；

    \item 根据需要配置 CS 建立时间和保持时间；

    \item 清除 \hyperref[fielddesc:SPIUSRCOMMAND]{SPI\_USR\_COMMAND}、\hyperref[fielddesc:SPIUSRADDR]{SPI\_USR\_ADDR}、\hyperref[fielddesc:SPIUSRDUMMY]{SPI\_USR\_DUMMY} 和 \hyperref[fielddesc:SPIUSRMISO]{SPI\_USR\_MISO} 位；

    \item 置位 \hyperref[fielddesc:SPIFWRITEOCT]{SPI\_FWRITE\_OCT} 和 \hyperref[fielddesc:SPIUSRMOSI]{SPI\_USR\_MOSI}，并将 \hyperref[fielddesc:SPIUSRMOSIDBITLEN]{SPI\_USR\_MOSI\_DBITLEN} 配置为 \hyperref[fielddesc:SPILCDHAWIDTH]{SPI\_LCD\_HA\_WIDTH}[11:0] * 8 -1；

    \item 配置 DMA TX 链表并在 TX buffer 中准备数据。此应用程序中建议使用 ring-buffer 或 ping-pang buffer，以减少传输开始前准备 buffer 花费的时间；

    \item 设置 I/O 线路。置位 \hyperref[fielddesc:SPIINTTRANSDONEEN]{SPI\_INT\_TRANS\_DONE\_EN} 或其它中断的使能位；

    \item 等待 LCD 从机做好传输准备；

    \item 置位 \hyperref[fielddesc:SPIUSR]{SPI\_USR} 开始数据传输。等待 \hyperref[fielddesc:SPITRANSDONE]{SPI\_TRANS\_DONE} 中断。
\end{enumerate}


\subsection{CS 建立时间和保持时间控制} \label{GP-SPI2 CS timing}

SPI CS 建立时间和保持时间对于满足各种 SPI 设备（如 flash 或 PSRAM）的时序要求非常重要。CS 建立时间为 CS 有效边沿至 SPI CLK 第一个锁存边沿的时间。模式 0 和模式 3 的第一锁存边沿为上升沿，模式 2 和模式 4 的第一锁存边沿为下降沿。CS 保持时间为最后一个锁存边沿到 CS 无效边沿之间的时间。

用户可通过设置寄存器 \hyperref[regdesc:SPIUSERREG]{SPI\_USER\_REG} 中 \hyperref[fielddesc:SPICSSETUP]{SPI\_CS\_SETUP} 位，以及寄存器 \hyperref[regdesc:SPICTRL2REG]{SPI\_CTRL2\_REG} 中 \hyperref[fielddesc:SPICSSETUPTIME]{SPI\_CS\_SETUP\_TIME} 位，来配置 CS 建立时间。
    \begin{itemize}
        \item \hyperref[fielddesc:SPICSSETUP]{SPI\_CS\_SETUP} = 0，则 SPI CS 建立时间为 0.5 x $T\_SPI\_CLK$；这里的 $T\_SPI\_CLK$ 指 SPI\_CLK 时钟周期。
        \item \hyperref[fielddesc:SPICSSETUP]{SPI\_CS\_SETUP} = 1，则 SPI CS 建立时间为 (\hyperref[fielddesc:SPICSSETUPTIME]{SPI\_CS\_SETUP\_TIME} + 1.5) x $T\_SPI\_CLK$。
    \end{itemize}

用户可通过配置寄存器 \hyperref[regdesc:SPIUSERREG]{SPI\_USER\_REG} 中 \hyperref[fielddesc:SPICSHOLD]{SPI\_CS\_HOLD} 位，以及寄存器 \hyperref[regdesc:SPICTRL2REG]{SPI\_CTRL2\_REG} 中 \hyperref[fielddesc:SPICSHOLDTIME]{SPI\_CS\_HOLD\_TIME} 位，来配置 CS 保持时间。
    \begin{itemize}
        \item \hyperref[fielddesc:SPICSHOLD]{SPI\_CS\_HOLD} = 0，则 SPI CS 保持时间为 0.5 x $T\_SPI\_CLK$；
        \item \hyperref[fielddesc:SPICSHOLD]{SPI\_CS\_HOLD} = 1，则 SPI CS 保持时间为 (\hyperref[fielddesc:SPICSHOLDTIME]{SPI\_CS\_HOLD\_TIME} + 1.5) x $T\_SPI\_CLK$。
    \end{itemize}

\section{GP-SPI2 从机模式} \label{GP-SPI2 slave mode}

GP-SPI2 可用作从机与另一 SPI 主机进行通信。用作从机时，GP-SPI2 支持 1-bit SPI、2-bit Dual SPI、4-bit Quad SPI 和 QPI 模式，同时支持章节 \ref{slave communication format} 所述的通信格式。置位寄存器 \hyperref[regdesc:SPISLAVEREG]{SPI\_SLAVE\_REG} 中 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE} 位使能从机模式。


在传输过程中，CS 信号应保持低电平，CS 信号的下降沿和上升沿代表一次传输的开始和结束。数据以字节为单位进行传输，否则多余的位将丢失。

\subsection{通信格式}\label{slave communication format}

GP-SPI2 从机模式支持 SPI 全双工通信和半双工通信。用户可配置寄存器 \hyperref[regdesc:SPIUSERREG]{SPI\_USER\_REG} 中 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN} 位选择通信模式：置 1 选择全双工模式，置 0 选择半双工模式。

全双工模式下，传输一开始，则数据同时输入和输出。在此模式下，所有比特将被视为输入/输出数据，即不需要命令、地址或 DUMMY 阶段。传输结束，触发 \hyperref[fielddesc:SPITRANSDONE]{SPI\_TRANS\_DONE} 中断。

半双工模式下，通信格式应为 CMD+ADDR+DUMMY+DATA（读或写 ）。读数据阶段，即 SPI 主机从 GP-SPI2 读取数据；写数据阶段，即 SPI 主机向 GP-SPI2 写入数据。这里的“读取数据”是相对主机而言。各个阶段的详细属性描述如下：

\begin{enumerate}
   \item CMD：用于识别 SPI 从机不同功能；一个字节，由主机发送至从机；仅支持表 \ref{table:SPI_slave_CMD_SPI} 和表 \ref{table:SPI_slave_CMD_QPI} 所述的 CMD 值；以 1-bit 模式或 4-bit QPI 模式发送；
   \item ADDR：用于确定 CPU 控制模式下 CMD1 和 CMD2 的数据地址，或在其它模式下用作占位；一个字节，由主机发送至从机；根据命令配置，以 1/2/4-bit 模式发送；
   \item DUMMY：具体值无意义，SPI 从机在此阶段准备数据；一个字节或两个字节，具体由命令决定；由主机发送至从机。

   \item 读数据或写数据阶段：CPU 控制模式下，数据可设置为 0 \verb+~+ 72 个字节；在 DMA 控制模式下，数据可设置为 0 \verb+~+ 2 MB；此阶段可根据 CMD 值，配置为 1/2/4-bit 模式。
   \end{enumerate}

半双工通信模式下，ADDR 和 DUMMY 阶段不可跳过，即便是在 DMA 控制的从机模式下，也不可跳过。

半双工传输结束后，传输的 CMD 和 ADDR 的值分别锁存至寄存器 \hyperref[regdesc:SPISLAVE1REG]{SPI\_SLAVE1\_REG} 中的 \hyperref[fielddesc:SPISLVLASTCOMMAND]{SPI\_SLV\_LAST\_COMMAND} 位和 \hyperref[fielddesc:SPISLVLASTADDR]{SPI\_SLV\_LAST\_ADDR} 位。如果 GP-SPI2 从机模式不支持传输的 CMD 值，寄存器 \hyperref[regdesc:SPISLAVE1REG]{SPI\_SLAVE1\_REG} 中 \hyperref[fielddesc:SPISLVCMDERR]{SPI\_SLV\_CMD\_ERR} 位将被置位。\hyperref[fielddesc:SPISLVCMDERR]{SPI\_SLV\_CMD\_ERR} 仅可由软件清零。

DMA 控制模式配置和 CPU 控制模式配置分别见章节 \ref{Subsection:GP-SPI2 CPU controlled Data Buffer} 和章节 \ref{Subsection:Usage of GP-SPI2 DMA controlled mode}。

\subsection{半双工模式支持的 CMD 值} \label{GP-SPI2 HD salve format}

在半双工传输中，CMD 定义的值将决定传输类型。不支持的 CMD 值及其相关数据传输均被忽略，且 \hyperref[fielddesc:SPISLVCMDERR]{SPI\_SLV\_CMD\_ERR} 将被置 1。传输格式为：CMD (8 bits) + ADDR (8 bits) + DUMMY + DATA，其中，在 1-bit SPI 模式下，DUMMY 为八个 SPI 时钟周期，在其它 SPI 模式下为四个 SPI 时钟周期；DATA 的单位为字节。CMD[3:0] 的详细说明如下：

\begin{enumerate}
    \item \hexprefix{}1 (Wr\_BUF)：CPU 控制写操作。主机发送数据，GP-SPI2 接收数据。数据将存储至相应地址的寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}。

    \item \hexprefix{}2 (Rd\_BUF)：CPU 控制读操作。主机接收 GP-SPI2 发送的数据。数据来自相应地址的寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}。

    \item \hexprefix{}3 (Wr\_DMA)：DMA 控制写操作。主机发送数据，GP-SPI2 接收数据。数据将存储至 GP-SPI2 的 DMA RX buffer 中。

    \item \hexprefix{}4 (Rd\_DMA)：DMA 控制读操作。主机接收 GP-SPI2 发送的数据。数据来自 GP-SPI2 的 DMA TX buffer。

    \item \hexprefix{}7 (CMD7)：用于生成 SPI\_SLV\_CMD7\_INT 中断。在从机连续传输模式下，使用 DMA RX 链表时，也可用于生成 SPI\_IN\_SUC\_EOF\_INT 中断。但不会结束 GP-SPI2 的从机连续传输。

    \item \hexprefix{}8 (CMD8)：仅用于生成 SPI\_SLV\_CMD8\_INT 中断，但不会结束 GP-SPI2 的从机连续传输。

    \item \hexprefix{}9 (CMD9)：仅用于生成 SPI\_SLV\_CMD9\_INT 中断，但不会结束 GP-SPI2 的从机连续传输。

    \item \hexprefix{}A (CMDA)：仅用于生成 SPI\_SLV\_CMDA\_INT 中断，但不会结束 GP-SPI2 的从机连续传输。

\end{enumerate}

CMD、ADDR 和 DATA 阶段均支持 1/2/4-bit 模式，具体由 CMD[7:4] 决定。DUMMY 仅支持 1-bit 模式，且在 1-bit SPI 模式下，持续八个 SPI 时钟周期，在其它 SPI 模式下持续四个 SPI 时钟周期。CMD[7:4] 的具体定义如下：

\begin{enumerate}
    \item \hexprefix{}0：CMD、ADDR 和 DATA 阶段均为 1-bit 模式。
    \item \hexprefix{}1：CMD 和 ADDR 均为 1-bit 模式。DATA 为 2-bit 模式。
    \item \hexprefix{}2：CMD 和 ADDR 均为 1-bit 模式。DATA 为 4-bit 模式。
    \item \hexprefix{}5：CMD 为 1-bit 模式。ADDR 和 DATA 均为 2-bit 模式。
    \item \hexprefix{}A：CMD 为 1-bit 模式，ADDR 和 DATA 均为 4-bit 模式。或 QPI 模式。注意，CMD 值为 \hexprefix{}A7、\hexprefix{}A8、\hexprefix{}A9 和 \hexprefix{}AA 时，仅在 QPI 模式下有效。
\end{enumerate}

此外，CMD[7:0] 的值为 \hexprefix{}05、\hexprefix{}A5、\hexprefix{}06 和 \hexprefix{}DD 时，将跳过 ADDR、DUMMY 和 DATA 阶段。具体定义如下：

\begin{enumerate}
    \item \hexprefix{}05 (End\_SEG\_TRANS)：主机发送 \hexprefix{}05 命令，结束 SPI 模式下从机连续传输。
    \item \hexprefix{}A5 (End\_SEG\_TRANS)：主机发送 \hexprefix{}A5 命令，结束 QPI 模式下从机连续传输。
    \item \hexprefix{}06 (En\_QPI)：GP-SPI2 接收到 \hexprefix{}06 命令后，进入 QPI 模式。此时，寄存器 \hyperref[regdesc:SPIUSERREG]{SPI\_USER\_REG} 中 \hyperref[fielddesc:SPIQPIMODE]{SPI\_QPI\_MODE} 置位。
    \item \hexprefix{}DD (Ex\_QPI)：GP-SPI2 接收到 \hexprefix{}DD 命令后，退出 QPI 模式。此时，\hyperref[fielddesc:SPIQPIMODE]{SPI\_QPI\_MODE} 位清零。
\end{enumerate}

\subsection{从机单次传输}\label{GP-SPI2 slave mode single transfer}


GP-SPI2 从机模式支持 CPU 及 DMA 控制的全双工和半双工单次数据传输。具体的寄存器配置如下：
\begin{enumerate}
    \item 按照章节  \ref{section:GP-SPI_Interfaces} 所示，配置 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN} 和 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE}；

    \item 根据具体模式，准备数据：
    \begin{itemize}
        \item 在 CPU 控制模式下，根据需要在寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG} 中准备数据；
        \item 在 DMA 控制模式下，配置 DMA 描述符并启动 DMA，见章节 \ref{Subsection:Usage of GP-SPI2 DMA controlled mode}；
    \end{itemize}

    \item 清零寄存器 \hyperref[regdesc:SPIDMACONFREG]{SPI\_DMA\_CONF\_REG} 中 \hyperref[fielddesc:SPIDMASLVSEGTRANSEN]{SPI\_DMA\_SLV\_SEG\_TRANS\_EN} 位，使能单次传输模式。

    \item 置位寄存器 \hyperref[regdesc:SPISLAVEREG]{SPI\_SLAVE\_REG} 中 \hyperref[fielddesc:SPIINTTRANSDONEEN]{SPI\_INT\_TRANS\_DONE\_EN}，等待 \hyperref[fielddesc:SPITRANSDONE]{SPI\_TRANS\_DONE} 中断。在 DMA 控制模式下，使用 DMA RX buffer 时，推荐等待 SPI\_IN\_SUC\_EOF\_INT 中断，即数据已存储至相应内存。
\end{enumerate}

\subsection{从机连续传输模式}\label{GP-SPI2 slave seg-trans}

GP-SPI2 支持从机连续传输功能，可提高通信效率和可靠性。由于寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG} 仅在 CPU 控制的从机半双工模式下可访问，所以 GP-SPI2 从机模式不支持 CPU 控制下的连续传输模式，仅支持 DMA 控制下的全双工和半双工连续传输。

在 GP-SPI2 从机连续传输模式下，主机可发送表 \ref{table:SPI_slave_CMD_SPI} 和表 \ref{table:SPI_slave_CMD_QPI} 中所有类型的数据传输，即在一次完整的连续传输过程中，可以包含 CPU 控制的数据传输，也可以包含 DMA 控制的数据传输。

在一次完整的连续传输过程中，推荐操作如下：

\begin{itemize}
    \item CPU 控制的数据传输可用于握手通信以及少量数据传输，相关中断见章节 \ref{Section:SPI_Interrupt}。
    \item DMA 控制的数据传输可用于传输大量数据，相关中断见章节 \ref{Section:SPI_Interrupt}。
\end{itemize}

GP-SPI2 收到 End\_SEG\_TRANS（SPI 模式下为 \hexprefix{}05，QPI 模式下为 \hexprefix{}A5），连续传输结束，并触发 \hyperref[fielddesc:SPIDMASEGTRANSDONE]{SPI\_DMA\_SEG\_TRANS\_DONE} 中断。具体的寄存器配置如下：

\begin{enumerate}
    \item 置位寄存器 \hyperref[regdesc:SPISLAVEREG]{SPI\_SLAVE\_REG} 中 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE} 位；
    \item 根据具体的模式，准备数据：
    \begin{itemize}
        \item  在 CPU 控制模式下，根据需要在寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG} 中准备数据；
         \item 在 DMA 控制模式下，配置 DMA 描述符并启动 DMA。清零 \hyperref[regdesc:SPIDMACONFREG]{SPI\_DMA\_CONF\_REG} 中 \hyperref[fielddesc:SPIRXEOFEN]{SPI\_RX\_EOF\_EN} 位；
    \end{itemize}
    \item 置位寄存器 \hyperref[regdesc:SPIDMACONFREG]{SPI\_DMA\_CONF\_REG} 中 \hyperref[fielddesc:SPIDMASLVSEGTRANSEN]{SPI\_DMA\_SLV\_SEG\_TRANS\_EN} 位，使能从机连续传输模式；
    \item 置位寄存器 \hyperref[regdesc:SPISLAVEREG]{SPI\_SLAVE\_REG} 中 \hyperref[fielddesc:SPIINTDMASEGTRANSEN]{SPI\_INT\_DMA\_SEG\_TRANS\_EN} 位，然后等待 \hyperref[fielddesc:SPIDMASEGTRANSDONE]{SPI\_DMA\_SEG\_TRANS\_DONE} 中断，即连续传输结束，且数据已放入相应内存。章节 \ref{Section:SPI_Interrupt} 所述中断也可用于该应用程序。
\end{enumerate}

在 DMA 全双工从机连续传输中，数据从 DMA buffer 中输入输出，因此这种传输不支持 CPU 控制模式。传输结束，触发 SPI\_IN\_SUC\_EOF\_INT\_ST 中断。具体的配置流程如下：

\begin{enumerate}
    \item 置位寄存器 \hyperref[regdesc:SPIUSERREG]{SPI\_USER\_REG} 中 \hyperref[fielddesc:SPIDOUTDIN]{SPI\_DOUTDIN} 位，和寄存器 \hyperref[regdesc:SPISLAVEREG]{SPI\_SLAVE\_REG} 中 \hyperref[fielddesc:SPISLAVEMODE]{SPI\_SLAVE\_MODE} 位；
    \item 配置 DMA 描述符并启动 DMA；

    \item 置位寄存器 \hyperref[regdesc:SPIDMACONFREG]{SPI\_DMA\_CONF\_REG} 中 \hyperref[fielddesc:SPIRXEOFEN]{SPI\_RX\_EOF\_EN} 位。在寄存器 \hyperref[regdesc:SPISLVRDBUFDLENREG]{SPI\_SLV\_RDBUF\_DLEN\_REG} 的 \hyperref[fielddesc:SPISLVDMARDBYTELEN]{SPI\_SLV\_DMA\_RD\_BYTELEN}[19:0] 中配置 DMA 接收数据长度；

    \item 置位寄存器 \hyperref[regdesc:SPIDMACONFREG]{SPI\_DMA\_CONF\_REG} 中 \hyperref[fielddesc:SPIDMASLVSEGTRANSEN]{SPI\_DMA\_SLV\_SEG\_TRANS\_EN} 位，使能从机连续传输模式；

    \item 置位寄存器 \hyperref[regdesc:SPIDMAINTENAREG]{SPI\_DMA\_INT\_ENA\_REG} 中 \hyperref[fielddesc:SPIINSUCEOFINTENA]{SPI\_IN\_SUC\_EOF\_INT\_ENA}，并等待 SPI\_IN\_SUC\_EOF\_INT\_ST 中断。
\end{enumerate}

\section{GP-SPI2 和 GP-SPI3 功能差异} \label{Subsection:Differences Between GP-SPI2 and GP-SPI3}
GP-SPI2 和 GP-SPI3 的功能差异如下：
\begin{itemize}
    \item GP-SPI2 各个阶段，如 CMD、ADDR、DOUT 或 DIN，其通信模式可单独配置。在主机模式下，数据可配置成 1-bit、2-bit、4-bit 或 8-bit 模式；在从机模式下，可配置为 1-bit、2-bit 和 4-bit 模式。由于 GP-SPI3 仅支持 1-bit 通信，暂不支持上述灵活配置。
    \item GP-SPI2 从机模式支持的 CMD 值见表 \ref{table:SPI_slave_CMD_SPI} 和表 \ref{table:SPI_slave_CMD_QPI}。GP-SPI3 从机模式仅支持的 CMD 值为：\hexprefix{}01、\hexprefix{}02、\hexprefix{}03、\hexprefix{}04、\hexprefix{}05、\hexprefix{}07、\hexprefix{}08、\hexprefix{}09 和 \hexprefix{}0A。
    \item GP-SPI2 I/O 可用过 GPIO 交换矩阵或 IO MUX 映射到芯片管脚。但 GP-SPI3 信号线仅可通过 GPIO 交换矩阵进行配置。
    \item GP-SPI2 在主机模式下具有六个 CS 信号。GP-SPI3 在主机模式下仅有三个 CS 信号。
\end{itemize}
除此之外，GP-SPI2 和 GP-SPI3 的功能相同。GP-SPI2 可使用所有通用 GP-SPI 寄存器，而 GP-SPI3 仅可使用部分 GP-SPI 寄存器，更多信息见表 \ref{tab:The deleted registers of GP-SPI3 vs to GP-SPI2}。

\section{CPU 控制下的传输模式}\label{Subsection:GP-SPI2 CPU controlled Data Buffer}

GP-SPI 主从机模式均支持 CPU 控制下的传输，其中数据从 GP-SPI 数据 buffer 读取，或写入 GP-SPI 数据 buffer。CPU 控制下的传输模式可以与全双工通信、半双工通信和章节 \ref{GP-SPI2 master mode} 以及章节 \ref{GP-SPI2 slave mode} 所描述的功能一起使用。如图 \ref{Figure:SPI_BUFFER} 所示，数据 buffer 大小为 18 x 32 bit，即 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}。待发送的数据应在传输前准备好。

\subsection{CPU 控制下的主机模式} \label{GP-SPI2 Master CPU}

在 CPU 控制模式下的全双工/半双工主机传输过程中，待读/写的数据来源于 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}[7:0] \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}[31:24] 或 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}[7:0] \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}[7:0]（高位模式）。寄存器 \hyperref[regdesc:SPIUSERREG]{SPI\_USER\_REG} 中 \hyperref[fielddesc:SPIUSRMOSIHIGHPART]{SPI\_USR\_MOSI\_HIGHPART} 位和 \hyperref[fielddesc:SPIUSRMISOHIGHPART]{SPI\_USR\_MISO\_HIGHPART} 位分别控制发送/接收数据的字节顺序。

\begin{itemize}
    \item \hyperref[fielddesc:SPIUSRMOSIHIGHPART]{SPI\_USR\_MOSI\_HIGHPART} = 0，则发送数据来源于 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}[7:0] \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}[31:24]，且每传输一个字节，数据地址增加 1。如果数据字节长度大于 72，则地址为 72 的模。\hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG} 存储的数据可多次发送。

    \item \hyperref[fielddesc:SPIUSRMOSIHIGHPART]{SPI\_USR\_MOSI\_HIGHPART} = 1，则发送数据来源于 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}[7:0] \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}[31:24]，且每传输一个字节，数据地址增加 1。如果数据字节长度大于 40，\hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}[7:0] \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}[31:24] 存储的数据可多次发送。

    \item \hyperref[fielddesc:SPIUSRMISOHIGHPART]{SPI\_USR\_MISO\_HIGHPART} = 0，则接收数据存储至 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}[7:0] \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}[31:24]，且每传输一个字节，数据地址增加 1。如果数据字节长度大于 72，则地址为 72 的模。\hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG} 存储的数据可覆盖。

    \item \hyperref[fielddesc:SPIUSRMISOHIGHPART]{SPI\_USR\_MISO\_HIGHPART} = 1，则接收数据存储至 \hyperref[regdesc:SPIW8REG]{SPI\_W8\_REG}[7:0] \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}[31:24]，且每传输一个字节，数据地址增加 1。如果数据字节长度大于 40，则 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG} 存储的数据可覆盖。

\end{itemize}

数据传输时，应清除寄存器 \hyperref[regdesc:SPIDMAINLINKREG]{SPI\_DMA\_IN\_LINK\_REG} 中 \hyperref[fielddesc:SPIDMARXENA]{SPI\_DMA\_RX\_ENA} 位和寄存器 \hyperref[regdesc:SPIDMAOUTLINKREG]{SPI\_DMA\_OUT\_LINK\_REG} 中 \hyperref[fielddesc:SPIDMATXENA]{SPI\_DMA\_TX\_ENA} 位。其中，\hyperref[fielddesc:SPIDMARXENA]{SPI\_DMA\_RX\_ENA} 置 1 时表示在 DMA 控制模式下接收数据，清零时表示在 CPU 控制模式下接收数据。\hyperref[fielddesc:SPIDMATXENA]{SPI\_DMA\_TX\_ENA} 置 1 时表示在 DMA 控制模式下发送数据，清零时表示在 CPU 控制模式下发送数据。


\subsection{CPU 控制下的从机模式} \label{GP-SPI2 slave CPU}

在 CPU 控制的从机全双工通信模式下，\hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG} 可字节寻址，数据地址从 0 开始，每发送一个字节，地址加 1。如果数据地址大于 71，则只会更新 \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}[31:24] 的内容。

在 CPU 控制下的从机半双工通信模式下，寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG} 也可按字节寻址。传输格式中的 ADDR 表示读写数据的开始地址，对应寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG}  \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}。读写地址按字节递增，对应 寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}。如果数据地址，即 \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}[31:24] 最高字节地址，大于 71，则地址将始终为 71，且仅更改 \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG}[31:24] 中内容。同时，寄存器 \hyperref[regdesc:SPISLAVE1REG]{SPI\_SLAVE1\_REG} 中 \hyperref[fielddesc:SPISLVADDRERR]{SPI\_SLV\_ADDR\_ERR} 将置 1。

根据具体的应用需要，寄存器 \hyperref[regdesc:SPIW0REG]{SPI\_W0\_REG} \verb+~+ \hyperref[regdesc:SPIW17REG]{SPI\_W17\_REG} 可全部用作数据 buffer，或部分用作数据 buffer，部分用作状态 buffer，或全部用作状态 buffer。此外，在 CPU 控制的从机模式下，\hyperref[fielddesc:SPIDMARXENA]{SPI\_DMA\_RX\_ENA} 和 \hyperref[fielddesc:SPIDMATXENA]{SPI\_DMA\_TX\_ENA} 可置 1。


\section{DMA 控制下的传输模式}\label{Subsection:Usage of GP-SPI2 DMA controlled mode}

GP-SPI 主从机模式均支持 DMA 控制下的传输，其中 DMA RX 模块接收数据，DMA TX 模块发送数据。DMA 将数据从 DMA 链接的内存中输入输出，无需 CPU 操作。

DMA 控制下的传输模式可以与全双工通信、半双工通信和章节 \ref{GP-SPI2 master mode} 以及章节  \ref{GP-SPI2 slave mode} 所描述的功能一起使用。同时，DMA RX 模块与 DMA TX 模块相互独立，即可以支持四种全双工通信：

\begin{itemize}
    \item 在 DMA 控制模式下接收数据，并在 DMA 控制模式下发送数据。

    \item 在 DMA 控制模式下接收数据，但在 CPU 控制模式下发送数据。

    \item 在 CPU 控制模式下接收数据，但在 DMA 控制模式下发送数据。

    \item 在 CPU 控制模式下接收数据，并在 CPU 控制模式下发送数据。

\end{itemize}

在半双工通信模式下，GP-SPI 从机连续传输模式支持传输 Wr\_BUF、Rd\_BUF、Wr\_DMA 和 Rd\_DMA，见章节 \ref{GP-SPI2 slave seg-trans}。

DMA 控制下的主从机通信模式具有以下特点：

\begin{itemize}
    \item 配置 DMA TX 描述符，并置位寄存器 \hyperref[regdesc:SPIDMAOUTLINKREG]{SPI\_DMA\_OUT\_LINK\_REG} 中 \hyperref[fielddesc:SPIOUTLINKSTART]{SPI\_OUTLINK\_START} 位，启动 DMA TX 控制器。在所有 DMA TX buffer 用完之前，或在复位 DMA TX 控制器之前，可置位 \hyperref[fielddesc:SPIOUTLINKRESTART]{SPI\_OUTLINK\_RESTART}，将新的 TX buffer 添加到最后一个使用中 TX buffer 的末尾。

    \item DMA RX buffer 链接方式与 DMA TX buffer 链接方式一样，置位 \hyperref[fielddesc:SPIINLINKSTART]{SPI\_INLINK\_START} 和 \hyperref[fielddesc:SPIINLINKRESTART]{SPI\_INLINK\_RESTART} 即可。

    \item 发送和接收的数据长度分别由配置的 DMA TX buffer 和 DMA RX buffer 确定，具体范围为：0 \verb+~+ 2 MB。

    \item DMA 启动前，应首先初始化 DMA 接收链表和发送链表。数据传输时，应置位寄存器  \hyperref[regdesc:SPIDMAINLINKREG]{SPI\_DMA\_IN\_LINK\_REG} 中 \hyperref[fielddesc:SPIDMARXENA]{SPI\_DMA\_RX\_ENA} 位和寄存器 \hyperref[regdesc:SPIDMAOUTLINKREG]{SPI\_DMA\_OUT\_LINK\_REG} 中 \hyperref[fielddesc:SPIDMATXENA]{SPI\_DMA\_TX\_ENA} 位。

\end{itemize}

DMA 控制的数据传输，在主机模式和从机模式下的唯一区别在于 DMA RX 控制方面：

\begin{itemize}
    \item 清除 \hyperref[fielddesc:SPIRXEOFEN]{SPI\_RX\_EOF\_EN} 位。
        \begin{itemize}
            \item 在主机模式下，单次数据传输结束；或一次 DMA 分段配置传输结束，则触发 SPI\_IN\_SUC\_EOF\_INT 中断。

            \item 在从机模式下，单次从机传输结束，且 \hyperref[fielddesc:SPIDMASLVSEGTRANSEN]{SPI\_DMA\_SLV\_SEG\_TRANS\_EN} 位为 0，则将触发 SPI\_IN\_SUC\_EOF\_INT 中断。在从机连续传输模式下，如果 \hyperref[fielddesc:SPIDMASLVSEGTRANSEN]{SPI\_DMA\_SLV\_SEG\_TRANS\_EN} 置 1，且正确收到 CMD7 或 End\_SEG\_TRANS 命令，也将触发 SPI\_IN\_SUC\_EOF\_INT 中断。
        \end{itemize}

    \item 置位 \hyperref[fielddesc:SPIRXEOFEN]{SPI\_RX\_EOF\_EN} 位。
        \begin{itemize}
            \item 在 GP-SPI 主机模式下，单次数据传输结束或一次 DMA 分段配置传输结束，且接收到的总 DMA 数据长度等于 \hyperref[fielddesc:SPIMSTDMARDBYTELEN]{SPI\_MST\_DMA\_RD\_BYTELEN} 的值，则将触发 SPI\_IN\_SUC\_EOF\_INT 中断。

            \item 在从机模式下，接收到的总 DMA 数据长度应等于 \hyperref[fielddesc:SPISLVDMARDBYTELEN]{SPI\_SLV\_DMA\_RD\_BYTELEN} 的值。单次从机传输结束，且 \hyperref[fielddesc:SPIDMASLVSEGTRANSEN]{SPI\_DMA\_SLV\_SEG\_TRANS\_EN} 位为 0，则将触发 SPI\_IN\_SUC\_EOF\_INT 中断。在从机连续传输模式下，如果 \hyperref[fielddesc:SPIDMASLVSEGTRANSEN]{SPI\_DMA\_SLV\_SEG\_TRANS\_EN} 置 1，且正确收到 CMD7 或 End\_SEG\_TRANS 命令，也将触发 SPI\_IN\_SUC\_EOF\_INT 中断。
        \end{itemize}

\end{itemize}

如果配置的 DMA TX buffer 长度小于实际传输的数据长度，则多出的数据将与上次传输的数据相同。同时将置位寄存器 \hyperref[regdesc:SPIDMAINTRAWREG]{SPI\_DMA\_INT\_RAW\_REG} 中 \hyperref[fielddesc:SPIOUTFIFOEMPTYERRINTRAW]{SPI\_OUTFIFO\_EMPTY\_ERR\_INT\_RAW} 和 \hyperref[fielddesc:SPIOUTEOFINTRAW]{SPI\_OUT\_EOF\_INT\_RAW} 位。

如果配置的 DMA TX buffer 长度大于实际传输的数据长度，则 TX buffer 未完全使用，即使以后链接了新的 TX buffer，上面未使用的 TX buffer 部分仍可使用。请特别注意上述情况，或保存未使用的数据并复位 DMA。

如果配置的 DMA RX buffer 长度小于实际传输的数据长度，则多出的数据将丢失。寄存器 \hyperref[regdesc:SPIDMAINTRAWREG]{SPI\_DMA\_INT\_RAW\_REG} 中 \hyperref[fielddesc:SPIINFIFOFULLERRINTRAW]{SPI\_INFIFO\_FULL\_ERR\_INT\_RAW} 位和寄存器 \hyperref[regdesc:SPISLAVEREG]{SPI\_SLAVE\_REG} 中 \hyperref[fielddesc:SPITRANSDONE]{SPI\_TRANS\_DONE} 有效。但不会生成有效的 \hyperref[fielddesc:SPIINSUCEOFINTRAW]{SPI\_IN\_SUC\_EOF\_INT\_RAW} 中断。

如果配置的 DMA RX buffer 长度大于实际传输的数据长度，则 RX buffer 未被完全使用，即使以后链接了新的 RX buffer，上述 buffer 未使用的部分仍可继续使用。请特别注意上述情况，或复位 DMA。


\section{GP-SPI 时钟控制}\label{Subsection:GP-SPI2 CLK}
主机模式下 GP-SPI 最高输出时钟频率为 $f_{\textrm{apb}}$。如果需要较低的时钟频率，可以采用如下分频方式：

    \[
    f_{\textrm{spi}}=\frac{f_{\textrm{apb}}} {\textrm{(\hyperref[fielddesc:SPICLKCNTN]{SPI\_CLKCNT\_N}+1)(\hyperref[fielddesc:SPICLKDIVPRE]{SPI\_CLKDIV\_PRE}+1)}}
    \] \label{division-formula}

 用户可配置寄存器 \hyperref[regdesc:SPICLOCKREG]{SPI\_CLOCK\_REG} 中 \hyperref[fielddesc:SPICLKCNTN]{SPI\_CLKCNT\_N} 和 \hyperref[fielddesc:SPICLKDIVPRE]{SPI\_CLKDIV\_PRE} 设置分频系数。寄存器 \hyperref[regdesc:SPICLOCKREG]{SPI\_CLOCK\_REG} 中 \hyperref[fielddesc:SPICLKEQUSYSCLK]{SPI\_CLK\_EQU\_SYSCLK} 位置 1 时，GP-SPI 的输出时钟频率为 $f_{\textrm{apb}}$。如果采用其它整数分频，则 \hyperref[fielddesc:SPICLKEQUSYSCLK]{SPI\_CLK\_EQU\_SYSCLK} 应置 0。

从机模式下，GP-SPI 支持的最高输入时钟频率为$f_{\textrm{apb}}$/2，低于此值的其它频率均可使用。

\subsection{GP-SPI 时序补偿} \label{subsubsection:timing-compensation}

SPI 输入输出信号可通过 GPIO 矩阵或 IO MUX 映射到芯片管脚，但 IO MUX 不支持时序调整。输入输出数据在 GPIO 矩阵模块中，可在上升沿或下降沿延迟 0、1、或 2 个 APB\_CLK 周期。寄存器配置详情见章节 \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}}。

在 GP-SPI 主机模式下，输入和输出数据均有时序调整模块，可在上升沿或下降沿延迟整数个 APB\_CLK 周期。寄存器 \hyperref[regdesc:SPIDINMODEREG]{SPI\_DIN\_MODE\_REG} 和 \hyperref[regdesc:SPIDINNUMREG]{SPI\_DIN\_NUM\_REG} 可用于选择锁存输入数据的时钟沿。寄存器 \hyperref[regdesc:SPIDOUTMODEREG]{SPI\_DOUT\_MODE\_REG} 和 \hyperref[regdesc:SPIDOUTNUMREG]{SPI\_DOUT\_NUM\_REG} 可用于选择锁存输出数据的时钟沿，更多信息见章节 \ref{Section:SPIREG}。同时，DUMMY 周期长可更改，用于补偿实际的 I/O 线路延迟，从而提高 GP-SPI 性能。

在 GP-SPI 从机模式下，如果寄存器 \hyperref[regdesc:SPICTRL1REG]{SPI\_CTRL1\_REG} 中 \hyperref[fielddesc:SPIRSCKDATAOUT]{SPI\_RSCK\_DATA\_OUT} 置 1，则在锁存沿发送输出数据，即提前半个 SPI 时钟周期。上述功能可用于从机模式时序补偿。

\subsection{GP-SPI 中断} \label{Section:SPI_INT}

SPI\_INT 中断源描述如下：
\begin{itemize}
\item \label{int:SPITRANSDONEINT}\hyperref[fielddesc:SPITRANSDONE]{SPI\_TRANS\_DONE}：主从机模式下，SPI 总线传输结束即触发此中断。
\item \label{int:SPISLVWRSTAINT}\hyperref[fielddesc:SPISLVWRDMADONE]{SPI\_SLV\_WR\_DMA\_DONE}：从机模式下，Wr\_DMA 传输结束即触发此中断。
\item \label{int:SPISLVRDSTAINT}\hyperref[fielddesc:SPISLVRDDMADONE]{SPI\_SLV\_RD\_DMA\_DONE}：从机模式下，Rd\_DMA 传输结束即触发此中断。
\item \label{int:SPISLVWRBUFINT}\hyperref[fielddesc:SPISLVWRBUFDONE]{SPI\_SLV\_WR\_BUF\_DONE}：从机模式下，Wr\_BUF 传输结束即触发此中断。
\item \label{int:SPISLVRDBUFINT}\hyperref[fielddesc:SPISLVRDBUFDONE]{SPI\_SLV\_RD\_BUF\_DONE}：从机模式下，Rd\_BUF 传输结束即触发此中断。
\item \label{int:SPISEGTRANSINT}\hyperref[fielddesc:SPIDMASEGTRANSDONE]{SPI\_DMA\_SEG\_TRANS\_DONE}：GP-SPI 从机连续传输模式下，End\_SEG\_TRANS 传输结束即触发此中断。主机模式下，分段配置传输结束也将触发此中断。
\item \hyperref[fielddesc:SPISEGMAGICERR]{SPI\_SEG\_MAGIC\_ERR\_INT}：主机 DMA 分段配置传输模式下，CONF buffer 中最近的 Magic 值有误。
\end{itemize}

\subsection{GP-SPI DMA 中断} \label{Section:SPI_DMA_INT}

SPI\_DMA\_INTR 中断源描述如下：
\begin{itemize}
\item \label{int:SPISLVCMDAINT} SPI\_SLV\_CMDA\_INT：GP-SPI 从机模式下，正确接收 CMDA，且 SPI 传输结束即触发此中断。

\item \label{int:SPISLVCMD9INT} SPI\_SLV\_CMD9\_INT：GP-SPI 从机模式下，正确接收 CMD9，且 SPI 传输结束即触发此中断。

\item \label{int:SPISLVCMD8INT} SPI\_SLV\_CMD8\_INT：GP-SPI 从机模式下，正确接收 CMD8，且 SPI 传输结束即触发此中断。

\item \label{int:SPISLVCMD7INT} SPI\_SLV\_CMD7\_INT：GP-SPI 从机模式下，正确接收 CMD7，且 SPI 传输结束即触发此中断。

\item \label{int:SPISLVCMD6INT} SPI\_SLV\_CMD6\_INT：GP-SPI 从机模式下，正确接收 En\_QPI 或 Ex\_QPI，且 SPI 传输结束即触发此中断。

\item \label{int:SPIOUTFIFOEMPTYERRINT}SPI\_OUTFIFO\_EMPTY\_ERR\_INT：DMA TX FIFO 小于实际传输的数据时即触发此中断。
\item \label{int:SPIINFIFOFULLERRINT}SPI\_INFIFO\_FULL\_ERR\_INT：DMA RX FIFO 小于实际传输的数据时即触发此中断。
\item \label{int:SPIOUTTOTALEOFINT}SPI\_OUT\_TOTAL\_EOF\_INT：发送链表 buffer 中所有数据发送结束即触发此中断。
\item \label{int:SPIOUTEOFINT}SPI\_OUT\_EOF\_INT：一个发送链表 buffer 用完即触发此中断。
\item \label{int:SPIOUTDONEINT}SPI\_OUT\_DONE\_INT：最后一个发送链表为 0 时即触发此中断。
\item \label{int:SPIINSUCEOFINT}SPI\_IN\_SUC\_EOF\_INT：所有接收链表 buffer 已满时即触发此中断。
\item \label{int:SPIINERREOFINT}SPI\_IN\_ERR\_EOF\_INT：接收链表出现错误即触发此中断。
\item \label{int:SPIINDONEINT}SPI\_IN\_DONE\_INT：最后一个接收链表为 0 时即触发此中断。
\item \label{int:SPIINLINKDSCRERRORINT}SPI\_INLINK\_DSCR\_ERROR\_INT：接收链表描述符出现错误时触发。
\item \label{int:SPIOUTLINKDSCRERRORINT}SPI\_OUTLINK\_DSCR\_ERROR\_INT：正在使用的发送链表无效时触发。
\item \label{int:SPIINLINKDSCREMPTYINT}SPI\_INLINK\_DSCR\_EMPTY\_INT：无有效的接收链表描述符时触发。
\end{itemize}
