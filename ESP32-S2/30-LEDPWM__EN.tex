\chapter{LED PWM Controller (LEDC)}
\label{mod:ledpwm}

\section{Overview}

The LED PWM controller is primarily designed to control LED devices, as well as generate PWM signals. It has 14-bit timers and waveform generators.

\section{Features}
The LED PWM controller has the following features:
\begin{itemize}
    \item Four independent timers that support division by fractions
    \item Eight independent waveform generators able to produce eight PWM signals
    \item Fading duty cycle of PWM signals without interference from any processors. An interrupt can be generated after the fade has completed
    \item Adjustable phase of PWM signal output
    \item PWM signal output in low-power mode
\end{itemize}

For the convenience of description, in the following sections the eight waveform generators are collectively referred to as PWM\regindex{n}, and the four timers are collectively referred to as Timer\regindex{x}.

\vspace{-2em}

\subsection{Timers}\label{ledc_timers}

\begin{figure}[ht]
    \centering
    \fbox{\includegraphics[width=0.9\textwidth]{\modulefiles/figures/ledc_frag_div}}
    \caption{LED\_PWM Divider}
    \label{Figure:led_pwm_divider}

The clock of the LED PWM controller, LEDC\_PWM\_CLK, has three clock sources: APB\_CLK, RC\_FAST\_CLK and XTAL\_CLK, selected by configuring \hyperref[fielddesc:LEDCAPBCLKSEL]{LEDC\_APB\_CLK\_SEL}[1:0]. The clock of each LED PWM timer, LEDC\_CLK\regindex{x}, has two clock sources: LEDC\_PWM\_CLK or REF\_TICK. When REF\_TICK is used as the clock source of a timer, \hyperref[fielddesc:LEDCAPBCLKSEL]{LEDC\_APB\_CLK\_SEL}[1:0] should be set 1 and the cycle of REF\_TICK should be an integral multiple of APB\_CLK cycles. Otherwise this clock will be not accurate. For more information on the clock sources, please see Chapter \hyperref[mod:resclk]{Reset and Clock}.

The output clock derived from LEDC\_CLK\regindex{x} is used as the base clock for the counter. The divider's divisor is configured by \hyperref[fielddesc:LEDCCLKDIVTIMERX]{LEDC\_CLK\_DIV\_TIMER\regindex{x}}. It is a fixed-point number: the highest 10 bits is the integer part represented as A, while the lowest eight bits is the fractional part represented as B. This divisor LEDC\_CLK\_DIV\regindex{x} is calculated as:
\begin{center}
$LEDC\_CLK\_DIV\regindex{x} = A + \frac{B}{256}$
\end{center}

When the fractional part B is not 0, the input and output clock of the divider is shown as in figure \ref{Figure:led_pwm_divider}. Among the 256 output clocks, B of them are divided by (A+1), whereas the remaining (256-B) are divided by A. Output clocks divided by (A+1) are evenly distributed in the total 256 output clocks.

The LED PWM controller has a 14-bit counter that counts up to $2^{\hyperref[fielddesc:LEDCTIMERXDUTYRES]{LEDC\_TIMER\regindex{x}\_DUTY\_RES}}-1$. If the counting value reaches $2^{\hyperref[fielddesc:LEDCTIMERXDUTYRES]{LEDC\_TIMER\regindex{x}\_DUTY\_RES}}-1$, the counter will overflow and restart counting from 0. The counting value can be reset, suspended or read by software. A LEDC\_TIMER\regindex{x}\_OVF\_INT interrupt can be generated every time when the counter overflows or when it overflows for (\hyperref[fielddesc:LEDCOVFNUMCHN]{LEDC\_OVF\_NUM\_CH\regindex{n}} + 1) times. The interrupt configuration is as follows:

 \begin{enumerate}
    \item Set \hyperref[fielddesc:LEDCOVFCNTENCHN]{LEDC\_OVF\_CNT\_EN\_CH\regindex{n}}
    \item Configure \hyperref[fielddesc:LEDCOVFNUMCHN]{LEDC\_OVF\_NUM\_CH\regindex{n}} with the times of overflow minus 1
    \item Set \hyperref[fielddesc:LEDCOVFCNTCHNINTENA]{LEDC\_OVF\_CNT\_CH\regindex{n}\_INT\_ENA}
    \item Set \hyperref[fielddesc:LEDCTIMERXDUTYRES]{LEDC\_TIMER\regindex{x}\_DUTY\_RES} to enable the timer and
    wait for a LEDC\_OVF\_CNT\_CH\regindex{n}\_INT interrupt
\end{enumerate}

The frequency of a PWM generator output signal, sig\_out\regindex{n}, depends on the frequency of the timer’s clock source LEDC\_CLK\regindex{x}, the division factor of the divider LEDC\_CLK\_DIV\regindex{x}, as well as the duty resolution (counter width) \hyperref[fielddesc:LEDCTIMERXDUTYRES]{LEDC\_TIMER\regindex{x}\_DUTY\_RES}:

    \[
    f_{\textrm{sig\_out\regindex{n}}}=\frac{f_{\textrm{LEDC\_CLK\regindex{x}}}}
                        {\textrm{LEDC\_CLK\_DIV\regindex{x}}\regindex{x}\cdot 2^{\hyperref[fielddesc:LEDCTIMERXDUTYRES]{LEDC\_TIMER\regindex{x}\_DUTY\_RES}}}
    \]

Based on the formula above, the desired duty resolution can be calculated as follows:

    \[
    \hyperref[fielddesc:LEDCTIMERXDUTYRES]{LEDC\_TIMER\regindex{x}\_DUTY\_RES}
    =\log_2\left(\frac{f_{\textrm{LEDC\_CLK\regindex{x}}}}
    {f_{\textrm{sig\_out\regindex{n}}}\cdot \textrm{LEDC\_CLK\_DIV\regindex{x}}}\right)
    \]

Table \ref{tab:ledc-common-freq-res} lists the commonly-used frequencies and their corresponding resolutions.

        \item[2] The lowest resolution is calculated when the clock divisor LEDC\_CLK\_DIV\regindex{x} is $1023+\frac{255}{256}$ and rounded up. If the lowest resolution calculated by the formula is lower than 0, then the lowest resolution should be 1.

To change the divisor and times of overflow, you should configure \hyperref[fielddesc:LEDCCLKDIVTIMERX]{LEDC\_CLK\_DIV\_TIMER\regindex{x}} and \\\hyperref[fielddesc:LEDCTIMERXDUTYRES]{LEDC\_TIMER\regindex{x}\_DUTY\_RES} respectively, and then set \hyperref[fielddesc:LEDCTIMERXPARAUP]{LEDC\_TIMER\regindex{x}\_PARA\_UP}; otherwise this change is not valid. The newly configured value is updated upon next overflow of the counter.

\subsection{PWM Generators}\label{LEDCPWM generator}
As shown in figure \ref{Figure:led_pwm_PWM generator}, each PWM generator has a high/low level comparator and two selectors. A PWM generator takes the 14-bit counting value of the selected timer, compares it to values of the comparator Hpoint\regindex{n} and Lpoint\regindex{n}, and therefore control the level of PWM signals.

\begin{itemize}
\item If Timer\regindex{x}\_cnt == Hpoint\regindex{n}, sig\_out\regindex{n} is 1.
\item If Timer\regindex{x}\_cnt == Lpoint\regindex{n}, sig\_out\regindex{n} is 0.
\end{itemize}

Hpoint\regindex{n} is updated by \hyperref[fielddesc:LEDCHPOINTCHN]{LEDC\_HPOINT\_CH\regindex{n}} when the counter overflows. The initial value of Lpoint\regindex{n} is the sum of \hyperref[fielddesc:LEDCDUTYCHN]{LEDC\_DUTY\_CH\regindex{n}}[18..4] and \hyperref[fielddesc:LEDCHPOINTCHN]{LEDC\_HPOINT\_CH\regindex{n}} when the counter overflows. By configuring these two fields, the relative phase and the duty cycle of the PWM output can be set.

Figure \ref{Figure:led_pwm_fixed_duty} illustrates PWM's waveform when the duty cycle is fixed.

\begin{figure}[ht]
    \centering
    \fbox{\includegraphics[width=0.6\textwidth]{\modulefiles/figures/ledc_2-3-1_723}}
    \caption{LED\_PWM Output Signal Diagram}
    \label{Figure:led_pwm_fixed_duty}
\end{figure}

\label{ledcfd:dithering}
\hyperref[fielddesc:LEDCDUTYCHN]{LEDC\_DUTY\_CH\regindex{n}} is a fixed-point register with four fractional bits. \hyperref[fielddesc:LEDCDUTYCHN]{LEDC\_DUTY\_CH\regindex{n}}[18..4] is the integral part used directly for PWM calculation. \hyperref[fielddesc:LEDCDUTYCHN]{LEDC\_DUTY\_CH\regindex{n}}[3..0] is the fractional part used to dither the output. If \hyperref[fielddesc:LEDCDUTYCHN]{LEDC\_DUTY\_CH\regindex{n}}[3..0] is non-zero, then among every 16 cycles of sig\_out\regindex{n}, \hyperref[fielddesc:LEDCDUTYCHN]{LEDC\_DUTY\_CH\regindex{n}}[3..0] have PWM pulses with width one timer cycle longer than that of (16-\hyperref[fielddesc:LEDCDUTYCHN]{LEDC\_DUTY\_CH\regindex{n}}[3..0]). This feature effectively increases the resolution of the PWM generator to 18 bits.

\subsection{Duty Cycle Fading}

The PWM generators is able to fade the duty cycle, that is to gradually change the duty cycle from one value to another. This is achieved by configuring \hyperref[fielddesc:LEDCDUTYCHN]{LEDC\_DUTY\_CH\regindex{n}}, \hyperref[fielddesc:LEDCDUTYSTARTCHN]{LEDC\_DUTY\_START\_CH\regindex{n}}, \hyperref[fielddesc:LEDCDUTYINCCHN]{LEDC\_DUTY\_INC\_CH\regindex{n}}, \hyperref[fielddesc:LEDCDUTYNUMCHN]{LEDC\_DUTY\_NUM\_CH\regindex{n}} and \hyperref[fielddesc:LEDCDUTYSCALECHN]{LEDC\_DUTY\_SCALE\_CH\regindex{n}}.

\hyperref[fielddesc:LEDCDUTYSTARTCHN]{LEDC\_DUTY\_START\_CH\regindex{n}} is used to update the value of Lpoint\regindex{n}. When this bit is set and the counter overflows, Lpoint\regindex{n} increments or decrements automatically, depending on whether the bit \hyperref[fielddesc:LEDCDUTYINCCHN]{LEDC\_DUTY\_INC\_CH\regindex{n}} is set or cleared.

The duty cycle of sig\_out\regindex{n} changes every \hyperref[fielddesc:LEDCDUTYCYCLECHN]{LEDC\_DUTY\_CYCLE\_CH\regindex{n}} PWM pulse cycles by adding or subtracting the value of \hyperref[fielddesc:LEDCDUTYSCALECHN]{LEDC\_DUTY\_SCALE\_CH\regindex{n}}.

Figure \ref{Figure:led_pwm_unfixed_duty} is a diagram of fading duty cycle. Upon reaching \hyperref[fielddesc:LEDCDUTYNUMCHN]{LEDC\_DUTY\_NUM\_CH\regindex{n}}, the fade stops and a \\LEDC\_DUTY\_CHNG\_END\_CH\regindex{n}\_INT interrupt is generated. When configured like this, the duty cycle of sig\_out\regindex{n} increases by \hyperref[fielddesc:LEDCDUTYSCALECHN]{LEDC\_DUTY\_SCALE\_CH\regindex{n}} every \hyperref[fielddesc:LEDCDUTYCYCLECHN]{LEDC\_DUTY\_CYCLE\_CH\regindex{n}} PWM pulse cycles.

\begin{figure}[ht]
    \centering
    \fbox{\includegraphics[width=0.8\textwidth]{\modulefiles/figures/ledc_f5}}
    \caption{Output Signal Diagram of Fading Duty Cycle}
    \label{Figure:led_pwm_unfixed_duty}
\end{figure}

\hyperref[fielddesc:LEDCSIGOUTENCHN]{LEDC\_SIG\_OUT\_EN\_CH\regindex{n}} used to enable PWM waveform output. When \hyperref[fielddesc:LEDCSIGOUTENCHN]{LEDC\_SIG\_OUT\_EN\_CH\regindex{n}} is 0, the level of sig\_out\regindex{n} is constant as specified in \hyperref[fielddesc:LEDCIDLELVCHN]{LEDC\_IDLE\_LV\_CH\regindex{n}}.

If \hyperref[fielddesc:LEDCHPOINTCHN]{LEDC\_HPOINT\_CH\regindex{n}}, \hyperref[fielddesc:LEDCDUTYSTARTCHN]{LEDC\_DUTY\_START\_CH\regindex{n}}, \hyperref[fielddesc:LEDCSIGOUTENCHN]{LEDC\_SIG\_OUT\_EN\_CH\regindex{n}},  \hyperref[fielddesc:LEDCTIMERSELCHN]{LEDC\_TIMER\_SEL\_CH\regindex{n}},  \\\hyperref[fielddesc:LEDCDUTYNUMCHN]{LEDC\_DUTY\_NUM\_CH\regindex{n}}, \hyperref[fielddesc:LEDCDUTYCYCLECHN]{LEDC\_DUTY\_CYCLE\_CH\regindex{n}},  \hyperref[fielddesc:LEDCDUTYSCALECHN]{LEDC\_DUTY\_SCALE\_CH\regindex{n}}, \hyperref[fielddesc:LEDCDUTYINCCHN]{LEDC\_DUTY\_INC\_CH\regindex{n}} and \\\hyperref[fielddesc:LEDCOVFCNTENCHN]{LEDC\_OVF\_CNT\_EN\_CH\regindex{n}} are reconfigured, \hyperref[fielddesc:LEDCPARAUPCHN]{LEDC\_PARA\_UP\_CH\regindex{n}} should be set to apply the new configuration.


\subsection{Interrupts}
\begin{itemize}
\item \label{int:LEDCOVFCNTCHnINT}LEDC\_OVF\_CNT\_CH\regindex{n}\_INT: Triggered when the timer counter overflows for (\hyperref[fielddesc:LEDCOVFNUMCHN]{LEDC\_OVF\_NUM\_CH\regindex{n}} + 1) times and register \hyperref[fielddesc:LEDCOVFCNTENCHN]{LEDC\_OVF\_CNT\_EN\_CH\regindex{n}} is set to 1.
\item \label{int:LEDCDUTYCHNGENDCHnINT}LEDC\_DUTY\_CHNG\_END\_CH\regindex{n}\_INT: Triggered when a fade on a LED PWM generator has finished.
\item \label{int:LEDCTIMERxOVFINT}LEDC\_TIMER\regindex{x}\_OVF\_INT: Triggered when a LED PWM timer has reached its maximum counter value.
\end{itemize}
