\hypertarget{systimer}{}
\chapter{System Timer (SYSTIMER)}
\label{mod:systimer}


\section{Overview}
System timer is a 64-bit timer specially for operating system. It can be used to schedule operating system tasks by generating periodical system ticks or certain time delay interrupts.

\section{Main Features}
\begin{itemize}
    \item A 64-bit timer
    \item Clocked with APB\_CLK
    \item Timer value increment step can be configured for each APB\_CLK cycle.
    \item Support automatic time compensation in case of APB\_CLK clock source switching between PLL\_CLK and XTAL\_CLK, to improve timer accuracy.
    \item Generate three independent interrupts based on different alarm values or periods (targets).
    \item Support for 64-bit alarm values and 30-bit periods.
    \item Software configuring the reference count value. For example, the system timer is able to load back the sleep time recorded by RTC timer via software after Light-sleep.
    \item Keep stalled if CPU is stalled or CPU is in on-chip-debugging mode.
\end{itemize}

\section{Clock Source Selection}

The System Timer is driven using XTAL\_CLK or PLL\_CLK clock. Selection between the two clock sources is described in Table CPU\_CLK Source in Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}}. For the specific clock frequency used for the System Timer, please refer to Table \ref{table:apb_clk_src} \textit{\nameref{table:apb_clk_src}}. On each clock period the timer will be increased by a step value configured in either \hyperref[fielddesc:SYSTIMERTIMERXTALSTEP]{SYSTIMER\_TIMER\_XTAL\_STEP} or \hyperref[fielddesc:SYSTIMERTIMERPLLSTEP]{SYSTIMER\_TIMER\_PLL\_STEP}, depending on which clock source is used.


\section{Functional Description}

Figure \ref{figure:System Timer Structure} shows the structure of system timer. The system timer can be enabled by setting the bit \hyperref[regdesc:SYSTEMPERIPCLKEN0REG]{SYSTEM\_SYSTIMER\_CLK\_EN} in register \hyperref[regdesc:SYSTEMPERIPCLKEN0REG]{SYSTEM\_PERIP\_CLK\_EN0\_REG} and be reset via software by setting the bit \hyperref[regdesc:SYSTEMPERIPRSTEN0REG]{SYSTEM\_SYSTIMER\_RST} in register \hyperref[regdesc:SYSTEMPERIPRSTEN0REG]{SYSTEM\_PERIP\_RST\_EN0\_REG}. For more information, please refer to Table \ref{table:peripheralcontrollingbit} \textit{\nameref{table:peripheralcontrollingbit}} in Chapter \ref{mod:sysreg} \textit{\nameref{mod:sysreg}}.

\subsection{Read System Timer Value}\label{section:read-timer-value}

\begin{enumerate}
    \item Set \hyperref[fielddesc:SYSTIMERTIMERUPDATE]{SYSTIMER\_TIMER\_UPDATE} to update the timer value into registers.
    \item Wait till \hyperref[fielddesc:SYSTIMERTIMERVALUEVALID]{SYSTIMER\_TIMER\_VALUE\_VALID} is set, which means users now can read the timer values from registers.
    \item Read the high 32 bits of the timer value from \hyperref[fielddesc:SYSTIMERTIMERVALUEHI]{ SYSTIMER\_TIMER\_VALUE\_HI}, and the low 32 bits from \hyperref[fielddesc:SYSTIMERTIMERVALUELO]{SYSTIMER\_TIMER\_VALUE\_LO}.

\end{enumerate}

\subsection{Configure a Time-Delay Alarm}

\begin{enumerate}
    \item Read the current value of system timer, see Section \ref{section:read-timer-value}. This value will be used to calculate the target in Step 3.
    \item Clear \hyperref[fielddesc:SYSTIMERTARGET0PERIODMODE]{ SYSTIMER\_TARGET\regindex{x}\_PERIOD\_MODE} to set the timer into a time-delay alarm mode.
    \item Write the high 32 bits of the target (alarm value) to \hyperref[fielddesc:SYSTIMERTIMERTARGET0HI]{SYSTIMER\_TIMER\_TARGET\regindex{x}\_HI}, and the low 32 bits to \hyperref[fielddesc:SYSTIMERTIMERTARGET0LO]{SYSTIMER\_TIMER\_TARGET\regindex{x}\_LO}.
    \item Set \hyperref[fielddesc:SYSTIMERTARGET0WORKEN]{ SYSTIMER\_TARGET\regindex{x}\_WORK\_EN} to enable the selected work mode.
    \item Set \hyperref[fielddesc:SYSTIMERINT0ENA]{SYSTIMER\_INT\regindex{x}\_ENA} to enable timer interrupt. When the timer counts to the alarm value, an interrupt will be triggered.
\end{enumerate}

\subsection{Configure Periodic Alarms}

\begin{enumerate}
     \item Write the target (alarm period) to \hyperref[fielddesc:SYSTIMERTARGET0PERIOD]{SYSTIMER\_TARGET\regindex{x}\_PERIOD}.
    \item Clear and then set \hyperref[fielddesc:SYSTIMERTARGET0PERIODMODE]{ SYSTIMER\_TARGET\regindex{x}\_PERIOD\_MODE} to configure the timer into periodic alarms mode.
    \item Set \hyperref[fielddesc:SYSTIMERTARGET0WORKEN]{ SYSTIMER\_TARGET\regindex{x}\_WORK\_EN} to enable periodical alarms mode.
    \item Set \hyperref[fielddesc:SYSTIMERINT0ENA]{SYSTIMER\_INT\regindex{x}\_ENA} to enable timer interrupt. An interrupt will be triggered when the timer counts to the target value set in Step 2.
\end{enumerate}

\subsection{Update after Light-sleep}

\begin{enumerate}

    \item Configure the RTC timer before the chip goes to Light-sleep, to record the exact sleep time. 
    \item Read the sleep time from the RTC timer when the chip is woken up from Light-sleep. 
    \item Read the current value of the system timer. See Section \ref{section:read-timer-value} for how to read the timer value.
    \item Add the current value of the system timer to the time that the RTC timer records in Light-sleep mode.
    \item Write the result into \hyperref[fielddesc:SYSTIMERTIMERLOADHI]{SYSTIMER\_TIMER\_LOAD\_HI} (high 32 bits), and into \hyperref[fielddesc:SYSTIMERTIMERLOADLO]{SYSTIMER\_TIMER\_LOAD\_LO} (low 32 bits).
    \item Set \hyperref[fielddesc:SYSTIMERTIMERLOAD]{SYSTIMER\_TIMER\_LOAD} to load the new value stored in \hyperref[fielddesc:SYSTIMERTIMERLOADHI]{SYSTIMER\_TIMER\_LOAD\_HI} and \hyperref[fielddesc:SYSTIMERTIMERLOADLO]{SYSTIMER\_TIMER\_LOAD\_LO} into the system timer. In such a way, the system timer is updated.
\end{enumerate}
