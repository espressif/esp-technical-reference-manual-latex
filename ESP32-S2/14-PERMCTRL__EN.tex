\hypertarget{permctrl}{}
\chapter{Permission Control (PMS)}
\label{mod:permctrl}


\section{Overview}
In ESP32-S2, different type of buses and modules can have different permissions to access the memory. The permission control module is used to manage all the access permissions through corresponding permission control registers.

\section{Features}

\begin{itemize}
    \item Permission controls for the CPU buses to access the internal memory
    \begin{itemize}
        \item SRAM partially adopts unified permission control and partially adopts split permission control.
        \item RTC FAST SRAM adopts split permission control.
        \item RTC SLOW SRAM adopts split permission control.
    \end{itemize}
    \item Access to the external memory is managed by both the memory management unit (MMU) and the permission control module.
\end{itemize}

\section{Functional Description}

\subsection{Internal Memory Permission Controls}
The internal memory resources in ESP32-S2 include ROM, SRAM, RTC FAST Memory, and RTC SLOW Memory. The breakdown of each memory is as follows:
\begin{itemize}
    \item The total ROM size is 128 KB and consists of two 64 KB blocks.
    \item The total SRAM size is 320 KB and consists of four 8 KB blocks and eighteen 16 KB blocks, numbered Block 0 \verb+~+ 21. The offset address range of each block is shown in Table \ref{table:SRAM_block_address}. For the base addresses accessed by different buses please refer to Section \ref{IBUSSRAM}, \ref{DUBS0PermissionControl}, and \ref{DMAPermissionControl}. Each of the four 8 KB blocks (Block 0 \verb+~+ 3) is equipped with an independent permission control register. The eighteen 16 KB blocks (Block 4 \verb+~+ 21) are regarded as a large storage space as a whole, which is  managed  in split parts (i.e., the storage space is split into high and low areas with separate permission controls). The split address cannot fall within the address range of Block 4 \verb+~+ 5.
    \item RTC FAST Memory adopts split permission control. The high and low areas have independent permission control registers.
    \item RTC SLOW Memory also adopts split permission control. The high and low areas have independent permission control registers.
\end{itemize}

The following sections describe the buses and modules in ESP32-S2 that can access the above-mentioned internal memory resources.

\subsubsection{Permission Control for the Instruction Bus (IBUS)}\label{Permission Control for the Instruction Bus (IBUS)}
The address range on IBUS that requires permission control is \hexprefix{}4002\_0000 \verb+~+ \hexprefix{}4007\_1FFF, where SRAM memory blocks and RTC FAST Memory are located. Access to different memory blocks is controlled by independent permission control registers which are configured by software.

The permission control registers related to IBUS are also controlled by the \hyperref[fielddesc:PMSPROIRAM0LOCK]{PMS\_PRO\_IRAM0\_LOCK} signal. When this signal is set to 1, the values configured in the permission control registers will be locked and cannot be changed. At the same time, the value of the \hyperref[fielddesc:PMSPROIRAM0LOCK]{PMS\_PRO\_IRAM0\_LOCK} signal will also remain at 1 and cannot be changed. The value of the \hyperref[fielddesc:PMSPROIRAM0LOCK]{PMS\_PRO\_IRAM0\_LOCK} signal is reset to 0 only when the CPU is reset.

\begin{tiplisting}
\textcolor{red}{*} {\bfseries Configuring split address:}

Split address is 32 bits in width and equal to base address plus the configured field value multiplied by 4. For example, if \hexprefix{}10000 is written to \hyperref[regdesc:PMSPROIRAM02REG]{PMS\_PRO\_IRAM0\_2\_REG} [16:0], the split address of IBUS in the SRAM Block 4 \verb+~+ 21 address range is equal to \hexprefix{}4004\_0000. Note that the split address on IBUS and DBUS0 cannot fall within the address range of Block 0 \verb+~+ 5.
\end{tiplisting}


If the type of access initiated by the CPU to SRAM through IBUS does not match the configured access type, the permission control module will directly deny this access. For example, if software allows the IBUS bus to only read and write SRAM Block5, but the CPU initiates an instruction fetch, the permission control module will deny this fetch access.

An access denial does not block the access initiated by the CPU, which means:

\begin{itemize}
    \item The write operation will not take effect and will not change the data in the memory block.
    \item Read and fetch operations will get invalid data.
\end{itemize}

If the interrupt that indicates an illegitimate access to SRAM via IBUS is enabled, the permission control module will record the current access address and access type, and give an interrupt signal at the same time. If the access is denied repeatedly, the hardware only records the first error message. The access error interrupt is a level signal and needs to be cleared by software. When the interrupt is cleared, the related error record is also cleared at the same time.

\subsubsection{Permission Control of Other Types of Internal Memory}
In ESP32-S2, software can read Trace memory to get information about the runtime status of the CPU. Additionally, it can also be used as a debug channel to communicate with software running on the CPU. Software can enable the Trace memory function by configuring register PMS\_PRO\_TRACE\_1. This register is also controlled by the \hyperref[fielddesc:PMSPROTRACELOCK]{PMS\_PRO\_TRACE\_LOCK} signal. When this signal is set to 1, the value configured in the register will be locked and cannot be changed. At the same time, the value of the \hyperref[fielddesc:PMSPROTRACELOCK]{PMS\_PRO\_TRACE\_LOCK} signal will also remain at 1 and cannot be changed. The value of the \hyperref[fielddesc:PMSPROTRACELOCK]{PMS\_PRO\_TRACE\_LOCK} signal is reset to 0 only when the CPU is reset.

After the Trace memory function is enabled, register PMS\_OCCUPY\_3 needs to be configured to select one block from SRAM Block 4 \verb+~+ 21 as Trace memory. Only one block can be used as Trace memory, in which case, it can no longer be accessed by the CPU. Register PMS\_OCCUPY\_3 is controlled by the \hyperref[fielddesc:PMSOCCUPYLOCK]{PMS\_OCCUPY\_LOCK} signal. When this signal is configured as 1, the value configured in the register will be locked and cannot be changed. At the same time, the value of the \hyperref[fielddesc:PMSOCCUPYLOCK]{PMS\_OCCUPY\_LOCK} signal will also remain at 1 and cannot be changed. The \hyperref[fielddesc:PMSOCCUPYLOCK]{PMS\_OCCUPY\_LOCK} signal value is reset to 0  the CPU is reset.

When an SRAM block is selected as Trace Memory, neither IBUS, DBUS, nor DMA can access this block.


\subsection{External Memory Permission Control}
The CPU can access the external flash and SRAM by SPI1, EDMA or cache. The former two support direct access, and the cache supports indirect access.

\subsubsection{Cache MMU}
The Cache MMU is used to control the permission for cache and EDMA to access the external memory. Its primary function is to convert virtual addresses to physical addresses. The MMU needs to be configured before enabling cache and EDMA. When a cache miss occurs or the data is written back to the external memory, the cache controller will automatically access the MMU and generate a physical address to access the external memory. When EDMA reads and writes the external SRAM, the EDMA controller also automatically accesses the MMU and generates a physical address to access the external SRAM.

\subsubsection{External Memory Permission Controls}\label{External Memory Permission Controls}
The hardware divides the physical address of the external memory (flash + SRAM) into eight (4 + 4) areas. Each area can be individually configured for W/R/X access. Software needs to configure the size of each area and its access type in advance.

The eight areas correspond to eight sets of registers, and the flash and SRAM each has four sets. Each set of registers contains three parts as follows:
\begin{enumerate}
    \item Attribute list: SYSCON\_${X}$\_ACE\_${n}$\_ATTR\_REG has 3 bits in total, which are W/R/X bits from high to low;
    \item Area start address: SYSCON\_${X}$\_ACE\_${n}$\_ADDR\_REG, which represents the physical address;
    \item Area length:  SYSCON\_${X}$\_ACE\_${n}$\_SIZE\_REG in multiples of 64 KB;
\end{enumerate}
Wherein “${X}$” represents flash or SRAM, “${n}$” represents 0 \verb+~+ 3. Note that the total size of the four areas corresponding to flash must be 1 GB, so is the total size of the four areas corresponding to SRAM.

When the CPU directly accesses external memory, the permission control module only monitors the CPU's write requests, not the read requests. The control module checks whether the access type matches the configured type according to the accessed physical address. If the area does not allow write access, the permission module directly rejects write access, records the information about the current access, and triggers an access error interrupt.

Cache miss, cache write back, and cache prefetch operations will trigger W/R/X requests from the cache to the external memory. The permission control module will query for the access attributes of the area according to the accessed physical address, and compare the the cache request against the pre-configured attributes. Only when the two match, the permission control module will pass the request to the external memory. If the cache initiates an R/F request, the permission control module will send the access attributes of the area to the cache for storage. When the access request is inconsistent with the current access attributes, the hardware records the information about the current access and triggers an access error interrupt.

When the CPU accesses the cache, the permission control module must check the attributes of the CPU access by comparing it with the local access attribute list. Only if the two match, the CPU access is allowed. When the access request is inconsistent with the current access attributes, the hardware records the information about the current access and triggers an access error interrupt.

If the access attribute check fails repeatedly, the hardware only records the first error message, and the subsequent error messages will be discarded directly. The access error interrupt is a level signal. When software clears the access error interrupt flag, the error record will be cleared at the same time.
