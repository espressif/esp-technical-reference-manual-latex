\hypertarget{intmtrx}{}
\chapter{中断矩阵}
\label{mod:intmtrx}


\section{概述}

ESP32-S2 中断矩阵 (Interrupt Matrix) 可以将任一外部中断源单独分配至 CPU 任一外部中断，以便在外设中断信号产生后，及时通知 CPU 进行处理。这一功能灵活强，可以满足不同的应用需求。

\section{主要特性}

\begin{itemize}
    \item 接收 95 个外部中断源作为输入
    \item 生成 26 个外部中断作为输出
    \item 关闭 CPU 的 NMI 类型中断源
    \item 查询外部中断源当前的中断状态
\end{itemize}

中断矩阵的结构如图 \ref{figure:InterruptMatrixStructure} 所示。

\subsection{CPU 中断}

CPU 共有 32 个中断，其中包括 26 个外部中断和六个内部中断。表 \ref{table:CPUInterrupt} 列出了所有中断：

\begin{itemize}
    \item 外部中断
        \begin{itemize}
            \item 电平触发型中断：由高电平触发，要求保持中断的电平状态直到 CPU 响应。
            \item 边沿触发型中断：由上升沿触发，此中断一旦发生，CPU 即可响应。
            \item NMI 类型中断：软件不可使用 CPU 寄存器屏蔽此类中断。
        \end{itemize}
    \item 内部中断
        \begin{itemize}
            \item 定时器类型中断：由内部定时器触发，可用于产生周期性中断。
            \item 软件类型中断：软件写特殊寄存器时将触发此中断。
            \item 解析类型中断：用于性能监测与分析。
        \end{itemize}
\end{itemize}

ESP32-S2 支持 6 级中断，在下表优先级一栏中，数字越大代表中断优先级越高。其中，NMI 拥有最高优先级，一旦 NMI 中断发生，CPU 必定会响应。

\subsection{分配外部中断源至 CPU 外部中断}

在本小节中，我们将使用以下术语描述中断矩阵相关操作：
\begin{itemize}

\item Source\_X：用于表示某个外部中断源，其中 X 为中断源编号，详见表 \ref{table:PeripheralInterruptConfigStatusSource}。
\item INTERRUPT\_PRO\_X\_MAP\_REG：用于表示 CPU 的某个外部中断配置寄存器。此外部中断配置寄存器与外部中断源 Source\_X 相对应。即表 \ref{table:PeripheralInterruptConfigStatusSource} 中“配置寄存器”一栏与“中断源”一栏应一一对应。例如：UHCI0\_INTR 的中断配置寄存器为处于同一行的 \hyperref[regdesc:INTERRUPTPROUHCI0INTRMAPREG]{INTERRUPT\_PRO\_UHCI0\_INTR\_MAP\_REG}。
\item Interrupt\_P：表示 CPU 中断序号为 Num\_P 的外部中断，Num\_P 的取值范围为 0 \verb+~+ 5、8 \verb+~+ 10、12 \verb+~+ 14、17 \verb+~+ 28、30 \verb+~+ 31，详见表 \ref{table:CPUInterrupt}。
\item Interrupt\_I：表示 CPU 中断序号为 Num\_I 的内部中断，Num\_I 的取值范围为 6、7、11、15、16、29，详见表 \ref{table:CPUInterrupt}。
\end{itemize}

\subsubsection{分配一个外部中断源 Source\_X 至 CPU 外部中断}

将外部中断源 Source\_X 对应的寄存器 INTERRUPT\_PRO\_X\_MAP\_REG 配置成 Num\_P，即可将该中断源分配至序号为 Num\_P 的外部中断（Interrupt\_P）。Num\_P 可以取任一 CPU 外部中断值，包括 0 \verb+~+ 5、8 \verb+~+ 10、12 \verb+~+ 14、17 \verb+~+ 28、30 \verb+~+ 31。一个 CPU 外部中断可以被多个外设共享。


\subsubsection{分配多个外部中断源 Source\_X\regindex{n} 至 CPU 外部中断}

将各个中断源对应的寄存器 INTERRUPT\_PRO\_X\regindex{n}\_MAP\_REG 均配置成相同的 Num\_P，即可将多个中断源 Source\_X\regindex{n} 分配至同一 CPU 外部中断 Interrupt\_P。上述任一外设中断源均会触发 CPU 外部中断 Interrupt\_P。待中断触发后，软件 需要查询中断状态寄存器，用于判断哪个外设产生中断。

\subsubsection{关闭 CPU 外部中断源 Source\_X}

将中断源对应的寄存器 INTERRUPT\_PRO\_X\_MAP\_REG 配置成任意 Num\_I，即可关闭外部中断源。这是因为任何被配置成 Num\_I 的外部中断均无法连接至 CPU，而且选择任一内部中断值 (6、7、11、15、16、29) 不会造成其他影响，可用于关闭外部中断。


\subsection{关闭 CPU 的 NMI 类型中断源}

中断矩阵内部的 Interrupt Reg 寄存器配置子模块可以产生 \hyperref[fielddesc:INTERRUPTPRONMIMASKHW]{INTERRUPT\_PRO\_NMI\_MASK\_HW} 信号，中断矩阵根据此内部信号可以通过硬件关闭所有被分配到 CPU 第 14 号 NMI 中断的外部中断源。
信号 \hyperref[fielddesc:INTERRUPTPRONMIMASKHW]{INTERRUPT\_PRO\_NMI\_MASK\_HW} 可以配置为高电平，此时 CPU 不响应 NMI 中断，也可以配置为低电平，则 CPU 响应 NMI 中断。具体结构请参考图 \ref{figure:InterruptMatrixStructure} 所示。


\subsection{查询外部中断源当前的中断状态}

读取寄存器 \hyperref[regdesc:INTERRUPTPROINTRSTATUSREG0REG]{INTERRUPT\_PRO\_INTR\_STATUS\_REG\_\regindex{n}}（只读）中的特定 Bit 值可以获取外部中断源当前的中断状态。
