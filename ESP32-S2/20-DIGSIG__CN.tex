\chapter{RSA 数字签名外设 (RSA\_DS)}
\label{mod:digsig}
\hypertarget{digsig}{}\hypertarget{rsads}{}


\section{概述}

RSA 数字签名外设 (RSA\_DS) 使用密码学算法，用于验证消息的真实性和完整性。该算法可用于向服务器验证设备身份，或验证消息是否经过篡改。

ESP32-S2 包含一个 RSA 数字签名外设 (RSA\_DS)，可高效生成 RSA 数字签名，而用户无法访问 RSA 私钥。

\section{主要特性}

\begin{itemize}
\item RSA 数字签名支持密钥长度最大为 4096 位
\item 支持仅限 RSA\_DS 外设读取的加密私钥数据
\item SHA-256 摘要用于保护私钥数据免遭攻击者篡改
\end{itemize}


\section{功能描述}
\subsection{概述}

RSA\_DS 外设计算 RSA 签名运算 \begin{math}Z = X^{Y} \bmod M\end{math}，其中 \begin{math}Z\end{math} 是签名，\begin{math}X\end{math} 是输入消息，\begin{math}Y\end{math} 和 \begin{math}M\end{math} 是 RSA 私钥参数。

私钥参数以加密形式存储在 flash 或其他存储器中。这些参数使用一个密钥进行加密，该密钥只能由 RSA\_DS 外设通过 HMAC 模块获取，并且，求解该密钥所需的一切输入信息只存放在 eFuse 中且只允许被 HMAC 模块访问。这意味着只有 RSA\_DS 硬件才能解密私钥，用户绝对不会获取私钥明文。

需要签名时，软件直接将输入消息 \begin{math}X\end{math} 发送到 RSA\_DS 外设。签名操作之后，软件将读取签名结果 \begin{math}Z\end{math}。


\subsection{私钥运算子}\label{Private Key Operands}

私钥运算子 \begin{math}Y\end{math}（私钥指数）和 \begin{math}M\end{math}（密钥模数）由用户生成。它们具有特定的 RSA 密钥长度（最大为 4096 位）。相应的公钥也将另外生成和存储，可独立用于验证 RSA\_DS 签名。

加密操作还需要两个运算子，即参数 \begin{math}\overline{r}\end{math} 和 \begin{math}M^{\prime}\end{math}。这两个参数由 \begin{math}Y\end{math} 和 \begin{math}M\end{math} 得出，但需要通过软件提前运算得到。

运算子 \begin{math}Y\end{math}、
\begin{math}M\end{math}、
\begin{math}\overline{r}\end{math} 和
\begin{math}M^{\prime}\end{math} 与验证摘要一起由用户加密并以密文 \begin{math}C\end{math} 的形式存储。密文 \begin{math}C\end{math} 输入到 RSA\_DS 外设之后由硬件解密并使用密钥生成签名。具体的加密过程请参考章节 \ref{Prerequisites}。

RSA\_DS 外设计算 RSA 签名运算 \begin{math}Z = X^{Y} \bmod M\end{math}，更多信息请参考章节 \ref{mod:rsa} \textit{\nameref{mod:rsa}}中第 \ref{subsection:large_modular_Exponentiation} 节\textit{\nameref{subsection:large_modular_Exponentiation}}。

\subsection{约定}
为方便描述，这里约定几个符号和函数，它们的作用域局限于本章。

\begin{itemize}
\item
1\begin{math}^s\end{math}
\hspace{0.75cm}
表示一个完全由 “1” 组成的长度为 \begin{math}s\end{math} 位的位串。

\item
    \begin{math}[x]_{s}\end{math}  \hspace{0.5cm}
       一个长度为 \begin{math}s\end{math} 位的位串。如果 \begin{math}x\end{math} 是一个数 (\begin{math}x\end{math} < 2\begin{math}^s\end{math})， 那么其在位串中遵循小端字节序。\begin{math}x\end{math} 可以是一个变量，例如 \begin{math}[Y]_{4096}\end{math}，或一个十六进制的常数，比如 [\hexprefix{}0C]\begin{math}_{8}\end{math}。根据需要，\begin{math}[x]\end{math} 右边可以加上 0，使长度变成 \begin{math}s\end{math} 位。例如：[\hexprefix{}5]\begin{math}_{4} = 0101\end{math}，[\hexprefix{}5]\begin{math}_{8} = 00000101\end{math}，[\hexprefix{}5]\begin{math}_{16} = 00000101 00000000\end{math}，[\hexprefix{}13]\begin{math}_{8} = 00010011\end{math}，[\hexprefix{}13]\begin{math}_{16} = 00010011 00000000\end{math}。


    \item
        \begin{math}{ || }\end{math}  \hspace{1.0cm}        表示位串粘接操作符，用于将两个位串前后粘成一个较长的位串。

        \end{itemize}


\subsection{软件存储私钥} \label{Prerequisites}

软件要存储用于 RSA\_DS 操作的私钥，需要做以下准备工作：

\begin{itemize}
\item 按照章节 \ref{Private Key Operands} 所述准备 RSA 私钥 (\begin{math}Y\end{math}, \begin{math}M\end{math}) 和运算子 \begin{math}\overline{r}\end{math} 和 \begin{math} M^{\prime}\end{math}。

\item 准备一个 256 位 HMAC 密钥 (\begin{math}[HMAC\_KEY]_{256}\end{math})，存储在 eFuse 中。HMAC 模块通过读取 HMAC 密钥来生成一个密钥，即 \begin{math}RSA\_DS\_KEY\end{math} = HMAC-SHA256(\begin{math}[HMAC\_KEY]_{256}\end{math}, \begin{math}1^{256}\end{math})，该密钥用于加解密 RSA 私钥。更多信息请参考章节 \ref{mod:hmac} \textit{\nameref{mod:hmac}}。

    \item 准备密文 \begin{math}C\end{math} 形式的加密密钥参数，长度为 1584 字节。
\end{itemize}

    \centering
    \includegraphics[width=0.7\textwidth]{\modulefiles/figures/DigitalSignature_20191022_cn}

图 \ref{figure:DigitalSignatureSoftwareHardwareOperationFlow} 中左半边给出了计算 \begin{math}C\end{math} 的过程。
用户需要依照图 \ref{figure:DigitalSignatureSoftwareHardwareOperationFlow} 指定的步骤来计算 \begin{math}C\end{math}。
更加详细的过程描述如下：

\begin{itemize}

    \item {\bfseries 步骤 1}：准备好大数 \begin{math}Y\end{math} 和 \begin{math}M\end{math}，它们应符合运算子的长度要求。
          记 \begin{math}[L]_{32} = \frac{N}{32}-1 \end{math}（比如，对于 RSA 4096，\begin{math}[L]_{32}\end{math} == [\hexprefix{}80-1]\begin{math}_{32}\end{math}）。另外，准备好 \begin{math}[RSA\_DS\_KEY]_{256}\end{math} 和一个随机的 \begin{math}[IV]_{128}\end{math}，他们都要符合 AES-CBC 块加密算法的要求。有关 AES 更多信息，请参考章节 \ref{mod:aes} \textit{\nameref{mod:aes}}。

    \item \label{Extract M_prime and Rb} {\bfseries 步骤 2}：
          根据 \begin{math}M\end{math} 求解 \begin{math}\overline{r}\end{math} 和 \begin{math}M^{\prime} \end{math}。

    \item \label{Expand big number} {\bfseries 步骤 3}：
          扩展 \begin{math}Y\end{math}、
               \begin{math}M\end{math} 和
               \begin{math}\overline{r}\end{math}，
          得到 \begin{math}[Y]_{4096}\end{math}、
               \begin{math}[M]_{4096}\end{math} 和
               \begin{math}[\overline{r}]_{4096}\end{math}。
          由于 \begin{math}Y\end{math}、\begin{math}M\end{math} 和 \begin{math}\overline{r}\end{math} 的最大位宽为 4096，运算子位宽小于 4096 需要扩展为 4096，位宽等于 4096 则不需要扩展。


    \item \label{Extract MD} {\bfseries 步骤 4}：
            使用 SHA-256 计算 MD 校验码：
                \begin{math}[MD]_{256}\end{math} = SHA256 (
            \begin{math}
                [Y]_{4096} ||
                [M]_{4096} ||
                [\overline{r}]_{4096} ||
                [M^{\prime}]_{32} ||
                [L]_{32} ||
                [IV]_{128}
            \end{math})

    \item {\bfseries 步骤 5}\label{Build Plaintext}：构造 \begin{math}[P]_{12672}\end{math} = (
            \begin{math}
                [Y]_{4096} ||
                [M]_{4096} ||
                [\overline{r}]_{4096} ||
                [MD]_{256} ||
                [M^{\prime}]_{32} ||
                [L]_{32} ||
                [\beta]_{64})
            \end{math}，其中 \begin{math}[\beta]_{64} \end{math} 是符合 PKCS\#7 封装方式的追加码，由 8 个值为 \hexprefix{}08 的字节组成的一个 64 位的位串 $[$\hexprefix{}0808080808080808$]_{64}$，目的在于使 \begin{math}P\end{math} 的长度为 128 位的整数倍。

    \item {\bfseries 步骤 6}\label{Extract Cipher}：
            计算 \begin{math}C\end{math} = \begin{math}[C]_{12672}\end{math} =
            AES-CBC-ENC (\begin{math}[P]_{12672}\end{math}, \begin{math}[RSA\_DS\_KEY]_{256}\end{math}, \begin{math}[IV]_{128}\end{math})。\begin{math}C\end{math} 以密文状态包含诸多信息，包括 RSA 运算子 \begin{math}Y\end{math}、 \begin{math}M\end{math}、\begin{math}\overline{r}\end{math}、\begin{math}M^{\prime} \end{math} 和 \begin{math}L\end{math}，用于校验的 MD 校验码和追加码 \begin{math}[\beta]_{64} \end{math}等其他信息。如前文 \ref{Prerequisites} 所述，\begin{math}RSA\_DS\_KEY\end{math} 由 eFuse 中的 \begin{math}HMAC\_KEY\end{math} 得出。

\end{itemize}

\subsection{硬件工作流程}\label{DS Operation at the Hardware Level}

每次需要计算 RSA 数字签名时，都会触发硬件操作。输入信息是预先生成的私钥密文 \begin{math}C\end{math}、唯一的消息 \begin{math}X\end{math}，和 \begin{math}IV\end{math}。

RSA\_DS 外设的工作流程可以视为 \ref{Prerequisites} 准备工作中计算 \begin{math}C\end{math} 的逆过程。可以分为如下三个阶段：

\begin{enumerate}

    \item {\bfseries 解析阶段，即图 \ref{figure:DigitalSignatureSoftwareHardwareOperationFlow} 中的步骤 7 和 步骤 8}

        解析过程是图 \ref{figure:DigitalSignatureSoftwareHardwareOperationFlow} 中步骤 6 的逆过程。RSA\_DS 外设将调用 AES 硬件加速器以 CBC 块模式对输入的密文信息 \begin{math}C\end{math} 进行解密，获取明文信息。
        该过程可以表示为
        \begin{math}P\end{math} = AES-CBC-DEC (\begin{math}C\end{math}, \begin{math}RSA\_DS\_KEY\end{math}, \begin{math}IV\end{math})，其中 \begin{math}IV\end{math} 就是 \begin{math}[IV]_{128}\end{math}，由用户直接指定； \begin{math}[RSA\_DS\_KEY]_{256}\end{math} 由硬件 HMAC 提供，由 存储在 eFuse 中的 \begin{math}HMAC\_KEY\end{math} 得到，软件无法获取。请参考章节 \ref{mod:hmac} \textit{\nameref{mod:hmac}} 获取更多信息。

        显然，RSA\_DS 外设能够通过 \begin{math}P\end{math} 解析出 \begin{math}[Y]_{4096}\end{math}、\begin{math}[M]_{4096}\end{math}、\begin{math}[\overline{r}]_{4096}\end{math}、\begin{math}[M^{\prime}]_{32}\end{math}、\begin{math}[L]_{32}\end{math}、MD 校验码和追加码 \begin{math}[\beta]_{64} \end{math}，这相当于步骤 5 的逆过程。

    \item {\bfseries 校验阶段，即图 \ref{figure:DigitalSignatureSoftwareHardwareOperationFlow} 中的步骤9 和 步骤 10}

        RSA\_DS 外设会执行两种校验操作： MD 校验和填充 (padding) 校验。
        由于填充校验和 MD 校验同步进行， 因此填充校验不在图 \ref{figure:DigitalSignatureSoftwareHardwareOperationFlow} 中体现。


        \begin{itemize}

            \item MD 校验 —— RSA\_DS 外设调用 SHA-256 进行哈希计算获取哈希结果值 \begin{math}[CALC\_MD]_{256}\end{math}（即步骤 4），然后将 \begin{math}[CALC\_MD]_{256}\end{math} 与 MD 校验码 \begin{math}[MD]_{256}\end{math} 作比较，当且仅当二者相同时，MD 校验通过。

            \item 填充校验 —— RSA\_DS 外设将检查解析阶段解析出的 追加码 \begin{math}[\beta]_{64} \end{math} 是否符合 PKCS\#7 标准，当且仅当符合标准时，填充校验通过。

        \end{itemize}

        如果 MD 校验通过，RSA\_DS 外设将执行后续计算；否则 RSA\_DS 外设拒绝执行。
        如果填充校验失败，将生成警告信息，但不会影响 RSA\_DS 外设的后续操作。

    \item {\bfseries 计算阶段，即图 \ref{figure:DigitalSignatureSoftwareHardwareOperationFlow} 中的步骤 11 和步骤 12}

        RSA\_DS 外设将把用户输入的 \begin{math}X\end{math}，
        以及解析得到的 \begin{math}Y\end{math}、\begin{math}M\end{math} 和 \begin{math}\overline{r}\end{math} 都视为大数，结合解析得到的 \begin{math}M^{\prime}\end{math}，构成了大数模幂运算 \begin{math} X^{Y} \bmod M\end{math} 的所有必要输入参数。大数模幂运算的运算长度由 \begin{math}L\end{math} 的值唯一指定。RSA\_DS 外设调用 RSA 加速器完成大数模幂运算 \begin{math}Z = X^{Y} \bmod M\end{math}，\begin{math}Z\end{math} 为签名结果。

\end{enumerate}


\subsection{软件层面的 RSA\_DS 操作}

每次需要计算 RSA 数字签名时，都应执行以下软件操作。输入消息是预先生成的私钥密文 \begin{math}C\end{math}、唯一的消息 \begin{math}X\end{math}，和 \begin{math}IV\end{math}。这些软件步骤触发章节 \ref{DS Operation at the Hardware Level} 中描述的硬件工作流程。

\begin{enumerate}

    \item {\bfseries 启动 RSA\_DS}：对寄存器 \hyperref[regdesc:DSSETSTARTREG]{DS\_SET\_START\_REG} 写 1。

    \item {\bfseries 检查 \begin{math}RSA\_DS\_KEY\end{math} 是否已经准备好}：轮询寄存器 \hyperref[regdesc:DSQUERYBUSYREG]{DS\_QUERY\_BUSY\_REG} 直到读到 0。

    如果 \hyperref[regdesc:DSQUERYBUSYREG]{DS\_QUERY\_BUSY\_REG} 超过 1 ms 还没读到 0，则说明 HMAC 未被调用。此时，软件应当读寄存器 \hyperref[regdesc:DSQUERYKEYWRONGREG]{DS\_QUERY\_KEY\_WRONG\_REG}，根据返回值判断具体是哪一种情况。
    \begin{itemize}

        \item 如果读到零值，说明 HMAC 未被调用。

        \item 如果读到非零值 (1 \verb+~+ 15)，则说明 HMAC 被调用过，但是 RSA\_DS 外设没有成功从 HMAC 模块获取 \begin{math}RSA\_DS\_KEY\end{math}，原因可能是有其他程序的干扰。

    \end{itemize}

    \item {\bfseries 配置寄存器}：将 \begin{math}IV\end{math} block 写入寄存器 DS\_IV\_\textcolor{red}{\it{m}}\_REG (\textcolor{red}{\it{m}}: 0-3)。有关 \begin{math}IV\end{math} block 的更多信息，请参考章节 \ref{mod:aes} \textit{\nameref{mod:aes}}。

    \item {\bfseries 将 \begin{math}X\end{math} 写入存储器 \hyperref[DSXMEM]{DS\_X\_MEM}}：将 \begin{math}X_{i}\end{math}
        (\begin{math}i \in [0,n) \cap \mathbb{N}\end{math})
        写入存储器 \hyperref[DSXMEM]{DS\_X\_MEM}，容量为 128 个字 (word)。
        每一个字刚好存放一个 \begin{math}b\end{math} 进制数。存储器的低地址存放运算子的低位进制数，高地址存放运算子的高位进制数。当 \begin{math}X\end{math} 的长度小于 128 个字时，存储器 \hyperref[DSXMEM]{DS\_X\_MEM} 中有一部分空间未使用，该部分空间中的数据可以是任意值。

    \item {\bfseries 将 \begin{math}C\end{math} 写入存储器 \hyperref[DSCMEM]{DS\_C\_MEM}}：将
        \begin{math}C_{i}\end{math}
        (\begin{math}i \in [0,396) \cap \mathbb{N}\end{math})
        写入存储器 \hyperref[DSCMEM]{DS\_C\_MEM}，容量为 396 个字。
        每一个字刚好存放一个 \begin{math}b\end{math} 进制数。


    \item {\bfseries 启动计算}：对寄存器 \hyperref[regdesc:DSSETMEREG]{DS\_SET\_ME\_REG} 写入 1。

    \item {\bfseries 等待运算结束}：轮询寄存器 \hyperref[regdesc:DSQUERYBUSYREG]{DS\_QUERY\_BUSY\_REG} 直到读到 0。

    \item {\bfseries 检查校验结果}：读寄存器 \hyperref[regdesc:DSQUERYCHECKREG]{DS\_QUERY\_CHECK\_REG}，根据返回值决定后续操作。

    \begin{itemize}

        \item 如果返回值为 0，则说明填充校验通过，MD 校验通过，可以继续读取 \begin{math}Z\end{math} 结果值。

        \item 如果返回值为 1，则说明填充校验通过，但 MD 校验失败。\begin{math}Z\end{math} 结果值全零无效，跳至步骤 \ref{Exit}。

        \item 如果返回值为 2，则说明填充校验通过失败，但 MD 校验通过，用户可以继续读取 \begin{math}Z\end{math} 结果值。

        \item 如果返回值为 3，则说明填充填充校验失败，且 MD 校验失败，\begin{math}Z\end{math} 结果值全零无效，跳至步骤 \ref{Exit}。

    \end{itemize}

    \item {\bfseries 读出运算结果}：从存储器 \hyperref[DSZMEM]{DS\_Z\_MEM} 读出运算结果 \begin{math}Z_{i}\end{math} (\begin{math}i \in \{0,1,2...,n\}\end{math})。\begin{math}Z\end{math} 以小端字节序存储在存储器中。

    \item {\bfseries 退出计算环境\label{Exit}}：对寄存器 \hyperref[regdesc:DSSETFINISHREG]{DS\_SET\_FINISH\_REG} 写入 1，然后轮询寄存器 \hyperref[regdesc:DSQUERYBUSYREG]{DS\_QUERY\_BUSY\_REG} 直到读到 0。

\end{enumerate}

RSA\_DS 退出计算环境后，所有输入/输出寄存器和存储器中的数据都已经被抹除（清零）。
