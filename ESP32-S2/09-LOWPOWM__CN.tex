\chapter{低功耗管理}
\label{mod:lowpowm}

\section{概述}

ESP32-S2 拥有一个先进的电源管理单元，可在低功耗协处理器的配合下，灵活打开或关闭芯片的不同电源域，协助客户在芯片工作性能、功耗控制和唤醒延迟之间实现最佳平衡。为了便利用户的使用，ESP32-S2 定义了五种最常见的电源域设置组合，对应 5 种预设功耗模式，可满足用户的常见场景需求，但也同时支持用户对某个电源域的独立控制，以满足一些复杂场景的功耗需求。ESP32-S2 内置 2 个超低功耗协处理器，允许芯片在绝大多数电源域均关闭的情况下正常工作，因此可以实现超低功耗。

\section{主要特性}
ESP32-S2 的低功耗管理具有以下主要特性：
\begin{itemize}
\item 各功耗模式下均支持超低功耗协处理器
\item 5 种预设功耗模式，可满足多种典型应用场景需求
\item 高达 16 KB 保留内存 (retention memory)
\item 8 个 32 位保留寄存器 (retention register)
\item 支持 RTC Boot 功能，用于快速唤醒
\end{itemize}

在本章节中，我们将首先介绍 ESP32-S2 低功耗管理的工作过程，其次介绍芯片的预设低功耗工作模式，最后介绍芯片的 RTC Boot 过程。

\section{功能描述}

ESP32-S2 的低功耗管理具有以下主要特性：

\begin{itemize}
\item 功耗管理单元 (PMU)：控制向模拟、RTC 和数字类电源域的供电；
\item 电源隔离单元：保证各电源域的独立工作，防止掉电电源域影响其他电源域的工作；
\item 低功耗时钟：为低功耗模式下工作的电源域提供时钟信号。
\item 定时器：
\begin{itemize}
\item RTC 定时器：在专用寄存器中记录 RTC 主状态机的状态；
\item ULP 定时器：在预设时间唤醒超低功耗协处理器。更多详情，请见章节 \ref{mod:ulp} \textit{\nameref{mod:ulp}}。
\item 触摸传感器定时器：在预设时间唤醒触摸传感器。更多详情，请见章节 \ref{mod:sensor}\textit{\nameref{mod:sensor}}。
\end{itemize}
\item 8 个 32 位 ``always-on" 保留寄存器：即这 8 个寄存器永远处于工作状态，不受芯片掉电影响，可用于存储一些不能丢失的数据。
\item 22 个 ``always-on" 管脚：即这 22 个管脚永远处于工作状态，不受芯片掉电影响，可用作低功耗模式下的唤醒源（详见第 \ref{WakeupSource} 节），也作为正常 GPIO 使用（详见 \ref{mod:iomuxgpio} \textit{\nameref{mod:iomuxgpio}} 章节）。
\item RTC 慢速内存：支持 8 KB SRAM，可用作保留内存或存储 ULP 指令和数据内存。
\item RTC 快速内存：支持 8 KB SRAM，可用作保留内存。
\item 调压器：调节向不同电源域的供电。
\end{itemize}

\subsection{功耗管理单元}

ESP32-S2 功耗管理单元可以控制向不同电源域的供电，其主要组成部分包括：

\begin{itemize}
\item RTC 主状态机：产生电源门控、时钟门控和复位信号。
\item 功耗控制器：根据 RTC 主状态机产生的电源门控信号，打开或关闭各电源域。
\item 睡眠和唤醒控制器：向 RTC 主状态机发送睡眠或唤醒请求。
\item 时钟控制器：选择并打开或关闭时钟源。
\end{itemize}

在 ESP32-S2 的电源管理单元中，睡眠和唤醒控制器向 RTC 主状态机发送睡眠或唤醒请求，RTC 主状态机接着产生电源门控、时钟门控和复位信号。此后，电源控制器和时钟控制器会根据 RTC 主状态机产生的信号，打开或关闭不同的电源域和时钟信号，从而让芯片进入或退出低功耗模式。电源管理单元的主要工作流程可见图 \ref{figure:RTC structure}。

\subsection{定时器}

ESP32-S2 的低功耗管理使用三个定时器：
\begin{itemize}
\item RTC 定时器
\item ULP 定时器
\item 触摸传感器定时器
\end{itemize}

本章节主要介绍 RTC 定时器。有关 ULP 定时器和触摸传感器定时器的内容，请见章节 \ref{mod:ulp} \textit{\nameref{mod:ulp}} 和 \ref{mod:sensor} \textit{\nameref{mod:sensor}}。

RTC 定时器是一个 48 位的可读计数器，可通过配置，使用 RTC 慢速时钟记录以下任一事件发生的时刻。更多详情，请见表 \ref{table:Update RTC TIMER VALUE}。

\begin{longtable}[c]{|p{6cm}|p{9cm}|}

\label{table:Update RTC TIMER VALUE}
\\ \hline

\hline
\hyperref[regdesc:RTCCNTLTIMEUPDATEREG]{RTC\_CNTL\_TIMER\_XTL\_OFF} & 1.RTC 主状态机关闭，或 2. 打开 40 MHz 晶振时均触发。\\

RTC 定时器会在每次触发时更新两组寄存器。其中第一组寄存器记录本次触发的信息，第二组寄存器记录之前触发的信息。这两组寄存器的具体情况见下：

\begin{itemize}
\item 寄存器组 0 用于记录 RTC 定时器在当前触发下的计数值。
\begin{itemize}
\item \hyperref[regdesc:RTCCNTLTIMEHIGH0REG]{RTC\_CNTL\_TIME\_HIGH0\_REG}
\item \hyperref[regdesc:RTCCNTLTIMELOW0REG]{RTC\_CNTL\_TIME\_LOW0\_REG}
\end{itemize}
\item 寄存器组 1 用于记录 RTC 定时器在上一次触发下的计数值。
\begin{itemize}
\item \hyperref[regdesc:RTCCNTLTIMEHIGH1REG]{RTC\_CNTL\_TIME\_HIGH1\_REG}
\item \hyperref[regdesc:RTCCNTLTIMELOW1REG]{RTC\_CNTL\_TIME\_LOW1\_REG}
\end{itemize}
\end{itemize}

每次有新的触发，上一次触发时的记录将从寄存器组 0 移至寄存器组 1（寄存器组 1 中之前的记录将被覆盖），而本次触发的记录将存储在寄存器组 0。因此，RTC 定时器最多可同时记录两次触发的值。

值得注意的是，除上电复位外的其余任何复位／睡眠均不会使 RTC 定时器停止或复位。

此外，RTC 定时器还能用作唤醒源。更多详情，请见第 \ref{WakeupSource} 节。

\subsection{调压器}

ESP32-S2 共有三个调压器，负责调节向不同电源域的供电：
\begin{itemize}
\item 数字系统调压器：负责数字类电源域；
\item 低功耗调压器：负责 RTC 类电源域；
\item Flash 调压器：负责数字类和 RTC 类之外的电源域。
\end{itemize}

\begin{tiplisting}
更多有关不同电源域的描述，请见第 \ref{rtc status} 节。
\end{tiplisting}


\subsubsection{数字系统调压器}

ESP32-S2 的内置数字系统调压器可以将外部电源电压（通常为 3.3 V）转换为 1.1 V，支持数字类电源域的正常工作，具有输出电压可调的特点，具体结构示意图可见下方图 \ref{figure:Digital regulator}。

\begin{figure}[H]
\centering
\includegraphics[width=0.5\textwidth]{\modulefiles/figures/REG_DIGITAL.pdf}
\caption{数字系统调压器}
\label{figure:Digital regulator}
\end{figure}


\begin{enumerate}
\item 当 XPD\_DIG\_REG == 1 时，该调压器的输出电压为 1.1 V，此时数字类电源域可以正常工作；当 XPD\_DIG\_REG == 0 时，该调压器和数字类电源域均关闭。
\item DIG\_REG\_DBIAS[2:0] 可以调节数字类电源域的供电电压：\\
$$\mathrm{VDD\_DIG = 0.90 + DBIAS \times 0.05 V}$$
\item 数字类电源域的输入电压来自管脚 VDD3P3\_CPU 和 VDD3P3\_RTC\_IO。
\end{enumerate}


\subsubsection{低功耗调压器}

ESP32-S2 的内置低功耗调压器可以将外部电源电压（通常为 3.3 V）转换为 1.1 V，支持 RTC 类电源域的正常工作，此外，该调压器的输出电压范围可调，可以在 Deep-sleep 和冬眠模式下输出更低的电压，从而降低功耗。具体结构示意图可见下方图 \ref{figure:Low-power regulator}。

\begin{figure}[H]
\centering
\includegraphics[width=0.5\textwidth]{\modulefiles/figures/REG_RTC.pdf}
\caption{低功耗调压器}
\label{figure:Low-power regulator}
\end{figure}


\begin{enumerate}
\item 当管脚 CHIP\_PU 为高电平时，低功耗调压器无法关闭，仅能在正常和 Deep-sleep 模式之间进行切换。
\item RTC\_DBIAS[2:0] 可以调节输出电压：\\
$$\mathrm{VDD\_RTC = 0.90 + DBIAS \times 0.05 V}$$
\item RTC 类电源域的输入电压来源为管脚 VDD3P3\_RTC\_IO。
\end{enumerate}


\subsubsection{Flash 调压器}

ESP32-S2 的内置 flash 调压器可以向数字类和 RTC 类之外的电源域（比如 flash）输出 3.3 V 或 1.8 V 电压，比如 flash，具体结构示意图可见下方图 \ref{figure:Flash regulator}。


\begin{figure}[H]
\centering
\includegraphics[width=0.7\textwidth]{\modulefiles/figures/REG1P8.pdf}
\caption{Flash 调压器}
\label{figure:Flash regulator}
\end{figure}

\begin{enumerate}
\item 当 XPD\_SDIO\_VREG == 1 时，调压器的输出电压为 3.3 V 或 1.8 V；当 XPD\_SDIO\_VREG == 0 时，调压器的输出电压为高阻抗，此时，电压由外部电源提供。
\item 当 SDIO\_TIEH == 1 时，调压器将管脚 VDD\_SDIO 和管脚 VDD3P3\_RTC 短路，电压输出为 3.3 V，即管脚 VDD3P3\_RTC 的电压。当 SDIO\_TIEH == 0 时，调压器的电压输出为参考电压 VREF，通常为 1.8 V。
\item DREFH\_SDIO、DREFM\_SDIO 和 DREFL\_SDIO 可以小幅调节参考电压 VREF，但这种操作可能会影响系统内环的稳定性，因此不推荐进行更改。
\item 当调压器输出为 3.3 V 或 1.8 V，输出电压来自管脚 VDD3P3\_RTC\_IO。
\end{enumerate}

Flash 调压器的配置可以通过 RTC 寄存器配置，也可以通过操作 eFuse 控制相关参数的配置。

\begin{itemize}
\item XPD\_SDIO\_VREG 的配置：
\begin{itemize}
\item 当芯片处于 Active 状态，且 RTC\_CNTL\_SDIO\_FORCE == 0，VDD\_SPI\_FORCE(EFUSE) == 1 时，XPD\_SDIO\_VREG 由 XPD\_VDD\_SPI\_REG(EFUSE) 决定；
\item 当芯片处于 sleep 状态，且 RTC\_CNTL\_SDIO\_REG\_PD\_EN == 1 时，XPD\_SDIO\_VREG 为 0；
\item 当 RTC\_CNTL\_SDIO\_FORCE == 1 时，XPD\_SDIO\_VREG 由 RTC\_CNTL\_XPD\_SDIO\_REG 决定。
\end{itemize}
\item SDIO\_TIEH 的配置：
\begin{itemize}
\item 当 RTC\_CNTL\_SDIO\_FORCE == 0 且 VDD\_SPI\_FORCE(EFUSE) == 1 时，SDIO\_TIEH = VDD\_SDIO\_TIEH(EFUSE)；
\item 否则，SDIO\_TIEH = RTC\_CNTL\_SDIO\_TIEH。
\end{itemize}
\end{itemize}

\subsubsection{欠压检测器}

ESP32-S2 的欠压检测器可以检查管脚 VDD3P3\_RTC\_IO，VDD3P3\_CPU，VDDA1 和 VDDA2 的电压，在电压快速下落至预设阈值以下时发出触发信号，并在触发信号持续一定时间后进行芯片或系统复位，从而关闭部分耗电模块（比如 LNA 和 PA 等），为数字模块争取更多时间，用以保存、转移重要数据。


欠压检测器的功耗非常低，在芯片开启时将永远保持开启。

\section{功耗模式管理}

\subsection{电源域}\label{powerdomain}

ESP32-S2 共有三大类共 10 个电源域：
\begin{itemize}

\begin{tiplisting}

\begin{enumerate}
\item ESP32-S2 的电源管理单元经过专门设计，一旦芯片上电即处于 ``always-on"（常开）状态，因此无法 FPU 和 FPD。
\item RTC 外设包括 \ref{mod:ulp} \textit{\nameref{mod:ulp}} 和 \ref{mod:sensor} \textit{\nameref{mod:sensor}}（比如温度传感器控制器、触摸传感器和 SAR ADC 控制器）。
\item RTC 低速内存支持 8 KB SRAM，可用作保留内存或存储 ULP 指令和数据内存，因此应强制打开。CPU 可通过 PeriBUS2 总线访问慢速内存，起始地址为 \hexprefix{}50000000。
\item RTC 快速内存支持 8 KB SRAM，可用作保留内存，因此应强制打开。CPU 可通过 IRAM0/DRAM0 访问快速内存。
\item 当数字内核关闭时，数字系统下的所有电源域均关闭。值得注意的是，ESP32-S2 的 ROM 与 SRAM 也属于数字内核电源域。因此，当数字内核关闭后，用户不可以单独强制打开 ROM 或 SRAM。
\item  Wi-Fi 数字电路电源域包括 Wi-Fi MAC 和 BB（基带）。
\end{enumerate}
\end{tiplisting}

\begin{tiplisting}
\vspace{-2em}
\begin{enumerate}
\item EXT0 仅可将芯片从 Light-sleep/Deep-sleep 模式中唤醒。当 \hyperref[fielddesc:RTCCNTLEXTWAKEUP0LV]{RTC\_CNTL\_EXT\_WAKEUP0\_LV} 为 1 时将触发管脚高电平，否则触发管脚低电平。用户可通过设置 RTCIO\_EXT\_WAKEUP0\_SEL 选择作为唤醒源的 RTC 管脚。
\item EXT1 经过专门设计，可将芯片从任何睡眠模式中唤醒，而且还支持多个管脚的组合。首先，应按照选定作为唤醒源的管脚位图，配置 \hyperref[fielddesc:RTCCNTLEXTWAKEUP1SEL]{RTC\_CNTL\_EXT\_WAKEUP1\_SEL[17:0]}。当 \hyperref[fielddesc:RTCCNTLEXTWAKEUP1LV]{RTC\_CNTL\_EXT\_WAKEUP1\_LV} == 1 时，任何选中管脚之一为高电压则将触发信号唤醒芯片；当 \hyperref[fielddesc:RTCCNTLEXTWAKEUP1LV]{RTC\_CNTL\_EXT\_WAKEUP1\_LV} == 0 时，任何选中管脚之一为低电压才能触发信号唤醒芯片。

\textbf{注意：}用户应确保 EXT1 信号的 hold 时长至少三个RTC\_SLOW\_CLK 周期，否则信号状态无法被\\ \hyperref[regdesc:RTCCNTLEXTWAKEUP1STATUSREG]{RTC\_CNTL\_EXT\_WAKEUP1\_STATUS} 记录。
\item 在 Deep-sleep 模式下，仅有 RTC GPIO 可以作为唤醒源。
\item 为了通过 Wi-Fi 唤醒芯片，芯片将在 Active、Modem-sleep 和 Light-sleep 之间进行切换，CPU 和 RF 模块均将在预设间隔中唤醒，保证 Wi-Fi 的正常连接和数据通信。
\item 当接收到的 RX 脉冲数量超过阈值寄存器中的设置时，即触发唤醒。
\item 当触摸传感器检测到触摸时，即触发唤醒。
\item 当超低功耗协处理器配置寄存器  \hyperref[fielddesc:RTCCNTLRTCSWCPUINT]{RTC\_CNTL\_RTC\_SW\_CPU\_INT}，即触发唤醒。
\item 32 kHz 晶振作为 RTC 慢速时钟时，当 32 kHz 看门狗定时器检测到任何时钟停振，即触发唤醒。
\item 当超低功耗协处理器进入捕获异常事件时（比如堆栈溢出），即触发唤醒。
\item 当 USB 主机发送指令让 USB 进入 RESUMING 状态时，即触发唤醒。
\item 表格中的所有唤醒源均可配置为拒绝睡眠原因，但 UART 除外。\\
\end{enumerate}
\end{tiplisting}

\subsection{拒绝睡眠}\label{sec:lpm-reject-source}

ESP32-S2 提供了硬件拒绝睡眠的机制，防止系统在某些外设仍在工作但是未被 CPU 检测到时进入睡眠，最终导致该外设不能正常工作。

ESP32-S2 的所有唤醒源也同时可以作为芯片拒绝睡眠的原因，因此具体的拒绝睡眠源请见表 \ref{table:wakeup source}（UART 除外）。

用户可以配置以下寄存器使能或禁用拒绝睡眠功能。
\begin{itemize}
    \item 配置 \hyperref[fielddesc:RTCCNTLRTCSLEEPREJECTENA]{RTC\_CNTL\_RTC\_SLEEP\_REJECT\_ENA} 整体使能或关闭拒绝休眠功能：
        \begin{itemize}
            \item 进一步配置 \hyperref[fielddesc:RTCCNTLLIGHTSLPREJECTEN]{RTC\_CNTL\_LIGHT\_SLP\_REJECT\_EN}，具体使能拒绝进入 Light\_sleep；
            \item 进一步配置 \hyperref[fielddesc:RTCCNTLDEEPSLPREJECTEN]{RTC\_CNTL\_DEEP\_SLP\_REJECT\_EN}，具体使能拒绝进入 Deep\_sleep；
        \end{itemize}
    \item 读取 \hyperref[fielddesc:RTCCNTLREJECTCAUSE]{RTC\_CNTL\_SLP\_REJECT\_CAUSE\_REG} 了解拒绝睡眠的原因。
\end{itemize}

\section{RTC Boot}
在 Deep-sleep 和冬眠模式下，芯片的 ROM 和 RAM 均将断电，因此在唤醒时 ROM 解包与 SPI 启动（从 flash 复制数据）所需时间更长。因此，相较于 Light-sleep 和 Modem-sleep 模式，Deep-sleep 和冬眠模式的唤醒时间要长的多。不过，值得注意的是，在 Deep-sleep 模式下，RTC 快速内存和慢速内存均可以处于上电状态。因此，用户可以将一些代码规模不大（即小于 8 KB 的 ``deep sleep wake stub" ）写入 RTC 快速内存或慢速内存，避免 ROM 解包或 SPI 启动带来的延迟，从而加速芯片唤醒过程。

\textbf{第一种方法：RTC 慢速内存}
\begin{enumerate}
\item 设置寄存器 \hyperref[fielddesc:RTCCNTLPROCPUSTATVECTORSEL]{RTC\_CNTL\_PROCPU\_STAT\_VECTOR\_SEL} 为 0。
\item 芯片进入睡眠模式。
\item 当 CPU 开启时，复位向量将从地址 \hexprefix{}50000000，而非 \hexprefix{}40000400 开始复位，整个过程并不需要进行 ROM 解包和 SPI 启动。RTC 内存中的代码仅需在 C 语言环境中进行部分初始化操作即可。
\end{enumerate}

\textbf{第二种方法：RTC 快速内存}
\begin{enumerate}
\item 设置寄存器 \hyperref[fielddesc:RTCCNTLPROCPUSTATVECTORSEL]{RTC\_CNTL\_PROCPU\_STAT\_VECTOR\_SEL} 为 1。
\item 计算 RTC 快速内存的 CRC 码，并将结果保存在寄存器 \hyperref[regdesc:RTCCNTLSTORE6REG]{RTC\_CNTL\_STORE6\_REG[31:0]} 中。
\item 设置寄存器 \hyperref[regdesc:RTCCNTLSTORE7REG]{RTC\_CNTL\_STORE7\_REG[31:0]} 为 RTC 快速内存的入口地址。
\item 芯片进入睡眠模式。
\item 当 CPU 开启时，开始进行 ROM 解包和部分初始化工作。此后，再次计算 RTC 快速内存的 CRC 码。如果与寄存器 \hyperref[regdesc:RTCCNTLSTORE6REG]{RTC\_CNTL\_STORE6\_REG[31:0]} 中保存的结果一致，则 CPU 跳转至 RTC 快速内存的入口地址。
\end{enumerate}
