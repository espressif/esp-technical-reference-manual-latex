\chapter{On-Chip Sensors and Analog Signal Processing}
\label{mod:sensor}


\section{Overview} \label{subsection:on-chip-sensor-overview}

ESP32-S2 provides the following on-chip sensor and signal processing peripherals:

\begin{itemize}

\item One temperature sensor for measuring the internal temperature of the ESP32-S2 chip.
\item Two 12-bit* Successive Approximation ADCs (SAR ADCs)  controlled by five dedicated controllers that can input analog signals from total of 20 channels. The SAR ADCs can operate in a high-performance mode or a low-power mode.
\item Two 8-bit independent DACs to  generate analog signals. The DACs also support a cosine wave output mode.
\end{itemize}

\begin{tiplisting}
SAR ADC in chip revision v0.0 has a resolution of 13 bits. For more information about chip revisions, please refer to \href{ https://espressif.com/sites/default/files/documentation/esp32-s2\_errata\_en.pdf}{ESP32-S2 Series SoC Errata}.
\end{tiplisting}
\section{SAR ADCs}\label{ssec:saradc}

\subsection{Overview}

ESP32-S2 integrates two 12-bit SAR ADCs, which are able to measure analog signals from up to 20 analog pads. It is also possible to measure internal signals, such as vdd33.
The SAR ADCs are managed by five dedicated controllers:

\begin{itemize}
   \item Two digital controllers: DIG ADC1 CTRL and DIG ADC2 CTRL, designed for high-performance multi-channel scanning and DMA continuous conversion.
   \item Two RTC controllers: RTC ADC1 CTRL and RTC ADC2 CTRL, designed for single conversion mode and low power mode.
\item One internal controller: PWDET/PKDET CTRL, for power and peak detection (only used by Wi-Fi).
\end{itemize}

A diagram of the SAR ADCs is shown in Figure \ref{figure:sar adc overview}.

\begin{figure}[H]

\subsection{Features}
\begin{itemize}
\item Two SAR ADCs that can operate separately (i.e., convert data simultaneously)
\item 12-bit sampling resolution
\item{Up to 20 analog input pads}
\item{Configurable channel-scanning duration}
\item{Two RTC ADC controllers, with the following features:}
    \begin{itemize}
        \item{Single conversion mode}
        \item{Operation in Deep-sleep mode}
        \item{Can be controlled by ULP coprocessor}
    \end{itemize}

\item{Two DIG ADC controllers, with the following features:}
    \begin{itemize}
        \item Sampled data is transferred using DMA
        \item Single conversion mode and continuous conversion mode
        \item{Multiple channel-scanning mode}
        \item{User-defined channel-scanning sequence}
        \item{Hardware IIR filter with configurable filter coefficient}
        \item{Threshold monitoring. An interrupt will be triggered when the value is greater or less than the threshold.}
    \end{itemize}
\item{One PWDET/PKDET controller only for Wi-Fi internal use to:}
    \begin{itemize}
        \item{Monitor power}
        \item{Monitor internal voltage}
    \end{itemize}
\end{itemize}

\end{tiplisting}

\subsubsection{ADC Conversion and Attenuation}

When the SAR ADCs convert (sample) an analog voltage, the resolution (12-bit) of the conversion spans voltage range from 0 mV to \(V_{ref}\) which is the SAR ADCs internal reference voltage (1100 mV by design). Thus the output value of the conversion (\({data}\)) will map to analog voltage \(V_{data}\) using the following formula:

\(V_{data} = \frac{V_{ref}}{k}\times{\frac{data}{4095}}\)

\(k\) is the coefficient corresponding to the configured attenuation.

In order to convert voltages larger than \(V_{ref}\), input signals can be attenuated before being input into the SAR ADCs. The supported attenuation levels are:

\begin{itemize}
    \item 0 dB (k≈100\%)
    \item 2.5 dB (k≈75\%)
    \item 6 dB (k≈50\%)
    \item 12 dB (k≈25\%)
\end{itemize}

\subsection{RTC ADC Controllers}
\label{subsubsec:rtc-adc-controllers}

The RTC ADC controllers (RTC ADC1 CTRL and RTC ADC2 CTRL) are powered in the RTC power domain, thus allow the SAR ADCs to conduct measurements at a low frequency with minimal power consumption. The outline of a single controller’s function is shown in Figure \ref{figure:sar adc ctrl}.

\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=1.0\textwidth]{\modulefiles/figures/saradc_rtc_ctrl.eps}}
    \caption{RTC SAR ADC Outline}
    \label{figure:sar adc ctrl}
\end{figure}

The RTC ADC controllers can be triggered to start a conversion by software or by the ULP coprocessor (ULP-FSM). \hyperref[fielddesc:SENSMEAS1STARTFORCE]{SENS\_MEAS\regindex{n}\_START\_FORCE} will select whether an RTC ADC controller is triggered by software or by the ULP-FSM.

To trigger the RTC ADC controller to start a conversion by software, set \hyperref[fielddesc:SENSMEAS1STARTSAR]{SENS\_MEAS\regindex{n}\_START\_SAR}. When the conversion is complete,  \hyperref[fielddesc:SENSMEAS1DONESAR]{SENS\_MEAS\regindex{n}\_DONE\_SAR} will be set and the conversion result will be stored in \hyperref[fielddesc:SENSMEAS1DATASAR]{SENS\_MEAS\regindex{n}\_DATA\_SAR}.

The RTC ADC controllers are intertwined with the ULP coprocessor, as the ULP coprocessor has a built-in instruction to start an ADC conversion. In many cases, the controllers need to cooperate with the ULP coprocessor, e.g.:

\begin{itemize}
    \item When the controllers periodically monitor a channel during Deep-sleep, ULP coprocessor is the only source to trigger ADC sampling by configuring RTC registers.
    \item Continuous scanning or DMA is not supported by the controllers. However, it is possible with the help of ULP coprocessor to scan channels continuously in a sequence.
\end{itemize}

\subsection{DIG ADC Controllers}

Compared to the RTC ADC controllers, the DIG ADC controllers are optimized for performance and throughput. The main features of the DIG ADC controllers are outlined below:

\begin{itemize}
    \item High performance: The clock of the DIG ADC controller is much faster, thus the sample rate is much higher.
\item Up to 12-bit sampling resolution
    \item Support single-channel scanning, double-channel scanning, or alternate-channel scanning mode. The measurement rules for each SAR ADC are defined in pattern tables.
    \item Scanning can be started by software or by DIG ADC timer, depending on the configuration of \hyperref[fielddesc:APBSARADCSTARTFORCE]{APB\_SARADC\_START\_FORCE}. Note that DIG ADC timer is enabled by \hyperref[fielddesc:APBSARADCTIMEREN]{APB\_SARADC\_TIMER\_EN} and its trigger period can be configured by \hyperref[fielddesc:APBSARADCTIMERTARGET]{APB\_SARADC\_TIMER\_TARGET}.
    \item Sampled data is transferred to memory via DMA.
    \item An interrupt will be generated when scanning is finished.
\end{itemize}

\subsubsection{Workflow of DIG ADC Controller}

\begin{tiplisting}
The term “start of conversion” is not used in this section, because there is no direct access to starting a single SAR analog-to-digital conversion. Each conversion is started automatically by the DIG ADC controller according to a pattern table. Instead, the term “start of scan” will be used to indicate that the DIG ADC controller will scan a sequence of channels according to a pattern table.
\end{tiplisting}

Figure \ref{figure:dig adc overview} shows a diagram of the DIG ADC controllers.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.9\textwidth]{\modulefiles/figures/saradc_dma}
    \caption{Diagram of DIG ADC Controllers}
    \label{figure:dig adc overview}
\end{figure}

A measurement rule instructs an ADC to sample a channel (and optionally attenuate the signal of the sampled channel). Each of the DIG ADC controllers has a rules table, and each table can store 16 rules. When scanning starts, the controller reads the measurement rules one-by-one from their pattern table. For each controller, a scanning sequence can include 16 different rules at most, before repeating itself.

The 8-bit item (pattern table register) is composed of three fields that contain channel and attenuation information, as shown in Table \ref{table:pattern table register}.

\begin{table}[H]

        \multicolumn{3}{|c|}{Pattern Table Register[7:0]} \\

        ch\_sel[3:0] & null & atten[1:0] \\

The scanning mode configured in \hyperref[fielddesc:APBSARADCWORKMODE]{APB\_SARADC\_WORK\_MODE} determines whether the DIG ADC controllers operate completely independently, or in an alternating or synchronized manner. The following scanning modes are supported:

\begin{itemize}
    \item Single-channel scanning: SAR ADC1 and SAR ADC2 will operate independently according to their own respective pattern tables.
    \item Double-channel scanning: SAR ADC1 and SAR ADC2 will sample simultaneously (i.e., the two DIG ADC controllers will progress down their own pattern tables in a synchronized manner).
    \item Alternate-channel scanning: SAR ADC1 and SAR ADC2 will sample alternately (i.e., the two DIG ADC controllers will take turns when progressing down their own pattern tables).
\end{itemize}

The final data sent to the DMA by each conversion is 16-bit. This data is composed of the 12/11-bit ADC result and some additional information related to the scanning mode:

\begin{itemize}
    \item For single-channel scanning, 4-bit information on channel selection is added.
    \item For double-channel scanning or alternate-channel scanning, 4-bit information on channel selection is added plus one extra bit indicating which SAR ADC is the data from.
\end{itemize}

Two DMA data formats, Type I and Type II, are used to store the data for the above-mentioned two situations, see Table \ref{table:type i dma data format} and Table \ref{table:type ii dma data format}.

\subsubsection{DMA}

DIG ADC controllers support direct memory access via the SPI3 DMA and is triggered by DIG ADC dedicated timer. Therefore, the DIG ADC controllers should not be used when SPI3 DMA is already being used. Users can switch the DMA data path to DIG ADC by configuring \hyperref[fielddesc:APBSARADCAPBADCTRANS]{APB\_SARADC\_APB\_ADC\_TRANS} via software. For specific DMA configuration, please refer to SPI3 DMA control.

\subsubsection{ADC Filter}

The DIG ADC controllers supports automatic filtering of sampled ADC data. The filter's formula is shown below:

\[data_{cur} = \frac{(k-1)data_{prev}}{k}+\frac{data_{in}}{k}-0.5\]

\begin{itemize}
    \item \(data_{cur}\) is the filtered data value.
    \item \(data_{in}\) is the sampled data value from the ADC.
    \item \(data_{prev}\) is the last filtered data value.
    \item \(k\) is the filter coefficient.
\end{itemize}

To configure the filter, set the filter coefficient using \hyperref[fielddesc:APBSARADCADC1FILTERFACTOR]{APB\_SARADC\_ADC\regindex{n}\_FILTER\_FACTOR} then set \hyperref[fielddesc:APBSARADCADC1FILTEREN]{APB\_SARADC\_ADC\regindex{n}\_FILTER\_EN} to enable the filter. The filter is applied to each DIG ADC controller. However, due to the filter being dependent on previous filtered values (\(data_{prev}\)), \textbf{only one channel on each DIG ADC controller should be scanned when filters are used} so that data of different channels are not mixed by the filter.
\subsubsection{Threshold Monitoring}

Each DIG ADC controller has a threshold monitor that will trigger an interrupt if any of the sampled ADC values exceeds (or falls below) a configured threshold.

\begin{itemize}
    \item Set the bit \hyperref[fielddesc:APBSARADCADC1THRESEN]{APB\_SARADC\_ADC\regindex{n}\_THRES\_EN} to enable threshold monitoring.
    \item Configure \hyperref[fielddesc:APBSARADCADC1THRES]{APB\_SARADC\_ADC\regindex{n}\_THRES} to set the threshold value.
    \item Configure \hyperref[fielddesc:APBSARADCADC1THRESMODE]{APB\_SARADC\_ADC\regindex{n}\_THRES\_MODE} to set the conditions:
    \begin{itemize}
        \item 0: When ADC\_DATA < \hyperref[fielddesc:APBSARADCADC1THRES]{APB\_SARADC\_ADC\regindex{n}\_THRES}, a monitor interrupt will be generated.
        \item 1: When ADC\_DATA >= \hyperref[fielddesc:APBSARADCADC1THRES]{APB\_SARADC\_ADC\regindex{n}\_THRES}, a monitor interrupt will be generated.
    \end{itemize}
\end{itemize}

Note:
    Threshold monitoring checks the ADC values on all the channels selected by SAR ADC1 or SAR ADC2, respectively.

\subsection{SAR ADC2 Arbiter}

SAR ADC2 can be controlled by three controllers, namely RTC ADC2 CTRL, DIG ADC2 CTRL, and PWDET/PKDET CTRL. To avoid any possible conflicts and to improve the efficiency of SAR ADC2, ESP32-S2 provides access arbitration for SAR ADC2. The arbiter supports fair arbitration and fixed priority arbitration.

\begin{itemize}
    \item Fair arbitration mode (cyclic priority arbitration) can be enabled by clearing \hyperref[fielddesc:APBSARADCADCARBFIXPRIORITY]{APB\_SARADC\_ADC\_ARB\_FIX\_PRIORITY} in register \hyperref[regdesc:APBSARADCARBCTRLREG]{APB\_SARADC\_ARB\_CTRL\_REG}.
    \item In fixed priority arbitration, users can set the bits in register \hyperref[regdesc:APBSARADCARBCTRLREG]{APB\_SARADC\_ARB\_CTRL\_REG}, including \hyperref[fielddesc:APBSARADCADCARBRTCPRIORITY]{APB\_SARADC\_ADC\_ARB\_RTC\_PRIORITY} (for RTC ADC2 CTRL), \hyperref[fielddesc:APBSARADCADCARBAPBPRIORITY]{APB\_SARADC\_ADC\_ARB\_APB\_PRIORITY} (for DIG ADC2 CTRL), and \hyperref[fielddesc:APBSARADCADCARBWIFIPRIORITY]{APB\_SARADC\_ADC\_ARB\_WIFI\_PRIORITY} (for PWDET CTRL), to configure the priorities for these controllers. A larger value indicates a higher priority.
\end{itemize}

The arbiter ensures that a higher priority controller can always start a conversion (sample) when required, regardless of whether a lower priority controller already has a conversion in progress. If a higher priority controller starts a conversion whilst the ADC already has a conversion in progress from a lower priority controller, the conversion in progress will be interrupted (stopped). The higher priority controller will then start its conversion.

If a lower priority controller attempts to start a conversion whilst the ADC has a conversion in progress from a higher priority controller, the lower priority controller will not start the conversion.

The value returned by a conversion that is interrupted or not started will be invalid. Therefore, certain data flags are embedded into the output data value to indicate whether the conversion is valid or not.

Note:
\begin{itemize}
    \item The data flag for RTC ADC2 CTRL is the two higher bits of \hyperref[fielddesc:SENSMEAS2DATASAR]{SENS\_MEAS2\_DATA\_SAR}.
    \begin{itemize}
        \item 2’b10: Conversion is interrupted.
        \item 2’b01: Conversion is not started.
        \item 2’b00: The data is valid.
    \end{itemize}
    \item The data flag for DIG ADC2 CTRL is the ch\_sel[3:0] bits in DMA data.
    \begin{itemize}
        \item 4’b1111: Conversion is interrupted.
        \item 4’b1110: Conversion is not started.
        \item Corresponding channel No.: The data is valid.
    \end{itemize}
    \item The data flag for PWDET/PKDET CTRL is the two higher bits of the sampling result.
    \begin{itemize}
        \item 2’b10: Conversion is interrupted.
        \item 2’b01: Conversion is not started.
        \item 2’b00: The data is valid.
    \end{itemize}
\end{itemize}

Users can configure \hyperref[fielddesc:APBSARADCADCARBGRANTFORCE]{APB\_SARADC\_ADC\_ARB\_GRANT\_FORCE} to mask the arbiter, and set the bits \hyperref[fielddesc:APBSARADCADCARBRTCFORCE]{APB\_SARADC\_ADC\_ARB\_RTC\_FORCE}, \hyperref[fielddesc:APBSARADCADCARBWIFIFORCE]{APB\_SARADC\_ADC\_ARB\_WIFI\_FORCE}, and \hyperref[fielddesc:APBSARADCADCARBAPBFORCE]{APB\_SARADC\_ADC\_ARB\_APB\_FORCE} to authorize corresponding controllers.

Note:
\begin{itemize}
    \item When the arbiter is masked, only one of the above APB\_SARADC\_ADC\_ARB\_XXX\_FORCE bits can be set to 1.
    \item The arbiter uses APB\_CLK as its clock source. When the clock frequency is 8 MHz or lower, the arbiter must be masked.
    \item In sleep mode, the \hyperref[fielddesc:SENSSAR2RTCFORCE]{SENS\_SAR2\_RTC\_FORCE} in register \hyperref[regdesc:SENSSARMEAS2MUXREG]{SENS\_SAR\_MEAS2\_MUX\_REG} should be set to 1, masking the arbiter and all the signals from controllers except the RTC controllers.
\end{itemize}

\section{DACs}
\label{ssec:dac}

\subsection{Overview}

ESP32-S2 comes with two built-in 8-bit DACs used to convert digital values into analog output signals (up to two of them). The DACs also support to output cosine waves.

\subsection{Features}

\begin{itemize}
    \item Two 8-bit DAC converters
    \item Independent or synchronous conversion in both channels
    \item Voltage reference from VDD3P3\_RTC\_IO pin
    \item Cosine waves output
    \item DMA capability
    \item Fully controlled by ULP coprocessor via registers. See Chapter \ref{mod:ulp} \textit{\nameref{mod:ulp}}.
\end{itemize}


\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.8\textwidth]{\modulefiles/figures/dac.pdf}}
    \caption{Diagram of DAC Function}
    \label{figure:dac mux}
\end{figure}


\subsection{DAC Conversion}

The two 8-bit DAC channels can be configured independently. For each DAC channel, the output analog voltage can be calculated as follows:
$$\mathrm{DAC\regindex{n}\_OUT = VDD3P3\_RTC\_IO \cdot PDAC\regindex{n}\_DAC / 255}$$
\begin{itemize}
    \item VDD3P3\_RTC\_IO is the voltage on pin VDD3P3\_RTC\_IO (usually 3.3 V).
    \item PDAC\regindex{n}\_DAC carries the digital value to be converted into an analog voltage and comes from multiple sources: \hyperref[ssec:cosine gen]{cosine waveform generator}, the register \hyperref[fielddesc:RTCIOPDAC1XPDDAC]{RTCIO\_PAD\_DAC\regindex{n}\_REG}, or DMA.
\end{itemize}

The conversion can be started by the register \hyperref[fielddesc:RTCIOPDAC1XPDDAC]{RTCIO\_PDAC\regindex{n}\_XPD\_DAC}, and controlled by the software or SAR ADC FSM. For more information, please see Figure \ref{figure:dac mux}.

\subsection{Cosine Wave Generator}
\label{ssec:cosine gen}

The cosine wave (CW) generator can be used to generate a cosine or sine tone. Figure \ref{figure:cw gen} shows The function of CW generator.

\begin{figure}[H]
    \centering
    \fbox{\includegraphics[width=0.8\textwidth]{\modulefiles/figures/dac2.eps}}
    \caption{Workflow of CW Generator}
    \label{figure:cw gen}
\end{figure}

CW generator features:
\begin{itemize}
    \item Adjustable frequency \\
    The frequency of CW can be adjusted by register \hyperref[fielddesc:SENSSWFSTEP]{SENS\_SW\_FSTEP}:
    $$\mathrm{freq = dig\_clk\_rtc\_freq \cdot \hyperref[fielddesc:SENSSWFSTEP]{SENS\_SW\_FSTEP} / 65535}$$
    Typically, the frequency of dig\_clk\_rtc is 8 MHz.
    \item Scaling\\
    The waveform amplitude can be scaled by 1, 1/2, 1/4, or 1/8 by configuring the register \hyperref[fielddesc:SENSDACSCALE1]{SENS\_DAC\_SCALE\regindex{n}}.
    \item DC offset\\
    The register \hyperref[fielddesc:SENSDACDC1]{SENS\_DAC\_DC\regindex{n}} may introduce DC offset, leading to a saturated result.
    \item Phase shift \\
    A phase-shift of 0° or 180° can be added by setting register \hyperref[fielddesc:SENSDACINV1]{SENS\_DAC\_INV\regindex{n}}.
    \begin{itemize}
       \item{2: 0°}
       \item{3: 180°}
    \end{itemize}
\end{itemize}

\subsection{DMA Support}

A DMA controller with dual DAC channels can be used to set the output of two DAC channels. By configuring the register \hyperref[fielddesc:SENSDACDIGFORCE]{SENS\_DAC\_DIG\_FORCE} and the bit \hyperref[fielddesc:APBSARADCAPBDACTRANS]{APB\_SARADC\_APB\_DAC\_TRANS} in the register \hyperref[regdesc:APBSARADCAPBDACCTRLREG]{APB\_SARADC\_APB\_DAC\_CTRL\_REG}, users can connect DIG\_SARADC\_CLK to DAC clk, and SPI3\_DMA\_OUT to DAC\_DATA for direct memory access.

For details, please refer to Chapter \ref{mod:dma} \textit{\nameref{mod:dma}}.

\section{Temperature Sensor}\label{ssec:tsensor}
\subsection{Overview}
ESP32-S2 provides a temperature sensor to monitor temperature changes in real time.
\subsection{Features}

\begin{itemize}
    \item Monitored in real time by ULP coprocessor when in low-power mode
    \item Triggered by software or by ULP coprocessor
    \item Configurable temperature offset based on the environment, to improve the accuracy
    \item Adjustable measurement range
\end{itemize}

\subsection{Operation Sequence}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{\modulefiles/figures/tsensor_struct.pdf}
    \caption{Structure of Temperature Sensor}
    \label{figure:tsensor_struct}
\end{figure}

As shown in Figure \ref{figure:tsensor_struct}, the temperature sensor can be started by software or by ULP coprocessor:


\begin{itemize}
   \item Started by software, i.e. by CPU or ULP-RISC-V configuring related registers:
    \begin{itemize}
        \item Start the temperature sensor by setting the registers \hyperref[fielddesc:SENSTSENSPOWERUPFORCE]{SENS\_TSENS\_POWER\_UP\_FORCE} and \hyperref[fielddesc:SENSTSENSPOWERUP]{SENS\_TSENS\_POWER\_UP}.
        \item Set \hyperref[fielddesc:SENSTSENSCLKGATEEN]{SENS\_TSENS\_CLKGATE\_EN} to enable temperature sensor clock.
        \item Wait for a while and then configure the register \hyperref[fielddesc:SENSTSENSDUMPOUT]{SENS\_TSENS\_DUMP\_OUT}. The output value gradually approaches the actual temperature linearly as the measurement time increases.
        \item Wait for \hyperref[fielddesc:SENSTSENSREADY]{SENS\_TSENS\_READY} and read the conversion result from \hyperref[fielddesc:SENSTSENSOUT]{SENS\_TSENS\_OUT}.
    \end{itemize}
    \item Started by ULP-FSM:
    \begin{itemize}
        \item Clear the register \hyperref[fielddesc:SENSTSENSPOWERUPFORCE]{SENS\_TSENS\_POWER\_UP\_FORCE}.
        \item ULP-FSM has a built-in instruction for temperature sampling. Executing the instruction can easily complete temperature sampling, see Section \ref{ulp-fsm} \textit{\nameref{ulp-fsm}}.
    \end{itemize}
\end{itemize}


\subsection{Temperature Conversion}

The actual temperature can be obtained by converting the output of temperature sensor via the following formula:\\

~
\begin{centering}
        T(°C) = 0.4386 * VALUE – 27.88 * offset – 20.52\\
\end{centering}
~\\
VALUE in the formula is the output of the temperature sensor, and the offset is determined by the temperature offset. The temperature offset varies in different actual environment (the temperature range). For details, refer to Table \ref{tab:tsensor-dac}.


\begin{longtable}[c]{|p{4cm}|p{4cm}|}
    \caption{Temperature Offset}
    \label{tab:tsensor-dac}
    \\\hline
    \rowcolor{lightgray}
    Measurement Range (°C)  & Temperature Offset (°C) \\\hline
    50 \verb+~+ 125 & -2 \\\hline
    20 \verb+~+ 100 & -1 \\\hline
    -10 \verb+~+ 80 & 0 \\\hline
    -30 \verb+~+ 50 & 1 \\\hline
    -40 \verb+~+ 20 & 2 \\\hline
\end{longtable}
