\chapter{System Registers}
\label{mod:sysreg}


\section{Overview}

The ESP32-S2 integrates a large number of peripherals, and enables the control of individual peripherals to achieve optimal characteristics in performance-vs-power-consumption scenarios. Specifically, ESP32-S2 has a various of system configuration registers that can be used for the chip's clock management (clock gating), power management, and the configuration of peripherals and core-system modules. This chapter lists all these system registers and their functions.

\section{Features}

ESP32-S2 system registers can be used to control the following peripheral blocks and core modules:

\begin{itemize}
    \item System and memory
    \item Reset and clock
    \item Interrupt matrix
    \item eFuse controller
    \item Low-power management
    \item Peripheral clock gating and reset
\end{itemize}

\subparagraph{ROM Power Consumption Control}

Registers \hyperref[regdesc:SYSTEMROMCTRL0REG]{SYSTEM\_ROM\_CTRL\_0\_REG} and  \hyperref[regdesc:SYSTEMROMCTRL1REG]{SYSTEM\_ROM\_CTRL\_1\_REG} can be used to control the power consumption of ESP32-S2's ROM. Specifically:

\begin{itemize}
    \item Setting different bits of the \hyperref[fielddesc:SYSTEMROMFO]{SYSTEM\_ROM\_FO} field in register \hyperref[regdesc:SYSTEMROMCTRL0REG]{SYSTEM\_ROM\_CTRL\_0\_REG} forces on the clock gates of different blocks of ROM.
    \item Setting different bits of the \hyperref[fielddesc:SYSTEMROMFORCEPD]{SYSTEM\_ROM\_FORCE\_PD} field in register \hyperref[regdesc:SYSTEMROMCTRL1REG]{SYSTEM\_ROM\_CTRL\_1\_REG} powers down different blocks of internal ROM.
    \item Setting different bits of the \hyperref[fielddesc:SYSTEMROMFORCEPU]{SYSTEM\_ROM\_FORCE\_PU} field in register \hyperref[regdesc:SYSTEMROMCTRL1REG]{SYSTEM\_ROM\_CTRL\_1\_REG} powers up different blocks of internal ROM.
\end{itemize}

\subsection{Reset and Clock Registers} \label{sec:peri-clock-gating-reset}
The following registers are used for reset and clock. For additional information, please refer to Chapter \ref{mod:resclk} \textit{\nameref{mod:resclk}}.

\begin{itemize}
    \item \hyperref[regdesc:SYSTEMCPUPERCONFREG]{SYSTEM\_CPU\_PER\_CONF\_REG}
    \item \hyperref[regdesc:SYSTEMSYSCLKCONFREG]{SYSTEM\_SYSCLK\_CONF\_REG}
    \item \hyperref[regdesc:SYSTEMBTLPCKDIVFRACREG]{SYSTEM\_BT\_LPCK\_DIV\_FRAC\_REG}
\end{itemize}

\begin{tiplisting}
ESP32-S2 features low power consumption. This is why some peripheral clocks are gated (disabled) by default. Before using any of these peripherals, it is mandatory to enable the clock for the given peripheral by setting the corresponding CLK\_EN bit to 1, and release the peripheral from reset state to make it operational by setting the RST\_EN bit to 0.
\end{tiplisting}

\begin{tiplisting}
\vspace{-3em}\label{table:reset_notes}
\begin{enumerate}
    \item Set the clock enable register to 1 to enable the clock, and to 0 to disable the clock;
    \item Set the reset enabling register to 1 to reset a peripheral, and to 0 to disable the reset.
    \item Reset registers are not cleared by hardware.
    \item UART memory is shared by all UART peripherals, meaning having any active UART peripherals will prevent the UART memory from entering the clock-gated state.
    \item Crypto DMA is shared by AES and SHA accelerators.
    \item Resetting this bit also resets the SHA accelerator.
    \item Resetting this bit also resets the AES, SHA, and RSA accelerators.
\end{enumerate}
\end{tiplisting}
